Spring Boot Actuator `HealthIndicator` that verifies Cassandra connectivity by querying the `system.local` table. Only activated when `spring.data.cassandra.enabled=true`.

## Key Components

| Element | Description |
|---|---|
| `CassandraHealthIndicator` | Spring `@Component` implementing `HealthIndicator` |
| `@ConditionalOnProperty` | Activates only when `spring.data.cassandra.enabled=true` |
| `health()` | Executes a lightweight CQL probe query; returns `UP` or `DOWN` with exception details |
| `CassandraOperations` | Injected dependency used to run the health check query |

## Usage Example

```java
// application.yml
spring:
  data:
    cassandra:
      enabled: true
      contact-points: localhost
      keyspace-name: my_keyspace
```

Once configured, the indicator is auto-registered with Spring Boot Actuator and exposed at `/actuator/health`:

```json
{
  "status": "UP",
  "components": {
    "cassandra": {
      "status": "UP"
    }
  }
}
```

On failure (e.g., cluster unreachable):

```json
{
  "status": "DOWN",
  "components": {
    "cassandra": {
      "status": "DOWN",
      "details": {
        "error": "com.datastax.oss.driver.api.core.AllNodesFailedException: ..."
      }
    }
  }
}
```

> The probe query `SELECT release_version FROM system.local` is a standard Cassandra connectivity check — it hits a local system table with negligible overhead and requires no application keyspace permissions.
