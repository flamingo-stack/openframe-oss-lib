Normalizes the Cassandra keyspace name at application startup by replacing dashes with underscores, ensuring compatibility with Cassandra's naming constraints (alphanumeric and underscores only).

## Key Components

- **`CassandraKeyspaceNormalizer`** — Implements `ApplicationContextInitializer<ConfigurableApplicationContext>` to run after Spring Cloud Config properties are loaded.
- **`initialize()`** — Reads `spring.data.cassandra.keyspace-name`, replaces `-` with `_`, and injects the normalized value as the highest-priority `MapPropertySource`.
- **`KEYSPACE_PROPERTY`** — Targets `spring.data.cassandra.keyspace-name`.
- **`PROPERTY_SOURCE_NAME`** — Registers the override source as `cassandra-keyspace-normalizer`.

## Usage Example

Register the initializer in `spring.factories` or programmatically:

```java
// Option 1: Register via SpringApplication
SpringApplication app = new SpringApplication(MyApplication.class);
app.addInitializers(new CassandraKeyspaceNormalizer());
app.run(args);
```

```text
# Option 2: META-INF/spring.factories
org.springframework.context.ApplicationContextInitializer=\
  com.openframe.data.cassandra.config.CassandraKeyspaceNormalizer
```

```yaml
# application.yml — tenant ID with dashes is automatically normalized
spring:
  data:
    cassandra:
      keyspace-name: tenant-abc-123   # normalized to: tenant_abc_123
```

## Behavior

| Input Keyspace | Action |
|---|---|
| `null` or empty | Skip — no-op |
| `my_keyspace` (no dashes) | Skip — already valid |
| `tenant-abc-123` | Normalized → `tenant_abc_123` |
