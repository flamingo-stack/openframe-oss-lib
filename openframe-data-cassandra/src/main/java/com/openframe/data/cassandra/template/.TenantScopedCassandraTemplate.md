Wraps `CassandraOperations` to enforce tenant isolation on every read and write by resolving a `tenant_id` via `TenantIdProvider` before delegating to the underlying Cassandra driver.

## Key Components

| Member | Description |
|---|---|
| `requireTenantId()` | Single guard point — resolves tenant ID or throws `CassandraTenantException` if absent/blank |
| `insert(entity, tenantIdSetter)` | Stamps entity with current tenant ID via a `BiConsumer` setter, then delegates to `CassandraOperations.insert()` |
| `findById(keyBuilder, entityType)` | Builds a fully-scoped primary key via a `Function<String, K>` that receives the tenant ID, then delegates to `selectOneById()` |
| `CassandraTenantException` | Thrown when no tenant scope is present — no escape hatch |

Activated only when `spring.data.cassandra.enabled=true`.

## Usage Example

```java
// Insert — BiConsumer stamps tenant_id onto the entity's key
DeviceEntity device = new DeviceEntity();
device.setHostname("ap-01");
tenantCassandra.insert(device, (e, tid) -> e.getKey().setTenantId(tid));

// Point lookup — Function builds the composite key with tenant scope
Optional<DeviceEntity> result = tenantCassandra.findById(
    tenantId -> new DeviceKey(tenantId, deviceId),
    DeviceEntity.class
);

// Aggregate/CQL queries — pull tenant_id explicitly and pass to repository
String tenantId = tenantCassandra.requireTenantId();
List<DeviceEntity> devices = deviceRepository.findByTenant(tenantId);
```

> For multi-row CQL queries not expressible through this API, call `requireTenantId()` directly and pass the value to a `@Query` method that includes `AND tenant_id = ?N` in its CQL.
