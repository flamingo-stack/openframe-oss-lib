Logs Cassandra session diagnostic information at application startup, providing visibility into the active connection configuration and cluster topology.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `CassandraSessionLogger(CqlSession)` | Constructor | Accepts an injected `CqlSession` instance |
| `logSessionDetails()` | `@PostConstruct` method | Fires after bean initialization to emit session details via SLF4J |

**Logged fields:**
- Connected nodes (cluster topology)
- Current datacenter (from first available node)
- Active keyspace
- Configured contact points
- Load balancing local datacenter

## Usage Example

```java
// Register as a Spring bean alongside your CqlSession bean
@Configuration
public class CassandraConfig {

    @Bean
    public CqlSession cqlSession() {
        return CqlSession.builder()
                .withKeyspace("my_keyspace")
                .build();
    }

    @Bean
    public CassandraSessionLogger cassandraSessionLogger(CqlSession session) {
        return new CassandraSessionLogger(session);
    }
}
```

On application startup, output will resemble:

```text
INFO  CassandraSessionLogger - Cassandra Session Details:
INFO  CassandraSessionLogger - Connected Nodes: {uuid=Node(...)}
INFO  CassandraSessionLogger - Current Datacenter: datacenter1
INFO  CassandraSessionLogger - Current Keyspace: my_keyspace
INFO  CassandraSessionLogger - Contact Points from Config: [127.0.0.1:9042]
INFO  CassandraSessionLogger - Load Balancing DC: datacenter1
```

> If the session is `null` or an exception occurs during introspection, the logger emits a `WARN` or `ERROR` message respectively, preventing startup failures from this diagnostic component.
