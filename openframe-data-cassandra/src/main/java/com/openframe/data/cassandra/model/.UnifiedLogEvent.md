Cassandra entity mapping a unified, NIST-compliant log event record for integrated MSP tools (Fleet, Meshcentral, etc.), stored in the `unified_logs` table with a composite primary key optimized for time-series queries.

## Key Components

### `UnifiedLogEvent` (Table: `unified_logs`)
Top-level entity annotated with `@Table` and `@Data`. Holds non-key event metadata:

| Field | Column | Description |
|---|---|---|
| `userId` | `user_id` | User associated with the event |
| `deviceId` | `device_id` | Device/machine identifier |
| `hostname` | `hostname` | Device hostname |
| `organizationId` | `organization_id` | Org identifier |
| `organizationName` | `organization_name` | Org display name |
| `severity` | `severity` | Level: `info`, `warning`, `error`, etc. |
| `message` | `message` | Human-readable event description |
| `debeziumMessage` | `debezium_message` | Raw CDC payload (JSON or plain text) |
| `details` | `details` | Flexible tool-specific JSON details |

### `UnifiedLogEventKey` (`@PrimaryKeyClass`)
Composite key designed for scalable Cassandra partitioning and filtering:

| Column | Key Type | Ordinal | Notes |
|---|---|---|---|
| `ingest_day` | PARTITIONED | 0 | `YYYY-MM-DD` for time-series bucketing |
| `tool_type` | PARTITIONED | 1 | e.g. `meshcentral`, `fleet-mdm` |
| `tenant_id` | CLUSTERED | 2 | Multi-tenant isolation |
| `event_type` | CLUSTERED | 3 | e.g. `login`, `logout`, `script` |
| `event_timestamp` | CLUSTERED | 4 | `Instant`, **DESCENDING** for recent-first queries |
| `tool_event_id` | CLUSTERED | 5 | Original tool event ID for uniqueness |

## Usage Example

```java
UnifiedLogEvent.UnifiedLogEventKey key = new UnifiedLogEvent.UnifiedLogEventKey();
key.setIngestDay("2025-07-01");
key.setToolType("fleet-mdm");
key.setTenantId("tenant-abc");
key.setEventType("script");
key.setEventTimestamp(Instant.now());
key.setToolEventId("evt-98765");

UnifiedLogEvent event = new UnifiedLogEvent();
event.setKey(key);
event.setHostname("workstation-01");
event.setOrganizationId("org-xyz");
event.setSeverity("info");
event.setMessage("Script executed successfully");
event.setDetails("{\"scriptName\": \"patch_update.ps1\", \"exitCode\": 0}");

cassandraTemplate.insert(event);
```

> **Partition strategy:** Combining `ingest_day` + `tool_type` prevents hot partitions while enabling efficient per-day, per-tool log queries at scale.
