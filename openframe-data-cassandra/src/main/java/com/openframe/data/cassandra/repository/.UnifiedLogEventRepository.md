Spring Data Cassandra repository interface for persisting and retrieving `UnifiedLogEvent` entities, conditionally enabled based on the `spring.data.cassandra.enabled` configuration property.

## Key Components

| Component | Description |
|-----------|-------------|
| `UnifiedLogEventRepository` | Extends `CassandraRepository` to provide CRUD and query operations for `UnifiedLogEvent` entities |
| `@ConditionalOnProperty` | Bean is only registered when `spring.data.cassandra.enabled=true`; defaults to **disabled** if the property is absent |
| `UnifiedLogEvent.UnifiedLogEventKey` | Composite primary key type used to uniquely identify log event records in Cassandra |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class UnifiedLogEventService {

    private final UnifiedLogEventRepository repository;

    public void saveEvent(UnifiedLogEvent event) {
        repository.save(event);
    }

    public Optional<UnifiedLogEvent> findEvent(UnifiedLogEvent.UnifiedLogEventKey key) {
        return repository.findById(key);
    }

    public List<UnifiedLogEvent> getAllEvents() {
        return repository.findAll();
    }
}
```

## Configuration

Enable the repository by setting the following property in `application.yml`:

```yaml
spring:
  data:
    cassandra:
      enabled: true
```

> If `spring.data.cassandra.enabled` is not set or is `false`, the repository bean will **not** be instantiated, preventing Cassandra connection errors in environments where Cassandra is unavailable.
