Spring `@Configuration` class that bootstraps a Cassandra connection for the OpenFrame data layer, conditionally activated via `spring.data.cassandra.enabled=true`.

## Key Components

| Component | Description |
|-----------|-------------|
| `CassandraConfig` | Extends `AbstractCassandraConfiguration`; wires keyspace, contact points, datacenter, and port from application properties |
| `cassandraSession()` | Overrides the default `CqlSessionFactoryBean` to inject a programmatic `DriverConfigLoader` with load-balancing DC, contact points, and server-side timestamp generation |
| `ensureKeyspaceExists()` | Opens a temporary `CqlSession` (no keyspace) before the main session is established and runs `CREATE KEYSPACE IF NOT EXISTS` with `SimpleStrategy` |
| `cassandraSessionLogger` | Registers a `CassandraSessionLogger` bean for post-connection diagnostics |
| `@EnableCassandraRepositories` | Scans `com.openframe.data.cassandra.repository` for Spring Data Cassandra repository interfaces |

## Configuration Properties

```yaml
spring:
  data:
    cassandra:
      enabled: true
      local-datacenter: datacenter1
      keyspace-name: openframe
      contact-points: 127.0.0.1
      port: 9042                  # default: 9042
      replication-factor: 1       # default: 1
```

## Usage Example

```java
// Repositories are auto-discovered; inject and use directly
@Service
public class DeviceService {

    private final DeviceRepository deviceRepository;

    public DeviceService(DeviceRepository deviceRepository) {
        this.deviceRepository = deviceRepository;
    }

    public Device findById(UUID id) {
        return deviceRepository.findById(id)
                .orElseThrow(() -> new EntityNotFoundException("Device not found: " + id));
    }
}
```

> **Note:** `ensureKeyspaceExists()` uses `SimpleStrategy` replication, which is suitable for single-datacenter or development environments. For production multi-DC deployments, update the CQL to use `NetworkTopologyStrategy` and increase `replication-factor` accordingly.
