A core interface defining asynchronous and synchronous message publishing capabilities for Kafka producers in the OpenFrame messaging system.

## Key Components

- **sendAsyncMessage()** - Sends messages asynchronously returning a CompletableFuture for non-blocking operations
- **sendAndAwaitMessage()** - Sends messages synchronously with blocking wait for confirmation
- **KafkaMessage** - Custom message model for structured data transport
- **SendResult** - Spring Kafka result wrapper containing metadata about the send operation

## Usage Example

```java
@Service
public class NotificationService {
    
    @Autowired
    private MessageProducer messageProducer;
    
    public void sendUserNotification(String userId, String content) {
        KafkaMessage message = new KafkaMessage(content);
        
        // Async sending for high-throughput scenarios
        messageProducer.sendAsyncMessage("user-notifications", message, userId)
            .thenAccept(result -> log.info("Message sent successfully"))
            .exceptionally(ex -> {
                log.error("Failed to send message", ex);
                return null;
            });
    }
    
    public void sendCriticalAlert(String alertData) {
        KafkaMessage message = new KafkaMessage(alertData);
        
        // Sync sending for critical operations requiring confirmation
        messageProducer.sendAndAwaitMessage("critical-alerts", message, "system");
    }
}
```

The interface provides flexibility between performance (async) and reliability (sync) based on use case requirements, with both methods supporting custom partition keys for message routing.
