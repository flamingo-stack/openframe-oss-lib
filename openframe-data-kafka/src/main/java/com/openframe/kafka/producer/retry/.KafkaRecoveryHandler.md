A simple interface that defines the contract for handling failed Kafka message publishing attempts, allowing implementations to determine how to process and potentially retry failed messages.

## Key Components

- **`enqueue` method** - Core contract method that receives failure context including the exception, target topic, message key, and payload for custom recovery processing

## Usage Example

```java
// Implementation example for dead letter queue recovery
public class DeadLetterQueueRecoveryHandler implements KafkaRecoveryHandler {
    
    @Override
    public void enqueue(Throwable ex, String topic, String key, Object payload) {
        // Log the failure
        log.error("Failed to publish message to topic: {}, key: {}", topic, key, ex);
        
        // Send to dead letter topic
        String deadLetterTopic = topic + ".dead-letter";
        kafkaTemplate.send(deadLetterTopic, key, payload);
        
        // Store in database for manual review
        failedMessageRepository.save(new FailedMessage(topic, key, payload, ex.getMessage()));
    }
}

// Usage in Kafka producer configuration
@Bean
public KafkaRecoveryHandler recoveryHandler() {
    return new DeadLetterQueueRecoveryHandler();
}
```

This interface enables flexible error handling strategies such as dead letter queues, database persistence, or custom retry mechanisms while maintaining separation of concerns in Kafka producer error handling.
