Service implementation that provides reliable message publishing for OSS tenant operations through automatic retry mechanisms and failure recovery.

## Key Components

- **OssTenantRetryingKafkaProducer** - Main service class extending `BaseRetryingKafkaProducer`
- **Constructor Dependencies** - Requires `OssTenantKafkaProducer` and `KafkaRecoveryHandler` for operation
- **Conditional Activation** - Only active when `spring.oss-tenant.kafka.enabled=true`
- **Service Registration** - Auto-registered as Spring service bean

## Usage Example

```java
// Configuration to enable the service
# application.yml
spring:
  oss-tenant:
    kafka:
      enabled: true

// Autowired usage in other components
@Autowired
private OssTenantRetryingKafkaProducer retryingProducer;

// The service automatically handles retries for tenant message publishing
// Inherits retry logic from BaseRetryingKafkaProducer
// Uses KafkaRecoveryHandler for failed message recovery
```

The service acts as a wrapper around the base OSS tenant Kafka producer, adding resilience through inherited retry mechanisms. It's conditionally loaded based on configuration and leverages dependency injection for its core components.
