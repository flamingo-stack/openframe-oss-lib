Abstract base class providing retry functionality for Kafka message publishing with configurable backoff strategies and recovery handling for failed messages.

## Key Components

- **BaseRetryingKafkaProducer** - Abstract base class with Spring Retry annotations
- **publish()** - Main entry point with exponential backoff retry logic (5 attempts, 500ms-5s delay)
- **recover()** - Overloaded recovery methods for handling non-retryable and exhausted retry scenarios
- **MessageProducer** - Injected dependency for actual Kafka message sending
- **KafkaRecoveryHandler** - Handler for processing failed messages after retry exhaustion

## Usage Example

```java
@Component
public class OrderEventProducer extends BaseRetryingKafkaProducer {
    
    public OrderEventProducer(MessageProducer producer, 
                             KafkaRecoveryHandler recoveryHandler) {
        super(producer, recoveryHandler);
    }
}

// Usage in service
@Service
public class OrderService {
    
    @Autowired
    private OrderEventProducer eventProducer;
    
    public void processOrder(Order order) {
        KafkaMessage message = new KafkaMessage(order);
        eventProducer.publish("order-events", order.getId(), message);
    }
}
```

The class automatically retries `TransientKafkaSendException` with exponential backoff, immediately fails on `NonRetryableKafkaException`, and delegates failed messages to the recovery handler for alternative processing strategies like dead letter queues.
