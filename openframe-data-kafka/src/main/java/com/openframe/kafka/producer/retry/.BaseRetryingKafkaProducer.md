Abstract base class that provides Spring Retry-backed Kafka message publishing with automatic exponential backoff and recovery routing for both transient and non-retryable failures.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `producer` | `MessageProducer` | Underlying Kafka producer used to send and await message delivery |
| `recoveryHandler` | `KafkaRecoveryHandler` | Handles failed messages after retries are exhausted (e.g. DLQ, outbox) |
| `publish()` | `@Retryable` method | Sends a message with up to 5 attempts and exponential backoff (500ms → 5000ms max) |
| `recover(NonRetryableKafkaException)` | `@Recover` | Immediately routes non-retryable failures to the recovery handler |
| `recover(TransientKafkaSendException)` | `@Recover` | Routes transient failures to the recovery handler after all attempts are exhausted |

### Retry Policy

```text
Max Attempts : 5
Initial Delay: 500ms
Multiplier   : 2.0x
Max Delay    : 5000ms
Retry On     : TransientKafkaSendException
No Retry On  : NonRetryableKafkaException
```

## Usage Example

```java
@Component
public class OrderEventProducer extends BaseRetryingKafkaProducer {

    public OrderEventProducer(MessageProducer producer,
                               KafkaRecoveryHandler recoveryHandler) {
        super(producer, recoveryHandler);
    }

    public void sendOrderCreated(OrderCreatedEvent event) {
        publish("orders.created", event.getOrderId(), event);
    }
}
```

Subclasses only need to inject dependencies and delegate to `publish()` — retry orchestration and recovery are fully handled by the base class via Spring AOP. Ensure Spring Retry is enabled with `@EnableRetry` on your application or configuration class.
