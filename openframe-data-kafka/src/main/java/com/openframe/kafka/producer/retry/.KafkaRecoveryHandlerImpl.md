Concrete implementation of Kafka recovery handling that logs failed message details for debugging and monitoring purposes. This service is invoked when messages cannot be successfully delivered after retry attempts.

## Key Components

- **KafkaRecoveryHandlerImpl**: Service class implementing `KafkaRecoveryHandler` interface
- **enqueue()**: Core method that processes failed Kafka messages and logs comprehensive error information
- **ObjectMapper**: Jackson mapper instance for potential JSON serialization (currently uses toString())

## Usage Example

This recovery handler is automatically invoked by the Kafka retry mechanism when message delivery fails:

```java
// Automatically called by Kafka retry infrastructure
@KafkaListener(topics = "user-events")
public void handleUserEvent(UserEvent event) {
    // If this method throws an exception after retries are exhausted,
    // KafkaRecoveryHandlerImpl.enqueue() will be called with:
    // - The final exception
    // - Topic name ("user-events")  
    // - Message key
    // - The UserEvent payload
    throw new RuntimeException("Processing failed");
}

// Manual usage (rarely needed)
@Autowired
private KafkaRecoveryHandler recoveryHandler;

public void handleFailure() {
    recoveryHandler.enqueue(
        new RuntimeException("Custom error"),
        "my-topic",
        "message-key", 
        new UserEvent("data")
    );
}
```

The recovery handler logs structured error information including topic, key, exception details, and payload data for troubleshooting failed message processing.
