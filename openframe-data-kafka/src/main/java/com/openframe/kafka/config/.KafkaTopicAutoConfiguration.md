Auto-configuration class that automatically creates Kafka topics based on inbound configuration properties for the OpenFrame MSP platform. Only activates when auto-creation is enabled and runs after the main OSS tenant Kafka configuration.

## Key Components

- **`kafkaTopics()`** - Bean that creates `KafkaAdmin.NewTopics` from inbound topic configurations
- **Topic Configuration Processing** - Iterates through inbound properties to build NewTopic objects
- **Conditional Activation** - Only runs when `openframe.oss-tenant.kafka.topics.auto-create` is true
- **Logging** - Reports each registered topic with its partition and replication settings

## Usage Example

```java
// Application properties configuration
openframe.oss-tenant.kafka.topics.auto-create=true
openframe.oss-tenant.kafka.topics.inbound.alerts.name=alert-events
openframe.oss-tenant.kafka.topics.inbound.alerts.partitions=3
openframe.oss-tenant.kafka.topics.inbound.alerts.replication-factor=2

// The auto-configuration will automatically create topics on startup
// Topics are built using Spring's TopicBuilder with specified partitions and replicas
// Logs will show: "Registered Kafka topic for auto-creation: alert-events (partitions=3, replicas=2)"
```

This configuration ensures Kafka topics for the OpenFrame platform are automatically provisioned with proper partitioning and replication settings during application startup, eliminating manual topic creation steps in MSP deployments.
