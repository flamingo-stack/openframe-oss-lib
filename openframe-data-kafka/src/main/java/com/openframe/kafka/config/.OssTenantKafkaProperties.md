Configuration properties class for OSS tenant Kafka cluster settings, extending Spring Boot's standard Kafka configuration with tenant-specific prefixing.

## Key Components

- **@ConfigurationProperties**: Binds properties with `spring.oss-tenant` prefix
- **enabled**: Boolean flag to enable/disable OSS Kafka cluster (default: true)
- **kafka**: Nested KafkaProperties instance containing all standard Kafka settings

## Usage Example

```java
// In application.yml
spring:
  oss-tenant:
    enabled: true
    kafka:
      bootstrap-servers: localhost:9092
      consumer:
        group-id: oss-tenant-group
        key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
        value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      producer:
        key-serializer: org.apache.kafka.common.serialization.StringSerializer
        value-serializer: org.apache.kafka.common.serialization.StringSerializer

// Inject in configuration class
@Autowired
private OssTenantKafkaProperties ossTenantProperties;

// Check if enabled and access Kafka properties
if (ossTenantProperties.isEnabled()) {
    KafkaProperties kafkaConfig = ossTenantProperties.getKafka();
    // Configure Kafka beans using kafkaConfig
}
```

The class inherits all standard Kafka configuration options from Spring Boot's KafkaProperties, allowing complete customization of consumer, producer, admin, and security settings while maintaining separation from other Kafka clusters in a multi-tenant environment.
