Configuration class that disables Spring Boot's default Kafka auto-configuration to allow for custom Kafka setup in the OpenFrame OSS project.

## Key Components

- **@EnableKafka** - Enables Kafka functionality and annotation-driven listener endpoints
- **@EnableAutoConfiguration** - Configures Spring Boot auto-configuration with exclusions
- **KafkaAutoConfiguration.class exclusion** - Prevents default Kafka configuration from loading

## Usage Example

```java
// This configuration class is automatically loaded by Spring Boot
// It allows you to define custom Kafka beans without conflicts

@Configuration
public class CustomKafkaProducerConfig {
    
    @Bean
    public ProducerFactory<String, Object> producerFactory() {
        Map<String, Object> configProps = new HashMap<>();
        configProps.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        configProps.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        configProps.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, JsonSerializer.class);
        return new DefaultKafkaProducerFactory<>(configProps);
    }
    
    @Bean
    public KafkaTemplate<String, Object> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

This configuration enables full control over Kafka producer, consumer, and listener configurations while preventing Spring Boot's default settings from interfering with custom implementations.
