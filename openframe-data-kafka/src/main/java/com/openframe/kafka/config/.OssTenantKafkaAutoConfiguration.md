Auto-configuration class that sets up Spring Kafka beans for the OSS tenant cluster in OpenFrame's multi-tenant messaging architecture. This configuration enables Kafka producers, consumers, and admin operations using the `spring.oss-tenant.kafka` property prefix.

## Key Components

- **`ossTenantKafkaProducerFactory`** - Creates producer factory with String/JSON serialization
- **`ossTenantKafkaTemplate`** - Main KafkaTemplate for sending messages to OSS cluster
- **`ossTenantKafkaConsumerFactory`** - Creates consumer factory with String/JSON deserialization  
- **`ossTenantKafkaListenerContainerFactory`** - Configures concurrent message listeners with customizable acknowledgment modes
- **`ossTenantKafkaProducer`** - High-level producer wrapper for OSS tenant operations
- **`ossTenantKafkaAdmin`** - Admin client for topic management and cluster operations
- **`kafkaTopics`** - Auto-creates topics based on `KafkaTopicProperties` configuration

## Usage Example

```java
@Service
public class NotificationService {
    
    @Autowired
    @Qualifier("ossTenantKafkaProducer")
    private OssTenantKafkaProducer kafkaProducer;
    
    public void sendNotification(String topic, NotificationEvent event) {
        kafkaProducer.send(topic, event);
    }
    
    @KafkaListener(
        topics = "user-events",
        containerFactory = "ossTenantKafkaListenerContainerFactory"
    )
    public void handleUserEvent(UserEvent event) {
        // Process OSS tenant events
        log.info("Received user event: {}", event);
    }
}
```

Configuration is activated when `spring.oss-tenant.kafka.enabled=true` and uses `OssTenantKafkaProperties` for cluster-specific settings like bootstrap servers, security, and listener concurrency.
