A data model representing integrated tool events for Kafka messaging, encapsulating security and monitoring event information from various tools and devices.

## Key Components

- **IntegratedToolEvent**: Main event class implementing `KafkaMessage` interface
- **Event Identification**: `toolEventId`, `eventType`, `toolType` for categorizing events
- **User Context**: `userId`, `deviceId`, `hostname` for tracking event sources
- **Organization Context**: `organizationId`, `organizationName` for multi-tenant support
- **Event Details**: `severity`, `summary`, `eventTimestamp` for event classification and description
- **Data Management**: `ingestDay` for partitioning and time-based queries

## Usage Example

```java
// Creating a new integrated tool event
IntegratedToolEvent event = new IntegratedToolEvent();
event.setToolEventId("evt-12345");
event.setUserId("user@company.com");
event.setDeviceId("laptop-001");
event.setHostname("workstation.local");
event.setOrganizationId("org-abc123");
event.setOrganizationName("Acme Corp");
event.setIngestDay("2024-01-15");
event.setToolType("antivirus");
event.setEventType("threat_detected");
event.setSeverity("HIGH");
event.setSummary("Malware detected and quarantined");
event.setEventTimestamp(System.currentTimeMillis());

// The event can now be published to Kafka topics
kafkaProducer.send("security-events", event);
```
