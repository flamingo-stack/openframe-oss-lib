A Kafka message model representing integrated tool events in the OpenFrame system, implementing the KafkaMessage interface for event streaming and processing.

## Key Components

- **KafkaMessage Interface**: Implements the base interface for Kafka message handling
- **Event Identification**: `toolEventId` for unique event identification
- **User Context**: `userId`, `deviceId`, and `hostname` for user and device tracking
- **Organization Data**: `organizationId` and `organizationName` for multi-tenant support
- **Event Classification**: `toolType`, `eventType`, and `severity` for categorization
- **Temporal Data**: `eventTimestamp` and `ingestDay` for time-based processing
- **Event Details**: `summary` field for event description

## Usage Example

```java
// Create a new integrated tool event
IntegratedToolEvent event = new IntegratedToolEvent();
event.setToolEventId("evt-12345");
event.setUserId("user-789");
event.setDeviceId("device-abc");
event.setHostname("workstation-01");
event.setOrganizationId("org-456");
event.setOrganizationName("Acme Corp");
event.setIngestDay("2024-01-15");
event.setToolType("security-scanner");
event.setEventType("vulnerability-detected");
event.setSeverity("HIGH");
event.setSummary("Critical vulnerability found in application");
event.setEventTimestamp(System.currentTimeMillis());

// Send to Kafka topic
kafkaTemplate.send("integrated-tool-events", event);
```

The class uses Lombok's `@Data` annotation to automatically generate getters, setters, toString, equals, and hashCode methods, reducing boilerplate code for this data transfer object.
