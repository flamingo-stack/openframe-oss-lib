A generic model class for handling Debezium change data capture (CDC) messages from Kafka streams, providing strongly-typed access to database change events including before/after states and metadata.

## Key Components

- **`DebeziumMessage<T>`** - Main container for Debezium CDC messages with generic payload type
- **`Payload<T>`** - Inner class containing the actual change data and metadata
- **`Source`** - Nested class with source system information (database, table, connector details)

Key fields:
- `before/after` - Entity states before and after the change
- `operation` - Type of database operation (INSERT, UPDATE, DELETE)
- `timestamp` - Event timestamp in milliseconds
- `source` - Metadata about the source database and connector

## Usage Example

```java
// Deserialize a Debezium message for a User entity
DebeziumMessage<User> message = objectMapper.readValue(kafkaRecord, 
    new TypeReference<DebeziumMessage<User>>() {});

// Access the change data
User beforeState = message.getPayload().getBefore();
User afterState = message.getPayload().getAfter();
String operation = message.getPayload().getOperation(); // "c", "u", "d", "r"

// Get source metadata
DebeziumMessage.Payload.Source source = message.getPayload().getSource();
String tableName = source.getTable();
String database = source.getDatabase();
Long eventTimestamp = message.getPayload().getTimestamp();

// Handle different operations
switch (operation) {
    case "c": // CREATE/INSERT
        processNewRecord(afterState);
        break;
    case "u": // UPDATE
        processUpdate(beforeState, afterState);
        break;
    case "d": // DELETE
        processDelete(beforeState);
        break;
}
```
