Kafka message model representing machine/device events published to a Kafka topic for ingestion into Apache Pinot.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `tenantId` | `String` | Tenant identifier for multi-tenancy support |
| `machineId` | `String` | Unique identifier for the machine/device |
| `organizationId` | `String` | Organization the machine belongs to |
| `deviceType` | `String` | Classification of the device |
| `status` | `String` | Current machine status |
| `osType` | `String` | Operating system type |
| `tags` | `List<String>` | Tag key names mapped to Pinot multi-value (MV) columns |
| `tagKeyValues` | `List<String>` | `"key:value"` pairs used for Pinot indexing |
| `ingestionTime` | `long` | Epoch timestamp of when the event was ingested |

Implements `KafkaMessage` and is triggered by changes in `Machine`, `MachineTag`, or `Tag` collections.

## Usage Example

```java
MachinePinotMessage message = MachinePinotMessage.builder()
    .tenantId("tenant-123")
    .machineId("machine-456")
    .organizationId("org-789")
    .deviceType("workstation")
    .status("online")
    .osType("Windows")
    .tags(List.of("environment", "region"))
    .tagKeyValues(List.of("environment:production", "region:us-east-1"))
    .ingestionTime(System.currentTimeMillis())
    .build();

kafkaTemplate.send("machines-topic", message.getMachineId(), message);
```

> **Note:** The `tags` and `tagKeyValues` lists serve different Pinot purposes — `tags` populates MV columns for filtering, while `tagKeyValues` enables compound key-value index lookups.
