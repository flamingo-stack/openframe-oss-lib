A data model representing machine events that are published to Kafka when changes occur in Machine, MachineTag, or Tag collections.

## Key Components

- **MachinePinotMessage**: Main data transfer object implementing `KafkaMessage` interface
- **Fields**: 
  - `machineId` - Unique machine identifier
  - `organizationId` - Organization that owns the machine
  - `deviceType` - Type/category of the device
  - `status` - Current operational status
  - `osType` - Operating system type
  - `tags` - List of associated tags
- **Lombok annotations**: Provides builder pattern, getters/setters, and constructors

## Usage Example

```java
// Create a machine event message
MachinePinotMessage message = MachinePinotMessage.builder()
    .machineId("machine-001")
    .organizationId("org-123")
    .deviceType("server")
    .status("active")
    .osType("linux")
    .tags(List.of("production", "web-server", "east-coast"))
    .build();

// Send to Kafka
kafkaTemplate.send("machine-events", message);

// Access fields
String machineId = message.getMachineId();
List<String> tags = message.getTags();
```

This message is typically used in event-driven architectures to notify downstream systems about machine state changes or updates.
