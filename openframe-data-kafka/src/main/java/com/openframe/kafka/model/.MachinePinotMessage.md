Represents machine event messages sent to Kafka when changes occur in Machine, MachineTag, or Tag collections, implementing the KafkaMessage interface for event-driven communication.

## Key Components

- **MachinePinotMessage** - Main message model class with Lombok annotations for automatic getter/setter generation
- **KafkaMessage** - Interface implementation for Kafka message compatibility
- **Builder pattern** - Enabled through `@Builder` annotation for fluent object construction
- **Core fields** - Machine identification, organization context, device metadata, and tag associations

## Usage Example

```java
// Creating a machine event message
MachinePinotMessage message = MachinePinotMessage.builder()
    .machineId("machine-123")
    .organizationId("org-456")
    .deviceType("SERVER")
    .status("ACTIVE")
    .osType("LINUX")
    .tags(Arrays.asList("production", "web-server", "critical"))
    .build();

// Sending to Kafka (typical usage in service layer)
kafkaTemplate.send("machine-events-topic", message);

// Accessing message data
String machineId = message.getMachineId();
List<String> machineTags = message.getTags();
```

The class uses Lombok annotations to reduce boilerplate code and provides a clean data structure for machine state changes that need to be propagated through the Kafka messaging system to downstream services like Apache Pinot for analytics.
