A runtime exception that represents a transient failure when sending messages to Kafka, indicating the operation can potentially be retried.

## Key Components

- **TransientKafkaSendException**: A custom runtime exception class that extends `RuntimeException`
- **Constructor**: Takes a message string and throwable cause for detailed error information

## Usage Example

This exception is typically thrown by Kafka producers when encountering temporary failures that may succeed on retry:

```java
try {
    kafkaProducer.send(producerRecord);
} catch (Exception e) {
    // Wrap retriable Kafka errors as transient exceptions
    throw new TransientKafkaSendException(
        "Failed to send message to topic: " + topic, e
    );
}

// In error handling code
try {
    sendMessageToKafka(message);
} catch (TransientKafkaSendException e) {
    // This exception indicates the operation can be retried
    logger.warn("Transient Kafka send failure: {}", e.getMessage());
    retryPolicy.executeWithRetry(() -> sendMessageToKafka(message));
}
```

The "transient" nature suggests this exception represents temporary issues like network timeouts, broker unavailability, or resource constraints rather than permanent configuration or data problems.
