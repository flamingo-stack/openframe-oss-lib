A custom runtime exception designed to represent transient failures when sending messages to Kafka that may succeed on retry.

## Key Components

- **TransientKafkaSendException**: A runtime exception class that extends `RuntimeException`
- **Constructor**: Takes a message string and causing throwable for detailed error context

## Usage Example

```java
// Throwing the exception when a Kafka send operation fails transiently
try {
    producer.send(record).get();
} catch (ExecutionException e) {
    if (isTransientError(e.getCause())) {
        throw new TransientKafkaSendException(
            "Failed to send message to topic: " + record.topic(), 
            e.getCause()
        );
    }
}

// Catching and handling transient exceptions with retry logic
try {
    kafkaService.sendMessage(message);
} catch (TransientKafkaSendException e) {
    logger.warn("Transient Kafka send failure, will retry: {}", e.getMessage());
    retryManager.scheduleRetry(() -> kafkaService.sendMessage(message));
}
```

This exception helps distinguish between permanent failures (which shouldn't be retried) and temporary issues like network timeouts or broker unavailability that may resolve themselves.
