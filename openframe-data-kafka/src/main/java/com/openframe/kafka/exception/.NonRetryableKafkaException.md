A custom runtime exception for handling non-retryable Kafka operations that should not be automatically retried by the messaging system.

## Key Components

- **NonRetryableKafkaException** - Runtime exception class that signals Kafka operations should not be retried
- **Constructor** - Takes a message and underlying cause for comprehensive error reporting

## Usage Example

```java
// Throwing when encountering a non-retryable condition
public void processMessage(String message) {
    try {
        // Process Kafka message
        validateMessageFormat(message);
    } catch (InvalidFormatException e) {
        // Don't retry malformed messages
        throw new NonRetryableKafkaException(
            "Message format is invalid and cannot be processed", e);
    }
}

// Catching in error handler
try {
    kafkaService.processMessage(message);
} catch (NonRetryableKafkaException e) {
    // Log error and move to dead letter queue
    logger.error("Non-retryable error: {}", e.getMessage());
    deadLetterService.send(message, e);
}
```

This exception is typically used in Kafka consumers to distinguish between temporary failures (network issues) that should be retried and permanent failures (invalid data format) that should be handled differently, such as routing to a dead letter queue.
