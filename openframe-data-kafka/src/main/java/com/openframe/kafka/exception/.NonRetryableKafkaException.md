A custom runtime exception for Kafka operations that should not be retried when they fail.

## Key Components

- **NonRetryableKafkaException**: A runtime exception class that extends `RuntimeException` and signals that a Kafka operation has failed in a way that should not trigger retry logic.

## Usage Example

```java
// Throw when encountering a non-retryable Kafka error
try {
    // Kafka operation that fails due to configuration issue
    producer.send(record);
} catch (ConfigurationException e) {
    throw new NonRetryableKafkaException(
        "Invalid Kafka configuration detected", e);
}

// In error handling logic
try {
    kafkaService.processMessage(message);
} catch (NonRetryableKafkaException e) {
    // Don't retry, log and move on
    logger.error("Non-retryable Kafka error: {}", e.getMessage());
    deadLetterQueue.send(message);
} catch (Exception e) {
    // Other exceptions might be retryable
    retryHandler.scheduleRetry(message);
}
```

This exception helps distinguish between transient Kafka failures (network issues, broker unavailability) that should be retried and permanent failures (serialization errors, invalid topics) that should be handled differently.
