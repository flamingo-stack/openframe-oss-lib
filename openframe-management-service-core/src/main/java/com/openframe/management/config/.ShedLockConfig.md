Configuration class that sets up ShedLock distributed locking for scheduled tasks, ensuring only one instance of scheduled jobs runs across multiple application instances using Redis as the lock store.

## Key Components

- **@EnableSchedulerLock** - Enables ShedLock with 10-minute default lock duration
- **@EnableScheduling** - Activates Spring's scheduled task execution
- **lockProvider()** - Creates Redis-based lock provider with tenant-scoped keys

## Configuration Details

The lock provider uses a hierarchical key structure for multi-tenant isolation:
- **Key Format**: `of:{tenantId}:job-lock:{environment}:{lockName}`
- **Environment Scoping**: Separates locks by deployment environment (dev, staging, prod)
- **Tenant Isolation**: Each tenant gets separate lock namespace using `OpenframeRedisKeyBuilder`

## Usage Example

```java
@Component
public class ScheduledTasks {
    
    @Scheduled(fixedRate = 60000)
    @SchedulerLock(name = "dataSync", lockAtMostFor = "5m", lockAtLeastFor = "1m")
    public void syncData() {
        // This method will only run on one instance at a time
        // Lock automatically released after execution or 5 minutes
    }
    
    @Scheduled(cron = "0 0 2 * * ?")
    @SchedulerLock(name = "dailyCleanup")
    public void dailyMaintenance() {
        // Uses default 10-minute lock from configuration
    }
}
```

The configuration prevents duplicate scheduled task execution in clustered deployments while maintaining tenant isolation and environment separation.
