Configures ShedLock distributed locking mechanism using Redis to prevent concurrent execution of scheduled tasks across multiple application instances.

## Key Components

- **@EnableSchedulerLock**: Activates ShedLock with 10-minute maximum lock duration
- **@EnableScheduling**: Enables Spring's task scheduling capabilities
- **lockProvider()**: Creates Redis-based lock provider for distributed coordination

## Usage Example

```java
@Component
public class ScheduledTasks {
    
    @Scheduled(cron = "0 0 * * * ?")
    @SchedulerLock(name = "hourlyDataSync", 
                   lockAtMostFor = "50m", 
                   lockAtLeastFor = "5m")
    public void syncData() {
        // This method will only run on one instance
        // even if multiple app instances are running
        processDataSync();
    }
    
    @Scheduled(fixedRate = 30000)
    @SchedulerLock(name = "healthCheck")
    public void performHealthCheck() {
        // Uses default 10m lockAtMostFor from config
        checkSystemHealth();
    }
}
```

The configuration ensures that scheduled tasks annotated with `@SchedulerLock` execute on only one application instance at a time, preventing duplicate processing in clustered environments.
