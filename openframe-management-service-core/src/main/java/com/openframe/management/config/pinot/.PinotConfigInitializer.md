Initializes Apache Pinot database schemas and table configurations on application startup, deploying predefined configurations for devices and logs tables to the Pinot controller via REST API with retry logic.

## Key Components

- **@PostConstruct init()** - Main initialization method that deploys all Pinot configurations on startup
- **deployPinotConfig()** - Orchestrates deployment of schema and table configs for a specific data type
- **deployWithRetry()** - Implements retry logic with configurable attempts and delays for resilient deployment
- **deploySchema()** / **deployTableConfig()** - Handle REST API calls to Pinot controller for schema and table creation/updates
- **PINOT_CONFIGS** - Static list defining configurations for devices and logs tables with their respective schema and config files

## Configuration Properties

The class uses several configurable properties:
- `pinot.controller.url` - Pinot controller endpoint
- `pinot.config.enabled` - Toggle for enabling/disabling configuration deployment
- `pinot.config.retry.max-attempts` - Maximum retry attempts (default: 5)
- `pinot.config.retry.delay-ms` - Delay between retries (default: 5000ms)

## Usage Example

```java
// Configuration is automatically deployed on application startup
// No manual invocation needed - uses @PostConstruct

// Example configuration files expected in classpath:
// - pinot/config/schema-devices.json
// - pinot/config/table-config-devices.json
// - pinot/config/schema-logs.json
// - pinot/config/table-config-logs-realtime.json
// - pinot/config/table-config-logs-offline.json

// To disable Pinot configuration deployment:
// pinot.config.enabled=false

// To configure retry behavior:
// pinot.config.retry.max-attempts=3
// pinot.config.retry.delay-ms=2000
```

The initializer automatically handles both table creation (for new tables) and updates (for existing tables) using HTTP PUT/POST operations with appropriate fallback logic.
