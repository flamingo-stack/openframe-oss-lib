A Spring Boot configuration class that automatically deploys Pinot schemas and table configurations to a Pinot cluster during application startup, with retry logic and error handling.

## Key Components

- **PinotConfigInitializer** - Main configuration class that manages Pinot deployment lifecycle
- **PINOT_CONFIGS** - Static list of predefined configurations for "devices" and "logs" tables
- **deployPinotConfig()** - Core deployment method that handles schema and table configuration
- **deployWithRetry()** - Retry mechanism with configurable attempts and delays
- **deploySchema()** / **deployTableConfig()** - HTTP client methods for Pinot REST API calls
- **loadResource()** - Utility to load JSON configuration files from classpath

## Usage Example

```java
// Configuration properties in application.yml
@Value("${pinot.controller.url}")
private String pinotControllerUrl = "localhost:9000";

@Value("${pinot.config.enabled:true}")
private boolean pinotConfigEnabled = true;

// The initializer runs automatically on startup
@PostConstruct
public void init() {
    // Deploys schemas and table configs for:
    // - devices table (schema-devices.json, table-config-devices.json)
    // - logs table (schema-logs.json, table-config-logs-realtime.json)
}

// Place configuration files in src/main/resources/pinot/config/
// - schema-devices.json
// - table-config-devices.json  
// - schema-logs.json
// - table-config-logs-realtime.json
```

The class uses Spring's `@PostConstruct` to automatically deploy Pinot configurations on application startup, with built-in retry logic for handling temporary connection issues. It attempts to update existing tables first, falling back to creation if the table doesn't exist.
