Automatically initializes Apache Pinot schemas and table configurations on application startup by deploying JSON configuration files to the Pinot controller via REST API with retry logic.

## Key Components

- **PinotConfigInitializer** - Main configuration class that handles automatic deployment
- **deployPinotConfig()** - Orchestrates schema and table configuration deployment for each defined config
- **deployWithRetry()** - Implements exponential backoff retry mechanism for network failures
- **deploySchema()** - Deploys schema configurations to Pinot controller
- **deployTableConfig()** - Handles both creation and updates of realtime/offline table configurations
- **PINOT_CONFIGS** - Static list defining devices and logs table configurations

## Configuration Properties

```java
# Enable/disable Pinot configuration deployment
pinot.config.enabled=true

# Pinot controller URL
pinot.controller.url=localhost:9000

# Retry configuration
pinot.config.retry.max-attempts=5
pinot.config.retry.delay-ms=5000
```

## Usage Example

The initializer automatically runs after Spring application startup and deploys configurations from the classpath:

```java
// Configuration files expected in classpath:pinot/config/
// - schema-devices.json
// - table-config-devices.json
// - schema-logs.json
// - table-config-logs-realtime.json

// Supports placeholder resolution in JSON files
{
  "tableName": "${pinot.table.prefix}_devices",
  "columns": [...]
}
```

The component handles both table creation (when tables don't exist) and updates (when tables already exist), making it suitable for both initial deployments and configuration updates.
