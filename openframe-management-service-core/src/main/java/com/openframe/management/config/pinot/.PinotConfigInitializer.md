A Spring Boot configuration class that automatically initializes and deploys Apache Pinot schemas and table configurations to a Pinot cluster on application startup.

## Key Components

- **PinotConfigInitializer**: Main configuration class with `@PostConstruct` initialization
- **PINOT_CONFIGS**: Static list defining device and log configurations to deploy
- **deployPinotConfig()**: Orchestrates schema and table deployment for each configuration
- **deployWithRetry()**: Implements retry logic with configurable attempts and delays
- **deploySchema()**: Deploys schema configurations via REST API
- **deployTableConfig()**: Handles table creation/updates with fallback logic
- **loadResource()**: Loads configuration files from classpath

## Usage Example

```java
// Application properties configuration
pinot.controller.url=localhost:9000
pinot.config.enabled=true
pinot.config.retry.max-attempts=3
pinot.config.retry.delay-ms=2000

// The component auto-initializes on Spring context startup
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
        // Pinot configs automatically deployed via @PostConstruct
    }
}

// Custom configuration files should be placed at:
// src/main/resources/pinot/config/schema-devices.json
// src/main/resources/pinot/config/table-config-devices.json
```

The initializer supports retry mechanisms for network failures and gracefully handles both table creation and updates based on existing table status.
