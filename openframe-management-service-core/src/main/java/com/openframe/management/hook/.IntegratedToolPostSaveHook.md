A lightweight hook interface for executing custom logic after an IntegratedTool entity is saved, providing an extension point without requiring Spring's event system overhead.

## Key Components

- **`onToolSaved(String, IntegratedTool)`** - Callback method triggered post-save with tool ID and entity data

## Usage Example

```java
@Component
public class ToolAuditHook implements IntegratedToolPostSaveHook {
    
    private final AuditService auditService;
    
    public ToolAuditHook(AuditService auditService) {
        this.auditService = auditService;
    }
    
    @Override
    public void onToolSaved(String toolId, IntegratedTool tool) {
        auditService.logToolUpdate(toolId, tool.getName(), 
            tool.getLastModified());
    }
}

@Component
public class CacheInvalidationHook implements IntegratedToolPostSaveHook {
    
    private final CacheManager cacheManager;
    
    @Override
    public void onToolSaved(String toolId, IntegratedTool tool) {
        cacheManager.evict("tools", toolId);
        cacheManager.evict("tool-configs", tool.getConfigId());
    }
}
```

This hook pattern allows multiple services to react to tool saves (auditing, caching, notifications) without tight coupling or Spring event complexity. Implementations are automatically discovered and invoked by the tool management service.
