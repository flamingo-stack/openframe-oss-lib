REST controller managing CRUD operations for integrated tools within the OpenFrame platform, handling tool configuration persistence, Debezium connector lifecycle, and post-save hook execution.

## Key Components

- **`getTools()`** — `GET /v1/tools` — Returns all tools for the current tenant
- **`getTool(id)`** — `GET /v1/tools/{id}` — Fetches a single tool by ID
- **`saveTool(id, request)`** — `POST /v1/tools/{id}` — Persists tool configuration, conditionally applies Debezium connectors based on tenant registration state, and triggers post-save hooks
- **`SaveToolRequest`** — Inner DTO wrapping the `IntegratedTool` payload
- **`IntegratedToolPostSaveHook`** — Extension point; all registered hooks are invoked after a successful save (failures are logged but non-fatal)

### Debezium Connector Handling

If no tenant is registered yet, connector definitions are stored to MongoDB only. Once a tenant is active, connectors are applied to Kafka Connect immediately via `DebeziumService`.

## Usage Example

```java
// GET all tools
GET /v1/tools
// Response: { "status": "success", "tools": [...] }

// Save/update a tool configuration
POST /v1/tools/zabbix
Content-Type: application/json

{
  "tool": {
    "name": "Zabbix",
    "debeziumConnectors": [...]
  }
}
// Response: { "status": "success", "tool": { ... } }
```

```java
// Implementing a post-save hook
@Component
public class MyHook implements IntegratedToolPostSaveHook {
    @Override
    public void onToolSaved(String toolId, IntegratedTool savedTool) {
        // custom logic after tool is saved
    }
}
```
