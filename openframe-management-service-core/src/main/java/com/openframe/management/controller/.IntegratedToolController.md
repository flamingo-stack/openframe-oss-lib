REST controller for managing integrated tools within the OpenFrame management system, providing endpoints for retrieving and saving tool configurations with automatic Debezium connector management.

## Key Components

- **IntegratedToolController** - Main REST controller class handling tool operations
- **IntegratedToolService** - Service layer for tool data management
- **DebeziumService** - Service for managing Debezium connectors
- **SaveToolRequest** - Request DTO for tool save operations

### Endpoints

- `GET /v1/tools` - Retrieves all available tools
- `GET /v1/tools/{id}` - Retrieves a specific tool by ID
- `POST /v1/tools/{id}` - Saves or updates a tool configuration

## Usage Example

```java
// GET all tools
// Response: {"status": "success", "tools": [...]}

// GET specific tool
// GET /v1/tools/my-tool-id
// Response: {"status": "success", "tool": {...}}
// or {"status": "error", "message": "Tool not found"}

// Save tool configuration
SaveToolRequest request = new SaveToolRequest();
IntegratedTool tool = new IntegratedTool();
tool.setName("My Integration Tool");
tool.setConfiguration(config);
request.setTool(tool);

// POST /v1/tools/my-tool-id
// Automatically sets tool.id = "my-tool-id" and tool.enabled = true
// Creates/updates associated Debezium connectors
// Response: {"status": "success", "tool": {...}}
```

The controller automatically enables saved tools and triggers Debezium connector updates for data streaming capabilities. All responses follow a consistent JSON structure with status indicators and error handling.
