REST controller managing CRUD operations for integrated tools within the OpenFrame platform, handling tool configuration persistence, Debezium connector lifecycle, and post-save hook execution.

## Key Components

| Element | Description |
|---|---|
| `GET /v1/tools` | Returns all configured tools for the current tenant |
| `GET /v1/tools/{id}` | Retrieves a single tool by its key identifier |
| `POST /v1/tools/{id}` | Saves or updates a tool configuration with full lifecycle management |
| `SaveToolRequest` | Inner DTO wrapping the `IntegratedTool` payload |
| `IntegratedToolPostSaveHook` | Extension points executed after a successful tool save |
| `DebeziumService` | Conditionally applies Kafka Connect connector changes post-tenant registration |
| `TenantIdProvider` | Supplies the current tenant context and registration status |

## Usage Example

```java
// POST /v1/tools/servicenow
// Body:
{
  "tool": {
    "name": "ServiceNow",
    "debeziumConnectors": [...]
  }
}

// Response (success):
{
  "status": "success",
  "tool": { ... }
}

// Response (not found - GET):
{
  "status": "error",
  "message": "Tool not found"
}
```

## Behavior Notes

- **Idempotent saves**: existing tools are looked up by `tenantId + key` to preserve MongoDB `_id` on updates
- **Deferred connector creation**: Debezium connectors are saved to MongoDB only when no tenant is registered yet; they are applied to Kafka Connect immediately on subsequent saves after tenant registration
- **Resilient hooks**: post-save hook failures are caught and logged as warnings, preventing a failed hook from rolling back the save operation
