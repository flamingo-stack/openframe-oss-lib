REST controller for managing OpenFrame cluster release version updates. Handles POST requests to update the current release version using image tag information.

## Key Components

- **ReleaseVersionController**: Main controller class with `/v1/cluster-registrations` base mapping
- **updateReleaseVersion()**: POST endpoint that accepts release version requests
- **ReleaseVersionService**: Injected service handling the business logic for version processing
- **ReleaseVersionRequest**: DTO containing image tag version information

## Usage Example

```java
// Example request to update release version
@PostMapping("/v1/cluster-registrations")
public void updateReleaseVersion(@RequestBody ReleaseVersionRequest request) {
    // Processes the image tag version (e.g., "v2.1.3", "latest")
    releaseVersionService.process(request.getImageTagVersion());
}

// Example request body:
// {
//   "imageTagVersion": "v2.1.3"
// }
```

The controller uses Spring Boot annotations for dependency injection (`@RequiredArgsConstructor`) and logging (`@Slf4j`). It delegates version processing logic to the service layer, maintaining clean separation of concerns in the application architecture.
