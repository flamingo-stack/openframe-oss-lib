REST controller that exposes an endpoint for updating the release version of cluster registrations by accepting an image tag version from the request body.

## Key Components

| Component | Description |
|-----------|-------------|
| `ReleaseVersionController` | Spring `@RestController` mapped to `/v1/cluster-registrations` |
| `updateReleaseVersion()` | `POST` handler that extracts `imageTagVersion` and delegates to `ReleaseVersionService` |
| `ReleaseVersionService` | Injected service responsible for processing the version update |

## Usage Example

```java
// POST /v1/cluster-registrations
// Content-Type: application/json

{
  "imageTagVersion": "v1.4.2"
}
```

```java
// Internally, the controller delegates to the service:
@PostMapping
public void updateReleaseVersion(@RequestBody ReleaseVersionRequest request) {
    String imageTagVersion = request.getImageTagVersion();
    releaseVersionService.process(imageTagVersion);
}
```

> **Note:** The endpoint returns no response body (`void`). A `200 OK` status indicates the version update was accepted and processed successfully.
