Scheduled fallback component that retries publishing unpublished agent version updates to NATS when initial publish attempts have failed, ensuring eventual consistency for `OpenFrameClientConfiguration` and `IntegratedToolAgent` entities.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `publishUnpublishedEntities()` | `@Scheduled` method | Main scheduled entry point; processes both client config and tool agents |
| `processOpenframeClient()` | Private method | Checks and retries publishing the `OpenFrameClientConfiguration` if unpublished |
| `processToolAgents()` | Private method | Iterates all enabled `IntegratedToolAgent` instances and retries publishing if needed |
| `shouldRetryPublish(PublishState)` | Private method | Returns `true` if the entity has never been published and has not exceeded `maxPublishAttempts` |
| `maxPublishAttempts` | `@Value` field | Max retry attempts before giving up; defaults to `5` |

**Activation:** Only loaded when `openframe.agent-version-update-publish-fallback.enabled=true`.

**Distributed locking:** Uses ShedLock (`agentVersionUpdatePublishFallback`) to prevent concurrent execution across multiple instances.

## Usage Example

```java
# application.yml

openframe:
  agent-version-update-publish-fallback:
    enabled: true
    interval: 30000        # Run every 30 seconds
    max-attempts: 5        # Stop retrying after 5 failed attempts
    lock-at-most-for: 5m   # ShedLock max hold duration
    lock-at-least-for: 10s # ShedLock min hold duration
```

**Retry logic summary:**

- `PublishState` is `null` → retry (entity never attempted)
- `isPublished() == true` → skip (already succeeded)
- `attempts < maxPublishAttempts` → retry (still within limit)
- `attempts >= maxPublishAttempts` → skip (give up)
