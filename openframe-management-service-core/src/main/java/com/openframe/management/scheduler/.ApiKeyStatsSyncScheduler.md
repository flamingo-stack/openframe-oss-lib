Scheduled component that periodically synchronizes API key statistics from Redis to MongoDB using distributed locking to prevent concurrent execution across multiple instances.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Activates only when `openframe.api-key-stats.enabled=true` (enabled by default) |
| `@SchedulerLock` | ShedLock annotation ensuring only one node runs the sync in a distributed environment |
| `syncStatsToMongo()` | Scheduled task that delegates to `ApiKeyStatsSyncService` for the actual Redis → MongoDB sync |
| `init()` | `@PostConstruct` lifecycle hook that logs scheduler initialization |

## Configuration Properties

| Property | Default | Description |
|---|---|---|
| `openframe.api-key-stats.enabled` | `true` | Enables/disables the scheduler |
| `openframe.api-key-stats.sync-interval` | *(required)* | Fixed delay between sync executions |
| `openframe.api-key-stats.lock-at-most-for` | `10m` | Maximum duration ShedLock holds the distributed lock |
| `openframe.api-key-stats.lock-at-least-for` | `1m` | Minimum duration ShedLock holds the lock (prevents re-entry on fast completion) |

## Usage Example

```java
// application.yml
openframe:
  api-key-stats:
    enabled: true
    sync-interval: 5m
    lock-at-most-for: 10m
    lock-at-least-for: 1m
```

```java
// Bean is auto-registered by Spring; no manual wiring needed.
// ShedLock requires a lock provider bean configured separately:
@Bean
public LockProvider lockProvider(MongoClient mongoClient) {
    return new MongoLockProvider(mongoClient.getDatabase("openframe"));
}
```

> **Note:** ShedLock's `lockAtLeastFor` prevents the same node from immediately re-acquiring the lock after a fast sync, which is critical in environments with sub-second scheduling intervals.
