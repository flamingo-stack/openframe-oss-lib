Scheduled Spring component that periodically detects and marks devices as offline when their heartbeat becomes stale, acting as the trigger layer for the `DeviceHeartbeatOfflineDetectionService`.

## Key Components

| Element | Description |
|---|---|
| `detectOfflineDevices()` | Scheduled task that invokes the offline detection sweep at a configurable fixed delay |
| `@ConditionalOnProperty` | Enables the scheduler only when `openframe.device.heartbeat.offline-detection.enabled=true` |
| `@Scheduled` | Runs with a fixed delay defaulting to `60000ms` (60s), configurable via `openframe.device.heartbeat.offline-detection.interval` |
| `DeviceHeartbeatOfflineDetectionService` | Injected service that performs the actual stale device marking logic |

## Configuration Properties

```yaml
openframe:
  device:
    heartbeat:
      offline-detection:
        enabled: true
        interval: 60000  # milliseconds (default: 60s)
```

## Usage Example

```java
// The scheduler is auto-registered as a Spring bean when enabled.
// No manual invocation needed — triggered automatically by Spring's task scheduler.

// To enable in application.yml:
// openframe.device.heartbeat.offline-detection.enabled=true

// The scheduler delegates directly to the detection service:
offlineDetectionService.markStaleDevicesOffline();
```

> **Note:** If `openframe.device.heartbeat.offline-detection.enabled` is missing or set to `false`, this scheduler bean is **not registered**, allowing the feature to be disabled without code changes — useful for environments where heartbeat detection is handled externally or not yet required.
