Scheduled Spring component that periodically checks whether the FleetMDM server integration requires setup, triggering API token generation and persistence when the tool is present.

## Key Components

| Element | Description |
|---------|-------------|
| `@ConditionalOnProperty` | Activates the scheduler only when `openframe.fleet-mdm.setup.enabled=true` |
| `FLEETMDM_SERVER` | Repository lookup key (`"fleetmdm-server"`) used to find the integrated tool record |
| `runSetupIfNeeded()` | Scheduled method that queries the tool repository and delegates to `FleetMdmSetupService` if the tool is found |
| `fixedDelayString` | Polling interval controlled by `openframe.fleet-mdm.setup.scheduler-interval-ms` (default: 30 seconds) |

## Usage Example

```java
// application.yml configuration to enable and tune the scheduler
openframe:
  fleet-mdm:
    setup:
      enabled: true
      scheduler-interval-ms: 60000  # Poll every 60 seconds
```

```java
// The scheduler automatically fires on the configured interval.
// If fleetmdm-server is registered in IntegratedToolRepository,
// FleetMdmSetupService handles token setup:
toolRepository.findByKey("fleetmdm-server")
    .ifPresent(fleetMdmSetupService::setupAndSaveApiToken);

// Errors are caught and logged; the scheduler retries on the next tick
// without crashing the application context.
```

> **Note:** Setting `openframe.fleet-mdm.setup.enabled=false` (or omitting it) disables the bean entirely — no polling occurs, and no Fleet MDM setup is attempted.
