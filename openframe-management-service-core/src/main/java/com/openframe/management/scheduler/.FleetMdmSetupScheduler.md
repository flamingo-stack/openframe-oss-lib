Scheduled component that periodically checks whether the Fleet MDM server integration requires setup, triggering token provisioning when the tool is present.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Activates only when `openframe.fleet-mdm.setup.enabled=true` |
| `FLEETMDM_SERVER` | Constant ID (`"fleetmdm-server"`) used to look up the tool in the repository |
| `runSetupIfNeeded()` | Scheduled task that queries `IntegratedToolRepository` and delegates to `FleetMdmSetupService` if the tool record exists |
| `scheduler-interval-ms` | Configurable polling interval (default: `30000` ms) |

## Usage Example

```java
// application.yml
openframe:
  fleet-mdm:
    setup:
      enabled: true
      scheduler-interval-ms: 60000  # override default 30s interval
```

```java
// The scheduler fires automatically at the configured interval.
// If the "fleetmdm-server" tool entry exists in the DB, setup is triggered:
toolRepository.findById("fleetmdm-server")
    .ifPresent(fleetMdmSetupService::setupAndSaveApiToken);
// Errors are caught and logged; the next tick will retry automatically.
```

## Behavior Notes

- **Conditional activation** — the bean is only registered when the feature flag is `true`; no-op in environments where Fleet MDM setup is disabled.
- **Non-fatal errors** — exceptions are caught, logged, and silently retried on the next scheduled tick to avoid crashing the scheduler thread.
- **Idempotent design** — `setupAndSaveApiToken` is expected to be safe to call repeatedly; the scheduler does not track state between ticks.
