Scheduled component that periodically checks for a Fleet MDM server integration and triggers setup (API token provisioning) if the tool is configured.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Only activates when `openframe.fleet-mdm.setup.enabled=true` |
| `FLEETMDM_SERVER` | Repository lookup key `"fleetmdm-server"` used to find the integrated tool |
| `runSetupIfNeeded()` | Scheduled method that queries the tool repository and delegates to `FleetMdmSetupService` if the Fleet MDM entry exists |

## Configuration Properties

| Property | Default | Description |
|---|---|---|
| `openframe.fleet-mdm.setup.enabled` | `false` | Enables or disables this scheduler |
| `openframe.fleet-mdm.setup.scheduler-interval-ms` | `30000` | Fixed delay (ms) between each execution |

## Usage Example

```yaml
# application.yml
openframe:
  fleet-mdm:
    setup:
      enabled: true
      scheduler-interval-ms: 60000
```

```java
// FleetMdmSetupService (consumed by scheduler)
@Service
public class FleetMdmSetupService {
    public void setupAndSaveApiToken(IntegratedTool tool) {
        // Provisions and persists the Fleet MDM API token
    }
}
```

The scheduler runs on a fixed delay (not a fixed rate), meaning the next execution starts only after the previous one completes. Errors are caught and logged without stopping the scheduler, allowing automatic retry on the next tick.
