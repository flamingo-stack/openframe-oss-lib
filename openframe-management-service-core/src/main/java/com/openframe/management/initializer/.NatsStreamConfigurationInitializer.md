Initializes and registers NATS JetStream stream configurations at application startup by implementing Spring's `ApplicationRunner` interface.

## Key Components

- **`CONFIGURATIONS`** — Static list of predefined `StreamConfiguration` objects covering five core streams:

| Stream Name | Subject Pattern | Purpose |
|---|---|---|
| `TOOL_INSTALLATION` | `machine.*.tool-installation` | Tool installation events |
| `CLIENT_UPDATE` | `machine.*.client-update` | Client update events |
| `TOOL_UPDATE` | `machine.*.tool.*.update` | Tool agent update events |
| `TOOL_CONNECTIONS` | `machine.*.tool-connection` | Tool connection events |
| `INSTALLED_AGENTS` | `machine.*.installed-agent` | Installed agent reporting |

- **`NatsStreamManagementService`** — Injected service responsible for persisting each stream configuration to NATS JetStream.
- **`AdditionalStreamConfigurationProvider`** — Supplies supplemental stream configurations beyond the static defaults, allowing extensibility without modifying this class.
- **`run(ApplicationArguments)`** — Entry point invoked at startup; iterates both static and additional configurations, registering each via `natsStreamManagementService::save`.

## Usage Example

```java
// Automatically executed by Spring Boot at startup — no manual invocation needed.
// To add custom streams, implement AdditionalStreamConfigurationProvider:

@Component
public class MyStreamProvider implements AdditionalStreamConfigurationProvider {

    @Override
    public List<StreamConfiguration> provide() {
        return List.of(
            StreamConfiguration.builder()
                .name("CUSTOM_STREAM")
                .subjects(List.of("machine.*.custom-event"))
                .storageType(StorageType.File)
                .retentionPolicy(RetentionPolicy.Limits)
                .build()
        );
    }
}
```

> **Note:** All streams use `StorageType.File` and `RetentionPolicy.Limits`. A future TODO indicates these configurations should be externalized to a JSON file for easier management.
