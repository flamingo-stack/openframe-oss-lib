This Spring component initializes predefined NATS JetStream configurations on application startup, setting up streams for machine tool management operations like installations, updates, and connections.

## Key Components

- **CONFIGURATIONS**: Static list of five pre-configured NATS streams for different machine operations:
  - `TOOL_INSTALLATION`: Handles tool installation events
  - `CLIENT_UPDATE`: Manages client update notifications  
  - `TOOL_UPDATE`: Processes tool agent updates
  - `TOOL_CONNECTIONS`: Tracks tool connection events
  - `INSTALLED_AGENTS`: Monitors installed agent information

- **run()**: `@PostConstruct` method that automatically creates all configured streams during application initialization

## Usage Example

```java
// The component runs automatically on startup, but you can also inject it
@Autowired
private NatsStreamConfigurationInitializer initializer;

// Manual initialization (if needed)
initializer.run();

// Example of how the streams are configured
StreamConfiguration toolInstallation = StreamConfiguration.builder()
    .name("TOOL_INSTALLATION")
    .subjects(List.of("machine.*.tool-installation"))
    .storageType(StorageType.File)
    .retentionPolicy(RetentionPolicy.Limits)
    .build();
```

All streams use file storage with limits-based retention policy and follow the subject pattern `machine.{machineId}.{operation}` for message routing.
