Initializes and synchronizes scripts with TacticalRMM on application startup. This component automatically creates or updates predefined scripts in TacticalRMM based on local resource files.

## Key Components

- **ApplicationRunner implementation** - Executes script synchronization during Spring Boot startup
- **SCRIPT_CONFIGS** - Static configuration list defining scripts to sync (currently includes OpenFrame Client updater)
- **processScript()** - Core method that handles create/update logic for individual scripts
- **loadScriptFromResources()** - Loads PowerShell script content from classpath resources
- **createScript()/updateScript()** - TacticalRMM API integration methods

## Usage Example

```java
// The initializer runs automatically on startup, but scripts are configured via:
private static final List<ScriptConfig> SCRIPT_CONFIGS = List.of(
    ScriptConfig.builder()
        .name("OpenFrame Client Latest Update")
        .resourcePath("classpath:scripts/openframe-client-update.ps1")
        .description("OpenFrame Client update to latest version")
        .shell("powershell")
        .category("OpenFrame")
        .defaultTimeout(30)
        .build()
);

// Add new scripts by extending the SCRIPT_CONFIGS list
// Place corresponding .ps1 files in src/main/resources/scripts/
```

The component requires TacticalRMM integration to be configured with valid API credentials and automatically handles script deduplication by name.
