Bootstraps Tactical RMM script synchronization on application startup, ensuring predefined PowerShell scripts exist (creating or updating as needed) in the connected Tactical RMM instance.

## Key Components

| Element | Description |
|---|---|
| `TOOL_ID` | Constant identifier `"tactical-rmm"` used to look up the integration config |
| `SCRIPT_CONFIGS` | Static list of `ScriptConfig` entries defining scripts to sync (currently: `OpenFrame Client Latest Update`) |
| `run()` | `ApplicationRunner` entry point — resolves API credentials, fetches existing scripts, and delegates to `processScript()` |
| `processScript()` | Loads script content from classpath resources, checks for an existing match by name, then routes to create or update |
| `loadScriptFromResources()` | Reads a classpath resource (`.ps1` file) into a UTF-8 string |
| `findScriptByName()` | Streams the existing script list to find a name match |
| `createScript()` | Builds a `CreateScriptRequest` and calls `TacticalRmmClient.addScript()` |
| `updateScript()` | Builds a `CreateScriptRequest` and calls `TacticalRmmClient.updateScript()` with the existing script ID |

## Usage Example

The initializer runs automatically at startup via `ApplicationRunner`. To add a new managed script, append an entry to `SCRIPT_CONFIGS` and place the script file on the classpath:

```java
ScriptConfig.builder()
    .name("OpenFrame Agent Health Check")
    .resourcePath("classpath:scripts/openframe-agent-health.ps1")
    .description("Verifies OpenFrame agent is running and healthy")
    .shell("powershell")
    .category("OpenFrame")
    .defaultTimeout(60)
    .build()
```

```text
src/main/resources/
└── scripts/
    ├── openframe-client-update.ps1   ← existing
    └── openframe-agent-health.ps1    ← new script file
```

On next startup, the initializer will detect the script is absent in Tactical RMM and create it automatically. If it already exists, it will be updated in place.
