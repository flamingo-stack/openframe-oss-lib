Bootstraps Tactical RMM script synchronization at application startup, ensuring all configured scripts are present and up-to-date in the connected Tactical RMM instance by creating or updating them as needed.

## Key Components

- **`TOOL_ID`** — Constant identifier (`"tactical-rmm"`) used to look up the integration in `IntegratedToolService`
- **`SCRIPT_CONFIGS`** — Static list of `ScriptConfig` entries defining scripts to sync (currently: `OpenFrame Client Latest Update`, a PowerShell script for updating the OpenFrame client)
- **`run(ApplicationArguments)`** — Entry point (`ApplicationRunner`); resolves API credentials, fetches existing remote scripts, then delegates per-script processing with success/failure tracking
- **`processScript(...)`** — Loads script content from classpath resources and routes to create or update based on name matching
- **`loadScriptFromResources(String)`** — Reads a classpath resource into a UTF-8 string, throwing if not found
- **`findScriptByName(...)`** — Streams existing remote scripts to find a name match
- **`createScript(...)`** / **`updateScript(...)`** — Builds a `CreateScriptRequest` and calls the appropriate `TacticalRmmClient` method

## Usage Example

```java
// Automatically invoked by Spring Boot on startup via ApplicationRunner.
// To add a new managed script, append a ScriptConfig entry:

private static final List<ScriptConfig> SCRIPT_CONFIGS = List.of(
    ScriptConfig.builder()
        .name("OpenFrame Client Latest Update")
        .resourcePath("classpath:scripts/openframe-client-update.ps1")
        .description("OpenFrame Client update to latest version")
        .shell("powershell")
        .category("OpenFrame")
        .defaultTimeout(30)
        .build(),

    ScriptConfig.builder()
        .name("My Custom Remediation Script")
        .resourcePath("classpath:scripts/my-remediation.ps1")
        .description("Runs custom remediation tasks")
        .shell("powershell")
        .category("OpenFrame")
        .defaultTimeout(60)
        .build()
);
```

> **Note:** The corresponding `.ps1` file must exist under `src/main/resources/scripts/`. If the Tactical RMM tool is not configured in `IntegratedToolService`, startup initialization will log an error and exit gracefully without crashing the application.
