A Spring Boot application initializer that automatically synchronizes PowerShell scripts from the classpath to a Tactical RMM server during application startup.

## Key Components

- **ApplicationRunner Implementation**: Executes automatically after Spring Boot application starts
- **Script Configuration**: Manages predefined script metadata including name, resource path, description, and execution parameters
- **Tactical RMM Integration**: Uses `TacticalRmmClient` to create and update scripts on the remote management server
- **Resource Loading**: Loads PowerShell script content from classpath resources using Spring's `ResourceLoader`
- **Error Handling**: Comprehensive logging and exception handling for script synchronization operations

## Usage Example

The initializer runs automatically on application startup, but here's how it processes scripts:

```java
// Script configuration is defined as a static list
private static final List<ScriptConfig> SCRIPT_CONFIGS = List.of(
    ScriptConfig.builder()
        .name("OpenFrame Client Latest Update")
        .resourcePath("classpath:scripts/openframe-client-update.ps1")
        .description("OpenFrame Client update to latest version")
        .shell("powershell")
        .category("OpenFrame")
        .defaultTimeout(30)
        .build()
);

// The initializer automatically:
// 1. Loads Tactical RMM connection details from IntegratedToolService
// 2. Fetches existing scripts from the server
// 3. Creates new scripts or updates existing ones based on configuration
// 4. Reports success/failure statistics
```

To add new scripts, simply add them to the `SCRIPT_CONFIGS` list and place the corresponding PowerShell files in the `src/main/resources/scripts/` directory. The initializer will handle the synchronization automatically on the next application restart.
