Initializes the OpenFrame client configuration from a JSON file during application startup, managing both new installations and updates to existing configurations.

## Key Components

- **@PostConstruct init()** - Main initialization method that loads configuration from classpath
- **processExistingConfiguration()** - Handles updates to existing configurations while preserving version and publish state
- **processNewConfiguration()** - Creates new configuration when none exists
- **CONFIG_FILE** - Points to `agent-configurations/client-configuration.json` resource

## Usage Example

The initializer runs automatically on Spring Boot startup:

```java
// Configuration is loaded from classpath resource
// agent-configurations/client-configuration.json

// Example client configuration JSON structure:
{
  "agentSettings": {
    "maxRetries": 3,
    "timeout": 30000
  },
  "features": {
    "autoUpdate": true,
    "telemetry": false
  }
}
```

The component automatically:
- Loads configuration from the embedded JSON file
- Sets the default ID using `OpenFrameClientConfigurationService.DEFAULT_ID`
- Preserves existing version numbers and publish states during updates
- Creates new configurations for fresh installations

This ensures consistent client configuration across OpenFrame deployments while maintaining upgrade compatibility.
