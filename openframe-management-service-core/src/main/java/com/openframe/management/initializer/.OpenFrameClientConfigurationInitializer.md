Initializes the OpenFrame client configuration at application startup by loading a default configuration from a bundled JSON file and persisting it via the configuration service.

## Key Components

| Element | Description |
|---|---|
| `OpenFrameClientConfigurationInitializer` | Spring `ApplicationRunner` component that runs at startup with `@Order(30)` |
| `CONFIG_FILE` | Classpath resource path: `agent-configurations/client-configuration.json` |
| `run()` | Reads the JSON config, assigns the default ID, and calls `updateConfigurationFields()` |

## Usage Example

This initializer runs automatically on startup. No manual invocation is required. The underlying JSON file must exist on the classpath:

```java
// Triggered automatically by Spring Boot at startup (Order = 30)
// Reads from: src/main/resources/agent-configurations/client-configuration.json

// Internally performs:
ClassPathResource resource = new ClassPathResource(CONFIG_FILE);
OpenFrameClientConfiguration fromConfig =
    objectMapper.readValue(resource.getInputStream(), OpenFrameClientConfiguration.class);
fromConfig.setId(DEFAULT_ID);
clientConfigurationService.updateConfigurationFields(fromConfig);
```

> **Note:** The `@Order(30)` annotation ensures this initializer runs after earlier-ordered startup components, maintaining a predictable initialization sequence across the OpenFrame platform.
