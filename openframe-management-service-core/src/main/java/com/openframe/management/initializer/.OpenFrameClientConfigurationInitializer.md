A Spring Boot component that initializes the OpenFrame client configuration from a JSON file on application startup, handling both new installations and existing configuration updates.

## Key Components

- **@PostConstruct init()** - Main initialization method that loads configuration from classpath
- **processExistingConfiguration()** - Handles updates to existing configurations while preserving version numbers
- **processNewConfiguration()** - Creates new default configuration for fresh installations
- **ObjectMapper** - Jackson JSON processor for deserializing configuration files
- **OpenFrameClientConfigurationService** - Service layer for configuration persistence

## Usage Example

```java
// The component automatically runs during Spring Boot startup
// Configuration is loaded from: agent-configurations/client-configuration.json

// Example configuration file structure:
{
  "displaySettings": {
    "resolution": "1920x1080",
    "refreshRate": 60
  },
  "networkSettings": {
    "timeout": 5000,
    "retryAttempts": 3
  }
}

// Manual service usage (if needed):
@Autowired
private OpenFrameClientConfigurationService configService;

public void updateConfig() {
    Optional<OpenFrameClientConfiguration> config = 
        configService.findById("default");
    
    if (config.isPresent()) {
        // Modify configuration
        configService.save(config.get());
    }
}
```

The initializer ensures version preservation during updates and provides comprehensive logging for troubleshooting configuration issues during application startup.
