Loads and applies the OpenFrame client configuration from a bundled JSON file at application startup, ensuring the client configuration is initialized before the application begins serving requests.

## Key Components

| Element | Description |
|---------|-------------|
| `OpenFrameClientConfigurationInitializer` | Spring `ApplicationRunner` component that executes during startup |
| `CONFIG_FILE` | Classpath resource path `agent-configurations/client-configuration.json` |
| `ObjectMapper` | Deserializes the JSON config file into an `OpenFrameClientConfiguration` document |
| `OpenFrameClientConfigurationService` | Applies the deserialized configuration via `updateConfigurationFields()` |
| `@Order(30)` | Controls execution order relative to other `ApplicationRunner` initializers |

## Usage Example

This component runs automatically on startup — no manual invocation required. To customize the client configuration, update the bundled JSON file:

```java
// Classpath resource resolved at startup:
// src/main/resources/agent-configurations/client-configuration.json

// Internally, the initializer performs:
ClassPathResource resource = new ClassPathResource("agent-configurations/client-configuration.json");
OpenFrameClientConfiguration config = objectMapper.readValue(
    resource.getInputStream(),
    OpenFrameClientConfiguration.class
);
clientConfigurationService.updateConfigurationFields(config);
```

To register a new initializer at a specific order relative to this one, use `@Order` with a value other than `30`:

```java
@Component
@Order(31) // Runs after OpenFrameClientConfigurationInitializer
public class MyPostConfigInitializer implements ApplicationRunner {
    // depends on client configuration being already applied
}
```

> **Note:** If the JSON file is missing or malformed, an `IOException` is thrown and will halt application startup. Ensure `client-configuration.json` is present and valid on every deployment.
