Loads and applies the OpenFrame client configuration at application startup by reading a JSON config file from the classpath and persisting it via the configuration service.

## Key Components

| Element | Description |
|---|---|
| `ApplicationRunner` | Spring Boot hook that triggers execution after context initialization |
| `@Order(30)` | Controls initialization sequence relative to other `ApplicationRunner` beans |
| `CONFIG_FILE` | Classpath resource path: `agent-configurations/client-configuration.json` |
| `ObjectMapper` | Deserializes the JSON config into an `OpenFrameClientConfiguration` document |
| `OpenFrameClientConfigurationService` | Persists/updates the loaded configuration fields |

## Usage Example

The initializer runs automatically on startup — no manual invocation required. To customize the client configuration, modify the source JSON file:

```java
// Classpath resource loaded at startup:
// src/main/resources/agent-configurations/client-configuration.json

// Internally, the runner executes:
ClassPathResource resource = new ClassPathResource("agent-configurations/client-configuration.json");
OpenFrameClientConfiguration config = objectMapper.readValue(
    resource.getInputStream(),
    OpenFrameClientConfiguration.class
);
clientConfigurationService.updateConfigurationFields(config);
```

To register a custom initializer at a different priority, adjust the `@Order` value:

```java
@Component
@Order(30) // Lower number = earlier execution
public class OpenFrameClientConfigurationInitializer implements ApplicationRunner {
    // ...
}
```

> **Note:** This initializer runs at order `30`, meaning it executes after lower-order initializers complete. Ensure any dependencies (e.g., database connectivity) are ready before this order value.
