Bootstraps `IntegratedToolAgent` configurations at application startup by reading JSON config files from the classpath and persisting them via `IntegratedToolAgentService`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `run()` | `ApplicationRunner` override | Entry point called on startup; validates config path list and iterates over each path |
| `processAgentConfiguration()` | Private method | Loads a single classpath JSON file, deserializes it into `IntegratedToolAgentConfiguration`, and applies it via the service layer |
| `@Order(20)` | Annotation | Controls initialization order relative to other `ApplicationRunner` beans |
| `AgentConfigurationProperties` | Dependency | Supplies the list of classpath-relative JSON file paths to load |
| `IntegratedToolAgentService` | Dependency | Persists or updates the deserialized configuration via `updateConfigurationFields()` |

## Behavior Notes

- Throws `IllegalStateException` at startup if no configuration paths are defined — treated as a hard failure.
- Missing individual config files emit a `WARN` log and are skipped gracefully, allowing partial initialization.
- Exceptions during deserialization or persistence are caught per-file and logged as `ERROR` without halting remaining configurations.

## Usage Example

```java
// application.yml — register config file paths
agent:
  configurations:
    - agents/mingo-agent.json
    - agents/fae-agent.json
```

```java
// Corresponding classpath resource (e.g. resources/agents/mingo-agent.json)
{
  "id": "mingo-agent",
  "name": "Mingo AI",
  "tools": ["ticket-lookup", "kb-search"]
}
```

On startup, the initializer deserializes each JSON file into `IntegratedToolAgentConfiguration` and calls `integratedToolAgentService.updateConfigurationFields(configuration)` to upsert the agent definition.
