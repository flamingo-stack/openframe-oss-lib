Initializes IntegratedToolAgent configurations from classpath resources during Spring application startup. Handles creation of new agents and updates to existing ones while preserving release versions and managing version-based notifications.

## Key Components

- `initializeToolAgents()` - Main `@PostConstruct` method that loads and processes all agent configurations
- `processAgentConfiguration()` - Loads agent configuration from classpath resource and handles parsing
- `processExistingAgent()` - Updates existing agents while preserving release version constraints
- `processNewAgent()` - Creates new agent configurations from resource files
- `processVersionUpdate()` - Publishes update notifications when agent versions change

## Usage Example

The initializer runs automatically during Spring Boot startup. Agent configurations are defined in the application properties:

```java
// Application configuration
@ConfigurationProperties("agent")
public class AgentConfigurationProperties {
    private List<String> agentConfigurations = List.of(
        "agents/data-processor-agent.json",
        "agents/monitoring-agent.json"
    );
}

// Example agent configuration JSON structure
{
    "id": "data-processor-v1",
    "version": "1.2.0",
    "name": "Data Processing Agent",
    "isReleaseVersion": false,
    "publishState": "DRAFT"
}
```

The component automatically:
- Loads agent configurations from classpath resources
- Creates new agents or updates existing ones
- Preserves versions for release agents
- Publishes update notifications for version changes
