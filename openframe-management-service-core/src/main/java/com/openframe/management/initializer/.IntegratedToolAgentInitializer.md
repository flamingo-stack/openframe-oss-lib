A Spring component that initializes IntegratedToolAgent configurations from classpath resources during application startup. It handles both creation of new agents and updates to existing agents while preserving release version integrity.

## Key Components

- **`@PostConstruct initializeToolAgents()`** - Main initialization method that processes all configured agent paths
- **`processAgentConfiguration(String)`** - Loads and parses individual agent configuration files from classpath
- **`processExistingAgent()`** - Updates existing agents while preserving release versions
- **`processNewAgent()`** - Creates new agent configurations
- **`processVersionUpdate()`** - Publishes version update events for non-release agents

## Usage Example

```java
// Configuration in application.yml
agent:
  agentConfigurations:
    - "agents/chatbot-agent.json"
    - "agents/analytics-agent.json"

// The initializer runs automatically on startup
// Agent JSON files should be in src/main/resources/agents/
// Example agent configuration structure:
{
  "id": "chatbot-v1",
  "version": "1.0.0",
  "releaseVersion": false,
  "configuration": {...}
}

// For programmatic access:
@Autowired
private IntegratedToolAgentService agentService;

// Access initialized agents
Optional<IntegratedToolAgent> agent = agentService.findById("chatbot-v1");
```

The component ensures release versions are never overwritten and publishes update events when agent versions change.
