A Mongock `@ChangeUnit` (order `003`) that seeds system ticket status definitions and migrates legacy `status` string fields on existing ticket documents to the new structured `statusId`/`statusKind` model. Guarded by the `openframe.features.tickets.lifecycle.enabled` feature flag and marked `runAlways=true` pending full rollout.

## Key Components

| Member | Description |
|---|---|
| `LEGACY_TO_KIND` | Static mapping from legacy status strings (`ACTIVE`, `TECH_REQUIRED`, `RESOLVED`, `ARCHIVED`) to `TicketStatusKind` enum values |
| `execution()` | Entry point — checks feature flag, seeds statuses, then migrates legacy ticket documents |
| `seedStatuses()` | Upserts system-level `TicketStatusDefinition` records; conditionally seeds the custom `ON_HOLD` status if absent |
| `migrateTickets()` | Finds all tenant tickets missing `statusId`, then delegates per-document migration with error isolation |
| `migrateOne()` | Resolves the correct `TicketStatusDefinition` for a ticket's legacy status and writes `statusId`, `statusKind`, `updatedAt`; removes the old `status` field |
| `resolveTarget()` | Handles `ON_HOLD` as a special case (name-based lookup with `TECH_REQUIRED` fallback); all other values go through kind-based lookup |
| `rollback()` | No-op — migration is forward-only |

## Usage Example

```java
// Mongock picks this up automatically via component scan.
// Triggered on application startup when the lifecycle flag is enabled:

// application.yml
// openframe:
//   features:
//     tickets:
//       lifecycle:
//         enabled: true

// Per-ticket before migration:
// { "_id": "abc", "tenantId": "t1", "status": "ACTIVE", ... }

// Per-ticket after migration:
// {
//   "_id": "abc",
//   "tenantId": "t1",
//   "statusId": "<TicketStatusDefinition._id>",
//   "statusKind": "AI_ASSISTANCE",
//   "updatedAt": "2025-01-15T10:30:00Z"
//   // "status" field removed
// }
```

> **Note:** `runAlways=true` is temporary. Once the lifecycle feature flag is permanently enabled, this should be converted to a standard one-shot migration and the flag guard removed — see the `TODO(lifecycle-rollout)` comments in the source.
