A Mongock `@ChangeUnit` migration (order `003`) that seeds system ticket status definitions and migrates existing tickets from a flat legacy `status` string field to the new structured `statusId`/`statusKind` model. Guarded by a feature flag and marked `runAlways=true` pending full lifecycle rollout.

## Key Components

| Member | Description |
|--------|-------------|
| `LEGACY_TO_KIND` | Static map translating legacy status strings (`ACTIVE`, `TECH_REQUIRED`, `RESOLVED`, `ARCHIVED`) to `TicketStatusKind` enum values |
| `execution()` | Mongock entry point — checks the `openframe.features.tickets.lifecycle.enabled` flag, seeds statuses, then migrates tickets |
| `rollback()` | No-op rollback (migration is idempotent by design) |
| `seedStatuses()` | Upserts system `TicketStatusDefinition` documents; inserts the custom `ON_HOLD` status only if absent |
| `migrateTickets()` | Finds all tenant tickets missing `statusId`, iterates and migrates each with per-document error isolation |
| `resolveTarget()` | Maps a legacy status string to a `TicketStatusDefinition`; handles `ON_HOLD` with a `TECH_REQUIRED` fallback |
| `applyUpdate()` | Writes `statusId`, `statusKind`, updated `updatedAt`, and unsets the legacy `status` field atomically per ticket |

## Usage Example

This class is auto-discovered and executed by Mongock at application startup. No manual invocation is needed.

```java
// Mongock scans the package and runs this unit once (or always, while runAlways=true).
// The execution method receives its dependencies via Spring injection:

@Execution
public void execution(
    MongoTemplate mongoTemplate,       // MongoDB operations
    Environment environment,           // reads the feature flag
    TenantIdProvider tenantIdProvider  // scopes migration to the current tenant
) {
    // Skips entirely when feature flag is off:
    // openframe.features.tickets.lifecycle.enabled=true (required to run)

    seedStatuses(mongoTemplate);       // ensure TicketStatusDefinition docs exist
    migrateTickets(mongoTemplate, tenantId); // patch legacy tickets
}
```

> **Lifecycle note:** The `runAlways=true` flag and the feature-flag guard are temporary. Once `openframe.features.tickets.lifecycle.enabled` is permanently enabled, both should be removed to convert this into a standard one-shot migration.
