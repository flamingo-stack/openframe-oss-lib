A Mongock database migration change unit that backfills LexoRank ordering values for existing tickets that lack an `order` field, grouped by ticket status.

## Key Components

- **`@ChangeUnit(id = "backfill-ticket-orders", order = "002")`** — Registers this as the second migration in the Mongock changelog sequence
- **`execution()`** — Entry point that iterates over all `TicketStatus` enum values and triggers per-status backfill
- **`backfillColumn()`** — Queries tickets for a given tenant and status where `order` is null, sorted by `createdAt` descending, then assigns sequential LexoRank values starting from the middle rank
- **`rollback()`** — No-op rollback; existing tickets without `order` are unaffected if the migration is reversed
- **`assignOrder()`** — Updates a single ticket document's `order` field using a targeted `updateFirst` MongoDB operation

## Usage Example

This class is auto-discovered and executed by Mongock at application startup. No manual invocation is required. The migration only targets tickets belonging to the current tenant where `order` is `null`.

```java
// Mongock picks this up automatically via @ChangeUnit.
// Internally, the execution flow per status looks like:

LexoRank rank = LexoRank.middle();
for (Ticket ticket : tickets) {           // sorted by createdAt DESC
    assignOrder(mongoTemplate, ticket.getId(), rank.format());
    rank = rank.genNext();                // e.g. "0|hzzzzz:" → "0|i00007:"
}
```

> **Note:** LexoRank values are generated forward from the middle bucket, so newly created tickets can always be ranked above or below existing ones without full re-indexing. Tickets already having an `order` value are excluded from this migration via the `.and(ORDER_FIELD).is(null)` query filter.
