A Mongock database migration change unit that backfills LexoRank ordering values for existing tickets that lack an `order` field, processing each ticket status group independently.

## Key Components

| Component | Description |
|-----------|-------------|
| `@ChangeUnit(id = "backfill-ticket-orders", order = "002")` | Mongock annotation registering this as the second migration in the sequence |
| `execution()` | Entry point iterating over all `TicketStatus` enum values and triggering backfill per status |
| `backfillColumn()` | Queries tickets with a null `order` field for a given status, sorted by `createdAt` descending, then assigns sequential LexoRank values |
| `assignOrder()` | Performs the individual MongoDB `updateFirst` to persist the generated rank string on a ticket document |
| `rollback()` | No-op rollback — order field additions are considered non-destructive |

## Usage Example

This class is auto-discovered and executed by Mongock at application startup. No manual invocation is needed.

```java
// Mongock scans the migration package and runs registered @ChangeUnit classes
// Execution order is determined by the `order` attribute

@ChangeUnit(id = "backfill-ticket-orders", order = "002", author = "openframe")
public class BackfillTicketOrdersChangeUnit {

    @Execution
    public void execution(MongoTemplate mongoTemplate) {
        // Iterates TicketStatus.values() → calls backfillColumn() for each
        // LexoRank.middle() seeds the ranking, then genNext() increments it
        for (TicketStatus status : TicketStatus.values()) {
            backfillColumn(mongoTemplate, status);
        }
    }
}
```

> **Note:** Only tickets where `order` is `null` are affected. Tickets already carrying an order value are skipped, making this migration safe to reason about in partial-migration scenarios.
