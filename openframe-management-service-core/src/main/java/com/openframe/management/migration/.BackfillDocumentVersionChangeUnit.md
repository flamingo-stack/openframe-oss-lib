A Mongock migration change unit that backfills a `documentVersion` field (set to `0L`) on existing documents that lack it, across three MongoDB collections for a given tenant.

## Key Components

| Element | Description |
|---|---|
| `@ChangeUnit(id = "backfill-document-version", order = "001")` | Mongock annotation registering this as migration step #1 |
| `execution()` | Main migration entry point; resolves the current tenant and runs the backfill across all three collections |
| `rollback()` | No-op rollback — adding a version field is considered safe and non-destructive |
| `backfill()` | Core helper that queries documents missing `documentVersion` for the given tenant and sets it to `0L` via `updateMulti` |

**Collections targeted:**

- `integrated_tool_agents`
- `openframe_client_configuration`
- `release_versions`

## Usage Example

```java
// Mongock discovers and executes this automatically on application startup.
// The change unit is idempotent — only documents without documentVersion are updated.

// Equivalent runtime query per collection:
Query query = new Query(
    Criteria.where("tenantId").is(tenantId)
            .and("documentVersion").exists(false)
);
Update update = new Update().set("documentVersion", 0L);
mongoTemplate.updateMulti(query, update, "integrated_tool_agents");
```

> **Note:** The rollback is intentionally empty. Since this migration only *adds* a previously absent field, reverting it would risk data loss and is not required for safe recovery.
