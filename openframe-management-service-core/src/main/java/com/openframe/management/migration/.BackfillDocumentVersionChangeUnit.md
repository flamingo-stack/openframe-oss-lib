Mongock migration change unit that backfills a `documentVersion` field (set to `0L`) on existing documents that lack it, targeting three MongoDB collections for a specific tenant.

## Key Components

| Element | Description |
|---|---|
| `@ChangeUnit(id = "backfill-document-version", order = "001")` | Registers this as the first Mongock migration to execute |
| `execution()` | Entry point — resolves the current tenant ID and triggers backfill on all three collections |
| `rollback()` | No-op rollback; adding a `documentVersion` field is considered safe and non-destructive |
| `backfill()` | Queries for tenant-scoped documents missing `documentVersion` and sets it to `0L` via `updateMulti` |

**Collections targeted:**
- `integrated_tool_agents`
- `openframe_client_configuration`
- `release_versions`

## Usage Example

```java
// Executed automatically by Mongock on application startup.
// No manual invocation needed. Mongock discovers @ChangeUnit beans and runs them in order.

// Internally, backfill() builds and executes a query equivalent to:
db.integrated_tool_agents.updateMany(
  { tenantId: "<tenantId>", documentVersion: { $exists: false } },
  { $set: { documentVersion: NumberLong(0) } }
)
```

> **Note:** The rollback method is intentionally empty. Since this migration only *adds* a field without removing existing data, reverting it would require a separate cleanup strategy outside the scope of this change unit.
