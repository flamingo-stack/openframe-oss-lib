Mongock migration change unit that backfills a `documentVersion` field (set to `0L`) on all existing documents that lack it, across three MongoDB collections.

## Key Components

| Element | Description |
|---|---|
| `@ChangeUnit(id = "backfill-document-version", order = "001")` | Mongock annotation registering this as the first migration to run |
| `execution(MongoTemplate)` | Iterates the three target collections and sets `documentVersion = 0` on any document missing the field |
| `rollback()` | No-op rollback — backfilling a version field is considered safe to leave in place |
| `backfill(MongoTemplate, String)` | Shared helper issuing an `updateMulti` query targeting documents where `documentVersion` does not exist |

**Targeted collections:**
- `integrated_tool_agents`
- `openframe_client_configuration`
- `release_versions`

## Usage Example

This class is auto-discovered and executed by Mongock at application startup. No manual invocation is needed. The underlying query it runs is equivalent to:

```java
Query query = new Query(Criteria.where("documentVersion").exists(false));
Update update = new Update().set("documentVersion", 0L);
mongoTemplate.updateMulti(query, update, "integrated_tool_agents");
```

Which translates to the following MongoDB operation:

```javascript
db.integrated_tool_agents.updateMany(
  { documentVersion: { $exists: false } },
  { $set: { documentVersion: NumberLong(0) } }
)
```

Each run logs the number of modified documents per collection. Subsequent runs are idempotent — documents already carrying a `documentVersion` field are unaffected.
