Default fallback implementation of `AgentRegistrationSecretManagementProcessor` that provides no-op behavior for agent registration secret lifecycle events.

## Key Components

| Element | Description |
|---|---|
| `DefaultAgentRegistrationSecretManagementProcessor` | Default `@Component` activated only when no other `AgentRegistrationSecretManagementProcessor` bean is present |
| `postProcessInitialSecretCreated` | No-op override that logs a debug message when an initial agent registration secret is created |

**Conditional Loading:** Uses `@ConditionalOnMissingBean` to ensure this default is replaced automatically when a custom `AgentRegistrationSecretManagementProcessor` implementation is registered in the application context.

## Usage Example

```java
// This bean is auto-registered by Spring Boot only when no custom implementation exists.
// To override, simply define your own bean:

@Component
public class CustomSecretProcessor implements AgentRegistrationSecretManagementProcessor {

    @Override
    public void postProcessInitialSecretCreated(AgentRegistrationSecret secret, String decryptedKey) {
        // Custom logic: e.g., store decryptedKey in a vault, send a notification, etc.
        notificationService.alertAdmin("New agent secret created: " + secret.getId());
        vaultService.store(secret.getId(), decryptedKey);
    }
}
```

Once `CustomSecretProcessor` is present, `DefaultAgentRegistrationSecretManagementProcessor` is skipped entirely by Spring's conditional bean resolution, ensuring no duplicate processing occurs.
