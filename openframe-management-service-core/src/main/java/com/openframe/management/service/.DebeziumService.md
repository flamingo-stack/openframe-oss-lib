Service for managing Debezium CDC (Change Data Capture) connectors through REST API operations. Provides functionality to create, update, monitor, and restart Debezium connectors for database change streaming.

## Key Components

- **createOrUpdateDebeziumConnector()** - Creates new connectors or updates existing ones based on configuration
- **listConnectors()** - Retrieves all registered connectors
- **getConnectorStatus()** - Gets detailed status information including task states
- **restartTask()** - Restarts specific failed connector tasks
- **checkAndRestartFailedTasks()** - Monitors all connectors and automatically restarts failed tasks
- **URL helper methods** - Internal methods for building Debezium REST API endpoints

## Usage Example

```java
@Autowired
private DebeziumService debeziumService;

// Create/update connectors from configuration
Object[] connectors = loadConnectorConfigs();
debeziumService.createOrUpdateDebeziumConnector(connectors);

// Monitor connector health
List<String> allConnectors = debeziumService.listConnectors();
for (String connector : allConnectors) {
    ConnectorStatus status = debeziumService.getConnectorStatus(connector);
    log.info("Connector {} status: {}", connector, status.getConnector().getState());
}

// Restart failed tasks automatically
debeziumService.checkAndRestartFailedTasks();

// Restart specific task manually
debeziumService.restartTask("my-connector", 0);
```

The service integrates with OpenFrame's CDC infrastructure to ensure reliable database change streaming across the platform.
