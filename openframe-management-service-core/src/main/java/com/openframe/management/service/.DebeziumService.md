This service manages Debezium connector lifecycle operations, providing functionality to create, update, monitor, and maintain database change data capture connectors through the Debezium REST API.

## Key Components

- **`createOrUpdateDebeziumConnector(Object[])`** - Creates new connectors or updates existing ones by checking current state
- **`listConnectors()`** - Retrieves all registered connector names
- **`getConnectorStatus(String)`** - Gets detailed status and task information for a specific connector
- **`restartTask(String, int)`** - Restarts a specific failed task within a connector
- **`checkAndRestartFailedTasks()`** - Monitors all connectors and automatically restarts failed tasks
- **`checkConnectorAndRestartFailedTasks(String)`** - Checks and restarts failed tasks for a specific connector

## Usage Example

```java
@Autowired
private DebeziumService debeziumService;

// Create or update connectors from configuration
Object[] connectors = getConnectorConfigurations();
debeziumService.createOrUpdateDebeziumConnector(connectors);

// Monitor connector health
List<String> activeConnectors = debeziumService.listConnectors();
for (String connector : activeConnectors) {
    ConnectorStatus status = debeziumService.getConnectorStatus(connector);
    log.info("Connector {} state: {}", connector, status.getConnector().getState());
}

// Automated failure recovery
debeziumService.checkAndRestartFailedTasks();

// Manual task restart
debeziumService.restartTask("my-connector", 0);
```

The service uses Spring's RestTemplate for HTTP communication and includes comprehensive error handling with automatic retry logic for failed connector tasks.
