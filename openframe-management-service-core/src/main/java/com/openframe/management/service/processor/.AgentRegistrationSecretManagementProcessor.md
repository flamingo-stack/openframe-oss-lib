Defines a processor interface for hooking into the agent registration secret lifecycle within the OpenFrame management service, specifically to enable post-creation synchronization to external systems.

## Key Components

| Element | Description |
|---|---|
| `AgentRegistrationSecretManagementProcessor` | Interface providing extension points for secret lifecycle events |
| `postProcessInitialSecretCreated` | Default no-op hook triggered after the initial agent registration secret is created |

## Usage Example

```java
@Component
public class VaultSecretSyncProcessor implements AgentRegistrationSecretManagementProcessor {

    private final VaultClient vaultClient;

    @Override
    public void postProcessInitialSecretCreated(
            AgentRegistrationSecret secret,
            String decryptedKey) {
        // Sync plaintext key to HashiCorp Vault after initial creation
        vaultClient.writeSecret(
            "agents/" + secret.getAgentId(),
            Map.of("registration_key", decryptedKey)
        );
    }
}
```

## Notes

- The `decryptedKey` parameter carries the **plaintext** secret key — implementations should handle this value securely and avoid logging it.
- The default no-op implementation means the interface can be introduced into the lifecycle without requiring all consumers to implement the hook.
- Designed for extension via Spring `@Component` beans or similar DI-managed implementations that need to react to new agent registration secrets (e.g., syncing to a secrets manager, notifying an external provisioning system).
