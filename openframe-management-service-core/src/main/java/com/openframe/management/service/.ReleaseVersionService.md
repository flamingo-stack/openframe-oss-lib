A Spring service that manages release version updates and propagates version changes to client configurations and tool agents throughout the OpenFrame system.

## Key Components

- **`process(String releaseVersion)`** - Main entry point that creates or updates the release version record
- **`updateExistingReleaseVersion()`** - Updates an existing version record if the version has changed
- **`createNewReleaseVersion()`** - Creates the initial release version record
- **`updateClientConfiguration()`** - Updates the default client configuration with the new version
- **`updateReleaseAgents()`** - Updates all release-enabled tool agents with the new version
- **Repository/Service Dependencies** - Integrates with version repository, client configuration service, and tool agent services
- **Publishers** - Publishes update events for client configurations and tool agents

## Usage Example

```java
@Autowired
private ReleaseVersionService releaseVersionService;

// Process a new release version
releaseVersionService.process("2.1.0");

// This will:
// 1. Create/update the release version record
// 2. Update default client configuration to v2.1.0
// 3. Update all release tool agents to v2.1.0
// 4. Publish update events for downstream systems
```

The service ensures version consistency across the OpenFrame ecosystem by automatically updating related configurations and publishing change events when a new release version is processed.
