Manages the lifecycle of release versions within the OpenFrame platform, handling creation, updates, and propagation of version changes across client configurations and integrated tool agents.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `process(String releaseVersion)` | Public Method | Entry point for version processing; creates or updates the persisted release version record with optimistic locking retry support |
| `updateExistingReleaseVersion(...)` | Private Method | Updates an existing version record if outdated, then triggers propagation (idempotent — propagates even if already up to date) |
| `createNewReleaseVersion(...)` | Private Method | Persists a new `ReleaseVersion` document using `ReleaseVersion.DEFAULT_ID` and triggers propagation |
| `propagate(String version)` | Private Method | Coordinates downstream updates: refreshes `OpenFrameClientConfiguration` and all release-flagged tool agents |
| `updateReleaseAgents(String version)` | Private Method | Fetches all `IntegratedToolAgent` records with `releaseVersion=true` and updates each to the new version |

## Usage Example

```java
// Typically invoked during a deployment or version sync event
@Autowired
private ReleaseVersionService releaseVersionService;

// Process (create or update) the platform release version
// Automatically propagates to client configs and release agents
releaseVersionService.process("2.4.1");
```

## Notes

- Annotated with `@RetryOnOptimisticLockingFailure` to handle concurrent version update conflicts gracefully.
- Propagation is always triggered after processing — even when the version is unchanged — ensuring idempotent behaviour across restarts or replays.
- Agents are filtered via `integratedToolAgentService.findByReleaseVersionTrue()`, meaning only agents explicitly marked as release agents receive the version update.
