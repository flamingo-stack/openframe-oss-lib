Service that manages OpenFrame release version updates and propagates them to client configurations and integrated tool agents.

## Key Components

- **process(String releaseVersion)** - Main entry point that either creates a new release version record or updates an existing one
- **updateExistingReleaseVersion()** - Updates existing version record and triggers downstream updates only if version has changed
- **createNewReleaseVersion()** - Creates initial version record and triggers configuration updates
- **updateClientConfiguration()** - Updates the default client configuration with new version and publishes update
- **updateReleaseAgents()** - Updates all release-flagged tool agents with new version and publishes updates

## Usage Example

```java
@Autowired
private ReleaseVersionService releaseVersionService;

// Process a new release version
releaseVersionService.process("1.2.3");

// The service will:
// 1. Check if version already exists in database
// 2. Update or create version record as needed
// 3. Update default client configuration
// 4. Update all release tool agents
// 5. Publish updates to subscribers
```

The service ensures version consistency across the OpenFrame system by maintaining a single source of truth for release versions and automatically propagating changes to dependent configurations and agents through event publishing.
