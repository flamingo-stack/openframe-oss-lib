Manages the lifecycle of OpenFrame platform release versions, handling creation, updates, and propagation of version changes across client configurations and integrated tool agents.

## Key Components

| Method | Description |
|--------|-------------|
| `process(String releaseVersion)` | Entry point decorated with `@RetryOnOptimisticLockingFailure`; upserts the release version record |
| `updateExistingReleaseVersion(...)` | Updates an existing `ReleaseVersion` document if the version has changed, then propagates |
| `createNewReleaseVersion(...)` | Persists a new `ReleaseVersion` document and triggers propagation |
| `isUpToDate(...)` | Equality check between the stored and incoming version strings |
| `propagate(...)` | Fans out the new version to `OpenFrameClientConfigurationService` and all release-flagged agents |
| `updateReleaseAgents(...)` | Fetches all `IntegratedToolAgent` records with `releaseVersion=true` and updates each to the new version |

### Dependencies

- **`ReleaseVersionRepository`** — MongoDB/data repository for `ReleaseVersion` documents
- **`OpenFrameClientConfigurationService`** — Pushes the new version to client-facing configurations
- **`IntegratedToolAgentService`** — Queries and updates tool agents marked as release agents

## Usage Example

```java
// Injected via Spring context
@Autowired
private ReleaseVersionService releaseVersionService;

// Called during a deployment pipeline or scheduled release job
releaseVersionService.process("2.5.1");

// Behavior:
// 1. Upserts the ReleaseVersion record in the data store
// 2. Updates OpenFrame client configurations to version 2.5.1
// 3. Updates all IntegratedToolAgents where releaseVersion=true
```

> **Idempotency:** If the incoming version matches the stored version, the upsert is skipped but propagation still runs — ensuring downstream systems remain consistent on repeated calls.
