REST controller that serves logging configuration files from the classpath with dynamic URL replacement for external references.

## Key Components

- **`LoggingConfigController`** - Main REST controller handling logging configuration requests
- **`getLoggingConfig(@PathVariable String filename, HttpServletRequest request)`** - Endpoint that retrieves and processes logging config files

## Core Functionality

The controller provides a single GET endpoint at `/logging/{filename}` that:

1. **File Retrieval** - Loads logging configuration files from the `logging/` directory in the classpath
2. **URL Rewriting** - Replaces relative resource references with absolute server URLs
3. **Content Processing** - Converts `resource="logging/"` patterns to `url="http://server/logging/"` format
4. **XML Response** - Returns processed content as `application/xml`

## Usage Example

```java
// The controller automatically handles requests like:
// GET /logging/logback-spring.xml
// GET /logging/log4j2.xml

// Original file content:
// <include resource="logging/appenders.xml"/>

// Processed response content:
// <include url="http://localhost:8080/logging/appenders.xml"/>

// Client usage:
@Autowired
private RestTemplate restTemplate;

String config = restTemplate.getForObject(
    "http://server:8080/logging/logback-spring.xml", 
    String.class
);
```

The controller is particularly useful for distributed logging configurations where config files need to reference other config files via absolute URLs rather than classpath resources.
