REST controller that serves logging configuration files from the classpath and dynamically updates resource URLs to point to the current server.

## Key Components

- **LoggingConfigController** - Main controller class handling logging configuration requests
- **getLoggingConfig()** - GET endpoint that retrieves and modifies logging config files
- **Dynamic URL replacement** - Converts local file references to absolute server URLs
- **Resource validation** - Checks if requested config files exist before serving

## Usage Example

```java
// Controller automatically handles requests like:
// GET /logging/logback.xml
// GET /logging/log4j2.xml

// The controller will:
// 1. Load the file from classpath:/logging/{filename}
// 2. Replace resource="logging/" with url="http://yourserver:port/logging/"
// 3. Return the modified XML content

// Example transformation:
// Original: <include resource="logging/appender-config.xml"/>
// Modified: <include url="http://localhost:8080/logging/appender-config.xml"/>

// Error handling - returns 404 if file not found
// GET /logging/nonexistent.xml -> 404 Not Found
```

The controller is particularly useful for distributed logging configurations where included files need to be accessible via HTTP URLs rather than local classpath references.
