A Spring Boot component that listens to NATS JetStream messages for tool connection events and processes them using a durable consumer with retry logic.

## Key Components

- **subscribeToToolConnections()** - Sets up JetStream subscription on application startup
- **buildConsumerConfig()** - Creates or updates durable consumer configuration with delivery group
- **handleMessage()** - Processes incoming tool connection messages with acknowledgment
- **cleanup()** - Gracefully shuts down subscription and dispatcher on application termination

## Usage Example

```java
// Component is automatically initialized by Spring Boot
// Listens to subject pattern: "machine.*.tool-connection"
// Messages are processed through ToolConnectionService

// Example message handling flow:
// 1. Message received on subject "machine.123.tool-connection"
// 2. Machine ID "123" extracted from subject
// 3. Message deserialized to ToolConnectionMessage
// 4. Tool connection processed via toolConnectionService.addToolConnection()
// 5. Message acknowledged on success, left unacked for retry on failure

// Consumer configuration includes:
// - Durable consumer name: "tool-connection-processor-v2" 
// - Delivery group: "tool-connection"
// - Max delivery attempts: 50
// - Acknowledgment wait time: 30 seconds
```

The component automatically handles consumer creation/updates, message retry logic, and graceful shutdown cleanup.
