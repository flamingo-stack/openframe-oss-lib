Listens for NATS messaging events related to client machine connections and disconnections, updating machine status accordingly via `MachineStatusService`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `machineConnectedConsumer` | `Bean<Consumer<String>>` | Processes machine connection events and marks the machine as online |
| `machineDisconnectionConsumer` | `Bean<Consumer<String>>` | Processes machine disconnection events and marks the machine as offline |
| `objectMapper` | `ObjectMapper` | Deserializes raw NATS message strings into `ClientConnectionEvent` |
| `machineStatusService` | `MachineStatusService` | Handles the persistence logic for machine online/offline state |

## Usage Example

These beans are auto-wired as Spring Cloud Stream function bindings. Bind them to NATS subjects in your application configuration:

```yaml
spring:
  cloud:
    function:
      definition: machineConnectedConsumer;machineDisconnectionConsumer
    stream:
      bindings:
        machineConnectedConsumer-in-0:
          destination: client.connected
        machineDisconnectionConsumer-in-0:
          destination: client.disconnected
```

Internally, each consumer deserializes the incoming message and delegates to `MachineStatusService`:

```java
// Triggered when a NATS "client connected" message arrives
ClientConnectionEvent event = objectMapper.readValue(message, ClientConnectionEvent.class);
String machineId = event.getClient().getName();
Instant eventTimestamp = Instant.parse(event.getTimestamp());
machineStatusService.updateToOnline(machineId, eventTimestamp);
```

> **Note:** A heartbeat fallback for NATS downtime is pending implementation (`// TODO: nats is down - heartbeat fallback`). Any processing failure throws a `NatsException` to signal upstream error handling.
