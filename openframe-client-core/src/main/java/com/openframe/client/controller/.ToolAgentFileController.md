A Spring Boot REST controller that serves tool agent files from embedded resources based on asset ID and operating system parameters. This is a temporary implementation serving hardcoded content for testing purposes.

## Key Components

- **`@RestController`** - Exposes REST endpoints for file serving
- **`@RequestMapping("/tool-agent/{assetId}")`** - Maps requests to `/tool-agent/{assetId}` path
- **`getToolAgentFile()`** - Main endpoint that returns binary file content based on asset ID and OS

## Usage Example

```java
// GET request examples:
// GET /tool-agent/my-tool?os=mac
// Returns: Binary content from /my-tool resource

// GET /tool-agent/my-tool?os=windows  
// Returns: Binary content from /my-tool.exe resource

// GET /tool-agent/meshcentral-core-module?os=linux
// Returns: Binary content from /meshcentral-core-module resource

// Error cases:
// GET /tool-agent/application-name?os=mac
// Throws: IllegalArgumentException("No asset available")

// GET /tool-agent/my-tool?os=linux
// Throws: IllegalArgumentException("Unknown os: linux")
```

The controller serves different file extensions based on the OS parameter - `.exe` for Windows, no extension for Mac, and has special handling for the `meshcentral-core-module` asset. Files are loaded from the classpath as embedded resources.
