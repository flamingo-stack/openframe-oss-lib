REST controller that serves tool agent files based on asset ID and operating system type. This is a temporary implementation with hardcoded content for testing, intended to be replaced with GitHub artifact integration.

## Key Components

- **ToolAgentFileController** - REST controller class handling tool agent file requests
- **getToolAgentFile()** - GET endpoint that returns binary file content based on asset ID and OS parameters
- Path mapping: `/tool-agent/{assetId}` with OS query parameter
- OS-specific file resolution (Mac, Windows, core module)

## Usage Example

```java
// GET request examples:
// GET /tool-agent/my-tool?os=mac
// Returns: binary content from /my-tool resource

// GET /tool-agent/mesh-agent?os=windows  
// Returns: binary content from /mesh-agent.exe resource

// GET /tool-agent/meshcentral-core-module?os=linux
// Returns: binary content from /meshcentral-core-module resource

// Error cases:
// GET /tool-agent/application-name?os=mac
// Throws: IllegalArgumentException("No asset available")

// GET /tool-agent/some-tool?os=linux
// Throws: IllegalArgumentException("Unknown os: linux")
```

The controller reads files from the classpath resources and returns them as byte arrays. It validates asset IDs (rejecting those containing "application") and supports "mac" and "windows" OS types, with special handling for the "meshcentral-core-module" asset.
