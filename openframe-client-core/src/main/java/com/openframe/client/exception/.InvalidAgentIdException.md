Thrown when a request contains an invalid or unrecognized agent ID, extending `BadRequestException` to signal a client-side 400-level error within the OpenFrame client exception hierarchy.

## Key Components

| Element | Description |
|---|---|
| `InvalidAgentIdException` | Custom exception class for invalid agent ID validation failures |
| `BadRequestException` | Parent class inherited from `com.openframe.core.exception` |
| `InvalidAgentIdException(String message)` | Single constructor accepting a descriptive error message |

## Usage Example

```java
public Agent resolveAgent(String agentId) {
    if (agentId == null || agentId.isBlank()) {
        throw new InvalidAgentIdException("Agent ID must not be null or empty: " + agentId);
    }
    return agentRepository.findById(agentId)
        .orElseThrow(() -> new InvalidAgentIdException("No agent found with ID: " + agentId));
}
```

> This exception propagates as an HTTP `400 Bad Request` response, making it suitable for REST API validation layers where an agent ID fails format checks or cannot be resolved.
