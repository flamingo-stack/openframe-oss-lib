Signals that a connection already exists when attempting to create a duplicate, extending the base `ConflictException` to represent HTTP 409 Conflict scenarios specific to connection conflicts.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `DuplicateConnectionException` | `class` | Custom exception extending `ConflictException` for duplicate connection detection |
| `DuplicateConnectionException(String message)` | Constructor | Accepts a descriptive error message passed up to the parent `ConflictException` |

## Usage Example

```java
// Throw when a connection with the same identifier already exists
public void createConnection(Connection connection) {
    if (connectionRepository.exists(connection.getId())) {
        throw new DuplicateConnectionException(
            "Connection with ID '" + connection.getId() + "' already exists."
        );
    }
    connectionRepository.save(connection);
}
```

> **Note:** This exception is part of the `com.openframe.client.exception` package and should be used specifically when a client attempts to register or create a connection that conflicts with an existing one. Catch it at the API layer to return a meaningful `409 Conflict` HTTP response.
