A custom runtime exception thrown when an invalid or unsupported tool type is encountered in the OpenFrame client application.

## Key Components

- **InvalidToolTypeException**: A runtime exception class that extends `RuntimeException`
- **Constructor**: Accepts a string message parameter to provide specific error details

## Usage Example

```java
// Throwing the exception when validating tool types
public void validateToolType(String toolType) {
    if (!SUPPORTED_TOOLS.contains(toolType)) {
        throw new InvalidToolTypeException(
            "Tool type '" + toolType + "' is not supported"
        );
    }
}

// Catching and handling the exception
try {
    toolManager.loadTool("unsupported-tool");
} catch (InvalidToolTypeException e) {
    logger.error("Invalid tool type: " + e.getMessage());
    // Handle gracefully or show user-friendly error
}
```

This exception is typically used in tool validation scenarios where the application needs to reject unsupported or malformed tool type identifiers, providing clear feedback about what went wrong.
