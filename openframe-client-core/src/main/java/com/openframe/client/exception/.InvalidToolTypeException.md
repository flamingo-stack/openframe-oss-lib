Custom exception thrown when a tool type provided in a client request is invalid or unrecognized, extending `BadRequestException` to signal a 400-level error.

## Key Components

- **`InvalidToolTypeException`** — Extends `BadRequestException`; wraps invalid tool type errors with a descriptive message passed up to the parent exception handler.

## Usage Example

```java
if (!SUPPORTED_TOOL_TYPES.contains(toolType)) {
    throw new InvalidToolTypeException(
        "Unsupported tool type: '" + toolType + "'. Must be one of: " + SUPPORTED_TOOL_TYPES
    );
}
```
