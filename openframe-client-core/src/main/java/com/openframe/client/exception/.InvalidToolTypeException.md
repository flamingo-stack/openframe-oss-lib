A custom runtime exception thrown when an invalid or unsupported tool type is encountered in the OpenFrame client application.

## Key Components

- **InvalidToolTypeException**: A runtime exception class that extends `RuntimeException`
- **Constructor**: Single parameterized constructor accepting an error message

## Usage Example

```java
// Throwing the exception when encountering an invalid tool type
public void validateToolType(String toolType) {
    if (!isValidToolType(toolType)) {
        throw new InvalidToolTypeException(
            "Unsupported tool type: " + toolType + 
            ". Supported types are: [HAMMER, SCREWDRIVER, WRENCH]"
        );
    }
}

// Catching and handling the exception
try {
    toolService.createTool("INVALID_TYPE");
} catch (InvalidToolTypeException e) {
    logger.error("Tool creation failed: " + e.getMessage());
    return ResponseEntity.badRequest()
        .body("Invalid tool type specified");
}
```

This exception provides a clean way to handle tool type validation errors throughout the application, allowing callers to catch and respond appropriately to invalid tool type scenarios.
