A Spring `ApplicationEvent` published when a device transitions from `PENDING` to `ONLINE` upon receiving its first heartbeat. Consumed by the SaaS layer to trigger downstream `DEVICE_REGISTERED` processing.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `machine` | `Machine` | The device document representing the newly connected machine |
| `DeviceFirstConnectedEvent(Object source, Machine machine)` | Constructor | Wraps the source and associated `Machine` for event propagation |

## Usage Example

**Publishing the event:**

```java
// Triggered when first heartbeat is received and device transitions PENDING → ONLINE
Machine machine = machineRepository.findById(deviceId).orElseThrow();
eventPublisher.publishEvent(new DeviceFirstConnectedEvent(this, machine));
```

**Listening for the event:**

```java
@Component
public class DeviceRegistrationListener {

    @EventListener
    public void onDeviceFirstConnected(DeviceFirstConnectedEvent event) {
        Machine machine = event.getMachine();
        // Trigger DEVICE_REGISTERED flow in SaaS layer
        saasService.notifyDeviceRegistered(machine);
    }
}
```

> This event is part of the OpenFrame device lifecycle pipeline. It fires **once per device** — on the first successful heartbeat — making it the appropriate hook for registration side-effects such as onboarding notifications, audit logging, or provisioning triggers.
