Handles tag creation and assignment during agent registration, ensuring each tag is validated, persisted (or reused if it already exists), and associated with the registering device.

## Key Components

| Component | Description |
|-----------|-------------|
| `assignTags()` | Public entry point; validates and assigns a list of tags to a machine by ID |
| `findOrCreateTag()` | Looks up a tag by `key + DEVICE` entity type; creates it if missing, or merges new values into existing options |
| `TagRepository` | Injected directly (bypasses `TagService`) for tag lookup and persistence |
| `TagAssignmentRepository` | Persists `TagAssignment` records; AOP aspect triggers Kafka → Pinot pipeline on save |
| `TagValidation` | Static utility called to validate key and value constraints before persistence |

## Usage Example

```java
// Called internally during agent registration flow
@Autowired
RegistrationTagAssignmentService registrationTagAssignmentService;

List<AgentRegistrationTagInput> tags = List.of(
    AgentRegistrationTagInput.builder()
        .key("environment")
        .values(List.of("production", "us-east"))
        .build()
);

registrationTagAssignmentService.assignTags("machine-uuid-1234", tags);
// → Validates key/values
// → Finds or creates Tag{key="environment", entityType=DEVICE}
// → Saves TagAssignment{entityId="machine-uuid-1234", tagId=..., entityType=DEVICE}
// → AOP fires → Kafka event → Pinot index updated
```

## Notes

- Uses repositories directly instead of `TagService` because `openframe-client-core` does not depend on `openframe-api-lib`.
- Tag value merging is **additive and deduplicated** — existing tag options are never removed.
- Failures per tag are caught individually and logged, so a single bad tag does not abort the entire registration.
- All assignments use `TagEntityType.DEVICE` as the entity scope.
