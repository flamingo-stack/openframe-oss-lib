Defines the contract for transforming raw agent tool identifiers into structured, tool-type-specific agent IDs during the agent registration process.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `getToolType()` | Method | Returns the `ToolType` this transformer handles, used for routing/dispatching to the correct implementation |
| `transform(...)` | Method | Converts a raw `agentToolId` and `machineId` into a normalized agent ID string; `lastAttempt` signals whether this is the final retry |

## Usage Example

```java
public class RmmToolAgentIdTransformer implements ToolAgentIdTransformer {

    @Override
    public ToolType getToolType() {
        return ToolType.RMM;
    }

    @Override
    public String transform(String machineId, String agentToolId, boolean lastAttempt) {
        // Normalize and combine identifiers for RMM tool type
        return machineId + "-" + agentToolId.toLowerCase();
    }
}
```

Implementations are typically discovered at runtime by matching `getToolType()` against the registering agent's tool type, allowing new tool integrations to plug in without modifying core registration logic.
