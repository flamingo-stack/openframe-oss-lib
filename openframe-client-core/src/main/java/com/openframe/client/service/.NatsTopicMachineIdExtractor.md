A Spring component that extracts machine IDs from NATS messaging topic subjects with validation and error handling.

## Key Components

- **extract(String subject)** - Main method that parses NATS subjects following the `machine.{machineId}.{suffix}` pattern
- **Validation Logic** - Comprehensive input validation with descriptive error messages
- **Logging** - Debug-level logging for successful extractions

## Usage Example

```java
@Autowired
private NatsTopicMachineIdExtractor extractor;

public void processMessage(String natsSubject) {
    try {
        // Extract machine ID from NATS subject
        String machineId = extractor.extract("machine.device-001.status");
        // Returns: "device-001"
        
        // Handle the machine-specific message
        processForMachine(machineId);
        
    } catch (IllegalArgumentException e) {
        log.error("Invalid NATS subject format: {}", e.getMessage());
    }
}

// Valid subject formats:
// "machine.server-123.heartbeat" → "server-123"
// "machine.device-001.telemetry" → "device-001"
// "machine.node-A.status.update" → "node-A"

// Invalid formats throw IllegalArgumentException:
// null, "", "invalid.format", "machine..suffix"
```

The extractor expects NATS subjects in the format `machine.{machineId}.{suffix}` where the machine ID is the second segment. It validates input thoroughly and provides clear error messages for debugging NATS routing issues.
