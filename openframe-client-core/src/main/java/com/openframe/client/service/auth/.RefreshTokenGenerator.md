A Spring component that generates JWT-based refresh tokens for OAuth2 authentication, with configurable expiration and refresh count tracking to prevent unlimited token refreshing.

## Key Components

- **`generate(String clientId)`** - Creates a new refresh token for the specified client with refresh count initialized to 0
- **`generateNext(String clientId, long refreshCount)`** - Generates a new refresh token with an incremented refresh count
- **`getMaxRefreshCount()`** - Returns the maximum allowed refresh count from configuration
- **Configuration properties**:
  - `security.oauth2.token.refresh.expiration-seconds` - Token expiration time
  - `security.oauth2.token.refresh.max-refresh-count` - Maximum refresh limit

## Usage Example

```java
@Autowired
private RefreshTokenGenerator refreshTokenGenerator;

// Generate initial refresh token
String refreshToken = refreshTokenGenerator.generate("client123");

// Generate subsequent refresh token (increment count)
String newRefreshToken = refreshTokenGenerator.generateNext("client123", 1L);

// Check refresh limit
if (currentRefreshCount >= refreshTokenGenerator.getMaxRefreshCount()) {
    throw new TokenRefreshLimitExceededException();
}
```

The component integrates with `JwtService` to create signed tokens and includes claims for client identification, refresh tracking, and expiration management.
