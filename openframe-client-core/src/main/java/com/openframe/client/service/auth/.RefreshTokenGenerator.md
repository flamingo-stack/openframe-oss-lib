A Spring component responsible for generating and refreshing JWT refresh tokens with configurable expiration and refresh count tracking for OAuth2 authentication flows.

## Key Components

- **RefreshTokenGenerator** - Main component class that generates refresh tokens
- **generate(String clientId)** - Creates initial refresh token with zero refresh count
- **generateNext(String clientId, long refreshCount)** - Creates new refresh token with incremented refresh count
- **buildClaims(String clientId)** - Builds JWT claims for initial token
- **buildNextTokenClaims(String clientId, long refreshCount)** - Builds JWT claims for subsequent tokens
- **maxRefreshCount** - Configurable limit for token refresh operations

## Usage Example

```java
@Autowired
private RefreshTokenGenerator refreshTokenGenerator;

// Generate initial refresh token
String initialToken = refreshTokenGenerator.generate("client123");

// Generate next refresh token (increment count)
String nextToken = refreshTokenGenerator.generateNext("client123", 1L);

// Check refresh limit
if (currentRefreshCount < refreshTokenGenerator.getMaxRefreshCount()) {
    String newToken = refreshTokenGenerator.generateNext(clientId, currentRefreshCount);
}
```

## Configuration Properties

The component requires these application properties:
- `security.oauth2.token.refresh.expiration-seconds` - Token expiration time
- `security.oauth2.token.refresh.max-refresh-count` - Maximum allowed refreshes
