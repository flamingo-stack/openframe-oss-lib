Transforms raw MeshCentral agent tool IDs into fully-qualified node path strings, supporting both legacy and tenant-scoped formats.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `LEGACY_NODE_PREFIX` | Constant | Legacy node path prefix `node//` used as the default format |
| `tenantId` | Field | Injected from `openframe.cluster-id` — used in tenant-scoped node ID construction |
| `tenantScopedNodeId` | Field | Injected from `openframe.client.meshcentral.tenant-scoped-node-id` — toggles ID format |
| `getToolType()` | Method | Returns `ToolType.MESHCENTRAL` to identify this transformer |
| `transform()` | Method | Core transformation logic — builds node path from `agentToolId` |

## Transformation Logic

```text
tenantScopedNodeId = true  →  "node/{tenantId}/{agentToolId}"
tenantScopedNodeId = false →  "node//{agentToolId}"  (legacy default)
agentToolId = blank        →  returns as-is with a warning log
```

## Usage Example

```java
// Injected automatically via Spring; called by the agent registration pipeline
@Autowired
MeshCentralAgentIdTransformer transformer;

String nodeId = transformer.transform(
    "machine-abc-123",       // machineId (for logging)
    "agent456",              // agentToolId
    false                    // unused flag (ignored)
);

// With tenantScopedNodeId=false (default):
// → "node//agent456"

// With tenantScopedNodeId=true and cluster-id="acme":
// → "node/acme/agent456"
```

> **Note:** The legacy `node//` format is marked for removal after the next MeshCentral release. Set `openframe.client.meshcentral.tenant-scoped-node-id=true` when upgrading to tenant-aware deployments.
