A Spring component that transforms MeshCentral agent IDs by adding a standardized node prefix for consistent identification across the OpenFrame system.

## Key Components

- **MeshCentralAgentIdTransformer** - Main transformer class implementing `ToolAgentIdTransformer`
- **getToolType()** - Returns `ToolType.MESHCENTRAL` to identify this transformer's purpose
- **transform()** - Adds "node//" prefix to agent IDs with validation and logging
- **NODE_PREFIX** - Constant defining the standardized prefix for MeshCentral nodes

## Usage Example

```java
@Autowired
private MeshCentralAgentIdTransformer transformer;

// Transform a MeshCentral agent ID
String originalId = "abc123def456";
String transformedId = transformer.transform(originalId, false);
// Result: "node//abc123def456"

// Check supported tool type
ToolType type = transformer.getToolType();
// Result: ToolType.MESHCENTRAL

// Handle blank input gracefully
String blankId = transformer.transform("", false);
// Result: "" (with warning logged)
```

The transformer ensures MeshCentral agent IDs follow a consistent format by prepending the "node//" prefix, enabling proper identification and routing within the OpenFrame agent registration system. It includes validation for blank inputs and comprehensive logging for debugging purposes.
