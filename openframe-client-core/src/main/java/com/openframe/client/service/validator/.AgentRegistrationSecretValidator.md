Validates agent registration secrets during the agent onboarding process by checking that a provided key is non-blank and matches the currently active encrypted secret stored in the database.

## Key Components

| Element | Description |
|---|---|
| `validate(String initialKey)` | Main validation method; checks for blank key, retrieves the active secret, decrypts it, and compares against the provided key |
| `AgentRegistrationSecretRepository` | Injected repository used to fetch the currently active `AgentRegistrationSecret` document |
| `EncryptionService` | Injected service used to decrypt the stored secret key before comparison |
| `AgentRegistrationSecretValidationException` | Thrown for recoverable validation failures (empty or invalid key), with a corresponding `ErrorCode` |
| `AgentRegistrationSecretValidationErrorException` | Thrown as a system-level error when no active secret exists in the database |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class AgentRegistrationService {

    private final AgentRegistrationSecretValidator secretValidator;

    public void registerAgent(AgentRegistrationRequest request) {
        // Validates the initial key before proceeding with registration.
        // Throws AgentRegistrationSecretValidationException if key is blank or invalid.
        // Throws AgentRegistrationSecretValidationErrorException if no active secret exists.
        secretValidator.validate(request.getInitialKey());

        // Continue with agent registration logic...
    }
}
```

> **Note:** Only one active `AgentRegistrationSecret` is expected to exist at any time. If none is found, the validator treats it as a system configuration error rather than a user input error, reflected by the use of distinct exception types.
