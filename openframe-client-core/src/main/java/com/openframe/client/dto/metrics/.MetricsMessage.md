A data transfer object that encapsulates system performance metrics collected from client machines, including CPU usage, memory utilization, and collection timestamp.

## Key Components

- **machineId** - Unique identifier for the source machine
- **cpu** - CPU utilization percentage (0.0 to 100.0)
- **memory** - Memory usage percentage (0.0 to 100.0) 
- **timestamp** - Exact time when metrics were collected
- **@Data** - Lombok annotation providing automatic getters, setters, toString, equals, and hashCode methods

## Usage Example

```java
// Creating a new metrics message
MetricsMessage metrics = new MetricsMessage();
metrics.setMachineId("server-01");
metrics.setCpu(75.5);
metrics.setMemory(62.3);
metrics.setTimestamp(Instant.now());

// Accessing metrics data
String machine = metrics.getMachineId(); // "server-01"
double cpuLoad = metrics.getCpu();       // 75.5
double memUsage = metrics.getMemory();   // 62.3
Instant when = metrics.getTimestamp();   // current timestamp

// Automatic toString for logging
System.out.println(metrics); // MetricsMessage(machineId=server-01, cpu=75.5, memory=62.3, timestamp=...)
```

This DTO is typically used for transmitting performance data between OpenFrame clients and servers, enabling system monitoring and resource tracking across distributed environments.
