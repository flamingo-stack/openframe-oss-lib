Represents a "resource not found" exception that maps to HTTP 404 responses, extending `BaseException` with a fixed `NOT_FOUND` HTTP status.

## Key Components

| Component | Description |
|-----------|-------------|
| `NotFoundException(String message)` | Convenience constructor using the default `ErrorCode.NOT_FOUND` code |
| `NotFoundException(ErrorCode, String)` | Constructor allowing a custom `ErrorCode` with a 404 status |
| `NotFoundException(ErrorCode, String, Throwable)` | Constructor supporting cause chaining for wrapping lower-level exceptions |

## Usage Example

```java
// Simple not-found scenario
public Ticket findById(Long id) {
    return ticketRepository.findById(id)
        .orElseThrow(() -> new NotFoundException("Ticket not found with id: " + id));
}

// With a specific error code
public Device findDevice(String deviceId) {
    return deviceRepository.findById(deviceId)
        .orElseThrow(() -> new NotFoundException(ErrorCode.DEVICE_NOT_FOUND, 
            "Device not found: " + deviceId));
}

// With cause chaining
public Client findClient(String clientId) {
    try {
        return externalService.fetchClient(clientId);
    } catch (ExternalServiceException e) {
        throw new NotFoundException(ErrorCode.CLIENT_NOT_FOUND, 
            "Client not found: " + clientId, e);
    }
}
```

> All constructors hardcode `HttpStatus.NOT_FOUND (404)`, ensuring consistent HTTP response mapping across the OpenFrame platform without requiring callers to specify the status code manually.
