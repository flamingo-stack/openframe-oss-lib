Custom exception for HTTP 401 Unauthorized errors, extending `BaseException` with a fixed `HttpStatus.UNAUTHORIZED` status. Used across the OpenFrame platform to signal authentication or authorization failures in a consistent, structured way.

## Key Components

- **`UnauthorizedException(String message)`** — Convenience constructor using the default `ErrorCode.UNAUTHORIZED` error code.
- **`UnauthorizedException(ErrorCode errorCode, String message)`** — Allows a custom `ErrorCode` while keeping the HTTP 401 status fixed.
- **`UnauthorizedException(ErrorCode errorCode, String message, Throwable cause)`** — Full constructor supporting cause chaining for wrapping lower-level exceptions.

## Usage Example

```java
// Simple unauthorized access
throw new UnauthorizedException("Access token is missing or invalid");

// With a specific error code
throw new UnauthorizedException(ErrorCode.TOKEN_EXPIRED, "JWT token has expired");

// Wrapping a lower-level cause
try {
    validateToken(token);
} catch (JwtException e) {
    throw new UnauthorizedException(ErrorCode.TOKEN_INVALID, "Token validation failed", e);
}
```

## Notes

- Always maps to `HttpStatus.UNAUTHORIZED` (HTTP 401) — use `ForbiddenException` for HTTP 403 scenarios where the user is authenticated but lacks permissions.
- Inherits structured error handling from `BaseException`, ensuring consistent error response shapes across the OpenFrame API.
