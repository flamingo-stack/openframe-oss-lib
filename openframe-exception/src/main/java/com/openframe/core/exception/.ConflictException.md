A specialized exception class for handling HTTP 409 Conflict errors, extending `BaseException` to represent resource conflict scenarios within the OpenFrame core exception hierarchy.

## Key Components

| Component | Description |
|-----------|-------------|
| `ConflictException(String message)` | Basic constructor using the default `ErrorCode.CONFLICT` code with HTTP 409 status |
| `ConflictException(ErrorCode, String)` | Constructor accepting a custom `ErrorCode` alongside the conflict message |
| `ConflictException(ErrorCode, String, Throwable)` | Full constructor supporting a root cause for exception chaining |

## Usage Example

```java
// Basic conflict - e.g., duplicate resource
throw new ConflictException("A user with this email already exists.");

// With a specific error code
throw new ConflictException(ErrorCode.DUPLICATE_ENTRY, "Ticket ID already assigned.");

// With root cause for exception chaining
try {
    userRepository.save(user);
} catch (DataIntegrityViolationException ex) {
    throw new ConflictException(
        ErrorCode.DUPLICATE_ENTRY,
        "User already exists in the system.",
        ex
    );
}
```

> All constructors automatically bind `HttpStatus.CONFLICT` (409), ensuring consistent HTTP response mapping across the OpenFrame platform without manual status code assignment.
