A `@RestControllerAdvice` base class that provides centralized, reusable exception handling across all Spring REST controllers in the OpenFrame platform, mapping exceptions to structured `ErrorResponse` payloads with appropriate HTTP status codes.

## Key Components

| Handler Method | Exception Handled | HTTP Status |
|---|---|---|
| `handleBaseException` | `BaseException` (+ `ValidationException` field errors) | Dynamic (from exception) |
| `handleMethodArgumentNotValid` | `MethodArgumentNotValidException` | `400 Bad Request` |
| `handleConstraintViolation` | `ConstraintViolationException` | `400 Bad Request` |
| `handleMissingRequestHeader` | `MissingRequestHeaderException` | `400 Bad Request` |
| `handleMissingServletRequestParameter` | `MissingServletRequestParameterException` | `400 Bad Request` |
| `handleIllegalArgument` | `IllegalArgumentException` | `400 Bad Request` |
| `handleIllegalState` | `IllegalStateException` | `409 Conflict` |
| `handleMethodNotSupported` | `HttpRequestMethodNotSupportedException` | `405 Method Not Allowed` |
| `handleMediaTypeNotSupported` | `HttpMediaTypeNotSupportedException` | `415 Unsupported Media Type` |
| `handleResponseStatusException` | `ResponseStatusException` | Dynamic (from exception) |
| `handleException` | `Exception` (catch-all) | `500 Internal Server Error` |

**Private helpers:**
- `buildValidationResponse` — assembles an `ErrorResponse` with field-level error details
- `toMessage` — flattens field errors into a human-readable comma-separated string

## Usage Example

Extend this class in a service-specific controller advice to inherit all base handlers while adding domain-specific ones:

```java
@RestControllerAdvice
public class OrderExceptionHandler extends BaseGlobalExceptionHandler {

    @ExceptionHandler(OrderNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ErrorResponse handleOrderNotFound(OrderNotFoundException ex) {
        return ErrorResponse.of(ErrorCode.NOT_FOUND, ex.getMessage());
    }
}
```

Validation errors on request bodies automatically include per-field details:

```java
// Response body for a failed @Valid check:
// {
//   "code": "VALIDATION_ERROR",
//   "message": "name: must not be blank, email: must be a valid email",
//   "fieldErrors": [
//     { "field": "name", "message": "must not be blank" },
//     { "field": "email", "message": "must be a valid email" }
//   ]
// }
```
