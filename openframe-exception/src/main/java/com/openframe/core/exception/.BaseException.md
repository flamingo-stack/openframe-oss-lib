Abstract base class for all custom exceptions in the OpenFrame platform, combining an `ErrorCode` enum with an HTTP status code to support structured, API-aware error handling.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `errorCode` | `ErrorCode` | Domain-specific error identifier |
| `httpStatus` | `HttpStatus` | HTTP status code associated with the error |
| `BaseException(ErrorCode, HttpStatus, String)` | Constructor | Creates exception with message |
| `BaseException(ErrorCode, HttpStatus, String, Throwable)` | Constructor | Creates exception with message and cause chain |
| `getErrorCode()` | Method | Returns the `ErrorCode` for programmatic handling |
| `getHttpStatus()` | Method | Returns the `HttpStatus` for response mapping |

## Usage Example

```java
// Define a concrete exception extending BaseException
public class ResourceNotFoundException extends BaseException {

    public ResourceNotFoundException(String resourceId) {
        super(
            ErrorCode.RESOURCE_NOT_FOUND,
            HttpStatus.NOT_FOUND,
            "Resource not found: " + resourceId
        );
    }
}

// Throw with a cause for exception chaining
public class ServiceUnavailableException extends BaseException {

    public ServiceUnavailableException(String message, Throwable cause) {
        super(
            ErrorCode.SERVICE_UNAVAILABLE,
            HttpStatus.SERVICE_UNAVAILABLE,
            message,
            cause
        );
    }
}

// Usage in a global exception handler
@ExceptionHandler(BaseException.class)
public ResponseEntity<ErrorResponse> handleBaseException(BaseException ex) {
    return ResponseEntity
        .status(ex.getHttpStatus())
        .body(new ErrorResponse(ex.getErrorCode(), ex.getMessage()));
}
```

> All custom exceptions in the OpenFrame stack should extend `BaseException` to ensure consistent error codes and HTTP status propagation through the global exception handler.
