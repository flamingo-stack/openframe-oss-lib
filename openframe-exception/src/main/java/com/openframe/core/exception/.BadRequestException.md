Custom exception representing HTTP 400 Bad Request errors, extending `BaseException` with pre-configured `HttpStatus.BAD_REQUEST` status.

## Key Components

| Constructor | Parameters | Description |
|-------------|-----------|-------------|
| `BadRequestException(String)` | `message` | Uses default `ErrorCode.BAD_REQUEST` |
| `BadRequestException(ErrorCode, String)` | `errorCode`, `message` | Custom error code with message |
| `BadRequestException(ErrorCode, String, Throwable)` | `errorCode`, `message`, `cause` | Includes root cause for chaining |

## Usage Example

```java
// Simple bad request with default error code
throw new BadRequestException("Invalid input: email format is incorrect");

// With a specific error code
throw new BadRequestException(ErrorCode.INVALID_FIELD, "Field 'phone' must be numeric");

// With root cause for exception chaining
try {
    parseRequest(payload);
} catch (ParseException e) {
    throw new BadRequestException(ErrorCode.MALFORMED_REQUEST, "Unable to parse request body", e);
}
```

> All constructors delegate to `BaseException` with `HttpStatus.BAD_REQUEST` hardcoded, ensuring consistent HTTP status mapping across the OpenFrame platform.
