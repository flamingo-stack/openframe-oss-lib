Represents a validation failure in the OpenFrame core exception hierarchy, mapping to HTTP 400 Bad Request with optional per-field error details.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `ValidationException(String message)` | Constructor | Creates a validation exception with a general message and no field errors |
| `ValidationException(String message, List<FieldError> fieldErrors)` | Constructor | Creates a validation exception with a message and a list of specific field-level errors |
| `getFieldErrors()` | Method | Returns the immutable list of `FieldError` records (never null) |
| `FieldError` | Record | Holds a `field` name and associated `message` for granular validation feedback |

**Error code:** `ErrorCode.VALIDATION_ERROR`
**HTTP status:** `HttpStatus.BAD_REQUEST` (400)

## Usage Example

```java
// Simple validation failure
throw new ValidationException("Request body is invalid");

// Field-level validation failures
List<ValidationException.FieldError> errors = List.of(
    new ValidationException.FieldError("email", "Must be a valid email address"),
    new ValidationException.FieldError("name", "Cannot be blank")
);

throw new ValidationException("One or more fields failed validation", errors);

// Inspecting field errors in a handler
catch (ValidationException ex) {
    ex.getFieldErrors().forEach(err ->
        log.warn("Field '{}': {}", err.field(), err.message())
    );
}
```
