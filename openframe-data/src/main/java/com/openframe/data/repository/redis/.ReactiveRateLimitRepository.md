A reactive Redis-based repository that provides atomic rate limiting operations with window-based counting and TTL management for Spring WebFlux applications.

## Key Components

- **checkAndIncrement()** - Atomically checks rate limit and increments counter, returning rate limit status
- **getStatus()** - Retrieves current rate limit status without incrementing the counter
- **RateLimitResult** - Builder pattern data class containing rate limit information (allowed, count, remaining, window times)

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class RateLimitService {
    private final ReactiveRateLimitRepository rateLimitRepo;
    
    public Mono<Boolean> checkUserRateLimit(String userId) {
        return rateLimitRepo.checkAndIncrement(
            userId,                    // key identifier
            "HOURLY",                 // window type
            "2024-01-15T14",          // window timestamp
            100L,                     // request limit
            Duration.ofHours(1)       // TTL
        ).map(RateLimitResult::isAllowed);
    }
    
    public Mono<RateLimitResult> getUserRateLimitStatus(String userId) {
        return rateLimitRepo.getStatus(
            userId, "HOURLY", "2024-01-15T14", 100L
        );
    }
}
```

The repository uses Redis hash operations to store count, firstRequest, and lastRequest timestamps with configurable TTL for automatic cleanup.
