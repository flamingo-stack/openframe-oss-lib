A fluent builder for constructing Pinot SQL queries with validation, security features, and specialized methods for analytics workloads.

## Key Components

**Core Builder Methods:**
- `select()`, `distinct()`, `selectCount()` - Column selection with aggregation support
- `where*()` methods - Type-safe filtering with SQL injection protection
- `orderBy*()`, `groupBy*()`, `limit()` - Query structuring and optimization
- `build()` - Generates final SQL string with validation

**Specialized Query Types:**
- `whereTextSearch()` - Full-text search using Pinot's TEXT_MATCH function
- `whereDateRange()` - Time-based filtering with epoch conversion
- `whereFilterOptionsExcluding()` - Complex multi-filter conditions
- `orderByTimestampDesc()` - Event ordering with secondary sort stability

**Security & Validation:**
- SQL injection prevention through value escaping
- Field name and limit validation
- Query state verification before building

## Usage Example

```java
// Basic analytics query with time filtering
PinotQueryBuilder builder = new PinotQueryBuilder("events_table")
    .select("userId", "eventType", "deviceId")
    .whereEquals("appVersion", "2.1.0")
    .whereDateRange("eventTimestamp", 
        LocalDate.now().minusDays(7), 
        LocalDate.now())
    .whereTextSearch("error", "message", "stackTrace")
    .orderByTimestampDesc()
    .limit(1000);

String query = builder.build();

// Aggregation query for analytics dashboard
String countQuery = new PinotQueryBuilder("user_events")
    .selectCount()
    .groupBy("eventType", "platform")
    .whereIn("status", "active", "pending")
    .orderByCountDesc()
    .build();

// Multi-filter query excluding specific field
String filterQuery = new PinotQueryBuilder("sessions")
    .select("sessionId", "duration")
    .whereFilterOptionsExcluding("browser", 
        Arrays.asList("chrome", "firefox"),
        "excludedField",
        Map.of("country", Arrays.asList("US", "CA")))
    .build();
```
