Repository interface for querying device data from Apache Pinot, providing filtering and aggregation capabilities for device analytics.

## Key Components

- **Filter Option Methods**: Five methods (`getStatusFilterOptions`, `getDeviceTypeFilterOptions`, `getOsTypeFilterOptions`, `getOrganizationFilterOptions`, `getTagFilterOptions`) that return aggregated counts for each filter category
- **Device Count Method**: `getFilteredDeviceCount()` returns the total count of devices matching the specified criteria
- **Common Parameters**: All methods accept the same filter parameters (statuses, deviceTypes, osTypes, organizationIds, tagNames)

## Usage Example

```java
@Autowired
private PinotDeviceRepository deviceRepository;

// Get available status options with counts
List<String> deviceTypes = Arrays.asList("mobile", "tablet");
List<String> statuses = Arrays.asList("active");
Map<String, Integer> statusOptions = deviceRepository.getStatusFilterOptions(
    statuses, deviceTypes, null, null, null
);
// Returns: {"active": 150, "inactive": 25, "maintenance": 10}

// Get total filtered device count
int totalDevices = deviceRepository.getFilteredDeviceCount(
    Arrays.asList("active"), 
    Arrays.asList("mobile"), 
    Arrays.asList("iOS", "Android"),
    null, 
    null
);
```
