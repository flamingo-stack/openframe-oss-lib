A reactive repository for tracking API key usage statistics in Redis, providing atomic increment operations for successful and failed request counters.

## Key Components

- **ReactiveApiKeyStatsRepository**: Main repository class for API key statistics operations
- **incrementSuccessful()**: Atomically increments success counters and updates last used timestamp
- **incrementFailed()**: Atomically increments failure counters and updates last used timestamp
- **ReactiveStringRedisTemplate**: Spring Data Redis reactive template for non-blocking operations

## Usage Example

```java
@Service
public class ApiKeyService {
    
    @Autowired
    private ReactiveApiKeyStatsRepository statsRepository;
    
    public Mono<String> processApiRequest(String apiKey, String requestData) {
        return authenticateKey(apiKey)
            .flatMap(keyId -> processRequest(requestData)
                .doOnSuccess(result -> 
                    statsRepository.incrementSuccessful(keyId, Duration.ofDays(30))
                        .subscribe())
                .doOnError(error -> 
                    statsRepository.incrementFailed(keyId, Duration.ofDays(30))
                        .subscribe())
            );
    }
    
    private Mono<String> authenticateKey(String apiKey) {
        // Authentication logic
        return Mono.just("key-123");
    }
    
    private Mono<String> processRequest(String data) {
        // Request processing logic
        return Mono.just("processed: " + data);
    }
}
```

The repository uses Redis hash operations to store counters (total, success, failed) and timestamps, with configurable TTL for automatic cleanup. All operations are atomic and non-blocking, suitable for high-throughput reactive applications.
