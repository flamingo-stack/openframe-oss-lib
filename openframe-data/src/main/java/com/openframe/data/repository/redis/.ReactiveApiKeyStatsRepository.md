A reactive Redis repository that manages API key usage statistics, providing atomic operations to track request counters and timestamps for successful and failed API calls.

## Key Components

- **incrementSuccessful()** - Atomically increments total and success counters while updating last used timestamp
- **incrementFailed()** - Atomically increments total and failed counters while updating last used timestamp
- **STATS_KEY_PREFIX** - Redis key prefix for organizing statistics data ("stats:")

## Usage Example

```java
@Service
public class ApiKeyStatsService {
    
    private final ReactiveApiKeyStatsRepository statsRepository;
    
    public Mono<Void> recordSuccessfulRequest(String apiKeyId) {
        Duration ttl = Duration.ofDays(30);
        return statsRepository.incrementSuccessful(apiKeyId, ttl);
    }
    
    public Mono<Void> recordFailedRequest(String apiKeyId) {
        Duration ttl = Duration.ofDays(30);
        return statsRepository.incrementFailed(apiKeyId, ttl);
    }
}
```

Both methods use Redis hash operations to store statistics as fields (total, success, failed, lastUsed) under keys like "stats:your-api-key-id". The repository automatically sets TTL on the keys to prevent indefinite storage and only operates in reactive web applications.
