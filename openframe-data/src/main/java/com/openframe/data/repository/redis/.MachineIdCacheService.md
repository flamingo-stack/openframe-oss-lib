This service provides cached access to machine and organization information using Spring's caching abstraction. It optimizes database lookups by storing lightweight DTOs and is conditionally enabled only when MongoDB repositories are available.

## Key Components

- **getMachine(String agentId)** - Retrieves cached machine information by agent ID, including machine ID, hostname, and organization ID
- **getOrganization(String organizationId)** - Fetches cached organization details by organization ID, returning ID and name
- **CachedMachineInfo/CachedOrganizationInfo** - Lightweight DTOs used to minimize cache storage and avoid serialization issues
- **@Cacheable annotations** - Enable automatic caching with "machineCache" and "organizationCache" cache regions

## Usage Example

```java
@Autowired
private MachineIdCacheService cacheService;

// Get machine information by agent ID
CachedMachineInfo machine = cacheService.getMachine("agent-123");
if (machine != null) {
    String machineId = machine.getMachineId();
    String hostname = machine.getHostname();
    String orgId = machine.getOrganizationId();
}

// Get organization information
CachedOrganizationInfo org = cacheService.getOrganization("org-456");
if (org != null) {
    String name = org.getName();
}
```

The service automatically handles caching, database fallbacks, and error handling. Subsequent calls with the same parameters will return cached results, improving performance for frequently accessed data.
