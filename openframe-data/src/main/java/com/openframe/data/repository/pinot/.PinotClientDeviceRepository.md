Repository implementation providing device data filtering and aggregation through Apache Pinot OLAP database queries. Supports dynamic filter options and count queries with multi-dimensional device attributes.

## Key Components

- **PinotClientDeviceRepository** - Spring repository implementing `PinotDeviceRepository` interface
- **Filter Option Methods** - `getStatusFilterOptions()`, `getDeviceTypeFilterOptions()`, `getOsTypeFilterOptions()`, `getOrganizationFilterOptions()`, `getTagFilterOptions()`
- **Count Method** - `getFilteredDeviceCount()` for paginated results
- **Query Builders** - `buildWhereClause()` and `buildWhereClauseExcluding()` for dynamic SQL construction
- **Query Executors** - `queryPinotForFilterOptions()` and `queryPinotForCount()` with error handling

## Usage Example

```java
@Service
public class DeviceService {
    
    @Autowired
    private PinotDeviceRepository deviceRepository;
    
    public Map<String, Object> getDeviceFilterData(List<String> currentStatuses) {
        // Get available status options excluding current filters
        Map<String, Integer> statusOptions = deviceRepository.getStatusFilterOptions(
            currentStatuses, null, null, null, null
        );
        
        // Get total count with current filters applied
        int totalDevices = deviceRepository.getFilteredDeviceCount(
            currentStatuses, null, null, null, null
        );
        
        return Map.of(
            "statusOptions", statusOptions,
            "totalCount", totalDevices
        );
    }
}
```

The repository automatically excludes DELETED devices and provides cross-dimensional filtering where each filter method excludes its own field to show available options based on other active filters.
