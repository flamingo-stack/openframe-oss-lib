Apache Pinot repository implementation for querying device data and providing filter options for the OpenFrame MSP platform. This class handles real-time analytics queries against device datasets stored in Pinot, enabling dynamic filtering capabilities for device management interfaces.

## Key Components

- **Filter Option Methods**: `getStatusFilterOptions()`, `getDeviceTypeFilterOptions()`, `getOsTypeFilterOptions()`, `getOrganizationFilterOptions()`, `getTagFilterOptions()` - Generate available filter values with counts
- **Device Count Method**: `getFilteredDeviceCount()` - Returns total count of devices matching filter criteria
- **Query Execution Methods**: `queryPinotForFilterOptions()`, `queryPinotForCount()` - Execute Pinot queries and handle results
- **WHERE Clause Builders**: `buildWhereClause()`, `buildWhereClauseExcluding()` - Construct dynamic SQL conditions based on filter parameters

## Usage Example

```java
@Autowired
private PinotClientDeviceRepository deviceRepository;

// Get status filter options excluding current status filters
Map<String, Integer> statusOptions = deviceRepository.getStatusFilterOptions(
    null, // exclude status filters
    Arrays.asList("laptop", "desktop"), // device types
    Arrays.asList("windows", "macos"), // OS types
    Arrays.asList("org-123"), // organization IDs
    Arrays.asList("production") // tag names
);

// Get filtered device count
int deviceCount = deviceRepository.getFilteredDeviceCount(
    Arrays.asList("active", "inactive"),
    Arrays.asList("laptop"),
    null,
    Arrays.asList("org-123"),
    null
);
```

The repository dynamically builds SQL WHERE clauses and executes queries against the configurable Pinot devices table, providing real-time analytics for MSP device management dashboards.
