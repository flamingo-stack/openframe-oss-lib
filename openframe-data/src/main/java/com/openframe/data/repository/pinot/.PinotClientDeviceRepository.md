A Spring repository implementation that provides filtered device data queries against Apache Pinot for analytics and reporting. The class supports dynamic filtering across multiple device attributes and returns aggregated counts for building filter options in user interfaces.

## Key Components

- **Filter Option Methods**: `getStatusFilterOptions()`, `getDeviceTypeFilterOptions()`, `getOsTypeFilterOptions()`, `getOrganizationFilterOptions()`, `getTagFilterOptions()` - Return available filter values with their counts
- **Device Count Method**: `getFilteredDeviceCount()` - Returns total count of devices matching filter criteria
- **Query Builders**: `buildWhereClause()`, `buildWhereClauseExcluding()` - Construct SQL WHERE clauses dynamically
- **Query Executors**: `queryPinotForFilterOptions()`, `queryPinotForCount()` - Execute Pinot queries and handle results

## Usage Example

```java
@Autowired
private PinotDeviceRepository deviceRepository;

// Get status filter options excluding current status filters
Map<String, Integer> statusOptions = deviceRepository.getStatusFilterOptions(
    null, // don't filter by status
    Arrays.asList("mobile", "tablet"), 
    Arrays.asList("iOS", "Android"),
    Arrays.asList("org1", "org2"),
    Arrays.asList("production")
);
// Returns: {"active": 1500, "inactive": 300, "maintenance": 50}

// Get total device count with filters applied
int totalDevices = deviceRepository.getFilteredDeviceCount(
    Arrays.asList("active"),
    Arrays.asList("mobile"),
    null, null, null
);
```
