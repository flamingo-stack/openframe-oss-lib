A Spring Data repository implementation that provides log querying capabilities against Apache Pinot, supporting filtered searches, pagination, and dynamic filter options retrieval.

## Key Components

- **PinotClientLogRepository** - Main repository class implementing `PinotLogRepository` interface
- **findLogs()** - Retrieves logs with filtering by date range, tool types, event types, severities, organization, and device
- **searchLogs()** - Performs text search across logs with same filtering capabilities
- **getEventTypeOptions()** - Returns available event type filter values
- **getSeverityOptions()** - Returns available severity filter values  
- **getToolTypeOptions()** - Returns available tool type filter values
- **getOrganizationOptions()** - Returns organization filter options with ID and name
- **getAvailableDateRanges()** - Returns available date ranges for logs

## Usage Example

```java
@Autowired
private PinotLogRepository logRepository;

// Find logs with filters
List<LogProjection> logs = logRepository.findLogs(
    LocalDate.of(2024, 1, 1),     // start date
    LocalDate.of(2024, 1, 31),    // end date
    List.of("SYSTEM", "APP"),      // tool types
    List.of("ERROR", "WARNING"),   // event types
    List.of("HIGH"),               // severities
    List.of("org1", "org2"),       // organization IDs
    "device123",                   // device ID
    null,                          // cursor for pagination
    100                            // limit
);

// Get filter options
List<String> eventTypes = logRepository.getEventTypeOptions(
    startDate, endDate, toolTypes, null, severities, organizationIds
);

// Search logs with text
List<LogProjection> searchResults = logRepository.searchLogs(
    startDate, endDate, toolTypes, eventTypes, severities, 
    organizationIds, deviceId, "connection timeout", cursor, 50
);
```
