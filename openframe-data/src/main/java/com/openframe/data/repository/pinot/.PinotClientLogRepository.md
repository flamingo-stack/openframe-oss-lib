Spring Data repository implementation for querying log data from Apache Pinot OLAP database, providing high-performance analytics queries with filtering, searching, and pagination capabilities.

## Key Components

**Core Methods:**
- `findLogs()` - Retrieves filtered log entries with pagination and sorting
- `searchLogs()` - Full-text search across log data with relevance ranking
- Filter option methods (`getEventTypeOptions()`, `getSeverityOptions()`, etc.) - Dynamic filter population
- `getOrganizationOptions()` - Returns available organizations with ID/name pairs
- `isSortableField()` / `getDefaultSortField()` - Sort validation and defaults

**Configuration:**
- `SORTABLE_COLUMNS` - Whitelisted fields for sorting operations
- `PRIMARY_KEY_FIELD` - Primary key for consistent pagination cursors
- `logsTable` - Configurable Pinot table name via properties

**Internal Utilities:**
- `executeLogQuery()` - Maps ResultSet to LogProjection objects
- `executeQuery()` - Generic query execution with custom mappers
- `buildColumnIndexMap()` - Creates column name to index mapping for efficient data access

## Usage Example

```java
@Autowired
private PinotLogRepository logRepository;

// Basic log filtering
List<LogProjection> logs = logRepository.findLogs(
    LocalDate.now().minusDays(7),  // startDate
    LocalDate.now(),               // endDate
    List.of("kubernetes", "docker"), // toolTypes
    List.of("error", "warning"),   // eventTypes
    List.of("HIGH", "CRITICAL"),   // severities
    List.of("org-123"),            // organizationIds
    "device-456",                  // deviceId
    null,                          // cursor (for pagination)
    100,                           // limit
    "eventTimestamp",              // sortField
    "DESC"                         // sortDirection
);

// Full-text search with relevance
List<LogProjection> searchResults = logRepository.searchLogs(
    LocalDate.now().minusDays(1),
    LocalDate.now(),
    null, null, null, null, null,
    "database connection failed",   // searchTerm
    null, 50, "eventTimestamp", "DESC"
);

// Get dynamic filter options
List<String> availableEventTypes = logRepository.getEventTypeOptions(
    startDate, endDate, toolTypes, null, severities, organizationIds
);
```
