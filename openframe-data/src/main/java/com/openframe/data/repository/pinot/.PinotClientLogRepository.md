Repository implementation that provides data access methods for querying log data from Apache Pinot, supporting filtering, search, and metadata retrieval operations for the OpenFrame platform.

## Key Components

- **PinotClientLogRepository** - Main repository class implementing PinotLogRepository interface
- **Log Query Methods** - `findLogs()` and `searchLogs()` with comprehensive filtering capabilities
- **Filter Options** - Methods to retrieve available event types, severities, tool types, date ranges, and organizations
- **Query Execution** - Generic query execution framework with result mapping and error handling
- **PinotQueryBuilder Integration** - Fluent query building for dynamic SQL construction

## Usage Example

```java
@Autowired
private PinotLogRepository logRepository;

// Query logs with filters
List<LogProjection> logs = logRepository.findLogs(
    LocalDate.now().minusDays(7),  // startDate
    LocalDate.now(),               // endDate
    List.of("RMM", "PSA"),        // toolTypes
    List.of("ERROR", "WARNING"),   // eventTypes
    List.of("HIGH", "MEDIUM"),     // severities
    List.of("org-123"),           // organizationIds
    "device-456",                 // deviceId
    null,                         // cursor
    50                            // limit
);

// Search logs with text query
List<LogProjection> searchResults = logRepository.searchLogs(
    LocalDate.now().minusDays(1),
    LocalDate.now(),
    null, null, null, null, null,
    "connection failed",          // searchTerm
    null, 25
);

// Get available filter options
List<String> eventTypes = logRepository.getEventTypeOptions(
    startDate, endDate, toolTypes, null, severities, organizationIds
);

List<OrganizationOption> orgs = logRepository.getOrganizationOptions(
    startDate, endDate, toolTypes, eventTypes, severities
);
```

The repository uses Apache Pinot's native client for high-performance analytics queries on large log datasets, with built-in error handling and configurable table names.
