A custom runtime exception specifically designed for handling errors that occur during Apache Pinot database query operations within the data repository layer.

## Key Components

- **PinotQueryException** - Custom runtime exception class extending `RuntimeException`
- **Constructor overloads** - Two constructors supporting message-only and message-with-cause initialization patterns

## Usage Example

```java
// Throwing exception with message only
if (query == null || query.isEmpty()) {
    throw new PinotQueryException("Query cannot be null or empty");
}

// Throwing exception with message and underlying cause
try {
    pinotConnection.execute(query);
} catch (SQLException e) {
    throw new PinotQueryException("Failed to execute Pinot query: " + query, e);
}

// Catching and handling the exception
try {
    List<Record> results = pinotRepository.findByFilter(filter);
} catch (PinotQueryException e) {
    logger.error("Pinot query failed: {}", e.getMessage(), e);
    // Handle gracefully or rethrow
}
```

This exception provides a clean way to propagate Pinot-specific errors up the call stack while maintaining the original error context through the cause chain.
