A custom runtime exception class specifically designed for handling Pinot query-related errors in the OpenFrame data repository layer.

## Key Components

- **PinotQueryException** - Custom runtime exception that extends `RuntimeException`
- **Constructor overloads** - Two constructors for different error scenarios:
  - Message-only constructor for simple error messages
  - Message and cause constructor for wrapping underlying exceptions

## Usage Example

```java
// Throwing with a simple message
throw new PinotQueryException("Invalid query syntax for table: users");

// Wrapping an underlying exception
try {
    executeQuery(query);
} catch (SQLException e) {
    throw new PinotQueryException("Failed to execute Pinot query", e);
}

// Catching in service layer
try {
    pinotRepository.findUserMetrics(userId);
} catch (PinotQueryException e) {
    log.error("Pinot query failed: {}", e.getMessage(), e);
    // Handle gracefully or propagate
}
```

This exception provides a clean abstraction layer for Pinot-specific query failures, allowing the application to distinguish between different types of database errors and handle them appropriately in the service layer.
