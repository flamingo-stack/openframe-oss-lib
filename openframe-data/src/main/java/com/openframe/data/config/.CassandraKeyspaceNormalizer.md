Normalizes Cassandra keyspace names by replacing dashes with underscores during Spring application startup. This ensures tenant IDs containing dashes can be used in configuration while maintaining Cassandra keyspace naming requirements.

## Key Components

- **initialize()** - Main entry point that processes the `spring.data.cassandra.keyspace-name` property
- **KEYSPACE_PROPERTY** - Configuration property for Cassandra keyspace name
- **PROPERTY_SOURCE_NAME** - Identifier for the normalized property source

## Usage Example

Add this initializer to your Spring Boot application:

```java
// In application.yml or properties
spring:
  data:
    cassandra:
      keyspace-name: tenant-123-data  # Contains dashes

// Register the initializer
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        new SpringApplicationBuilder(Application.class)
            .initializers(new CassandraKeyspaceNormalizer())
            .run(args);
    }
}
```

The normalizer automatically converts `tenant-123-data` to `tenant_123_data` before Cassandra connection initialization, ensuring valid keyspace names while allowing flexible tenant ID formats in configuration.
