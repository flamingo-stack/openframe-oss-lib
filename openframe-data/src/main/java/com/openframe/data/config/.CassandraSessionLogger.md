A utility class that logs detailed Cassandra session information for debugging and monitoring purposes after dependency injection initialization.

## Key Components

- **Constructor**: Accepts a `CqlSession` instance for inspection
- **logSessionDetails()**: `@PostConstruct` method that logs comprehensive session configuration and connection details
- **Logger Integration**: Uses SLF4J for structured logging output

## Usage Example

```java
// In a Spring configuration class
@Configuration
public class CassandraConfig {
    
    @Bean
    public CqlSession cassandraSession() {
        return CqlSession.builder()
            .addContactPoint(new InetSocketAddress("127.0.0.1", 9042))
            .withLocalDatacenter("datacenter1")
            .build();
    }
    
    @Bean
    public CassandraSessionLogger sessionLogger(CqlSession session) {
        return new CassandraSessionLogger(session);
    }
}

// The logger will automatically output session details like:
// Cassandra Session Details:
// Connected Nodes: [Node(endPoint=127.0.0.1:9042, hostId=...)]
// Current Datacenter: datacenter1
// Current Keyspace: my_keyspace
// Contact Points from Config: [127.0.0.1:9042]
// Load Balancing DC: datacenter1
```

This class is particularly useful for troubleshooting connection issues and verifying Cassandra cluster configuration in different environments.
