Utility class for logging Cassandra database session configuration and connection details during application startup. Provides comprehensive visibility into the active Cassandra connection state for debugging and monitoring purposes.

## Key Components

- **Constructor**: `CassandraSessionLogger(CqlSession session)` - Accepts the Cassandra session to monitor
- **`@PostConstruct logSessionDetails()`** - Logs detailed session information after bean initialization
- **Session Metadata Logging** - Extracts and displays connected nodes, datacenter, keyspace, and configuration details
- **Error Handling** - Gracefully handles null sessions and logging exceptions

## Usage Example

```java
@Configuration
public class CassandraConfig {
    
    @Bean
    public CqlSession cassandraSession() {
        return CqlSession.builder()
            .addContactPoint(new InetSocketAddress("127.0.0.1", 9042))
            .withLocalDatacenter("datacenter1")
            .withKeyspace("myapp")
            .build();
    }
    
    @Bean
    public CassandraSessionLogger sessionLogger(CqlSession session) {
        return new CassandraSessionLogger(session);
    }
}

// Example log output:
// INFO: Cassandra Session Details:
// INFO: Connected Nodes: {127.0.0.1=DefaultNode...}
// INFO: Current Datacenter: datacenter1
// INFO: Current Keyspace: myapp
// INFO: Contact Points from Config: [127.0.0.1:9042]
// INFO: Load Balancing DC: datacenter1
```

This class is typically used as a Spring bean to automatically log connection details when the application starts, helping developers verify their Cassandra configuration is correct.
