Spring Data Cassandra configuration class that handles database connection setup, keyspace creation, and session management for OpenFrame's Cassandra integration.

## Key Components

- **CassandraConfig** - Main configuration class extending `AbstractCassandraConfiguration`
- **Connection Properties** - Configurable datacenter, keyspace, contact points, and port settings
- **ensureKeyspaceExists()** - Automatic keyspace creation with configurable replication factor
- **cassandraSession()** - Custom session factory with driver configuration
- **CassandraSessionLogger** - Bean for session monitoring and logging

## Usage Example

Configure Cassandra connection in `application.yml`:

```yaml
spring:
  data:
    cassandra:
      enabled: true
      local-datacenter: datacenter1
      keyspace-name: openframe
      contact-points: localhost
      port: 9042
      replication-factor: 1
```

The configuration automatically:
- Creates the keyspace if it doesn't exist
- Sets up connection pooling and load balancing
- Enables repository scanning in `com.openframe.data.repository.cassandra`
- Configures server-side timestamp generation

```java
// Repositories are automatically discovered and configured
@Repository
public interface UserRepository extends CassandraRepository<User, UUID> {
    // Repository methods available immediately
}
```

The `@ConditionalOnProperty` annotation ensures Cassandra is only configured when explicitly enabled, allowing for flexible deployment scenarios.
