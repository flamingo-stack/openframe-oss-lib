Spring Data Cassandra configuration class that establishes database connectivity and manages session lifecycle. Conditionally enables Cassandra repositories when `spring.data.cassandra.enabled` is true.

## Key Components

- **AbstractCassandraConfiguration** - Extended base class providing Cassandra configuration structure
- **CqlSessionFactoryBean** - Custom session factory with programmatic driver configuration
- **CassandraSessionLogger** - Custom bean for session monitoring and logging
- **SchemaAction.CREATE_IF_NOT_EXISTS** - Automatic keyspace creation strategy
- **@ConditionalOnProperty** - Conditional activation based on application properties

## Usage Example

```java
# application.yml
spring:
  data:
    cassandra:
      enabled: true
      contact-points: "127.0.0.1"
      port: 9042
      keyspace-name: "openframe_keyspace"
      local-datacenter: "datacenter1"

# Repository usage
@Repository
public interface UserRepository extends CassandraRepository<User, UUID> {
    // Repository methods automatically available
}

# Service injection
@Service
public class DataService {
    @Autowired
    private CqlSession cqlSession; // Auto-configured session
    
    @Autowired
    private UserRepository userRepository; // Auto-enabled repository
}
```

The configuration automatically sets up connection pooling, load balancing for the specified datacenter, and enables server-side timestamp generation for consistent data ordering across the cluster.
