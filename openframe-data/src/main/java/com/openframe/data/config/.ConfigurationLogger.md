A Spring Boot component that logs database configuration properties when the application starts up, providing visibility into the configured data store connections.

## Key Components

- `@Component` - Spring-managed bean that automatically runs during application startup
- `@Value` annotations - Inject configuration properties with null defaults using SpEL expressions
- `@EventListener` - Listens for `ApplicationReadyEvent` to trigger logging after full application initialization
- `logConfiguration()` - Logs all configured database connection details including MongoDB, Cassandra, Redis, and Pinot

## Usage Example

```java
// Configuration properties in application.yml or application.properties
spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/mydb
    cassandra:
      contact-points: localhost:9042
    redis:
      host: localhost

pinot:
  controller:
    url: http://localhost:9000
  broker:
    url: http://localhost:8000

// The component automatically logs these values on startup:
// INFO - Application Configuration:
// INFO - MongoDB URI: mongodb://localhost:27017/mydb  
// INFO - Cassandra Contact Points: localhost:9042
// INFO - Redis Host: localhost
// INFO - Pinot Controller URL: http://localhost:9000
// INFO - Pinot Broker URL: http://localhost:8000
```
