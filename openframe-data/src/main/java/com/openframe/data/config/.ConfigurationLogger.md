A Spring Boot component that automatically logs database and service configuration values at application startup for troubleshooting and verification purposes.

## Key Components

- **@Component**: Spring-managed bean that auto-registers with the application context
- **@Value annotations**: Inject configuration properties from application.properties/yml with null defaults
- **@EventListener**: Listens for ApplicationReadyEvent to trigger logging after full startup
- **Configuration properties**: MongoDB URI, Cassandra contact points, Redis host, and Pinot service URLs
- **SLF4J Logger**: Structured logging output for configuration values

## Usage Example

This component runs automatically when the Spring Boot application starts. Configure your database connections in `application.yml`:

```java
# application.yml
spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/mydb
    cassandra:
      contact-points: localhost:9042
    redis:
      host: localhost

pinot:
  controller:
    url: http://localhost:9000
  broker:
    url: http://localhost:8000
```

The component will automatically log these values at startup:

```java
// Sample log output:
// INFO  - Application Configuration:
// INFO  - MongoDB URI: mongodb://localhost:27017/mydb
// INFO  - Cassandra Contact Points: localhost:9042
// INFO  - Redis Host: localhost
// INFO  - Pinot Controller URL: http://localhost:9000
// INFO  - Pinot Broker URL: http://localhost:8000
```

This is particularly useful for verifying configuration in different environments and debugging connectivity issues during deployment.
