Spring Cache configuration class that sets up Redis as the caching provider with JSON serialization and 6-hour TTL for cached entries.

## Key Components

- **CacheManager Bean**: Configures Redis-based cache manager with custom serialization
- **RedisCacheConfiguration**: Defines default cache behavior including TTL and null value handling
- **StringRedisSerializer**: Handles cache key serialization as strings
- **GenericJackson2JsonRedisSerializer**: Serializes cached values as JSON for cross-language compatibility

## Usage Example

This configuration enables caching annotations throughout the application:

```java
@Service
public class UserService {
    
    @Cacheable("users")
    public User findById(Long id) {
        // Expensive database operation
        return userRepository.findById(id);
    }
    
    @CacheEvict(value = "users", key = "#user.id")
    public void updateUser(User user) {
        userRepository.save(user);
    }
}
```

The configuration automatically:
- Caches method results for 6 hours
- Prevents caching of null values
- Stores keys as strings and values as JSON in Redis
- Enables use of `@Cacheable`, `@CacheEvict`, and other Spring Cache annotations
