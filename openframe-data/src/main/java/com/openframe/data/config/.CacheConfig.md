Configures Spring Cache integration with Redis as the backing cache store, enabling application-wide caching with JSON serialization and 6-hour TTL.

## Key Components

- **`CacheConfig`** - Main configuration class annotated with `@Configuration` and `@EnableCaching`
- **`cacheManager(RedisConnectionFactory)`** - Bean that creates and configures a Redis-based cache manager
- **TTL Configuration** - Sets default cache entry expiration to 6 hours
- **Serialization Setup** - Uses String serializer for keys and Jackson JSON serializer for values

## Usage Example

```java
// Service class using caching
@Service
public class UserService {
    
    @Cacheable("users")
    public User findById(Long id) {
        // Expensive database operation
        return userRepository.findById(id);
    }
    
    @CacheEvict(value = "users", key = "#user.id")
    public void updateUser(User user) {
        userRepository.save(user);
    }
    
    @CachePut(value = "users", key = "#result.id")
    public User createUser(User user) {
        return userRepository.save(user);
    }
}
```

The configuration automatically handles Redis connection management, key-value serialization, and cache expiration. Cache entries are stored as JSON objects in Redis with string-based keys and will automatically expire after 6 hours.
