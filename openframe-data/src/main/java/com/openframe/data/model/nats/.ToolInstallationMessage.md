A NATS message data model that defines the structure for tool installation requests in the OpenFrame platform, containing tool metadata, download configurations, and installation commands.

## Key Components

- **ToolInstallationMessage** - Main message class containing tool installation details
- **Asset** - Nested class representing downloadable tool assets with source information
- **LocalFilenameConfiguration** - Platform-specific filename configuration
- **AssetSource** - Enum defining supported asset sources (Artifactory, Tool API, GitHub)

Key fields include tool identification (`toolAgentId`, `toolId`, `toolType`), versioning, session type, download configurations, and various command argument lists for installation lifecycle management.

## Usage Example

```java
// Create a tool installation message
ToolInstallationMessage message = new ToolInstallationMessage();
message.setToolAgentId("agent-001");
message.setToolId("docker");
message.setToolType("containerization");
message.setVersion("20.10.17");
message.setSessionType(SessionType.DOCKER);
message.setReinstall(false);

// Set installation commands
message.setInstallationCommandArgs(List.of("apt-get", "install", "-y", "docker-ce"));
message.setRunCommandArgs(List.of("docker", "run"));
message.setUninstallationCommandArgs(List.of("apt-get", "remove", "-y", "docker-ce"));

// Create asset with download configuration
ToolInstallationMessage.Asset asset = new ToolInstallationMessage.Asset();
asset.setId("docker-binary");
asset.setVersion("20.10.17");
asset.setSource(ToolInstallationMessage.AssetSource.GITHUB);
asset.setPath("/usr/bin/docker");
asset.setExecutable(true);

message.setAssets(List.of(asset));
```

This message structure enables the OpenFrame platform to orchestrate tool installations across different environments with flexible asset management and platform-specific configurations.
