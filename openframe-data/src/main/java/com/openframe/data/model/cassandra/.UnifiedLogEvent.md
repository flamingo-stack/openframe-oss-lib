A Cassandra entity representing unified log events from multiple monitoring tools (Fleet, Meshcentral, Tactical-RMM) with NIST-compliant structure and time-series partitioning for scalable storage.

## Key Components

- **UnifiedLogEvent**: Main entity class annotated with `@Table("unified_logs")`
- **UnifiedLogEventKey**: Composite primary key class with multi-level partitioning strategy
- **Partitioning Strategy**: Primary partitions by `ingest_day` and `tool_type` for optimal distribution
- **Clustering Strategy**: Clusters by `event_type`, `event_timestamp` (descending), and `tool_event_id`
- **Event Metadata**: Includes user/device/organization context, severity levels, and flexible detail storage

## Usage Example

```java
// Create a new unified log event
UnifiedLogEvent event = new UnifiedLogEvent();

// Set up the composite key
UnifiedLogEvent.UnifiedLogEventKey key = new UnifiedLogEvent.UnifiedLogEventKey();
key.setIngestDay("2024-01-15");
key.setToolType("meshcentral");
key.setEventType("user_login");
key.setEventTimestamp(Instant.now());
key.setToolEventId("mc_evt_12345");

// Set event properties
event.setKey(key);
event.setUserId("user123");
event.setDeviceId("device456");
event.setHostname("workstation-01");
event.setOrganizationId("org789");
event.setSeverity("info");
event.setMessage("User successfully logged in");
event.setDetails("{\"ip_address\":\"192.168.1.100\",\"user_agent\":\"Chrome/120.0\"}");

// Save using Spring Data Cassandra repository
unifiedLogRepository.save(event);
```

The partitioning design enables efficient queries by date range and tool type while maintaining chronological ordering for recent events.
