An enumeration that defines different message types for integrated tool events, specifying their routing and processing configuration within the OpenFrame data pipeline.

## Key Components

- **MESHCENTRAL_EVENT** - Events from MeshCentral remote management tool
- **TACTICAL_RMM_AUDIT_EVENT** - Audit events from Tactical RMM system
- **TACTICAL_RMM_AGENT_HISTORY_EVENT** - Agent history events from Tactical RMM
- **FLEET_MDM_EVENT** - Events from Fleet Mobile Device Management
- **FLEET_MDM_QUERY_RESULT_EVENT** - Query result events from Fleet MDM

Each enum value contains:
- `integratedToolType` - Source tool identification
- `dataEnrichmentServiceType` - Processing service category
- `destinationList` - Target destinations (Cassandra, Kafka)
- `eventHandlerType` - Handler categorization for processing

## Usage Example

```java
// Get message type configuration
MessageType messageType = MessageType.MESHCENTRAL_EVENT;

// Access routing destinations
List<Destination> destinations = messageType.getDestinationList();
// Returns [CASSANDRA, KAFKA]

// Check source tool type
IntegratedToolType toolType = messageType.getIntegratedToolType();
// Returns MESHCENTRAL

// Determine processing service
DataEnrichmentServiceType serviceType = messageType.getDataEnrichmentServiceType();
// Returns INTEGRATED_TOOLS_EVENTS

// Get handler type
EventHandlerType handlerType = messageType.getEventHandlerType();
// Returns COMMON_TYPE
```

This enum centralizes message routing configuration, ensuring consistent processing and delivery across all integrated security and management tools in the OpenFrame ecosystem.
