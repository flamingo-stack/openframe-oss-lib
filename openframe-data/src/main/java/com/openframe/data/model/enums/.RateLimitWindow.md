Enum that defines time windows for rate limiting operations, providing duration and timestamp formatting for different granularities.

## Key Components

- **MINUTE** - 1-minute window with format `yyyy-MM-dd-HH-mm`
- **HOUR** - 1-hour window with format `yyyy-MM-dd-HH` 
- **DAY** - 1-day window with format `yyyy-MM-dd`
- **duration** - `Duration` object representing the time window length
- **timestampFormat** - String pattern for formatting timestamps within the window
- **getSeconds()** - Utility method returning window duration in seconds

## Usage Example

```java
// Get rate limit window properties
RateLimitWindow window = RateLimitWindow.HOUR;
long seconds = window.getSeconds(); // 3600
String format = window.getTimestampFormat(); // "yyyy-MM-dd-HH"

// Use in rate limiting logic
Duration windowDuration = RateLimitWindow.MINUTE.getDuration();
LocalDateTime now = LocalDateTime.now();
DateTimeFormatter formatter = DateTimeFormatter.ofPattern(
    RateLimitWindow.DAY.getTimestampFormat()
);
String timeKey = now.format(formatter); // "2023-12-25"
```

This enum is typically used in rate limiting systems to bucket requests by time periods and generate consistent timestamp keys for tracking usage across different time granularities.
