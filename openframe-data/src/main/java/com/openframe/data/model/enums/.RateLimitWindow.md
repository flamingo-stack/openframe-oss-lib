An enumeration that defines time windows for rate limiting functionality, providing duration and timestamp formatting for different time intervals.

## Key Components

- **MINUTE** - 1-minute window with `yyyy-MM-dd-HH-mm` format
- **HOUR** - 1-hour window with `yyyy-MM-dd-HH` format  
- **DAY** - 1-day window with `yyyy-MM-dd` format
- **duration** - Java Duration object for each time window
- **timestampFormat** - String pattern for formatting timestamps within each window
- **getSeconds()** - Convenience method returning duration in seconds

## Usage Example

```java
// Configure rate limiting for different time windows
RateLimitWindow window = RateLimitWindow.HOUR;

// Get the duration for rate limit calculations
Duration windowDuration = window.getDuration(); // 1 hour
long windowSeconds = window.getSeconds();       // 3600 seconds

// Format timestamps for bucketing requests
String timestampKey = window.getTimestampFormat(); // "yyyy-MM-dd-HH"

// Use in rate limiting logic
LocalDateTime now = LocalDateTime.now();
String bucketKey = now.format(DateTimeFormatter.ofPattern(window.getTimestampFormat()));
// Result: "2024-01-15-14" for hour-based bucketing
```

This enum is typically used in rate limiting systems to group requests into time-based buckets and track usage patterns across different temporal granularities.
