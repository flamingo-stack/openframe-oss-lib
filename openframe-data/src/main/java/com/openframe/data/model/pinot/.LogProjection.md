Data model class representing a log projection for Pinot database storage, capturing essential log event information with organizational and device context.

## Key Components

- **LogProjection** - Main data class with comprehensive log event fields
- **Lombok annotations** - `@Data`, `@Builder`, `@AllArgsConstructor`, `@NoArgsConstructor` for automatic code generation
- **Event identification** - `toolEventId`, `eventType`, `toolType` for log categorization
- **Contextual data** - `userId`, `deviceId`, `hostname`, `organizationId/Name` for traceability
- **Temporal data** - `eventTimestamp`, `ingestDay` for time-based queries
- **Log metadata** - `severity`, `summary` for log classification and content

## Usage Example

```java
// Create a log projection using builder pattern
LogProjection logEntry = LogProjection.builder()
    .toolEventId("evt-12345")
    .ingestDay("2024-01-15")
    .toolType("security-scanner")
    .eventType("vulnerability_detected")
    .severity("HIGH")
    .userId("user123")
    .deviceId("device-456")
    .hostname("server01.example.com")
    .organizationId("org-789")
    .organizationName("Acme Corp")
    .summary("SQL injection vulnerability found")
    .eventTimestamp(Instant.now())
    .build();

// Use with Pinot queries for log analytics
// Typically serialized to JSON for Pinot ingestion
```
