A simple enumeration that defines severity levels for logging, alerting, or error classification within the OpenFrame data model system.

## Key Components

- **INFO** - Informational messages or events
- **WARNING** - Warning conditions that should be noted but don't prevent operation
- **ERROR** - Error conditions that may affect functionality
- **CRITICAL** - Critical issues requiring immediate attention

## Usage Example

```java
import com.openframe.data.model.enums.Severity;

// Assigning severity levels
Severity logLevel = Severity.ERROR;
Severity alertLevel = Severity.CRITICAL;

// Using in conditional logic
if (severity == Severity.CRITICAL) {
    sendImmediateAlert();
} else if (severity == Severity.ERROR) {
    logError();
}

// Using in switch statements
switch (severity) {
    case INFO:
        handleInfo();
        break;
    case WARNING:
        handleWarning();
        break;
    case ERROR:
        handleError();
        break;
    case CRITICAL:
        handleCritical();
        break;
}
```

This enum follows standard severity hierarchies commonly used in logging frameworks and monitoring systems, providing a consistent way to categorize the importance of events or conditions throughout the application.
