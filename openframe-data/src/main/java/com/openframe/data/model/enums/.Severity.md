An enumeration that defines severity levels for categorizing the importance or urgency of events, messages, or issues within the application.

## Key Components

- **INFO** - Informational messages for general system events
- **WARNING** - Non-critical issues that may require attention
- **ERROR** - Error conditions that affect functionality but don't halt operations
- **CRITICAL** - Severe errors that may cause system failure or require immediate action

## Usage Example

```java
// Assigning severity levels
Severity logLevel = Severity.ERROR;
Severity alertLevel = Severity.CRITICAL;

// Using in conditional logic
if (severity == Severity.CRITICAL) {
    sendImmediateAlert();
} else if (severity == Severity.ERROR) {
    logError();
}

// Switch statement usage
switch (severity) {
    case INFO:
        System.out.println("Info: " + message);
        break;
    case WARNING:
        System.out.println("Warning: " + message);
        break;
    case ERROR:
        handleError(message);
        break;
    case CRITICAL:
        handleCriticalError(message);
        break;
}

// Comparing severity levels (ordinal-based)
if (currentSeverity.ordinal() >= Severity.ERROR.ordinal()) {
    escalateIssue();
}
```
