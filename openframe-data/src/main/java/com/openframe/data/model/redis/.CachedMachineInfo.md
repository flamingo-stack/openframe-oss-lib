A lightweight Data Transfer Object designed for efficiently caching essential machine information in Redis storage. This class minimizes serialization overhead by containing only the most critical machine identifiers.

## Key Components

- **CachedMachineInfo**: Main DTO class implementing `Serializable` for Redis storage
- **machineId**: Unique identifier for the machine
- **hostname**: Network hostname of the machine  
- **organizationId**: Associated organization identifier
- **Lombok annotations**: `@Data`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic getter/setter generation

## Usage Example

```java
// Creating cached machine info
CachedMachineInfo machineInfo = new CachedMachineInfo(
    "machine-001", 
    "web-server-01.example.com", 
    "org-12345"
);

// Using with Redis template
@Autowired
private RedisTemplate<String, CachedMachineInfo> redisTemplate;

// Store in cache
redisTemplate.opsForValue().set("machine:machine-001", machineInfo);

// Retrieve from cache
CachedMachineInfo cached = redisTemplate.opsForValue().get("machine:machine-001");
System.out.println("Hostname: " + cached.getHostname());
```

The class is optimized for Redis caching scenarios where quick lookups of basic machine information are needed without the overhead of full machine entity serialization.
