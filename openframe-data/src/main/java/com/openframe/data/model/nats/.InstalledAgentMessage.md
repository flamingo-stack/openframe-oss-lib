A simple data model representing information about an installed agent, used for NATS messaging communication between system components.

## Key Components

- **machineId** - Unique identifier for the machine where the agent is installed
- **agentType** - Type classification of the installed agent
- **version** - Version string of the installed agent
- **@Data** - Lombok annotation that automatically generates getters, setters, toString, equals, and hashCode methods

## Usage Example

```java
// Creating an installed agent message
InstalledAgentMessage message = new InstalledAgentMessage();
message.setMachineId("machine-123");
message.setAgentType("monitoring-agent");
message.setVersion("1.2.3");

// Using with NATS messaging
String json = objectMapper.writeValueAsString(message);
natsConnection.publish("agents.installed", json.getBytes());

// Receiving and processing
InstalledAgentMessage received = objectMapper.readValue(
    messageData, 
    InstalledAgentMessage.class
);
System.out.println("Agent " + received.getAgentType() + 
                  " v" + received.getVersion() + 
                  " installed on " + received.getMachineId());
```

This message type is typically used in distributed systems to track agent deployments and maintain an inventory of installed components across different machines.
