A Spring component that maps download configuration objects from document format to NATS messaging format, handling version resolution and installation type conversion.

## Key Components

- **`map()`** - Main mapping method that converts a list of document-based download configurations to NATS format
- **`mapToNatsDownloadConfiguration()`** - Private method that handles individual configuration object mapping
- **`mapInstallationType()`** - Utility method that converts installation type enums between document and NATS models
- **`downloadConfigurationResolver`** - Injected dependency for resolving download links with version information

## Usage Example

```java
@Autowired
private DownloadConfigurationMapper mapper;

public void processDownloadConfigs() {
    // Document configurations from database/storage
    List<DownloadConfiguration> documentConfigs = configRepository.findAll();
    String version = "1.0.2";
    
    // Map to NATS format for messaging
    List<com.openframe.data.model.nats.DownloadConfiguration> natsConfigs = 
        mapper.map(documentConfigs, version);
    
    // Use mapped configurations for NATS publishing
    natsPublisher.publishConfigurations(natsConfigs);
}
```

The mapper ensures proper field mapping between different data models while resolving download links through the injected resolver service. Installation types default to STANDARD when null, and the mapper handles null input gracefully by returning null.
