A Spring component that resolves download links by replacing version placeholders in URL templates, with special handling for "latest" version requests.

## Key Components

- **`resolve(DownloadConfiguration, String)`** - Main method that resolves a download link template with the specified version
- **`resolveLink(String, String)`** - Private helper that performs the actual template replacement logic
- **`isLatestVersion(String)`** - Utility method to check if the version is "latest"
- **Constants** - Predefined path patterns and placeholders for URL template processing

## Usage Example

```java
@Autowired
private DownloadConfigurationLinkResolver linkResolver;

// For a specific version
DownloadConfiguration config = new DownloadConfiguration();
config.setLinkTemplate("https://releases.example.com/download/{version}/app.zip");

String specificVersionLink = linkResolver.resolve(config, "v1.2.3");
// Result: "https://releases.example.com/download/v1.2.3/app.zip"

// For latest version
String latestVersionLink = linkResolver.resolve(config, "latest");
// Result: "https://releases.example.com/latest/download/app.zip"
```

The resolver handles two scenarios: replacing `{version}` placeholders with actual version numbers, or converting `download/{version}` paths to `latest/download` when the version is "latest". This enables flexible URL generation for both versioned and latest release downloads.
