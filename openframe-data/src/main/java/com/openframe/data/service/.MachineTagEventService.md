Service interface for handling repository events and publishing Kafka messages when Machine, MachineTag, or Tag entities are modified. Encapsulates business logic for processing entity changes and maintaining data consistency across distributed systems.

## Key Components

- **processMachineSave()** - Handles single machine save events and publishes MachinePinotMessage
- **processMachineSaveAll()** - Batch processes multiple machine saves
- **processMachineTagSave()** - Handles machine-tag association changes
- **processMachineTagSaveAll()** - Batch processes multiple machine-tag associations
- **processTagSave()** - Handles tag modifications (only when tag name changes)
- **processTagSaveAll()** - Batch processes multiple tag modifications

## Usage Example

```java
@Component
public class MachineTagEventServiceImpl implements MachineTagEventService {
    
    @Autowired
    private KafkaProducer kafkaProducer;
    
    @Override
    public void processMachineSave(Machine machine) {
        // Fetch all tags for the machine
        List<Tag> tags = tagRepository.findByMachineId(machine.getId());
        
        // Create and send MachinePinotMessage
        MachinePinotMessage message = buildMessage(machine, tags);
        kafkaProducer.send("machine-topic", message);
    }
    
    @Override
    public void processMachineTagSave(MachineTag machineTag) {
        // Fetch associated machine and tag data
        Machine machine = machineRepository.findById(machineTag.getMachineId());
        Tag tag = tagRepository.findById(machineTag.getTagId());
        
        // Send updated machine data
        processMachineSave(machine);
    }
}
```

This service acts as an event handler that responds to repository changes and ensures downstream systems receive updated machine-tag relationship data through Kafka messaging.
