Processes command line arguments for tools by resolving dynamic placeholders like registration secrets based on tool configuration.

## Key Components

- **`process(String toolId, List<String> commandArgs)`** - Main method that processes a list of command arguments and replaces placeholders with actual values
- **`processArgument(String toolId, String argument)`** - Handles individual argument processing and placeholder replacement
- **`getRegistrationSecret(String toolId)`** - Retrieves the appropriate registration secret for a given tool ID
- **`REGISTRATION_SECRET_PLACEHOLDER`** - Constant defining the placeholder pattern `${server.registrationSecret}`

## Usage Example

```java
@Autowired
private ToolCommandParamsResolver resolver;

public void executeToolCommand() {
    String toolId = "my-tool";
    List<String> args = Arrays.asList(
        "--config", "/path/to/config",
        "--secret", "${server.registrationSecret}",
        "--port", "8080"
    );
    
    List<String> processedArgs = resolver.process(toolId, args);
    // Result: ["--config", "/path/to/config", "--secret", "actual-secret-value", "--port", "8080"]
    
    // Execute command with resolved parameters
    executeCommand(processedArgs);
}
```

The component is conditionally enabled via the `openframe.integration.tool.enabled` property and automatically injects registration secrets by matching tool IDs with registered secret retrievers.
