A Spring component that resolves command parameters for tool agents by replacing placeholder values with actual secrets during runtime processing.

## Key Components

- **`process(String toolId, List<String> commandArgs)`** - Main method that processes a list of command arguments and replaces placeholders with resolved values
- **`processArgument(String toolId, String argument)`** - Processes individual arguments to detect and replace the registration secret placeholder
- **`getRegistrationSecret(String toolId)`** - Retrieves the appropriate registration secret for a specific tool ID using registered retrievers
- **`isSuitable(String toolId, ToolAgentRegistrationSecretRetriever retriever)`** - Determines if a retriever is suitable for the given tool ID
- **`REGISTRATION_SECRET_PLACEHOLDER`** - Static constant defining the placeholder pattern `${server.registrationSecret}`

## Usage Example

```java
@Service
public class ToolExecutionService {
    
    @Autowired
    private ToolCommandParamsResolver paramsResolver;
    
    public void executeTool(String toolId, List<String> rawArgs) {
        // Raw command arguments with placeholders
        List<String> commandArgs = List.of(
            "--api-key", 
            "abc123",
            "--registration-secret", 
            "${server.registrationSecret}",
            "--endpoint",
            "https://api.example.com"
        );
        
        // Resolve placeholders to actual values
        List<String> resolvedArgs = paramsResolver.process(toolId, commandArgs);
        
        // resolvedArgs now contains actual registration secret instead of placeholder
        executeToolCommand(resolvedArgs);
    }
}
```

The component is conditionally enabled via the `openframe.integration.tool.enabled=true` property and uses dependency injection to access multiple secret retrievers for different tool types.
