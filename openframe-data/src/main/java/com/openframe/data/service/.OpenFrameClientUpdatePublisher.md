Publishes client update messages to all OpenFrame machines via NATS messaging when new client configurations are available. This service is conditionally enabled based on Spring Cloud Stream and feature flags.

## Key Components

**Main Service**
- `OpenFrameClientUpdatePublisher` - Spring service for broadcasting client updates to all machines

**Key Methods**
- `publish(OpenFrameClientConfiguration)` - Publishes update message to all machines if feature is enabled
- `buildMessage(OpenFrameClientConfiguration)` - Constructs NATS message with version and download configurations

**Dependencies**
- `NatsMessagePublisher` - Handles NATS message publishing
- `DownloadConfigurationMapper` - Maps configuration data for message payload

**Configuration Properties**
- `openframe.client.update.feature.enabled` - Feature flag to enable/disable publishing
- `spring.cloud.stream.enabled` - Conditional property for service activation

## Usage Example

```java
@Autowired
private OpenFrameClientUpdatePublisher publisher;

// Publish update to all machines when new configuration is ready
OpenFrameClientConfiguration newConfig = configurationService.getLatestConfig();
publisher.publish(newConfig);
// Logs: "Published client update message for all machines with version: 1.2.3"

// If feature is disabled, publish call is ignored
// Logs: "Client update publishing is disabled, skipping publish for version: 1.2.3"
```

The service publishes to the `machine.all.client-update` NATS topic, allowing all connected OpenFrame machines to receive configuration updates simultaneously. Messages include version information and mapped download configurations for client deployment.
