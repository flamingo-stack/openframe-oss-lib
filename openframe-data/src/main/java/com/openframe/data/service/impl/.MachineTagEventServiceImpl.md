Service implementation that processes machine, machine tag, and tag events, publishing corresponding Kafka messages to keep downstream systems synchronized with entity changes.

## Key Components

**Main Service Methods:**
- `processMachineSave()` / `processMachineSaveAll()` - Handles machine entity save events
- `processMachineTagSave()` / `processMachineTagSaveAll()` - Processes machine-tag association changes
- `processTagSave()` / `processTagSaveAll()` - Manages tag entity updates

**Helper Methods:**
- `fetchMachineTags()` - Retrieves all tags associated with a machine
- `fetchMachine()` - Loads machine entity by ID
- `buildMachinePinotMessage()` - Constructs Kafka message payload
- `sendMachineEventToKafka()` - Publishes machine events to Kafka topic

**Key Features:**
- Deduplication logic in batch operations to prevent duplicate processing
- Comprehensive error handling with logging
- Conditional activation via `openframe.device.aspect.enabled` property

## Usage Example

```java
@Autowired
private MachineTagEventService eventService;

// Process single machine save
Machine machine = new Machine();
machine.setMachineId("machine-123");
eventService.processMachineSave(machine);

// Process tag changes that affect multiple machines
Tag updatedTag = new Tag();
updatedTag.setId("tag-456");
updatedTag.setName("Production");
eventService.processTagSave(updatedTag); // Updates all machines with this tag
```
