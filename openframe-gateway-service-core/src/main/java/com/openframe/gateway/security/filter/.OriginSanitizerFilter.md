A Spring WebFlux security filter that sanitizes malicious "null" string values in the Origin header to prevent CORS bypass attacks.

## Key Components

- **OriginSanitizerFilter**: Main filter class implementing `WebFilter` interface
- **@Order(Ordered.HIGHEST_PRECEDENCE)**: Ensures this filter runs before all other filters in the chain
- **filter()**: Core method that inspects and sanitizes Origin headers
- **Header Mutation**: Uses Spring's immutable request builder to safely remove problematic headers

## Usage Example

```java
// Filter automatically registers as a Spring component
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class OriginSanitizerFilter implements WebFilter {
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
        String origin = exchange.getRequest().getHeaders().getFirst("Origin");
        
        // Remove Origin header if it contains the literal string "null"
        if (origin != null && "null".equalsIgnoreCase(origin.trim())) {
            ServerHttpRequest mutated = exchange.getRequest()
                .mutate()
                .headers(h -> h.remove("Origin"))
                .build();
            return chain.filter(exchange.mutate().request(mutated).build());
        }
        
        return chain.filter(exchange);
    }
}
```

This filter prevents potential security vulnerabilities where attackers send `Origin: null` headers to bypass CORS policies. It runs at the highest precedence to sanitize requests before they reach other security filters or application logic.
