A utility class that defines standardized path prefixes used throughout the OpenFrame Gateway for routing and security configuration.

## Key Components

- **CLIENTS_PREFIX** - Path prefix for client-related endpoints (`/clients`)
- **API_PREFIX** - Path prefix for API endpoints (`/api`)
- **TOOLS_PREFIX** - Path prefix for tool endpoints (`/tools`)
- **WS_TOOLS_PREFIX** - Path prefix for WebSocket tool endpoints (`/ws/tools`)
- **CHAT_PREFIX** - Path prefix for chat endpoints (`/chat`)

## Usage Example

```java
// Using path constants in security configuration
@Configuration
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(PathConstants.API_PREFIX + "/**").authenticated()
                .requestMatchers(PathConstants.CLIENTS_PREFIX + "/**").hasRole("CLIENT")
                .requestMatchers(PathConstants.TOOLS_PREFIX + "/**").hasRole("ADMIN")
            )
            .build();
    }
}

// Using in controller mappings
@RestController
@RequestMapping(PathConstants.API_PREFIX)
public class ApiController {
    
    @GetMapping("/health")
    public ResponseEntity<String> health() {
        return ResponseEntity.ok("OK");
    }
}
```

The constants provide a centralized way to manage endpoint paths, ensuring consistency across the application and making path changes easier to maintain.
