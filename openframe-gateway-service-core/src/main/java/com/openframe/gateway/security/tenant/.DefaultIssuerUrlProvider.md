Fallback `IssuerUrlProvider` implementation for self-hosted (OSS/single-tenant) OpenFrame deployments, resolving JWT issuer URLs from static configuration rather than a database lookup.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `allowedIssuerBase` | `@Value` field | Base URL for the issuer, from `openframe.security.jwt.allowed-issuer-base` |
| `superTenantId` | `@Value` field | Fixed tenant ID, from `openframe.security.jwt.super-tenant-id` |
| `resolveIssuerUrls()` | `Mono<List<String>>` | Wraps the cached URL list in a reactive `Mono` |
| `getCachedIssuerUrl()` | `List<String>` | Returns a singleton list of the fully-composed issuer URL |

**Activation condition:** Registered only when no other `IssuerUrlProvider` bean exists (e.g. `SaasIssuerUrlProvider` in managed/cloud deployments), controlled via `@ConditionalOnMissingBean`.

## Usage Example

```java
// application.yml (self-hosted deployment)
// openframe:
//   security:
//     jwt:
//       allowed-issuer-base: https://auth.example.com
//       super-tenant-id: my-org

// Resolved issuer URL produced:
// → https://auth.example.com/my-org

@Autowired
IssuerUrlProvider issuerUrlProvider;

// Reactive usage
issuerUrlProvider.resolveIssuerUrls()
    .subscribe(urls -> System.out.println(urls));
// Output: [https://auth.example.com/my-org]

// Synchronous usage (e.g. during filter initialization)
List<String> urls = issuerUrlProvider.getCachedIssuerUrl();
```

> **Note:** In SaaS/multi-tenant deployments, this bean is skipped in favour of a database-backed provider. For single-tenant OSS installs, no DB lookup is required — the issuer URL is fully resolved from static config at startup.
