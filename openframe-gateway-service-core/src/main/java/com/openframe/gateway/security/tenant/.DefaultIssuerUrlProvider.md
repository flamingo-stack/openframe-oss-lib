Fallback `IssuerUrlProvider` implementation for self-hosted (OSS/single-tenant) OpenFrame deployments. Resolves JWT issuer URLs from static configuration rather than a database lookup.

## Key Components

| Member | Description |
|--------|-------------|
| `@ConditionalOnMissingBean` | Auto-activates only when no other `IssuerUrlProvider` bean (e.g. `SaasIssuerUrlProvider`) is registered |
| `allowedIssuerBase` | Base URL for the issuer, injected from `openframe.security.jwt.allowed-issuer-base` |
| `superTenantId` | Tenant identifier injected from `openframe.security.jwt.super-tenant-id` |
| `resolveIssuerUrls()` | Returns a `Mono<List<String>>` wrapping the single constructed issuer URL |
| `getCachedIssuerUrl()` | Synchronously returns `[allowedIssuerBase + "/" + superTenantId]` — no caching layer needed |

## Usage Example

```java
// application.yml
openframe:
  security:
    jwt:
      allowed-issuer-base: https://auth.example.com
      super-tenant-id: tenant-001
```

```java
// Resolved issuer URL produced at runtime:
// → https://auth.example.com/tenant-001

// Typical injection in a JWT validation component
@Autowired
private IssuerUrlProvider issuerUrlProvider;

issuerUrlProvider.resolveIssuerUrls()
    .subscribe(urls -> {
        // urls = ["https://auth.example.com/tenant-001"]
    });
```

> **Note:** In multi-tenant (SaaS) deployments, this bean is **not activated** — a `SaasIssuerUrlProvider` takes precedence. This default is intended for self-hosted OpenFrame instances where the issuer is fixed at startup.
