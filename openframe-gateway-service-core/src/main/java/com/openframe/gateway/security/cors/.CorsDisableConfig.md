A Spring Boot configuration class that conditionally disables CORS restrictions for gateway applications, creating a permissive CORS filter when the `openframe.gateway.disable-cors` property is set to true.

## Key Components

- **CorsDisableConfig**: Main configuration class annotated with `@ConditionalOnProperty` to activate only when `openframe.gateway.disable-cors=true`
- **disabledCorsWebFilter()**: Bean method that creates a `CorsWebFilter` with permissive CORS settings, allowing all origins, methods, and headers with credential support

## Usage Example

```java
// In application.yml or application.properties
openframe.gateway.disable-cors=true

// The configuration automatically creates a permissive CORS filter
// No additional code needed - the filter is applied to all routes ("/**")

// Example of what this enables:
// - All origins allowed: config.addAllowedOriginPattern("*")
// - All HTTP methods: config.addAllowedMethod("*") 
// - All headers: config.addAllowedHeader("*")
// - Credentials supported: config.setAllowCredentials(true)
// - Cache preflight for 1 hour: config.setMaxAge(3600L)
```

**Important**: This configuration is intended for SaaS deployments where frontend and backend share the same domain. For production environments with different origins, implement proper CORS policies instead of disabling restrictions entirely.
