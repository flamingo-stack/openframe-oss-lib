Configuration class that creates a permissive CORS filter to completely disable CORS restrictions when the `openframe.gateway.disable-cors` property is set to true.

## Key Components

- **`@ConditionalOnProperty`** - Activates only when `openframe.gateway.disable-cors=true`
- **`disabledCorsWebFilter()`** - Creates a CORS filter that allows all origins, methods, and headers with credentials support
- **`@Primary`** - Ensures this filter takes precedence over default CORS configurations

## Usage Example

Enable CORS bypass in application properties:

```java
# application.yml
openframe:
  gateway:
    disable-cors: true
```

The configuration automatically creates a permissive CORS filter:

```java
// Resulting CORS configuration allows:
// - All origins (*) 
// - All HTTP methods (GET, POST, PUT, DELETE, etc.)
// - All headers
// - Credentials (cookies, authorization headers)
// - 1-hour preflight cache (3600 seconds)
```

**⚠️ Security Note**: This configuration is intended for SaaS deployments where frontend and backend share the same domain. It should NOT be used in OSS deployments or when different origins require proper CORS validation, as it completely bypasses CORS security protections.
