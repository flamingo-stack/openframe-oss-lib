This service provides dynamic JWT issuer URL resolution for multi-tenant security configurations. It manages a cached list of allowed issuer URLs based on tenant data from the database, with support for a super tenant configuration.

## Key Components

- **`resolveIssuerUrls()`** - Reactively resolves and caches issuer URLs from tenant repository
- **`getCachedIssuerUrl()`** - Provides synchronous access to cached issuer URLs
- **`allowedIssuerBase`** - Configurable base URL for issuer generation
- **`superTenantId`** - Optional configuration for super tenant support
- **`AtomicReference<Mono<List<String>>>`** - Thread-safe caching mechanism for reactive operations

## Usage Example

```java
@Component
public class JwtValidator {
    
    private final IssuerUrlProvider issuerUrlProvider;
    
    public JwtValidator(IssuerUrlProvider issuerUrlProvider) {
        this.issuerUrlProvider = issuerUrlProvider;
    }
    
    public Mono<Boolean> validateToken(String token) {
        return issuerUrlProvider.resolveIssuerUrls()
            .map(allowedIssuers -> {
                // Extract issuer from JWT token
                String tokenIssuer = extractIssuer(token);
                return allowedIssuers.contains(tokenIssuer);
            });
    }
    
    // For synchronous contexts
    public boolean isValidIssuer(String issuer) {
        List<String> cachedIssuers = issuerUrlProvider.getCachedIssuerUrl();
        return cachedIssuers.contains(issuer);
    }
}
```

The component implements thread-safe lazy loading with error recovery, ensuring issuer URLs are efficiently cached while maintaining data consistency across reactive operations.
