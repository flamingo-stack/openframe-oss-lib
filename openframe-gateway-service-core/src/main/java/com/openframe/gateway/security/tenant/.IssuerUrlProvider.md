Defines the contract for resolving JWT issuer URLs accepted by the gateway, supporting both single-tenant (OSS) and multi-tenant (SaaS) deployments.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `resolveIssuerUrls()` | `Mono<List<String>>` | Reactively fetches all accepted JWT issuer URLs (e.g., from DB or remote registry) |
| `getCachedIssuerUrl()` | `List<String>` | Returns the last resolved issuer URLs synchronously from cache, avoiding a blocking call on hot paths |

## Usage Example

```java
@Component
public class JwtAuthFilter {

    private final IssuerUrlProvider issuerUrlProvider;

    public JwtAuthFilter(IssuerUrlProvider issuerUrlProvider) {
        this.issuerUrlProvider = issuerUrlProvider;
    }

    // Reactive resolution — used during startup or cache refresh
    public Mono<Void> refreshIssuers() {
        return issuerUrlProvider.resolveIssuerUrls()
            .doOnNext(urls -> log.info("Accepted issuers: {}", urls))
            .then();
    }

    // Synchronous cache read — used during JWT validation on every request
    public boolean isKnownIssuer(String issuer) {
        return issuerUrlProvider.getCachedIssuerUrl().contains(issuer);
    }
}
```

## Implementation Notes

- **OSS default** — reads a single `Tenant` record from the local database
- **SaaS override** — a `@Primary` bean reads from `tenant_cluster_registrations`, enabling per-cluster issuer resolution across multiple tenants
- Implementations should populate the synchronous cache inside `resolveIssuerUrls()` so `getCachedIssuerUrl()` remains safe to call without blocking
