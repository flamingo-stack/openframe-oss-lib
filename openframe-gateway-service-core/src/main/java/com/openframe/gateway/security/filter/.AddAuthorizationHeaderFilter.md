A Spring WebFlux filter that ensures protected endpoints have an Authorization header by extracting bearer tokens from multiple sources (cookies, custom headers, or query parameters) and adding them to the request.

## Key Components

- **AddAuthorizationHeaderFilter**: Main WebFilter implementation that processes incoming requests
- **filter()**: Core method that checks for existing Authorization headers and adds them when missing
- **isPrivatePath()**: Determines which paths require authentication based on path prefixes
- **resolveBearerToken()**: Extracts bearer tokens from cookies, Access-Token header, or query parameters

## Usage Example

```java
// The filter automatically processes requests - no direct usage needed
// It runs as part of the Spring WebFlux filter chain

// For protected paths like:
// GET /dashboard/users - will have Authorization header added
// GET /tools/api/data - will have Authorization header added
// GET /clients/api/agents/register - public, no header added

// Token resolution priority:
// 1. access_token cookie
// 2. Access-Token header  
// 3. authorization query parameter

// Example of how a request gets transformed:
// Original: GET /dashboard/users (no Authorization header)
// After filter: GET /dashboard/users 
//               Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc...
```
