A reactive `WebFilter` that normalizes bearer token delivery by extracting tokens from multiple sources (cookie, custom header, query parameter) and injecting them as a standard `Authorization` header before downstream authentication processing.

## Key Components

| Member | Description |
|--------|-------------|
| `filter()` | Main filter method — skips public paths, short-circuits if `Authorization` already present, otherwise resolves and injects the token |
| `isPrivatePath()` | Determines whether the request path requires authentication, covering API, tools, WebSocket, chat, content, and scoped client routes |
| `resolveBearerToken()` | Priority-ordered token resolution: cookie → `Access-Token` header → `authorization` query param |
| `CookieService` | Injected dependency that handles cookie-based token extraction |

**Token resolution priority:**
```text
1. access_token cookie
2. Access-Token custom header
3. authorization query parameter
```

## Usage Example

This filter is auto-registered as a Spring `@Component` in the reactive filter chain. No manual wiring is required. Internally, it works as follows:

```java
// Incoming request (no Authorization header, but has a cookie)
// GET /api/tickets HTTP/1.1
// Cookie: access_token=eyJhbGci...

// After filter executes, the mutated request carries:
// GET /api/tickets HTTP/1.1
// Authorization: Bearer eyJhbGci...

// Public paths are passed through unchanged:
// GET /clients/oauth/token  →  no header injected (excluded path)
// GET /clients/metrics/cpu  →  no header injected (excluded path)

// Private paths that trigger token resolution:
// /api/**        /tools/**      /ws-tools/**
// /chat/**       /content/**    /internal/authz/probe
// /clients/**    (with exclusions above)
```

> **Note:** If no token is found from any source, the request is forwarded as-is, allowing downstream security filters (e.g., Spring Security OAuth2 resource server) to handle the unauthenticated case.
