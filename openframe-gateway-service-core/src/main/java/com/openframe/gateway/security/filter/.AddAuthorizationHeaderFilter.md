A reactive Spring `WebFilter` that normalizes incoming requests by injecting a standard `Authorization: Bearer` header when absent, resolving the token from cookies, a custom header, or a query parameter.

## Key Components

| Member | Description |
|--------|-------------|
| `filter()` | Core filter method — skips public paths, short-circuits if `Authorization` already exists, otherwise resolves and injects the token |
| `isPrivatePath()` | Determines whether a request path requires authentication, covering API, tools, WebSocket, chat, content, internal probe, and client routes (with explicit public exclusions) |
| `resolveBearerToken()` | Cascading token resolver: checks cookie → `Access-Token` header → `authorization` query param, returns first non-blank value |
| `CookieService` | Injected dependency responsible for extracting the access token from the request cookie store |

## Usage Example

```java
// Registered automatically as a Spring @Component — no manual wiring needed.
// The filter intercepts all reactive requests and mutates protected ones:

// Example: request arrives with token in cookie, no Authorization header
// Before filter:  GET /api/tickets  (no Authorization header)
// After filter:   GET /api/tickets  Authorization: Bearer eyJhbGci...

// Example: public path — filter is bypassed entirely
// GET /clients/oauth/token  →  passed through unchanged

// To extend protected paths, update isPrivatePath() or PathConstants:
public static final String API_PREFIX = "/api";
public static final String CONTENT_PREFIX = "/content";
```

## Token Resolution Priority

```text
1. HTTP Cookie     → access_token cookie   (cookieService)
2. Custom Header   → Access-Token header   (ACCESS_TOKEN_HEADER)
3. Query Parameter → ?authorization=...    (AUTHORIZATION_QUERY_PARAM)
```

> Paths explicitly excluded from client-private routing (e.g. `/clients/oauth/token`, `/clients/api/agents/register`) are treated as public and will never have a token injected.
