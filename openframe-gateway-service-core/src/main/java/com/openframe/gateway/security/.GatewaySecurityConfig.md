Configures Spring Security for the OpenFrame API Gateway in a reactive (WebFlux) environment, defining JWT authentication, role-based access control, and the security filter chain.

## Key Components

| Component | Description |
|-----------|-------------|
| `reactiveJwtAuthenticationConverter()` | Builds a JWT converter that extracts both `roles` (prefixed `ROLE_`) and `scope` (prefixed `SCOPE_`) claims, using `sub` as the principal |
| `springSecurityFilterChain()` | Defines the main security filter chain: disables CSRF/CORS/form login, enables OAuth2 resource server with a multi-issuer resolver, and enforces path-based RBAC |
| `passwordEncoder()` | Exposes a `BCryptPasswordEncoder` bean for password hashing |
| `AddAuthorizationHeaderFilter` | Custom pre-authentication filter injected before the authentication stage |

## Access Control Rules

| Path Pattern | Required Role |
|-------------|--------------|
| `OPTIONS /**` | Public |
| `/clients/metrics/**`, `/oauth/token`, `/api/agents/register` | Public |
| `/api/**` | `ADMIN` |
| `/tools/agent/**`, `/ws-tools/agent/**` | `AGENT` |
| NATS WebSocket endpoints | `ADMIN` or `AGENT` |
| `/chat/**` | `ADMIN` or `AGENT` |
| `/guide/**` | `ADMIN` |
| `/clients/**` | `AGENT` |
| `/tools/**`, `/ws-tools/**` | `ADMIN` |

## Usage Example

```java
// JWT token must carry claims in this shape for correct role resolution:
// {
//   "sub": "user-id-123",
//   "roles": ["ADMIN"],
//   "scope": "read write"
// }

// Spring auto-wires the filter chain — no manual wiring needed.
// To protect a new route, add a pathMatchers() rule in springSecurityFilterChain():

.pathMatchers(NEW_PREFIX + "/**").hasRole(ADMIN)
```

> **Note:** The `ReactiveAuthenticationManagerResolver` bean (not shown here) must support multi-issuer JWT validation. The filter chain is only active for reactive (`REACTIVE`) web applications via `@ConditionalOnWebApplication`.
