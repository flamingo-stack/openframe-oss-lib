Configures Spring WebFlux security for the OpenFrame API gateway, defining JWT-based OAuth2 resource server authentication, role-based access control, and filter chain setup for reactive web applications.

## Key Components

| Bean | Description |
|------|-------------|
| `reactiveJwtAuthenticationConverter()` | Extracts `roles` (prefixed `ROLE_`) and `scope` (prefixed `SCOPE_`) claims from JWT tokens, merging both into a single authority stream |
| `springSecurityFilterChain()` | Builds the reactive security filter chain: disables CSRF/CORS/BasicAuth/FormLogin, wires the multi-issuer JWT resolver, injects `AddAuthorizationHeaderFilter`, and enforces path-level RBAC |
| `passwordEncoder()` | Provides a BCrypt password encoder bean |

## Access Control Rules

```text
OPTIONS /**                              → permitAll (preflight)
/clients/metrics/**                      → permitAll
/clients/api/agents/register|reinstall   → permitAll
/clients/oauth/token                     → permitAll
/api/**                                  → ROLE_ADMIN
/tools/agent/**                          → ROLE_AGENT
/ws-tools/agent/**                       → ROLE_AGENT
NATS WebSocket endpoints                 → ROLE_AGENT or ROLE_ADMIN
/chat/**                                 → ROLE_AGENT or ROLE_ADMIN
/content/**                              → ROLE_ADMIN
/clients/**                              → ROLE_AGENT
/tools/**                                → ROLE_ADMIN
/ws-tools/**                             → ROLE_ADMIN
/**                                      → permitAll (catch-all)
```

## Usage Example

```java
// JWT must carry a "roles" claim for RBAC to work
// Example decoded JWT payload:
{
  "sub": "agent-service-01",
  "roles": ["AGENT"],
  "scope": "read write"
}

// Spring will resolve this to:
// GrantedAuthority: ROLE_AGENT, SCOPE_read, SCOPE_write

// Bean is auto-wired by Spring Boot — no manual instantiation needed.
// Customize path rules via PathConstants:
// API_PREFIX, CLIENTS_PREFIX, TOOLS_PREFIX, etc.
```

> **Note:** The class activates only for reactive (`REACTIVE`) web applications via `@ConditionalOnWebApplication`. A multi-issuer `ReactiveAuthenticationManagerResolver` bean must be provided separately to support multiple JWT issuers.
