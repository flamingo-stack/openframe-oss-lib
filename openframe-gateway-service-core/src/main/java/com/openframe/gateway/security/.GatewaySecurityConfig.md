Configures Spring Security for the OpenFrame API Gateway, defining JWT-based OAuth2 resource server authentication and role-based access control for all gateway routes in a reactive WebFlux environment.

## Key Components

| Bean | Description |
|------|-------------|
| `reactiveJwtAuthenticationConverter()` | Extracts both `roles` (prefixed `ROLE_`) and `scope` (prefixed `SCOPE_`) claims from JWT tokens and merges them as granted authorities |
| `springSecurityFilterChain()` | Builds the security filter chain: disables CSRF/CORS/Basic/Form auth, enables multi-issuer JWT OAuth2, injects `AddAuthorizationHeaderFilter`, and defines path-level authorization rules |
| `passwordEncoder()` | Provides a `BCryptPasswordEncoder` bean for password hashing |

## Authorization Rules Summary

| Path Pattern | Required Role |
|---|---|
| `OPTIONS /**` | Permit all |
| `/clients/metrics/**`, `/clients/api/agents/register`, `/clients/oauth/token`, etc. | Permit all (public agent bootstrap endpoints) |
| `/api/**` | `ROLE_ADMIN` |
| `/tools/agent/**`, `/ws-tools/agent/**` | `ROLE_AGENT` |
| NATS WebSocket endpoints | `ROLE_AGENT` or `ROLE_ADMIN` |
| `/chat/**` | `ROLE_AGENT` or `ROLE_ADMIN` |
| `/content/**`, `/tools/**`, `/ws-tools/**` | `ROLE_ADMIN` |
| `/clients/**` | `ROLE_AGENT` |
| `/**` (catch-all) | Permit all |

## Usage Example

```java
// The filter chain is auto-applied by Spring Security.
// JWT tokens must include a "roles" claim for role-based access:
{
  "sub": "agent-123",
  "roles": ["AGENT"],
  "scope": "read write"
}

// Injecting the security config in tests:
@SpringBootTest
@Import(GatewaySecurityConfig.class)
class GatewaySecurityConfigTest {

    @Autowired
    SecurityWebFilterChain filterChain;

    @Test
    void filterChainShouldNotBeNull() {
        assertNotNull(filterChain);
    }
}
```

> **Note:** This config is only active for reactive (`REACTIVE`) web applications (`@ConditionalOnWebApplication`). Multi-issuer JWT validation is delegated to the injected `ReactiveAuthenticationManagerResolver<ServerWebExchange>` bean (`issuerResolver`).
