Resolves HTTP authentication headers for integrated tools based on their configured API key type, mapping tool credentials to the appropriate header format for outbound gateway requests.

## Key Components

| Element | Description |
|---|---|
| `ToolApiKeyHeadersResolver` | Spring `@Component` that handles API key header resolution |
| `resolve(IntegratedTool tool)` | Main method returning a `Map<String, String>` of headers based on the tool's credential type |
| `APIKeyType.HEADER` | Returns a single custom-named header with the raw API key value |
| `APIKeyType.BEARER_TOKEN` | Returns an `Authorization: Bearer <key>` header |
| `APIKeyType.NONE` / `default` | Returns an empty map (no authentication headers) |

## Usage Example

```java
@Service
public class ToolRequestService {

    private final ToolApiKeyHeadersResolver headersResolver;

    public ToolRequestService(ToolApiKeyHeadersResolver headersResolver) {
        this.headersResolver = headersResolver;
    }

    public HttpHeaders buildHeaders(IntegratedTool tool) {
        HttpHeaders httpHeaders = new HttpHeaders();
        
        // Resolves to e.g. {"X-Api-Key": "abc123"} or {"Authorization": "Bearer abc123"}
        Map<String, String> authHeaders = headersResolver.resolve(tool);
        authHeaders.forEach(httpHeaders::set);
        
        return httpHeaders;
    }
}
```

> **Note:** If the tool has no credentials or the API key is `null`, the resolver safely defaults to `APIKeyType.NONE` and returns an empty map, preventing null pointer exceptions during header construction.
