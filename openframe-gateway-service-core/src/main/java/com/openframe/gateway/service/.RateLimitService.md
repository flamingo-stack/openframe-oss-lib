Provides reactive rate limiting functionality for API requests using Redis-backed atomic operations across multiple time windows (minute, hour, day).

## Key Components

- **`isAllowed(String keyId)`** - Main rate limiting check that validates requests against all time windows and atomically increments counters
- **`getRateLimitStatus(String keyId)`** - Returns comprehensive rate limit status for monitoring without incrementing counters
- **Rate limit windows** - Supports MINUTE, HOUR, and DAY time windows with configurable limits
- **Fail-open strategy** - Configurable behavior for Redis failures (allow or deny requests)
- **Reactive operations** - Fully reactive using `Mono` for non-blocking Redis interactions

## Usage Example

```java
@Autowired
private RateLimitService rateLimitService;

// Check if request is allowed for a user/API key
public Mono<Boolean> validateRequest(String userId) {
    return rateLimitService.isAllowed("user:" + userId)
        .doOnNext(allowed -> {
            if (!allowed) {
                log.warn("Rate limit exceeded for user: {}", userId);
            }
        });
}

// Get detailed rate limit status for monitoring
public Mono<RateLimitStatus> getUserStatus(String userId) {
    return rateLimitService.getRateLimitStatus("user:" + userId)
        .map(status -> {
            log.info("User {} has {}/{} requests remaining this minute", 
                userId, status.getMinuteLimit() - status.getMinuteRequests(), 
                status.getMinuteLimit());
            return status;
        });
}
```
