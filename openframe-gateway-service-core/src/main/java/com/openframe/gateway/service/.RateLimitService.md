A reactive rate limiting service that provides configurable API request throttling across multiple time windows using Redis atomic operations with fail-open error handling.

## Key Components

- **`isAllowed(String keyId)`** - Main rate limiting check that validates requests across minute, hour, and day windows
- **`getRateLimitStatus(String keyId)`** - Retrieves comprehensive rate limit status for monitoring and debugging
- **`checkAndIncrement()`** - Private method that atomically checks and increments counters using repository operations
- **`getStatus()`** - Private method that retrieves current status without incrementing counters
- **Error handling methods** - `createFailureResult()` and `createEmptyStatus()` for graceful degradation

## Usage Example

```java
@Autowired
private RateLimitService rateLimitService;

public Mono<String> handleApiRequest(String clientId) {
    return rateLimitService.isAllowed(clientId)
        .flatMap(allowed -> {
            if (allowed) {
                return processRequest();
            } else {
                return Mono.error(new RateLimitExceededException("Rate limit exceeded"));
            }
        });
}

// Get detailed rate limit information
public Mono<RateLimitStatus> getClientStatus(String clientId) {
    return rateLimitService.getRateLimitStatus(clientId);
}
```

The service supports configurable limits per time window, atomic Redis operations through `ReactiveRateLimitRepository`, and implements a fail-open strategy that allows requests when Redis is unavailable (configurable). All operations are fully reactive using Project Reactor.
