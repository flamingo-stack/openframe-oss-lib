Reactive Spring service that proxies HTTP requests to integrated tools (e.g., RMM agents, APIs) by resolving the tool's upstream URL, injecting appropriate authentication headers, and forwarding the request via a configured `WebClient` with SSL and timeout settings.

## Key Components

| Method | Description |
|---|---|
| `proxyApiRequest()` | Proxies REST API calls to a tool's upstream endpoint under `/tools`, injecting API key headers |
| `proxyAgentRequest()` | Proxies agent-level requests under `/tools/agent`, forwarding `Tool-Authorization` headers from the incoming request |
| `buildApiRequestHeaders()` | Builds standard JSON headers plus resolved API key headers for tool API calls |
| `buildAgentRequestHeaders()` | Builds headers for agent requests, passing through `Tool-Authorization` as `Authorization` |
| `proxy()` | Core proxy method — constructs a `WebClient`, sends the request with a 60s timeout, and maps the response to `ResponseEntity<String>` |
| `buildHttpClient()` | Configures a Reactor Netty `HttpClient` with SSL bypass (insecure trust manager), 30s connect timeout, 16KB decoder limits, and curl-style request logging |
| `processErrorResponse()` | Converts 4xx/5xx upstream responses into `WebClientResponseException` for consistent error handling |
| `buildErrorResponse()` | Maps exceptions to structured `ResponseEntity` error responses |

## Usage Example

```java
// Injected via Spring context
@Autowired
RestProxyService restProxyService;

// Proxy an API request to an integrated tool
Mono<ResponseEntity<String>> response = restProxyService.proxyApiRequest(
    "tactical-rmm",       // toolId
    serverHttpRequest,    // incoming reactive request
    "{\"agentId\": \"*\"}" // request body
);

response.subscribe(entity -> {
    System.out.println("Status: " + entity.getStatusCode());
    System.out.println("Body:   " + entity.getBody());
});
```

> **Note:** SSL verification is intentionally bypassed via `InsecureTrustManagerFactory` to support self-signed certificates common in on-premise MSP tool deployments. The in-memory buffer is capped at **16 MB** to handle large upstream responses. Device-level access validation for agent requests is a known TODO pending tool connection feature implementation.
