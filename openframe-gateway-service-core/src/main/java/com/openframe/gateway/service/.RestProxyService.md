Reactive proxy service that forwards HTTP requests from the OpenFrame gateway to integrated tools (RMM agents, APIs), handling SSL, authentication headers, and error responses.

## Key Components

| Method | Description |
|---|---|
| `proxyApiRequest` | Proxies generic API calls to a tool's REST endpoint, injecting API key headers |
| `proxyAgentRequest` | Proxies agent/device-scoped requests, forwarding `Tool-Authorization` headers |
| `buildHttpClient` | Constructs a Reactor Netty `HttpClient` with SSL bypass, timeouts, and curl logging |
| `buildApiRequestHeaders` | Builds standard JSON headers + resolved API key credentials for tool API calls |
| `buildAgentRequestHeaders` | Extracts and forwards `Tool-Authorization` from the incoming request |
| `proxy` | Core dispatch method — creates a `WebClient`, applies headers/body, handles errors reactively |
| `buildErrorResponse` | Maps `WebClientResponseException` or generic errors to `ResponseEntity<String>` |

## Usage Example

```java
// Inject the service
@Autowired
private RestProxyService restProxyService;

// Proxy an API request to a registered tool
Mono<ResponseEntity<String>> response = restProxyService.proxyApiRequest(
    "tool-abc123",
    serverHttpRequest,
    "{\"query\": \"status\"}"
);

// Proxy an agent-scoped request (e.g., Tactical RMM)
Mono<ResponseEntity<String>> agentResponse = restProxyService.proxyAgentRequest(
    "tool-rmm456",
    serverHttpRequest,
    "{\"agentId\": \"device-001\"}"
);
```

## Notes

- SSL verification is **disabled** via `InsecureTrustManagerFactory` — intended for internal/self-signed tool endpoints
- Response and connection timeouts are set to **60s** and **30s** respectively
- In-memory codec buffer is capped at **16 MB** to support large upstream responses
- `CurlLoggingHandler` is attached per connection for debug logging
- ⚠️ Device-level access validation for agent requests is **not yet implemented** (see TODO in source)
