A reactive service that manages API key request statistics using Redis for atomic counter operations with configurable TTL.

## Key Components

- **ReactiveApiKeyStatsRepository** - Redis repository for atomic counter operations
- **incrementSuccessful()** - Atomically increments successful request counters for an API key
- **incrementFailed()** - Atomically increments failed request counters for an API key
- **redisTtl** - Configurable TTL for Redis entries via `openframe.api-key-stats.redis-ttl` property

## Usage Example

```java
@RestController
public class ApiController {
    
    @Autowired
    private ApiKeyStatsService statsService;
    
    @GetMapping("/api/data")
    public Mono<ResponseEntity<String>> getData(@RequestHeader("X-API-Key") String apiKey) {
        return processRequest(apiKey)
            .doOnSuccess(result -> statsService.incrementSuccessful(apiKey))
            .doOnError(error -> statsService.incrementFailed(apiKey))
            .map(data -> ResponseEntity.ok(data));
    }
}
```

The service provides fire-and-forget statistics tracking with built-in error handling. Failed Redis operations are logged but don't affect the main request flow. TTL ensures automatic cleanup of old statistics entries.
