Service class responsible for tracking API key usage statistics using atomic Redis operations with configurable TTL.

## Key Components

- **ApiKeyStatsService** - Main service class for managing API key metrics
- **incrementSuccessful(String keyId)** - Atomically increments successful request counter for an API key
- **incrementFailed(String keyId)** - Atomically increments failed request counter for an API key
- **ReactiveApiKeyStatsRepository** - Injected repository for Redis operations
- **redisTtl** - Configurable TTL for Redis entries from `openframe.api-key-stats.redis-ttl` property

## Usage Example

```java
@Autowired
private ApiKeyStatsService apiKeyStatsService;

// Track successful API request
public void handleSuccessfulRequest(String apiKeyId) {
    apiKeyStatsService.incrementSuccessful(apiKeyId);
}

// Track failed API request
public void handleFailedRequest(String apiKeyId) {
    apiKeyStatsService.incrementFailed(apiKeyId);
}

// In application.yml
// openframe:
//   api-key-stats:
//     redis-ttl: 86400  # 24 hours
```

Both methods use fire-and-forget semantics with error handling that logs failures but doesn't propagate exceptions, making them safe for use in request processing pipelines.
