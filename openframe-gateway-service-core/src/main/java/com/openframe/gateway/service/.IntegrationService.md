Provides reactive service logic for testing connectivity to integrated MSP tools by resolving their API URLs and performing health check requests.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `testIntegrationConnection(String toolId)` | `Mono<String>` | Looks up a tool by ID, validates it is enabled, resolves its API URL, and performs a `GET /health` request |
| `adjustUrl(String originalUrl)` | `private String` | Rewrites the host to `localhost` when running under the `local` Spring profile, preserving port, path, query, and fragment |
| `isLocalProfile()` | `private boolean` | Returns `true` if the `local` profile is active or no profiles are configured |

**Dependencies injected:**
- `ToolUrlService` — resolves `ToolUrl` entries by type (`API`)
- `ReactiveIntegratedToolRepository` — reactive repository for fetching integrated tool documents
- `WebClient.Builder` — used to build the health-check HTTP client
- `Environment` — inspects active Spring profiles

## Usage Example

```java
// Injected into a controller or another service
@RestController
@RequiredArgsConstructor
public class IntegrationController {

    private final IntegrationService integrationService;

    @GetMapping("/integrations/{toolId}/test")
    public Mono<ResponseEntity<String>> testConnection(@PathVariable String toolId) {
        return integrationService.testIntegrationConnection(toolId)
            .map(response -> ResponseEntity.ok(response))
            .onErrorResume(ResponseStatusException.class, ex ->
                Mono.just(ResponseEntity.status(ex.getStatusCode()).body(ex.getReason()))
            );
    }
}
```

> **Note:** When no Spring profile is active, `isLocalProfile()` defaults to `true`, so all tool URLs are rewritten to `localhost`. Ensure an explicit profile (e.g., `prod`) is set in non-local deployments to prevent unintended URL rewriting.
