Provides integration connectivity testing for tools registered in the OpenFrame gateway, resolving tool API URLs and verifying reachability via HTTP health checks.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `testIntegrationConnection(String toolId)` | `Mono<String>` | Looks up a tool by ID, validates it is enabled, resolves its API URL, and performs a `GET /health` request |
| `adjustUrl(String originalUrl)` | `private String` | Rewrites the host to `localhost` when the active Spring profile is `local` (or no profile is set), preserving scheme, port, path, query, and fragment |
| `isLocalProfile()` | `private boolean` | Returns `true` if the `local` profile is active or no profiles are configured |

**Dependencies injected:**

- `ToolUrlService` — resolves the `API` URL from a tool document
- `ReactiveIntegratedToolRepository` — reactive lookup of integrated tool records
- `WebClient.Builder` — issues the health-check HTTP request
- `Environment` — reads active Spring profiles for local-mode detection

## Usage Example

```java
// Inject the service and test connectivity for a registered tool
integrationService.testIntegrationConnection("zabbix-prod")
    .subscribe(
        response -> System.out.println("Health response: " + response),
        error   -> System.err.println("Connection failed: " + error.getMessage())
    );
```

**Error conditions:**

- `404 NOT_FOUND` — no tool exists for the given `toolId`
- `403 FORBIDDEN` — tool exists but `isEnabled()` returns `false`
- `500 INTERNAL_SERVER_ERROR` — tool has no `API`-type URL configured
