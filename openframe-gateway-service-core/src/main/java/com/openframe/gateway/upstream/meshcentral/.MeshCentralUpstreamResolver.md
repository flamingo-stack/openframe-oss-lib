Resolves upstream URIs for MeshCentral proxy routing within the OpenFrame gateway, handling both REST and WebSocket traffic without a MongoDB lookup on each request. On a shared multi-tenant pod it rewrites the upstream namespace and the tenant-scoped path prefix for the calling tenant per request.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `TOOL_ID` | `String` constant | Identifies this resolver as `"meshcentral-server"` |
| `props` | `MeshCentralRoutingProperties` | Config-bound upstream addresses from `openframe.tools.meshcentral.*` |
| `proxyUrlResolver` | `ProxyUrlResolver` | Builds the base proxy URI from host/port/path |
| `supportsToolId()` | Method | Binds this resolver exclusively to `"meshcentral-server"` |
| `resolveRest()` | Method | Builds an `http://` upstream URI from `props.getApi()` |
| `resolveWs()` | Method | Builds a `ws://` upstream URI from `props.getWebsocket()` |
| `prependPathPrefix()` | Method | Prepends tenant-scoped path prefix, skipping double-prefixing for relay tunnel URLs |
| `resolveTenantPathPrefix()` | Method | Replaces the `tenant-uuid` placeholder token in the configured path prefix with the per-request `X-Tenant-Id` (no-op without the header) |
| `withPath()` | Method | Reconstructs a URI with a new path using raw component concatenation to avoid strict query-param validation failures on base64-padded values |

## Usage Example

```java
// Wired automatically by Spring; consumed via ToolUpstreamResolver interface
MeshCentralUpstreamResolver resolver = context.getBean(MeshCentralUpstreamResolver.class);

// REST upstream (e.g. API calls to MeshCentral)
URI restUri = resolver.resolveRest(tool, serverHttpRequest, "/meshcentral");
// → http://meshcentral-host:4430/meshcentral/api/v1/...

// WebSocket upstream (e.g. agent relay connections)
URI wsUri = resolver.resolveWs(tool, serverHttpRequest, "/meshcentral");
// → ws://meshcentral-host:4431/meshcentral/meshrelay.ashx?...
```

## Notes

- Reads upstream config statically from `application.yml` — avoids per-request DB lookups that `DefaultToolUpstreamResolver` performs.
- The `withPath()` method deliberately bypasses `UriComponentsBuilder.build(true)` to prevent failures when query values contain `=` characters (e.g. base64 padding in mesh auth cookies).
- Double-prefix guard in `prependPathPrefix()` prevents malformed URLs like `/<domain>/<domain>/meshrelay.ashx` that MeshCentral's relay logic would otherwise produce.
- On a shared multi-tenant pod the upstream host carries a `tenant-ns` placeholder and the path prefix a `tenant-uuid` token; both are swapped for the calling tenant per request from the trusted `X-Tenant-Ns` / `X-Tenant-Id` headers. Single-tenant pods send neither header, so host and prefix are used verbatim.
