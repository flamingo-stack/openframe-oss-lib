Configuration properties class that maps `openframe.tools.meshcentral.*` YAML properties to typed Java objects for routing HTTP and WebSocket traffic to a MeshCentral upstream service.

## Key Components

| Element | Description |
|---|---|
| `MeshCentralRoutingProperties` | Root `@ConfigurationProperties` bean bound to `openframe.tools.meshcentral` |
| `Upstream` (inner class) | Holds `url`, `port`, and optional `pathPrefix` for a single upstream endpoint |
| `api` field | HTTP upstream — handles REST, dashboard, and plugin routes (root-bound, `pathPrefix` must be empty) |
| `websocket` field | WebSocket upstream — handles `control.ashx`, `meshrelay.ashx`, and `agent.ashx` |

## Usage Example

Define upstreams in your application YAML:

```yaml
openframe:
  tools:
    meshcentral:
      api:
        url: http://meshcentral.integrated-tools.svc.cluster.local
        port: "8383"
      websocket:
        url: ws://meshcentral.integrated-tools.svc.cluster.local
        port: "8383"
        pathPrefix: "/${TENANT_ID}"
```

Inject and use in a gateway filter or route locator:

```java
@Component
public class MeshCentralRouteConfig {

    private final MeshCentralRoutingProperties props;

    public MeshCentralRouteConfig(MeshCentralRoutingProperties props) {
        this.props = props;
    }

    public String resolveApiTarget() {
        MeshCentralRoutingProperties.Upstream api = props.getApi();
        return api.getUrl() + ":" + api.getPort();
    }

    public String resolveWsTarget(String tenantId) {
        MeshCentralRoutingProperties.Upstream ws = props.getWebsocket();
        String prefix = ws.getPathPrefix() != null
            ? ws.getPathPrefix().replace("${TENANT_ID}", tenantId)
            : "";
        return ws.getUrl() + ":" + ws.getPort() + prefix;
    }
}
```

> **Note:** Because MeshCentral REST plugin routes are root-bound, the `api` upstream's `pathPrefix` must remain empty. Only the `websocket` upstream should carry a `pathPrefix` (e.g. a tenant-scoped domain path).
