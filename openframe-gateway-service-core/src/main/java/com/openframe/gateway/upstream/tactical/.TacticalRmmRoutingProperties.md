Configuration properties class for routing Tactical RMM traffic across its three upstream services within the OpenFrame gateway, replacing the fan-out logic previously handled by `tactical-nginx`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `backend` | `Upstream` | Django/uWSGI upstream for all REST traffic |
| `websocket` | `Upstream` | Daphne ASGI upstream for `/ws/` WebSocket traffic |
| `nats` | `Upstream` | NATS WebSocket listener for `/natsws` traffic |
| `Upstream` | Static nested class | Holds `url`, `port`, and optional `pathPrefix` for each upstream |

Bound to the `openframe.tools.tactical-rmm` configuration prefix via Spring Boot's `@ConfigurationProperties`.

## Usage Example

```yaml
openframe:
  tools:
    tactical-rmm:
      backend:
        url: http://tactical-backend.integrated-tools.svc.cluster.local
        port: "8080"
      websocket:
        url: ws://tactical-backend.integrated-tools.svc.cluster.local
        port: "8383"
        path-prefix: /ws/
      nats:
        url: ws://tactical-nats.integrated-tools.svc.cluster.local
        port: "9235"
        path-prefix: /natsws
```

```java
@Service
public class TacticalRmmRouter {

    private final TacticalRmmRoutingProperties props;

    public TacticalRmmRouter(TacticalRmmRoutingProperties props) {
        this.props = props;
    }

    public String resolveUpstreamUrl(String path) {
        if (path.startsWith(props.getNats().getPathPrefix())) {
            return props.getNats().getUrl() + ":" + props.getNats().getPort();
        }
        if (path.startsWith(props.getWebsocket().getPathPrefix())) {
            return props.getWebsocket().getUrl() + ":" + props.getWebsocket().getPort();
        }
        return props.getBackend().getUrl() + ":" + props.getBackend().getPort();
    }
}
```

> **Note:** All three upstreams must be configured — the gateway uses `pathPrefix` on the `websocket` and `nats` upstreams to perform the path-based fan-out previously managed by `tactical-nginx`.
