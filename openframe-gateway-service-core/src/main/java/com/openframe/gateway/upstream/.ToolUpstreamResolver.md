Strategy interface for resolving the upstream target URI when proxying requests to integrated tools. Implementations encapsulate all routing logic per tool, keeping the gateway's proxy layer decoupled from tool-specific URL schemes.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `supportsToolId()` | `Optional<String>` | Returns the `toolId` this resolver handles. Return `Optional.empty()` for the default fallback (exactly one resolver must do this). |
| `resolveRest(...)` | `URI` | Builds the upstream target URI for REST/HTTP requests, stripping the given path prefix before forwarding. |
| `resolveWs(...)` | `URI` | Builds the upstream target URI for WebSocket requests with the same prefix-stripping semantics. |

## Built-in Implementations

| Class | Behavior |
|-------|----------|
| `DefaultToolUpstreamResolver` | Reads the URL from the `IntegratedTool` Mongo document via `ToolUrlType.API` / `ToolUrlType.WS`. Serves as the universal fallback. |
| `TacticalRmmUpstreamResolver` *(example)* | Reads URLs from typed config properties and may fan out by request path. |

## Usage Example

```java
// Create a custom resolver for a new tool
@Component
public class MyToolUpstreamResolver implements ToolUpstreamResolver {

    @Override
    public Optional<String> supportsToolId() {
        return Optional.of("my-tool-id"); // handles only this toolId
    }

    @Override
    public URI resolveRest(IntegratedTool tool,
                           ServerHttpRequest request,
                           String stripPrefix) {
        String path = request.getPath().value().replace(stripPrefix, "");
        return URI.create("https://my-tool-backend.internal" + path);
    }

    @Override
    public URI resolveWs(IntegratedTool tool,
                         ServerHttpRequest request,
                         String stripPrefix) {
        String path = request.getPath().value().replace(stripPrefix, "");
        return URI.create("wss://my-tool-backend.internal" + path);
    }
}
```

> **Extending routing:** Add `@Component` implementing `ToolUpstreamResolver` and return the target `toolId` from `supportsToolId()`. Spring auto-discovers it — no other code changes required.
