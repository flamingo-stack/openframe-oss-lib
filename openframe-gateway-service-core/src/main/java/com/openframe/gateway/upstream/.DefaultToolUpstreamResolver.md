Fallback implementation of `ToolUpstreamResolver` that resolves upstream URLs for integrated tools by reading configuration from MongoDB via `ToolUrlService`, used when no dedicated resolver is registered for a given tool.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `toolUrlService` | Dependency | Retrieves `ToolUrl` documents (REST/WS) from MongoDB for a given `IntegratedTool` |
| `proxyUrlResolver` | Dependency | Builds the final proxy `URI` from the tool key, base URL, port, and incoming request URI |
| `supportsToolId()` | Method | Returns `Optional.empty()` — marks this as the catch-all fallback (no specific tool ID claimed) |
| `resolveRest()` | Method | Resolves the upstream REST URI using `ToolUrlType.API` |
| `resolveWs()` | Method | Resolves the upstream WebSocket URI using `ToolUrlType.WS` |
| `resolveByType()` | Private Method | Shared logic — fetches the configured `ToolUrl` by type and delegates to `ProxyUrlResolver` |

## Usage Example

```java
// Injected automatically when no dedicated ToolUpstreamResolver matches the tool ID.
// The gateway picks this resolver as the default fallback.

// REST upstream resolution example:
URI upstreamUri = defaultToolUpstreamResolver.resolveRest(
    integratedTool,   // MongoDB document for the tool (e.g., Zabbix, Grafana)
    serverHttpRequest, // incoming reactive HTTP request
    "/api/zabbix"      // prefix to strip before forwarding
);

// WebSocket upstream resolution example:
URI wsUri = defaultToolUpstreamResolver.resolveWs(
    integratedTool,
    serverHttpRequest,
    "/ws/zabbix"
);

// Throws IllegalStateException if no URL of the requested type
// is configured on the IntegratedTool document.
```

## Notes

- `supportsToolId()` returning `Optional.empty()` is the convention used by the resolver registry to identify this as the **default/fallback** resolver.
- Throws `IllegalStateException` if the tool has no URL configured for the requested `ToolUrlType`, preventing silent misrouting.
