Serves as the default fallback implementation of `ToolUpstreamResolver`, resolving upstream REST and WebSocket URLs for integrated tools by reading configuration from MongoDB via `ToolUrlService`. Used when no dedicated resolver is registered for a given tool.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `toolUrlService` | Dependency | Fetches `ToolUrl` entries from the `IntegratedTool` document by URL type |
| `proxyUrlResolver` | Dependency | Builds the final proxy `URI` from tool ID, base URL, port, and incoming request URI |
| `supportsToolId()` | Method | Returns `Optional.empty()` — signals this resolver applies to all tools without a dedicated resolver |
| `resolveRest()` | Method | Resolves the upstream REST (`API`) URI for a given tool and request |
| `resolveWs()` | Method | Resolves the upstream WebSocket (`WS`) URI for a given tool and request |
| `resolveByType()` | Private | Shared logic — looks up the `ToolUrl` by type and delegates to `ProxyUrlResolver` |

## Usage Example

```java
// Injected automatically by Spring; called by the gateway routing pipeline
URI upstreamUri = defaultToolUpstreamResolver.resolveRest(
    integratedTool,
    serverHttpRequest,
    "/api/v1/tools/my-tool"
);

// For WebSocket upgrades
URI wsUri = defaultToolUpstreamResolver.resolveWs(
    integratedTool,
    serverHttpRequest,
    "/ws/my-tool"
);
```

> **Note:** Throws `IllegalStateException` if the target tool has no URL configured for the requested `ToolUrlType` (`API` or `WS`). Ensure all integrated tools have both URL types populated in MongoDB before routing traffic through this resolver.
