Registry component that aggregates all `ToolUpstreamResolver` beans and dispatches upstream resolution requests by `toolId`, using a fallback strategy when no specific resolver matches.

## Key Components

| Member | Description |
|--------|-------------|
| `byToolId` | Immutable map of `toolId → ToolUpstreamResolver` for tool-specific strategies |
| `fallback` | Default `ToolUpstreamResolver` used when no tool-specific match is found |
| `ToolUpstreamResolverRegistry(List<ToolUpstreamResolver>)` | Constructor that wires all resolver beans, validates uniqueness, and enforces exactly one default |
| `resolve(String toolId)` | Returns the matching resolver for a given `toolId`, or the fallback if none is registered |

## Wiring Rules (enforced at startup)

- Resolvers returning a **non-empty** `supportsToolId()` are registered as tool-specific strategies
- Exactly **one** resolver must return `Optional.empty()` — this becomes the fallback
- **Duplicate** `toolId` registrations → `IllegalStateException`
- **Missing or duplicate** default resolver → `IllegalStateException`

## Usage Example

```java
// Resolvers are auto-discovered by Spring — no manual registration needed.

// Tool-specific resolver (e.g., handles "my-tool")
@Component
public class MyToolResolver implements ToolUpstreamResolver {
    @Override
    public Optional<String> supportsToolId() {
        return Optional.of("my-tool");
    }
}

// Default fallback resolver
@Component
public class DefaultToolResolver implements ToolUpstreamResolver {
    @Override
    public Optional<String> supportsToolId() {
        return Optional.empty(); // marks this as the fallback
    }
}

// At runtime, the registry dispatches automatically
ToolUpstreamResolver resolver = registry.resolve("my-tool");   // → MyToolResolver
ToolUpstreamResolver resolver = registry.resolve("unknown");   // → DefaultToolResolver
```
