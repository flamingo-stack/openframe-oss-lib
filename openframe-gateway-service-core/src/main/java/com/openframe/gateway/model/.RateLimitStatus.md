A record class that encapsulates rate limiting status information for API keys, tracking request counts and limits across different time windows (minute, hour, day).

## Key Components

- **Record Fields**: Immutable data structure containing:
  - `keyId` - Unique identifier for the API key
  - `minuteRequests/minuteLimit` - Current requests and limit for minute window
  - `hourRequests/hourLimit` - Current requests and limit for hour window  
  - `dayRequests/dayLimit` - Current requests and limit for day window
  - `isMinuteExceeded/isHourExceeded/isDayExceeded` - Boolean flags indicating if limits are exceeded

- **Lombok Integration**: Uses `@Builder` annotation for convenient object construction

## Usage Example

```java
// Create rate limit status using builder pattern
RateLimitStatus status = RateLimitStatus.builder()
    .keyId("api-key-123")
    .minuteRequests(45)
    .minuteLimit(60)
    .hourRequests(1200)
    .hourLimit(3600)
    .dayRequests(15000)
    .dayLimit(100000)
    .isMinuteExceeded(false)
    .isHourExceeded(false)
    .isDayExceeded(false)
    .build();

// Check if any limits are exceeded
if (status.isMinuteExceeded()) {
    throw new RateLimitExceededException("Minute limit exceeded");
}

// Access current usage
System.out.println("Hour usage: " + status.hourRequests() + "/" + status.hourLimit());
```
