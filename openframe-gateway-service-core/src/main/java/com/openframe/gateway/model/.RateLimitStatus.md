A data record that encapsulates rate limiting status and quota information for API keys, tracking request counts and limits across different time windows.

## Key Components

- **RateLimitStatus** - Immutable record with builder pattern support
- **keyId** - String identifier for the API key
- **Request counters** - `minuteRequests`, `hourRequests`, `dayRequests` track current usage
- **Limit thresholds** - `minuteLimit`, `hourLimit`, `dayLimit` define maximum allowed requests
- **Exceeded flags** - `isMinuteExceeded`, `isHourExceeded`, `isDayExceeded` indicate quota violations

## Usage Example

```java
// Create rate limit status using builder
RateLimitStatus status = RateLimitStatus.builder()
    .keyId("api_key_123")
    .minuteRequests(45)
    .minuteLimit(100)
    .hourRequests(2500)
    .hourLimit(5000)
    .dayRequests(15000)
    .dayLimit(50000)
    .isMinuteExceeded(false)
    .isHourExceeded(false)
    .isDayExceeded(false)
    .build();

// Check if any limits are exceeded
boolean hasViolations = status.isMinuteExceeded() || 
                       status.isHourExceeded() || 
                       status.isDayExceeded();

// Access remaining capacity
int minuteRemaining = status.minuteLimit() - status.minuteRequests();
```
