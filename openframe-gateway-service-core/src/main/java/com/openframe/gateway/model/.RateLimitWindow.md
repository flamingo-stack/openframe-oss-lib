Defines the time window granularities available for rate limiting in the OpenFrame API gateway, mapping each window to a `Duration` and a timestamp format string used for key bucketing.

## Key Components

| Enum Constant | Duration | Timestamp Format |
|---------------|----------|-----------------|
| `MINUTE` | 1 minute | `yyyy-MM-dd-HH-mm` |
| `HOUR` | 1 hour | `yyyy-MM-dd-HH` |
| `DAY` | 1 day | `yyyy-MM-dd` |

**Fields & Methods:**
- `duration` — `java.time.Duration` representing the window length
- `timestampFormat` — date pattern used to bucket requests into time-keyed rate limit slots
- `getSeconds()` — convenience method returning the window duration in seconds (useful for TTL configuration in Redis or similar stores)

## Usage Example

```java
RateLimitWindow window = RateLimitWindow.HOUR;

// Get TTL for Redis key expiry
long ttlSeconds = window.getSeconds(); // 3600

// Format current timestamp into a bucket key
String bucket = LocalDateTime.now()
    .format(DateTimeFormatter.ofPattern(window.getTimestampFormat()));
// e.g. "2024-11-15-14" for the 2pm bucket

String rateLimitKey = "rate:" + clientId + ":" + bucket;
```

> **Note:** The `timestampFormat` pattern is designed so that all requests within the same window produce an identical bucket string, enabling atomic counter increments per client per time window without additional rounding logic.
