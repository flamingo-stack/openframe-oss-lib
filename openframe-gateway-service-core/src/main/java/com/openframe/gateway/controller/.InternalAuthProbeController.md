A Spring WebFlux REST controller that provides an internal authorization probe endpoint for health checks and monitoring purposes. This controller is conditionally enabled based on configuration properties.

## Key Components

- **InternalAuthProbeController**: Main controller class exposing internal authorization endpoints
- **probe()**: GET endpoint that returns an empty Mono with HTTP 200 status
- **@ConditionalOnProperty**: Enables the controller only when `openframe.gateway.internal.enable=true`
- **@RequestMapping("/internal/authz")**: Base path for all authorization-related internal endpoints

## Usage Example

```java
// Configuration to enable the controller
# application.yml
openframe:
  gateway:
    internal:
      enable: true

// Making a request to the probe endpoint
WebClient client = WebClient.create();
Mono<Void> response = client.get()
    .uri("http://localhost:8080/internal/authz/probe")
    .retrieve()
    .bodyToMono(Void.class);

// The endpoint returns HTTP 200 OK with empty body
// Useful for health checks or load balancer probes
```

The controller serves as a lightweight health check endpoint that can be used by monitoring systems or load balancers to verify the authorization service is responsive without performing actual authorization logic.
