A Spring WebFlux REST controller that provides an internal authorization probe endpoint for health checks and monitoring, conditionally enabled through configuration properties.

## Key Components

- **InternalAuthProbeController**: Main controller class with conditional activation
- **@ConditionalOnProperty**: Enables controller only when `openframe.gateway.internal.enable=true`
- **probe()**: GET endpoint that returns an empty Mono with HTTP 200 status
- **Reactive Support**: Uses Spring WebFlux with `Mono<Void>` return type

## Usage Example

```java
// Configuration to enable the controller
# application.yml
openframe:
  gateway:
    internal:
      enable: true

// The probe endpoint will be available at:
// GET /internal/authz/probe
// Returns: HTTP 200 OK with empty body

// Example client call
WebClient client = WebClient.create();
Mono<Void> response = client.get()
    .uri("/internal/authz/probe")
    .retrieve()
    .bodyToMono(Void.class);
```

This controller is typically used by internal monitoring systems, load balancers, or Kubernetes health checks to verify that the authorization service is running and responsive. The conditional property allows it to be disabled in production environments where internal endpoints should not be exposed.
