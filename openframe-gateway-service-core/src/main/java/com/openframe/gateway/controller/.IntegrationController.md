REST controller that handles external tool integrations by providing health checks, testing capabilities, and HTTP request proxying for both API and agent endpoints.

## Key Components

- **Health Check Endpoint** (`GET /{toolId}/health`) - Verifies connectivity to external tools
- **Test Integration Endpoint** (`POST /{toolId}/test`) - Tests integration functionality for specific tools
- **API Proxy Endpoint** (`{toolId}/**`) - Proxies all HTTP methods to external tool APIs
- **Agent Proxy Endpoint** (`agent/{toolId}/**`) - Proxies requests specifically for agent-based tool interactions

## Usage Example

```java
// Health check for a specific tool
GET /tools/slack/health

// Test integration connectivity
POST /tools/jira/test

// Proxy API request to external tool
GET /tools/github/repos/user/project/issues
POST /tools/slack/api/chat.postMessage
{
  "channel": "#general",
  "text": "Hello from gateway"
}

// Proxy agent request
POST /tools/agent/openai/chat/completions
{
  "model": "gpt-4",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

The controller uses reactive programming with `Mono<ResponseEntity<String>>` for non-blocking operations and includes authentication support through the `Authentication` parameter.
