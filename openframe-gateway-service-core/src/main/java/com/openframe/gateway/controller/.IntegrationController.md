A reactive REST controller that provides integration management and HTTP proxy functionality for external tools. It handles health checks, connection testing, and proxying of both API and agent requests to integrated tools.

## Key Components

- **IntegrationController**: Main controller class annotated with `@RestController` and `@RequestMapping("/tools")`
- **IntegrationService**: Service dependency for testing integration connections
- **RestProxyService**: Service dependency for proxying HTTP requests to external tools

### Endpoints

- **GET** `/tools/{toolId}/health` - Health check for a specific tool integration
- **POST** `/tools/{toolId}/test` - Test connection to a specific tool integration  
- **ANY** `/tools/{toolId}/**` - Proxy API requests to external tools (supports all HTTP methods)
- **ANY** `/tools/agent/{toolId}/**` - Proxy agent-specific requests to external tools

## Usage Example

```java
// Health check request
GET /tools/slack-bot/health

// Test integration connection
POST /tools/github-api/test

// Proxy API request to external tool
GET /tools/jira/api/v2/projects
POST /tools/slack-bot/api/chat.postMessage

// Proxy agent request
POST /tools/agent/openai-assistant/chat/completions
```

The controller uses reactive programming with Spring WebFlux, returning `Mono<ResponseEntity<String>>` for all operations. Error handling is implemented using `onErrorResume` to return appropriate HTTP error responses. All requests are logged with tool IDs and paths for monitoring and debugging purposes.
