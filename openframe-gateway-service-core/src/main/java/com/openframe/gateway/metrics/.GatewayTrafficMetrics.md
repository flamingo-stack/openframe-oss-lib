Tracks and exposes real-time WebSocket connection metrics for the OpenFrame API gateway using Micrometer.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `activeWebSocketConnections` | `AtomicInteger` | Thread-safe counter for currently active WebSocket connections |
| `GatewayTrafficMetrics(MeterRegistry)` | Constructor | Registers the `openframe.tenant.gateway.websocket.connections.active` gauge with Micrometer |
| `webSocketOpened(sessionId, path, sub)` | Method | Increments the active connection counter and logs the event |
| `webSocketClosed(sessionId, path, sub)` | Method | Decrements the counter (floor of 0) and logs the event |

## Usage Example

```java
@Component
public class GatewayWebSocketHandler {

    private final GatewayTrafficMetrics metrics;

    public GatewayWebSocketHandler(GatewayTrafficMetrics metrics) {
        this.metrics = metrics;
    }

    public void afterConnectionEstablished(WebSocketSession session) {
        metrics.webSocketOpened(session.getId(), session.getUri().getPath(), getTenantSub(session));
    }

    public void afterConnectionClosed(WebSocketSession session, CloseStatus status) {
        metrics.webSocketClosed(session.getId(), session.getUri().getPath(), getTenantSub(session));
    }
}
```

## Notes

- The gauge metric is safe for concurrent access via `AtomicInteger`, preventing race conditions under high connection throughput.
- `webSocketClosed` guards against negative counts using `updateAndGet(v -> v > 0 ? v - 1 : 0)`.
- The registered metric name `openframe.tenant.gateway.websocket.connections.active` can be scraped by Prometheus or any Micrometer-compatible monitoring backend.
