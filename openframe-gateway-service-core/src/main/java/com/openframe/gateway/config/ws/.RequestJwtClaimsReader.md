Extracts and parses JWT claims from incoming HTTP/WebSocket requests in the OpenFrame API Gateway, supporting both standard `Authorization` headers and a pre-stored original authorization attribute.

## Key Components

| Method | Description |
|--------|-------------|
| `getExpiration(ServerWebExchange)` | Returns the JWT token's expiration as an `Instant` |
| `getSubject(ServerWebExchange)` | Safely returns the `sub` claim as an `Optional<String>`, returning empty on failure |
| `getClaims(ServerWebExchange)` *(private)* | Parses the unsigned JWT claims part from the `Bearer` token, applying configured clock skew tolerance |
| `extractAuthorization(ServerWebExchange)` *(private)* | Resolves the authorization string — first from `ORIGINAL_AUTHORIZATION_ATTR` (set by `ToolWebSocketProxyUrlFilter`), falling back to the raw `Authorization` header (e.g. for `/ws/nats` routes) |

## Usage Example

```java
@Component
@RequiredArgsConstructor
public class MyWebSocketHandler {

    private final RequestJwtClaimsReader jwtClaimsReader;

    public void handleExchange(ServerWebExchange exchange) {
        // Check token expiration
        Instant expiry = jwtClaimsReader.getExpiration(exchange);
        if (Instant.now().isAfter(expiry)) {
            throw new IllegalStateException("Token has expired");
        }

        // Retrieve the subject (e.g. user ID or tenant)
        String subject = jwtClaimsReader.getSubject(exchange)
                .orElseThrow(() -> new IllegalStateException("Missing subject claim"));

        // Proceed with authenticated logic
    }
}
```

> **Note:** Claims are parsed without signature verification — only the payload segment is decoded. Full signature validation is handled upstream by the Spring Security filter chain.
