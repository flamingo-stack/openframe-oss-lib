Configures a reactive `WebClient.Builder` bean with standardized timeout settings for HTTP communication within the OpenFrame gateway.

## Key Components

| Component | Description |
|-----------|-------------|
| `webClientBuilder()` | Spring `@Bean` providing a pre-configured `WebClient.Builder` |
| `HttpClient` (Reactor Netty) | Underlying HTTP client with connection, read, and write timeout handlers |
| `ReactorClientHttpConnector` | Bridges Reactor Netty's `HttpClient` with Spring's `WebClient` |

**Timeout Settings (all 30 seconds):**

| Setting | Value | Scope |
|---------|-------|-------|
| `CONNECT_TIMEOUT_MILLIS` | 30,000 ms | TCP connection establishment |
| `SO_LINGER` | 0 | Immediate socket close on disconnect |
| `responseTimeout` | 30s | Full response wait |
| `ReadTimeoutHandler` | 30s | Inbound data read |
| `WriteTimeoutHandler` | 30s | Outbound data write |

## Usage Example

Inject the builder wherever outbound HTTP calls are needed, then customize per-service as required:

```java
@Service
public class TicketingService {

    private final WebClient webClient;

    public TicketingService(WebClient.Builder webClientBuilder) {
        this.webClient = webClientBuilder
            .baseUrl("https://api.example-psa.com")
            .defaultHeader(HttpHeaders.AUTHORIZATION, "Bearer " + apiToken)
            .build();
    }

    public Mono<TicketResponse> fetchTicket(String ticketId) {
        return webClient.get()
            .uri("/tickets/{id}", ticketId)
            .retrieve()
            .bodyToMono(TicketResponse.class);
    }
}
```

> The shared `WebClient.Builder` bean ensures all outbound calls from the gateway inherit consistent timeout behavior, preventing hung connections from cascading into broader service degradation.
