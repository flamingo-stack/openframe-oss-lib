Configuration properties class for controlling WebSocket debug logging in the OpenFrame Gateway, allowing path-based filtering of WebSocket connections that should emit debug-level log output.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `debugPathPrefixes` | `List<String>` | Path prefixes that trigger debug logging; defaults to `["/ws/"]` |
| `isDebugPath(String path)` | `boolean` | Returns `true` if the given path starts with any configured debug prefix |

## Configuration

Bind values via `application.yml` under the `openframe.gateway.websocket.logging` namespace:

```yaml
openframe:
  gateway:
    websocket:
      logging:
        debug-path-prefixes:
          - /ws/
          - /internal/ws/
```

## Usage Example

```java
@Component
public class WebSocketHandler {

    private final WebSocketLoggingProperties loggingProperties;

    public WebSocketHandler(WebSocketLoggingProperties loggingProperties) {
        this.loggingProperties = loggingProperties;
    }

    public void handleConnection(String path) {
        if (loggingProperties.isDebugPath(path)) {
            log.debug("WebSocket connection established on debug path: {}", path);
        }
    }
}
```

## Notes

- `null` path or `null` prefix list both short-circuit to `false` safely
- Registered as a Spring `@Component`, so it is injectable anywhere in the application context
- Prefix matching uses `String#startsWith`, so prefix specificity matters when multiple entries are configured
