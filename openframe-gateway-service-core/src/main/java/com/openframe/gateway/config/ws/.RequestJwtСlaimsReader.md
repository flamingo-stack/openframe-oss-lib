A Spring WebFlux component that extracts and parses JWT claims from HTTP requests without signature verification, primarily used for reading token metadata like expiration times.

## Key Components

- **`getExpiration(ServerWebExchange)`** - Extracts the expiration time from JWT token as an Instant
- **`getClaims(ServerWebExchange)`** - Private method that parses JWT claims from the Authorization header
- **JWT Parsing Logic** - Extracts the claims portion of the JWT and parses it without signature validation

## Usage Example

```java
@Autowired
private RequestJwt–°laimsReader claimsReader;

public Mono<String> processRequest(ServerWebExchange exchange) {
    try {
        Instant expiration = claimsReader.getExpiration(exchange);
        
        if (expiration.isBefore(Instant.now())) {
            return Mono.error(new RuntimeException("Token expired"));
        }
        
        return Mono.just("Token valid until: " + expiration);
    } catch (IllegalStateException e) {
        return Mono.error(new RuntimeException("Invalid authorization header"));
    }
}
```

**Note:** This implementation only parses JWT claims without signature verification, making it suitable for reading metadata but not for security validation. The component expects Bearer tokens in the Authorization header and will throw `IllegalStateException` for missing or malformed headers.
