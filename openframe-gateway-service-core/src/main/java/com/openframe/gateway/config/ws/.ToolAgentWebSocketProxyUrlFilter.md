A Spring WebSocket proxy URL filter specifically designed for tool agent WebSocket connections, extending the base tool WebSocket functionality with agent-specific path parsing and endpoint configuration.

## Key Components

- **ToolAgentWebSocketProxyUrlFilter**: Main component that extends `ToolWebSocketProxyUrlFilter`
- **Constructor**: Dependency injection for tool repository, URL service, and proxy resolver
- **getRequestToolId()**: Extracts tool ID from the 5th path segment (index 4)
- **getEndpointPrefix()**: Returns the agent-specific WebSocket endpoint prefix

## Usage Example

```java
// Spring automatically registers this component
@Autowired
private ToolAgentWebSocketProxyUrlFilter toolAgentFilter;

// The filter automatically processes WebSocket URLs like:
// /ws/tools/agent/{toolId}/connect
// where getRequestToolId() extracts the toolId from position [4]

// Path parsing example:
// URL: "/ws/tools/agent/my-tool-123/connect"
// Split by '/': ["", "ws", "tools", "agent", "my-tool-123", "connect"]
// getRequestToolId() returns: "my-tool-123" (index 4)

// Endpoint prefix configuration
String prefix = filter.getEndpointPrefix(); // Returns TOOLS_AGENT_WS_ENDPOINT_PREFIX
```

This filter enables WebSocket proxy functionality for tool agents by implementing agent-specific URL parsing logic while leveraging the shared infrastructure from the parent `ToolWebSocketProxyUrlFilter` class.
