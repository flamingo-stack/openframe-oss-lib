Specializes the `ToolWebSocketProxyUrlFilter` for the tool agent WebSocket endpoint, extracting the tool ID from agent-specific URL path segments and routing connections to the appropriate upstream tool service.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `ToolAgentWebSocketProxyUrlFilter` | Constructor | Wires in the reactive tool repository and upstream resolver registry via the parent class |
| `getRequestToolId(String path)` | Method | Extracts the tool ID from path segment index `4` (e.g. `/ws/tools/agent/{toolId}/...`) |
| `getEndpointPrefix()` | Method | Returns `TOOLS_AGENT_WS_ENDPOINT_PREFIX` to scope this filter to the agent WebSocket route |

## Usage Example

This filter is auto-registered as a Spring `@Component` and participates in the gateway's WebSocket proxy pipeline. No manual instantiation is required.

```java
// Incoming agent WebSocket path handled by this filter:
// /ws/tools/agent/{toolId}/session

// getRequestToolId resolves:
String toolId = path.split("/")[4]; // extracts {toolId}

// getEndpointPrefix scopes the filter to:
// TOOLS_AGENT_WS_ENDPOINT_PREFIX = "/ws/tools/agent"
```

## Path Segment Breakdown

```text
Index:  0    1       2      3        4
Path:   "" / "ws" / "tools" / "agent" / "{toolId}" / ...
```

The parent class `ToolWebSocketProxyUrlFilter` uses both `getRequestToolId` and `getEndpointPrefix` to resolve and proxy the WebSocket connection to the correct upstream tool service registered in `ToolUpstreamResolverRegistry`.
