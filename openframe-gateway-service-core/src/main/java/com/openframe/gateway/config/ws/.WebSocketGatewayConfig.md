Configures WebSocket routing and security for the OpenFrame API Gateway, defining route mappings for tool agents, tool APIs, and NATS messaging endpoints, while wiring proxy cleanup and security decorator beans.

## Key Components

| Component | Type | Description |
|---|---|---|
| `TOOLS_AGENT_WS_ENDPOINT_PREFIX` | Constant | Route prefix `/ws/tools/agent` for tool agent WebSocket connections |
| `TOOLS_API_WS_ENDPOINT_PREFIX` | Constant | Route prefix `/ws/tools` for tool API WebSocket connections |
| `NATS_WS_ENDPOINT_PATH` | Constant | Path `/ws/nats` for direct NATS WebSocket proxying |
| `NATS_API_WS_ENDPOINT_PATH` | Constant | Path `/ws/nats-api` for NATS API WebSocket proxying |
| `customRouteLocator` | `@Bean` | Registers four Spring Cloud Gateway routes; tool routes delegate to custom proxy URL filters, NATS routes forward to the configured `nats-ws-url` with `NamespaceRewriteGatewayFilter` rewriting the host namespace per tenant on a shared pod |
| `proxyCleanupWebSocketClient` | `@Bean @Primary` | Conditionally wraps the Reactor Netty `WebSocketClient` with `ProxySessionCleanupWebSocketClient` when `openframe.gateway.websocket.proxy-cleanup.enabled=true` |
| `webSocketServiceDecorator` | `@Bean @Primary` | Wraps the default `WebSocketService` with `WebSocketServiceSecurityDecorator` to enforce JWT claims validation and record traffic metrics |

## Usage Example

```java
# application.yml
nats-ws-url: ws://nats-server:4223

openframe:
  gateway:
    websocket:
      proxy-cleanup:
        enabled: true   # wraps client with ProxySessionCleanupWebSocketClient
```

```java
// Routes registered at startup — clients connect using the defined prefixes:
// Tool agent:  ws://gateway/ws/tools/agent/{toolId}/...
// Tool API:    ws://gateway/ws/tools/{toolId}/...
// NATS direct: ws://gateway/ws/nats
// NATS API:    ws://gateway/ws/nats-api
```

> **Note:** Both `proxyCleanupWebSocketClient` and `webSocketServiceDecorator` are marked `@Primary`, overriding Spring Boot's auto-configured defaults. The cleanup client is a no-op passthrough when disabled, returning the original Reactor Netty delegate unchanged.
