Configures WebSocket routing and security for a Spring Cloud Gateway, defining endpoints for tool agents, tool APIs, and NATS messaging with JWT-based authentication.

## Key Components

- **Route Constants**: Defines WebSocket endpoint prefixes for tools agent (`/ws/tools/agent`), tools API (`/ws/tools`), and NATS (`/ws/nats`)
- **`customRouteLocator()`**: Creates routing configuration for WebSocket connections with custom filters for different tool endpoints
- **`webSocketServiceDecorator()`**: Provides a security-enhanced WebSocket service that reads JWT claims for authentication

## Usage Example

```java
// The configuration automatically sets up these WebSocket routes:

// For tool agent connections
// ws://localhost:8080/ws/tools/agent/{toolId}/...

// For tool API connections  
// ws://localhost:8080/ws/tools/{toolId}/...

// For NATS messaging
// ws://localhost:8080/ws/nats

// Example WebSocket client connection:
WebSocketClient client = new StandardWebSocketClient();
client.doHandshake(handler, "ws://localhost:8080/ws/tools/agent/my-tool/session", 
    headers).get();

// JWT authentication is automatically handled by the WebSocketServiceSecurityDecorator
// which reads claims from the request for authorization
```
