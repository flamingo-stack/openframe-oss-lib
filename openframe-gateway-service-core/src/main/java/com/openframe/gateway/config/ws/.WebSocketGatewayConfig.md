Configures WebSocket routing and security for the OpenFrame gateway, defining custom routes for tool agents, tool APIs, and NATS messaging with JWT-based authentication.

## Key Components

- **Route Constants**: Predefined WebSocket endpoint prefixes for different service types
- **customRouteLocator()**: Creates dynamic WebSocket routes with custom filters for tool proxying
- **webSocketServiceDecorator()**: Wraps the default WebSocket service with JWT security validation

## Usage Example

```java
// The configuration automatically registers these WebSocket routes:

// Agent WebSocket - routed through ToolAgentWebSocketProxyUrlFilter
// ws://gateway/ws/tools/agent/{toolId}/connect

// API WebSocket - routed through ToolApiWebSocketProxyUrlFilter  
// ws://gateway/ws/tools/{toolId}/stream

// NATS WebSocket - direct proxy to configured NATS WebSocket URL
// ws://gateway/ws/nats

// All WebSocket connections are secured with JWT validation
// through the WebSocketServiceSecurityDecorator
```

The configuration uses Spring Cloud Gateway's route builder pattern to create three distinct WebSocket routes: agent tools (with tool ID routing), API tools (with tool ID routing), and direct NATS messaging. Each route applies appropriate filtering and security through the JWT claims reader integration.
