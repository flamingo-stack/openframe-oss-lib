A WebSocket proxy filter that handles URL routing for tool API endpoints by extracting tool IDs from specific path segments and applying appropriate endpoint prefixes.

## Key Components

- **ToolApiWebSocketProxyUrlFilter**: Main filter class extending `ToolWebSocketProxyUrlFilter`
- **getRequestToolId()**: Extracts tool ID from the 4th path segment (index 3)
- **getEndpointPrefix()**: Returns the tools API WebSocket endpoint prefix constant

## Usage Example

```java
// Spring automatically instantiates this component
@Autowired
private ToolApiWebSocketProxyUrlFilter toolApiFilter;

// The filter processes WebSocket URLs like:
// /api/v1/tools/tool123/websocket -> extracts "tool123" as toolId
// Uses TOOLS_API_WS_ENDPOINT_PREFIX for routing

// Path structure expected: /segment0/segment1/segment2/toolId/...
// Example: /api/v1/tools/my-tool-id/websocket
String toolId = filter.getRequestToolId("/api/v1/tools/my-tool-id/websocket");
// Returns: "my-tool-id"
```

The filter integrates with the reactive tool repository and URL service to resolve proxy destinations for tool-specific WebSocket connections, handling the API-specific path structure where tool IDs appear in the fourth URL segment.
