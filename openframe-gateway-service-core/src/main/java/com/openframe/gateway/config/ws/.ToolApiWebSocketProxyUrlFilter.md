Handles WebSocket proxy URL filtering for tool API connections, extending the base `ToolWebSocketProxyUrlFilter` to inject API key authentication headers into proxied WebSocket requests.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `ToolApiWebSocketProxyUrlFilter` | `@Component` | Spring component extending `ToolWebSocketProxyUrlFilter` for API-authenticated WebSocket proxying |
| `apiKeyHeadersResolver` | `ToolApiKeyHeadersResolver` | Resolves API key headers for a given `IntegratedTool` |
| `getRequestToolId(String path)` | Method | Extracts the tool ID from path segment index `[3]` (e.g. `/ws/tools/api/{toolId}/...`) |
| `getEndpointPrefix()` | Method | Returns `TOOLS_API_WS_ENDPOINT_PREFIX` to match the correct WS route |
| `mutateExchange(...)` | Method | Strips the `Origin` header and injects resolved API key headers into the proxied request |

## Usage Example

```java
// Incoming WebSocket request path:
// /ws/tools/api/{toolId}/some/downstream/path

// 1. getRequestToolId extracts toolId from path[3]
String toolId = getRequestToolId("/ws/tools/api/my-tool/endpoint"); // → "my-tool"

// 2. mutateExchange strips Origin and injects API key headers
ServerWebExchange mutated = filter.mutateExchange(exchange, integratedTool);
// Resulting request headers:
// - Origin: <removed>
// - X-Api-Key: <resolved from ToolApiKeyHeadersResolver>
```

> **Note:** This filter is part of the OpenFrame gateway's WebSocket proxy layer. API key resolution is delegated to `ToolApiKeyHeadersResolver`, keeping authentication logic decoupled from routing. For related WebSocket configuration, see `WebSocketGatewayConfig.TOOLS_API_WS_ENDPOINT_PREFIX`.
