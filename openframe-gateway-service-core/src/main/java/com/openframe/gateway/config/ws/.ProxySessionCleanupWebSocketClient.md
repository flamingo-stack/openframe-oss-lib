A decorator/wrapper around a reactive `WebSocketClient` that ensures downstream proxy WebSocket sessions are properly closed after relay completion, preventing connection leaks in the OpenFrame API gateway.

## Key Components

| Component | Description |
|-----------|-------------|
| `ProxySessionCleanupWebSocketClient` | Implements `WebSocketClient`; delegates execution while adding cleanup logic |
| `delegate` | The underlying `WebSocketClient` to which actual connection handling is forwarded |
| `loggingProperties` | `WebSocketLoggingProperties` used to enable per-path debug logging |
| `wrapHandler()` | Wraps the original `WebSocketHandler` to inject a `doFinally` cleanup hook |
| `CLOSE_TIMEOUT` | 5-second timeout applied when forcefully closing a lingering session |

## Behavior

After the relay `Mono<Void>` completes (normally or via error/cancel), the `doFinally` hook checks whether the proxy session is still open:

- **Still open** → logs a warning, then calls `session.close(GOING_AWAY)` with a 5-second timeout; errors during close are swallowed silently.
- **Already closed** → logs a debug message (if the path is in the debug list) and does nothing.

## Usage Example

```java
// Bean definition in Spring configuration
@Bean
public WebSocketClient webSocketClient(
        ReactorNettyWebSocketClient nettyClient,
        WebSocketLoggingProperties loggingProps) {

    return new ProxySessionCleanupWebSocketClient(nettyClient, loggingProps);
}

// The wrapper is transparent — use it exactly like a standard WebSocketClient
webSocketClient.execute(
    URI.create("ws://backend-service/ws/events"),
    httpHeaders,
    session -> session.receive()
        .doOnNext(msg -> process(msg))
        .then()
);
// Cleanup fires automatically via doFinally when the relay ends
```
