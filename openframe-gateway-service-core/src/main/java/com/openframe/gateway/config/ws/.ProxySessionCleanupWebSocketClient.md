A decorator implementation of `WebSocketClient` that ensures downstream proxy WebSocket sessions are properly closed after relay completion, preventing session leaks in the OpenFrame API gateway.

## Key Components

- **`ProxySessionCleanupWebSocketClient`** — Implements `WebSocketClient` by wrapping a delegate client with cleanup logic
- **`delegate`** — The underlying `WebSocketClient` that performs actual connection handling
- **`loggingProperties`** — `WebSocketLoggingProperties` instance controlling per-path debug logging
- **`wrapHandler(URI, WebSocketHandler)`** — Wraps the original handler in a `doFinally` hook that forcibly closes any session still open after relay completion
- **`CLOSE_TIMEOUT`** — 5-second timeout applied to the close call to avoid hanging indefinitely

## Behavior

After the relay pipeline finishes (any termination signal), the wrapper checks `proxySession.isOpen()`:

- If still open → issues `CloseStatus.GOING_AWAY` with a 5s timeout, logs a warning, and silently swallows close errors
- If already closed → logs a debug message (when the path is debug-enabled)

## Usage Example

```java
@Bean
public WebSocketClient webSocketClient(
        ReactorNettyWebSocketClient nettyClient,
        WebSocketLoggingProperties loggingProperties) {

    return new ProxySessionCleanupWebSocketClient(nettyClient, loggingProperties);
}

// The cleanup wrapper is transparent — usage is identical to a plain WebSocketClient
webSocketClient.execute(
    URI.create("ws://backend-service/ws"),
    headers,
    session -> session.send(outbound).then()
);
```

> **Note:** The `onErrorResume` on the close call ensures a failed close attempt never propagates as an unhandled error in the reactive pipeline.
