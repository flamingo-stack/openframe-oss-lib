A Spring Boot configuration properties class that defines configurable rate limiting settings for the OpenFrame gateway, allowing customization of rate limits, error handling, and logging behavior.

## Key Components

- **Rate Limit Defaults**: Configurable per-minute, per-hour, and per-day request limits
- **Fail-Safe Behavior**: `failOpen` property to control behavior when Redis is unavailable
- **Monitoring Controls**: `logViolations` and `includeHeaders` for observability
- **Global Toggle**: `enabled` flag to disable rate limiting entirely
- **Spring Configuration**: Uses `@ConfigurationProperties` with prefix `openframe.rate-limit`

## Usage Example

**application.yml configuration:**

```java
openframe:
  rate-limit:
    enabled: true
    default-requests-per-minute: 60
    default-requests-per-hour: 1000
    default-requests-per-day: 10000
    fail-open: true
    log-violations: true
    include-headers: true
```

**Injecting properties:**

```java
@Service
public class RateLimitService {
    
    private final RateLimitProperties properties;
    
    public RateLimitService(RateLimitProperties properties) {
        this.properties = properties;
    }
    
    public boolean isRateLimitEnabled() {
        return properties.isEnabled();
    }
    
    public int getHourlyLimit() {
        return properties.getDefaultRequestsPerHour();
    }
}
```

The class uses Lombok's `@Data` annotation for automatic getter/setter generation and Spring's configuration binding to map YAML properties to Java fields.
