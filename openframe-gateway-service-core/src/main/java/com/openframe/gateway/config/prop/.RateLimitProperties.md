Configuration properties class that manages rate limiting settings for the OpenFrame gateway, allowing customization of request limits, failure handling, and logging behavior.

## Key Components

- **Rate Limit Defaults**: `defaultRequestsPerMinute`, `defaultRequestsPerHour`, `defaultRequestsPerDay` - configurable request thresholds
- **Failure Handling**: `failOpen` - controls behavior when Redis is unavailable
- **Feature Toggles**: `enabled`, `logViolations`, `includeHeaders` - control rate limiting features
- **Configuration Binding**: Uses `@ConfigurationProperties` with prefix `openframe.rate-limit`

## Usage Example

```java
// In application.yml
openframe:
  rate-limit:
    enabled: true
    default-requests-per-minute: 100
    default-requests-per-hour: 5000
    default-requests-per-day: 100000
    fail-open: true
    log-violations: true
    include-headers: true

// Injecting properties
@Autowired
private RateLimitProperties rateLimitProperties;

public void configureRateLimit() {
    if (rateLimitProperties.isEnabled()) {
        int minuteLimit = rateLimitProperties.getDefaultRequestsPerMinute();
        boolean shouldFailOpen = rateLimitProperties.isFailOpen();
        // Configure rate limiting logic
    }
}
```

The properties are automatically bound from configuration files and can be used throughout the application to control rate limiting behavior.
