A Netty channel handler that intercepts outgoing HTTP requests and logs them as executable curl commands for debugging and monitoring purposes.

## Key Components

- **CurlLoggingHandler** - Main handler class extending `ChannelDuplexHandler`
- **TARGET_URI** - Attribute key for storing target URI information
- **write()** - Core method that intercepts outgoing messages and builds curl commands
- **channelRegistered()** - Lifecycle method for channel registration logging

## Usage Example

```java
// Add to Netty pipeline for request logging
ChannelPipeline pipeline = channel.pipeline();
pipeline.addLast("curlLogger", new CurlLoggingHandler());

// Set target URI in channel attributes (optional)
channel.attr(CurlLoggingHandler.TARGET_URI).set(URI.create("https://api.example.com"));

// Handler will automatically log requests like:
// curl 'https://api.example.com/users' \
//   -X 'POST' \
//   -H 'Content-Type: application/json' \
//   -H 'Authorization: Bearer token' \
//   --data-raw '{"name":"John","email":"john@example.com"}'
```

The handler captures HTTP method, headers (excluding Host), request body, and constructs a complete curl command that can be executed directly for testing or debugging purposes.
