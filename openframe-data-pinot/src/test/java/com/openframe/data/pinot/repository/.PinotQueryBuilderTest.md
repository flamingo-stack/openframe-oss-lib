Unit test suite for `PinotQueryBuilder`, validating SQL query construction for Apache Pinot with multi-tenant isolation, input validation, and SQL injection prevention.

## Key Components

| Test Class | Coverage Area |
|---|---|
| `TenantIdValidation` | Null, empty, and blank `tenantId` rejection |
| `TenantIdEscaping` | SQL injection prevention via single-quote escaping |
| `AutoAddedTenantFilter` | Automatic `tenantId` WHERE clause injection and ordering |
| `FullQueryComposition` | SELECT, COUNT, facet, date range, IN, OR, and NOT EQUALS queries |
| `TableNameValidation` | Null/empty table name rejection and FROM clause quoting |
| `BuildStateValidation` | Missing SELECT columns and non-positive LIMIT guards |

## Usage Example

```java
// Typical builder usage exercised by these tests
String query = new PinotQueryBuilder("logs", "uuid-aaa")
    .select("id")
    .whereDateRange("eventTimestamp", LocalDate.of(2026, 1, 1), LocalDate.of(2026, 1, 31))
    .whereIn("severity", List.of("ERROR", "WARN"))
    .limit(100)
    .build();
// → SELECT id FROM "logs" WHERE tenantId = 'uuid-aaa' AND eventTimestamp ...
//   AND severity IN ('ERROR','WARN') LIMIT 100

// Facet / aggregation pattern
String facet = new PinotQueryBuilder("logs", "uuid-aaa")
    .select("status")
    .selectCount()
    .groupBy("status")
    .orderByCountDesc()
    .build();
// → SELECT status, COUNT(*) as count FROM "logs"
//   WHERE tenantId = 'uuid-aaa' GROUP BY status ORDER BY count DESC
```

## Key Behaviors Verified

- `tenantId` is **always** the first condition in the WHERE clause, even when other filters are present
- Single quotes in `tenantId` are doubled (`'` → `''`) to prevent SQL injection
- Table names are always double-quoted in the FROM clause (`FROM "logs"`)
- `build()` throws `PinotQueryException` if called with no SELECT columns or a non-positive LIMIT
