Unit test suite for `AbstractPinotRepository`, verifying the core query execution methods of the Pinot repository base class using mocked Apache Pinot client objects.

## Key Components

| Component | Description |
|---|---|
| `TestRepository` | Concrete inner subclass that exposes `AbstractPinotRepository`'s `protected` methods for direct testing |
| `ExecuteQuery` | Tests generic row mapping and exception wrapping into `PinotQueryException` |
| `ExecuteSingleColumnQuery` | Tests single-column string extraction with null/empty value filtering |
| `ExecuteKeyCountQuery` | Tests `Map<String, Integer>` aggregation results with insertion-order preservation and blank-key skipping |
| `ExecuteCountQuery` | Tests scalar count retrieval from the first result cell, including empty result handling |
| `BuildColumnIndexMap` | Tests column-name-to-index mapping from a `ResultSet` |

## Usage Example

```java
// The test class uses Mockito to simulate Pinot responses.
// Example: verifying key-count query returns ordered results with null keys skipped.

when(pinotConnection.execute(anyString())).thenReturn(resultSetGroup);
when(resultSetGroup.getResultSet(0)).thenReturn(resultSet);
when(resultSet.getRowCount()).thenReturn(2);
when(resultSet.getString(0, 0)).thenReturn("ACTIVE");
when(resultSet.getLong(0, 1)).thenReturn(100L);
when(resultSet.getString(1, 0)).thenReturn(null); // skipped

Map<String, Integer> result =
    repository.doExecuteKeyCountQuery("SELECT status, COUNT(*) FROM t GROUP BY status");

assertEquals(1, result.size());
assertEquals(100, result.get("ACTIVE"));
```

## Notes

- Uses `@ExtendWith(MockitoExtension.class)` with `@Mock` for `Connection`, `ResultSetGroup`, and `ResultSet`
- `TestRepository` is the only way to reach `protected` methods — no reflection needed
- Exception wrapping is validated: any `RuntimeException` from the Pinot connection must surface as `PinotQueryException`
- Key-count map insertion order is explicitly asserted using `List.copyOf(result.keySet())`
