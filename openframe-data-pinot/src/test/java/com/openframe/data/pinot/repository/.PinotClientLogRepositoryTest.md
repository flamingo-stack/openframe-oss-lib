Unit test suite for `PinotClientLogRepository`, verifying query construction, tenant ID propagation, sort field validation, and result-set-to-model mapping against a mocked Apache Pinot connection.

## Key Components

| Class / Member | Purpose |
|---|---|
| `TenantIdPropagation` | Nested test group asserting every public repository method injects a `tenantId = '...'` SQL filter and that null/blank tenant IDs throw `PinotQueryException` |
| `SortValidation` | Nested test group checking `isSortableField()` accepts known columns (`eventTimestamp`, `severity`), rejects unknown/null/blank values, and confirms the default sort field |
| `ResultMapping` | Nested test group verifying that `ResultSet` rows are correctly mapped to `LogProjection` and that `getOrganizationOptions` filters out null/blank `organizationId` entries |
| `whenQueryReturnsEmptyRows()` | Helper that stubs `pinotConnection.execute()` to return an empty result set, used for query-content assertions |
| `captureExecutedQuery()` | Helper using `ArgumentCaptor` to retrieve the exact SQL string passed to `pinotConnection.execute()` |
| `setField()` | Reflection utility to inject the `logsTable` value (`"logs"`) into the repository without a Spring context |

## Usage Example

```java
// Verifying tenantId is always injected into generated SQL
@Test
void findLogsIncludesTenantId() {
    whenQueryReturnsEmptyRows();
    repository.findLogs(TENANT_ID, START, END,
            List.of(), List.of(), List.of(), List.of(),
            null, null, 100, "eventTimestamp", "DESC");

    String query = captureExecutedQuery();
    assertTrue(query.contains("tenantId = 'uuid-aaa'"));
}

// Verifying result-set rows are mapped to LogProjection
List<LogProjection> logs = repository.findLogs(...);
assertEquals("evt-1",     logs.get(0).toolEventId);
assertEquals("FLEET_MDM", logs.get(0).toolType);
assertNotNull(            logs.get(0).eventTimestamp);
```
