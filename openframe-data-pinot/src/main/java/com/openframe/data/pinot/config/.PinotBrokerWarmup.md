Executes a lightweight warm-up query against the Apache Pinot broker on application startup to pre-initialize the broker connection and reduce cold-start latency for the first real query.

## Key Components

| Component | Description |
|---|---|
| `@ConditionalOnProperty` | Only activates when `pinot.broker.warmup.enabled=true` |
| `pinotConnection` | Injected `Connection` bean qualified as `pinotBrokerConnection` |
| `devicesTableName` | Table name resolved from `pinot.tables.devices.name` property |
| `warmUp()` | Event listener that fires on `ApplicationReadyEvent` and runs a `COUNT(*)` probe query |

## Usage Example

Enable the warm-up via application properties:

```yaml
pinot:
  broker:
    warmup:
      enabled: true
  tables:
    devices:
      name: devices_REALTIME
```

The warm-up fires automatically after Spring Boot finishes startup:

```java
// No manual invocation needed.
// On ApplicationReadyEvent, the following is executed internally:
pinotConnection.execute("SELECT COUNT(*) FROM \"devices_REALTIME\" LIMIT 1");
```

Failures are caught and logged as warnings (`WARN` level) so they never block application startup.

> **Note:** The warm-up is non-blocking by design. If the Pinot broker is temporarily unavailable at startup, the application will still start successfully and log a warning rather than throwing an exception.
