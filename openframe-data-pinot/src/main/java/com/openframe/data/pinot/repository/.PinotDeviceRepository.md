Repository interface for querying device data from Apache Pinot, providing multi-dimensional filter option aggregations and device count queries scoped by tenant.

## Key Components

| Method | Returns | Description |
|--------|---------|-------------|
| `getStatusFilterOptions` | `Map<String, Integer>` | Aggregates device counts grouped by status |
| `getDeviceTypeFilterOptions` | `Map<String, Integer>` | Aggregates device counts grouped by device type |
| `getOsTypeFilterOptions` | `Map<String, Integer>` | Aggregates device counts grouped by OS type |
| `getOrganizationFilterOptions` | `Map<String, Integer>` | Aggregates device counts grouped by organization |
| `getTagKeyFilterOptions` | `Map<String, Integer>` | Aggregates device counts from the tags multi-value column, grouped by tag key |
| `getFilteredDeviceCount` | `int` | Returns total device count matching all active filters |

All methods accept the same filter parameter set, enabling cross-filter faceted search (i.e., applying current selections to narrow counts in other filter dimensions).

## Usage Example

```java
// Fetch available status options while filters are already applied
Map<String, Integer> statusOptions = pinotDeviceRepository.getStatusFilterOptions(
    "tenant-abc",
    List.of(),                        // no status filter (showing all)
    List.of("laptop", "server"),      // device type filter active
    List.of("windows"),               // OS type filter active
    List.of("org-1", "org-2"),        // organization filter active
    List.of("environment"),           // tag key filter active
    List.of("environment:production") // tag key-value filter active
);
// Result: {"online": 42, "offline": 8, "warning": 3}

int total = pinotDeviceRepository.getFilteredDeviceCount(
    "tenant-abc",
    List.of(), List.of("laptop"), List.of(), List.of(), List.of(), List.of()
);
```

## Notes

- All queries are **tenant-scoped** via `tenantId` — required on every call.
- Filter parameters accept empty lists to indicate "no filter applied" for that dimension.
- Tag filtering supports two levels: `tagKeys` (key-only) and `tagKeyValues` (key:value pairs) sourced from a Pinot multi-value column.
