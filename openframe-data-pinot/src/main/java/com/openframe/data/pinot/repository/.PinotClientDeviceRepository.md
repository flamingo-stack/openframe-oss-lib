A Spring `@Repository` that queries Apache Pinot for device filter facets and counts within the OpenFrame platform, implementing `PinotDeviceRepository` by extending `AbstractPinotRepository`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `devicesTable` | `@Value` field | Configurable Pinot table name (default: `"devices"`) |
| `getStatusFilterOptions` | Method | Returns `status → count` facet map |
| `getDeviceTypeFilterOptions` | Method | Returns `deviceType → count` facet map |
| `getOsTypeFilterOptions` | Method | Returns `osType → count` facet map |
| `getOrganizationFilterOptions` | Method | Returns `organizationId → count` facet map |
| `getTagKeyFilterOptions` | Method | Returns `tagKeyValues → count` facet map |
| `getFilteredDeviceCount` | Method | Returns total device count matching all active filters |
| `executeFacetQuery` | Private | Builds and runs a `GROUP BY … COUNT(*)` facet query, excluding the facet's own filter |
| `applyDeviceFilters` | Private | Applies shared `WHERE` clauses; always excludes `DELETED` devices; skips the `excludeField` filter for self-exclusion in faceted queries |

## Usage Example

```java
// Inject the repository
@Autowired
private PinotDeviceRepository deviceRepo;

// Retrieve available status options while other filters are active
Map<String, Integer> statusOptions = deviceRepo.getStatusFilterOptions(
    "tenant-123",
    null,                          // statuses excluded (facet field)
    List.of("LAPTOP", "DESKTOP"), // deviceTypes
    List.of("WINDOWS"),           // osTypes
    List.of("org-456"),           // organizationIds
    null,                          // tagKeys
    null                           // tagKeyValues
);
// Result: {"ONLINE": 42, "OFFLINE": 7, "MAINTENANCE": 3}

// Get total matching device count
int total = deviceRepo.getFilteredDeviceCount(
    "tenant-123",
    List.of("ONLINE"),
    List.of("LAPTOP"),
    null, null, null, null
);
```

> **Note:** `DELETED` devices are always excluded from all queries regardless of the `statuses` filter input. The `excludeField` pattern enables accurate faceted filter counts — when computing options for a given facet, that facet's own active filter is intentionally omitted so all available values are returned.
