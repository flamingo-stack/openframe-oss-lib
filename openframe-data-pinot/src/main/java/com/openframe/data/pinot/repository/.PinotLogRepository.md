Spring Data-style repository interface for querying IT operations log data from Apache Pinot, supporting multi-tenant filtering, full-text search, faceted options, and cursor-based pagination.

## Key Components

| Method | Description |
|--------|-------------|
| `findLogs(...)` | Retrieves paginated log entries with multi-dimensional filters (tenant, date range, tool/event type, severity, organization, device) and cursor-based pagination |
| `searchLogs(...)` | Extends `findLogs` with a `searchTerm` parameter for full-text search across log records |
| `getToolTypeOptions(...)` | Returns distinct tool type values available given the current filter context |
| `getEventTypeOptions(...)` | Returns distinct event type values available given the current filter context |
| `getSeverityOptions(...)` | Returns distinct severity levels available given the current filter context |
| `getAvailableDateRanges(...)` | Returns valid date boundaries for a tenant and filter combination |
| `getOrganizationOptions(...)` | Returns `OrganizationOption` objects (id + label) available under the current filters |
| `isSortableField(String field)` | Validates whether a given field name supports sorting in Pinot |
| `getDefaultSortField()` | Returns the fallback sort field when none is specified by the caller |

## Usage Example

```java
@Service
public class LogQueryService {

    private final PinotLogRepository logRepository;

    public LogQueryService(PinotLogRepository logRepository) {
        this.logRepository = logRepository;
    }

    public List<LogProjection> fetchLogs(String tenantId) {
        String sortField = logRepository.isSortableField("timestamp")
                ? "timestamp"
                : logRepository.getDefaultSortField();

        return logRepository.findLogs(
                tenantId,
                LocalDate.now().minusDays(7),
                LocalDate.now(),
                List.of("RMM", "PSA"),
                List.of("ALERT", "INCIDENT"),
                List.of("HIGH", "CRITICAL"),
                List.of("org-123"),
                null,      // no device filter
                null,      // first page, no cursor
                50,
                sortField,
                "DESC"
        );
    }

    public List<String> fetchSeverities(String tenantId) {
        return logRepository.getSeverityOptions(
                tenantId,
                LocalDate.now().minusDays(30),
                LocalDate.now(),
                List.of(), List.of(), List.of(), List.of()
        );
    }
}
```

> **Note:** All query methods are scoped by `tenantId`, ensuring strict multi-tenant data isolation at the repository layer. Cursor-based pagination (`cursor` + `limit`) is used instead of offset pagination for efficient traversal of large Pinot datasets.
