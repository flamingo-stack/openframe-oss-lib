Spring `@Configuration` class that sets up Apache Pinot client connections as Spring beans, reading broker and controller URLs from application properties.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `pinotBrokerConnection()` | `@Bean` | Creates a Pinot `Connection` via `ConnectionFactory.fromProperties()`, using the `brokerList` property |
| `pinotControllerConnection()` | `@Bean` | Creates a Pinot `Connection` via `ConnectionFactory.fromHostList()` pointing to the controller URL |
| `pinot.broker.url` | `@Value` | Injected broker URL from application properties |
| `pinot.controller.url` | `@Value` | Injected controller URL from application properties |

## Usage Example

Define the required properties in your `application.yml` (or `.properties`), then inject the beans wherever Pinot queries are needed:

```yaml
pinot:
  broker:
    url: localhost:8099
  controller:
    url: localhost:9000
```

```java
@Service
public class PinotQueryService {

    private final Connection pinotBrokerConnection;

    public PinotQueryService(Connection pinotBrokerConnection) {
        this.pinotBrokerConnection = pinotBrokerConnection;
    }

    public ResultSetGroup query(String sql) {
        PreparedStatement statement = pinotBrokerConnection
            .prepareStatement("SELECT * FROM myTable WHERE status = 'OPEN'");
        return statement.execute();
    }
}
```

## Notes

- Both beans are named after their method — use `@Qualifier("pinotBrokerConnection")` or `@Qualifier("pinotControllerConnection")` if injecting both in the same class.
- The broker connection is preferred for **query execution**; the controller connection is typically used for **schema/metadata operations**.
