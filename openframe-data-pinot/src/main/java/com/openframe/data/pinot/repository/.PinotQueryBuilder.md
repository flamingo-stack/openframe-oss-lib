A fluent SQL query builder for Apache Pinot that constructs tenant-scoped queries with built-in safety validations and SQL injection prevention.

## Key Components

| Method/Field | Description |
|---|---|
| `PinotQueryBuilder(tableName, tenantId)` | Constructor — validates both inputs and automatically injects a `tenantId` filter into every query |
| `select(String... columns)` | Adds columns to the SELECT clause |
| `selectCount()` / `selectCountAll()` | Adds `COUNT(*) as count` or `COUNT(*)` to the query |
| `distinct()` | Adds DISTINCT modifier to the SELECT clause |
| `where*()` methods | Family of filter methods: `whereEquals`, `whereIn`, `whereOr`, `whereLike`, `whereDateRange`, `whereTextSearch`, `whereNotEquals`, `whereGreaterThan`, etc. |
| `whereFilterOptionsExcluding()` | Applies a map of filters while skipping a designated field (useful for faceted search) |
| `orderBy*()` methods | Ordering helpers: `orderByTimestampDesc`, `orderByTimestampAsc`, `orderByCountDesc`, `orderBySortInput` |
| `groupBy()` / `groupByWithCount()` | Adds GROUP BY columns |
| `limit(int)` | Sets query result limit (validated between `MIN_LIMIT` and `MAX_LIMIT = 10000`) |
| `build()` | Assembles and returns the final SQL string |
| `escapeSqlValue()` | Sanitizes values by escaping single quotes and backslashes |

## Usage Example

```java
String sql = new PinotQueryBuilder("events_REALTIME", tenantId)
    .select("toolEventId", "eventTimestamp", "toolName", "severity")
    .whereDateRange("eventTimestamp", LocalDate.now().minusDays(7), LocalDate.now())
    .whereIn("severity", "HIGH", "CRITICAL")
    .whereTextSearch("timeout", "message", "description")
    .orderByTimestampDesc()
    .limit(100)
    .build();
// SELECT toolEventId, eventTimestamp, toolName, severity
// FROM "events_REALTIME"
// WHERE tenantId = 'acme' AND eventTimestamp >= ... AND severity IN ('HIGH','CRITICAL')
// AND (TEXT_MATCH(message, 'timeout') OR TEXT_MATCH(description, 'timeout'))
// ORDER BY eventTimestamp DESC, toolEventId DESC
// LIMIT 100
```
