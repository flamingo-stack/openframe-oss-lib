A functional interface for resolving OAuth redirect targets with tenant-aware validation and security controls in reactive Spring applications.

## Key Components

- **`resolve()`** - Core method that validates and resolves redirect URLs based on tenant context and HTTP request details, returning a reactive `Mono<String>` for non-blocking operations

## Usage Example

```java
@Component
public class WhitelistRedirectResolver implements RedirectTargetResolver {
    
    @Override
    public Mono<String> resolve(String tenantId, String requestedRedirectTo, ServerHttpRequest request) {
        // Validate against tenant-specific whitelist
        if (isValidRedirectForTenant(tenantId, requestedRedirectTo)) {
            return Mono.just(requestedRedirectTo);
        }
        
        // Fallback to default tenant redirect
        return Mono.just(getDefaultRedirectForTenant(tenantId));
    }
    
    private boolean isValidRedirectForTenant(String tenantId, String redirectUrl) {
        // Implementation logic for validation
        return redirectUrl.startsWith("https://") && 
               whitelistedDomains.get(tenantId).contains(extractDomain(redirectUrl));
    }
}
```

This interface enables secure multi-tenant OAuth flows by allowing custom redirect target validation strategies while maintaining reactive programming patterns for high-performance applications.
