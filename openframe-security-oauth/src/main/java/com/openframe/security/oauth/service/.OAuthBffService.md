Backend-for-Frontend (BFF) service that handles the OAuth2 PKCE authorization flow for the OpenFrame gateway, including authorization URL construction, callback processing, token exchange, token refresh, and token revocation.

## Key Components

| Method | Description |
|--------|-------------|
| `buildAuthorizeRedirect` | Generates PKCE code verifier/challenge, state, and the full authorization URL for redirecting users to the auth server |
| `handleCallback` | Validates the OAuth state cookie, extracts session data, and exchanges the authorization code for tokens |
| `refreshTokensPublic` | Refreshes tokens using a known `tenantId` and refresh token |
| `refreshTokensByLookup` | Best-effort token refresh that resolves `tenantId` from MongoDB when not explicitly provided |
| `revokeRefreshToken` | Revokes a refresh token against the auth server's `/oauth2/revoke` endpoint |
| `revokeRefreshTokenByLookup` | Best-effort revocation that resolves `tenantId` from MongoDB by looking up the refresh token |
| `OAuthSessionData` | Private record holding `codeVerifier`, `tenantId`, and `redirectTo` extracted from the state cookie |

**Activation:** Only loaded when `openframe.gateway.oauth.enable=true`.

## Usage Example

```java
// Build an authorization redirect (called before sending user to IdP)
Mono<AuthorizeData> redirect = oAuthBffService.buildAuthorizeRedirect(
    "tenant-abc",
    "https://app.example.com/dashboard",
    "google",
    serverHttpRequest
);

// Handle the callback after IdP redirects back
Mono<OAuthCallbackResult> result = oAuthBffService.handleCallback(
    authorizationCode,
    stateParam,
    serverHttpRequest
);

// Refresh tokens when tenantId is unknown (e.g. from a cookie-only context)
Mono<TokenResponse> refreshed = oAuthBffService.refreshTokensByLookup(
    refreshToken,
    serverHttpRequest
);

// Revoke a refresh token without knowing the tenantId
Mono<Void> revoked = oAuthBffService.revokeRefreshTokenByLookup(refreshToken);
```
