OAuth Backend-for-Frontend (BFF) controller that handles the complete OAuth2 authorization code flow with PKCE for OpenFrame's multi-tenant security gateway. Manages authorization redirects, token exchange, refresh operations, and secure logout with cookie-based session management.

## Key Components

- **OAuth Flow Endpoints**: `/login`, `/continue`, `/callback` for standard OAuth2 PKCE flow
- **Token Management**: `/refresh` for token renewal, `/logout` for session termination
- **Development Support**: `/dev-exchange` endpoint for development token exchange via tickets
- **Security Services**: Integration with `OAuthBffService`, `CookieService`, and `OAuthDevTicketStore`
- **Configuration Properties**: Conditional enablement via `openframe.gateway.oauth.enable` property

## Usage Example

```java
// OAuth login initiation
GET /oauth/login?tenantId=acme&redirectTo=/dashboard&provider=google
// Returns 302 redirect to OAuth provider with PKCE parameters

// OAuth callback handling
GET /oauth/callback?code=auth_code&state=csrf_state
// Exchanges code for tokens, sets secure cookies, redirects to original destination

// Token refresh
POST /oauth/refresh
Cookie: refresh_token=jwt_refresh_token
// Returns new access/refresh tokens in secure cookies

// Secure logout
GET /oauth/logout?tenantId=acme
// Revokes tokens, clears all auth cookies

// Development token exchange (when dev-ticket-enabled=true)
GET /oauth/dev-exchange?ticket=dev_ticket_123
// Returns tokens as headers for development/testing
```

The controller implements a secure BFF pattern with state management via JWT cookies, CSRF protection through state parameters, and optional development tooling for local testing environments.
