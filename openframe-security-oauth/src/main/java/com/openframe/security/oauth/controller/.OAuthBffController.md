Backend-for-Frontend (BFF) OAuth controller that handles OAuth2 authentication flows including login, callback, token refresh, and logout operations for multi-tenant applications.

## Key Components

- **`/oauth/login`** - Initiates OAuth flow with provider redirect and state management
- **`/oauth/callback`** - Handles OAuth provider callback with authorization code exchange
- **`/oauth/refresh`** - Refreshes expired access tokens using refresh tokens
- **`/oauth/logout`** - Revokes tokens and clears authentication cookies
- **`/oauth/dev-exchange`** - Development-only endpoint for token exchange via tickets
- **Helper methods** - URL validation, cookie management, and error handling utilities

## Usage Example

```java
// Login initiation
GET /oauth/login?tenantId=company1&redirectTo=/dashboard&provider=google

// OAuth callback handling (automatically called by provider)
GET /oauth/callback?code=auth_code&state=encrypted_state

// Token refresh
POST /oauth/refresh?tenantId=company1
// With refresh token in cookie or Authorization header

// Logout
GET /oauth/logout?tenantId=company1

// Development token exchange (when dev mode enabled)
GET /oauth/dev-exchange?ticket=dev_ticket_123
```

The controller manages secure cookie-based authentication, supports multiple OAuth providers per tenant, and includes development-friendly features for token debugging. All endpoints handle errors gracefully with appropriate redirects and status codes.
