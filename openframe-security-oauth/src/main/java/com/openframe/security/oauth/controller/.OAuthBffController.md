Reactive Spring WebFlux REST controller implementing the Backend-for-Frontend (BFF) OAuth2 pattern for OpenFrame's gateway. It manages the full OAuth2 authorization code flow (with PKCE/state), token refresh, logout, and a dev-ticket exchange mechanism, exposing all operations under the `/oauth` path.

## Key Components

| Endpoint | Method | Description |
|---|---|---|
| `/oauth/login` | `GET` | Initiates OAuth2 login, clears SAS cookies, sets state cookie, redirects to IdP |
| `/oauth/continue` | `GET` | Same as login but skips cookie clearing — used for SSO continuation flows |
| `/oauth/callback` | `GET` | Handles IdP callback, exchanges code for tokens, sets auth cookies |
| `/oauth/refresh` | `POST` | Refreshes tokens via cookie or header; supports tenant-scoped or lookup-based refresh |
| `/oauth/logout` | `GET` | Revokes refresh token and clears auth cookies |
| `/oauth/dev-exchange` | `GET` | Consumes a one-time dev ticket to retrieve tokens as response headers |

**Key Dependencies:**
- `OAuthBffService` — core OAuth2 logic (authorize URL, callback handling, token refresh/revoke)
- `OAuthDevTicketStore` — creates/consumes short-lived dev tickets for local development token exchange
- `CookieService` — manages auth, state, and SAS cookie lifecycle

**Configuration Properties:**

| Property | Default | Description |
|---|---|---|
| `openframe.gateway.oauth.enable` | — | Must be `true` to activate this controller |
| `openframe.gateway.oauth.state-cookie-ttl-seconds` | `180` | TTL for the OAuth state cookie |
| `openframe.gateway.oauth.dev-ticket-enabled` | `true` | Enables dev-ticket flow |
| `openframe.auth.error-url` | — | Redirect target on auth failure |

## Usage Example

```java
// Trigger login flow (browser redirect)
// GET /oauth/login?tenantId=acme&redirectTo=/dashboard&provider=google

// Handle callback after IdP redirect
// GET /oauth/callback?code=abc123&state=xyz789

// Refresh tokens using refresh cookie
// POST /oauth/refresh

// Refresh tokens with explicit tenantId
// POST /oauth/refresh?tenantId=acme

// Exchange a dev ticket for token headers (local dev only)
// GET /oauth/dev-exchange?ticket=<one-time-ticket>
```

> **Note:** This controller is conditionally enabled via `@ConditionalOnProperty`. In production, `devTicketEnabled` should be set to `false` to disable the `/oauth/dev-exchange` endpoint and suppress dev token headers from refresh responses.
