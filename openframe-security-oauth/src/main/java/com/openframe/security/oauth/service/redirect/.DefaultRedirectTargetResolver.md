Default implementation of the OAuth redirect target resolution service that determines where to redirect users after authentication, with fallback logic using HTTP referer headers.

## Key Components

- **DefaultRedirectTargetResolver** - Spring component implementing `RedirectTargetResolver` interface
- **resolve()** - Main method that determines redirect target using priority-based fallback logic
- **@ConditionalOnMissingBean** - Auto-configuration annotation ensuring this bean is only created when no custom implementation exists

## Usage Example

```java
@Autowired
private RedirectTargetResolver redirectTargetResolver;

public Mono<String> handleAuthSuccess(String tenantId, String requestedTarget, ServerHttpRequest request) {
    return redirectTargetResolver.resolve(tenantId, requestedTarget, request)
        .map(target -> "redirect:" + target);
}

// Example scenarios:
// 1. Explicit redirect: resolve("tenant1", "/dashboard", request) → "/dashboard"
// 2. Referer fallback: resolve("tenant1", null, request) → referer header value
// 3. Root fallback: resolve("tenant1", "", requestWithoutReferer) → "/"
```

The resolver follows a three-tier priority system: first using the explicitly requested redirect target, then falling back to the HTTP `Referer` header if available, and finally defaulting to the root path ("/") if neither option provides a valid target. This ensures users are always redirected to a meaningful location after OAuth authentication.
