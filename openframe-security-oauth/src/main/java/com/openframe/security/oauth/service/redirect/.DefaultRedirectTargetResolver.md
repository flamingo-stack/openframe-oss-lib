Default implementation of redirect target resolution for OAuth flows. This resolver determines where users should be redirected after authentication by prioritizing explicit redirect parameters, falling back to the HTTP referer header, or defaulting to the root path.

## Key Components

- **DefaultRedirectTargetResolver** - Component implementing `RedirectTargetResolver` interface
- **resolve()** - Core method that determines redirect target based on tenant ID, requested redirect, and HTTP request context
- **@ConditionalOnMissingBean** - Auto-configuration annotation ensuring this bean is only created when no other `RedirectTargetResolver` is present

## Usage Example

```java
@Autowired
private RedirectTargetResolver redirectResolver;

public Mono<String> handleOAuthCallback(ServerHttpRequest request) {
    String tenantId = "tenant1";
    String requestedRedirect = request.getQueryParams().getFirst("redirect_uri");
    
    return redirectResolver.resolve(tenantId, requestedRedirect, request)
        .map(target -> {
            // Use resolved target for redirect
            return "redirect:" + target;
        });
}

// Custom implementation to override default behavior
@Component
public class CustomRedirectTargetResolver implements RedirectTargetResolver {
    @Override
    public Mono<String> resolve(String tenantId, String requestedRedirectTo, ServerHttpRequest request) {
        // Custom logic here will automatically replace the default
        return Mono.just("/custom-dashboard");
    }
}
```

The resolver follows a fallback strategy: requested redirect → referer header → root path ("/").
