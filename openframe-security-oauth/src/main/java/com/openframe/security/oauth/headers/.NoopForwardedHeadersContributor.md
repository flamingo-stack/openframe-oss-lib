A no-operation implementation of ForwardedHeadersContributor that serves as a default fallback when no other forwarded headers contributor is configured in the Spring Boot application.

## Key Components

- **NoopForwardedHeadersContributor** - Primary implementation that performs no operations on HTTP headers
- **contribute()** - Empty method implementation that adds no forwarded headers to requests
- **Spring Boot Annotations** - Uses `@Primary` and `@ConditionalOnMissingBean` for auto-configuration

## Usage Example

This component is automatically registered when no custom ForwardedHeadersContributor bean is present:

```java
// This bean is auto-configured - no manual setup needed
// When Spring Boot starts, it will use this no-op implementation
// unless you provide your own ForwardedHeadersContributor

// To override with custom implementation:
@Component
public class CustomForwardedHeadersContributor implements ForwardedHeadersContributor {
    @Override
    public void contribute(HttpHeaders headers, ServerHttpRequest request) {
        headers.add("X-Forwarded-Proto", "https");
        headers.add("X-Forwarded-Host", request.getHeaders().getFirst("Host"));
    }
}
```

The `@ConditionalOnMissingBean` annotation ensures this no-op version only activates when no other ForwardedHeadersContributor implementations are available, making it a safe default that won't interfere with custom header forwarding logic.
