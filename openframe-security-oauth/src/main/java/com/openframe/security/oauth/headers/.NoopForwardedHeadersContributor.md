A no-operation implementation of ForwardedHeadersContributor that serves as a default fallback when no other forwarded headers contributor is configured.

## Key Components

- **NoopForwardedHeadersContributor** - Primary component that implements ForwardedHeadersContributor interface
- **contribute()** - Empty method that performs no operations on HTTP headers
- **@ConditionalOnMissingBean** - Auto-configures only when no other ForwardedHeadersContributor exists

## Usage Example

This component is automatically registered by Spring Boot when no custom ForwardedHeadersContributor is present:

```java
// This will be auto-configured if no custom implementation exists
@Autowired
private ForwardedHeadersContributor contributor;

public void processRequest(ServerHttpRequest request) {
    HttpHeaders headers = new HttpHeaders();
    // Calls the no-op implementation - does nothing
    contributor.contribute(headers, request);
    // headers remain unchanged
}

// To override with custom implementation:
@Component
public class CustomForwardedHeadersContributor implements ForwardedHeadersContributor {
    @Override
    public void contribute(HttpHeaders headers, ServerHttpRequest request) {
        headers.add("X-Forwarded-For", request.getRemoteAddress().toString());
    }
}
```

The no-op implementation ensures the application doesn't break when forwarded headers functionality isn't needed, while allowing easy replacement with custom logic.
