A development-only interface for storing and retrieving OAuth tokens using single-use tickets for localhost token exchanges.

## Key Components

- **`createTicket(TokenResponse tokens)`** - Creates a single-use ticket and returns its ID
- **`consumeTicket(String ticketId)`** - Retrieves and removes tokens by ticket ID
- **`TokenResponse`** - DTO containing OAuth token data

## Usage Example

```java
@Service
public class OAuthCallbackService {
    
    @Autowired
    private OAuthDevTicketStore ticketStore;
    
    // After successful OAuth flow
    public Mono<String> handleCallback(String authCode) {
        return exchangeCodeForTokens(authCode)
            .flatMap(tokens -> ticketStore.createTicket(tokens))
            .doOnNext(ticketId -> 
                redirectToLocalhost("http://localhost:3000/callback?ticket=" + ticketId)
            );
    }
    
    // Frontend retrieves tokens
    public Mono<TokenResponse> getTokens(String ticketId) {
        return ticketStore.consumeTicket(ticketId);
    }
}
```

This interface enables secure token handoff in development environments by creating temporary tickets instead of exposing tokens directly in redirect URLs. The default implementation uses in-memory storage, but applications can provide custom implementations for different storage mechanisms.
