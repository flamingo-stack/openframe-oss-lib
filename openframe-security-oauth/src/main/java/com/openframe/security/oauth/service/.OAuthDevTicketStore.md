An interface for storing temporary OAuth exchange tickets during development, allowing secure token transfer between OAuth providers and localhost applications.

## Key Components

- **`createTicket(TokenResponse tokens)`** - Generates a single-use ticket ID for the provided OAuth tokens
- **`consumeTicket(String ticketId)`** - Retrieves and removes tokens associated with a ticket ID, ensuring single use
- **Development-only scope** - Designed specifically for localhost OAuth flows where direct token passing is needed

## Usage Example

```java
@Service
public class OAuthCallbackHandler {
    
    private final OAuthDevTicketStore ticketStore;
    
    public Mono<String> handleOAuthCallback(TokenResponse tokens) {
        // Store tokens and get ticket ID for redirect
        return ticketStore.createTicket(tokens)
            .map(ticketId -> "http://localhost:3000/auth/callback?ticket=" + ticketId);
    }
    
    public Mono<TokenResponse> exchangeTicket(String ticketId) {
        // Consume ticket to get tokens (single use)
        return ticketStore.consumeTicket(ticketId)
            .switchIfEmpty(Mono.error(new IllegalArgumentException("Invalid or expired ticket")));
    }
}
```

This interface enables secure OAuth flows in development environments by creating temporary, single-use tickets that bridge the gap between OAuth provider callbacks and localhost applications. The default implementation uses in-memory storage, but applications can provide custom implementations for different storage backends.
