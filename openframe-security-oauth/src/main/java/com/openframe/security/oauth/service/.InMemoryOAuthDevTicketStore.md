In-memory implementation of OAuth development ticket storage using a concurrent hash map for temporary token storage during OAuth flows.

## Key Components

- **InMemoryOAuthDevTicketStore**: Main component implementing `OAuthDevTicketStore` interface
- **createTicket(TokenResponse)**: Generates UUID ticket ID and stores token response
- **consumeTicket(String)**: Retrieves and removes token by ticket ID (one-time use)
- **ConcurrentHashMap**: Thread-safe storage backend for ticket-token pairs

## Usage Example

```java
@Autowired
private OAuthDevTicketStore ticketStore;

// Store OAuth tokens with generated ticket
TokenResponse tokens = new TokenResponse("access_token", "refresh_token");
Mono<String> ticketId = ticketStore.createTicket(tokens);

// Later, consume the ticket to retrieve tokens
ticketId.flatMap(id -> ticketStore.consumeTicket(id))
    .subscribe(retrievedTokens -> {
        // Use the tokens for OAuth operations
        System.out.println("Access token: " + retrievedTokens.getAccessToken());
    });

// Subsequent calls to consumeTicket with same ID return empty
ticketStore.consumeTicket("same-id"); // Returns Mono.empty()
```

**Note**: This implementation is automatically registered only when no other `OAuthDevTicketStore` bean exists, making it suitable for development environments where persistence isn't required.
