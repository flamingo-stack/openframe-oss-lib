A thread-safe in-memory implementation of OAuth development ticket storage that uses a concurrent hash map to temporarily store authentication tokens during the OAuth flow.

## Key Components

- **InMemoryOAuthDevTicketStore**: Main component implementing `OAuthDevTicketStore` interface
- **ConcurrentHashMap store**: Thread-safe storage for ticket ID to token mappings
- **createTicket()**: Generates unique ticket IDs and stores token responses
- **consumeTicket()**: Retrieves and removes tokens by ticket ID (one-time use)
- **@ConditionalOnMissingBean**: Auto-configures only when no other implementation exists

## Usage Example

This component is automatically registered as a Spring bean and used internally by the OAuth service:

```java
@Autowired
private OAuthDevTicketStore ticketStore;

// Create a temporary ticket for token exchange
TokenResponse tokens = new TokenResponse(accessToken, refreshToken);
Mono<String> ticketId = ticketStore.createTicket(tokens);

// Later, consume the ticket (removes from storage)
Mono<TokenResponse> retrievedTokens = ticketStore.consumeTicket(ticketId);
```

The implementation provides thread-safe operations suitable for development environments where tokens need temporary storage during OAuth callback flows. Each ticket is consumed only once, automatically removing it from memory after retrieval.
