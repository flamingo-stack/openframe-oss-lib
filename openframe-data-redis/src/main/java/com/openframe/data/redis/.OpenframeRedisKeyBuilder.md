Centralized Redis key builder that provides consistent key naming conventions with global prefixing and tenant namespacing support for the OpenFrame platform.

## Key Components

- **OpenframeRedisKeyBuilder** - Main class for building Redis keys with configurable prefixes and tenant isolation
- **tenantKey(String relativeKey)** - Builds keys under tenant namespace with mandatory tenant ID validation
- **cacheKeyPrefix(String domainPrefix, String cacheName)** - Generates Redis cache prefixes with proper formatting
- **requireTenantId()** - Validates and retrieves tenant ID, failing fast if not configured

## Key Features

- **Tenant Isolation**: Automatically namespaces keys by tenant ID for multi-tenant applications
- **Global Prefixing**: Supports environment and application-level key prefixes
- **Hash Tag Support**: Optional Redis Cluster hash tag formatting for tenant keys
- **Cache Integration**: Helper methods for Spring Cache prefix generation
- **Validation**: Strict tenant ID validation to prevent cross-tenant data leaks

## Usage Example

```java
@Configuration
public class RedisConfig {
    
    @Autowired
    private OpenframeRedisKeyBuilder keyBuilder;
    
    public void configureCache() {
        // Build tenant-scoped key
        String userKey = keyBuilder.tenantKey("user:session:123");
        // Result: "app:prod:{tenant-123}:user:session:123"
        
        // Generate cache prefix
        String cachePrefix = keyBuilder.cacheKeyPrefix("users", "sessions");
        // Result: "app:prod:{tenant-123}:users:sessions::"
        
        // Access tenant ID safely
        String tenantId = keyBuilder.requireTenantId();
    }
}
```

The builder ensures consistent key formatting while preventing tenant data isolation issues through mandatory tenant ID validation.
