A Spring Boot configuration properties class that manages Redis key namespacing and prefix settings for OpenFrame, providing consistent key naming across different services and supporting multi-tenant environments.

## Key Components

- **`OpenframeRedisProperties`** - Main configuration class annotated with `@ConfigurationProperties` for mapping `openframe.redis` properties
- **`keyPrefix`** - Global key prefix (defaults to "of") to prevent collisions with other applications sharing Redis
- **`tenantId`** - Mandatory tenant identifier for key namespacing (shared services use "shared", tenant services use actual tenant ID)
- **`tenantHashTag`** - Boolean flag to wrap tenant ID with Redis Cluster hashtag braces for optimal hash slot distribution
- **`Keys`** - Nested configuration class containing specific key prefixes for various OpenFrame services

## Usage Example

```java
@Service
public class RedisKeyService {
    
    @Autowired
    private OpenframeRedisProperties redisProperties;
    
    public String buildUserKey(String userId) {
        String prefix = redisProperties.getKeyPrefix();
        String tenantId = redisProperties.getTenantId();
        
        if (redisProperties.isTenantHashTag()) {
            return String.format("%s:{%s}:user:%s", prefix, tenantId, userId);
        } else {
            return String.format("%s:%s:user:%s", prefix, tenantId, userId);
        }
    }
    
    public String buildPasswordResetKey(String token) {
        return String.format("%s:{%s}:%s:%s", 
            redisProperties.getKeyPrefix(),
            redisProperties.getTenantId(),
            redisProperties.getKeys().getPasswordResetPrefix(),
            token);
    }
}
```

The configuration supports both single-instance and Redis Cluster deployments, with tenant-based key isolation and service-specific prefixes for organized data management.
