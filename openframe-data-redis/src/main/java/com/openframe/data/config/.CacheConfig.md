Configures Spring Boot caching with Redis backend for the OpenFrame platform. This configuration enables tenant-aware caching with automatic key prefixing and JSON serialization.

## Key Components

- **CacheConfig class**: Main configuration class that sets up Redis-backed caching
- **cacheManager()**: Bean factory method that creates a customized RedisCacheManager
- **Conditional activation**: Only activates when Redis is enabled via `spring.redis.enabled=true`
- **OpenframeRedisKeyBuilder integration**: Ensures all cache keys follow tenant-aware naming conventions

## Usage Example

The cache manager is automatically configured and can be used with Spring's caching annotations:

```java
@Service
public class UserService {
    
    @Cacheable(value = "users", key = "#userId")
    public User findById(Long userId) {
        // Database lookup - result will be cached
        return userRepository.findById(userId);
    }
    
    @CacheEvict(value = "users", key = "#user.id")
    public User updateUser(User user) {
        // Cache entry will be evicted after update
        return userRepository.save(user);
    }
}
```

Cache keys will automatically include tenant context:
- Format: `<tenant-prefix>:users::<userId>`
- TTL: 6 hours default
- Serialization: JSON for values, String for keys
- Null values are not cached to avoid storage waste
