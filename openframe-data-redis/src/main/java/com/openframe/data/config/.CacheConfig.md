Configures Spring Cache with Redis as the backing store, enabling tenant-aware cache key prefixing and per-cache TTL settings for the OpenFrame platform.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Only activates when `spring.redis.enabled=true` |
| `@ConditionalOnMissingBean` | Allows downstream services to override the `CacheManager` |
| `defaultConfig` | Global cache config with 6-hour TTL, JSON value serialization, and null-value eviction |
| `fleetCacheConfig` | Overrides TTL to 1 hour for fleet-related caches (`fleetPolicyCache`, `fleetQueryCache`) |
| `computePrefixWith` | Delegates to `OpenframeRedisKeyBuilder` to generate tenant-scoped key prefixes in the format `<prefix>:<cacheName>::<key>` |

## Usage Example

```java
// Enable Redis in application properties
// spring.redis.enabled=true

// Use Spring's @Cacheable with the configured cache names
@Service
public class FleetPolicyService {

    // Uses fleetPolicyCache — 1-hour TTL
    @Cacheable("fleetPolicyCache")
    public FleetPolicy getPolicy(String policyId) {
        return policyRepository.findById(policyId);
    }

    // Uses default cache — 6-hour TTL, tenant-aware key prefix
    @Cacheable("deviceCache")
    public Device getDevice(String deviceId) {
        return deviceRepository.findById(deviceId);
    }
}
```

> Fleet caches (`fleetPolicyCache`, `fleetQueryCache`) use a shorter 1-hour TTL since policies and queries can be renamed or deleted frequently. All other caches inherit the 6-hour default with tenant-scoped key prefixes automatically applied via `OpenframeRedisKeyBuilder`.
