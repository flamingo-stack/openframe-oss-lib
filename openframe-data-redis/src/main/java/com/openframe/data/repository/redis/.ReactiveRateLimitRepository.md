Reactive repository for rate limiting operations using Redis with atomic counter operations and window-based tracking. Provides thread-safe rate limit checks and increments for OpenFrame's reactive web applications.

## Key Components

- **ReactiveRateLimitRepository**: Main repository class for rate limiting operations
- **checkAndIncrement()**: Atomically checks rate limit and increments counter
- **getStatus()**: Retrieves current rate limit status without incrementing
- **RateLimitResult**: Result object containing limit status, counts, and window timing
- **buildRedisKey()**: Constructs tenant-aware Redis keys for rate limit data
- **getTimestamp()**: Retrieves and parses timestamps from Redis hash fields

## Usage Example

```java
@Autowired
private ReactiveRateLimitRepository rateLimitRepo;

// Check and increment rate limit
public Mono<Boolean> isRequestAllowed(String userId) {
    String window = "1h";
    String timestamp = LocalDateTime.now().truncatedTo(ChronoUnit.HOURS).toString();
    Duration ttl = Duration.ofHours(1);
    
    return rateLimitRepo.checkAndIncrement(userId, window, timestamp, 100, ttl)
        .map(ReactiveRateLimitRepository.RateLimitResult::isAllowed)
        .doOnNext(allowed -> {
            if (!allowed) {
                log.warn("Rate limit exceeded for user: {}", userId);
            }
        });
}

// Get current status without incrementing
public Mono<Long> getRemainingRequests(String apiKey) {
    return rateLimitRepo.getStatus(apiKey, "1m", getCurrentMinute(), 60)
        .map(ReactiveRateLimitRepository.RateLimitResult::getRemaining);
}
```

The repository uses Redis hash operations to store request counts, timestamps, and metadata per rate limit window, with automatic expiration and tenant-aware key prefixing.
