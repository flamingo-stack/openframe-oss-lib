A reactive Redis repository that tracks API key usage statistics with atomic increment operations for successful and failed requests.

## Key Components

- **incrementSuccessful()** - Atomically increments total and successful request counters, updates last used timestamp, and sets TTL
- **incrementFailed()** - Atomically increments total and failed request counters, updates last used timestamp, and sets TTL  
- **buildStatsKey()** - Constructs tenant-aware Redis keys using configured prefix and key builder
- **ReactiveStringRedisTemplate** - Spring Data Redis reactive template for non-blocking operations
- **OpenframeRedisKeyBuilder** - Utility for building tenant-scoped Redis keys
- **@ConditionalOnWebApplication** - Only activates in reactive web application contexts

## Usage Example

```java
@Service
public class ApiKeyStatsService {
    
    private final ReactiveApiKeyStatsRepository statsRepository;
    
    public Mono<Void> recordSuccessfulRequest(String keyId) {
        Duration ttl = Duration.ofDays(30); // 30-day retention
        return statsRepository.incrementSuccessful(keyId, ttl);
    }
    
    public Mono<Void> recordFailedRequest(String keyId) {
        Duration ttl = Duration.ofDays(30);
        return statsRepository.incrementFailed(keyId, ttl);
    }
}
```

The repository uses Redis hash operations to store statistics like total requests, successful/failed counts, and last used timestamps. All operations are atomic and reactive, making it suitable for high-throughput API key monitoring in multi-tenant environments.
