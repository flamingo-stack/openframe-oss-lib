Facade service that handles organization command operations (Create, Update, Delete) by mapping DTOs to domain entities and delegating to the core `OrganizationService`. Sits in `api-lib` to manage DTO concerns and prevent circular dependencies.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `createOrganization` | Method | Maps `CreateOrganizationRequest` DTO to entity and persists a new organization |
| `updateOrganization` | Method | Fetches existing org, applies partial updates (non-null fields only) from `UpdateOrganizationRequest`, and saves |
| `updateOrganizationStatus` | Method | Converts DTO status enum to domain `OrganizationStatus` and delegates status update |
| `organizationMapper` | Dependency | Handles DTO ↔ entity conversions |
| `organizationService` | Dependency | Core data-layer service for persistence operations |

## Usage Example

```java
// Create a new organization
CreateOrganizationRequest createReq = new CreateOrganizationRequest("Acme Corp");
Organization created = organizationCommandService.createOrganization(createReq);

// Partially update an existing organization (only non-null fields applied)
UpdateOrganizationRequest updateReq = new UpdateOrganizationRequest("Acme Corp Updated", null);
Organization updated = organizationCommandService.updateOrganization("org-123", updateReq);

// Update organization status
UpdateOrganizationStatusRequest statusReq = new UpdateOrganizationStatusRequest(StatusAction.ACTIVE);
organizationCommandService.updateOrganizationStatus("org-123", statusReq);
```

> **Note:** `updateOrganization` throws `IllegalArgumentException` if the organization ID does not exist. Callers should handle this case or let it propagate to a global exception handler.
