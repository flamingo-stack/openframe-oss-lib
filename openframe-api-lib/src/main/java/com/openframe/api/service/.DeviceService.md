Service layer class responsible for querying, filtering, and updating device (machine) records in the OpenFrame platform, with support for cursor-based pagination, tag-based filtering, and status lifecycle management.

## Key Components

| Method / Member | Description |
|---|---|
| `findByMachineId` | Looks up a single `Machine` by its unique machine ID |
| `queryDevices` | Executes a paginated, filterable, and sortable device query returning a `CountedGenericQueryResult` |
| `updateStatusByMachineId` | Updates a device's `DeviceStatus` and triggers post-processing via `DeviceStatusProcessor` |
| `resolveTagFilterToMachineIds` | Translates tag key/value filter criteria into a list of matching machine IDs via `TagAssignment` lookups |
| `buildDeviceQuery` | Composes a MongoDB `Query` from filter criteria and resolved tag constraints |
| `fetchPageItems` | Applies cursor-based page slicing using `MachineRepository.findMachinesWithCursor` |
| `buildPageInfo` | Encodes cursor values and assembles `PageInfo` for GraphQL-style pagination |
| `validateSortField` | Guards against invalid sort fields, falling back to the repository default |

## Usage Example

```java
// Query devices with filtering, pagination, and sorting
DeviceFilterCriteria filter = DeviceFilterCriteria.builder()
    .statuses(List.of(DeviceStatus.ONLINE))
    .tagKeys(List.of("environment"))
    .tagValues(List.of("production"))
    .build();

CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
    .limit(20)
    .cursor(null) // first page
    .build();

SortInput sort = new SortInput("updatedAt", SortDirection.DESC);

CountedGenericQueryResult<Machine> result =
    deviceService.queryDevices(filter, pagination, "web-server", sort);

// Update a device status
deviceService.updateStatusByMachineId("machine-abc-123", DeviceStatus.OFFLINE);
```

## Notes

- Tag filtering resolves through two repository layers (`TagRepository` → `TagAssignmentRepository`) and short-circuits to an empty result set if tag keys exist but no matches are found.
- `updateStatusByMachineId` is idempotent — it skips the write if the device already holds the target status.
- Post-processing failures in `DeviceStatusProcessor` are caught and logged without rolling back the status update.
