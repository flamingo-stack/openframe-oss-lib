Manages the full lifecycle of tags in the OpenFrame platform, providing CRUD operations, search/autocomplete support, and batch tag retrieval scoped to device entities.

## Key Components

| Method | Description |
|--------|-------------|
| `listTags()` | Returns all tenant-wide tags for the `DEVICE` entity type |
| `searchTagKeys(search, limit)` | Autocomplete search over tag keys with configurable limit (default: 20, max: 100) |
| `searchTagValues(tagKey, search, limit)` | Autocomplete search over values for a specific tag key |
| `getTagValueOptions(tagKey)` | Returns distinct tag values with device counts for filter UI dropdowns |
| `getTagsForMachines(machineIds)` | Batch-loads tags for multiple machines, preserving per-device assignment values |
| `getTagsForMachine(machineId)` | Loads tags for a single machine |
| `createTag(key, entityType, description, color)` | Creates a new tag; returns existing tag if key already exists |
| `updateTag(tagId, key, description, color)` | Partially updates tag metadata |
| `deleteTag(tagId)` | Deletes a tag and cascades removal of all its assignments |

## Usage Example

```java
// Autocomplete tag keys
List<Tag> suggestions = tagService.searchTagKeys("env", 10);

// Get filter dropdown options with device counts
List<DeviceFilterOption> options = tagService.getTagValueOptions("environment");
// → [{ value: "production", label: "production", count: 42 }, ...]

// Batch-load tags for a list of machines
List<String> machineIds = List.of("machine-1", "machine-2");
List<List<Tag>> tagsByMachine = tagService.getTagsForMachines(machineIds);

// Create a new device tag
Tag tag = tagService.createTag("environment", TagEntityType.DEVICE, "Deployment env", "#00FF00");

// Delete tag with cascade
tagService.deleteTag("tag-id-123");
```

> **Note:** All read operations are wrapped in a read-only transaction by default (`@Transactional(readOnly = true)`). Write operations (`createTag`, `updateTag`, `deleteTag`) override this with their own `@Transactional` annotation.
