Manages file attachments for Knowledge Base articles, providing batch loading via DataLoader, presigned URL generation for GCS uploads/downloads, and attachment lifecycle operations.

## Key Components

| Method | Description |
|--------|-------------|
| `getAttachmentsByArticleIds` | Batch loads attachments for multiple article IDs, preserving order for GraphQL DataLoader |
| `generateDownloadUrl` | Generates a time-limited GCS presigned download URL for a specific attachment |
| `createUploadUrl` | Creates an attachment record and returns a presigned GCS upload URL |
| `deleteAttachment` | Removes attachment metadata and deletes the file from GCS storage |
| `validateArticleExists` | Guards upload creation against non-existent articles |
| `buildAttachmentPath` | Constructs the GCS storage path using `kb-attachments/{articleId}/{fileName}` |

**Configuration:**
- `openframe.kb.presigned-url-expiration-minutes` — controls URL TTL (default: `15` minutes)

## Usage Example

```java
// Batch load attachments for DataLoader
List<List<KnowledgeBaseItemAttachment>> result =
    attachmentService.getAttachmentsByArticleIds(List.of("article-1", "article-2"));

// Generate a presigned download URL
String downloadUrl = attachmentService.generateDownloadUrl("attachment-abc123");

// Create an upload URL for a new attachment
KnowledgeBaseAttachmentUpload upload = attachmentService.createUploadUrl(
    "user-456",       // uploaderId
    "article-789",    // articleId
    "diagram.png",    // fileName
    "image/png",      // contentType
    204800L           // fileSize in bytes
);

String presignedUploadUrl = upload.getUploadUrl();
KnowledgeBaseItemAttachment savedAttachment = upload.getAttachment();

// Delete an attachment and its GCS object
attachmentService.deleteAttachment("attachment-abc123");
```
