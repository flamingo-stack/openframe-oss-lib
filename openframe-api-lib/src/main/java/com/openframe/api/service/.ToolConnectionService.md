Service layer responsible for retrieving `ToolConnection` records from the repository, supporting both single-machine and batch machine lookups optimized for GraphQL DataLoader patterns.

## Key Components

| Method | Description |
|--------|-------------|
| `findById(String id)` | Retrieves a single `ToolConnection` by its ID, returning an `Optional` |
| `getToolConnectionsForMachines(List<String> machineIds)` | Batch-fetches tool connections for multiple machines, returning results grouped and ordered to match the input list (DataLoader-compatible) |
| `getToolConnectionsForMachine(String machineId)` | Fetches all tool connections associated with a single machine ID |

## Usage Example

```java
// Single machine lookup
List<ToolConnection> connections = toolConnectionService
    .getToolConnectionsForMachine("machine-123");

// Batch lookup (DataLoader pattern)
List<String> machineIds = List.of("machine-1", "machine-2", "machine-3");
List<List<ToolConnection>> batchResults = toolConnectionService
    .getToolConnectionsForMachines(machineIds);

// batchResults.get(0) → connections for "machine-1"
// batchResults.get(1) → connections for "machine-2"
// batchResults.get(2) → connections for "machine-3"
```

## Notes

- Annotated with `@Transactional(readOnly = true)` — all operations are read-only, optimizing database performance.
- `getToolConnectionsForMachines` issues a **single batch query** (`findByMachineIdIn`) and groups results in-memory, avoiding N+1 query issues when used with a GraphQL DataLoader.
- Returns an empty `ArrayList` early when `machineIds` is empty to avoid unnecessary database calls.
