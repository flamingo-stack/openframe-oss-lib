Manages the assignment of items (such as knowledge articles) to targets (organizations, devices, tickets) within the OpenFrame platform, providing CRUD operations and cursor-based paginated querying of assignments.

## Key Components

| Method | Description |
|--------|-------------|
| `countAssignmentsByTargetType` | Returns a map of assignment counts grouped by `AssignmentTargetType` for a given item |
| `queryAssignments` | Fetches a paginated, filtered, and sorted list of `ItemAssignment` records with cursor-based navigation |
| `assignItem` | Creates and persists a new `ItemAssignment`, resolving the target's human-readable display name |
| `unassignItem` | Removes a specific assignment by item ID, target type, and target ID |
| `unassignAllByType` | Bulk-removes all assignments of a given target type from an item |
| `resolveDisplayName` | (private) Resolves a display name for the target by querying the appropriate repository |
| `validateSortField` | (private) Guards against invalid sort fields, falling back to the repository default |
| `buildPageInfo` | (private) Constructs `PageInfo` with cursor encoding for pagination metadata |

## Usage Example

```java
// Assign a knowledge article to an organization
ItemAssignment assignment = assignmentService.assignItem(
    "kb-item-123",
    AssignmentItemType.KNOWLEDGE_ARTICLE,
    AssignmentTargetType.ORGANIZATION,
    "org-456"
);

// Query assignments with cursor pagination
CountedGenericQueryResult<ItemAssignment> result = assignmentService.queryAssignments(
    "kb-item-123",
    AssignmentTargetType.ORGANIZATION,
    new CursorPaginationCriteria(10, null),
    "Acme",
    new SortInput("createdAt", SortDirection.ASC)
);

// Remove a specific assignment
assignmentService.unassignItem("kb-item-123", AssignmentTargetType.DEVICE, "device-789");

// Count assignments by target type
Map<AssignmentTargetType, Long> counts = assignmentService.countAssignmentsByTargetType("kb-item-123");
```

> **Note:** Write operations (`assignItem`, `unassignItem`, `unassignAllByType`) are explicitly `@Transactional`. The class defaults to `readOnly = true` to optimize read performance across all query methods.
