Spring service class for managing installed agents across machines in the OpenFrame system. Provides core business logic for retrieving agent information with optimized batch operations.

## Key Components

- **InstalledAgentRepository**: Data access layer dependency for agent persistence
- **getInstalledAgentsForMachines()**: Batch retrieval method that efficiently fetches agents for multiple machines
- **getInstalledAgentsForMachine()**: Single machine agent lookup
- **getAllInstalledAgents()**: Retrieves all agents in the system
- **getInstalledAgent()**: Finds agent by unique ID
- **getInstalledAgentByMachineIdAndType()**: Locates specific agent type on a machine

## Usage Example

```java
@Autowired
private InstalledAgentService agentService;

// Get agents for multiple machines (optimized for DataLoader)
List<String> machineIds = Arrays.asList("machine1", "machine2", "machine3");
List<List<InstalledAgent>> agentsByMachine = agentService.getInstalledAgentsForMachines(machineIds);

// Get agents for a single machine
List<InstalledAgent> agents = agentService.getInstalledAgentsForMachine("machine1");

// Find specific agent type
Optional<InstalledAgent> agent = agentService.getInstalledAgentByMachineIdAndType("machine1", "monitoring");
```

The service uses stream operations to efficiently group and map agents by machine ID, making it particularly useful for bulk operations in data loading scenarios.
