Lifecycle listener that automatically stamps the `publishedAt` timestamp on a `KnowledgeBaseItem` the first time it transitions to `PUBLISHED` status, before MongoDB persistence occurs.

## Key Components

| Element | Description |
|---|---|
| `KnowledgeBasePublishLifecycleListener` | Spring `@Component` extending `AbstractMongoEventListener<KnowledgeBaseItem>` |
| `onBeforeConvert()` | Intercepts MongoDB pre-save events to conditionally set `publishedAt` |
| Idempotency guard | Only stamps `publishedAt` when it is `null`, preserving the original publication moment across subsequent status changes |

## Usage Example

```java
// No manual wiring required — Spring picks up the @Component automatically.
// The timestamp is set transparently before any save operation:

KnowledgeBaseItem article = new KnowledgeBaseItem();
article.setTitle("Getting Started");
article.setStatus(KnowledgeBaseArticleStatus.DRAFT);
repository.save(article); // publishedAt remains null

article.setStatus(KnowledgeBaseArticleStatus.PUBLISHED);
repository.save(article); // publishedAt = Instant.now() (stamped by listener)

article.setStatus(KnowledgeBaseArticleStatus.ARCHIVED);
repository.save(article); // publishedAt unchanged — original timestamp preserved

article.setStatus(KnowledgeBaseArticleStatus.PUBLISHED);
repository.save(article); // publishedAt still unchanged — idempotent
```

## Behavior Notes

- Follows the **"first published at"** semantic aligned with [Schema.org `datePublished`](https://schema.org/datePublished) and Atom RFC 4287 `atom:published`
- Operates at the `BeforeConvert` lifecycle phase, ensuring the timestamp is written to the document on the same save that triggers the status change
- Stateless and thread-safe — no mutable fields
