Service class that provides comprehensive log querying, filtering, and detailed retrieval capabilities using both Pinot and Cassandra data sources for optimized log analytics and audit trail functionality.

## Key Components

- **queryLogs()** - Main log querying method with filtering, pagination, and search capabilities
- **findLogDetails()** - Retrieves detailed log information by composite key from Cassandra
- **getLogFilters()** - Provides available filter options (tool types, event types, severities, organizations)
- **buildLogQueryResult()** - Constructs paginated query results with cursor-based pagination
- **Mapping methods** - Convert between data projections and DTOs (mapToLogEvent, mapToLogDetails)

## Usage Example

```java
@Autowired
private LogService logService;

// Query logs with filters and pagination
LogFilterOptions filters = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .toolTypes(List.of("SECURITY", "SYSTEM"))
    .severities(List.of("ERROR", "WARN"))
    .build();

CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
    .limit(50)
    .build();

GenericQueryResult<LogEvent> result = logService.queryLogs(filters, pagination, "authentication");

// Get detailed log information
Optional<LogDetails> details = logService.findLogDetails(
    "2024-01-15", 
    "SECURITY", 
    "LOGIN_ATTEMPT", 
    Instant.now(), 
    "evt_12345"
);

// Retrieve available filter options
LogFilters availableFilters = logService.getLogFilters(filters);
```
