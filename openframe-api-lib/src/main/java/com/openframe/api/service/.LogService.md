A Spring service that provides comprehensive log querying, filtering, and detail retrieval capabilities using both Pinot and Cassandra data sources for high-performance log analytics.

## Key Components

**Main Methods:**
- `queryLogs()` - Searches and filters logs with cursor pagination, supporting both full-text search and exact field filtering
- `findLogDetails()` - Retrieves detailed log information by composite key from Cassandra
- `getLogFilters()` - Provides available filter options (tool types, event types, severities, organizations)

**Helper Methods:**
- `buildLogQueryResult()` - Constructs paginated query results with cursor information
- `createLogCursor()` - Generates cursor strings for pagination using timestamp and tool event ID
- `mapToLogEvent()` - Converts Pinot projections to API log events
- `mapToLogDetails()` - Converts Cassandra entities to detailed log objects

**Dependencies:**
- `PinotLogRepository` - High-performance log querying and aggregation
- `UnifiedLogEventRepository` - Detailed log storage and retrieval

## Usage Example

```java
// Query logs with filters and pagination
LogFilterOptions filters = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .toolTypes(List.of("security-scanner", "backup"))
    .severities(List.of("ERROR", "WARN"))
    .build();

CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
    .limit(50)
    .cursor("1234567890_event123")
    .build();

GenericQueryResult<LogEvent> results = logService.queryLogs(filters, pagination, "failed authentication");

// Get detailed log information
Optional<LogDetails> details = logService.findLogDetails(
    "2024-01-15", 
    "security-scanner", 
    "login-failure",
    Instant.now(),
    "event123"
);

// Retrieve available filter options
LogFilters availableFilters = logService.getLogFilters(filters);
```
