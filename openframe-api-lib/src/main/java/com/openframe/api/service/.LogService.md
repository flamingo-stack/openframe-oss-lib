A service class that provides comprehensive log querying, filtering, and retrieval capabilities using both Pinot and Cassandra data stores for audit log management.

## Key Components

- **`queryLogs()`** - Performs paginated log queries with filtering and optional search functionality
- **`findLogDetails()`** - Retrieves detailed log information by composite key from Cassandra
- **`getLogFilters()`** - Returns available filter options (tool types, event types, severities, organizations)
- **`buildLogQueryResult()`** - Constructs paginated query results with cursor-based navigation
- **Mapping methods** - Convert between data projections and DTOs (`mapToLogEvent`, `mapToLogDetails`)

## Usage Example

```java
@Autowired
private LogService logService;

// Query logs with filters and pagination
LogFilterOptions filters = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .toolTypes(List.of("security", "application"))
    .severities(List.of("ERROR", "WARN"))
    .build();

CursorPaginationCriteria pagination = new CursorPaginationCriteria(null, 50);
GenericQueryResult<LogEvent> results = logService.queryLogs(filters, pagination, "failed login");

// Get detailed log information
Optional<LogDetails> details = logService.findLogDetails(
    "2024-01-15", 
    "security", 
    "authentication", 
    Instant.now(), 
    "evt-12345"
);

// Retrieve available filter options
LogFilters availableFilters = logService.getLogFilters(filters);
```
