Provides read-only query operations for retrieving and searching `Ticket` documents in MongoDB, acting as the service layer between controllers and the `TicketRepository`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `findById` | Method | Fetches a single ticket by its ID, returning an `Optional<Ticket>` |
| `searchTickets` | Method | Queries tickets using a `TicketQueryFilter`, optional search string, and result limit; results are sorted by `createdAt` descending |
| `SORT_FIELD_CREATED_AT` | Constant | MongoDB sort field (`"createdAt"`) used for default ordering |
| `SORT_DIRECTION_DESC` | Constant | Default sort direction (`"DESC"`) applied to all search results |

## Usage Example

```java
// Inject the service
@Autowired
private TicketQueryService ticketQueryService;

// Find a ticket by ID
Optional<Ticket> ticket = ticketQueryService.findById("ticket-123");
ticket.ifPresent(t -> System.out.println(t.getTitle()));

// Search tickets with a filter
TicketQueryFilter filter = new TicketQueryFilter();
filter.setStatus("OPEN");
filter.setAssigneeId("tech-456");

List<Ticket> results = ticketQueryService.searchTickets(filter, "network outage", 25);
```

## Notes

- Annotated with `@Transactional(readOnly = true)` — all operations are non-mutating and optimized for read performance.
- Cursor-based pagination is partially implemented; full pagination and sorting support is planned when the PSA module migrates to the OSS stack.
- Delegates query construction to `TicketRepository.buildTicketQuery()` and result fetching to `findTicketsWithCursor()`.
