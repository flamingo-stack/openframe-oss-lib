A default fallback implementation of the DeviceStatusProcessor interface that provides basic logging for device status changes when no custom processor is configured.

## Key Components

- **DefaultDeviceStatusProcessor**: Default implementation that logs device status updates
- **postProcessStatusUpdated()**: Method that logs machine ID and status changes at debug level
- **@ConditionalOnMissingBean**: Spring annotation ensuring this bean is only active when no other DeviceStatusProcessor implementation exists
- **DeviceStatusProcessor**: Interface implemented for handling device status post-processing

## Usage Example

This processor automatically activates when no custom implementation is provided:

```java
// Custom implementation would override this default
@Component
public class CustomDeviceStatusProcessor implements DeviceStatusProcessor {
    @Override
    public void postProcessStatusUpdated(Machine machine) {
        // Custom logic here - email alerts, webhooks, etc.
        notifyAdministrators(machine);
    }
}

// When Machine status is updated, the processor is called
Machine machine = new Machine();
machine.setMachineId("SERVER-001");
machine.setStatus("OFFLINE");
// DefaultDeviceStatusProcessor logs: "Device status updated: SERVER-001, new status: OFFLINE"
```

The default processor serves as a safe fallback that prevents errors while providing basic observability through debug logging. Custom implementations can replace it for specific business requirements like notifications, integrations, or advanced monitoring workflows.
