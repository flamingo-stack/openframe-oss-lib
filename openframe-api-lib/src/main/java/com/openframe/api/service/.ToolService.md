Service layer responsible for querying and filtering integrated tools registered in the OpenFrame platform, delegating persistence operations to `IntegratedToolRepository`.

## Key Components

| Method | Description |
|--------|-------------|
| `findById(String id)` | Retrieves a single `IntegratedTool` by its ID, returning an `Optional` |
| `queryTools(...)` | Fetches a filtered, searched, and sorted list of tools wrapped in a `ToolList` |
| `getToolFilters()` | Returns distinct filter values (types, categories, platform categories) available for the tool catalog |
| `buildQueryFilter(...)` | Private helper that maps `ToolFilterCriteria` to the repository-level `ToolQueryFilter` |
| `validateSortField(...)` | Private helper that sanitizes and validates the requested sort field, falling back to the repository default |

## Usage Example

```java
// Inject ToolService via Spring DI
@Autowired
private ToolService toolService;

// Query tools with filters, search, and sort
ToolFilterCriteria criteria = ToolFilterCriteria.builder()
    .enabled(true)
    .type("monitoring")
    .category("infrastructure")
    .build();

SortInput sort = SortInput.builder()
    .field("name")
    .direction(SortDirection.ASC)
    .build();

ToolList results = toolService.queryTools(criteria, "grafana", sort);

// Retrieve available filter options for UI dropdowns
ToolFilters filters = toolService.getToolFilters();

// Look up a specific tool by ID
Optional<IntegratedTool> tool = toolService.findById("tool-abc123");
tool.ifPresent(t -> log.info("Found tool: {}", t.getName()));
```

## Notes

- Invalid or blank sort fields are gracefully handled — a warning is logged and the repository's default sort field is applied automatically.
- A `null` `ToolFilterCriteria` argument to `queryTools` produces an empty (unfiltered) `ToolQueryFilter` rather than throwing an exception.
