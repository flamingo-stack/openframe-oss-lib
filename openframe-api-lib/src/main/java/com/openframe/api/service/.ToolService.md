A Spring service that manages MSP tool querying and filtering operations in the OpenFrame platform, providing search, sorting, and metadata retrieval functionality for integrated tools.

## Key Components

- **`queryTools()`** - Main query method that accepts filter options, search terms, and sorting parameters to retrieve filtered tool lists
- **`getToolFilters()`** - Returns available filter options including types, categories, and platform categories
- **`buildQueryFilter()`** - Private helper that converts DTO filter options into repository query filters
- **`validateSortField()`** - Ensures requested sort fields are valid and falls back to defaults if invalid

## Usage Example

```java
@Autowired
private ToolService toolService;

public void queryToolsExample() {
    // Create filter options
    ToolFilterOptions filters = ToolFilterOptions.builder()
            .enabled(true)
            .type("monitoring")
            .category("infrastructure")
            .build();
    
    // Set up sorting
    SortInput sort = SortInput.builder()
            .field("name")
            .direction(SortDirection.ASC)
            .build();
    
    // Query tools with filters
    ToolList results = toolService.queryTools(filters, "prometheus", sort);
    
    // Get available filter options
    ToolFilters availableFilters = toolService.getToolFilters();
    List<String> toolTypes = availableFilters.getTypes();
}
```

The service integrates with `IntegratedToolRepository` to perform database operations and includes validation for sort fields with fallback to repository defaults. All operations are logged for debugging purposes.
