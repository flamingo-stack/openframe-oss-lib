Provides MongoDB-backed querying of `Organization` documents with support for filtering, cursor-based pagination, full-text search, and dynamic sorting.

## Key Components

| Method | Description |
|---|---|
| `findByOrganizationId(String)` | Looks up a single organization by its ID, returning an `Optional<Organization>` |
| `queryOrganizations(...)` | Main query entry point; accepts filter options, pagination, search string, and sort input; returns a `CountedGenericQueryResult` |
| `fetchPageItems(...)` | Executes the paginated MongoDB query, fetching one extra item to detect next-page availability |
| `buildPageInfo(...)` | Constructs `PageInfo` with cursor values and `hasNextPage`/`hasPreviousPage` flags |
| `buildQueryFilter(...)` | Maps `OrganizationFilterOptions` to the internal `OrganizationQueryFilter` used at the MongoDB level |
| `validateSortField(...)` | Guards against invalid sort fields by falling back to the repository's default sort field |

## Usage Example

```java
// Inject service
@Autowired
private OrganizationQueryService organizationQueryService;

// Build filter options
OrganizationFilterOptions filters = OrganizationFilterOptions.builder()
        .status("ACTIVE")
        .hasActiveContract(true)
        .minEmployees(10)
        .build();

// Build pagination and sort
CursorPaginationCriteria pagination = CursorPaginationCriteria.of(null, 20);
SortInput sort = SortInput.of("name", SortDirection.ASC);

// Execute query
CountedGenericQueryResult<Organization> result =
        organizationQueryService.queryOrganizations(filters, pagination, "Acme", sort);

List<Organization> orgs = result.getItems();
PageInfo pageInfo = result.getPageInfo();
String nextCursor = pageInfo.getEndCursor();
```

> **Note:** All filtering is applied at the MongoDB query level via `OrganizationRepository.buildOrganizationQuery()`, avoiding in-memory filtering for better performance on large datasets.
