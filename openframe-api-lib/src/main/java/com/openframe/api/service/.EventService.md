A comprehensive service class that manages event operations in the OpenFrame API platform, providing CRUD operations, query functionality with cursor-based pagination, and filtering capabilities for event data stored in MongoDB.

## Key Components

**Main Service Methods:**
- `queryEvents()` - Executes complex event queries with filtering, pagination, search, and sorting
- `findById()` - Retrieves a single event by its unique identifier
- `createEvent()` - Creates new events with automatic ID and timestamp assignment
- `updateEvent()` - Updates existing events with validation
- `getEventFilters()` - Returns available filter options (user IDs and event types)

**Private Helper Methods:**
- `fetchPageItems()` - Handles cursor-based pagination logic
- `buildQueryFilter()` - Constructs MongoDB query filters from filter options
- `buildPageInfo()` - Creates pagination metadata for query results
- `validateSortField()` - Ensures sort fields are valid and supported

## Usage Example

```java
@Autowired
private EventService eventService;

// Query events with filters and pagination
EventFilterOptions filters = EventFilterOptions.builder()
    .userIds(List.of("user123"))
    .eventTypes(List.of("LOGIN", "LOGOUT"))
    .startDate(Instant.now().minus(7, ChronoUnit.DAYS))
    .build();

CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
    .limit(20)
    .build();

SortInput sort = SortInput.builder()
    .field("timestamp")
    .direction(SortDirection.DESC)
    .build();

GenericQueryResult<Event> result = eventService.queryEvents(filters, pagination, "search term", sort);

// Create a new event
Event newEvent = Event.builder()
    .eventType("USER_ACTION")
    .userId("user456")
    .data(Map.of("action", "click", "element", "button"))
    .build();

Event savedEvent = eventService.createEvent(newEvent);
```
