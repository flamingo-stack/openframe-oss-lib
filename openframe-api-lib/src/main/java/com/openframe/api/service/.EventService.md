Service layer component that handles event management operations including querying, creating, updating, and filtering events with cursor-based pagination support. Acts as the business logic layer between the API controllers and the data repository.

## Key Components

- **queryEvents()** - Executes filtered event queries with cursor pagination and search capabilities
- **findById()** - Retrieves a specific event by its unique identifier
- **createEvent()** - Creates new events with auto-generated IDs and timestamps
- **updateEvent()** - Updates existing events with validation
- **getEventFilters()** - Provides available filter options (user IDs and event types)
- **Helper methods** - Private utilities for pagination, query building, and page info construction

## Usage Example

```java
@Autowired
private EventService eventService;

// Query events with filters and pagination
EventFilterOptions filters = EventFilterOptions.builder()
    .eventTypes(List.of("LOGIN", "LOGOUT"))
    .startDate(Instant.now().minus(7, ChronoUnit.DAYS))
    .build();
    
CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
    .limit(20)
    .cursor("cursor-token")
    .build();

GenericQueryResult<Event> results = eventService.queryEvents(filters, pagination, "search-term");

// Create a new event
Event newEvent = Event.builder()
    .type("USER_ACTION")
    .userId("user-123")
    .data(Map.of("action", "click"))
    .build();
    
Event savedEvent = eventService.createEvent(newEvent);

// Get available filter options
EventFilters availableFilters = eventService.getEventFilters();
```
