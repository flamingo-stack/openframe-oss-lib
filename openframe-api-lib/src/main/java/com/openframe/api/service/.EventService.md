Service layer for managing event lifecycle operations including querying with cursor-based pagination, CRUD operations, and filter discovery against a MongoDB-backed event repository.

## Key Components

| Method | Description |
|--------|-------------|
| `queryEvents()` | Paginated event query with filtering, full-text search, and sort support |
| `findById()` | Retrieves a single event by its string ID |
| `createEvent()` | Persists a new event, auto-assigning UUID and timestamp |
| `updateEvent()` | Updates an existing event, throwing `RuntimeException` if not found |
| `getEventFilters()` | Returns distinct user IDs and event types for filter UI population |
| `fetchPageItems()` | Private helper implementing cursor-based page slicing |
| `buildQueryFilter()` | Maps `EventFilterCriteria` DTO to the internal `EventQueryFilter` |
| `buildPageInfo()` | Constructs `PageInfo` with encoded start/end cursors |
| `validateSortField()` | Guards against unsupported sort fields, falling back to repository default |

## Usage Example

```java
// Query events with filters, cursor pagination, and sorting
EventFilterCriteria filters = EventFilterCriteria.builder()
        .eventTypes(List.of("LOGIN", "LOGOUT"))
        .startDate(Instant.parse("2024-01-01T00:00:00Z"))
        .build();

CursorPaginationCriteria pagination = CursorPaginationCriteria.builder()
        .cursor("eyJpZCI6IjEyMyJ9")
        .limit(20)
        .build();

SortInput sort = SortInput.builder()
        .field("timestamp")
        .direction(SortDirection.DESC)
        .build();

GenericQueryResult<Event> result = eventService.queryEvents(filters, pagination, "admin", sort);
List<Event> events = result.getItems();
PageInfo pageInfo = result.getPageInfo();

// Create a new event
Event newEvent = new Event();
newEvent.setEventType("USER_ACTION");
newEvent.setUserId("user-456");
Event created = eventService.createEvent(newEvent);

// Retrieve available filter options
EventFilters availableFilters = eventService.getEventFilters();
```

> **Note:** Sort field validation is enforced via `EventRepository.isSortableField()`. Invalid fields fall back to the repository default silently with a `WARN` log. Community support is available at [OpenMSP Slack](https://join.slack.com/t/openmsp/shared_invite/zt-36bl7mx0h-3~U2nFH6nqHqoTPXMaHEHA).
