Provides device filtering capabilities by retrieving filter options with counts from a Pinot database and enriching organization data with human-readable names. Uses asynchronous processing to efficiently fetch multiple filter categories simultaneously.

## Key Components

- **`getDeviceFilters(DeviceFilterOptions)`** - Main method that returns `CompletableFuture<DeviceFilters>` with all available filter options and counts
- **`convertMapToFilterOptions(Map<String, Integer>)`** - Converts repository results to standard filter options
- **`convertMapToTagFilterOptions(Map<String, Integer>)`** - Specialized converter for tag filter options
- **`convertMapToOrganizationFilterOptions(Map<String, Integer>)`** - Enriches organization IDs with names from MongoDB

## Usage Example

```java
@Autowired
private DeviceFilterService deviceFilterService;

// Get all available filters
CompletableFuture<DeviceFilters> filters = deviceFilterService.getDeviceFilters(null);

// Get filters based on current selections
DeviceFilterOptions options = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE))
    .deviceTypes(List.of(DeviceType.MOBILE))
    .organizationIds(List.of("org-123"))
    .build();

CompletableFuture<DeviceFilters> filteredResults = deviceFilterService.getDeviceFilters(options);

// Access results
DeviceFilters result = filteredResults.join();
List<DeviceFilterOption> statusOptions = result.getStatuses();
Integer totalCount = result.getFilteredCount();
```
