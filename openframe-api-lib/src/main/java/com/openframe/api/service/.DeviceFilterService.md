A Spring service that provides device filtering capabilities by aggregating filter options and counts from multiple data sources asynchronously.

## Key Components

- **getDeviceFilters()** - Main method that returns `CompletableFuture<DeviceFilters>` with all available filter options
- **convertMapToFilterOptions()** - Converts repository data to standard filter options
- **convertMapToTagFilterOptions()** - Specialized converter for tag-based filters  
- **convertMapToOrganizationFilterOptions()** - Converts organization IDs to filter options with readable names
- **PinotDeviceRepository** - Data access layer for device metrics
- **OrganizationRepository** - Repository for organization name lookups

## Usage Example

```java
@Autowired
private DeviceFilterService deviceFilterService;

public void getFilters() {
    // Create filter options
    DeviceFilterOptions options = DeviceFilterOptions.builder()
        .statuses(List.of(DeviceStatus.ACTIVE))
        .organizationIds(List.of("org-123"))
        .build();
    
    // Get filter data asynchronously
    CompletableFuture<DeviceFilters> filtersFuture = 
        deviceFilterService.getDeviceFilters(options);
    
    // Process results
    filtersFuture.thenAccept(filters -> {
        System.out.println("Filtered device count: " + filters.getFilteredCount());
        System.out.println("Available statuses: " + filters.getStatuses().size());
    });
}
```

The service uses parallel CompletableFuture execution to efficiently fetch filter counts for statuses, device types, OS types, organizations, and tags simultaneously.
