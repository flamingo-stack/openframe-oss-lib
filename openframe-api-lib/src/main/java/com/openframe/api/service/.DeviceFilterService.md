Provides device filter options by executing parallel async queries against Apache Pinot, resolving organization names from MongoDB, and assembling structured filter facets for the device list UI.

## Key Components

| Method | Description |
|--------|-------------|
| `getDeviceFilters(DeviceFilterCriteria)` | Fires six concurrent `CompletableFuture` queries (statuses, device types, OS types, organizations, tag keys, filtered count) and joins them into a `DeviceFilters` response |
| `buildTagKeyValuesFilter(DeviceFilterCriteria)` | Constructs `"key:value"` pair strings for Pinot tag filtering; builds a cross-product when both keys and values are present |
| `convertMapToFilterOptions(Map)` | Maps a `String → Integer` count map into a list of `DeviceFilterOption` (value, label, count) |
| `convertMapToTagFilterOptions(Map)` | Parses `"key:value"` Pinot entries into typed `TagFilterOption` objects, splitting on the first colon |
| `convertMapToOrganizationFilterOptions(Map)` | Batch-loads only the organizations present in the filter result from MongoDB and substitutes their names as display labels |

## Usage Example

```java
// Inject the service
@Autowired
private DeviceFilterService deviceFilterService;

// Build criteria with active filters
DeviceFilterCriteria criteria = DeviceFilterCriteria.builder()
        .statuses(List.of(DeviceStatus.ONLINE))
        .organizationIds(List.of("org-123", "org-456"))
        .tagKeys(List.of("env"))
        .tagValues(List.of("production"))
        .build();

// Fetch filter facets asynchronously
CompletableFuture<DeviceFilters> future = deviceFilterService.getDeviceFilters(criteria);

future.thenAccept(filters -> {
    System.out.println("Matched devices: " + filters.getFilteredCount());
    filters.getStatuses().forEach(s ->
            System.out.println(s.getLabel() + ": " + s.getCount()));
});
```

> **Note:** All six Pinot queries run concurrently via `CompletableFuture.allOf`, minimizing total latency. Organization name resolution is handled with a targeted MongoDB batch lookup rather than a full collection scan or cache layer.
