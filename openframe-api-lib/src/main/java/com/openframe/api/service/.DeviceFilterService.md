A Spring service that provides asynchronous device filtering capabilities, retrieving filter options with counts for device management interfaces.

## Key Components

- **getDeviceFilters()** - Main method that asynchronously fetches all device filter options (statuses, types, OS, organizations, tags) with their counts
- **convertMapToFilterOptions()** - Converts repository data maps to standard filter option DTOs
- **convertMapToTagFilterOptions()** - Specialized converter for tag-based filter options  
- **convertMapToOrganizationFilterOptions()** - Enhanced converter that enriches organization IDs with human-readable organization names

## Usage Example

```java
@Autowired
private DeviceFilterService deviceFilterService;

// Get all available filter options
DeviceFilterOptions filters = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE))
    .deviceTypes(List.of(DeviceType.MOBILE))
    .build();

CompletableFuture<DeviceFilters> filtersFuture = deviceFilterService.getDeviceFilters(filters);

DeviceFilters deviceFilters = filtersFuture.get();
// Access filter options with counts
List<DeviceFilterOption> statuses = deviceFilters.getStatuses();
List<TagFilterOption> tags = deviceFilters.getTags();
int totalFilteredDevices = deviceFilters.getFilteredCount();
```

The service leverages asynchronous processing to fetch multiple filter types concurrently from Apache Pinot, significantly improving response times for device management dashboards.
