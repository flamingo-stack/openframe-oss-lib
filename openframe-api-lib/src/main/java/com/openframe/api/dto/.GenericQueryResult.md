A generic container class that wraps query results with pagination metadata, providing a standardized response format for paginated API endpoints.

## Key Components

- **`items`** - List of generic type T containing the actual query results
- **`pageInfo`** - CursorPageInfo object containing pagination metadata (cursors, hasNext, etc.)
- **Generic Type T** - Allows reuse with any data type while maintaining type safety

## Usage Example

```java
// For user queries
GenericQueryResult<User> userResult = GenericQueryResult.<User>builder()
    .items(userList)
    .pageInfo(CursorPageInfo.builder()
        .hasNext(true)
        .startCursor("cursor123")
        .endCursor("cursor456")
        .build())
    .build();

// For product queries  
GenericQueryResult<Product> productResult = GenericQueryResult.<Product>builder()
    .items(productList)
    .pageInfo(pageInfo)
    .build();

// Accessing results
List<User> users = userResult.getItems();
boolean hasMorePages = userResult.getPageInfo().isHasNext();
String nextCursor = userResult.getPageInfo().getEndCursor();
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern, making it easy to create consistent paginated responses across different entity types.
