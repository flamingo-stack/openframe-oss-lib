A DTO (Data Transfer Object) used to encapsulate filter parameters when querying knowledge base items in the OpenFrame API.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `parentId` | `String` | Filters items by their parent node ID |
| `type` | `KnowledgeBaseItemType` | Filters by item type (e.g., folder, article) |
| `tagIds` | `List<String>` | Filters items matching any of the provided tag IDs |
| `statuses` | `List<KnowledgeBaseArticleStatus>` | Filters articles by one or more statuses (e.g., draft, published) |

## Usage Example

```java
// Build a filter to find published articles under a specific parent with given tags
KnowledgeBaseFilterCriteria criteria = KnowledgeBaseFilterCriteria.builder()
    .parentId("folder-123")
    .type(KnowledgeBaseItemType.ARTICLE)
    .tagIds(List.of("tag-abc", "tag-xyz"))
    .statuses(List.of(KnowledgeBaseArticleStatus.PUBLISHED))
    .build();

// Pass criteria to the knowledge base service
List<KnowledgeBaseItem> results = knowledgeBaseService.findItems(criteria);
```

> **Note:** All fields are optional. When a field is `null`, it is ignored in the filter query, allowing partial or fully open-ended searches across the knowledge base.
