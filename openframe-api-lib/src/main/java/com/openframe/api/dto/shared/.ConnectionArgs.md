DTO implementing [Relay Connection Specification](https://relay.dev/graphql/connections.htm) pagination arguments, supporting both forward and backward cursor-based pagination.

## Key Components

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `first` | `Integer` | 1–100 | Number of items to return (forward pagination) |
| `after` | `String` | — | Cursor to start after (forward pagination) |
| `last` | `Integer` | 1–100 | Number of items to return (backward pagination) |
| `before` | `String` | — | Cursor to start before (backward pagination) |

## Usage Example

```java
// Forward pagination — fetch first 10 items after a cursor
ConnectionArgs forwardPage = ConnectionArgs.builder()
    .first(10)
    .after("cursor_abc123")
    .build();

// Backward pagination — fetch last 5 items before a cursor
ConnectionArgs backwardPage = ConnectionArgs.builder()
    .last(5)
    .before("cursor_xyz789")
    .build();

// First page (no cursor required)
ConnectionArgs firstPage = ConnectionArgs.builder()
    .first(20)
    .build();
```

> **Note:** `first` and `last` are mutually exclusive per the Relay spec. Bean validation (`@Min`, `@Max`) enforces a page size between 1 and 100 on each. Cursors (`after`, `before`) are opaque strings — typically Base64-encoded identifiers.
