A shared DTO (Data Transfer Object) for organization responses that can be used by both GraphQL and REST API endpoints to ensure consistent data structure across different API implementations.

## Key Components

- **OrganizationResponse** - Main DTO class with comprehensive organization data fields
- **Lombok annotations** - `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation
- **Core fields** - Basic organization info (id, name, category, employee count)
- **Business fields** - Financial data (monthlyRevenue), contract dates, website
- **Metadata fields** - Timestamps, deletion status, default flag
- **Nested DTO** - `ContactInformationDto` for structured contact data

## Usage Example

```java
// Building an organization response
OrganizationResponse response = OrganizationResponse.builder()
    .id("org-123")
    .name("Tech Corp")
    .organizationId("TECH001")
    .category("Technology")
    .numberOfEmployees(150)
    .websiteUrl("https://techcorp.com")
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractStartDate(LocalDate.of(2024, 1, 1))
    .contractEndDate(LocalDate.of(2025, 12, 31))
    .isDefault(false)
    .deleted(false)
    .createdAt(Instant.now())
    .build();

// Accessing data
String orgName = response.getName();
BigDecimal revenue = response.getMonthlyRevenue();
```
