A shared DTO class that represents organization data in API responses, used by both GraphQL and REST endpoints to ensure consistent organization data structure across different API interfaces.

## Key Components

- **OrganizationResponse**: Main DTO class with comprehensive organization fields
- **Lombok annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for boilerplate reduction
- **Core fields**: `id`, `name`, `organizationId`, `category`, `numberOfEmployees`
- **Business fields**: `monthlyRevenue`, contract dates, contact information
- **Metadata fields**: timestamps, deletion status, default flag
- **ContactInformationDto**: Nested DTO for contact details

## Usage Example

```java
// Building an organization response
OrganizationResponse response = OrganizationResponse.builder()
    .id("org-123")
    .name("Acme Corp")
    .organizationId("ACME-001")
    .category("Technology")
    .numberOfEmployees(150)
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractStartDate(LocalDate.of(2024, 1, 1))
    .isDefault(false)
    .deleted(false)
    .createdAt(Instant.now())
    .build();

// Using in REST controller
@GetMapping("/organizations/{id}")
public OrganizationResponse getOrganization(@PathVariable String id) {
    return organizationService.getOrganizationResponse(id);
}
```
