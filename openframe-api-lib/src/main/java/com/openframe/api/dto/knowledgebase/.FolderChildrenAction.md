Defines the action strategy for child items when a parent folder is deleted in the knowledge base.

## Key Components

| Constant | Description |
|----------|-------------|
| `MOVE` | Re-parents all child items to another specified folder |
| `ARCHIVE` | Recursively archives all child items in place |

## Usage Example

```java
FolderDeletionRequest request = FolderDeletionRequest.builder()
    .folderId("folder-123")
    .childrenAction(FolderChildrenAction.MOVE)
    .targetFolderId("folder-456")
    .build();

// Or archive all children recursively
FolderDeletionRequest archiveRequest = FolderDeletionRequest.builder()
    .folderId("folder-123")
    .childrenAction(FolderChildrenAction.ARCHIVE)
    .build();
```

> Use `MOVE` when child content should remain accessible under a new parent. Use `ARCHIVE` to soft-delete the entire subtree without permanent data loss.
