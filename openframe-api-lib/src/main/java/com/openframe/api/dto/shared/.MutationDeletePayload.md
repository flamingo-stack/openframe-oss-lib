A standardized response payload DTO for delete mutation operations in the OpenFrame API, encapsulating either a successful deletion result or a list of user-facing errors.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `deletedId` | `String` | ID of the successfully deleted entity |
| `userErrors` | `List<MutationError>` | List of errors; defaults to empty list on success |
| `success(String deletedId)` | Static factory | Builds a success payload with the deleted entity's ID |
| `error(String message)` | Static factory | Builds an error payload with a single `MutationError` message |

## Usage Example

```java
// Success case — return after a successful delete operation
MutationDeletePayload payload = MutationDeletePayload.success("ticket-123");

// Error case — return when deletion fails (e.g., entity not found)
MutationDeletePayload payload = MutationDeletePayload.error("Record not found or already deleted.");

// Checking the result
if (payload.getUserErrors().isEmpty()) {
    System.out.println("Deleted: " + payload.getDeletedId());
} else {
    payload.getUserErrors().forEach(e -> System.err.println(e.getMessage()));
}
```

> **Pattern:** This follows the GraphQL-style mutation payload convention — always return a structured response with either a result or `userErrors`, never throw raw exceptions to the client layer.
