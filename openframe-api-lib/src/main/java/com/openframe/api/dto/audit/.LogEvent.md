A Data Transfer Object (DTO) representing audit log events in the OpenFrame platform, designed to capture and structure logging data from various MSP tools and systems.

## Key Components

- **Event Identification**: `id`, `toolEventId`, `eventType` for unique identification and categorization
- **Tool Context**: `toolType`, `ingestDay` for tracking the source and timing of events
- **Security & Severity**: `severity`, `userId` for prioritization and user attribution
- **Device & Network**: `deviceId`, `hostname` for asset tracking and network context
- **Organization**: `organizationId`, `organizationName` for multi-tenant support
- **Event Details**: `summary`, `timestamp` for human-readable descriptions and precise timing
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Create a log event for a security incident
LogEvent securityEvent = LogEvent.builder()
    .id("log-001")
    .toolEventId("splunk-alert-123")
    .eventType("SECURITY_ALERT")
    .toolType("SIEM")
    .severity("HIGH")
    .userId("user-456")
    .deviceId("workstation-789")
    .hostname("ws-finance-01")
    .organizationId("org-123")
    .organizationName("Acme Corp")
    .summary("Failed login attempts detected")
    .timestamp(Instant.now())
    .ingestDay("2024-01-15")
    .build();

// Use in audit trail processing
auditService.processLogEvent(securityEvent);
```

This DTO enables the OpenFrame platform to standardize log data from different MSP tools (SIEM, RMM, PSA) into a unified audit trail for Flamingo's AI-powered analysis and automation.
