A DTO (Data Transfer Object) class that defines filtering criteria for device queries in the OpenFrame API, allowing clients to specify multiple filter parameters to narrow down device search results.

## Key Components

- **statuses** - Filter devices by their operational status (active, inactive, maintenance, etc.)
- **deviceTypes** - Filter by device category (server, workstation, mobile, etc.)
- **osTypes** - Filter by operating system type (Windows, Linux, macOS, etc.)
- **organizationIds** - Filter devices belonging to specific organizations
- **tagKeys** - Filter devices that have specific tag keys
- **tagValues** - Filter devices with specific tag values

## Usage Example

```java
// Create filter criteria for active Windows workstations
DeviceFilterCriteria criteria = DeviceFilterCriteria.builder()
    .statuses(List.of(DeviceStatus.ACTIVE))
    .deviceTypes(List.of(DeviceType.WORKSTATION))
    .osTypes(List.of("Windows 11", "Windows 10"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagKeys(List.of("environment", "department"))
    .build();

// Use with device service to filter results
List<Device> filteredDevices = deviceService.findDevices(criteria);
```

The class uses Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern implementation, making it easy to construct complex filter combinations for device queries.
