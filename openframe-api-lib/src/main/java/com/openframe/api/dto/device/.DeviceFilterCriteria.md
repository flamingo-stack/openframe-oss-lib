Data Transfer Object for filtering devices in OpenFrame's device management system. This criteria class defines the various parameters that can be used to filter devices based on their properties and metadata.

## Key Components

- **Status Filtering**: Filter devices by their operational status (active, inactive, etc.)
- **Device Type Filtering**: Filter by device types (workstation, server, mobile, etc.)  
- **OS Type Filtering**: Filter by operating system types (Windows, Linux, macOS, etc.)
- **Organization Filtering**: Filter devices by organization membership
- **Tag-based Filtering**: Filter devices by custom tag keys and values for flexible categorization

## Usage Example

```java
// Filter for active Windows workstations in specific organizations
DeviceFilterCriteria criteria = DeviceFilterCriteria.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.WORKSTATION))
    .osTypes(List.of("Windows 10", "Windows 11"))
    .organizationIds(List.of("org-123", "org-456"))
    .build();

// Filter devices with specific monitoring tags
DeviceFilterCriteria monitoringCriteria = DeviceFilterCriteria.builder()
    .tagKeys(List.of("monitoring", "critical"))
    .tagValues(List.of("enabled", "high-priority"))
    .build();

// Use criteria in device query
List<Device> filteredDevices = deviceService.findDevicesByCriteria(criteria);
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it easy to construct complex filter combinations for device queries.
