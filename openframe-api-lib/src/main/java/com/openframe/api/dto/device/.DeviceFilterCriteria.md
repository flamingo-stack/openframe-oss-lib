A DTO (Data Transfer Object) used to encapsulate filter parameters when querying devices in the OpenFrame API.

## Key Components

| Field | Type | Description |
|---|---|---|
| `statuses` | `List<DeviceStatus>` | Filter by one or more device statuses |
| `deviceTypes` | `List<DeviceType>` | Filter by device type (e.g., workstation, server) |
| `osTypes` | `List<String>` | Filter by operating system types |
| `organizationIds` | `List<String>` | Scope results to specific organizations |
| `tagKeys` | `List<String>` | Filter by tag key presence |
| `tagValues` | `List<String>` | Filter by tag value presence |

Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) provide boilerplate for getters/setters, builder pattern, and constructors.

## Usage Example

```java
// Build a filter for active Windows workstations in a specific org
DeviceFilterCriteria criteria = DeviceFilterCriteria.builder()
    .statuses(List.of(DeviceStatus.ACTIVE))
    .deviceTypes(List.of(DeviceType.WORKSTATION))
    .osTypes(List.of("Windows"))
    .organizationIds(List.of("org-123"))
    .tagKeys(List.of("environment"))
    .tagValues(List.of("production"))
    .build();

// Pass to a device query service
List<Device> devices = deviceService.findDevices(criteria);
```

> All fields are optional — supply only the filters relevant to your query. Omitted fields are ignored during filtering, allowing flexible, composable queries.
