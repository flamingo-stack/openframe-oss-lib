A data transfer object that defines filtering criteria for device queries in the OpenFrame API, allowing users to filter devices by multiple attributes such as status, type, and organization.

## Key Components

**Main Class:**
- `DeviceFilterCriteria` - DTO with filtering parameters for device searches

**Filter Fields:**
- `statuses` - List of device status enums to filter by
- `deviceTypes` - List of device type enums to filter by  
- `osTypes` - List of operating system types (strings)
- `organizationIds` - List of organization identifiers
- `tagKeys` - List of tag keys for tag-based filtering
- `tagValues` - List of tag values for tag-based filtering

**Annotations:**
- Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern

## Usage Example

```java
// Create filter criteria using builder pattern
DeviceFilterCriteria criteria = DeviceFilterCriteria.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.PENDING))
    .deviceTypes(List.of(DeviceType.LAPTOP, DeviceType.DESKTOP))
    .osTypes(List.of("Windows", "macOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagKeys(List.of("department", "location"))
    .build();

// Use in service layer for filtering devices
List<Device> filteredDevices = deviceService.findDevices(criteria);
```

This DTO is typically used in REST API endpoints to accept filtering parameters and in service layers to build dynamic database queries for device retrieval.
