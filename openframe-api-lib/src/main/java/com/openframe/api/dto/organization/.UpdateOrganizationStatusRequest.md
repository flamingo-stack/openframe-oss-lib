Request DTO for updating an organization's status, enforcing allowed status transitions via API.

## Key Components

- **`UpdateOrganizationStatusRequest`** — Java record carrying the validated status payload
- **`status`** — Required field (`@NotNull`) of type `OrganizationStatusAction`
- **`OrganizationStatusAction`** — Enum restricting allowed transitions to `ACTIVE` and `ARCHIVED`; `DELETED` is intentionally excluded from API access

## Usage Example

```java
// Valid request — activate an organization
UpdateOrganizationStatusRequest request = new UpdateOrganizationStatusRequest(
    UpdateOrganizationStatusRequest.OrganizationStatusAction.ACTIVE
);

// Valid request — archive an organization
UpdateOrganizationStatusRequest archiveRequest = new UpdateOrganizationStatusRequest(
    UpdateOrganizationStatusRequest.OrganizationStatusAction.ARCHIVED
);

// Null status will trigger validation failure:
// "Status is required"
UpdateOrganizationStatusRequest invalid = new UpdateOrganizationStatusRequest(null);
```

## Notes

- `DELETED` is not exposed through the API — deletion must be handled through a separate, controlled flow
- Bean Validation (`jakarta.validation`) is applied at the record level; ensure `@Valid` is present on the corresponding controller parameter to trigger constraint evaluation
