A data transfer object that encapsulates filtering criteria for querying audit logs and system events within the OpenFrame platform.

## Key Components

- **toolTypes** - List of tool type identifiers to filter log entries
- **eventTypes** - List of event type categories for log filtering  
- **severities** - List of severity levels (e.g., INFO, WARN, ERROR)
- **organizations** - List of organization filter options for multi-tenant log access
- **Lombok annotations** - Provides automatic getters, setters, builders, and constructors

## Usage Example

```java
// Create log filters for API requests
LogFilters filters = LogFilters.builder()
    .toolTypes(Arrays.asList("mingo-ai", "fae-client"))
    .eventTypes(Arrays.asList("authentication", "api-request"))
    .severities(Arrays.asList("ERROR", "WARN"))
    .organizations(Arrays.asList(
        new OrganizationFilterOption("org-123", "Acme Corp")
    ))
    .build();

// Use in audit log service
List<AuditLogEntry> logs = auditService.getFilteredLogs(filters);

// Check if specific filters are applied
if (!filters.getSeverities().isEmpty()) {
    System.out.println("Filtering by severities: " + filters.getSeverities());
}
```

The class supports the builder pattern for flexible filter construction and integrates with OpenFrame's multi-tenant audit logging system to provide scoped access to log data based on organizational boundaries.
