A data transfer object (DTO) that encapsulates filter criteria for querying organizations. This class provides a structured way to pass filtering parameters for organization searches and queries.

## Key Components

- **category** - String filter for organization category/type
- **minEmployees** - Integer minimum employee count threshold
- **maxEmployees** - Integer maximum employee count threshold  
- **hasActiveContract** - Boolean flag to filter by contract status
- **Lombok annotations** - `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for boilerplate code generation

## Usage Example

```java
// Create filter for tech companies with 50-500 employees and active contracts
OrganizationFilterOptions filters = OrganizationFilterOptions.builder()
    .category("Technology")
    .minEmployees(50)
    .maxEmployees(500)
    .hasActiveContract(true)
    .build();

// Use with service layer
List<Organization> results = organizationService.findWithFilters(filters);

// Create filter for any size organizations in healthcare
OrganizationFilterOptions healthcareFilter = OrganizationFilterOptions.builder()
    .category("Healthcare")
    .build();
```

The builder pattern makes it easy to create filter combinations, and nullable fields allow for optional filtering criteria.
