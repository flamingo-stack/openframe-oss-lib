A Data Transfer Object (DTO) that encapsulates filtering criteria for organization queries, providing flexible search and filtering capabilities for organization-related operations.

## Key Components

- **category** - String field for filtering organizations by category/type
- **minEmployees** - Integer field for minimum employee count threshold
- **maxEmployees** - Integer field for maximum employee count threshold  
- **hasActiveContract** - Boolean field for filtering by contract status
- **Lombok annotations** - `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for boilerplate reduction

## Usage Example

```java
// Create filter for medium-sized tech companies with active contracts
OrganizationFilterOptions techFilter = OrganizationFilterOptions.builder()
    .category("Technology")
    .minEmployees(50)
    .maxEmployees(500)
    .hasActiveContract(true)
    .build();

// Create filter for large organizations regardless of contract status
OrganizationFilterOptions largeOrgFilter = OrganizationFilterOptions.builder()
    .minEmployees(1000)
    .build();

// Use in service layer for filtered queries
List<Organization> filteredOrgs = organizationService.findByFilter(techFilter);
```

This DTO is typically used internally by service layers to construct dynamic queries and provide flexible organization search functionality across the application.
