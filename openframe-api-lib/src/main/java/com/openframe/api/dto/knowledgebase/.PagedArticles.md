A lightweight pagination wrapper for knowledge base articles, using a `record` to carry a page of results alongside a `hasNextPage` signal.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `items` | `List<KnowledgeBaseItem>` | The articles returned for the current page |
| `hasNextPage` | `boolean` | Indicates whether more pages exist beyond the current one |

## Usage Example

The `hasNextPage` flag is derived by fetching `limit + 1` items — if the result size exceeds `limit`, there is a next page, and the extra item is trimmed before wrapping.

```java
int limit = 10;
List<KnowledgeBaseItem> fetched = repository.findArticles(offset, limit + 1);

boolean hasNextPage = fetched.size() > limit;
List<KnowledgeBaseItem> page = hasNextPage
    ? fetched.subList(0, limit)
    : fetched;

PagedArticles result = new PagedArticles(page, hasNextPage);
```

> **Why `limit + 1`?** A common pitfall is checking `size == limit` to detect the next page — this produces false negatives when `totalCount % limit == 0` (e.g., exactly 20 results with a limit of 10). Fetching one extra item avoids this edge case entirely without requiring a separate `COUNT` query.
