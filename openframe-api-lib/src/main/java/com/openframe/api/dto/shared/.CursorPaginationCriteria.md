A DTO that encapsulates cursor-based pagination parameters, supporting both GraphQL Relay Connection Spec and REST API pagination patterns with built-in limit normalization and cursor decoding.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `limit` | `Integer` | Number of records to fetch per page |
| `cursor` | `String` | Decoded opaque cursor for positional reference |
| `backward` | `boolean` | Direction flag; `true` for reverse (last/before) pagination |
| `DEFAULT_PAGE_SIZE` | `int` | Default page size constant (`20`) |
| `MIN_PAGE_SIZE` / `MAX_PAGE_SIZE` | `int` | Bounds constants (`1`–`100`) |
| `fromConnectionArgs()` | static factory | Builds criteria from GraphQL Relay `ConnectionArgs`; backward pagination takes precedence |
| `fromRest()` | static factory | Builds criteria from REST query parameters |
| `normalize()` | instance method | Clamps `limit` within `MIN_PAGE_SIZE`–`MAX_PAGE_SIZE`, applying default if absent |
| `hasCursor()` | instance method | Returns `true` if a non-blank cursor is present |

## Usage Example

```java
// GraphQL Relay Connection (forward pagination)
ConnectionArgs args = ConnectionArgs.builder()
        .first(10)
        .after("encoded-cursor-abc")
        .build();

CursorPaginationCriteria criteria = CursorPaginationCriteria
        .fromConnectionArgs(args)
        .normalize();

// REST API pagination
CursorPaginationCriteria restCriteria = CursorPaginationCriteria
        .fromRest("encoded-cursor-xyz", 50)
        .normalize(); // limit clamped to [1, 100]

// Check if paginating from a specific position
if (criteria.hasCursor()) {
    // apply WHERE id > cursor in query
}

// Backward pagination takes precedence when both directions supplied
ConnectionArgs mixedArgs = ConnectionArgs.builder()
        .first(10).after("fwd-cursor")
        .last(5).before("bwd-cursor") // this wins
        .build();
CursorPaginationCriteria backward = CursorPaginationCriteria.fromConnectionArgs(mixedArgs);
// backward.isBackward() == true
```
