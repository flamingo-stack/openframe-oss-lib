A shared DTO class for implementing cursor-based pagination in API requests, providing limit and cursor parameters with validation constraints.

## Key Components

**Main Class:**
- `CursorPaginationInput` - Data transfer object for cursor pagination parameters

**Fields:**
- `limit` - Maximum number of items to return (1-100, validated)
- `cursor` - Opaque cursor string for pagination position

**Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode
- `@Builder` - Enables builder pattern for object construction
- `@Min/@Max` - Validates limit field boundaries

## Usage Example

```java
// Create pagination input with builder pattern
CursorPaginationInput paginationInput = CursorPaginationInput.builder()
    .limit(25)
    .cursor("eyJpZCI6MTIzLCJ0aW1lc3RhbXAiOiIyMDI0LTAxLTE1In0=")
    .build();

// Use in API controller
@GetMapping("/items")
public ResponseEntity<PagedResponse<Item>> getItems(
    @Valid @ModelAttribute CursorPaginationInput pagination) {
    
    return itemService.getItems(pagination.getLimit(), pagination.getCursor());
}

// First page request (no cursor)
CursorPaginationInput firstPage = CursorPaginationInput.builder()
    .limit(50)
    .build();
```

The validation ensures limit values between 1-100, preventing oversized result sets while maintaining API performance and consistency across paginated endpoints.
