A DTO (Data Transfer Object) for handling cursor-based pagination input parameters with built-in validation constraints.

## Key Components

- **CursorPaginationInput**: Main DTO class with Lombok annotations for automatic getter/setter generation
- **limit**: Integer field constrained between 1-100 for controlling page size
- **cursor**: String field representing the pagination cursor position
- **Validation annotations**: `@Min` and `@Max` constraints on the limit field

## Usage Example

```java
// Create pagination input programmatically
CursorPaginationInput pagination = CursorPaginationInput.builder()
    .limit(25)
    .cursor("eyJpZCI6MTIzfQ==")
    .build();

// Or use in a REST controller
@GetMapping("/items")
public ResponseEntity<ItemsResponse> getItems(@Valid CursorPaginationInput pagination) {
    List<Item> items = itemService.findItems(pagination);
    return ResponseEntity.ok(new ItemsResponse(items));
}

// Default constructor usage
CursorPaginationInput pagination = new CursorPaginationInput();
pagination.setLimit(50);
pagination.setCursor("nextPageToken123");
```

The validation annotations ensure that API requests automatically reject invalid limit values, providing clear error messages for values outside the 1-100 range.
