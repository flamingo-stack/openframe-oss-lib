Data Transfer Object (DTO) that encapsulates filtering criteria for device queries. This class provides a structured way to specify multiple filter parameters when searching or retrieving devices from the system.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter by device operational status
- **deviceTypes** - List of `DeviceType` enums to filter by device category/type
- **osTypes** - List of operating system strings for OS-based filtering
- **organizationIds** - List of organization identifiers for multi-tenant filtering
- **tagNames** - List of tag names for tag-based device categorization

## Usage Example

```java
// Create filter options using builder pattern
DeviceFilterOptions filterOptions = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.PENDING))
    .deviceTypes(List.of(DeviceType.MOBILE, DeviceType.TABLET))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "customer-facing"))
    .build();

// Or create empty filter and set specific criteria
DeviceFilterOptions filter = new DeviceFilterOptions();
filter.setStatuses(List.of(DeviceStatus.ACTIVE));
filter.setOrganizationIds(List.of("my-org"));
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation.
