A data transfer object (DTO) that encapsulates filtering criteria for device queries, allowing multiple filter parameters to be applied simultaneously.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their current status
- **deviceTypes** - List of `DeviceType` enums to filter by device category
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagNames** - List of tag name strings for tag-based device categorization

## Usage Example

```java
// Create filter options using builder pattern
DeviceFilterOptions filterOptions = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.MOBILE, DeviceType.TABLET))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "testing"))
    .build();

// Use with device service/repository
List<Device> filteredDevices = deviceService.findDevices(filterOptions);

// Create minimal filter for specific use case
DeviceFilterOptions statusFilter = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE))
    .build();
```
