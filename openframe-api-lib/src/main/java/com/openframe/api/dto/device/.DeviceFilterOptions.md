A data transfer object (DTO) that encapsulates filtering criteria for device queries, allowing API consumers to specify multiple filter parameters for device searches.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their operational status
- **deviceTypes** - List of `DeviceType` enums to filter by device categories (mobile, tablet, desktop, etc.)
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagNames** - List of tag name strings for label-based device categorization

## Usage Example

```java
// Build filter options for active mobile devices
DeviceFilterOptions mobileFilter = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ONLINE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.MOBILE, DeviceType.TABLET))
    .osTypes(List.of("Android", "iOS"))
    .build();

// Filter by organization and tags
DeviceFilterOptions orgFilter = DeviceFilterOptions.builder()
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "test-fleet"))
    .build();

// Use in device service query
List<Device> filteredDevices = deviceService.findDevices(mobileFilter);
```

The class uses Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern implementation, making it ideal for REST API request parameters and service layer filtering operations.
