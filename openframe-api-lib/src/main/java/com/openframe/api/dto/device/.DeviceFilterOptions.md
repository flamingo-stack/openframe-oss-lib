A data transfer object that encapsulates filtering criteria for device queries, allowing clients to specify multiple filter parameters to narrow down device search results.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their operational status
- **deviceTypes** - List of `DeviceType` enums to filter by hardware/device category
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagNames** - List of tag name strings for metadata-based filtering

## Usage Example

```java
DeviceFilterOptions filterOptions = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ONLINE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.TABLET, DeviceType.SMARTPHONE))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "kiosk"))
    .build();

// Pass to device service for filtering
List<Device> filteredDevices = deviceService.findDevices(filterOptions);
```

The class uses Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern implementation.
