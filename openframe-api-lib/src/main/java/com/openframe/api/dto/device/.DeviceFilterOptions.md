A data transfer object (DTO) that encapsulates filtering criteria for device queries, providing a structured way to specify multiple filter parameters for device search and retrieval operations.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their operational status
- **deviceTypes** - List of `DeviceType` enums to filter by hardware/device categories
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagKeys** - List of tag key strings for key-based device tag filtering
- **tagValues** - List of tag value strings for value-based device tag filtering

## Usage Example

```java
// Create filter options for active Android devices with specific tags
DeviceFilterOptions filterOptions = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.TABLET, DeviceType.PHONE))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagKeys(List.of("site", "environment"))
    .tagValues(List.of("production", "retail"))
    .build();

// Use with device service for filtered queries
List<Device> filteredDevices = deviceService.findDevices(filterOptions);

// Create minimal filter for just device types
DeviceFilterOptions simpleFilter = DeviceFilterOptions.builder()
    .deviceTypes(List.of(DeviceType.DISPLAY))
    .build();
```
