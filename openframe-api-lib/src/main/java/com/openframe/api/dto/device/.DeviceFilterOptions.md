A data transfer object that encapsulates filter criteria for querying devices, allowing filtering by multiple device attributes simultaneously.

## Key Components

- **statuses**: List of `DeviceStatus` enums to filter devices by their current status
- **deviceTypes**: List of `DeviceType` enums to filter by device categories
- **osTypes**: List of operating system type strings for OS-based filtering
- **organizationIds**: List of organization identifier strings for multi-tenant filtering
- **tagNames**: List of tag name strings for label-based filtering

## Usage Example

```java
// Create filter for active Android devices in specific organizations
DeviceFilterOptions filter = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.MOBILE, DeviceType.TABLET))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "monitoring"))
    .build();

// Use in device service query
List<Device> filteredDevices = deviceService.findDevices(filter);
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it easy to construct complex filter combinations.
