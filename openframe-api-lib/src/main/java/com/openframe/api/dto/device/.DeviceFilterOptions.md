A data transfer object that encapsulates filtering criteria for device queries, providing flexible filtering options across multiple device attributes.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their operational state
- **deviceTypes** - List of `DeviceType` enums to filter by hardware/device category
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagNames** - List of tag name strings for label-based device categorization

## Usage Example

```java
// Create filter for active Android devices in specific organizations
DeviceFilterOptions filter = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.TABLET, DeviceType.PHONE))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "field-device"))
    .build();

// Use with device service/repository
List<Device> filteredDevices = deviceService.findDevices(filter);

// Empty filter returns all devices
DeviceFilterOptions noFilter = new DeviceFilterOptions();
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it ideal for constructing complex filter combinations in device management APIs.
