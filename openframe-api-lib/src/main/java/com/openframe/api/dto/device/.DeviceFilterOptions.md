Data Transfer Object for filtering device queries with multiple criteria options including status, type, OS, organization, and tags.

## Key Components

- **statuses** - List of `DeviceStatus` enum values to filter devices by their current operational status
- **deviceTypes** - List of `DeviceType` enum values to filter by hardware/device category
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifier strings for multi-tenant filtering
- **tagNames** - List of tag name strings for label-based device categorization

## Usage Example

```java
// Create filter for active Android tablets in specific organizations
DeviceFilterOptions filters = DeviceFilterOptions.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.TABLET))
    .osTypes(List.of("Android"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "customer-facing"))
    .build();

// Use with device service for filtered queries
List<Device> filteredDevices = deviceService.findDevices(filters);
```

This DTO leverages Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern implementation, making it ideal for flexible device search and filtering operations across the OpenFrame API.
