A data transfer object that encapsulates complete contact information for an organization, including contact persons and address details.

## Key Components

- **ContactInformationDto**: Immutable record class using Lombok's `@Builder` annotation
- **contacts**: List of `ContactPersonDto` objects representing individual contact persons
- **physicalAddress**: `AddressDto` containing the organization's physical location
- **mailingAddress**: `AddressDto` for correspondence and mail delivery
- **mailingAddressSameAsPhysical**: Boolean flag indicating if mailing and physical addresses are identical

## Usage Example

```java
// Create contact information with builder pattern
ContactInformationDto contactInfo = ContactInformationDto.builder()
    .contacts(List.of(
        ContactPersonDto.builder()
            .name("John Doe")
            .email("john@company.com")
            .build()
    ))
    .physicalAddress(AddressDto.builder()
        .street("123 Main St")
        .city("Springfield")
        .build())
    .mailingAddress(AddressDto.builder()
        .street("PO Box 456")
        .city("Springfield")
        .build())
    .mailingAddressSameAsPhysical(false)
    .build();

// Access data
List<ContactPersonDto> contacts = contactInfo.contacts();
boolean sameAddress = contactInfo.mailingAddressSameAsPhysical();
```

The record structure ensures immutability and provides automatic implementations of `equals()`, `hashCode()`, and `toString()` methods, making it ideal for API data transfer scenarios.
