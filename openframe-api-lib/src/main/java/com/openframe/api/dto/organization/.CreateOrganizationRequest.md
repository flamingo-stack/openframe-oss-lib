Data transfer object for creating new organizations with validation constraints and shared usage across GraphQL and REST APIs.

## Key Components

- **CreateOrganizationRequest** - Immutable record with builder pattern for organization creation
- **Validation Annotations**:
  - `@NotBlank` on name field (required)
  - `@PositiveOrZero` on numberOfEmployees and monthlyRevenue
  - `@Valid` on nested contactInformation object
- **Fields**: name, category, numberOfEmployees, websiteUrl, notes, contactInformation, monthlyRevenue, contract dates

## Usage Example

```java
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Tech Solutions Inc")
    .category("Technology")
    .numberOfEmployees(50)
    .websiteUrl("https://techsolutions.com")
    .monthlyRevenue(new BigDecimal("25000.00"))
    .contractStartDate(LocalDate.of(2024, 1, 1))
    .contractEndDate(LocalDate.of(2024, 12, 31))
    .contactInformation(ContactInformationDto.builder()
        .email("contact@techsolutions.com")
        .build())
    .build();
```

The organizationId is automatically generated as a UUID on the backend, so it's not included in the request. All monetary values use BigDecimal for precision, and validation ensures data integrity before processing.
