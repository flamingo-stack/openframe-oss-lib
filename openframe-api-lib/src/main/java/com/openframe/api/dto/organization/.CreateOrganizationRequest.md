Data Transfer Object (DTO) record for creating new organizations in the system. This shared DTO is used by both GraphQL and REST API endpoints, with automatic UUID generation for organization IDs on the backend.

## Key Components

- **CreateOrganizationRequest** - Builder-enabled record containing organization creation fields
- **Validation Annotations** - Jakarta validation constraints for required fields and positive values
- **ContactInformationDto** - Nested DTO for contact information (marked with `@Valid`)
- **Core Fields** - Name (required), category, employee count, website, notes, revenue, and contract dates

## Usage Example

```java
// Create organization request with required fields
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Acme Corporation")
    .category("Technology")
    .numberOfEmployees(150)
    .websiteUrl("https://acme.com")
    .contactInformation(ContactInformationDto.builder()
        .email("info@acme.com")
        .phone("+1-555-0123")
        .build())
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractStartDate(LocalDate.now())
    .contractEndDate(LocalDate.now().plusYears(1))
    .build();

// The request can be validated using Jakarta validation
// organizationId will be auto-generated as UUID on backend
```
