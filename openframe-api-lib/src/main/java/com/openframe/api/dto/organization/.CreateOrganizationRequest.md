A Data Transfer Object (DTO) record for creating new organizations in the system, featuring comprehensive validation constraints and automatic UUID generation on the backend.

## Key Components

- **CreateOrganizationRequest** - Main record class with `@Builder` support for creating organization requests
- **Validation annotations** - `@NotBlank` for required name field, `@PositiveOrZero` for numeric constraints, `@Valid` for nested objects
- **ContactInformationDto** - Nested DTO for organization contact details
- **Financial fields** - Monthly revenue tracking with `BigDecimal` precision
- **Contract management** - Start and end date fields for contract lifecycle

## Usage Example

```java
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Acme Corporation")
    .category("Technology")
    .numberOfEmployees(150)
    .websiteUrl("https://acme.com")
    .contactInformation(ContactInformationDto.builder()
        .email("contact@acme.com")
        .phone("+1-555-0123")
        .build())
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractStartDate(LocalDate.now())
    .contractEndDate(LocalDate.now().plusYears(1))
    .notes("New client onboarding")
    .build();

// Used in both GraphQL mutations and REST endpoints
// organizationId is auto-generated as UUID by backend
```
