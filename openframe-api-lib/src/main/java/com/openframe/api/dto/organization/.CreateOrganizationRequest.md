A data transfer object (DTO) for creating new organizations in the OpenFrame API, shared between GraphQL and REST endpoints with comprehensive validation rules.

## Key Components

- **Record Class**: Immutable data structure with automatic constructor, getters, and validation
- **@Builder**: Lombok annotation enabling fluent builder pattern for object creation
- **Validation Annotations**: Jakarta Bean Validation constraints for required fields and value ranges
- **Contact Information**: Nested DTO for structured contact details
- **Financial Fields**: Monthly revenue tracking with proper decimal handling
- **Contract Dates**: Start and end date fields for contract management

## Usage Example

```java
// Using builder pattern to create organization request
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Tech Solutions Inc")
    .category("Software Development")
    .numberOfEmployees(25)
    .websiteUrl("https://techsolutions.com")
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractStartDate(LocalDate.now())
    .contractEndDate(LocalDate.now().plusYears(1))
    .contactInformation(ContactInformationDto.builder()
        .email("contact@techsolutions.com")
        .phone("+1-555-0123")
        .build())
    .notes("Premier client with annual contract")
    .build();

// The organizationId will be auto-generated as UUID on the backend
```

The validation ensures data integrity with required name fields, positive employee counts, and proper financial values before organization creation.
