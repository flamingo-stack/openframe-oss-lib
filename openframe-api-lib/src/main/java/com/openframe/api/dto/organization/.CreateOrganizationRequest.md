Data Transfer Object for creating new organizations in the OpenFrame API system. This shared DTO is used by both GraphQL and REST endpoints to ensure consistent organization creation across different API interfaces.

## Key Components

- **CreateOrganizationRequest** - Main record class with validation annotations
- **Required Fields** - `name` (validated as not blank)
- **Optional Fields** - `category`, `numberOfEmployees`, `websiteUrl`, `notes`, `monthlyRevenue`, contract dates
- **Nested Object** - `contactInformation` (ContactInformationDto with validation)
- **Validation Constraints** - Bean validation annotations for data integrity

## Usage Example

```java
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Acme Corporation")
    .category("Technology")
    .numberOfEmployees(150)
    .websiteUrl("https://acme.com")
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contactInformation(ContactInformationDto.builder()
        .email("contact@acme.com")
        .phone("+1-555-0123")
        .build())
    .contractStartDate(LocalDate.now())
    .contractEndDate(LocalDate.now().plusYears(1))
    .notes("Key client in tech sector")
    .build();
```

The organizationId is automatically generated as a UUID on the backend, so it's not included in the request payload.
