A DTO (Data Transfer Object) record for updating organization entities, designed to work across both GraphQL and REST APIs with optional field updates.

## Key Components

- **UpdateOrganizationRequest**: Builder-enabled record with optional fields for organization updates
- **Validation Annotations**: `@PositiveOrZero` constraints on `numberOfEmployees` and `monthlyRevenue`
- **ContactInformationDto**: Nested DTO for contact details with `@Valid` validation
- **Field Types**: Supports strings, integers, BigDecimal for financial data, and LocalDate for contract dates

## Usage Example

```java
// Create partial update request
UpdateOrganizationRequest updateRequest = UpdateOrganizationRequest.builder()
    .name("Updated Company Name")
    .numberOfEmployees(150)
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contractEndDate(LocalDate.of(2024, 12, 31))
    .build();

// Only specified fields will be updated
organizationService.updateOrganization(orgId, updateRequest);

// Minimal update - just change website
UpdateOrganizationRequest minimalUpdate = UpdateOrganizationRequest.builder()
    .websiteUrl("https://newdomain.com")
    .build();
```

The record's optional nature allows clients to send partial updates, making it efficient for both small tweaks and comprehensive organization data changes.
