A DTO record for updating organization data with optional fields and validation constraints, designed for use across both GraphQL and REST APIs.

## Key Components

- **UpdateOrganizationRequest** - Builder-enabled record containing all updatable organization fields
- **Validation Annotations** - `@PositiveOrZero` constraints on numeric fields, `@Valid` for nested objects
- **Optional Fields** - All fields are nullable, allowing partial updates
- **ContactInformationDto** - Nested DTO for contact details with validation

## Usage Example

```java
// Create partial update request
UpdateOrganizationRequest updateRequest = UpdateOrganizationRequest.builder()
    .name("Acme Corporation Updated")
    .numberOfEmployees(150)
    .monthlyRevenue(new BigDecimal("50000.00"))
    .contactInformation(ContactInformationDto.builder()
        .email("contact@acme.com")
        .phone("+1-555-0123")
        .build())
    .build();

// Only name, employee count, revenue, and contact info will be updated
// Other fields remain unchanged in the organization
```

The record supports flexible partial updates where only non-null fields are processed, making it efficient for both bulk updates and single field modifications across different API endpoints.
