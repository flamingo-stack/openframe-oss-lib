A shared DTO record for updating existing organizations, used across both GraphQL and REST APIs with optional field-based updates.

## Key Components

- **UpdateOrganizationRequest** - Record class with builder pattern for organization updates
- **Validation Annotations** - `@PositiveOrZero` constraints on `numberOfEmployees` and `monthlyRevenue`
- **Optional Fields** - All fields are nullable, enabling partial updates
- **ContactInformationDto** - Nested DTO with `@Valid` annotation for cascading validation

## Usage Example

```java
// Create a partial update request
UpdateOrganizationRequest updateRequest = UpdateOrganizationRequest.builder()
    .name("Acme Corporation Updated")
    .numberOfEmployees(150)
    .monthlyRevenue(new BigDecimal("75000.00"))
    .contactInformation(
        ContactInformationDto.builder()
            .email("contact@acme-updated.com")
            .build()
    )
    .build();

// Only specified fields will be updated
organizationService.updateOrganization(orgId, updateRequest);
```

The record supports incremental updates where clients only provide fields they want to change, making it efficient for both small tweaks and comprehensive updates across different API endpoints.
