A data transfer object (DTO) that encapsulates filtering criteria for tool-related queries. This class provides a structured way to pass multiple optional filter parameters when searching or filtering tools in the system.

## Key Components

- **enabled** - Boolean flag to filter tools by their enabled/disabled status
- **type** - String filter for tool type classification
- **category** - String filter for tool category grouping
- **platformCategory** - String filter for platform-specific tool categorization

## Usage Example

```java
// Create filter options using builder pattern
ToolFilterOptions filters = ToolFilterOptions.builder()
    .enabled(true)
    .type("API")
    .category("integration")
    .platformCategory("web")
    .build();

// Create filter with only specific criteria
ToolFilterOptions enabledOnly = ToolFilterOptions.builder()
    .enabled(true)
    .build();

// Use with service layer
List<Tool> tools = toolService.findTools(filters);

// Create empty filter for all tools
ToolFilterOptions allTools = new ToolFilterOptions();
```

The class leverages Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern implementation, reducing boilerplate code while maintaining clean filtering functionality.
