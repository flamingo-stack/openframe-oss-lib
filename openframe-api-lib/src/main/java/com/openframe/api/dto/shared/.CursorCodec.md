Utility class for encoding and decoding opaque Relay-style pagination cursors using Base64, hiding internal implementation details (e.g., MongoDB ObjectIds, composite keys) from API consumers.

## Key Components

| Method | Description |
|--------|-------------|
| `encode(String rawCursor)` | Encodes an internal cursor value to a Base64 opaque string. Returns `null` for null/empty input. |
| `decode(String opaqueCursor)` | Decodes a Base64 cursor back to its raw internal value. Returns `null` for null, empty, or invalid Base64 input. |

## Usage Example

```java
// Encoding an internal MongoDB ObjectId before sending to the client
String rawId = "507f1f77bcf86cd799439011";
String opaqueCursor = CursorCodec.encode(rawId);
// opaqueCursor → "NTA3ZjFmNzdiY2Y4NmNkNzk5NDM5MDEx"

// Decoding a cursor received from a pagination request
String decoded = CursorCodec.decode(opaqueCursor);
// decoded → "507f1f77bcf86cd799439011"

// Safe handling of invalid input
String result = CursorCodec.decode("!!!invalid-base64!!!");
// result → null (no exception thrown)
```

## Notes

- The class is **non-instantiable** (private constructor) and all methods are `static`.
- Encoding uses `UTF-8` consistently for both encode and decode operations.
- Invalid Base64 input during decoding is caught silently, returning `null` rather than propagating an `IllegalArgumentException`.
- Designed to be a drop-in cursor codec for Relay-compatible pagination APIs.
