A generic DTO that extends `GenericQueryResult<T>` to include a filtered record count alongside the standard query result payload.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `filteredCount` | `int` | The number of records matching the applied filters, separate from the full result set |

**Inherited from `GenericQueryResult<T>`:** core query result fields (data payload, pagination info, etc.)

**Lombok annotations used:**
- `@Data` — generates getters, setters, `toString`, `equals`, `hashCode`
- `@SuperBuilder` — enables builder pattern with parent class field support
- `@NoArgsConstructor` / `@AllArgsConstructor` — generates both constructors
- `@EqualsAndHashCode(callSuper = true)` — includes parent fields in equality checks

## Usage Example

```java
// Building a paginated, filtered response
CountedGenericQueryResult<DeviceDTO> result = CountedGenericQueryResult.<DeviceDTO>builder()
    .data(deviceList)
    .filteredCount(42)
    .build();

// Accessing the filtered count
int count = result.getFilteredCount(); // 42
```

> `filteredCount` is useful when returning paginated results where the client needs to know the total number of matching records (for pagination controls) without returning all records at once.
