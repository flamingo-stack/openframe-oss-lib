A specialized extension of `GenericQueryResult` that includes a filtered count field for paginated query results that need to track the total number of filtered records.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class extending `GenericQueryResult<T>` with count tracking
- **filteredCount** - Integer field storing the total count of records matching the filter criteria
- **Lombok annotations** - Provides automatic generation of constructors, getters, setters, equals, hashCode, and builder pattern

## Usage Example

```java
// Creating a counted query result for user data
CountedGenericQueryResult<User> result = CountedGenericQueryResult.<User>builder()
    .data(userList)
    .success(true)
    .message("Users retrieved successfully")
    .filteredCount(150) // Total matching records before pagination
    .build();

// Accessing the filtered count
int totalMatching = result.getFilteredCount();
List<User> currentPage = result.getData();

// Useful for pagination UI components
int totalPages = (int) Math.ceil((double) result.getFilteredCount() / pageSize);
```

This class is typically used in API responses where the client needs to know both the current page of results and the total count of records that match the applied filters, enabling proper pagination controls and result summaries.
