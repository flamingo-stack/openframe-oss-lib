A specialized extension of `GenericQueryResult` that includes a filtered count for pagination and result tracking in query operations.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class that extends `GenericQueryResult<T>` with pagination metadata
- **filteredCount** - Integer field tracking the number of filtered results, useful for pagination controls and result summaries
- **Lombok annotations** - Provides automatic generation of constructors, getters, setters, equals, hashCode, and builder pattern

## Usage Example

```java
// Creating a counted query result for user search
CountedGenericQueryResult<User> userResult = CountedGenericQueryResult.<User>builder()
    .data(userList)           // from parent GenericQueryResult
    .success(true)            // from parent GenericQueryResult  
    .filteredCount(25)        // total matching the filter criteria
    .build();

// Using in a paginated API response
@GetMapping("/users")
public ResponseEntity<CountedGenericQueryResult<User>> getUsers(
    @RequestParam(defaultValue = "0") int page,
    @RequestParam(defaultValue = "10") int size
) {
    Page<User> userPage = userService.findUsers(page, size);
    
    CountedGenericQueryResult<User> result = CountedGenericQueryResult.<User>builder()
        .data(userPage.getContent())
        .filteredCount((int) userPage.getTotalElements())
        .success(true)
        .build();
        
    return ResponseEntity.ok(result);
}
```

This class is particularly useful for API endpoints that need to return both the actual data and metadata about how many total records matched the filter criteria, enabling proper pagination controls in frontend applications.
