A DTO class that extends `GenericQueryResult` to include filtered count information for paginated query responses.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class that adds filtering capabilities to query results
- **filteredCount** - Integer field representing the number of items that match the filter criteria
- **Generic Type Parameter <T>** - Allows the class to work with any data type for query results

## Usage Example

```java
// Create a counted query result for user entities
CountedGenericQueryResult<User> userResult = CountedGenericQueryResult.<User>builder()
    .data(userList)              // Inherited from GenericQueryResult
    .totalCount(1000)            // Inherited from GenericQueryResult  
    .filteredCount(25)           // New field - users matching filter
    .build();

// Access the filtered count
int matchingUsers = userResult.getFilteredCount();

// Use in API responses for pagination with filtering
@GetMapping("/users")
public CountedGenericQueryResult<User> getUsers(
    @RequestParam String filter,
    @RequestParam int page) {
    
    List<User> users = userService.findFiltered(filter, page);
    int totalFiltered = userService.countFiltered(filter);
    
    return CountedGenericQueryResult.<User>builder()
        .data(users)
        .filteredCount(totalFiltered)
        .build();
}
```

This class is particularly useful for API endpoints that support both pagination and filtering, where clients need to know both the total available items and how many match the current filter criteria.
