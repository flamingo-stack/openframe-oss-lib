A DTO class that extends GenericQueryResult to include a filtered count alongside query results, useful for pagination scenarios where you need both the data and the count of filtered records.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class extending GenericQueryResult
- **filteredCount** - Integer field representing the number of records after filtering
- **Lombok annotations** - Provides builders, getters/setters, equals/hashCode, and constructors

## Usage Example

```java
// Creating a counted result for paginated user search
CountedGenericQueryResult<User> userResult = CountedGenericQueryResult.<User>builder()
    .data(userList)
    .success(true)
    .message("Users retrieved successfully")
    .filteredCount(150)
    .build();

// Accessing the filtered count for pagination
int totalFilteredUsers = userResult.getFilteredCount();
List<User> currentPageUsers = userResult.getData();

// Useful for API responses that need both data and count
return ResponseEntity.ok(userResult);
```

The class leverages Lombok's `@SuperBuilder` to enable fluent building while maintaining inheritance from the parent GenericQueryResult class, making it ideal for REST API responses that require pagination metadata.
