A data transfer object that extends GenericQueryResult to include a count of filtered results, typically used for paginated queries where you need to know both the results and how many items matched the filter criteria.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class extending GenericQueryResult
- **filteredCount** - Integer field storing the number of items that matched the filter
- **Lombok annotations** - Provides boilerplate code generation (getters, setters, builders, etc.)
- **Generic type T** - Allows the result to contain any type of data

## Usage Example

```java
// Creating a counted result for user queries
CountedGenericQueryResult<User> userResult = CountedGenericQueryResult.<User>builder()
    .data(userList)
    .success(true)
    .message("Users retrieved successfully")
    .filteredCount(150)
    .build();

// Accessing the filtered count
int totalMatches = userResult.getFilteredCount();

// Using with pagination info
System.out.println("Showing " + userResult.getData().size() + 
                  " of " + userResult.getFilteredCount() + " matching users");
```

This class is particularly useful for API responses where clients need to implement pagination controls or display "X of Y results" information to users.
