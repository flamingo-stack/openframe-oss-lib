A specialized query result DTO that extends `GenericQueryResult` to include count information for filtered results, typically used for paginated API responses that need to display total filtered records.

## Key Components

- **`CountedGenericQueryResult<T>`** - Generic DTO class extending `GenericQueryResult<T>`
- **`filteredCount`** - Integer field storing the number of records matching filter criteria
- **Lombok annotations** - `@Data`, `@SuperBuilder`, `@NoArgsConstructor`, `@AllArgsConstructor` for boilerplate code generation
- **`@EqualsAndHashCode(callSuper = true)`** - Includes parent class fields in equality/hash calculations

## Usage Example

```java
// Creating a counted result for user search
CountedGenericQueryResult<User> userResults = CountedGenericQueryResult.<User>builder()
    .data(userList)
    .total(1000)
    .filteredCount(150)
    .build();

// Accessing the filtered count
int matchingUsers = userResults.getFilteredCount();
List<User> users = userResults.getData();
int totalUsers = userResults.getTotal();

// Use in paginated API responses
@GetMapping("/users")
public CountedGenericQueryResult<User> searchUsers(
    @RequestParam String filter,
    @RequestParam int page) {
    
    List<User> filteredUsers = userService.searchUsers(filter, page);
    int totalFiltered = userService.countFilteredUsers(filter);
    
    return CountedGenericQueryResult.<User>builder()
        .data(filteredUsers)
        .filteredCount(totalFiltered)
        .build();
}
```
