A data transfer object that extends `GenericQueryResult` to include pagination metadata, specifically tracking the number of filtered results in query operations.

## Key Components

- **CountedGenericQueryResult<T>**: Generic DTO class extending `GenericQueryResult<T>`
- **filteredCount**: Integer field storing the count of items after filtering
- **Lombok annotations**: Provides automatic generation of getters, setters, builders, and equality methods

## Usage Example

```java
// Creating a counted query result for user data
CountedGenericQueryResult<User> result = CountedGenericQueryResult.<User>builder()
    .data(userList)
    .totalCount(1000)
    .filteredCount(250)
    .build();

// Accessing the filtered count
int filtered = result.getFilteredCount();
System.out.println("Showing " + result.getData().size() + 
                   " of " + filtered + " filtered results");

// Using with pagination info
if (result.getFilteredCount() < result.getTotalCount()) {
    System.out.println("Filters applied - showing " + 
                       result.getFilteredCount() + " filtered items");
}
```

This class is typically used in REST API responses where you need to return both the actual data and metadata about how many items matched the applied filters, enabling proper pagination controls in frontend applications.
