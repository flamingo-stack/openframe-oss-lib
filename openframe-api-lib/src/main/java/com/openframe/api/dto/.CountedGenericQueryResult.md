A generic data transfer object that extends `GenericQueryResult` to include a filtered count, providing both paginated results and the total number of items that match filtering criteria.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class that inherits all functionality from `GenericQueryResult<T>`
- **filteredCount** - Integer field representing the total count of items after filters are applied
- **Lombok annotations** - Provides automatic generation of getters, setters, constructors, and builder pattern

## Usage Example

```java
// Creating a counted query result for user data
CountedGenericQueryResult<User> userResult = CountedGenericQueryResult.<User>builder()
    .data(userList)           // inherited from GenericQueryResult
    .totalCount(1000)         // inherited from GenericQueryResult  
    .filteredCount(150)       // new field - users matching filters
    .page(1)                  // inherited from GenericQueryResult
    .pageSize(20)             // inherited from GenericQueryResult
    .build();

// Accessing the filtered count
int matchingUsers = userResult.getFilteredCount(); // 150

// Use case: API response with search/filter applied
// totalCount = all users in database (1000)
// filteredCount = users matching search criteria (150)
// data = actual page of results (20 items)
```

The class is particularly useful for APIs that need to return both the total dataset size and the count after search/filtering operations, enabling proper pagination controls in user interfaces.
