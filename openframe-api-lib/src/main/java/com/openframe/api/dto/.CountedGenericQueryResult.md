A specialized data transfer object that extends `GenericQueryResult` to include filtered result count information, typically used for paginated queries that need to report both the filtered subset and total available records.

## Key Components

- **CountedGenericQueryResult<T>** - Generic DTO class extending `GenericQueryResult<T>`
- **filteredCount** - Integer field storing the count of filtered results
- **Lombok annotations** - Provides standard object methods and builder pattern support

## Usage Example

```java
// Creating a counted query result for user search
CountedGenericQueryResult<User> result = CountedGenericQueryResult.<User>builder()
    .data(Arrays.asList(user1, user2, user3))
    .filteredCount(3)
    .totalCount(150) // inherited from parent
    .build();

// Using in a paginated API response
@GetMapping("/users")
public CountedGenericQueryResult<User> searchUsers(
    @RequestParam String query,
    @RequestParam int page,
    @RequestParam int size) {
    
    List<User> users = userService.searchUsers(query, page, size);
    int filteredCount = userService.countFilteredUsers(query);
    
    return CountedGenericQueryResult.<User>builder()
        .data(users)
        .filteredCount(filteredCount)
        .build();
}
```

This class is particularly useful for frontend applications that need to display both "showing X of Y filtered results" and total available records for pagination controls.
