A DTO class that encapsulates filtering criteria for audit log queries, providing flexible options to filter logs by date ranges, event types, tools, severity levels, and organizational scope.

## Key Components

- **Date Filtering**: `startDate` and `endDate` fields for time-based log filtering using `LocalDate`
- **Event Classification**: `eventTypes` and `toolTypes` lists for filtering by specific event categories and tools
- **Severity Filtering**: `severities` list for filtering logs by severity levels (e.g., INFO, WARN, ERROR)
- **Organizational Scope**: `organizationIds` list for multi-tenant filtering by organization
- **Device Targeting**: `deviceId` string for filtering logs from specific devices
- **Lombok Annotations**: Uses `@Data`, `@Builder`, `@NoArgsConstructor`, and `@AllArgsConstructor` for boilerplate code generation

## Usage Example

```java
// Build comprehensive filter options
LogFilterOptions filterOptions = LogFilterOptions.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.of(2024, 1, 31))
    .eventTypes(List.of("LOGIN", "LOGOUT", "DATA_ACCESS"))
    .toolTypes(List.of("WEB_APP", "MOBILE_APP"))
    .severities(List.of("WARN", "ERROR"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use with audit service
List<AuditLog> filteredLogs = auditService.getLogs(filterOptions);

// Create date-only filter
LogFilterOptions dateFilter = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .build();
```
