A data transfer object (DTO) that encapsulates filtering criteria for audit log queries, providing flexible search and filtering capabilities across multiple dimensions.

## Key Components

**Fields**:
- `startDate` / `endDate` - Date range filtering using LocalDate
- `eventTypes` - Filter by specific audit event types
- `toolTypes` - Filter by tool or application types
- `severities` - Filter by log severity levels
- `organizationIds` - Multi-tenant filtering by organization
- `deviceId` - Filter by specific device identifier

**Annotations**:
- `@Data` - Generates getters, setters, toString, equals, and hashCode
- `@Builder` - Enables fluent builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` - Provides constructors

## Usage Example

```java
// Using builder pattern for complex filtering
LogFilterOptions filter = LogFilterOptions.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "LOGOUT", "DATA_ACCESS"))
    .severities(List.of("ERROR", "WARN"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Simple date range filtering
LogFilterOptions dateFilter = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .build();

// Organization-specific filtering
LogFilterOptions orgFilter = new LogFilterOptions();
orgFilter.setOrganizationIds(List.of("tenant-abc"));
orgFilter.setEventTypes(List.of("USER_CREATED", "USER_DELETED"));
```
