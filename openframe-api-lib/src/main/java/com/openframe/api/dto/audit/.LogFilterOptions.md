A DTO (Data Transfer Object) class that encapsulates filtering criteria for audit log queries, providing flexible options to narrow down log entries by various attributes.

## Key Components

- **Date Range Filtering**: `startDate` and `endDate` fields for temporal filtering using LocalDate
- **Event Classification**: `eventTypes` list for filtering by specific event categories
- **Tool Categorization**: `toolTypes` list for filtering by tool or service types
- **Severity Levels**: `severities` list for filtering by log severity (e.g., INFO, WARN, ERROR)
- **Multi-tenancy Support**: `organizationIds` list for filtering by organization scope
- **Device Targeting**: `deviceId` string for filtering by specific device identifier
- **Lombok Annotations**: Uses `@Data`, `@Builder`, `@NoArgsConstructor`, and `@AllArgsConstructor` for boilerplate code generation

## Usage Example

```java
// Create filter with builder pattern
LogFilterOptions filter = LogFilterOptions.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "LOGOUT", "DATA_ACCESS"))
    .severities(List.of("ERROR", "WARN"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use in service layer
List<AuditLog> filteredLogs = auditService.findLogs(filter);

// Create simple date range filter
LogFilterOptions dateFilter = LogFilterOptions.builder()
    .startDate(LocalDate.now().minusDays(30))
    .endDate(LocalDate.now())
    .build();
```
