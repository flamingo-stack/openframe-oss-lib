Data Transfer Object (DTO) that encapsulates filtering options and metadata for device queries in the OpenFrame MSP platform. This class provides structured filter criteria for device searches and includes the count of filtered results.

## Key Components

- **statuses** - Filter options for device operational states (active, inactive, etc.)
- **deviceTypes** - Filter options for device categories (workstation, server, mobile, etc.)  
- **osTypes** - Filter options for operating system types (Windows, macOS, Linux, etc.)
- **organizationIds** - Filter options for organization-based device grouping
- **tagKeys** - Filter options for device tags and labels
- **filteredCount** - Total number of devices matching the applied filters

## Usage Example

```java
// Building device filters for a search query
DeviceFilters filters = DeviceFilters.builder()
    .statuses(Arrays.asList(
        DeviceFilterOption.builder().value("active").label("Active").build(),
        DeviceFilterOption.builder().value("maintenance").label("Under Maintenance").build()
    ))
    .deviceTypes(Arrays.asList(
        DeviceFilterOption.builder().value("workstation").label("Workstations").build()
    ))
    .osTypes(Arrays.asList(
        DeviceFilterOption.builder().value("windows").label("Windows").build()
    ))
    .filteredCount(127)
    .build();

// Using in API response to show available filter options
return ResponseEntity.ok(filters);
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it ideal for API responses that provide both filter options and result metadata to frontend applications.
