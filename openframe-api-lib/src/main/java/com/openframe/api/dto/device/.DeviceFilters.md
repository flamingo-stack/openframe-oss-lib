A data transfer object that encapsulates filtering options and criteria for device queries, containing various filter categories and the count of filtered results.

## Key Components

- **Filter Collections**: Lists of `DeviceFilterOption` objects for statuses, device types, OS types, and organization IDs
- **Tag Filters**: Specialized `TagFilterOption` list for tag-based filtering
- **Result Metadata**: `filteredCount` field to track the number of devices matching the applied filters
- **Lombok Annotations**: Auto-generated getters, setters, constructors, and builder pattern support

## Usage Example

```java
// Create device filters for querying active Android devices
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(
        new DeviceFilterOption("active", "Active"),
        new DeviceFilterOption("online", "Online")
    ))
    .deviceTypes(List.of(
        new DeviceFilterOption("tablet", "Tablet"),
        new DeviceFilterOption("phone", "Phone")
    ))
    .osTypes(List.of(
        new DeviceFilterOption("android", "Android")
    ))
    .organizationIds(List.of(
        new DeviceFilterOption("org123", "Main Organization")
    ))
    .tags(List.of(
        new TagFilterOption("production", "Production Environment")
    ))
    .filteredCount(25)
    .build();

// Use in API response
return ResponseEntity.ok(filters);
```
