A data transfer object (DTO) that encapsulates various filter options for querying and filtering devices within the OpenFrame platform. This class provides a comprehensive set of filtering criteria including device status, type, operating system, organization, and tags.

## Key Components

- **statuses** - List of device status filter options (active, inactive, maintenance, etc.)
- **deviceTypes** - List of device type filter options (server, workstation, mobile, etc.)
- **osTypes** - List of operating system filter options (Windows, Linux, macOS, etc.)
- **organizationIds** - List of organization filter options for multi-tenant filtering
- **tagKeys** - List of tag-based filter options for custom device categorization
- **filteredCount** - Integer representing the total count of devices matching applied filters

## Usage Example

```java
// Create device filters for querying active Windows servers
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(
        DeviceFilterOption.builder().value("active").build(),
        DeviceFilterOption.builder().value("maintenance").build()
    ))
    .deviceTypes(List.of(
        DeviceFilterOption.builder().value("server").build()
    ))
    .osTypes(List.of(
        DeviceFilterOption.builder().value("windows").build()
    ))
    .organizationIds(List.of(
        DeviceFilterOption.builder().value("org-123").build()
    ))
    .tagKeys(List.of(
        TagFilterOption.builder().key("environment").value("production").build()
    ))
    .build();

// Apply filters to device query
DeviceQueryRequest request = DeviceQueryRequest.builder()
    .filters(filters)
    .build();
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it ideal for API request/response handling in the OpenFrame device management system.
