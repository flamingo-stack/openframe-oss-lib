A data transfer object that encapsulates various filtering options for device queries, including status, type, OS, organization, and tag-based filters along with result count information.

## Key Components

- **DeviceFilters**: Main DTO class with Lombok annotations for automatic getter/setter generation
- **statuses**: Filter options for device status (active, inactive, etc.)
- **deviceTypes**: Filter options for different device types
- **osTypes**: Operating system filter options
- **organizationIds**: Organization-based filtering options
- **tags**: Tag-based filter options using `TagFilterOption` type
- **filteredCount**: Total count of devices matching the applied filters

## Usage Example

```java
// Building device filters
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(new DeviceFilterOption("ACTIVE", "Active Devices")))
    .deviceTypes(List.of(new DeviceFilterOption("MOBILE", "Mobile Devices")))
    .osTypes(List.of(new DeviceFilterOption("ANDROID", "Android")))
    .organizationIds(List.of(new DeviceFilterOption("org123", "Engineering Team")))
    .tags(List.of(new TagFilterOption("production", "Production Environment")))
    .filteredCount(150)
    .build();

// Using in a service method
public List<Device> getFilteredDevices(DeviceFilters filters) {
    return deviceRepository.findByFilters(filters);
}
```
