A data transfer object (DTO) that encapsulates filter options for device queries, containing various categorical filters and metadata about filtered results.

## Key Components

- **DeviceFilters**: Main DTO class with Lombok annotations for automatic getter/setter/builder generation
- **Filter Collections**: Lists of `DeviceFilterOption` objects for statuses, device types, OS types, and organization IDs
- **Tag Filters**: Specialized `TagFilterOption` list for tag-based filtering
- **Result Metadata**: `filteredCount` integer to track the number of matching devices

## Usage Example

```java
// Create device filters using builder pattern
DeviceFilters filters = DeviceFilters.builder()
    .statuses(Arrays.asList(
        new DeviceFilterOption("ACTIVE", 15),
        new DeviceFilterOption("INACTIVE", 3)
    ))
    .deviceTypes(Arrays.asList(
        new DeviceFilterOption("MOBILE", 10),
        new DeviceFilterOption("TABLET", 8)
    ))
    .osTypes(Arrays.asList(
        new DeviceFilterOption("iOS", 12),
        new DeviceFilterOption("Android", 6)
    ))
    .filteredCount(18)
    .build();

// Access filter data
int totalDevices = filters.getFilteredCount();
List<DeviceFilterOption> availableStatuses = filters.getStatuses();
```

This class is typically used in API responses to provide clients with available filter options and current filter state for device management interfaces.
