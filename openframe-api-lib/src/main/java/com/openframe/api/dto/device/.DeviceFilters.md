A data transfer object that encapsulates all available filter options for device queries in the OpenFrame platform, including status, type, and organizational filters.

## Key Components

- **Device Filters**: Lists of `DeviceFilterOption` objects for statuses, device types, OS types, and organization IDs
- **Tag Filters**: `TagFilterOption` list for tag-based filtering
- **Result Metadata**: `filteredCount` integer providing the total number of devices matching current filters
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic getter/setter generation and builder pattern support

## Usage Example

```java
// Build device filters for API response
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(
        new DeviceFilterOption("ONLINE", "Online", 150),
        new DeviceFilterOption("OFFLINE", "Offline", 25)
    ))
    .deviceTypes(List.of(
        new DeviceFilterOption("LAPTOP", "Laptops", 89),
        new DeviceFilterOption("MOBILE", "Mobile Devices", 86)
    ))
    .osTypes(List.of(
        new DeviceFilterOption("WINDOWS", "Windows", 120),
        new DeviceFilterOption("MACOS", "macOS", 55)
    ))
    .organizationIds(List.of(
        new DeviceFilterOption("org-123", "Acme Corp", 100)
    ))
    .tagKeys(List.of(
        new TagFilterOption("department", List.of("IT", "Sales", "Marketing"))
    ))
    .filteredCount(175)
    .build();

// Use in device listing endpoints
return ResponseEntity.ok(filters);
```
