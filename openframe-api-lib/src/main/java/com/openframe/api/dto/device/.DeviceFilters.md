A data transfer object that represents available filter options and counts for device search and filtering operations in the OpenFrame API.

## Key Components

- **statuses** - List of available device status filter options (active, inactive, etc.)
- **deviceTypes** - List of device type filter options (mobile, tablet, desktop, etc.)
- **osTypes** - List of operating system filter options (iOS, Android, Windows, etc.)
- **organizationIds** - List of organization ID filter options for multi-tenant filtering
- **tags** - List of tag-based filter options for custom device categorization
- **filteredCount** - Total number of devices matching current filter criteria

## Usage Example

```java
// Building device filters for a search interface
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(
        DeviceFilterOption.builder().value("active").count(150).build(),
        DeviceFilterOption.builder().value("inactive").count(25).build()
    ))
    .deviceTypes(List.of(
        DeviceFilterOption.builder().value("mobile").count(100).build(),
        DeviceFilterOption.builder().value("tablet").count(50).build()
    ))
    .osTypes(List.of(
        DeviceFilterOption.builder().value("iOS").count(80).build(),
        DeviceFilterOption.builder().value("Android").count(70).build()
    ))
    .filteredCount(175)
    .build();

// Accessing filter information
int totalDevices = filters.getFilteredCount();
List<DeviceFilterOption> availableStatuses = filters.getStatuses();
```

This DTO is typically used to populate filter dropdowns and display counts in device management interfaces, enabling users to refine device searches based on multiple criteria.
