Data Transfer Object (DTO) used to encapsulate filter criteria for querying audit logs within the OpenFrame API.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `startDate` | `LocalDate` | Start of the date range filter |
| `endDate` | `LocalDate` | End of the date range filter |
| `eventTypes` | `List<String>` | Filter by specific audit event types |
| `toolTypes` | `List<String>` | Filter by tool/integration types |
| `severities` | `List<String>` | Filter by log severity levels |
| `organizationIds` | `List<String>` | Filter logs scoped to specific organizations |
| `deviceId` | `String` | Filter logs for a specific device |

## Usage Example

```java
LogFilterCriteria criteria = LogFilterCriteria.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "CONFIG_CHANGE"))
    .severities(List.of("ERROR", "WARNING"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();
```

> **Note:** All fields are optional. Only populated fields will be applied as active filters when querying the audit log store, allowing flexible partial filtering.
