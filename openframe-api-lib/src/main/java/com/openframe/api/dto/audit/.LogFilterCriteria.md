Data Transfer Object (DTO) for audit log filtering that provides structured criteria to query and filter audit logs across different dimensions including time ranges, event types, and organizational boundaries.

## Key Components

**Fields:**
- `startDate` / `endDate` - Time range boundaries for log filtering (LocalDate)
- `eventTypes` - List of specific audit event types to include
- `toolTypes` - List of tool/service types that generated the logs
- `severities` - List of severity levels to filter by
- `organizationIds` - List of organization identifiers for multi-tenant filtering
- `deviceId` - Specific device identifier for device-scoped queries

**Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode
- `@Builder` - Enables fluent builder pattern for object creation
- `@NoArgsConstructor` / `@AllArgsConstructor` - Default and full constructors

## Usage Example

```java
// Build filter criteria using builder pattern
LogFilterCriteria criteria = LogFilterCriteria.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "LOGOUT", "DATA_ACCESS"))
    .severities(List.of("ERROR", "WARN"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use in audit service
List<AuditLog> filteredLogs = auditService.getLogsByCriteria(criteria);

// Create simple time-based filter
LogFilterCriteria timeFilter = LogFilterCriteria.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .build();
```
