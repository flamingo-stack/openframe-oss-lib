A Data Transfer Object (DTO) that defines filtering criteria for querying audit logs within the OpenFrame platform, enabling precise log retrieval based on temporal, categorical, and organizational parameters.

## Key Components

- **Temporal Filters**: `startDate` and `endDate` fields for date range filtering using `LocalDate`
- **Event Classification**: `eventTypes` list for filtering by specific log event categories
- **Tool Integration**: `toolTypes` list for filtering logs by MSP tool sources
- **Severity Levels**: `severities` list for filtering by log importance/criticality levels
- **Multi-tenancy Support**: `organizationIds` list for filtering logs across specific organizations
- **Device Targeting**: `deviceId` field for filtering logs from a specific device
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Create comprehensive filter criteria
LogFilterCriteria criteria = LogFilterCriteria.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "DEVICE_ACCESS", "CONFIG_CHANGE"))
    .toolTypes(List.of("RMM", "PSA", "BACKUP"))
    .severities(List.of("ERROR", "WARNING"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use for targeted log queries
auditService.findLogs(criteria);
```

The builder pattern and comprehensive filtering options make this class ideal for constructing complex audit log queries across the distributed MSP platform ecosystem.
