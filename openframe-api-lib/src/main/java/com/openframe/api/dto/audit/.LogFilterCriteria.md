A data transfer object that encapsulates filtering criteria for querying audit logs in the OpenFrame platform, supporting time-based, type-based, and organizational filtering.

## Key Components

- **Date Range Filtering**: `startDate` and `endDate` fields for temporal log queries
- **Event Classification**: `eventTypes` and `toolTypes` lists for categorizing log entries
- **Severity Levels**: `severities` list for filtering by log importance/criticality
- **Multi-tenant Support**: `organizationIds` list for filtering logs across multiple organizations
- **Device-specific Filtering**: `deviceId` field for targeting specific device logs
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Create filter for critical errors in the last 7 days
LogFilterCriteria criteria = LogFilterCriteria.builder()
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .severities(List.of("ERROR", "CRITICAL"))
    .eventTypes(List.of("SECURITY_BREACH", "SYSTEM_FAILURE"))
    .organizationIds(List.of("org-123", "org-456"))
    .build();

// Filter for specific device monitoring events
LogFilterCriteria deviceCriteria = LogFilterCriteria.builder()
    .deviceId("device-789")
    .toolTypes(List.of("MONITORING", "ALERT"))
    .startDate(LocalDate.now().minusHours(24))
    .build();
```

The class supports flexible filtering combinations, allowing MSP administrators to query audit logs across multiple dimensions for compliance reporting, incident investigation, and operational monitoring within the Flamingo platform.
