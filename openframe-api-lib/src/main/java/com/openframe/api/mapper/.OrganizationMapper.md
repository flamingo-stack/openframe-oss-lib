A Spring `@Component` mapper that handles bidirectional conversion between `Organization` DTOs and entities, shared across both GraphQL and REST API layers in the OpenFrame platform.

## Key Components

| Method | Description |
|--------|-------------|
| `toEntity(CreateOrganizationRequest)` | Converts a creation request to an `Organization` entity, auto-generating a UUID `organizationId` |
| `updateEntity(Organization, UpdateOrganizationRequest)` | Applies partial updates to an existing entity; skips `null` fields and treats `organizationId` as immutable |
| `toResponse(Organization)` | Converts an `Organization` entity to an `OrganizationResponse` DTO including audit timestamps and status |
| `toContactInformationEntity(ContactInformationDto)` | Maps contact info DTO to entity, automatically copying physical address to mailing address when `mailingAddressSameAsPhysical` is `true` |
| Private address/contact helpers | `toAddressEntity`, `toAddressDto`, `toContactPersonEntity`, `toContactPersonDto`, `copyAddress` — handle nested object mapping |

## Usage Example

```java
@Service
public class OrganizationService {

    private final OrganizationMapper mapper;
    private final OrganizationRepository repository;

    // Create a new organization
    public OrganizationResponse create(CreateOrganizationRequest request) {
        Organization entity = mapper.toEntity(request);
        Organization saved = repository.save(entity);
        return mapper.toResponse(saved);
    }

    // Partially update an existing organization
    public OrganizationResponse update(String id, UpdateOrganizationRequest request) {
        Organization existing = repository.findById(id).orElseThrow();
        Organization updated = mapper.updateEntity(existing, request);
        return mapper.toResponse(repository.save(updated));
    }
}
```

> **Note:** `organizationId` is immutable after creation — `updateEntity` will never overwrite it, even if the request contains a value.
