A custom runtime exception thrown when a device cannot be found or accessed within the OpenFrame API system.

## Key Components

- **DeviceNotFoundException** - Custom runtime exception class extending `RuntimeException`
- **Constructor** - Takes a string message parameter to provide context about the missing device

## Usage Example

```java
// Throwing the exception when a device lookup fails
public Device findDeviceById(String deviceId) {
    Device device = deviceRepository.findById(deviceId);
    if (device == null) {
        throw new DeviceNotFoundException("Device with ID " + deviceId + " not found");
    }
    return device;
}

// Catching and handling the exception
try {
    Device device = deviceService.findDeviceById("DEV-123");
    // Process device...
} catch (DeviceNotFoundException e) {
    logger.error("Device lookup failed: " + e.getMessage());
    // Handle missing device scenario
}
```

This exception is typically used in service layers, repositories, or controllers when device-related operations fail due to the target device being unavailable, deleted, or non-existent in the system.
