Resolves and caches the active tenant ID for use in Pinot shared tables, falling back to a cluster-level environment variable when no tenant is registered in MongoDB.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `getTenantId()` | `public String` | Returns the active tenant ID — from MongoDB cache or `TENANT_ID` env var fallback |
| `isTenantRegistered()` | `public boolean` | Checks whether a tenant exists in MongoDB; used to gate deferred operations (e.g. Debezium connector setup) |
| `getCachedTenant()` | `private Tenant` | Lazy-loads and permanently caches the first tenant found in MongoDB for pod lifetime |
| `fallbackTenantId` | `@Value` field | Defaults to `"oss"` if `TENANT_ID` env var is not set |
| `cachedTenant` | in-memory field | Pod-lifetime cache; safe because one tenant per cluster never changes while pod is alive |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class DebeziumConnectorService {

    private final TenantIdProvider tenantIdProvider;

    public void setupConnector() {
        // Gate connector creation until a tenant is registered
        if (!tenantIdProvider.isTenantRegistered()) {
            log.warn("Skipping Debezium setup — tenant not yet registered");
            return;
        }

        String tenantId = tenantIdProvider.getTenantId();
        // Use tenantId to scope Pinot shared table operations
        createConnectorForTenant(tenantId);
    }
}
```

> **Caching behaviour:** The first successful MongoDB lookup is cached for the pod's lifetime. Pre-registration calls continue hitting MongoDB until a tenant appears, then cache forever. Cluster recycling destroys the pod, so stale cache is not a concern.
