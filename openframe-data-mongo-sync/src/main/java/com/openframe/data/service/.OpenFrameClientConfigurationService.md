Manages persistence and lifecycle of `OpenFrameClientConfiguration` documents per tenant, handling version tracking, publish state transitions, and configuration field merging.

## Key Components

| Method | Description |
|--------|-------------|
| `get()` | Retrieves the current tenant's configuration or throws if not found |
| `findById(String id)` | Looks up a configuration by document ID |
| `save(config)` | Persists a configuration document |
| `markAsNonPublished(config)` | Transitions publish state to non-published |
| `markAsPublished(config)` | Transitions publish state to published |
| `updateVersion(newVersion)` | Updates version string and marks config as pending publish; no-op if version unchanged |
| `updateConfigurationFields(fromConfig)` | Upserts configuration for the current tenant, merging download settings and triggering pending state if changed |

> `updateVersion` and `updateConfigurationFields` are annotated with `@RetryOnOptimisticLockingFailure` to safely handle concurrent write conflicts.

## Usage Example

```java
// Retrieve and publish the current tenant's configuration
OpenFrameClientConfiguration config = configService.get();
configService.markAsPublished(config);

// Update the client version (idempotent if version unchanged)
configService.updateVersion("2.4.1");

// Sync configuration fields from a remote/imported source
OpenFrameClientConfiguration incoming = buildConfigFromRemote();
configService.updateConfigurationFields(incoming);
```
