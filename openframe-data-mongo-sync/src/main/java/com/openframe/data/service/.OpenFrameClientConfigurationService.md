Service layer for managing `OpenFrameClientConfiguration` documents, handling CRUD operations, publish state transitions, and version/configuration field updates with optimistic locking retry support.

## Key Components

| Method | Description |
|--------|-------------|
| `get()` | Retrieves the configuration for the current tenant; throws `IllegalStateException` if not found |
| `findById(String id)` | Looks up a configuration by document ID, returning an `Optional` |
| `save(OpenFrameClientConfiguration)` | Persists a configuration document via the repository |
| `markAsNonPublished(config)` | Transitions publish state to non-published using the current state |
| `markAsPublished(config)` | Sets publish state to published |
| `updateVersion(String newVersion)` | Updates the client version and marks configuration as pending publish; no-op if version unchanged |
| `updateConfigurationFields(config)` | Upserts configuration fields for the current tenant; merges into existing or inserts if absent |
| `mergeAndSave(existing, fromConfig)` | Merges `DownloadConfiguration` from a source config into an existing one, marking pending if changed |

> Methods annotated with `@RetryOnOptimisticLockingFailure` (`updateVersion`, `updateConfigurationFields`) automatically retry on concurrent write conflicts.

## Usage Example

```java
// Retrieve and publish the current tenant's configuration
OpenFrameClientConfiguration config = configService.get();
configService.markAsPublished(config);

// Update the client version (retries on optimistic lock conflict)
configService.updateVersion("2.5.0");

// Upsert configuration fields from a remote/fetched config
configService.updateConfigurationFields(fetchedConfig);
```
