Spring `@Service` bean that provides CRUD and lookup operations for `IntegratedTool` documents stored in MongoDB, conditionally enabled via the `openframe.integration.tool.enabled` property.

## Key Components

| Method | Description |
|--------|-------------|
| `getAllTools()` | Returns all `IntegratedTool` documents for the current tenant |
| `getTool(String toolType)` | Looks up a tool by its **type** field |
| `getToolById(String key)` | Looks up a tool by its **human-readable key** (e.g. `"fleetmdm-server"`); tenant-scoped automatically via `TenantAwareMongoTemplate` |
| `findByUuid(String uuid)` | Looks up a tool by its raw MongoDB `_id` UUID — use only when the UUID is already known |
| `saveTool(IntegratedTool tool)` | Persists (insert or update) an `IntegratedTool` document |

> **Conditional activation:** This service only loads when `openframe.integration.tool.enabled=true` is set in application properties.

## Usage Example

```java
// Inject the service
@Autowired
private IntegratedToolService integratedToolService;

// Retrieve a tool by human-readable key
Optional<IntegratedTool> tool = integratedToolService.getToolById("fleetmdm-server");
tool.ifPresent(t -> log.info("Found tool: {}", t.getType()));

// Retrieve all registered tools
List<IntegratedTool> tools = integratedToolService.getAllTools();

// Persist a new or updated tool
IntegratedTool newTool = new IntegratedTool();
newTool.setKey("zabbix-server");
newTool.setType("monitoring");
integratedToolService.saveTool(newTool);
```
