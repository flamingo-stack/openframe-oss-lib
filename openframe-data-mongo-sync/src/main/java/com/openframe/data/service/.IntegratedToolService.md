Provides CRUD operations for managing `IntegratedTool` documents in the OpenFrame platform, enabling retrieval and persistence of integrated MSP tool configurations.

## Key Components

| Method | Return Type | Description |
|--------|-------------|-------------|
| `getAllTools()` | `List<IntegratedTool>` | Retrieves all registered integrated tools |
| `getTool(String toolType)` | `Optional<IntegratedTool>` | Looks up a tool by its type identifier |
| `getToolById(String toolId)` | `Optional<IntegratedTool>` | Looks up a tool by its unique document ID |
| `saveTool(IntegratedTool tool)` | `IntegratedTool` | Persists a new or updated tool configuration |

> **Conditional Activation:** This service is only instantiated when the property `openframe.integration.tool.enabled=true` is set, allowing tool integration to be toggled per environment.

## Usage Example

```java
// Retrieve all tools
List<IntegratedTool> tools = integratedToolService.getAllTools();

// Find a tool by type (e.g., "psa", "rmm")
Optional<IntegratedTool> rmm = integratedToolService.getTool("rmm");
rmm.ifPresent(tool -> log.info("Found tool: {}", tool.getId()));

// Find a tool by its document ID
Optional<IntegratedTool> tool = integratedToolService.getToolById("abc123");

// Persist a new tool configuration
IntegratedTool newTool = new IntegratedTool();
newTool.setType("psa");
IntegratedTool saved = integratedToolService.saveTool(newTool);
```

## Configuration

Ensure the following property is set in your environment to activate this service:

```yaml
openframe:
  integration:
    tool:
      enabled: true
```
