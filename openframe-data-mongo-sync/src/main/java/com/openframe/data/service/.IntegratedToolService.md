Spring service providing CRUD operations for `IntegratedTool` documents, conditionally activated via the `openframe.integration.tool.enabled` property and automatically scoped to the current tenant via `TenantAwareMongoTemplate`.

## Key Components

| Method | Description |
|--------|-------------|
| `getAllTools()` | Returns all integrated tool documents for the current tenant |
| `getTool(String toolType)` | Finds a tool by its type field |
| `getToolById(String key)` | Looks up a tool by human-readable key (e.g. `"fleetmdm-server"`) |
| `findByUuid(String uuid)` | Looks up a tool directly by MongoDB document UUID |
| `saveTool(IntegratedTool tool)` | Persists a new or updated tool document |

## Usage Example

```java
// Inject the service
@RequiredArgsConstructor
public class MyComponent {
    private final IntegratedToolService toolService;

    public void example() {
        // Look up by human-readable key (preferred)
        Optional<IntegratedTool> tool = toolService.getToolById("fleetmdm-server");

        // Look up by type
        Optional<IntegratedTool> byType = toolService.getTool("MDM");

        // Persist a new tool
        IntegratedTool newTool = new IntegratedTool();
        newTool.setKey("zabbix-server");
        newTool.setType("MONITORING");
        toolService.saveTool(newTool);
    }
}
```

## Notes

- Activated only when `openframe.integration.tool.enabled=true` is set in application properties.
- All repository calls are automatically tenant-scoped — no manual tenant filtering required.
- Prefer `getToolById(key)` over `findByUuid(uuid)` unless you explicitly hold a MongoDB document UUID.
