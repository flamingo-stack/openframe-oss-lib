Manages the read/unread state lifecycle for notifications, providing operations to create, mark, delete, and query notification read states for individual recipients or audiences.

## Key Components

| Method | Description |
|--------|-------------|
| `createForAudience` | Bulk-inserts `UNREAD` read-state records for a list of recipient IDs |
| `hasUnread` | Checks if a recipient has any unread notifications |
| `markRead` | Marks a single notification as read for a specific recipient |
| `markAllAsRead` | Marks all notifications as read for a recipient, returns affected count |
| `deleteNotification` | Soft-deletes a single notification for a recipient |
| `deleteAllRead` | Soft-deletes all read notifications for a recipient |
| `unreadCountsByCategory` | Returns a `Map<NotificationCategory, Long>` of unread counts grouped by category |

## Usage Example

```java
// Broadcast a new notification to multiple users
notificationReadStateService.createForAudience(
    "notif-001",
    NotificationCategory.ALERT,
    "System maintenance scheduled",
    RecipientType.USER,
    List.of("user-1", "user-2", "user-3")
);

// Check for unread notifications before rendering badge
boolean hasBadge = notificationReadStateService.hasUnread("user-1", RecipientType.USER);

// Mark a single notification as read
notificationReadStateService.markRead("user-1", RecipientType.USER, "notif-001");

// Get per-category unread counts for a notification center UI
Map<NotificationCategory, Long> counts =
    notificationReadStateService.unreadCountsByCategory("user-1", RecipientType.USER);
// e.g., { ALERT: 3, BILLING: 1 }
```
