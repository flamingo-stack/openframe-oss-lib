Service layer for managing `IntegratedToolAgent` MongoDB documents, handling CRUD operations, configuration synchronization, versioning, and publish-state lifecycle for tool agents in the OpenFrame platform.

## Key Components

| Method | Description |
|--------|-------------|
| `getAll()` / `getAllEnabled()` | Retrieve all agents or only `ENABLED` ones |
| `findByKey(String)` / `getByKey(String)` | Lookup by human-readable key (e.g. `"fleetmdm-agent"`) |
| `findByUuid(String)` / `getByUuid(String)` | Lookup by MongoDB document UUID |
| `findById` / `getById` | **Deprecated** — prefer key- or UUID-based variants |
| `updateConfigurationFields(...)` | Upserts an agent from a configuration; creates if missing, merges if existing |
| `mergeAndSave(...)` | Applies config changes and flags `PublishState.pending()` if version or assets changed |
| `updateReleaseAgentVersion(...)` | Updates the version on a release agent and marks it for publish |
| `markAsPublished(...)` / `markAsNonPublished(...)` | Transitions agent publish state |
| `hasAssetChanges(...)` | Compares asset versions between existing agent and incoming config |

Both `updateConfigurationFields` and `updateReleaseAgentVersion` are annotated with `@RetryOnOptimisticLockingFailure` to handle concurrent MongoDB document updates safely.

## Usage Example

```java
// Upsert agent config from a configuration document
IntegratedToolAgentConfiguration config = loadConfiguration("fleetmdm-agent");
integratedToolAgentService.updateConfigurationFields(config);

// Fetch by human-readable key
IntegratedToolAgent agent = integratedToolAgentService.getByKey("fleetmdm-agent");

// Promote a release agent to a new version
integratedToolAgentService.updateReleaseAgentVersion(agent.getId(), "2.1.0");

// Mark as published after deployment
integratedToolAgentService.markAsPublished(agent);

// List only enabled agents for runtime use
List<IntegratedToolAgent> activeAgents = integratedToolAgentService.getAllEnabled();
```
