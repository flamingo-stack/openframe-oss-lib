Service layer for managing `IntegratedToolAgent` documents, handling CRUD operations, publish state transitions, and configuration synchronization with optimistic locking retry support.

## Key Components

| Method | Description |
|--------|-------------|
| `getAll()` | Returns all tool agents |
| `getAllEnabled()` | Returns only agents with `ENABLED` status |
| `findById(String)` / `getById(String)` | Lookup by ID; `getById` throws if not found |
| `findByReleaseVersionTrue()` | Returns agents flagged as release versions |
| `markAsPublished(agent)` / `markAsNonPublished(agent)` | Transitions publish state and persists |
| `updateConfigurationFields(configuration)` | Upserts an agent from a configuration object; creates if absent, merges if present |
| `updateReleaseAgentVersion(id, newVersion)` | Updates version on release agents and marks them pending publish |
| `mergeAndSave(existing, configuration)` | Detects version/asset drift and sets publish state to `pending` accordingly |
| `hasAssetChanges(existing, configuration)` | Compares asset versions by ID to detect changes |

## Usage Example

```java
// Upsert an agent from a configuration (create or merge)
IntegratedToolAgentConfiguration config = loadConfiguration();
integratedToolAgentService.updateConfigurationFields(config);

// Mark a release agent's version as updated
integratedToolAgentService.updateReleaseAgentVersion("agent-123", "2.1.0");

// Retrieve all enabled agents
List<IntegratedToolAgent> activeAgents = integratedToolAgentService.getAllEnabled();

// Transition publish state
IntegratedToolAgent agent = integratedToolAgentService.getById("agent-123");
integratedToolAgentService.markAsPublished(agent);
```

## Notes

- `updateConfigurationFields` and `updateReleaseAgentVersion` are annotated with `@RetryOnOptimisticLockingFailure` to handle concurrent write conflicts safely.
- Publish state is automatically set to `pending` when a non-release agent's version or assets change during a merge.
- Release agents (`isReleaseVersion() == true`) have their version managed separately via `updateReleaseAgentVersion` and are skipped during standard configuration version writes.
