Service layer for managing `IntegratedToolAgent` MongoDB documents, handling CRUD operations, configuration synchronization, versioning, and publish-state lifecycle for tool agents in the OpenFrame platform.

## Key Components

| Method | Description |
|---|---|
| `getAll()` / `getAllEnabled()` | Fetch all agents or only those with `ENABLED` status |
| `findByKey(String)` / `getByKey(String)` | Look up agents by human-readable key (e.g. `"fleetmdm-agent"`) |
| `findByUuid(String)` / `getByUuid(String)` | Look up agents by MongoDB document UUID |
| `findById` / `getById` | **Deprecated** — prefer key or UUID variants |
| `updateConfigurationFields(...)` | Upsert an agent from a configuration object; triggers publish-state changes on version/asset drift |
| `updateReleaseAgentVersion(...)` | Bumps the version on a release-flagged agent and marks it `pending` for publish |
| `markAsPublished(...)` / `markAsNonPublished(...)` | Transition agent publish state |
| `hasAssetChanges(...)` | Detects asset version drift between existing agent and incoming configuration |

Both `updateConfigurationFields` and `updateReleaseAgentVersion` are annotated with `@RetryOnOptimisticLockingFailure` to handle concurrent MongoDB write conflicts.

## Usage Example

```java
// Upsert agent from a configuration payload
IntegratedToolAgentConfiguration config = loadFromRegistry("fleetmdm-agent");
integratedToolAgentService.updateConfigurationFields(config);

// Fetch enabled agents for runtime dispatch
List<IntegratedToolAgent> active = integratedToolAgentService.getAllEnabled();

// Promote a release agent to a new version
integratedToolAgentService.updateReleaseAgentVersion(agentUuid, "2.1.0");

// Publish state transitions
IntegratedToolAgent agent = integratedToolAgentService.getByKey("fleetmdm-agent");
integratedToolAgentService.markAsPublished(agent);
```
