Manages CRUD and lifecycle operations for `Organization` entities within the OpenFrame platform, including status transitions, soft deletion, and default organization provisioning for new tenants.

## Key Components

| Method | Description |
|--------|-------------|
| `getOrganizationById` | Fetches a non-deleted organization by document ID |
| `getOrganizationByOrganizationId` | Fetches a non-deleted organization by its unique identifier |
| `getOrganizationByName` | Fetches a non-deleted organization by name |
| `getDefaultOrganization` | Returns the active default organization for the tenant |
| `createOrganization` | Persists a new organization |
| `updateOrganization` | Saves updates to an existing organization |
| `canArchiveOrganization` | Checks whether all associated devices are archived or deleted |
| `updateOrganizationStatus` | Transitions status between `ACTIVE` and `ARCHIVED` with device validation |
| `deleteOrganization` | *(Deprecated)* Soft-deletes an organization; use `archiveOrganization` instead |
| `createDefaultOrganization` | Bootstraps a default `"General"` organization for a new tenant |

**Constant:** `DEFAULT_ORGANIZATION_NAME = "Default"` — the reserved name for the tenant fallback organization.

## Usage Example

```java
// Bootstrap default org during tenant registration
organizationService.createDefaultOrganization();

// Retrieve active default org
Optional<Organization> defaultOrg = organizationService.getDefaultOrganization();

// Archive a non-default org (fails if active devices exist)
try {
    organizationService.updateOrganizationStatus("org-uuid-123", OrganizationStatus.ARCHIVED);
} catch (OrganizationHasMachinesException e) {
    log.warn("Cannot archive: organization still has active machines");
}

// Pre-flight check before archiving
boolean canArchive = organizationService.canArchiveOrganization("org-uuid-123");
```

> **Note:** The default organization cannot be archived or deleted. Soft deletion (`deleteOrganization`) is deprecated — archived organizations must remain in the database to support devices with `DELETED` status that may come back online.
