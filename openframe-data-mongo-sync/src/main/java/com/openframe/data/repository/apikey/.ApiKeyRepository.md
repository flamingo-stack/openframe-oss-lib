Spring Data MongoDB repository interface for managing `ApiKey` documents, providing CRUD operations alongside custom queries for user-scoped key lookup and expired key detection.

## Key Components

| Method | Description |
|--------|-------------|
| `findByIdAndUserId` | Finds an API key by its ID and owning user ID (MongoDB `$and` query) |
| `findByIdAndUserIdOrElseThrow` | Wraps `findByIdAndUserId`, throwing `ApiKeyNotFoundException` if no match is found |
| `findByUserId` | Retrieves all API keys belonging to a specific user |
| `findExpiredKeys` | Returns all enabled keys with an `expiresAt` timestamp before the provided `Instant` |

## Usage Example

```java
// Fetch a key or throw a domain exception
ApiKey key = apiKeyRepository.findByIdAndUserIdOrElseThrow("key-id-123", "user-id-456");

// List all keys for a user
List<ApiKey> userKeys = apiKeyRepository.findByUserId("user-id-456");

// Find all expired (but still enabled) keys — useful for cleanup jobs
List<ApiKey> expiredKeys = apiKeyRepository.findExpiredKeys(Instant.now());
```

> **Note:** `findExpiredKeys` only returns keys where `enabled: true`, so keys already disabled are excluded from expiry sweeps. Pair this with a scheduled task to bulk-disable or purge stale keys.
