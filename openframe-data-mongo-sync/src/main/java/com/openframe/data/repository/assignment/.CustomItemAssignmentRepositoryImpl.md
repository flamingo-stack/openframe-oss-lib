Custom MongoDB repository implementation for querying and paginating `ItemAssignment` documents, supporting cursor-based pagination, filtering by target type, full-text search, and grouped aggregation counts.

## Key Components

| Method | Description |
|---|---|
| `findAssignmentsWithCursor()` | Fetches a paginated list of assignments using cursor-based (keyset) pagination with configurable sort field and direction |
| `countAssignments()` | Returns total count of assignments matching itemId, targetType, and optional search term |
| `countByItemIdGroupedByTargetType()` | Aggregates assignment counts grouped by `AssignmentTargetType` for a given item |
| `buildAssignmentQuery()` | Constructs a base `Query` with required filters and optional case-insensitive display name search |
| `applyCursorCriteria()` | Appends cursor boundary conditions to the query, handling tie-breaking on `_id` when sort values collide |
| `isSortableField()` | Validates whether a given field name is in the allowed sort fields (`createdAt`, `displayName`) |
| `getDefaultSortField()` | Returns `createdAt` as the fallback sort field |

## Usage Example

```java
// Inject the repository (via Spring)
@Autowired
private CustomItemAssignmentRepository assignmentRepository;

// Paginate assignments with cursor
List<ItemAssignment> page = assignmentRepository.findAssignmentsWithCursor(
    "item-abc-123",             // itemId
    AssignmentTargetType.USER,  // targetType
    "john",                     // search term (display name, case-insensitive)
    "createdAt",                // sort field
    "DESC",                     // sort direction
    "64f1a2b3c4d5e6f7a8b9c0d1", // cursor (last seen document _id)
    20                          // page size
);

// Count grouped by type
Map<AssignmentTargetType, Long> counts =
    assignmentRepository.countByItemIdGroupedByTargetType("item-abc-123");
// e.g. { USER=42, DEVICE=7, GROUP=3 }

// Validate sort field before use
String sortField = assignmentRepository.isSortableField("displayName")
    ? "displayName"
    : assignmentRepository.getDefaultSortField();
```

> **Note:** Cursor pagination uses `_id` as a tiebreaker when two documents share the same sort field value, ensuring stable, consistent result ordering across pages.
