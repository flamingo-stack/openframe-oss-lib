MongoDB repository interface for managing `User` documents, providing standard CRUD operations alongside custom query methods for email-based lookups, role filtering, and status checks.

## Key Components

| Method | Description |
|---|---|
| `findByEmail(String email)` | Retrieves a user by email, returning an `Optional<User>` |
| `existsByEmail(String email)` | Checks if a user exists with the given email |
| `existsByEmailAndStatus(String email, UserStatus status)` | Validates email existence against a specific `UserStatus` |
| `findByRolesInAndStatus(Collection<UserRole> roles, UserStatus status)` | Fetches all users matching any of the provided roles and a given status |

## Inheritance

```text
MongoRepository<User, String>   → Standard CRUD + pagination
BaseUserRepository<...>         → Shared user query contracts
CustomUserRepository            → Custom query implementations
```

## Usage Example

```java
// Lookup user by email
Optional<User> user = userRepository.findByEmail("tech@flamingo.run");

// Check if active user exists
boolean isActive = userRepository.existsByEmailAndStatus(
    "tech@flamingo.run",
    UserStatus.ACTIVE
);

// Find all admins that are active
List<User> admins = userRepository.findByRolesInAndStatus(
    List.of(UserRole.ADMIN, UserRole.SUPER_ADMIN),
    UserStatus.ACTIVE
);
```

> **Note:** Query methods are resolved automatically by Spring Data MongoDB at runtime via method name convention — no implementation required.
