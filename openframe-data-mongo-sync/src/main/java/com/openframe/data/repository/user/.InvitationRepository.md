Spring Data MongoDB repository interface for managing `Invitation` documents, providing query methods for lookup by email/status and paginated filtering by status exclusion.

## Key Components

| Method | Description |
|--------|-------------|
| `findByEmailAndStatus(String email, String status)` | Retrieves a single invitation matching both email address and status value |
| `findByStatusNotIn(Collection<InvitationStatus> statuses, Pageable pageable)` | Returns a paginated list of invitations whose status is **not** in the provided collection |

## Usage Example

```java
// Look up a pending invitation by email
Optional<Invitation> pending = invitationRepository
    .findByEmailAndStatus("user@example.com", "PENDING");

pending.ifPresent(invitation -> {
    // process the invitation
});

// Fetch all non-terminal invitations (exclude ACCEPTED and REVOKED)
Collection<InvitationStatus> excludedStatuses = List.of(
    InvitationStatus.ACCEPTED,
    InvitationStatus.REVOKED
);

Pageable pageable = PageRequest.of(0, 20);
Page<Invitation> activeInvitations = invitationRepository
    .findByStatusNotIn(excludedStatuses, pageable);
```

## Notes

- Extends `MongoRepository<Invitation, String>`, inheriting standard CRUD operations with a `String` document ID.
- Query methods are resolved automatically by Spring Data at runtime — no implementation class required.
- The `status` parameter in `findByEmailAndStatus` accepts a raw `String`, while `findByStatusNotIn` uses the typed `InvitationStatus` enum for safer filtering.
