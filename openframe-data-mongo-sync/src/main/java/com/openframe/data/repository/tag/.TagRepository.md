Spring Data MongoDB repository interface for managing `Tag` documents, providing query methods to look up tags by entity type, key, and combinations thereof.

## Key Components

| Method | Description |
|--------|-------------|
| `findByEntityType` | Retrieves all tags for a given `TagEntityType` |
| `findByKeyAndEntityType` | Fetches a single tag matching both key and entity type |
| `findByKeyInAndEntityType` | Batch lookup of tags by a list of keys and entity type |
| `existsByKeyAndEntityType` | Checks existence of a tag without fetching the full document |
| `findByEntityTypeAndKeyContainingIgnoreCase` | Case-insensitive partial key search within an entity type |
| `findValuesByKeyAndEntityType` | Projecting query returning only the `values` field for a matched tag |

## Usage Example

```java
@Service
public class TagService {

    private final TagRepository tagRepository;

    public TagService(TagRepository tagRepository) {
        this.tagRepository = tagRepository;
    }

    public List<Tag> getTagsForEntity(TagEntityType entityType) {
        return tagRepository.findByEntityType(entityType);
    }

    public boolean tagExists(String key, TagEntityType entityType) {
        return tagRepository.existsByKeyAndEntityType(key, entityType);
    }

    public List<Tag> searchTags(TagEntityType entityType, String partialKey) {
        return tagRepository.findByEntityTypeAndKeyContainingIgnoreCase(entityType, partialKey);
    }

    public Tag getTagValues(String key, TagEntityType entityType) {
        // Returns only the 'values' field via MongoDB projection
        return tagRepository.findValuesByKeyAndEntityType(key, entityType);
    }
}
```

> The `findValuesByKeyAndEntityType` method uses a `@Query` projection (`{ 'values': 1 }`) to minimise data transfer when only tag values are needed, avoiding full document retrieval.
