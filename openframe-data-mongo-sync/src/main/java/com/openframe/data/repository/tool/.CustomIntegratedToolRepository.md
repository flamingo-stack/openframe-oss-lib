Custom repository interface for advanced MongoDB querying of `IntegratedTool` documents, supporting dynamic filtering, full-text search, sorting, and distinct field lookups beyond standard Spring Data capabilities.

## Key Components

| Method | Description |
|--------|-------------|
| `buildToolQuery` | Constructs a MongoDB `Query` object from a `ToolQueryFilter` and optional search string |
| `findToolsWithFilters` | Returns filtered and searched tools using default sort order |
| `findToolsWithFiltersAndSort` | Returns filtered and searched tools with explicit sort field and direction |
| `findDistinctTypes` | Retrieves all unique tool type values from the collection |
| `findDistinctCategories` | Retrieves all unique category values from the collection |
| `findDistinctPlatformCategories` | Retrieves all unique platform category values from the collection |
| `isSortableField` | Validates whether a given field name is eligible for sorting |
| `getDefaultSortField` | Returns the default field used for sorting results |

## Usage Example

```java
@Service
public class ToolService {

    private final CustomIntegratedToolRepository toolRepository;

    public ToolService(CustomIntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public List<IntegratedTool> searchTools(ToolQueryFilter filter, String search) {
        // Validate sort field before applying
        String sortField = toolRepository.isSortableField("name")
            ? "name"
            : toolRepository.getDefaultSortField();

        return toolRepository.findToolsWithFiltersAndSort(filter, search, sortField, "ASC");
    }

    public List<String> getAvailableCategories() {
        return toolRepository.findDistinctCategories();
    }
}
```

> **Note:** This interface follows the Spring Data custom repository pattern. The implementation class (`CustomIntegratedToolRepositoryImpl`) must be registered in the same package to be auto-detected by Spring.
