Custom MongoDB repository implementation for bulk-inserting `NotificationReadState` documents using unordered bulk operations, with idempotent duplicate-key error handling.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `MONGO_DUPLICATE_KEY` | `int` constant | MongoDB error code `11000` for duplicate key violations |
| `mongoTemplate` | `MongoTemplate` | Spring Data MongoDB template for executing bulk operations |
| `bulkInsertUnordered()` | Method | Inserts a list of `NotificationReadState` documents in an unordered bulk operation |
| `allDuplicateKeyErrors()` | Private helper | Returns `true` only if every error in a `BulkOperationException` is a duplicate key violation |

## Behavior Notes

- **Unordered mode** — MongoDB continues processing remaining documents even if individual inserts fail, maximizing throughput.
- **Idempotent inserts** — If all failures are duplicate-key errors (`11000`), the exception is silently swallowed and logged at `DEBUG` level, making repeated calls safe.
- **Partial failures** — If any error is *not* a duplicate key, the `BulkOperationException` is re-thrown to the caller.
- **Null/empty guard** — Returns early without hitting MongoDB if `rows` is `null` or empty.

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class NotificationService {

    private final CustomNotificationReadStateRepository readStateRepository;

    public void markNotificationsAsRead(List<NotificationReadState> states) {
        // Safe to call multiple times — duplicate keys are silently ignored
        readStateRepository.bulkInsertUnordered(states);
    }
}
```
