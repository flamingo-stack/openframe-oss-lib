Custom repository interface that extends MongoDB ticket querying capabilities beyond standard Spring Data methods, providing advanced filtering, cursor-based pagination, bulk operations, and aggregation support for `Ticket` documents.

## Key Components

| Method | Description |
|--------|-------------|
| `buildTicketQuery` | Constructs a MongoDB `Query` from a `TicketQueryFilter`, with optional full-text search, ticket ID restriction, and machine owner scoping |
| `findTicketsWithCursor` | Retrieves a paginated list of tickets using cursor-based pagination with configurable sort field and direction |
| `countTickets` | Returns the total count of tickets matching a given `Query` |
| `countTicketsByStatus` | Aggregates ticket counts grouped by `TicketStatus` |
| `countTicketsByStatusKind` | Aggregates ticket counts grouped by `TicketStatusKind` |
| `countTicketsByStatusId` | Aggregates ticket counts grouped by status ID string |
| `getAverageResolutionTimeMs` | Returns the optional average resolution time in milliseconds across resolved tickets |
| `updateStatusBulk` | Bulk-updates all tickets from one `TicketStatus` to another |
| `reassignTicketsToStatus` | Reassigns tickets from one status ID to another, updating the `TicketStatusKind` |
| `updateTitle` | Updates the title of a single ticket by ID |
| `isSortableField` | Validates whether a given field name is allowed for sorting |
| `getDefaultSortField` | Returns the default field used for ticket sorting |

## Usage Example

```java
// Build a filtered query with search and cursor-based pagination
Query query = ticketRepository.buildTicketQuery(
    filter,
    "network issue",       // full-text search term
    List.of("t1", "t2"),  // restrict to specific ticket IDs
    "machine-001"          // owner machine scope
);

List<Ticket> page = ticketRepository.findTicketsWithCursor(
    query,
    "last-cursor-id",
    25,
    ticketRepository.getDefaultSortField(),
    "DESC"
);

// Aggregate counts by status kind
Map<TicketStatusKind, Long> summary = ticketRepository.countTicketsByStatusKind();

// Bulk transition open tickets to closed
int updated = ticketRepository.updateStatusBulk(TicketStatus.OPEN, TicketStatus.CLOSED);
```
