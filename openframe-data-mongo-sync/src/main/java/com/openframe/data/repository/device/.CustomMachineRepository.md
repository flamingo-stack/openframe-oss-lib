Defines the custom repository interface for advanced Machine document queries in MongoDB, extending beyond standard Spring Data repository capabilities.

## Key Components

| Method | Description |
|--------|-------------|
| `buildDeviceQuery` | Constructs a MongoDB `Query` object from a `MachineQueryFilter` and optional search string |
| `findMachinesWithCursor` | Retrieves a paginated list of `Machine` documents using cursor-based pagination with configurable sort |
| `countMachines` | Returns the total count of `Machine` documents matching a given query |
| `isSortableField` | Validates whether a given field name is eligible for sorting |
| `getDefaultSortField` | Returns the default field used for sorting machine results |

## Usage Example

```java
// Build a filtered query
Query query = customMachineRepository.buildDeviceQuery(filter, "windows");

// Fetch paginated results using cursor-based navigation
List<Machine> machines = customMachineRepository.findMachinesWithCursor(
    query,
    "cursor_abc123",
    25,
    "hostname",
    "ASC"
);

// Count total matches for pagination metadata
long total = customMachineRepository.countMachines(query);

// Validate sort field before applying user input
if (customMachineRepository.isSortableField("lastSeen")) {
    // apply sort
}
```

## Notes

- Designed to be implemented alongside a standard Spring Data `MongoRepository` via the repository fragment pattern
- Cursor-based pagination avoids the performance pitfalls of offset pagination on large device datasets
- `MachineQueryFilter` encapsulates filtering logic, keeping query construction decoupled from service layers
