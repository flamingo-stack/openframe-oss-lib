Spring Data MongoDB repository interface for managing `ReleaseVersion` documents, designed to maintain a single document representing the current release version of the application.

## Key Components

| Component | Description |
|-----------|-------------|
| `ReleaseVersionRepository` | Extends `MongoRepository<ReleaseVersion, String>` providing standard CRUD operations |
| `findFirstByOrderByCreatedAtAsc()` | Retrieves the oldest (and expected sole) release version document, ordered by creation date ascending |

## Usage Example

```java
@Service
public class ReleaseVersionService {

    private final ReleaseVersionRepository releaseVersionRepository;

    public ReleaseVersionService(ReleaseVersionRepository releaseVersionRepository) {
        this.releaseVersionRepository = releaseVersionRepository;
    }

    public ReleaseVersion getCurrentVersion() {
        return releaseVersionRepository.findFirstByOrderByCreatedAtAsc()
                .orElseThrow(() -> new RuntimeException("No release version found"));
    }

    public ReleaseVersion initializeVersion(ReleaseVersion version) {
        return releaseVersionRepository.save(version);
    }
}
```

## Notes

- The repository follows the **singleton document pattern** — only one `ReleaseVersion` document is expected to exist in the collection at any time.
- `findFirstByOrderByCreatedAtAsc()` uses Spring Data's derived query mechanism, sorting by `createdAt` ascending to consistently return the earliest record if duplicates exist.
- Standard inherited methods from `MongoRepository` (e.g., `save()`, `delete()`, `findAll()`) are also available.
