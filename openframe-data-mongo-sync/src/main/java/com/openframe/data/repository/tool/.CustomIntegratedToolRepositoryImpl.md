Custom MongoDB repository implementation for querying and filtering `IntegratedTool` documents, providing dynamic query construction, sorting, and distinct value lookups.

## Key Components

| Method | Description |
|--------|-------------|
| `buildToolQuery` | Constructs a MongoDB `Query` from a `ToolQueryFilter` and optional search string |
| `findToolsWithFilters` | Returns tools matching filter and search criteria |
| `findToolsWithFiltersAndSort` | Returns filtered tools with configurable sort field and direction |
| `findDistinctTypes` | Returns all distinct, sorted tool type values |
| `findDistinctCategories` | Returns all distinct, sorted category values |
| `findDistinctPlatformCategories` | Returns all distinct, sorted platform category values |
| `isSortableField` | Validates whether a given field name is in the allowed sort whitelist |
| `getDefaultSortField` | Returns the default sort field (`_id`) |

**Sortable fields:** `_id`, `name`, `type`, `category`, `enabled`

## Usage Example

```java
// Inject the repository
@Autowired
private CustomIntegratedToolRepository toolRepository;

// Build a filter
ToolQueryFilter filter = new ToolQueryFilter();
filter.setEnabled(true);
filter.setCategory("monitoring");

// Retrieve filtered tools with sorting
List<IntegratedTool> tools = toolRepository.findToolsWithFiltersAndSort(
    filter,
    "grafana",       // search term (matches name or description)
    "name",          // sort field
    "ASC"            // sort direction
);

// Fetch distinct categories for dropdowns
List<String> categories = toolRepository.findDistinctCategories();

// Validate a user-supplied sort field before querying
String userField = "name";
String sortField = toolRepository.isSortableField(userField)
    ? userField
    : toolRepository.getDefaultSortField();
```

> **Note:** Search matching uses case-insensitive regex against both `name` and `description` fields. All `findDistinct*` methods strip null values and return alphabetically sorted results.
