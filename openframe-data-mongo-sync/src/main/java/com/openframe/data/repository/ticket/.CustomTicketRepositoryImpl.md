Custom MongoDB repository implementation for ticket querying, providing advanced filtering, cursor-based pagination, and aggregation-based counting operations on the `Ticket` collection.

## Key Components

| Method | Description |
|--------|-------------|
| `buildTicketQuery()` | Constructs a `Query` from a `TicketQueryFilter`, optional free-text search, ticket ID restrictions, and owner machine ID |
| `findTicketsWithCursor()` | Executes cursor-based pagination with configurable sort field/direction |
| `countTickets()` | Returns total count for a given query |
| `countTicketsByStatus()` | Aggregates ticket counts grouped by `TicketStatus` enum |
| `countTicketsByStatusKind()` | Aggregates ticket counts grouped by `TicketStatusKind` enum |
| `countTicketsByStatusId()` | Aggregates ticket counts grouped by raw status ID string |

**Internal helpers:**
- `applySearchCriteria()` — case-insensitive regex search across title, hostname, org name, and assignee name
- `applyCursorCriteria()` — builds stable keyset pagination criteria handling tie-breaking by `_id`
- `getSortFieldValue()` — reflective-style field extraction from a `Ticket` instance for cursor comparison

## Usage Example

```java
// Build a filtered, paginated query
TicketQueryFilter filter = TicketQueryFilter.builder()
    .statusKinds(List.of(TicketStatusKind.OPEN))
    .organizationIds(List.of("org-123"))
    .build();

Query query = repo.buildTicketQuery(filter, "network", null, null);

List<Ticket> page = repo.findTicketsWithCursor(
    query,
    "64f1a2b3c4d5e6f7a8b9c0d1", // cursor (last seen _id)
    25,                          // page size
    "createdAt",                 // sort field
    "DESC"                       // sort direction
);

// Aggregate counts by status
Map<TicketStatus, Long> statusCounts = repo.countTicketsByStatus();
// e.g. { OPEN=42, CLOSED=17, IN_PROGRESS=8 }
```

> **Note:** Cursor-based pagination uses keyset logic — when sorting by a non-ID field, ties are broken by `_id` to ensure stable, gap-free page traversal across concurrent writes.
