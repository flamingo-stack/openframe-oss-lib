Custom MongoDB repository implementation for the `Ticket` document, providing advanced query building, cursor-based pagination, aggregation analytics, and bulk update operations beyond standard Spring Data capabilities.

## Key Components

| Method | Description |
|--------|-------------|
| `buildTicketQuery()` | Constructs a dynamic MongoDB `Query` from filter criteria, search terms, ticket ID restrictions, and machine owner constraints |
| `findTicketsWithCursor()` | Executes cursor-based paginated queries with configurable sort field and direction |
| `countTickets()` | Returns total count for a pre-built query |
| `countTicketsByStatus()` | Aggregates ticket counts grouped by `TicketStatus` enum values |
| `getAverageResolutionTimeMs()` | Computes average resolution time (ms) across all resolved tickets using MongoDB aggregation |
| `updateStatusBulk()` | Bulk-updates ticket status from one state to another, returning modified count |
| `updateTitle()` | Updates the title of a single ticket by ID |
| `isSortableField()` | Validates whether a given field name is in the allowed sort fields list |

**Supported sortable fields:** `_id`, `ticketNumber`, `status`, `organizationName`, `assignedName`, `deviceHostname`, `createdAt`, `updatedAt`, `resolvedAt`, `order`

## Usage Example

```java
// Build a filtered query
TicketQueryFilter filter = TicketQueryFilter.builder()
        .statuses(List.of(TicketStatus.OPEN))
        .organizationIds(List.of("org-123"))
        .createdAtFrom(Instant.now().minus(7, ChronoUnit.DAYS))
        .build();

Query query = repo.buildTicketQuery(filter, "printer", null, null);

// Fetch first page (cursor-based)
List<Ticket> page1 = repo.findTicketsWithCursor(query, null, 20, "createdAt", "DESC");

// Fetch next page using last ticket's ID as cursor
List<Ticket> page2 = repo.findTicketsWithCursor(query, page1.getLast().getId(), 20, "createdAt", "DESC");

// Analytics
Map<TicketStatus, Long> counts = repo.countTicketsByStatus();
Optional<Long> avgMs = repo.getAverageResolutionTimeMs();

// Bulk status transition
int updated = repo.updateStatusBulk(TicketStatus.OPEN, TicketStatus.CLOSED);
```

> **Cursor pagination note:** When sorting by a non-ID field, the cursor fetches the pivot document to retrieve its sort value, then applies a compound OR predicate (`past sort value OR same sort value + past ID`) to ensure stable, gap-free pagination.
