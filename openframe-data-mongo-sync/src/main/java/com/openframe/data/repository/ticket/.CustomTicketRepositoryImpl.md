Custom MongoDB repository implementation for the `Ticket` document, providing advanced querying, cursor-based pagination, filtering, and aggregation operations beyond standard Spring Data capabilities.

## Key Components

| Method | Description |
|--------|-------------|
| `buildTicketQuery` | Constructs a `Query` from a `TicketQueryFilter`, optional search string, ticket ID restrictions, and owner machine ID |
| `findTicketsWithCursor` | Executes cursor-based pagination with configurable sort field and direction |
| `countTickets` | Returns the total count for a given query |
| `countTicketsByStatus` | Aggregates ticket counts grouped by `TicketStatus` enum |
| `countTicketsByStatusKind` | Aggregates ticket counts grouped by `TicketStatusKind` enum |
| `countTicketsByStatusId` | Aggregates ticket counts grouped by raw status ID string |

**Internal helpers:**

- `applySearchCriteria` — Applies case-insensitive regex OR search across `title`, `deviceHostname`, `organizationName`, and `assignedName`
- `applyCursorCriteria` — Builds tie-breaking cursor logic using the sort field value + `_id` for stable pagination
- `getSortFieldValue` — Reflectively maps sort field names to `Ticket` getter values

## Usage Example

```java
// Build a filtered, paginated query
TicketQueryFilter filter = TicketQueryFilter.builder()
        .statusKinds(List.of(TicketStatusKind.OPEN))
        .organizationIds(List.of("org-123"))
        .build();

Query query = repo.buildTicketQuery(filter, "printer", null, null);

// Fetch first page (no cursor)
List<Ticket> page1 = repo.findTicketsWithCursor(query, null, 20, "createdAt", "DESC");

// Fetch next page using last document's ObjectId as cursor
List<Ticket> page2 = repo.findTicketsWithCursor(query, "64f1a2b3c4d5e6f7a8b9c0d1", 20, "createdAt", "DESC");

// Count by status
Map<TicketStatus, Long> statusCounts = repo.countTicketsByStatus();
// → { OPEN=42, CLOSED=15, IN_PROGRESS=7 }
```

> **Note:** Cursor pagination uses a compound tie-breaker (`sortField` + `_id`) to guarantee stable, duplicate-free pages even when sort field values are non-unique.
