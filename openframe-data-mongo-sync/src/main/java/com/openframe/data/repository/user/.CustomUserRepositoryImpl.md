Custom MongoDB repository implementation for querying `User` documents using dynamic filters, supporting regex-based search on email and name fields with optional status filtering.

## Key Components

| Component | Description |
|---|---|
| `findUsersBySearch(UserQueryFilter, int)` | Executes a filtered, limited, and sorted MongoDB query returning a list of `User` documents |
| `buildQuery(UserQueryFilter)` | Constructs a dynamic `Query` object from a `UserQueryFilter`, applying criteria conditionally |
| `UserQueryFilter` | Filter DTO carrying optional `status`, `emailRegex`, and `nameRegex` fields |
| `MongoTemplate` | Spring Data MongoDB low-level template used to execute the built query |

### Query Behavior

- Results are always sorted by `createdAt` **descending**
- Email and name regex matches are **case-insensitive**
- Name regex applies an **OR** across `firstName` and `lastName`
- A `null` filter returns all documents (up to `limit`)

## Usage Example

```java
// Build a filter to find active users matching "john" in name or email
UserQueryFilter filter = UserQueryFilter.builder()
        .status(UserStatus.ACTIVE)
        .nameRegex("john")
        .emailRegex("john")
        .build();

// Execute with a max of 50 results
List<User> users = customUserRepository.findUsersBySearch(filter, 50);
```

```java
// Name-only search, no status filter
UserQueryFilter nameFilter = UserQueryFilter.builder()
        .nameRegex("^doe")
        .build();

List<User> users = customUserRepository.findUsersBySearch(nameFilter, 10);
```

> **Note:** Regex patterns follow standard MongoDB regex syntax. Use anchors (`^`, `$`) for prefix/exact matching and avoid overly broad patterns on large collections for performance reasons.
