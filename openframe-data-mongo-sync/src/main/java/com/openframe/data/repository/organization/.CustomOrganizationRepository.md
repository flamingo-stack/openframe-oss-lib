Custom repository interface for MongoDB-based organization queries, providing dynamic filtering, cursor-based pagination, and sorting capabilities beyond standard Spring Data repository methods.

## Key Components

| Method | Description |
|--------|-------------|
| `buildOrganizationQuery` | Constructs a MongoDB `Query` object from an `OrganizationQueryFilter` and optional search string |
| `findOrganizationsWithFilters` | Returns a filtered list of `Organization` documents using the provided filter and search criteria |
| `findOrganizationsWithCursor` | Fetches a paginated slice of organizations using cursor-based pagination with configurable sort field, direction, and limit |
| `isSortableField` | Validates whether a given field name is eligible for sorting |
| `getDefaultSortField` | Returns the default field used for sorting when none is specified |

## Usage Example

```java
// Inject via Spring — typically paired with OrganizationRepository
@Autowired
private CustomOrganizationRepository customOrgRepo;

// Build and reuse a query
OrganizationQueryFilter filter = new OrganizationQueryFilter();
filter.setStatus("ACTIVE");

Query query = customOrgRepo.buildOrganizationQuery(filter, "acme");

// Cursor-based pagination — fetch the next 25 orgs sorted by name
List<Organization> page = customOrgRepo.findOrganizationsWithCursor(
    query,
    "lastSeenOrgId",   // cursor value from previous page
    25,
    customOrgRepo.getDefaultSortField(),
    "ASC"
);

// Validate sort field before accepting user input
if (customOrgRepo.isSortableField(userProvidedSortField)) {
    // safe to sort
}
```

> **Note:** This interface follows the Spring Data custom repository fragment pattern. The implementation class (`CustomOrganizationRepositoryImpl`) is auto-detected by Spring and composed into the main `OrganizationRepository`.
