Spring Data MongoDB repository interface for `Organization` document persistence, providing standard CRUD operations alongside custom query methods for organization lookup and batch retrieval.

## Key Components

| Method | Description |
|--------|-------------|
| `findByOrganizationId(String)` | Looks up a single organization by its unique identifier |
| `findByName(String)` | Retrieves an organization matching the given name |
| `findByIsDefaultTrue()` | Fetches the designated default organization |
| `findByOrganizationIdIn(Set<String>)` | Batch loads multiple organizations by a set of IDs |

**Extends:**
- `MongoRepository<Organization, String>` — inherited CRUD and pagination operations
- `CustomOrganizationRepository` — project-specific query extensions

## Usage Example

```java
@Service
public class OrganizationService {

    private final OrganizationRepository organizationRepository;

    public OrganizationService(OrganizationRepository organizationRepository) {
        this.organizationRepository = organizationRepository;
    }

    // Lookup by unique ID
    public Organization getById(String orgId) {
        return organizationRepository.findByOrganizationId(orgId)
                .orElseThrow(() -> new NotFoundException("Organization not found: " + orgId));
    }

    // Fetch default org
    public Organization getDefault() {
        return organizationRepository.findByIsDefaultTrue()
                .orElseThrow(() -> new NotFoundException("No default organization configured"));
    }

    // Batch load
    public List<Organization> getByIds(Set<String> ids) {
        return organizationRepository.findByOrganizationIdIn(ids);
    }
}
```

> **Note:** Query methods follow Spring Data naming conventions and are auto-implemented at runtime — no manual query definitions required.
