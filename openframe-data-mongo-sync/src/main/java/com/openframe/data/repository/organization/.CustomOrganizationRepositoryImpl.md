Custom MongoDB repository implementation for querying `Organization` documents with dynamic filtering, text search, and cursor-based pagination support.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `SORTABLE_FIELDS` | `List<String>` | Whitelist of fields allowed for sorting (`_id`, `name`, `organizationId`, `createdAt`, `updatedAt`) |
| `DEFAULT_SORT_FIELD` | `String` | Fallback sort field (`_id`) |
| `buildOrganizationQuery` | Method | Constructs a MongoDB `Query` from an `OrganizationQueryFilter` and optional search string |
| `findOrganizationsWithFilters` | Method | Executes the built query and returns matching `Organization` documents |
| `findOrganizationsWithCursor` | Method | Applies cursor-based pagination, limit, and sort to an existing query |
| `isSortableField` | Method | Validates whether a given field name is in the sort whitelist |
| `getDefaultSortField` | Method | Returns the default sort field string |

## Usage Example

```java
// Inject the repository
@Autowired
private CustomOrganizationRepository orgRepository;

// Build and execute a filtered query
OrganizationQueryFilter filter = OrganizationQueryFilter.builder()
    .status("ACTIVE")
    .category("Technology")
    .minEmployees(10)
    .hasActiveContract(true)
    .build();

List<Organization> results = orgRepository.findOrganizationsWithFilters(filter, "acme");

// Cursor-based pagination (e.g. second page)
Query query = orgRepository.buildOrganizationQuery(filter, "acme");
List<Organization> page = orgRepository.findOrganizationsWithCursor(
    query,
    "64f1a2b3c4d5e6f7a8b9c0d1", // last seen _id as cursor
    25,                          // page size
    "createdAt",
    "DESC"
);
```

## Filter Behavior Notes

- **Status**: defaults to `ACTIVE` (or documents with no status field) when not explicitly set
- **Category**: matched case-insensitively via regex
- **Search**: applied as case-insensitive regex across `name`, `organizationId`, and `category`
- **Active contract**: checks `contractStartDate ≤ now ≤ contractEndDate` at query time
- All criteria are combined using MongoDB `$and` for predictable compound filtering
