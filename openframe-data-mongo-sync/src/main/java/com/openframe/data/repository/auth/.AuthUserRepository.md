MongoDB repository interface for managing `AuthUser` documents, providing query methods for user lookup by email, tenant, and status combinations, as well as tenant-scoped deletion.

## Key Components

| Method | Description |
|--------|-------------|
| `findByEmailAndStatus` | Finds an active user by email regardless of tenant |
| `findByEmailAndTenantIdAndStatus` | Finds an active user by email within a specific tenant |
| `existsByEmailAndTenantId` | Checks if a user exists for a given email and tenant |
| `findByEmailAndTenantId` | Finds a user by email and tenant regardless of status |
| `deleteByTenantId` | Removes all users belonging to a specific tenant |

## Usage Example

```java
@Service
public class AuthService {

    private final AuthUserRepository authUserRepository;

    // Find an active user by email (global lookup)
    public AuthUser getActiveUser(String email) {
        return authUserRepository
            .findByEmailAndStatus(email, UserStatus.ACTIVE)
            .orElseThrow(() -> new UserNotFoundException(email));
    }

    // Find an active user scoped to a tenant
    public AuthUser getTenantUser(String email, String tenantId) {
        return authUserRepository
            .findByEmailAndTenantIdAndStatus(email, tenantId, UserStatus.ACTIVE)
            .orElseThrow(() -> new UserNotFoundException(email));
    }

    // Check for duplicate users before registration
    public boolean isEmailTaken(String email, String tenantId) {
        return authUserRepository.existsByEmailAndTenantId(email, tenantId);
    }

    // Remove all users when a tenant is deprovisioned
    public void removeTenant(String tenantId) {
        authUserRepository.deleteByTenantId(tenantId);
    }
}
```

> **Note:** Methods returning `Optional<AuthUser>` should always use `.orElseThrow()` or `.orElse()` to handle the case where no matching user is found, avoiding `NoSuchElementException` at runtime.
