MongoDB repository interface for managing `IntegratedToolAgent` documents, providing data access operations for tool agents within the OpenFrame platform.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `IntegratedToolAgentRepository` | Interface | Spring Data MongoDB repository extending `MongoRepository<IntegratedToolAgent, String>` |
| `findByStatus` | Query Method | Retrieves all tool agents matching a given `ToolAgentStatus` |
| `findByReleaseVersionTrue` | Query Method | Retrieves all tool agents flagged as release versions |

## Usage Example

```java
@Service
public class ToolAgentService {

    private final IntegratedToolAgentRepository repository;

    public ToolAgentService(IntegratedToolAgentRepository repository) {
        this.repository = repository;
    }

    // Fetch all active tool agents
    public List<IntegratedToolAgent> getActiveAgents() {
        return repository.findByStatus(ToolAgentStatus.ACTIVE);
    }

    // Fetch only release-ready agents
    public List<IntegratedToolAgent> getReleaseAgents() {
        return repository.findByReleaseVersionTrue();
    }
}
```

## Notes

- Annotated with `@Repository`, enabling Spring component scanning and exception translation.
- Inherits standard CRUD operations (`save`, `findById`, `deleteById`, etc.) from `MongoRepository`.
- Query methods follow Spring Data naming conventions — no manual query definitions required.
- `ToolAgentStatus` is an enum used to filter agents by lifecycle state (e.g., `ACTIVE`, `INACTIVE`).
