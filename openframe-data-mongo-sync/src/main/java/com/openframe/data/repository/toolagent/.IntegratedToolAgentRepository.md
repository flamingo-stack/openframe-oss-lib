MongoDB repository interface for managing `IntegratedToolAgent` documents, providing CRUD operations and custom queries for tool agent lifecycle management.

## Key Components

| Method | Description |
|--------|-------------|
| `findByStatus(ToolAgentStatus status)` | Retrieves all tool agents matching a given status (e.g., `ACTIVE`, `INACTIVE`) |
| `findByReleaseVersionTrue()` | Returns all tool agents flagged as release versions |
| `findByKey(String key)` | Looks up a single tool agent by its unique key identifier |

Extends `MongoRepository<IntegratedToolAgent, String>`, inheriting standard CRUD operations (`save`, `findById`, `deleteById`, etc.).

## Usage Example

```java
@Service
public class ToolAgentService {

    private final IntegratedToolAgentRepository repository;

    public ToolAgentService(IntegratedToolAgentRepository repository) {
        this.repository = repository;
    }

    public List<IntegratedToolAgent> getActiveAgents() {
        return repository.findByStatus(ToolAgentStatus.ACTIVE);
    }

    public List<IntegratedToolAgent> getReleasedAgents() {
        return repository.findByReleaseVersionTrue();
    }

    public Optional<IntegratedToolAgent> findAgent(String key) {
        return repository.findByKey(key);
    }
}
```

> **Note:** This repository is backed by MongoDB via Spring Data. The `@Repository` annotation enables Spring's exception translation, converting MongoDB-specific exceptions into Spring's `DataAccessException` hierarchy.
