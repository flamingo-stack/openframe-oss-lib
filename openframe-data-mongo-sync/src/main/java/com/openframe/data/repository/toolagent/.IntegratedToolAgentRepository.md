MongoDB repository interface for managing `IntegratedToolAgent` documents, providing query methods for filtering agents by status, release version, and unique key.

## Key Components

| Method | Description |
|--------|-------------|
| `findByStatus(ToolAgentStatus status)` | Retrieves all tool agents matching a given `ToolAgentStatus` enum value |
| `findByReleaseVersionTrue()` | Returns all agents flagged as a release version |
| `findByKey(String key)` | Looks up a single agent by its unique string key, returning an `Optional` |

## Usage Example

```java
@Service
public class ToolAgentService {

    private final IntegratedToolAgentRepository repository;

    public ToolAgentService(IntegratedToolAgentRepository repository) {
        this.repository = repository;
    }

    public List<IntegratedToolAgent> getActiveAgents() {
        return repository.findByStatus(ToolAgentStatus.ACTIVE);
    }

    public List<IntegratedToolAgent> getReleasedAgents() {
        return repository.findByReleaseVersionTrue();
    }

    public Optional<IntegratedToolAgent> getAgentByKey(String key) {
        return repository.findByKey(key);
    }
}
```

> Standard CRUD operations (`save`, `findById`, `deleteById`, etc.) are inherited from `MongoRepository<IntegratedToolAgent, String>` and require no additional implementation.
