MongoDB repository interface for persisting and querying `ExternalApplicationEvent` documents, providing time-range lookups by user and tag-based filtering by event type.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `findByUserIdAndTimestampBetween` | Derived Query | Retrieves events for a specific user within a start/end `Instant` range |
| `findByTypeAndTags` | `@Query` | Fetches events matching a given type and metadata tag map using a native MongoDB query |

## Usage Example

```java
@Service
public class EventService {

    private final ExternalApplicationEventRepository eventRepository;

    public EventService(ExternalApplicationEventRepository eventRepository) {
        this.eventRepository = eventRepository;
    }

    // Fetch events for a user within the last 24 hours
    public List<ExternalApplicationEvent> getRecentUserEvents(String userId) {
        Instant end = Instant.now();
        Instant start = end.minus(24, ChronoUnit.HOURS);
        return eventRepository.findByUserIdAndTimestampBetween(userId, start, end);
    }

    // Fetch events by type and metadata tags
    public List<ExternalApplicationEvent> getTaggedEvents(String type) {
        Map<String, String> tags = Map.of("source", "integration", "env", "production");
        return eventRepository.findByTypeAndTags(type, tags);
    }
}
```

> **Note:** The `findByTypeAndTags` method uses a native MongoDB query targeting `metadata.tags` — ensure the `ExternalApplicationEvent` document includes a properly indexed `metadata` field for efficient querying at scale.
