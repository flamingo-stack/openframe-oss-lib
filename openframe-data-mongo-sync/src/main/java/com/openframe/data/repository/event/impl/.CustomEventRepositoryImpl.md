Custom MongoDB repository implementation for querying and paginating `Event` documents using cursor-based pagination, dynamic filtering, and field-level sorting.

## Key Components

| Method | Description |
|--------|-------------|
| `buildEventQuery` | Constructs a MongoDB `Query` from an `EventQueryFilter` (user IDs, event types, date range) and an optional full-text search pattern |
| `findEventsWithCursor` | Executes cursor-based pagination using MongoDB `ObjectId` comparisons (`$lt`/`$gt`), with configurable sort field and direction |
| `findDistinctUserIds` | Returns a sorted, deduplicated list of all `userId` values across the events collection |
| `findDistinctEventTypes` | Returns a sorted, deduplicated list of all `type` values across the events collection |
| `isSortableField` | Validates whether a given field name is in the allowed sortable fields allowlist (`_id`, `type`, `userId`, `timestamp`) |
| `getDefaultSortField` | Returns the default sort field (`_id`) |

## Usage Example

```java
// Build a filtered query
EventQueryFilter filter = EventQueryFilter.builder()
        .userIds(List.of("user123"))
        .eventTypes(List.of("LOGIN", "LOGOUT"))
        .startDate(LocalDate.of(2024, 1, 1))
        .endDate(LocalDate.of(2024, 12, 31))
        .build();

Query query = repository.buildEventQuery(filter, "login");

// Paginate using cursor
String sortField = repository.isSortableField("timestamp")
        ? "timestamp"
        : repository.getDefaultSortField();

List<Event> events = repository.findEventsWithCursor(
        query,
        "65f1a2b3c4d5e6f7a8b9c0d1", // cursor (last seen ObjectId)
        20,                           // page size
        sortField,
        "DESC"
);

// Fetch filter metadata
List<String> userIds   = repository.findDistinctUserIds();
List<String> eventTypes = repository.findDistinctEventTypes();
```

> **Note:** The cursor must be a valid MongoDB `ObjectId` hex string. Invalid cursors are logged as warnings and ignored, falling back to an unconstrained query.
