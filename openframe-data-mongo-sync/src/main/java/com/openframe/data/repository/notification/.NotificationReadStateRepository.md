MongoDB repository interface for managing notification read states, providing CRUD operations and bulk update capabilities for tracking per-recipient notification status.

## Key Components

| Method | Description |
|--------|-------------|
| `existsByRecipientIdAndRecipientTypeAndStatus` | Checks if a read state record exists for a given recipient and status |
| `markAsRead` | Marks a specific notification as `READ` for a recipient, setting `readAt` timestamp |
| `markAllAsRead` | Bulk-marks all `UNREAD` notifications as `READ` for a recipient |
| `softDelete` | Soft-deletes a specific notification by setting status to `DELETED` (skips already deleted) |
| `softDeleteAllRead` | Bulk soft-deletes all `READ` notifications for a recipient |
| `unreadCountsByCategory` | Aggregates unread notification counts grouped by category for a recipient |

Extends both `MongoRepository<NotificationReadState, String>` and `CustomNotificationReadStateRepository` for standard CRUD plus custom query support.

## Usage Example

```java
// Check if any unread notifications exist
boolean hasUnread = repository.existsByRecipientIdAndRecipientTypeAndStatus(
    "user-123", RecipientType.USER, ReadStatus.UNREAD
);

// Mark a single notification as read
long updated = repository.markAsRead("user-123", RecipientType.USER, "notif-456");

// Mark all notifications as read
long totalMarked = repository.markAllAsRead("user-123", RecipientType.USER);

// Soft-delete a specific notification
long deleted = repository.softDelete("user-123", RecipientType.USER, "notif-456");

// Get unread counts per category
List<CategoryCount> counts = repository.unreadCountsByCategory("user-123", RecipientType.USER);
```

> **Note:** All mutating methods (`markAsRead`, `markAllAsRead`, `softDelete`, `softDeleteAllRead`) use MongoDB's `$set` operator via `@Update`, returning the count of modified documents. Timestamps use MongoDB's `$$NOW` server-side expression for consistency.
