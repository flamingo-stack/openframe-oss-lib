Custom repository interface for MongoDB-backed event querying, providing advanced filtering, cursor-based pagination, and field discovery beyond standard Spring Data repository capabilities.

## Key Components

| Method | Description |
|--------|-------------|
| `buildEventQuery` | Constructs a MongoDB `Query` object from an `EventQueryFilter` and optional search string |
| `findEventsWithCursor` | Retrieves a paginated list of `Event` documents using cursor-based pagination with configurable sort field and direction |
| `findDistinctUserIds` | Returns all unique user IDs present across event documents |
| `findDistinctEventTypes` | Returns all unique event type identifiers present across event documents |
| `isSortableField` | Validates whether a given field name is eligible for sorting |
| `getDefaultSortField` | Returns the default field used for sorting event queries |

## Usage Example

```java
// Inject the repository in your service
@Autowired
private CustomEventRepository customEventRepository;

// Build a filtered query
EventQueryFilter filter = new EventQueryFilter();
filter.setEventType("USER_LOGIN");
filter.setStartDate(LocalDateTime.now().minusDays(7));

Query query = customEventRepository.buildEventQuery(filter, "admin");

// Fetch paginated results using cursor
String cursor = null; // null for first page
int limit = 25;
String sortField = customEventRepository.getDefaultSortField();

if (customEventRepository.isSortableField(sortField)) {
    List<Event> events = customEventRepository.findEventsWithCursor(
        query, cursor, limit, sortField, "DESC"
    );
}

// Retrieve available filter options
List<String> userIds = customEventRepository.findDistinctUserIds();
List<String> eventTypes = customEventRepository.findDistinctEventTypes();
```

> This interface follows the Spring Data custom repository pattern and should be implemented alongside a concrete class (e.g., `CustomEventRepositoryImpl`) that uses `MongoTemplate` for query execution.
