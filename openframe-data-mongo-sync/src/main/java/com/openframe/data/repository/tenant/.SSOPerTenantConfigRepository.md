Spring Data MongoDB repository interface for managing per-tenant SSO (Single Sign-On) configurations, providing query methods to retrieve and manage SSO settings scoped by tenant, provider, and domain.

## Key Components

| Method | Description |
|--------|-------------|
| `findByTenantIdAndEnabledTrue` | Retrieves all active SSO configurations for a specific tenant |
| `findByProvider` | Looks up an SSO config by its identity provider (e.g., Google, Azure AD) |
| `findFirstByTenantIdAndProviderAndEnabledTrue` | Fetches the first active SSO config matching both tenant and provider |
| `findByAllowedDomainsIn` | Queries configs that permit any of the specified email domains |
| `deleteByTenantId` | Removes all SSO configurations associated with a given tenant |

## Usage Example

```java
// Retrieve all enabled SSO configurations for a tenant
List<SSOPerTenantConfig> activeConfigs = ssoRepo
    .findByTenantIdAndEnabledTrue("tenant-abc");

// Find a config by provider
Optional<SSOPerTenantConfig> googleConfig = ssoRepo
    .findByProvider("google");

// Find the first active config for a specific tenant + provider combination
Optional<SSOPerTenantConfig> config = ssoRepo
    .findFirstByTenantIdAndProviderAndEnabledTrue("tenant-abc", "azure-ad");

// Find configs that allow login from specific domains
List<SSOPerTenantConfig> domainConfigs = ssoRepo
    .findByAllowedDomainsIn(List.of("acme.com", "partner.org"));

// Remove all SSO configs when offboarding a tenant
ssoRepo.deleteByTenantId("tenant-abc");
```

## Notes

- Extends `MongoRepository<SSOPerTenantConfig, String>`, inheriting standard CRUD operations
- All query methods are resolved at runtime via Spring Data's method name convention — no manual query implementation required
- Domain-based lookup (`findByAllowedDomainsIn`) supports multi-domain SSO scenarios where a single tenant may allow multiple email domains
