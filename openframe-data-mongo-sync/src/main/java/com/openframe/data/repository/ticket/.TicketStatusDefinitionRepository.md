MongoDB repository interface for managing `TicketStatusDefinition` documents, providing CRUD operations and custom queries for ticket status configurations in the OpenFrame platform.

## Key Components

| Method | Description |
|--------|-------------|
| `findAllByOrderByPositionAsc()` | Retrieves all status definitions sorted by position |
| `findByKind(TicketStatusKind kind)` | Looks up a single status definition by its kind enum |
| `findByName(String name)` | Looks up a single status definition by name |
| `findByKindOrderByPositionAsc(TicketStatusKind kind)` | Retrieves all statuses of a specific kind, ordered by position |
| `countByKind(TicketStatusKind kind)` | Returns the count of statuses for a given kind |
| `existsByName(String name)` | Checks whether a status definition with the given name exists |

## Usage Example

```java
@Service
public class TicketStatusService {

    private final TicketStatusDefinitionRepository statusRepository;

    public TicketStatusService(TicketStatusDefinitionRepository statusRepository) {
        this.statusRepository = statusRepository;
    }

    public List<TicketStatusDefinition> getOpenStatuses() {
        // Fetch all "OPEN" kind statuses in display order
        return statusRepository.findByKindOrderByPositionAsc(TicketStatusKind.OPEN);
    }

    public void createStatus(TicketStatusDefinition newStatus) {
        // Guard against duplicate names before persisting
        if (statusRepository.existsByName(newStatus.getName())) {
            throw new IllegalArgumentException("Status name already exists: " + newStatus.getName());
        }
        statusRepository.save(newStatus);
    }
}
```

## Notes

- Extends `MongoRepository<TicketStatusDefinition, String>` — inherits full Spring Data CRUD and pagination support
- Query methods are resolved at runtime by Spring Data MongoDB via method name conventions — no manual query definitions needed
- `TicketStatusKind` is an enum that categorizes statuses (e.g., `OPEN`, `CLOSED`, `PENDING`)
