Spring Data MongoDB repository interface for managing `TicketStatusDefinition` documents, providing CRUD operations and custom queries for ticket status configurations.

## Key Components

| Method | Description |
|--------|-------------|
| `findAllByOrderByPositionAsc()` | Retrieves all status definitions sorted by display position |
| `findByKind(TicketStatusKind)` | Looks up a single status definition by its kind enum value |
| `findByName(String)` | Looks up a single status definition by name |
| `findByKindOrderByPositionAsc(TicketStatusKind)` | Retrieves all statuses of a given kind, ordered by position |
| `countByKind(TicketStatusKind)` | Counts how many definitions exist for a specific kind |
| `existsByName(String)` | Checks whether a status definition with the given name already exists |

## Usage Example

```java
@Service
public class TicketStatusService {

    private final TicketStatusDefinitionRepository repository;

    public TicketStatusService(TicketStatusDefinitionRepository repository) {
        this.repository = repository;
    }

    public List<TicketStatusDefinition> getOpenStatuses() {
        // Retrieve all statuses of kind OPEN, ordered by position
        return repository.findByKindOrderByPositionAsc(TicketStatusKind.OPEN);
    }

    public void createStatus(TicketStatusDefinition definition) {
        // Guard against duplicate names before persisting
        if (repository.existsByName(definition.getName())) {
            throw new IllegalArgumentException("Status name already exists: " + definition.getName());
        }
        repository.save(definition);
    }
}
```

## Notes

- Extends `MongoRepository<TicketStatusDefinition, String>`, inheriting full CRUD and pagination support.
- The `position` field governs display ordering across multiple query methods, ensuring consistent UI rendering of ticket status pipelines in OpenFrame.
