Custom repository interface defining advanced query operations for `KnowledgeBaseItem` documents beyond standard CRUD, supporting filtered browsing, pagination, and archival access patterns.

## Key Components

| Method | Description |
|--------|-------------|
| `findFoldersForParent` | Retrieves folder items under a given parent, optionally filtered by search term and item ID allowlist |
| `findArticles` | Paginated article query with filters for parent, type, status list, and cursor-based pagination |
| `countArticles` | Returns total count matching the same filter criteria as `findArticles` (without pagination) |
| `findArchivedArticles` | Fetches archived articles with cursor-based pagination and optional search/ID filters |
| `countArchivedArticles` | Returns count of archived articles matching search and ID filters |
| `findAllArticles` | Retrieves all non-archived articles with no filtering |

## Usage Example

```java
// Injected via Spring Data repository composition
@Autowired
private KnowledgeBaseItemRepository repository; // extends CustomKnowledgeBaseItemRepository

// Paginated article lookup by status
List<KnowledgeBaseItem> articles = repository.findArticles(
    "folder-123",                          // parentId
    "vpn setup",                           // search term
    KnowledgeBaseItemType.ARTICLE,         // type filter
    List.of("item-1", "item-2"),           // allowlisted IDs
    List.of(KnowledgeBaseArticleStatus.PUBLISHED), // status filter
    "cursor-abc",                          // pagination cursor
    20                                     // page size
);

// Count before paginating
long total = repository.countArticles(
    "folder-123", "vpn setup",
    KnowledgeBaseItemType.ARTICLE,
    List.of("item-1", "item-2"),
    List.of(KnowledgeBaseArticleStatus.PUBLISHED)
);
```

> **Note:** This interface is typically paired with a `CustomKnowledgeBaseItemRepositoryImpl` providing the MongoDB query implementation, and composed into the main Spring Data repository interface.
