MongoDB repository interface for managing `AuthInvitation` documents, providing tenant-scoped query and cleanup operations for user invitation workflows.

## Key Components

| Method | Description |
|--------|-------------|
| `findByTenantIdAndId` | Retrieves a single invitation by tenant and invitation ID |
| `findByTenantIdAndEmailAndStatus` | Looks up an invitation for a specific email/status combination within a tenant |
| `findByTenantIdAndStatusOrderByCreatedAtDesc` | Lists all invitations for a tenant filtered by status, sorted newest first |
| `deleteByTenantIdAndExpiresAtBefore` | Purges expired invitations older than a given cutoff timestamp |
| `deleteByTenantId` | Removes all invitations belonging to a tenant (e.g. tenant offboarding) |

## Usage Example

```java
// Check for a pending invitation before sending a duplicate
Optional<AuthInvitation> existing = authInvitationRepository
    .findByTenantIdAndEmailAndStatus(tenantId, "user@example.com", "PENDING");

if (existing.isEmpty()) {
    // Proceed to create and save a new invitation
}

// List all pending invitations for a tenant dashboard
List<AuthInvitation> pending = authInvitationRepository
    .findByTenantIdAndStatusOrderByCreatedAtDesc(tenantId, "PENDING");

// Clean up expired invitations during a scheduled job
authInvitationRepository
    .deleteByTenantIdAndExpiresAtBefore(tenantId, Instant.now());
```

> All queries are tenant-scoped, enforcing data isolation across the multi-tenant OpenFrame platform. The expiry cleanup method is well-suited for use in a scheduled `@Scheduled` maintenance task.
