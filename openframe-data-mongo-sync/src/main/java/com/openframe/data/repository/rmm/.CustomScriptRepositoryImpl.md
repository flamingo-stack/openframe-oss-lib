Implements cursor-based pagination queries for `Script` documents in MongoDB, providing tenant-scoped filtering, sorting, and search capabilities backed by `MongoTemplate`.

## Key Components

| Member | Description |
|--------|-------------|
| `findPageForTenant()` | Core pagination method — builds a `Query` with all active filters and executes it against MongoDB |
| `applyStatusFilter()` | Filters by script status; defaults to excluding `DELETED` documents via `$ne` |
| `applyShellsFilter()` | Restricts results to specified shell types |
| `applyPlatformsFilter()` | Matches scripts whose `supportedPlatforms` array contains any of the requested values |
| `applyTagFilter()` | Case-insensitive exact tag match using an anchored, quoted regex |
| `applySearch()` | Case-insensitive substring match on `name` using `Pattern.quote` to escape user input |
| `applyCursor()` | Parses a hex `ObjectId` cursor and applies `$lt`/`$gt` based on sort + pagination direction |
| `isSortableField()` | Validates a field against the allowlist: `_id`, `name`, `createdAt`, `updatedAt` |
| `getDefaultSortField()` | Returns `_id` as the fallback sort field |
| `flip()` | Inverts sort direction for backward pagination |

## Usage Example

```java
List<Script> page = customScriptRepository.findPageForTenant(
    "tenant-123",
    ScriptQueryFilter.builder()
        .statuses(List.of(ScriptStatus.ACTIVE))
        .shells(List.of("bash", "powershell"))
        .supportedPlatforms(List.of("LINUX"))
        .tag("monitoring")
        .build(),
    "deploy",                  // search term (substring match on name)
    "_id",                     // sortField
    Sort.Direction.DESC,       // sortDirection
    "64f1a2b3c4d5e6f7a8b9c0d1", // cursor (ObjectId hex)
    false,                     // backward
    25                         // limit
);
```

> **Cursor behavior:** An invalid or blank cursor is silently treated as "no cursor", returning the first page. Direction logic uses XOR between `sortDirection` and `backward` to determine `$lt` vs `$gt` comparisons, ensuring consistent ordering across forward and backward traversal.
