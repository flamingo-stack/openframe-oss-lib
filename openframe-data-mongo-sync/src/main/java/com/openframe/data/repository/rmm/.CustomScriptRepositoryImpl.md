Implements cursor-based paginated MongoDB queries for `Script` documents, scoped per tenant, with filtering, search, and sort support backed by `MongoTemplate`.

## Key Components

| Method / Member | Description |
|---|---|
| `findPageForTenant()` | Main query method — builds and executes a tenant-scoped `Query` with all filters applied, cursor pagination, and directional sort |
| `isSortableField()` | Validates a field name against the allowlist (`_id`, `name`, `createdAt`, `updatedAt`) |
| `getDefaultSortField()` | Returns `_id` as the fallback sort field |
| `SORTABLE_FIELDS` | Static allowlist of fields safe to sort on |
| `applyStatusFilter()` | Filters by status set; defaults to excluding `DELETED` documents (handles legacy docs with no status field via `$ne`) |
| `applyShellsFilter()` | Restricts results to specified shell types |
| `applyPlatformsFilter()` | Matches scripts supporting any of the requested platforms using array `$in` |
| `applyTagFilter()` | Case-insensitive exact tag match using anchored, quoted regex |
| `applySearch()` | Case-insensitive substring match on script name using `Pattern.quote` to sanitize input |
| `applyCursor()` | Parses cursor string to `ObjectId` and applies `$lt` / `$gt` based on sort + pagination direction; falls back to first page on invalid cursor |
| `flip()` | Reverses sort direction for backward pagination |

## Usage Example

```java
List<Script> page = customScriptRepository.findPageForTenant(
    "tenant-abc",
    ScriptQueryFilter.builder()
        .statuses(List.of(ScriptStatus.ACTIVE))
        .shells(List.of("powershell"))
        .supportedPlatforms(List.of("WINDOWS"))
        .tag("onboarding")
        .build(),
    "backup",              // search term
    "_id",                 // sort field
    Sort.Direction.DESC,   // newest first
    "64f1a2b3c4d5e6f7a8b9c0d1", // cursor from previous page
    false,                 // forward pagination
    25                     // page size
);
```

> **Cursor safety:** Any cursor value that is not a valid 24-character hex `ObjectId` is logged as a warning and silently ignored, returning the first page instead of throwing an error.
