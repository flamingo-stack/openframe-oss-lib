Spring Data MongoDB repository interface for tracking `ConnectorRecreationEvent` documents, providing query methods to count and clean up connector recreation events by base name and timestamp.

## Key Components

| Method | Description |
|--------|-------------|
| `countByBaseNameAndCreatedAtAfter` | Returns the count of recreation events for a given connector base name that occurred after a specified `Instant` cutoff |
| `deleteByCreatedAtBefore` | Deletes all recreation events created before a given `Instant` cutoff, returning the number of deleted records |

## Usage Example

```java
@Service
public class ConnectorRecreationService {

    private final ConnectorRecreationEventRepository repository;

    public ConnectorRecreationService(ConnectorRecreationEventRepository repository) {
        this.repository = repository;
    }

    public boolean hasExceededRecreationLimit(String baseName, int limit, Duration window) {
        Instant cutoff = Instant.now().minus(window);
        long recentEvents = repository.countByBaseNameAndCreatedAtAfter(baseName, cutoff);
        return recentEvents >= limit;
    }

    public void purgeStaleEvents(Duration retention) {
        Instant cutoff = Instant.now().minus(retention);
        long deleted = repository.deleteByCreatedAtBefore(cutoff);
        log.info("Purged {} stale connector recreation events", deleted);
    }
}
```

> **Note:** The `deleteByCreatedAtBefore` method is useful for scheduled maintenance jobs to prevent unbounded growth of the events collection in MongoDB.
