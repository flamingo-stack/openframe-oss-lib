Custom repository interface for cursor-based pagination of notifications filtered by recipient, read status, and search terms.

## Key Components

- **`findPageForRecipient(... Sort.Direction direction ...)`** — Full method for fetching a paginated `NotificationPage` with explicit sort direction, supporting forward/backward cursor navigation.
- **`findPageForRecipient(... int limit)`** *(default)* — Convenience overload that defaults sort direction to `Sort.Direction.DESC`, delegating to the full method.

## Usage Example

```java
// Forward pagination with default DESC sort
NotificationPage page = notificationRepository.findPageForRecipient(
    "user-123",
    RecipientType.USER,
    false,       // unread only
    "alert",     // search term
    null,        // no cursor (first page)
    false,       // forward direction
    20           // page size
);

// Backward pagination with explicit ASC sort
NotificationPage prevPage = notificationRepository.findPageForRecipient(
    "user-123",
    RecipientType.USER,
    null,                  // no read filter
    null,                  // no search
    "cursor-abc",          // cursor from previous page
    true,                  // backward
    Sort.Direction.ASC,
    20
);
```

## Parameters

| Parameter | Description |
|-----------|-------------|
| `recipientId` | ID of the target user or entity |
| `recipientType` | Enum distinguishing recipient kinds (e.g., `USER`, `TEAM`) |
| `readFilter` | `true`/`false` to filter by read status; `null` returns all |
| `search` | Optional text search against notification content |
| `cursor` | Opaque pagination cursor; `null` starts from the beginning |
| `backward` | Traversal direction relative to the cursor |
| `direction` | Sort order; defaults to `DESC` |
| `limit` | Maximum number of results per page |
