Defines a custom MongoDB repository interface for paginated notification queries scoped to a specific recipient.

## Key Components

| Element | Description |
|---|---|
| `CustomNotificationRepository` | Interface extending Spring Data's repository pattern with custom query logic |
| `findPageForRecipient(...)` | Retrieves a paginated, filterable page of notifications for a given recipient |

### `findPageForRecipient` Parameters

| Parameter | Type | Description |
|---|---|---|
| `recipientId` | `String` | Unique identifier of the notification recipient |
| `recipientType` | `RecipientType` | Enum distinguishing recipient kind (e.g., user, team) |
| `readFilter` | `Boolean` | Optional filter for read/unread notifications; `null` returns all |
| `search` | `String` | Optional text search term |
| `cursor` | `String` | Cursor value for keyset pagination |
| `backward` | `boolean` | Direction flag; `true` paginates backward |
| `limit` | `int` | Maximum number of results per page |

## Usage Example

```java
NotificationPage page = customNotificationRepository.findPageForRecipient(
    "user-123",
    RecipientType.USER,
    false,       // unread only
    "alert",     // search term
    null,        // no cursor (first page)
    false,       // forward direction
    20           // page size
);
```

> This interface is typically implemented by a `CustomNotificationRepositoryImpl` class using `MongoTemplate` for fine-grained query control, and composed into the main `NotificationRepository` via Spring Data's multi-interface inheritance pattern.
