A Spring Cache-backed service that provides cached lookups for machine and organization metadata, reducing repeated database queries by storing lightweight DTOs in Redis.

## Key Components

| Component | Description |
|-----------|-------------|
| `getMachine(String agentId)` | Looks up machine info by agent ID via `ToolConnectionRepository`, then `MachineRepository`; caches result in `machineCache` |
| `getOrganization(String organizationId)` | Looks up organization info by ID via `OrganizationRepository`; caches result in `organizationCache` |
| `CachedMachineInfo` | Lightweight DTO holding `machineId`, `hostname`, and `organizationId` |
| `CachedOrganizationInfo` | Lightweight DTO holding `organizationId` and `name` |

**Conditional activation:** Only loaded when `openframe.machine-id.cache.enabled=true` is set and MongoDB repositories are available.

## Usage Example

```java
// Inject the service
@Autowired
private MachineIdCacheService machineIdCacheService;

// Resolve machine info from agent ID (cache-first)
CachedMachineInfo machine = machineIdCacheService.getMachine("agent-abc-123");
if (machine != null) {
    String machineId = machine.getMachineId();
    String hostname  = machine.getHostname();
    String orgId     = machine.getOrganizationId();
}

// Resolve organization info (cache-first)
CachedOrganizationInfo org = machineIdCacheService.getOrganization("org-xyz-456");
if (org != null) {
    String orgName = org.getName();
}
```

## Cache Behavior

- **`machineCache`** — keyed by `agentId`; resolves the most recent `ToolConnection` entry first, then maps to the corresponding machine document.
- **`organizationCache`** — keyed by `organizationId`; maps directly from the organization document.
- `unless = "#result == null"` ensures null results (not-found cases) are never stored in cache, preventing stale negative entries.
