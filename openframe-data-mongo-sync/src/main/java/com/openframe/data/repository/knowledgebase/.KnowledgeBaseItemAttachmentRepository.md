MongoDB repository interface for managing file attachments associated with knowledge base items, providing CRUD operations and item-scoped queries.

## Key Components

| Method | Description |
|--------|-------------|
| `findByItemId(String itemId)` | Retrieves all attachments belonging to a single knowledge base item |
| `findByItemIdIn(List<String> itemIds)` | Batch-fetches attachments for multiple knowledge base items |
| `deleteByItemId(String itemId)` | Removes all attachments associated with a specific item |

Extends `MongoRepository<KnowledgeBaseItemAttachment, String>`, inheriting standard CRUD operations (`save`, `findById`, `findAll`, `delete`, etc.).

## Usage Example

```java
@Service
public class KnowledgeBaseAttachmentService {

    private final KnowledgeBaseItemAttachmentRepository attachmentRepository;

    // Fetch all attachments for a single item
    public List<KnowledgeBaseItemAttachment> getAttachments(String itemId) {
        return attachmentRepository.findByItemId(itemId);
    }

    // Batch-load attachments for multiple items (e.g., list view)
    public List<KnowledgeBaseItemAttachment> getAttachmentsForItems(List<String> itemIds) {
        return attachmentRepository.findByItemIdIn(itemIds);
    }

    // Cascade-delete attachments when a KB item is removed
    public void removeItemAttachments(String itemId) {
        attachmentRepository.deleteByItemId(itemId);
    }
}
```

> **Note:** `deleteByItemId` is transactional by Spring Data convention. Ensure cascade deletion is called explicitly when removing a parent `KnowledgeBaseItem` to avoid orphaned attachment documents in MongoDB.
