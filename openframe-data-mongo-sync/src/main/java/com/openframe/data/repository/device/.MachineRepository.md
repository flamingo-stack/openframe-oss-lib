Spring Data MongoDB repository interface for managing `Machine` documents, providing both standard CRUD operations and specialized query methods for device lifecycle management, status filtering, and organization-level lookups.

## Key Components

| Method | Description |
|--------|-------------|
| `findByMachineId` | Lookup a single machine by its unique machine ID |
| `findByHostnameContainingIgnoreCase` | Case-insensitive hostname search across all machines |
| `findByTypeAndHostnameContainingIgnoreCase` | Filter machines by `DeviceType` and partial hostname match |
| `findByMachineIdIn` | Batch fetch machines by a collection of machine IDs |
| `findByMachineIdInAndStatus` | Batch fetch filtered by both ID set and `DeviceStatus` |
| `findByStatusIn` | Retrieve machines matching any of the provided statuses |
| `countByStatusIn` | Count machines by status set (useful for dashboards/metrics) |
| `findByStatusInAndRegisteredAtBefore` | Find machines with specific statuses registered before a timestamp |
| `findByStatusAndLastSeenBefore` | Detect stale/offline machines that haven't reported in since a threshold |
| `findByStatusAndRegisteredAtBeforeAndStuckNotifiedAtIsNull` | Find stuck machines not yet notified — drives alerting workflows |
| `existsByOrganizationId` | Check if any machine belongs to an organization |
| `existsByOrganizationIdAndStatusNotIn` | Check for active machines in an org, excluding certain statuses |

Extends both `MongoRepository<Machine, String>` and `CustomMachineRepository` for custom query support.

## Usage Example

```java
// Find all online machines for a batch of known IDs
List<Machine> activeMachines = machineRepository
    .findByMachineIdInAndStatus(knownIds, DeviceStatus.ONLINE);

// Detect machines that appear stuck and haven't been notified yet
Instant cutoff = Instant.now().minus(Duration.ofMinutes(30));
List<Machine> stuckMachines = machineRepository
    .findByStatusAndRegisteredAtBeforeAndStuckNotifiedAtIsNull(
        DeviceStatus.REGISTERING, cutoff
    );

// Check if an org has any non-decommissioned machines before deletion
boolean hasActive = machineRepository.existsByOrganizationIdAndStatusNotIn(
    orgId, List.of(DeviceStatus.DECOMMISSIONED)
);
```
