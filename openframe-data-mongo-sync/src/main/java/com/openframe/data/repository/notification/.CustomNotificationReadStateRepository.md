Custom repository interface for bulk insertion of `NotificationReadState` documents, bypassing standard ordered write operations for improved performance.

## Key Components

| Element | Type | Description |
|---|---|---|
| `CustomNotificationReadStateRepository` | Interface | Custom Spring Data repository extension for notification read states |
| `bulkInsertUnordered` | Method | Inserts multiple `NotificationReadState` documents in an unordered bulk operation |

## Usage Example

```java
@Repository
public class CustomNotificationReadStateRepositoryImpl
    implements CustomNotificationReadStateRepository {

    private final MongoTemplate mongoTemplate;

    @Override
    public void bulkInsertUnordered(List<NotificationReadState> rows) {
        BulkOperations bulkOps = mongoTemplate.bulkOps(
            BulkOperations.BulkMode.UNORDERED,
            NotificationReadState.class
        );
        rows.forEach(bulkOps::insert);
        bulkOps.execute();
    }
}
```

> **Note:** Unordered bulk inserts allow MongoDB to optimize write order, improving throughput when inserting large batches of notification read state records. Individual failures do not halt the entire operation.
