A Spring Data MongoDB `@WritingConverter` that serializes a `NotificationContext` object into a BSON `Document` for persistence, using Jackson's `ObjectMapper` as the serialization bridge.

## Key Components

- **`NotificationContextWriteConverter`** — Implements `Converter<NotificationContext, Document>`, registered via `@WritingConverter` for automatic invocation by Spring Data MongoDB during write operations.
- **`convert(NotificationContext source)`** — Serializes the source object to a JSON string via `ObjectMapper`, then parses it into a BSON `Document`. Returns `null` for null input and throws `IllegalStateException` on serialization failure, including the context type in the error message for easier debugging.

## Usage Example

```java
// Register with Spring Data MongoDB custom conversions
@Configuration
public class MongoConfig extends AbstractMongoClientConfiguration {

    @Autowired
    private ObjectMapper objectMapper;

    @Override
    public MongoCustomConversions customConversions() {
        return new MongoCustomConversions(List.of(
            new NotificationContextWriteConverter(objectMapper)
        ));
    }
}

// Converter is then invoked automatically during save operations:
NotificationContext ctx = new NotificationContext("ALERT", payload);
notificationRepository.save(notification); // ctx is converted to Document internally
```

> **Note:** Pair this converter with a corresponding `NotificationContextReadConverter` to ensure symmetric read/write behavior for `NotificationContext` documents in MongoDB.
