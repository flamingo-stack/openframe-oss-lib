A Spring Data `@WritingConverter` that serializes a `NotificationContext` object into a MongoDB `Document` for persistence, using Jackson's `ObjectMapper` for JSON conversion.

## Key Components

| Element | Description |
|---|---|
| `@WritingConverter` | Marks this as a write-side converter registered with Spring Data MongoDB |
| `convert(NotificationContext)` | Serializes the source object to JSON, then parses it into a BSON `Document` |
| `ObjectMapper` | Injected Jackson mapper used for JSON serialization |

## Usage Example

```java
// Register the converter in your MongoDB configuration
@Configuration
public class MongoConfig extends AbstractMongoClientConfiguration {

    @Autowired
    private ObjectMapper objectMapper;

    @Override
    public MongoCustomConversions customConversions() {
        return new MongoCustomConversions(List.of(
            new NotificationContextWriteConverter(objectMapper)
        ));
    }
}

// The converter is then invoked automatically when persisting a document
// containing a NotificationContext field — no manual invocation needed.
NotificationContext ctx = new NotificationContext();
ctx.setType("ALERT");
notificationRepository.save(notification); // converter fires here
```

## Error Handling

If Jackson fails to serialize the `NotificationContext`, an `IllegalStateException` is thrown including the context's `type` field to aid debugging — for example:

```text
IllegalStateException: Failed to serialise NotificationContext (type=ALERT)
```

> **Note:** A `null` source is handled gracefully, returning `null` rather than throwing an exception.
