MongoDB repository interface for managing temporary ticket attachments, providing query methods for batch retrieval, user-based lookups, and time-based cleanup operations.

## Key Components

| Method | Description |
|--------|-------------|
| `findByIdIn(List<String> ids)` | Retrieves multiple `TempAttachment` documents by a list of IDs in a single query |
| `findByUploadedBy(String uploadedBy)` | Fetches all temporary attachments uploaded by a specific user |
| `findByCreatedAtBefore(Instant cutoffTime)` | Returns attachments created before a given timestamp, used for expiry/cleanup jobs |

## Usage Example

```java
@Service
public class TempAttachmentService {

    private final TempAttachmentRepository tempAttachmentRepository;

    // Batch fetch by IDs (e.g., when finalizing a ticket)
    public List<TempAttachment> getSelectedAttachments(List<String> ids) {
        return tempAttachmentRepository.findByIdIn(ids);
    }

    // Fetch all uploads for a specific user
    public List<TempAttachment> getUserUploads(String userId) {
        return tempAttachmentRepository.findByUploadedBy(userId);
    }

    // Cleanup stale attachments older than 24 hours
    public void purgeExpiredAttachments() {
        Instant cutoff = Instant.now().minus(24, ChronoUnit.HOURS);
        List<TempAttachment> expired = tempAttachmentRepository.findByCreatedAtBefore(cutoff);
        tempAttachmentRepository.deleteAll(expired);
    }
}
```

> **Note:** `findByCreatedAtBefore` is typically invoked by a scheduled cleanup task to prevent orphaned attachments from accumulating in the database when tickets are abandoned mid-creation.
