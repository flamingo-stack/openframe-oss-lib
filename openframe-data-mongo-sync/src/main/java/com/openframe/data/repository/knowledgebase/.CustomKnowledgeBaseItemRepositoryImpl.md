Custom MongoDB repository implementation for querying `KnowledgeBaseItem` documents, supporting folder/article retrieval with filtering, full-text search, status filtering, and cursor-based pagination.

## Key Components

| Method | Description |
|--------|-------------|
| `findFoldersForParent` | Retrieves `FOLDER`-type items under a given parent, with optional name search and ID filtering |
| `findArticles` | Fetches paginated articles filtered by parent, type, status, search, and cursor |
| `countArticles` | Returns total count matching article query criteria (no pagination) |
| `findArchivedArticles` | Retrieves paginated `ARCHIVED` articles with optional search and ID filter |
| `countArchivedArticles` | Returns total count of archived articles matching criteria |
| `findAllArticles` | Fetches all non-archived articles sorted by name |
| `buildItemQuery` | Shared query builder for article filters; resolves `itemIds` vs. `parentId` precedence |
| `buildArchivedArticlesQuery` | Constructs the query for archived article lookups |
| `addComposites` | Combines `$or` search and cursor criteria into a single `$and` to avoid Spring Data MongoDB key collisions |
| `buildCursorCriteria` | Translates an `ObjectId` cursor string into stable keyset pagination criteria on `updatedAt` + `_id` |
| `executeWithSort` | Applies descending `updatedAt`/`_id` sort and limit, then executes the query |

## Usage Example

```java
// Inject via Spring and call from a service
List<KnowledgeBaseItem> folders = repository.findFoldersForParent(
    "parentFolderId",
    "network",        // search term
    null              // no ID filter
);

// Paginated article fetch with cursor
List<KnowledgeBaseItem> page = repository.findArticles(
    "parentFolderId",
    "vpn",
    KnowledgeBaseItemType.ARTICLE,
    null,
    List.of(KnowledgeBaseArticleStatus.PUBLISHED),
    "lastSeenObjectId",  // cursor from previous page
    20                   // page size
);

long total = repository.countArticles(
    "parentFolderId", "vpn",
    KnowledgeBaseItemType.ARTICLE,
    null,
    List.of(KnowledgeBaseArticleStatus.PUBLISHED)
);
```

> **Pagination note:** Cursor-based pagination uses a stable `updatedAt` + `_id` keyset strategy. If two documents share the same `updatedAt`, the `_id` tiebreaker prevents duplicates or skipped records across pages.
