A Spring Data MongoDB `@ReadingConverter` that deserializes a raw BSON `Document` into a `NotificationContext` object using Jackson, bridging MongoDB's BSON format with Java type mapping.

## Key Components

| Component | Description |
|-----------|-------------|
| `@ReadingConverter` | Registers this converter for MongoDB read operations |
| `RELAXED_JSON` | `JsonWriterSettings` using `JsonMode.RELAXED` to produce standard JSON (e.g., numbers as numbers, not `$numberLong`) |
| `ObjectMapper` | Jackson mapper injected via constructor for JSON-to-POJO deserialization |
| `convert(Document source)` | Core method: converts BSON `Document` → JSON string → `NotificationContext` |

## Usage Example

```java
// Register the converter in your MongoDB configuration
@Configuration
public class MongoConfig extends AbstractMongoClientConfiguration {

    @Bean
    public NotificationContextReadConverter notificationContextReadConverter(ObjectMapper objectMapper) {
        return new NotificationContextReadConverter(objectMapper);
    }

    @Override
    public MongoCustomConversions customConversions() {
        return new MongoCustomConversions(List.of(
            notificationContextReadConverter(objectMapper())
        ));
    }
}
```

## Error Handling

If deserialization fails, an `IllegalStateException` is thrown with diagnostic context including the document `_id` and `type` field, making it straightforward to identify the offending record in logs.

```java
// Example error message produced on failure:
// "Failed to deserialise NotificationContext (id=64abc123, type=ALERT)"
```

## Notes

- **Null safety:** Returns `null` if the source `Document` is `null`
- **RELAXED mode:** Avoids MongoDB Extended JSON quirks (e.g., `{"$numberLong": "1"}`) ensuring Jackson can parse standard numeric types without custom deserializers
