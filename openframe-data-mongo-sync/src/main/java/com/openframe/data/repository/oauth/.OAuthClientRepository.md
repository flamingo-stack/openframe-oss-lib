MongoDB repository interface for managing OAuth client documents, providing query methods for client lookup and existence checks.

## Key Components

| Method | Return Type | Description |
|--------|-------------|-------------|
| `findByClientId` | `Optional<OAuthClient>` | Looks up an OAuth client by its `clientId` field |
| `findByMachineId` | `Optional<OAuthClient>` | Looks up an OAuth client by its `machineId` field |
| `existsByMachineId` | `boolean` | Checks whether an OAuth client exists for a given `machineId` |

Inherits full CRUD operations from `MongoRepository<OAuthClient, String>`.

## Usage Example

```java
@Service
public class OAuthClientService {

    private final OAuthClientRepository repository;

    public OAuthClientService(OAuthClientRepository repository) {
        this.repository = repository;
    }

    public OAuthClient getByClientId(String clientId) {
        return repository.findByClientId(clientId)
                .orElseThrow(() -> new ClientNotFoundException(clientId));
    }

    public boolean isMachineRegistered(String machineId) {
        return repository.existsByMachineId(machineId);
    }

    public OAuthClient getByMachineId(String machineId) {
        return repository.findByMachineId(machineId)
                .orElseThrow(() -> new ClientNotFoundException(machineId));
    }
}
```

## Notes

- Backed by MongoDB via Spring Data; no implementation class is required.
- `machineId` methods support machine-to-machine (M2M) OAuth flows where clients are identified by a machine identifier rather than a manually assigned `clientId`.
- The `String` type parameter in `MongoRepository<OAuthClient, String>` corresponds to MongoDB's `_id` field type for `OAuthClient` documents.
