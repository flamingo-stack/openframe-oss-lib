Custom MongoDB repository implementation for paginated, cursor-based notification retrieval. Supports filtering by read status, full-text search, and bidirectional pagination for a given recipient.

## Key Components

| Method | Description |
|--------|-------------|
| `findPageForRecipient` | Main entry point — builds a `NotificationPage` for a recipient using cursor pagination and optional filters |
| `fetchReadStates` | Queries `NotificationReadState` collection with all filters applied; returns an ordered map of notification IDs to read statuses |
| `fetchNotificationsPreservingOrder` | Fetches full `Notification` documents by ID and reorders them to match the cursor-sorted result from `fetchReadStates` |
| `effectiveDirection` | Flips sort direction when paginating backward |
| `applyStatusFilter` | Adds a `status` criterion: excludes `DELETED` (no filter), or matches `READ`/`UNREAD` explicitly |
| `applyCursor` | Appends a `lt`/`gt` bound on `notificationId` depending on sort direction |
| `applySearch` | Adds a case-insensitive regex filter on the `title` field using `Pattern.quote` to prevent injection |

## Usage Example

```java
// Fetch the first page of unread notifications for a user, newest first
NotificationPage page = repo.findPageForRecipient(
    "user-123",
    RecipientType.USER,
    false,          // unread only
    null,           // no search
    null,           // no cursor (first page)
    false,          // forward pagination
    Sort.Direction.DESC,
    20              // page size
);

// Fetch the next page using the last seen notification ID as cursor
NotificationPage nextPage = repo.findPageForRecipient(
    "user-123",
    RecipientType.USER,
    false,
    null,
    "663f1a2b4e0a1c0012345678", // cursor from previous page
    false,
    Sort.Direction.DESC,
    20
);
```

> **Design note:** Pagination is implemented as a two-phase query — read states are fetched first (lightweight), then full notification documents are hydrated in a single `$in` query, preserving cursor order without relying on MongoDB sort stability across collections.
