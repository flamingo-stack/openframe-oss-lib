MongoDB repository interface for managing `Tenant` documents, providing domain-based lookup, existence checks, and projection support via Spring Data MongoDB.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `TenantRepository` | Interface | Main repository extending `MongoRepository<Tenant, String>` and `BaseTenantRepository` |
| `findByDomain(String)` | Method | Returns an `Optional<Tenant>` for a given domain string |
| `existsByDomain(String)` | Method | Returns `Boolean` indicating whether a domain is already registered |
| `findByDomainIn(List<String>)` | Method | Bulk domain lookup returning a lightweight `DomainView` projection |
| `DomainView` | Inner Interface | Projection interface exposing only `getDomain()` for efficient queries |
| `count()` | Method | Returns total number of tenant documents in the collection |

## Usage Example

```java
@Service
public class TenantService {

    private final TenantRepository tenantRepository;

    public TenantService(TenantRepository tenantRepository) {
        this.tenantRepository = tenantRepository;
    }

    public Optional<Tenant> resolveTenant(String domain) {
        return tenantRepository.findByDomain(domain);
    }

    public boolean isDomainTaken(String domain) {
        return tenantRepository.existsByDomain(domain);
    }

    public List<TenantRepository.DomainView> getActiveDomains(List<String> domains) {
        return tenantRepository.findByDomainIn(domains);
    }

    public long getTenantCount() {
        return tenantRepository.count();
    }
}
```

> **Note:** `DomainView` projections are preferred for bulk domain lookups — they avoid loading the full `Tenant` document, reducing memory overhead in multi-tenant routing scenarios common in the OpenFrame platform.
