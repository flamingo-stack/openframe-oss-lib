Spring Data MongoDB repository interface for managing `OpenFrameClientConfiguration` documents, providing CRUD operations and a custom query to retrieve the most recent configuration record.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `OpenFrameClientConfigurationRepository` | Interface | Spring `@Repository` extending `MongoRepository` |
| `findFirstByOrderByCreatedAtDesc()` | Method | Returns the latest configuration sorted by `createdAt` descending |

## Usage Example

```java
@Service
public class ClientConfigurationService {

    private final OpenFrameClientConfigurationRepository repository;

    public ClientConfigurationService(OpenFrameClientConfigurationRepository repository) {
        this.repository = repository;
    }

    public OpenFrameClientConfiguration getLatestConfig() {
        return repository.findFirstByOrderByCreatedAtDesc()
                .orElseThrow(() -> new RuntimeException("No client configuration found"));
    }

    public OpenFrameClientConfiguration saveConfig(OpenFrameClientConfiguration config) {
        return repository.save(config);
    }
}
```

## Notes

- Inherits standard CRUD methods (`save`, `findById`, `findAll`, `delete`, etc.) from `MongoRepository`
- `findFirstByOrderByCreatedAtDesc()` uses Spring Data's derived query mechanism — no manual query annotation required
- Returns `Optional<OpenFrameClientConfiguration>` to safely handle the case where no configuration document exists in MongoDB
