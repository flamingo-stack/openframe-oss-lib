Spring Data MongoDB repository interface for managing `OpenFrameClientConfiguration` documents, providing tenant-aware configuration lookup with standard CRUD operations.

## Key Components

| Method | Description |
|--------|-------------|
| `findFirstByOrderByCreatedAtDesc()` | Retrieves the most recently created client configuration document |
| `findByTenantId(String tenantId)` | Looks up a client configuration by its associated tenant identifier |

Extends `MongoRepository<OpenFrameClientConfiguration, String>`, inheriting full CRUD and pagination support.

## Usage Example

```java
@Service
public class ClientConfigurationService {

    private final OpenFrameClientConfigurationRepository repository;

    public ClientConfigurationService(OpenFrameClientConfigurationRepository repository) {
        this.repository = repository;
    }

    public OpenFrameClientConfiguration getLatestConfig() {
        return repository.findFirstByOrderByCreatedAtDesc()
                .orElseThrow(() -> new RuntimeException("No configuration found"));
    }

    public OpenFrameClientConfiguration getConfigByTenant(String tenantId) {
        return repository.findByTenantId(tenantId)
                .orElseThrow(() -> new RuntimeException("No config found for tenant: " + tenantId));
    }
}
```

> Both query methods return `Optional<OpenFrameClientConfiguration>`, ensuring null-safe access to configuration documents in a multi-tenant OpenFrame environment.
