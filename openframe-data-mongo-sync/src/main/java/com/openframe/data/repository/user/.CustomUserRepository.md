Defines a custom Spring Data repository interface for executing filtered user search queries beyond standard repository capabilities.

## Key Components

| Component | Type | Description |
|---|---|---|
| `CustomUserRepository` | Interface | Custom repository contract for user-specific query operations |
| `findUsersBySearch` | Method | Retrieves a filtered, paginated list of `User` documents |
| `UserQueryFilter` | Parameter | Encapsulates search criteria applied to the query |
| `limit` | Parameter | Caps the maximum number of results returned |

## Usage Example

```java
@Repository
public class CustomUserRepositoryImpl implements CustomUserRepository {

    private final MongoTemplate mongoTemplate;

    public CustomUserRepositoryImpl(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    @Override
    public List<User> findUsersBySearch(UserQueryFilter filter, int limit) {
        Query query = new Query();

        if (filter.getName() != null) {
            query.addCriteria(Criteria.where("name").regex(filter.getName(), "i"));
        }

        if (filter.getRole() != null) {
            query.addCriteria(Criteria.where("role").is(filter.getRole()));
        }

        query.limit(limit);
        return mongoTemplate.find(query, User.class);
    }
}
```

## Notes

- Intended to be extended by a concrete `CustomUserRepositoryImpl` class using `MongoTemplate` or similar.
- Pair with a standard Spring Data repository by extending both interfaces to combine auto-generated queries with this custom search logic.
- `UserQueryFilter` should encapsulate all dynamic search criteria (e.g., name, role, status) to keep the method signature stable as filter options grow.
