Tenant-scoped MongoDB repository interface for managing `Script` documents within the RMM module, extending both `MongoRepository` and `CustomScriptRepository` for standard CRUD plus custom cursor-based pagination.

## Key Components

| Method | Description |
|--------|-------------|
| `findByTenantIdAndId` | Retrieves a single script by its ID, scoped to a specific tenant |
| `findByTenantIdAndName` | Retrieves a single script by name within a tenant; enforces the compound `(tenantId, name)` unique index |
| `existsByTenantIdAndName` | Checks for name uniqueness within a tenant |
| `existsByTenantIdAndNameAndIdNot` | Checks for name uniqueness while excluding a specific script (useful for update validation) |
| `deleteByTenantIdAndId` | Hard-deletes a script by ID within a tenant; returns `0` or `1` indicating documents removed |

## Usage Example

```java
// Lookup a script by name for a given tenant
Optional<Script> script = scriptRepository.findByTenantIdAndName(tenantId, "restart-service");

script.ifPresent(s -> System.out.println("Found: " + s.getId()));

// Validate name uniqueness before updating
boolean nameConflict = scriptRepository.existsByTenantIdAndNameAndIdNot(
    tenantId,
    "restart-service",
    currentScriptId
);

if (nameConflict) {
    throw new DuplicateScriptNameException("Name already in use.");
}

// Hard-delete a script
long removed = scriptRepository.deleteByTenantIdAndId(tenantId, scriptId);
if (removed == 0) {
    throw new ScriptNotFoundException(scriptId);
}
```

> All operations are explicitly tenant-scoped. List and pagination queries are handled by `CustomScriptRepository` using `MongoTemplate` for cursor-style paging.
