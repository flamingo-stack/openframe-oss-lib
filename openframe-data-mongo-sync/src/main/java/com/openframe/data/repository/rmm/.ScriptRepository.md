Tenant-scoped MongoDB repository interface for managing `Script` documents within the RMM (Remote Monitoring & Management) module.

## Key Components

| Method | Description |
|--------|-------------|
| `findByTenantIdAndId` | Retrieves a single script by its document ID, scoped to a tenant |
| `findByTenantIdAndName` | Retrieves a script by name using the compound `(tenantId, name)` unique index |
| `existsByTenantIdAndName` | Checks for name uniqueness within a tenant |
| `existsByTenantIdAndNameAndIdNot` | Checks for name conflicts excluding a specific document (useful for update validation) |
| `deleteByTenantIdAndId` | Hard-deletes a script by ID within a tenant; returns `0` or `1` |

> List and pagination operations are handled by `CustomScriptRepository` via MongoTemplate for cursor-style paging.

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class ScriptService {

    private final ScriptRepository scriptRepository;

    public Script getScript(String tenantId, String scriptId) {
        return scriptRepository.findByTenantIdAndId(tenantId, scriptId)
                .orElseThrow(() -> new ScriptNotFoundException(scriptId));
    }

    public boolean isNameAvailable(String tenantId, String name, String excludeId) {
        return !scriptRepository.existsByTenantIdAndNameAndIdNot(tenantId, name, excludeId);
    }

    public long removeScript(String tenantId, String scriptId) {
        return scriptRepository.deleteByTenantIdAndId(tenantId, scriptId);
    }
}
```
