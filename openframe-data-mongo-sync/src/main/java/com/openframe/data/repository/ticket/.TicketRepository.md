MongoDB repository interface for `Ticket` documents, providing both standard CRUD operations and specialized aggregation queries for ticket management within the OpenFrame platform.

## Key Components

### Legacy Methods (lifecycle feature flag OFF)

| Method | Description |
|--------|-------------|
| `findByTicketNumber(Integer)` | Lookup a single ticket by its human-readable number |
| `findByStatus(TicketStatus)` | Retrieve all tickets matching an enum status |
| `findByOrganizationId(String)` | Fetch tickets scoped to an organization |
| `findByAssignedTo(String)` | Fetch tickets assigned to a specific user |
| `findByDeviceId(String)` | Fetch tickets linked to a device |
| `findByIdIn(List<String>)` | Batch-fetch tickets by a list of IDs |
| `findByIdAndOwnerMachineId(String, String)` | Find a ticket by ID and owner machine (custom `@Query`) |
| `findFirstInColumn` / `findLastInColumn` | Get first/last ordered ticket in a status column |
| `findFirstAfter` / `findFirstBefore` | Navigate adjacent tickets by `order` field |

### Lifecycle Methods (lifecycle feature flag ON)

Uses `statusId` (String) instead of `TicketStatus` enum, enabling dynamic status configurations.

| Method | Description |
|--------|-------------|
| `countByStatusId(String)` | Count tickets in a given lifecycle status |
| `findFirstInColumnByStatusId` | First ordered ticket for a `statusId`/tenant pair |
| `findFirstAfterByStatusId` | Next ticket after a given order value |
| `findFirstBeforeByStatusId` | Previous ticket before a given order value |

## Usage Example

```java
// Legacy: fetch open tickets for an organization
List<Ticket> openTickets = ticketRepository.findByStatus(TicketStatus.OPEN);

// Kanban navigation: find the first ticket in a column
Optional<Ticket> first = ticketRepository
    .findFirstInColumn(TicketStatus.IN_PROGRESS, "tenant-123");

// Lifecycle mode: navigate to the next ticket after a given order
Optional<Ticket> next = ticketRepository
    .findFirstAfterByStatusId("status-abc", "order-xyz", "tenant-123");

// Custom query: validate ticket ownership by machine
Optional<Ticket> ticket = ticketRepository
    .findByIdAndOwnerMachineId("ticket-id", "machine-id");
```

> **Note:** Legacy methods using `TicketStatus` enum are scheduled for removal after the lifecycle feature flag rollout is complete. Prefer `*ByStatusId` variants in new code.
