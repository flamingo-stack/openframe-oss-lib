MongoDB repository interface for `Ticket` documents, providing standard CRUD operations alongside specialized query and aggregation methods for ticket lookup, filtering, and kanban-style ordering.

## Key Components

| Method | Type | Description |
|--------|------|-------------|
| `findByTicketNumber` | `@Query` (derived) | Lookup a ticket by its human-readable number |
| `findByStatus` | Derived | Fetch all tickets matching a given `TicketStatus` |
| `findByOrganizationId` | Derived | Retrieve tickets scoped to an organization |
| `findByAssignedTo` | Derived | List tickets assigned to a specific technician |
| `findByDeviceId` | Derived | Fetch tickets linked to a device |
| `findByIdIn` | Derived | Bulk fetch tickets by a list of IDs |
| `findByIdAndOwnerMachineId` | `@Query` | Secure lookup combining ticket ID and machine owner |
| `findFirstInColumn` | `@Aggregation` | Retrieves the first ordered ticket in a status column |
| `findLastInColumn` | `@Aggregation` | Retrieves the last ordered ticket in a status column |
| `findFirstAfter` | `@Aggregation` | Gets the next ticket after a given order value |
| `findFirstBefore` | `@Aggregation` | Gets the previous ticket before a given order value |

## Usage Example

```java
// Fetch all open tickets for an organization
List<Ticket> openTickets = ticketRepository.findByStatus(TicketStatus.OPEN);

// Find a specific ticket by number
Optional<Ticket> ticket = ticketRepository.findByTicketNumber(1042);

// Kanban board: find the bottom ticket in the IN_PROGRESS column
Optional<Ticket> last = ticketRepository.findLastInColumn(TicketStatus.IN_PROGRESS);

// Find the next ticket after a given order position (for drag-and-drop reordering)
Optional<Ticket> next = ticketRepository.findFirstAfter(TicketStatus.OPEN, "order_abc123");

// Validate ticket ownership by machine
Optional<Ticket> owned = ticketRepository.findByIdAndOwnerMachineId(ticketId, machineId);
```

> The aggregation methods (`findFirstInColumn`, `findLastInColumn`, `findFirstAfter`, `findFirstBefore`) power kanban board ordering logic, enabling precise ticket positioning within status columns.
