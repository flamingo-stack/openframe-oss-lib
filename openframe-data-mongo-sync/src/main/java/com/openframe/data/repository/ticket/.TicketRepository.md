MongoDB repository interface for `Ticket` documents, providing both standard CRUD operations and specialized queries for ticket lookup, filtering, and kanban-style column ordering. Supports two query modes: a legacy path using `TicketStatus` enum and a lifecycle-feature path using dynamic `statusId` strings.

## Key Components

**Standard Finders**
- `findByTicketNumber(Integer)` — Lookup a ticket by its human-readable number
- `findByStatus(TicketStatus)` — Filter tickets by legacy enum status
- `findByOrganizationId / findByAssignedTo / findByDeviceId` — Common relationship-based filters
- `findByIdIn(List<String>)` — Batch fetch by ID list
- `findByIdAndOwnerMachineId(String, String)` — Scoped lookup using a custom `@Query`

**Legacy Column Ordering (enum-based)**
| Method | Description |
|---|---|
| `findFirstInColumn` | First ticket (lowest order) in a status column |
| `findLastInColumn` | Last ticket (highest order) in a status column |
| `findFirstAfter` | Next ticket after a given order value |
| `findFirstBefore` | Previous ticket before a given order value |

**Lifecycle Column Ordering (statusId-based)**
Mirrors the legacy ordering methods but queries against `statusId` (string) instead of `status` enum, used when the lifecycle feature flag is enabled.

- `countByStatusId` — Count tickets in a dynamic status
- `findFirstInColumnByStatusId / findFirstAfterByStatusId / findFirstBeforeByStatusId`

## Usage Example

```java
// Legacy: fetch tickets in a column and navigate ordering
Optional<Ticket> first = ticketRepository.findFirstInColumn(TicketStatus.OPEN, "tenant-123");
Optional<Ticket> next  = ticketRepository.findFirstAfter(TicketStatus.OPEN, first.get().getOrder(), "tenant-123");

// Lifecycle flag ON: same navigation using dynamic statusId
Optional<Ticket> firstNew = ticketRepository.findFirstInColumnByStatusId("status-abc", "tenant-123");
long count = ticketRepository.countByStatusId("status-abc");

// Scoped lookup by owner machine
Optional<Ticket> ticket = ticketRepository.findByIdAndOwnerMachineId("ticket-id", "machine-xyz");
```

> **Note:** Legacy methods (`findByStatus`, `findFirstInColumn`, etc.) are marked for removal after the lifecycle feature flag rollout is complete.
