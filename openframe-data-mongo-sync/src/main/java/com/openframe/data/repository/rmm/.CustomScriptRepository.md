Defines the custom repository interface for paginated, filterable `Script` document queries backed by `MongoTemplate`, supplementing standard Spring Data derived methods with cursor-based pagination support.

## Key Components

- **`findPageForTenant`** — Core list query scoped to a tenant, supporting optional filter (`ScriptQueryFilter`), regex name search, dynamic sort, and `ObjectId`-based cursor pagination in both forward and backward directions
- **`isSortableField`** — Guard method to validate that a requested sort field is permitted before passing it to the query
- **`getDefaultSortField`** — Returns the fallback sort field when none is provided by the caller

## Usage Example

```java
// Fetch the first page of active scripts, sorted by name ascending
List<Script> page = customScriptRepository.findPageForTenant(
    "tenant-abc",            // tenantId
    ScriptQueryFilter.ACTIVE, // filter
    "deploy",                // search substring (regex, case-insensitive)
    "name",                  // sortField
    Sort.Direction.ASC,      // sortDirection
    null,                    // cursor = first page
    false,                   // forward pagination
    26                       // pageSize + 1 to detect hasNextPage
);

// Detect if a next page exists
boolean hasNextPage = page.size() > 25;
List<Script> results = hasNextPage ? page.subList(0, 25) : page;
```

## Notes

- Implementation lives in `CustomScriptRepositoryImpl`
- DELETED records are always excluded regardless of the supplied `filter`
- For backward pagination (`before`/`last`), the returned list should be **reversed** by the caller before display
- The cursor is expected as a raw `ObjectId` hex string (base64-decoded by the caller before passing in)
