Defines the custom MongoTemplate-backed query contract for paginated, filterable `Script` document retrieval — supplementing Spring Data's derived query limitations with cursor-based pagination support.

## Key Components

| Member | Description |
|--------|-------------|
| `findPageForTenant(...)` | Fetches a single page of scripts scoped to a tenant, supporting optional filter, regex search, sorting, and cursor-based forward/backward pagination |
| `isSortableField(String field)` | Validates whether a given field name is permitted as a sort key |
| `getDefaultSortField()` | Returns the default sort field used when none is explicitly provided |

## Usage Example

```java
// Inject repository where CustomScriptRepository is mixed into the main repo
@Autowired
private ScriptRepository scriptRepository;

// Fetch first page (no cursor) with a filter and search term
List<Script> page = scriptRepository.findPageForTenant(
    "tenant-abc-123",
    ScriptQueryFilter.ACTIVE,       // optional filter
    "deploy",                       // regex substring search on name
    scriptRepository.getDefaultSortField(),
    Sort.Direction.ASC,
    null,                           // null = first page, no cursor
    false,                          // forward pagination
    26                              // pageSize + 1 to detect next page
);

boolean hasNextPage = page.size() > 25;
List<Script> results = hasNextPage ? page.subList(0, 25) : page;
```

## Notes

- Implementation lives in `CustomScriptRepositoryImpl`
- Cursor value is the raw `ObjectId` hex (base64-decoded before passing)
- Deleted scripts are always excluded regardless of filter input
- When `backward=true`, the returned list should be reversed by the caller for correct display order
