Provides MongoDB persistence operations for `Event` documents by combining Spring Data's standard CRUD capabilities with custom query extensions.

## Key Components

| Component | Description |
|---|---|
| `MongoRepository<Event, String>` | Supplies built-in CRUD, pagination, and sorting operations with `String` as the document ID type |
| `CustomEventRepository` | Mixes in project-specific query methods beyond the standard Spring Data interface |

## Usage Example

```java
@Service
public class EventService {

    private final EventRepository eventRepository;

    public EventService(EventRepository eventRepository) {
        this.eventRepository = eventRepository;
    }

    public Event saveEvent(Event event) {
        return eventRepository.save(event);
    }

    public Optional<Event> findById(String id) {
        return eventRepository.findById(id);
    }

    public void deleteEvent(String id) {
        eventRepository.deleteById(id);
    }
}
```

> Spring Data automatically provides the implementation at runtime — no concrete class is required.
