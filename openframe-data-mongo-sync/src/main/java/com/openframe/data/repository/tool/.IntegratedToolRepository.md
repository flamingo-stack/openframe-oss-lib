MongoDB repository interface for managing `IntegratedTool` documents, providing standard CRUD operations alongside custom and base lookup methods.

## Key Components

| Element | Description |
|---|---|
| `IntegratedToolRepository` | Main repository interface extending Spring Data MongoDB and custom base interfaces |
| `findByType(String type)` | Retrieves an `IntegratedTool` by its tool type identifier |
| `findByKey(String key)` | Retrieves an `IntegratedTool` by its unique key |

**Extends:**
- `MongoRepository<IntegratedTool, String>` — standard Spring Data CRUD operations
- `BaseIntegratedToolRepository<Optional<IntegratedTool>, Boolean, String>` — shared base query contracts
- `CustomIntegratedToolRepository` — project-specific custom query logic

## Usage Example

```java
@Service
public class ToolService {

    private final IntegratedToolRepository toolRepository;

    public ToolService(IntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public IntegratedTool getToolByType(String type) {
        return toolRepository.findByType(type)
                .orElseThrow(() -> new ToolNotFoundException("No tool found for type: " + type));
    }

    public IntegratedTool getToolByKey(String key) {
        return toolRepository.findByKey(key)
                .orElseThrow(() -> new ToolNotFoundException("No tool found for key: " + key));
    }
}
```

Both lookup methods return `Optional<IntegratedTool>`, ensuring null-safe handling when a tool is not found in the MongoDB collection.
