Spring Data MongoDB repository interface for managing `IntegratedTool` documents, providing lookup operations by tool type and key.

## Key Components

| Element | Description |
|---|---|
| `IntegratedToolRepository` | Primary repository interface extending `MongoRepository<IntegratedTool, String>` |
| `BaseIntegratedToolRepository` | Base interface providing typed signatures for find/exists operations |
| `CustomIntegratedToolRepository` | Custom query extension interface for additional repository logic |
| `findByType(String type)` | Retrieves an `IntegratedTool` by its tool type identifier |
| `findByKey(String key)` | Retrieves an `IntegratedTool` by its unique key |

## Usage Example

```java
@Service
public class IntegratedToolService {

    private final IntegratedToolRepository repository;

    public IntegratedToolService(IntegratedToolRepository repository) {
        this.repository = repository;
    }

    public IntegratedTool getToolByType(String type) {
        return repository.findByType(type)
                .orElseThrow(() -> new ToolNotFoundException("No tool found for type: " + type));
    }

    public IntegratedTool getToolByKey(String key) {
        return repository.findByKey(key)
                .orElseThrow(() -> new ToolNotFoundException("No tool found for key: " + key));
    }
}
```

## Notes

- Both query methods return `Optional<IntegratedTool>`, ensuring null-safe handling when a tool is not found.
- Inherits full CRUD operations from Spring Data's `MongoRepository`.
- The `@Repository` annotation enables Spring component scanning and exception translation for MongoDB persistence errors.
