Spring Data MongoDB repository interface for managing `IntegratedTool` documents, providing CRUD operations and custom query support for integrated MSP tools within the OpenFrame platform.

## Key Components

| Component | Description |
|-----------|-------------|
| `MongoRepository<IntegratedTool, String>` | Base Spring Data MongoDB CRUD operations |
| `BaseIntegratedToolRepository` | Generic base interface typed for `Optional<IntegratedTool>`, `Boolean`, and `String` |
| `CustomIntegratedToolRepository` | Custom query extensions for tool-specific operations |
| `findByType(String type)` | Derived query method returning an `Optional<IntegratedTool>` by tool type |

## Usage Example

```java
@Service
public class IntegratedToolService {

    private final IntegratedToolRepository toolRepository;

    public IntegratedToolService(IntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public IntegratedTool getToolByType(String type) {
        return toolRepository.findByType(type)
                .orElseThrow(() -> new ToolNotFoundException("No tool found for type: " + type));
    }

    public IntegratedTool saveTool(IntegratedTool tool) {
        return toolRepository.save(tool);
    }

    public boolean toolExists(String id) {
        return toolRepository.existsById(id);
    }
}
```

## Notes

- Annotated with `@Repository` for Spring component scanning and exception translation
- `findByType` returns `Optional<IntegratedTool>` to safely handle missing tool configurations
- Inherits the full Spring Data MongoDB query API alongside any custom repository logic defined in `CustomIntegratedToolRepository`
