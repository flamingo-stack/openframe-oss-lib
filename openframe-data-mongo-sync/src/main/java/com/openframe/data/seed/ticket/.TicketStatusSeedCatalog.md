Defines the seed catalog for default ticket status definitions used during system initialization, providing pre-configured statuses with names, colors, and LexoRank-based ordering positions.

## Key Components

**Constants**
- `NAME_*` — Display names for built-in statuses: `AI Assistance`, `Tech Required`, `Resolved`, `Archived`, `On Hold`
- `COLOR_*` — Hex color codes associated with each status (e.g., `#B39DDB` for AI Assistance, `#5EA62E` for Resolved)

**Methods**
- `systemStatuses()` — Returns an ordered `List<TicketStatusDefinition>` of the four core system statuses, with LexoRank positions generated relative to a middle anchor: `AI Assistance → Tech Required → [middle] → Resolved → Archived`
- `onHoldCustom()` — Returns a single `CUSTOM`-kind `On Hold` status placed at the LexoRank middle position, used as an example or default custom status

## Usage Example

```java
// Seed system-wide default statuses on tenant initialization
List<TicketStatusDefinition> defaults = TicketStatusSeedCatalog.systemStatuses();
ticketStatusRepository.saveAll(defaults);

// Optionally seed a custom "On Hold" status
TicketStatusDefinition onHold = TicketStatusSeedCatalog.onHoldCustom();
ticketStatusRepository.save(onHold);
```

## Notes

- The class is `final` with a private constructor — it is a non-instantiable utility class
- LexoRank ordering ensures statuses render in a meaningful default sequence without requiring numeric indices
- `CUSTOM` kind on `onHoldCustom()` signals that tenant operators can modify or remove it, unlike system-locked statuses
