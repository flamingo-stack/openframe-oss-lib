Defines the default seed catalog for ticket statuses in the OpenFrame platform, providing pre-configured system status definitions and a sample custom status used during tenant initialization.

## Key Components

### Constants
| Constant | Value | Description |
|---|---|---|
| `NAME_AI_ASSISTANCE` | `"AI Assistance"` | Status for tickets handled by Mingo AI |
| `NAME_TECH_REQUIRED` | `"Tech Required"` | Status escalated to a human technician |
| `NAME_RESOLVED` | `"Resolved"` | Completed ticket status |
| `NAME_ARCHIVED` | `"Archived"` | Long-term archived ticket status |
| `NAME_ON_HOLD` | `"On Hold"` | Sample custom status |

### Methods

- **`systemStatuses()`** — Returns an ordered `List<TicketStatusDefinition>` of four built-in system statuses (`AI_ASSISTANCE`, `TECH_REQUIRED`, `RESOLVED`, `ARCHIVED`) with LexoRank-based positions ensuring a deterministic sort order.
- **`onHoldCustom()`** — Returns a single `CUSTOM` kind `TicketStatusDefinition` representing an "On Hold" status, positioned at the LexoRank middle. Useful as a demo or default custom status during onboarding.

## Usage Example

```java
// Seed all system statuses during tenant initialization
List<TicketStatusDefinition> defaults = TicketStatusSeedCatalog.systemStatuses();
ticketStatusRepository.saveAll(defaults);

// Optionally seed a sample custom status
TicketStatusDefinition onHold = TicketStatusSeedCatalog.onHoldCustom();
ticketStatusRepository.save(onHold);
```

## Notes

- The class is `final` with a private constructor — it is a pure utility/catalog class, not meant to be instantiated.
- LexoRank ordering places statuses as: `AI Assistance → Tech Required → [middle] → Resolved → Archived`, matching a typical MSP ticket lifecycle.
- Colors follow the OpenFrame UI palette (e.g., amber `#E1B32F` for tech escalation, green `#5EA62E` for resolved).
