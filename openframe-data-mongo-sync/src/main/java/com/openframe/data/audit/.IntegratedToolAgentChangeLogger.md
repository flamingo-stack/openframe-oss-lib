Monitors and logs MongoDB save lifecycle events for `IntegratedToolAgent` documents, capturing key metadata before and after each persistence operation.

## Key Components

- **`IntegratedToolAgentChangeLogger`** — Spring `@Component` extending `AbstractMongoEventListener<IntegratedToolAgent>` that hooks into MongoDB save events
- **`onBeforeSave`** — Intercepts the `BeforeSaveEvent` and logs agent state prior to persistence
- **`onAfterSave`** — Intercepts the `AfterSaveEvent` and logs agent state after successful persistence
- **`log(String phase, IntegratedToolAgent agent)`** — Extracts and emits structured log fields: document ID, document version, agent version, release flag, and publish state
- **`formatPublishState(PublishState ps)`** — Converts a `PublishState` object into a human-readable string (`none`, `published`, `pending`, or `pending(attempts=N)`)

## Usage Example

This component is auto-registered by Spring via `@Component` and requires no manual wiring. It activates automatically on any MongoDB save of an `IntegratedToolAgent` document.

```java
// Sample log output emitted on save
// Before:
// save-before id=abc123 v=5 agentVersion=1.2.0 release=false publish=pending(attempts=2)

// After:
// save-after id=abc123 v=6 agentVersion=1.2.0 release=true publish=published
```

Typical audit log fields:

| Field | Source |
|---|---|
| `id` | `agent.getId()` |
| `v` | `agent.getDocumentVersion()` |
| `agentVersion` | `agent.getVersion()` |
| `release` | `agent.isReleaseVersion()` |
| `publish` | Formatted `PublishState` |
