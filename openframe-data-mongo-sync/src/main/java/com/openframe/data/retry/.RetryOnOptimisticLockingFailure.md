A custom composed annotation that wraps Spring Retry's `@Retryable` to handle `OptimisticLockingFailureException`, providing automatic retry logic with exponential backoff for database concurrency conflicts.

## Key Components

| Element | Value | Description |
|---|---|---|
| `retryFor` | `OptimisticLockingFailureException` | Triggers retry on optimistic lock conflicts |
| `maxAttempts` | `5` | Maximum number of retry attempts |
| `delay` | `50ms` | Initial backoff delay |
| `multiplier` | `2` | Exponential backoff multiplier |
| `random` | `true` | Adds jitter to prevent retry storms |
| `listeners` | `optimisticLockingRetryListener` | Bean name for retry lifecycle hooks |

## Usage Example

```java
@Service
public class TicketService {

    @RetryOnOptimisticLockingFailure
    public Ticket updateTicketStatus(Long ticketId, TicketStatus newStatus) {
        Ticket ticket = ticketRepository.findById(ticketId)
                .orElseThrow(() -> new EntityNotFoundException(ticketId));
        ticket.setStatus(newStatus);
        return ticketRepository.save(ticket); // retried on version conflict
    }
}
```

> **Note:** The target bean must be managed by Spring and called via proxy (i.e., not a self-invocation) for the retry behaviour to activate. Ensure `@EnableRetry` is present on your application or configuration class.
