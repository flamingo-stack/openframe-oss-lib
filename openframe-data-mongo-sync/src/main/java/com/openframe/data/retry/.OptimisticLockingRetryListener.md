Spring Retry listener that logs optimistic locking exception (OLE) retry lifecycle events — warnings on each failed attempt, success/exhaustion summaries on close.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `METHOD_NAME_KEY` | `String` constant | Context attribute key (`"context.name"`) used to extract the retried method name for log messages |
| `onError()` | `RetryListener` override | Logs a `WARN` on each failed retry attempt, including attempt count, method name, and exception message |
| `close()` | `RetryListener` override | Logs `INFO` on successful recovery or `ERROR` when retries are exhausted; silently skips logging if no retries were attempted |

## Usage Example

Register the listener on a `@Retryable` operation or wire it into a `RetryTemplate`:

```java
@Configuration
public class RetryConfig {

    @Bean
    public RetryTemplate retryTemplate(OptimisticLockingRetryListener listener) {
        RetryTemplate template = new RetryTemplate();

        template.setRetryPolicy(
            new SimpleRetryPolicy(3, Map.of(ObjectOptimisticLockingFailureException.class, true))
        );

        template.registerListener(listener);
        return template;
    }
}
```

```java
// Usage in a service
retryTemplate.execute(ctx -> {
    ctx.setAttribute("context.name", "TicketService.updateStatus");
    return ticketRepository.save(ticket); // retried on OLE
});
```

> **Note:** The `"context.name"` attribute must be set manually on the `RetryContext` when using `RetryTemplate` directly. With `@Retryable`, Spring populates this automatically from the method signature.
