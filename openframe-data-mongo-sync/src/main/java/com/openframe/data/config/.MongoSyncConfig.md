Configures synchronous (non-reactive) MongoDB repositories for OpenFrame when tenant isolation is disabled, conditionally enabling standard Spring Data Mongo repositories while excluding tenant-aware ones.

## Key Components

- **`@ConditionalOnExpression`** — Activates only when `spring.data.mongodb.enabled=true` **and** `openframe.tenant-isolation.enabled=false`
- **`@EnableMongoRepositories`** — Scans `com.openframe.data.repository` for standard MongoDB repositories
- **Exclusion Filter** — Skips any repository annotated with `@TenantAwareRepository` to prevent conflicts with multi-tenant configurations

## Usage Example

```java
# application.properties

# Enable standard (sync) Mongo config
spring.data.mongodb.enabled=true

# Must be false to activate MongoSyncConfig
openframe.tenant-isolation.enabled=false

spring.data.mongodb.uri=mongodb://localhost:27017/openframe
```

```java
// Standard repository — picked up by MongoSyncConfig
public interface DeviceRepository extends MongoRepository<Device, String> {
    List<Device> findByTenantId(String tenantId);
}

// Annotated with @TenantAwareRepository — excluded from this config
@TenantAwareRepository
public interface TenantDeviceRepository extends MongoRepository<Device, String> { }
```

> **Note:** When `openframe.tenant-isolation.enabled=true`, this configuration is skipped entirely and the tenant-aware MongoDB setup takes over, ensuring repositories are routed through the correct tenant database context.
