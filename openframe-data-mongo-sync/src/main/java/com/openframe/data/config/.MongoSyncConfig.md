Configures MongoDB repositories for standard (non-tenant-isolated) sync mode, enabling Spring Data MongoDB repositories while explicitly excluding tenant-aware repository implementations.

## Key Components

- **`@ConditionalOnExpression`** — Activates only when `spring.data.mongodb.enabled=true` **and** `openframe.tenant-isolation.enabled=false`, preventing conflicts with multi-tenant configurations
- **`@EnableMongoRepositories`** — Scans `com.openframe.data.repository` for MongoDB repositories, filtering out any interface annotated with `@TenantAwareRepository`

## Usage Example

```java
# application.properties

# Enable standard MongoDB (single-tenant mode)
spring.data.mongodb.enabled=true
openframe.tenant-isolation.enabled=false

spring.data.mongodb.uri=mongodb://localhost:27017/openframe
```

```java
// A standard repository picked up by this config
@Repository
public interface DeviceRepository extends MongoRepository<Device, String> {
    List<Device> findByStatus(String status);
}

// This repository is excluded — handled by a separate tenant-aware config
@TenantAwareRepository
public interface TenantDeviceRepository extends MongoRepository<Device, String> { }
```

## Notes

- Mutually exclusive with tenant-isolation mode — when `openframe.tenant-isolation.enabled=true`, this config is skipped entirely and a separate multi-tenant MongoDB config takes over
- The `@TenantAwareRepository` exclusion filter prevents duplicate bean registration when both config classes are present on the classpath
