Configures Spring Data MongoDB for synchronous (blocking) access, activated only when `spring.data.mongodb.enabled=true` is set.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Guards the entire configuration — MongoDB beans are only registered when the property is explicitly enabled |
| `@EnableMongoRepositories` | Scans `com.openframe.data.repository` for Spring Data repository interfaces |
| `@EnableMongoAuditing` | Enables automatic population of auditing fields (`@CreatedDate`, `@LastModifiedDate`, etc.) |
| `mappingMongoConverter()` | Custom `MappingMongoConverter` bean that wires custom type conversions and replaces `.` in map keys with `__dot__` to avoid MongoDB key restriction errors |

## Usage Example

Enable the configuration in your `application.yml`:

```yaml
spring:
  data:
    mongodb:
      enabled: true
      uri: mongodb://localhost:27017/openframe
```

Use a custom conversion or map field safely:

```java
@Document(collection = "tickets")
public class Ticket {

    @Id
    private String id;

    // Keys with dots (e.g. "meta.region") are stored as "meta__dot__region"
    private Map<String, Object> metadata;

    @CreatedDate
    private Instant createdAt;

    @LastModifiedDate
    private Instant updatedAt;
}
```

## Notes

- The `__dot__` replacement is critical when storing `Map<String, Object>` fields whose keys may contain `.` — MongoDB rejects dotted key names natively.
- Setting `matchIfMissing = false` means MongoDB is **opt-in**; services that don't need persistence won't load any Mongo beans, keeping startup fast and avoiding connection errors in non-database environments.
