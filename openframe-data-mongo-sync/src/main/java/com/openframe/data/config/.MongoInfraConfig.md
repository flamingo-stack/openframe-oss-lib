Configures MongoDB infrastructure for the OpenFrame data layer, conditionally activated when `spring.data.mongodb.enabled=true` and enabling MongoDB auditing support.

## Key Components

- **`MongoInfraConfig`** — Spring `@Configuration` class guarded by `@ConditionalOnProperty`, ensuring MongoDB beans are only registered when explicitly enabled.
- **`@EnableMongoAuditing`** — Activates automatic population of auditing fields (e.g., `@CreatedDate`, `@LastModifiedDate`) on MongoDB documents.
- **`mappingMongoConverter()`** — Overrides the default `MappingMongoConverter` bean to:
  - Wire a `DefaultDbRefResolver` for `@DBRef` resolution.
  - Apply any registered `MongoCustomConversions`.
  - Replace dot characters (`.`) in map keys with `__dot__` to avoid MongoDB key constraint violations.

## Usage Example

```java
# application.yml
spring:
  data:
    mongodb:
      enabled: true
      uri: mongodb://localhost:27017/openframe
```

```java
@Document(collection = "devices")
public class Device {

    @Id
    private String id;

    // Map keys containing dots (e.g., "192.168.1.1") are safely stored
    // as "192__dot__168__dot__1__dot__1" in MongoDB
    private Map<String, Object> metadata;

    @CreatedDate
    private Instant createdAt;

    @LastModifiedDate
    private Instant updatedAt;
}
```

> **Note:** If `spring.data.mongodb.enabled` is `false` or absent, this configuration is skipped entirely — no MongoDB beans are created, allowing the application to run without a MongoDB connection (e.g., in environments using an alternative data store).
