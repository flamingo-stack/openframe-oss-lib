Configures MongoDB infrastructure for the OpenFrame data layer, conditionally activated when `spring.data.mongodb.enabled=true` with MongoDB auditing enabled.

## Key Components

- **`MongoInfraConfig`** — Spring `@Configuration` class that bootstraps core MongoDB infrastructure beans
- **`mappingMongoConverter`** — Custom `MappingMongoConverter` bean that wires together the `DbRefResolver`, mapping context, and custom conversions; replaces MongoDB's default converter
- **Dot Replacement** — Substitutes `.` characters in map keys with `__dot__` to avoid conflicts with MongoDB's dot-notation field addressing

## Annotations

| Annotation | Purpose |
|---|---|
| `@ConditionalOnProperty` | Only loads if `spring.data.mongodb.enabled=true` |
| `@EnableMongoAuditing` | Activates `@CreatedDate`, `@LastModifiedDate`, etc. on entities |

## Usage Example

```java
# application.yml
spring:
  data:
    mongodb:
      enabled: true
      uri: mongodb://localhost:27017/openframe
```

```java
// Entity using auditing fields enabled by this config
@Document
public class Ticket {
    @Id
    private String id;

    @CreatedDate
    private Instant createdAt;

    @LastModifiedDate
    private Instant updatedAt;

    // Map keys with dots are stored as __dot__ in MongoDB
    private Map<String, Object> metadata;
}
```

> **Note:** The `__dot__` key replacement prevents runtime errors when storing `Map<String, Object>` fields whose keys contain `.` characters — common in dynamic metadata payloads across the OpenFrame platform.
