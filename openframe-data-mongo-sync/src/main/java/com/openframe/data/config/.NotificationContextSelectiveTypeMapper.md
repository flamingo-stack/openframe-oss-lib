Custom MongoDB type mapper that suppresses `_class` type hints for `NotificationContext` documents while preserving default type mapping behavior for all other types.

## Key Components

- **`NotificationContextSelectiveTypeMapper`** — Extends `DefaultMongoTypeMapper` to selectively skip type metadata writes for `NotificationContext` and its subclasses.
- **`writeType(TypeInformation<?>, Bson)`** — Overrides type writing via `TypeInformation`; skips if the resolved type is assignable to `NotificationContext`.
- **`writeType(Class<?>, Bson)`** — Overrides type writing via raw `Class`; applies the same suppression logic.

## Usage Example

```java
@Configuration
public class MongoConfig extends AbstractMongoClientConfiguration {

    @Bean
    public MappingMongoConverter mappingMongoConverter(
            MongoDatabaseFactory factory,
            MongoMappingContext mappingContext) {

        NotificationContextSelectiveTypeMapper typeMapper =
            new NotificationContextSelectiveTypeMapper(mappingContext);

        MappingMongoConverter converter =
            new MappingMongoConverter(new DefaultDbRefResolver(factory), mappingContext);

        converter.setTypeMapper(typeMapper);
        return converter;
    }
}
```

With this mapper registered, documents of type `NotificationContext` (and subclasses) are persisted **without** a `_class` field in MongoDB, preventing deserialization conflicts when the class hierarchy evolves or when documents are shared across services that may not have `NotificationContext` on the classpath.
