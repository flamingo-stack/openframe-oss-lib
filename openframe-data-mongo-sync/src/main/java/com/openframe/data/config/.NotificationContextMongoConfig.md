Spring `@Configuration` class that wires MongoDB custom conversions and type mapping for the `NotificationContext` domain, enabling serialization/deserialization of notification context objects via Jackson-backed read/write converters.

## Key Components

| Bean | Description |
|------|-------------|
| `notificationContextReadConverter` | Registers `NotificationContextReadConverter` for deserializing MongoDB documents into `NotificationContext` objects using Jackson |
| `notificationContextWriteConverter` | Registers `NotificationContextWriteConverter` for serializing `NotificationContext` objects into MongoDB documents |
| `notificationContextCustomConversions` | Bundles both converters into a `MongoCustomConversions` instance consumed by Spring Data MongoDB |
| `notificationContextTypeMapperPostProcessor` | Static `BeanPostProcessor` that intercepts `MappingMongoConverter` beans and replaces the default type mapper with `NotificationContextSelectiveTypeMapper`, controlling which documents store `_class` metadata |

**Imports:** `NotificationContextJacksonConfig` via `@Import`, ensuring the correctly configured `ObjectMapper` is available before converter beans are initialized.

## Usage Example

```java
// Automatically applied via Spring component scan or explicit @Import
@Import(NotificationContextMongoConfig.class)
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// Once registered, Spring Data MongoDB transparently invokes the converters:
// Write path: NotificationContext → MongoDocument (via NotificationContextWriteConverter)
// Read path:  MongoDocument → NotificationContext (via NotificationContextReadConverter)
NotificationContext ctx = mongoTemplate.findById(id, NotificationContext.class);
```

> **Note:** The `BeanPostProcessor` is declared `static` to ensure early initialization, preventing it from missing `MappingMongoConverter` bean registration during the Spring context lifecycle.
