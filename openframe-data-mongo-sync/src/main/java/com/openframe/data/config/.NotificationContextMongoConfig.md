Configures MongoDB integration for the notification context, registering custom read converters, conversion mappings, and a selective type mapper for `MappingMongoConverter`.

## Key Components

| Bean | Type | Purpose |
|------|------|---------|
| `notificationContextReadConverter` | `NotificationContextReadConverter` | Deserializes MongoDB documents into notification context objects using Jackson |
| `notificationContextCustomConversions` | `MongoCustomConversions` | Registers the read converter with Spring Data MongoDB's conversion infrastructure |
| `notificationContextTypeMapperPostProcessor` | `BeanPostProcessor` | Intercepts `MappingMongoConverter` initialization to inject a `NotificationContextSelectiveTypeMapper` |

## Usage Example

This configuration is auto-applied via Spring's component scanning. It imports `NotificationContextJacksonConfig` to ensure the `ObjectMapper` is properly configured before wiring converters.

```java
// Triggered automatically on application startup — no manual wiring needed.
// The BeanPostProcessor intercepts MappingMongoConverter and applies
// the selective type mapper to control _class field handling.

@SpringBootApplication
@Import(NotificationContextMongoConfig.class)
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

> **Note:** The `notificationContextTypeMapperPostProcessor` is declared `static` to ensure it is instantiated early in the Spring lifecycle, before other beans are processed — a requirement for `BeanPostProcessor` beans defined inside `@Configuration` classes.
