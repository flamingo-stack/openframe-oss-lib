Spring `@Configuration` class that registers MongoDB custom converters and type mapping for `NotificationContext` persistence, enabling JSON-based serialization/deserialization of notification context objects stored in MongoDB.

## Key Components

| Bean | Description |
|------|-------------|
| `notificationContextReadConverter` | Reads/deserializes `NotificationContext` documents from MongoDB using Jackson `ObjectMapper` |
| `notificationContextWriteConverter` | Writes/serializes `NotificationContext` objects to MongoDB documents using Jackson `ObjectMapper` |
| `notificationContextCustomConversions` | Registers both converters as `MongoCustomConversions` so Spring Data MongoDB uses them automatically |
| `notificationContextTypeMapperPostProcessor` | `BeanPostProcessor` that injects a `NotificationContextSelectiveTypeMapper` into `MappingMongoConverter` to control `_class` type hint storage selectively |

**Imports:** `NotificationContextJacksonConfig` via `@Import` to ensure the shared `ObjectMapper` bean is configured before converters are instantiated.

## Usage Example

```java
// No manual wiring needed — Spring auto-detects the configuration.
// Simply ensure the package is component-scanned or explicitly imported:

@SpringBootApplication
@Import(NotificationContextMongoConfig.class)
public class OpenFrameApplication {
    public static void main(String[] args) {
        SpringApplication.run(OpenFrameApplication.class, args);
    }
}

// At runtime, saving a NotificationContext entity routes through the registered converters:
// Write path: NotificationContext → NotificationContextWriteConverter → MongoDB Document
// Read path:  MongoDB Document   → NotificationContextReadConverter  → NotificationContext
```

> The `BeanPostProcessor` runs statically after `MappingMongoConverter` initialization, replacing the default type mapper with `NotificationContextSelectiveTypeMapper` to avoid writing unnecessary `_class` fields on all MongoDB documents — keeping the schema clean for non-polymorphic notification context records.
