Registers a Jackson `Module` that dynamically maps `NotificationContext` subtypes using all available `NotificationContextDescriptor` beans, enabling polymorphic JSON serialization/deserialization for notification context objects.

## Key Components

| Component | Description |
|-----------|-------------|
| `notificationContextSubtypesModule` | Spring `@Bean` that builds a `SimpleModule` registering each descriptor's context class as a named Jackson subtype |
| `NotificationContextDescriptor` | Interface/contract providing the concrete class (`contextClass()`) and its type discriminator name (`type()`) |

## Usage Example

```java
// Implement NotificationContextDescriptor to register a new subtype
@Component
public class AlertNotificationContextDescriptor implements NotificationContextDescriptor {

    @Override
    public String type() {
        return "alert"; // used as the JSON type discriminator value
    }

    @Override
    public Class<AlertNotificationContext> contextClass() {
        return AlertNotificationContext.class;
    }
}

// Resulting JSON deserialization behavior:
// {
//   "@type": "alert",
//   "severity": "HIGH",
//   "message": "Disk usage critical"
// }
// → deserialized as AlertNotificationContext
```

> **Design Note:** This configuration follows an open/closed pattern — new notification context types are registered automatically by adding a `NotificationContextDescriptor` bean, without modifying this config class. This is essential in OpenFrame's extensible notification pipeline where multiple MSP integrations contribute their own context shapes.
