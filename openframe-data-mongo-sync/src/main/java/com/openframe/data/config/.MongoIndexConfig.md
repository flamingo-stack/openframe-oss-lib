Spring Boot `@Configuration` class that initializes and maintains MongoDB indexes at application startup, ensuring required indexes exist and removing legacy/stale indexes from evolving schema versions.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `mongoTemplate` | `MongoTemplate` | Injected Spring Data MongoDB template for index operations |
| `initIndexes()` | `@PostConstruct` method | Runs after bean initialization to create required indexes and drop stale ones |
| `dropStaleIndex(collection, indexName)` | Private helper | Safely drops a named index, silently skipping if it no longer exists |

## Index Operations

**Created indexes on `application_events`:**

| Fields | Direction | Purpose |
|--------|-----------|---------|
| `userId` ASC, `timestamp` DESC | Compound | User activity timeline queries |
| `type` ASC, `metadata.tags` ASC | Compound | Event type and tag filtering |

**Dropped stale indexes from `tags`:**

| Index Name | Reason |
|------------|--------|
| `name` | Legacy unique index from old schema |
| `key_org_idx` | Replaced by tenant-wide tag uniqueness model |
| `key_org_entity_idx` | Replaced by `key_entity_idx` compound index |

## Usage Example

This bean is auto-configured by Spring — no manual invocation required. The `@PostConstruct` lifecycle hook fires once during application startup:

```java
// No manual wiring needed — Spring detects @Configuration and runs @PostConstruct automatically.
// To add a new index, extend initIndexes():

mongoTemplate.indexOps("your_collection")
    .ensureIndex(new Index()
        .on("fieldName", Sort.Direction.ASC)
        .unique());

// To clean up a deprecated index safely:
dropStaleIndex("your_collection", "old_index_name");
```

> **Note:** `dropStaleIndex` is intentionally fault-tolerant — if the index does not exist, it logs at `DEBUG` level and continues, making it safe to run on both fresh and migrated deployments.
