Unit tests for `NotificationContextWriteConverter`, validating the conversion of `NotificationContext` objects into BSON `Document` instances for MongoDB persistence.

## Key Components

| Test | Description |
|------|-------------|
| `given_null_source_when_converting_then_returns_null` | Verifies that a `null` input short-circuits conversion and returns `null` without invoking Jackson |
| `given_generic_context_when_converting_then_document_contains_type_and_payload` | Asserts that a populated `GenericContext` produces a BSON `Document` with the correct `type` discriminator and `payload` fields |
| `given_jackson_failure_when_converting_then_throws_with_type_in_message` | Confirms that a Jackson serialisation failure wraps the underlying `JsonProcessingException` in an `IllegalStateException` whose message includes the context type for diagnosability |

## Usage Example

```java
// Typical converter instantiation and happy-path usage mirrored by the tests
ObjectMapper objectMapper = new ObjectMapper();
NotificationContextWriteConverter converter = new NotificationContextWriteConverter(objectMapper);

GenericContext context = GenericContext.builder()
        .type("GENERIC")
        .payload("hello")
        .build();

Document bsonDoc = converter.convert(context);
// bsonDoc.getString("type")    → "GENERIC"
// bsonDoc.getString("payload") → "hello"

// Null-safe – converter returns null without throwing
assertThat(converter.convert(null)).isNull();
```

## Notes

- Uses **Mockito BDD-style** stubbing (`given(…).willThrow(…)`) to simulate Jackson failures without real serialisation overhead.
- The `IllegalStateException` message deliberately includes the type discriminator so production log searches can pinpoint which context variant caused a persistence failure.
- Tests follow the **Given / When / Then** naming convention enforced via `@DisplayName`.
