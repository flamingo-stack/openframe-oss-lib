Unit test suite for `NotificationContextReadConverter`, verifying correct behavior when converting MongoDB `Document` objects to `NotificationContext` instances via Jackson deserialization.

## Key Components

| Method | Description |
|---|---|
| `given_null_source_when_converting_then_returns_null` | Asserts that passing `null` returns `null` without invoking the `ObjectMapper` |
| `given_jackson_failure_when_converting_then_throws_with_id_and_type_in_message` | Asserts that a Jackson parse failure wraps into an `IllegalStateException` containing the document `_id` and `type` for production log diagnosability |

## Usage Example

```java
// Null input — converter short-circuits cleanly
ObjectMapper objectMapper = mock(ObjectMapper.class);
NotificationContextReadConverter converter =
    new NotificationContextReadConverter(objectMapper);

assertThat(converter.convert(null)).isNull();

// Malformed document — Jackson failure surfaces as IllegalStateException
given(objectMapper.readValue(anyString(), eq(NotificationContext.class)))
    .willThrow(new JsonParseException(null, "simulated parse failure"));

Document bad = new Document("_id", "doc-id-42").append("type", "weird-type");

assertThatThrownBy(() -> converter.convert(bad))
    .isInstanceOf(IllegalStateException.class)
    .hasMessageContaining("doc-id-42")
    .hasMessageContaining("weird-type")
    .hasCauseInstanceOf(IOException.class);
```

## Key Behaviors Verified

- **Null safety** — converter returns `null` for `null` input without side effects
- **Error context** — deserialization failures always include `_id` and `type` in the exception message, ensuring failures are traceable in production logs
- **Exception chaining** — the original `IOException` is preserved as the cause of the `IllegalStateException`
