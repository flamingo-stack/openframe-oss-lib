Utility class for executing and asserting MongoDB `explain` commands in test contexts, wrapping Spring Data's `MongoTemplate` to capture query execution statistics and validate index usage.

## Key Components

### `MongoExplain` (static factory methods)

| Method | Description |
|---|---|
| `explainFind` | Explains a `find` query with optional sort, projection, and limit |
| `explainCount` | Explains a `count` operation |
| `explainAggregation` | Explains an aggregation pipeline (handles `$cursor` stage unwrapping) |
| `explainUpsert` | Explains an upsert `update` operation |

### `Stats` (inner result class)

Parsed from MongoDB's `executionStats` verbosity output. Exposes a fluent assertion API and raw metrics.

| Member | Description |
|---|---|
| `assertNoCollectionScan()` | Fails if any leaf stage is `COLLSCAN` |
| `assertUsesIndex(name)` | Fails if the named index is not found in `IXSCAN` nodes |
| `assertUsesAnyOf(names...)` | Fails if none of the named indexes are used |
| `assertExaminedAtMost(n)` | Asserts `totalKeysExamined ≤ n` |
| `assertExecutionTimeBelow(ms)` | Asserts `executionTimeMillis < ms` |
| `keysExamined()` | Returns raw keys examined count |
| `executionTimeMillis()` | Returns raw execution time |
| `indexesUsed()` | Returns list of index names hit by `IXSCAN` nodes |

## Usage Example

```java
@Test
void deviceLookupUsesIndex() {
    Query query = new Query(Criteria.where("tenantId").is("acme")
        .and("status").is("ACTIVE"));

    MongoExplain.explainFind(mongoTemplate, "devices", query)
        .assertNoCollectionScan()
        .assertUsesIndex("tenantId_1_status_1")
        .assertExaminedAtMost(50)
        .assertExecutionTimeBelow(10);
}

@Test
void aggregationPipelineUsesIndex() {
    List<Document> pipeline = List.of(
        new Document("$match", new Document("tenantId", "acme")),
        new Document("$group", new Document("_id", "$category"))
    );

    MongoExplain.explainAggregation(mongoTemplate, "devices", pipeline)
        .assertNoCollectionScan()
        .assertUsesAnyOf("tenantId_1", "tenantId_1_category_1");
}
```

> All assertion methods return `this`, enabling fluent chaining. Assertions use AssertJ under the hood and include the full explain plan JSON in failure messages for easy debugging.
