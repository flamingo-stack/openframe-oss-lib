A Spring component that registers the `TestExtendedContext` type with the notification system by implementing `NotificationContextDescriptor`, enabling deserialization of extended notification contexts in integration tests.

## Key Components

| Element | Description |
|---|---|
| `TestExtendedContextDescriptor` | Spring `@Component` implementing `NotificationContextDescriptor` |
| `type()` | Returns `TestExtendedContext.TYPE` to identify this context variant |
| `contextClass()` | Returns `TestExtendedContext.class` for polymorphic deserialization |

## Usage Example

```java
// Automatically picked up by Spring and used by the notification framework
// to resolve the correct context class when deserializing notification payloads

@Component
public class TestExtendedContextDescriptor implements NotificationContextDescriptor {

    @Override
    public String type() {
        return TestExtendedContext.TYPE; // e.g. "test_extended"
    }

    @Override
    public Class<? extends NotificationContext> contextClass() {
        return TestExtendedContext.class;
    }
}
```

> This descriptor is part of the test integration layer. It mirrors the pattern used by production `NotificationContextDescriptor` implementations, allowing the notification framework to correctly map a type string to its concrete `NotificationContext` subclass during document processing tests.
