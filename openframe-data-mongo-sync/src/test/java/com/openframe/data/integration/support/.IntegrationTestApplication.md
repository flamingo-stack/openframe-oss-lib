Minimal Spring Boot application context used exclusively for integration tests within the `com.openframe.data` module.

## Key Components

| Element | Description |
|---|---|
| `@SpringBootConfiguration` | Marks this as the bootstrap configuration class for the test context |
| `@EnableAutoConfiguration` | Activates Spring Boot's auto-configuration for the test environment |
| `@EnableMongoAuditing` | Enables MongoDB auditing (e.g., `@CreatedDate`, `@LastModifiedDate`) during tests |
| `@EnableMongoRepositories` | Scans for MongoDB repositories starting from `NotificationRepository` |
| `@Import` | Manually wires required beans not covered by auto-configuration |

**Imported beans:**

- `CustomNotificationRepositoryImpl` — custom MongoDB query logic for notifications
- `CustomNotificationReadStateRepositoryImpl` — custom read-state persistence logic
- `NotificationReadStateService` — service layer for managing notification read states
- `NotificationContextMongoConfig` — MongoDB configuration for notification context
- `TestExtendedContextDescriptor` — test-specific extended context descriptor

## Usage Example

```java
@SpringBootTest(classes = IntegrationTestApplication.class)
@AutoConfigureDataMongo
class NotificationRepositoryIntegrationTest {

    @Autowired
    private NotificationRepository notificationRepository;

    @Test
    void shouldPersistNotification() {
        // arrange, act, assert against a real (or embedded) MongoDB instance
    }
}
```

> This class contains no business logic. Its sole purpose is to bootstrap a minimal, focused application context for integration tests — importing only the beans required by notification-related repository and service tests.
