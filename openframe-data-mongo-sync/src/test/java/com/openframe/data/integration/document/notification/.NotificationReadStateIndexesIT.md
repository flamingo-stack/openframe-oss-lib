Integration test that verifies MongoDB index definitions on the `NotificationReadState` document collection are correctly resolved and applied, and that deprecated legacy indexes are absent.

## Key Components

| Element | Description |
|---|---|
| `NotificationReadStateIndexesIT` | JUnit 5 integration test class extending `BaseMongoIntegrationTest` |
| `MongoTemplate` | Spring Data autowired bean used to access index operations |
| `MongoPersistentEntityIndexResolver` | Resolves and applies entity-declared indexes to MongoDB |
| `@EnabledIfSystemProperty` | Guards execution — only runs when `-Dintegration.tests=true` is set |

## Test: `given_read_state_entity_when_indexes_resolved_then_required_compound_indexes_exist`

Resolves and ensures all indexes declared on `NotificationReadState`, then asserts:

- **`recipient_notification_unique`** — compound unique index on `(recipientId, recipientType, notificationId)`
- **`recipient_status`** — compound index on `(recipientId, recipientType, status)`
- **`recipient_category_status`** — compound index on `(recipientId, recipientType, category, status)`
- **`user_read_at`** — legacy index confirmed **absent**

## Usage Example

```bash
# Run integration tests locally (requires a running MongoDB instance)
./mvnw verify -Dintegration.tests=true -Pit
```

```java
// Index resolution pattern used inside the test
IndexOperations indexOps = mongoTemplate.indexOps(NotificationReadState.class);
new MongoPersistentEntityIndexResolver(mongoTemplate.getConverter().getMappingContext())
    .resolveIndexFor(NotificationReadState.class)
    .forEach(indexOps::ensureIndex);

List<IndexInfo> indexes = indexOps.getIndexInfo();
```

> **Note:** This test is skipped by default in CI unless the `integration.tests` system property is explicitly enabled. It is tagged `integration` for selective test suite filtering.
