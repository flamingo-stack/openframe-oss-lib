Integration test suite for `NotificationReadStateService`, verifying read/unread state lifecycle, per-recipient isolation, bulk operations, and soft-delete semantics against a real MongoDB instance.

## Key Components

| Element | Purpose |
|---|---|
| `unread_to_read_round_trip` | Validates the full UNREAD → READ state transition for a single recipient |
| `blank_input_throws` | Ensures `null`/blank `recipientId` or `recipientType` raises `ConstraintViolationException` |
| `mark_read_no_audience` | Confirms `markRead` returns `false` when the caller has no read-state row |
| `audience_isolation_by_id` | Verifies marking read for one recipient does not affect others sharing the same notification |
| `audience_isolation_by_type` | Confirms `USER` and `MACHINE` recipient types are tracked independently |
| `mark_all_as_read` | Validates bulk read operation flips all UNREAD rows and returns the correct count |
| `delete_notification_soft_deletes` | Asserts `deleteNotification` sets `status=DELETED` without removing the document |
| `delete_all_read` | Verifies only READ rows transition to DELETED; UNREAD rows remain intact |
| `unread_counts_by_category` | Validates per-category aggregation returns correct counts scoped to the recipient |

## Usage Example

```java
// Typical test flow mirrors real service usage:

// 1. Seed audience — creates UNREAD rows in MongoDB
service.createForAudience("notif-1", NotificationCategory.TICKETS, "title",
    RecipientType.USER, Set.of("user-alice"));

// 2. Assert unread state
assertThat(service.hasUnread("user-alice", RecipientType.USER)).isTrue();

// 3. Mark read and verify transition
assertThat(service.markRead("user-alice", RecipientType.USER, "notif-1")).isTrue();
assertThat(service.hasUnread("user-alice", RecipientType.USER)).isFalse();

// 4. Aggregate unread by category
Map<NotificationCategory, Long> counts =
    service.unreadCountsByCategory("user-alice", RecipientType.USER);
```

> Tests run only when the system property `integration.tests=true` is set and require a live MongoDB instance via `BaseMongoIntegrationTest`. Each test resets both collections in `@BeforeEach` to guarantee isolation.
