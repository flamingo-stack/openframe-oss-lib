Abstract base class for MongoDB integration tests, providing automatic container lifecycle management via Testcontainers with optional override to an external MongoDB instance.

## Key Components

| Component | Description |
|-----------|-------------|
| `EXTERNAL_URI` | JVM system property `mongo.external.uri`; when set, skips Testcontainers and connects to an existing MongoDB instance |
| `MONGO` | Shared `MongoDBContainer` running `mongo:7`, lazily started on first use |
| `mongoProperties()` | `@DynamicPropertySource` hook that injects `spring.data.mongodb.uri` and enables auto-index creation into the Spring context |
| `resolveUri()` | Resolution logic: external URI → running container → start container, returning a `directConnection=true` URI pointed at the `test` database |

## Usage Example

```java
@SpringBootTest
@Testcontainers
class UserRepositoryIntegrationTest extends BaseMongoIntegrationTest {

    @Autowired
    private UserRepository userRepository;

    @Test
    void shouldPersistAndFindUser() {
        User user = new User("alice@example.com");
        userRepository.save(user);

        Optional<User> found = userRepository.findByEmail("alice@example.com");
        assertThat(found).isPresent();
    }
}
```

To target an external MongoDB instance instead of Testcontainers, pass the JVM argument at test runtime:

```bash
./mvnw test -Dmongo.external.uri=mongodb://localhost:27017/test
```

## Notes

- The `MONGO` container is `static`, so it is shared across all subclasses in the same JVM, avoiding redundant container startups.
- Auto-index creation is always enabled, keeping entity `@Indexed` annotations effective during tests without manual setup.
