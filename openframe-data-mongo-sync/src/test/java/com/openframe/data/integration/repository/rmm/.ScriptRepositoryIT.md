Integration test suite for `ScriptRepository`, verifying MongoDB persistence, multi-tenant data isolation, cursor-based pagination, soft-delete filtering, and compound unique index enforcement for the `Script` document.

## Key Components

- **`resetCollection()`** — Clears all `Script` documents before each test while preserving indexes (avoids losing the compound unique constraint that Spring Data only creates at startup).
- **Auditing tests** — Validates that `createdAt`/`updatedAt` are populated automatically via Spring Data MongoDB auditing on save.
- **Tenant isolation tests** — Covers `findByTenantIdAndId`, `findByTenantIdAndName`, `existsByTenantIdAndName`, and `deleteByTenantIdAndId` to ensure no cross-tenant data leakage.
- **Pagination tests** — Exercises `findPageForTenant` for no-cursor (first page), forward cursor (older rows), backward cursor (newer rows, ASC), limit enforcement, tenant filtering, soft-delete exclusion, and graceful fallback on invalid cursor values.
- **Collision detection tests** — Validates `existsByTenantIdAndNameAndIdNot` correctly ignores self-collision during edits and detects real name conflicts within a tenant.
- **Compound unique index tests** — Confirms `DuplicateKeyException` is thrown for duplicate `(tenantId, name)` pairs, and that the same name is allowed across different tenants.

## Usage Example

```java
// Seed sequential scripts and paginate newest-first
List<Script> seeded = seedSequentialForTenant("tenant-a", 5);

List<Script> page = scriptRepository.findPageForTenant(
    "tenant-a",
    null, null,
    "_id", Sort.Direction.DESC,
    null, false, 10
);
// Returns all 5 scripts, newest first

// Forward cursor — fetch rows older than cursor
String cursor = seeded.get(2).getId();
List<Script> olderPage = scriptRepository.findPageForTenant(
    "tenant-a", null, null,
    "_id", Sort.Direction.DESC,
    cursor, false, 10
);
// Returns seeded[1], seeded[0]
```
