Integration test suite for `ScriptRepository`, validating MongoDB persistence, multi-tenant isolation, cursor-based pagination, soft-delete filtering, and compound unique index enforcement for the `Script` document.

## Key Components

- **`resetCollection()`** — Clears all `Script` documents before each test while preserving indexes (avoids losing the compound unique constraint that Spring Data only creates at startup).
- **Audit tests** — Verifies that `createdAt`/`updatedAt` fields are auto-populated on save.
- **Tenant isolation tests** — Confirms `findByTenantIdAndId`, `findByTenantIdAndName`, `existsByTenantIdAndName`, `deleteByTenantIdAndId`, and `findPageForTenant` never leak data across tenant boundaries.
- **Cursor pagination tests** — Covers no-cursor (newest-first), forward cursor (older rows), backward cursor (newer rows ascending), page size limit, and graceful fallback on invalid cursor values.
- **Soft-delete filtering** — Ensures `DELETED` status scripts are excluded from default list queries.
- **Collision detection** — Tests `existsByTenantIdAndNameAndIdNot` for self-edit (no false collision) and real same-name conflict.
- **Compound unique index** — Asserts duplicate `(tenantId, name)` throws `DuplicateKeyException`, while the same name across different tenants is allowed.

## Usage Example

```java
// Run integration tests with the required system property
// mvn verify -Dintegration.tests=true

// Typical test pattern used throughout the suite
Script script = newScript(TENANT_A, "Restart Spooler");
Script saved = scriptRepository.save(script);

// Tenant-scoped lookup
Optional<Script> found = scriptRepository.findByTenantIdAndId(TENANT_A, saved.getId());

// Cursor-based pagination (newest-first, page size 10)
List<Script> page = scriptRepository.findPageForTenant(
    TENANT_A, null, null, "_id", Sort.Direction.DESC, null, false, 10);

// Cross-tenant delete guard
long removed = scriptRepository.deleteByTenantIdAndId(TENANT_B, saved.getId()); // returns 0
```
