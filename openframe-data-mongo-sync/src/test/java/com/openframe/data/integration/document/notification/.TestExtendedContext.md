A concrete test implementation of `NotificationContext` used to verify polymorphic serialization and deserialization of extended notification context types, particularly validating that subclass-specific fields are preserved through JSON processing.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `TYPE` | `String` constant | Discriminator value `"TEST_EXTENDED_CONTEXT"` identifying this context subtype |
| `extraField` | `String` | Additional string field extending the base context |
| `extraNumber` | `int` | Additional numeric field extending the base context |
| `getType()` | Method | Returns `TYPE`; the `type` field is write-ignored via `@JsonIgnoreProperties` to prevent deserialization conflicts |

## Usage Example

```java
// Build an instance using the Lombok SuperBuilder
TestExtendedContext ctx = TestExtendedContext.builder()
    .extraField("some-value")
    .extraNumber(42)
    .build();

// Serializes with type discriminator included via getter
String json = objectMapper.writeValueAsString(ctx);
// {"type":"TEST_EXTENDED_CONTEXT","extraField":"some-value","extraNumber":42}

// Polymorphic deserialization resolves back to TestExtendedContext
NotificationContext base = objectMapper.readValue(json, NotificationContext.class);
assert base instanceof TestExtendedContext;
```

## Notes

- `@JsonIgnoreProperties(value = "type", allowGetters = true)` allows `type` to be **serialized** (via getter) but **ignored on deserialization**, preventing conflicts with the parent class's type discriminator field.
- Lombok's `@SuperBuilder`, `@EqualsAndHashCode(callSuper = true)`, and `@ToString(callSuper = true)` ensure proper inheritance-aware behavior across builder construction, equality checks, and string representation.
