A concrete test implementation of `NotificationContext` used to verify polymorphic serialization/deserialization behavior in notification context integration tests, adding extra fields beyond the base context.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `TYPE` | `String` constant | Discriminator value `"TEST_EXTENDED_CONTEXT"` returned by `getType()` |
| `extraField` | `String` | Additional string field extending the base context |
| `extraNumber` | `int` | Additional numeric field extending the base context |
| `getType()` | Method | Overrides base class to return the `TYPE` discriminator |

## Usage Example

```java
// Build a test context instance using the Lombok SuperBuilder
TestExtendedContext ctx = TestExtendedContext.builder()
        .extraField("some-value")
        .extraNumber(42)
        // ...set base NotificationContext fields here
        .build();

// Type discriminator is serialized as a getter-only field
String type = ctx.getType(); // "TEST_EXTENDED_CONTEXT"

// Jackson ignores "type" on deserialization but exposes it on serialization
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(ctx);
// json includes: { "type": "TEST_EXTENDED_CONTEXT", "extraField": "some-value", "extraNumber": 42, ... }
```

## Notes

- `@JsonIgnoreProperties(value = "type", allowGetters = true)` ensures `type` is **written** during serialization but **ignored** during deserialization, preventing unknown field errors.
- Lombok `@SuperBuilder` and `@EqualsAndHashCode(callSuper = true)` propagate builder and equality logic to the parent `NotificationContext`.
- Intended purely for **test use**; not a production notification context type.
