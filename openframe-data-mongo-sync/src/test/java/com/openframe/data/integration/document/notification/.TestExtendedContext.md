A test implementation of `NotificationContext` that extends the base class with additional fields, used for validating polymorphic notification context handling in integration tests.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `TYPE` | `String` constant | Discriminator value `"TEST_EXTENDED_CONTEXT"` identifying this context subtype |
| `extraField` | `String` | Additional string field for testing context extension |
| `extraNumber` | `int` | Additional numeric field for testing context extension |

## Usage Example

```java
// Build a test extended context instance
TestExtendedContext context = TestExtendedContext.builder()
    .extraField("someValue")
    .extraNumber(42)
    .build();

// Use type discriminator for routing/assertions
String type = TestExtendedContext.TYPE; // "TEST_EXTENDED_CONTEXT"

// Polymorphic usage via base class
NotificationContext base = context;
System.out.println(base.toString()); // includes superclass fields via callSuper
```

## Notes

- Lombok `@SuperBuilder` enables fluent builder chaining with inherited fields from `NotificationContext`
- `@EqualsAndHashCode(callSuper = true)` and `@ToString(callSuper = true)` ensure parent class fields are included in equality checks and string representation
- Intended for use in integration tests only — not a production notification context type
