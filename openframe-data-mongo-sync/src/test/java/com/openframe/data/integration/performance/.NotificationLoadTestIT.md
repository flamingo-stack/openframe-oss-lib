Integration performance test that benchmarks MongoDB notification query and write operations at scale (up to 1,000,000 documents), validating index efficiency, latency budgets, and query plan coverage across five real-world inbox scenarios.

## Key Components

| Component | Description |
|-----------|-------------|
| `seed()` | `@BeforeAll` setup — drops and rebuilds collections, generates hot-user slices, sparse-match data, and noise documents via `BatchInserter` |
| `captureEnvironment()` | Records JVM, OS, CPU, heap, and MongoDB version metadata into the performance report |
| `installSectionDescriptions()` | Registers human-readable descriptions for each of the five benchmark scenarios |
| `PerfResultRecorder` | Collects timing samples, environment metadata, and scenario descriptions for structured report output |
| `BatchInserter` | Internal helper that buffers documents and bulk-inserts at `BATCH_FLUSH_SIZE` (5,000) intervals |
| `HOT_SLICES` | User inbox sizes under test: 100, 1,000, 10,000, 100,000 notifications |

## Scenarios Benchmarked

- **S1 — List inbox:** First-page fetch across all hot-user slice sizes; validates `recipient_notification_unique` index prevents linear scans
- **S2 — Dense search:** Title-regex with 100% hit rate; measures read-state collection pushdown
- **S3 — Sparse search:** ~1% match rate over 30,000 rows; must stay within `SEARCH_BUDGET_MS` (1,500 ms)
- **S4 — Bulk mark-as-read:** Single `updateMany` flipping 100,000 rows `UNREAD → READ`; budget 5,000 ms
- **S5 — Unread counts:** Category-grouped aggregation; must be index-covered with no `FETCH` stage

## Usage Example

```bash
# Run only load tests (disabled by default)
mvn verify -Dloadtest.tests=true -Dnotification.loadtest.total=500000
```

```java
// Key tuning constants (configurable via system properties)
private static final int TOTAL        = Integer.getInteger("notification.loadtest.total", 1_000_000);
private static final int SPARSE_SLICE = Integer.getInteger("notification.loadtest.sparse", 30_000);
private static final int SAMPLES = 20;
private static final int WARMUP  = 5;

// Latency budgets enforced per scenario
private static final long LIST_BUDGET_MS   = 500;
private static final long SEARCH_BUDGET_MS = 1_500;
private static final long WRITE_BUDGET_MS  = 5_000;
```

> **Note:** The test is gated by `@EnabledIfSystemProperty(named = "loadtest.tests", matches = "true")` and tagged `@Tag("loadtest")`, so it only executes in dedicated performance CI pipelines. Reports are written to `target/perf-results/`.
