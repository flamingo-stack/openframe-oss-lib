Integration/performance test that load-tests the `Notification` and `NotificationReadState` MongoDB collections at scale (up to 1 million documents), measuring latency for inbox listing, search, bulk updates, and unread count aggregations.

## Key Components

| Component | Description |
|-----------|-------------|
| `seed()` (@BeforeAll) | Seeds MongoDB with hot-user slices (100–100k), sparse users, and noise recipients up to `TOTAL` documents |
| `captureEnvironment()` | Records JVM, OS, and MongoDB version metadata into the perf report |
| `installSectionDescriptions()` | Registers human-readable descriptions for each scenario in the `PerfResultRecorder` |
| `HOT_SLICES` | Array `{100, 1_000, 10_000, 100_000}` — inbox sizes for latency regression testing |
| `SPARSE_SLICE` / `SPARSE_MATCH_EVERY` | Models a ~1% search hit-rate scenario (30k docs, match every 100th) |
| `BatchInserter` | Helper for bulk-flushing documents into MongoDB collections in batches of 5,000 |
| `PerfResultRecorder` | Captures wall-time percentiles (P50/P95/P99), explain output, and writes a perf report to `target/perf-results/` |

### Scenarios Measured

| ID | Scenario | Budget |
|----|----------|--------|
| S1 | Inbox list — first page, no search | 500 ms |
| S2 | Dense search — 100% title match | 500 ms |
| S3 | Sparse search — ~1% title match | 1,500 ms |
| S4 | Mark all as read — bulk status flip | 5,000 ms |
| S5 | Unread counts by category (aggregation) | — |

## Usage Example

```bash
# Run load tests explicitly (disabled by default)
mvn verify -Dloadtest.tests=true

# Tune dataset size and sparse slice
mvn verify -Dloadtest.tests=true \
  -Dnotification.loadtest.total=500000 \
  -Dnotification.loadtest.sparse=15000
```

```java
// Tests only activate when system property is set
@EnabledIfSystemProperty(named = "loadtest.tests", matches = "true")
@Tag("loadtest")
class NotificationLoadTestIT extends BaseMongoIntegrationTest { ... }
```

> **Note:** Results are written to `target/perf-results/` after each run. The test is annotated `@Tag("loadtest")` so it is excluded from standard CI and must be opted into explicitly.
