Integration test that benchmarks notification query and write performance against a MongoDB database seeded with up to 1 million documents, validating latency budgets for inbox listing, search, bulk updates, and unread count aggregations.

## Key Components

| Component | Description |
|-----------|-------------|
| `seed()` | `@BeforeAll` setup: drops collections, rebuilds indexes, and inserts hot-user slices, sparse-user data, and noise recipients up to `TOTAL` documents |
| `captureEnvironment()` | Records JVM, OS, CPU, heap, and MongoDB version metadata into the `PerfResultRecorder` |
| `installSectionDescriptions()` | Attaches human-readable descriptions to each benchmark scenario for the generated report |
| `BatchInserter` | Internal helper that buffers documents and flushes in configurable batch sizes (`BATCH_FLUSH_SIZE = 5,000`) |
| `PerfResultRecorder` | Collects timing samples, environment metadata, and scenario descriptions; writes the final performance report |
| `HOT_SLICES` | Array of inbox sizes (100, 1k, 10k, 100k) used to test that query latency does not scale with inbox size |

**Scenarios measured:**

| Scenario | Budget |
|----------|--------|
| S1 — List inbox (first page, no search) | 500 ms |
| S2 — Dense search (100% hit rate) | 1,500 ms |
| S3 — Sparse search (~1% hit rate) | 1,500 ms |
| S4 — Mark all as read (bulk update, 100k rows) | 5,000 ms |
| S5 — Unread counts by category (aggregation) | — |

## Usage Example

```bash
# Run only load tests (disabled by default via @EnabledIfSystemProperty)
./mvnw verify -Dloadtest.tests=true \
               -Dnotification.loadtest.total=1000000 \
               -Dnotification.loadtest.sparse=30000 \
               -pl openframe-data
```

```java
// The test is activated only when the system property is set
@EnabledIfSystemProperty(named = "loadtest.tests", matches = "true")
@Tag("loadtest")
class NotificationLoadTestIT extends BaseMongoIntegrationTest { ... }
```

> **Note:** Results are written to `target/perf-results/`. Tests require a real MongoDB instance provided by `BaseMongoIntegrationTest`. Join the [OpenMSP Slack](https://join.slack.com/t/openmsp/shared_invite/zt-36bl7mx0h-3~U2nFH6nqHqoTPXMaHEHA) to discuss benchmarking and contribute.
