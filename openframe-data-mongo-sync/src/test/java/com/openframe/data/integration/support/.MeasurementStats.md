Immutable record for capturing latency statistics over repeated executions of a measurable operation, supporting warmup runs and percentile calculations.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `samples` | `int` | Total number of measured samples |
| `min` / `max` | `long` | Minimum and maximum observed values |
| `mean` | `long` | Arithmetic mean across all samples |
| `p50` / `p95` / `p99` | `long` | 50th, 95th, and 99th percentile latencies |
| `measure(warmup, samples, op)` | `static` | Runs the operation `warmup + samples` times, discards warmup results, and returns a populated `MeasurementStats` |
| `timeMillis(op)` | `static` | Convenience helper that times a `Runnable` in milliseconds using `System.nanoTime()` |
| `percentile(sorted, pct)` | `private static` | Ceiling-based percentile index calculation over a pre-sorted array |

## Usage Example

```java
// Measure a database query over 5 warmup + 100 sample runs
MeasurementStats stats = MeasurementStats.measure(5, 100, () ->
    MeasurementStats.timeMillis(() -> myRepository.findAll())
);

System.out.printf(
    "samples=%d  min=%dms  mean=%dms  p95=%dms  p99=%dms  max=%dms%n",
    stats.samples(),
    stats.min(),
    stats.mean(),
    stats.p95(),
    stats.p99(),
    stats.max()
);

// Quick one-off timing
long elapsed = MeasurementStats.timeMillis(() -> expensiveOperation());
System.out.println("Elapsed: " + elapsed + "ms");
```

> **Note:** All measurement values are in whatever unit `LongSupplier` returns. Pair with `timeMillis()` for millisecond latency benchmarks, or supply nanosecond values directly for higher precision.
