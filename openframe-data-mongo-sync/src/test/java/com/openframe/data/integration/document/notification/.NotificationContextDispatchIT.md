Integration test verifying that `Notification` documents with polymorphic context objects are correctly persisted to and deserialized from MongoDB, using `type` as the sole discriminator field (no `_class` field).

## Key Components

| Component | Role |
|-----------|------|
| `NotificationContextDispatchIT` | Main test class extending `BaseMongoIntegrationTest` |
| `NotificationRepository` | Spring Data repository under test for saving/retrieving notifications |
| `MongoTemplate` | Used for raw document inspection to validate storage structure |
| `TestExtendedContext` | Concrete `GenericContext` subclass with extra fields used to exercise polymorphic dispatch |
| `resetCollection()` | `@BeforeEach` teardown that drops the `Notification` collection before each test |

## Test Coverage

| Test | Validates |
|------|-----------|
| `given_extended_context_when_reread_then_dispatches_to_concrete_subclass` | A saved `Notification` with a typed extended context round-trips back as the correct concrete subclass with all extra fields intact |
| `given_generic_context_when_reread_then_round_trips_as_generic` | A `GenericContext` with an arbitrary payload round-trips correctly without type loss |
| `given_saved_notification_when_inspecting_raw_doc_then_context_has_no_class_field` | The raw MongoDB document uses `type` as the sole discriminator — no Spring `_class` metadata is written to the embedded context |

## Usage Example

```java
// Typical pattern exercised by these tests:
Notification notification = Notification.builder()
        .title("My Event")
        .createdAt(Instant.now())
        .context(TestExtendedContext.builder()
                .type(TestExtendedContext.TYPE)
                .extraField("value")
                .extraNumber(99)
                .build())
        .build();

Notification saved = repository.save(notification);
Notification loaded = repository.findById(saved.getId()).orElseThrow();

// Polymorphic dispatch — resolves to concrete subclass via `type`
assertThat(loaded.getContext()).isInstanceOf(TestExtendedContext.class);
```

> **Run condition:** Tests are gated by the system property `integration.tests=true` and require a live MongoDB instance configured via `BaseMongoIntegrationTest`.
