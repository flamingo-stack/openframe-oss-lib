Integration test verifying that `Notification` documents with polymorphic context fields are correctly serialized to and deserialized from MongoDB, using `type` as the sole discriminator (no `_class` field).

## Key Components

| Component | Role |
|-----------|------|
| `NotificationContextDispatchIT` | Main test class extending `BaseMongoIntegrationTest` |
| `NotificationRepository` | Spring Data repository under test |
| `MongoTemplate` | Used for raw document inspection and collection teardown |
| `TestExtendedContext` | Concrete `GenericContext` subclass with extra fields used in tests |
| `@EnabledIfSystemProperty` | Guards execution behind `-Dintegration.tests=true` |

## Key Components

**`given_extended_context_when_reread_then_dispatches_to_concrete_subclass`**
Saves a `Notification` with a `TestExtendedContext` payload and asserts the deserialized result is the concrete subclass with all extra fields intact.

**`given_generic_context_when_reread_then_round_trips_as_generic`**
Saves a `Notification` with a `GenericContext` and asserts the `type` and raw `payload` string survive the round-trip.

**`given_saved_notification_when_inspecting_raw_doc_then_context_has_no_class_field`**
Reads the raw MongoDB `Document` directly and asserts the embedded `context` subdocument contains a `type` field but **no** `_class` field, confirming Spring Data's default class-discriminator is suppressed.

## Usage Example

```java
// Run integration tests with the required system property
// mvn verify -Dintegration.tests=true

// Typical assertion pattern used in this test
Optional<Notification> retrieved = repository.findById(persisted.getId());

assertThat(retrieved.get().getContext()).isInstanceOf(TestExtendedContext.class);

// Raw document check — no _class discriminator
Document context = raw.get("context", Document.class);
assertThat(context.containsKey("_class")).isFalse();
assertThat(context.getString("type")).isEqualTo(TestExtendedContext.TYPE);
```

> **Note:** All tests require a running MongoDB instance. The collection is dropped via `@BeforeEach` to ensure test isolation. These tests are part of the OpenFrame data layer and validate the polymorphic context dispatch strategy used across the Flamingo notification pipeline.
