Integration test suite that validates cursor-based pagination, filtering, audience isolation, and title search behavior for the `NotificationRepository` against a real MongoDB instance.

## Key Components

| Element | Description |
|---------|-------------|
| `BaseMongoIntegrationTest` | Base class providing MongoDB setup and seeding helpers |
| `NotificationRepository` | Repository under test with custom pagination queries |
| `NotificationWithStatus` | Projection pairing a `Notification` with its `ReadStatus` |
| `NotificationFixtures` | Factory for building test `Notification` documents |
| `page(...)` | Helper that invokes the repository's paginated listing method |
| `seedSequentialForRecipient(...)` | Creates ordered notifications + read states for a given recipient |

## Test Coverage Summary

| Scenario | Test Method |
|----------|-------------|
| Default descending order (no cursor) | `given_no_cursor_returns_descending` |
| Forward cursor (older rows) | `given_forward_cursor_returns_older_rows` |
| Backward cursor (newer rows) | `given_backward_cursor_returns_newer_rows` |
| Recipient ID isolation | `audience_isolation_by_id` |
| Recipient type isolation (`USER` vs `MACHINE`) | `audience_isolation_by_type` |
| Page size limit enforcement | `limit_honored` |
| Empty result when no rows | `empty_when_no_rows` |
| `DELETED` status excluded by default | `deleted_excluded_by_default` |
| Read/unread filter | `read_filter_true`, `read_filter_false` |
| Title search (case-insensitive) | `search_filters_by_title` |
| Streaming until full page collected | `search_streams_until_full_page_collected` |
| Sparse matches fewer than limit | `search_exhausts_when_fewer_matches_than_limit` |
| Batch growth for distant matches | `search_grows_batch_to_reach_distant_matches` |

## Usage Example

```java
// Only runs when -Dintegration.tests=true is set
// Seed notifications and assert pagination behavior:
List<Notification> seeded = seedSequentialForRecipient(ALICE, RecipientType.USER, 5);

// Forward cursor — fetch items older than seeded[2]
List<NotificationWithStatus> page = page(
    ALICE,
    RecipientType.USER,
    null,          // readFilter (null = all non-deleted)
    null,          // search term
    seeded.get(2).getId(),  // cursor
    false,         // backward = false
    10             // limit
);

assertThat(page).extracting(it -> it.notification().getId())
    .containsExactly(seeded.get(1).getId(), seeded.get(0).getId());
```

> **Note:** Tests are guarded by `@EnabledIfSystemProperty(named = "integration.tests", matches = "true")` and require a live MongoDB instance configured via `BaseMongoIntegrationTest`. Each test resets both `Notification` and `NotificationReadState` collections in `@BeforeEach`.
