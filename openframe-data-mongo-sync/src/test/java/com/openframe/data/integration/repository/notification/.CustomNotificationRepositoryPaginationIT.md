Integration test suite for cursor-based pagination of the `NotificationRepository`, verifying filtering, isolation, search, and streaming behavior against a real MongoDB instance.

## Key Components

| Component | Role |
|-----------|------|
| `CustomNotificationRepositoryPaginationIT` | Main test class extending `BaseMongoIntegrationTest` |
| `NotificationRepository` | Repository under test, injected via `@Autowired` |
| `MongoTemplate` | Used in `@BeforeEach` to reset `Notification` and `NotificationReadState` collections |
| `NotificationFixtures` | Helper for building seed `Notification` documents |
| `NotificationWithStatus` | Result type pairing a `Notification` with its `ReadStatus` |
| `NotificationPage` | Cursor-based page wrapper returned by the repository |

## Covered Test Scenarios

| Scenario | Test Method |
|----------|-------------|
| No cursor → newest-first descending order | `given_no_cursor_returns_descending` |
| Forward cursor → older rows only | `given_forward_cursor_returns_older_rows` |
| Backward cursor → newer rows only | `given_backward_cursor_returns_newer_rows` |
| Audience isolation by recipient ID | `audience_isolation_by_id` |
| Audience isolation by recipient type (`USER` vs `MACHINE`) | `audience_isolation_by_type` |
| `limit` parameter honored | `limit_honored` |
| Empty result when no rows exist | `empty_when_no_rows` |
| `DELETED` status excluded by default | `deleted_excluded_by_default` |
| `readFilter=true` returns only READ rows | `read_filter_true` |
| `readFilter=false` returns only UNREAD rows | `read_filter_false` |
| Title search (case-insensitive) | `search_filters_by_title` |
| Streaming fills a full page when matches are sparse | `search_streams_until_full_page_collected` |
| Streaming exhausts gracefully when matches < limit | `search_exhausts_when_fewer_matches_than_limit` |
| Batch growth when matches are beyond initial 64-row window | `search_grows_batch_to_reach_distant_matches` |

## Usage Example

```java
// Tests run only when the system property is set:
// mvn verify -Dintegration.tests=true

// Typical pagination call exercised in these tests:
List<NotificationWithStatus> page = repository.findPage(
    recipientId,       // e.g. "user-alice"
    recipientType,     // RecipientType.USER or RecipientType.MACHINE
    readFilter,        // Boolean: true=READ, false=UNREAD, null=both
    search,            // String: case-insensitive title filter, or null
    cursor,            // String: ObjectId of last seen notification, or null
    backward,          // boolean: true = fetch newer rows (previous page)
    limit              // int: max rows to return
);
```

> **Note:** Tests are guarded by `@EnabledIfSystemProperty(named = "integration.tests", matches = "true")` and require a running MongoDB instance configured in `BaseMongoIntegrationTest`. Each test resets both the `Notification` and `NotificationReadState` collections in `@BeforeEach`.
