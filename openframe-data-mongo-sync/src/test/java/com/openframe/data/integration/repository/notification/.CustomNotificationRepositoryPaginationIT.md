Integration test suite for the `NotificationRepository`'s custom pagination logic, verifying cursor-based navigation, recipient isolation, read/delete filtering, and title search behavior against a real MongoDB instance.

## Key Components

| Component | Role |
|-----------|------|
| `CustomNotificationRepositoryPaginationIT` | Main test class extending `BaseMongoIntegrationTest` |
| `NotificationRepository` | Repository under test, injected via Spring context |
| `MongoTemplate` | Used in `@BeforeEach` to reset `Notification` and `NotificationReadState` collections |
| `NotificationFixtures` | Factory helpers for seeding test documents |
| `NotificationWithStatus` | DTO returned by pagination queries, carrying notification + read status |
| `NotificationPage` | Pagination result wrapper |

## Test Coverage Summary

| Test | What It Validates |
|------|-------------------|
| `given_no_cursor_returns_descending` | Default newest-first ordering |
| `given_forward_cursor_returns_older_rows` | Forward cursor excludes newer results |
| `given_backward_cursor_returns_newer_rows` | Backward cursor fetches newer results |
| `audience_isolation_by_id` | Different recipient IDs don't bleed data |
| `audience_isolation_by_type` | `USER` vs `MACHINE` type isolation |
| `limit_honored` | Page size cap respected |
| `deleted_excluded_by_default` | `DELETED` status filtered out by default |
| `read_filter_true/false` | `READ`/`UNREAD` filtering |
| `search_filters_by_title` | Case-insensitive title search |
| `search_streams_until_full_page_collected` | Streaming fills full page despite sparse matches |
| `search_grows_batch_to_reach_distant_matches` | Batch growth handles matches past initial 64-document window |

## Usage Example

```java
// Tests are guarded by a system property — run integration tests with:
// -Dintegration.tests=true

// Typical test flow pattern used internally:
List<Notification> seeded = seedSequentialForRecipient(ALICE, U, 5);
String cursor = seeded.get(2).getId();

// Forward page: returns items older than the cursor
List<NotificationWithStatus> page = page(ALICE, U, null, null, cursor, false, 10);

// Search page: streams until `limit` matching titles are found
List<NotificationWithStatus> results = page(ALICE, U, null, "alert", null, false, 3);
```

> **Note:** Tests are only activated when `-Dintegration.tests=true` is set, requiring a live MongoDB instance configured via `BaseMongoIntegrationTest`. Community support is available on [OpenMSP Slack](https://join.slack.com/t/openmsp/shared_invite/zt-36bl7mx0h-3~U2nFH6nqHqoTPXMaHEHA).
