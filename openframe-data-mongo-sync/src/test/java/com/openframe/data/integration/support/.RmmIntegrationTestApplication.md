Minimal Spring Boot test application context for RMM repository integration tests, scoping MongoDB repository scanning to the RMM package and importing required custom repository fragments.

## Key Components

| Element | Description |
|---|---|
| `@SpringBootConfiguration` | Marks this as a Spring Boot configuration class for the test context |
| `@EnableAutoConfiguration` | Activates Spring Boot's auto-configuration for MongoDB and related infrastructure |
| `@EnableMongoAuditing` | Enables `@CreatedDate` / `@LastModifiedDate` population on `Script` documents, matching production behavior |
| `@EnableMongoRepositories` | Restricts repository scanning to the RMM package via `ScriptRepository` as the base class anchor |
| `@Import(CustomScriptRepositoryImpl.class)` | Explicitly registers the custom repository fragment since `@EnableMongoRepositories` does not auto-detect it in a test context |

## Usage Example

```java
// Reference this configuration in your RMM integration test
@DataMongoTest
@Import(RmmIntegrationTestApplication.class)
class ScriptRepositoryIntegrationTest {

    @Autowired
    private ScriptRepository scriptRepository;

    @Test
    void shouldPersistScriptWithAuditDates() {
        Script script = new Script("cleanup-script", "rm -rf /tmp/*");
        Script saved = scriptRepository.save(script);

        assertNotNull(saved.getCreatedDate());
        assertNotNull(saved.getLastModifiedDate());
    }
}
```

## Notes

- Intentionally minimal — excludes notification, organization, and other unrelated repositories to keep the test context lightweight and fast
- `CustomScriptRepositoryImpl` must be explicitly imported here because in production it is resolved via the tenant application's component scan, not by `@EnableMongoRepositories` alone
- Mirrors the pattern of the broader `IntegrationTestApplication` but is scoped exclusively to RMM concerns
