Minimal Spring Boot test application context for RMM repository integration tests, scoping MongoDB repository scanning exclusively to the RMM package to avoid wiring unrelated repositories into the test context.

## Key Components

| Element | Purpose |
|---|---|
| `@SpringBootConfiguration` | Marks this class as a Spring Boot configuration source for the test context |
| `@EnableAutoConfiguration` | Activates Spring Boot's auto-configuration for MongoDB and related infrastructure |
| `@EnableMongoAuditing` | Ensures `@CreatedDate` / `@LastModifiedDate` fields on `Script` entities are populated, matching production behavior |
| `@EnableMongoRepositories` | Restricts repository scanning to the RMM package using `ScriptRepository` as the base class anchor |
| `@Import(CustomScriptRepositoryImpl.class)` | Explicitly registers the custom repository fragment, which is not auto-discovered by `@EnableMongoRepositories` in this minimal context |

## Usage Example

Reference this class as the application context for any RMM repository integration test:

```java
@DataMongoTest
@SpringBootTest(classes = RmmIntegrationTestApplication.class)
class ScriptRepositoryIntegrationTest {

    @Autowired
    private ScriptRepository scriptRepository;

    @Test
    void shouldPersistAndAuditScript() {
        Script script = new Script("example", "echo hello");
        Script saved = scriptRepository.save(script);

        assertNotNull(saved.getCreatedDate());
        assertNotNull(saved.getLastModifiedDate());
    }
}
```

> **Note:** This configuration intentionally excludes notification, organization, and other repository packages. If your test requires those, use the broader `IntegrationTestApplication` context instead.
