Integration test that verifies MongoDB index usage and query performance for the `NotificationReadState` collection, ensuring all major notification read-state operations (list, mark-read, unread counts, etc.) use covered index plans rather than collection scans.

## Key Components

| Component | Description |
|-----------|-------------|
| `seed()` / `@BeforeAll` | Drops and re-seeds `notifications` + `notification_read_states` collections with configurable volumes (default 30k notifications, 3k hot admin read states, 20k noise records) |
| `list_for_recipient_step_one_uses_indexed_plan` | Asserts `findPageForRecipient` uses `recipient_notification_unique` or `recipient_status` index, no collection scan, ≤ `limit × 4` keys examined |
| `has_unread_uses_recipient_status_index` | Verifies the existence check short-circuits on `recipient_status`, examining at most 2 keys |
| `mark_read_uses_unique_compound_index` | Confirms single-row `markRead` lookup hits `recipient_notification_unique` with exactly 1 key examined |
| `mark_all_as_read_uses_recipient_status_index` | Validates bulk update query plan against `recipient_status` within hot-recipient slice |
| `unread_counts_by_category_uses_recipient_category_status_index` | Ensures `$match` stage of aggregation hits `recipient_category_status` or `recipient_status` |
| `MongoExplain.Stats` | Helper wrapper around MongoDB `explain()` output; provides assertion methods for index names, execution time, and keys examined |
| `PerfResultRecorder` | Writes per-test timing/key stats to `target/perf-results/` as a Markdown report |

## Usage Example

```bash
# Run only performance tests (disabled by default)
mvn verify -Dperformance.tests=true

# Tune dataset sizes via system properties
mvn verify -Dperformance.tests=true \
  -Dperf.notifications.count=50000 \
  -Dperf.readStates.hotAdmin=5000 \
  -Dperf.readStates.noise=30000
```

```java
// Core assertion pattern used in every test
Stats stats = MongoExplain.explainFind(mongoTemplate, READS_COLL, query);

stats.assertNoCollectionScan()
     .assertUsesAnyOf(IDX_RECIPIENT_NOTIF, IDX_RECIPIENT_STATUS)
     .assertExecutionTimeBelow(FIND_BUDGET_MS);  // 200 ms
stats.assertExaminedAtMost(limit * 4L);
```

> **Note:** Tests are gated by `@EnabledIfSystemProperty(named = "performance.tests", matches = "true")` and tagged `@Tag("performance")` so they never run in standard CI pipelines — only in dedicated performance verification runs.
