Integration test that validates MongoDB index usage and query performance for the `NotificationReadState` collection, ensuring critical notification read-state operations use optimal index plans and stay within defined execution time budgets.

## Key Components

| Component | Description |
|-----------|-------------|
| `seed()` / `@BeforeAll` | Drops and re-seeds `notifications` and `notification_read_states` collections with configurable data volumes before the test suite runs |
| `ensureIndexes()` | Programmatically resolves and applies MongoDB entity indexes via `MongoPersistentEntityIndexResolver` |
| `MongoExplain` | Test support utility that wraps MongoDB's `explain()` for find, upsert, and aggregation operations, exposing an assertion-friendly `Stats` object |
| `PerfResultRecorder` | Captures execution metrics and writes a Markdown performance report to `target/perf-results/` |
| Index constants | Named references to the three key indexes: `recipient_notification_unique`, `recipient_status`, `recipient_category_status` |

## Test Coverage

| Test Method | Operation Verified | Expected Index |
|---|---|---|
| `list_for_recipient_step_one_uses_indexed_plan` | Paginated find for USER recipient | `recipient_notification_unique` or `recipient_status` |
| `list_for_recipient_machine_uses_indexed_plan` | Paginated find for MACHINE recipient | `recipient_notification_unique` or `recipient_status` |
| `has_unread_uses_recipient_status_index` | Existence check for UNREAD status | `recipient_status` |
| `mark_read_uses_unique_compound_index` | Single-document update by recipient + notificationId | `recipient_notification_unique` |
| `mark_all_as_read_uses_recipient_status_index` | Bulk update UNREAD → READ | `recipient_status` |
| `unread_counts_by_category_uses_recipient_category_status_index` | Aggregation group by category | `recipient_category_status` or `recipient_status` |

## Usage Example

```java
// Run performance tests explicitly via Maven system property
mvn verify -Dperformance.tests=true

// Override data volume for local runs
mvn verify -Dperformance.tests=true \
           -Dperf.notifications.count=5000 \
           -Dperf.readStates.hotAdmin=500

// All tests are tagged "performance" and skipped unless the system property is set
// Results are written to target/perf-results/<ClassName>.md
```

> **Note:** Tests are conditionally enabled via `@EnabledIfSystemProperty(named = "performance.tests", matches = "true")` and tagged `@Tag("performance")`, keeping them isolated from the standard CI test run.
