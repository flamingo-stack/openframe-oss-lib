Orchestrates the installation of integrated tool agents on machines by resolving command parameters and publishing installation messages via NATS.

## Key Components

| Element | Description |
|---|---|
| `process(machineId, toolAgent)` | Initiates a fresh tool installation for a given machine |
| `process(machineId, toolAgent, reinstall)` | Core installation handler; resolves install/run command args, retrieves tool data, and publishes via NATS |
| `getIntegratedToolData(toolId)` | Fetches `IntegratedTool` metadata by ID; returns an empty stub if no `toolId` is provided |
| `@ConditionalOnProperty` | Service is only active when `spring.cloud.stream.enabled` is set |

**Dependencies:**

- `IntegratedToolService` — retrieves tool configuration from the data layer
- `ToolCommandParamsResolver` — resolves dynamic parameters in installation and run command arguments
- `ToolInstallationNatsPublisher` — publishes the installation event to the NATS message bus

## Usage Example

```java
// Trigger a standard installation
toolInstallationService.process("machine-abc123", toolAgent);

// Trigger a reinstallation on an existing machine
toolInstallationService.process("machine-abc123", toolAgent, true);
```

## Notes

- Errors during publishing are caught and logged; no retry/fallback mechanism exists yet (marked as TODO).
- When `toolId` is empty, an empty `IntegratedTool` stub is returned — this is a temporary workaround pending a `hasToolServer` flag on agent configuration.
- Command args for both `installationCommandArgs` and `runCommandArgs` are resolved before publishing to ensure all dynamic tokens are substituted.
