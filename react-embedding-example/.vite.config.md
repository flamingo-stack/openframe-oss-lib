Vite configuration for the OpenFrame frontend dev server and preview server, wiring up React, proxy rules, module deduplication, and pre-bundling for the linked `@flamingo-stack/openframe-frontend-core` library.

## Key Components

- **`hubTarget` / `injectHeaders`** — proxy helpers that resolve the hub origin and inject chat secrets + identity headers server-side (secrets stay in Node config, never in the bundle)
- **`CONTENT_PREFIX`** proxy rule — forwards `/content/api/*` → `${HUB_ORIGIN}/api/*` with path rewriting
- **Bare `/api` fallbacks** — temporary dev-only routes (`/api/docs/search`, `/api/chat/agent`) for lib hooks that still hardcode bare paths
- **`resolve.dedupe`** — enforces a single copy of React, React DOM, React Router, and TanStack Query across the app and the linked lib (prevents "invalid hook call" and split context provider issues)
- **`next/font/google` stub** — aliases the Next.js font macro to a local stub so accidental imports don't break the Vite build
- **`optimizeDeps.include`** — pre-bundles all `openframe-frontend-core` entry points via esbuild, preventing mid-rebuild 404s and Babel 500 KB deoptimisation warnings during dev

## Usage Example

```bash
# Dev server — force re-bundle on every start (picks up lib edits)
npm run dev -- --force

# Preview server (uses the same proxy config)
npm run preview
```

```typescript
// proxy/inject.mjs contract expected by this config
export function hubTarget(env: Record<string, string>): string
export function injectHeaders(proxyReq: { setHeader: ... }, env: Record<string, string>): void
export function rewrite(path: string): string
```

> **Note:** All environment variables are loaded with an empty prefix (`''`) via `loadEnv`, so secrets like `HUB_ORIGIN` and the chat secret are available to the Node config layer only — they are never injected into the client bundle.
