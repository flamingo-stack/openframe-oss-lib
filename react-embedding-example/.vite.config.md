Vite configuration for the OpenFrame frontend dev server, setting up React plugin, proxy rules for hub API routing, dependency deduplication, and pre-bundling of the `@flamingo-stack/openframe-frontend-core` linked library.

## Key Components

- **Proxy rules** — Routes `/content/*`, `/api/docs/search`, and `/api/chat/agent` to the hub origin (resolved via `hubTarget`), injecting auth/identity headers server-side via `injectHeaders` on each `proxyReq` event
- **`resolve.dedupe`** — Enforces a single copy of React, React Router, React Query, and related singletons to prevent "invalid hook call" and split context issues when the linked lib ships its own node_modules
- **`resolve.alias`** — Stubs `next/font/google` to prevent accidental imports of a Next.js compiler macro from breaking the Vite build
- **`optimizeDeps.include`** — Pre-bundles all `openframe-frontend-core` entry points via esbuild at dev-server start, avoiding missing-file errors during `tsup` rebuilds and Babel deopt warnings on large chunks

## Usage Example

```bash
# Standard dev start — uses cached dep bundle
npm run dev

# Force re-bundle (required after lib changes; mirrors the VSCode F5 launch config)
npm run dev -- --force

# Preview production build with the same proxy rules
npm run preview
```

## Notes

- Secrets are loaded via `loadEnv(mode, cwd, '')` (all vars, no prefix) and only used inside the Node config closure — they are **never** included in the browser bundle
- The two bare `/api/*` fallback routes (`/api/docs/search`, `/api/chat/agent`) are intentionally temporary; they should be removed once the upstream lib exposes `searchEndpoint` / `agentBaseUrl` seams
