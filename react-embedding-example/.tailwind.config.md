Tailwind CSS configuration for the application, composing design tokens, fonts, and animations from the OpenFrame Frontend Core preset while extending content scanning to include both local source files and the core library's compiled output.

## Key Components

- **`openframeCorePreset`** — Imported preset from `@flamingo-stack/openframe-frontend-core` providing ODS tokens, custom fonts, and animations
- **`presets`** — Merges the core preset using Tailwind's native preset composition (not object-spread)
- **`content`** — Glob patterns covering local HTML/TS/TSX sources and pre-compiled library dist files to prevent class purging

## Usage Example

```typescript
// Extend the config locally without overriding core preset tokens
import type { Config } from 'tailwindcss'
import openframeCorePreset from '@flamingo-stack/openframe-frontend-core/tailwind.config.ts'

export default {
  presets: [openframeCorePreset as Config],
  content: [
    './index.html',
    './src/**/*.{ts,tsx}',
    './node_modules/@flamingo-stack/openframe-frontend-core/dist/**/*.{js,mjs,cjs}',
  ],
  // Add app-level extensions here — core tokens remain intact
  theme: {
    extend: {
      colors: {
        'app-accent': '#FF6B35',
      },
    },
  },
} satisfies Config
```

> **Note:** Always use `presets: [corePreset]` instead of spreading the core config object. Tailwind's preset system handles deep merging of `theme`, `plugins`, and `variants` — object-spread will silently drop inherited rules.
