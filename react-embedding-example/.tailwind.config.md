Tailwind CSS configuration for the application, composing the OpenFrame core design system preset (ODS tokens, fonts, animations) with local content sources.

## Key Components

- **`openframeCorePreset`** — Imported preset from `@flamingo-stack/openframe-frontend-core` providing shared ODS design tokens, typography, and animation utilities
- **`presets`** — Merges the core preset via Tailwind's official composition mechanism (not object-spread)
- **`content`** — Purge paths covering local HTML/TSX sources and pre-built dist files from the core library

## Usage Example

```typescript
// tailwind.config.ts
import type { Config } from 'tailwindcss'
import openframeCorePreset from '@flamingo-stack/openframe-frontend-core/tailwind.config.ts'

export default {
  presets: [openframeCorePreset as Config],
  content: [
    './index.html',
    './src/**/*.{ts,tsx}',
    // Required: include dist/ to prevent purging lib classes
    './node_modules/@flamingo-stack/openframe-frontend-core/dist/**/*.{js,mjs,cjs}',
  ],
} satisfies Config
```

## Notes

- Use `presets` array (not object-spread) — this is how Tailwind correctly merges preset configs including plugins and theme extensions
- The `node_modules` dist glob is **required**; omitting it causes Tailwind's purge step to strip classes used only inside the pre-compiled library
- Any app-level `theme.extend` additions go alongside `presets`, inheriting all ODS tokens automatically
