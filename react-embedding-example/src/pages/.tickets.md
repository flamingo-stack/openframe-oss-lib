Thin wrapper page component that renders the `HelpCenterList` from the OpenFrame frontend core library, providing the full help center UI including ticket list, creation form, and customer identification via `ChatRuntime`.

## Key Components

- **`TicketsPage`** — Page-level component that delegates entirely to `HelpCenterList`
- **`HelpCenterList`** *(imported)* — Core library component that composes the `DevSectionPage` chrome, ticket creation form, and ticket list; resolves customer identity via the proxy-injected `ChatRuntime`

## Usage Example

```typescript
// Route registration (e.g. in your router config)
import { TicketsPage } from './tickets'

const routes = [
  {
    path: '/tickets',
    element: <TicketsPage />,
  },
]
```

## Notes

- API calls target `/api/chat/agent/*` — covered by the dev-only proxy fallback defined in `vite.config.ts`
- Once the optional `agentBaseUrl` seam lands in the core lib, the proxy dependency can be replaced with a direct base URL prop
- No local state or props — all logic is encapsulated within `HelpCenterList`
