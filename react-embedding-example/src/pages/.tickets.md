Thin page-level wrapper that renders the `HelpCenterList` component from the OpenFrame frontend core library as the Tickets/Help Center route.

## Key Components

- **`TicketsPage`** — Stateless React page component; delegates all rendering and logic to `HelpCenterList`
- **`HelpCenterList`** *(from `@flamingo-stack/openframe-frontend-core`)* — Composes the full Help Center UI including:
  - Dev section page chrome
  - Ticket creation form
  - Ticket list
  - Customer identification via `ChatRuntime` (proxy-injected)

## Usage Example

```typescript
// Typically registered as a route in your router config
import { TicketsPage } from './tickets'

const routes = [
  {
    path: '/tickets',
    element: <TicketsPage />,
  },
]
```

## Notes

- API calls within `HelpCenterList` target `/api/chat/agent/*` and rely on the dev-only proxy fallback configured in `vite.config.ts`
- Once the optional `agentBaseUrl` seam is available in the core lib, the proxy dependency can be removed
- No props or local state — all behaviour is encapsulated inside the core library component
