Route component that renders a full product release detail page by composing injected section components (`RoadmapSection`, `VideoDisplaySection`, `DeliverySection`) with the core library's `ReleaseDetailPage`.

## Key Components

- **`ReleaseDetailRoute`** — Main exported route component; reads `slug` from URL params and wires all dependencies into `ReleaseDetailPage`
- **`useRelease`** — Host-supplied data hook that fetches a single release by slug via `EP.productReleaseBySlug`, integrated with the app's `QueryClient`
- **`RoadmapSection`** — Wraps `RoadmapGrid` with hub-scoped endpoints for voting and refresh
- **`VideoDisplaySection`** — Injects `EntityVideoSection` to render a single tabbed player (Full Video / Highlights) instead of the lib's default multiple-player fallback
- **`DeliverySection`** — Renders `DeliveryTable` for completed and in-progress delivery items from the `DeliveryResponse`; gates on data presence

## Usage Example

```typescript
// Registered as a React Router route:
import { ReleaseDetailRoute } from './routes/release-detail'

<Route path="/releases/:slug" element={<ReleaseDetailRoute />} />
```

```typescript
// ReleaseDetailRoute resolves to:
<ReleaseDetailPage
  slug={slug}
  useRelease={useRelease}
  RoadmapSection={RoadmapSection}
  DeliverySection={DeliverySection}
  VideoDisplaySection={VideoDisplaySection}
  roadmapApiEndpoint={EP.roadmap}
  deliveryApiEndpoint={EP.delivery}
  backButton={{ label: 'Back to releases', href: '/releases' }}
/>
```

> All section components are structural wrappers — they accept props typed by the core lib (`VideoDisplaySectionProps`, `DeliveryResponse`, `RoadmapItem`) and forward them verbatim, keeping the host app decoupled from lib internals.
