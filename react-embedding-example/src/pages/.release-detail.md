Route component that renders the full release detail view by wiring host-app data fetching and injected UI sections into the core library's `ReleaseDetailPage`.

## Key Components

| Export / Symbol | Purpose |
|---|---|
| `ReleaseDetailRoute` | Main page component; reads `:slug` from the URL and mounts `ReleaseDetailPage` with all required dependencies |
| `useRelease` | Host-supplied data hook that fetches a single release by slug via `EP.productReleaseBySlug` through the app's `QueryClient` |
| `RoadmapSection` | Injects `RoadmapGrid` so roadmap items vote and refresh against `/content` endpoints |
| `VideoDisplaySection` | Replaces the lib's default multi-player fallback with the single tabbed `EntityVideoSection` (Full Video / Highlights) |
| `DeliverySection` | Renders completed and in-progress `DeliveryTable` instances from the `DeliveryResponse` fetched by `ReleaseDetailPage` |

## Usage Example

```typescript
// In your router config (e.g. react-router-dom v6)
import { ReleaseDetailRoute } from './routes/release-detail'

<Route path="/releases/:slug" element={<ReleaseDetailRoute />} />
```

```typescript
// ReleaseDetailRoute internally composes to:
<ReleaseDetailPage
  slug={slug}
  useRelease={useRelease}
  RoadmapSection={RoadmapSection}
  DeliverySection={DeliverySection}
  VideoDisplaySection={VideoDisplaySection}
  roadmapApiEndpoint={EP.roadmap}
  deliveryApiEndpoint={EP.delivery}
  backButton={{ label: 'Back to releases', href: '/releases' }}
/>
```

> All four injected props (`useRelease`, `RoadmapSection`, `DeliverySection`, `VideoDisplaySection`) are required by `ReleaseDetailPage`. Omitting any causes the lib to either crash or silently skip that section — see inline comments for per-prop fallback behaviour.
