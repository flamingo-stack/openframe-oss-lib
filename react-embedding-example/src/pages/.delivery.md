Thin page-level component that wires the Delivery section's API endpoints into the shared `DevSectionPage` + `DeliveryLists` shell from `openframe-frontend-core`.

## Key Components

- **`DeliveryPage`** — Stateless page component; its sole responsibility is supplying the two API route constants (`EP.deliveryCompleted`, `EP.deliveryInProgress`) to `DeliveryLists`. All UI chrome (hero, search, task-type filter, URL-param wiring) and table rendering are delegated to the core library.

## Usage Example

```typescript
// Route registration (e.g. React Router / Next.js)
import { DeliveryPage } from './pages/delivery'

<Route path="/delivery" element={<DeliveryPage />} />
```

```typescript
// Endpoint config (config/endpoints.ts)
export const EP = {
  deliveryCompleted:   '/api/delivery/completed',
  deliveryInProgress:  '/api/delivery/in-progress',
}
```

## Notes

- No local state or hooks — fully controlled by URL search params via `DevSectionPage`.
- To change displayed data, update `EP.deliveryCompleted` / `EP.deliveryInProgress` in the endpoints config; no changes to this file are needed.
- Filter and search behaviour lives in `DeliveryLists` (core library); extend it there, not here.
