Thin config-only page component that wires the Delivery section's API endpoints into the shared `DevSectionPage` + `DeliveryLists` shell from `openframe-frontend-core`.

## Key Components

| Export | Type | Description |
|---|---|---|
| `DeliveryPage` | React component | Renders the full Delivery page by composing core UI components with local endpoint config |

## Usage Example

```typescript
// Route registration (e.g. in your router config)
import { DeliveryPage } from './pages/delivery'

const routes = [
  { path: '/delivery', element: <DeliveryPage /> },
]
```

```typescript
// EP config shape expected (config/endpoints.ts)
export const EP = {
  deliveryCompleted: '/api/delivery/completed',
  deliveryInProgress: '/api/delivery/in-progress',
  // ...
}
```

## Notes

- **No local state or logic** — all filtering (`search`, `task_type`), URL-param wiring, and table rendering is delegated to `openframe-frontend-core`.
- `DevSectionPage sectionKey="delivery"` provides the hero, search bar, and task-type filter.
- `DeliveryLists` consumes `completedApiEndpoint` and `inProgressApiEndpoint` to populate the completed and active delivery tables.
- To change data sources, update `EP.deliveryCompleted` / `EP.deliveryInProgress` in `config/endpoints.ts`.
