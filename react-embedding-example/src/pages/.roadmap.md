Thin page-level wrapper that wires the Roadmap feature to its API endpoints with zero UI logic of its own.

## Key Components

- **`RoadmapPage`** ‚Äî Exported page component; composes `DevSectionPage` and `RoadmapView` with endpoint config
- **`DevSectionPage`** (`sectionKey="roadmap"`) ‚Äî Provides chrome: hero, search input, and status filter pills, all URL-param-driven
- **`RoadmapView`** ‚Äî Handles data fetching, filtered grid rendering, and voting interactions by consuming URL params
- **`EP`** ‚Äî Centralized endpoint constants (`EP.roadmap`, `EP.roadmapById`, `EP.roadmapVote`)

## Usage Example

```typescript
// Route registration (e.g., React Router / Next.js)
import { RoadmapPage } from './pages/roadmap'

// Simply mount the page ‚Äî all filtering, fetching, and voting
// behavior is driven by URL params and endpoint config
<Route path="/roadmap" element={<RoadmapPage />} />
```

## Endpoint Wiring

| Prop | Endpoint | Purpose |
|------|----------|---------|
| `endpoint` | `EP.roadmap` | Fetch filtered roadmap item list |
| `buildRefreshUrl` | `EP.roadmapById` | Refresh a single item after interaction |
| `votingOptions.voteApiEndpoint` | `EP.roadmapVote` | Submit upvotes/downvotes |

> All UI state (search query, status filters, pagination) lives in URL params managed by `DevSectionPage`, keeping this file intentionally config-only ‚Äî add new behavior by extending `EP` or `RoadmapView` props, not here.
