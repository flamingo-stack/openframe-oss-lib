Thin page component that wires the roadmap feature to its API endpoints, delegating all UI, filtering, and state management to `DevSectionPage` and `RoadmapView` from the shared component library.

## Key Components

- **`RoadmapPage`** — Default page export; config-only, contains no local state or logic
- **`DevSectionPage`** (`sectionKey="roadmap"`) — Provides the canonical chrome: hero, search input, and status filter pills, all wired to URL params
- **`RoadmapView`** — Reads URL params, fetches filtered roadmap items, renders the card grid, and handles user voting

## Props / Endpoint Bindings

| Prop | Value | Purpose |
|---|---|---|
| `endpoint` | `EP.roadmap` | List fetch (filtered) |
| `buildRefreshUrl` | `EP.roadmapById` | Per-item refresh after vote |
| `votingOptions.voteApiEndpoint` | `EP.roadmapVote` | Submit vote |

## Usage Example

```typescript
// Registered in your router config
import { RoadmapPage } from './pages/roadmap'

const routes = [
  {
    path: '/roadmap',
    element: <RoadmapPage />,
  },
]
```

> **Note:** All runtime behavior (URL param parsing, data fetching, optimistic vote updates) lives inside `RoadmapView`. To change API routes, update `EP` in `config/endpoints` — no changes needed here.
