Thin page component that wires the onboarding guides catalog into the app shell by supplying API endpoint configuration — all rendering logic (RAG search, section pills, guide grid) is delegated to the library component.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `OnboardingCatalogPage` | React component | Page entry point that composes `DevSectionPage` and `OnboardingGuidesCatalogView` |

**Dependencies:**
- `DevSectionPage` — provides page chrome (hero + back button) via `sectionKey="onboarding"`
- `OnboardingGuidesCatalogView` — library component handling RAG search, section pills, guide grid, and internal data fetching
- `EP` — centralized endpoint config supplying `onboarding` and `onboardingSections` routes

## Usage Example

```typescript
// Route registration (e.g., React Router / Next.js equivalent)
import { OnboardingCatalogPage } from './pages/onboarding-catalog'

// In your router config:
{ path: '/onboarding-guides', element: <OnboardingCatalogPage /> }
```

```typescript
// The component itself requires no props — endpoints and base path are
// resolved from the centralized EP config
export function OnboardingCatalogPage() {
  return (
    <DevSectionPage sectionKey="onboarding">
      <OnboardingGuidesCatalogView
        guidesEndpoint={EP.onboarding}
        sectionsEndpoint={EP.onboardingSections}
        basePath="/onboarding-guides"
      />
    </DevSectionPage>
  )
}
```

> **Note:** URL-based filtering is handled internally by `OnboardingGuidesCatalogView` via the `?section=` query param — no additional state management is needed at the page level.
