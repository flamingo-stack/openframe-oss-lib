Renders the public contact page by mounting the self-contained `ContactForm` component from the OpenFrame frontend core library.

## Key Components

- **`ContactPage`** — Page-level wrapper component that renders `ContactForm` inside a padded container. No local state, hooks, or schema imports required.
- **`ContactForm`** *(from `@flamingo-stack/openframe-frontend-core`)* — Handles its own submission logic via `useContactSubmission`, which reads the `contactUrl` endpoint (`EP.contact`) from the nearest `EndpointsRuntime` provider in the tree.

## Usage Example

```typescript
// Registered as a route, e.g. /contact
import { ContactPage } from './contact'

export const routes = [
  { path: '/contact', element: <ContactPage /> },
]
```

```typescript
// ContactForm accepts optional display props
<ContactForm
  title="Contact us"
  subtitle="Questions about OpenFrame? Send us a note."
/>
```

## Notes

- The host app must have an `EndpointsRuntime` provider mounted higher in the tree so `useContactSubmission` can resolve `EP.contact`.
- No props are required on `ContactPage` itself — all configuration is handled by the shared core component and the runtime provider.
