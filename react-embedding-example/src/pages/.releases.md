Thin page component that composes the Releases list view by wiring the configured API endpoint into the shared `ProductReleasesView` and wrapping it with `DevSectionPage` chrome (hero, search, and release-status filter — all URL-param-driven).

## Key Components

- **`ReleasesPage`** — Default page export; config-only, supplies no layout or state logic of its own
- **`DevSectionPage`** (`sectionKey="releases"`) — Provides hero, search, and release-status filter UI; pulls the page title from the section registry
- **`ProductReleasesView`** — Reads URL params, fetches, paginates, and renders the release list
- **`EP.productReleases`** — The sole configuration this page contributes; maps to the API route for product releases

## Usage Example

```typescript
// Route registration (e.g. React Router / Next.js)
import { ReleasesPage } from './releases'

// In your router config:
{ path: '/releases', element: <ReleasesPage /> }
```

```typescript
// EP.productReleases (config/endpoints.ts) — the only value this page depends on
export const EP = {
  productReleases: '/api/products/releases',
  // ...
}
```

> **Design note:** All filter state (search query, release status) lives in URL params managed by `DevSectionPage`. To change the data source, update `EP.productReleases` — no changes to this file are needed.
