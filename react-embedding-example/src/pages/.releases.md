Thin config-only page component that wires the product releases list view to its API endpoint within the developer section chrome.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `ReleasesPage` | React Component | Page shell for the product releases list |

**Dependencies:**
- `DevSectionPage` — provides hero, search, and release-status filter UI, all URL-param-wired; pulls the page title from the section registry
- `ProductReleasesView` — handles param reading, data fetching, pagination, and rendering
- `EP.productReleases` — endpoint constant injected as the API route

## Usage Example

```typescript
// Registered as a route in the app router
import { ReleasesPage } from './releases'

// router config (e.g. React Router / TanStack Router)
{
  path: '/dev/releases',
  element: <ReleasesPage />,
}
```

## Notes

- **Zero local state** — all filter/search/pagination state lives in URL params managed by `DevSectionPage` and consumed by `ProductReleasesView`
- **No manual heading** — the `<h1>` title is sourced from the section registry via `sectionKey="releases"`, not hardcoded here
- Adding a new endpoint or swapping the data source requires only changing `EP.productReleases`; no layout or filter logic needs to be touched
