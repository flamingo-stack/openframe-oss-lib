Navigation hub component for the OpenFrame embedding demo app, rendering a grid of links to all available embedded surfaces with brief descriptions of each route and its backing API.

## Key Components

- **`SURFACES`** — Static array defining the 7 demo routes, each with a `to` path, display `title`, and short `desc` explaining the backing API or hook.
- **`HomePage`** — Default export component. Renders a responsive 1→2→3 column grid of `<Link>` cards, one per surface entry.

## Usage Example

```typescript
import { HomePage } from './home'

// Used as the index route in your router config
const router = createBrowserRouter([
  { path: '/', element: <HomePage /> },
  { path: '/onboarding-guides', element: <OnboardingGuidesPage /> },
  // ...remaining SURFACES routes
])
```

## Surface Routes

| Path | Description |
|------|-------------|
| `/onboarding-guides` | Catalog + detail, props-driven from `/content/api/onboarding-guides` |
| `/roadmap` | Voting grid; votes + refresh hit `/content/api/roadmap` |
| `/delivery` | Bug-fix + enhancement tables via `/content/api/delivery/*` |
| `/releases` | Product releases list + roadmap-injected detail |
| `/legal/privacy` | Privacy / terms via `/content/api/legal/*` |
| `/contact` | `ContactForm` → `/content/api/contact` (EndpointsRuntime) |
| `/tickets` | `HelpCenterList` via tickets hooks |

> Each card links to a real `@flamingo-stack/openframe-frontend-core` component proxied through `/content`, which injects the chat secret and a fixed identity for the floating **Ask AI** widget.
