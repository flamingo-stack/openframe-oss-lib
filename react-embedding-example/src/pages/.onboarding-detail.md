Thin page component that renders an onboarding guide detail view by extracting the `slug` route parameter and wiring it to the shared `OnboardingGuideDetailView` from the core library.

## Key Components

- **`OnboardingDetailPage`** — Route-level page component; reads `slug` from URL params and delegates all data fetching and rendering to `OnboardingGuideDetailView`
- **`OnboardingGuideDetailView`** — Core library component (from `@flamingo-stack/openframe-frontend-core`) that handles guide fetching and display internally
- **`EP.onboardingBySlug`** — API endpoint constant injected as `guideEndpoint` prop

## Props Passed to `OnboardingGuideDetailView`

| Prop | Value | Description |
|------|-------|-------------|
| `slug` | `useParams().slug` | Guide identifier extracted from the URL |
| `guideEndpoint` | `EP.onboardingBySlug` | API route used to fetch the guide |
| `basePath` | `"/onboarding-guides"` | Base navigation path for back-links |

## Usage Example

```typescript
// Registered in your router (e.g. react-router-dom)
{
  path: '/onboarding-guides/:slug',
  element: <OnboardingDetailPage />,
}
```

```typescript
// The component itself — no additional configuration needed
export function OnboardingDetailPage() {
  const { slug = '' } = useParams()
  return (
    <OnboardingGuideDetailView
      slug={slug}
      guideEndpoint={EP.onboardingBySlug}
      basePath="/onboarding-guides"
    />
  )
}
```

> This page intentionally contains no business logic. All data fetching, loading states, and error handling are owned by `OnboardingGuideDetailView`, keeping this component a pure routing adapter.
