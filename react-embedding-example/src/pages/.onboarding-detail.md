Thin route-level page component that wires the current URL slug and API endpoint configuration into the shared `OnboardingGuideDetailView` library component.

## Key Components

- **`OnboardingDetailPage`** — Default page export; reads the `slug` param from the URL via `useParams` and delegates all data-fetching and rendering to `OnboardingGuideDetailView`.

## Usage Example

```typescript
// Typical React Router v6 route registration
import { OnboardingDetailPage } from './pages/onboarding-detail'

<Route path="/onboarding-guides/:slug" element={<OnboardingDetailPage />} />
```

```typescript
// Internal wiring (what this component does)
const { slug = '' } = useParams()

<OnboardingGuideDetailView
  slug={slug}
  guideEndpoint={EP.onboardingBySlug}  // e.g. "/api/onboarding/:slug"
  basePath="/onboarding-guides"
/>
```

## Notes

- **No local state or data-fetching** — all logic lives inside `OnboardingGuideDetailView` from `@flamingo-stack/openframe-frontend-core`.
- `slug` defaults to `''` if the param is absent, matching the lib component's expected type.
- `EP.onboardingBySlug` is the only app-specific configuration injected, keeping the page decoupled from the API layer.
