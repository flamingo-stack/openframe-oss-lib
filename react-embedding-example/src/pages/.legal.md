Renders legal document pages (Privacy Policy and Terms of Service) by resolving the `docType` route parameter and supplying the appropriate copy strings to the shared `LegalDocumentPage` component.

## Key Components

- **`COPY`** — Static record mapping `docType` keys (`privacy`, `terms`) to their respective UI strings: title, fallback description, contact email, error messages, and empty-state text.
- **`LegalPage`** — Route-aware component that reads `:docType` from the URL, resolves the correct copy block (defaulting to `privacy`), and delegates rendering to `LegalDocumentPage`.

## Usage Example

```typescript
// Route definition (e.g. in your router config)
<Route path="/legal/:docType" element={<LegalPage />} />

// Navigating to a specific doc
// /legal/privacy  → renders Privacy Policy
// /legal/terms    → renders Terms of Service
// /legal/unknown  → falls back to Privacy Policy copy
```

```typescript
// EP.legal() resolves the API endpoint for document content
EP.legal('privacy') // → e.g. "/api/v1/legal/privacy"
EP.legal('terms')   // → e.g. "/api/v1/legal/terms"
```

> **Note:** `backButton` is explicitly set to `false`, meaning no back navigation is rendered — intended for standalone legal pages accessed directly via URL rather than from within a flow.
