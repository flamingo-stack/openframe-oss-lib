Renders the legal document page for OpenFrame, supporting both Privacy Policy and Terms of Service routes via a single parameterized component.

## Key Components

- **`COPY`** — Static record mapping `docType` slugs (`"privacy"` | `"terms"`) to per-document UI strings (title, contact email, error/empty state messages)
- **`LegalPage`** — Route-level component that reads the `docType` URL param, resolves the matching copy, and delegates rendering to `LegalDocumentPage` from `openframe-frontend-core`

## Usage Example

```typescript
// Route setup (e.g. react-router-dom)
<Route path="/legal/:docType" element={<LegalPage />} />

// Navigating to a specific doc
// /legal/privacy  → renders Privacy Policy
// /legal/terms    → renders Terms of Service
// /legal/unknown  → falls back to Privacy Policy copy
```

## Notes

- Falls back to `COPY.privacy` for any unrecognised `docType` param
- The API endpoint is resolved via `EP.legal(docType)` from the shared endpoint config
- `backButton` is disabled (`false`) — this is a standalone destination page, not a modal or nested view
- Replaces the hub's previous separate `/privacy` and `/terms` pages with a single parameterized surface
