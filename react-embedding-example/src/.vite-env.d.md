Vite environment type declarations that extend the default `ImportMetaEnv` interface with application-specific environment variables.

## Key Components

- **`ImportMetaEnv`** — Extends Vite's built-in env interface with a custom optional variable:
  - `VITE_HUB_ORIGIN` — Public hub origin URL used for new-tab content links in embed mode
- **`ImportMeta`** — Re-declares `env` as the typed `ImportMetaEnv` for full `import.meta.env` type safety

## Usage Example

```typescript
// Access the typed environment variable anywhere in the app
const hubOrigin = import.meta.env.VITE_HUB_ORIGIN

// Example: constructing a link in embed mode
const contentUrl = hubOrigin
  ? `${hubOrigin}/content/123`
  : '/content/123'
```

```bash
# .env or .env.local
VITE_HUB_ORIGIN=https://hub.flamingo.run
```

> **Note:** The `VITE_` prefix is required by Vite to expose variables to client-side code. Variables without this prefix remain server-side only and will be `undefined` at runtime.
