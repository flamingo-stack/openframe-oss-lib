Vite environment type declarations that extend the default `ImportMeta` interface with application-specific environment variables.

## Key Components

- **`ImportMetaEnv`** — Augments Vite's built-in env interface with a custom variable:
  - `VITE_HUB_ORIGIN` *(optional string)* — Public hub origin URL used for new-tab content links in embed mode
- **`ImportMeta`** — Re-declares `env` as the extended `ImportMetaEnv` type to enable full IntelliSense on `import.meta.env`

## Usage Example

```typescript
// Access the hub origin at runtime with full type safety
const hubOrigin = import.meta.env.VITE_HUB_ORIGIN ?? 'https://default.hub.example.com'

const contentUrl = `${hubOrigin}/path/to/resource`
window.open(contentUrl, '_blank')
```

```bash
# .env or .env.local
VITE_HUB_ORIGIN=https://hub.flamingo.run
```

> **Note:** Variables prefixed with `VITE_` are exposed to client-side code by Vite's build pipeline. Any variable without this prefix remains server-side only and will be `undefined` in the browser bundle.
