Vite alias stub that replaces `next/font/google` with harmless no-ops, preventing build crashes when a bundler encounters Next.js compiler macros like `Azeret_Mono()` or `DM_Sans()` that cannot be evaluated outside the Next.js compiler.

## Key Components

- **`FontResult`** — Type describing the shape returned by Next.js font functions (`className`, `variable`, `style.fontFamily`)
- **`noop`** — Factory returning an empty `FontResult`; used as a safe stand-in for any font constructor call
- **`Azeret_Mono`** / **`DM_Sans`** — Named exports matching the real `next/font/google` API, each resolving to `noop`
- **`default` (Proxy)** — Catch-all default export that returns `noop` for any property access, covering any other Google font name not explicitly stubbed

## Usage Example

This module is **not imported directly** — it is wired in via a Vite alias in `vite.config.ts`:

```typescript
// vite.config.ts
export default defineConfig({
  resolve: {
    alias: {
      'next/font/google': path.resolve(__dirname, './stubs/next-font-google.ts'),
    },
  },
})
```

Any transitive import of `next/font/google` then resolves to the stub:

```typescript
// Resolved transparently to the stub — no build error
import { DM_Sans } from 'next/font/google'

const font = DM_Sans({ subsets: ['latin'] })
// → { className: '', variable: '', style: { fontFamily: '' } }
```

> **Note:** Fonts in this project are delivered via the `./styles` CSS import, so this stub acts purely as a defensive guard — it is not expected to be exercised at runtime in normal operation.
