Application entry point that bootstraps the React app by registering the embed router bridge, applying global styles, and mounting the component tree (`AppProviders` + `AppRoutes`) inside a `BrowserRouter`.

## Key Components

- **`registerEmbedRouterBridge()`** — Called before first render to wire react-router navigation into the embed shims layer, replacing the fallback `popstate`-only handler
- **`AppProviders`** — Wraps the app with all global context providers
- **`AppRoutes`** — Defines the top-level route structure
- **`@flamingo-stack/openframe-frontend-core/styles`** — Imports ODS design tokens, chat keyframes, and fonts; must load before any lib surface renders

## Usage Example

```typescript
// This file is the Vite entry point — referenced in index.html:
// <script type="module" src="/src/main.tsx"></script>

// The render tree looks like:
// <BrowserRouter>
//   <AppProviders>   ← auth, query client, theme, etc.
//     <AppRoutes />  ← <Routes> with all page-level components
//   </AppProviders>
// </BrowserRouter>
```

## Notes

- **No `<React.StrictMode>`** — intentional. The reference app's network panel is kept production-faithful; StrictMode's dev-only double-invoke adds cancelled fetch noise without adding coverage value (the lib is StrictMode-safe via its own Vitest suite)
- Global styles are imported once here to guarantee paint order before any lib surface mounts
