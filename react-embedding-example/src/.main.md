Application entry point that bootstraps the React app by mounting the component tree into the DOM, registering the embed router bridge, and importing global styles.

## Key Components

- **`registerEmbedRouterBridge()`** — Called before first render to wire `react-router` navigation into the embed shims layer, replacing the fallback `popstate`-only mechanism
- **`AppProviders`** — Wraps the app with all required context providers
- **`AppRoutes`** — Defines the top-level route configuration
- **`BrowserRouter`** — Provides client-side routing context
- **`@flamingo-stack/openframe-frontend-core/styles`** — Imports ODS design tokens, chat keyframes, and fonts exactly once before any lib surface renders

## Usage Example

```typescript
// This file is the Vite/bundler entrypoint — referenced in index.html:
// <script type="module" src="/src/main.tsx"></script>

// Execution order:
// 1. Global styles imported (ODS tokens, fonts, keyframes)
// 2. registerEmbedRouterBridge() called synchronously
// 3. ReactDOM.createRoot mounts the tree:
//    <BrowserRouter>
//      <AppProviders>   ← all context/providers
//        <AppRoutes />  ← route definitions
//      </AppProviders>
//    </BrowserRouter>
```

## Notes

- **No `<React.StrictMode>`** by design — this reference app mirrors production network behavior exactly. StrictMode's dev-only double-invocation produces duplicate cancelled fetch requests in the network panel, which is misleading for demos. The underlying library remains StrictMode-safe (covered by its Vitest suite).
