Defines the application's client-side route tree using React Router v6, mapping URL paths to page components within a shared `AppShell` layout.

## Key Components

- **`AppRoutes`** — Root routing component that wraps all routes inside `AppShell` and exports a single registry of all application surfaces
- **`AppShell`** — Persistent layout wrapper (nav, shell chrome) rendered as the parent `Route` element
- **`PageError`** — Catch-all `*` route rendering a "Page not found" error for unmatched paths

## Route Map

| Path | Component |
|------|-----------|
| `/` | `HomePage` |
| `/onboarding-guides` | `OnboardingCatalogPage` |
| `/onboarding-guides/:slug` | `OnboardingDetailPage` |
| `/roadmap` | `RoadmapPage` |
| `/delivery` | `DeliveryPage` |
| `/releases` | `ReleasesPage` |
| `/releases/:slug` | `ReleaseDetailRoute` |
| `/legal/:docType` | `LegalPage` |
| `/contact` | `ContactPage` |
| `/tickets` | `TicketsPage` |
| `*` | `PageError` |

## Usage Example

```typescript
// Mount at the application entry point
import { BrowserRouter } from 'react-router-dom'
import { AppRoutes } from './app-routes'

export function App() {
  return (
    <BrowserRouter>
      <AppRoutes />
    </BrowserRouter>
  )
}
```

> **Adding a new surface:** declare a single `<Route>` inside `AppRoutes` — the shared shell and error boundary are inherited automatically.
