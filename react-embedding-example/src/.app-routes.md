Defines the top-level React Router v6 route tree for the application, mapping URL paths to page components within a shared `AppShell` layout.

## Key Components

- **`AppRoutes`** — Root route configuration component; all routes are nested under `AppShell` so the shell (nav, layout) persists across navigation
- **`AppShell`** — Wrapping layout component rendered as the parent `<Route element>`
- **`PageError`** — Catch-all `*` route renders a "Page not found" error for unmatched paths

## Route Map

| Path | Component |
|------|-----------|
| `/` | `HomePage` |
| `/onboarding-guides` | `OnboardingCatalogPage` |
| `/onboarding-guides/:slug` | `OnboardingDetailPage` |
| `/roadmap` | `RoadmapPage` |
| `/delivery` | `DeliveryPage` |
| `/releases` | `ReleasesPage` |
| `/releases/:slug` | `ReleaseDetailRoute` |
| `/legal/:docType` | `LegalPage` |
| `/contact` | `ContactPage` |
| `/tickets` | `TicketsPage` |
| `*` | `PageError` (404) |

## Usage Example

```typescript
// Mount at the application entry point
import { BrowserRouter } from 'react-router-dom'
import { AppRoutes } from './app-routes'

export function App() {
  return (
    <BrowserRouter>
      <AppRoutes />
    </BrowserRouter>
  )
}
```

> **Adding a new surface:** Declare a single `<Route>` inside `AppRoutes` — the shared shell and 404 fallback are inherited automatically.
