Thin wrapper component that renders an embedded AI chat interface for the embedder side, delegating all configuration, identity resolution, and routing logic to the shared `EmbeddableChat` library component.

## Key Components

**`AskAi`** *(exported function component)*
Renders `EmbeddableChat` with `guide` mode active and an empty guide config, which signals the SSE adapter to fall back to the lib's built-in `defaultTableIdForDocumentType` lookup — no hand-written source map required.

## Design Notes

| Concern | Resolution |
|---|---|
| Identity | Resolved server-side via proxy `act-as` headers |
| Source wiring | Resolved server-side; no static per-embed override |
| Chat-history namespace | Falls back to lib defaults |
| Doc-card routing | Config-driven via `content-runtime.ts` `docPlatformTargets` |
| Mixed-source routing | Each doc type routes to its own platform (knowledge hub vs. company hub) |
| Entity content | Routes through `composeContentUrl` |

## Usage Example

```typescript
// Drop into any embedder page — no props required
import { AskAi } from './ask-ai'

export default function SupportPage() {
  return (
    <main>
      <AskAi />
    </main>
  )
}
```

> This is the embedder-side equivalent of the hub's `global-ask-ai-client.tsx`. Both wrap the same `EmbeddableChat` component — the embedder version carries no auth gate, as identity is handled transparently by the proxy layer.
