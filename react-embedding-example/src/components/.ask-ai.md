Thin wrapper component that renders an AI chat interface for the embedder context, delegating all configuration to the `EmbeddableChat` library component with guide mode active by default.

## Key Components

- **`AskAi`** — Default-exported functional component that mounts `EmbeddableChat` in guide mode with an empty config, allowing the SSE adapter to fall back to `defaultTableIdForDocumentType` automatically.

## Design Notes

- **No auth gate** — identity (act-as headers) is resolved server-side via the proxy; no client-side auth wiring needed.
- **No source wiring** — source resolution, chat-history namespace, and link decisions all fall back to lib defaults defined in `content-runtime.ts`.
- **Doc-card routing** — driven by `content-runtime.ts`'s `docPlatformTargets` per `documentType`; each doc chip resolves its URL per row to `getBaseUrl(platform)/<basePath>/<path>` and opens in a new tab. Chats mixing multiple sources route each card to its respective platform independently.
- **Entity content** — releases, podcasts, and similar content route through `composeContentUrl`.

## Usage Example

```typescript
// Drop into any embedder page — no props required
import { AskAi } from './ask-ai'

export default function EmbedPage() {
  return (
    <main>
      <AskAi />
    </main>
  )
}
```
