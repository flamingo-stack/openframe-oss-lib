Provides the top-level layout shell for the OpenFrame embed demo app, composing a sticky navigation header, announcement bar, route outlet, and floating AI assistant into a single persistent wrapper rendered on every route.

## Key Components

- **`AppShell`** — Root layout component rendered by the router; wraps all child routes via `<Outlet />`
- **`NAV`** — Static navigation config array defining all top-level routes with their labels and `end` matching flags
- **`AnnouncementBar`** — Imported from `@flamingo-stack/openframe-frontend-core`; reads its endpoint autonomously from `EndpointsRuntime.announcementsUrl` — no props required
- **`AskAi`** — Floating AI assistant trigger rendered persistently across all routes

## Routing Notes

The shell deliberately avoids any manual router bridging. React Router is registered once into the library's embed shims via `providers/embed-router-bridge`, allowing all embedded surfaces (chat chips, markdown anchors, entity cards, onboarding cards, release lists) to soft-navigate through the registered router directly — no `runtime.navigation` callback or document-click interceptor needed.

## Usage Example

```typescript
// Typically registered as the root layout route
import { createBrowserRouter } from 'react-router-dom'
import { AppShell } from './app-shell'

const router = createBrowserRouter([
  {
    path: '/',
    element: <AppShell />,
    children: [
      { index: true, element: <Home /> },
      { path: 'roadmap', element: <Roadmap /> },
      { path: 'tickets', element: <Tickets /> },
    ],
  },
])
```
