The root layout shell for the OpenFrame embed demo app, rendering the persistent navigation bar, announcement banner, page outlet, and floating AI assistant across all routes.

## Key Components

- **`AppShell`** — Top-level layout component wrapping all routes via React Router's `<Outlet />`
- **`NAV`** — Static navigation config array defining route links for the demo app
- **`AnnouncementBar`** — Reads its endpoint from `EndpointsRuntime.announcementsUrl` automatically; requires no props
- **`AskAi`** — Floating AI assistant trigger rendered on every route
- **Router bridging** — Navigation is wired once via `providers/embed-router-bridge`; no manual `runtime.navigation` callbacks needed

## Usage Example

```typescript
// main.tsx — mount AppShell as the root layout route
import { createBrowserRouter, RouterProvider } from 'react-router-dom'
import { AppShell } from './app-shell'

const router = createBrowserRouter([
  {
    path: '/',
    element: <AppShell />,
    children: [
      { index: true, element: <Home /> },
      { path: 'roadmap', element: <Roadmap /> },
      { path: 'tickets', element: <Tickets /> },
      // ...other child routes
    ],
  },
])

export default function App() {
  return <RouterProvider router={router} />
}
```

## Notes

- All ODS design tokens (`bg-ods-bg`, `text-ods-text-primary`, etc.) are applied at the shell level, ensuring consistent theming across every child route
- The sticky header uses `backdrop-blur` with a semi-transparent `bg-ods-card/95` for legibility over scrolled content
- Adding a new top-level route requires only a new entry in the `NAV` array plus a corresponding child route definition
