Shared UI component for displaying route-level error states using ODS design tokens (no hardcoded colors).

## Key Components

- **`PageError`** — A presentational component that renders a styled error heading and optional detail message using ODS semantic color tokens (`ods-text-primary`, `ods-text-secondary`).

## Usage Example

```typescript
import { PageError } from './page-state'

// Minimal usage
<PageError title="Not Found" />

// With detail message
<PageError
  title="Failed to load data"
  detail="The requested resource could not be retrieved. Please try again."
/>
```

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `title` | `string` | ✅ | Primary error heading |
| `detail` | `string` | ❌ | Optional secondary descriptive message |
