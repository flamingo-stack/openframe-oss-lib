Wraps the entire embed application in the required runtime and data-fetching providers, establishing a single shared context tree for chat identity, API endpoints, and server-state caching.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `AppProviders` | React component | Root provider tree; accepts `children: ReactNode` |
| `queryClient` | `QueryClient` | Module-level singleton; shared across all hooks using `@tanstack/react-query` |

**Provider nesting order (outermost → innermost):**

```text
QueryClientProvider         ← react-query cache (tickets, onboarding, chat hooks)
  └─ ChatRuntimeContext     ← memoized chat runtime (buildChatRuntime)
       └─ ChatIdentityProvider  ← single /identity fetch for the whole embed
            └─ EndpointsRuntimeContext  ← memoized endpoints runtime
                 └─ {children}
```

> `ChatIdentityProvider` must sit **inside** `ChatRuntimeContext` (reads `identityUrl` from it) and **above** `EndpointsRuntimeContext` so that the persistent chat shell, `/tickets` page, and contact form all share one identity resolution instead of triggering duplicate fetches.

## Usage Example

```typescript
// main.tsx / entry point
import { AppProviders } from './app-providers'
import { AppShell } from './app-shell'

export function App() {
  return (
    <AppProviders>
      <AppShell />
    </AppProviders>
  )
}
```

## Notes

- Both runtimes (`chatRuntime`, `endpointsRuntime`) are wrapped in `useMemo` with empty deps — stable references required by the lib's embedder contract to prevent context re-renders.
- Theme is applied via `data-theme="dark"` on `<html>` (in `index.html`) rather than a provider; ODS design tokens resolve directly from that attribute.
