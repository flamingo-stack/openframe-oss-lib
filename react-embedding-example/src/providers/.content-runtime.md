Constructs and exports two memoized runtime configuration objects — `buildChatRuntime` and `buildEndpointsRuntime` — that wire all endpoint URLs, navigation mode, and content URL resolution for the OpenFrame embedder's chat and announcements subsystems.

## Key Components

**`buildChatRuntime()`**
Returns an `Omit<ChatRuntime, 'source'>` object containing:
- `endpoints` — Maps all chat-related URLs (stream, approval, commands, attachments, identity, image proxy) using the `EP` config and `CONTENT_PREFIX` proxy base
- `navigation` — Sets mode to `'host'`, delegating routing to the app's react-router via the lib's embed-router-bridge
- `resolvePlaceholderUrl` — OG placeholder generator per title
- `composeContentUrl` — Built via `makeComposeContentUrl` with `HOSTED_TYPES` for in-app soft navigation vs. canonical hub links
- `docPlatformTargets` — Per-document-type viewer routing (`markdown` → flamingo knowledge hub, `data_room_doc` → company-hub)

**`buildEndpointsRuntime()`**
Returns an `EndpointsRuntime` object with announcements, access code (validate/consume), and contact URLs.

**`HOSTED_TYPES`**
A `Set` declaring which content types (`onboarding_guide`, `product_release`) have local `/:slug` detail routes, enabling in-app soft navigation instead of redirecting to the hub origin.

## Usage Example

```typescript
// app-providers.tsx — memoized once at the provider level
import { buildChatRuntime, buildEndpointsRuntime } from './runtime/content-runtime'
import { useMemo } from 'react'

function AppProviders({ children }) {
  const chatRuntime = useMemo(() => buildChatRuntime(), [])
  const endpointsRuntime = useMemo(() => buildEndpointsRuntime(), [])

  return (
    <ChatRuntimeProvider runtime={chatRuntime}>
      <EndpointsRuntimeProvider runtime={endpointsRuntime}>
        {children}
      </EndpointsRuntimeProvider>
    </ChatRuntimeProvider>
  )
}
```

> **Note:** `source` is intentionally omitted from `ChatRuntime` — it is resolved server-side from the proxied hub's `currentPlatform()`, making this factory platform-agnostic across any hub origin.
