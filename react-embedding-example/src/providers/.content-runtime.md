Single factory module that constructs both `ChatRuntime` and `EndpointsRuntime` configuration objects for the OpenFrame embedder, wiring all API endpoints, navigation behavior, and content URL resolution from a single source.

## Key Components

**`HOSTED_TYPES`**
A `Set` of content types (`onboarding_guide`, `product_release`) whose detail routes are owned by this embedder (e.g. `/onboarding-guides/:slug`). Used by `makeComposeContentUrl` to emit relative in-app hrefs for soft navigation vs. canonical hub URLs for everything else.

**`buildChatRuntime()`**
Returns an `Omit<ChatRuntime, 'source'>` object containing:
- `endpoints` — all proxied API URLs via `EP.*` and `CONTENT_PREFIX` (chat stream, approval, commands, uploads, identity, image proxy)
- `navigation.mode: 'host'` — delegates routing to the embedded react-router instance via the lib's embed-router-bridge
- `composeContentUrl` — resolves content item URLs using `makeComposeContentUrl`, with explicit overrides for `roadmap_item` and `delivery_item` that deep-link into filtered list views (`?search=<id>`) rather than non-existent detail pages
- `docPlatformTargets` — routes `markdown` docs to Flamingo's knowledge hub and `data_room_doc` to the company hub

**`buildEndpointsRuntime()`**
Returns an `EndpointsRuntime` with announcement, access code validation/consumption, and contact endpoints.

## Usage Example

```typescript
// app-providers.tsx (memoized at app root)
import { buildChatRuntime, buildEndpointsRuntime } from './content-runtime'

const chatRuntime = useMemo(() => buildChatRuntime(), [])
const endpointsRuntime = useMemo(() => buildEndpointsRuntime(), [])

// Pass into lib context providers
<ChatRuntimeProvider value={chatRuntime}>
  <EndpointsRuntimeProvider value={endpointsRuntime}>
    {children}
  </EndpointsRuntimeProvider>
</ChatRuntimeProvider>
```

> **Note:** `source` is intentionally omitted from `ChatRuntime` — it is resolved server-side from the proxied hub's `currentPlatform()`, making this factory platform-agnostic.
