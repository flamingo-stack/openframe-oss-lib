Adapts React Router DOM's navigation hooks and `<Link>` component into the shape expected by `@flamingo-stack/openframe-frontend-core`'s embed shims, ensuring all lib surfaces navigate correctly within the SPA.

## Key Components

- **`LinkAdapter`** — A `forwardRef` component that maps Next.js-style `<Link>` props (`href`, `prefetch`, `replace`, `scroll`) to React Router's `<Link to="...">`, dropping Next-only props.
- **`registerEmbedRouterBridge()`** — One-time registration function (idempotent via `registered` guard) that calls `registerNavigation` and `registerLink` with React Router-adapted implementations.
- **`registerNavigation` adapters** — Wraps `useNavigate` into a `RouterStub` (`push`, `replace`, `back`, `forward`, `refresh`, `prefetch`), extracts a bare `string` from `useLocation().pathname`, and unwraps React Router's `[params, setter]` tuple from `useSearchParams()` to return a bare `URLSearchParams`.

## Usage Example

```typescript
// main.tsx — call ONCE before any lib surface renders
import { registerEmbedRouterBridge } from './embed-router-bridge'

registerEmbedRouterBridge()

createRoot(document.getElementById('root')!).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)
```

## Notes

- Must be called **once** before the first lib surface renders; subsequent calls are no-ops.
- The `useRouter()` return value is memoized on `nav` to prevent unnecessary re-renders in consumers that depend on stable references (e.g. `useDocSearch`).
- The file is `.tsx` (not `.ts`) solely because `LinkAdapter` contains JSX.
