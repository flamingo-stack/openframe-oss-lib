Adapts React Router's navigation hooks and `<Link>` component into the shape expected by `@flamingo-stack/openframe-frontend-core`'s embed-shims, enabling all lib surfaces to navigate and re-render correctly within this SPA.

## Key Components

- **`LinkAdapter`** — A `forwardRef` wrapper that maps the Next.js `<Link>` prop shape (`href`, `prefetch`, `replace`, `scroll`) to React Router's `<Link to=...>`, dropping Next-only props.
- **`registerEmbedRouterBridge()`** — One-time registration function (guarded by a `registered` flag) that calls `registerNavigation` and `registerLink` from the embed-shims with adapted React Router hooks.
- **`registerNavigation` adapters** — Bridges the hook shape mismatch:
  - `useRouter()` → wraps `useNavigate()` into a `RouterStub` (`push`, `replace`, `back`, `forward`, `refresh`, `prefetch`), memoized to prevent unnecessary re-renders.
  - `usePathname()` → returns `useLocation().pathname` as a bare string.
  - `useSearchParams()` → unwraps React Router's `[params, setter]` tuple to return only the bare `URLSearchParams`.

## Usage Example

Call **once** before any lib surface renders, typically in `main.tsx`:

```typescript
import { registerEmbedRouterBridge } from './embed-router-bridge'

registerEmbedRouterBridge()

ReactDOM.createRoot(document.getElementById('root')!).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)
```

> **Note:** `registerEmbedRouterBridge()` is idempotent — subsequent calls after the first are a no-op. Never register the raw React Router hooks directly into the shims; the hook return shapes are incompatible with the `NavigationImpl` contract.
