Centralized API endpoint registry for the Flamingo content hub. All route strings are derived once from the `CONTENT` base URL — no path is duplicated across the runtime factory, data layer, or pages.

## Key Components

**`EP` (const object)**
The single source of truth for all hub API paths, organized by domain:

| Group | Endpoints |
|-------|-----------|
| Chat | `chatStream`, `commands`, `docsSearch`, `agentBase`, `approval`, `attachmentUpload`, `attachmentViewPrefix`, `identity`, `imageProxy` |
| Roadmap | `roadmap`, `roadmapVote`, `roadmapById(id)` |
| Delivery | `delivery`, `deliveryCompleted`, `deliveryInProgress` |
| Onboarding | `onboarding`, `onboardingBySlug(slug)`, `onboardingSections` |
| Releases | `productReleases`, `productReleaseBySlug(slug)` |
| Misc | `legal(docType)`, `contact`, `announcements`, `accessValidate`, `accessConsume`, `ogPlaceholder(title)` |

Static paths are plain strings; dynamic paths are typed factory functions returning a string.

**`HUB_PUBLIC_ORIGIN`**
The public hub origin URL, read from `VITE_HUB_ORIGIN` at build time, falling back to `http://localhost:3000`. Used for embed-mode "open in full" links.

## Usage Example

```typescript
import { EP, HUB_PUBLIC_ORIGIN } from './endpoints'

// Static endpoint
const res = await fetch(EP.chatStream, { method: 'POST', body: payload })

// Dynamic endpoint — slug/id factories
const release = await fetch(EP.productReleaseBySlug('v2.1.0'))
const guide   = await fetch(EP.onboardingBySlug('getting-started'))
const legal   = await fetch(EP.legal('privacy-policy'))

// Embed-mode origin
const fullUrl = `${HUB_PUBLIC_ORIGIN}/releases/v2.1.0`
```
