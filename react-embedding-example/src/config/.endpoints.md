Centralizes all API endpoint paths for the hub application, deriving every URL from a single `CONTENT` base constant to eliminate duplication across the runtime factory, data layer, and pages.

## Key Components

**`EP` (const object)**
A frozen map of all hub API endpoints. Grouped by domain:

| Group | Keys |
|---|---|
| Chat | `chatStream`, `commands`, `docsSearch`, `agentBase`, `approval`, `attachmentUpload`, `attachmentViewPrefix`, `identity`, `imageProxy` |
| Roadmap | `roadmap`, `roadmapVote`, `roadmapById(id)` |
| Delivery | `delivery`, `deliveryCompleted`, `deliveryInProgress` |
| Onboarding | `onboarding`, `onboardingBySlug(slug)`, `onboardingSections` |
| Releases | `productReleases`, `productReleaseBySlug(slug)` |
| Misc | `legal(docType)`, `contact`, `announcements`, `accessValidate`, `accessConsume`, `ogPlaceholder(title)` |

**`HUB_PUBLIC_ORIGIN`**
Public hub origin URL, read from `VITE_HUB_ORIGIN` env var, defaulting to `http://localhost:3000`. Used for new-tab "open in full" links in embed mode.

**`AGENT_BASE` (internal)**
Private constant anchoring the `/chat/agent` sub-tree so `approval` and any future agent paths derive from one place.

## Usage Example

```typescript
import { EP, HUB_PUBLIC_ORIGIN } from './endpoints'

// Static path
const res = await fetch(EP.chatStream)

// Parameterized path
const release = await fetch(EP.productReleaseBySlug('v2.4.0'))

// Dynamic title placeholder
const placeholderUrl = EP.ogPlaceholder('Getting Started with OpenFrame')

// Embed mode — open full content in new tab
window.open(`${HUB_PUBLIC_ORIGIN}/releases`, '_blank')
```
