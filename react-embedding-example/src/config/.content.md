Centralized base URL constant for all content hub API endpoint calls, derived from the shared proxy prefix to ensure namespace consistency.

## Key Components

- **`CONTENT`** — Exported string constant combining `CONTENT_PREFIX` with `/api`, used as the base path for every hub endpoint request.
- **`CONTENT_PREFIX`** — Imported from `proxy/content-prefix.mjs`, the single source of truth for the proxy namespace prefix.

## Usage Example

```typescript
import { CONTENT } from './content'

const response = await fetch(`${CONTENT}/articles`)
const data = await response.json()
```

> Changing the proxy namespace requires only a single edit in `proxy/content-prefix.mjs` — all hub calls update automatically.
