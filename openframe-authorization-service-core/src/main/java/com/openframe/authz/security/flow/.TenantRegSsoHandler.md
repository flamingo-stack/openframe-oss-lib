Handles SSO-based tenant registration flow by processing authentication callbacks and creating new tenants from OIDC user information stored in secure cookies.

## Key Components

- **`handle()`** - Main flow method that processes SSO authentication, extracts user data from OIDC claims, and registers a new tenant
- **`cookieName()`** - Returns the SSO registration cookie identifier
- **`requireOidcUser()`** - Validates and extracts OIDC user from authentication context
- **`requireEmail()`** - Extracts and validates email from OIDC user claims
- **`clearCookie()`** - Removes SSO registration cookie after processing
- **`clearSession()`** - Invalidates security context and HTTP session
- **`buildRedirect()`** - Constructs redirect URL with tenant ID parameter

## Usage Example

```java
// Used automatically by Spring Security during SSO flow
@Autowired
private TenantRegSsoHandler tenantRegHandler;

// The handler processes requests when SSO registration cookie is present
// Cookie contains: tenantName, tenantDomain, accessCode, redirectTo
// OIDC user provides: email, given_name, family_name

// After successful tenant registration:
// - Cookie is cleared
// - Session invalidated  
// - User redirected to target URL with tenantId parameter
// Example redirect: "/dashboard?tenantId=abc-123-def"
```

The handler integrates with Spring Security's OAuth2/OIDC flow to automatically complete tenant registration when users authenticate via SSO providers.
