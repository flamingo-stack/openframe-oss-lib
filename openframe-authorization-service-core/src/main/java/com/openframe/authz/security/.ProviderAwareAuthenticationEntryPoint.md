A custom authentication entry point that intelligently routes unauthenticated requests to the appropriate OAuth2 provider (Google/Microsoft) or the default login page based on request parameters. It preserves tenant context in the session for multi-tenant SSO scenarios.

## Key Components

- **`commence()`** - Main entry point method that handles unauthenticated requests
- **Provider Detection** - Extracts OAuth2 provider from request parameters
- **Tenant Context Preservation** - Stores tenant ID in session for cross-request continuity
- **Dynamic Routing** - Redirects to provider-specific OAuth2 endpoints or default login

## Usage Example

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public AuthenticationEntryPoint authenticationEntryPoint() {
        return new ProviderAwareAuthenticationEntryPoint();
    }
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .exceptionHandling(config -> 
                config.authenticationEntryPoint(authenticationEntryPoint()))
            .build();
    }
}

// Usage in requests:
// GET /protected-resource?provider=google → redirects to /oauth2/authorization/google
// GET /protected-resource?provider=microsoft → redirects to /oauth2/authorization/microsoft
// GET /protected-resource → redirects to /login
```

The entry point automatically detects the desired OAuth2 provider from request parameters and preserves tenant context for multi-tenant applications, ensuring proper SSO flow continuation after authentication.
