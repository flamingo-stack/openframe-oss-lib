Custom Spring Security authentication entry point that routes users to appropriate OAuth2 login endpoints based on the provider parameter and preserves tenant context.

## Key Components

- **commence()** - Main entry point method that handles unauthenticated requests
- **Provider routing** - Supports Google and Microsoft OAuth2 providers via URL parameter
- **Tenant preservation** - Stores tenant ID in HTTP session for multi-tenant applications
- **Fallback handling** - Defaults to standard login page for unknown providers

## Usage Example

```java
// Configure in Spring Security
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .exceptionHandling(exceptions -> exceptions
            .authenticationEntryPoint(new ProviderAwareAuthenticationEntryPoint())
        );
    return http.build();
}

// Usage via URL parameters
// Redirect to Google OAuth2: /secured-endpoint?provider=google
// Redirect to Microsoft OAuth2: /secured-endpoint?provider=microsoft
// Redirect to login page: /secured-endpoint (no provider parameter)
```

The entry point automatically redirects to `/oauth2/authorization/{provider}` for supported OAuth2 providers or `/login` as fallback, while maintaining tenant context in the session for dynamic client registration resolution.
