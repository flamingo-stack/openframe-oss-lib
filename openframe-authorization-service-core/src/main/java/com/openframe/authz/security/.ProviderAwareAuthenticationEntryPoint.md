A Spring Security authentication entry point that redirects unauthenticated users to appropriate login endpoints based on OAuth2 provider selection, with tenant-aware session management.

## Key Components

- **commence()**: Main entry point method that handles authentication failures by redirecting to provider-specific OAuth2 endpoints or default login
- **Provider Detection**: Extracts and validates OAuth2 provider from request parameters (Google, Microsoft)
- **Tenant Preservation**: Stores current tenant ID in HTTP session for multi-tenant OAuth2 flows
- **Dynamic Routing**: Routes users to provider-specific OAuth2 authorization endpoints or fallback login page

## Usage Example

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .exceptionHandling(exceptions -> exceptions
                .authenticationEntryPoint(new ProviderAwareAuthenticationEntryPoint())
            )
            .oauth2Login(oauth2 -> oauth2
                .authorizationEndpoint(authorization -> 
                    authorization.baseUri("/oauth2/authorization"))
            );
        return http.build();
    }
}

// Usage in controller or filter
// GET /secure-endpoint?provider=google
// -> Redirects to /oauth2/authorization/google

// GET /secure-endpoint?provider=microsoft  
// -> Redirects to /oauth2/authorization/microsoft

// GET /secure-endpoint (no provider)
// -> Redirects to /login
```

The entry point automatically preserves tenant context in the session and supports case-insensitive provider matching for Google and Microsoft OAuth2 flows.
