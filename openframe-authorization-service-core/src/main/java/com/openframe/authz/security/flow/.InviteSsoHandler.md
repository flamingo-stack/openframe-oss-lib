Handles SSO authentication flow for users accepting invitations by processing invitation cookies and registering new users through the OpenFrame authorization system.

## Key Components

- **cookieName()** - Returns the SSO invitation cookie name constant
- **handle()** - Main flow method that processes invitation cookies, extracts user data from OIDC authentication, and registers the invited user
- **SsoCookieCodec** - Dependency for encoding/decoding SSO cookies containing invitation data
- **InvitationRegistrationService** - Service for registering users through invitation flow
- **SsoInviteCookiePayload** - Contains invitation ID, tenant switching flag, and redirect URL

## Usage Example

```java
@Autowired
private InviteSsoHandler inviteSsoHandler;

// The handler is automatically invoked during SSO flow
// when an invitation cookie is present
public void processSsoAuthentication(HttpServletRequest request, 
                                   HttpServletResponse response,
                                   Authentication auth) {
    if (inviteSsoHandler.cookieName().equals(cookieFound)) {
        inviteSsoHandler.handle(request, response, auth);
        // User will be registered and redirected to target tenant
    }
}

// Example of what the handler does internally:
InvitationRegistrationRequest registrationRequest = 
    InvitationRegistrationRequest.builder()
        .invitationId(UUID.fromString("invitation-uuid"))
        .firstName("John")
        .lastName("Doe")
        .password(UUID.randomUUID().toString())
        .switchTenant(true)
        .build();
```

The handler automatically extracts user information from OIDC authentication, generates a random password, and completes the invitation registration process while preserving the OAuth session.
