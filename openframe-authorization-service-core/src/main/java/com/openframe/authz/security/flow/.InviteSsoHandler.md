Handles the SSO flow for invited users, processing an OIDC-authenticated user's invitation cookie and registering them via the `InvitationRegistrationService`.

## Key Components

| Element | Description |
|---|---|
| `cookieName()` | Returns the SSO invite cookie identifier (`COOKIE_SSO_INVITE`) |
| `handle()` | Core SSO flow method — validates the invite cookie, extracts OIDC user claims, builds and submits a registration request, then clears the flow cookie and redirects |
| `SsoCookieCodec` | Decodes the encrypted/signed invite cookie into a `SsoInviteCookiePayload` |
| `InvitationRegistrationService` | Performs the actual user registration against the target tenant |
| `SsoInviteCookiePayload` | Carries `invitationId`, `switchTenant`, and `redirectTo` from the invite flow |

## Usage Example

```java
// Called automatically by the SSO flow dispatcher when COOKIE_SSO_INVITE is present
// Typical flow: user clicks invite link → authenticates via OIDC → handler fires

InviteSsoHandler handler = new InviteSsoHandler(ssoCookieCodec, invitationRegistrationService);

// Resolves cookie, decodes payload, registers user, then redirects
handler.handle(httpRequest, httpResponse, authentication);
// → user registered under the invited tenant
// → SSO flow cookie cleared
// → browser redirected to payload.redirectTo() within targetTenantId
```

## Notes

- A random UUID is used as the initial password since SSO users authenticate via OIDC, not credentials.
- Throws `IllegalStateException` if the invite cookie payload cannot be decoded (invalid or expired session).
- `requireEmail()` validates email presence on the OIDC principal even though the field is not directly used in the registration payload.
