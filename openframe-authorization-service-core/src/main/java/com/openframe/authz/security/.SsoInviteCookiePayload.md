A record class that represents the payload structure for SSO invitation cookies, containing authentication and routing information for user invitations.

## Key Components

- **Record Fields:**
  - `s` - Session or signature identifier
  - `invitationId` - Unique identifier for the invitation
  - `switchTenant` - Boolean flag indicating if tenant switching is required
  - `provider` - Authentication provider name
  - `redirectTo` - URL to redirect user after processing
  - `iat` - Issued at timestamp (Unix epoch)
  - `exp` - Expiration timestamp (Unix epoch)

## Usage Example

```java
// Creating an SSO invite cookie payload
SsoInviteCookiePayload payload = new SsoInviteCookiePayload(
    "session123",
    "inv-456789",
    true,
    "google",
    "/dashboard",
    System.currentTimeMillis() / 1000,
    System.currentTimeMillis() / 1000 + 3600 // 1 hour expiry
);

// Accessing payload data
String inviteId = payload.invitationId();
boolean needsTenantSwitch = payload.switchTenant();
String redirectUrl = payload.redirectTo();

// Check if token is expired
boolean isExpired = System.currentTimeMillis() / 1000 > payload.exp();
```

This immutable record provides a type-safe way to handle SSO invitation cookie data, ensuring all necessary authentication and routing parameters are properly structured and accessible.
