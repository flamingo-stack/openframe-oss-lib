Interface for handling SSO authentication flows with cookie-based state management and OIDC user processing. Provides utilities for cookie resolution, user validation, and post-authentication redirects.

## Key Components

- **`cookieName()`** - Returns the name of the SSO flow cookie
- **`resolveCookie()`** - Finds the SSO cookie in the request, returns null if not found
- **`requireCookie()`** - Gets the SSO cookie or throws exception if missing
- **`isActivated()`** - Checks if the SSO flow is active based on cookie presence
- **`handle()`** - Main method to process the SSO flow after authentication
- **`requireOidcUser()`** - Extracts and validates the OIDC user from authentication
- **`requireEmail()`** - Gets the user's email or throws if unavailable
- **`resolveNames()`** - Extracts first and last names from OIDC user claims
- **`clearFlowCookieAndRedirect()`** - Cleans up cookies and redirects to continue URL

## Usage Example

```java
public class InviteFlowHandler implements SsoFlowHandler {
    @Override
    public String cookieName() {
        return "invite_flow";
    }

    @Override
    public void handle(HttpServletRequest request, 
                      HttpServletResponse response, 
                      Authentication authentication) throws Exception {
        Cookie flowCookie = requireCookie(request);
        OidcUser user = requireOidcUser(authentication);
        String email = requireEmail(user);
        String[] names = resolveNames(user);
        
        // Process invite logic here
        processInvite(email, names[0], names[1]);
        
        // Clean up and redirect
        clearFlowCookieAndRedirect(response, flowCookie, "tenant-id", null);
    }
}
```
