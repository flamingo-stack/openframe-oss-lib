Defines the contract for handling SSO (Single Sign-On) flow operations, including cookie management and authentication processing. This interface provides default implementations for cookie resolution and activation status checking.

## Key Components

- **`cookieName()`** - Abstract method that returns the SSO cookie name
- **`resolveCookie(HttpServletRequest)`** - Default method to find and return the SSO cookie from the request
- **`isActivated(HttpServletRequest)`** - Default method to check if SSO flow is active based on cookie presence
- **`handle(HttpServletRequest, HttpServletResponse, Authentication)`** - Abstract method for processing SSO flow logic

## Usage Example

```java
public class CustomSsoFlowHandler implements SsoFlowHandler {
    @Override
    public String cookieName() {
        return "SSO_TOKEN";
    }
    
    @Override
    public void handle(HttpServletRequest request, HttpServletResponse response, 
                      Authentication authentication) throws Exception {
        // Check if SSO is activated
        if (isActivated(request)) {
            Cookie ssoCookie = resolveCookie(request);
            // Process SSO logic with the cookie
            processAuthentication(ssoCookie, authentication);
        }
    }
    
    private void processAuthentication(Cookie cookie, Authentication auth) {
        // Custom SSO authentication logic
    }
}
```
