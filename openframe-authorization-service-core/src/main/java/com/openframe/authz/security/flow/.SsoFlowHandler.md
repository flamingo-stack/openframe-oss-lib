Handles Single Sign-On (SSO) authentication flows by managing flow cookies, extracting user information from OIDC tokens, and orchestrating redirects during OAuth authentication processes.

## Key Components

- **`cookieName()`** - Abstract method to define the SSO flow cookie identifier
- **`resolveCookie()`/`requireCookie()`** - Cookie retrieval and validation methods
- **`isActivated()`** - Checks if SSO flow is active based on cookie presence
- **`handle()`** - Abstract method for processing the complete SSO flow
- **`requireOidcUser()`** - Extracts and validates OIDC user from authentication
- **`requireEmail()`** - Retrieves and normalizes user email from OIDC claims
- **`resolveNames()`** - Parses given and family names from OIDC user data
- **`clearFlowCookieAndRedirect()`** - Cleans up flow state and redirects to continuation URL

## Usage Example

```java
public class LoginSsoFlowHandler implements SsoFlowHandler {
    @Override
    public String cookieName() {
        return "openframe_login_flow";
    }

    @Override
    public void handle(HttpServletRequest request, HttpServletResponse response, 
                      Authentication authentication) throws Exception {
        Cookie flowCookie = requireCookie(request);
        OidcUser user = requireOidcUser(authentication);
        String email = requireEmail(user);
        String[] names = resolveNames(user);
        
        // Process user authentication
        authenticateUser(email, names[0], names[1]);
        
        // Clear flow and redirect
        clearFlowCookieAndRedirect(response, flowCookie, "tenant-123", "/dashboard");
    }
}
```
