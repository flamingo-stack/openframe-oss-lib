MongoDB-based implementation of Spring Security OAuth2's RegisteredClientRepository interface that provides persistence for OAuth2 client registrations using MongoDB as the data store.

## Key Components

- **MongoRegisteredClientRepository**: Main repository class implementing `RegisteredClientRepository`
- **save()**: Converts Spring Security RegisteredClient to MongoDB document and persists it
- **findById()**: Retrieves client registration by ID
- **findByClientId()**: Retrieves client registration by client ID  
- **toRegistered()**: Private method converting MongoDB document back to RegisteredClient

## Usage Example

```java
@Autowired
private MongoRegisteredClientRepository clientRepository;

// Save a new OAuth2 client
RegisteredClient client = RegisteredClient.withId(UUID.randomUUID().toString())
    .clientId("my-client")
    .clientSecret("{noop}secret")
    .clientAuthenticationMethod(ClientAuthenticationMethod.CLIENT_SECRET_BASIC)
    .authorizationGrantType(AuthorizationGrantType.AUTHORIZATION_CODE)
    .redirectUri("http://localhost:8080/callback")
    .scope("read")
    .build();

clientRepository.save(client);

// Retrieve client by ID or client ID
RegisteredClient found = clientRepository.findByClientId("my-client");
RegisteredClient foundById = clientRepository.findById(client.getId());
```

The repository handles mapping between Spring Security's RegisteredClient objects and MongoDB documents, including conversion of authentication methods, grant types, and token/client settings.
