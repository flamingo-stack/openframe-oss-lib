Data Transfer Object (DTO) for initiating SSO-based tenant registration in the OpenFrame authorization system. This class captures the essential information needed to start the tenant onboarding process through Single Sign-On.

## Key Components

- **Email Validation**: Uses `@ValidEmail` and `@NotBlank` to ensure a valid email address is provided
- **Organization Details**: Captures `tenantName` with regex pattern validation and `tenantDomain` with custom validation
- **SSO Configuration**: Includes `provider` field for specifying the SSO identity provider
- **Access Control**: Optional `accessCode` for restricted registration flows
- **Redirect Handling**: Optional `redirectTo` field for post-registration navigation

## Usage Example

```java
// Create a new SSO tenant registration request
SsoTenantRegistrationInitRequest request = new SsoTenantRegistrationInitRequest();
request.setEmail("admin@acmecorp.com");
request.setTenantName("ACME Corporation");
request.setTenantDomain("acme");
request.setProvider("google");
request.setAccessCode("INVITE123");
request.setRedirectTo("https://acme.openframe.com/dashboard");

// Using constructor
SsoTenantRegistrationInitRequest request = new SsoTenantRegistrationInitRequest(
    "admin@acmecorp.com",
    "INVITE123", 
    "ACME Corporation",
    "acme",
    "google",
    "https://acme.openframe.com/dashboard"
);
```

The tenant name validation allows letters, numbers, and common punctuation marks (2-100 characters), making it suitable for diverse organization names while preventing malicious input.
