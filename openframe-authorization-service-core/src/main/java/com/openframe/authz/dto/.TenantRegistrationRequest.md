A DTO class for handling multi-tenant user registration requests, extending CoreUserRequest with tenant-specific fields and validation.

## Key Components

- **TenantRegistrationRequest**: Main DTO class extending `CoreUserRequest`
- **email**: User email with `@ValidEmail` validation
- **tenantName**: Organization name with pattern validation (2-100 characters, supports international characters)
- **tenantDomain**: Tenant domain with custom `@TenantDomain` validation
- **accessCode**: Optional access code for registration

## Usage Example

```java
// Create a new tenant registration request
TenantRegistrationRequest request = TenantRegistrationRequest.builder()
    .email("admin@example.com")
    .tenantName("Example Corporation")
    .tenantDomain("example")
    .accessCode("INVITE123")
    .build();

// Validate and process registration
@PostMapping("/register")
public ResponseEntity<?> registerTenant(@Valid @RequestBody TenantRegistrationRequest request) {
    // Registration logic here
    return ResponseEntity.ok().build();
}

// Access validated fields
String organizationName = request.getTenantName();
String domain = request.getTenantDomain();
```

The class uses Lombok annotations for boilerplate code generation and Jakarta validation for input sanitization, ensuring secure tenant creation with proper organization naming conventions.
