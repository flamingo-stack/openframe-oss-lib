A DTO (Data Transfer Object) class that handles multi-tenant user registration requests, extending CoreUserRequest with tenant-specific fields and validation constraints.

## Key Components

- **TenantRegistrationRequest**: Main DTO class extending `CoreUserRequest` for tenant-aware registration
- **Validation Annotations**: Custom validators including `@ValidEmail`, `@TenantDomain`, and standard Jakarta validation
- **Lombok Annotations**: Uses `@Data`, `@SuperBuilder`, `@EqualsAndHashCode` for automatic code generation
- **Key Fields**:
  - `email`: Validated email address using custom `@ValidEmail`
  - `tenantName`: Organization name with regex pattern validation (2-100 characters)
  - `tenantDomain`: Custom validated tenant domain identifier
  - `accessCode`: Optional access code for registration

## Usage Example

```java
// Create a tenant registration request
TenantRegistrationRequest request = TenantRegistrationRequest.builder()
    .email("user@example.com")
    .tenantName("Acme Corporation")
    .tenantDomain("acme")
    .accessCode("INVITE123")
    .build();

// The request inherits additional fields from CoreUserRequest
// Validation occurs automatically when processed by the controller
```

The class supports both programmatic creation and JSON deserialization for REST API endpoints, with comprehensive validation ensuring data integrity for multi-tenant user registration workflows.
