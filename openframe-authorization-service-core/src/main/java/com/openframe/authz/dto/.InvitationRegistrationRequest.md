A DTO for handling user registration requests initiated through an invitation system, extending the base user registration functionality with invitation-specific fields.

## Key Components

- **InvitationRegistrationRequest** - Main DTO class extending `CoreUserRequest`
- **invitationId** - Required field linking the registration to a specific invitation
- **switchTenant** - Optional field indicating whether to switch tenants during registration

## Usage Example

```java
// Create invitation-based registration request
InvitationRegistrationRequest request = InvitationRegistrationRequest.builder()
    .invitationId("inv-123456789")
    .switchTenant(true)
    // Additional CoreUserRequest fields inherited
    .build();

// Validate the request
if (request.getInvitationId() != null && !request.getInvitationId().isBlank()) {
    // Process invitation registration
    userService.registerViaInvitation(request);
}

// Check tenant switching preference
if (Boolean.TRUE.equals(request.getSwitchTenant())) {
    tenantService.switchUserTenant(request.getInvitationId());
}
```

The class uses Jakarta validation to ensure the invitation ID is provided, and Lombok annotations for automatic generation of constructors, getters, setters, and builder pattern support. The `SuperBuilder` annotation enables builder inheritance from the parent `CoreUserRequest` class.
