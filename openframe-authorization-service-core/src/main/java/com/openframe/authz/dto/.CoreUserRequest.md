Base DTO for core user creation fields used across registration and SSO flows in the OpenFrame authorization module.

## Key Components

| Field | Validation | Notes |
|-------|-----------|-------|
| `firstName` | `@NotBlank`, unicode-aware name regex (max 50 chars) | Supports accented chars, hyphens, apostrophes |
| `lastName` | `@NotBlank`, same unicode pattern as first name | Same constraints as `firstName` |
| `password` | `@NotBlank`, min 8 chars, requires upper/lowercase, digit, and special char | Strong password enforcement |
| `pictureUrl` | None | Optional; populated internally from SSO/OIDC `picture` claim — not user-supplied |

The class uses `@SuperBuilder` to allow subclasses to extend it with their own builder chains, and is annotated with Lombok `@Data`, `@NoArgsConstructor`, and `@AllArgsConstructor`.

## Usage Example

```java
// Direct instantiation
CoreUserRequest request = CoreUserRequest.builder()
    .firstName("Jane")
    .lastName("Doe")
    .password("Secure@123")
    .build();

// Typical subclass extension pattern
@Data
@SuperBuilder
@EqualsAndHashCode(callSuper = true)
public class CreateUserRequest extends CoreUserRequest {
    @NotBlank
    @Email
    private String email;
}

// SSO flow — pictureUrl set internally, not from client input
CoreUserRequest ssoRequest = CoreUserRequest.builder()
    .firstName("Jane")
    .lastName("Doe")
    .password(generatedPassword)
    .pictureUrl(oidcClaims.get("picture")) // populated by SSO handler
    .build();
```

> **Note:** `pictureUrl` must never be accepted from user-facing API payloads. It is reserved for internal SSO/OIDC flow handlers to prevent unauthorized profile picture injection.
