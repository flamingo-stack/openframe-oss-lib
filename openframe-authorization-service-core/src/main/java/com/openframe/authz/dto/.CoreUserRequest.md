A data transfer object (DTO) that defines the core request structure for user creation or updates with comprehensive validation rules for user credentials.

## Key Components

- **firstName**: Required field with Unicode letter validation, supporting international names with spaces, hyphens, and apostrophes (max 50 chars)
- **lastName**: Required field with identical validation pattern as firstName
- **password**: Required field with minimum 8 characters, enforcing strong password policy with uppercase, lowercase, digit, and special character requirements

## Usage Example

```java
// Creating a valid user request
CoreUserRequest userRequest = CoreUserRequest.builder()
    .firstName("María")
    .lastName("García-López")
    .password("SecurePass123!")
    .build();

// Manual construction
CoreUserRequest request = new CoreUserRequest();
request.setFirstName("John");
request.setLastName("O'Connor");
request.setPassword("MyPassword1@");

// The validation annotations will automatically validate:
// - Names must start with a letter and contain only letters, marks, spaces, hyphens, apostrophes
// - Password must be 8+ chars with mixed case, numbers, and special characters
```

The class leverages Lombok annotations for automatic getter/setter generation and builder pattern support, while Jakarta validation ensures data integrity before processing.
