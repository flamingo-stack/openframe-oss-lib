A Data Transfer Object (DTO) for core user registration and update requests with comprehensive validation rules for user identity and password security.

## Key Components

- **firstName** - User's first name with Unicode letter support and length validation
- **lastName** - User's last name with matching validation rules as firstName
- **password** - Secure password field with complexity requirements including mixed case, digits, and special characters
- **Validation Annotations** - Jakarta Bean Validation constraints for input sanitization and security
- **Lombok Annotations** - SuperBuilder pattern support for inheritance-friendly object construction

## Usage Example

```java
// Creating a user request with builder pattern
CoreUserRequest userRequest = CoreUserRequest.builder()
    .firstName("John")
    .lastName("Doe")
    .password("SecureP@ss123")
    .build();

// Validation will automatically check:
// - Names contain only Unicode letters, marks, apostrophes, spaces, hyphens
// - Names are 1-50 characters long
// - Password is at least 8 characters
// - Password contains uppercase, lowercase, digit, and special character

// Using in a service method
@PostMapping("/users")
public ResponseEntity<User> createUser(@Valid @RequestBody CoreUserRequest request) {
    // Validation happens automatically due to @Valid annotation
    return userService.createUser(request);
}
```

The class uses strict regex patterns for name validation supporting international characters while preventing injection attacks, and enforces enterprise-grade password complexity requirements.
