Contains nested DTO classes for handling password reset operations, including request initiation and password confirmation with validation.

## Key Components

- **ResetRequest**: DTO for initiating password reset requests
  - `email` - Validated email address field
- **ResetConfirm**: DTO for confirming password reset with new password
  - `token` - Required reset token
  - `newPassword` - New password with comprehensive validation rules

## Usage Example

```java
// Initiating a password reset
PasswordResetDtos.ResetRequest resetRequest = new PasswordResetDtos.ResetRequest();
resetRequest.setEmail("user@example.com");

// Confirming password reset
PasswordResetDtos.ResetConfirm resetConfirm = new PasswordResetDtos.ResetConfirm();
resetConfirm.setToken("reset-token-123");
resetConfirm.setNewPassword("SecurePass123!");

// The validation annotations ensure:
// - Email format is valid
// - Token is not blank
// - Password is at least 8 characters with uppercase, lowercase, digit, and special character
```

The DTOs leverage Jakarta validation annotations and Lombok's `@Data` for automatic getter/setter generation, providing type-safe password reset functionality with built-in validation.
