This strategy class provides Microsoft-specific OIDC client registration configuration by extending the base OIDC registration strategy with Microsoft SSO properties.

## Key Components

- **`MicrosoftClientRegistrationStrategy`** - Main strategy class that extends `BaseOidcClientRegistrationStrategy`
- **`providerId()`** - Returns the Microsoft provider identifier constant
- **`props()`** - Provides Microsoft-specific OIDC configuration properties
- **Dependencies**: `SSOConfigService` and `MicrosoftSSOProperties`

## Usage Example

```java
@Autowired
private MicrosoftClientRegistrationStrategy microsoftStrategy;

// Get the provider ID for Microsoft
String providerId = microsoftStrategy.providerId(); // Returns "microsoft"

// The strategy is automatically used by the OIDC framework
// when Microsoft SSO is configured in application properties:
// openframe.authz.oidc.microsoft.client-id=your-client-id
// openframe.authz.oidc.microsoft.client-secret=your-client-secret
```

The strategy follows the Strategy pattern, allowing the authentication framework to handle Microsoft OIDC flows without hardcoding provider-specific logic. It's automatically registered as a Spring component and used when Microsoft SSO is enabled.
