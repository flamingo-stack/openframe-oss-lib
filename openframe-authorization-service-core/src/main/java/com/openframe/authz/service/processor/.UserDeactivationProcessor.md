This interface defines a contract for handling post-processing operations after a user has been deactivated in the authentication system.

## Key Components

- **`postProcessDeactivation(AuthUser existing)`** - Method called after user deactivation to perform cleanup, notifications, or other side effects

## Usage Example

```java
@Service
public class EmailNotificationProcessor implements UserDeactivationProcessor {
    
    @Autowired
    private EmailService emailService;
    
    @Override
    public void postProcessDeactivation(AuthUser existing) {
        // Send deactivation notification
        emailService.sendDeactivationNotice(existing.getEmail());
        
        // Log the deactivation event
        log.info("User {} has been deactivated", existing.getUsername());
    }
}

// Multiple processors can be implemented
@Service  
public class SessionCleanupProcessor implements UserDeactivationProcessor {
    
    @Override
    public void postProcessDeactivation(AuthUser existing) {
        // Invalidate all user sessions
        sessionManager.invalidateAllUserSessions(existing.getId());
        
        // Clear user caches
        cacheManager.evictUserData(existing.getId());
    }
}
```

This processor pattern allows for modular, extensible handling of user deactivation workflows while maintaining separation of concerns.
