Service that helps users discover which tenants they can access based on their email address, supporting multiple authentication provider discovery strategies.

## Key Components

- **`discoverTenantForEmail(String email)`** - Main discovery method that returns tenant access information
- **`getAvailableAuthProviders(Tenant tenant)`** - Retrieves available SSO and authentication providers for a tenant
- **Configuration Properties** - `localTenant` flag and `baseDomain` for tenant resolution
- **Multi-layered Discovery** - Cascading fallback strategy through existing users, domain-based auto-provisioning, and global domain policies

## Usage Example

```java
@Autowired
private TenantDiscoveryService discoveryService;

// Discover tenants for a user email
TenantDiscoveryResponse response = discoveryService.discoverTenantForEmail("user@company.com");

if (response.isHasExistingAccounts()) {
    String tenantId = response.getTenantId();
    List<String> authProviders = response.getAuthProviders();
    
    // User has access - show available login options
    log.info("User {} can access tenant {} via providers: {}", 
             response.getEmail(), tenantId, authProviders);
} else {
    // No tenant access found
    log.info("No tenant access found for {}", response.getEmail());
}
```

The service implements a three-tier discovery strategy: first checking for existing user accounts, then domain-based SSO auto-provisioning rules, and finally global domain policies. Always includes the default "openframe-sso" provider alongside any configured SSO providers for the discovered tenant.
