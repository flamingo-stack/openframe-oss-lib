Handles secure password reset functionality through time-limited tokens stored in Redis with email delivery and validation.

## Key Components

- **createResetToken(String email)** - Generates and stores a secure reset token for valid user emails
- **resetPassword(String token, String newPassword)** - Validates reset token and updates user password
- **Token Management** - Uses Redis for temporary token storage with configurable TTL (default 30 minutes)
- **Email Integration** - Sends reset tokens via EmailService
- **Security Features** - Token-based validation, automatic cleanup, and user existence verification

## Usage Example

```java
@Autowired
private PasswordResetService passwordResetService;

// Initiate password reset flow
passwordResetService.createResetToken("user@example.com");

// Complete password reset with token
try {
    passwordResetService.resetPassword("abc123token", "newSecurePassword");
    log.info("Password reset successful");
} catch (IllegalArgumentException e) {
    log.error("Reset failed: {}", e.getMessage());
}
```

The service uses Redis for scalable token storage with tenant-aware key prefixing and automatic expiration. Reset tokens are cryptographically secure and single-use, automatically deleted after successful password updates. Failed reset attempts for non-existent emails are logged but don't reveal user existence to prevent enumeration attacks.
