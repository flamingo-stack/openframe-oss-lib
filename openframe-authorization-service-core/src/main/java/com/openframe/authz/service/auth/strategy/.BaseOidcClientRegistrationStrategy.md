Base abstract strategy class for creating OAuth2/OIDC client registrations dynamically based on tenant-specific SSO configurations. This class provides common functionality for building OAuth2 client configurations while allowing specific providers to customize their properties.

## Key Components

- **BaseOidcClientRegistrationStrategy** - Abstract base class implementing `ClientRegistrationStrategy`
- **buildClient(String tenantId)** - Main method that constructs OAuth2 `ClientRegistration` instances
- **props()** - Abstract method for subclasses to provide provider-specific properties
- **SSOConfigService** - Injected service for retrieving SSO configurations
- **capitalize(String)** - Utility method for formatting provider names

## Usage Example

```java
public class AzureOidcClientRegistrationStrategy extends BaseOidcClientRegistrationStrategy {
    
    private final AzureOidcProviderProperties azureProps;
    
    public AzureOidcClientRegistrationStrategy(SSOConfigService ssoConfigService, 
                                             AzureOidcProviderProperties azureProps) {
        super(ssoConfigService);
        this.azureProps = azureProps;
    }
    
    @Override
    protected AbstractOidcProviderProperties props() {
        return azureProps;
    }
    
    @Override
    public String providerId() {
        return "azure";
    }
}

// Usage
ClientRegistration registration = strategy.buildClient("tenant-123");
```

The strategy pattern allows different OIDC providers (Azure AD, Google, etc.) to implement provider-specific URL construction while sharing common OAuth2 client building logic.
