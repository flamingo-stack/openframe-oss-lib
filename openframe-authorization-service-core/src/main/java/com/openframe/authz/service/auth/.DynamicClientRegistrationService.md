A service that manages dynamic OAuth2 client registration by delegating to provider-specific strategies based on provider ID and tenant context.

## Key Components

- **DynamicClientRegistrationService** - Main service class that orchestrates client registration
- **loadClient()** - Core method that selects appropriate strategy and builds OAuth2 client configuration
- **strategies** - Injected list of `ClientRegistrationStrategy` implementations for different SSO providers

## Usage Example

```java
@Autowired
private DynamicClientRegistrationService registrationService;

// Load client configuration for Google SSO in a specific tenant
ClientRegistration googleClient = registrationService.loadClient("google", "tenant-123");

// Load client configuration for Microsoft Azure AD
ClientRegistration azureClient = registrationService.loadClient("azure", "tenant-456");

// This would throw IllegalArgumentException for unsupported providers
try {
    ClientRegistration unsupportedClient = registrationService.loadClient("unknown-provider", "tenant-789");
} catch (IllegalArgumentException e) {
    log.error("Provider not supported: {}", e.getMessage());
}
```

The service uses the Strategy pattern to support multiple SSO providers, automatically selecting the correct implementation based on the provider ID and building tenant-specific OAuth2 client configurations.
