Service that dynamically loads OAuth2 client registrations based on provider ID and tenant ID using a strategy pattern approach.

## Key Components

- **loadClient(String, String)** - Main method that retrieves appropriate client registration strategy and builds OAuth2 client configuration
- **strategies** - List of `ClientRegistrationStrategy` implementations injected for handling different OAuth2 providers
- Uses strategy pattern to support multiple SSO providers (Google, Azure, etc.)

## Usage Example

```java
@Autowired
private DynamicClientRegistrationService registrationService;

// Load client registration for Google SSO for specific tenant
ClientRegistration googleClient = registrationService.loadClient("google", "tenant-123");

// Load client registration for Azure AD for different tenant  
ClientRegistration azureClient = registrationService.loadClient("azure", "tenant-456");

// Use the client registration with OAuth2 flow
OAuth2AuthorizedClientManager clientManager = new DefaultOAuth2AuthorizedClientManager(
    clientRegistrationRepository, authorizedClientService);
```

The service automatically selects the correct strategy implementation based on the provider ID and delegates client building to the strategy, enabling tenant-specific OAuth2 configurations across multiple identity providers.
