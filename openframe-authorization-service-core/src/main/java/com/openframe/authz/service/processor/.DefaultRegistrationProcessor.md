Default fallback implementation of `RegistrationProcessor` that provides no-op lifecycle hooks for tenant, invitation, and auto-provision registration flows. Activated automatically via `@ConditionalOnMissingBean` when no custom processor is registered.

## Key Components

| Method | Description |
|--------|-------------|
| `preProcessTenantRegistration` | No-op hook called before tenant registration is processed |
| `postProcessTenantRegistration` | No-op hook called after a tenant and user are created |
| `postProcessInvitationRegistration` | No-op hook called after a user registers via invitation |
| `postProcessAutoProvision` | No-op hook called after a user is auto-provisioned |

## Usage Example

Override the default behavior by providing a custom `RegistrationProcessor` bean — once detected, Spring will skip loading `DefaultRegistrationProcessor`:

```java
@Slf4j
@Component
public class CustomRegistrationProcessor implements RegistrationProcessor {

    @Override
    public void postProcessTenantRegistration(
            Tenant tenant, AuthUser user, TenantRegistrationRequest request) {
        // Send welcome email, provision resources, etc.
        log.info("New tenant registered: {}", tenant.getId());
        notificationService.sendWelcome(user.getEmail());
    }

    @Override
    public void postProcessAutoProvision(AuthUser user, String pictureUrl) {
        // Sync profile picture to external system
        profileService.syncAvatar(user.getId(), pictureUrl);
    }

    // Implement remaining hooks as needed...
}
```

> All methods default to debug-level logging only. Custom implementations should extend or replace this bean to add business logic such as notifications, audit trails, or third-party provisioning.
