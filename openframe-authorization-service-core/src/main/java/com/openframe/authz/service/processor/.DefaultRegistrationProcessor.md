Default implementation of the RegistrationProcessor interface that provides no-operation (no-op) implementations for all registration processing hooks. This component serves as a fallback when no custom registration processor is configured.

## Key Components

- **DefaultRegistrationProcessor**: Main class implementing `RegistrationProcessor` interface
- **preProcessTenantRegistration()**: Hook executed before tenant registration
- **postProcessTenantRegistration()**: Hook executed after tenant and user creation
- **postProcessInvitationRegistration()**: Hook executed after invitation-based registration
- **postProcessAutoProvision()**: Hook executed after auto-provisioning users

## Usage Example

The default processor is automatically used when no custom implementation is provided:

```java
// Custom processor that overrides default behavior
@Component
public class CustomRegistrationProcessor implements RegistrationProcessor {
    
    @Override
    public void postProcessTenantRegistration(Tenant tenant, AuthUser user, 
                                            TenantRegistrationRequest request) {
        // Send welcome email
        emailService.sendWelcome(user.getEmail());
        
        // Initialize default settings
        settingsService.createDefaults(tenant.getId());
    }
    
    @Override
    public void postProcessInvitationRegistration(AuthUser user, String invitationId,
                                                 InvitationRegistrationRequest request) {
        // Add user to invited team
        teamService.addMember(user.getId(), request.getTeamId());
    }
}
```

The `@ConditionalOnMissingBean` annotation ensures the default processor only loads when no custom implementation exists, enabling easy customization of registration workflows.
