A default no-operation implementation of the RegistrationProcessor interface that provides fallback behavior when no custom registration processor is configured. This component uses Spring's conditional bean creation to only activate when no other RegistrationProcessor implementation is available.

## Key Components

- **DefaultRegistrationProcessor** - Main component implementing RegistrationProcessor interface
- **preProcessTenantRegistration()** - Hook for pre-processing tenant registration requests
- **postProcessTenantRegistration()** - Hook for post-processing completed tenant registrations
- **postProcessInvitationRegistration()** - Hook for post-processing invitation-based user registrations
- **postProcessAutoProvision()** - Hook for post-processing auto-provisioned user accounts

## Usage Example

```java
// This component is automatically registered when no custom processor exists
// Custom implementations can override specific methods:

@Component
public class CustomRegistrationProcessor implements RegistrationProcessor {
    
    @Override
    public void postProcessTenantRegistration(Tenant tenant, AuthUser user, 
                                            TenantRegistrationRequest request) {
        // Send welcome email
        emailService.sendWelcomeEmail(user.getEmail(), tenant.getName());
        
        // Create default workspace
        workspaceService.createDefaultWorkspace(tenant.getId());
    }
    
    // Other methods will fall back to default no-op behavior
}

// The conditional annotation ensures only one processor is active:
// @ConditionalOnMissingBean(value = RegistrationProcessor.class, 
//                          ignored = DefaultRegistrationProcessor.class)
```

The conditional bean annotation ensures this default processor is only used when no custom implementation is provided, allowing for easy customization of registration workflows while maintaining a working fallback.
