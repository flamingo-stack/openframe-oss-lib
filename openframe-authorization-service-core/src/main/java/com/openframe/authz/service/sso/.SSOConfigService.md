Manages SSO (Single Sign-On) configuration resolution for multi-tenant environments, providing tenant-specific or fallback provider configurations with encrypted credential handling.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `getSSOConfig` | Method | Retrieves an active, enabled `SSOPerTenantConfig` by tenant ID and provider |
| `getEffectiveSSOConfig` | Method | Returns the best available `SSOConfig` — tenant-specific first, then falls back to application properties defaults |
| `getActiveForTenant` | Method | Lists all enabled SSO configurations for a given tenant |
| `getEffectiveProvidersForTenant` | Method | Returns a de-duplicated union of active tenant providers and configured default providers |
| `getDefaultProviders` | Method | Lists providers available from application properties (non-blank client ID and secret required) |
| `findAutoProvisionByDomain` | Method | Locates an enabled, auto-provisioning SSO config matching an email domain |
| `getDecryptedClientSecret` | Method | Decrypts a stored client secret via `EncryptionService` |
| `localTenant` | Field | When `true`, ignores tenant scoping (single-tenant/local mode) |

## Usage Example

```java
// Resolve effective SSO config for a tenant and provider
Optional<SSOConfig> config = ssoConfigService.getEffectiveSSOConfig("tenant-123", "google");

config.ifPresent(cfg -> {
    String clientId = cfg.getClientId();
    String secret   = ssoConfigService.getDecryptedClientSecret(cfg);
    // Use clientId and secret to initiate OAuth2 flow
});

// List all active providers for a tenant (includes property-based defaults)
List<String> providers = ssoConfigService.getEffectiveProvidersForTenant("tenant-123");
// e.g. ["google", "microsoft", "okta"]

// Auto-provision lookup by inbound email domain
Optional<SSOPerTenantConfig> autoProvision =
    ssoConfigService.findAutoProvisionByDomain("acme.com");

autoProvision.ifPresent(cfg -> {
    // Automatically enroll user under cfg.getTenantId()
});
```

## Notes

- Tenant ID scoping is bypassed when `openframe.tenancy.local-tenant=true`, enabling single-tenant deployments.
- Default provider configs (from application properties) are only active when **both** `clientId` and `clientSecret` are non-blank.
- Client secrets are stored encrypted and must be decrypted via `getDecryptedClientSecret` before use in OAuth2 flows.
