A strategy implementation for Google OAuth2/OIDC client registration that extends the base OIDC registration strategy with Google-specific configuration.

## Key Components

- **GoogleClientRegistrationStrategy** - Main strategy class extending `BaseOidcClientRegistrationStrategy`
- **providerId()** - Returns the Google provider identifier constant
- **props()** - Provides Google-specific OIDC configuration properties
- **GoogleSSOProperties** - Google-specific SSO configuration properties dependency
- **SSOConfigService** - Service for managing SSO configurations (inherited from base)

## Usage Example

This strategy is automatically registered as a Spring component and used by the OIDC client registration system when Google authentication is configured:

```java
// The strategy is automatically discovered and used when:
// 1. Google SSO is enabled in application configuration
// 2. A client registration request is made for the "google" provider

// Example configuration that would trigger this strategy:
@Configuration
public class AuthConfig {
    
    @Bean
    public GoogleSSOProperties googleSSOProperties() {
        GoogleSSOProperties props = new GoogleSSOProperties();
        props.setClientId("your-google-client-id");
        props.setClientSecret("your-google-client-secret");
        props.setEnabled(true);
        return props;
    }
}

// The strategy automatically handles:
// - Provider ID resolution ("google")
// - Google-specific OIDC configuration
// - Integration with base registration logic
```

The strategy follows the strategy pattern to provide Google-specific implementation while leveraging common OIDC registration functionality from the base class.
