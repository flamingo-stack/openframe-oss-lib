Interface defining hooks for customizing registration workflows in the authorization service. Provides default no-op implementations for pre and post-processing steps during tenant and user registration.

## Key Components

- **preProcessTenantRegistration()** - Hook executed before tenant registration begins
- **postProcessTenantRegistration()** - Hook executed after successful tenant and user creation
- **postProcessInvitationRegistration()** - Hook for invitation-based user registration completion
- **postProcessAutoProvision()** - Hook for SSO auto-provisioned user creation

## Usage Example

```java
@Component
public class CustomRegistrationProcessor implements RegistrationProcessor {
    
    @Override
    public void preProcessTenantRegistration(TenantRegistrationRequest request) {
        // Validate business rules or prepare external systems
        validateDomain(request.getDomain());
    }
    
    @Override
    public void postProcessTenantRegistration(Tenant tenant, AuthUser user, TenantRegistrationRequest request) {
        // Send welcome emails, create default settings, etc.
        emailService.sendWelcomeEmail(user.getEmail());
        setupDefaultTenantSettings(tenant);
    }
    
    @Override
    public void postProcessInvitationRegistration(AuthUser user, String invitationId, InvitationRegistrationRequest request) {
        // Track invitation completion metrics
        analyticsService.trackInvitationAccepted(invitationId);
    }
}
```

The interface uses default methods, allowing implementations to override only the hooks they need while maintaining a clean, extensible registration pipeline.
