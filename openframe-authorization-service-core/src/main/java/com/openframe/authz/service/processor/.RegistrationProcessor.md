Interface for extending registration workflows with customizable pre and post-processing hooks. Provides a plugin mechanism for adding custom logic around tenant creation, invitation-based registration, and auto-provisioning flows.

## Key Components

- **preProcessTenantRegistration()** - Hook executed before tenant registration begins
- **postProcessTenantRegistration()** - Hook executed after successful tenant and user creation
- **postProcessInvitationRegistration()** - Hook executed after invitation-based user registration
- **postProcessAutoProvision()** - Hook executed after SSO auto-provisioning of new users

## Usage Example

```java
@Component
public class CustomRegistrationProcessor implements RegistrationProcessor {
    
    @Override
    public void preProcessTenantRegistration(TenantRegistrationRequest request) {
        // Validate business rules before registration
        validateCompanyDomain(request.getDomain());
        logRegistrationAttempt(request);
    }
    
    @Override
    public void postProcessTenantRegistration(Tenant tenant, AuthUser user, 
                                            TenantRegistrationRequest request) {
        // Send welcome emails, setup default data, etc.
        emailService.sendWelcomeEmail(user.getEmail());
        setupDefaultTenantData(tenant);
        analyticsService.trackTenantCreation(tenant.getId());
    }
    
    @Override
    public void postProcessInvitationRegistration(AuthUser user, String invitationId, 
                                                 InvitationRegistrationRequest request) {
        // Assign roles based on invitation type
        roleService.assignInvitationRoles(user, invitationId);
        notificationService.notifyInviter(invitationId, user);
    }
}
```

All methods provide default no-op implementations, allowing selective override of only needed hooks.
