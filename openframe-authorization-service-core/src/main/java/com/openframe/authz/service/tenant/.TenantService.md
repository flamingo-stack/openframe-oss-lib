Service layer component for managing tenant operations in a multi-tenant architecture, providing CRUD operations and domain validation for tenant entities.

## Key Components

- **createTenant()** - Creates new tenant with validation, auto-generated ID, and default configuration
- **existByDomain()** - Checks if tenant exists by domain name
- **findById()** - Retrieves tenant by unique identifier
- **findFirst()** - Gets the first available tenant (utility method)
- **save()** - Persists tenant entity changes
- **TenantRepository** - Data access layer dependency
- **RegistrationValidationService** - Domain validation dependency

## Usage Example

```java
@Autowired
private TenantService tenantService;

// Create a new tenant
Tenant newTenant = tenantService.createTenant("Acme Corp", "acme.example.com");

// Check if domain is already taken
boolean exists = tenantService.existByDomain("acme.example.com");

// Find tenant by ID
Optional<Tenant> tenant = tenantService.findById("tenant-123");

// Update tenant
if (tenant.isPresent()) {
    Tenant updated = tenant.get();
    updated.setStatus(TenantStatus.SUSPENDED);
    tenantService.save(updated);
}

// Get any available tenant
Optional<Tenant> firstTenant = tenantService.findFirst();
```

The service automatically sets new tenants to ACTIVE status with FREE plan and validates domain availability before creation.
