Manages tenant lifecycle operations within OpenFrame's multi-tenant architecture, providing creation, lookup, and persistence capabilities with domain uniqueness validation.

## Key Components

| Method | Description |
|--------|-------------|
| `createTenant(String tenantName, String domain)` | Creates and persists a new active tenant after validating domain availability |
| `existByDomain(String domain)` | Checks whether a tenant with the given domain already exists |
| `findById(String tenantId)` | Retrieves a tenant by its unique ID, returning an `Optional` |
| `findFirst()` | Returns the first tenant found in the repository, useful for single-tenant bootstrapping |
| `save(Tenant tenant)` | Persists an existing tenant entity |

## Usage Example

```java
// Create a new tenant
Tenant tenant = tenantService.createTenant("Acme Corp", "acme.openframe.ai");

// Check domain availability before registration
if (!tenantService.existByDomain("acme.openframe.ai")) {
    tenantService.createTenant("Acme Corp", "acme.openframe.ai");
}

// Lookup a tenant by ID
Optional<Tenant> found = tenantService.findById("tenant-abc123");
found.ifPresent(t -> log.info("Tenant: {}", t.getName()));
```

> **Note:** All methods are executed within a Spring `@Transactional` context. Domain uniqueness is enforced via `RegistrationValidationService.ensureTenantDomainAvailable()` before persisting a new tenant, preventing duplicate domain registrations across the platform.
