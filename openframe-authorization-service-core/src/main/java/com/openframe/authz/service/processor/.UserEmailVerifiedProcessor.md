This interface defines a hook for processing user email verification events in the Authorization Server, allowing implementations to react to successful email verifications with actions like publishing Kafka messages or creating audit logs.

## Key Components

- **`UserEmailVerifiedProcessor`** - Interface for handling post-email-verification logic
- **`postProcessEmailVerified(AuthUser user)`** - Method called after a user's email verification is completed

## Usage Example

```java
@Component
public class EmailVerificationNotificationProcessor implements UserEmailVerifiedProcessor {
    
    private final KafkaTemplate<String, Object> kafkaTemplate;
    private final AuditService auditService;
    
    public EmailVerificationNotificationProcessor(
        KafkaTemplate<String, Object> kafkaTemplate,
        AuditService auditService
    ) {
        this.kafkaTemplate = kafkaTemplate;
        this.auditService = auditService;
    }
    
    @Override
    public void postProcessEmailVerified(AuthUser user) {
        // Publish notification event
        kafkaTemplate.send("user.email.verified", user.getId(), user);
        
        // Log audit event
        auditService.logEvent("EMAIL_VERIFIED", user.getId(), 
            "User email verification completed");
        
        // Additional business logic
        enableUserFeatures(user);
    }
    
    private void enableUserFeatures(AuthUser user) {
        // Enable features that require verified email
    }
}
```

This processor pattern enables decoupled handling of email verification events, making it easy to add new functionality without modifying core authentication logic.
