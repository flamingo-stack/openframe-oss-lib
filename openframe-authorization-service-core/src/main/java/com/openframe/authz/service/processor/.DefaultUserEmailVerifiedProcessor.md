Default no-op implementation of the UserEmailVerifiedProcessor interface that simply logs when a user's email is verified. This component serves as a fallback when no custom email verification processor is configured.

## Key Components

- **DefaultUserEmailVerifiedProcessor** - Spring component implementing UserEmailVerifiedProcessor
- **postProcessEmailVerified()** - Method that logs the user ID when email verification occurs
- **@ConditionalOnMissingBean** - Ensures this default is only used when no custom implementation exists

## Usage Example

This processor automatically activates when no custom UserEmailVerifiedProcessor bean is defined:

```java
// When email verification occurs, this processor will log:
// "User email verified: {userId}"

// To override with custom behavior, create your own component:
@Component
public class CustomEmailVerifiedProcessor implements UserEmailVerifiedProcessor {
    @Override
    public void postProcessEmailVerified(AuthUser user) {
        // Send welcome email
        // Update user status
        // Trigger analytics event
    }
}
```

The default processor provides basic logging functionality while allowing applications to easily override the behavior by providing their own UserEmailVerifiedProcessor implementation.
