Interface for implementing OAuth2 client registration strategies that can dynamically build client configurations for different providers and tenants.

## Key Components

- **`providerId()`** - Returns a unique identifier for the OAuth2 provider (e.g., "google", "github", "okta")
- **`buildClient(String tenantId)`** - Constructs a `ClientRegistration` instance configured for the specified tenant and provider

## Usage Example

```java
@Component
public class GoogleClientRegistrationStrategy implements ClientRegistrationStrategy {
    
    @Override
    public String providerId() {
        return "google";
    }
    
    @Override
    public ClientRegistration buildClient(String tenantId) {
        return ClientRegistration.withRegistrationId("google-" + tenantId)
            .clientId(getClientIdForTenant(tenantId))
            .clientSecret(getClientSecretForTenant(tenantId))
            .scope("openid", "profile", "email")
            .authorizationUri("https://accounts.google.com/o/oauth2/auth")
            .tokenUri("https://oauth2.googleapis.com/token")
            .userInfoUri("https://www.googleapis.com/oauth2/v3/userinfo")
            .userNameAttributeName("sub")
            .clientName("Google")
            .build();
    }
}
```

This strategy pattern enables multi-tenant applications to support different OAuth2 providers with tenant-specific configurations.
