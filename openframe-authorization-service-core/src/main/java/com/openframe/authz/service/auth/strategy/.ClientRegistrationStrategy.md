Defines a strategy interface for building OAuth2 client registrations dynamically based on provider and tenant context. This interface enables flexible authentication provider configuration across multi-tenant applications.

## Key Components

- **`providerId()`** - Returns the unique identifier for the OAuth2 provider (e.g., "google", "github", "azure")
- **`buildClient(String tenantId)`** - Constructs a Spring Security `ClientRegistration` object for the specified tenant

## Usage Example

```java
@Component
public class GoogleClientRegistrationStrategy implements ClientRegistrationStrategy {
    
    @Override
    public String providerId() {
        return "google";
    }
    
    @Override
    public ClientRegistration buildClient(String tenantId) {
        return ClientRegistration.withRegistrationId("google-" + tenantId)
            .clientId(getClientId(tenantId))
            .clientSecret(getClientSecret(tenantId))
            .authorizationUri("https://accounts.google.com/oauth/authorize")
            .tokenUri("https://oauth2.googleapis.com/token")
            .userInfoUri("https://openidconnect.googleapis.com/v1/userinfo")
            .redirectUri("{baseUrl}/login/oauth2/code/google")
            .scope("openid", "profile", "email")
            .authorizationGrantType(AuthorizationGrantType.AUTHORIZATION_CODE)
            .build();
    }
    
    private String getClientId(String tenantId) {
        // Retrieve tenant-specific Google client ID
        return configService.getGoogleClientId(tenantId);
    }
}
```

This pattern allows the authentication system to support multiple OAuth2 providers with tenant-specific configurations, enabling each tenant to use their own OAuth2 credentials while sharing the same application infrastructure.
