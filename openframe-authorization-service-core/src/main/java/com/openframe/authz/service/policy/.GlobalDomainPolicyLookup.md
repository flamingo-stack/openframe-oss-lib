Abstraction for looking up global domain policies on a per-tenant basis without creating hard dependencies. This interface enables SaaS authentication servers to provide domain-to-tenant mapping while allowing default implementations to gracefully handle absence of such functionality.

## Key Components

- **findTenantIdByDomainIfAutoAllowed()** - Core lookup method that returns tenant ID if domain is configured for automatic tenant association

## Usage Example

```java
// Service implementation
@Service
public class AuthenticationService {
    private final GlobalDomainPolicyLookup domainPolicyLookup;
    
    public Optional<String> resolveTenantFromDomain(String emailDomain) {
        return domainPolicyLookup.findTenantIdByDomainIfAutoAllowed(emailDomain);
    }
}

// Default implementation (returns empty)
@Component
public class DefaultDomainPolicyLookup implements GlobalDomainPolicyLookup {
    @Override
    public Optional<String> findTenantIdByDomainIfAutoAllowed(String domain) {
        return Optional.empty();
    }
}

// SaaS implementation example
@Component
@ConditionalOnProperty("saas.enabled")
public class SaasDomainPolicyLookup implements GlobalDomainPolicyLookup {
    @Override
    public Optional<String> findTenantIdByDomainIfAutoAllowed(String domain) {
        // Query tenant configuration for domain auto-assignment
        return tenantRepository.findByAutoAllowedDomain(domain)
            .map(Tenant::getId);
    }
}
```
