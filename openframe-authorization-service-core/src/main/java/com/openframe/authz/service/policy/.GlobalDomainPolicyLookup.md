Interface for abstracting global domain policy lookups in multi-tenant environments. Enables tenant identification by domain without creating hard dependencies between components.

## Key Components

- **`findTenantIdByDomainIfAutoAllowed(String domain)`** - Looks up tenant ID for a given domain, returning result only if auto-allowed by policy

## Usage Example

```java
@Service
public class AuthService {
    private final GlobalDomainPolicyLookup policyLookup;
    
    public AuthService(GlobalDomainPolicyLookup policyLookup) {
        this.policyLookup = policyLookup;
    }
    
    public Optional<String> resolveTenant(String userDomain) {
        return policyLookup.findTenantIdByDomainIfAutoAllowed(userDomain);
    }
}

// In a SaaS environment
Optional<String> tenantId = policyLookup
    .findTenantIdByDomainIfAutoAllowed("company.com");

if (tenantId.isPresent()) {
    // Process request for resolved tenant
    processForTenant(tenantId.get());
}
```

The interface follows a plugin architecture pattern where the default implementation returns `Optional.empty()`, while specialized implementations (like SaaS Auth Server) provide actual domain-to-tenant resolution logic.
