Default implementation of the `UserDeactivationProcessor` interface that provides basic logging functionality when users are deactivated in the authorization service.

## Key Components

- **DefaultUserDeactivationProcessor** - Spring component implementing `UserDeactivationProcessor` interface
- **postProcessDeactivation()** - Method that logs user deactivation events at debug level
- **@ConditionalOnMissingBean** - Ensures this bean is only used when no custom implementation is provided

## Usage Example

This component is automatically registered by Spring when no custom `UserDeactivationProcessor` implementation exists:

```java
// Custom implementation to override default behavior
@Component
public class CustomUserDeactivationProcessor implements UserDeactivationProcessor {
    @Override
    public void postProcessDeactivation(AuthUser existing) {
        // Custom logic - audit logging, notifications, cleanup, etc.
        auditService.logUserDeactivation(existing.getId());
        notificationService.sendDeactivationAlert(existing);
    }
}

// The default processor will be bypassed when custom implementation is present
// Default behavior: only logs "User deactivated: {userId}" at debug level
```

The processor integrates seamlessly into the authorization service's user lifecycle management, providing extensibility for post-deactivation tasks while maintaining a minimal default implementation.
