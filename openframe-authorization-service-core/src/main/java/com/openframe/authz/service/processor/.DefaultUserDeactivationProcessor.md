A default implementation of the UserDeactivationProcessor interface that provides basic logging functionality when users are deactivated in the authorization service.

## Key Components

- **DefaultUserDeactivationProcessor**: Spring component implementing `UserDeactivationProcessor` interface
- **postProcessDeactivation()**: Main processing method that logs user deactivation events
- **@ConditionalOnMissingBean**: Ensures this default implementation is only used when no custom processor is provided

## Usage Example

This component is automatically registered by Spring when no other `UserDeactivationProcessor` implementation exists:

```java
// This default processor will be used automatically
// Custom implementation would override the default:

@Component
public class CustomUserDeactivationProcessor implements UserDeactivationProcessor {
    @Override
    public void postProcessDeactivation(AuthUser existing) {
        // Custom deactivation logic (e.g., cleanup tasks, notifications)
        cleanupUserSessions(existing);
        sendDeactivationEmail(existing);
    }
}
```

The default processor provides minimal functionality with debug-level logging. Applications can override this behavior by providing their own `UserDeactivationProcessor` implementation for custom post-deactivation workflows like session cleanup, audit logging, or notification sending.
