Service that handles the initiation of SSO (Single Sign-On) invitation acceptance flows by validating invitations, configuring SSO providers, and generating secure authorization data.

## Key Components

- **`startAccept(SsoInvitationAcceptRequest)`** - Main method that initiates the SSO invitation acceptance process
- **`COOKIE_TTL_SECONDS`** - Constant defining cookie expiration time (600 seconds/10 minutes)
- **Dependencies**:
  - `InvitationValidator` - Validates and loads invitation data
  - `SsoProviderValidator` - Validates and normalizes SSO provider configurations
  - `SsoCookieCodec` - Encodes secure cookie payloads for SSO state management

## Usage Example

```java
@Autowired
private SsoInvitationService ssoInvitationService;

public void handleInvitationAcceptance() {
    SsoInvitationAcceptRequest request = SsoInvitationAcceptRequest.builder()
        .invitationId("inv-123")
        .provider("google")
        .switchTenant(true)
        .redirectTo("/dashboard")
        .build();
    
    SsoAuthorizeData authData = ssoInvitationService.startAccept(request);
    
    // Use authData to set secure cookies and redirect user
    String redirectUrl = authData.getRedirectPath();
    String secureToken = authData.getToken();
    int cookieMaxAge = authData.getCookieTtlSeconds();
}
```

The service creates a secure state token, validates the invitation and SSO provider, then returns authorization data including a redirect path to the OAuth2 authorization endpoint with the appropriate tenant context.
