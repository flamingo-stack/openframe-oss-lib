Handles HTTP endpoints for tenant registration in the OpenFrame authorization service, supporting both standard credential-based registration and SSO-initiated flows.

## Key Components

| Element | Description |
|---|---|
| `POST /oauth/register` | Registers a new tenant using a JSON payload (`TenantRegistrationRequest`) |
| `GET /oauth/register/sso` | Initiates an SSO-based tenant registration flow, clears auth state, sets a secure registration cookie, and redirects to the IdP |
| `buildSsoRegistrationCookie()` | Constructs an `HttpOnly`, `Secure` cookie scoped to `/` to carry the SSO registration state across the OAuth redirect |
| `authErrorUrl` | Injected via `openframe.auth.error-url`; used as the redirect target when SSO initialization fails |

## Usage Example

**Standard tenant registration:**

```java
POST /oauth/register
Content-Type: application/json

{
  "tenantName": "acme-corp",
  "adminEmail": "admin@acme.com",
  "password": "s3cur3!"
}
// Returns: Tenant object (HTTP 200)
```

**SSO registration initiation:**

```java
GET /oauth/register/sso?provider=google&returnUrl=https://app.openframe.ai/setup
// Clears existing auth cookies
// Sets COOKIE_SSO_REG with TTL from SsoAuthorizeData
// Redirects (303) to IdP authorization URL
// On failure → redirects to /error?error=<encoded-message>
```

## Error Handling

SSO initiation errors are caught at the controller level — exceptions are URL-encoded and forwarded to the configured `authErrorUrl` as a query parameter, preventing stack traces from leaking to the client.
