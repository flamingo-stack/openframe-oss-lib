REST controller that handles tenant registration workflows, supporting both direct JSON registration and SSO-initiated registration flows.

## Key Components

- **`register()`** - POST endpoint for direct tenant registration using JSON payload
- **`startSsoRegistration()`** - GET endpoint that initiates SSO-based tenant registration flow
- **`buildSsoRegistrationCookie()`** - Creates secure HTTP-only cookies for SSO registration state
- **Dependencies**:
  - `TenantRegistrationService` - Handles direct tenant registration logic
  - `SsoTenantRegistrationService` - Manages SSO registration workflow

## Usage Example

```java
// Direct tenant registration via POST
@PostMapping("/oauth/register")
public Tenant register(@Valid @RequestBody TenantRegistrationRequest request) {
    return registrationService.registerTenant(request);
}

// SSO registration initiation via GET
@GetMapping("/oauth/register/sso")  
public void startSsoRegistration(@Valid @ModelAttribute SsoTenantRegistrationInitRequest request,
                                HttpServletRequest httpRequest,
                                HttpServletResponse httpResponse) {
    // Clears auth state, creates SSO cookie, redirects to OAuth provider
    clearAuthState(httpRequest, httpResponse);
    SsoAuthorizeData data = ssoRegistrationService.startRegistration(request);
    httpResponse.addCookie(buildSsoRegistrationCookie(data.cookieValue(), data.cookieTtlSeconds()));
    seeOther(httpResponse, data.redirectPath());
}
```

The controller provides two registration paths: direct API registration returning a `Tenant` object, and SSO registration that redirects users through OAuth2 authorization flow with secure cookie-based state management.
