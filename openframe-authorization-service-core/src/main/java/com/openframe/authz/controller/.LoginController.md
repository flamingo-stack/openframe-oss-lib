Spring MVC controller that handles login page rendering and index page display for OpenFrame's multi-tenant authorization system with optional password reset integration.

## Key Components

- **LoginController**: Main Spring MVC controller class
- **login()**: Handles `/login` endpoint with error message display and optional password reset URL
- **index()**: Handles root `/` endpoint with welcome message
- **passwordResetPageUrl**: Configurable property for password reset page integration

## Usage Example

```java
// Configuration for password reset integration
# application.properties
openframe.password-reset.page-url=https://reset.example.com/password

// Error handling - URL with error parameter
GET /login?error=true
// Returns login view with "Invalid credentials" message

// Standard login page access
GET /login
// Returns login view with optional password reset link

// Index page access
GET /
// Returns index view with OpenFrame welcome message
```

The controller integrates with Spring Security's authentication flow, displaying error messages when authentication fails and conditionally showing password reset options based on configuration. The `login` method processes authentication errors and conditionally adds a password reset URL to the model if configured, while the `index` method provides a simple landing page with the OpenFrame authorization message.
