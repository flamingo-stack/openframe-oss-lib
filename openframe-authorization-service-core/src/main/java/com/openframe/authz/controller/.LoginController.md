Handles login page routing and display logic for the OpenFrame multi-tenant authorization system. Manages login form presentation with error and logout message handling.

## Key Components

- **LoginController**: Spring MVC controller class
- **login()**: GET endpoint for `/login` that handles authentication feedback messages
- **index()**: GET endpoint for root path `/` that displays the main landing page

## Usage Example

```java
// The controller automatically handles these URL patterns:

// Basic login page access
// GET /login -> returns "login" view

// Login with error parameter (after failed authentication)
// GET /login?error -> adds "Invalid credentials" message to model

// Login with logout parameter (after successful logout)  
// GET /login?logout -> adds "Logged out successfully" message to model

// Root page access
// GET / -> returns "index" view with welcome message

// Example of accessing error messages in a Thymeleaf template:
// <div th:if="${errorMessage}" th:text="${errorMessage}"></div>
// <div th:if="${logoutMessage}" th:text="${logoutMessage}"></div>
```

The controller integrates with Spring Security's default authentication flow, automatically receiving error and logout parameters from the security framework.
