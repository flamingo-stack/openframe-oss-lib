REST controller that handles tenant discovery requests for multi-tenant authentication flows. It provides an endpoint to identify available tenants and authentication providers based on a user's email address.

## Key Components

- **TenantDiscoveryController** - Main REST controller class
- **discoverTenants()** - GET endpoint for tenant discovery by email
- **TenantDiscoveryService** - Injected service handling business logic
- **TenantDiscoveryResponse** - DTO containing discovery results

## Usage Example

```java
// GET request to discover tenants for a user
// Endpoint: GET /tenant/discover?email=user@company.com

@Autowired
private TenantDiscoveryService tenantDiscoveryService;

// The controller validates email format and delegates to service
public TenantDiscoveryResponse discoverTenants(@RequestParam @Email @NotBlank String email) {
    // Email is automatically converted to lowercase
    return tenantDiscoveryService.discoverTenantForEmail(email.toLowerCase(Locale.ROOT));
}

// Example client usage:
// curl -X GET "https://api.example.com/tenant/discover?email=john.doe@acme.com"
// Response: TenantDiscoveryResponse with available tenants and auth providers
```

The endpoint validates email format, converts to lowercase, and returns tenant information needed for the authentication flow. Primarily used during login to determine which tenant and authentication method a user should use.
