This Spring Boot REST controller handles tenant discovery functionality for multi-tenant authentication systems, allowing clients to identify which tenants and authentication providers are available for a given email address.

## Key Components

- **TenantDiscoveryController**: Main REST controller class with `/tenant` base mapping
- **discoverTenants()**: GET endpoint at `/discover` that accepts an email parameter and returns tenant discovery information
- **TenantDiscoveryService**: Injected service that performs the actual tenant lookup logic
- **TenantDiscoveryResponse**: DTO containing the discovery results

## Usage Example

```java
// Client request to discover tenants
GET /tenant/discover?email=user@company.com

// Controller processes request
@RestController
public class ClientExample {
    
    @Autowired
    private RestTemplate restTemplate;
    
    public TenantDiscoveryResponse findUserTenants(String email) {
        String url = "/tenant/discover?email=" + email;
        return restTemplate.getForObject(url, TenantDiscoveryResponse.class);
    }
}
```

The endpoint validates the email format using `@Email` and `@NotBlank` annotations, normalizes it to lowercase, and delegates to the service layer for tenant discovery. This is typically used in login flows where users need to be directed to their appropriate tenant and authentication provider.
