A REST controller that handles user invitation acceptance workflows, supporting both direct registration and SSO-based invitation flows.

## Key Components

- **`register()`** - POST endpoint that accepts invitation registration requests and creates new authenticated users
- **`acceptViaSso()`** - GET endpoint that initiates SSO-based invitation acceptance, setting secure cookies and redirecting to SSO provider
- **Dependencies**:
  - `InvitationRegistrationService` - Handles direct invitation registration
  - `SsoInvitationService` - Manages SSO invitation workflows

## Usage Example

```java
// Direct invitation acceptance (POST /invitations/accept)
InvitationRegistrationRequest request = InvitationRegistrationRequest.builder()
    .invitationToken("abc123")
    .email("user@example.com")
    .password("securePassword")
    .build();

AuthUser newUser = restTemplate.postForObject(
    "/invitations/accept", 
    request, 
    AuthUser.class
);

// SSO invitation acceptance (GET /invitations/accept/sso)
// Redirects to: /invitations/accept/sso?token=xyz789&tenantId=tenant123
// Sets secure HMAC cookie and redirects to SSO provider
```

The controller provides two distinct invitation acceptance paths - direct registration for users creating accounts with credentials, and SSO integration for users authenticating through external identity providers.
