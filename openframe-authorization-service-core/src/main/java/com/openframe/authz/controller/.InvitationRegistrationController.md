Handles HTTP endpoints for accepting tenant invitations via standard credentials or SSO-based flows within the OpenFrame authorization service.

## Key Components

| Element | Description |
|---|---|
| `POST /invitations/accept` | Accepts a JSON invitation request and registers the user via `InvitationRegistrationService` |
| `GET /invitations/accept/sso` | Initiates an SSO-based invitation acceptance, sets a secure HTTP-only cookie, and redirects to the SSO provider |
| `InvitationRegistrationService` | Handles credential-based invitation registration |
| `SsoInvitationService` | Orchestrates the SSO invitation acceptance flow and returns redirect/cookie data |
| `SsoRegistrationConstants.COOKIE_SSO_INVITE` | Cookie name used to persist SSO invite state across the redirect |
| `authErrorUrl` | Configurable fallback redirect target on SSO failure, populated from `openframe.auth.error-url` |

## Usage Example

```java
// Standard invitation acceptance (POST /invitations/accept)
InvitationRegistrationRequest request = new InvitationRegistrationRequest();
request.setToken("invite-token-abc123");
request.setPassword("SecurePass1!");

// Returns AuthUser on 200 OK
AuthUser user = restTemplate.postForObject("/invitations/accept", request, AuthUser.class);

// SSO invitation acceptance (GET /invitations/accept/sso)
// Client redirects browser to:
// GET /invitations/accept/sso?token=invite-token-abc123&tenantId=tenant-xyz
//
// Controller will:
// 1. Clear existing auth state cookies
// 2. Start the SSO accept flow via SsoInvitationService
// 3. Set COOKIE_SSO_INVITE as a secure, HttpOnly cookie
// 4. Redirect (303 See Other) to the SSO provider
// On failure: redirects to authErrorUrl?error=<encoded-message>
```

> **Note:** The SSO flow relies on a stateful cookie (`COOKIE_SSO_INVITE`) to carry invite context through the external provider redirect. Ensure `openframe.auth.error-url` is configured in your application properties so failures redirect gracefully.
