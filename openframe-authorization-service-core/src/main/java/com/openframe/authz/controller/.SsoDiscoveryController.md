REST controller for discovering available SSO providers based on context (invitation or registration). Supports different provider sets depending on whether a user is accepting an invitation or registering independently.

## Key Components

- **`providersForInvitation`** - Returns SSO providers available for a specific tenant based on invitation ID
- **`providersForRegistration`** - Returns default SSO providers for new user registration
- **`ProvidersResponse`** - Record containing the list of provider names
- **Dependencies**: `InvitationValidator` for invitation validation, `SSOConfigService` for provider configuration

## Usage Example

```java
// Get providers for invitation acceptance
GET /sso/providers/invite?invitationId=abc123
// Response: {"providers": ["google", "microsoft", "okta"]}

// Get providers for registration
GET /sso/providers/registration
// Response: {"providers": ["google", "microsoft"]}

// Using the response in client code
ProvidersResponse response = ssoDiscoveryController.providersForInvitation("abc123");
List<String> availableProviders = response.providers();
```

The controller automatically determines the appropriate SSO providers based on context - tenant-specific providers for invitations or system defaults for registration, enabling flexible SSO configuration per tenant while maintaining fallback options.
