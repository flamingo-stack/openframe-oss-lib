REST controller for discovering available Single Sign-On (SSO) providers based on invitation context or registration scenarios. Handles tenant-specific and system-default SSO provider lookups.

## Key Components

- **`SsoDiscoveryController`** - Main REST controller handling SSO provider discovery
- **`providersForInvitation()`** - Returns SSO providers available for a specific invitation/tenant
- **`providersForRegistration()`** - Returns default SSO providers for new user registration
- **`ProvidersResponse`** - Record class for JSON response containing provider list
- **`InvitationValidator`** - Service for validating and loading invitation data
- **`SSOConfigService`** - Service for retrieving SSO configuration and provider lists

## Usage Example

```java
// Example API calls this controller handles:

// Get SSO providers for accepting an invitation
// GET /sso/providers/invite?invitationId=abc123
// Response: {"providers": ["google", "okta", "azure"]}

// Get default SSO providers for registration
// GET /sso/providers/registration
// Response: {"providers": ["google", "github"]}

// Using the controller programmatically
@Autowired
private SsoDiscoveryController ssoController;

public void handleInvitation(String invitationId) {
    ProvidersResponse response = ssoController.providersForInvitation(invitationId);
    List<String> availableProviders = response.providers();
    // Use provider list to render SSO options
}
```

The controller supports two discovery scenarios: invitation-based lookup (tenant-specific providers) and registration lookup (system defaults). Both endpoints return JSON with a list of available SSO provider identifiers.
