REST controller that handles password reset functionality, providing endpoints for requesting password reset tokens and confirming password changes.

## Key Components

- **PasswordResetController** - Main REST controller class handling password reset operations
- **requestReset()** - POST endpoint (`/password-reset/request`) that initiates password reset by email
- **confirmReset()** - POST endpoint (`/password-reset/confirm`) that completes password reset with token
- **PasswordResetService** - Injected service handling the business logic for password reset operations

## Usage Example

```java
// Request password reset (POST /password-reset/request)
{
    "email": "user@example.com"
}
// Returns HTTP 202 ACCEPTED

// Confirm password reset (POST /password-reset/confirm)
{
    "token": "abc123-reset-token",
    "newPassword": "newSecurePassword123"
}
// Returns HTTP 204 NO CONTENT
```

The controller follows REST best practices with proper HTTP status codes (202 for async operations, 204 for successful completion without response body) and JSON content negotiation. Email addresses are automatically normalized to lowercase for consistent processing. Both endpoints use validated DTOs to ensure proper request structure and data integrity.
