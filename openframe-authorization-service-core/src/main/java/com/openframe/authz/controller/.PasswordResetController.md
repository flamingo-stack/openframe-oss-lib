REST controller that handles password reset functionality through a two-step process: requesting a reset token and confirming the password change.

## Key Components

- **`requestReset()`** - POST endpoint that initiates password reset by generating a token for the provided email address
- **`confirmReset()`** - POST endpoint that validates the reset token and updates the user's password
- **`PasswordResetService`** - Injected service that handles the business logic for token creation and password updates
- **DTO Classes** - `ResetRequest` and `ResetConfirm` for request validation

## Usage Example

```java
// Request password reset
POST /password-reset/request
Content-Type: application/json

{
    "email": "user@example.com"
}

// Response: 202 Accepted

// Confirm password reset with token
POST /password-reset/confirm
Content-Type: application/json

{
    "token": "abc123-reset-token",
    "newPassword": "newSecurePassword123"
}

// Response: 204 No Content
```

The controller follows REST conventions with appropriate HTTP status codes (202 for async token generation, 204 for successful password update) and uses Jakarta validation for input sanitization.
