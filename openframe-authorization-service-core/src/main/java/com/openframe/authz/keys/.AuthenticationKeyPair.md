Immutable value object (Java record) that holds a generated RSA key pair along with its PEM-encoded representations and a key ID (`kid`), used for JWT signing and verification in the OpenFrame authorization layer.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `publicKey` | `RSAPublicKey` | Raw RSA public key object |
| `privateKey` | `RSAPrivateKey` | Raw RSA private key object |
| `publicPem` | `String` | PEM-encoded public key string |
| `privatePem` | `String` | PEM-encoded private key string |
| `kid` | `String` | Key ID used to identify this key pair in a JWKS |

## Usage Example

```java
// Typically constructed by a key generation service
AuthenticationKeyPair keyPair = new AuthenticationKeyPair(
    rsaPublicKey,
    rsaPrivateKey,
    "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----",
    "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----",
    "key-id-2024"
);

// Access individual components
RSAPublicKey publicKey = keyPair.publicKey();
String kid             = keyPair.kid();
String publicPem       = keyPair.publicPem();
```

> As a Java `record`, all fields are final and immutable. Accessor methods are auto-generated matching each field name (e.g., `keyPair.publicKey()`, `keyPair.kid()`). No setters are available by design, ensuring key material integrity throughout the authorization lifecycle.
