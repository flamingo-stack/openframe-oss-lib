Generates RSA public/private key pairs for authentication purposes, implementing the `AuthenticationKeyPairGenerator` interface using configurable cryptographic parameters.

## Key Components

| Element | Description |
|---|---|
| `generate()` | Creates a new RSA key pair using a `SecureRandom` instance and returns an `AuthenticationKeyPair` |
| `KeyGeneratorProperties` | Injected configuration supplying algorithm name, secure random algorithm, and key size |
| `PemUtil` | Utility used to serialize RSA keys into PEM-encoded strings |
| `KID_PREFIX` | Constant prefix (`kid-`) prepended to a UUID to form a unique key ID |
| `AuthenticationKeyPair` | Return type wrapping the public/private key objects, their PEM representations, and the generated `kid` |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class TokenIssuanceService {

    private final AuthenticationKeyPairGenerator keyPairGenerator;

    public void rotateSigningKey() {
        AuthenticationKeyPair keyPair = keyPairGenerator.generate();

        String kid        = keyPair.getKid();        // e.g. "kid-550e8400-e29b-..."
        String publicPem  = keyPair.getPublicPem();  // PEM-encoded RSA public key
        String privatePem = keyPair.getPrivatePem(); // PEM-encoded RSA private key

        // Store or register keys for JWT signing/verification
    }
}
```

> **Configuration note:** Key generation behaviour is driven by `KeyGeneratorProperties`. Ensure the `algorithm` (e.g. `RSA`), `secureRandomAlgorithm` (e.g. `SHA1PRNG`), and `keySize` (e.g. `2048` or `4096`) are correctly set in your application properties before the bean is used.
