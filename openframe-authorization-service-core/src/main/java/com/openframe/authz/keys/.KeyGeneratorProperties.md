Configuration properties class for the OpenFrame authorization key generator, binding `application.properties`/`application.yml` values under the `openframe.authz.key-generator` prefix.

## Key Components

| Property | Default | Description |
|----------|---------|-------------|
| `algorithm` | `RSA` | Cryptographic algorithm used for key generation |
| `keySize` | `2048` | Key size in bits |
| `secureRandomAlgorithm` | `NativePRNGNonBlocking` | `SecureRandom` algorithm for entropy sourcing |

## Usage Example

Override defaults in `application.yml`:

```yaml
openframe:
  authz:
    key-generator:
      algorithm: RSA
      key-size: 4096
      secure-random-algorithm: NativePRNGNonBlocking
```

Inject into a Spring component:

```java
@Service
@RequiredArgsConstructor
public class KeyGeneratorService {

    private final KeyGeneratorProperties properties;

    public KeyPair generateKeyPair() throws NoSuchAlgorithmException {
        KeyPairGenerator generator = KeyPairGenerator
            .getInstance(properties.getAlgorithm());

        SecureRandom secureRandom = SecureRandom
            .getInstance(properties.getSecureRandomAlgorithm());

        generator.initialize(properties.getKeySize(), secureRandom);
        return generator.generateKeyPair();
    }
}
```

> `NativePRNGNonBlocking` is preferred in containerized environments to avoid entropy starvation (`/dev/urandom` backed). Switch to `SHA1PRNG` only if the target OS does not expose a native PRNG.
