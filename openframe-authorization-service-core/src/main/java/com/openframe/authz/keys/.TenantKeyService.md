Manages RSA key pairs for tenant-specific JWT signing operations, providing secure key generation, storage, and retrieval with automatic encryption of private keys.

## Key Components

- **getOrCreateActiveKey(String tenantId)**: Retrieves the active RSA signing key for a tenant, or creates a new one if none exists. Returns a `RSAKey` object suitable for JWT operations.
- **createAndStore(String tenantId)**: Private method that generates a new RSA key pair, encrypts the private key, and stores it in the database.

## Usage Example

```java
@Autowired
private TenantKeyService tenantKeyService;

// Get or create an active signing key for a tenant
public String signTokenForTenant(String tenantId, JWTClaimsSet claims) {
    RSAKey signingKey = tenantKeyService.getOrCreateActiveKey(tenantId);
    
    // Use the key for JWT signing
    JWSSigner signer = new RSASSASigner(signingKey);
    SignedJWT signedJWT = new SignedJWT(
        new JWSHeader.Builder(JWSAlgorithm.RS256)
            .keyID(signingKey.getKeyID())
            .build(),
        claims
    );
    
    signedJWT.sign(signer);
    return signedJWT.serialize();
}
```

The service handles key lifecycle management, warns about multiple active keys to prevent JWT verification issues, and ensures secure storage through encryption of private key material.
