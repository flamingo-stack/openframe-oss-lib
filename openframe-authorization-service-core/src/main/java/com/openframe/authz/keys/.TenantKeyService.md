Manages RSA signing key lifecycle for tenants in the OpenFrame authorization service, providing retrieval or on-demand creation of active tenant-scoped JWK keys with encrypted private key storage.

## Key Components

| Component | Description |
|-----------|-------------|
| `getOrCreateActiveKey(String tenantId)` | Retrieves the active RSA signing key for a tenant, or generates and persists a new one if none exists. Warns on multiple active keys to prevent `kid` mismatches. |
| `createAndStore(String tenantId)` | Generates a new RSA key pair, encrypts the private key via `EncryptionService`, and persists the `TenantKey` document to the repository. |
| `TenantKeyRepository` | MongoDB/document store repository for querying and persisting `TenantKey` records by tenant and active status. |
| `EncryptionService` | Handles symmetric encryption/decryption of the RSA private key PEM before storage. |
| `AuthenticationKeyPairGenerator` | Generates a new RSA key pair (`AuthenticationKeyPair`) including public PEM, private PEM, and a `kid`. |
| `PemUtil` | Parses PEM-encoded strings into `RSAPublicKey` / `RSAPrivateKey` instances. |

## Usage Example

```java
// Injected via Spring IoC
@Autowired
private TenantKeyService tenantKeyService;

// Retrieve or auto-create the active RSA JWK for a tenant
RSAKey jwk = tenantKeyService.getOrCreateActiveKey("tenant-abc-123");

// Use the JWK for token signing
RSASSASigner signer = new RSASSASigner(jwk);
JWSHeader header = new JWSHeader.Builder(JWSAlgorithm.RS256)
    .keyID(jwk.getKeyID())
    .build();
```

## Notes

- Only one active key per tenant is expected; multiple active keys trigger a `WARN` log indicating a potential misconfiguration.
- Private keys are **never stored in plaintext** — they are encrypted via `EncryptionService` before persistence and decrypted only at runtime during JWK construction.
- Key IDs (`kid`) are propagated into the built `RSAKey` to support JWKS endpoint resolution and token verification flows.
