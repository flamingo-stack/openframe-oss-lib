Utility class for managing authentication state and session cleanup in web applications, providing methods to invalidate sessions and clear authentication-related cookies.

## Key Components

- **`clearAuthState(request, response)`** - Completely clears authentication state by invalidating the session and removing JSESSIONID cookies
- **`clearCookie(response, name, path)`** - Clears a specific cookie by setting it to expire immediately with secure flags
- **`clearCookie(response, name)`** - Convenience method to clear a cookie with default root path
- **`JSESSIONID`** - Constant for the standard Java session ID cookie name

## Usage Example

```java
// Clear all authentication state during logout
@PostMapping("/logout")
public ResponseEntity<String> logout(HttpServletRequest request, HttpServletResponse response) {
    AuthStateUtils.clearAuthState(request, response);
    return ResponseEntity.ok("Logged out successfully");
}

// Clear specific cookies manually
@PostMapping("/clear-session")
public void clearSession(HttpServletResponse response) {
    AuthStateUtils.clearCookie(response, "CUSTOM_AUTH_TOKEN");
    AuthStateUtils.clearCookie(response, "USER_PREFS", "/app");
}
```

The utility ensures secure cookie clearing by setting `HttpOnly` and `Secure` flags, and handles both root path and context-specific cookie removal for comprehensive session cleanup.
