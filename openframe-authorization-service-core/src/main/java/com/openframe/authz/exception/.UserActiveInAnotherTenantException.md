A custom runtime exception thrown when attempting to perform operations on a user who is already active in a different tenant context.

## Key Components

- **UserActiveInAnotherTenantException** - Runtime exception class that extends `RuntimeException`
- **Constructor** - Accepts user email and generates a descriptive error message

## Usage Example

```java
// Throw exception when user conflict is detected
public void activateUserInTenant(String email, String tenantId) {
    if (isUserActiveInDifferentTenant(email, tenantId)) {
        throw new UserActiveInAnotherTenantException(email);
    }
    // Proceed with activation
}

// Catch and handle the exception
try {
    userService.activateUserInTenant("user@example.com", "tenant-123");
} catch (UserActiveInAnotherTenantException e) {
    logger.error("Tenant conflict: {}", e.getMessage());
    // Handle multi-tenant user conflict
}
```

This exception helps enforce tenant isolation by preventing users from being simultaneously active across multiple tenants in a multi-tenant application architecture.
