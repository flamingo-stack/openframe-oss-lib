A specialized exception thrown when a user attempts to authenticate or register in a tenant while already being active in a different tenant.

## Key Components

- **`UserActiveInAnotherTenantException`** — Extends `ConflictException`, mapping to HTTP 409 Conflict responses.
- **`ErrorCode.USER_ACTIVE_IN_ANOTHER_TENANT`** — Structured error code providing machine-readable conflict identification.
- **Constructor `(String email)`** — Accepts the conflicting user's email to include in the exception message for debugging and logging clarity.

## Usage Example

```java
public void registerUserToTenant(String email, String tenantId) {
    if (userService.isActiveInAnotherTenant(email, tenantId)) {
        throw new UserActiveInAnotherTenantException(email);
    }
    // proceed with registration
}
```

The thrown exception propagates as a `ConflictException`, typically serialized by a global exception handler into a structured error response:

```json
{
  "errorCode": "USER_ACTIVE_IN_ANOTHER_TENANT",
  "message": "User is active in another tenant: john.doe@example.com"
}
```

> This exception is part of OpenFrame's multi-tenant authorization layer (`com.openframe.authz`), enforcing tenant isolation by preventing a single user identity from being simultaneously active across multiple tenants.
