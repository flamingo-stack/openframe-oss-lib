A centralized exception handler that intercepts and transforms various exceptions into standardized HTTP error responses for the OpenFrame authorization service.

## Key Components

- **@RestControllerAdvice**: Global exception handling across all REST controllers
- **ValidationException Handlers**: Process `ConstraintViolationException` and `MethodArgumentNotValidException` with detailed field-level error messages
- **HTTP Status Mapping**: Maps specific exceptions to appropriate HTTP status codes (400, 409, 500)
- **Custom Exception Handler**: Handles domain-specific `UserActiveInAnotherTenantException`
- **Generic Fallback**: Catches all unhandled exceptions as internal server errors
- **Structured Logging**: Logs exceptions at appropriate levels (warn/error) with contextual information

## Usage Example

The handler automatically processes exceptions thrown by any controller method:

```java
@RestController
public class UserController {
    
    @PostMapping("/users")
    public User createUser(@Valid @RequestBody CreateUserRequest request) {
        // If validation fails, MethodArgumentNotValidException is thrown
        // and handled by GlobalExceptionHandler, returning:
        // {
        //   "code": "VALIDATION_ERROR",
        //   "message": "email: must be a well-formed email address"
        // }
        return userService.createUser(request);
    }
    
    @PostMapping("/users/{userId}/activate")
    public void activateUser(@PathVariable String userId) {
        // If user is active in another tenant, custom exception is thrown
        // and returns HTTP 409 with structured error response
        userService.activateUser(userId);
    }
}
```
