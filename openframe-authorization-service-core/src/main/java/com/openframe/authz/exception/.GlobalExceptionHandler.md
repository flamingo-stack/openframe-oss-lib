A centralized exception handler for the OpenFrame authorization service that provides consistent error responses and logging across all endpoints using Spring's `@RestControllerAdvice`.

## Key Components

- **@RestControllerAdvice**: Global exception handling annotation that intercepts exceptions from all controllers
- **Exception Handlers**: Methods for specific exception types including validation errors, user conflicts, and system errors
- **ErrorResponse**: Standardized error format with error codes and messages
- **Logging**: Structured logging with SLF4J for error tracking and debugging
- **HTTP Status Mapping**: Automatic status code assignment based on exception type

## Usage Example

```java
// The handler automatically catches exceptions from controllers
@RestController
public class UserController {
    
    @PostMapping("/users")
    public User createUser(@Valid @RequestBody CreateUserRequest request) {
        // If validation fails, GlobalExceptionHandler catches MethodArgumentNotValidException
        // Returns: {"code": "VALIDATION_ERROR", "message": "field: error details"}
        
        if (userExists(request.getEmail())) {
            // Handler catches IllegalStateException 
            // Returns 409 CONFLICT with {"code": "CONFLICT", "message": "User already exists"}
            throw new IllegalStateException("User already exists");
        }
        
        return userService.create(request);
    }
    
    @GetMapping("/users/{id}")
    public User getUser(@PathVariable Long id) {
        // Handler catches UserNotFoundException
        // Returns 404 NOT_FOUND with {"code": "USER_NOT_FOUND", "message": "User not found"}
        return userService.findById(id);
    }
}
```

The handler ensures all API errors follow a consistent format with appropriate HTTP status codes and structured error messages for better client integration.
