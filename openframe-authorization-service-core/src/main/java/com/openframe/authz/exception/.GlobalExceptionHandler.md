Centralized exception handling for the OpenFrame authorization module that converts various exceptions into standardized HTTP error responses. Uses Spring's `@RestControllerAdvice` to provide consistent error handling across all REST endpoints.

## Key Components

- **`handleConstraintViolation()`** - Handles Bean Validation constraint violations (400 Bad Request)
- **`handleIllegalArgument()`** - Handles illegal argument exceptions (400 Bad Request)
- **`handleIllegalState()`** - Handles illegal state exceptions (409 Conflict)
- **`handleMissingParam()`** - Handles missing request parameters (400 Bad Request)
- **`handleValidation()`** - Handles method argument validation errors (400 Bad Request)
- **`handleUserActiveInAnotherTenant()`** - Handles custom tenant conflict exceptions (409 Conflict)
- **`handleException()`** - Catches all unhandled exceptions (500 Internal Server Error)

## Usage Example

```java
// The handler automatically catches exceptions from any REST controller
@RestController
public class UserController {
    
    @PostMapping("/users")
    public User createUser(@Valid @RequestBody CreateUserRequest request) {
        // If validation fails, handleValidation() is automatically called
        // Returns: ErrorResponse("VALIDATION_ERROR", "field errors...")
        return userService.createUser(request);
    }
    
    @GetMapping("/users/{id}")
    public User getUser(@PathVariable String id) {
        // If IllegalArgumentException is thrown, handleIllegalArgument() responds
        // Returns: ErrorResponse("BAD_REQUEST", "Invalid user ID format")
        return userService.findById(id);
    }
}
```
