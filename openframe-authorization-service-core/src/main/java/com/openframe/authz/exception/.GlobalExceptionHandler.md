Centralized REST exception handler for the `authz` service, mapping Spring MVC exceptions to structured `ErrorResponse` objects.

## Key Components

| Element | Description |
|---|---|
| `@RestControllerAdvice` | Intercepts exceptions thrown across all REST controllers |
| `handleNoResourceFound` | Catches `NoResourceFoundException` and returns a `404 NOT_FOUND` response |
| `ErrorResponse.of(...)` | Wraps the error using `ErrorCode.NOT_FOUND` from the core module |

## Usage Example

No direct invocation required — Spring auto-applies this handler. When a controller route is not found, the handler responds automatically:

```java
// Triggered automatically when a request hits an unmapped endpoint
// e.g. GET /api/unknown-path

// Response: HTTP 404
// Body:
{
  "code": "NOT_FOUND",
  "message": "No static resource api/unknown-path."
}
```

To extend the handler with additional exception mappings:

```java
@ExceptionHandler(AccessDeniedException.class)
@ResponseStatus(HttpStatus.FORBIDDEN)
public ErrorResponse handleAccessDenied(AccessDeniedException ex) {
    log.warn("Access denied: {}", ex.getMessage());
    return ErrorResponse.of(ErrorCode.FORBIDDEN, ex.getMessage());
}
```

> **Note:** Debug-level logging is used for `NoResourceFoundException` to avoid log noise from routine 404s. Use `log.warn` or `log.error` for higher-severity exceptions added in future handlers.
