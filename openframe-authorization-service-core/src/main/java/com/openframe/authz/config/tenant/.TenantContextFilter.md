A servlet filter that extracts tenant information from HTTP requests and manages multi-tenant context throughout the request lifecycle.

## Key Components

- **TenantContextFilter**: Main filter class extending `OncePerRequestFilter` with highest precedence
- **TENANT_ID**: Static constant for session attribute key
- **EXCLUDED_CONTEXTS**: Set of paths excluded from tenant extraction (login, sso, public, etc.)
- **doFilterInternal()**: Core method that extracts tenant from URL path, query parameters, or session
- **getString()**: Helper method to normalize request URI by removing context path

## Usage Example

```java
// Tenant extraction from URL path
// Request: /tenant123/oauth2/authorize
// Extracted tenant: "tenant123"

// Fallback to query parameter
// Request: /login?tenant=acme
// Extracted tenant: "acme"

// Access tenant context in other components
@Service
public class MyService {
    public void processRequest() {
        String currentTenant = TenantContext.getTenantId();
        // Process with tenant-specific logic
    }
}

// Configuration in Spring Boot
@Configuration
public class FilterConfig {
    // Filter is automatically registered as @Component
    // Order ensures it runs before other security filters
}
```

The filter automatically invalidates sessions when tenant context changes and ensures proper cleanup via `TenantContext.clear()` in the finally block.
