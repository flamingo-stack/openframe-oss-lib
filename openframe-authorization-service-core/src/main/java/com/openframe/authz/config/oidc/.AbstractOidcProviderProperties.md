Abstract base class for OIDC provider configuration properties, providing common OAuth 2.0/OpenID Connect endpoints and credential management with optional tenant-specific URL resolution.

## Key Components

- **Configuration Properties**: Core OIDC endpoints (`authorizationUrl`, `tokenUrl`, `userInfoUrl`, `jwkSetUri`, `issuerUri`)
- **Credential Management**: Default client credentials (`defaultClientId`, `defaultClientSecret`) with fallback support
- **Redirect URIs**: Registration and login redirect endpoints for OAuth flows
- **Tenant Resolution**: Abstract methods for tenant-specific URL resolution (currently pass-through implementations)
- **Scope Configuration**: List of OAuth 2.0 scopes to request during authentication

## Usage Example

```java
@Component
@ConfigurationProperties(prefix = "app.oidc.azure")
public class AzureOidcProperties extends AbstractOidcProviderProperties {
    
    @Override
    public String effectiveAuthorizationUrl(String msTenantId) {
        // Override for tenant-specific URLs
        return getAuthorizationUrl().replace("{tenant}", msTenantId);
    }
}

// Configuration usage
@Autowired
private AzureOidcProperties oidcProperties;

public void configureOAuth() {
    String clientId = oidcProperties.getDefaultClientId();
    List<String> scopes = oidcProperties.getScopes();
    String authUrl = oidcProperties.effectiveAuthorizationUrl("tenant-123");
}
```

The class serves as a foundation for provider-specific OIDC configurations, allowing subclasses to implement custom tenant resolution logic while maintaining consistent property structure across different identity providers.
