A thread-safe utility class for managing tenant context in multi-tenant applications using ThreadLocal storage. This ensures tenant isolation across concurrent requests without requiring explicit parameter passing.

## Key Components

- **setTenantId(String tenantId)** - Sets the current tenant ID for the executing thread
- **getTenantId()** - Retrieves the tenant ID associated with the current thread
- **clear()** - Removes the tenant context from the current thread (important for thread pool cleanup)
- **CURRENT_TENANT** - ThreadLocal variable that stores tenant ID per thread

## Usage Example

```java
// Set tenant context at request start
TenantContext.setTenantId("tenant-123");

// Access tenant ID anywhere in the same thread
String currentTenant = TenantContext.getTenantId();
System.out.println("Processing request for: " + currentTenant);

// Clear context when request completes (crucial for thread pools)
try {
    // Process tenant-specific logic
    performTenantOperation();
} finally {
    TenantContext.clear(); // Prevents memory leaks
}
```

**Note:** Always call `clear()` in a finally block or use try-with-resources pattern to prevent memory leaks in application servers that reuse threads.
