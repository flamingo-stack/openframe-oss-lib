A thread-safe utility class that manages tenant context information using ThreadLocal storage for multi-tenant applications. This enables tenant isolation by storing the current tenant ID at the thread level.

## Key Components

- **setTenantId(String tenantId)** - Sets the tenant ID for the current thread
- **getTenantId()** - Retrieves the tenant ID for the current thread
- **clear()** - Removes the tenant ID from the current thread to prevent memory leaks

## Usage Example

```java
// Set tenant context at the beginning of request processing
TenantContext.setTenantId("tenant-123");

// Retrieve tenant ID anywhere in the same thread
String currentTenant = TenantContext.getTenantId();
System.out.println("Current tenant: " + currentTenant);

// Clear context when done (important for thread pool environments)
try {
    // Process tenant-specific logic
    processRequest();
} finally {
    TenantContext.clear(); // Prevents memory leaks
}
```

**Important:** Always call `clear()` in a finally block or use try-with-resources pattern to prevent memory leaks in thread pool environments where threads are reused across different tenant requests.
