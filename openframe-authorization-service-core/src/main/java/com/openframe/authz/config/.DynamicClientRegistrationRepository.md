A Spring Security OAuth2 client registration repository that dynamically loads client configurations based on tenant context, enabling multi-tenant OAuth2 authentication.

## Key Components

- **`DynamicClientRegistrationRepository`** - Main component implementing `ClientRegistrationRepository` for tenant-aware client registration lookup
- **`findByRegistrationId(String)`** - Resolves OAuth2 client registrations dynamically using tenant context
- **`resolveTenantId()`** - Multi-source tenant ID resolution from context or HTTP session
- **`DynamicClientRegistrationService`** - Injected service handling the actual client configuration loading

## Usage Example

```java
@Configuration
@EnableWebSecurity
public class OAuth2Config {
    
    @Autowired
    private DynamicClientRegistrationRepository clientRepo;
    
    @Bean
    public OAuth2AuthorizedClientService authorizedClientService() {
        return new InMemoryOAuth2AuthorizedClientService(clientRepo);
    }
    
    // The repository automatically resolves tenant-specific clients
    // For tenant "acme", registration "google" loads acme-specific Google OAuth config
    // For tenant "corp", same "google" registration loads corp-specific config
}

// Example client resolution flow:
// 1. OAuth2 login request for "google" provider
// 2. Repository resolves tenant ID from TenantContext or session
// 3. Calls dynamic.loadClient("google", "tenant-123")  
// 4. Returns tenant-specific ClientRegistration or null if not found
```

The repository provides graceful fallback behavior, returning `null` for missing tenant context or failed client resolution, allowing the OAuth2 flow to handle unavailable configurations appropriately.
