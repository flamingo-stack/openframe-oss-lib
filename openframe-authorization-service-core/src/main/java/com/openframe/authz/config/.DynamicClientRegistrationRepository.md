A Spring Security component that dynamically resolves OAuth2 client registrations based on tenant context, enabling multi-tenant authentication workflows.

## Key Components

- **`DynamicClientRegistrationRepository`** - Main repository implementing `ClientRegistrationRepository`
- **`findByRegistrationId(String)`** - Resolves client registration by ID using tenant context
- **`resolveTenantId()`** - Extracts tenant ID from `TenantContext` or HTTP session
- **`DynamicClientRegistrationService`** - Injected service for loading tenant-specific clients

## Usage Example

```java
@Autowired
private DynamicClientRegistrationRepository clientRepo;

// The repository automatically resolves tenant context
ClientRegistration registration = clientRepo.findByRegistrationId("google");

// Tenant ID resolution order:
// 1. TenantContext.getTenantId()
// 2. HTTP session attribute "TENANT_ID"
// 3. Returns null if no tenant context found

// Set tenant context before resolution
TenantContext.setTenantId("tenant123");
ClientRegistration tenantSpecific = clientRepo.findByRegistrationId("oauth-provider");
```

The repository integrates with Spring Security's OAuth2 client framework, automatically handling tenant-aware client registration lookup during authentication flows. Returns `null` when tenant context is unavailable or client resolution fails.
