Configuration class that sets up Spring Security for OAuth2/OIDC authentication with multi-tenant support, auto-provisioning capabilities, and Microsoft Azure AD integration.

## Key Components

- **defaultSecurityFilterChain**: Main security filter configuration with OAuth2 login and endpoint permissions
- **oidcUserService**: Custom OIDC user service that handles auto-provisioning and user mapping
- **microsoftAwareJwtDecoderFactory**: JWT decoder factory with special handling for Microsoft multi-tenant scenarios
- **SsoAuthorizationRequestResolver**: Custom authorization request resolver for SSO flows
- **SsoTenantRegistrationSuccessHandler**: Success handler for tenant registration after SSO login

## Usage Example

```java
// The configuration is automatically applied by Spring Boot
// Key endpoints that are publicly accessible:
// - /oauth2/** - OAuth2 endpoints
// - /login - Login page
// - /tenant/** - Tenant management
// - /sso/providers/** - SSO provider endpoints

// Auto-provisioning behavior:
// 1. User authenticates via OIDC provider (e.g., Microsoft, Google)
// 2. System checks tenant SSO configuration
// 3. If auto-provision is enabled and domain is allowed, creates new user
// 4. Assigns ADMIN role by default to auto-provisioned users

// Microsoft-specific features:
// - Handles multi-tenant issuer patterns
// - Custom JWT validation for Azure AD tokens
// - Support for various Microsoft claim formats
```

The configuration handles both explicit tenant SSO configurations and global domain policies for automatic user provisioning across tenants.
