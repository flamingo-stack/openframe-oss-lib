Configures Spring Security for non-Authorization Server requests, handling OAuth2/OIDC login flows, SSO provider integration, user auto-provisioning, and Microsoft multi-tenant JWT validation.

## Key Components

| Component | Description |
|---|---|
| `defaultSecurityFilterChain` | Defines HTTP security rules, permitted endpoints, form login, and OAuth2 login with custom resolvers and handlers |
| `oauth2LoginFailureHandler` | Redirects to a configurable error URL with an encoded error message on OAuth2 login failure |
| `oidcUserService` | Custom OIDC user service that auto-provisions users and resolves principal claims after delegating to the standard service |
| `microsoftAwareJwtDecoderFactory` | Builds a `JwtDecoderFactory` that applies a regex-based issuer validator for Microsoft multi-tenant tokens |
| `autoProvisionIfNeeded` | Internal helper that checks SSO config, validates allowed domains, and triggers user provisioning via `RegistrationProcessor` |

## Usage Example

```java
// The beans are auto-wired by Spring Security. No manual instantiation is needed.
// Example: customizing the permitted paths or SSO behavior via application config

# application.yml
openframe:
  auth:
    error-url: https://your-app.example.com/auth-error
```

```java
// Microsoft multi-tenant JWT validation is applied automatically
// when the ClientRegistration registrationId equals "microsoft":
@Bean
public JwtDecoderFactory<ClientRegistration> microsoftAwareJwtDecoderFactory() {
    // Validates issuer against:
    // ^https://login\.microsoftonline\.com/[^/]+/v2\.0/?$
}
```

> **Note:** CSRF and CORS are disabled in this configuration. Ensure your deployment environment provides equivalent protections at the network or gateway layer.
