Configuration class that sets up Spring Security OAuth2 Authorization Server with multi-tenant support, JWT token generation, and custom user authentication for the OpenFrame authorization service.

## Key Components

- **authorizationServerSecurityFilterChain()** - Configures OAuth2 authorization server security with OIDC support and multi-issuer capabilities
- **jwkSource()** - Provides tenant-specific JSON Web Key Sets (JWKS) for JWT token signing
- **tokenCustomizer()** - Adds custom claims (tenant_id, userId, roles) to JWT access tokens
- **userDetailsService()** - Integrates with custom UserService for tenant-aware user authentication
- **authenticationManager()** - Provides programmatic authentication support for registration flows
- **Filter beans** - Handles forwarded headers and tenant prefix routing

## Usage Example

```java
// The configuration automatically provides these beans for injection:

@Autowired
private AuthenticationManager authenticationManager;

@Autowired  
private JwtEncoder jwtEncoder;

@Autowired
private UserDetailsService userDetailsService;

// Programmatic authentication (e.g., in registration)
UsernamePasswordAuthenticationToken authRequest = 
    new UsernamePasswordAuthenticationToken(email, password);
Authentication auth = authenticationManager.authenticate(authRequest);

// JWT tokens automatically include custom claims:
// - tenant_id: Current tenant identifier
// - userId: User's database ID  
// - roles: User roles (OWNER automatically includes ADMIN)
```

This configuration enables OAuth2/OIDC flows with tenant isolation, where each tenant gets its own signing keys and user namespace, while supporting standard Spring Security authentication patterns.
