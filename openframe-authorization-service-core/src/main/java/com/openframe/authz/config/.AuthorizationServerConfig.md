Configures a multi-tenant OAuth2 Authorization Server with JWT token support, custom user authentication, and tenant-aware security filters.

## Key Components

- **authorizationServerSecurityFilterChain**: Main security filter chain for OAuth2 endpoints with multi-issuer support
- **jwkSource**: Tenant-specific JSON Web Key source for JWT signing/verification
- **jwtEncoder/jwtDecoder**: JWT encoding and decoding components
- **tokenCustomizer**: Adds custom claims (tenant_id, userId, roles) to access tokens
- **userDetailsService**: Loads user details from database with tenant context
- **authenticationManager**: Handles programmatic authentication with DAO provider
- **passwordEncoder**: BCrypt password encoder for secure hashing
- **Filter beans**: Forwarded headers and tenant prefix handling

## Usage Example

```java
// The configuration automatically provides these beans:

// JWT token will include custom claims:
{
  "tenant_id": "acme-corp",
  "userId": "user123", 
  "roles": ["USER", "ADMIN"],
  // ... standard JWT claims
}

// User authentication with tenant context:
@Autowired
private AuthenticationManager authManager;

public void authenticateUser(String email, String password) {
    Authentication auth = authManager.authenticate(
        new UsernamePasswordAuthenticationToken(email, password)
    );
}

// Keys are automatically resolved per tenant:
// GET /oauth2/jwks -> returns tenant-specific public keys
```
