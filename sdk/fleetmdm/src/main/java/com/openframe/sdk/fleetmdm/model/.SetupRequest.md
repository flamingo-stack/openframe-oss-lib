Request payload model for initializing a Fleet MDM instance, encapsulating admin credentials, organization details, and server configuration.

## Key Components

| Class/Field | Type | Description |
|---|---|---|
| `SetupRequest` | Root class | Top-level payload for Fleet MDM setup endpoint |
| `admin` | `AdminInfo` | Admin user credentials and display name |
| `orgInfo` | `OrgInfo` | Organization name (serialized as `org_info`) |
| `serverUrl` | `String` | Fleet MDM server URL (serialized as `server_url`) |
| `AdminInfo` | Static nested class | Holds `email`, `password`, and `name` fields |
| `OrgInfo` | Static nested class | Holds `orgName` (serialized as `org_name`) |

## Usage Example

```java
SetupRequest request = new SetupRequest();

// Configure admin credentials
SetupRequest.AdminInfo admin = new SetupRequest.AdminInfo();
admin.setEmail("admin@example.com");
admin.setPassword("securepassword");
admin.setName("Fleet Admin");

// Configure organization info
SetupRequest.OrgInfo orgInfo = new SetupRequest.OrgInfo();
orgInfo.setOrgName("Acme IT Services");

// Assemble the request
request.setAdmin(admin);
request.setOrgInfo(orgInfo);
request.setServerUrl("https://fleet.acme.com");
```

The resulting JSON serialization maps to:

```json
{
  "admin": {
    "email": "admin@example.com",
    "password": "securepassword",
    "name": "Fleet Admin"
  },
  "org_info": {
    "org_name": "Acme IT Services"
  },
  "server_url": "https://fleet.acme.com"
}
```

> `SetupRequest` is typically used once during Fleet MDM bootstrapping. Lombok `@Getter`/`@Setter` annotations apply only to the root class — nested classes (`AdminInfo`, `OrgInfo`) use manually defined accessors.
