Request model for updating a Fleet MDM policy, carrying the mutable fields that can be modified on an existing policy. Null fields are excluded from serialization via `@JsonInclude(NON_NULL)`, enabling partial updates.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | Display name of the policy |
| `query` | `String` | SQL query that defines the policy check |
| `description` | `String` | Human-readable description of the policy |
| `platform` | `String` | Target platform (e.g., `darwin`, `windows`, `linux`) |

**Lombok annotations:**
- `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString`
- `@NoArgsConstructor` / `@AllArgsConstructor` — generates both default and full constructors
- `@JsonInclude(NON_NULL)` — omits null fields from the serialized JSON payload

## Usage Example

```java
// Full update
UpdatePolicyRequest fullUpdate = new UpdatePolicyRequest(
    "Disk Encryption Check",
    "SELECT 1 FROM disk_encryption WHERE encrypted = 1;",
    "Verifies disk encryption is enabled",
    "darwin"
);

// Partial update — only rename the policy (null fields are omitted)
UpdatePolicyRequest partialUpdate = new UpdatePolicyRequest();
partialUpdate.setName("Renamed Policy");

fleetMdmClient.updatePolicy(policyId, partialUpdate);
```
