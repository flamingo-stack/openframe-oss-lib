A custom runtime exception class specifically designed for Fleet MDM (Mobile Device Management) operations, providing standardized error handling within the OpenFrame SDK.

## Key Components

- **FleetMdmException**: Custom runtime exception extending `RuntimeException`
- **Constructor with message**: Creates exception with descriptive error message
- **Constructor with message and cause**: Creates exception with message and underlying cause for exception chaining

## Usage Example

```java
// Throwing a simple FleetMdmException with message
public void enrollDevice(String deviceId) {
    if (deviceId == null || deviceId.isEmpty()) {
        throw new FleetMdmException("Device ID cannot be null or empty");
    }
    // enrollment logic...
}

// Throwing FleetMdmException with underlying cause
public void syncDeviceData(Device device) {
    try {
        // network operation
        apiClient.sync(device);
    } catch (NetworkException e) {
        throw new FleetMdmException("Failed to sync device data", e);
    }
}

// Catching FleetMdmException
try {
    fleetManager.enrollDevice("device123");
} catch (FleetMdmException e) {
    logger.error("Fleet MDM operation failed: " + e.getMessage(), e);
}
```
