A custom runtime exception class for handling Fleet MDM (Mobile Device Management) related errors in the OpenFrame SDK.

## Key Components

- **FleetMdmException** - Custom runtime exception that extends `RuntimeException`
- **Two constructors**:
  - `FleetMdmException(String message)` - Creates exception with error message
  - `FleetMdmException(String message, Throwable cause)` - Creates exception with message and underlying cause

## Usage Example

```java
// Throwing a FleetMdmException with just a message
if (deviceNotFound) {
    throw new FleetMdmException("Device with ID " + deviceId + " not found");
}

// Throwing with an underlying cause
try {
    // Some Fleet MDM operation
    performDeviceAction(device);
} catch (IOException e) {
    throw new FleetMdmException("Failed to communicate with device", e);
}

// Catching and handling FleetMdmException
try {
    fleetService.enrollDevice(deviceInfo);
} catch (FleetMdmException e) {
    logger.error("Fleet MDM operation failed: " + e.getMessage(), e);
    // Handle Fleet-specific error
}
```

This exception provides a way to wrap and identify Fleet MDM-specific errors, making it easier to handle device management failures separately from other application exceptions. Since it extends `RuntimeException`, it's an unchecked exception that doesn't require explicit handling in method signatures.
