A Fleet MDM compliance policy model that maps JSON API responses to a Java object, representing policy definitions including their SQL query, platform scope, authorship, team assignment, and host compliance counts.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `Long` | Unique policy identifier |
| `name` / `description` | `String` | Policy display name and details |
| `query` | `String` | osquery SQL used to evaluate compliance |
| `resolution` | `String` | Remediation instructions for failing hosts |
| `platform` | `String` | Target platform (e.g., `windows`, `darwin`, `linux`) |
| `critical` | `Boolean` | Marks the policy as high-priority |
| `authorId` / `authorName` / `authorEmail` | `Long` / `String` | Policy creator metadata |
| `teamId` / `teamName` | `Long` / `String` | Team scope assignment |
| `passingHostCount` / `failingHostCount` | `Long` | Compliance status aggregates |
| `hostsIncludeAny` | `List<AssignedHost>` | Hosts explicitly included in policy scope |

## Usage Example

```java
// Deserialize a Fleet MDM API response into a Policy object
ObjectMapper mapper = new ObjectMapper();
Policy policy = mapper.readValue(jsonResponse, Policy.class);

// Inspect compliance results
System.out.println("Policy: " + policy.getName());
System.out.println("Passing hosts: " + policy.getPassingHostCount());
System.out.println("Failing hosts: " + policy.getFailingHostCount());

// Check if policy is critical and has failing hosts
if (Boolean.TRUE.equals(policy.getCritical()) && policy.getFailingHostCount() > 0) {
    System.out.println("Critical violation - Resolution: " + policy.getResolution());
}
```

> Unknown JSON fields are silently ignored via `@JsonIgnoreProperties(ignoreUnknown = true)`, ensuring forward compatibility with Fleet MDM API updates.
