Request body model for the Fleet MDM `POST /api/v1/fleet/queries/run` endpoint, used to create a distributed live-query campaign against targeted hosts.

## Key Components

| Element | Description |
|---|---|
| `query` | Ad-hoc SQL string to execute (mutually exclusive with `queryId`) |
| `queryId` | ID of an existing saved query to run |
| `selected` | Nested object defining the target scope |
| `Selected.hostIds` | Target specific host IDs |
| `Selected.labelIds` | Target hosts matching label IDs |
| `Selected.teamIds` | Target all hosts belonging to team IDs |

> **Note:** Either `query` or `queryId` must be provided. `@JsonInclude(NON_NULL)` ensures omitted fields are excluded from the serialized payload.

## Usage Example

```java
// Run an ad-hoc query against specific hosts
RunLiveQueryRequest request = new RunLiveQueryRequest();
request.setQuery("SELECT * FROM os_version;");
request.setSelected(new RunLiveQueryRequest.Selected(
    List.of(101L, 102L),  // hostIds
    null,                  // labelIds
    null                   // teamIds
));

// Run a saved query targeting a team
RunLiveQueryRequest savedRequest = new RunLiveQueryRequest();
savedRequest.setQueryId(42L);
savedRequest.setSelected(new RunLiveQueryRequest.Selected(
    null,
    null,
    List.of(3L)  // teamIds
));
```
