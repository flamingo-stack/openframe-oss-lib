Represents a Fleet distributed live-query campaign returned by the `POST /queries/run` endpoint. This model captures the campaign metadata used to track and stream query results asynchronously over the Fleet live-query WebSocket connection.

## Key Components

| Field | Type | JSON Property | Description |
|-------|------|---------------|-------------|
| `id` | `Long` | `id` | Unique campaign identifier; used to correlate WebSocket stream results |
| `queryId` | `Long` | `query_id` | Reference to the originating query definition |
| `query` | `String` | `query` | The raw SQL query string executed across the fleet |
| `status` | `String` | `status` | Current campaign lifecycle state (e.g., `pending`, `completed`) |
| `userId` | `Long` | `user_id` | ID of the Fleet user who initiated the campaign |
| `createdAt` | `String` | `created_at` | ISO 8601 timestamp of campaign creation |

## Usage Example

```java
// Deserialize a campaign response from Fleet's POST /queries/run
ObjectMapper mapper = new ObjectMapper();
LiveQueryCampaign campaign = mapper.readValue(responseBody, LiveQueryCampaign.class);

// Use the campaign ID to subscribe to the live-query WebSocket stream
Long campaignId = campaign.getId();
System.out.println("Campaign started: " + campaignId);
System.out.println("Status: " + campaign.getStatus());

// Connect to WebSocket and filter incoming result frames by campaign ID
webSocketClient.connect(campaignId);
```

> **Note:** Unknown JSON fields are silently ignored (`@JsonIgnoreProperties(ignoreUnknown = true)`), ensuring forward compatibility as the Fleet API evolves.
