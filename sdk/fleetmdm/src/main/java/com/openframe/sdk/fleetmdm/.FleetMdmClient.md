HTTP client for interacting with the Fleet MDM REST API, supporting host management, live queries, policies, and scheduled queries.

## Key Components

| Method / Element | Description |
|---|---|
| `FleetMdmClient(String, String)` | Primary constructor — accepts base URL and API token |
| `FleetMdmClient(String, String, HttpClient)` | Package-private constructor for unit testing with a mocked `HttpClient` |
| `getHostById(long)` | Fetches a single host by its numeric ID; returns `null` on 404 |
| `searchHosts(HostSearchRequest)` | Full-featured host search with pagination, ordering, and query parameters |
| `searchHosts(String)` | Convenience overload using default pagination |
| `searchHosts(String, Integer, Integer)` | Convenience overload with explicit page and page size |
| `runQuery(long, String)` | Executes an osquery SQL statement on a specific host synchronously |
| `runQueryAsync(long, String)` | Non-blocking variant returning a `CompletableFuture<QueryResult>` |
| `MAPPER` | Shared, thread-safe `ObjectMapper` instance reused across all requests |

## Usage Example

```java
// Initialize the client
FleetMdmClient client = new FleetMdmClient(
    "https://fleet.example.com",
    "your-api-token"
);

// Fetch a host by ID
Host host = client.getHostById(42);

// Search hosts with pagination
List<Host> hosts = client.searchHosts("macbook", 0, 25);

// Run a synchronous osquery on a host
QueryResult result = client.runQuery(42, "SELECT * FROM os_version;");

// Run an async osquery
client.runQueryAsync(42, "SELECT * FROM users;")
      .thenAccept(r -> System.out.println(r.getRows()))
      .exceptionally(ex -> { ex.printStackTrace(); return null; });
```

> All methods throw `FleetMdmApiException` for HTTP-level errors (4xx/5xx) and `FleetMdmException` for client-side processing failures. Authentication errors (HTTP 401) are surfaced immediately on every call.
