A lightweight HTTP client for bootstrapping a Fleet MDM instance, handling the initial setup, admin login, and API-only user creation workflows.

## Key Components

| Method | Description |
|--------|-------------|
| `FleetMdmSetupClient(String baseUrl)` | Constructor — initializes the client with a base URL and a 5-second connect timeout |
| `setup(SetupRequest)` | Calls `POST /api/v1/setup` to initialize a new Fleet MDM instance |
| `login(LoginRequest)` | Calls `POST /api/v1/fleet/login` to authenticate and retrieve an admin token |
| `createApiOnlyUser(String adminToken, CreateUserRequest)` | Calls `POST /api/v1/fleet/users/admin` with Bearer auth to provision an API-only user |

**Private helpers:**

- `post(path, body, bearerToken)` — serializes the request body to JSON and executes the HTTP POST, optionally attaching a Bearer token
- `requireSuccess(response, operation)` — throws `FleetMdmApiException` on non-2xx responses
- `deserialize(body, type)` — deserializes the JSON response body via Jackson; throws `FleetMdmException` on parse failure

## Usage Example

```java
FleetMdmSetupClient client = new FleetMdmSetupClient("https://fleet.example.com");

// 1. Bootstrap the Fleet instance
SetupResponse setup = client.setup(new SetupRequest("admin@example.com", "password123", "MyOrg"));

// 2. Log in to obtain an admin token
LoginResponse login = client.login(new LoginRequest("admin@example.com", "password123"));
String token = login.getToken();

// 3. Create an API-only service account
CreateUserRequest userReq = new CreateUserRequest("api-service", "service@example.com", true);
CreateUserResponse apiUser = client.createApiOnlyUser(token, userReq);
```

> **Note:** This client is intended for **one-time provisioning workflows** only. For ongoing Fleet MDM operations, use the main `FleetMdmClient` with a persistent token.
