Request model for creating a scheduled query in Fleet MDM. Encapsulates all parameters needed to register a new scheduled query via the OpenFrame Fleet MDM API.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | Identifier name for the scheduled query |
| `query` | `String` | The SQL query to be executed |
| `description` | `String` | Optional description of the query's purpose |
| `interval` | `Integer` | Execution frequency in seconds |
| `platform` | `String` | Target platform filter (e.g., `darwin`, `windows`, `linux`) |

**Annotations:**
- `@JsonInclude(NON_NULL)` — null fields are omitted from serialization
- `@Data` / `@NoArgsConstructor` / `@AllArgsConstructor` — Lombok-generated boilerplate

## Usage Example

```java
// Using all-args constructor
CreateScheduledQueryRequest request = new CreateScheduledQueryRequest(
    "disk-usage-check",
    "SELECT * FROM disk_info;",
    "Monitors disk usage across endpoints",
    3600,
    "linux"
);

// Using builder-style setters (no-args + setters)
CreateScheduledQueryRequest request = new CreateScheduledQueryRequest();
request.setName("active-users");
request.setQuery("SELECT * FROM logged_in_users;");
request.setInterval(600);
request.setPlatform("darwin");

fleetMdmClient.createScheduledQuery(packId, request);
```
