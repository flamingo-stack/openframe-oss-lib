Model class representing the request payload for updating a scheduled query in the Fleet MDM integration layer.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | Display name of the scheduled query |
| `query` | `String` | SQL query string to execute |
| `description` | `String` | Human-readable description of the query |
| `interval` | `Integer` | Execution interval in seconds |
| `platform` | `String` | Target platform filter (e.g., `darwin`, `linux`, `windows`) |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates default and full constructors
- `@JsonInclude(NON_NULL)` — Omits `null` fields from JSON serialization, enabling partial updates

## Usage Example

```java
// Partial update - only change interval and platform
UpdateScheduledQueryRequest request = new UpdateScheduledQueryRequest();
request.setInterval(3600);
request.setPlatform("linux");

// Full update using all-args constructor
UpdateScheduledQueryRequest fullRequest = new UpdateScheduledQueryRequest(
    "disk-usage-check",
    "SELECT * FROM disk_info WHERE percentage_used > 80",
    "Alerts on high disk usage",
    300,
    "darwin"
);
```

> **Note:** Because `@JsonInclude(NON_NULL)` is applied, only fields explicitly set will be included in the outgoing request body. This allows callers to perform partial updates without overwriting unset fields on the Fleet MDM side.
