A specialized exception class for handling HTTP API errors in the Fleet MDM SDK, extending FleetMdmException to provide additional context about failed API requests.

## Key Components

- **FleetMdmApiException**: Main exception class that captures HTTP status codes and response bodies
- **statusCode**: Integer field storing the HTTP status code from failed API calls
- **responseBody**: String field containing the raw response body for debugging
- **getStatusCode()**: Accessor method for retrieving the HTTP status code
- **getResponseBody()**: Accessor method for accessing the response body content
- **toString()**: Overridden method for formatted exception display

## Usage Example

```java
try {
    // API call that might fail
    fleetMdmClient.updateDevice(deviceId, config);
} catch (FleetMdmApiException e) {
    // Handle API-specific errors with detailed context
    int statusCode = e.getStatusCode();
    String responseBody = e.getResponseBody();
    
    if (statusCode == 401) {
        logger.error("Authentication failed: " + e.getMessage());
    } else if (statusCode >= 500) {
        logger.error("Server error: " + responseBody);
    }
    
    // Log full exception details
    logger.error("API Exception: " + e.toString());
}
```

This exception type is particularly useful for API client implementations where you need to handle different HTTP error scenarios with access to both status codes and response content for debugging.
