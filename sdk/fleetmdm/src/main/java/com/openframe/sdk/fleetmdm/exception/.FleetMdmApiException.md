A specialized exception class that handles API-related errors in the Fleet MDM SDK, providing detailed error information including HTTP status codes and response bodies.

## Key Components

- **FleetMdmApiException** - Main exception class extending `FleetMdmException`
- **statusCode** - HTTP status code from the failed API request
- **responseBody** - Raw response body content from the API
- **Constructor** - Creates exception with message, status code, and response body
- **Getters** - Access methods for status code and response body
- **toString()** - Custom string representation for debugging

## Usage Example

```java
// Throwing the exception when API call fails
try {
    // API call logic here
} catch (IOException e) {
    throw new FleetMdmApiException(
        "Failed to connect to Fleet MDM API", 
        500, 
        "Internal Server Error"
    );
}

// Handling the exception
try {
    fleetMdmService.performOperation();
} catch (FleetMdmApiException e) {
    System.out.println("API Error: " + e.getMessage());
    System.out.println("Status Code: " + e.getStatusCode());
    System.out.println("Response: " + e.getResponseBody());
    
    // Handle specific status codes
    if (e.getStatusCode() == 401) {
        // Handle authentication error
    } else if (e.getStatusCode() == 404) {
        // Handle not found error
    }
}
```
