A request model for creating a new policy in the Fleet MDM integration, encapsulating the required policy definition fields.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | The display name of the policy |
| `query` | `String` | The SQL/osquery query used to evaluate policy compliance |
| `description` | `String` | Human-readable description of the policy's purpose |
| `platform` | `String` | Target platform (e.g., `windows`, `darwin`, `linux`) |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonInclude(NON_NULL)` — Omits `null` fields during JSON serialization

## Usage Example

```java
// Build a policy request for macOS disk encryption compliance
CreatePolicyRequest request = new CreatePolicyRequest(
    "FileVault Enabled",
    "SELECT 1 FROM disk_encryption WHERE user_uuid IS NOT NULL AND filevault_status = 'on';",
    "Checks that FileVault disk encryption is enabled",
    "darwin"
);

// Or using the no-args constructor with setters
CreatePolicyRequest request = new CreatePolicyRequest();
request.setName("Windows Firewall Active");
request.setQuery("SELECT 1 FROM windows_firewall WHERE global_state = 1;");
request.setPlatform("windows");
```

> **Note:** Fields left as `null` are automatically excluded from the serialized JSON payload, keeping API requests clean when optional fields are not provided.
