Unit test suite for the FleetMdmClient class, verifying HTTP-based interactions with Fleet MDM API endpoints using mocked dependencies.

## Key Components

- **Test Class**: `FleetMdmClientTest` - Main test suite with Mockito extensions
- **Mock Dependencies**: 
  - `HttpClient httpClient` - Mocked HTTP client for network requests
  - `HttpResponse<String> httpResponse` - Mocked HTTP response handling
- **Test Subject**: `FleetMdmClient client` - Instance under test with injected mocks
- **Test Methods**:
  - `getHostById_success()` - Tests successful host retrieval with valid JSON response
  - `getHostById_notFound()` - Tests 404 handling returning null
  - `getHostById_error()` - Tests error handling for 500 status codes

## Usage Example

```java
// Test setup with mocked dependencies
@BeforeEach
void setUp() {
    client = new FleetMdmClient("https://fleet.example.com", "token", httpClient);
}

// Testing successful API response
@Test
void getHostById_success() throws IOException, InterruptedException {
    String body = """
        { "host": { "id": 1, "hostname": "mac" } }
        """;
    when(httpResponse.statusCode()).thenReturn(200);
    when(httpResponse.body()).thenReturn(body);
    when(httpClient.send(any(HttpRequest.class), any(HttpResponse.BodyHandler.class)))
        .thenReturn(httpResponse);

    Host host = client.getHostById(1);
    assertEquals(1L, host.getId());
    assertEquals("mac", host.getHostname());
}
```

The tests cover the complete behavior spectrum: successful responses with JSON parsing, 404 not-found scenarios, and error handling with RuntimeException throwing for server errors.
