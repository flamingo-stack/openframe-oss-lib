Unit test class that validates the functionality of the `FleetMdmClient` class using mocked HTTP dependencies. Tests cover successful host retrieval, not found scenarios, and error handling.

## Key Components

- **Test Setup**: Mocks `HttpClient` and `HttpResponse` dependencies using Mockito
- **FleetMdmClient**: Instance under test initialized with test URL, token, and mocked HTTP client
- **getHostById Tests**: Three test methods covering different response scenarios:
  - `getHostById_success()`: Tests successful host retrieval with 200 status
  - `getHostById_notFound()`: Tests 404 not found handling
  - `getHostById_error()`: Tests server error (500) exception handling

## Usage Example

```java
// Test demonstrates expected FleetMdmClient usage
@Test
void getHostById_success() throws IOException, InterruptedException {
    // Mock successful response
    String jsonResponse = """
        { "host": { "id": 1, "hostname": "mac" } }
        """;
    when(httpResponse.statusCode()).thenReturn(200);
    when(httpResponse.body()).thenReturn(jsonResponse);
    
    // Execute client method
    Host host = client.getHostById(1);
    
    // Verify results
    assertNotNull(host);
    assertEquals(1L, host.getId());
    assertEquals("mac", host.getHostname());
}
```

The tests validate that the client properly handles JSON deserialization, HTTP status codes, and error conditions when communicating with Fleet MDM APIs.
