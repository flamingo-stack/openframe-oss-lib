Unit tests for the Fleet MDM client's query-related functionality, testing successful query retrieval, error handling, and authentication scenarios using mocked HTTP responses.

## Key Components

- **FleetMdmClientQueryTest** - Test class for Fleet MDM client query operations
- **setUp()** - Initializes mock HTTP client and Fleet MDM client for testing
- **testGetQueryById_Success()** - Tests successful query retrieval with complete response parsing
- **testGetQueryById_NotFound()** - Tests 404 error handling when query doesn't exist
- **testGetQueryById_Unauthorized()** - Tests 401 authentication failure scenarios

## Usage Example

```java
// This is a test class - typical usage would be running tests
// The tests validate the FleetMdmClient's getQueryById method:

// Setup test environment
FleetMdmClient client = new FleetMdmClient("https://fleet.test.com", "test-token", mockHttpClient);

// Test successful query retrieval
Query query = client.getQueryById(2L);
assertEquals("System Info Query", query.getName());
assertEquals("SELECT * FROM system_info", query.getQuery());

// Test error handling
Query notFound = client.getQueryById(999L);
assertNull(notFound); // Returns null for 404

// Test authentication failures
FleetMdmApiException exception = assertThrows(FleetMdmApiException.class, 
    () -> client.getQueryById(1L)); // Throws exception for 401
```

The test suite verifies proper JSON parsing, error handling for different HTTP status codes, and ensures the client correctly handles authentication failures and missing resources.
