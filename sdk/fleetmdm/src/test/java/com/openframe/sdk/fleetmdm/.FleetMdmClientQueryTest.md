Test suite for Fleet MDM Client query-related operations, validating API interactions for retrieving queries by ID with comprehensive error handling scenarios.

## Key Components

- **FleetMdmClientQueryTest**: Main test class containing unit tests for query operations
- **setUp()**: Test fixture initialization with mock HTTP client
- **testGetQueryById_Success()**: Validates successful query retrieval with proper JSON deserialization
- **testGetQueryById_NotFound()**: Tests handling of 404 responses for non-existent queries
- **testGetQueryById_Unauthorized()**: Verifies proper exception handling for authentication failures

## Usage Example

```java
// Example of what the client method being tested might look like
FleetMdmClient client = new FleetMdmClient("https://fleet.example.com", "api-token");

// Successful query retrieval
Query query = client.getQueryById(2L);
if (query != null) {
    System.out.println("Query: " + query.getName());
    System.out.println("SQL: " + query.getQuery());
    System.out.println("Is scheduled: " + query.isScheduled());
}

// Handle cases where query doesn't exist
Query missingQuery = client.getQueryById(999L);
// Returns null for 404 responses

// Authentication errors throw FleetMdmApiException
try {
    client.getQueryById(1L);
} catch (FleetMdmApiException e) {
    System.err.println("API Error: " + e.getMessage());
}
```
