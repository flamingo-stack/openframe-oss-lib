Response model for the Tactical RMM script schedule agents API (`POST`/`DELETE`/`PUT /script-schedules/<pk>/agents/`). It captures the resulting agent assignment count and the number of `TaskResult` rows synced during the operation.

## Key Components

| Field | JSON Property | Type | Description |
|---|---|---|---|
| `agentsCount` | `agents_count` | `Integer` | Total number of agents currently assigned to the schedule |
| `taskResultsCreated` | `task_results_created` | `Integer` | TaskResult rows created for newly assigned agents |
| `taskResultsDeleted` | `task_results_deleted` | `Integer` | TaskResult rows removed for unassigned agents |

**Annotations:**
- `@JsonIgnoreProperties(ignoreUnknown = true)` — safely ignores any extra fields returned by the API
- `@JsonProperty` — maps snake_case JSON keys to camelCase Java fields

## Usage Example

```java
// After modifying agent assignments on a script schedule
ScriptScheduleAgentsResult result = tacticalRmmClient
    .scriptSchedules()
    .updateAgents(scheduleId, agentUpdateRequest);

System.out.println("Agents assigned: " + result.getAgentsCount());
System.out.println("Task results created: " + result.getTaskResultsCreated());
System.out.println("Task results deleted: " + result.getTaskResultsDeleted());

// Check if any cleanup occurred
if (result.getTaskResultsDeleted() > 0) {
    log.info("Cleaned up {} stale task result rows", result.getTaskResultsDeleted());
}
```
