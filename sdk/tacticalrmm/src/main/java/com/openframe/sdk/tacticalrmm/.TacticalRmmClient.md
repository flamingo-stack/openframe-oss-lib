HTTP client for interacting with the Tactical RMM REST API, providing methods for agent management, script execution, and scheduled task operations.

## Key Components

| Method | Description |
|--------|-------------|
| `getInstallationSecret` | Retrieves an agent registration/installation secret |
| `runCommand` | Executes a shell command on a specific agent (blocking) |
| `runCommandAsync` | Executes a shell command on a specific agent (non-blocking, returns `CompletableFuture`) |
| `getAgentInfo` | Fetches detailed platform and OS information for a single agent |
| `getAllAgents` | Lists all agents with minimal detail (`detail=false`) |
| `getAllScripts` | Retrieves all available scripts from the Tactical RMM server |

## Usage Example

```java
TacticalRmmClient client = new TacticalRmmClient();

// Run a blocking command
CommandResult result = client.runCommand(
    "https://rmm.example.com",
    "your-api-key",
    "agent-id-123",
    "powershell",
    "Get-ComputerInfo",
    60,
    false
);

// Run a command asynchronously
CompletableFuture<CommandResult> future = client.runCommandAsync(
    "https://rmm.example.com",
    "your-api-key",
    "agent-id-123",
    "bash",
    "uname -a",
    30,
    false
);

future.thenAccept(r -> System.out.println(r.getOutput()));

// List all agents
List<AgentListItem> agents = client.getAllAgents(
    "https://rmm.example.com",
    "your-api-key"
);
```

## Notes

- HTTP connection timeout: **5 seconds**; per-request timeouts vary (10–30 seconds)
- Throws `TacticalRmmApiException` for non-2xx HTTP responses (with status code and body)
- Throws `TacticalRmmException` for network/parsing failures
- All methods validate `tacticalServerUrl` and `apiKey` before making requests
