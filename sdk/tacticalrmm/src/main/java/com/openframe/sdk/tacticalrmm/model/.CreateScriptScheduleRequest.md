Represents the request payload for creating a scheduled script task in the TacticalRMM integration, mapping task scheduling parameters to their JSON API equivalents.

## Key Components

| Field | JSON Property | Description |
|-------|--------------|-------------|
| `name` | `name` | Display name for the scheduled task |
| `taskType` | `task_type` | Type of scheduled task (e.g., `daily`, `weekly`) |
| `runTimeDate` | `run_time_date` | ISO date/time string for task execution |
| `enabled` | `enabled` | Whether the schedule is active |
| `runTimeBitWeekdays` | `run_time_bit_weekdays` | Bitmask representing days of the week |
| `weeklyInterval` | `weekly_interval` | Interval in weeks between executions |
| `dailyInterval` | `daily_interval` | Interval in days between executions |
| `taskSupportedPlatforms` | `task_supported_platforms` | List of OS platforms the task targets |
| `actions` | `actions` | List of [`TaskAction`](./TaskAction) objects to execute |

> `@JsonInclude(NON_NULL)` ensures only populated fields are serialized, keeping API payloads clean.

## Usage Example

```java
CreateScriptScheduleRequest request = new CreateScriptScheduleRequest();
request.setName("Weekly Maintenance");
request.setTaskType("weekly");
request.setEnabled(true);
request.setWeeklyInterval(1);
request.setRunTimeBitWeekdays(0b0111110); // Monday–Friday bitmask

TaskAction action = new TaskAction();
action.setScript(42);
request.setActions(List.of(action));

// Serialize to JSON (null fields omitted automatically)
// {"name":"Weekly Maintenance","task_type":"weekly","enabled":true,
//  "weekly_interval":1,"run_time_bit_weekdays":62,"actions":[...]}
```
