A specialized exception class for handling HTTP API errors when communicating with the Tactical RMM service. This exception captures both the HTTP status code and response body to provide detailed error information for API failures.

## Key Components

- **TacticalRmmApiException** - Main exception class extending `TacticalRmmException`
- **statusCode** - HTTP status code from the failed API request
- **responseBody** - Raw response content from the API error
- **Constructor** - Accepts error message, status code, and response body
- **Getters** - Access methods for status code and response body
- **toString()** - Custom string representation including error details

## Usage Example

```java
try {
    // API call to Tactical RMM
    tacticalRmmClient.getAgents();
} catch (TacticalRmmApiException e) {
    // Handle specific API errors
    int statusCode = e.getStatusCode();
    String responseBody = e.getResponseBody();
    
    if (statusCode == 401) {
        logger.error("Authentication failed: " + e.getMessage());
    } else if (statusCode == 404) {
        logger.warn("Resource not found: " + responseBody);
    } else {
        logger.error("API error [" + statusCode + "]: " + responseBody);
    }
}

// Throwing the exception
if (response.getStatusCode() >= 400) {
    throw new TacticalRmmApiException(
        "API request failed", 
        response.getStatusCode(), 
        response.getBody()
    );
}
```
