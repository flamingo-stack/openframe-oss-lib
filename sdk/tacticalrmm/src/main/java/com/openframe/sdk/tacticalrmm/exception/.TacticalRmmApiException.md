A specialized exception class that extends TacticalRmmException to handle HTTP API-related errors with additional context about response status codes and response bodies.

## Key Components

- **TacticalRmmApiException(String, int, String)** - Constructor that accepts error message, HTTP status code, and response body
- **getStatusCode()** - Returns the HTTP status code from the failed API call
- **getResponseBody()** - Returns the raw response body content from the API
- **toString()** - Provides formatted string representation including status code and response body

## Usage Example

```java
try {
    // API call that might fail
    tacticalRmmClient.getAgents();
} catch (TacticalRmmApiException e) {
    System.err.println("API Error: " + e.getMessage());
    System.err.println("Status Code: " + e.getStatusCode());
    System.err.println("Response: " + e.getResponseBody());
    
    // Handle specific HTTP status codes
    if (e.getStatusCode() == 401) {
        // Handle authentication error
        refreshAuthToken();
    } else if (e.getStatusCode() >= 500) {
        // Handle server errors
        scheduleRetry();
    }
}
```

This exception provides detailed context for debugging API failures by preserving both the HTTP status code and the server's response body.
