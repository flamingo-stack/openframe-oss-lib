Custom runtime exception class for handling errors specific to TacticalRMM operations within the OpenFrame SDK. This exception provides standardized error handling for TacticalRMM API interactions and service operations.

## Key Components

- **TacticalRmmException(String message)** - Constructor for creating exceptions with descriptive error messages
- **TacticalRmmException(String message, Throwable cause)** - Constructor for wrapping underlying exceptions while preserving the original cause and adding context

## Usage Example

```java
// Throwing a TacticalRMM-specific exception
public void connectToAgent(String agentId) throws TacticalRmmException {
    if (agentId == null || agentId.isEmpty()) {
        throw new TacticalRmmException("Agent ID cannot be null or empty");
    }
    
    try {
        // Attempt connection to TacticalRMM agent
        rmmClient.connect(agentId);
    } catch (ConnectionException e) {
        throw new TacticalRmmException("Failed to connect to TacticalRMM agent: " + agentId, e);
    }
}

// Catching and handling the exception
try {
    tacticalRmmService.connectToAgent("invalid-id");
} catch (TacticalRmmException e) {
    logger.error("TacticalRMM operation failed: {}", e.getMessage());
    // Handle the specific TacticalRMM error
}
```
