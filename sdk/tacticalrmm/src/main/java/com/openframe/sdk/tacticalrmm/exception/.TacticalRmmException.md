A custom runtime exception class for the Tactical RMM SDK that provides standardized error handling for RMM-related operations and API interactions.

## Key Components

- **TacticalRmmException** - Custom runtime exception extending `RuntimeException`
- **Constructor overloads** - Support for message-only and message-with-cause error scenarios

## Usage Example

```java
// Throwing exception with message only
if (apiResponse.getStatusCode() != 200) {
    throw new TacticalRmmException("Failed to connect to Tactical RMM API");
}

// Throwing exception with message and underlying cause
try {
    // API call that might fail
    rmmClient.getAgents();
} catch (IOException e) {
    throw new TacticalRmmException("Unable to retrieve agents from RMM", e);
}

// Catching the exception
try {
    rmmService.deployScript(scriptId);
} catch (TacticalRmmException e) {
    logger.error("RMM operation failed: " + e.getMessage(), e);
    // Handle RMM-specific error
}
```

This exception class allows the SDK to throw consistent, identifiable exceptions for all Tactical RMM related failures while preserving the original cause when wrapping other exceptions.
