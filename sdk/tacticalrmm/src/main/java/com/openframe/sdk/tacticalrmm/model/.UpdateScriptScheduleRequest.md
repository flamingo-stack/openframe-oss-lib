Request model for updating an existing script schedule in the TacticalRMM integration, mapping schedule configuration fields to their JSON API counterparts.

## Key Components

| Field | JSON Property | Type | Description |
|-------|--------------|------|-------------|
| `name` | `name` | `String` | Schedule name |
| `taskType` | `task_type` | `String` | Type of scheduled task |
| `runTimeDate` | `run_time_date` | `String` | Scheduled execution date/time |
| `enabled` | `enabled` | `Boolean` | Enable/disable the schedule |
| `runTimeBitWeekdays` | `run_time_bit_weekdays` | `Integer` | Bitmask for weekday selection |
| `weeklyInterval` | `weekly_interval` | `Integer` | Recurrence in weeks |
| `dailyInterval` | `daily_interval` | `Integer` | Recurrence in days |
| `taskSupportedPlatforms` | `task_supported_platforms` | `List<String>` | Target OS platforms |
| `actions` | `actions` | `List<TaskAction>` | Task actions to execute |

**Annotations:** `@JsonInclude(NON_NULL)` ensures only populated fields are serialized, making partial updates safe without overwriting existing schedule values.

## Usage Example

```java
UpdateScriptScheduleRequest request = new UpdateScriptScheduleRequest();
request.setName("Weekly Maintenance");
request.setTaskType("scheduled");
request.setEnabled(true);
request.setWeeklyInterval(1);
request.setRunTimeBitWeekdays(0b0111110); // Mon–Fri bitmask
request.setRunTimeDate("2024-01-15T02:00:00");
request.setTaskSupportedPlatforms(List.of("windows", "linux"));

TaskAction action = new TaskAction();
// configure action...
request.setActions(List.of(action));

tacticalRmmClient.updateScriptSchedule(scheduleId, request);
```
