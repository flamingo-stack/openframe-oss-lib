A utility class that manages authentication cookies in a thread-safe manner for test execution. It provides centralized cookie management with automatic login fallback and thread-local storage.

## Key Components

- **ThreadLocal<Map<String, String>> cookies** - Thread-safe storage for authentication cookies
- **getCookies()** - Retrieves current cookies or performs automatic login if none exist
- **setCookies(Map<String, String>)** - Manually sets authentication cookies for current thread
- **clearCookies()** - Removes cookies from current thread storage

## Usage Example

```java
// Automatic login and cookie retrieval
Map<String, String> authCookies = AuthHelper.getCookies();

// Manual cookie management
Map<String, String> customCookies = new HashMap<>();
customCookies.put("sessionId", "abc123");
AuthHelper.setCookies(customCookies);

// Clean up after test
@AfterEach
void cleanup() {
    AuthHelper.clearCookies();
}

// Thread-safe usage in parallel tests
@Test
@Execution(ExecutionMode.CONCURRENT)
void testWithAuthentication() {
    Map<String, String> cookies = AuthHelper.getCookies();
    // Each thread gets its own cookie storage
}
```

The class ensures thread isolation for parallel test execution while providing convenient authentication state management across test methods.
