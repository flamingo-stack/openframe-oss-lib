Builds and configures RestAssured `RequestSpecification` objects for OpenFrame API tests, providing authorized, unauthorized, and auth-flow request specs with retry logic, SSL relaxation, and SLF4J-routed logging.

## Key Components

| Member | Description |
|--------|-------------|
| `getAuthorizedSpec()` | Returns a `RequestSpecification` with session cookies attached (auto-fetches via `AuthHelper` if not set) |
| `getUnAuthorizedSpec()` | Returns a bare `RequestSpecification` with no credentials |
| `getAuthFlowRequestSpec()` | Returns a spec targeting the auth URL (used during login/token flows) |
| `graphqlSuccess()` | `ResponseSpecification` asserting HTTP 200 and no `errors` field in the GraphQL response body |
| `setBaseUrl(String)` / `setCookies(Map)` | Thread-local overrides for base URL and cookies, safe for parallel test execution |
| `clearCookies()` | Removes the thread-local cookie store (forces re-authentication on next use) |
| `setEnableLogging(boolean)` | Toggles `RequestLoggingFilter` globally |
| `retryingHttpClientConfig()` | Configures 10 s connect / 32 s socket timeouts with up to 2 connect-failure retries |
| `Slf4jOutputStream` (inner) | Bridges RestAssured's `PrintStream` logging to SLF4J line-by-line |

## Usage Example

```java
// Authorized JSON request
given()
    .spec(RequestSpecHelper.getAuthorizedSpec())
    .body(payload)
.when()
    .post("/api/devices")
.then()
    .statusCode(201);

// GraphQL request with success assertion
given()
    .spec(RequestSpecHelper.getAuthorizedSpec())
    .body(graphqlQuery)
.when()
    .post("/graphql")
.then()
    .spec(RequestSpecHelper.graphqlSuccess());

// Override base URL per thread (e.g. in parallel test suites)
RequestSpecHelper.setBaseUrl("https://staging.openframe.ai");

// Unauthenticated call (e.g. testing 401 responses)
given()
    .spec(RequestSpecHelper.getUnAuthorizedSpec())
.when()
    .get("/api/protected")
.then()
    .statusCode(401);
```

> **Retry behaviour:** Only connect-establishment failures (`ConnectException`, `ConnectTimeoutException`) are retried — never socket/read timeouts — ensuring mutations are not replayed after the server has already received the request.
