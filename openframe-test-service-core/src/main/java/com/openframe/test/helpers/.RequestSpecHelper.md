Test helper utility for creating REST-assured request specifications with proper authentication and reporting setup for OpenFrame test automation.

## Key Components

- **`getAuthorizedSpec()`** - Creates a request specification with authentication cookies, JSON content type, and Allure reporting filter
- **`getUnAuthorizedSpec()`** - Creates a request specification without authentication, suitable for testing unauthorized endpoints

Both methods automatically configure:
- JSON content type for API requests
- Allure REST-assured filter for test reporting integration
- Request specification builder pattern for clean test code

## Usage Example

```java
import static com.openframe.test.helpers.RequestSpecHelper.*;

public class ApiTests {
    
    @Test
    public void testProtectedEndpoint() {
        given()
            .spec(getAuthorizedSpec())
        .when()
            .get("/api/protected/users")
        .then()
            .statusCode(200);
    }
    
    @Test
    public void testPublicEndpoint() {
        given()
            .spec(getUnAuthorizedSpec())
        .when()
            .get("/api/public/health")
        .then()
            .statusCode(200);
    }
    
    @Test
    public void testUnauthorizedAccess() {
        given()
            .spec(getUnAuthorizedSpec())
        .when()
            .get("/api/protected/admin")
        .then()
            .statusCode(401);
    }
}
```

This helper eliminates boilerplate code and ensures consistent request configuration across all API tests while maintaining proper authentication flow and test reporting.
