Configures REST Assured for API testing with default timeouts, SSL handling, and logging settings. This utility class sets up the global REST Assured configuration used across all test cases.

## Key Components

- **`configure()`** - Sets up REST Assured with the default base URL from ApiConfig
- **`configure(String baseUrl)`** - Configures REST Assured with a custom base URL
- **`DEFAULT_TIMEOUT`** - 60-second timeout constant for HTTP connections and socket operations

## Configuration Features

- **SSL Configuration**: Relaxed HTTPS validation for testing environments
- **Logging**: Automatic request/response logging when validation fails
- **Timeouts**: 60-second connection and socket timeouts
- **Base URL**: Integration with ApiConfig for centralized URL management

## Usage Example

```java
// Configure with default base URL
RestAssuredConfig.configure();

// Or configure with custom base URL
RestAssuredConfig.configure("https://test-api.example.com");

// REST Assured is now configured globally for all requests
given()
    .when()
    .get("/api/users")
    .then()
    .statusCode(200);
```

The configuration is applied globally to REST Assured, so it only needs to be called once per test suite, typically in a `@BeforeAll` setup method or test configuration class.
