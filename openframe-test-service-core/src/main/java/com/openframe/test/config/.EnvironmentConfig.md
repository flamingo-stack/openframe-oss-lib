Centralized configuration class for resolving test environment URLs and settings from environment variables or programmatic setup.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `OSS`, `DEV`, `STAGE`, `PROD` | `String` constants | Supported environment mode identifiers |
| `DEFAULT_BASE_URL`, `GRAPHQL`, `CHAT_GRAPHQL` | `String` constants | Default URL and GraphQL endpoint paths |
| `configure(baseUrl, userDomain)` | Static method | Programmatically set base URL and user domain, bypassing env vars |
| `random()` | Static method | Generates a random `test-??????` user domain using DataFaker |
| `getEnvMode()` | Static method | Returns `TEST_ENV_MODE` env var, defaults to `DEV` |
| `getBaseUrl()` | Static method | Resolves tenant-aware base URL, handles localhost special case |
| `getAuthUrl()` | Static method | Returns auth endpoint using the raw base URL |
| `getUserDomain()` | Static method | Returns `localhost` in OSS mode, full subdomain otherwise |
| `getRegistrationUrl()` | Static method | Returns registration URL with optional port |

## Usage Example

```java
// Option 1: Load from environment variables
// Requires TEST_BASE_URL and TEST_USER_DOMAIN to be set
String baseUrl = EnvironmentConfig.getBaseUrl();
String graphqlEndpoint = baseUrl + EnvironmentConfig.GRAPHQL;

// Option 2: Programmatic setup (e.g., in test setup/BeforeAll)
EnvironmentConfig.configure("flamingo.run", "acme");
String tenantUrl = EnvironmentConfig.getBaseUrl(); 
// → "https://acme.flamingo.run/"

// Option 3: Random domain for isolated test runs
EnvironmentConfig.random();
String domain = EnvironmentConfig.getUserDomain();

// Check environment mode
if (EnvironmentConfig.getEnvMode().equals(EnvironmentConfig.OSS)) {
    // OSS-specific test behavior
}
```

## Required Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `TEST_BASE_URL` | ✅ Yes | Base domain for the test target |
| `TEST_USER_DOMAIN` | ✅ Yes | Tenant subdomain for the test user |
| `TEST_ENV_MODE` | ❌ No | Environment mode; defaults to `DEV` |
| `PORT` | ❌ No | Optional port appended to URLs |
