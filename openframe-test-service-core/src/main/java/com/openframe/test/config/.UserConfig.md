Test configuration utility that manages user credentials and settings with support for environment variable and system property overrides.

## Key Components

- **Constants**: Default test user credentials (`DEFAULT_EMAIL`, `DEFAULT_PASSWORD`, `NEW_PASSWORD`) and domain settings
- **getUser()**: Returns a singleton User object built from configured email, password, and domain
- **getEmail()**: Retrieves user email from system property → environment variable → default fallback
- **getPassword()**: Retrieves user password with same priority order as email
- **getDomain()**: Retrieves user domain with same priority order, defaults to "localhost"

## Usage Example

```java
// Get default test user with auto@flamingo.cx credentials
User testUser = UserConfig.getUser();

// Override via system properties
System.setProperty("user.email", "test@example.com");
System.setProperty("user.password", "CustomPass123!");
User customUser = UserConfig.getUser();

// Override via environment variables
// USER_EMAIL=qa@flamingo.cx USER_PASSWORD=QAPass123! java -jar test.jar
String email = UserConfig.getEmail(); // Returns "qa@flamingo.cx"
```

The configuration follows a priority hierarchy: system properties override environment variables, which override hardcoded defaults. This allows flexible test user management across different environments while maintaining consistent fallback behavior.
