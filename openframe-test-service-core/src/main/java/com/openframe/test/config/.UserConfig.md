Provides static configuration for test user credentials, supporting both explicit setup and environment variable-based resolution for use in automated test suites.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `configure(email, password)` | `static void` | Explicitly sets email and password for the test user |
| `random()` | `static void` | Generates a random email using `DataFaker` (e.g., `abcdef@flamingo.cx`) |
| `getUser()` | `static User` | Builds and returns a `User` DTO with email, password, and domain |
| `getEmail()` | `static String` | Returns email, falling back to `TEST_USER_EMAIL` env var |
| `getPassword()` | `static String` | Returns password, falling back to `TEST_USER_PASSWORD` env var |
| `getDomain()` | `static String` | Delegates to `EnvironmentConfig.getUserDomain()` |

## Usage Example

```java
// Option 1: Explicit configuration
UserConfig.configure("technician@flamingo.cx", "securePassword123");

// Option 2: Resolve from environment variables
// Requires TEST_USER_EMAIL and TEST_USER_PASSWORD to be set
User user = UserConfig.getUser();

// Option 3: Random user (email only)
UserConfig.random();
String email = UserConfig.getEmail(); // e.g., "xkrtpq@flamingo.cx"
```

> **Note:** If `TEST_USER_EMAIL` or `TEST_USER_PASSWORD` environment variables are not set and no explicit configuration was applied, a `RuntimeException` is thrown. Always ensure these are configured in your CI/CD pipeline or local environment before running tests.
