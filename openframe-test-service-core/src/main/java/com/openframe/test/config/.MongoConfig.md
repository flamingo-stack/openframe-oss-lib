Provides lazy-loaded, environment-variable-backed static accessors for MongoDB connection settings used across the OpenFrame test suite.

## Key Components

| Method | Env Variable | Description |
|--------|-------------|-------------|
| `getMongoDbUri()` | `MONGODB_URI` | Full MongoDB connection URI |
| `getDatabaseName()` | `MONGODB_DATABASE` | Target database name |
| `getMongoUser()` | `MONGODB_USERNAME` | Authentication username |
| `getMongoPassword()` | `MONGODB_PASSWORD` | Authentication password |
| `getAuthDatabase()` | `MONGODB_AUTH_DATABASE` | Database used for credential authentication |

All fields are lazily initialized on first access and cached as static values. A `RuntimeException` is thrown if any required environment variable is missing or blank.

## Usage Example

```java
// Retrieve MongoDB connection details at test setup
String uri      = MongoConfig.getMongoDbUri();
String database = MongoConfig.getDatabaseName();
String user     = MongoConfig.getMongoUser();
String password = MongoConfig.getMongoPassword();
String authDb   = MongoConfig.getAuthDatabase();

MongoCredential credential = MongoCredential.createCredential(
    user, authDb, password.toCharArray()
);
MongoClient client = MongoClients.create(uri);
MongoDatabase db   = client.getDatabase(database);
```

Ensure the following environment variables are set before running tests:

```bash
export MONGODB_URI="mongodb://localhost:27017"
export MONGODB_DATABASE="openframe_test"
export MONGODB_USERNAME="testuser"
export MONGODB_PASSWORD="testpass"
export MONGODB_AUTH_DATABASE="admin"
```
