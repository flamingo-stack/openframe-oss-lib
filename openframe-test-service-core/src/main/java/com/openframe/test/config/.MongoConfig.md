A configuration utility class that provides MongoDB connection parameters for OpenFrame applications, supporting both environment variables and system properties with fallback defaults.

## Key Components

- **Static Configuration Methods**: Five getter methods that retrieve MongoDB connection parameters
  - `getMongoDbUri()` - MongoDB connection URI
  - `getDatabaseName()` - Target database name  
  - `getMongoUser()` - Authentication username
  - `getMongoPassword()` - Authentication password
  - `getAuthDatabase()` - Authentication database name
- **Configuration Priority**: System properties → Environment variables → Default values
- **Lazy Loading**: Values are cached on first access to avoid repeated lookups
- **Default Values**: Preconfigured for Kubernetes cluster deployment with "openframe" database

## Usage Example

```java
// Get MongoDB connection parameters
String uri = MongoConfig.getMongoDbUri();
String database = MongoConfig.getDatabaseName();
String user = MongoConfig.getMongoUser();
String password = MongoConfig.getMongoPassword();
String authDb = MongoConfig.getAuthDatabase();

// Configure MongoDB client
MongoClientSettings settings = MongoClientSettings.builder()
    .applyConnectionString(new ConnectionString(uri))
    .credential(MongoCredential.createCredential(user, authDb, password.toCharArray()))
    .build();

MongoClient client = MongoClients.create(settings);
MongoDatabase db = client.getDatabase(database);
```

Configuration can be overridden via environment variables (`MONGODB_URI`, `MONGODB_DATABASE`, etc.) or system properties (`mongodb.uri`, `mongodb.database`, etc.).
