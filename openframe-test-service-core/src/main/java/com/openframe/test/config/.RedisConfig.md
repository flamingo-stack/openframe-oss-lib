Configuration utility class for managing Redis connection URI settings with fallback priority and lazy initialization.

## Key Components

- **DEFAULT_REDIS_URI**: Default Kubernetes cluster Redis endpoint
- **getRedisUri()**: Static method that resolves Redis URI with configuration priority:
  1. System property (`redis.uri`)
  2. Environment variable (`REDIS_URI`)
  3. Default cluster URI fallback
- **Lazy initialization**: URI is resolved once and cached for subsequent calls
- **Logging**: Debug-level logging for configuration transparency

## Usage Example

```java
// Get Redis URI (auto-resolves from system property, env var, or default)
String redisConnection = RedisConfig.getRedisUri();

// Configure Redis client with resolved URI
RedisClient client = RedisClient.create(redisConnection);

// Set custom Redis URI via system property
System.setProperty("redis.uri", "redis://localhost:6379");
String customUri = RedisConfig.getRedisUri();

// Set via environment variable
// REDIS_URI=redis://prod-redis:6379
// String prodUri = RedisConfig.getRedisUri();
```

This configuration follows the **12-factor app** methodology by prioritizing environment-specific configuration over defaults, making it suitable for both development and production Kubernetes deployments in the Flamingo/OpenFrame stack.
