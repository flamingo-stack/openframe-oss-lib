Programmatic JUnit 5 test runner that discovers, filters, and executes tests by tag using the JUnit Platform Launcher API with JAR-aware classpath scanning.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `TestRunner(TestRunnerConfig)` | Constructor | Initializes the JUnit `Launcher` and registers execution listeners from config |
| `run(String... tags)` | Method | Discovers and executes tests matching the given tag(s) |
| `run(LauncherDiscoveryRequest)` | Method | Executes tests from a pre-built discovery request |
| `run(TestPlan)` | Method | Executes a pre-discovered `TestPlan` directly |
| `discover(String... tags)` | Method | Returns a `TestPlan` for the given tags without executing |
| `list(TestPlan)` | Method | Maps a `TestPlan` to a flat `List<Test>` with display name and tags |
| `discoverTestClasses()` | Private | Scans JAR entries under the configured package path, falling back to `selectPackage` |

## Usage Example

```java
// Configure and build the runner
TestRunnerConfig config = TestRunnerConfig.builder()
    .testPackage("com.openframe.tests")
    .testListeners(new MyExecutionListener())
    .build();

TestRunner runner = new TestRunner(config);

// Discover tests tagged "smoke" without running them
TestPlan plan = runner.discover("smoke");

// List all discovered tests
List<Test> tests = runner.list(plan);
tests.forEach(t -> System.out.println(t.getDisplayName()));

// Execute all "smoke" tagged tests
runner.run("smoke");

// Or execute with a custom discovery request
LauncherDiscoveryRequest request = LauncherDiscoveryRequestBuilder.request()
    .selectors(selectPackage("com.openframe.tests"))
    .filters(includeTags("integration"))
    .build();
runner.run(request);
```

## Notes

- **JAR scanning**: `discoverTestClasses()` opens `JarURLConnection` to enumerate `.class` entries, skipping anonymous/inner classes (`$`). Falls back to `selectPackage` when no JAR resources are found (e.g., in local IDE runs).
- **Listener registration** happens once at construction time via `TestRunnerConfig.getTestListeners()`, making the runner reusable across multiple `run()` calls.
