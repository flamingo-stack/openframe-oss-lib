Configuration data class for the `TestRunner`, holding the test package path and optional JUnit `TestExecutionListener` instances to attach during test execution.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `testPackage` | `String` | Package path to scan for tests |
| `testListeners` | `TestExecutionListener[]` | JUnit Platform listeners for execution events |
| `TestRunnerConfigBuilder` | Inner class | Fluent builder for constructing `TestRunnerConfig` instances |

## Usage Example

```java
// Using the fluent builder
TestRunnerConfig config = TestRunnerConfig.builder()
    .testPackage("com.openframe.tests.integration")
    .testListeners(new MyCustomListener(), new LoggingListener())
    .build();

// Using the all-args constructor (Lombok)
TestRunnerConfig config = new TestRunnerConfig(
    "com.openframe.tests.unit",
    new TestExecutionListener[]{ new MyCustomListener() }
);

// Passing to a TestRunner
TestRunner runner = new TestRunner(config);
runner.execute();
```

> **Note:** `testListeners` is variadic in the builder (`TestExecutionListener...`), so multiple listeners can be passed directly without wrapping in an array. The no-args constructor (Lombok `@NoArgsConstructor`) is also available for frameworks requiring default instantiation.
