Defines the contract for a multi-step authentication flow, providing a fluent (builder-style) interface for executing OAuth/auth sequences in sequence.

## Key Components

| Method | Description |
|---|---|
| `discoverTenant()` | Resolves the tenant/realm for the authentication context |
| `startFlow()` | Initiates the authentication flow (e.g., opens a login session) |
| `initAuth()` | Performs initial authorization setup or redirects |
| `postCredentials()` | Submits user credentials to the identity provider |
| `getAuthCode()` | Retrieves the authorization code after credential validation |
| `extractTokens()` | Finalizes the flow and returns the extracted token map (e.g., `access_token`, `refresh_token`) |

## Usage Example

```java
Map<String, String> tokens = authFlow
    .discoverTenant()
    .startFlow()
    .initAuth()
    .postCredentials()
    .getAuthCode()
    .extractTokens();

String accessToken = tokens.get("access_token");
String refreshToken = tokens.get("refresh_token");
```

Each method returns `IAuthFlow` to enable method chaining, with `extractTokens()` terminating the chain and returning the resulting token pairs. Implementations are expected to maintain internal state across each step.
