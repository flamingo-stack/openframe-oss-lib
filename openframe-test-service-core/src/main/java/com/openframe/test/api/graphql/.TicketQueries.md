Defines GraphQL query and mutation string constants used for ticket-related API testing in the OpenFrame platform.

## Key Components

| Constant | Type | Description |
|---|---|---|
| `TICKET_LABELS` | Query | Fetches all ticket label metadata (id, key, color, etc.) |
| `GET_TICKETS` | Query | Paginated ticket list with filtering, search, and cursor-based pagination |
| `GET_TICKET` | Query | Full ticket detail by ID, including notes, attachments, dialog, and AI token usage |
| `CREATE_TICKET` | Mutation | Creates a new ticket and returns the full ticket payload |
| `ARCHIVE_TICKET` | Mutation | Archives a ticket by ID |
| `RESOLVE_TICKET` | Mutation | Marks a ticket as resolved |
| `PUT_TICKET_ON_HOLD` | Mutation | Places a ticket in on-hold status |
| `REORDER_TICKET` | Mutation | Updates a ticket's sort order |

## Usage Example

```java
// Execute GET_TICKETS query in an integration test
GraphQLResponse response = graphQLClient.query(
    TicketQueries.GET_TICKETS,
    Map.of(
        "filter", Map.of("status", "OPEN"),
        "pagination", Map.of("first", 10),
        "search", "network issue"
    )
);

// Execute CREATE_TICKET mutation
GraphQLResponse createResponse = graphQLClient.mutation(
    TicketQueries.CREATE_TICKET,
    Map.of("input", Map.of(
        "title", "VPN not connecting",
        "organizationId", "org_123"
    ))
);
```

## Notes

- All queries follow cursor-based pagination via `CursorPaginationInput`
- `owner` fields use GraphQL union types: `ClientTicketOwner` (machine-linked) or `AdminTicketOwner` (user-linked)
- `GET_TICKET` includes `dialog.tokenUsage` for tracking Mingo AI conversation context
- All mutations return a `userErrors` array for field-level validation feedback
