REST-Assured API client for user-related endpoints, providing methods to interact with user management, authentication, and password reset operations in the OpenFrame test suite.

## Key Components

| Method | Endpoint | Description |
|--------|----------|-------------|
| `me()` | `GET api/me` | Returns current authenticated user profile |
| `getUsers()` | `GET api/users` | Retrieves full list of authenticated users |
| `getUsers(UserRole)` | `GET api/users` | Filters user list by role |
| `getUser(String)` | `GET api/users/{id}` | Fetches a single user by ID |
| `deleteUser(String)` | `DELETE api/users/{id}` | Deletes a user, returns HTTP status code |
| `resetPassword(User)` | `POST sas/password-reset/request` | Initiates password reset flow (unauthenticated) |
| `confirmReset(ResetConfirmRequest)` | `POST sas/password-reset/confirm` | Confirms a password reset (expects `204`) |
| `attemptConfirmReset(ResetConfirmRequest)` | `POST sas/password-reset/confirm` | Confirms reset expecting a `400` error response |

**Auth strategy:** Authorized endpoints use `getAuthorizedSpec()`; password reset endpoints use `getUnAuthorizedSpec()`.

## Usage Example

```java
// Get current user
MeResponse me = UserApi.me();

// Get all users with a specific role
List<AuthUser> admins = UserApi.getUsers(UserRole.ADMIN);

// Fetch a user by ID
AuthUser user = UserApi.getUser("user-123");

// Delete a user and assert status
int status = UserApi.deleteUser("user-123");

// Trigger and confirm password reset
UserApi.resetPassword(user);
UserApi.confirmReset(new ResetConfirmRequest(token, "newPassword123"));

// Test invalid reset token - expect 400 error
ErrorResponse error = UserApi.attemptConfirmReset(new ResetConfirmRequest("bad-token", "pass"));
```
