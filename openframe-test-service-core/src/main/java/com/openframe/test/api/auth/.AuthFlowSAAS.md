Implements the `IAuthFlow` interface to handle the full OAuth2 SAAS authentication flow for the OpenFrame test framework, guiding a user through tenant discovery, OAuth login initiation, credential submission, and token extraction via a step-by-step redirect chain.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `AuthFlowSAAS(User user)` | Constructor | Initializes the flow with a target user |
| `discoverTenant()` | Method | Resolves the tenant ID via `TenantApi` using the user's identity |
| `startFlow()` | Method | Initiates the OAuth login redirect at `/oauth/login` with the tenant ID |
| `initAuth()` | Method | Follows the first redirect to initialize the auth provider session |
| `postCredentials()` | Method | Submits username/password form using the active `JSESSIONID` session cookie |
| `getAuthCode()` | Method | Follows the authorization code redirect step |
| `extractTokens()` | Method | Completes the flow and extracts `access_token` and `refresh_token` from response cookies |
| `allCookies` | Field | Accumulates session cookies across all redirect steps |

## Usage Example

```java
User user = new User("admin@example.com", "securepassword");

Map<String, String> tokens = new AuthFlowSAAS(user)
    .discoverTenant()
    .startFlow()
    .initAuth()
    .postCredentials()
    .getAuthCode()
    .extractTokens();

String accessToken  = tokens.get("access_token");
String refreshToken = tokens.get("refresh_token");
```

> Each method follows exactly one redirect step and returns `this`, enabling fluent chaining. Cookies are accumulated across steps to maintain session state throughout the OAuth redirect chain.
