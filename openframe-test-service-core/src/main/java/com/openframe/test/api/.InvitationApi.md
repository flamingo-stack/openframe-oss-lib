Test API client providing convenient methods for testing invitation workflows including user invites, acceptance, and revocation operations.

## Key Components

- **`inviteUser()`** - Creates a new user invitation with authorized credentials, expects 201 status
- **`attemptInviteUser()`** - Tests invitation creation that should fail with conflict (409 status)
- **`acceptInvitation()`** - Processes invitation acceptance without authorization, expects 200 status
- **`attemptAcceptInvitation()`** - Tests invitation acceptance that should fail with conflict (409 status)
- **`revokeInvitation()`** - Deletes an invitation by ID with authorized credentials, expects 204 status

## Usage Example

```java
// Create invitation for a new user
InvitationRequest request = new InvitationRequest("user@example.com", "USER");
Invitation invitation = InvitationApi.inviteUser(request);

// Accept the invitation
AcceptInvitationRequest acceptRequest = new AcceptInvitationRequest(
    invitation.getToken(), "password", "John", "Doe"
);
AcceptInvitationResponse response = InvitationApi.acceptInvitation(acceptRequest);

// Test conflict scenarios
InvitationConflictResponse conflict = InvitationApi.attemptInviteUser(request);
assertThat(conflict.getErrorCode()).isEqualTo("USER_ALREADY_EXISTS");

// Revoke invitation if needed
InvitationApi.revokeInvitation(invitation.getId());
```

This API wrapper handles both successful flows and expected failure scenarios (409 conflicts) for comprehensive invitation testing.
