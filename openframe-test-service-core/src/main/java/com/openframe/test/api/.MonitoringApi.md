REST-Assured API client for interacting with FleetMDM monitoring policy endpoints, providing CRUD operations for device compliance policies within the OpenFrame test framework.

## Key Components

| Method | Description |
|--------|-------------|
| `getPolicy(Integer policyId)` | Fetches a single policy by ID |
| `getPolicies()` | Retrieves all policies as a list |
| `createPolicy(CreatePolicyRequest request)` | Creates a new monitoring policy |
| `deletePolicy(Integer... policyIds)` | Bulk deletes policies by IDs, returns deleted IDs |
| `selectPolicyDevices(Integer policyId, Integer... hostIds)` | Associates host devices with a specific policy |

**Endpoint Constants:**

| Constant | Path |
|----------|------|
| `POLICIES` | `tools/fleetmdm-server/api/latest/fleet/policies` |
| `POLICY` | `…/policies/{id}` |
| `POLICY_DELETE` | `…/policies/delete` |
| `POLICY_HOSTS` | `…/v1/fleet/policies/{id}/hosts` |

## Usage Example

```java
// Create a new policy
CreatePolicyRequest request = new CreatePolicyRequest("Disk Encryption Check", "SELECT 1 FROM disk_encryption;");
Policy created = MonitoringApi.createPolicy(request);

// Retrieve all policies
List<Policy> policies = MonitoringApi.getPolicies();

// Assign hosts to a policy
MonitoringApi.selectPolicyDevices(created.getId(), 101, 102, 103);

// Fetch a specific policy
Policy policy = MonitoringApi.getPolicy(created.getId());

// Delete policies in bulk
List<Integer> deleted = MonitoringApi.deletePolicy(created.getId(), 55, 56);
```

> All methods use `getAuthorizedSpec()` to inject authentication headers automatically, expecting HTTP `200` on success and deserializing responses into typed `Policy` objects.
