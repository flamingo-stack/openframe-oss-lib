A utility class containing static GraphQL query string constants used for device-related API testing in the OpenFrame platform.

## Key Components

| Constant | Description |
|---|---|
| `DEVICE_IDS` | Fetches `machineId` values with optional filter |
| `ALL_DEVICE_IDS` | Paginated fetch of `machineId` values using cursor-based pagination |
| `DEVICE_HOSTNAMES` | Fetches device hostnames with optional filter |
| `FULL_DEVICE` | Fetches complete device details including tags, tool connections, and installed agents |
| `DEVICES_WITH_FILTER` | Fetches a broad set of device fields with filter support, includes `filteredCount` |
| `SEARCH_DEVICE` | Fetches device fields using both a filter and a search string |
| `ALL_DEVICES` | Paginated fetch focused on `toolConnections` data |
| `DEVICE_FILTERS` | Fetches available filter facets (statuses, device types, OS types, etc.) with counts |
| `DEVICE_FILTERS_WITH_FILTER` | Fetches filtered status facets with a `DeviceFilterInput` variable |

## Usage Example

```java
// Using FULL_DEVICE query in a GraphQL test client
Map<String, Object> variables = Map.of("machineId", "machine-abc-123");

GraphQLResponse response = graphQLClient.query(
    DeviceQueries.FULL_DEVICE,
    variables
);

// Using paginated ALL_DEVICE_IDS query
Map<String, Object> pageVars = Map.of(
    "filter", filterInput,
    "first", 50,
    "after", "cursor-token"
);

GraphQLResponse pagedResponse = graphQLClient.query(
    DeviceQueries.ALL_DEVICE_IDS,
    pageVars
);
```

> **Note:** All queries accept a `DeviceFilterInput` variable for scoped filtering. Pagination queries (`ALL_DEVICE_IDS`, `ALL_DEVICES`) follow the [Relay cursor spec](https://relay.dev/graphql/connections.htm) using `first`/`after` with `pageInfo`.
