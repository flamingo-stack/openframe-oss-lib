A utility class containing predefined GraphQL query strings for device-related operations in the OpenFrame test API framework.

## Key Components

- **DEVICE_IDS** - Query to retrieve machine IDs for devices matching a filter
- **DEVICE_HOSTNAMES** - Query to fetch hostnames for filtered devices
- **FULL_DEVICE** - Comprehensive query to get complete device details by machine ID
- **DEVICES_WITH_FILTER** - Query for paginated device list with filtering capabilities
- **SEARCH_DEVICE** - Query combining device filtering with text search functionality
- **DEVICE_FILTERS** - Query to retrieve available filter options and their counts

## Usage Example

```java
// Example usage in a test class
public class DeviceApiTest {
    
    @Test
    public void testGetDeviceById() {
        String query = DeviceQueries.FULL_DEVICE;
        Map<String, Object> variables = Map.of("machineId", "test-machine-123");
        
        GraphQLResponse response = graphQLClient.execute(query, variables);
        Device device = response.extractValueAsObject("data.device", Device.class);
        
        assertThat(device.getMachineId()).isEqualTo("test-machine-123");
        assertThat(device.getHostname()).isNotNull();
    }
    
    @Test
    public void testSearchDevices() {
        String query = DeviceQueries.SEARCH_DEVICE;
        Map<String, Object> variables = Map.of(
            "filter", Map.of("status", "ONLINE"),
            "search", "web-server"
        );
        
        GraphQLResponse response = graphQLClient.execute(query, variables);
        List<Device> devices = response.extractValueAsList("data.devices.edges[*].node", Device.class);
        
        assertThat(devices).isNotEmpty();
    }
}
```

This class centralizes all device-related GraphQL queries, making them reusable across test suites and ensuring consistency in API interactions. Each query is optimized for specific use cases, from simple ID retrieval to comprehensive device management operations.
