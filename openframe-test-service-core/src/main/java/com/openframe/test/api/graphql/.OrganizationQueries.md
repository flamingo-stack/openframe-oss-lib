Defines reusable GraphQL query string constants for fetching organization data in the OpenFrame API test suite.

## Key Components

| Constant | Description |
|---|---|
| `ORGANIZATIONS` | Paginated query returning a list of organizations with full field details and `pageInfo` for cursor-based navigation |
| `ORGANIZATION_BY_ORGANIZATION_ID` | Single-organization lookup by `organizationId`, returning the same full field set |

Both queries retrieve a comprehensive organization payload including contact info, physical/mailing addresses, contract dates, revenue, status, and audit timestamps.

## Usage Example

```java
// Paginated list query
Map<String, Object> variables = new HashMap<>();
variables.put("first", 10);
variables.put("after", null);

GraphQLResponse response = graphQLClient.query(
    OrganizationQueries.ORGANIZATIONS,
    variables
);

// Lookup by organizationId
Map<String, Object> variables = new HashMap<>();
variables.put("organizationId", "org-abc123");

GraphQLResponse response = graphQLClient.query(
    OrganizationQueries.ORGANIZATION_BY_ORGANIZATION_ID,
    variables
);
```

## Notes

- Both queries return identical organization fields for consistency across list and detail views.
- `ORGANIZATIONS` supports cursor-based pagination via `$first` (page size) and `$after` (cursor) variables, with `pageInfo.hasNextPage` and `pageInfo.endCursor` for traversal.
- `ORGANIZATION_BY_ORGANIZATION_ID` requires a non-null `$organizationId: String!` argument.
