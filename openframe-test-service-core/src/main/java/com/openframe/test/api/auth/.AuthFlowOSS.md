Implements the `IAuthFlow` interface for the OSS (open-source) authentication flow, orchestrating a multi-step OAuth2 authorization code exchange for a given user against an OpenFrame tenant.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `AuthFlowOSS(User)` | Constructor | Initializes the flow with a target user |
| `discoverTenant()` | Method | Resolves the tenant ID and generates initial `AuthParts` |
| `startFlow()` | Method | Kicks off the OAuth flow, capturing `state`, `code_challenge`, and session cookies |
| `initAuth()` | Method | Initiates authorization and merges returned cookies into session state |
| `postCredentials()` | Method | Submits user credentials and accumulates session cookies |
| `getAuthCode()` | Method | Follows the redirect to extract the authorization code |
| `extractTokens()` | Method | Exchanges the auth code for `access_token` and `refresh_token` cookies |

## Usage Example

```java
User user = new User("admin@tenant.example.com", "password");

Map<String, String> tokens = new AuthFlowOSS(user)
    .discoverTenant()
    .startFlow()
    .initAuth()
    .postCredentials()
    .getAuthCode()
    .extractTokens();

String accessToken  = tokens.get("access_token");
String refreshToken = tokens.get("refresh_token");
```

Each step in the chain validates the expected HTTP `302` redirect response and propagates cookies and state parameters through the shared `AuthParts` object, making the full OAuth code-exchange flow reproducible in automated API tests.
