A utility class containing static GraphQL query and mutation string constants for all Knowledge Base API operations used in integration tests.

## Key Components

| Constant | Type | Description |
|---|---|---|
| `KNOWLEDGE_BASE_ITEM` | Query | Fetch a single KB item by ID with author and tags |
| `KNOWLEDGE_BASE_FOLDER_TREE` | Query | Retrieve full folder hierarchy |
| `KNOWLEDGE_BASE_ARTICLE_TREE` | Query | Retrieve full article hierarchy with status |
| `KNOWLEDGE_BASE_TAGS` | Query | List tags, optionally filtered by folder |
| `KNOWLEDGE_BASE_ITEMS` | Query | Paginated, filterable list of KB items |
| `ARCHIVED_ARTICLES` | Query | Paginated list of archived articles |
| `CREATE_ARTICLE` | Mutation | Create a new article with full author/tag response |
| `UPDATE_ARTICLE` | Mutation | Update article content, name, and summary |
| `PUBLISH_ARTICLE` | Mutation | Transition article to published status |
| `ARCHIVE_ARTICLE` | Mutation | Archive an article |
| `UNARCHIVE_ARTICLE` | Mutation | Restore an archived article to a folder |
| `CREATE_FOLDER` | Mutation | Create a folder, optionally nested |
| `RENAME_FOLDER` | Mutation | Rename an existing folder |
| `MOVE_TO_FOLDER` | Mutation | Move an item to a different folder |
| `DELETE_FOLDER` | Mutation | Delete a folder via input object |
| `ADD_TAG_TO_ITEM` | Mutation | Associate a tag with a KB item |

Private field fragments (`FOLDER_FIELDS`, `ARTICLE_LIFECYCLE_FIELDS`, `TAG_OP_FIELDS`) are reused via `String.formatted()` to keep response shapes consistent across related mutations.

## Usage Example

```java
// Executing a paginated knowledge base items query
Map<String, Object> variables = Map.of(
    "first", 10,
    "after", null,
    "search", "VPN setup",
    "filter", Map.of("status", "PUBLISHED")
);

graphQLClient.execute(KnowledgeBaseQueries.KNOWLEDGE_BASE_ITEMS, variables);

// Creating a new article
Map<String, Object> input = Map.of(
    "name", "How to Reset MFA",
    "content", "Step-by-step guide...",
    "parentId", "folder-123"
);

graphQLClient.execute(
    KnowledgeBaseQueries.CREATE_ARTICLE,
    Map.of("input", input)
);
```
