GraphQL query string constants for the Logs API, used in integration/API tests to query log data, filters, and detailed log entries.

## Key Components

| Constant | Description |
|---|---|
| `LOG_FILTERS` | Fetches available filter options (tool types, event types, severities, organizations) for the log listing UI |
| `LOGS` | Paginated log listing query supporting filter and search inputs, returning edges with cursor-based pagination and `pageInfo` |
| `LOG_DETAILS` | Fetches full details of a single log entry identified by `ingestDay`, `toolType`, `eventType`, `timestamp`, and `toolEventId` |

## Usage Example

```java
// Fetch available log filters
Map<String, Object> variables = new HashMap<>();
variables.put("filter", Map.of("organizationId", "org-123"));

GraphQLResponse response = graphQLClient.query(LogQueries.LOG_FILTERS, variables);

// Paginated log search
Map<String, Object> logVars = new HashMap<>();
logVars.put("search", "authentication failure");
logVars.put("filter", Map.of("severity", "HIGH"));

GraphQLResponse logs = graphQLClient.query(LogQueries.LOGS, logVars);

// Fetch single log detail
Map<String, Object> detailVars = Map.of(
    "ingestDay", "2024-01-15",
    "toolType",  "ENDPOINT",
    "eventType", "ALERT",
    "timestamp", Instant.parse("2024-01-15T10:30:00Z"),
    "toolEventId", "evt-abc123"
);

GraphQLResponse detail = graphQLClient.query(LogQueries.LOG_DETAILS, detailVars);
```

> **Note:** `LOG_DETAILS` requires all five arguments (`ingestDay`, `toolType`, `eventType`, `timestamp`, `toolEventId`) as they are marked non-null in the GraphQL schema. The `details` and `message` fields are only available on the detail query, not the list query.
