Contains GraphQL query constants for OpenFrame log management operations including filtering, pagination, and detailed log retrieval.

## Key Components

- **LOG_FILTERS** - Query to retrieve available filter options (tool types, event types, severities, organizations)
- **LOGS** - Paginated query for log entries with filtering and search capabilities
- **LOG_DETAILS** - Query to fetch detailed information for a specific log entry

## Usage Example

```java
// Execute log filters query to populate UI dropdowns
String filtersQuery = LogQueries.LOG_FILTERS;
GraphQLRequest filterRequest = new GraphQLRequest(filtersQuery, filterVariables);

// Query logs with pagination and filtering
String logsQuery = LogQueries.LOGS;
Map<String, Object> variables = Map.of(
    "filter", Map.of(
        "toolTypes", List.of("MONITORING", "SECURITY"),
        "severities", List.of("ERROR", "WARN")
    ),
    "search", "authentication failed"
);
GraphQLRequest logsRequest = new GraphQLRequest(logsQuery, variables);

// Get detailed view of specific log entry
String detailsQuery = LogQueries.LOG_DETAILS;
Map<String, Object> detailVariables = Map.of(
    "ingestDay", "2024-01-15",
    "toolType", "MONITORING",
    "eventType", "ALERT",
    "timestamp", Instant.now(),
    "toolEventId", "log-123"
);
GraphQLRequest detailRequest = new GraphQLRequest(detailsQuery, detailVariables);
```

The queries support OpenFrame's unified log aggregation system, enabling technicians to filter across multiple MSP tools and access detailed troubleshooting information through a single interface.
