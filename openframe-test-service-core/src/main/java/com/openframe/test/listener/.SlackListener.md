A JUnit 5 `TestExecutionListener` that aggregates test results and posts a formatted summary report to Slack, tracking pass/fail/skip counts across a test plan execution.

## Key Components

| Member | Description |
|---|---|
| `SlackListener(SlackClient)` | Constructor accepting a `SlackClient` for posting messages |
| `testPlanExecutionStarted(TestPlan)` | Resets counters and result list when a new test plan begins |
| `executionStarted(TestIdentifier)` | Increments the discovered test counter |
| `executionFinished(TestIdentifier, TestExecutionResult)` | Records SUCCESSFUL / FAILED / ABORTED outcomes with Slack emoji prefixes |
| `executionSkipped(TestIdentifier, String)` | Logs and records skipped tests with reason |
| `truncateMessage(String)` | Trims failure messages to the first line, capped at 300 characters |
| `sendResults(String tag, String domain, String baseUrl)` | Builds and dispatches a threaded Slack report with summary + per-test details |

## Usage Example

```java
// Register the listener programmatically via the JUnit Launcher API
SlackClient slackClient = new SlackClient(webhookUrl);
SlackListener slackListener = new SlackListener(slackClient);

LauncherDiscoveryRequest request = LauncherDiscoveryRequestBuilder.request()
    .selectors(selectPackage("com.openframe.test"))
    .build();

Launcher launcher = LauncherFactory.create();
launcher.discover(request);
launcher.execute(request, slackListener);

// After execution, send the aggregated report
slackListener.sendResults("@smoke", "acme", "flamingo.run");
// Posts to Slack:
// *Test Report* 🟢
// Environment: `https://acme.flamingo.run`
// Tag: @smoke
```

> **Note:** `sendResults` must be called explicitly after test execution completes. The `reported` flag prevents double-reporting if `testPlanExecutionStarted` fires multiple times before results are flushed.
