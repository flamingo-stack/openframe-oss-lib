Utility class for retrieving password reset tokens from a Redis cache, used in OpenFrame's test data layer.

## Key Components

| Element | Description |
|---|---|
| `REDIS_URI` | Hardcoded connection URI targeting the `redis-master` service in the `datasources` Kubernetes namespace |
| `KEY` | Scan pattern `of:{default}:pwdreset:*` used to locate all password reset entries |
| `getResetToken(String email)` | Scans Redis keys matching the reset pattern and returns the token associated with the given email |

## Usage Example

```java
// Retrieve the password reset token for a given user email
String token = Redis.getResetToken("user@example.com");

if (token != null) {
    System.out.println("Reset token found: " + token);
} else {
    System.out.println("No reset token found for this email.");
}
```

## Notes

- Uses a **cursor-based SCAN** loop to safely iterate large key sets without blocking Redis, stopping early once a match is found.
- The token is extracted by splitting the matched key on `:pwdreset:` and taking the second segment.
- The Redis connection is explicitly closed after each call via `redis.close()`.
- This class is intended for **test automation use only** — it connects directly to the internal cluster Redis service and is not suitable for production use.
