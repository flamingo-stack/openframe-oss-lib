A simple Data Transfer Object (DTO) representing a user entity within the Fleet device management context, used for deserializing JSON responses in OpenFrame test scenarios.

## Key Components

| Element | Description |
|---------|-------------|
| `username` | The identifier string for the fleet user |
| `type` | The role or classification type of the user |

**Lombok Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables the builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unrecognized JSON fields during deserialization

## Usage Example

```java
// Builder pattern
FleetUser user = FleetUser.builder()
    .username("john.doe")
    .type("admin")
    .build();

// No-args constructor with setters
FleetUser user = new FleetUser();
user.setUsername("jane.doe");
user.setType("technician");

// All-args constructor
FleetUser user = new FleetUser("john.doe", "admin");

// JSON deserialization (via Jackson)
// { "username": "john.doe", "type": "admin", "extraField": "ignored" }
FleetUser user = objectMapper.readValue(jsonString, FleetUser.class);
```

> **Note:** The `@JsonIgnoreProperties(ignoreUnknown = true)` annotation makes this DTO resilient to API changes — additional fields returned by the Fleet service will not cause deserialization failures.
