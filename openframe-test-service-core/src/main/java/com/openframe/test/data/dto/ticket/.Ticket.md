Data Transfer Object (DTO) representing a support ticket entity within the OpenFrame test data layer, mapping ticket attributes returned from the API.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique ticket identifier |
| `ticketNumber` | `Integer` | Human-readable ticket number |
| `status` | `String` | Current ticket status |
| `owner` | `TicketOwner` | Ticket owner details |
| `labels` | `List<TicketLabel>` | Associated labels |
| `dialog` | `TicketDialog` | Conversation thread |
| `attachments` | `List<TicketAttachment>` | File attachments |
| `notes` | `List<TicketNote>` | Internal notes |
| `createdAt` / `updatedAt` / `resolvedAt` | `Instant` | Lifecycle timestamps |

**Lombok Annotations:** `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` generate boilerplate (getters, setters, builder pattern, constructors).  
**`@JsonIgnoreProperties(ignoreUnknown = true)`** ensures forward compatibility by silently ignoring unmapped API response fields.

## Usage Example

```java
// Build a ticket using the Lombok builder
Ticket ticket = Ticket.builder()
    .id("ticket-001")
    .ticketNumber(42)
    .title("Network connectivity issue")
    .status("OPEN")
    .organizationId("org-123")
    .organizationName("Acme Corp")
    .createdAt(Instant.now())
    .build();

// Access fields via generated getters
String title = ticket.getTitle();
List<TicketNote> notes = ticket.getNotes();

// Deserialize from API JSON response (unknown fields are ignored)
Ticket fromApi = objectMapper.readValue(apiResponse, Ticket.class);
```
