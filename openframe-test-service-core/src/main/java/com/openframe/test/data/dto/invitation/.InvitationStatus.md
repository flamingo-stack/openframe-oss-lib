Enumeration defining the possible states of an invitation within the OpenFrame test system, used to track invitation lifecycle from creation to final resolution.

## Key Components

- **PENDING** - Initial state when invitation is created but not yet responded to
- **ACCEPTED** - State when recipient has accepted the invitation
- **REVOKED** - State when invitation has been cancelled or withdrawn

## Usage Example

```java
// Create invitation with initial status
InvitationStatus status = InvitationStatus.PENDING;

// Check invitation state
if (status == InvitationStatus.PENDING) {
    // Send reminder or allow acceptance
    processActivInvitation();
}

// Update status based on user action
status = InvitationStatus.ACCEPTED;

// Handle revoked invitations
if (status == InvitationStatus.REVOKED) {
    cleanupInvitationResources();
}

// Use in switch statements for state handling
switch (status) {
    case PENDING -> handlePendingInvitation();
    case ACCEPTED -> processAcceptedInvitation();
    case REVOKED -> handleRevokedInvitation();
}
```

This enum provides type-safe status management for invitation workflows, ensuring consistent state representation across the application and preventing invalid status assignments.
