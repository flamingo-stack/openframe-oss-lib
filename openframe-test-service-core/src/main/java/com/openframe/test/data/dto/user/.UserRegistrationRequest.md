A test data transfer object (DTO) representing the payload structure for user registration API requests in integration/automated tests.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `email` | `String` | User's email address |
| `firstName` | `String` | User's first name |
| `lastName` | `String` | User's last name |
| `password` | `String` | User's password |
| `tenantName` | `String` | Name of the tenant/organization |
| `tenantDomain` | `String` | Domain associated with the tenant |
| `accessCode` | `String` | Optional access/invite code |

**Lombok Annotations:**
- `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder(toBuilder = true)` — enables builder pattern and object cloning via `toBuilder()`
- `@NoArgsConstructor` / `@AllArgsConstructor` — generates both default and full constructors
- `@JsonInclude(NON_NULL)` — omits `null` fields from serialized JSON payloads

## Usage Example

```java
// Full registration payload
UserRegistrationRequest request = UserRegistrationRequest.builder()
    .email("john.doe@acme.com")
    .firstName("John")
    .lastName("Doe")
    .password("SecurePass123!")
    .tenantName("Acme Corp")
    .tenantDomain("acme.com")
    .accessCode("INVITE-XYZ")
    .build();

// Clone and override a field using toBuilder()
UserRegistrationRequest noCode = request.toBuilder()
    .accessCode(null)
    .build();
```

> `@JsonInclude(NON_NULL)` ensures that optional fields like `accessCode` are cleanly excluded from the request body when not set, keeping API payloads minimal during tests.
