A data transfer object (DTO) representing pagination metadata for ticket list responses, encapsulating cursor-based navigation information.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `hasNextPage` | `Boolean` | Indicates if a next page of tickets exists |
| `hasPreviousPage` | `Boolean` | Indicates if a previous page of tickets exists |
| `startCursor` | `String` | Cursor pointing to the start of the current page |
| `endCursor` | `String` | Cursor pointing to the end of the current page |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables the builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unrecognized JSON fields during deserialization

## Usage Example

```java
// Build via builder pattern
TicketPageInfo pageInfo = TicketPageInfo.builder()
    .hasNextPage(true)
    .hasPreviousPage(false)
    .startCursor("cursor_abc123")
    .endCursor("cursor_xyz789")
    .build();

// Check pagination state
if (pageInfo.getHasNextPage()) {
    fetchNextPage(pageInfo.getEndCursor());
}
```

> This DTO is typically used alongside a ticket list payload to support cursor-based pagination, allowing clients to navigate forward and backward through paginated ticket results without relying on offset-based indexing.
