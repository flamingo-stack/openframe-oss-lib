A data transfer object (DTO) representing the response payload returned after successful user registration in the OpenFrame platform.

## Key Components

- **User Identity Fields**: `id`, `name`, `ownerId` - Core user identification data
- **Domain Management**: `domain` - Associated domain for the registered user
- **Account Status**: `status`, `active` - Current account state and activation status
- **Subscription Info**: `plan` - User's subscription plan level
- **CRM Integration**: `hubspotId` - External CRM system identifier for sales tracking
- **Audit Trail**: `createdAt`, `updatedAt` - Registration and modification timestamps
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` - Auto-generated getters, setters, constructors, and builder pattern

## Usage Example

```java
// Building a registration response
UserRegistrationResponse response = UserRegistrationResponse.builder()
    .id("usr_123456789")
    .name("Acme Corp")
    .domain("acme.example.com")
    .ownerId("owner_987654321")
    .status("ACTIVE")
    .plan("PROFESSIONAL")
    .hubspotId("hs_contact_456789")
    .active(true)
    .createdAt("2024-01-15T10:30:00Z")
    .updatedAt("2024-01-15T10:30:00Z")
    .build();

// Accessing response data
String userId = response.getId();
Boolean isActive = response.getActive();
String subscriptionPlan = response.getPlan();
```

This DTO is typically returned by REST endpoints handling user registration workflows, providing clients with comprehensive information about the newly created user account including subscription details and CRM integration data.
