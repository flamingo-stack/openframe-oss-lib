A Data Transfer Object (DTO) that models the standard error response structure returned by the OpenFrame API, capturing the error code and descriptive message.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `code` | `String` | Machine-readable error identifier (e.g., `BAD_REQUEST`, `VALIDATION_ERROR`) |
| `message` | `String` | Human-readable error description |

**Lombok Annotations:**
- `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — enables fluent builder pattern
- `@NoArgsConstructor` / `@AllArgsConstructor` — generates both default and full constructors

**Jackson Annotation:**
- `@JsonIgnoreProperties(ignoreUnknown = true)` — tolerates extra fields in API responses without throwing deserialization errors

## Usage Example

```java
// Deserializing from API response JSON:
// {"code":"BAD_REQUEST","message":"Registration is closed for this organization"}

ErrorResponse error = objectMapper.readValue(responseBody, ErrorResponse.class);

System.out.println(error.getCode());    // "BAD_REQUEST"
System.out.println(error.getMessage()); // "Registration is closed for this organization"

// Building manually in tests
ErrorResponse mockError = ErrorResponse.builder()
    .code("VALIDATION_ERROR")
    .message("Email address is invalid")
    .build();

// Asserting in test context
assertEquals("BAD_REQUEST", error.getCode());
assertNotNull(error.getMessage());
```

## Notes

- `@JsonIgnoreProperties(ignoreUnknown = true)` makes this DTO forward-compatible — if the API adds new fields to error responses, deserialization will not fail.
- Intended for use in automated test suites validating OpenFrame API error handling behavior.
