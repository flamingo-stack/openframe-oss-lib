A simple DTO for cursor-based pagination request parameters, used to pass `limit` and `cursor` values in API or test data queries.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `limit` | `Integer` | Maximum number of records to return per page |
| `cursor` | `String` | Opaque pointer to the current position in the dataset |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonInclude(NON_NULL)` — Omits `null` fields from JSON serialization

## Usage Example

```java
// Builder pattern
CursorPaginationInput pagination = CursorPaginationInput.builder()
    .limit(25)
    .cursor("eyJpZCI6MTAwfQ==")
    .build();

// First page (no cursor)
CursorPaginationInput firstPage = CursorPaginationInput.builder()
    .limit(10)
    .build(); // cursor is null and will be excluded from JSON output
```

> **Note:** Because `@JsonInclude(NON_NULL)` is applied, omitting `cursor` on the first page request will produce clean JSON without a `null` cursor field — e.g., `{"limit": 10}` instead of `{"limit": 10, "cursor": null}`.
