Data Transfer Object (DTO) representing a user invitation within the OpenFrame test data layer, encapsulating invitation metadata including recipient email, assigned roles, expiry, and lifecycle status.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier for the invitation |
| `email` | `String` | Recipient's email address |
| `roles` | `List<UserRole>` | Assigned roles (defaults to empty list) |
| `expiresAt` | `Instant` | Invitation expiration timestamp |
| `status` | `InvitationStatus` | Current state (defaults to `PENDING`) |
| `createdAt` | `Instant` | Creation timestamp |
| `updatedAt` | `Instant` | Last update timestamp |

## Usage Example

```java
// Build a new invitation with roles and expiry
Invitation invitation = Invitation.builder()
    .id("inv-001")
    .email("technician@example.com")
    .roles(List.of(UserRole.TECHNICIAN))
    .expiresAt(Instant.now().plusSeconds(86400))
    .status(InvitationStatus.PENDING)
    .build();

// Default construction with no-args constructor
Invitation emptyInvitation = new Invitation();
// roles defaults to new ArrayList<>()
// status defaults to InvitationStatus.PENDING

// Deserialize from API response (unknown JSON fields are ignored)
Invitation fromJson = objectMapper.readValue(jsonResponse, Invitation.class);
```

## Notes

- Annotated with `@JsonIgnoreProperties(ignoreUnknown = true)` for safe deserialization of API responses containing extra fields.
- `@Builder.Default` ensures `roles` and `status` are never `null` when using the builder pattern.
- Relies on `InvitationStatus` enum and `UserRole` enum for type-safe field values.
