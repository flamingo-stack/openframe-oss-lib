A DTO (Data Transfer Object) that represents a device along with its associated tool connections, used for deserializing API responses in test scenarios.

## Key Components

| Element | Type | Description |
|---|---|---|
| `toolConnections` | `List<ToolConnection>` | List of tool connections associated with the device |

**Lombok Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables the builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unrecognized JSON fields during deserialization

## Usage Example

```java
// Builder pattern
DeviceWithConnections device = DeviceWithConnections.builder()
    .toolConnections(List.of(
        new ToolConnection("connectwise"),
        new ToolConnection("ninjaone")
    ))
    .build();

// Accessing connections
List<ToolConnection> connections = device.getToolConnections();

// Deserialization via Jackson (e.g., in a test)
DeviceWithConnections response = objectMapper.readValue(jsonResponse, DeviceWithConnections.class);
```

> **Note:** This class resides in the `com.openframe.test.data.dto.device` package, indicating it is part of the [OpenFrame](https://openframe.ai) test data layer. It is typically used to validate device-connection relationships in integration or API tests.
