A data transfer object (DTO) representing a dialog/conversation context associated with a ticket, including mode state and token consumption tracking.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier for the dialog |
| `currentMode` | `String` | Active mode of the dialog (e.g., AI response mode) |
| `tokenUsage` | `List<TicketTokenUsage>` | List of token usage records for the dialog |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern instantiation
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unmapped JSON fields during deserialization

## Usage Example

```java
// Builder pattern
TicketDialog dialog = TicketDialog.builder()
    .id("dialog-001")
    .currentMode("ai-assist")
    .tokenUsage(List.of(
        TicketTokenUsage.builder().tokens(150).build()
    ))
    .build();

// Access fields
String mode = dialog.getCurrentMode();
List<TicketTokenUsage> usage = dialog.getTokenUsage();
```
