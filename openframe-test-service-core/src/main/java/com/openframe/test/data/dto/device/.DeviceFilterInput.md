A DTO (Data Transfer Object) used to encapsulate filter criteria when querying devices in the OpenFrame test layer.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `statuses` | `List<DeviceStatus>` | Filter by one or more device status values |
| `deviceTypes` | `List<DeviceType>` | Filter by device type classifications |
| `osTypes` | `List<String>` | Filter by operating system types |
| `organizationIds` | `List<String>` | Scope results to specific organizations |
| `tagNames` | `List<String>` | Filter by assigned tag names |

**Annotations:**
- `@Data` / `@Builder` / `@NoArgsConstructor` / `@AllArgsConstructor` — Lombok-generated boilerplate
- `@JsonInclude(NON_NULL)` — Null fields are omitted from serialization, keeping payloads clean

## Usage Example

```java
// Build a filter for active Windows workstations in a specific org
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.WORKSTATION))
    .osTypes(List.of("Windows"))
    .organizationIds(List.of("org-abc-123"))
    .tagNames(List.of("managed", "patched"))
    .build();

// Pass to a device query service
List<Device> devices = deviceService.findDevices(filter);
```

> **Note:** All filter fields are optional. `@JsonInclude(NON_NULL)` ensures unset filters are excluded from the serialized JSON body, avoiding unintended empty-list filtering on the API side.
