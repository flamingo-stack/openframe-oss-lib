A data transfer object (DTO) for filtering device queries in the OpenFrame test platform, providing comprehensive filtering criteria for device management operations.

## Key Components

**Main Class:**
- `DeviceFilterInput` - Filter criteria container for device queries

**Filter Fields:**
- `statuses` - Filter by device status (active, inactive, etc.)
- `deviceTypes` - Filter by device type classification
- `osTypes` - Filter by operating system types
- `organizationIds` - Filter by organization ownership
- `tagNames` - Filter by assigned device tags

**Lombok Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode
- `@Builder` - Enables builder pattern for object construction
- `@NoArgsConstructor/@AllArgsConstructor` - Constructor generation

## Usage Example

```java
// Create filter for Windows devices in specific organizations
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.MAINTENANCE))
    .deviceTypes(List.of(DeviceType.LAPTOP, DeviceType.DESKTOP))
    .osTypes(List.of("Windows 11", "Windows 10"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "critical"))
    .build();

// Use in device service
List<Device> filteredDevices = deviceService.findDevices(filter);

// Empty filter for all devices
DeviceFilterInput allDevices = DeviceFilterInput.builder().build();
```

This DTO supports flexible device filtering across multiple dimensions, enabling complex queries for device management and monitoring operations in the OpenFrame platform.
