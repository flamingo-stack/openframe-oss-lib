Data Transfer Object (DTO) representing the request payload for accepting a user invitation in the OpenFrame platform.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `invitationId` | `String` | Unique identifier of the invitation being accepted |
| `firstName` | `String` | First name of the user accepting the invitation |
| `lastName` | `String` | Last name of the user accepting the invitation |
| `password` | `String` | Password to set for the new account |
| `switchTenant` | `boolean` | Whether to switch to the invited tenant context; defaults to `false` |

## Key Behaviors

- **`@JsonInclude(NON_NULL)`** — Null fields are omitted from JSON serialization, keeping payloads lean
- **`@Builder.Default`** — Ensures `switchTenant` defaults to `false` even when using the Lombok builder pattern
- Lombok annotations (`@Data`, `@Builder`, `@AllArgsConstructor`, `@NoArgsConstructor`) auto-generate getters, setters, constructors, and builder

## Usage Example

```java
// Accepting an invitation with full account setup
AcceptInvitationRequest request = AcceptInvitationRequest.builder()
    .invitationId("inv-abc123")
    .firstName("Jane")
    .lastName("Doe")
    .password("SecureP@ss!")
    .switchTenant(true)
    .build();

// Minimal acceptance (tenant switch disabled by default)
AcceptInvitationRequest minimal = AcceptInvitationRequest.builder()
    .invitationId("inv-xyz789")
    .firstName("John")
    .lastName("Smith")
    .password("AnotherP@ss!")
    .build();
// minimal.isSwitchTenant() == false
```
