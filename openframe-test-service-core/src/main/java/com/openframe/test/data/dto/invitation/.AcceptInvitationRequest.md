A Data Transfer Object (DTO) that encapsulates the request payload for accepting user invitations in the OpenFrame platform. This class defines the structure for invitation acceptance operations with user details and tenant switching options.

## Key Components

**Main Properties:**
- `invitationId` - Unique identifier for the invitation being accepted
- `firstName` - User's first name for account setup
- `lastName` - User's last name for account setup  
- `password` - User-defined password for the new account
- `switchTenant` - Boolean flag to switch to the inviting tenant (defaults to false)

**Lombok Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode methods
- `@Builder` - Enables builder pattern for object construction
- `@AllArgsConstructor` / `@NoArgsConstructor` - Generates constructors

## Usage Example

```java
// Create invitation acceptance request using builder pattern
AcceptInvitationRequest request = AcceptInvitationRequest.builder()
    .invitationId("inv-12345")
    .firstName("John")
    .lastName("Doe")
    .password("securePassword123")
    .switchTenant(true)
    .build();

// Alternative constructor approach
AcceptInvitationRequest request2 = new AcceptInvitationRequest(
    "inv-67890", 
    "Jane", 
    "Smith", 
    "myPassword456", 
    false
);

// Access properties
String inviteId = request.getInvitationId();
boolean willSwitch = request.isSwitchTenant();
```

The `switchTenant` field uses `@Builder.Default` to automatically set `false` as the default value when using the builder pattern, allowing users to optionally switch to the inviting organization's tenant upon acceptance.
