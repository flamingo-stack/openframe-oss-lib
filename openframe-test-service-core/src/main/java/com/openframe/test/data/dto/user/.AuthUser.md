A data transfer object (DTO) representing an authenticated user in the OpenFrame system, containing authentication details, profile information, and authorization data.

## Key Components

### Fields
- **`id`** - Unique user identifier
- **`tenantId`** - Multi-tenant organization identifier
- **`passwordHash`** - Encrypted password storage
- **`emailVerified`** - Email verification status flag
- **`loginProvider`** - Authentication provider (OAuth, SAML, etc.)
- **`email`** - User's email address
- **`firstName/lastName`** - User profile information
- **`roles`** - List of assigned user roles for authorization
- **`status`** - Current user account status

### Annotations
- **`@Data`** - Generates getters, setters, toString, equals, and hashCode
- **`@Builder`** - Enables fluent builder pattern for object creation
- **`@NoArgsConstructor/@AllArgsConstructor`** - Generates constructors

## Usage Example

```java
// Create authenticated user with builder pattern
AuthUser authUser = AuthUser.builder()
    .id("user-123")
    .tenantId("tenant-456")
    .email("user@example.com")
    .firstName("John")
    .lastName("Doe")
    .emailVerified(true)
    .loginProvider("OAuth2")
    .roles(List.of(UserRole.USER, UserRole.ADMIN))
    .status(UserStatus.ACTIVE)
    .build();

// Access user information
String userEmail = authUser.getEmail();
boolean isVerified = authUser.getEmailVerified();
List<UserRole> userRoles = authUser.getRoles();
```

This DTO is typically used in authentication flows, JWT token payloads, and authorization checks throughout the OpenFrame MSP platform.
