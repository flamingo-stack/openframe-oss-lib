Provides static utility methods for querying the `users` MongoDB collection, returning `AuthUser` documents filtered by various criteria such as ID, status, and role.

## Key Components

| Method | Description |
|--------|-------------|
| `findUser()` | Returns the first user document in the collection |
| `findUser(String id)` | Finds a user by their `_id` field |
| `findUser(UserStatus status)` | Finds an `ADMIN` user matching the given status |
| `findUser(UserRole role)` | Finds an `ACTIVE` user matching the given role |
| `findUser(String field, String value)` | Finds a user by any arbitrary field/value pair |
| `findUser(UserStatus status, UserRole role)` | Finds a user matching both status and role |

## Usage Example

```java
// Fetch the first available user
AuthUser anyUser = UsersCollection.findUser();

// Fetch a user by ID
AuthUser userById = UsersCollection.findUser("abc123");

// Fetch an active technician
AuthUser activeTech = UsersCollection.findUser(UserStatus.ACTIVE, UserRole.TECHNICIAN);

// Fetch any active admin (default role)
AuthUser activeAdmin = UsersCollection.findUser(UserStatus.ACTIVE);

// Fetch by custom field
AuthUser byEmail = UsersCollection.findUser("email", "tech@example.com");
```

> All methods return the **first matching document** or `null` if no match is found. Queries are composed using MongoDB's `Filters.eq` and `Filters.and` / `Filters.in` combinators via the `MongoDB.getCollection` helper.
