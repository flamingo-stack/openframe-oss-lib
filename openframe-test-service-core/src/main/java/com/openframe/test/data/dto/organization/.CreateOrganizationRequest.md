Data Transfer Object (DTO) for submitting a new organization creation request to the OpenFrame backend. The `organizationId` is auto-generated as a UUID server-side and is not included in this request.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | Organization name |
| `category` | `String` | Business category |
| `numberOfEmployees` | `Integer` | Headcount |
| `websiteUrl` | `String` | Organization website |
| `notes` | `String` | Free-text notes |
| `contactInformation` | `ContactInformationDto` | Nested contact details |
| `monthlyRevenue` | `String` | Monthly revenue value |
| `contractStartDate` | `LocalDate` | Contract start date |
| `contractEndDate` | `LocalDate` | Contract end date |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern for construction
- `@JsonInclude(NON_NULL)` — Omits `null` fields from serialization

## Usage Example

```java
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Acme Corp")
    .category("IT Services")
    .numberOfEmployees(150)
    .websiteUrl("https://acme.com")
    .monthlyRevenue("25000")
    .contractStartDate(LocalDate.of(2024, 1, 1))
    .contractEndDate(LocalDate.of(2025, 1, 1))
    .contactInformation(ContactInformationDto.builder()
        .email("contact@acme.com")
        .phone("+1-555-0100")
        .build())
    .build();
```

> All fields are optional at the DTO level (`@JsonInclude(NON_NULL)` ensures unset fields are excluded from the JSON payload). Backend validation rules determine which fields are required.
