Data Transfer Object for creating new organizations in the OpenFrame test suite. This request DTO captures all necessary information for organization creation, with automatic UUID generation handled by the backend.

## Key Components

**Main Properties:**
- `name` - Organization name
- `category` - Business category/type
- `numberOfEmployees` - Company size indicator
- `websiteUrl` - Organization website
- `notes` - Additional information
- `contactInformation` - Embedded contact details (ContactInformationDto)
- `monthlyRevenue` - Revenue information as string
- `contractStartDate`/`contractEndDate` - Contract period dates

**Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode methods
- `@Builder` - Enables builder pattern for object creation

## Usage Example

```java
CreateOrganizationRequest request = CreateOrganizationRequest.builder()
    .name("Acme Corporation")
    .category("Technology")
    .numberOfEmployees(150)
    .websiteUrl("https://acme.com")
    .monthlyRevenue("$50,000")
    .contractStartDate(LocalDate.of(2024, 1, 1))
    .contractEndDate(LocalDate.of(2024, 12, 31))
    .notes("Key client for Q1 2024")
    .contactInformation(contactDto)
    .build();

// Use in service layer or API endpoints
organizationService.createOrganization(request);
```

The organizationId field is intentionally omitted as it's automatically generated as a UUID by the backend service, ensuring unique identification for each new organization.
