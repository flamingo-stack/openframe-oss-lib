Provides static factory methods for generating test data related to scripts, including script creation, execution, scheduling, and response formatting for use in automated tests.

## Key Components

| Method | Description |
|--------|-------------|
| `addScriptRequest()` | Builds a `Script` object representing a PowerShell directory-listing script with predefined metadata |
| `addScriptResponse(Script)` | Returns the expected API response string after a script is added |
| `runSpeedTestScriptRequest(String...)` | Builds a `RunScriptRequest` targeting specific agents using script ID `14` |
| `createOnlineCheckSchedule()` | Builds a `CreateScriptScheduleRequest` for a one-time network online check scheduled 5 minutes from now |
| `editScriptResponse(Script)` | Returns the expected API response string after a script is edited |

## Usage Example

```java
// Generate a script creation payload
Script script = ScriptGenerator.addScriptRequest();
String expectedAddResponse = ScriptGenerator.addScriptResponse(script);
// expectedAddResponse → "Dir was added!"

// Generate a run request targeting specific agents
RunScriptRequest runRequest = ScriptGenerator.runSpeedTestScriptRequest("agent-001", "agent-002");

// Generate a scheduled task for an online check
CreateScriptScheduleRequest schedule = ScriptGenerator.createOnlineCheckSchedule();
// Scheduled ~5 minutes from now, targets windows, pings 8.8.8.8

// Generate an edit response
String expectedEditResponse = ScriptGenerator.editScriptResponse(script);
// expectedEditResponse → "Dir was edited!"
```

> **Note:** `createOnlineCheckSchedule()` uses `Instant.now()` at call time, so the scheduled `runTimeDate` is dynamic. Avoid caching the result across test steps if timing precision matters.
