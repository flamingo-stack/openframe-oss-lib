Data Transfer Object (DTO) representing the response payload for the "current user" (`/me`) endpoint, encapsulating the authenticated user's details and authentication status.

## Key Components

| Field | Type | Description |
|---|---|---|
| `user` | `User` | The authenticated user object |
| `authenticated` | `boolean` | Whether the current session is authenticated |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern for object construction
- `@AllArgsConstructor` / `@NoArgsConstructor` — Generates full and no-arg constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unrecognized JSON fields during deserialization

## Usage Example

```java
// Deserializing a /me API response
MeResponse meResponse = objectMapper.readValue(jsonBody, MeResponse.class);

if (meResponse.isAuthenticated()) {
    User currentUser = meResponse.getUser();
    System.out.println("Logged in as: " + currentUser.getName());
}

// Building a MeResponse using the builder pattern
MeResponse response = MeResponse.builder()
    .user(existingUser)
    .authenticated(true)
    .build();
```

> **Note:** The `@JsonIgnoreProperties(ignoreUnknown = true)` annotation makes this DTO resilient to API changes — extra fields returned by the server won't cause deserialization failures, which is useful in test environments targeting evolving APIs.
