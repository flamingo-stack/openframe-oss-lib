A data transfer object (DTO) that encapsulates filtering options and metadata for device queries in the OpenFrame testing framework. This class provides a structured way to represent available filter criteria and the count of devices that match the applied filters.

## Key Components

- **Filter Lists**: Contains lists of `DeviceFilterOption` objects for different device attributes (status, type, OS, organization)
- **Tag Filters**: Specialized `TagFilterOption` list for tag-based filtering
- **Result Metadata**: `filteredCount` field to track the number of devices matching current filter criteria
- **Lombok Annotations**: Uses `@Data`, `@Builder`, `@NoArgsConstructor`, and `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Build device filters for a query interface
DeviceFilters filters = DeviceFilters.builder()
    .statuses(List.of(
        new DeviceFilterOption("ACTIVE", "Active Devices"),
        new DeviceFilterOption("INACTIVE", "Inactive Devices")
    ))
    .deviceTypes(List.of(
        new DeviceFilterOption("LAPTOP", "Laptops"),
        new DeviceFilterOption("DESKTOP", "Desktops")
    ))
    .osTypes(List.of(
        new DeviceFilterOption("WINDOWS", "Windows"),
        new DeviceFilterOption("MACOS", "macOS")
    ))
    .filteredCount(142)
    .build();

// Use in a search API response
return ResponseEntity.ok(filters);
```

This DTO is typically used in REST API responses to provide frontend applications with both the available filtering options and the current result count for dynamic UI updates.
