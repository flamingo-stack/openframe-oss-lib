A utility class that provides factory methods for creating pre-configured `DeviceFilterInput` objects with common device filtering criteria. This generator simplifies test data setup by offering ready-to-use filter configurations for different device scenarios.

## Key Components

- **`onlineDevicesFilter()`** - Creates filter for online devices only
- **`offlineDevicesFilter()`** - Creates filter for offline devices only
- **`statusAndOSDevicesFilter()`** - Creates filter combining device status and OS type criteria
- **`listedDevicesFilter()`** - Creates filter including all standard device statuses
- **`filterDevicesByStatus()`** - Creates filter for multiple specified device statuses using varargs

## Usage Example

```java
// Filter for online devices only
DeviceFilterInput onlineFilter = DeviceGenerator.onlineDevicesFilter();

// Filter for offline devices only  
DeviceFilterInput offlineFilter = DeviceGenerator.offlineDevicesFilter();

// Filter for active Windows devices
DeviceFilterInput windowsActiveFilter = DeviceGenerator.statusAndOSDevicesFilter(
    DeviceStatus.ACTIVE, "Windows"
);

// Filter for devices with specific statuses
DeviceFilterInput customFilter = DeviceGenerator.filterDevicesByStatus(
    DeviceStatus.ONLINE, 
    DeviceStatus.ACTIVE, 
    DeviceStatus.MAINTENANCE
);

// Filter for all listed device statuses
DeviceFilterInput allStatusesFilter = DeviceGenerator.listedDevicesFilter();
```

The class leverages the builder pattern from `DeviceFilterInput` to construct filters with appropriate status lists and optional OS type constraints, making it ideal for test scenarios requiring consistent device filtering logic.
