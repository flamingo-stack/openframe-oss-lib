Data Transfer Object representing a tenant in OpenFrame's multi-tenant architecture. This DTO defines the core attributes of an organization or company using the OpenFrame platform.

## Key Components

- **@BsonId** - MongoDB document identifier for database persistence
- **name** - Display name of the tenant organization
- **domain** - Unique domain identifier for tenant isolation
- **ownerId** - Reference to the tenant administrator user
- **status** - Current operational state (enum: TenantStatus)
- **plan** - Subscription tier information (enum: TenantPlan)
- **hubspotId** - External CRM integration identifier
- **createdAt/updatedAt** - Audit timestamps for tenant lifecycle

## Usage Example

```java
// Create a new tenant
Tenant tenant = Tenant.builder()
    .name("Acme Corporation")
    .domain("acme-corp")
    .ownerId("user-123")
    .status(TenantStatus.ACTIVE)
    .plan(TenantPlan.PROFESSIONAL)
    .hubspotId("hs-456789")
    .createdAt(LocalDateTime.now())
    .build();

// Access tenant properties
String tenantDomain = tenant.getDomain();
TenantStatus currentStatus = tenant.getStatus();
```

The Lombok annotations provide automatic generation of getters, setters, constructors, and builder pattern methods, making this DTO suitable for JSON serialization, database operations, and service layer data transfer in OpenFrame's multi-tenant MSP platform.
