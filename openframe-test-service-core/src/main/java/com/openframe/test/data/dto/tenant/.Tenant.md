Data Transfer Object (DTO) representing an organization/company (tenant) within OpenFrame's multi-tenant architecture.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | BSON primary key (`@BsonId`) |
| `name` | `String` | Organization display name |
| `domain` | `String` | Tenant's domain identifier |
| `ownerId` | `String` | Reference to the owning user |
| `status` | `TenantStatus` | Current tenant lifecycle state |
| `plan` | `TenantPlan` | Subscription/service plan tier |
| `hubspotId` | `String` | CRM integration reference |
| `createdAt` / `updatedAt` | `LocalDateTime` | Audit timestamps |

## Usage Example

```java
// Build a new Tenant instance
Tenant tenant = Tenant.builder()
    .name("Acme Corp")
    .domain("acme.com")
    .ownerId("user-123")
    .status(TenantStatus.ACTIVE)
    .plan(TenantPlan.PROFESSIONAL)
    .hubspotId("hs-456")
    .createdAt(LocalDateTime.now())
    .build();

// Deserialize from JSON (unknown fields are safely ignored)
Tenant parsed = objectMapper.readValue(json, Tenant.class);
```

## Notes

- Annotated with Lombok `@Data`, `@Builder`, `@NoArgsConstructor`, and `@AllArgsConstructor` — no boilerplate getters/setters required.
- `@JsonIgnoreProperties(ignoreUnknown = true)` ensures forward compatibility when API responses include additional fields.
- `@BsonId` maps `id` to MongoDB's `_id` field for direct document storage.
