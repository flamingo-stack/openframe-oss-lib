A Data Transfer Object (DTO) used to encapsulate filter criteria when querying tickets, currently supporting filtering by ticket status values.

## Key Components

| Element | Type | Description |
|---|---|---|
| `statuses` | `List<String>` | List of status values to filter tickets by |

**Lombok Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors

**Jackson Annotation:**
- `@JsonInclude(NON_NULL)` — Omits `null` fields during JSON serialization

## Usage Example

```java
// Using builder pattern
TicketFilterInput filter = TicketFilterInput.builder()
    .statuses(List.of("OPEN", "IN_PROGRESS"))
    .build();

// Using all-args constructor
TicketFilterInput filter = new TicketFilterInput(List.of("CLOSED"));

// Using no-args constructor with setter
TicketFilterInput filter = new TicketFilterInput();
filter.setStatuses(List.of("OPEN", "PENDING"));
```

> **Note:** When serialized to JSON, if `statuses` is `null`, the field is excluded from the output entirely — useful for partial filter payloads in API requests.
