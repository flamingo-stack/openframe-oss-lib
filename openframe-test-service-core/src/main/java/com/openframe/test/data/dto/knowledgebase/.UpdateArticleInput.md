A Data Transfer Object (DTO) used to carry input data for updating an existing knowledge base article in the OpenFrame test suite.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier of the article to update |
| `name` | `String` | Updated title/name of the article |
| `parentId` | `String` | Reference to the parent category or folder |
| `content` | `String` | Full body content of the article |
| `summary` | `String` | Brief description or excerpt of the article |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonInclude(NON_NULL)` — Omits `null` fields from JSON serialization, supporting partial updates

## Usage Example

```java
// Partial update - only name and content are serialized (nulls excluded)
UpdateArticleInput input = UpdateArticleInput.builder()
    .id("article-123")
    .name("Updated Networking Guide")
    .content("## Overview\nThis guide covers...")
    .build();

// Full update
UpdateArticleInput fullUpdate = UpdateArticleInput.builder()
    .id("article-456")
    .name("VPN Setup Guide")
    .parentId("category-789")
    .content("Step-by-step VPN configuration...")
    .summary("How to configure VPN for remote access")
    .build();
```

> The `@JsonInclude(NON_NULL)` annotation makes this DTO ideal for **partial update** (PATCH-style) operations — only fields explicitly set will be included in the outgoing JSON payload.
