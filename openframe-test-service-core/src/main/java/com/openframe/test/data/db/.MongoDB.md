Provides a thread-safe MongoDB connection utility for test data access, managing per-thread `MongoClient` and `MongoDatabase` instances with POJO codec support.

## Key Components

| Method | Description |
|--------|-------------|
| `getDatabase()` | Returns a thread-local `MongoDatabase` instance with automatic POJO codec registry configured |
| `getCollection(String, Class<T>)` | Returns a typed `MongoCollection` by name and document class |
| `openConnection()` | Lazily initializes a thread-local `MongoClient` with credentials, URI, and timeout settings |
| `closeConnection()` | Closes and removes the thread-local `MongoClient` and `MongoDatabase` instances |

**Thread Safety:** Uses `ThreadLocal` for both `MongoClient` and `MongoDatabase`, ensuring isolation across parallel test threads.

**Timeouts:** All connection, read, and server selection timeouts are set to `10` seconds.

## Usage Example

```java
// Retrieve a typed collection and query documents
MongoCollection<User> users = MongoDB.getCollection("users", User.class);
User found = users.find(eq("email", "test@example.com")).first();

// Always close the connection after test execution
MongoDB.closeConnection();
```

> **Note:** Connection configuration (URI, credentials, database name) is sourced from `MongoConfig`. Ensure environment variables or config properties are set before invoking any method. Call `closeConnection()` in a teardown/`@AfterEach` hook to prevent thread-local leaks in parallel test suites.
