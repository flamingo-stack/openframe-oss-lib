A thread-safe MongoDB connection manager that provides database and collection access with automatic POJO codec registration for the OpenFrame test framework.

## Key Components

**Static Methods:**
- `getDatabase()` - Returns configured MongoDatabase instance with POJO codec support
- `getCollection(String, Class<T>)` - Gets typed collection for specified document class
- `openConnection()` - Establishes authenticated MongoDB connection using configuration
- `closeConnection()` - Closes connection and cleans up thread-local resources

**Thread Safety:**
- Uses `ThreadLocal<MongoClient>` and `ThreadLocal<MongoDatabase>` for thread-safe operations
- Automatic connection management per thread

**Configuration Integration:**
- Imports connection settings from `MongoConfig` class
- Supports MongoDB authentication with credentials
- Configures POJO codec registry for automatic object mapping

## Usage Example

```java
// Get a typed collection for User documents
MongoCollection<User> userCollection = MongoDB.getCollection("users", User.class);

// Insert a user document
User newUser = new User("john_doe", "john@example.com");
userCollection.insertOne(newUser);

// Query users
List<User> users = userCollection.find().into(new ArrayList<>());

// Clean up connection when done
MongoDB.closeConnection();
```

The class automatically handles connection pooling, authentication, and POJO serialization, making it easy to work with MongoDB collections in test scenarios while maintaining thread safety across concurrent test execution.
