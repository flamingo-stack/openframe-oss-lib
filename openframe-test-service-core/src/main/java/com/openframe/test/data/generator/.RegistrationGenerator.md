Generates test data objects for user registration workflows, providing factory methods for building `UserRegistrationRequest`, `UserRegistrationResponse`, and `ErrorResponse` DTOs used in automated testing.

## Key Components

| Method | Description |
|--------|-------------|
| `newUserRegistrationRequest()` | Creates a request with fixed name constants (`Auto Tester`) and environment-aware access code (appended for non-OSS environments) |
| `userRegistrationRequest()` | Creates a fully randomized registration request using Datafaker for email, name, and company fields |
| `existingUserRequest(AuthUser)` | Builds a registration request reusing credentials from an existing `AuthUser` |
| `newUserRegistrationResponse(UserRegistrationRequest)` | Builds an expected response DTO with `ACTIVE` status and `FREE` plan, derived from the input request |
| `registrationClosedResponse()` | Returns a `BAD_REQUEST` error DTO for closed-registration scenarios |
| `existingUserResponse()` | Returns a `BAD_REQUEST` error DTO for duplicate/existing user scenarios |

## Usage Example

```java
// Generate a new standard user request (env-aware)
UserRegistrationRequest request = RegistrationGenerator.newUserRegistrationRequest();

// Generate a fully randomized user request
UserRegistrationRequest randomRequest = RegistrationGenerator.userRegistrationRequest();

// Build expected response to assert against API
UserRegistrationResponse expectedResponse = RegistrationGenerator.newUserRegistrationResponse(request);
```
