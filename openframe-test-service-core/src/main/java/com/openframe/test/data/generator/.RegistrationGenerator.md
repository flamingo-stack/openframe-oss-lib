A utility class for generating test data objects related to user registration workflows in the OpenFrame platform testing framework.

## Key Components

**Static Generator Methods:**
- `newUserRegistrationRequest()` - Creates a new user registration request with faker-generated tenant name
- `existingUserRequest(AuthUser)` - Creates a registration request for an existing user
- `newUserRegistrationResponse(UserRegistrationRequest)` - Generates a successful registration response
- `registrationClosedResponse()` - Creates an error response for closed registration
- `existingUserResponse()` - Creates an error response for existing user scenarios

**Configuration Constants:**
- `TENANT_DOMAIN_NAME` - Default localhost domain for testing
- `FIRST_NAME` / `LAST_NAME` - Standard test user names ("Auto Tester")
- Uses `UserConfig.DEFAULT_EMAIL` and `DEFAULT_PASSWORD` for consistent test credentials

## Usage Example

```java
// Generate a new user registration request
UserRegistrationRequest newRequest = RegistrationGenerator.newUserRegistrationRequest();

// Create request for existing user
AuthUser existingUser = getExistingUser();
UserRegistrationRequest existingRequest = RegistrationGenerator.existingUserRequest(existingUser);

// Generate successful response
UserRegistrationResponse response = RegistrationGenerator.newUserRegistrationResponse(newRequest);

// Generate error responses for test scenarios
ErrorResponse closedError = RegistrationGenerator.registrationClosedResponse();
ErrorResponse existingError = RegistrationGenerator.existingUserResponse();
```

The class leverages the DataFaker library to generate realistic company names for tenant creation while sanitizing special characters to ensure valid tenant names.
