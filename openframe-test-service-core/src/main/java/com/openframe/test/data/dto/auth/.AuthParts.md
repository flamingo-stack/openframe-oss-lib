A data transfer object (DTO) representing the various components required during an OAuth2/authentication flow, including credentials, PKCE challenge parameters, and session state.

## Key Components

| Field | Type | Description |
|---|---|---|
| `email` | `String` | User email credential |
| `password` | `String` | User password credential |
| `tenantId` | `String` | Target tenant identifier |
| `state` | `String` | OAuth2 state parameter |
| `codeChallenge` | `String` | PKCE code challenge |
| `codeVerifier` | `String` | PKCE code verifier secret |
| `cookies` | `Map<String, String>` | Session cookies key-value store |
| `authorizationCode` | `String` | OAuth2 authorization code |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern construction
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Tolerates unknown JSON fields during deserialization

## Usage Example

```java
AuthParts authParts = AuthParts.builder()
    .email("technician@example.com")
    .password("securePassword123")
    .tenantId("tenant-abc")
    .state("random-oauth-state")
    .codeChallenge("E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM")
    .codeVerifier("dBjftJeZ4CVP-mB92K27uhbUJU1p1r_wW1gFWFOEjXk")
    .cookies(Map.of("session", "abc123"))
    .authorizationCode("SplxlOBeZQQYbYS6WxSbIA")
    .build();

String tenantId = authParts.getTenantId();
```

> This DTO is typically used in test infrastructure to pass authentication state across multiple steps of an OAuth2 PKCE flow (e.g., login, consent, token exchange) during integration testing.
