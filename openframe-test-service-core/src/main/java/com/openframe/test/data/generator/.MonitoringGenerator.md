Utility class for generating test data related to monitoring policies in the OpenFrame platform, providing factory methods for creating policy requests and querying policy collections.

## Key Components

| Method | Description |
|--------|-------------|
| `windowsVersionPolicy()` | Creates a `CreatePolicyRequest` that validates Windows OS major version is ≥ 10 |
| `emptyPolicy()` | Creates a `CreatePolicyRequest` named "Empty" with no associated devices |
| `policyHost(Machine device)` | Builds a `Policy.Host` object from a `Machine` device, mapping fleet ID and hostname |
| `findPolicyByName(List<Policy>, String)` | Searches a policy list by name, returning an `Optional<Policy>` |

## Usage Example

```java
// Create a Windows version monitoring policy
CreatePolicyRequest request = MonitoringGenerator.windowsVersionPolicy();

// Map a device to a policy host
Machine device = ...; // retrieved from DeviceGenerator or test setup
Policy.Host host = MonitoringGenerator.policyHost(device);

// Find a specific policy from a list
List<Policy> policies = ...; // fetched from API response
Optional<Policy> found = MonitoringGenerator.findPolicyByName(policies, "Windows version");
found.ifPresent(p -> System.out.println("Found: " + p.getName()));

// Create a policy with no devices attached
CreatePolicyRequest empty = MonitoringGenerator.emptyPolicy();
```

> **Note:** This class is part of the OpenFrame test data layer. Policy queries use OSQuery SQL syntax targeting tables like `os_version`. Fleet IDs are resolved via `DeviceGenerator.getFleetId()`.
