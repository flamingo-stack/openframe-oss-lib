A GraphQL-style connection wrapper DTO representing a paginated collection of tickets, containing edges, pagination metadata, and a filtered count.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `edges` | `List<TicketEdge>` | List of ticket edge nodes in the current page |
| `pageInfo` | `TicketPageInfo` | Pagination metadata (cursors, hasNextPage, etc.) |
| `filteredCount` | `Integer` | Total count of tickets matching the applied filter |

## Usage Example

```java
// Deserializing a paginated ticket response
TicketConnection connection = objectMapper.readValue(jsonResponse, TicketConnection.class);

// Accessing edges and page info
List<TicketEdge> edges = connection.getEdges();
TicketPageInfo pageInfo = connection.getPageInfo();
int total = connection.getFilteredCount();

// Builder pattern usage
TicketConnection connection = TicketConnection.builder()
    .edges(List.of(ticketEdge1, ticketEdge2))
    .pageInfo(TicketPageInfo.builder()
        .hasNextPage(true)
        .endCursor("cursor_xyz")
        .build())
    .filteredCount(42)
    .build();
```

## Notes

- Annotated with `@JsonIgnoreProperties(ignoreUnknown = true)` to safely handle extra fields in API responses without throwing deserialization errors.
- Follows the **Relay-style GraphQL connection pattern**, making it compatible with cursor-based pagination.
- Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate boilerplate getters, setters, and constructors.
