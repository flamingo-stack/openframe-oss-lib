Data Transfer Object (DTO) representing a host/device within a Fleet inventory, mapping hardware details, users, software, and derived vulnerability data from JSON responses.

## Key Components

| Member | Type | Description |
|---|---|---|
| `computerName` | `String` | Mapped from `computer_name` |
| `hardwareVendor` / `hardwareModel` / `hardwareSerial` | `String` | Hardware identification fields |
| `cpuType` / `cpuSubtype` / `cpuBrand` | `String` | CPU descriptor fields |
| `cpuPhysicalCores` / `cpuLogicalCores` | `int` | Core count metrics |
| `status` | `String` | Current host status |
| `memory` / `uptime` | `long` | System resource metrics |
| `gigsTotalDiskSpace` | `double` | Mapped from `gigs_total_disk_space` |
| `users` | `List<FleetUser>` | Users associated with the host |
| `software` | `List<FleetSoftware>` | Installed software inventory |
| `getVulnerabilities()` | `List<FleetVulnerability>` | Aggregates CVEs across all installed software |

## Usage Example

```java
// Deserialize from Fleet API response
FleetHost host = objectMapper.readValue(json, FleetHost.class);

// Access hardware info
System.out.println(host.getComputerName());     // e.g. "DESKTOP-ABC123"
System.out.println(host.getCpuBrand());          // e.g. "Intel Core i7"

// Retrieve all vulnerabilities across installed software
List<FleetVulnerability> cves = host.getVulnerabilities();
cves.forEach(v -> System.out.println(v.getCve() + " in " + v.getSoftwareName()));

// Build programmatically
FleetHost host = FleetHost.builder()
        .computerName("SERVER-01")
        .hardwareVendor("Dell")
        .cpuPhysicalCores(8)
        .status("online")
        .build();
```

> `getVulnerabilities()` safely returns an empty list when `software` is `null`, flattening CVE entries from all `FleetSoftware` items and enriching each with its parent software name.
