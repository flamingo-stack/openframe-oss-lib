A simple data transfer object representing a conflict response when invitation operations fail due to validation or business rule violations.

## Key Components

- **code** - Unique identifier for the specific conflict type
- **message** - Human-readable description of the conflict

## Usage Example

```java
// Create a conflict response for duplicate invitation
InvitationConflictResponse conflict = InvitationConflictResponse.builder()
    .code("INVITATION_ALREADY_EXISTS")
    .message("An invitation has already been sent to this email address")
    .build();

// Example usage in a REST controller
@PostMapping("/invitations")
public ResponseEntity<?> createInvitation(@RequestBody InvitationRequest request) {
    try {
        return ResponseEntity.ok(invitationService.create(request));
    } catch (InvitationConflictException e) {
        InvitationConflictResponse conflict = InvitationConflictResponse.builder()
            .code(e.getErrorCode())
            .message(e.getMessage())
            .build();
        return ResponseEntity.status(409).body(conflict);
    }
}
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it ideal for API error responses with HTTP 409 status codes.
