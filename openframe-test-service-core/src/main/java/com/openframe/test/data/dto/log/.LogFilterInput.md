Data Transfer Object (DTO) for filtering log entries, encapsulating optional query parameters such as date ranges, event types, severities, and organization/device identifiers.

## Key Components

| Field | Type | Description |
|---|---|---|
| `startDate` | `String` | Start of the date range filter |
| `endDate` | `String` | End of the date range filter |
| `eventTypes` | `List<String>` | Filter by specific event type(s) |
| `toolTypes` | `List<String>` | Filter by tool type(s) |
| `severities` | `List<String>` | Filter by severity level(s) |
| `organizationIds` | `List<String>` | Filter by organization ID(s) |
| `deviceId` | `String` | Filter by a specific device ID |

**Annotations:**
- `@JsonInclude(NON_NULL)` — Omits `null` fields during serialization, keeping payloads clean
- `@Builder` — Enables fluent builder-pattern construction
- `@Data` / `@NoArgsConstructor` / `@AllArgsConstructor` — Lombok-generated boilerplate

## Usage Example

```java
LogFilterInput filter = LogFilterInput.builder()
    .startDate("2024-01-01")
    .endDate("2024-01-31")
    .eventTypes(List.of("LOGIN", "LOGOUT"))
    .severities(List.of("ERROR", "WARN"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();
```

> **Note:** All fields are optional. Fields left as `null` are excluded from the serialized JSON payload due to `@JsonInclude(NON_NULL)`, making this DTO flexible for partial filter queries.
