A DTO (Data Transfer Object) class for filtering log entries in the OpenFrame test data system, providing comprehensive filter criteria for log queries.

## Key Components

- **Date Range Filtering**: `startDate` and `endDate` fields for temporal filtering
- **Event Classification**: `eventTypes` list for filtering by specific event categories
- **Tool-based Filtering**: `toolTypes` list to filter logs by originating tools
- **Severity Levels**: `severities` list for filtering by log severity (INFO, WARN, ERROR, etc.)
- **Organization Context**: `organizationIds` list for multi-tenant log filtering
- **Device Targeting**: `deviceId` field for device-specific log retrieval
- **Lombok Annotations**: Full builder pattern, getters/setters, and constructors

## Usage Example

```java
// Create filter for error logs from specific tools
LogFilterInput filter = LogFilterInput.builder()
    .startDate("2024-01-01T00:00:00Z")
    .endDate("2024-01-31T23:59:59Z")
    .eventTypes(List.of("AUTHENTICATION", "AUTHORIZATION"))
    .toolTypes(List.of("MSP_CONNECTOR", "DEVICE_MONITOR"))
    .severities(List.of("ERROR", "CRITICAL"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use in service layer
List<LogEntry> filteredLogs = logService.findLogs(filter);
```

```java
// Simple date range filter
LogFilterInput simpleFilter = LogFilterInput.builder()

    .endDate("2024-02-29T23:59:59Z")

```

This class is typically used as input for log query services, REST API endpoints, or GraphQL resolvers to provide flexible log filtering capabilities across the OpenFrame MSP platform.
