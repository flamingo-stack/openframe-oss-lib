Utility class for generating test ticket data objects used in automated testing of OpenFrame's ticket management functionality.

## Key Components

| Method | Description |
|--------|-------------|
| `createTicketRequest(...)` | Builds a `CreateTicketInput` DTO with a default title/description and provided org, device, assignee, and label identifiers |
| `activeTickets()` | Returns a `TicketFilterInput` DTO pre-configured to filter tickets with `ACTIVE` status |

## Usage Example

```java
// Create a ticket request with associated context
CreateTicketInput ticket = TicketGenerator.createTicketRequest(
    "org-123",
    "device-456",
    "user-789",
    List.of("label-001", "label-002")
);

// Generate a filter for active tickets
TicketFilterInput filter = TicketGenerator.activeTickets();
```

## Notes

- All methods are `static` — no instantiation required
- Default title is `"Test ticket"` and description is `"Manually created ticket for tests"`
- `activeTickets()` hardcodes the `ACTIVE` status filter, suitable for most ticket listing test scenarios
