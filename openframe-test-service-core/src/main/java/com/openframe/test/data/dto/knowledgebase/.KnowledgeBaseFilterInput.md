A DTO (Data Transfer Object) used to encapsulate filter criteria when querying knowledge base items within the OpenFrame test data layer.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `parentId` | `String` | Filters items by their parent node ID |
| `type` | `KnowledgeBaseItemType` | Filters by item type (e.g., folder, article) |
| `tagIds` | `List<String>` | Filters items matching one or more tag IDs |

**Annotations:**
- `@JsonInclude(NON_NULL)` — omits `null` fields from serialization, keeping payloads clean
- `@Builder` — enables fluent construction via Lombok's builder pattern
- `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString`

## Usage Example

```java
// Filter by parent and type
KnowledgeBaseFilterInput filter = KnowledgeBaseFilterInput.builder()
    .parentId("folder-123")
    .type(KnowledgeBaseItemType.ARTICLE)
    .build();

// Filter by tags only
KnowledgeBaseFilterInput tagFilter = KnowledgeBaseFilterInput.builder()
    .tagIds(List.of("tag-abc", "tag-xyz"))
    .build();

// All fields
KnowledgeBaseFilterInput fullFilter = KnowledgeBaseFilterInput.builder()
    .parentId("folder-456")
    .type(KnowledgeBaseItemType.FOLDER)
    .tagIds(List.of("tag-001"))
    .build();
```

> All fields are optional. When serialized, any `null` field is excluded from the JSON output, making this DTO safe to use in partial filter scenarios.
