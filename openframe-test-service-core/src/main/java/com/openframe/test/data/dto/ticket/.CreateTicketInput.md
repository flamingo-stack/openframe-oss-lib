A DTO (Data Transfer Object) used to encapsulate input data when creating a new support ticket in the OpenFrame test suite.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `title` | `String` | Ticket title/subject |
| `description` | `String` | Detailed ticket description |
| `organizationId` | `String` | Associated organization identifier |
| `deviceId` | `String` | Related device identifier |
| `assigneeId` | `String` | Technician or user to assign the ticket |
| `labelIds` | `List<String>` | Labels/tags to categorize the ticket |
| `tempAttachmentIds` | `List<String>` | Temporary attachment references |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern for object construction
- `@JsonInclude(NON_NULL)` — Excludes `null` fields from JSON serialization

## Usage Example

```java
CreateTicketInput input = CreateTicketInput.builder()
    .title("Printer not responding")
    .description("The printer on the 2nd floor stopped working after the update.")
    .organizationId("org-abc123")
    .deviceId("dev-xyz789")
    .assigneeId("tech-001")
    .labelIds(List.of("hardware", "urgent"))
    .tempAttachmentIds(List.of("tmp-attach-001"))
    .build();
```

> Only non-null fields are included in the serialized JSON payload, keeping requests minimal and clean when optional fields are omitted.
