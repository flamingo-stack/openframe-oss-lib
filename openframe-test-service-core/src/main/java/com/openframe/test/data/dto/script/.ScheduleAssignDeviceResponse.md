Data Transfer Object (DTO) representing the response payload returned when scheduling a device assignment operation in the OpenFrame test framework.

## Key Components

| Field | Type | JSON Property | Description |
|-------|------|---------------|-------------|
| `agentsCount` | `Integer` | `agents_count` | Number of agents involved in the assignment |
| `taskResultsCreated` | `Integer` | `task_results_created` | Number of task results created during scheduling |
| `taskResultsDeleted` | `Integer` | `task_results_deleted` | Number of task results removed during scheduling |

## Usage Example

```java
// Deserializing a JSON response
String json = """
    {
        "agents_count": 5,
        "task_results_created": 10,
        "task_results_deleted": 2
    }
    """;

ScheduleAssignDeviceResponse response = objectMapper
    .readValue(json, ScheduleAssignDeviceResponse.class);

System.out.println(response.getAgentsCount());        // 5
System.out.println(response.getTaskResultsCreated()); // 10
System.out.println(response.getTaskResultsDeleted()); // 2

// Building via Lombok builder
ScheduleAssignDeviceResponse built = ScheduleAssignDeviceResponse.builder()
    .agentsCount(3)
    .taskResultsCreated(6)
    .taskResultsDeleted(1)
    .build();
```

## Notes

- Annotated with `@JsonIgnoreProperties(ignoreUnknown = true)` — unknown fields in the response payload are silently ignored, making the DTO resilient to API changes.
- Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate getters, setters, constructors, and the builder pattern.
