A lightweight Data Transfer Object (DTO) representing a user associated with a ticket, used in test data contexts within the OpenFrame platform.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier for the user |
| `firstName` | `String` | User's first name |
| `lastName` | `String` | User's last name |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern for object construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Safely deserializes JSON payloads with extra fields

## Usage Example

```java
// Builder pattern
TicketUser user = TicketUser.builder()
    .id("usr-001")
    .firstName("Jane")
    .lastName("Doe")
    .build();

// All-args constructor
TicketUser user = new TicketUser("usr-001", "Jane", "Doe");

// Access fields
String fullName = user.getFirstName() + " " + user.getLastName();
```

> **Note:** `@JsonIgnoreProperties(ignoreUnknown = true)` makes this DTO resilient when deserializing ticket API responses that may contain additional user fields not defined here — useful for test data scenarios where API payloads evolve over time.
