A Data Transfer Object (DTO) representing a single tag-based filter option for device filtering, including the tag's key, value, and occurrence count.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `key` | `String` | The tag attribute name used for filtering |
| `value` | `String` | The tag attribute value |
| `count` | `Integer` | Number of devices matching this tag key/value pair |

**Lombok Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern construction
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates both default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Safely deserializes JSON payloads with extra fields

## Usage Example

```java
// Builder pattern
TagFilterOption filterOption = TagFilterOption.builder()
    .key("os")
    .value("Windows 11")
    .count(42)
    .build();

// Deserialize from JSON response
// { "key": "location", "value": "us-east", "count": 15, "extraField": "ignored" }
TagFilterOption option = objectMapper.readValue(jsonString, TagFilterOption.class);

// Access fields
String tagKey   = option.getKey();    // "location"
String tagValue = option.getValue();  // "us-east"
Integer devices = option.getCount();  // 15
```

> This DTO is typically used in device inventory queries within the OpenFrame platform, where tag-based filtering helps technicians segment and manage endpoints by attributes such as OS, location, or custom labels.
