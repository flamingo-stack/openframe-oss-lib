A GraphQL-style pagination edge DTO that wraps a `Ticket` node with its associated cursor for use in paginated ticket queries.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `cursor` | `String` | Opaque pagination cursor identifying the position of this edge in a result set |
| `node` | `Ticket` | The ticket entity associated with this edge |

## Usage Example

```java
// Building a TicketEdge manually
TicketEdge edge = TicketEdge.builder()
    .cursor("eyJpZCI6MTIzfQ==")
    .node(Ticket.builder()
        .id("123")
        .title("Network issue")
        .build())
    .build();

// Accessing fields from a paginated response
List<TicketEdge> edges = ticketConnection.getEdges();
for (TicketEdge edge : edges) {
    String cursor = edge.getCursor();
    Ticket ticket = edge.getNode();
}
```

## Notes

- Annotated with `@JsonIgnoreProperties(ignoreUnknown = true)` to safely deserialize partial GraphQL responses without failing on unexpected fields
- Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate boilerplate getters, setters, and constructors
- Typically used as part of a connection/pagination wrapper (e.g., `TicketConnection`) following the [Relay cursor-based pagination spec](https://relay.dev/graphql/connections.htm)
