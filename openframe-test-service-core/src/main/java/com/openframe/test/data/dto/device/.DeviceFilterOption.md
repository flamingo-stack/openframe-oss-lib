A simple DTO representing a single filter option for device filtering, containing a value, display label, and count of matching devices.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `value` | `String` | The filter option's identifier/key |
| `label` | `String` | Human-readable display name for the option |
| `count` | `Integer` | Number of devices matching this filter option |

**Annotations:**
- `@Data` — Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@Builder` — Enables fluent builder pattern instantiation
- `@NoArgsConstructor` / `@AllArgsConstructor` — Generates default and full constructors
- `@JsonIgnoreProperties(ignoreUnknown = true)` — Silently ignores unrecognized JSON fields during deserialization

## Usage Example

```java
// Builder pattern
DeviceFilterOption option = DeviceFilterOption.builder()
    .value("windows")
    .label("Windows")
    .count(42)
    .build();

// All-args constructor
DeviceFilterOption option = new DeviceFilterOption("linux", "Linux", 17);

// Access fields
String label = option.getLabel();   // "Linux"
Integer count = option.getCount();  // 17
```

This DTO is typically used within device filtering APIs in the OpenFrame platform, where filter dropdowns or faceted search results need to convey both the selectable option and how many devices fall under each category.
