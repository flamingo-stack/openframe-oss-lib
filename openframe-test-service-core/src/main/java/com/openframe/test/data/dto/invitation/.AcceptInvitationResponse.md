A DTO representing the server response returned after a user successfully accepts a platform invitation, containing the newly activated user's profile and account metadata.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique user identifier |
| `email` | `String` | User's email address |
| `firstName` / `lastName` / `fullName` | `String` | User name fields |
| `roles` | `List<UserRole>` | Assigned roles for the user |
| `status` | `UserStatus` | Current account status |
| `tenantId` | `String` | Associated tenant identifier |
| `emailVerified` | `Boolean` | Whether the email has been verified |
| `loginProvider` | `String` | Authentication provider (e.g., local, OAuth) |
| `externalUserId` | `String` | ID from an external identity provider |
| `lastLogin` | `Instant` | Timestamp of the user's most recent login |
| `createdAt` / `updatedAt` | `LocalDateTime` | Record lifecycle timestamps |

**Lombok annotations:** `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` auto-generate boilerplate.  
**`@JsonIgnoreProperties(ignoreUnknown = true)`** ensures forward compatibility when the API returns additional fields.

## Usage Example

```java
AcceptInvitationResponse response = objectMapper.readValue(apiResponse, AcceptInvitationResponse.class);

// Access user details after invitation accepted
String userId     = response.getId();
String tenantId   = response.getTenantId();
UserStatus status = response.getStatus();
List<UserRole> roles = response.getRoles();

// Builder pattern for test data setup
AcceptInvitationResponse mockResponse = AcceptInvitationResponse.builder()
    .id("user-123")
    .email("tech@example.com")
    .tenantId("tenant-456")
    .status(UserStatus.ACTIVE)
    .emailVerified(true)
    .loginProvider("local")
    .build();
```
