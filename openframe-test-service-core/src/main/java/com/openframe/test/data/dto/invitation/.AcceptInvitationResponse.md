Data Transfer Object (DTO) response class that represents the complete user information returned after successfully accepting an invitation in the OpenFrame platform.

## Key Components

- **User Identity Fields**: `id`, `email`, `firstName`, `lastName`, `fullName` for user identification
- **Authorization Fields**: `roles` (List<UserRole>), `status` (UserStatus), `tenantId` for access control
- **Authentication Fields**: `passwordHash`, `emailVerified`, `loginProvider`, `externalUserId` for login management
- **Audit Fields**: `createdAt`, `updatedAt`, `lastLogin` for tracking user lifecycle
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Build response after invitation acceptance
AcceptInvitationResponse response = AcceptInvitationResponse.builder()
    .id("user_123")
    .email("john.doe@company.com")
    .firstName("John")
    .lastName("Doe")
    .fullName("John Doe")
    .roles(List.of(UserRole.MEMBER))
    .status(UserStatus.ACTIVE)
    .tenantId("tenant_456")
    .emailVerified(true)
    .createdAt(LocalDateTime.now())
    .updatedAt(LocalDateTime.now())
    .lastLogin(Instant.now())
    .build();

// Return in REST endpoint
@PostMapping("/invitations/{token}/accept")
public ResponseEntity<AcceptInvitationResponse> acceptInvitation(
    @PathVariable String token,
    @RequestBody AcceptInvitationRequest request) {
    
    AcceptInvitationResponse response = invitationService.acceptInvitation(token, request);
    return ResponseEntity.ok(response);
}
```

This response DTO provides comprehensive user information after invitation acceptance, supporting both internal authentication and external providers through the `loginProvider` and `externalUserId` fields.
