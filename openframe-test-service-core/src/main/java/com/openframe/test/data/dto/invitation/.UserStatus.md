A simple enumeration that defines the possible status states for users in the invitation system, providing binary classification for user account lifecycle management.

## Key Components

- **ACTIVE** - Represents users with active accounts who can participate in invitations and system operations
- **DELETED** - Represents users whose accounts have been marked as deleted or deactivated

## Usage Example

```java
package com.openframe.test.data.dto.invitation;

// Setting user status when creating or updating user records
public class UserInvitationService {
    
    public void processUserInvitation(User user) {
        if (user.getStatus() == UserStatus.ACTIVE) {
            // Send invitation to active user
            sendInvitation(user);
        } else if (user.getStatus() == UserStatus.DELETED) {
            // Skip deleted users or handle cleanup
            logSkippedUser(user.getId());
        }
    }
    
    public void deactivateUser(User user) {
        user.setStatus(UserStatus.DELETED);
        userRepository.save(user);
    }
    
    public boolean isUserActive(User user) {
        return user.getStatus() == UserStatus.ACTIVE;
    }
}
```

This enum provides a clean, type-safe way to manage user states throughout the invitation workflow, ensuring consistent status handling across the application.
