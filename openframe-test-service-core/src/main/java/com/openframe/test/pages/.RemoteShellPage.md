Page Object for the Remote Shell interface rendered by xterm.js, supporting command execution, terminal output inspection, and connection management for CMD and PowerShell sessions.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `getPageHeading()` | Method | Returns the "Remote Shell" page heading text |
| `getDeviceName()` | Method | Returns the device name from the connection status card |
| `getShellType()` | Method | Extracts `cmd` or `powershell` from the URL query parameter |
| `isConnected()` | Method | Checks green success CSS class and non-empty terminal output |
| `executeCommand()` | Method | Types a command into the xterm input proxy and presses Enter |
| `typeText()` | Method | Types text without Enter — useful for password prompts |
| `pressKey()` | Method | Sends a key or combo (e.g. `Control+C`) to the terminal |
| `getTerminalLines()` | Method | Returns non-empty lines from xterm.js DOM rows |
| `getTerminalOutput()` | Method | Returns full terminal output as a newline-joined string |
| `waitForOutputContaining()` | Method | Polls terminal DOM until expected text appears or timeout fires |
| `terminalContainer()` | Method | Returns the `.terminal.xterm` `Locator` for visibility assertions |
| `clickDisconnect()` | Method | Terminates the active shell session |
| `clickBackToDevice()` | Method | Navigates back to the Device Details page |

## Usage Example

```java
// Navigate to the Remote Shell page (opened from DeviceDetailsPage)
RemoteShellPage shell = new RemoteShellPage(page);

// Verify shell type and connection
assertThat(shell.getShellType()).isEqualTo("powershell");
assertThat(shell.isConnected()).isTrue();

// Execute a command and wait for output
shell.executeCommand("Get-Date");
shell.waitForOutputContaining("2025", 5000);
assertThat(shell.terminalOutputContains("2025")).isTrue();

// Send Ctrl+C to cancel a running process
shell.pressKey("Control+C");

// Disconnect and return to device details
shell.clickDisconnect();
shell.clickBackToDevice();
```

> **Note:** Terminal input targets xterm.js's hidden `textarea[aria-label="Terminal input"]` proxy. Output is read directly from `.xterm-rows > div` DOM elements — no canvas scraping required.
