Post-login Dashboard page object for Playwright-based UI tests. Provides locators and helper queries for asserting successful login and verifying the OpenFrame dashboard shell loaded correctly.

## Key Components

| Member | Type | Description |
|---|---|---|
| `URL_FRAGMENT` | Constant | URL segment (`/dashboard/`) used for navigation assertions |
| `userBadge()` | Locator | Top-right header avatar button showing user initials |
| `pageHeading()` | Locator | First `h1`/`h2` inside `<main>` content area |
| `sidebar()` | Locator | Main navigation sidebar — reliable app-shell load indicator |
| `isLoaded()` | Query | Returns `true` when URL, user badge, and sidebar are all present |
| `getCurrentUrl()` | Query | Returns the browser's current URL string |
| `getUserBadgeText()` | Query | Returns trimmed inner text of the user avatar badge (e.g. `"TQ"`) |

## Usage Example

```java
// Instantiate after a successful login action
DashboardPage dashboard = new DashboardPage(page);

// Assert the dashboard loaded correctly
assertTrue(dashboard.isLoaded(), "Dashboard should be visible after login");

// Verify user initials are displayed in the header badge
assertEquals("TQ", dashboard.getUserBadgeText());

// Assert the current URL contains the expected fragment
assertThat(dashboard.getCurrentUrl()).contains(DashboardPage.URL_FRAGMENT);
```

> **Note:** `isLoaded()` combines three checks — URL fragment, user badge visibility, and sidebar visibility — making it the recommended single assertion for post-login verification in test flows.
