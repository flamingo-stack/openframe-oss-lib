Page Object for the OpenFrame authentication entry page (`https://openframe.build/auth/`), implementing step 1 of the login flow using the Playwright Java API.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `URL` | `static final String` | Resolved auth entry URL from `EnvironmentConfig` |
| `LOGIN_CARD` | CSS selector | Scopes all locators to the "Already Have an Account?" card to avoid collision with the "Create Organization" card |
| `navigate()` | Method | Navigates to the auth URL, waits for `DOMCONTENTLOADED` then `NETWORKIDLE` |
| `enterEmail(String)` | Action | Fills the email input field; returns `this` for chaining |
| `clickContinue()` | Action | Clicks Continue, waits for redirect to `/auth/login/`, returns `AuthMethodPage` |
| `submitEmail(String)` | Convenience | Chains `enterEmail` + `clickContinue` in one call |
| `isContinueEnabled()` | Query | Returns whether the Continue button is currently enabled |

## Usage Example

```java
// Navigate to auth entry and proceed to step 2
AuthMethodPage methodPage = new AuthEntryPage(page)
    .navigate()
    .submitEmail("technician@example.com");

// Or step-by-step with an assertion
AuthEntryPage entryPage = new AuthEntryPage(page).navigate();

assertThat(entryPage.loginCardHeading()).hasText("Already Have an Account?");

entryPage.enterEmail("admin@acme.com");
assertTrue(entryPage.isContinueEnabled());

AuthMethodPage nextStep = entryPage.clickContinue();
```

> **Note:** Locators for `emailInput()` and `continueButton()` use `.nth(1)` to select the second matching element on the page, since the "Create Organization" card renders a similar input first.
