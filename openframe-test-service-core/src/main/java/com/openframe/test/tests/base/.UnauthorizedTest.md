Abstract base class for API tests that require an unauthorized/unauthenticated context, providing common setup and teardown for REST API testing without authentication headers.

## Key Components

- **@BeforeAll config()** - Static setup method that configures RestAssured settings and opens MongoDB connection
- **@AfterAll close()** - Static teardown method that closes the MongoDB connection
- **Abstract class design** - Must be extended by concrete test classes

## Usage Example

```java
public class PublicEndpointsTest extends UnauthorizedTest {
    
    @Test
    public void testHealthCheckEndpoint() {
        given()
            .when()
                .get("/health")
            .then()
                .statusCode(200);
    }
    
    @Test
    public void testUnauthorizedAccess() {
        given()
            .when()
                .get("/protected/users")
            .then()
                .statusCode(401);
    }
}
```

This base class is ideal for testing public endpoints, authentication flows, or verifying that protected resources properly return 401/403 status codes when accessed without proper credentials. The MongoDB connection setup suggests tests may need to verify database state changes from unauthorized requests.
