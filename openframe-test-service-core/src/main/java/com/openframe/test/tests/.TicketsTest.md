Integration test suite for the OpenFrame ticket management API, covering the full lifecycle of ticket operations including creation, status transitions, ordering, and retrieval.

## Key Components

| Method | Tags | Description |
|--------|------|-------------|
| `testListTickets()` | `saas`, `read` | Verifies paginated ticket listing returns valid edges with mandatory fields |
| `testListTicketLabels()` | `saas`, `read` | Validates that ticket labels exist and contain required `id` and `key` fields |
| `testCreateTicket()` | — | Creates a fully-populated ticket with assignee, organization, device, and label; asserts all returned fields |
| `testResolveTicket()` | Order 4 | Transitions an `ACTIVE` ticket to the `RESOLVED` system status via `transitionTicket` and verifies `resolvedAt` is set |
| `testArchiveActiveTicketRejected()` | — | Confirms transitioning a non-`RESOLVED` ticket straight to `ARCHIVED` is rejected with a top-level GraphQL error (`TICKET_INVALID_TRANSITION`) and the ticket's status kind is unchanged |
| `testArchiveTicket()` | Order 5 | Transitions a `RESOLVED` ticket (found via `statusIds`) to the `ARCHIVED` system status; verifies the archived status definition |
| `testReorderTicket()` | Order 6 | Within a single lifecycle column (statusId), moves the bottom ticket to the top and asserts the `order` key changes while the column (statusDefinition) is unchanged |
| `testCreateTicketStatus()` | Order 7 | Creates a custom status, asserts its fields (kind `CUSTOM`, not system) and presence in `ticketStatuses`, then deletes it |
| `testDeleteTicketStatus()` | Order 8 | Creates then deletes a custom status; asserts the delete returns `true` and the status is gone from `ticketStatuses` |
| `testDeleteSystemStatusRejected()` | — | Confirms deleting a system status is rejected with a top-level GraphQL error (`TICKET_STATUS_SYSTEM_PROTECTED`) and the status still exists |
| `testGetTicket()` | `saas`, `read` | Fetches a single ticket by ID and verifies mandatory fields including `owner` |

## Usage Example

```java
// Run only read-safe SaaS tests (no mutations)
// mvn test -Dgroups="saas & read"

// Full lifecycle: create → resolve → archive (via transitionTicket)
CreateTicketInput input = TicketGenerator.createTicketRequest(
        organization.getOrganizationId(),
        device.getMachineId(),
        assigneeId,
        List.of(labelId));

Ticket ticket   = TicketApi.createTicket(input);                                          // ACTIVE
Ticket resolved = TicketApi.transitionTicket(ticket.getId(), TicketApi.resolveSystemStatusId("RESOLVED")); // RESOLVED
Ticket archived = TicketApi.transitionTicket(ticket.getId(), TicketApi.resolveSystemStatusId("ARCHIVED")); // ARCHIVED
```

> **Lifecycle note:** legacy status mutations (`putTicketOnHold`, `resolveTicket`, `archiveTicket`) are disabled while the ticket lifecycle is enabled — use `transitionTicket` with a status definition id instead. `ON_HOLD` has no lifecycle status definition, so it is no longer covered. Direct `ACTIVE → ARCHIVED` is rejected by the API and validated in `testArchiveActiveTicketRejected()`.
