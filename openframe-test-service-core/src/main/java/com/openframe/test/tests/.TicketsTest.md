Integration test suite for the OpenFrame ticket management API, covering the full ticket lifecycle from creation through status transitions (active → on hold → resolved → archived) and supporting operations like listing, label retrieval, reordering, and validation of invalid state transitions.

## Key Components

| Method | Tags/Order | Description |
|---|---|---|
| `testListTickets()` | `saas`, `read` | Fetches paginated active tickets and validates mandatory fields (`id`, `ticketNumber`, `title`, `status`) |
| `testListTicketLabels()` | `saas`, `read` | Retrieves all ticket labels and asserts `id` and `key` are present |
| `testCreateTicket()` | Order 1 | Creates a ticket with assignee, organization, device, and label; validates all returned fields |
| `testPutTicketOnHold()` | Order 2 | Transitions an `ACTIVE` ticket to `ON_HOLD` |
| `testResolveTicket()` | Order 3 | Transitions an `ACTIVE` ticket to `RESOLVED` and checks `resolvedAt` is set |
| `testArchiveActiveTicketRejected()` | — | Asserts that archiving an `ACTIVE` ticket returns user errors with an invalid transition message |
| `testArchiveTicket()` | Order 4 | Resolves then archives a ticket; validates `ARCHIVED` status |
| `testReorderTicket()` | Order 5 | Reorders tickets using before/after anchors and verifies the order key changes |
| `testGetTicket()` | `saas`, `read` | Fetches a single ticket by ID and validates fields including `owner` |

## Usage Example

```java
// Run only read-safe SaaS tests (no mutations)
// mvn test -Dgroups="saas & read"

// Full lifecycle test run (requires at least 3 active tickets and a valid assignee)
TicketConnection connection = TicketApi.getTickets(
    TicketGenerator.activeTickets(),
    CursorPaginationInput.builder().limit(20).build(),
    null);

CreateTicketInput input = TicketGenerator.createTicketRequest(
    organization.getOrganizationId(),
    device.getMachineId(),
    assigneeId,
    List.of(labelId));

Ticket ticket = TicketApi.createTicket(input);
// Lifecycle: createTicket → putTicketOnHold → resolveTicket → archiveTicket
```

> **Note:** The class is annotated `@Disabled` at the class level. Remove or override this annotation to enable execution. Tests tagged `saas`+`read` are safe for read-only environments; ordered tests (1–5) require a writable environment with pre-existing users, organizations, and devices.
