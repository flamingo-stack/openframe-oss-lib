Integration test suite for the OpenFrame ticket management API, covering the full lifecycle of ticket operations including creation, status transitions, ordering, and retrieval.

## Key Components

| Method | Tags | Description |
|--------|------|-------------|
| `testListTickets()` | `saas`, `read` | Verifies paginated ticket listing returns valid edges with mandatory fields |
| `testListTicketLabels()` | `saas`, `read` | Validates that ticket labels exist and contain required `id` and `key` fields |
| `testCreateTicket()` | — | Creates a fully-populated ticket with assignee, organization, device, and label; asserts all returned fields |
| `testPutTicketOnHold()` | Order 2 | Transitions an `ACTIVE` ticket to `ON_HOLD` status |
| `testResolveTicket()` | Order 3 | Transitions an `ACTIVE` ticket to `RESOLVED` and verifies `resolvedAt` is set |
| `testArchiveActiveTicketRejected()` | — | Confirms that archiving an `ACTIVE` ticket is rejected with a user error describing the invalid status transition |
| `testArchiveTicket()` | Order 4 | Resolves then archives a ticket; verifies `ARCHIVED` status |
| `testReorderTicket()` | Order 5 | Reorders a ticket among its neighbors and asserts the `order` key changes |
| `testGetTicket()` | `saas`, `read` | Fetches a single ticket by ID and verifies mandatory fields including `owner` |

## Usage Example

```java
// Run only read-safe SaaS tests (no mutations)
// mvn test -Dgroups="saas & read"

// Full lifecycle: create → hold → resolve → archive
CreateTicketInput input = TicketGenerator.createTicketRequest(
        organization.getOrganizationId(),
        device.getMachineId(),
        assigneeId,
        List.of(labelId));

Ticket ticket  = TicketApi.createTicket(input);           // ACTIVE
Ticket onHold  = TicketApi.putTicketOnHold(ticket.getId()); // ON_HOLD
Ticket resolved = TicketApi.resolveTicket(ticket.getId());  // RESOLVED
Ticket archived = TicketApi.archiveTicket(ticket.getId());  // ARCHIVED
```

> **Status transition rules enforced:** Direct `ACTIVE → ARCHIVED` is rejected by the API and validated in `testArchiveActiveTicketRejected()`. Tickets must be `RESOLVED` before they can be archived.
