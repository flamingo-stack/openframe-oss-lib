End-to-end integration test suite for the OpenFrame ticket management API, covering the full ticket lifecycle from creation through archival, plus listing, labeling, reordering, and status-transition validation.

## Key Components

| Method | Tags | Description |
|---|---|---|
| `testListTickets()` | `saas`, `read` | Fetches paginated active tickets and asserts mandatory fields (`id`, `ticketNumber`, `title`, `status`) |
| `testListTicketLabels()` | `saas`, `read` | Retrieves all ticket labels and validates `id` and `key` presence |
| `testCreateTicket()` | — | Creates a ticket with assignee, organization, device, and label; asserts all fields including owner type |
| `testPutTicketOnHold()` | Order 2 | Transitions an ACTIVE ticket to `ON_HOLD` and verifies the status change |
| `testResolveTicket()` | Order 3 | Resolves an ACTIVE ticket and asserts `RESOLVED` status and `resolvedAt` timestamp |
| `testArchiveActiveTicketRejected()` | — | Confirms that archiving an ACTIVE ticket returns `userErrors` citing an invalid status transition |
| `testArchiveTicket()` | Order 4 | Resolves then archives a ticket, asserting the `ARCHIVED` final state |
| `testReorderTicket()` | Order 5 | Reorders the third ticket between the first two and confirms the `order` key changes |
| `testGetTicket()` | `saas`, `read` | Fetches a single ticket by ID and validates fields including the owner relationship |

## Usage Example

```java
// Run only SaaS read-only tests against a live environment
// (class is @Disabled by default — enable per environment as needed)

@Tag("saas")
@Tag("read")
@Test
@DisplayName("List tickets")
public void testListTickets() {
    TicketConnection connection = TicketApi.getTickets(
            TicketGenerator.activeTickets(),
            CursorPaginationInput.builder().limit(20).build(),
            null);

    assertThat(connection.getEdges()).isNotEmpty();
    assertThat(connection.getEdges()).allSatisfy(edge -> {
        Ticket ticket = edge.getNode();
        assertThat(ticket.getId()).isNotNull();
        assertThat(ticket.getStatus()).isNotEmpty();
    });
}
```

> **Note:** The class is annotated `@Disabled` at the suite level. Remove or override this annotation to activate tests in a target environment. Ordered tests (`@Order 1–5`) follow the lifecycle: **Create → Hold → Resolve → Archive → Reorder**.
