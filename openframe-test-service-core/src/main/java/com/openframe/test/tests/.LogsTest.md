JUnit test suite for the OpenFrame logs monitoring system, validating log retrieval, filtering, searching, and detail access functionality.

## Key Components

- **AuthorizedTest** - Base class providing authentication context for API tests
- **LogsApi** - API client for logs service interactions (filters, retrieval, search, details)
- **LogEvent** - Data model for log entries with metadata (tool type, event type, timestamp, severity)
- **LogDetails** - Extended log information with full event details
- **LogFilters** - Available filter options (tool types, event types, severities)
- **LogGenerator** - Test utility for creating filter configurations

## Test Coverage

The test suite validates core logs functionality:

- **Filter Management** - Retrieves available filter options and validates presence of required categories
- **Log Listing** - Fetches logs and validates required fields (tool event ID, event type, timestamp, hostname)
- **Detail Retrieval** - Gets detailed log information and verifies data consistency
- **Search Functionality** - Searches logs by tool type and validates filtered results
- **Advanced Filtering** - Applies severity and tool type filters with result validation

## Usage Example

```java
// Run all logs tests
@Test
public void validateLogsSystemHealth() {
    // Test basic log retrieval
    List<LogEvent> logs = LogsApi.getLogs();
    assertThat(logs).isNotEmpty();
    
    // Test filtering capabilities
    LogFilters filters = LogsApi.getLogFilters();
    List<LogEvent> filtered = LogsApi.getLogs(
        LogGenerator.severityAndToolFilter("ERROR", "MSP_TOOL")
    );
    
    // Validate log details access
    LogDetails details = LogsApi.getLogDetails(logs.getFirst());
    assertThat(details.getToolEventId()).isNotNull();
}
```

All tests are tagged with `@Tag("shared")` and `@Tag("monitor")` for selective test execution in CI/CD pipelines.
