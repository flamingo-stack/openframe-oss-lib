JUnit 5 test suite for the OpenFrame Logs API, validating log retrieval, filtering, search, and detail endpoints.

## Key Components

| Method | Tag | Description |
|---|---|---|
| `testGetLogFilters()` | `read` | Verifies the log filters endpoint returns non-null filters with at least one tool type, event type, and severity |
| `testListLogs()` | `read` | Asserts the log list is non-empty and each `LogEvent` contains required fields |
| `testGetLogDetails()` | `read` | Fetches the first log event and validates that its details match by `toolEventId`, `eventType`, and `toolType` |
| `testSearchLogs()` | `read` | Extracts a search term from an existing log summary and confirms results contain that term |
| `testFilterLogs()` | `read` | Applies a severity filter and asserts all returned logs match the selected severity |

## Usage Example

```java
// Run all Logs tests tagged as "saas"
@Tag("saas")
public class LogsTest extends BaseTest {

    // Filter logs by a specific severity from available filters
    @Test
    public void testFilterLogs() {
        LogFilters filters = LogsApi.getLogFilters();
        String severity = filters.getSeverities().getFirst();
        List<LogEvent> logs = LogsApi.getLogs(LogGenerator.severityFilter(severity));
        assertThat(logs).allSatisfy(log ->
            assertThat(log.getSeverity()).isEqualTo(severity)
        );
    }
}
```

> All tests are tagged `read` and `saas`, making them suitable for scoped execution in SaaS environment CI pipelines without requiring write permissions.
