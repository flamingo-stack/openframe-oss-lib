UI test suite validating remote access features (Remote Shell, Remote Desktop, File Manager) for a specific device (`vm115982`) via the MeshCentral agent in the OpenFrame platform.

## Key Components

| Component | Description |
|-----------|-------------|
| `@Tag("device")` / `@Tag("remote")` | JUnit 5 tags for test categorization and selective execution |
| `@BeforeEach openDeviceDetails()` | Navigates to device `vm115982` and opens the Agents tab before each test |
| `testRemoteShell()` | Verifies PowerShell connection, command execution (`dir`), and disconnect |
| `testRemoteDesktop()` | Validates that the remote desktop canvas is visible and fully loaded |
| `testRemoteFileManager()` | Confirms the file manager lists files with valid folder path and row data |

## Usage Example

```java
// Run only remote tests via Maven
mvn test -Dgroups="device & remote"

// Example assertion pattern used in this suite
RemoteShellPage remoteShellPage = deviceDetailsPage.openRemoteShellPowerShell();
List<String> before = remoteShellPage.getTerminalLines();
remoteShellPage.executeCommand("dir");
assertThat(remoteShellPage.getTerminalLines())
    .hasSizeGreaterThan(before.size());
```

## Prerequisites

- Device `vm115982` must be reachable with MeshCentral agent reporting **ONLINE** status
- All three tests assert agent status before proceeding — tests will fail fast if the agent is offline
- Extends `BaseUITest`, which is expected to provide the `navigationSidebar` fixture and browser/driver setup
