Collects and aggregates OS version statistics across all OpenFrame tenants by querying device fleet data in parallel.

## Key Components

| Component | Description |
|-----------|-------------|
| `getDomains()` | Reads tenant domain list from `/openframe.all.tenants.json` classpath resource |
| `collectStats()` | Main test method — queries all tenant domains concurrently (15 threads) and aggregates OS version counts |
| `normalizeOsVersion()` | Strips trailing build numbers from Windows OS strings (e.g. `Windows 10 10.0.19041` → `Windows 10`) |
| `saveStats()` | Writes sorted OS version frequency data to `stats.txt` |

## Usage Example

```java
// Run manually (class is @Disabled by default)
// 1. Provide tenant list at: src/test/resources/openframe.all.tenants.json
// [{"domain": "tenant1.openframe.ai"}, {"domain": "tenant2.openframe.ai"}]

// 2. Enable and run the test
@Test
public void collectStats() throws Exception { ... }

// Output (console + stats.txt):
// 1500    Windows 11 Pro
// 942     Windows 10 Pro
// 210     macOS 14.4
```

## Notes

- The test is marked `@Disabled` — it must be explicitly enabled before running
- Uses a 15-thread pool with a 50ms delay per device request to avoid rate limiting
- OS version aggregation is thread-safe via `ConcurrentHashMap.merge()`
- Results are sorted descending by device count before output
