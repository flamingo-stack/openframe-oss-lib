Ordered integration test suite that validates the complete owner registration flow in the OpenFrame platform, covering user registration, authentication login, and default organization creation.

## Key Components

| Component | Description |
|-----------|-------------|
| `testRegisterNewUser()` | Registers a new owner user via `RegistrationApi` and asserts the response fields (id, ownerId, updatedAt) match expected values |
| `testLoginNewUser()` | Authenticates the registered user via `AuthFlow.login()` and verifies the `/me` endpoint returns `authenticated: true` |
| `testDefaultOrganizationCreated()` | Confirms a default organization is automatically provisioned after registration |
| `setup()` | `@BeforeAll` hook that initializes random environment and user configs via `EnvironmentConfig.random()` and `UserConfig.random()` |

## Usage Example

```java
// Tests run in strict order via @TestMethodOrder(MethodOrderer.OrderAnnotation.class)
// and are tagged for selective execution

// Run only registration tests
./mvnw test -Dgroups="registration"

// Run only the login step
./mvnw test -Dgroups="login"
```

## Notes

- Extends `BaseTest` and must execute **before all other test suites** since it seeds the owner user and organization required by downstream tests
- Uses `RegistrationGenerator` to produce deterministic request/response DTOs for assertion via AssertJ's `usingRecursiveComparison()`, ignoring volatile fields (`id`, `ownerId`, `hubspotId`, `createdAt`, `updatedAt`, `plan`)
- `RequestSpecHelper` and `AuthHelper` both receive session cookies after login to propagate auth state across subsequent API calls
