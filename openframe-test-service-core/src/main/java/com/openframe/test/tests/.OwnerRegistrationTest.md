A comprehensive integration test suite that validates the complete owner registration workflow, from user creation to default organization setup. This test ensures the critical path for new system owners is working correctly.

## Key Components

- **testRegisterNewUser()** - Validates user registration with generated test data and verifies response completeness
- **testLoginNewUser()** - Tests authentication flow for newly registered users using cookies
- **testDefaultOrganizationCreated()** - Confirms automatic creation of default organization after registration
- **@Order annotations** - Ensures tests execute in proper sequence (registration → login → org validation)
- **@Tag("registration")** - Groups this test with other registration-related tests

## Usage Example

```java
// Run specific registration test
@Test
@Order(1)
public void testRegisterNewUser() {
    UserRegistrationRequest request = RegistrationGenerator.newUserRegistrationRequest();
    UserRegistrationResponse response = RegistrationApi.registerUser(request);
    
    // Verify essential fields are populated
    assertThat(response.getId()).isNotNull();
    assertThat(response.getOwnerId()).isNotNull();
    
    // Compare response structure ignoring auto-generated fields
    assertThat(response).usingRecursiveComparison()
            .ignoringFields("id", "ownerId", "hubspotId", "createdAt", "updatedAt")
            .isEqualTo(expectedResponse);
}
```

This test class executes before all other tests to establish a valid owner account and serves as a foundation for subsequent test scenarios requiring authenticated users.
