JUnit 5 integration test class that verifies the deletion of monitoring policies via the `MonitoringApi`, covering both empty policies and policies with associated devices.

## Key Components

| Component | Description |
|---|---|
| `DeletePolicyTest` | Main test class extending `BaseTest`, tagged as `"archive"` |
| `testDeleteEmptyPolicy()` | Verifies deletion of a policy with no devices assigned |
| `testDeletePolicyWithDevices()` | Verifies deletion of a policy named `"Windows version"` that has devices associated |

**Dependencies:**
- `MonitoringApi` — provides `getPolicies()` and `deletePolicy(id)` API calls
- `MonitoringGenerator.findPolicyByName()` — utility to locate a policy by name from a list
- `Policy` DTO — represents a monitoring policy with at least an `id` field

## Usage Example

```java
// Both tests follow the same pattern:

// 1. Fetch all existing policies
List<Policy> policies = MonitoringApi.getPolicies();

// 2. Locate the target policy by name
Optional<Policy> policy = findPolicyByName(policies, "Empty");
assertThat(policy).as("Policy 'Empty' should exist").isPresent();

// 3. Delete the policy by ID and assert it was included in the deleted list
List<Integer> deleted = MonitoringApi.deletePolicy(policy.get().getId());
assertThat(deleted).contains(policy.get().getId());
```

## Notes

- The `@Tag("archive")` annotation marks these tests as part of an archived suite — they may not run in the default CI pipeline.
- Both tests assume pre-existing policies (`"Empty"` and `"Windows version"`) are present in the test environment before execution.
- The API's `deletePolicy()` returns a list of deleted IDs, allowing assertion that the correct record was removed.
