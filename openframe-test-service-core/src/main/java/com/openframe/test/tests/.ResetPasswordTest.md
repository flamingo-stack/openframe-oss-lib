Automated integration tests for the password reset flow, verifying both successful reset and error handling for invalid tokens.

## Key Components

| Component | Description |
|-----------|-------------|
| `testResetPassword()` | End-to-end happy path: triggers reset, fetches token from Redis, confirms reset with new password, and validates authentication |
| `testResetPasswordWithInvalidToken()` | Negative test: attempts reset confirmation with a bogus token and asserts the correct error response |

## Usage Example

```java
// Run only reset-tagged tests via JUnit tag filter
// mvn test -Dgroups="reset"

// Happy path flow internals:
User user = UserConfig.getUser();
UserApi.resetPassword(user);                          // trigger reset email
String token = Redis.getResetToken(user.getEmail());  // fetch token from Redis
ResetConfirmRequest req = AuthGenerator.resetConfirmRequest(token);
UserApi.confirmReset(req);
user.setPassword(req.getNewPassword());
MeResponse me = UserApi.me();
assertThat(me.isAuthenticated()).isTrue();

// Invalid token flow:
ResetConfirmRequest bad = AuthGenerator.resetConfirmRequest("invalid");
ErrorResponse err = UserApi.attemptConfirmReset(bad);
assertThat(err).isEqualTo(AuthGenerator.invalidTokenResponse());
```

## Notes

- Tagged `@Tag("reset")` for selective test execution
- Relies on `Redis.getResetToken()` to intercept reset tokens out-of-band, avoiding real email delivery in test environments
- `UserApi.attemptConfirmReset()` is used (vs. `confirmReset()`) for negative cases to return an `ErrorResponse` without throwing
- Extends `BaseTest` for shared setup/teardown and session management
