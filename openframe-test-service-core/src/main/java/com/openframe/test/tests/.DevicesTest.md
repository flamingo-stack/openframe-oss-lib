JUnit 5 test suite validating the OpenFrame device management API endpoints, covering device filtering, retrieval, fleet info, search, OS filtering, archiving, and deletion.

## Key Components

| Test Method | Tag(s) | Description |
|---|---|---|
| `testGetDeviceFilters()` | `saas`, `read` | Validates device filter response includes statuses, OS types, and organization IDs with non-empty values |
| `testGetDevice()` | `saas`, `read` | Retrieves a device and asserts all core fields plus required tool connections (`MESHCENTRAL`, `FLEET_MDM`) |
| `testFleetInfo()` | `saas`, `read` | Validates Fleet MDM host data including hardware specs, users, software inventory, and CVE vulnerabilities |
| `testSearchDevices()` | `saas`, `read` | Searches devices by hostname and confirms the result matches the queried hostname |
| `testFilterDevices()` | `saas`, `read` | Filters devices by OS type and asserts all results return `WINDOWS` |
| `testArchiveDevice()` | `archive` | Archives an offline device and confirms it no longer appears in listed devices |
| `testDeleteDevice()` | `archive` | Deletes an offline device and confirms it no longer appears in listed devices |

## Usage Example

```java
// Run only read-safe SaaS tests (no mutations)
// mvn test -Dgroups="saas & read"

// Run archive/mutation tests separately
// mvn test -Dgroups="archive"

// Internally, filters are built via DeviceGenerator static imports:
Machine device = DeviceApi.getAnyDevice(onlineDevicesFilter(), offlineDevicesFilter());
device = DeviceApi.getDevice(device.getMachineId());

// Fleet info lookup uses the machineId → fleetId resolution
String fleetId = getFleetId(device);
FleetHost fleet = DeviceApi.getFleetInfo(fleetId);
```

## Notes

- All tests extend `BaseTest` for shared setup/teardown
- `archive`-tagged tests are **destructive** — they modify device state and should run in isolated environments
- Fleet vulnerability assertions are conditionally skipped when the vulnerability list is empty, avoiding false negatives on clean environments
