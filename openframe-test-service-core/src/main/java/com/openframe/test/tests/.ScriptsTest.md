JUnit 5 integration test suite for the Scripts API, validating CRUD operations (list, get, add, edit) against the OpenFrame scripts endpoint.

## Key Components

| Method | Tags | Description |
|--------|------|-------------|
| `testListScripts()` | `saas`, `read` | Verifies scripts list is non-empty and all entries contain required fields (`id`, `name`, `category`, `shell`) |
| `testGetScript()` | `saas`, `read` | Fetches a single script by ID and asserts the response includes `scriptBody` and matches the listed record |
| `testAddScript()` | — | Creates a new script via `ScriptGenerator` and validates the API response string |
| `testEditScript()` | — | Updates a script's description and confirms both the response string and re-fetched record reflect the change |

## Usage Example

```java
// Run only read-tagged tests against SaaS environment
@Tag("saas")
@Tag("read")
// -> executes testListScripts() and testGetScript()

// Run ordered write tests sequentially
@Order(1) // testAddScript
@Order(2) // testEditScript
```

```java
// Typical assertion pattern used throughout
List<Script> scripts = ScriptApi.listScripts();
assertThat(scripts).allSatisfy(script -> {
    assertThat(script.getId()).isNotNull();
    assertThat(script.getName()).isNotEmpty();
});
```

## Notes

- Extends `BaseTest` for shared setup/teardown logic
- `testEditScript` depends on at least one existing script (populated by `testAddScript` via `@Order`)
- Uses AssertJ's `usingRecursiveComparison().ignoringFields(...)` to allow partial field matching between listed and fetched script objects
- Script data is generated via `ScriptGenerator` to keep tests self-contained
