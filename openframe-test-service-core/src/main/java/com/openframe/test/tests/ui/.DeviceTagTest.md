Playwright-based UI test class for verifying device tag filtering functionality in the OpenFrame devices management interface.

## Key Components

| Element | Description |
|---|---|
| `DeviceTagTest` | JUnit 5 test class extending `BaseUITest` for tag-based device filtering |
| `TAG_KEY` / `TAG_VALUE` | Static test data constants (`purpose` / `auto_test`) |
| `EXPECTED_CHIP` | Expected filter chip label composed as `purpose:auto_test` |
| `DEVICE_NAME` | Target device identifier (`vm115982`) used to validate filtered results |
| `openDeviceDetails()` | `@BeforeEach` setup method that navigates to the Devices page via sidebar |
| `testFilterByTag()` | Verifies tag filtering returns results, shows the target device, and displays the active filter chip |

## Usage Example

```java
// The test navigates to Devices, applies a tag filter, then asserts:
// 1. At least one device row is returned
// 2. The specific device "vm115982" is visible in results
// 3. The active filter chip "purpose:auto_test" is displayed

devicesPage.filterByTag("purpose", "auto_test");

assertThat(devicesPage.deviceRows().count()).isNotZero();

PlaywrightAssertions.assertThat(devicesPage.deviceRowByName("vm115982")).isVisible();
PlaywrightAssertions.assertThat(devicesPage.activeFilterChip("purpose:auto_test")).isVisible();
```

## Notes

- Annotated with `@Disabled` — test is skipped in CI by default until preconditions are confirmed stable
- Tagged with `@Tag("device")` for selective test suite execution
- Combines AssertJ (`assertThat`) for count checks and Playwright assertions for DOM visibility checks
