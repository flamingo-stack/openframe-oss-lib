JUnit 5 integration test class that validates the OpenFrame User API endpoints, covering user listing, retrieval, and deletion protection rules for owner accounts.

## Key Components

| Method | Tag | Description |
|--------|-----|-------------|
| `testListUsers()` | `read` | Validates all fields returned by the list users endpoint |
| `testGetUsers()` | `read` | Verifies a single user fetch matches the listed user data |
| `testDeleteOwner()` | `delete` | Asserts that owner-role users cannot be deleted (expects HTTP 409) |

## Usage Example

```java
// Run all user tests
@Tag("oss")
public class UserTest extends BaseTest { ... }

// Run only read tests via Maven/Gradle tag filter
// mvn test -Dgroups="read"

// Fetch and validate a user directly via UserApi
List<AuthUser> users = UserApi.getUsers();
AuthUser user = UserApi.getUser(users.getFirst().getId());
assertThat(user).isEqualTo(users.getFirst());

// Attempt to delete an owner-role user — expects 409 Conflict
List<AuthUser> owners = UserApi.getUsers(UserRole.OWNER);
int statusCode = UserApi.deleteUser(owners.getFirst().getId());
assertThat(statusCode).isEqualTo(409);
```

## Notes

- Extends `BaseTest` for shared test lifecycle and configuration setup.
- Uses AssertJ fluent assertions with descriptive `as()` messages for clear failure output.
- `testDeleteOwner()` also verifies the user persists post-attempt with `UserStatus.ACTIVE`, ensuring no partial deletion side effects.
- Tagged `oss` at class level, enabling selective execution in OpenFrame CI pipelines.
