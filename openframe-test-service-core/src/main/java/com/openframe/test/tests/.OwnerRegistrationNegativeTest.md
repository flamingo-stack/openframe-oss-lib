Negative test suite validating that owner user registration fails gracefully under invalid conditions, such as when registration is closed or a duplicate email is used.

## Key Components

| Component | Description |
|-----------|-------------|
| `testRegistrationClosed()` | Verifies the API returns a registration-closed error when registration is disabled |
| `testRegisterExistingUser()` | Verifies the API rejects registration attempts using an already-registered email |
| `RegistrationApi` | Handles HTTP calls to the registration endpoint |
| `RegistrationGenerator` | Factory for building request payloads and expected error responses |
| `UserApi` | Fetches existing users to source a duplicate email for testing |

## Usage Example

```java
// Run the full negative test suite
// testRegistrationClosed: attempts registration and asserts the closed-error response
UserRegistrationRequest request = RegistrationGenerator.userRegistrationRequest();
ErrorResponse response = RegistrationApi.attemptRegistration(request);
assertThat(response).isEqualTo(RegistrationGenerator.registrationClosedResponse());

// testRegisterExistingUser: fetches an existing OWNER, attempts re-registration,
// and asserts the duplicate-user error response
List<AuthUser> owners = UserApi.getUsers(UserRole.OWNER);
UserRegistrationRequest dupRequest = RegistrationGenerator.existingUserRequest(owners.getLast());
ErrorResponse dupResponse = RegistrationApi.attemptRegistration(dupRequest);
assertThat(dupResponse).isEqualTo(RegistrationGenerator.existingUserResponse());
```
