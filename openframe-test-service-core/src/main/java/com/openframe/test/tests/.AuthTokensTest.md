JUnit test suite for validating authentication token operations including refresh and logout functionality across tenant-scoped and global contexts. Extends UnauthorizedTest to test authentication workflows without requiring pre-authenticated state.

## Key Components

- **testRefreshTokens()** - Validates token refresh with tenant context, ensuring new tokens are generated and authentication is maintained
- **testRefreshTokensWithoutTenantId()** - Tests token refresh in global context without tenant scoping
- **testLogout()** - Verifies logout functionality with tenant context, confirming cookies are cleared
- **testLogoutWithoutTenantId()** - Tests logout in global context without tenant scoping

## Usage Example

```java
// Run specific test method
@Test
public void testRefreshTokens() {
    User user = UserConfig.getUser();
    Tenant tenant = TenantsCollection.findTenantByDomain(user.getDomain());
    
    // Login and get initial cookies
    Map<String, String> oldCookies = AuthFlow.login(user);
    
    // Refresh tokens
    Map<String, String> newCookies = AuthApi.refresh(tenant, oldCookies);
    assertThat(newCookies).isNotEqualTo(oldCookies);
    
    // Verify authentication with new tokens
    AuthHelper.setCookies(newCookies);
    MeResponse response = UserApi.me();
    assertThat(response.isAuthenticated()).isTrue();
}
```

The test class uses AssertJ for fluent assertions and includes monitoring tags for CI/CD pipeline integration. Note that some logout verification tests are commented out due to a known issue where 500 status codes are returned instead of expected 401 responses.
