JUnit 5 test class validating OpenFrame authentication token lifecycle — token refresh (with and without `tenantId`) and logout (with and without `tenantId`).

## Key Components

| Method | Tag | Description |
|--------|-----|-------------|
| `testRefreshTokens()` | `token` | Refreshes access tokens using `tenantId` + old cookies, asserts new cookies differ and user remains authenticated |
| `testRefreshTokensWithoutTenantId()` | `token` | Same refresh flow omitting `tenantId`, verifying the endpoint works without it |
| `testLogout()` | `logout` | Logs out with `tenantId`, asserts cookies are cleared to expected blank state |
| `testLogoutWithoutTenantId()` | `logout` | Same logout flow omitting `tenantId` |

**Class-level tags:** `oss` (OSS test suite)

> ⚠️ **Known Issue:** Both logout tests have commented-out assertions verifying that a `401` is returned on refresh after logout. Currently the API returns `500` instead — flagged for a future fix.

## Usage Example

```java
// Run all token refresh tests
@Tag("token")

// Run all logout tests
@Tag("logout")

// Run full auth token suite
@Tag("oss")
```

```bash
# Run via Maven by tag
mvn test -Dgroups="token"
mvn test -Dgroups="logout"
mvn test -Dgroups="oss"
```

## Dependencies

- `AuthFlow` — handles full login sequence and returns session cookies
- `AuthApi` — wraps `/refresh` and `/logout` endpoints
- `TenantApi` — resolves `tenantId` from user config
- `UserApi.me()` — validates active session post-refresh
- `AuthGenerator.clearedCookies()` — provides the expected empty-cookie reference for logout assertions
