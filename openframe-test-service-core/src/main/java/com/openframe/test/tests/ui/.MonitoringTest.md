JUnit 5 UI test class for the Monitoring section of the OpenFrame platform, verifying that monitoring policy statuses display correctly in the browser.

## Key Components

| Component | Description |
|-----------|-------------|
| `MonitoringTest` | Extends `BaseUITest`; tagged as `scheduled` for CI/CD scheduled runs |
| `openMonitoring()` | `@BeforeEach` setup that navigates to the Monitoring page via the sidebar |
| `testEmptyPolicyStatus()` | Asserts that the "Empty" policy reports a `Compliant` status |
| `testWindowsVersionPolicyStatus()` | Asserts that the "Windows version" policy reports a `Compliant` status |

## Usage Example

```java
// Test execution via Maven (runs all @Tag("scheduled") tests)
mvn test -Dgroups="scheduled"

// Typical flow replicated by the test:
MonitoringPage monitoringPage = navigationSidebar.goToMonitoring();

assertThat(monitoringPage.getPolicyStatusByName("Empty"))
    .isEqualTo("Compliant");

assertThat(monitoringPage.getPolicyStatusByName("Windows version"))
    .isEqualTo("Compliant");
```

## Notes

- Inherits browser setup/teardown and `navigationSidebar` from `BaseUITest`
- Uses **AssertJ** fluent assertions for readable failure messages
- The `@Tag("scheduled")` annotation enables selective execution in automated nightly or scheduled CI pipelines
- `MonitoringPage` follows the **Page Object Model (POM)** pattern, encapsulating the `getPolicyStatusByName()` lookup logic away from the test layer
