Ordered JUnit 5 integration test suite validating the full user invitation lifecycle in the OpenFrame platform, covering invite creation, acceptance, revocation, conflict handling, and user deletion scenarios.

## Key Components

| Test Method | Order | Description |
|---|---|---|
| `testInviteUser()` | 1 | Creates a new invitation and validates API response matches DB record |
| `testAcceptInvitation()` | 2 | Accepts a pending invitation and verifies email consistency |
| `testAcceptAcceptedInvitation()` | 3 | Asserts conflict response when re-accepting an already accepted invitation |
| `testRevokeInvitation()` | 4 | Revokes an invitation and confirms `REVOKED` status in DB |
| `testAcceptRevokedInvitation()` | 5 | Asserts conflict response when attempting to accept a revoked invitation |
| `testInviteActiveUser()` | 6 | Asserts conflict response when inviting an already active user |
| `testDeleteUser()` | 8 | Deletes an admin user and confirms `DELETED` status via API |
| `testInviteDeletedUser()` | 9 | Verifies a deleted user can be re-invited with `PENDING` status |

## Usage Example

```java
// Tests are ordered and executed sequentially via @TestMethodOrder
// Run the full suite with the "oss" tag:
// mvn test -Dgroups="oss"

// Run only deletion-related tests:
// mvn test -Dgroups="oss & delete"

// Key assertion pattern used throughout:
Invitation apiInvitation = InvitationApi.inviteUser(invitationRequest);
Invitation dbInvitation = InvitationsCollection.findInvitation(invitationRequest.getEmail());

assertThat(apiInvitation)
    .usingRecursiveComparison()
    .ignoringFields("updatedAt", "expiresAt", "createdAt")
    .isEqualTo(dbInvitation);

assertThat(apiInvitation.getCreatedAt())
    .isCloseTo(dbInvitation.getCreatedAt(), within(1, ChronoUnit.SECONDS));
```

> **Note:** Tests are stateful and interdependent — later tests rely on DB state created by earlier ones (e.g., `testAcceptAcceptedInvitation` depends on the accepted record from `testAcceptInvitation`). Always run the full suite in order.
