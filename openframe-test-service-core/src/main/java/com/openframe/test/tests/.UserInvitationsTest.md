Comprehensive integration test suite for OpenFrame's user invitation and user management workflow, testing the complete lifecycle from invitation creation through acceptance and edge cases.

## Key Components

- **Test Flow Management**: Uses `@Order` annotations to execute tests sequentially, ensuring proper data state between tests
- **API Integration Testing**: Tests both `InvitationApi` and `UserApi` endpoints with database validation
- **Data Validation**: Compares API responses against database state using `InvitationsCollection` and `UsersCollection`
- **Edge Case Coverage**: Tests conflict scenarios like accepting already-accepted invitations, inviting existing users, and deleting protected accounts
- **Test Data Generation**: Leverages `InvitationGenerator` for creating test data and expected responses

## Usage Example

```java
// Run the complete invitation workflow test suite
@Test
public void exampleTestFlow() {
    // 1. Invite new user
    InvitationRequest request = InvitationGenerator.newUserInvitationRequest();
    Invitation invitation = InvitationApi.inviteUser(request);
    
    // 2. Validate database state
    Invitation dbInvitation = InvitationsCollection.findInvitation(request.getEmail());
    assertThat(dbInvitation.getStatus()).isEqualTo(InvitationStatus.PENDING);
    
    // 3. Accept invitation
    AcceptInvitationRequest acceptRequest = InvitationGenerator.acceptInvitationRequest(invitation);
    AcceptInvitationResponse response = InvitationApi.acceptInvitation(acceptRequest);
    
    // 4. Test edge cases
    InvitationConflictResponse conflict = InvitationApi.attemptAcceptInvitation(acceptRequest);
    assertThat(conflict).isEqualTo(InvitationGenerator.alreadyAcceptedResponse());
}
```

The test suite ensures invitation system integrity by validating API-database consistency, testing authorization boundaries (owner deletion protection), and verifying proper status transitions throughout the user lifecycle.
