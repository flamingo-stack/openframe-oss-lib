JUnit 5 integration test suite validating the OpenFrame Knowledge Base API, covering CRUD operations for folders and articles, tag retrieval, and item lifecycle management (draft → publish).

## Key Components

| Test Method | Tags | Description |
|---|---|---|
| `testGetRootFolders` | `saas`, `read` | Verifies root folder listing returns valid items with required fields |
| `testGetArticlesInFolder` | `saas`, `read` | Fetches articles within a folder and validates filtered count consistency |
| `testGetAllTags` | `saas`, `read` | Asserts at least one tag exists with non-blank `id` and `key` |
| `testCreateDraftArticle` | `saas` | Creates a folder + draft article with tags, validates all returned fields |
| `testPublishArticle` | `saas` | Creates a draft then publishes it, asserting status transitions to `PUBLISHED` |
| `testUpdateArticle` | `saas` | Updates name, content, and summary; asserts `updatedAt` changes |
| `testCreateAndDeleteFolder` | `saas` | Creates a root folder and hard-deletes it, asserting `true` on delete |
| `testRenameFolder` | `saas` | Renames a folder and verifies the new name is persisted |
| `testMoveToFolder` | `saas` | Moves a child folder under a parent and checks `parentId` update |
| `testGetKnowledgeBaseItemById` | `saas` | Fetches both a folder and article by ID and validates their fields |

## Usage Example

```java
// Run only read-scoped SaaS tests via Maven Surefire tag filter
mvn test -Dgroups="saas & read" -Dtest=KnowledgeBaseTest

// Run the full suite (remove @Disabled to enable)
mvn test -Dtest=KnowledgeBaseTest
```

> **Note:** The class is annotated with `@Disabled` — remove this annotation to include these tests in CI runs. All write tests (`testCreate*`, `testPublish*`, etc.) generate isolated resources using `randomFolderName()` and `draftArticle()` from `KnowledgeBaseGenerator`.
