JUnit 5 integration test suite for the OpenFrame Knowledge Base API via `KnowledgeBaseApi`. Tests operate on data that already exists in the environment; mutation tests change real data **in place** and are ordered (`@Order`) so reads run first and dependent mutations run in sequence. The exception is the ordered create-folder → create-article → delete-folder trio, which creates its own data to cover the create/delete paths and cleans up at the end (the created folder is shared between them via a static field).

## Key Components

| Test Method | Order | Tag(s) | Description |
|---|---|---|---|
| `testGetRootFolders` | 1 | `saas`, `read` | Validates root folder listing — id, name, type, parentId, timestamps |
| `testGetAllTags` | 2 | `saas`, `read` | Asserts at least one tag exists with non-blank id and key |
| `testGetFolderTree` | 3 | `saas`, `read` | Asserts the folder tree is non-empty, all `FOLDER`, and contains an existing root folder |
| `testGetArticleTree` | 4 | `saas`, `read` | Asserts the article tree is non-empty and every node is `ARTICLE` with a non-blank id |
| `testGetKnowledgeBaseItemById` | 5 | `saas`, `read` | Fetches an existing folder and article by id and validates type/name |
| `testRenameFolder` | 6 | `saas` | Renames an existing root folder in place, verifies the new name |
| `testMoveToFolder` | 7 | `saas` | Moves an existing root folder under another existing folder, validates `parentId` |
| `testUpdateArticle` | 8 | `saas` | Updates an existing article's name/content/summary, verifies field changes and `updatedAt` mutation |
| `testPublishArticle` | 9 | `saas` | Publishes an existing `DRAFT` article, verifies status transitions to `PUBLISHED` |
| `testAddAndRemoveTagOnArticle` | 10 | `saas` | Adds then removes a tag on an existing article |
| `testArchiveArticle` | 11 | `saas` | Archives an existing article (stays archived), verifies it appears in `archivedArticles` |
| `testUnarchiveArticle` | 12 | `saas` | Unarchives an existing archived article into an existing folder, verifies status + `parentId` |
| `testCreateFolder` | 13 | `saas` | Creates a root folder (asserts id/type/name/null-parent); stores it in a static field for the next two tests |
| `testCreateArticle` | 14 | `saas` | Creates a draft article in the folder from `testCreateFolder` (asserts fields, author, attached tag) |
| `testDeleteFolder` | 15 | `saas` | Deletes the folder from `testCreateFolder` with `ARCHIVE` children (archives the article), asserts `true` — cleanup |

## Discovery helpers

The suite finds reusable data instead of creating it, via discovery helpers on `KnowledgeBaseApi` (they fail fast with `AssertionError` when the environment lacks the data):

- `KnowledgeBaseApi.rootFolders()` / `anyRootFolder()` — existing root folders (filter `type = FOLDER`).
- `KnowledgeBaseApi.anyArticle()` — first article from `knowledgeBaseArticleTree`.
- `KnowledgeBaseApi.anyDraftArticle()` — first article in the tree whose status is `DRAFT`.
- `KnowledgeBaseApi.tagNotOn(item)` — a tag not already attached to the given item.

> **Environment assumptions:** at least two root folders, at least one article, at least one `DRAFT` article (for publish), and at least one tag must already exist. Mutation tests permanently change real data (renamed/moved folders, archived article) — by design.
