Spring Boot auto-configuration class that automatically registers a `TacticalRmmClient` bean in the application context when the TacticalRMM SDK is on the classpath.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `TacticalRmmConfig` | `@AutoConfiguration` class | Spring Boot auto-configuration entry point for the TacticalRMM SDK |
| `tacticalRmmClient()` | `@Bean` method | Factory method that instantiates and registers `TacticalRmmClient` as a managed Spring bean |

## Usage Example

No manual configuration is required. Once the SDK dependency is declared, Spring Boot auto-discovery handles registration automatically.

```java
// Simply inject TacticalRmmClient anywhere in your Spring application
@Service
public class MyMspService {

    private final TacticalRmmClient tacticalRmmClient;

    public MyMspService(TacticalRmmClient tacticalRmmClient) {
        this.tacticalRmmClient = tacticalRmmClient;
    }

    public void performAction() {
        tacticalRmmClient.someOperation();
    }
}
```

> **Note:** For auto-configuration to be picked up, ensure the SDK's `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` file references `com.openframe.sdk.tacticalrmm.config.TacticalRmmConfig`. No `@ComponentScan` or manual `@Bean` declaration is needed in your application.
