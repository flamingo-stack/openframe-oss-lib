Reactive MongoDB repository interface for managing `ApiKey` documents, providing non-blocking database operations using Project Reactor's `Mono` and `Flux` types.

## Key Components

| Method | Return Type | Description |
|--------|-------------|-------------|
| `findByIdAndUserId` | `Mono<ApiKey>` | Fetches a single API key by its ID and owner's user ID |
| `findByUserId` | `Flux<ApiKey>` | Streams all API keys belonging to a specific user |
| `findExpiredKeys` | `Flux<ApiKey>` | Streams all enabled keys whose `expiresAt` is before the given timestamp |

## Inheritance & Generics

Extends both `ReactiveMongoRepository<ApiKey, String>` and `BaseApiKeyRepository`, binding the generic return types as:

```text
MonoApiKey  → Mono<ApiKey>
MonoBoolean → Mono<Boolean>
FluxApiKey  → Flux<ApiKey>
KeyId       → String
```

## Usage Example

```java
@Service
public class ApiKeyService {

    private final ReactiveApiKeyRepository repository;

    public ApiKeyService(ReactiveApiKeyRepository repository) {
        this.repository = repository;
    }

    // Fetch a single key scoped to a user
    public Mono<ApiKey> getKey(String keyId, String userId) {
        return repository.findByIdAndUserId(keyId, userId);
    }

    // Fetch all keys for a user
    public Flux<ApiKey> getUserKeys(String userId) {
        return repository.findByUserId(userId);
    }

    // Find expired but still-enabled keys for cleanup
    public Flux<ApiKey> getExpiredKeys() {
        return repository.findExpiredKeys(Instant.now());
    }
}
```

## Notes

- `findByIdAndUserId` uses a custom `@Query` to enforce user-scoped key lookups, preventing cross-user data access.
- `findExpiredKeys` uses a MongoDB range query (`$lt`) combined with an `enabled: true` filter, making it suitable for scheduled cleanup jobs or token invalidation pipelines.
