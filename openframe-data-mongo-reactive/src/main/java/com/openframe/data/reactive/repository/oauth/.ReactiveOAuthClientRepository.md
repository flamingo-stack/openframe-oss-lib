Reactive MongoDB repository interface for managing `OAuthClient` documents, providing non-blocking database operations using Project Reactor.

## Key Components

| Element | Description |
|---|---|
| `ReactiveMongoRepository<OAuthClient, String>` | Base interface providing standard CRUD operations reactively |
| `findByClientId(String clientId)` | Custom query method returning a single `OAuthClient` wrapped in a `Mono` |

## Usage Example

```java
@Service
public class OAuthClientService {

    private final ReactiveOAuthClientRepository repository;

    public OAuthClientService(ReactiveOAuthClientRepository repository) {
        this.repository = repository;
    }

    public Mono<OAuthClient> getClientById(String clientId) {
        return repository.findByClientId(clientId)
                .switchIfEmpty(Mono.error(new ClientNotFoundException(clientId)));
    }
}
```

## Notes

- Annotated with `@Repository` for Spring component scanning and exception translation
- All operations are non-blocking, suitable for reactive pipelines
- `findByClientId` returns `Mono.empty()` when no matching client is found — handle accordingly with `.switchIfEmpty()` or `.defaultIfEmpty()`
