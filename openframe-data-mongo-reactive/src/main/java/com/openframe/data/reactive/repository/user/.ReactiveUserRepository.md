Reactive MongoDB repository interface for `User` documents, providing non-blocking, asynchronous user lookup and existence-check operations using Project Reactor's `Mono` type.

## Key Components

| Member | Return Type | Description |
|--------|-------------|-------------|
| `findByEmail(String email)` | `Mono<User>` | Retrieves a single user by email address |
| `existsByEmail(String email)` | `Mono<Boolean>` | Checks if a user exists with the given email |
| `existsByEmailAndStatus(String email, UserStatus status)` | `Mono<Boolean>` | Checks if a user exists with the given email and specific `UserStatus` |

**Extends:**
- `ReactiveMongoRepository<User, String>` — provides full reactive CRUD operations against MongoDB
- `BaseUserRepository<Mono<User>, Mono<Boolean>, String>` — shared user query contract across reactive and blocking implementations

## Usage Example

```java
@Service
public class UserService {

    private final ReactiveUserRepository userRepository;

    public UserService(ReactiveUserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public Mono<User> getActiveUserByEmail(String email) {
        return userRepository.existsByEmailAndStatus(email, UserStatus.ACTIVE)
                .filter(Boolean::booleanValue)
                .flatMap(exists -> userRepository.findByEmail(email));
    }

    public Mono<Boolean> isEmailRegistered(String email) {
        return userRepository.existsByEmail(email);
    }
}
```

> **Note:** This repository is designed for non-blocking, reactive pipelines (e.g., Spring WebFlux). For traditional blocking contexts, use the equivalent imperative repository that shares the same `BaseUserRepository` contract.
