Reactive MongoDB repository interface for `Tenant` documents, providing non-blocking database operations using Project Reactor types.

## Key Components

| Component | Description |
|-----------|-------------|
| `ReactiveTenantRepository` | Spring `@Repository` interface extending `ReactiveMongoRepository` and `BaseTenantRepository` |
| `findByDomain(String domain)` | Reactively retrieves a `Tenant` by its domain name, returning `Mono<Tenant>` |
| `existsByDomain(String domain)` | Reactively checks if a tenant exists for a given domain, returning `Mono<Boolean>` |

## Usage Example

```java
@Service
public class TenantService {

    private final ReactiveTenantRepository tenantRepository;

    public TenantService(ReactiveTenantRepository tenantRepository) {
        this.tenantRepository = tenantRepository;
    }

    public Mono<Tenant> getTenantByDomain(String domain) {
        return tenantRepository.findByDomain(domain)
                .switchIfEmpty(Mono.error(new TenantNotFoundException(domain)));
    }

    public Mono<Boolean> isDomainRegistered(String domain) {
        return tenantRepository.existsByDomain(domain);
    }
}
```

## Notes

- Extends both `ReactiveMongoRepository<Tenant, String>` (full CRUD via Spring Data) and `BaseTenantRepository` (shared contract with blocking counterpart)
- All operations return `Mono<T>` — suitable for reactive pipelines and non-blocking I/O
- Pairs with a blocking `TenantRepository` sharing the same `BaseTenantRepository` contract, enabling consistent querying across both paradigms
