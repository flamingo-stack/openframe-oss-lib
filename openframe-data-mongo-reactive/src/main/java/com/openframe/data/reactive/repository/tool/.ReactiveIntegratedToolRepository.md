Reactive MongoDB repository interface for managing `IntegratedTool` documents, combining Spring's `ReactiveMongoRepository` with the base tool repository contract to provide non-blocking data access.

## Key Components

| Element | Description |
|---------|-------------|
| `ReactiveIntegratedToolRepository` | Main repository interface extending both `ReactiveMongoRepository<IntegratedTool, String>` and `BaseIntegratedToolRepository` |
| `findByType(String type)` | Reactively queries an `IntegratedTool` document by its type field |
| `findByKey(String key)` | Reactively queries an `IntegratedTool` document by its key field |

## Usage Example

```java
@Service
public class IntegratedToolService {

    private final ReactiveIntegratedToolRepository toolRepository;

    public IntegratedToolService(ReactiveIntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public Mono<IntegratedTool> getToolByType(String type) {
        return toolRepository.findByType(type)
                .switchIfEmpty(Mono.error(new ToolNotFoundException("No tool found for type: " + type)));
    }

    public Mono<IntegratedTool> getToolByKey(String key) {
        return toolRepository.findByKey(key)
                .doOnSuccess(tool -> log.info("Retrieved tool: {}", tool.getKey()));
    }
}
```

## Notes

- Returns are wrapped in `Mono<IntegratedTool>` for fully non-blocking, reactive data access
- Backed by **MongoDB** via Spring Data's reactive driver
- Designed for use in reactive pipelines (e.g., WebFlux-based services)
- Inherits standard CRUD operations from `ReactiveMongoRepository` in addition to the custom lookup methods
