Reactive MongoDB repository interface for managing `IntegratedTool` documents, combining Spring's `ReactiveMongoRepository` with the shared `BaseIntegratedToolRepository` contract to provide non-blocking data access.

## Key Components

| Element | Description |
|---|---|
| `ReactiveMongoRepository<IntegratedTool, String>` | Base Spring Data reactive MongoDB CRUD operations |
| `BaseIntegratedToolRepository<Mono<...>>` | Shared repository contract using `Mono` as the reactive return type |
| `findByType(String type)` | Looks up a single `IntegratedTool` by its integration type |
| `findByKey(String key)` | Looks up a single `IntegratedTool` by its unique key |

## Usage Example

```java
@Service
public class ToolService {

    private final ReactiveIntegratedToolRepository toolRepository;

    public ToolService(ReactiveIntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public Mono<IntegratedTool> getToolByType(String type) {
        return toolRepository.findByType(type)
                .switchIfEmpty(Mono.error(
                    new ToolNotFoundException("No tool found for type: " + type)
                ));
    }

    public Mono<IntegratedTool> getToolByKey(String key) {
        return toolRepository.findByKey(key)
                .doOnNext(tool -> log.info("Found tool: {}", tool.getKey()));
    }
}
```

> Both query methods return `Mono<IntegratedTool>`, meaning they emit at most one result and integrate cleanly into reactive pipelines using Project Reactor operators such as `flatMap`, `switchIfEmpty`, or `doOnNext`.
