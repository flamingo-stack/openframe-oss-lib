Reactive MongoDB repository interface for managing `IntegratedTool` documents, providing non-blocking database operations using Project Reactor types.

## Key Components

| Element | Type | Description |
|---|---|---|
| `ReactiveIntegratedToolRepository` | Interface | Spring Data reactive repository for `IntegratedTool` documents |
| `findByType(String type)` | `Mono<IntegratedTool>` | Queries a single integrated tool by its type identifier |

**Extends:**
- `ReactiveMongoRepository<IntegratedTool, String>` — Full reactive CRUD support for MongoDB
- `BaseIntegratedToolRepository<Mono<IntegratedTool>, Mono<Boolean>, String>` — Shared contract across reactive and blocking repository implementations

## Usage Example

```java
@Service
public class ToolService {

    private final ReactiveIntegratedToolRepository toolRepository;

    public ToolService(ReactiveIntegratedToolRepository toolRepository) {
        this.toolRepository = toolRepository;
    }

    public Mono<IntegratedTool> getToolByType(String type) {
        return toolRepository.findByType(type)
                .switchIfEmpty(Mono.error(
                    new ToolNotFoundException("No tool found for type: " + type)
                ));
    }

    public Mono<IntegratedTool> saveTool(IntegratedTool tool) {
        return toolRepository.save(tool);
    }
}
```

> **Note:** This repository is part of the OpenFrame reactive data layer. Use it in non-blocking, reactive pipelines (e.g., WebFlux controllers or reactive service chains) to avoid blocking the event loop. For blocking/imperative contexts, use the non-reactive counterpart that also implements `BaseIntegratedToolRepository`.
