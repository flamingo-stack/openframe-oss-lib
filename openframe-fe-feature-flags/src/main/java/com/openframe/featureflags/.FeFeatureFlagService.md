Resolves effective frontend feature flags for a given cluster by merging YAML-configured defaults with per-tenant database overrides, using a short-lived Caffeine cache to minimize database calls on hot paths.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `CACHE_TTL` | `Duration` (5 min) | TTL for the Caffeine override cache |
| `overridesCache` | `Cache<String, Map<String, Boolean>>` | Per-cluster cache keyed by `clusterId` |
| `getEffectiveFlags()` | `public` method | Returns the merged flag map (defaults + DB overrides) |
| `loadOverrides()` | `private` method | Fetches `FeFeatureFlags` from the repository with cache-aside logic |

## Merge Strategy

```text
Priority: DB override (FeFeatureFlags document) > YAML default (FeFeatureFlagProperties)
```

1. YAML defaults populate the base `LinkedHashMap` (insertion order preserved)
2. DB overrides are applied via `putAll`, overwriting matching keys and appending new ones
3. On repository failure, an empty map is returned silently (warn-logged) so defaults always serve

## Usage Example

```java
@Component
public class FlagResolver {

    private final FeFeatureFlagService flagService;

    public FlagResolver(FeFeatureFlagService flagService) {
        this.flagService = flagService;
    }

    public boolean isEnabled(String flagName) {
        return flagService.getEffectiveFlags()
                .getOrDefault(flagName, false);
    }
}
```

## Configuration

| Property | Description |
|----------|-------------|
| `openframe.cluster-id` | Injected via `@Value`; used as the cache key and repository lookup ID |
| `FeFeatureFlagProperties` | YAML-bound default flag definitions |
| `FeFeatureFlagsRepository` | MongoDB/document repository for per-tenant overrides |
