Spring Boot auto-configuration entry point for the OpenFrame feature flags module. Activates automatically when `DgsComponent` (Netflix DGS GraphQL) is on the classpath, enabling component scanning and binding configuration properties for feature flag management.

## Key Components

| Element | Description |
|---|---|
| `@AutoConfiguration` | Registers this class as a Spring Boot auto-configuration |
| `@ConditionalOnClass(DgsComponent.class)` | Only activates when Netflix DGS GraphQL is present on the classpath |
| `@EnableConfigurationProperties` | Binds `FeFeatureFlagProperties` to the Spring environment |
| `@ComponentScan` | Scans all components within the `com.openframe.featureflags` package |

## Usage Example

No manual wiring is required. Include the module as a dependency and Spring Boot will auto-configure it via `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`.

```java
// application.yml — configure feature flags via bound properties
fe:
  feature-flags:
    enabled: true
    some-flag: false
```

```java
// Any Spring-managed bean can then inject the properties
@Service
public class MyService {

    private final FeFeatureFlagProperties featureFlags;

    public MyService(FeFeatureFlagProperties featureFlags) {
        this.featureFlags = featureFlags;
    }
}
```

> **Prerequisite:** The Netflix DGS GraphQL dependency must be present on the classpath. If `DgsComponent` is not detected, this auto-configuration is skipped entirely and no feature flag beans are registered.
