GraphQL data fetcher that exposes feature flag state via a DGS query, allowing clients to retrieve one or all feature flags by name.

## Key Components

- **`FeFeatureFlagDataFetcher`** — DGS component implementing the `feFeatureFlags` GraphQL query
- **`feFeatureFlags(@InputArgument List<String> names)`** — Query resolver that accepts an optional list of flag names; returns all flags when `names` is null or empty, otherwise filters to the requested subset
- **`FeFeatureFlagService`** — Injected service dependency providing the resolved flag map via `getEffectiveFlags()`
- **`FeFeatureFlag`** — Builder-based model representing a flag with `name` and `enabled` fields

## Usage Example

```java
// GraphQL query — fetch all feature flags
query {
  feFeatureFlags {
    name
    enabled
  }
}

// GraphQL query — fetch specific flags by name
query {
  feFeatureFlags(names: ["dark_mode", "new_dashboard"]) {
    name
    enabled
  }
}
```

```java
// Internal resolution logic (simplified)
Map<String, Boolean> flags = featureFlagService.getEffectiveFlags();
// names = null → returns all flags
// names = ["dark_mode"] → returns only that flag
return keys.stream()
    .map(name -> FeFeatureFlag.builder()
        .name(name)
        .enabled(flags.getOrDefault(name, false))
        .build())
    .toList();
```

> Unknown flag names not present in `getEffectiveFlags()` default to `enabled: false` via `getOrDefault`.
