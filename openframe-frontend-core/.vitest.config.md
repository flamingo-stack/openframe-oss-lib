Vitest configuration file for the project's test suite, defining the test environment, coverage settings, path aliases, and exclusion rules.

## Key Components

| Option | Value | Purpose |
|--------|-------|---------|
| `environment` | `jsdom` | Simulates a browser DOM for component/UI tests |
| `globals` | `true` | Enables global test APIs (`describe`, `it`, `expect`) without imports |
| `setupFiles` | `./vitest.setup.ts` | Runs setup script before each test file |
| `exclude` | `node_modules`, `dist`, `.yalc` | Prevents vitest from scanning compiled/linked artifacts |
| `coverage.provider` | `v8` | Uses Node's built-in V8 coverage engine |
| `coverage.reporter` | `text`, `json`, `html` | Outputs coverage in multiple formats |
| `resolve.alias` | `@` → `./src` | Allows `@/` absolute imports throughout the codebase |

## Notable Exclusions

The `.yalc/**` exclusion is intentional — when a downstream consumer mounts this package via `yalc` during local development, the `.yalc/` folder contains compiled-but-not-test-runnable source copies that break Vitest's setup file resolution.

## Usage Example

```bash
# Run all tests
npx vitest

# Run with coverage report
npx vitest run --coverage

# Watch mode during development
npx vitest watch
```

```typescript
// Leverage the @ alias in any test file
import { MyComponent } from '@/components/MyComponent'

describe('MyComponent', () => {
  it('renders correctly', () => {
    // test using jsdom environment
  })
})
```
