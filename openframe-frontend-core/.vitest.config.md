Configuration file for Vitest testing framework that sets up DOM testing environment, global test utilities, coverage reporting, and path aliases for the Flamingo platform's testing infrastructure.

## Key Components

- **Test Environment**: Configures `jsdom` for browser-like testing environment with global test functions
- **Setup Files**: References `vitest.setup.ts` for test initialization and configuration
- **Coverage Configuration**: Uses V8 provider with multiple output formats (text, JSON, HTML)
- **Path Alias**: Maps `@` to the `src` directory for clean imports in tests
- **Coverage Exclusions**: Excludes common non-testable directories and files

## Usage Example

This configuration enables testing with the following capabilities:

```typescript
// Tests can use global functions without imports
describe('Component Tests', () => {
  it('should render correctly', () => {
    // DOM testing with jsdom environment
    const element = document.createElement('div')
    expect(element).toBeDefined()
  })
})

// Clean imports using @ alias
import { Component } from '@/components/Component'
import { utils } from '@/lib/utils'
```

Run tests with coverage:

```bash
# Run all tests with coverage report
vitest --coverage

# Watch mode for development
vitest --watch
```

The configuration generates coverage reports in multiple formats and excludes test files, setup files, and build artifacts from coverage calculations.
