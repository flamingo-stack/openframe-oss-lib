Global Vitest test setup file that configures shared mock state for navigation hooks, router functions, and URL search parameters across all test suites.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `mockReplace` | `vi.fn()` | Spy for `router.replace()` calls |
| `mockPush` | `vi.fn()` | Spy for `router.push()` calls |
| `mockSearchParams` | `Proxy<URLSearchParams>` | Live proxy that always reflects the current search params state |
| `setMockSearchParams` | `function` | Swaps the active `URLSearchParams` instance between test assertions |

## Architecture Note

Because `useApiParams` and related hooks consume the embed-shim at `src/embed-shims/next-navigation.tsx` rather than importing directly from `next/navigation`, `vi.mock('next/navigation')` has no effect. Navigation is mocked by calling `registerNavigation()` with a test-only implementation instead.

## Usage Example

```typescript
import { setMockSearchParams, mockPush, mockReplace } from '../vitest.setup'

it('reads query params and navigates', () => {
  // Arrange: inject URL state before rendering
  setMockSearchParams(new URLSearchParams('page=2&filter=active'))

  const { result } = renderHook(() => useApiParams())

  // Assert: values from the mocked search params
  expect(result.current.page).toBe('2')

  // Trigger navigation, then assert on the spy
  result.current.navigate({ page: '3' })
  expect(mockReplace).toHaveBeenCalledWith('/?page=3&filter=active')
})
```

## Automatic Reset

A `beforeEach` hook automatically clears `mockReplace` and `mockPush` call history and resets `currentSearchParams` to an empty `URLSearchParams`, keeping assertions isolated between tests without any per-test teardown boilerplate.

## `window.location` Shim

`window.location` is redefined via `Object.defineProperty` to expose `assign`, `replace`, and `reload` as `vi.fn()` stubs, preventing `TypeError: ... is not a function` errors from code paths that bypass the router (e.g. `permanentRedirect`).
