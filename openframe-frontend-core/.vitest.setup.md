Vitest test setup configuration that provides global test utilities, mocks Next.js navigation hooks, and ensures clean test isolation between test runs.

## Key Components

- **mockReplace/mockPush**: Mock functions for Next.js router navigation methods
- **mockSearchParams**: Mutable URLSearchParams instance for testing search parameter interactions
- **setMockSearchParams()**: Helper function to update search parameters in tests
- **Navigation mocks**: Complete mocking of `next/navigation` hooks including `useRouter`, `useSearchParams`, and `usePathname`
- **Window location mock**: Configurable `window.location` object for testing location-dependent code
- **beforeEach cleanup**: Automatic reset of all mocks between tests

## Usage Example

```typescript
import { mockPush, mockSearchParams, setMockSearchParams } from './vitest.setup'

describe('Navigation Component', () => {
  test('should navigate on button click', () => {
    render(<NavigationButton />)
    
    fireEvent.click(screen.getByText('Navigate'))
    
    expect(mockPush).toHaveBeenCalledWith('/dashboard')
  })

  test('should handle search parameters', () => {
    setMockSearchParams(new URLSearchParams('?tab=settings'))
    
    render(<TabComponent />)
    
    expect(mockSearchParams.get('tab')).toBe('settings')
  })
})
```

This setup file is automatically loaded by Vitest and provides consistent mocking across all test files, eliminating the need to manually mock Next.js navigation in individual tests.
