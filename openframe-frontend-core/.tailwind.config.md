Tailwind CSS configuration for the Flamingo/OpenFrame frontend, extending the default theme with ODS (Open Design System) tokens, custom typography utilities, and responsive breakpoints.

## Key Components

### `odsTypographyPlugin`
A custom Tailwind plugin that registers `.text-h1` through `.text-h6` utility classes, each mapped to CSS variables for font family, weight, size, line height, and letter spacing.

### `config` (default export)
The full Tailwind config with:

- **ODS Color Palette** — Semantic color tokens under the `ods.*` namespace (backgrounds, borders, text, accent, status: success/error/warning/info, links, and brand colors)
- **Custom Breakpoints** — `md` (800px), `lg` (1280px), `xl` (1440px) aligned with ODS Figma tokens
- **Typography** — `font-heading-1` through `font-heading-6` font-size utilities backed by CSS variables
- **Shadows** — Named shadow utilities (`card`, `modal`, `dropdown`, `tooltip`, `focus`, `accent`)
- **Transitions** — Duration (`fast`, `normal`, `slow`, `slower`) and easing (`bounce`, `elastic`) tokens
- **Z-Index** — Layering scale (`dropdown` → `modal` → `tooltip` → `notification`)
- **Animations** — Accordion expand/collapse and fade-in keyframes
- **Plugins** — `tailwindcss-animate`, `odsTypographyPlugin`, `@tailwindcss/container-queries`

## Usage Example

```typescript
// Apply ODS semantic colors and typography utilities in a component
<div className="bg-ods-card border border-ods-border text-ods-text-primary shadow-card">
  <h1 className="text-h1 text-ods-accent">Dashboard</h1>
  <p className="text-ods-text-secondary">Welcome back</p>
  <button className="bg-ods-accent hover:bg-ods-accent-hover transition-fast">
    Get Started
  </button>
</div>
```
