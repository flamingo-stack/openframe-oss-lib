Tailwind CSS configuration that implements the Open Design System (ODS) color palette with shadcn/ui compatibility and table functionality utilities.

## Key Components

- **Content scanning**: Includes all TypeScript/JavaScript source files for class detection
- **Safelist**: Preserves responsive visibility classes for dynamic table functionality
- **ODS Color System**: Comprehensive color palette using CSS custom properties
- **Dual color structure**: Both flat (`ods-bg`) and nested (`ods.bg`) naming conventions
- **Typography**: Custom font families with DM Sans and Azeret Mono
- **Animation plugins**: Includes tailwindcss-animate for UI components

## Usage Example

```typescript
// Using ODS colors in components
<div className="bg-ods-card border border-ods-border text-ods-text-primary">
  <h1 className="text-ods-accent font-heading">Header</h1>
  <p className="text-ods-text-secondary">Content</p>
</div>

// Responsive table visibility (preserved by safelist)
<td className="hidden sm:flex lg:hidden">Data</td>

// Using nested color structure
<button className="bg-ods-accent hover:bg-ods-accent-hover">
  Click me
</button>
```

The configuration maintains backward compatibility with shadcn/ui components while providing a unified design system for consistent theming across Flamingo applications. All colors reference CSS custom properties defined in the application's global styles.
