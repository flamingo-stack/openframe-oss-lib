Provides a React context for configuring the chat panel's API endpoints, navigation behavior, and identity settings across different host environments (hub vs. embedded apps).

## Key Components

### `ChatRuntime` Interface
The primary configuration shape with four top-level concerns:

- **`endpoints`** — All API URLs: `chatStreamUrl`, `approvalToolUrl`, `commandsUrl`, `buildListUrl`, attachment upload/view, identity, image proxy, and Supabase storage origin
- **`navigation`** — Mode (`'host'` | `'embed'`), `defaultContentOrigin`, `openExternal`, `navigate`, and `decideNewTab` callbacks
- **`resolvePlaceholderUrl`** — Optional OG placeholder URL builder
- **`composeContentUrl`** — Optional platform-aware content href composer (single authority for all embeddable surfaces)
- **`source`** — Optional platform identifier (hub only; embedders leave unset)

### `ChatRuntimeContext`
A `React.createContext<ChatRuntime | null>(null)` — the single seam between host/embed environments and the chat panel internals.

### `useChatRuntime()`
Returns the active `ChatRuntime` or `null`. Null is a valid signal — optional consumers fall back to no-op behavior.

### `useRequiredChatRuntime()`
Strict variant used inside the chat panel. Throws a descriptive error if called outside a provider, guiding developers to wrap with `<HubRuntimeProvider>` or a mock provider.

## Usage Example

```typescript
// Hub: already handled by <HubRuntimeProvider>

// Embedded app
const runtime = useMemo<ChatRuntime>(() => ({
  endpoints: {
    chatStreamUrl: 'https://hub.openframe.ai/api/docs/chat',
    approvalToolUrl: 'https://hub.openframe.ai/api/chat/agent/confirm-tool',
    commandsUrl: 'https://hub.openframe.ai/api/docs/commands',
    buildListUrl: (type, ids) => null,
    attachmentUploadUrl: 'https://hub.openframe.ai/api/storage/upload',
    attachmentViewUrlPrefix: 'https://hub.openframe.ai/api/storage/view/chat-attachments/',
    identityUrl: 'https://hub.openframe.ai/api/chat/identity',
  },
  navigation: {
    mode: 'embed',
    defaultContentOrigin: 'https://hub.openframe.ai',
  },
}), [])

return (
  <ChatRuntimeContext.Provider value={runtime}>
    <ChatPanel />
  </ChatRuntimeContext.Provider>
)
```

> **Memoize the provider value.** Every reference change invalidates downstream `useMemo` consumers across the entire chat tree (slash-commands binding, `useNavLink`, `useDocChat` stream factory).
