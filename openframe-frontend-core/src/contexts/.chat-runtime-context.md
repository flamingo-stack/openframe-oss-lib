Provides the React context and typed interface that supplies the chat panel with environment-specific API endpoints, navigation behavior, and identity configuration ‚Äî enabling the same chat component to run in both the hub and embedded/reverse-proxied deployments without code changes.

## Key Components

- **`ChatRuntime`** ‚Äî Core interface with four top-level concerns:
  - `endpoints` ‚Äî All API URLs (`chatStreamUrl`, `approvalToolUrl`, `commandsUrl`, `buildListUrl`, attachment + identity endpoints, image proxy config, Supabase storage origin)
  - `navigation` ‚Äî Mode (`'host'` | `'embed'`), `defaultContentOrigin`, `openExternal`, `navigate`, and `decideNewTab` callbacks
  - `resolvePlaceholderUrl` / `composeContentUrl` / `docPlatformTargets` ‚Äî Optional content-URL and OG-placeholder wiring
  - `source` ‚Äî Optional platform identifier (hub only; embedders omit)
- **`ChatRuntimeContext`** ‚Äî `React.createContext<ChatRuntime | null>(null)`
- **`useChatRuntime()`** ‚Äî Optional consumer; returns `null` when no provider is mounted
- **`useRequiredChatRuntime()`** ‚Äî Strict consumer used inside the chat panel; throws with a descriptive error if called outside a provider

## Usage Example

```typescript
// Hub ‚Äî memoize to avoid re-render cascade across the chat tree
const runtime = useMemo<ChatRuntime>(() => ({
  endpoints: {
    chatStreamUrl: '/api/docs/chat',
    approvalToolUrl: '/api/chat/agent/confirm-tool',
    commandsUrl: '/api/docs/commands',
    buildListUrl: (type, ids) => buildHubListUrl(type, ids),
    attachmentUploadUrl: '/api/storage/upload/chat-attachments',
    attachmentViewUrlPrefix: '/api/storage/view/chat-attachments/',
    identityUrl: '/api/chat/identity',
  },
  navigation: { mode: 'host', navigate: hubNavigate },
}), [hubNavigate])

return (
  <ChatRuntimeContext.Provider value={runtime}>
    <ChatPanel />
  </ChatRuntimeContext.Provider>
)

// Embedded app ‚Äî forces new-tab for all in-chat links
const runtime = useMemo<ChatRuntime>(() => ({
  endpoints: {
    chatStreamUrl: '/api/mingo-guide/docs/chat',
    // ... other endpoints proxied under /api/mingo-guide/*
    attachmentUploadUrl: '/api/mingo-guide/storage/upload/chat-attachments',
    attachmentViewUrlPrefix: 'https://hub.openframe.ai/api/storage/view/chat-attachments/',
    identityUrl: '/api/mingo-guide/chat/identity',
    buildListUrl: () => null,
  },
  navigation: {
    mode: 'embed',
    defaultContentOrigin: 'https://hub.openframe.ai',
  },
}), [])

// Inside chat internals ‚Äî throws if no provider
const { endpoints, navigation } = useRequiredChatRuntime()
```

> **Performance note:** Always `useMemo` the value passed to `<ChatRuntimeContext.Provider>`. An unstable reference invalidates the slash-commands binding, `useNavLink`'s embed-resolution memo, and `useDocChat`'s stream factory on every render.
