Defines the centralized source of truth for operating system type metadata across the Flamingo platform, normalizing OS values from Fleet MDM, Tactical RMM, and GraphQL into a unified structure.

## Key Components

- **`OSTypeValues`** — Const enum-like object with `WINDOWS`, `MACOS`, and `LINUX` string literals
- **`OSType`** — Union type derived from `OSTypeValues`
- **`OSTypeDefinition`** — Interface describing a full OS entry: `id`, `label`, `value`, `icon`, `platformId`, and `aliases`
- **`OS_TYPES`** — Primary export; array of `OSTypeDefinition` objects for all three platforms, including brand icons and alias lists for fuzzy matching
- **`osLabels`** — Convenience record mapping each `OSType` to its display label

## Usage Example

```typescript
import { OS_TYPES, OSTypeValues, osLabels } from './os.types'

// Resolve an OS from an arbitrary string (e.g. from Fleet MDM)
function resolveOS(raw: string) {
  const normalized = raw.toLowerCase()
  return OS_TYPES.find(os =>
    os.aliases.includes(normalized)
  )
}

const match = resolveOS('darwin')
// match.id    → 'MACOS'
// match.label → 'macOS'
// match.platformId → 'darwin'

// Render display label from known type
const label = osLabels[OSTypeValues.WINDOWS]
// → 'Windows'

// Render the OS icon in a React component
const OsIcon = match?.icon
// <OsIcon className="w-4 h-4" />
```

## Notes

- Aliases are **order-sensitive** — more specific strings (e.g. `'darwin'`) precede shorter ones (e.g. `'mac'`) to prevent false-positive matches
- Linux aliases cover major distros (`ubuntu`, `debian`, `centos`, `fedora`, `arch`, etc.)
- Icons are pulled from the internal `icons-v2-generated` brand logo set
