Centralized operating system type definitions and normalization utilities for the Flamingo platform. Provides a single source of truth for OS information across Fleet MDM, Tactical RMM, and GraphQL integrations.

## Key Components

- **`OSTypeValues`** - Constant enum defining the three supported OS types (WINDOWS, MACOS, LINUX)
- **`OSType`** - TypeScript type derived from OSTypeValues for type safety
- **`OSTypeDefinition`** - Interface defining OS metadata including label, icon, platform ID, and aliases
- **`OS_TYPES`** - Array of complete OS definitions with React icons and normalization aliases
- **`osLabels`** - Simple mapping from OS type to display label

## Usage Example

```typescript
import { OS_TYPES, OSTypeValues, type OSType } from './os.types'

// Find OS by alias (handles normalization)
const normalizeOS = (rawOS: string): OSType | null => {
  const osDefinition = OS_TYPES.find(os => 
    os.aliases.some(alias => 
      alias.toLowerCase() === rawOS.toLowerCase()
    )
  )
  return osDefinition?.id || null
}

// Use in components
const DeviceIcon = ({ osType }: { osType: OSType }) => {
  const osDefinition = OS_TYPES.find(os => os.id === osType)
  const IconComponent = osDefinition?.icon
  return IconComponent ? <IconComponent /> : null
}

// Type-safe OS checking
if (device.os === OSTypeValues.MACOS) {
  // Handle macOS-specific logic
}
```
