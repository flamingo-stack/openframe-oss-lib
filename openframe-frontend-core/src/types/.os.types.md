This file provides a centralized type system and utilities for handling operating system information across different data sources and APIs.

## Key Components

### Types
- **`OSType`** - Union type for standardized OS identifiers: `'WINDOWS' | 'MACOS' | 'LINUX'`
- **`OSTypeDefinition`** - Complete OS metadata including ID, label, icon component, platform ID, and aliases

### Constants
- **`OS_TYPES`** - Master array containing all OS type definitions with icons and aliases for normalization
- **`osLabels`** - Simple mapping from OS type to display labels

### Core Functions
- **`normalizeOSType()`** - Converts raw OS strings to standardized `OSType` enum values
- **`getOSLabel()`** - Returns display-friendly labels for OS types
- **`getOSIcon()`** - Retrieves React icon components for OS types
- **`getOSTypeDefinition()`** - Gets complete OS metadata by type string
- **`getOSPlatformId()`** - Maps OS types to platform IDs for cross-referencing
- **`isOSPlatform()`** - Checks if a device OS matches a specific platform

## Usage Example

```typescript
import { normalizeOSType, getOSLabel, getOSIcon, isOSPlatform } from './os.types'

// Normalize various OS string formats
const osType1 = normalizeOSType('Darwin') // 'MACOS'
const osType2 = normalizeOSType('win32') // 'WINDOWS' 
const osType3 = normalizeOSType('Ubuntu 20.04') // 'LINUX'

// Get display information
const label = getOSLabel('windows') // 'Windows'
const IconComponent = getOSIcon('macos') // MacOSIcon

// Platform checking
const isWindows = isOSPlatform('Windows 10', 'windows') // true
const isMac = isOSPlatform('Darwin', 'darwin') // true

// Use in React components
function DeviceCard({ deviceOS }: { deviceOS: string }) {
  const IconComponent = getOSIcon(deviceOS)
  const label = getOSLabel(deviceOS)
  
  return (
    <div>
      {IconComponent && <IconComponent />}
      <span>{label}</span>
    </div>
  )
}
```
