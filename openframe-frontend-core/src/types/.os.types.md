This file provides a centralized system for managing operating system types and normalizing OS values from various sources across the platform.

## Key Components

- **`OSType`** - Union type for the three supported OS categories: `'WINDOWS'`, `'MACOS'`, `'LINUX'`
- **`OSTypeDefinition`** - Interface defining OS metadata including ID, label, icon component, platform ID, and aliases
- **`OS_TYPES`** - Master array containing all OS definitions with their icons, labels, and alias mappings
- **`normalizeOSType()`** - Converts raw OS strings to standardized `OSType` values using fuzzy matching
- **`getOSLabel()`** - Returns user-friendly display labels for OS types
- **`getOSIcon()`** - Retrieves appropriate icon components for OS types
- **`getOSTypeDefinition()`** - Returns complete OS metadata for a given OS string
- **`isOSPlatform()`** - Checks if a device OS matches a specific platform

## Usage Example

```typescript
import { normalizeOSType, getOSLabel, getOSIcon } from './os.types'

// Normalize various OS strings
const osType1 = normalizeOSType('Darwin') // 'MACOS'
const osType2 = normalizeOSType('windows 10') // 'WINDOWS'
const osType3 = normalizeOSType('Ubuntu 20.04') // 'LINUX'

// Get display components
const label = getOSLabel('darwin') // 'macOS'
const IconComponent = getOSIcon('windows')

// Check platform compatibility
if (isOSPlatform(deviceOS, 'darwin')) {
  // Handle macOS-specific logic
}
```
