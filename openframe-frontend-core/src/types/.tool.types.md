Centralized type definitions for MSP tool types used throughout the Flamingo/OpenFrame platform. Provides a single source of truth for tool type constants and display labels.

## Key Components

- **`ToolTypeValues`** - Const assertion object containing all available tool type constants (TACTICAL_RMM, FLEET_MDM, MESHCENTRAL, etc.)
- **`ToolType`** - Union type derived from ToolTypeValues for type-safe tool type references
- **`toolLabels`** - Record mapping tool types to human-readable display names

## Usage Example

```typescript
import { ToolType, ToolTypeValues, toolLabels } from './tool.types'

// Type-safe tool type usage
const currentTool: ToolType = ToolTypeValues.TACTICAL_RMM

// Get display label for UI
const displayName = toolLabels[currentTool] // "Tactical"

// Use in component props
interface ToolBadgeProps {
  type: ToolType
  label?: string
}

function ToolBadge({ type, label }: ToolBadgeProps) {
  const defaultLabel = toolLabels[type]
  return <span>{label || defaultLabel}</span>
}

// Usage with type safety
<ToolBadge type={ToolTypeValues.OPENFRAME_CHAT} />
```

The const assertion pattern ensures type safety while maintaining runtime access to the values, making it ideal for components like ToolBadge and ToolIcon that need both compile-time type checking and runtime value access.
