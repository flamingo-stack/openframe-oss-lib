Provides centralized type definitions and utilities for all tool-related components across the platform. This file serves as the single source of truth for tool types, ensuring consistency between components like ToolBadge and ToolIcon.

## Key Components

- **`ToolType`** - Union type defining all supported platform tools (Tactical RMM, Fleet MDM, MeshCentral, etc.)
- **`toolLabels`** - Record mapping tool types to human-readable display names
- **`getToolLabel()`** - Utility function to safely retrieve display labels with fallback

## Usage Example

```typescript
import { ToolType, getToolLabel, toolLabels } from './tool.types'

// Type-safe tool identification
const currentTool: ToolType = 'TACTICAL_RMM'

// Get display label
const displayName = getToolLabel(currentTool) // Returns "Tactical"

// Direct label access
const allLabels = Object.entries(toolLabels)
// [['TACTICAL_RMM', 'Tactical'], ['FLEET_MDM', 'Fleet'], ...]

// Component usage
function ToolBadge({ tool }: { tool: ToolType }) {
  return <span>{getToolLabel(tool)}</span>
}
```

The centralized approach prevents inconsistent tool naming and makes it easy to add new tools or update existing labels across the entire platform.
