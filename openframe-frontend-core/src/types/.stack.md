This file defines the complete type system for MSP (Managed Service Provider) technology stack management, including vendor selections, cost calculations, and stack sharing functionality.

## Key Components

**Core Types:**
- `MSP` - Managed Service Provider interface with company details
- `UserStack` - Main stack structure containing vendor selections
- `StackVendor` - Extended vendor with stack-specific metadata
- `StackVendorSelection` - Individual vendor choice within a category

**Selection Types:**
- `SelectionSource` - Tracks if vendor was AI-selected or manually chosen
- `StackVendorType` - Distinguishes between 'current' and 'alternative' vendors

**Cost Analysis:**
- `StackCostCalculation` - Complete cost breakdown and savings analysis
- `CategoryCostBreakdown` - Per-category cost comparison
- `MarginAnalysis` - Profit margin improvements

**API Interfaces:**
- Request/response types for CRUD operations
- Component prop interfaces for React components
- Filter and sorting options

## Usage Example

```typescript
// Create a new stack
const stackRequest: CreateStackRequest = {
  name: "Enterprise MSP Stack",
  description: "Technology stack for 500+ seat MSP",
  is_public: true
};

// Add vendor selection
const vendorSelection: StackVendorSelectionRequest = {
  category_slug: "rmm",
  subcategory_slug: "endpoint-management", 
  vendor_slug: "connectwise-automate",
  vendor_type: "current",
  selection_source: "manual"
};

// Cost calculation options
const costOptions: CostCalculationOptions = {
  seat_count: 500,
  include_implementation_costs: true,
  msp_markup_percentage: 30
};
```
