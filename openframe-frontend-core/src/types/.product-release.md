TypeScript definitions for product release management system with comprehensive change tracking and external integrations.

## Key Components

- **ProductRelease**: Main interface defining release structure with changelog categories, metadata, and external links
- **ChangelogEntry**: Individual changelog item with title and optional description
- **ReleaseMedia**: Media attachments (images, videos, screenshots, demos) with ordering
- **External Link Interfaces**: GitHub releases, knowledge base articles, ClickUp tasks for roadmap/delivery tracking
- **CreateProductReleaseData**: Data transfer object for creating new releases with platform and tag associations
- **ProductReleaseFilters**: Query parameters for filtering releases by various criteria
- **Configuration Objects**: Pre-defined options for release types, statuses, and changelog labels

## Usage Example

```typescript
import { 
  ProductRelease, 
  CreateProductReleaseData, 
  releaseTypeOptions,
  SEMVER_REGEX 
} from './product-release'

// Create a new product release
const newRelease: CreateProductReleaseData = {
  title: "API Gateway v2.1.0",
  slug: "api-gateway-v2-1-0",
  version: "2.1.0",
  summary: "Enhanced authentication and rate limiting",
  release_type: "minor",
  release_status: "stable",
  release_date: "2024-03-15",
  features_added: [
    { title: "JWT token validation", description: "Improved security" }
  ],
  bugs_fixed: [
    { title: "Memory leak in connection pool" }
  ],
  platforms: ["platform-uuid"],
  tags: [1, 2, 3],
  status: "published",
  author_id: "user-123"
}

// Validate semantic version
const isValidVersion = SEMVER_REGEX.test("2.1.0") // true

// Filter releases
const filters: ProductReleaseFilters = {
  platform: "web",
  release_type: ["major", "minor"],
  has_breaking_changes: false,
  search: "authentication"
}
```
