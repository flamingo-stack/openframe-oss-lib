A TypeScript interface definition for standardized icon component props in React applications.

## Key Components

**IconProps Interface**
- Extends React's `SVGProps<SVGSVGElement>` while omitting `width` and `height`
- Provides consistent prop structure for all icon components
- Includes optional styling and sizing properties

## Usage Example

```typescript
import { IconProps } from './icons';

// Icon component implementation
const HomeIcon: React.FC<IconProps> = ({ 
  size = 24, 
  color = 'currentColor', 
  className,
  ...props 
}) => (
  <svg
    width={size}
    height={size}
    viewBox="0 0 24 24"
    fill="none"
    className={className}
    {...props}
  >
    <path d="M3 9l9-7 9 7v11a2 2 0 01-2 2H5a2 2 0 01-2-2z" stroke={color} />
  </svg>
);

// Usage in components
<HomeIcon size={32} color="#blue-500" className="hover:opacity-80" />
```

This interface ensures all icon components have consistent APIs while maintaining full SVG element compatibility and allowing custom sizing through the `size` prop instead of separate `width`/`height` properties.
