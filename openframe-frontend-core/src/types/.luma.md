A comprehensive TypeScript type definition file for managing Luma events across admin interfaces, providing data models and component props for event management systems.

## Key Components

**Core Types:**
- `LumaEvent` - Main database model interface with event details, location, timing, and metadata
- `EventStatus` - Status enumeration (upcoming, past, live, cancelled, all)
- `EventFilters` - Filter parameters for event queries with pagination support

**API Response Types:**
- `EventsResponse` - Paginated event list response with metadata
- `EventStatsResponse` - Event statistics summary

**Component Props:**
- `EventCardProps` - Props for individual event card components
- `EventsDashboardProps` - Props for event dashboard container

**Helper Types:**
- `EventTimeInfo` - Processed time display information with status flags
- `EventLocationInfo` - Location details for online/in-person events
- `LumaEventHost/Guest` - User participant models

## Usage Example

```typescript
import { LumaEvent, EventFilters, EventsResponse } from './luma';

// Filter upcoming events by platform
const filters: EventFilters = {
  platform: 'luma',
  status: 'upcoming',
  search: 'workshop',
  pageSize: 20
};

// Handle event data
function displayEvent(event: LumaEvent) {
  console.log(`${event.name} - ${event.start_at}`);
  if (event.meeting_url) {
    console.log(`Join: ${event.meeting_url}`);
  }
}

// Process API response
function handleEventsResponse(response: EventsResponse) {
  response.events.forEach(displayEvent);
  console.log(`Page ${response.pagination.currentPage} of ${response.pagination.totalPages}`);
}
```
