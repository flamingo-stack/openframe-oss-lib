Provides type definitions and interfaces for managing vendor links in a normalized database structure, supporting both legacy and modern link management systems.

## Key Components

**Types:**
- `VendorLinkType` - Union type defining supported link categories (website, github_repo, docs, social media, etc.)
- `VendorLinksGrouped` - Groups vendor links by their type for organized display

**Interfaces:**
- `VendorLink` - Complete database record with timestamps and metadata
- `VendorLinkInput` - Simplified interface for creating/updating links
- `LinkDisplayConfig` - UI configuration for rendering links with icons and colors
- `VendorWithLinks` - Comprehensive vendor entity supporting both legacy columns and normalized links
- `VendorLinkFallback` - Utility interface for handling legacy-to-modern system transitions

## Usage Example

```typescript
// Creating a new vendor link
const newLink: VendorLinkInput = {
  vendor_id: 123,
  link_type: 'github_repo',
  link_url: 'https://github.com/vendor/repo',
  link_title: 'Main Repository',
  is_primary: true,
  display_order: 1
};

// Working with grouped links
const groupedLinks: VendorLinksGrouped = {
  website: [websiteLink],
  github_repo: [repoLink],
  docs: [docsLink]
};

// Handling legacy fallback
const fallbackLink: VendorLinkFallback = {
  url: vendor.website || '',
  type: 'website',
  source: 'legacy'
};
```
