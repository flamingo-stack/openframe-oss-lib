This file defines the core authentication types and interfaces for an application with SSO support and role-based access control.

## Key Components

**Types:**
- `UserRole` - Defines user permission levels (`'user' | 'super_admin'`)
- `AuthStatus` - Tracks authentication state (`'loading' | 'authenticated' | 'unauthenticated'`)
- `SSOProvider` - Supported SSO providers (`'google' | 'slack' | 'microsoft'`)

**Interfaces:**
- `AuthUser` - User profile data including ID, email, role, and avatar
- `AuthContextType` - React context interface for authentication state and methods
- `JWTClaims` - JWT token payload structure with standard and custom claims

## Usage Example

```typescript
import { AuthUser, AuthContextType, SSOProvider } from './auth'

// Type a user object
const user: AuthUser = {
  id: '123',
  email: 'user@example.com',
  name: 'John Doe',
  role: 'user',
  provider: 'google'
}

// Use in React context
const authContext: AuthContextType = {
  user,
  status: 'authenticated',
  isLoading: false,
  isSuperAdmin: user.role === 'super_admin',
  signInWithSSO: async (provider: SSOProvider) => { /* implementation */ },
  signOut: async () => { /* implementation */ }
}
```
