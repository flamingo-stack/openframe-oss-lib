Defines TypeScript interfaces for employee data structures used across the Flamingo platform for managing and displaying employee information.

## Key Components

| Interface | Description |
|-----------|-------------|
| `Employee` | Full employee record including identity, role, and optional location |
| `EmployeeResponse` | API response wrapper containing a success flag and array of `Employee` records |
| `EmployeeProfileData` | Slim profile subset used for display contexts (excludes `id`, `about`, `location_country`) |

## Usage Example

```typescript
import { Employee, EmployeeResponse, EmployeeProfileData } from './employee';

// Typing an API response
const response: EmployeeResponse = await fetchEmployees();
if (response.success) {
  const employees: Employee[] = response.data;
}

// Using the profile subset for UI display
const profile: EmployeeProfileData = {
  full_name: 'Jane Doe',
  department: 'Engineering',
  avatar_url: 'https://cdn.example.com/avatars/jane.png',
  role: 'admin',
  job_title: 'Senior Engineer',
};
```

## Notes

- `location_country` is the only optional field on `Employee` — all other fields are required.
- `EmployeeProfileData` is a strict subset of `Employee`, suitable for profile cards or directory listings where sensitive/verbose fields (`about`, `id`) are not needed.
