This file defines unified platform types and interfaces used across the application for managing different platforms like OpenMSP, TMCG, and Flamingo. It provides consistent type definitions for database records, API configurations, and UI components.

## Key Components

### Core Types
- **`PlatformName`** - Union type defining all supported platform identifiers
- **`PlatformFilter`** - Platform names plus 'all' option for filtering
- **`LegacyPlatform`** - Backward compatibility type for older platform names

### Database Interfaces
- **`PlatformRecord`** - Complete database model with metadata and timestamps
- **`PlatformStats`** - Platform statistics with content counts

### UI/API Interfaces
- **`PlatformConfig`** - Full platform configuration with display properties and defaults
- **`PlatformOption`** - Simplified key-value pairs for dropdowns and filters

## Usage Example

```typescript
import { PlatformName, PlatformConfig, PlatformFilter } from './platform';

// Type-safe platform selection
const activePlatform: PlatformName = 'flamingo';

// Filter configuration

// Platform dropdown configuration

  value: 'flamingo',
  label: 'Flamingo Documentation',
  name: 'flamingo',
  display_name: 'Flamingo Documentation',
  default_color: '#3B82F6',
  default_icon: 'flame',
  description: 'Modern documentation platform'
};

// Database query result
function getPlatformStats(platform: PlatformName): Promise<PlatformStats> {
  // Implementation would fetch from database
  return Promise.resolve({
    platform,
    display_name: 'Platform Name',
    announcement_count: 5,

    is_active: true
  });
}
```
