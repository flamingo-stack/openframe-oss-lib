Centralized type definition hub that provides a single import point for all TypeScript types used across the Flamingo UI Kit ecosystem.

## Key Components

**Platform Types**
- `PlatformName`, `PlatformRecord`, `PlatformSettings` - Core platform configuration and identification
- `PlatformOption`, `PlatformFilter`, `PlatformStats` - Platform filtering and analytics types

**Content & Media Types**
- `Announcement`, `Category`, `CategoryCardProps` - UI component prop definitions
- Blog, case study, product release, and media content types
- Luma event integration and vendor management types

**Authentication & User Types**
- Authentication flow and user session types
- `ProfileData`, `ProfileResponse` - User profile management
- Team and employee organizational types

**Database & API Types**
- Supabase database schema types
- API request/response interfaces

**Business Logic Types**
- Stack analysis and reporting types
- FAQ, waitlist, and communication (Slack) types
- Navigation, tool, shell, and OS integration types

## Usage Example

```typescript
import { 
  PlatformName, 
  CategoryCardProps, 
  ProfileData,
  AnnouncementFormData 
} from '@flamingo/ui-kit'

// Type-safe platform handling
const platform: PlatformName = 'web'

// Component props with full type safety
const categoryProps: CategoryCardProps = {
  title: 'Development Tools',
  description: 'Essential dev tools',
  platforms: ['web', 'mobile']
}

// User profile management
const profile: ProfileData = {
  id: 'user-123',
  username: 'developer',
  email: 'dev@example.com'
}
```
