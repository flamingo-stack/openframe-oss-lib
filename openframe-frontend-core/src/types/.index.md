Centralized type definition index that provides a single import point for all TypeScript types used across the Flamingo UI Kit platform, including core platform, authentication, content, user management, and business logic types.

## Key Components

**Core Platform Types**
- `announcement` - System announcement and notification types
- `categories`, `category` - Content categorization and filtering types
- `media` - Media asset and file handling types
- `platform` - Platform configuration, stats, and filtering types including `LegacyPlatform`, `PlatformFilter`, `PlatformName`, `PlatformOption`, `PlatformRecord`, `PlatformSettings`, and `PlatformStats`

**Data & Authentication**
- `supabase` - Database schema and query types
- `auth` - Authentication and authorization types
- `profile`, `user`, `employee`, `team` - User management and organizational types

**Content Management**
- `blog`, `case-study`, `customer-interview`, `product-release` - Content type definitions
- `vendor`, `vendor-links` - Vendor and integration types
- `faq`, `report`, `stack` - Documentation and reporting types

**System Integration**
- `slack`, `luma` - Third-party service integration types
- `tool.types`, `shell.types`, `os.types` - System and tooling types
- `navigation` - UI navigation and routing types
- `marketing`, `waitlist` - Marketing and lead generation types

## Usage Example

```typescript
import { 
  PlatformName, 
  PlatformStats, 
  CategoryCardProps,
  UserProfile,
  BlogPost 
} from '@flamingo/ui-kit/types'

// Use platform types
const platformConfig: PlatformName = 'openframe'
const stats: PlatformStats = { users: 1000, tools: 50 }

// Use content types
const category: CategoryCardProps = {
  name: 'MSP Tools',
  description: 'Essential MSP platform tools'
}

// Use combined types across modules
interface DashboardProps {
  platform: PlatformName
  user: UserProfile
  featuredContent: BlogPost[]
}
```
