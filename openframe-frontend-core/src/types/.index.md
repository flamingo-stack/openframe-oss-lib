Centralized type definition export hub that provides unified access to all TypeScript types across the @flamingo/ui-kit package, eliminating the need for multiple individual imports throughout the application.

## Key Components

- **Platform Types**: Core platform configurations, names, records, and statistics
- **Content Types**: Blog posts, case studies, product releases, announcements, and categories
- **Authentication Types**: User authentication, profiles, teams, and employee data
- **Database Types**: Supabase integration and API response types
- **Communication Types**: Slack integration, waitlist management, and marketing tools
- **System Types**: Navigation, tools, shell commands, and OS-specific definitions
- **Business Logic Types**: Stack configurations, reports, FAQs, and vendor management

## Usage Example

```typescript
import { 
  PlatformName, 
  Announcement, 
  ProfileData, 
  Category,
  ToolType 
} from '@flamingo/ui-kit'

// Use platform types
const platform: PlatformName = 'github'

// Use content types
const announcement: Announcement = {
  id: '123',
  title: 'New Feature',
  content: 'Description...'
}

// Use profile types
const profile: ProfileData = {
  id: 'user-123',
  name: 'John Doe',
  email: 'john@example.com'
}
```

This barrel export pattern simplifies imports and provides a clean API surface for consuming applications while maintaining type safety across the entire Flamingo ecosystem.
