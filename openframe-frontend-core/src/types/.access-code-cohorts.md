This file defines TypeScript interfaces for managing access code cohorts, a system for organizing users into groups with time-limited access codes for registration or participation.

## Key Components

**Core Interfaces:**
- `AccessCodeCohort` - Main cohort entity with launch date and capacity
- `CreateAccessCodeCohort` / `UpdateAccessCodeCohort` - Data transfer objects for cohort operations
- `AccessCode` - Individual access codes linked to cohorts and waitlist entries
- `CohortMembership` - User membership tracking with status progression
- `CohortWithStats` - Extended cohort view with usage analytics

**Validation & Response Types:**
- `AccessCodeValidation` / `AccessCodeValidationResponse` - Code verification workflow
- `AccessCodeConsumptionResponse` - Code usage tracking

## Usage Example

```typescript
// Creating a new cohort
const newCohort: CreateAccessCodeCohort = {
  name: "Beta Program Q1",
  description: "Early access to new features",
  launch_date: "2024-03-01",
  capacity: 100
};

// Validating an access code
const validation: AccessCodeValidation = {
  email: "user@example.com",
  code: "BETA2024"
};

// Working with cohort statistics
const cohortStats: CohortWithStats = {
  id: "cohort-123",
  name: "Beta Program",
  launch_date: "2024-03-01",
  member_count: 45,
  codes_generated: 100,
  codes_used: 45,
  created_at: "2024-02-01T00:00:00Z",
  updated_at: "2024-02-15T00:00:00Z"
};
```
