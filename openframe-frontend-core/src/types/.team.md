Defines TypeScript interfaces for team roster data structures, supporting organized display of team members grouped by department sections.

## Key Components

| Interface | Description |
|-----------|-------------|
| `TeamMember` | Full profile of an individual employee including personal info, employment details, skills, and social links |
| `TeamSection` | Groups team members by department with a title and description |
| `TeamData` | Top-level container holding an optional founder and an array of department sections |
| `GetTeamOptions` | Query options for filtering team data by active status or department |

## Usage Example

```typescript
import type { TeamData, TeamMember, GetTeamOptions } from './team';

const options: GetTeamOptions = {
  includeInactive: false,
  department: 'Engineering',
};

const member: TeamMember = {
  id: 'usr_01',
  employee_id: 'EMP-001',
  first_name: 'Jane',
  last_name: 'Doe',
  employment_status: 'active',
  is_public: true,
  is_manager: true,
  created_at: '2024-01-01T00:00:00Z',
  updated_at: '2024-06-01T00:00:00Z',
  skills: ['TypeScript', 'Node.js'],
  social_links: { linkedin: 'https://linkedin.com/in/janedoe' },
};

const teamData: TeamData = {
  founder: member,
  sections: [
    {
      title: 'Engineering',
      department: 'engineering',
      description: 'Builds and maintains the platform',
      members: [member],
    },
  ],
};
```

## Notes

- `full_name` is optional and may be derived from `first_name` + `last_name`
- `work_email` and `email` are both optional to support personal vs. corporate email separation
- `social_links` uses a flexible `Record<string, string>` map to accommodate any platform
