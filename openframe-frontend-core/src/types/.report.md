This file defines TypeScript interfaces for margin analysis reports and cost comparisons in a vendor analysis system. It centralizes type definitions used by margin analysis wizards and data access layer utilities.

## Key Components

- **`MarginReportCostStructure`** - Financial breakdown with current costs, alternative costs, and savings
- **`MarginReportReplacementEntry`** - Individual vendor replacement recommendation with current/alternative vendors and cost structure
- **`MarginReport`** - Complete margin analysis report containing cost structures, recommendations, and metadata
- **`CreateMarginReportPayload`** - Request payload for generating new margin reports with stack configuration

## Usage Example

```typescript
// Creating a margin report payload
const reportPayload: CreateMarginReportPayload = {
  mspProfile: { id: "msp-123", name: "TechServices Inc" },
  stack: userStack,
  currentStack: { "security": "vendor-a", "backup": "vendor-b" },
  manualAltMap: { "security": "vendor-c" },
  aiAltMap: { "backup": "vendor-d" },
  stackVendorsToSlugs: [
    { category: "security", vendorSlug: "vendor-a" }
  ]
};

// Accessing report data
const handleReport = (report: MarginReport) => {
  const totalSavings = report.cost_structure.total_savings;
  const replacements = report.replacements.map(r => ({
    current: r.current_vendor,
    alternative: r.alternative_vendor,
    savings: r.cost_structure.total_savings
  }));
};
```
