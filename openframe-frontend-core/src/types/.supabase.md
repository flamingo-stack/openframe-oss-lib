TypeScript type definitions for a Supabase database schema, providing comprehensive type safety for database operations across multiple data models including announcements, Slack integration, and file assets.

## Key Components

- **`Json`** - Recursive JSON type definition for flexible data structures
- **`Database`** - Complete database schema with tables for announcements, Slack data, and assets
- **`Tables<T>`** - Generic utility type for extracting row types from any table
- **`TablesInsert<T>`** - Generic utility type for insert operations with optional fields
- **`TablesUpdate<T>`** - Generic utility type for update operations with all optional fields

### Database Tables

- **`announcements`** - Platform announcements with styling and metadata
- **`announcement_assets`** - File attachments linked to announcements
- **`slack_channels`** - Slack channel information and metadata
- **`slack_messages`** - Slack message data with threading support
- **`slack_users`** - Slack user profiles and permissions

## Usage Example

```typescript
import { Database, Tables, TablesInsert } from './supabase'
import { createClient } from '@supabase/supabase-js'

const supabase = createClient<Database>(url, key)

// Type-safe row access
type Announcement = Tables<'announcements'>
type SlackMessage = Tables<'slack_messages'>

// Type-safe insert operations
const newAnnouncement: TablesInsert<'announcements'> = {
  title: 'System Maintenance',
  description: 'Scheduled downtime tonight',
  background_color: '#ff6b6b'
}

// Type-safe database operations
const { data, error } = await supabase
  .from('announcements')
  .insert(newAnnouncement)
  .select()
```
