Defines TypeScript interfaces and types for vendor management in a marketplace or directory system, including media, pricing, GitHub metrics, and feature comparison capabilities.

## Key Components

**Core Interfaces:**
- `Vendor` - Main vendor entity with metadata, relations, and optional AI selection tracking
- `VendorMedia` - Media assets (logo, image, video) with URLs
- `VendorPricing` - Pricing model information
- `VendorGithub` - GitHub repository metrics (stars, forks, commits, license)
- `VendorClassification` - Vendor categorization (open source, commercial, selected)

**Feature Comparison Types:**
- `SubcategoryFeature` - Feature definitions with importance levels
- `VendorFeatureValue` - Boolean feature values with display formatting
- `EnhancedVendorFeature` - Feature data enriched for UI display
- `RelatedVendorFeature` - Cross-vendor feature comparison data

**Utility Types:**
- `VendorSearchResult` - Lightweight vendor data for search results
- `VendorProsCons` - Structured pros and cons lists
- `CategoryWithFeatures` - Hierarchical feature organization

## Usage Example

```typescript
// Creating a vendor with media and GitHub data
const vendor: Vendor = {
  id: 1,
  title: "Example Vendor",
  slug: "example-vendor",
  description: "AI-powered analytics platform",
  vendor_media: [
    { media_type: "logo", media_url: "/logos/example.png" }
  ],
  vendor_github: {
    github_stars: 1500,
    github_forks: 200,
    github_license: "MIT"
  },
  selection_source: "ai"
};

// Feature comparison
const feature: EnhancedVendorFeature = {
  id: "real-time-analytics",
  name: "Real-time Analytics",
  importance: "Critical",
  feature_value: true,
  display_value: "Yes, sub-second latency"
};
```
