A TypeScript interface definition for waitlist entry management with HubSpot integration and cohort tracking capabilities. Defines the data structures for collecting user signups with comprehensive metadata tracking and CRM synchronization.

## Key Components

- **`WaitlistEntry`** - Complete waitlist entry interface with all fields including HubSpot integration metadata, platform relationships, and cohort membership data
- **`CreateWaitlistEntry`** - Input interface for creating new waitlist entries with required and optional user data plus tracking fields
- **`WaitlistResponse`** - API response wrapper interface with success/error handling and optional data payload

## Usage Example

```typescript
// Creating a new waitlist entry
const newEntry: CreateWaitlistEntry = {
  email: "user@example.com",
  name: "John Doe",
  company: "Acme Corp",
  platform_id: "platform-123",
  referral_source: "google",
  utm_source: "newsletter",
  utm_campaign: "launch2024",
  ip_address: "192.168.1.1",
  user_agent: "Mozilla/5.0..."
};

// API response structure
const response: WaitlistResponse = {
  success: true,
  data: {
    id: "entry-456",
    email: "user@example.com",
    platform_id: "platform-123",
    created_at: "2024-01-15T10:00:00Z",
    updated_at: "2024-01-15T10:00:00Z",
    hubspot_sync_status: "success",
    cohort: {
      cohort_id: "cohort-789",
      cohort_name: "Beta Launch",
      access_code: "BETA2024",
      access_code_used: false,
      membership_status: "active",
      // ...other cohort fields
    }
  }
};
```
