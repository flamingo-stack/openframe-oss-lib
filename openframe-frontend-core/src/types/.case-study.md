TypeScript type definitions for a case study management system that integrates with MSP (Managed Service Provider) profiles and platform relationships.

## Key Components

- **`CaseStudy`** - Main interface defining case study structure with story sections (challenge, solution, results), SEO metadata, publishing status, and relational data to users, MSPs, platforms, and tags
- **`CreateCaseStudyData`** - Input interface for creating new case studies with required fields and platform/tag associations
- **`UpdateCaseStudyData`** - Partial update interface extending CreateCaseStudyData
- **`CaseStudyFilters`** - Query parameters for filtering case studies by platform, tags, industry, company size, search terms, and pagination
- **`CaseStudyListResponse`** - API response wrapper containing filtered case studies and total count

## Usage Example

```typescript
// Creating a new case study
const newCaseStudy: CreateCaseStudyData = {
  title: "Cloud Migration Success",
  slug: "cloud-migration-success",
  summary: "How we migrated 500+ users to Microsoft 365",
  challenge: "Legacy infrastructure limitations",
  solution: "Phased migration approach",
  results: "99.9% uptime achieved",
  status: "published",
  author_id: "author-uuid",
  platforms: ["microsoft-365", "azure"],
  tags: [1, 5, 12]
}

// Filtering case studies
const filters: CaseStudyFilters = {
  platform: "microsoft-365",
  industry: "healthcare",
  company_size: "medium",
  featured: true,
  limit: 10
}
```
