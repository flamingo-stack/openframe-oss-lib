This TypeScript file defines the type structure for a case study system that showcases MSP (Managed Service Provider) success stories and customer testimonials.

## Key Components

- **`CaseStudy`** - Main interface defining the complete case study structure with story sections (challenge/solution/results), SEO fields, and relationships to users, MSPs, platforms, and tags
- **`CreateCaseStudyData`** - Input interface for creating new case studies with required fields and platform/tag associations
- **`UpdateCaseStudyData`** - Partial type for updating existing case studies
- **`CaseStudyFilters`** - Query parameters for filtering and searching case studies by platform, industry, company size, etc.
- **`CaseStudyListResponse`** - API response wrapper containing paginated case study data

## Usage Example

```typescript
// Creating a new case study
const newCaseStudy: CreateCaseStudyData = {
  title: "How TechCorp Reduced Downtime by 90%",
  slug: "techcorp-reduced-downtime",
  summary: "A comprehensive case study on infrastructure optimization",
  user_id: "uuid-of-msp-user",
  challenge: "Frequent system outages affecting productivity",
  solution: "Implemented redundant systems and monitoring",
  results: "90% reduction in downtime, $50k annual savings",
  status: "published",
  author_id: "author-uuid",
  platforms: ["platform-uuid-1", "platform-uuid-2"],
  tags: [1, 3, 7]
}

// Filtering case studies
const filters: CaseStudyFilters = {
  platform: "microsoft-365",
  industry: "healthcare",
  company_size: "50-200",
  featured: true,
  limit: 10
}
```
