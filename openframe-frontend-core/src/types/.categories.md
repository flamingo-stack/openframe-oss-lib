Defines TypeScript interfaces for category data structures used throughout the application, supporting both legacy and current database schemas with statistical information.

## Key Components

- **`Category`** - Main category interface with basic properties (id, title, description, counts, icons, slug)
- **`CategoryCardProps`** - Legacy interface for backward compatibility with existing category card components
- **`RealCategory`** - Database-aligned interface matching the actual data structure with subcategories
- **`RealCategoryCardProps`** - Props interface for components using the real category structure
- **`CategoryWithStats`** - Extended category interface that includes computed statistics (vendor and subcategory counts)

## Usage Example

```typescript
// Using the main Category interface
const category: Category = {
  id: "electronics",
  title: "Electronics",
  description: "Electronic devices and gadgets",
  subcategoriesCount: 5,
  productsCount: 150,
  iconUrls: ["icon1.png", "icon2.png"],
  slug: "electronics"
};

// Using RealCategory with database data
const dbCategory: RealCategory = {
  id: 1,
  name: "Electronics",
  slug: "electronics", 
  description: "Electronic devices",
  vendor_subcategory: [
    { id: 1, name: "Smartphones", slug: "smartphones" }
  ]
};

// Enhanced category with statistics
const categoryWithStats: CategoryWithStats = {
  ...dbCategory,
  vendorCount: 25,
  subcategoryCount: 8
};
```
