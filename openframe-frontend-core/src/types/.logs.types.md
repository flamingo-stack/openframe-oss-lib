TypeScript type definitions and interfaces for a logging system, providing structured data models for log entries, severity levels, and UI component props.

## Key Components

- **`LogSeverity`** - Union type defining log levels (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- **`LogEntry`** - Core interface for log data with fields like id, severity, timestamp, and metadata
- **`LogsListProps`** - Props interface for rendering a list of logs with customization options
- **`LogSeverityDotProps`** - Props for visual severity indicators with size variants
- **`ToolIconProps`** - Props for tool-specific icons with configurable sizing
- **`LOG_SEVERITY_COLORS`** - Color mapping for each severity level
- **`LOG_SEVERITY_LABELS`** - Human-readable labels for severity levels

## Usage Example

```typescript
import type { LogEntry, LogSeverity, LogsListProps } from './logs.types'

// Creating a log entry
const logEntry: LogEntry = {
  id: 'log-123',
  ingestDay: '2024-01-15',
  eventType: 'deployment',
  severity: 'ERROR',
  title: 'Build Failed',
  timestamp: new Date(),
  toolType: 'github',
  message: 'Compilation error in main.ts'
}

// Using in a component
const LogsList: React.FC<LogsListProps> = ({ logs, onLogClick }) => {
  return (
    <div>
      {logs.map(log => (
        <div 
          key={log.id} 
          onClick={() => onLogClick?.(log)}
          style={{ color: LOG_SEVERITY_COLORS[log.severity] }}
        >
          {LOG_SEVERITY_LABELS[log.severity]}: {log.title}
        </div>
      ))}
    </div>
  )
}
```
