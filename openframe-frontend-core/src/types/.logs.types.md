TypeScript definitions for the logging system, including severity levels, log entry structures, and UI component props for displaying logs in the Flamingo platform.

## Key Components

- **LogSeverity**: Union type defining log levels (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- **LogEntry**: Main interface for log entries with metadata, timestamps, and tool associations
- **LogsListProps**: Props interface for the logs list component with display options
- **LogSeverityDotProps**: Props for severity indicator components
- **LOG_SEVERITY_COLORS**: Color mapping for each severity level in the UI
- **LOG_SEVERITY_LABELS**: Human-readable labels for severity levels

## Usage Example

```typescript
import type { LogEntry, LogSeverity, LogsListProps } from './logs.types'

// Create a log entry
const logEntry: LogEntry = {
  id: 'log-001',
  ingestDay: '2024-01-15',
  eventType: 'AUTH_FAILURE',
  severity: 'ERROR',
  title: 'Authentication Failed',
  timestamp: new Date(),
  toolType: 'SECURITY',
  toolIcon: 'shield',
  message: 'Invalid credentials provided',
  metadata: { userId: '12345', ip: '192.168.1.100' }
}

// Configure logs list component
const logsProps: LogsListProps = {
  logs: [logEntry],
  maxHeight: '400px',
  showConnector: true,
  onLogClick: (log) => console.log('Selected log:', log.id),
  loading: false,
  emptyMessage: 'No logs available'
}
```

The severity colors provide consistent visual indicators across the platform, with gray for debug/info, yellow for warnings, and red tones for errors and critical issues.
