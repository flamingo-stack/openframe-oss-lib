Type definitions for the logging system, providing interfaces for log entries, UI components, and severity level configuration.

## Key Components

**Types & Interfaces:**
- `LogSeverity` - Union type for log severity levels (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- `LogEntry` - Core interface defining log structure with id, timestamp, severity, and optional metadata
- `LogsListProps` - Props interface for log list component with customization options
- `LogSeverityDotProps` - Props for severity indicator components
- `ToolIconProps` - Props for tool icon components

**Constants:**
- `LOG_SEVERITY_COLORS` - Color mapping for each severity level
- `LOG_SEVERITY_LABELS` - Human-readable labels for severity levels

## Usage Example

```typescript
import type { LogEntry, LogSeverity } from './logs.types'

// Create a log entry
const logEntry: LogEntry = {
  id: 'log-123',
  ingestDay: '2024-01-15',
  eventType: 'api_call',
  severity: 'ERROR',
  title: 'API Request Failed',
  timestamp: new Date(),
  toolType: 'webhook',
  message: 'Failed to connect to external service'
}

// Use severity colors
const errorColor = LOG_SEVERITY_COLORS['ERROR'] // '#f36666'

// Implement logs list component
const LogsList: React.FC<LogsListProps> = ({ logs, onLogClick }) => {
  return (
    <div>
      {logs.map(log => (
        <div key={log.id} onClick={() => onLogClick?.(log)}>
          {log.title}
        </div>
      ))}
    </div>
  )
}
```
