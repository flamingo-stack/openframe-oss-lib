Unified content-href authority for all embeddable surfaces — resolves content links consistently across page views and chat surfaces using a single configurable seam.

## Key Components

**`DEFAULT_CONTENT_SUFFIXES`** — Maps content `documentType` keys (e.g. `product_release`) to URL path segments (e.g. `releases`). Embedder default; kept manually in sync with the hub's `PUBLIC_URL_PATHS`.

**`ComposeContentUrlInput`** — Shared input shape for both callers: page views (pass `type` + slug as `identifier`) and chat rows (pass `type` + primary-key `identifier` + `externalUrl` for slug recovery).

**`ContentHrefOptions`** — Config for `makeComposeContentUrl`: `hostedTypes` (in-app soft-nav set), `contentOrigin` (fallback hub origin), optional `suffixes` override, and per-type `overrides`.

**`makeComposeContentUrl(opts)`** — Builds the embedder's `composeContentUrl` function. Resolution order:
1. `overrides[type]` — explicit per-type href
2. `hostedTypes` member → relative `/<suffix>/<slug>` (soft-nav)
3. `externalUrl` present → use verbatim (RAG-authoritative)
4. Fallback → `${contentOrigin}/<suffix>/<identifier>`

**`buildDefaultHref(basePath, slug)`** — Single-platform fallback when no composer is wired.

**`resolveContentHref(composeContentUrl, args)`** — Centralizes the composer-vs-fallback ternary used by every catalog/detail view.

## Usage Example

```typescript
// Wire the embedder's composer (e.g. in ChatRuntime setup)
const composeContentUrl = makeComposeContentUrl({
  hostedTypes: new Set(['onboarding_guide', 'product_release']),
  contentOrigin: 'https://openframe.app',
  overrides: {
    webinar: (id) => ({ href: `/live/${id}`, targetPlatform: null }),
  },
})

// Page view — slug passed directly
composeContentUrl({ type: 'product_release', identifier: 'my-release' })
// → { href: '/releases/my-release', targetPlatform: null }

// Chat row — slug recovered from hub URL
composeContentUrl({
  type: 'product_release',
  identifier: '42',
  externalUrl: 'https://openframe.app/releases/my-release',
})
// → { href: '/releases/my-release', targetPlatform: null }

// Non-hosted type — opens out to hub
composeContentUrl({ type: 'blog_post', identifier: 'some-post' })
// → { href: 'https://openframe.app/blog/some-post', targetPlatform: null }

// In a catalog/detail view — handles missing composer gracefully
resolveContentHref(composeContentUrl, {
  type: 'onboarding_guide',
  slug: 'getting-started',
  basePath: '/onboarding-guides',
})
```
