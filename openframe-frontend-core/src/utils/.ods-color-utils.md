Provides runtime utilities for working with ODS (Open Design System) color tokens across multiple platforms. This module handles platform detection, theme switching, and color token management for dynamic theming in web applications.

## Key Components

**Core Functions:**
- `getODSToken()` - Retrieves raw ODS token values by category, color, and variant
- `getPlatformAccentColor()` - Gets CSS custom property for platform-specific accent colors
- `getCurrentPlatform()` - Detects current platform from environment or DOM attributes
- `switchPlatformTheme()` - Dynamically changes platform theme via CSS custom properties

**Utility Functions:**
- `tokenToTailwindClass()` - Converts ODS tokens to Tailwind CSS classes
- `getTokensByCategory()` - Retrieves all tokens for a specific color category
- `generatePlatformCSS()` - Creates CSS custom property declarations for platforms
- `interpolateColors()` - Blends between two ODS color tokens with progress value

**React Integration:**
- `usePlatformColors()` - Hook providing platform-aware color utilities and theme controls

## Usage Example

```typescript
import { 
  getCurrentPlatform, 
  getPlatformAccentColor, 
  getODSToken,
  switchPlatformTheme,
  usePlatformColors 
} from './ods-color-utils';

// Get platform-specific accent color
const platform = getCurrentPlatform(); // 'openmsp' | 'admin-hub' | 'openframe' | 'flamingo'
const accentColor = getPlatformAccentColor(platform); // 'var(--ods-open-yellow-base)'

// Get specific ODS token
const primaryButton = getODSToken('open', 'blue', 'hover');

// Switch theme dynamically
switchPlatformTheme('flamingo');

// React component usage
function ThemeAwareComponent() {
  const { accentColor, isDarkTheme, switchTheme } = usePlatformColors();
  
  return (
    <div style={{ backgroundColor: accentColor }}>
      <button onClick={() => switchTheme('openframe')}>
        Switch to OpenFrame Theme
      </button>
    </div>
  );
}
```
