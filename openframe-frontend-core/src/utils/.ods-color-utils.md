Runtime utility library for working with ODS (OpenFrame Design System) color tokens across the `openmsp`, `openframe`, and `flamingo` platforms.

## Key Components

### Types
- `Platform` — `'openmsp' | 'openframe' | 'flamingo'`
- `ColorCategory` — `'open' | 'flamingo' | 'system' | 'attention'`
- `ColorVariant` — `'base' | 'hover' | 'active' | 'focus' | 'disabled'`

### Token Lookup
- `getODSToken(category, color, variant?)` — Returns a raw token value by key
- `getTokensByCategory(category)` — Returns all tokens for a given category
- `isValidODSToken(tokenKey)` — Validates token existence
- `tokenToTailwindClass(tokenKey, type?)` — Maps a token key to a Tailwind CSS class

### Platform Theming
- `getCurrentPlatform()` — Reads platform from DOM attribute or `NEXT_PUBLIC_APP_TYPE` env var
- `getPlatformAccentColor(platform?)` — Returns the CSS variable for the platform's accent color
- `getPlatformConfig(platform?)` — Returns accent color, theme mode, and brand name for a platform
- `switchPlatformTheme(platform)` — Updates `data-app-type` on `<html>` and dispatches `platformThemeChanged`
- `generatePlatformCSS(platform)` — Generates scoped CSS custom property declarations
- `getSemanticColor(semanticName, platform?)` — Reads a computed `--color-*` CSS variable at runtime

### Color Math
- `hexToRgb(hex)` / `rgbToHex(r, g, b)` — Hex ↔ RGB conversion
- `rgbToHsl(r, g, b)` / `hslToRgb(h, s, l)` — RGB ↔ HSL conversion
- `deriveHoverColor(hex)` / `deriveActiveColor(hex)` — Darken a hex color by a fixed step (10/20) for interaction states
- `interpolateColors(startToken, endToken, progress)` — Linear RGB interpolation between two tokens
- `getReadableTextColor(hex)` — Returns `#212121` or `#fafafa` for contrast using Rec. 601 luminance
- `applyColorToken(element, property, tokenKey)` — Sets a CSS custom property directly on a DOM element

### React Integration
- `usePlatformColors(platform?)` — Returns platform config and helper accessors for use in React components

## Usage Example

```typescript
import {
  getCurrentPlatform,
  getPlatformConfig,
  deriveHoverColor,
  getReadableTextColor,
  usePlatformColors,
} from './ods-color-utils';

// Detect current platform
const platform = getCurrentPlatform(); // 'openframe'

// Get full platform config
const config = getPlatformConfig();
// { platform: 'openframe', accentColor: 'var(--ods-open-yellow-base)', isDarkTheme: true, ... }

// Derive interaction states from a custom hex color
const base = '#3b82f6';
const hover = deriveHoverColor(base);   // '#315cf6' (darkened by 10)
const active = deriveActiveColor(base); // '#2752f6' (darkened by 20)

// Pick a readable text color for a background
const textColor = getReadableTextColor('#FFC008'); // '#212121'

// In a React component
function MyButton() {
  const { accentColor, isDarkTheme, switchTheme } = usePlatformColors();
  return (
    <button style={{ background: accentColor }}>
      {isDarkTheme ? 'Dark Mode' : 'Light Mode'}
    </button>
  );
}
```

> **Note:** `getSemanticColor` and `applyColorToken` are browser-only; they return `undefined` or no-op on the server side. Platform detection falls back to the `NEXT_PUBLIC_APP_TYPE` environment variable when `window` is unavailable.
