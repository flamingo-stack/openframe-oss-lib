A cancellation-proof scroll utility that animates a target element to the top of the viewport using a self-driven `requestAnimationFrame` tween, bypassing native smooth scroll's vulnerability to browser scroll anchoring and competing programmatic scrolls.

## Key Components

**`scrollElementIntoView(target, options)`** — Main export. Scrolls the page so `target` lands at the top of the viewport. SSR-safe; accepts `null`/`undefined` targets without defensive branching.

**`ScrollElementIntoViewOptions`** — Configuration interface:
| Option | Type | Default | Description |
|---|---|---|---|
| `headerOffset` | `number` | `0` | Pixels subtracted from target `top` to clear sticky chrome |
| `behavior` | `ScrollBehavior` | `'smooth'` | `'instant'`/`'auto'` for synchronous jump; `'smooth'` for tween |
| `adjustTargetY` | `(raw: number) => number` | — | Per-frame target override for known layout shifts |
| `durationMs` | `number` | `320` | Tween duration in ms |

**`cancelActiveScroll()`** — Internal. Cancels any in-flight tween and removes gesture listeners. Only one page-scroll animation runs at a time.

**`easeOutCubic`** — `1 - (1-t)³` easing curve applied to the tween.

## Usage Example

```typescript
import { scrollElementIntoView } from '@/lib/scroll-into-view'

// Basic scroll with hub header offset
scrollElementIntoView(ref.current, { headerOffset: 96 })

// Instant jump (deep-link land / programmatic focus)
scrollElementIntoView(anchorEl, { behavior: 'instant', headerOffset: 96 })

// Compensate for a sibling drawer still collapsing
scrollElementIntoView(rowEl, {
  headerOffset: 96,
  adjustTargetY: (raw) => raw - DRAWER_HEIGHT,
})
```

> **Why not `behavior: 'smooth'`?** Browser scroll anchoring synchronously corrects `scrollTop` when content shifts, which aborts any in-flight native smooth scroll per the CSSOM-View spec. This tween writes instant positions every frame — nothing to abort — and re-computes the target each frame to track elements still settling.
