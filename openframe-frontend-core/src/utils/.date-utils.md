Unified date formatting utilities that provide consistent timestamp handling and display formatting across the OpenMSP platform. This module serves as the single source of truth for all date and time operations.

## Key Components

- `formatRelativeTime()` - Primary relative time formatter (e.g., "2h ago", "3d ago")
- `formatAbsoluteDate()` - Absolute date formatting with customizable options
- `formatDateTime()` - Combined date and time formatting
- `getDetailedTimeDifference()` - Verbose time difference descriptions
- `isToday()` - Check if timestamp is within current day
- `isWithinMinutes()` - Check if timestamp is within specified time window
- `createUTCTimestamp()` - Generate UTC timestamp strings for database storage

## Usage Example

```typescript
import { 
  formatRelativeTime, 
  formatAbsoluteDate, 
  formatDateTime,
  isWithinMinutes 
} from './date-utils';

// Relative time formatting (most common use case)
const postTime = formatRelativeTime('2024-01-01T12:00:00Z');
// Returns: "2h ago", "3d ago", or "Jan 1" for older dates

// Absolute date formatting
const eventDate = formatAbsoluteDate('2024-06-15T10:30:00Z');
// Returns: "Jun 15, 2024"

// Date with time
const fullDateTime = formatDateTime('2024-06-15T10:30:00Z');
// Returns: "Jun 15, 2024 at 10:30 AM"

// Check recent activity
const isRecent = isWithinMinutes('2024-01-01T12:00:00Z', 30);
// Returns: true if within last 30 minutes

// Generate database timestamp
const dbTimestamp = createUTCTimestamp();
// Returns: "2024-01-01T12:00:00.000Z"
```

All functions handle invalid dates gracefully with console warnings and fallback values. The `formatRelativeTime()` function automatically switches to absolute dates for content older than 30 days to maintain readability.
