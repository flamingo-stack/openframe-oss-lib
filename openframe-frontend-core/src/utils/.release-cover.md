Resolves the cover image for `ProductReleaseCard` using a deterministic priority chain, ensuring consistent rendering across all surfaces (catalog, chat inline, chat wrapper).

## Key Components

### `ReleaseCoverSource`
Input interface describing the optional image fields sourced from a release entity.

| Field | Description |
|---|---|
| `featured_image` | Editor-curated cover — highest priority |
| `highlight_video_thumbnail` | AI-generated highlight reel thumbnail |
| `main_video_thumbnail` | Full-release video thumbnail |
| `og_image_url` | SEO/OG fallback image |

### `ReleaseCoverResult`
Return shape from `resolveReleaseCover`.

| Field | Description |
|---|---|
| `cover` | Resolved image URL, or `null` if none available |
| `hasVideoCover` | `true` only when the active cover is a video thumbnail (priority 2 or 3) — used to conditionally render a Play overlay |

### `resolveReleaseCover(r: ReleaseCoverSource): ReleaseCoverResult`
Applies the priority chain using `||` (not `??`) so empty strings fall through rather than rendering a broken `<Image src="">`. Sets `hasVideoCover` to `false` when a `featured_image` is present, suppressing misleading Play overlays on still images.

## Usage Example

```typescript
import { resolveReleaseCover } from './release-cover'

const { cover, hasVideoCover } = resolveReleaseCover({
  featured_image: null,
  highlight_video_thumbnail: 'https://cdn.example.com/highlight.jpg',
  main_video_thumbnail: 'https://cdn.example.com/main.jpg',
  og_image_url: 'https://cdn.example.com/og.jpg',
})

// cover         → 'https://cdn.example.com/highlight.jpg'
// hasVideoCover → true  (no featured_image, video thumb is active)
```

> To add a new fallback source or reorder priority, edit **only this file** — all surfaces update automatically.
