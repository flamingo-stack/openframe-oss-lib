Resolves the cover image for a `ProductReleaseCard` using a deterministic fallback chain, ensuring consistent rendering across all surfaces (catalog row, chat-inline dispatcher, chat wrapper).

## Key Components

**`ReleaseCoverSource`** — Input interface describing the available image fields on a release entity.

**`ReleaseCoverResult`** — Output shape containing the resolved `cover` URL (or `null`) and a `hasVideoCover` flag.

**`resolveReleaseCover(r)`** — Core resolver that applies the priority chain and returns a `ReleaseCoverResult`.

### Cover Priority

| Priority | Field | Notes |
|----------|-------|-------|
| 1 | `featured_image` | Editor-curated, always wins |
| 2 | `highlight_video_thumbnail` | AI-generated highlight reel |
| 3 | `main_video_thumbnail` | Full-release video thumb |
| 4 | `og_image_url` | SEO fallback |
| 5 | `null` | Caller renders entity placeholder |

> `hasVideoCover` is `true` only when priority 2 or 3 wins. A Play overlay is suppressed when a still `featured_image` is active to avoid misleading UI.

## Usage Example

```typescript
import { resolveReleaseCover, ReleaseCoverSource } from './release-cover'

const source: ReleaseCoverSource = {
  featured_image: null,
  highlight_video_thumbnail: 'https://cdn.example.com/thumb.jpg',
  og_image_url: 'https://cdn.example.com/og.jpg',
}

const { cover, hasVideoCover } = resolveReleaseCover(source)
// cover        → 'https://cdn.example.com/thumb.jpg'
// hasVideoCover → true
```

> **Note:** Uses `||` (not `??`) so empty-string DB values fall through to the next fallback, preventing `<Image src="">` renders.
