A centralized configuration module that provides unified branding assets (icons, colors, names, descriptions) for all platforms in the Flamingo ecosystem including OpenFrame, OpenMSP, and various specialized hubs.

## Key Components

- **Platform Asset Maps**: Exports for `platformIcons`, `platformColors`, `platformDisplayNames`, `platformDescriptions`, `platformSlogans`, `platformHexColors`, and `platformIconNames`
- **Utility Functions**: Helper functions `getDefaultColorForPlatform`, `getDefaultIconForPlatform`, `getPlatformIcon`, `getPlatformColor`, etc.
- **Icon Components**: Specialized icon getters `getSmallPlatformIcon` (4x4) and `getPlatformIconComponent` (6x6) with platform-specific styling
- **Data Transformation**: `transformPlatformConfigsToOptions` converts platform configs to selectable options

## Usage Example

```typescript
import { 
  getPlatformIcon, 
  getPlatformColor, 
  getPlatformDisplayName,
  getSmallPlatformIcon 
} from './platform-config';

// Get platform assets
const flamingoIcon = getPlatformIcon('flamingo');
const openFrameColor = getPlatformColor('openframe'); // 'bg-[#8B5CF6]'
const displayName = getPlatformDisplayName('openmsp'); // 'OpenMSP'

// Use in components
function PlatformSelector({ platformName }: { platformName: string }) {
  return (
    <div className={getPlatformColor(platformName)}>
      {getSmallPlatformIcon(platformName)}
      <span>{getPlatformDisplayName(platformName)}</span>
    </div>
  );
}
```

The module ensures consistent branding across the entire platform by centralizing all visual and textual representations of each service in the Flamingo stack.
