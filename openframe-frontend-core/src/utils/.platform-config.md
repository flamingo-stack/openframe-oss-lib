Centralized configuration module for all platform branding assets and metadata across the Flamingo/OpenFrame ecosystem, providing icons, colors, display names, descriptions, and utility functions for consistent platform presentation.

## Key Components

### Lookup Maps (exported constants)

| Export | Purpose |
|--------|---------|
| `platformIcons` | React icon components keyed by platform name |
| `platformColors` | Tailwind `bg-*` class strings per platform |
| `platformDisplayNames` | Human-readable names (e.g., `"Flamingo Marketing Hub"`) |
| `platformDescriptions` | Full marketing copy per platform |
| `platformSlogans` | Short taglines for branding use |
| `platformHexColors` | Raw hex values for non-Tailwind contexts |
| `platformIconNames` | String icon identifiers for config serialization |

Supported platform keys: `openframe`, `openmsp`, `flamingo`, `flamingo-teaser`, `marketing-hub`, `product-hub`, `revenue-hub`, `people-hub`, `company-hub`, `tmcg`, `universal`

### Utility Functions

| Function | Returns | Notes |
|----------|---------|-------|
| `getPlatformIcon(name)` | `React.ReactNode` | Falls back to `universal` Globe icon |
| `getPlatformColor(name)` | `string` | Tailwind `bg-*` class |
| `getPlatformDisplayName(name)` | `string` | Falls back to raw name |
| `getPlatformDescription(name)` | `string` | Falls back to raw name |
| `getPlatformSlogan(name)` | `string` | Falls back to raw name |
| `getDefaultColorForPlatform(name)` | `string` | Hex value, falls back to `universal` |
| `getDefaultIconForPlatform(name)` | `string` | Icon name string, falls back to `universal` |
| `getSmallPlatformIcon(name)` | `React.ReactNode` | 4×4 (`h-4 w-4`), uses CSS variables for hub variants |
| `getPlatformIconComponent(name, className?)` | `React.ReactNode` | 6×6 default, accepts custom `className` |
| `transformPlatformConfigsToOptions(configs)` | `SelectableOption[]` | Maps `PlatformConfig[]` to UI-selectable options with icon and color |

## Usage Example

```typescript
import {
  getPlatformDisplayName,
  getPlatformColor,
  getSmallPlatformIcon,
  transformPlatformConfigsToOptions
} from './platform-config';

// Render a platform badge
const PlatformBadge = ({ name }: { name: string }) => (
  <span className={`px-2 py-1 rounded ${getPlatformColor(name)}`}>
    {getSmallPlatformIcon(name)}
    {getPlatformDisplayName(name)}
  </span>
);

// Populate a platform selector dropdown
const options = transformPlatformConfigsToOptions(platformConfigs);
```

> All lookup functions fall back gracefully to the `universal` platform or the raw input string, so unknown platform keys never produce `undefined` in UI renders.
