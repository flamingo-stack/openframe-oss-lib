Defines the canonical URL query parameter keys shared between the dev-center chrome registry and list views, ensuring consistent read/write contracts for section-level filtering.

## Key Components

- **`DEV_SECTION_PARAM_KEYS`** — A frozen `as const` object mapping logical filter names to their URL query string keys:

| Key | URL Param | Used By |
|-----|-----------|---------|
| `search` | `search` | All dev-center sections |
| `status` | `status` | Roadmap & Help Center tickets |
| `releaseStatus` | `release_status` | Product Releases |
| `deliveryTaskType` | `task_type` | Delivery (bugs / enhancements) |

## Usage Example

```typescript
import { DEV_SECTION_PARAM_KEYS } from '@/lib/dev-section-param-keys'

// Chrome registry writing to the URL
const url = new URL(window.location.href)
url.searchParams.set(DEV_SECTION_PARAM_KEYS.status, 'in-progress')

// List view reading from the URL
const status = searchParams.get(DEV_SECTION_PARAM_KEYS.status) // 'in-progress'

// Type-safe access to all available keys
type ParamKey = typeof DEV_SECTION_PARAM_KEYS[keyof typeof DEV_SECTION_PARAM_KEYS]
// => 'search' | 'status' | 'release_status' | 'task_type'
```

## Notes

- **Zero dependencies** — no React or runtime imports, safe to use in both server-bundled utilities and client-side views.
- Centralizing these literals makes chrome ↔ view key mismatches a compile-time problem rather than a silent runtime filter failure.
