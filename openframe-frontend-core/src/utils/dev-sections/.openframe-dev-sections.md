Canonical metadata registry for all OpenFrame dev-center sections (Roadmap, Bug-fixes & Enhancements, Releases, Onboarding Guides, Help Center). Provides a single source of truth for icon, navigator card copy, hero copy, search config, and filter config consumed by both the homepage navigator grid and destination page heroes.

## Key Components

**Filter Option Constants**
- `ROADMAP_STATUS_OPTIONS` — `all | completed | in_progress` status pills for the Roadmap section
- `DELIVERY_TASK_TYPE_OPTIONS` — `all | Bug | Request` type pills for Bug-fixes & Enhancements
- `TICKET_STATUS_OPTIONS` — `all | open | closed` status pills for the Help Center

**`OpenframeDevSection` Interface**
Defines the shape of each registry entry:
- `href` — route path for navigator card links and cross-links
- `icon` — Lucide icon component reference (never rendered at module load)
- `navigator` — short title + description (~50 chars) for homepage cards
- `hero` — longer title + description (~120–150 chars) for destination page heroes
- `search` — placeholder text + URL param key, or `null` (onboarding)
- `filter` — label, param key, default value, and options array, or `null` (onboarding)

**`OPENFRAME_DEV_SECTIONS`**
The registry object (`as const satisfies Record<string, OpenframeDevSection>`) with five keys: `roadmap`, `delivery`, `releases`, `onboarding`, `tickets`.

**`OpenframeDevSectionKey`**
Derived union type (`keyof typeof OPENFRAME_DEV_SECTIONS`) for type-safe key references.

## Usage Example

```typescript
import { OPENFRAME_DEV_SECTIONS, type OpenframeDevSectionKey } from './openframe-dev-sections'

// Render all homepage navigator cards
Object.values(OPENFRAME_DEV_SECTIONS).map((section) => ({
  href: section.href,
  title: section.navigator.title,
  description: section.navigator.description,
  Icon: section.icon,
}))

// Skip onboarding when building DevSectionView-hosted pages only
const devCenterSections = Object.values(OPENFRAME_DEV_SECTIONS).filter(
  (s) => s.href !== '/onboarding-guides'
)

// Type-safe key access
const key: OpenframeDevSectionKey = 'roadmap'
const { search, filter } = OPENFRAME_DEV_SECTIONS[key]
```

> **Embedder note:** The `onboarding` entry has `search: null` and `filter: null` — its search and filter controls are injected by the host via `DevSectionPage`'s `preControls` slot. Any consumer iterating the registry for `DevSectionView`-hosted pages should filter out `href === '/onboarding-guides'`.
