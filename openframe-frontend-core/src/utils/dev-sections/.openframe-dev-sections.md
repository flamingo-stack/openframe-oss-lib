Canonical metadata registry for all OpenFrame dev-center surfaces, co-locating icon, copy, search config, and filter config for each section in a single source of truth consumed by both the homepage navigator grid and destination page heroes.

## Key Components

**Exported Constants**

| Constant | Purpose |
|---|---|
| `ROADMAP_STATUS_OPTIONS` | Filter options for the Roadmap section (`all` / `completed` / `in_progress`) |
| `DELIVERY_TASK_TYPE_OPTIONS` | Filter options for Bug-fixes & Enhancements (`all` / `Bug` / `Request`) |
| `TICKET_STATUS_OPTIONS` | Filter options for Help Center tickets (`all` / `open` / `closed`) |
| `OPENFRAME_DEV_SECTIONS` | Master registry of all five sections: `roadmap`, `delivery`, `releases`, `onboarding`, `tickets` |

**Exported Types**

- `OpenframeDevSection` — interface describing the shape of each section entry (`href`, `icon`, `navigator`, `hero`, `search`, `filter`)
- `OpenframeDevSectionKey` — union type of valid section keys (`"roadmap" | "delivery" | "releases" | "onboarding" | "tickets"`)

## Usage Example

```typescript
import {
  OPENFRAME_DEV_SECTIONS,
  type OpenframeDevSectionKey,
} from './openframe-dev-sections'

// Render all homepage navigator cards (skip onboarding if route not hosted)
const navSections = Object.values(OPENFRAME_DEV_SECTIONS).filter(
  (s) => s.href !== '/onboarding-guides'
)

// Access a specific section's hero copy
const { title, description } = OPENFRAME_DEV_SECTIONS.roadmap.hero

// Type-safe section lookup
function getSectionConfig(key: OpenframeDevSectionKey) {
  return OPENFRAME_DEV_SECTIONS[key]
}

// Build filter pills for a section (guard for null)
const roadmapFilter = OPENFRAME_DEV_SECTIONS.roadmap.filter
if (roadmapFilter) {
  roadmapFilter.options.forEach(({ value, label }) => {
    console.log(`${label} → ?${roadmapFilter.paramKey}=${value}`)
  })
}
```

> **Note:** The `onboarding` section has `search: null` and `filter: null` by design — its search and filter controls are injected by the host via `DevSectionPage`'s `preControls` slot. Embedders iterating the registry for sections that use `DevSectionView` chrome should filter it out explicitly.
