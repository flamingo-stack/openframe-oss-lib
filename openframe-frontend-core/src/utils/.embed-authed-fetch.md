Shared `fetch` wrapper for embedded surfaces (chat, ticket center, widgets) that automatically attaches bearer-act-as proxy authentication headers and supports optional host-supplied auth adapters with 401 self-healing.

## Key Components

### `EmbedAuthAdapter` (interface)
Opt-in auth adapter interface for hosts with custom auth models (e.g., OpenFrame JWT, OAuth tokens). Exposes:
- `getHeaders()` — returns per-request headers merged over proxy defaults
- `credentials` — overrides the default `'same-origin'` credentials mode
- `refresh()` — optional 401 self-heal callback; resolves `true` to trigger a single retry with refreshed headers

### `setEmbedAuthAdapter(adapter)`
Registers (or clears) a host-owned auth adapter at module level. Stored on `globalThis` to survive bundler chunk duplication. Warns in dev if overwriting an existing registration.

### `embedAuthedFetch(url, init)`
Drop-in `fetch` replacement that:
- Enforces same-origin URLs synchronously (throws on cross-origin to prevent bearer leakage)
- Merges proxy-auth headers via `applyProxyAuth` (proxy values win over caller headers)
- Defaults to `Content-Type: application/json` when no headers are provided
- Triggers `dedupedRefresh()` + single retry on `401` when an adapter with `refresh` is registered

### `dedupedRefresh()`
Deduplicates concurrent 401 refresh attempts via a `globalThis`-scoped in-flight promise — ensures at most one refresh call fires per expiry cycle regardless of how many requests 401 simultaneously.

## Usage Example

```typescript
// Basic usage — proxy auth applied automatically
const res = await embedAuthedFetch('/api/chat/messages', {
  method: 'POST',
  body: JSON.stringify({ message: 'Hello' }),
})

// Register a host adapter (e.g., OpenFrame JWT + cross-origin cookies)
setEmbedAuthAdapter({
  getHeaders: () => ({
    Authorization: token ? `Bearer ${token}` : undefined,
  }),
  credentials: 'include',
  refresh: async () => {
    const ok = await refreshAccessToken()
    return ok
  },
})

// Clear on provider unmount
setEmbedAuthAdapter(null)
```

> **Note:** Use `embedAuthedFetch` only for routes requiring bearer-act-as identity (`/api/chat/*`, `/api/docs/chat/*`, `/api/storage/generate-upload-url`). Routes like `/api/profile/me` should use vanilla `fetch`.
