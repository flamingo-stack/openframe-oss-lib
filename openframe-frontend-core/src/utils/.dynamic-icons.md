A unified icon registry system that provides dynamic icon rendering by mapping database icon names to imported React components, ensuring type safety and consistent sizing across the application.

## Key Components

- **ICON_REGISTRY**: Maps exact icon names from database to React components (UI Kit social icons + Lucide icons)
- **SIZE_CLASSES**: Tailwind CSS size presets (xs through xl) for consistent icon dimensions
- **ICON_COLORS**: Optional color overrides for specific platform icons
- **getDynamicIcon()**: Main function that dynamically renders icons by name with fallback handling
- **getPlatformLogo**: Re-exported platform-specific logo function for unified API

## Usage Example

```typescript
import { getDynamicIcon, DynamicIconSize } from './dynamic-icons';

// Basic usage with default medium size
const icon = getDynamicIcon('LinkedInIcon');

// With custom size and additional classes
const largeIcon = getDynamicIcon('YouTubeIcon', 'lg', 'text-red-500');

// Handle missing icons (returns Globe fallback)
const fallbackIcon = getDynamicIcon('UnknownIcon');

// Size options
const sizes: DynamicIconSize[] = ['xs', 'sm', 'md', 'lg', 'xl'];
```

The system ensures all icons are explicitly imported (no runtime tricks), provides consistent sizing through Tailwind classes, and includes comprehensive error handling with fallback icons for missing components.
