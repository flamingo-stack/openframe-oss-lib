Maps product release types to their corresponding `StatusBadge` color schemes for the `ProductReleaseCard` large variant's metadata grid.

## Key Components

**Types**
- `ReleaseType` — Union of valid release type strings: `'major' | 'minor' | 'patch' | 'beta' | 'alpha'`
- `ReleaseTypeBadgeColor` — Union of badge color tokens: `'error' | 'cyan' | 'success' | 'warning'`

**Function**
- `releaseTypeToBadgeColor(t)` — Converts a release type string to its badge color token. Accepts `ReleaseType | string | null | undefined` to handle legacy API values gracefully; returns `undefined` for unknown inputs (triggers the card's em-dash fallback rather than crashing).

**Color mapping**

| Release Type | Color | Semantic |
|---|---|---|
| `major` | `error` (red) | Breaking change |
| `minor` | `cyan` | New feature |
| `patch` | `success` (green) | Safe bump |
| `beta` / `alpha` | `warning` (yellow) | Preview |

## Usage Example

```typescript
import { releaseTypeToBadgeColor } from './release-badge'

// Known release type
const color = releaseTypeToBadgeColor('major') // → 'error'

// In a ProductReleaseCard render
const releaseTypeBadgeColor = releaseTypeToBadgeColor(release.release_type)

// Card renders colored badge only when BOTH values are present
// If release_type is unknown/null, color is undefined → card shows '—'
<ProductReleaseCard
  size="lg"
  releaseType={release.release_type}
  releaseTypeBadgeColor={releaseTypeToBadgeColor(release.release_type)}
/>
```

> **Note:** The admin form's `releaseTypeOptions.color` array uses a separate color vocabulary — this utility is intentionally scoped to `ProductReleaseCard` display logic only.
