Maps product release types to their corresponding `StatusBadge` color schemes for use in `ProductReleaseCard` lg variant metadata rendering.

## Key Components

**Types**
- `ReleaseType` — Union of valid release type strings: `'major' | 'minor' | 'patch' | 'beta' | 'alpha'`
- `ReleaseTypeBadgeColor` — Union of badge color tokens: `'error' | 'cyan' | 'success' | 'warning'`

**Functions**
- `releaseTypeToBadgeColor(t)` — Converts a release type string to its badge color token; accepts loose input (`string | null | undefined`) to handle legacy API values and returns `undefined` for unknowns

## Color Mapping

| Release Type | Badge Color | Semantic Intent |
|---|---|---|
| `major` | `error` (red) | Breaking change |
| `minor` | `cyan` | New feature |
| `patch` | `success` (green) | Safe bump |
| `beta` / `alpha` | `warning` (yellow) | Preview/unstable |

## Usage Example

```typescript
import { releaseTypeToBadgeColor } from './release-badge'

const color = releaseTypeToBadgeColor(release.release_type)

// Pass to ProductReleaseCard — badge renders only when both props are defined
<ProductReleaseCard
  size="lg"
  releaseType={release.release_type}
  releaseTypeBadgeColor={color}  // undefined → falls back to '—' placeholder
/>
```

> **Note:** An `undefined` return is the intentional failure mode for unknown/legacy values — the card silently falls back to an em-dash rather than crashing. Do not conflate `releaseTypeOptions.color` used in the admin form with these tokens; they serve a different concern.
