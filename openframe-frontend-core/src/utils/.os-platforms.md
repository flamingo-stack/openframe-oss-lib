Operating system platform definitions and configuration used throughout the application to provide OS-specific UI elements and functionality.

## Key Components

**Types:**
- `OSPlatformId` - Union type for supported OS identifiers ('windows', 'linux', 'darwin')
- `OSPlatformOption` - Interface defining platform structure with id, display name, and icon component

**Exports:**
- `OS_PLATFORMS` - Array of all available OS platform configurations
- `DEFAULT_OS_PLATFORM` - Default platform selection ('windows')

## Usage Example

```typescript
import { OS_PLATFORMS, DEFAULT_OS_PLATFORM, type OSPlatformId } from './os-platforms';

// Render platform selector
function PlatformSelector() {
  return (
    <div>
      {OS_PLATFORMS.map(platform => (
        <button key={platform.id}>
          <platform.icon />
          {platform.name}
        </button>
      ))}
    </div>
  );
}

// Platform detection utility
function getCurrentPlatform(): OSPlatformId {
  const userAgent = navigator.userAgent.toLowerCase();
  
  if (userAgent.includes('mac')) return 'darwin';
  if (userAgent.includes('linux')) return 'linux';
  
  return DEFAULT_OS_PLATFORM; // fallback to windows
}
```

This module provides a centralized way to handle OS-specific functionality like installation instructions, file paths, or platform-specific features across the application.
