A localStorage utility for managing onboarding walkthrough state with atomic writes and cross-tab synchronization. Provides a complete API for tracking completed steps, skipped steps, and dismissal status.

## Key Components

- **OnboardingState interface**: Defines the structure for onboarding data (completed/skipped steps, dismissed status, timestamps)
- **saveOnboardingState/loadOnboardingState**: Core storage operations with error handling and SSR safety
- **Step management functions**: `markStepComplete`, `markStepSkipped`, `markMultipleComplete` for tracking progress
- **Status checkers**: `isStepComplete`, `isStepSkipped`, `isOnboardingDismissed` for querying state
- **dismissOnboarding**: Mark the entire onboarding as dismissed

## Usage Example

```typescript
// Initialize onboarding for a feature
const ONBOARDING_KEY = 'app-onboarding'

// Mark steps as complete
markStepComplete(ONBOARDING_KEY, 'welcome-step')
markMultipleComplete(ONBOARDING_KEY, ['step-1', 'step-2'])

// Check progress
if (isStepComplete(ONBOARDING_KEY, 'welcome-step')) {
  console.log('Welcome step completed')
}

// Skip a step
markStepSkipped(ONBOARDING_KEY, 'optional-step')

// Dismiss entire onboarding
dismissOnboarding(ONBOARDING_KEY)

// Load current state
const currentState = loadOnboardingState(ONBOARDING_KEY)
console.log('Progress:', currentState.completedSteps.length)
```
