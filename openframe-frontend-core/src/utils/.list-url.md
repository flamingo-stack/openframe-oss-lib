Builds type-safe list API URLs for chat entity card fetching — the single authoritative source for per-type endpoint shapes used by both the hub's RAG mappers and embedder clients.

## Key Components

**`buildListUrl(contentRefType, ids, base?)`** — Main export. Returns a fully-formed list API URL string for the given content type and IDs, or `null` if the type has no list endpoint (preventing fetch attempts).

**`BUILDERS`** — Internal map of 12 canonical `documentType` keys to URL-builder functions. Each builder's output is byte-parity tested against its corresponding hub RAG mapper closure.

**`ALIASES`** — Legacy `contentRefType` remapping (e.g. `blog_post_existing` → `blog_post`) for callers that predate RAG `documentType` unification.

## Supported Content Types

| Type | Endpoint Pattern |
|---|---|
| `roadmap_item`, `delivery_item`, `internal_task` | `?task_ids=` (ClickUp-backed) |
| `blog_post` | `?ids=&pageSize=` |
| `webinar`, `podcast`, `event` | `?ids=&limit=&filter=all` |
| `onboarding_guide`, `case_study`, `product_release`, `customer_interview`, `investor_update` | `?ids=&limit=` |
| `marketing_campaign` | Static literal branch (`/api/admin/...`) |
| `github_*`, `slack_message`, financials | Returns `null` (no list endpoint) |

## Usage Example

```typescript
import { buildListUrl } from '@flamingo-stack/openframe-frontend-core/utils'

// Hub-relative (default base)
buildListUrl('roadmap_item', ['abc', 'def'])
// → '/api/roadmap?task_ids=abc,def'

// With reverse-proxy prefix (embedder usage)
buildListUrl('blog_post_existing', ['x1', 'x2'], '/content')
// → '/content/api/blog/posts?ids=x1,x2&pageSize=2'

// Types with no list endpoint return null
buildListUrl('github_pr', ['abc'])
// → null

// Wire into embedder endpoints once:
endpoints.buildListUrl = (type, ids) => buildListUrl(type, ids, '/content')
```

> **Security note:** `marketing_campaign` is handled as a static literal branch rather than a `BUILDERS` entry, allowing CodeQL to prove no user-controlled dynamic dispatch can reach `/api/admin`.
