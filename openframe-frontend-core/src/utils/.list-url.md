Builds a type-safe list-API URL for fetching full entity rows by ID in OpenFrame's chat entity-card pipeline, serving as the single source of truth for per-content-type endpoint shapes across hub RAG mappers and embedders.

## Key Components

- **`ALIASES`** — Maps legacy `contentRefType` strings (e.g. `blog_post_existing`) to their canonical `documentType` equivalents before URL lookup.
- **`BUILDERS`** — A keyed registry of URL constructor functions for each fetch-mode content type. Covers 12 entity types across roadmap/delivery tasks, blog, programs, guides, case studies, releases, interviews, and investor updates.
- **`buildListUrl(contentRefType, ids, base?)`** — The exported entry point. Resolves aliases, handles the `marketing_campaign` admin endpoint as a static literal branch (CodeQL-safe), performs a `hasOwnProperty` guard to prevent prototype-chain dispatch, and returns `null` for unknown/no-list types.

## Usage Example

```typescript
import { buildListUrl } from '@flamingo-stack/openframe-frontend-core/utils'

// Hub-relative (default base)
buildListUrl('roadmap_item', ['abc', 'def'])
// → '/api/roadmap?task_ids=abc,def'

// Legacy alias resolution
buildListUrl('blog_post_existing', ['abc', 'def'])
// → '/api/blog/posts?ids=abc,def&pageSize=2'

// With reverse-proxy prefix (embedder usage)
buildListUrl('webinar', ['abc'], '/content')
// → '/content/api/programs/webinars?ids=abc&limit=1&filter=all'

// Type with no list endpoint → null (fetch skipped)
buildListUrl('github_pr', ['abc'])
// → null

// Embedder wiring
endpoints.buildListUrl = (type, ids) => buildListUrl(type, ids, '/content')
```

> **Note:** Returns `null` for empty `ids` arrays or unregistered types. Non-RAG types (`github_*`, `slack_message`, financials) are intentionally absent from `BUILDERS` — their `null` return disables fetching rather than fabricating a URL.
