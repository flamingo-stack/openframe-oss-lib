Provides email domain validation utilities for SSO domain allowlist features, enabling validation of domain formats used in auto-provisioning restrictions.

## Key Components

- **`isValidEmailDomain(domain: string): boolean`** - Core validation function that checks if a domain follows proper email domain format
- **`validateEmailDomain(domain: string): EmailDomainValidationResult`** - Enhanced validation with detailed error messages and input cleaning
- **`validateEmailDomainList(domains: string[]): ValidationResult`** - Validates arrays of domains with duplicate detection
- **`cleanEmailDomain(domain: string): string`** - Cleans domain input by removing common user mistakes (protocols, @, www, etc.)
- **`EmailDomainValidationResult`** - Interface for validation results with error details

## Usage Example

```typescript
import { validateEmailDomain, validateEmailDomainList, isValidEmailDomain } from './validation-utils'

// Basic validation
const isValid = isValidEmailDomain('example.com') // true
const isInvalid = isValidEmailDomain('not-a-domain') // false

// Detailed validation with error messages
const result = validateEmailDomain('@company.com')
// { valid: true, cleanedDomain: 'company.com' }

const errorResult = validateEmailDomain('http://example.com')
// { valid: false, error: 'Enter email domain only (e.g., openframe.ai), not a URL' }

// Validate multiple domains
const listResult = validateEmailDomainList(['company.com', 'partner.org'])
// { valid: true, cleanedDomains: ['company.com', 'partner.org'] }
```
