Utility module providing class name merging with custom Tailwind typography extensions, and environment-aware URL resolution for Flamingo platform deployments.

## Key Components

- **`cn(...inputs)`** — Merges class names using `clsx` + a custom `tailwind-merge` instance extended with ODS typography classes (`text-h1` through `text-h6`).
- **`getPlatformProductionUrl(platform)`** — Returns the production URL for a named platform, with support for `NEXT_PUBLIC_*` environment variable overrides.
- **`getAllPlatformBaseDomains()`** — Extracts all unique base domains from known platforms; handles localhost, Vercel preview, and production cases (useful for cross-domain cookie scoping).
- **`getBaseUrl(platform?)`** — Resolves the application base URL for the current environment, respecting dev/preview/production context and optional platform targeting.

## Usage Example

```typescript
import { cn, getBaseUrl, getPlatformProductionUrl } from '@/lib/cn'

// Merge Tailwind classes (deduplicates conflicting utilities)
const classes = cn('text-h1 font-bold', isActive && 'text-amber-400')

// Resolve URL for current environment
const appUrl = getBaseUrl()                    // http://localhost:3000 (dev)
const flamingoUrl = getBaseUrl('flamingo')     // https://www.flamingo.run (prod)
const openframeUrl = getBaseUrl('openframe')   // https://openframe.ai (prod)

// Get a platform URL directly (always production, with env override)
const dashboardUrl = getPlatformProductionUrl('openframe-dashboard')
// → process.env.NEXT_PUBLIC_OPENFRAME_DASHBOARD_URL || 'https://openframe.ai'

// Get all cookie-scoped domains (e.g. for cross-platform auth)
const domains = getAllPlatformBaseDomains()
// → ['.flamingo.run', 'flamingo.run', '.openmsp.ai', 'openmsp.ai', ...]
```
