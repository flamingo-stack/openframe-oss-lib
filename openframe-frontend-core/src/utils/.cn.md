Utility module providing class name merging with extended Tailwind support, and platform URL resolution helpers for the Flamingo/OpenFrame multi-platform monorepo.

## Key Components

- **`cn(...inputs)`** — Merges class names using `clsx` + a custom `tailwind-merge` instance extended with ODS typography classes (`text-h1` through `text-h6`).
- **`getPlatformProductionUrl(platform)`** — Returns the production base URL for a named platform (e.g. `flamingo`, `openframe`, `openmsp`, hub variants), with `NEXT_PUBLIC_*` environment variable overrides.
- **`getAllPlatformBaseDomains()`** — Extracts all unique base domains across every platform (handles localhost, Vercel preview, and production). Useful for cross-domain cookie scoping.
- **`getBaseUrl(platform?)`** — Returns the effective base URL for the current environment, respecting `NODE_ENV`, `VERCEL_PROJECT_PRODUCTION_URL`, and optional platform targeting.

## Usage Example

```typescript
import { cn, getBaseUrl, getPlatformProductionUrl } from '@/lib/cn'

// Merge Tailwind classes safely (including ODS typography)
const className = cn('text-h1 font-bold', isActive && 'text-amber-500')

// Get the current app's base URL (respects environment)
const appUrl = getBaseUrl() // 'http://localhost:3000' (dev) | 'https://www.openmsp.ai' (prod)

// Get a specific platform's URL with env override support
const openframeUrl = getPlatformProductionUrl('openframe')
// → process.env.NEXT_PUBLIC_OPENFRAME_URL || 'https://openframe.ai'

// Get platform-specific base URL (useful for cross-app links/CTAs)
const flamingoUrl = getBaseUrl('flamingo') // 'https://www.flamingo.run'

// Get all cookie-scoped domains for the current environment
import { getAllPlatformBaseDomains } from '@/lib/cn'
const domains = getAllPlatformBaseDomains()
// Production → ['.flamingo.run', 'flamingo.run', '.openmsp.ai', 'openmsp.ai', ...]
// Vercel preview → ['.vercel.app', 'vercel.app']
// Localhost → []
```

> **Note:** `openframe-dashboard` is intentionally distinct from `openframe` — the `openframe` key may be overridden to `hub.openframe.ai` in production, while `openframe-dashboard` always resolves to the product/CTA target (`https://openframe.ai`).
