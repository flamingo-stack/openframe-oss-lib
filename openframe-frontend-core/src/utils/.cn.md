A utility module providing class name merging, formatting functions, and platform URL management for the Flamingo/OpenMSP ecosystem.

## Key Components

- **`cn(...inputs)`** - Combines and merges Tailwind CSS class names using clsx and tailwind-merge
- **`formatDate(date, options)`** - Formats dates to human-readable strings with validation
- **`formatNumber(num)`** - Adds thousands separators to numbers
- **`formatPrice(price, currency)`** - Formats prices with currency symbols
- **`formatBytes(bytes, decimals)`** - Converts bytes to human-readable sizes (KB, MB, GB, etc.)
- **`getBaseUrl(platform?)`** - Gets application base URLs with environment-specific logic
- **`getAllPlatformBaseDomains()`** - Extracts all unique base domains for cross-platform cookie sharing
- **`getPlatformProductionUrl(platform)`** - Maps platform identifiers to their production URLs

## Usage Example

```typescript
import { cn, formatDate, formatBytes, getBaseUrl } from './cn'

// Merge CSS classes with Tailwind conflict resolution
const buttonClass = cn('px-4 py-2', 'px-6', 'bg-blue-500', { 'bg-red-500': isError })

// Format various data types
const displayDate = formatDate(new Date(), { month: 'short', day: 'numeric' })
const fileSize = formatBytes(1024000) // "1000 KB"
const price = formatPrice(99.99, 'USD') // "$99.99"

// Get platform URLs
const flamingoUrl = getBaseUrl('flamingo') // https://flamingo.run (prod) or localhost (dev)
const currentUrl = getBaseUrl() // Current app's base URL

// Get all domains for cookie configuration
const domains = getAllPlatformBaseDomains() // ['.flamingo.so', 'flamingo.so', ...]
```

The module handles environment detection, supports Vercel deployments, and provides consistent URL resolution across the Flamingo platform ecosystem including OpenMSP, Flamingo, TMCG, and OpenFrame applications.
