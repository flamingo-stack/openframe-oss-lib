A configuration module that defines the application's core settings and provides utilities for accessing app-specific configuration values.

## Key Components

- **`APP_CONFIG`** - Immutable configuration object containing app metadata and feature flags
- **`getAppConfig()`** - Function that returns the complete application configuration
- **`getAppType()`** - Function that retrieves the app type from environment variables with fallback

## Usage Example

```typescript
import { APP_CONFIG, getAppConfig, getAppType } from './app-config';

// Access static configuration
console.log(APP_CONFIG.app.name); // 'OpenMSP'
console.log(APP_CONFIG.features.announcements); // true

// Get configuration dynamically
const config = getAppConfig();
if (config.features.notifications) {
  // Enable notification features
}

// Check app type (useful for multi-tenant applications)
const appType = getAppType();
if (appType === 'openmsp') {
  // OpenMSP-specific logic
}
```

The configuration uses TypeScript's `as const` assertion to ensure type safety and immutability. The `getAppType()` function allows runtime determination of the application type through environment variables, making it suitable for multi-tenant or white-label applications.
