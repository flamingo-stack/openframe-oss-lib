This file provides centralized application configuration for an OpenMSP application, defining core app metadata and feature flags.

## Key Components

- **`APP_CONFIG`** - Main configuration object containing app metadata (`type`, `name`, `domain`) and feature toggles (`announcements`, `notifications`)
- **`getAppConfig()`** - Returns the complete application configuration object
- **`getAppType()`** - Retrieves the app type from environment variables with fallback to 'openmsp'

## Usage Example

```typescript
import { getAppConfig, getAppType, APP_CONFIG } from './app-config';

// Get full configuration
const config = getAppConfig();
console.log(config.app.name); // 'OpenMSP'
console.log(config.features.announcements); // true

// Get app type with environment override
const appType = getAppType(); // 'openmsp' or process.env.NEXT_PUBLIC_APP_TYPE

// Direct access to config
if (APP_CONFIG.features.notifications) {
  // Enable notification features
}

// Conditional rendering based on app domain
const isProduction = config.app.domain === 'openmsp.ai';
```
