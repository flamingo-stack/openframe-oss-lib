Storybook configuration file defining interactive stories for the MobileFilterModal component, showcasing various filtering and sorting scenarios for mobile interfaces.

## Key Components

- **Meta Configuration**: Storybook metadata with component documentation and control definitions
- **Story Variants**: Eight different story examples covering common use cases
- **Default Data**: Predefined filter groups and configurations for consistent testing
- **Interactive Examples**: Stateful stories demonstrating real-world usage patterns

## Usage Example

```typescript
// Basic story implementation
export const Default: Story = {
  args: {
    isOpen: true,
    title: 'Sort and Filter',
    filterGroups: defaultFilterGroups,
    currentFilters: defaultCurrentFilters,
    onFilterChange: fn(),
    onClose: fn(),
  },
}

// Interactive story with state management
export const Interactive: Story = {
  render: function InteractiveStory() {
    const [isOpen, setIsOpen] = useState(false)
    const [currentFilters, setCurrentFilters] = useState<TableFilters>(defaultCurrentFilters)

    const handleFilterChange = (filters: TableFilters) => {
      setCurrentFilters(filters)
      setIsOpen(false)
    }

    return (
      <div className="flex flex-col items-center gap-4">
        <Button onClick={() => setIsOpen(true)} variant="outline">
          Open Filters Modal
        </Button>
        <MobileFilterModal
          isOpen={isOpen}
          onClose={() => setIsOpen(false)}
          filterGroups={defaultFilterGroups}
          currentFilters={currentFilters}
          onFilterChange={handleFilterChange}
        />
      </div>
    )
  },
}
```

The stories demonstrate filtering with status counts, multiple filter groups, sorting capabilities, and interactive state management scenarios typical in MSP platform interfaces.
