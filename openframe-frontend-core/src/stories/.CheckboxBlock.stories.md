Storybook configuration file showcasing all variants and use cases of the CheckboxBlock component, including interactive examples for testing component behavior and visual states.

## Key Components

- **meta**: Main Storybook configuration object defining component metadata, parameters, and controls
- **Story exports**: Multiple story variants demonstrating different states and configurations:
  - `Default` - Basic unchecked state
  - `RepeatScriptRun` - Figma design reference implementation
  - `Checked` - Pre-checked state
  - `Disabled/DisabledChecked` - Non-interactive states
  - `Controlled` - Interactive state management example
  - `WithDescription` variants - Taller layout with secondary text
  - `AllVariants` - Comprehensive comparison view

## Usage Example

```typescript
import { CheckboxBlock } from '../components/ui/checkbox-block';
import { useState } from 'react';

// Basic usage
<CheckboxBlock label="Enable notifications" />

// Controlled state
const [isEnabled, setIsEnabled] = useState(false);
<CheckboxBlock
  label="Controlled Option"
  checked={isEnabled}
  onCheckedChange={setIsEnabled}
/>

// With description
<CheckboxBlock
  label="Run as User"
  description="Windows Only"
  defaultChecked={true}
/>

// Disabled state
<CheckboxBlock
  label="Unavailable Option"
  disabled={true}
/>
```

The stories provide comprehensive testing scenarios for the ODS design system checkbox component, including dark theme styling, text wrapping behavior, and state management patterns.
