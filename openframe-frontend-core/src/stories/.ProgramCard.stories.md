Storybook stories for the `ProgramCard` entity card component, covering the three supported program doc types (`podcast`, `webinar`, `event`) at both `sm` (chat-dispatch) and `default` density sizes.

## Key Components

| Export | Description |
|---|---|
| `PodcastSm` | Podcast card at `sm` size — matches chat dispatch rendering |
| `WebinarSm` | Webinar card at `sm` size — matches chat dispatch rendering |
| `EventSm` | Event card at `sm` size — matches chat dispatch rendering |
| `PodcastDefault` | Podcast card at `default` density — catalog page layout |

## Usage Example

```typescript
// Rendering ProgramCard with minimum safe config for chat
import { ProgramCard } from '../components/chat/entity-cards'
import { DEFAULT_PROGRAM_CONFIGS } from '../components/chat/entity-cards/program-card-defaults'

<ProgramCard
  config={DEFAULT_PROGRAM_CONFIGS.podcast}
  item={podcastItem}
  size="sm"
  href={`/podcasts/${podcastItem.id}`}
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column width context
- `DEFAULT_PROGRAM_CONFIGS[type]` provides safe minimum config for compact chat rendering; embedders can override per type with a full catalog config
- `sm` size is the canonical size used by the chat dispatch; `default` size targets catalog page rendering
