Storybook stories for `ProgramCard`, a generic entity card component supporting `podcast`, `webinar`, and `event` doc types, demonstrating both compact (`sm`) and full (`default`) size variants.

## Key Components

| Export | Description |
|---|---|
| `PodcastSm` | Podcast card at `sm` size — used by chat dispatch |
| `WebinarSm` | Webinar card at `sm` size — used by chat dispatch |
| `EventSm` | Event card at `sm` size — used by chat dispatch |
| `PodcastDefault` | Podcast card at `default` density — mirrors catalog page rendering |

## Usage Example

```typescript
import { ProgramCard } from '../components/chat/entity-cards'
import { DEFAULT_PROGRAM_CONFIGS } from '../components/chat/entity-cards/program-card-defaults'

// Compact chat variant (sm)
<ProgramCard
  config={DEFAULT_PROGRAM_CONFIGS.podcast}
  item={podcastItem}
  size="sm"
  href={`/podcasts/${podcastItem.id}`}
  target="_blank"
  rel="noopener noreferrer"
/>

// Full density variant (catalog page)
<ProgramCard
  config={DEFAULT_PROGRAM_CONFIGS.podcast}
  item={podcastItem}
  size="default"
  href={`/podcasts/${podcastItem.id}`}
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout context
- `DEFAULT_PROGRAM_CONFIGS[type]` provides safe minimum config for chat rendering; embedders can override per type with full catalog config
- `sm` is the size used by chat dispatch; `default` is used on catalog/detail pages
- The `config` prop drives rendering behavior, allowing per-type customization without changing the component itself
