Storybook stories for the `CustomerInterviewCard` component, demonstrating two size variants (`sm` and `default`) of the `customer_interview` doc type entity card within a chat column layout.

## Key Components

- **`meta`** — Storybook metadata registering the component under `Chat/EntityCards/CustomerInterviewCard` with fullscreen layout
- **`ChatColumnDecorator`** — Wraps stories in a simulated chat column context
- **`customerInterview`** — Shared fixture providing mock interview data (slug, video URL, user/MSP badge info)
- **`ChatInlineSm`** — Story rendering the card at `size: 'sm'` (inline/compact chat variant)
- **`Default`** — Story rendering the card at `size: 'default'` (standard chat card variant)

## Usage Example

```typescript
import { CustomerInterviewCard } from '../components/chat/entity-cards'

<CustomerInterviewCard
  interview={customerInterview}
  href={`/customer-interviews/${customerInterview.slug}`}
  size="default"
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- The `main_video_url` field on the interview fixture controls whether the play-button overlay renders on the card cover
- Both stories pass `target="_blank"` and `rel="noopener noreferrer"` for external link safety
- The user + MSP badge overlay is present in both `sm` and `default` size variants
