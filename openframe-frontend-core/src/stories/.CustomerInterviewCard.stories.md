Storybook stories for the `CustomerInterviewCard` component, covering both `sm` and `default` size variants within a chat column layout.

## Key Components

- **`meta`** — Storybook metadata registering the component under `Chat/EntityCards/CustomerInterviewCard` with fullscreen layout
- **`ChatColumnDecorator`** — Wraps each story in a chat-column context to simulate real rendering environment
- **`customerInterview`** — Shared fixture providing mock interview data for all stories
- **`ChatInlineSm`** — Story rendering the card at `size: 'sm'` (inline/compact variant)
- **`Default`** — Story rendering the card at `size: 'default'` (standard variant)

## Usage Example

```typescript
// Render the CustomerInterviewCard in a story
export const Default: Story = {
  args: {
    interview: customerInterview,
    href: '/customer-interviews/' + customerInterview.slug,
    size: 'default',
    target: '_blank',
    rel: 'noopener noreferrer',
  },
}
```

## Notes

- The `main_video_url` field on the interview fixture controls whether the play-button overlay appears on the card cover
- Both stories open links in a new tab via `target: '_blank'` and `rel: 'noopener noreferrer'`
- The User + MSP badge overlay renders in both `sm` and `default` sizes
