Storybook stories for the `BlogCard` entity card component, covering both compact chat-inline (`sm`) and full vertical catalog (`default`) display densities.

## Key Components

| Export | Description |
|--------|-------------|
| `ChatInlineSm` | Compact horizontal form (~80px tall) used inline in chat responses |
| `ChatInlineWithVideoBadge` | `sm` variant with `hasEmbeddedVideo` flag to render a video badge indicator |
| `Default` | Full vertical card for catalog/related-content rail use cases |

## Usage Example

```typescript
import { BlogCard } from '../components/chat/entity-cards'

// Chat-inline usage (dispatched automatically with size="sm")
<BlogCard
  post={blogPostSummary}
  href={`/blog/${blogPostSummary.slug}`}
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
/>

// With embedded video badge
<BlogCard
  post={blogPostSummary}
  href={`/blog/${blogPostSummary.slug}`}
  size="sm"
  hasEmbeddedVideo={true}
  target="_blank"
  rel="noopener noreferrer"
/>

// Full vertical catalog card (related-content rail)
<BlogCard
  post={blogPostSummary}
  href={`/blog/${blogPostSummary.slug}`}
  size="default"
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- The card composes its own `<a>` tag internally from the `href` prop — callers are responsible for URL resolution via `buildContentURL` or `useNavLink`
- Chat dispatch always passes `size="sm"`; the `default` size is reserved for the related-content rail
- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout context
