Storybook stories for the `GuideWelcome` component, covering the guide-mode chat empty state with configurable quick actions, slash-command onboarding lists, and greeting copy variants.

## Key Components

| Export | Description |
|--------|-------------|
| `Default` | Full empty state with slash-command list and quick-action chips |
| `NoCommands` | Greeting and quick actions only, no slash-command list |
| `NoQuickActions` | Slash-command list only, chip row omitted |
| `CustomCopy` | Custom `title` and `subtitle` overriding default greeting text |

**Supporting fixtures:**
- `SAMPLE_QUICK_ACTIONS` — seven action chips demonstrating inline display and `⋯` overflow menu
- `SAMPLE_LIST` — three `MingoOnboardingCard` entries (`/release-notes`, `/blogs`, `/getting-started`)
- `ACTIONS` — shared Recent / Search / Find callbacks passed to each card

## Usage Example

```typescript
import { GuideWelcome } from '../components/chat/guide-welcome'
import { GuideModeBanner } from '../components/chat/guide-mode-banner'

<div className="flex h-[760px] w-[440px] flex-col">
  <GuideModeBanner />
  <GuideWelcome
    title="Ask the Guide"
    subtitle="Explore docs and tickets."
    quickActions={[
      { id: 'how-to-start', label: 'How to start' },
      { id: 'connect-device', label: 'Connect device' },
    ]}
    onQuickAction={(action) => console.log(action.id)}
  >
    {/* Optional slash-command list */}
  </GuideWelcome>
</div>
```

## Notes

- Decorator wraps every story in a `440×760px` dark `ods-bg` panel with `GuideModeBanner` pinned at the top, matching the Figma node `7532:328214` drawer layout.
- `quickActions` and `children` are both optional; omitting either removes that UI region entirely.
- `onQuickAction` and `quickActions` controls are disabled in Storybook (`control: false`) — pass them programmatically.
