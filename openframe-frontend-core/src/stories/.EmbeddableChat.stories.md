Storybook story file that renders interactive visual states for the `EmbeddableChat` component, covering new-user, returning-user, loading, error, and archive UI variations using mocked NATS adapters and runtime contexts.

## Key Components

| Export / Constant | Purpose |
|---|---|
| `createMockRuntime()` | Builds a minimal `ChatRuntime` with stub endpoints (no real server calls) |
| `createMockMingoConfig()` | Base NATS adapter config; `getNatsWsUrl` returns `null` to suppress WebSocket connections |
| `createMockMingoConfigWithDialogs()` | Extends base config with a populated dialog history, rename/archive/unarchive callbacks, and timestamp bucketing (Today / Yesterday) |
| `createMockMingoConfigChatsLoading()` | `fetchDialogs` never resolves — pins `isDialogsLoading: true` for skeleton UI |
| `createMockMingoConfigChatLoading()` | `fetchDialogMessages` never resolves — pins `isMessagesLoading: true` after opening a dialog |
| `createMockMingoConfigChatsError()` | `fetchDialogs` rejects — triggers `dialogsLoadError` and the retry affordance |
| `createMockMingoConfigArchiveEmpty()` | `fetchArchivedDialogs` resolves empty — renders the archive empty state |
| `SAMPLE_DIALOGS` | 8 canned `DialogItem` entries representing a returning-user history |
| `SAMPLE_QUICK_ACTIONS` | 5 `MingoQuickAction` chip definitions (Weekly Log Summary, Run a Script, etc.) |
| `SAMPLE_SLASH_COMMANDS` | 5 `SlashCommandSummary` entries served via a `window.fetch` shim at `/__story__/commands` |

## Usage Example

```typescript
// Typical story wrapper pattern used throughout this file
function StoryWrapper({ mingoConfig }: { mingoConfig: UseNatsChatAdapterConfig }) {
  const queryClient = useMemo(() => new QueryClient(), [])
  const runtime = useMemo(() => createMockRuntime(), [])

  return (
    <QueryClientProvider client={queryClient}>
      <ChatRuntimeContext.Provider value={runtime}>
        <EmbeddableChat
          mingoConfig={mingoConfig}
          quickActions={SAMPLE_QUICK_ACTIONS}
        />
      </ChatRuntimeContext.Provider>
    </QueryClientProvider>
  )
}

// Example: render the returning-user variation with populated dialog history
export const ReturningUser: StoryObj = {
  render: () => <StoryWrapper mingoConfig={createMockMingoConfigWithDialogs()} />,
}

// Example: render the dialogs loading skeleton
export const ChatsLoading: StoryObj = {
  render: () => <StoryWrapper mingoConfig={createMockMingoConfigChatsLoading()} />,
}
```

> **Note:** The `pending<T>()` helper (`new Promise<T>(() => {})`) is intentionally used for loading states — it never settles, keeping skeleton UIs visible without fake timers or flicker.
