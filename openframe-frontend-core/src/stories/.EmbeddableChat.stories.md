Brief description of the file's purpose: Storybook story file for the `EmbeddableChat` component, providing mock runtimes, NATS adapter configs, and interactive story scenarios covering loading, error, empty, and active chat states.

## Key Components

| Export / Symbol | Description |
|----------------|-------------|
| `createMockRuntime()` | Builds a minimal `ChatRuntime` with stub endpoints (no real server calls) |
| `createMockMingoConfig()` | Base NATS adapter config with a null WS URL and canned message thread |
| `createMockMingoConfigWithDialogs()` | Extends base config with paginated dialog history, rename, archive, and unarchive callbacks |
| `createMockMingoConfigChatsLoading()` | Config where `fetchDialogs` never resolves â€” pins the history skeleton |
| `createMockMingoConfigChatLoading()` | Config where `fetchDialogMessages` never resolves â€” pins the message-list skeleton |
| `createMockMingoConfigChatsError()` | Config where `fetchDialogs` rejects â€” triggers the load-error + retry UI |
| `createMockMingoConfigArchiveEmpty()` | Config with an empty archived-dialogs page â€” renders the archive empty state |
| `SAMPLE_DIALOGS` | Eight canned `DialogItem` objects split across Today / Yesterday groups |
| `SAMPLE_QUICK_ACTIONS` | Five `MingoQuickAction` chip definitions (Search, Script, Health, Tickets, Patch) |
| `SAMPLE_SLASH_COMMANDS` | Six `SlashCommandSummary` entries served via a `window.fetch` shim at `/__story__/commands` |
| `pending<T>()` | Helper returning a never-settling Promise â€” keeps any loading flag pinned `true` |
| `COMMANDS_URL` | Shared constant (`/__story__/commands`) used by both the runtime mock and the fetch shim |

## Usage Example

```typescript
// Returning-user story (dialogs populated, quick-actions rendered)
const meta: Meta<typeof EmbeddableChat> = {
  component: EmbeddableChat,
  decorators: [
    (Story) => {
      const runtime = useMemo(createMockRuntime, [])
      const queryClient = new QueryClient()
      return (
        <QueryClientProvider client={queryClient}>
          <ChatRuntimeContext.Provider value={runtime}>
            <Story />
          </ChatRuntimeContext.Provider>
        </QueryClientProvider>
      )
    },
  ],
}

export const ReturningUser: StoryObj<typeof EmbeddableChat> = {
  args: {
    mingoConfig: createMockMingoConfigWithDialogs(),
    quickActions: SAMPLE_QUICK_ACTIONS,
  },
  // play function uses `within` + `userEvent` to click the first dialog
  // and verify the conversation view loads with canned messages.
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement)
    await userEvent.click(canvas.getByText('PowerShell script for bulk user creation'))
  },
}
```

> **Note:** All fetch calls inside stories are intercepted by a `window.fetch` shim that serves `SAMPLE_SLASH_COMMANDS` for `COMMANDS_URL` and returns `404` for any other story-scoped path â€” no real network traffic is produced.
