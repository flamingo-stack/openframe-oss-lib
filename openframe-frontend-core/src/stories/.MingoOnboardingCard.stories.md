Storybook stories for the `MingoOnboardingCard` component, covering all visual states and a full chip-command stack that mirrors the chat empty-state UI (Figma node `7363:205938`).

## Key Components

| Export | Description |
|---|---|
| `Default` | Base card with icon, title, slash command, and description |
| `WithoutDescription` | Card omitting the description field |
| `WithoutSlashCommand` | Card omitting the slash command badge |
| `LongTitleTruncates` | Validates truncation on long titles and slash commands |
| `Clickable` | Card with an `onClick` handler wired up |
| `WithActions` | Card with inline action chips (Recent, Search, Find) |
| `ChipCommands` | Full stacked list matching the runtime chat empty-state |

## Usage Example

```typescript
// Minimal card
<MingoOnboardingCard
  icon={<CompassIcon size={16} />}
  title="ClickUp Roadmap"
  slashCommand="/roadmap"
  description="Public product roadmap with upcoming features and releases"
/>

// Card with action chips
<MingoOnboardingCard
  icon={<CompassIcon size={16} />}
  title="ClickUp Roadmap"
  slashCommand="/roadmap"
  description="Public product roadmap"
  actions={[
    { id: 'recent', label: 'Recent', onClick: () => {} },
    { id: 'search', label: 'Search', onClick: () => {} },
  ]}
/>
```

The `ChipCommands` story uses `resolveOnboardingIcon(iconName)` to source icons from the shared registry, keeping stories in sync with the chat runtime mapping across 13 sample commands (ClickUp, Slack, HubSpot, OpenFrame Docs, Blog Posts, etc.).
