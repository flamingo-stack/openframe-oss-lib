Storybook stories for the `MingoOnboardingCard` component, covering all prop combinations and a full chip-command stack that mirrors the chat empty-state UI (Figma node `7363:205938`).

## Key Components

| Export | Description |
|---|---|
| `Default` | Base card with icon, title, slash command, and description |
| `WithoutDescription` | Card omitting the optional description field |
| `WithoutSlashCommand` | Card omitting the optional slash command badge |
| `LongTitleTruncates` | Validates truncation behavior for long titles and slash commands |
| `Clickable` | Card with an `onClick` handler wired to console output |
| `WithActions` | Card rendering inline outline-chip action buttons |
| `ChipCommands` | Full stacked list using `resolveOnboardingIcon` to mirror runtime rendering |

## Usage Example

```typescript
// Render a card with actions
<MingoOnboardingCard
  icon={<CompassIcon size={16} />}
  title="ClickUp Roadmap"
  slashCommand="/roadmap"
  description="Public product roadmap with upcoming features and releases"
  actions={[
    { id: 'recent', label: 'Recent', onClick: () => console.log('recent') },
    { id: 'search', label: 'Search', onClick: () => console.log('search') },
  ]}
/>
```

## Notes

- All stories are wrapped in a `w-[520px]` decorator that replicates the `rounded-md` container used in the live chat UI
- The `ChipCommands` story sources icons via `resolveOnboardingIcon`, keeping it in sync with the runtime icon registry — add new slash commands to `SAMPLE_CHIP_COMMANDS` to preview them
- The `icon` and `onClick` argTypes have `control: false` set, as they accept React nodes/functions rather than serializable values
