Storybook decorator and helper utilities for rendering chat entity-card stories in an isolated, realistic context without the full chat runtime.

## Key Components

**`ChatColumnDecorator`** — A React wrapper component that reproduces the assistant message column layout (constrained `max-w-[420px]`, chat background `bg-ods-bg`, ODS padding). Use this as a Storybook decorator to preview entity-cards at their real rendered width.

**`makeAnchorProps`** — Factory function that builds the `{ href, onClick }` pair expected by card inner `<a>` elements. The `onClick` calls `preventDefault` to prevent iframe navigation in Storybook while preserving anchor semantics (hover, copy-link, etc.).

## Usage Example

```typescript
import { ChatColumnDecorator, makeAnchorProps } from './chat-card-decorator'
import type { Meta, StoryObj } from '@storybook/react'
import { TicketCard } from './ticket-card'

const meta: Meta<typeof TicketCard> = {
  component: TicketCard,
  decorators: [
    (Story) => (
      <ChatColumnDecorator>
        <Story />
      </ChatColumnDecorator>
    ),
  ],
}
export default meta

export const Default: StoryObj<typeof TicketCard> = {
  args: {
    title: 'Printer not working',
    anchorProps: makeAnchorProps('https://example.com/tickets/42'),
  },
}
```

> The decorator targets the assistant message column width (`420px`) so card stories reflect their true production layout without spinning up the dispatch pipeline or chat runtime.
