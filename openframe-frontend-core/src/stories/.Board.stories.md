Storybook stories for the `Board` feature component, covering drag-and-drop kanban board layouts used in OpenFrame's ticket management UI.

## Key Components

**Exported Stories**

| Story | Description |
|-------|-------------|
| `Default` | Five canonical-status columns (`ACTIVE`, `TECH_REQUIRED`, `ON_HOLD`, `RESOLVED`, `ARCHIVED`) with full DnD wired up |
| `SystemAndCustom` | Mixed system + custom columns matching the OpenFrame Tickets layout, with drop/drag restrictions |
| `CustomColumns` | Fully custom columns (`Backlog`, `In Review`, `Blocked`, `Done`) decoupled from `TicketStatus` |
| `Collapsible` | Column collapse toggle persisted to `localStorage` via `collapseStorageKey` |
| `Pagination` | Lazy-load pattern — starts at 10 of 50 tickets, appends 10 more per `onLoadMore` with a 600ms simulated delay |
| `InitialLoading` | All columns begin in skeleton/loading state, resolving to seeded data after 1 second |
| `Empty` | Empty columns validating drop placeholder rendering |

**Seed Helpers**

- `baseTicket(i, status, overrides)` — generates a `BoardTicket` from rotating fixture data
- `buildSeedColumns()` — builds the default five-column layout using `columnFromTicketStatus`
- `applyChange(columns, change)` — immutably applies a `BoardChange` (move/reorder) to column state

## Usage Example

```typescript
// Minimal interactive board story pattern
export const MyStory: Story = {
  render: function Render() {
    const [columns, setColumns] = React.useState(buildSeedColumns())
    return (
      <div className="h-[80vh] bg-ods-bg">
        <Board
          columns={columns}
          onChange={c => setColumns(prev => applyChange(prev, c))}
          getTicketHref={id => `#${id}`}
          onAddTicket={id => console.log('add to', id)}
        />
      </div>
    )
  },
}
```
