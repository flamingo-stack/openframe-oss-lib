Storybook stories for the `Board` component, covering drag-and-drop kanban board layouts with various column configurations, loading states, pagination, and collapse behavior.

## Key Components

- **`Default`** — Five canonical ticket-status columns (`ACTIVE`, `TECH_REQUIRED`, `ON_HOLD`, `RESOLVED`, `ARCHIVED`) with full DnD wired up
- **`SystemAndCustom`** — Mixed system + custom columns matching the OpenFrame Tickets layout; AI-Assistance blocks drops, Resolved tickets are drag-disabled
- **`CustomColumns`** — Fully custom columns (`Backlog`, `In Review`, `Blocked`, `Done`), demonstrating column color is decoupled from `TicketStatus`
- **`Collapsible`** — Column collapse toggle that persists via `localStorage` using `collapseStorageKey`
- **`Pagination`** — Single column starts with 10 of 50 tickets; `onLoadMore` appends 10 more after a 600ms simulated delay
- **`InitialLoading`** — All columns render skeleton/loading state, resolving to seeded data after 1 second
- **`Empty`** — Empty columns to verify the drop placeholder UI

## Usage Example

```typescript
// Minimal interactive board with immutable change handling
const [columns, setColumns] = React.useState(buildSeedColumns())

function applyChange(columns: BoardColumnDef[], change: BoardChange): BoardColumnDef[] {
  const next = columns.map(c => ({ ...c, tickets: [...c.tickets] }))
  const fromCol = next.find(c => c.id === change.fromColumnId)
  const toCol   = next.find(c => c.id === change.toColumnId)
  if (!fromCol || !toCol) return columns

  const [moved] = fromCol.tickets.splice(
    fromCol.tickets.findIndex(t => t.id === change.ticketId), 1
  )
  toCol.tickets.splice(0, 0, moved)
  return next
}

<Board
  columns={columns}
  onChange={c => setColumns(prev => applyChange(prev, c))}
  getTicketHref={id => `/tickets/${id}`}
  onAddTicket={colId => console.log('add to', colId)}
/>
```
