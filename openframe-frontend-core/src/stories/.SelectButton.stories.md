Storybook configuration file that defines interactive stories for the SelectButton component, showcasing various states and usage patterns including selectable options with icons, images, and interactive grouping examples.

## Key Components

- **Meta Configuration**: Storybook metadata with controls for `title`, `description`, `selected`, and `disabled` properties
- **Static Stories**: Individual story variants demonstrating different SelectButton states:
  - `Default` - Basic unselected button
  - `Selected` - Selected state button  
  - `WithIcon`/`WithIconSelected` - Buttons with Lucide React icons
  - `WithImage`/`WithImageSelected` - Buttons with placeholder images
  - `NoDescription` - Title-only variant
  - `Disabled` - Non-interactive disabled state
- **Interactive Stories**: Dynamic examples showing real selection behavior:
  - `InteractiveGroup` - Full-featured selectable group with icons and descriptions
  - `InteractiveIconTitleOnly` - Compact selectable group with just icons and titles

## Usage Example

The stories demonstrate how SelectButton can be used in different scenarios:

```typescript
// Static display (from stories)
<SelectButton
  title="API"
  description="REST endpoints"
  icon={<Zap size={24} />}
  selected={true}
/>

// Interactive group selection (from InteractiveGroup story)
const [selectedId, setSelectedId] = useState<string | null>('monitor')
const options = [
  { id: 'monitor', title: 'Monitor', description: 'System monitoring', icon: <Monitor size={24} /> },
  { id: 'web', title: 'Web App', description: 'Browser based', icon: <Globe size={24} /> }
]

// Render with selection state management
{options.map((option) => (
  <SelectButton
    key={option.id}
    title={option.title}
    description={option.description}
    icon={option.icon}
    selected={selectedId === option.id}
    onClick={() => setSelectedId(option.id)}
  />
))}
```
