Storybook stories for the `PageActions` UI component, covering all layout variants, action states, and responsive behaviors.

## Key Components

**Exported Stories:**
- `IconButtons` — Icon-labeled buttons that collapse to a dropdown menu on mobile
- `PrimaryButtons` / `PrimaryButtonsWithIcons` / `SinglePrimaryButton` — Accent + outline button combos for form/detail pages
- `SingleIconButton` — Single action rendered as icon (not collapsed to menu) on mobile
- `ThreeIconButtons` — Multiple icon actions
- `WithStates` — Loading and disabled button states
- `MenuPrimary` / `MenuPrimarySingleAction` / `MenuPrimaryLoading` — `ActionsMenuDropdown ("...")` + primary button pattern
- `MobileOnlyActions` — Actions hidden on desktop via `showOnlyMobile`
- `LinkActions` — Actions rendered as Next.js `<Link>` using `href`
- `SplitButtonWithMenu` — Primary button with chevron submenu alongside `menuActions`
- `SplitButtonTwoActions` — Two independently clickable halves via `iconAction`

**Variants covered:** `icon-buttons`, `primary-buttons`, `menu-primary`

## Usage Example

```typescript
// Basic icon-buttons variant
<PageActions
  variant="icon-buttons"
  actions={[
    { label: 'Add Script', onClick: handleAdd, icon: <PlusCircleIcon size={24} /> },
    { label: 'Edit Categories', onClick: handleEdit, icon: <ColorsIcon size={24} /> },
  ]}
/>

// menu-primary with split button and submenu
<PageActions
  variant="menu-primary"
  actions={[{
    label: 'Remote Shell',
    variant: 'outline',
    submenu: [
      { id: 'cmd', label: 'CMD', onClick: handleCmd, icon: <TerminalIcon size={24} /> },
    ],
  }]}
  menuActions={[{
    items: [{ id: 'edit', label: 'Edit Device', onClick: handleEdit }],
  }]}
/>
```

Use with `PageContainer`'s `headerActions` prop for consistent page-level layouts.
