Storybook stories for the `PageActions` UI component, covering all layout variants, states, and interaction patterns used in page headers.

## Key Components

### Stories Exported

| Story | Variant | Description |
|-------|---------|-------------|
| `IconButtons` | `icon-buttons` | Icon buttons, collapses to dropdown on mobile |
| `PrimaryButtons` | `primary-buttons` | Accent + outline buttons, fixed bottom bar on mobile |
| `PrimaryButtonsWithIcons` | `primary-buttons` | Primary buttons with leading icons |
| `SinglePrimaryButton` | `primary-buttons` | Single accent CTA |
| `SingleIconButton` | `icon-buttons` | Single icon stays visible on mobile (no collapse) |
| `ThreeIconButtons` | `icon-buttons` | Three-action icon row |
| `WithStates` | `primary-buttons` | Loading and disabled button states |
| `MenuPrimary` | `menu-primary` | `"..."` menu + primary button; full bar on mobile |
| `MenuPrimaryLoading` | `menu-primary` | Primary button in loading state |
| `MobileOnlyActions` | `icon-buttons` | Actions hidden on desktop via `showOnlyMobile` |
| `LinkActions` | `menu-primary` | Actions rendered as Next.js `<Link>` via `href` |
| `SplitButtonWithMenu` | `menu-primary` | Chevron dropdown for alternative actions |
| `SplitButtonTwoActions` | `menu-primary` | Two independent click targets via `iconAction` |

### `PageActions` Props Covered

- `variant` — `'icon-buttons'` | `'primary-buttons'` | `'menu-primary'`
- `actions` — primary action buttons (supports `onClick`, `href`, `icon`, `submenu`, `iconAction`, `loading`, `disabled`, `showOnlyMobile`)
- `menuActions` — grouped items for the `"..."` overflow dropdown
- `className` — additional CSS classes

## Usage Example

```typescript
import { PageActions } from '../components/ui/page-actions'
import { PlusCircleIcon } from '../components/icons-v2-generated'

<PageActions
  variant="menu-primary"
  actions={[
    {
      label: 'Run Script',
      onClick: handleRun,
      variant: 'accent',
      icon: <PlayCircleIcon size={24} />,
    },
  ]}
  menuActions={[
    {
      items: [
        { id: 'edit', label: 'Edit Script', onClick: handleEdit },
        { id: 'delete', label: 'Delete', onClick: handleDelete },
      ],
    },
  ]}
/>
```

> Use with `PageContainer`'s `headerActions` prop for consistent page-level layouts across the Flamingo platform.
