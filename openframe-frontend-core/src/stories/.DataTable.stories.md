Storybook showcase for the `DataTable` component, demonstrating sorting, filtering, row selection, column visibility, and other TanStack Table-powered features through interactive stories.

## Key Components

- **`Basic`** — Minimal setup: data + columns, no extra configuration
- **`WithSorting`** — Client-side/server-side sort state management via `DataTableSortState`
- **`multiSelectFilterFn`** — Column filter function for multi-value filtering (OS, Status)
- **`useDataTable`** — Hook that creates the TanStack Table instance passed to `<DataTable>`
- **`baseColumns()`** — Reusable column factory with responsive `hideAt` breakpoints and filter metadata
- **Cell renderers** — `StatusTag`, `OSBadge`, `OwnerCell`, `TagsCell`, `LoadCell`, `LastSeenCell`
- **Sample data** — `makeDevices(n)` deterministic device generator; pre-sliced sets (`DEVICES_6`–`DEVICES_50`)

## Usage Example

```typescript
import { DataTable, useDataTable, type ColumnDef } from '../components/ui/data-table'

const columns: ColumnDef<Device>[] = [
  { accessorKey: 'hostname', header: 'Hostname', meta: { width: 'w-[200px]' } },
  {
    accessorKey: 'status',
    header: 'Status',
    cell: ({ row }) => <StatusTag status={row.original.status} />,
    meta: { width: 'w-[140px]' },
  },
]

function DeviceTable() {
  const table = useDataTable<Device>({ data, columns })
  return (
    <DataTable table={table}>
      <DataTable.Header />
      <DataTable.Body emptyMessage="No devices" />
    </DataTable>
  )
}
```

> **Note:** `useDataTable` defaults to `manualSorting: true` and `manualFiltering: true` (server-side assumed). Pass `clientSideSorting: true` / `clientSideFiltering: true` for purely in-memory tables. Use `data-no-row-click` on interactive cell wrappers to prevent click events from bubbling to `onRowClick`.
