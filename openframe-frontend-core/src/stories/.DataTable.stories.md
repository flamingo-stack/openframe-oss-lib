Storybook showcase for the `DataTable` component, demonstrating all interactive features (sorting, filtering, row selection, column visibility) built on TanStack Table with a card-row UI.

## Key Components

- **`Basic`** — Minimal setup story: data + columns only, using `useDataTable` defaults
- **`WithSorting`** — Consumer-controlled sort state with manual toggle cycle and client-side data sorting
- **`baseColumns()`** — Reusable column factory covering hostname, IP, OS, status, owner, tags, CPU, memory, and last-seen
- **Cell renderers** — `StatusTag`, `OSBadge`, `OwnerCell`, `TagsCell`, `LoadCell`, `LastSeenCell` — composable display components for each data type
- **`makeDevices()`** — Deterministic fixture generator producing `Device[]` datasets of varying sizes (`DEVICES_6` through `DEVICES_50`)

## Usage Example

```typescript
// Minimal table
const columns = useMemo<ColumnDef<Device>[]>(() => [
  { accessorKey: 'hostname', header: 'Hostname', meta: { width: 'w-[260px]' } },
  { accessorKey: 'status', header: 'Status', cell: ({ row }) => <StatusTag status={row.original.status} />, meta: { width: 'w-[140px]' } },
], [])

const table = useDataTable<Device>({ data: devices, columns })

return (
  <DataTable table={table}>
    <DataTable.Header />
    <DataTable.Body emptyMessage="No devices" />
  </DataTable>
)

// Consumer-owned sorting (manual/server-side)
const [sort, setSort] = useState<DataTableSortState | null>({ id: 'hostname', desc: false })
const handleSortChange = useCallback((columnId: string) => {
  setSort(prev => {
    if (prev?.id !== columnId) return { id: columnId, desc: false }
    if (!prev.desc) return { id: columnId, desc: true }
    return null
  })
}, [])
```

> **Note:** Use `data-no-row-click` on interactive cell wrappers to prevent click events from bubbling to `onRowClick`. Pass `clientSideSorting: true` / `clientSideFiltering: true` to `useDataTable` for non-server-side tables.
