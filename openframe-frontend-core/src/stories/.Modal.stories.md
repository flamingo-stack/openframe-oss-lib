Storybook stories for the `Modal` UI component, demonstrating two usage patterns: a basic modal with header/content/footer, and a modal containing an `Autocomplete` dropdown.

## Key Components

- **`Default`** — Standard modal story with `ModalHeader`, `ModalContent`, and `ModalFooter`, controlled via local `useState`
- **`WithAutocomplete`** — Modal story embedding a multi-select `Autocomplete` component; uses a `useCallback` ref (`modalRef`) to capture the modal DOM node and avoid z-index conflicts with the dropdown portal
- **`meta`** — Storybook metadata registering the component under `UI/Modal` with `isOpen` (boolean) and `className` (text) controls

## Usage Example

```typescript
// Default modal pattern
export const Default: Story = {
  render: function Render(args) {
    const [isOpen, setIsOpen] = useState(args.isOpen)
    return (
      <>
        <Button onClick={() => setIsOpen(true)}>Open Modal</Button>
        <Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
          <ModalHeader>
            <ModalTitle>Modal Title</ModalTitle>
          </ModalHeader>
          <ModalContent className="px-6 py-4">
            <p>Content goes here.</p>
          </ModalContent>
          <ModalFooter>
            <Button variant="accent" onClick={() => setIsOpen(false)}>Confirm</Button>
          </ModalFooter>
        </Modal>
      </>
    )
  },
}

// Capturing modal DOM ref for Autocomplete portal containment
const modalRef = useCallback((node: HTMLDivElement | null) => {
  setModalElement(node)
}, [])

<Modal ref={modalRef} isOpen={isOpen} onClose={() => setIsOpen(false)} className="overflow-visible">

    <Autocomplete multiple options={options} value={selected} onChange={setSelected} />
  </ModalContent>
</Modal>
```

> **Note:** When embedding dropdowns or popovers inside a modal, pass `portalContainer` or capture a modal `ref` to ensure the dropdown renders within the modal's DOM subtree and avoids z-index stacking issues.
