Storybook stories for the `Modal` UI component, demonstrating interactive open/close behavior and composition with sub-components (`ModalHeader`, `ModalContent`, `ModalFooter`).

## Key Components

| Export | Description |
|--------|-------------|
| `Default` | Basic modal with header, body text, Cancel and Confirm footer buttons |
| `WithAutocomplete` | Modal containing a multi-select `Autocomplete` dropdown; uses a `ref`-based `portalContainer` pattern to avoid z-index conflicts |

## Usage Example

```typescript
// Default modal pattern
const [isOpen, setIsOpen] = useState(false)

<Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
  <ModalHeader>
    <ModalTitle>Modal Title</ModalTitle>
  </ModalHeader>
  <ModalContent className="px-6 py-4">
    <p>Content goes here.</p>
  </ModalContent>
  <ModalFooter>
    <Button variant="outline" onClick={() => setIsOpen(false)}>Cancel</Button>
    <Button variant="accent" onClick={() => setIsOpen(false)}>Confirm</Button>
  </ModalFooter>
</Modal>

// Autocomplete inside modal — capture modal DOM node via ref
const [modalElement, setModalElement] = useState<HTMLDivElement | null>(null)
const modalRef = useCallback((node: HTMLDivElement | null) => {
  setModalElement(node)
}, [])

<Modal ref={modalRef} isOpen={isOpen} onClose={...} className="overflow-visible">
  <ModalContent className="px-6 py-4">
    <Autocomplete
      multiple
      options={options}
      value={selected}
      onChange={setSelected}
      portalContainer={modalElement}
    />
  </ModalContent>
</Modal>
```

> **Note:** When rendering dropdowns inside a modal, pass the modal's DOM node as `portalContainer` to the `Autocomplete` component. This keeps the dropdown within the modal's stacking context and prevents z-index conflicts with the backdrop.
