Storybook configuration file that defines interactive stories for the Modal component, demonstrating different use cases including basic modals and modals containing autocomplete functionality.

## Key Components

- **Default Story** - Basic modal with header, content, and footer sections
- **WithAutocomplete Story** - Modal containing an autocomplete dropdown with proper z-index handling
- **Meta Configuration** - Storybook metadata with component controls for `isOpen` and `className` props
- **AutocompleteOptions** - Sample data for industry selection in the autocomplete story

## Usage Example

```typescript
// Basic modal story structure
export const MyModalStory: Story = {
  args: {
    isOpen: true,
    onClose: fn(),
  },
  render: function Render(args) {
    const [isOpen, setIsOpen] = useState(args.isOpen)
    
    return (
      <Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
        <ModalHeader>
          <ModalTitle>Example Modal</ModalTitle>
        </ModalHeader>
        <ModalContent className="px-6 py-4">
          <p>Modal content goes here</p>
        </ModalContent>
        <ModalFooter>
          <Button onClick={() => setIsOpen(false)}>Close</Button>
        </ModalFooter>
      </Modal>
    )
  }
}
```

The stories demonstrate proper modal state management, portal rendering for dropdowns, and responsive design patterns within the Flamingo design system.
