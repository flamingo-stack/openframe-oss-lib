Storybook stories for the `CampaignCardAdmin` chat-inline entity card, covering the `marketing_campaign` doc type with variants for missing description and empty goals.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `Default` | `Story` | Full campaign card with name, start date, goal count, and description |
| `NoDescription` | `Story` | Campaign card with `description: null` — tests graceful omission |
| `NoGoals` | `Story` | Campaign card with an empty `goals` array — tests zero-count rendering |

## Usage Example

```typescript
// Rendered inside ChatColumnDecorator to simulate chat column constraints
export const Default: Story = {
  args: {
    campaign: campaignItem,
    anchorProps: makeAnchorProps('/admin/campaigns/' + campaignItem.id),
  },
}
```

## Notes

- All stories are wrapped in `ChatColumnDecorator` to replicate the narrow chat column layout.
- `makeAnchorProps` generates link props pointing to `/admin/campaigns/:id`.
- This card is the **chat-only projection** — the full admin card with delete/manage actions lives in the hub.
- Fixture data is sourced from `./__fixtures__/chat-cards` (`campaignItem`) and `./__fixtures__/chat-card-decorator`.
