Storybook stories for the `CampaignCardAdmin` chat-inline entity card, covering default display and edge cases where optional campaign fields are absent.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `Default` | `Story` | Full campaign card with name, start date, goal count, and description |
| `NoDescription` | `Story` | Card rendered when `description` is `null` |
| `NoGoals` | `Story` | Card rendered when `goals` is an empty array |

## Usage Example

```typescript
import { CampaignCardAdmin } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { campaignItem } from './__fixtures__/chat-cards'

// Minimal usage
<CampaignCardAdmin
  campaign={campaignItem}
  anchorProps={makeAnchorProps('/admin/campaigns/' + campaignItem.id)}
/>

// With missing optional fields
<CampaignCardAdmin
  campaign={{ ...campaignItem, description: null, goals: [] }}
  anchorProps={makeAnchorProps('/admin/campaigns/' + campaignItem.id)}
/>
```

## Notes

- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout context.
- This card is the **chat-only projection** of the `marketing_campaign` doc type — the full admin card with delete/manage actions lives in the hub.
- Displayed fields: campaign name, start date, goal count, and a one-line description.
