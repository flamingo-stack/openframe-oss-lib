Storybook stories for the `ActionsMenu` and `ActionsMenuDropdown` UI components, covering all supported item variants and interaction patterns.

## Key Components

- **`Default`** — Single group with icon-labeled action items
- **`GroupedWithDanger`** — Multi-group layout with a separator and a destructive (danger) delete row
- **`WithCheckboxes`** — Toggle-style items that maintain a `checked` state without closing the menu
- **`WithSubmenu`** — Nested submenu via `ActionsMenuDropdown` (requires Radix DropdownMenu context)
- **`WithDisabledItem`** — Demonstrates dimmed, non-interactive row behavior
- **`WithSecondaryAction`** — Row with an independent secondary icon button (e.g., "open in new tab")
- **`Dropdown`** — Full `ActionsMenuDropdown` attached to a default ellipsis trigger

## Usage Example

```typescript
import { ActionsMenu, ActionsMenuDropdown } from '../components/ui/actions-menu';
import { PencilIcon, TrashBlankIcon } from '../components/icons-v2-generated';

// Inline panel
<ActionsMenu
  groups={[
    {
      id: 'primary',
      separator: true,
      items: [
        { id: 'edit', label: 'Edit', icon: <PencilIcon />, onClick: () => {} },
      ],
    },
    {
      id: 'destructive',
      items: [
        { id: 'delete', label: 'Delete', icon: <TrashBlankIcon />, danger: true, onClick: () => {} },
      ],
    },
  ]}
/>

// Dropdown (attaches to a trigger button)
<ActionsMenuDropdown
  groups={groups}
  onItemClick={(id) => console.log(id)}
/>
```

## Notes

- Use `ActionsMenu` to render the panel inline; use `ActionsMenuDropdown` to attach it to a trigger
- Submenus require `ActionsMenuDropdown` — they depend on the Radix DropdownMenu context
- `iconAction` on a row renders a separate 40px button on the right with its own `href` or `onClick`
