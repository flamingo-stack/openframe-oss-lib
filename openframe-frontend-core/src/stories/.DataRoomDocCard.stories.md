Storybook stories for the `DataRoomDocCard` chat entity card component, covering both `data_room_doc` and `markdown` doc types rendered inline in a chat column layout.

## Key Components

| Export | Type | Description |
|---|---|---|
| `DataRoom` | `Story` | Card variant for data room documents with "Data room" badge |
| `KnowledgeBaseMarkdown` | `Story` | Card variant for OpenFrame Docs markdown files with "OpenFrame Docs" badge |

## Usage Example

```typescript
import { DataRoomDocCard } from '../components/chat/entity-cards'

// badgeText is REQUIRED — resolved via SOURCE_LABELS_BY_TABLE in dispatch
<DataRoomDocCard
  item={dataRoomDocItem}
  badgeText="Data room"
  anchorProps={makeAnchorProps(dataRoomDocItem.url ?? '#')}
/>

// Markdown doc type — same component, different badge driven by sourceRepo
<DataRoomDocCard
  item={openframeDocsMarkdownItem}
  badgeText="OpenFrame Docs"
  anchorProps={makeAnchorProps(openframeDocsMarkdownItem.url ?? '#')}
/>
```

## Notes

- `sourceRepo` on the item drives the badge label (e.g. `"openframe-docs"` → `"OpenFrame Docs"`)
- `badgeText` must be pre-resolved by the caller via `SOURCE_LABELS_BY_TABLE` before passing to the component — it is not derived internally
- Both stories use `ChatColumnDecorator` to simulate the constrained chat column width context
