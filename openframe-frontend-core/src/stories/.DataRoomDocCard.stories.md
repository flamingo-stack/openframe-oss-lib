Storybook stories for `DataRoomDocCard`, an inline chat entity card that renders `data_room_doc` and `markdown` document types. The same component handles both variants — the `sourceRepo` field drives the badge label while `badgeText` is a required resolved prop.

## Key Components

| Export | Description |
|--------|-------------|
| `DataRoom` | Story showing the card with a Data Room document item and `"Data room"` badge |
| `KnowledgeBaseMarkdown` | Story showing the card with an OpenFrame Docs markdown item and `"OpenFrame Docs"` badge |

## Usage Example

```typescript
// Rendering the card directly
<DataRoomDocCard
  item={dataRoomDocItem}
  badgeText="Data room"          // resolved via SOURCE_LABELS_BY_TABLE before render
  anchorProps={makeAnchorProps(dataRoomDocItem.url ?? '#')}
/>

// Markdown / OpenFrame Docs variant
<DataRoomDocCard
  item={openframeDocsMarkdownItem}
  badgeText="OpenFrame Docs"     // driven by sourceRepo: "openframe-docs"
  anchorProps={makeAnchorProps(openframeDocsMarkdownItem.url ?? '#')}
/>
```

## Notes

- `badgeText` is **required** — callers must resolve it (e.g. via `SOURCE_LABELS_BY_TABLE`) before passing it to the component; it is not derived internally.
- Both stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout.
- Story path: `Chat → EntityCards → DataRoomDocCard`
