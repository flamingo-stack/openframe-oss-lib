Storybook stories for the `SlackMessageCard` component, covering both display variants used when rendering Slack message references inside chat.

## Key Components

- **`Compact`** — Story rendering the card in `compact` variant, used inline within chat messages
- **`RowVariant`** — Story rendering the card in `row` variant, used in list/search result contexts
- **`ChatColumnDecorator`** — Wraps stories in a chat-column layout to simulate real rendering context
- **`makeAnchorProps`** — Helper that generates link props from the item's URL

## Usage Example

```typescript
// Render the compact variant manually
import { SlackMessageCard } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { slackMessageItem } from './__fixtures__/chat-cards'

<SlackMessageCard
  item={slackMessageItem}
  variant="compact"
  anchorProps={makeAnchorProps(slackMessageItem.url ?? '#')}
/>

// Row variant for list contexts
<SlackMessageCard
  item={slackMessageItem}
  variant="row"
  anchorProps={makeAnchorProps(slackMessageItem.url ?? '#')}
/>
```

## Notes

- Maps to the `slack_message` doc type surfaced by Mingo AI in chat
- Previews are **PII-sanitized server-side** before reaching this component — no client-side scrubbing needed
- Pure presentation component; no data fetching or side effects
- Falls back to `'#'` if `item.url` is nullish
