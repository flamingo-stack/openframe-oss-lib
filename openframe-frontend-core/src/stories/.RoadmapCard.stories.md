Storybook stories for the `RoadmapCard` component, covering all three roadmap doc types (`roadmap_item`, `delivery_item`, `internal_task`) in both `sm` (chat-inline) and `default` (full rich card) size variants.

## Key Components

- **`RoadmapItemSm`** — Compact inline card for a roadmap feature item, used inside chat responses
- **`DeliveryItemSm`** — Compact inline card for a delivery/release item
- **`InternalTaskSm`** — Compact inline card for an internal task
- **`RoadmapItemDefault`** — Full rich card with vote buttons, screenshot gallery, status pill, and target version; used on the public roadmap page

## Usage Example

```typescript
import { RoadmapCard } from '../components/chat/entity-cards'

// Chat-inline compact variant
<RoadmapCard
  item={roadmapFeatureItem}
  href={`/roadmap/${roadmapFeatureItem.id}`}
  size="sm"
  cardType="roadmap_item"
  target="_blank"
  rel="noopener noreferrer"
/>

// Full rich card (public roadmap page)
<RoadmapCard
  item={roadmapFeatureItem}
  size="default"
  cardType="roadmap_item"
/>
```

## Notes

- The `cardType` prop controls the icon rendered in the `sm` variant (`Feature` / `Delivery` / `Bug`-shaped task)
- All `sm` stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout
- In the `default` size, vote handlers are real; in chat (`sm`) they are stubbed
- Fixture data is sourced from `./__fixtures__/chat-cards`
