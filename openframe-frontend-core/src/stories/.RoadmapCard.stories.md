Storybook stories for `RoadmapCard`, covering all three roadmap doc types (`roadmap_item`, `delivery_item`, `internal_task`) in both `sm` (chat-inline) and `default` (full rich card) sizes.

## Key Components

| Export | Description |
|---|---|
| `RoadmapItemSm` | Chat-inline card for a feature roadmap item with external link |
| `DeliveryItemSm` | Chat-inline card for a delivery item with external link |
| `InternalTaskSm` | Chat-inline card for an internal task with external link |
| `RoadmapItemDefault` | Full rich card with vote buttons, screenshot gallery, status pill, and target version |

## Usage Example

```typescript
// Rendering RoadmapCard in sm (chat-inline) variant
<RoadmapCard
  item={roadmapFeatureItem}
  href={'/roadmap/' + roadmapFeatureItem.id}
  size="sm"
  cardType="roadmap_item"
  target="_blank"
  rel="noopener noreferrer"
/>

// Rendering RoadmapCard in full default variant
<RoadmapCard
  item={roadmapFeatureItem}
  size="default"
  cardType="roadmap_item"
/>
```

## Notes

- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout context.
- The `cardType` prop controls the icon displayed in the `sm` variant — `roadmap_item` (Feature), `delivery_item` (Delivery), `internal_task` (Bug-shaped task).
- In the `default` size, vote handlers are active; in the `sm` chat variant they are stubbed.
- Fixture data (`roadmapFeatureItem`, `roadmapDeliveryItem`, `roadmapInternalTaskItem`) is sourced from `./__fixtures__/chat-cards`.
