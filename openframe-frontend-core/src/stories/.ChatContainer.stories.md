Storybook stories for the `ChatContainer` component, demonstrating the Fae client-facing chat shell across four layout and feature variants.

## Key Components

- **`FaeChatShell`** — Interactive story shell that wires up controlled state for `ChatInput` and `ChatMessageList`, allowing message sending (user bubbles only, no assistant reply transport)
- **`FAE_MESSAGES`** — Mock conversation seed data simulating a Fae/client printer-support exchange
- **`Fae`** — Default story: standard centered layout with Fae persona header
- **`FaeWithTicket`** — Story with `ticketInfo` populated on the header, showing active ticket title, meta, and status badge
- **`FaeFullWidth`** — Full-width variant for narrow side panels or slide-in drawers
- **`FaeBareHeader`** — Headerless-chrome variant for embedding inside a host-supplied card or drawer

## Usage Example

```typescript
// Render the default Fae chat shell in a story
export const Fae: Story = {
  render: () => <FaeChatShell />,
}

// With an active ticket displayed in the header
export const FaeWithTicket: Story = {
  render: () => <FaeChatShell withTicketInfo />,
}

// Full-width layout for narrow containers (mobile / drawer)
export const FaeFullWidth: Story = {
  render: () => <FaeChatShell fullWidth />,
}

// Bare header — no card chrome, flush with host container
export const FaeBareHeader: Story = {
  render: () => <FaeChatShell bare fullWidth />,
}
```

### `FaeChatShell` Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `withTicketInfo` | `boolean` | `false` | Adds ticket title, meta, and status to the header |
| `bare` | `boolean` | `false` | Removes card chrome (background, border, shadow) from the header |
| `fullWidth` | `boolean` | `false` | Drops the centered 600px column constraint on header, messages, and footer |
