Storybook stories for the `CaseStudyCard` component, covering both inline chat (`sm`) and catalog grid (`default`) size variants.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `meta` | `Meta<CaseStudyCard>` | Storybook metadata — registers component under `Chat/EntityCards/CaseStudyCard` with `fullscreen` layout |
| `ChatInlineSm` | `Story` | Compact card variant used inline within chat messages (`size: 'sm'`) |
| `Default` | `Story` | Full catalog grid card variant (`size: 'default'`) |

## Usage Example

```typescript
// Render the compact chat-inline variant
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
/>

// Render the catalog grid variant
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  size="default"
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- Both stories share the same `caseStudy` fixture from `__fixtures__/chat-cards` and are wrapped in `ChatColumnDecorator` to simulate the chat column layout context.
- The `sm` size renders a compact inline card; `default` renders the full catalog grid card — both display the user + MSP badge overlay.
- Component lives at `Chat/EntityCards/CaseStudyCard` in the Storybook hierarchy.
