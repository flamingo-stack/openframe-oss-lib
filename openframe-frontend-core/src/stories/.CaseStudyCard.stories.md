Storybook stories for the `CaseStudyCard` entity card component, covering both inline chat (`sm`) and catalog grid (`default`) size variants.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `meta` | `Meta<CaseStudyCard>` | Storybook metadata — mounts under `Chat/EntityCards/CaseStudyCard`, wrapped in `ChatColumnDecorator` |
| `ChatInlineSm` | `Story` | Compact card variant rendered inline within chat messages |
| `Default` | `Story` | Full-size catalog grid card variant |

## Usage Example

```typescript
// Render the compact chat-inline card
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
/>

// Render the catalog grid card
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  size="default"
  target="_blank"
  rel="noopener noreferrer"
/>
```

## Notes

- Both stories use the shared `caseStudy` fixture from `__fixtures__/chat-cards`
- `ChatColumnDecorator` constrains the story viewport to simulate a realistic chat column width
- The `size` prop is the only differentiator between the two stories — `"sm"` for inline chat, `"default"` for grid catalog
- User + MSP badge overlay is rendered in both size variants per the component spec
