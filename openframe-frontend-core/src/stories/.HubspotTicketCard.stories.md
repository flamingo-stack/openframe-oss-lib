Storybook stories for the `HubspotTicketCard` chat entity card, covering all three doc type variants (`hubspot_ticket`, `hubspot_ticket_anon`, `hubspot_ticket_self`) and both layout modes.

## Key Components

| Export | Description |
|---|---|
| `Full` | Complete ticket card with customer company, clickable URL, and anchor props |
| `Anon` | Anonymous variant with `customerCompany` and `url` nulled out server-side |
| `SelfTicket` | Ticket owned by the chatting user; admin URL stripped, status set to `NEW` |
| `ClosedTicket` | Closed/low-priority ticket with anchor navigation enabled |
| `RowVariant` | Same full data rendered in `row` layout instead of `compact` |

## Usage Example

```typescript
import { HubspotTicketCard } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { hubspotTicketItem } from './__fixtures__/chat-cards'

// Full (linked) variant
<HubspotTicketCard
  item={hubspotTicketItem}
  variant="compact"
  anchorProps={makeAnchorProps(hubspotTicketItem.url ?? '#')}
/>

// Anonymous variant — server redacts these fields before sending
<HubspotTicketCard
  item={{ ...hubspotTicketItem, customerCompany: null, url: null }}
  variant="compact"
/>
```

## Notes

- The same `HubspotTicketCard` component handles all three doc types; field redaction (`customerCompany`, `url`) is done server-side before the payload reaches the client.
- `status` and `priority` values drive badge styling — stories cover `NEW`, `CLOSED`, `MEDIUM`, and `LOW` states.
- All stories are wrapped in `ChatColumnDecorator` to simulate the chat column width constraint.
