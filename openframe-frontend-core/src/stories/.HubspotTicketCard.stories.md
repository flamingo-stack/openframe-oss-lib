Storybook stories for the `HubspotTicketCard` chat entity card, covering the three server-side doc types (`hubspot_ticket`, `hubspot_ticket_anon`, `hubspot_ticket_self`) and both visual variants.

## Key Components

| Export | Description |
|--------|-------------|
| `Full` | Complete ticket card with company, URL, and anchor navigation |
| `Anon` | Anonymous variant — `customerCompany` and `url` nulled server-side, renders non-clickable shell |
| `SelfTicket` | User's own ticket — admin URL stripped, status `NEW`, priority `MEDIUM` |
| `ClosedTicket` | `CLOSED` / `LOW` priority state with clickable anchor |
| `RowVariant` | Same full data rendered in `row` layout instead of `compact` |

## Usage Example

```typescript
import { HubspotTicketCard } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { hubspotTicketItem } from './__fixtures__/chat-cards'

// Full ticket (technician view)
<HubspotTicketCard
  item={hubspotTicketItem}
  variant="compact"
  anchorProps={makeAnchorProps(hubspotTicketItem.url ?? '#')}
/>

// Anonymous variant (server redacts PII before sending)
<HubspotTicketCard
  item={{ ...hubspotTicketItem, customerCompany: null, url: null }}
  variant="compact"
/>

// Row layout
<HubspotTicketCard
  item={hubspotTicketItem}
  variant="row"
  anchorProps={makeAnchorProps(hubspotTicketItem.url ?? '#')}
/>
```

## Notes

- All stories use `ChatColumnDecorator` to simulate the chat column width context.
- The single `HubspotTicketCard` component handles all three doc types — redaction of `customerCompany` (anon) and `url` (anon + self) happens server-side before the payload reaches the card.
- Status/priority badge appearance is driven by the `status` / `statusLabel` / `priority` fields on the item.
