Storybook stories for the `TicketInfoSection` component, covering collapsed, expanded, interactive, and minimal states of the collapsible ticket info bar.

## Key Components

- **`Collapsed`** — Default header-only view showing organization, user, device, and status fields without expanded content
- **`Expanded`** — Fully populated state including assignee selector, description, attachments, tags, and notes
- **`Interactive`** — Stateful story using `useState` to demonstrate real expand/collapse toggling with all note CRUD handlers wired
- **`Minimal`** — Expanded state with all fields set to "Unassigned", useful for empty/default data validation

## Usage Example

```typescript
// Interactive expand/collapse with full data
import { TicketInfoSection } from '../components/ui/ticket-info-section';
import { useState } from 'react';
import { Monitor } from 'lucide-react';

function Example() {
  const [expanded, setExpanded] = useState(false);

  return (
    <TicketInfoSection
      organization={{ name: 'Acme Corp' }}
      user="John Doe"
      device={{ name: 'WIN-SERVER-01', icon: <Monitor className="size-4" /> }}
      status="ACTIVE"
      expanded={expanded}
      onExpand={() => setExpanded(prev => !prev)}
      assigned={{
        currentAssignee: { id: 'u1', name: 'Roman Smith' },
        options: [{ value: 'u1', label: 'Roman Smith' }],
        onAssign={(userId) => console.log('Assigned to', userId)}
      }}
      createdAt="2025/07/12, 22:27"
      description="Ticket description here."
      tags={['urgent', 'production']}
      notes={[]}
      onAddNote={(text) => console.log('New note:', text)}
      onEditNote={(id) => console.log('Edit:', id)}
      onDeleteNote={(id) => console.log('Delete:', id)}
    />
  );
}
```

> **Note:** The `Interactive` story is the primary reference for wiring expand/collapse state — all other stories use static `expanded` props for snapshot/visual testing purposes.
