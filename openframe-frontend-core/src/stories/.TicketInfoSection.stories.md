Storybook stories for the `TicketInfoSection` component, covering collapsed, expanded, interactive, and minimal states of the collapsible ticket info bar.

## Key Components

- **`Collapsed`** — Default header-only view showing organization, user, device, and status fields
- **`Expanded`** — Full view with all sections populated: assignee, description, attachments, tags, notes, and knowledge base articles
- **`Interactive`** — Stateful story demonstrating live expand/collapse toggling via `useState`
- **`Minimal`** — Expanded state with no optional data, showing the bare assignee selector and timestamp

## Usage Example

```typescript
import { TicketInfoSection } from '../components/ui/ticket-info-section';

// Collapsed header bar
<TicketInfoSection
  organization={{ name: 'Acme Corp' }}
  user="John Doe"
  device={{ name: 'WIN-SERVER-01', icon: <Monitor className="size-4" /> }}
  status="ACTIVE"
  expanded={false}
  onExpand={() => setExpanded(prev => !prev)}
/>

// Expanded with notes and attachments
<TicketInfoSection
  organization={{ name: 'Acme Corp' }}
  user="John Doe"
  device={{ name: 'WIN-SERVER-01' }}
  status="TECH_REQUIRED"
  expanded={true}
  description="Ticket description with **markdown** support."
  tags={['Production', 'High-Risk']}
  notes={notes}
  attachments={attachments}
  onAddNote={(text) => handleAdd(text)}
  onEditNote={(id) => handleEdit(id)}
  onDeleteNote={(id) => handleDelete(id)}
/>
```

> **Note:** The component renders at `900px` width in Storybook via a decorator. The `Interactive` story demonstrates the recommended pattern for controlling `expanded` state externally using `onExpand`.
