Storybook stories for the `ChatTicketList` component, providing visual test cases covering ticket list rendering, single items, status variants, and scrollable overflow states with fade effects.

## Key Components

- **`meta`** — Storybook metadata registering `ChatTicketList` under the `Chat/ChatTicketList` category with `ticketClicked` action logging
- **`mockTickets`** — 5-item dataset covering all ticket statuses: `ACTIVE`, `ON_HOLD`, `TECH_REQUIRED`, `RESOLVED`
- **`manyTickets`** — 15-item dataset extending `mockTickets` for scroll/overflow scenarios
- **`constrainedDecorator`** — Factory function returning a story decorator that wraps the component in a constrained `maxWidth: 600` container with an optional fixed height

## Exported Stories

| Story | Description |
|---|---|
| `Default` | Renders all 5 mock tickets with no height constraint |
| `SingleTicket` | Renders only the first ticket |
| `AllStatuses` | First 4 tickets, one per status variant |
| `ScrollFading` | 15 tickets in a `400px` height container showing scroll fade |
| `ScrollFadingCompact` | 15 tickets in a `280px` height container for pronounced fade effect |
