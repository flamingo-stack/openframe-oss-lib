Storybook stories for the `ChatTicketList` component, providing visual test cases covering ticket list rendering across different data sets, ticket counts, and scroll/fade behaviors.

## Key Components

- **`meta`** — Storybook metadata registering `ChatTicketList` under the `Chat/ChatTicketList` title with `onTicketClick` action logging
- **`mockTickets`** — Dataset of 5 tickets covering all status variants: `ACTIVE`, `ON_HOLD`, `TECH_REQUIRED`, and `RESOLVED`
- **`manyTickets`** — Extended 15-ticket dataset for testing scroll and fade UI behavior
- **`constrainedDecorator`** — Factory that wraps stories in a max-width 600px container with an optional fixed height to simulate real chat panel constraints

## Stories

| Story | Description |
|---|---|
| `Default` | 5 tickets, unconstrained height |
| `SingleTicket` | Single ticket rendering |
| `AllStatuses` | First 4 tickets showing all 4 status types |
| `ScrollFading` | 15 tickets in a 400px container; tests top/bottom fade scroll indicators |
| `ScrollFadingCompact` | 15 tickets in a 280px container; exaggerates fade visibility |

## Usage Example

```typescript
// Reuse mock data in other stories or tests
import { Default, ScrollFading } from "./ChatTicketList.stories";

// Render the default list story
<Default />

// Simulate a constrained panel with scroll fading
<ScrollFading />
```
