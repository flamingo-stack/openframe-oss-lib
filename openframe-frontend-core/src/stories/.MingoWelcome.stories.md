Storybook stories for the `MingoWelcome` chat empty-state component, covering layout variants, user personalization, and Guide Chat feature toggling.

## Key Components

| Export | Description |
|--------|-------------|
| `Default` | Full empty state with Guide CTA promo card and yellow chip wired |
| `WithUserName` | Personalized greeting with a first name (e.g., "Grace") |
| `WithoutGuide` | Guide mode suppressed — promo card and "Start Guide Chat" chip hidden |
| `WithQuickActions` | Extra quick-action chips appended after the built-in Guide chip |
| `ReturningUser` | Returning user state — hides "New to OpenFrame?" promo, renders muted chip + dialog history |
| `Wide` | Wide panel layout (960px) simulating desktop/in-layout rendering |

## Usage Example

```typescript
import { MingoWelcome } from '../components/chat/mingo-welcome'

// Basic usage with Guide Chat wired
<MingoWelcome
  userName="Grace"
  onStartGuideChat={() => console.log('switch to guide mode')}
  quickActions={[
    { id: 'weekly', label: 'Weekly Log Summary' },
  ]}
/>

// Returning user with existing chat history
<MingoWelcome
  hasExistingChats={true}
  onStartGuideChat={() => console.log('switch to guide mode')}
  dialogHistory={<MingoChatHistory dialogs={[...]} />}
/>
```

## Notes

- Decorator wraps each story in a `440×760px` panel (mimicking the drawer context) so `flex-1` centring and pinned grid/chips render correctly; the `Wide` story overrides this to `960px`.
- `hasExistingChats` is the only boolean arg exposed via Storybook controls — all other props disable controls to prevent unintended overrides.
- Corresponds to Figma node **`7532:222444`**.
