Storybook stories for the `MingoWelcome` component, covering the chat empty-state UI across its key display variants and configurations.

## Key Components

| Export | Description |
|--------|-------------|
| `Default` | Full welcome state with Guide CTA (promo card + chip) wired |
| `WithUserName` | Personalised greeting with a first name passed via `userName` |
| `WithoutGuide` | Guide mode disabled — promo card and "Start Guide Chat" chip suppressed |
| `WithQuickActions` | Additional quick-action chips appended after the built-in Guide chip |
| `ReturningUser` | Returning user state (`hasExistingChats: true`) — hides promo notification, mutes Guide chip, shows dialog history |
| `Wide` | Wide panel layout (960px) for desktop/in-layout rendering |

## Usage Example

```typescript
import { MingoWelcome } from '../components/chat/mingo-welcome'

// First-time user with Guide CTA
<MingoWelcome
  onStartGuideChat={() => switchToGuideMode()}
/>

// Returning user with chat history and quick actions
<MingoWelcome
  userName="Grace"
  hasExistingChats={true}
  dialogHistory={<MingoChatHistory dialogs={[...]} />}
  onStartGuideChat={() => switchToGuideMode()}
  quickActions={[
    { id: 'weekly', label: 'Weekly Log Summary' },
  ]}
/>
```

## Notes

- All stories render inside a `440×760px` constrained flex container mimicking the narrow drawer panel; the `Wide` story overrides this to `960px`
- `SAMPLE_HISTORY` provides a reusable `MingoChatHistory` fixture with 6 sample dialogs for the `ReturningUser` story
