Storybook stories file that demonstrates various configurations and states of the Input component, including different adornment combinations, disabled/invalid states, and preview modes.

## Key Components

- **meta**: Storybook metadata defining the component path and configuration
- **Story exports**: Individual story variants showcasing different Input component states:
  - `Default` - Basic input without adornments
  - `WithEndAdornmentText` - Input with text suffix (e.g., "Seconds")
  - `WithStartAdornmentIcon` - Input with leading icon
  - `WithBothAdornments` - Input with both start and end adornments
  - `WithEndAdornmentIcon` - Input with trailing icon
  - `DisabledWithAdornments` - Disabled state with adornments
  - `InvalidWithAdornments` - Error state with adornments
  - `WithValueAndEndAdornment` - Pre-filled input with suffix
  - `NumberWithUnit` - Numeric input with unit display
  - `Preview` - Non-interactive display mode
  - `PreviewWithBothAdornments` - Preview with start/end adornments
  - `AllVariants` - Comparison view of multiple states

## Usage Example

```typescript
// Import stories in Storybook
import { Default, WithEndAdornmentText } from './Input.stories';

// Each story demonstrates specific Input configurations
export const MyStory = {
  args: {
    placeholder: 'Custom placeholder',
    disabled: true,
    className: 'max-w-md'
  }
};
```

The stories cover the full range of Input component functionality including adornments, validation states, preview modes, and accessibility considerations for comprehensive UI testing and documentation.
