A Storybook stories file that defines interactive component demos for the OSTypeBadgeGroup component, showcasing different configurations for displaying multiple OS platform badges in a horizontal group.

## Key Components

- **Default Meta Configuration**: Sets up Storybook metadata with component title, parameters, and control definitions
- **Story Variants**: Multiple story configurations demonstrating different use cases:
  - `Default` - Standard three-OS display (Windows, macOS, Linux)
  - `WindowsAndMac` - Two-platform configuration
  - `SingleOS` - Single platform display
  - `WithAliases` - OS name alias handling (Darwin → macOS, win32 → Windows)
  - `WithMaxDisplay` - Truncation with "+N more" indicator
  - `SmallIcons`/`LargeIcons` - Icon size variations
  - `Empty` - Edge case handling
  - `Comparison` - Side-by-side configuration comparison

## Usage Example

```typescript
// Run Storybook to view component variations
npm run storybook

// Stories demonstrate these configurations:
<OSTypeBadgeGroup osTypes={['windows', 'darwin', 'linux']} />
<OSTypeBadgeGroup osTypes={['ubuntu', 'fedora']} maxDisplay={2} />
<OSTypeBadgeGroup osTypes={['Darwin', 'win32']} iconSize="w-4 h-4" />
```

The stories file provides comprehensive testing scenarios for the OS badge group component, including platform compatibility, truncation behavior, icon sizing, and alias normalization. Each story includes descriptive comments and demonstrates real-world usage patterns for multi-platform script documentation.
