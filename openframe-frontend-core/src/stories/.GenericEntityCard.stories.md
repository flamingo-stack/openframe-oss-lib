Storybook stories for `GenericEntityCard`, a fallback chat entity card renderer covering financial document types that don't require bespoke UI treatment.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `meta` | `Meta` | Storybook configuration for `Chat/EntityCards/GenericEntityCard` |
| `FinancialKpi` | `Story` | Renders a financial KPI card |
| `CapTable` | `Story` | Renders a cap table card |
| `ProfitLoss` | `Story` | Renders a profit & loss card |
| `BalanceSheet` | `Story` | Renders a balance sheet card |
| `CashFlow` | `Story` | Renders a cash flow card |

## Design Notes

- **Decorator:** All stories are wrapped in `ChatColumnDecorator` to simulate the chat column layout context.
- **Anchor props:** Each story uses `makeAnchorProps` to wire up navigation from the card's URL.
- **Supported doc types:** `financial_kpi`, `cap_table`, `profit_loss`, `balance_sheet`, `cash_flow` — all driven by a shared `facts` key-value strip and a `badge` colored chip.

## Usage Example

```typescript
import { GenericEntityCard } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { financialKpiItem } from './__fixtures__/chat-cards'

<GenericEntityCard
  item={financialKpiItem}
  anchorProps={makeAnchorProps(financialKpiItem.url ?? '#')}
/>
```

> **Note:** `GenericEntityCard` acts as a catch-all renderer. Prefer bespoke card components for doc types with richer display requirements; use this component when a `facts` array + `badge` chip is sufficient.
