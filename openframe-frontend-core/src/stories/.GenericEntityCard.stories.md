Storybook stories for `GenericEntityCard`, the fallback chat entity card renderer used for financial document types that don't require a bespoke display component.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `meta` | `Meta` | Storybook configuration — mounts under `Chat/EntityCards/GenericEntityCard`, wrapped in `ChatColumnDecorator` |
| `FinancialKpi` | `Story` | Renders a financial KPI card |
| `CapTable` | `Story` | Renders a cap table card |
| `ProfitLoss` | `Story` | Renders a profit & loss card |
| `BalanceSheet` | `Story` | Renders a balance sheet card |
| `CashFlow` | `Story` | Renders a cash flow card |

## Covered Doc Types

`GenericEntityCard` acts as a catch-all renderer for these document types:
- `financial_kpi`
- `cap_table`
- `profit_loss`
- `balance_sheet`
- `cash_flow`

Each story passes a fixture `item` and anchor props generated by `makeAnchorProps`, which simulates the clickable link behavior used in the chat column.

## Usage Example

```typescript
import { GenericEntityCard } from '../components/chat/entity-cards'
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'
import { financialKpiItem } from './__fixtures__/chat-cards'

// Minimal render — mirrors the FinancialKpi story
<GenericEntityCard
  item={financialKpiItem}
  anchorProps={makeAnchorProps(financialKpiItem.url ?? '#')}
/>
```

> The `facts` array on each `item` drives the key-value strip at the bottom of the card; the `badge` field controls the colored chip rendered on the right side.
