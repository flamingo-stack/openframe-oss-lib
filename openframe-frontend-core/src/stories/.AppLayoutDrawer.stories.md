Storybook stories for `AppLayoutDrawer`, demonstrating an in-layout drawer component that renders within `AppLayout`'s main content area without covering the sidebar or header.

## Key Components

- **`Right`** — Default right-side drawer; persistent (only closes via X button or Escape, not by clicking app chrome)
- **`ResizableRight`** — Right-side drawer with drag-to-resize handle; clamped to the main content area bounds, opens by default
- **`Left`** — Left-side drawer variant, suitable for filters or secondary navigation panels

## Usage Example

```typescript
import { useState } from 'react'
import {
  AppLayout,
  AppLayoutDrawer,
  AppLayoutDrawerBody,
  AppLayoutDrawerContent,
  AppLayoutDrawerHeader,
  AppLayoutDrawerTitle,
} from '../components/navigation'

function MyPage() {
  const [open, setOpen] = useState(false)

  return (
    <AppLayout
      sidebarConfig={{ items: navigationItems, onNavigate: () => {} }}
      drawer={
        <AppLayoutDrawer open={open} onOpenChange={setOpen}>
          {/* Resizable right-side panel */}
          <AppLayoutDrawerContent
            side="right"
            resizable
            minSize={360}
            defaultSize={560}
            storageKey="my-app:drawer:right"
          >
            <AppLayoutDrawerHeader>
              <AppLayoutDrawerTitle>Chat</AppLayoutDrawerTitle>
            </AppLayoutDrawerHeader>
            <AppLayoutDrawerBody>
              <p>Panel content here.</p>
            </AppLayoutDrawerBody>
          </AppLayoutDrawerContent>
        </AppLayoutDrawer>
      }
    >
      <button onClick={() => setOpen(true)}>Open Drawer</button>
    </AppLayout>
  )
}
```

> **Note:** Use `AppLayoutDrawer` for page-scoped panels (chat, details, filters). For app-level overlays that should cover the full viewport (e.g. auth modals, notifications), use the standard `Drawer` from `components/ui` instead.
