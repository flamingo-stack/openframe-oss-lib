Storybook stories file for `MingoInfoCard` chat entity cards, providing visual test cases for each migrated `CHAT_CARD_REGISTRY` type rendered inside a simulated chat column layout.

## Key Components

- **`meta`** — Storybook metadata config; sets title `Chat/ChatMingoInfoCards`, wraps all stories in `ChatColumnDecorator` for accurate in-chat width/background
- **`openMenu(href, openDetails?)`** — Shared factory for the "⋯" overflow menu; produces "Ask Mingo" + "Open Details" items with an ↗ new-tab side button
- **`HubspotTicket`** — Story covering `hubspot_ticket` registry variants: Open (warning), Closed (success), Waiting (grey), and no-status states
- **`GithubActivity`** — Story covering `github_commit`, `github_pull_request`, and `github_pr_review` variants with per-kind icons and review state pills (Approved/Changes/Comment)
- **`SlackMessage`** — Story covering `slack_message` with no status pill; title format `"Author · #channel"`

## Usage Example

```typescript
// Each story mirrors the entity-cards/dispatch.tsx field mapping:
export const HubspotTicket: Story = {
  render: () => (
    <MingoInfoCard
      title="Login page returns 500 after SSO redirect"
      description="Customer cannot sign in via Google SSO…"
      icon={<TagIcon size={24} />}
      status={{ label: 'Open', variant: 'warning' }}
      anchorProps={makeAnchorProps(HREF)}
      menuGroups={openMenu(HREF, {
        label: 'Open Ticket Details',
        icon: <TagIcon size={20} />,
      })}
      menuAriaLabel="Ticket actions"
    />
  ),
}
```

Each story uses `makeAnchorProps` to neutralize navigation inside the Storybook iframe, and `ChatColumnDecorator` to replicate the exact column width and background the dispatch renderer provides at runtime.
