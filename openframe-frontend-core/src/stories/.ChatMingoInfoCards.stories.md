Storybook stories file that documents and renders visual variants of `MingoInfoCard` as in-chat entity cards, mirroring the field mappings (icon, status pill, description, overflow menu) from the `CHAT_CARD_REGISTRY` dispatch without requiring the chat runtime.

## Key Components

- **`meta`** — Storybook `Meta` config registering stories under `Chat/ChatMingoInfoCards`, with `fullscreen` layout and `ChatColumnDecorator` wrapping all stories to replicate chat column width/background
- **`openMenu(href, openDetails?)`** — Shared overflow menu factory producing the standard "Ask Mingo" + "Open Details" (with ↗ new-tab side button) menu group used across all card stories
- **`HubspotTicket`** — Story covering `hubspot_ticket` registry types; renders four card states (Open → amber, Closed → green, Waiting → grey, no status)
- **`GithubActivity`** — Story covering `github_commit`, `github_pull_request`, and `github_pr_review` types; demonstrates per-kind icons and review state pills (Approved → green, Changes → red, Comment → grey)
- **`SlackMessage`** — Story covering `slack_message`; no status pill, author+channel title format, message preview as description

## Usage Example

```typescript
// All stories use the shared openMenu helper and makeAnchorProps fixture
import { makeAnchorProps } from './__fixtures__/chat-card-decorator'

<MingoInfoCard
  title="fix(auth): guard SSO redirect"
  description="a1b2c3d · May 28, 2026"
  icon={<CodingCommitIcon size={24} />}
  status={{ label: 'Commit', variant: 'grey' }}
  anchorProps={makeAnchorProps('https://github.com/...')}
  menuGroups={openMenu('https://github.com/...')}
  menuAriaLabel="Activity actions"
/>
```

> Each story's `render` function directly reproduces the icon/status/description mapping from `entity-cards/dispatch.tsx`, so visual regressions in card appearance are caught in Storybook before touching the chat runtime.
