Storybook stories for the `NotificationDrawer` feature, covering all tile variants, live animation states, drawer open/close behavior, and an interactive playground for firing notifications programmatically.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `AllTileVariants` | `Story` | Static render of all five `NotificationVariant` tile states (`default`, `success`, `error`, `warning`, `info`) |
| `LiveTile` | `Story` | Animated tile showing the 4-second progress bar / dismiss phase before settling; includes a replay button |
| `DrawerEmpty` | `Story` | Drawer auto-opened with no notifications (empty state) |
| `DrawerWithSeedData` | `Story` | Drawer pre-populated with six seeded notifications mirroring the Figma reference |
| `LivePlayground` | `Story` | Full interactive sandbox — fires all variants, toggles the drawer, marks-all-read, and clears notifications |
| `AutoOpen` | Helper | Calls `useNotifications().open()` on mount so drawer stories open immediately |
| `PlaygroundControls` | Helper | Internal panel exposing buttons for every `addNotification` variant and drawer actions |

## Usage Example

```typescript
// Simulating how a NATS/realtime source would push notifications
import { NotificationsProvider, NotificationDrawer, NotificationPopups, useNotifications } from '../components/features/notifications';

function AppRoot() {
  return (
    <NotificationsProvider onHistoryClick={() => router.push('/notifications')}>
      <NotificationDrawer />
      <NotificationPopups />
      <App />
    </NotificationsProvider>
  );
}

// From any child component or subscription handler:
function NatsListener() {
  const { addNotification } = useNotifications();

  useEffect(() => {
    nats.subscribe('alerts', (msg) => {
      addNotification({
        variant: 'error',
        title: msg.title,
        description: msg.body,
        onClick: () => router.push(`/alerts/${msg.id}`),
      });
    });
  }, [addNotification]);

  return null;
}
```

> The `LivePlayground` story mirrors this exact pattern — `addNotification` is called externally while `<NotificationPopups>` surfaces incoming items as top-right toasts. Opening the drawer automatically suppresses pop-ups to avoid duplicate display.
