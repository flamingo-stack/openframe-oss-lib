Storybook stories for the `MingoInfoCard` chat component, covering all Figma-defined variants and optional-element permutations.

## Key Components

| Export | Description |
|--------|-------------|
| `WithImage` | Card using a 40px avatar image as the leading media element |
| `WithIcon` | Card using a boxed glyph (`MonitorIcon`) instead of an image |
| `WithoutStatus` | Omits the status pill |
| `WithoutDescription` | Omits the subtitle/description text |
| `WithoutMenu` | Omits the trailing overflow (`⋯`) menu |
| `Clickable` | Entire card row is interactive via `onClick` |
| `StatusVariants` | Renders all four `Tag` palette variants: `success`, `warning`, `error`, `grey` |
| `LongContentTruncates` | Validates title/description truncation without layout overflow |

## Usage Example

```typescript
import { MingoInfoCard } from '../components/chat/mingo-info-card'
import { MonitorIcon } from '../components/icons-v2-generated/devices/monitor-icon'

<MingoInfoCard
  title="Production Server"
  description="All systems operational"
  icon={<MonitorIcon size={24} />}
  status={{ label: 'Online', variant: 'success' }}
  menuGroups={[
    {
      items: [
        {
          id: 'copy',
          label: 'Copy link',
          icon: <Copy01Icon size={16} />,
          onClick: () => console.log('copied'),
        },
      ],
    },
  ]}
  onClick={() => console.log('card clicked')}
/>
```

## Notes

- The card is `w-full` — stories use a `fullscreen` layout (not `centered`) so the card correctly fills its block container and reflows on viewport resize.
- `SAMPLE_MENU` defines reusable "Copy link" and "Remove" (danger) overflow actions shared across all stories.
