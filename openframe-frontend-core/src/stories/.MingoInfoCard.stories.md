Storybook stories for the `MingoInfoCard` chat component, covering all Figma-defined variants and edge cases including media type, optional elements, status pill variants, clickability, and text truncation behavior.

## Key Components

| Export | Description |
|--------|-------------|
| `WithImage` | Card with a 40px avatar image as the leading media element |
| `WithIcon` | Card with a boxed glyph icon instead of an avatar |
| `WithoutStatus` | Omits the status pill |
| `WithoutDescription` | Omits the subtitle/description line |
| `WithoutMenu` | Omits the trailing overflow "⋯" menu |
| `Clickable` | Full card acts as a clickable row (logs to console) |
| `StatusVariants` | Renders all four `Tag` palette variants: `success`, `warning`, `error`, `grey` |
| `LongContentTruncates` | Validates that long titles and descriptions truncate cleanly |

## Usage Example

```typescript
import { MingoInfoCard } from '../components/chat/mingo-info-card'
import { MonitorIcon } from '../components/icons-v2-generated/devices/monitor-icon'

<MingoInfoCard
  title="Production Host"
  description="All systems operational"
  icon={<MonitorIcon size={24} />}
  status={{ label: 'Online', variant: 'success' }}
  menuGroups={[
    {
      items: [
        {
          id: 'copy',
          label: 'Copy link',
          icon: <Copy01Icon size={16} />,
          onClick: () => console.log('copy'),
        },
      ],
    },
  ]}
  onClick={() => console.log('card clicked')}
/>
```

> **Layout note:** The card is `w-full` and must be placed inside a block container. The Storybook layout is set to `fullscreen` (not `centered`) so the card can fill 100% width and reflow correctly on viewport resize.
