Storybook stories for the `PageContainer` layout component, covering legacy usage, variant configurations, header customization, and padding/background options.

## Key Components

- **`meta`** — Storybook metadata registering `PageContainer` under `Layout/PageContainer` with `argType` controls for `variant`, `padding`, `background`, and `showHeader`
- **`PlaceholderContent`** — Internal helper component rendering a styled card placeholder to fill story layouts

## Exported Stories

| Story | Description |
|---|---|
| `Legacy` | Basic container with full-width background and custom className |
| `LegacyCustomBackground` | Legacy mode with custom `backgroundClassName` |
| `LegacyContentWidthBackground` | Legacy mode with content-scoped background only |
| `ListPage` | List variant with title, subtitle, and a single action button |
| `ListPageMultipleActions` | List variant with multiple header actions (Edit + Create) |
| `DetailPage` | Detail variant with back button navigation |
| `DetailPageWithActions` | Detail variant combining back button and action buttons |
| `FormPage` | Form variant with Cancel back button and Save/Publish actions |
| `ContentPage` | Generic content variant with a title |
| `CustomHeaderContent` | List variant with a fully custom `headerContent` ReactNode |
| `WithHeaderActions` | List variant combining `headerActions` and `actions` props |
| `HiddenHeader` | Content variant with `showHeader: false` |
| `WithPaddingAndBackground` | Content variant demonstrating `padding: 'lg'` and `background: 'card'` |

## Usage Example

```typescript
// Typical list page usage
<PageContainer
  variant="list"
  title="Devices"
  subtitle="24 devices found"
  actions={[
    { label: 'Add Device', onClick: () => {}, icon: <Plus />, variant: 'accent' }
  ]}
>
  <DeviceTable />
</PageContainer>

// Detail page with back navigation
<PageContainer
  variant="detail"
  title="Device Details"
  backButton={{ label: 'Back to Devices', onClick: () => router.back() }}
>
  <DeviceDetail />
</PageContainer>
```
