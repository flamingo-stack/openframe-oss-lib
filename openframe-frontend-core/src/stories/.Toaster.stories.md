Storybook stories for the `Toaster` component system, providing interactive and static previews of toast notification variants, command approval flows, and live trigger behavior.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `AllVariants` | `Story` | Static grid of all `ToastCard` variant states (default, success, warning, error, info) plus edge cases |
| `Playground` | `Story` | Interactive args-driven preview; tweak variant, title, description, duration, and dismissible via Storybook controls |
| `CommandApprovalCollapsed` | `Story` | Command approval toast in its collapsed (header-only) state |
| `CommandApprovalExpanded` | `Story` | Command approval toast with details panel pre-opened via `defaultExpanded` |
| `CommandApprovalPerTool` | `Story` | Renders one approval toast per supported tool type to verify icon mapping (Fleet MDM, TacticalRMM, MeshCentral, Authentik, osquery, OpenFrame) |
| `CommandApprovalLive` | `Story` | Fires real Sonner command approval toasts with `onApprove` / `onReject` callbacks |
| `LiveTriggers` | `Story` | Buttons that invoke `toast()` and `showToast()` at runtime, covering all variants and edge cases |

## Usage Example

```typescript
// Render the Toaster once near your app root
import { Toaster } from '@/components/ui/toaster';
export default function RootLayout({ children }) {
  return (
    <>
      {children}
      <Toaster />
    </>
  );
}

// Fire a toast from anywhere
import { showToast } from '@/components/ui/toaster';
import { showCommandApprovalToast } from '@/components/ui/toaster';

showToast({ variant: 'success', title: 'Saved', description: 'Changes applied.' });

showCommandApprovalToast({
  title: 'Tech Required',
  description: 'Approval required to execute.',
  command: 'fleetctl query --hosts mac-*',
  toolType: 'fleet_mdm',
  onApprove: () => console.log('approved'),
  onReject: () => console.log('rejected'),
});
```
