Storybook stories configuration for the StatusTag UI component, providing comprehensive examples and interactive controls for all component variants and configurations.

## Key Components

- **meta**: Storybook configuration defining the component's title, controls, and argument types
- **Individual Variant Stories**: `Success`, `Warning`, `Error`, `Critical`, `Info`, `Active`, `Inactive`, `Offline` - showcasing each status variant
- **Feature Stories**: `Mobile`, `WithLeftIcon`, `WithRightIcon`, `WithBothIcons` - demonstrating component features
- **Collection Stories**: `AllVariants`, `AllVariantsMobile`, `AllWithIcons` - displaying multiple variants together

## Usage Example

```typescript
// Basic status tag story
export const Success: Story = {
  args: { label: 'Success', variant: 'success' },
};

// Status tag with icon
export const WithIcon: Story = {
  args: {
    label: 'Online',
    variant: 'active',
    leftIcon: <Wifi className="size-4" />,
  },
};

// Mobile variant
export const Mobile: Story = {
  args: { 
    label: 'Mobile', 
    variant: 'success', 
    isMobile: true 
  },
};
```

The stories demonstrate all 8 status variants (`success`, `warning`, `error`, `critical`, `info`, `active`, `inactive`, `offline`) and support both mobile and desktop layouts with optional left/right icons using Lucide React icons.
