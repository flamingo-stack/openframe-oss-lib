Storybook configuration file that defines interactive stories for the StatusTag component, showcasing all available variants, states, and icon combinations for development and testing purposes.

## Key Components

- **Meta Configuration**: Defines component metadata with title "UI/StatusTag" and interactive controls for `variant` and `isMobile` properties
- **Individual Variant Stories**: Separate stories for each status variant (Success, Warning, Error, Critical, Info, Active, Inactive, Offline)
- **Icon Integration Stories**: Examples showing StatusTag with left icons, right icons, and both icons using Lucide React icons
- **Collection Stories**: Grouped displays showing all variants together in normal and mobile modes
- **Interactive Controls**: Storybook controls for testing different variant combinations and mobile responsiveness

## Usage Example

```typescript
// Individual variant story
export const Success: Story = {
  args: { label: 'Success', variant: 'success' },
};

// Story with icons
export const WithLeftIcon: Story = {
  args: {
    label: 'Online',
    variant: 'active',
    leftIcon: <Wifi className="size-4" />,
  },
};

// Collection story showing all variants
export const AllVariants: Story = {
  render: () => (
    <div className="flex flex-wrap gap-3">
      <StatusTag label="Success" variant="success" />
      <StatusTag label="Warning" variant="warning" />
      {/* ... more variants */}
    </div>
  ),
};
```

The stories demonstrate 8 status variants (success, warning, error, critical, info, active, inactive, offline) with support for mobile layouts and optional left/right icons from Lucide React.
