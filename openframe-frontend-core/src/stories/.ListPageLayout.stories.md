A comprehensive Storybook configuration file that showcases the `ListPageLayout` component through multiple interactive stories and usage examples. This file demonstrates different layout configurations, states, and real-world implementations for standardized list-based pages in the OpenFrame application.

## Key Components

- **Story Configuration**: Complete Storybook meta configuration with layout documentation
- **Sample Data**: Mock device data and table column definitions for realistic examples
- **Story Variants**: 15+ different story configurations covering various use cases
- **Interactive Examples**: Working search functionality and state management demonstrations
- **Real-world Examples**: Scripts page and Logs page layout implementations

## Usage Example

```typescript
// Basic story implementation
export const Basic: Story = {
  args: {
    title: 'Devices',
    searchPlaceholder: 'Search devices...',
    searchValue: '',
    onSearch: () => {},
    children: (
      <Table
        data={sampleDevices}
        columns={deviceColumns}
        rowKey="id"
      />
    )
  }
}

// Interactive story with state management
export const Interactive: Story = {
  render: function InteractiveStory() {
    const [searchValue, setSearchValue] = useState('')
    
    const filteredDevices = sampleDevices.filter(device =>
      device.name.toLowerCase().includes(searchValue.toLowerCase())
    )

    return (
      <ListPageLayout
        title="Devices"
        searchValue={searchValue}
        onSearch={setSearchValue}
      >
        <Table data={filteredDevices} columns={deviceColumns} rowKey="id" />
      </ListPageLayout>
    )
  }
}
```

The file includes comprehensive documentation showing the layout structure, key features like responsive design and accessibility support, and demonstrates various states including error handling, empty states, and different padding/background configurations.
