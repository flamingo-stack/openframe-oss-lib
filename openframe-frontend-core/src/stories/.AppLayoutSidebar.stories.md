Storybook stories for the `AppLayoutSidebar` component, demonstrating how `AppLayout` integrates with `AppLayoutDrawer` to render the Mingo AI chat sidebar as a togglable right-side panel within the main content area.

## Key Components

- **`Default`** — Story showing the Mingo AI header button toggling the right-side drawer open/closed; navigation sidebar and header remain interactive while the drawer is open
- **`Resizable`** — Story demonstrating a draggable/resizable drawer variant; starts open with a configurable `minSize`, `defaultSize`, and `storageKey` for persisting width across sessions
- **`DashboardChildren`** — Internal helper rendering a mock dashboard (device stats, active tickets) used as the main content backdrop for both stories
- **`navigationItems`** — Shared `NavigationSidebarConfig` fixture with Dashboard, Customers, Devices, Scripts, and Settings entries

## Usage Example

```typescript
import { AppLayout, AppLayoutDrawer, AppLayoutDrawerContent } from '../components/navigation'

// Fixed-width drawer
<AppLayout
  sidebarConfig={{ items: navigationItems, onNavigate: handleNav, onToggleMinimized: handleMinimize }}
  headerProps={{
    showMingoAI: true,
    onMingoAI: () => setOpen(prev => !prev),
    isMingoAIActive: open,
  }}
  drawer={
    <AppLayoutDrawer open={open} onOpenChange={setOpen}>
      <AppLayoutDrawerContent side="right">
        {/* chat UI */}
      </AppLayoutDrawerContent>
    </AppLayoutDrawer>
  }
>
  <YourPageContent />
</AppLayout>

// Resizable drawer
<AppLayoutDrawerContent
  side="right"
  resizable
  minSize={360}
  defaultSize={560}
  storageKey="my-app:mingo-sidebar"
/>
```

> The drawer renders **inside** the layout content area — the nav sidebar and header stay visible and fully interactive while it is open. Close via the X button or `Escape` key.
