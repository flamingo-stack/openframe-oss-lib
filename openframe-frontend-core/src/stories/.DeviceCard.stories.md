A comprehensive Storybook configuration file that defines stories for the DeviceCard component, showcasing various device states, operating systems, and interaction patterns across different use cases.

## Key Components

- **Meta Configuration**: Storybook metadata with component info, parameter settings, and argument type definitions
- **Base Device Template**: Reusable device object for consistent story data
- **Story Variants**: Multiple stories covering different device types (Windows, Linux, macOS), status states, and feature combinations
- **Interactive Examples**: Stories with custom actions, click handlers, and status badges
- **Layout Stories**: Grid layout demonstration and fully-loaded device examples

## Usage Example

```typescript
// Basic device story
export const Default: Story = {
  args: {
    device: {
      id: '1',
      name: 'MacBook Pro',
      operatingSystem: 'darwin',
      organization: 'Engineering Team',
      status: 'active',
      lastSeen: new Date().toISOString(),
    },
  },
}

// Device with custom actions and status badge
export const WithCustomFeatures: Story = {
  args: {
    device: baseDevice,
    actions: {
      moreButton: { visible: true },
      customActions: [
        { label: 'Remote Access', visible: true },
        { label: 'Run Script', visible: true },
      ],
    },
    statusBadgeComponent: (
      <div className="bg-green-500/20 text-green-500 px-2 py-1 rounded text-sm">
        Online
      </div>
    ),
    onDeviceClick: (device) => console.log('Device clicked:', device),
  },
}
```

The stories cover essential testing scenarios including different operating systems, status states, custom actions, tags, and interactive behaviors for comprehensive component documentation and testing.
