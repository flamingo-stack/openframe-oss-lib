Storybook configuration file defining visual documentation and interactive examples for the OSTypeBadge component system. This file creates comprehensive stories showcasing different OS type displays, normalization capabilities, and component variants.

## Key Components

- **OSTypeBadge Stories** - Main component stories with various OS types and display modes
- **OSTypeIcon Stories** - Icon-only variant stories with different sizes
- **OSTypeLabel Stories** - Label-only variant stories
- **Alias Demonstrations** - Stories showing OS type string normalization from different sources
- **Configuration Objects** - Meta configurations for each component variant

## Usage Example

```typescript
// Example story configuration for Windows OS display
export const Windows: Story = {
  args: {
    osType: 'windows',
  },
};

// Custom render function for comparison stories
export const AllOSTypes: Story = {
  render: () => (
    <div style={{ display: 'flex', flexDirection: 'column', gap: '1rem' }}>
      <OSTypeBadge osType="windows" />
      <OSTypeBadge osType="macos" />
      <OSTypeBadge osType="linux" />
    </div>
  ),
};
```

The stories demonstrate OS type normalization from various MDM sources (Fleet, Tactical RMM) with support for aliases like "Darwin" → "macOS" and "win32" → "Windows". Includes examples for icon-only, label-only, and combined display modes with customizable sizing options.
