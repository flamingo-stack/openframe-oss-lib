React hooks for subscribing to NATS subjects via the shared `NatsProvider` connection, with support for raw byte messages and auto-decoded JSON payloads.

## Key Components

### `useNatsSubscription`
Core hook that subscribes to a NATS subject using the connection from `<NatsProvider>`. Automatically resubscribes on subject changes, connection readiness changes, and reconnections (`reconnectionCount` dep). Cleans up the subscription on unmount or dependency change.

**Parameters:**
- `subject` — NATS subject string (or `null` to disable)
- `onMessage` — callback receiving raw `Msg`; stored in a ref to avoid stale closures
- `options` — optional queue group, max messages, and `enabled` flag

### `useNatsJsonSubscription`
Wraps `useNatsSubscription` to transparently decode the `Uint8Array` payload via `TextDecoder` and `JSON.parse`, calling `onPayload` with the typed result. Silently drops malformed payloads.

### Types
| Type | Description |
|------|-------------|
| `UseNatsSubscriptionOptions` | Extends `NatsSubscribeOptions` with an `enabled` flag |
| `UseNatsSubscriptionReturn` | `{ isSubscribed, isReady }` state |
| `UseNatsJsonSubscriptionOptions` | Alias of `UseNatsSubscriptionOptions` |
| `NatsJsMsg` | Re-export of `JsMsg` for JetStream consumers |

## Usage Example

```typescript
// Raw bytes
useNatsSubscription('telemetry.raw', (msg) => {
  console.log('bytes received:', msg.data)
}, { enabled: isLoggedIn })

// Typed JSON — resubscribes automatically after reconnects
interface DeviceEvent { id: string; status: 'online' | 'offline' }

const { isSubscribed, isReady } = useNatsJsonSubscription<DeviceEvent>(
  `devices.${deviceId}.events`,
  (payload, msg) => {
    console.log('device status:', payload.status)
  },
  { queue: 'ui-workers' },
)
```

> Pass `subject: null` or `enabled: false` to suspend the subscription without unmounting the component.
