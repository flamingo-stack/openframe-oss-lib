React hooks for subscribing to NATS subjects using the shared connection from `NatsProvider`. Provides both raw byte and JSON-decoded subscription variants with automatic re-subscription on reconnection.

## Key Components

- **`useNatsSubscription`** — Core hook that subscribes to a NATS subject, accepting raw `Msg` callbacks. Automatically resubscribes when the subject changes, the connection becomes ready, or a reconnection occurs.
- **`useNatsJsonSubscription`** — Wrapper around `useNatsSubscription` that decodes incoming byte payloads as JSON before invoking the callback. Silently ignores malformed messages.
- **`UseNatsSubscriptionOptions`** — Extends `NatsSubscribeOptions` with an `enabled` flag to conditionally pause subscriptions.
- **`UseNatsSubscriptionReturn`** — Returns `isSubscribed` (active subscription state) and `isReady` (underlying NATS connection readiness).

## Usage Example

```typescript
// Raw bytes subscription
function MyComponent() {
  const { isSubscribed } = useNatsSubscription(
    'events.alerts',
    (msg) => console.log('Received:', msg.data),
    { enabled: true, queue: 'workers' }
  )
  return <span>{isSubscribed ? 'Listening' : 'Disconnected'}</span>
}

// JSON subscription with typed payload
interface AlertPayload {
  level: 'info' | 'warn' | 'error'
  message: string
}

function AlertListener() {
  const { isSubscribed, isReady } = useNatsJsonSubscription<AlertPayload>(
    'events.alerts',
    (payload, msg) => {
      console.log(`[${payload.level}] ${payload.message}`)
    },
    { enabled: isReady }
  )
}
```

> **Note:** Both hooks use a stable `handlerRef` to avoid resubscribing when only the callback reference changes. Pass `enabled: false` or `subject: null` to cleanly unsubscribe.
