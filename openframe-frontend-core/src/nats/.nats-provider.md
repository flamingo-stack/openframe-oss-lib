React context provider that manages a shared NATS WebSocket connection lifecycle, exposing connection state and client access to child components via context hooks.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `NatsProvider` | Component | Root provider managing connection lifecycle, reconnection backoff, and status tracking |
| `useNats` | Hook | Returns `NatsContextValue`; throws if used outside `<NatsProvider>` |
| `useOptionalNats` | Hook | Returns `NatsContextValue | null`; safe to call outside provider |
| `NatsProviderProps` | Interface | Provider configuration including `getWsUrl`, `onBeforeReconnect`, `clientConfig`, `reconnectionBackoff`, `urlRevision` |
| `NatsContextValue` | Interface | `{ client, status, isReady, reconnectionCount }` consumed by hooks |

## Usage Example

```typescript
// 1. Wrap your app (or subtree) with NatsProvider
function App() {
  const { token } = useAuth()

  return (
    <NatsProvider
      getWsUrl={() => token ? `wss://nats.example.com?token=${token}` : null}
      onBeforeReconnect={async () => await refreshToken()}
      reconnectionBackoff={{ initialMs: 500, maxMs: 30_000 }}
      urlRevision={token} // forces re-evaluation when auth changes
    >
      <Dashboard />
    </NatsProvider>
  )
}

// 2. Consume in child components
function Dashboard() {
  const { client, isReady, status, reconnectionCount } = useNats()

  if (!isReady) return <p>Connecting… ({status})</p>

  return <p>Connected — reconnections: {reconnectionCount}</p>
}

// 3. Optional access (no throw outside provider)
function MaybeSubscriber() {
  const nats = useOptionalNats()
  if (!nats?.isReady) return null
  return <Subscriber client={nats.client} />
}
```

## Notes

- **URL laziness** — `getWsUrl()` returning `null` cleanly tears down any existing connection without error.
- **`urlRevision` prop** — increment or change this value (e.g. pass the auth token itself) to trigger URL re-evaluation; the provider does not watch external auth state on its own.
- **Shared connections** — delegates to `acquireClient` / `releaseClient` from `shared-connection`, so multiple provider instances pointing at the same URL share a single underlying connection.
- All callback refs (`getWsUrl`, `onBeforeReconnect`, `reconnectionBackoff`) are kept in refs to avoid restarting the connection lifecycle on every render.
