React context provider that manages a shared NATS WebSocket connection lifecycle, exposing connection state and client access to child components via context hooks.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `NatsProvider` | Component | Context provider that owns the NATS connection lifecycle |
| `useNats` | Hook | Consume NATS context; throws if used outside `<NatsProvider>` |
| `useOptionalNats` | Hook | Consume NATS context; returns `null` if no provider is present |
| `NatsProviderProps` | Interface | Props for configuring the provider |
| `NatsContextValue` | Interface | Shape of the context value exposed to consumers |
| `NatsReconnectionBackoff` | Type (re-export) | Backoff strategy for reconnection attempts |

## `NatsContextValue` Shape

| Field | Type | Description |
|-------|------|-------------|
| `client` | `NatsClient \| null` | Active NATS client, or `null` when disconnected |
| `status` | `NatsStatus` | Current connection status (`connecting`, `connected`, `closed`, etc.) |
| `isReady` | `boolean` | `true` only when `status === 'connected'` and `client !== null` |
| `reconnectionCount` | `number` | Increments on each successful reconnect (useful for re-subscribing) |

## Usage Example

```typescript
// Wrap your app (or subtree) with NatsProvider
function App() {
  const getWsUrl = () => isAuthenticated ? 'wss://nats.example.com' : null

  return (
    <NatsProvider
      getWsUrl={getWsUrl}
      urlRevision={authToken}          // re-evaluates URL when auth changes
      onBeforeReconnect={refreshToken} // refresh credentials before each retry
      reconnectionBackoff={{ initialMs: 500, maxMs: 30_000 }}
    >
      <MyFeature />
    </NatsProvider>
  )
}

// Consume inside any child component
function MyFeature() {
  const { client, isReady, reconnectionCount } = useNats()

  React.useEffect(() => {
    if (!isReady || !client) return
    const sub = client.subscribe('events.>')
    // handle messages ...
    return () => sub.unsubscribe()
  }, [isReady, reconnectionCount]) // re-subscribe on reconnect

  return <div>{isReady ? 'Connected' : 'Connecting…'}</div>
}
```

> **Note:** Pass `urlRevision` whenever external auth state changes (e.g. a new JWT). The provider does not watch auth state on its own — bumping `urlRevision` is the signal to call `getWsUrl()` again and reconnect if the URL changed.
