A browser/WebSocket NATS client abstraction built on `nats.ws`, providing typed publish/subscribe, request/reply, JetStream ordered consumer support, and connection lifecycle management with exponential backoff reconnection.

## Key Components

### Interfaces & Types
- **`NatsClient`** — Main client interface with publish, subscribe, request/reply, JetStream, and status methods
- **`NatsClientOptions`** — Connection configuration including servers, auth, reconnect behavior, and exponential backoff
- **`JetStreamOrderedSubscribeOptions`** — Options for ephemeral JetStream ordered consumer subscriptions
- **`NatsStatusEvent`** / **`NatsStatus`** — Connection lifecycle event types (`connecting`, `connected`, `disconnected`, `reconnecting`, `closed`, `error`)
- **`NatsSubscribeOptions`** / **`NatsPublishOptions`** / **`NatsRequestOptions`** — Per-operation options

### Factory Function
- **`createNatsClient(options)`** — Returns a `NatsClient` instance; lazily imports `nats.ws` (browser/Tauri only, throws if `window` is undefined)

### Internal Utilities
- **`createExponentialBackoffHandler`** — Configurable backoff with optional jitter using `crypto.getRandomValues`
- **`toNatsHeaders`** — Converts `Record<string, string>` or existing `NatsHeaders` into a `nats.ws` headers object
- **`mapNatsTypeToStatus`** — Normalizes raw `nats.ws` status strings to `NatsStatus` values

## Usage Example

```typescript
import { createNatsClient } from './nats'

const client = createNatsClient({
  servers: 'wss://nats.example.com:443',
  name: 'my-app',
  token: 'secret-token',
  exponentialBackoff: {
    initialDelayMs: 1000,
    maxDelayMs: 30000,
    multiplier: 2,
    jitter: true,
  },
})

// Listen for connection status changes
client.onStatus((event) => {
  console.log('NATS status:', event.status)
})

await client.connect()

// Publish & subscribe JSON
client.publishJson('events.created', { id: 1, name: 'test' })

client.subscribeJson<{ id: number; name: string }>('events.created', (payload) => {
  console.log('Received:', payload)
})

// Request/reply
const response = await client.requestJson<{ ok: boolean }>('rpc.ping', {})

// JetStream ordered consumer (live-tail)
const handle = await client.subscribeJetStreamOrdered(
  (msg) => console.log('JetStream msg seq:', msg.seq),
  {
    streamName: 'EVENTS',
    filterSubject: 'events.>',
    deliverPolicy: 'new',
  },
)

// Cleanup
handle.unsubscribe()
await client.close()
```

> **Note:** This client is browser/Tauri-only. Importing in a Next.js server bundle is safe, but calling `connect()` server-side will throw.
