Manages a singleton NATS WebSocket connection shared across multiple consumers, providing reference-counted acquire/release lifecycle and an exponential-backoff reconnection loop with single-owner coordination.

## Key Components

**`NATS_DEFAULTS`** — Compile-time constants for timeouts, ping intervals, and retry backoff parameters.

**`SharedConnection`** — Shape of the singleton: holds the `NatsClient`, `refCount`, pending `connectPromise`, and `retryOwner` (the token of the consumer currently driving reconnects).

**`acquireClient(url, opts?)`** — Increments `refCount`, creates a new `NatsClient` if the URL changed (closing the old one), and cancels any pending close timer.

**`releaseClient(url, opts?)`** — Decrements `refCount`; when it reaches zero, schedules a delayed `close()` (default 3 s) to allow rapid re-acquire without reconnecting.

**`startConnectionLifecycle(options)`** — Attaches a status listener and drives the reconnect loop. Only the first caller to claim `retryOwner` runs retries; subsequent callers observe status only. Returns a `stop()` handle that clears timers and releases ownership.

**`NatsReconnectionBackoff`** — Optional configuration for the retry strategy: a fast-retry phase (fixed delay) followed by an exponential phase with jitter and a configurable ceiling.

## Usage Example

```typescript
import { acquireClient, releaseClient, startConnectionLifecycle } from './shared-connection'

// Acquire the shared connection
const conn = acquireClient('wss://nats.example.com', { user: 'app', pass: 'secret' })

// Start the connection lifecycle (connect + auto-retry)
const lifecycle = startConnectionLifecycle({
  conn,
  wsUrl: 'wss://nats.example.com',
  getFreshUrl: () => 'wss://nats.example.com',
  backoff: { fastRetries: 2, fastRetryDelayMs: 500, maxDelayMs: 15_000 },
  onStatusChange: (status) => console.log('NATS status:', status),
})

// On teardown
lifecycle.stop()
releaseClient('wss://nats.example.com')
```
