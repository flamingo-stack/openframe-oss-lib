React Query hooks for **public-read** access to onboarding guides, plus a shared cache key builder (`onboardingGuideKeys`) used by both client and admin (hub-side) code.

## Key Components

### `onboardingGuideKeys`
Hierarchical cache key factory for the `['onboarding-guides']` namespace. Exposes both public sub-keys and an `admin` sub-namespace so a single `invalidateQueries({ queryKey: onboardingGuideKeys.all })` clears both public and admin caches simultaneously.

| Key | Purpose |
|-----|---------|
| `all` | Root namespace — invalidates everything |
| `list(filters)` | Filtered guide list |
| `detail(slug)` | Single guide by slug |
| `sections()` | Section summaries |
| `admin.*` | Admin sub-namespace (mutations live hub-side) |

### `useOnboardingGuides(filters?)`
Fetches a paginated, filterable list of published guides from `GET /api/onboarding-guides`. Accepts `search`, `section`, `limit`, and `offset` filters.

### `useOnboardingGuide(slug)`
Fetches a single guide by slug from `GET /api/onboarding-guides/:slug`. Query is disabled when `slug` is `undefined`.

### `useOnboardingGuideSections()`
Fetches all section summaries from `GET /api/onboarding-guides/sections`. Always fetches fresh data (`staleTime: 0`).

## Usage Example

```typescript
import {
  useOnboardingGuides,
  useOnboardingGuide,
  useOnboardingGuideSections,
  onboardingGuideKeys,
} from './use-onboarding-guides'

// Filtered guide list
const { data: guides } = useOnboardingGuides({ section: 'networking', limit: 10 })

// Single guide (disabled until slug is available)
const { data: guide } = useOnboardingGuide(slug)

// All section summaries
const { data: sections } = useOnboardingGuideSections()

// Invalidate all public + admin guide caches from an admin hook
queryClient.invalidateQueries({ queryKey: onboardingGuideKeys.all })

// Invalidate only the admin cache
queryClient.invalidateQueries({ queryKey: onboardingGuideKeys.admin.all })
```

> **Non-Next.js embedders:** Routes under `/api/onboarding-guides*` must be reverse-proxied to the host. Admin mutations (`create`, `update`, `delete`, `publish`, `stats`) are managed hub-side and import `onboardingGuideKeys.admin` from this module.
