React Query hooks for public-read access to Onboarding Guides, plus a shared cache key builder used by both client-side reads and hub-side admin mutations.

## Key Components

### `onboardingGuideKeys`
Hierarchical cache key factory for the `['onboarding-guides']` namespace. Exposes both public sub-keys and an `admin` sub-namespace so hub-side mutations can invalidate the full cache with a single `invalidateQueries({ queryKey: onboardingGuideKeys.all })` call.

| Key | Description |
|-----|-------------|
| `all` | Root namespace — invalidates everything |
| `lists()` / `list(filters)` | Paginated/filtered list queries |
| `details()` / `detail(slug)` | Single guide lookups |
| `sections()` | Section summary list |
| `admin.*` | Sub-namespace for hub-side mutations |

### `useOnboardingGuides(filters?)`
Fetches a paginated, filterable list of guides from `/api/onboarding-guides`. Supports `search`, `section`, `limit`, and `offset` filter params.

### `useOnboardingGuide(slug)`
Fetches a single guide by slug from `/api/onboarding-guides/:slug`. Query is disabled until a slug is provided.

### `useOnboardingGuideSections()`
Fetches all section summaries from `/api/onboarding-guides/sections`. Always refetches on mount (`staleTime: 0`).

## Usage Example

```typescript
import {
  useOnboardingGuides,
  useOnboardingGuide,
  useOnboardingGuideSections,
  onboardingGuideKeys,
} from './use-onboarding-guides'

// Filtered list
const { data: guides } = useOnboardingGuides({ section: 'networking', limit: 10 })

// Single guide (query skipped until slug is known)
const { data: guide } = useOnboardingGuide(slug)

// Section nav
const { data: sections } = useOnboardingGuideSections()

// Hub-side admin: invalidate everything including public reads
queryClient.invalidateQueries({ queryKey: onboardingGuideKeys.all })

// Hub-side admin: invalidate only admin cache
queryClient.invalidateQueries({ queryKey: onboardingGuideKeys.admin.all })
```
