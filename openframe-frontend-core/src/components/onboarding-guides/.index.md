Barrel export file for the Onboarding Guides product surface within the OpenFrame frontend, exposing views, skeletons, and query hooks while intentionally omitting entity types to prevent TS2308 ambiguous re-export conflicts.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `OnboardingGuidesCatalogView` | Component | Main catalog/list view for browsing onboarding guides |
| `OnboardingGuideDetailView` | Component | Detail view for a single onboarding guide |
| `OnboardingGuidesCatalogSkeleton` | Component | Loading skeleton for the catalog view |
| `useOnboardingGuides` | Hook | Query hook for fetching the guides list |
| `useOnboardingGuide` | Hook | Query hook for fetching a single guide |
| `useOnboardingGuideSections` | Hook | Query hook for fetching sections of a guide |
| `onboardingGuideKeys` | Object | React Query key factory for cache management |

## Usage Example

```typescript
import {
  OnboardingGuidesCatalogView,
  OnboardingGuideDetailView,
  OnboardingGuidesCatalogSkeleton,
  useOnboardingGuides,
  useOnboardingGuide,
} from '@flamingo-stack/openframe-frontend-core/components/onboarding-guides'

// Fetch and render the catalog
function GuidesPage() {
  const { data, isLoading } = useOnboardingGuides()

  if (isLoading) return <OnboardingGuidesCatalogSkeleton />
  return <OnboardingGuidesCatalogView guides={data} />
}

// Fetch and render a single guide
function GuidePage({ id }: { id: string }) {
  const { data } = useOnboardingGuide(id)
  return <OnboardingGuideDetailView guide={data} />
}
```

## ⚠️ Type Import Note

`OnboardingGuide` and related entity types are **not** exported from this barrel to avoid TS2308 ambiguous re-export conflicts. Import them directly from the chat surface:

```typescript
import type { OnboardingGuide } from
  '@flamingo-stack/openframe-frontend-core/components/chat'
```
