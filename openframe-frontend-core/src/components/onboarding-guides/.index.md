Barrel export file for the **Onboarding Guides** product surface, exposing views, skeletons, and data-fetching hooks for OpenFrame's onboarding guide feature.

## Key Components

### Views
- **`OnboardingGuidesCatalogView`** / **`OnboardingGuidesCatalogViewProps`** — Catalog listing view for browsing available onboarding guides
- **`OnboardingGuideDetailView`** / **`OnboardingGuideDetailViewProps`** — Detail view for a single onboarding guide
- **`OnboardingGuidesCatalogSkeleton`** — Loading skeleton for the catalog view

### Hooks & Query Keys
- **`useOnboardingGuides`** — Fetches the list of onboarding guides
- **`useOnboardingGuide`** — Fetches a single onboarding guide by ID
- **`useOnboardingGuideSections`** — Fetches sections for a specific guide
- **`onboardingGuideKeys`** — React Query key factory for cache management

## Usage Example

```typescript
import {
  OnboardingGuidesCatalogView,
  OnboardingGuideDetailView,
  OnboardingGuidesCatalogSkeleton,
  useOnboardingGuides,
  useOnboardingGuide,
  onboardingGuideKeys,
} from '@flamingo-stack/openframe-frontend-core/components/onboarding-guides'

// For row/entity types, import from the chat surface — NOT from here:
import type { OnboardingGuide } from
  '@flamingo-stack/openframe-frontend-core/components/chat'

function GuideCatalog() {
  const { data, isLoading } = useOnboardingGuides()
  if (isLoading) return <OnboardingGuidesCatalogSkeleton />
  return <OnboardingGuidesCatalogView guides={data} />
}
```

## ⚠️ Important: Avoid Duplicate Re-exports

Do **not** import `OnboardingGuide`, `OnboardingGuideFilters`, `OnboardingGuideListResponse`, or `OnboardingGuideSectionSummary` from this barrel. Those types are owned by `components/chat` — re-exporting them here triggers TypeScript **TS2308** (ambiguous re-export) at the top-level barrel.
