Public-facing catalog view component for the `/onboarding-guides` route, supporting both SSR-controlled (hub) and self-fetching (config-only embed) data modes.

## Key Components

- **`OnboardingGuidesCatalogView`** — Main export. Renders the guide catalog body with search, section pill filters, grouped guide cards, loading/error states, and empty-state UI. Does **not** render page chrome (hero, back button) — callers wrap it in `<DevSectionPage>`.
- **`OnboardingGuidesCatalogViewProps`** — Interface defining controlled (`initialGuides`, `initialSections`) vs. self-fetch (`guidesEndpoint`, `sectionsEndpoint`) modes, plus optional `renderCard` override and `basePath`.
- **`SectionSummary`** — Internal type: `{ section, section_order, count }` for building filter pill options.
- **`useSelfFetch`** — Hook used in self-fetch mode to retrieve guides (re-fetches on `?section=` URL change) and section summaries.
- **`useDocSearch`** — RAG-backed search bar scoped to `onboarding-guides` table IDs.
- **`FilterPillRow`** — Dynamic section pills derived from available sections in data.

## Usage Example

**Controlled (SSR/hub) mode:**

```typescript
<OnboardingGuidesCatalogView
  initialGuides={serverGuides}
  initialSections={serverSections}
  initialSection={searchParams.section ?? ''}
  basePath="/onboarding-guides"
/>
```

**Self-fetching (embed/config-only) mode:**

```typescript
<OnboardingGuidesCatalogView
  guidesEndpoint="/api/onboarding-guides"
  sectionsEndpoint="/api/onboarding-guides/sections"
  basePath="/onboarding-guides"
/>
```

**Custom card renderer:**

```typescript
<OnboardingGuidesCatalogView
  initialGuides={guides}
  renderCard={(guide) => <MyCustomCard guide={guide} />}
/>
```

> **Data mode detection:** if `initialGuides` is `undefined` and `guidesEndpoint` is provided, self-fetch mode activates. Section filter changes push `?section=X` to the router; in controlled mode the host RSC re-renders, in self-fetch mode the hook re-fetches automatically.
