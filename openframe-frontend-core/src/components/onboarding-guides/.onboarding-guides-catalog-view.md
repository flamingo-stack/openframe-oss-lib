Public-facing catalog view for the `/onboarding-guides` route, supporting both SSR-controlled (hub) and self-fetching (config-only embed) data modes with section filtering and RAG-powered search.

## Key Components

- **`OnboardingGuidesCatalogView`** — Main export. Renders the guide grid, section filter pills, and `<DocSearchBar>`. Does NOT render page chrome (caller wraps in `<DevSectionPage>`).
- **`OnboardingGuidesCatalogViewProps`** — Interface controlling data mode:
  - *Controlled mode*: pass `initialGuides` + `initialSections` (server-fetched).
  - *Self-fetch mode*: pass `guidesEndpoint` + `sectionsEndpoint` (client-fetched, refetches on `?section=` URL change).
- **`useSelfFetch`** — Drives client-side data fetching when no initial data is provided.
- **`useDocSearch`** — RAG search scoped to `onboarding-guides` table IDs.
- **`grouped`** (memoized) — Buckets guides by section, sorted by `section_order` then `step_order`.
- **`renderCard`** — Optional override for per-guide card rendering; defaults to `<OnboardingGuideCard>` with runtime-composed hrefs via `resolveContentHref`.

## Usage Example

```typescript
// Controlled (SSR / hub) mode — host provides server-fetched data
<OnboardingGuidesCatalogView
  initialGuides={guides}
  initialSections={sections}
  initialSection={searchParams.section ?? ''}
  basePath="/onboarding-guides"
/>

// Self-fetch (config-only embed) mode — view fetches its own data
<OnboardingGuidesCatalogView
  guidesEndpoint="/api/onboarding-guides"
  sectionsEndpoint="/api/onboarding-guides/sections"
  basePath="/onboarding-guides"
/>

// Custom card renderer override
<OnboardingGuidesCatalogView
  initialGuides={guides}
  renderCard={(guide) => <MyCustomCard guide={guide} />}
/>
```

**Loading states:** Shows `<OnboardingGuidesCatalogSkeleton>` only on cold first load; subsequent section toggles dim the grid (`opacity-60`) while refetching. Fetch errors surface a retryable `<LoadError>`.
