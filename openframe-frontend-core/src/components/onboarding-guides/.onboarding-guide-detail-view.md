Renders the public-facing detail page for a single onboarding guide at `/onboarding-guides/<slug>`, supporting both SSR-controlled and self-fetching embed modes.

## Key Components

- **`OnboardingGuideDetailView`** — Main exported component. Displays guide title, author metadata, video player (Mux/MP4 or YouTube), markdown body, video bites, and related guides in the same section.
- **`OnboardingGuideDetailViewProps`** — Interface controlling two operation modes:
  - **Controlled/SSR**: pass `initialData` (pre-fetched `OnboardingGuide`)
  - **Self-fetching**: pass `slug` + `guideEndpoint` factory; the component fetches its own data via `useSelfFetch`
- **`useSelfFetch`** — Drives data loading; no-ops when `initialData` is provided.
- **`useVideoWarmup`** — Pre-warms the video connection before playback.
- **`resolveContentHref`** — Builds related-card hrefs via `runtime.composeContentUrl` or falls back to `basePath`.

## Usage Example

```typescript
// SSR / controlled mode (hub pattern)
const guide = await fetchGuide(params.slug)

<OnboardingGuideDetailView
  initialData={guide}
  related={relatedGuides}
  backHref="/getting-started"
/>

// Self-fetching / config-only embed (no host data layer)
<OnboardingGuideDetailView
  slug={params.slug}
  guideEndpoint={(s) => `/content/api/onboarding-guides/${s}`}
  related={[]}
  basePath="/onboarding-guides"
/>

// Custom markdown renderer and related-card renderer
<OnboardingGuideDetailView
  initialData={guide}
  MarkdownRenderer={MyMDXRenderer}
  renderRelatedCard={(g) => <MyGuideCard guide={g} />}
/>
```

## Render Sections (in order)

1. Back navigation link
2. Guide title (`h1`)
3. Author card with section · step metadata
4. Video player (skipped if no `main_video_url` or `youtube_url`)
5. Markdown body via `MarkdownRenderer`
6. Video bites strip
7. Related guides (same section, filtered from `related` prop)
