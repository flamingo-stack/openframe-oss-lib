Public-facing detail view for rendering a single onboarding guide at `/onboarding-guides/<slug>`. Supports both SSR-controlled mode (via `initialData`) and self-fetching config-only embed mode (via `slug` + `guideEndpoint`).

## Key Components

- **`OnboardingGuideDetailView`** — Main export. Renders the full guide detail page including back navigation, title, author metadata, video player, markdown body, video bites, and related guides.
- **`OnboardingGuideDetailViewProps`** — Interface defining all props, supporting both controlled and self-fetching modes.
- **`useSelfFetch`** — Handles data fetching in self-fetch mode; no-ops when `initialData` is provided.
- **`useVideoWarmup`** — Preconnects to the video origin before playback to reduce latency.
- **`defaultRenderRelatedCard`** — Internal fallback that resolves `href` via `runtime.composeContentUrl` and renders an `OnboardingGuideCard`.

## Usage Example

**Controlled (SSR) mode:**

```typescript
<OnboardingGuideDetailView
  initialData={serverFetchedGuide}
  related={relatedGuides}
  basePath="/onboarding-guides"
/>
```

**Self-fetching (config-only embed) mode:**

```typescript
<OnboardingGuideDetailView
  slug={params.slug}
  guideEndpoint={(s) => `/content/api/onboarding-guides/${s}`}
  related={relatedGuides}
  backHref="/getting-started"
  backLabel="Back to Getting Started"
/>
```

**With custom markdown renderer and related card:**

```typescript
<OnboardingGuideDetailView
  initialData={guide}
  MarkdownRenderer={MyCustomRenderer}
  renderRelatedCard={(g) => <MyGuideCard guide={g} />}
  basePath="/onboarding-guides"
/>
```

> **Loading states:** Renders `<DetailPageSkeleton>` while fetching and `<LoadError>` on failure or missing guide — consistent with catalog/roadmap/release detail pages.
