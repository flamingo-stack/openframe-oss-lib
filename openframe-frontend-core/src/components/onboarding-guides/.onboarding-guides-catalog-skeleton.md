Suspense fallback skeleton for the `/onboarding-guides` catalog page, rendering shimmer placeholders that mirror the loaded `<OnboardingGuidesCatalogView>` layout to prevent layout shift during data fetching.

## Key Components

- **`OnboardingGuidesCatalogSkeleton`** — Page-level skeleton component used as a `<Suspense>` fallback, composing:
  - `<DevSectionPage sectionKey="onboarding">` — preserves hero, back button, and page scaffold
  - `preControls` slot — reserves space for the search bar (`h-12`) and section pill row (`~74px`)
  - `3` skeleton sections with `4 + 3 + 3 = 10` cards matching the typical onboarding dataset
  - `<OnboardingGuideCardSkeleton size="catalog">` — per-card skeleton at `288px` height to eliminate per-card shift on resolve

## Usage Example

```typescript
import { Suspense } from 'react'
import { OnboardingGuidesCatalogSkeleton } from './onboarding-guides-catalog-skeleton'
import { OnboardingGuidesCatalogView } from './onboarding-guides-catalog-view'

export default function OnboardingGuidesPage() {
  return (
    <Suspense fallback={<OnboardingGuidesCatalogSkeleton />}>
      <OnboardingGuidesCatalogView />
    </Suspense>
  )
}
```

> **Layout shift guarantee:** The skeleton uses identical wrapper classes, section structure, and card heights as the loaded view. The `preControls` slot pre-allocates space for controls rendered post-load, ensuring a zero-shift Suspense → resolved transition.
