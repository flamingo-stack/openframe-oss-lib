Suspense fallback skeleton for the `/onboarding-guides` catalog page, rendering shimmer placeholders that mirror the loaded `<OnboardingGuidesCatalogView>` layout to eliminate cumulative layout shift (CLS) on resolve.

## Key Components

- **`OnboardingGuidesCatalogSkeleton`** — Default export; a page-level Suspense fallback that renders:
  - `<DevSectionPage sectionKey="onboarding">` — shared page scaffold (hero, back button) identical to the loaded view
  - `preControls` slot — shimmer for search bar (`h-12`) + filter pill row (`~74px`) to prevent vertical shift
  - `10` `<OnboardingGuideCardSkeleton size="catalog">` cards distributed across 3 sections (`4 + 3 + 3`) matching the typical onboarding dataset
  - Section heading placeholders with title and count badge shimmers

## Usage Example

```typescript
import { Suspense } from 'react'
import { OnboardingGuidesCatalogSkeleton } from './onboarding-guides-catalog-skeleton'
import { OnboardingGuidesCatalogView } from './onboarding-guides-catalog-view'

export default function OnboardingGuidesPage() {
  return (
    <Suspense fallback={<OnboardingGuidesCatalogSkeleton />}>
      <OnboardingGuidesCatalogView />
    </Suspense>
  )
}
```

> **Layout contract:** Card height is fixed at `288px` and section structure is `4 + 3 + 3`, byte-identical to loaded cards — ensuring zero per-card vertical shift when the Suspense boundary resolves.
