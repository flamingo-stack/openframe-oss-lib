A reusable React component that renders contextual empty states for vendors, posts, search results, and generic scenarios, with smart CTA logic that adapts based on the current URL path.

## Key Components

### `EmptyStateProps` Interface
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'vendors' \| 'posts' \| 'search' \| 'generic'` | — | Determines default icon, title, and description |
| `title` | `string` | auto | Overrides default title for the given type |
| `description` | `string` | auto | Overrides default description |
| `showBackButton` | `boolean` | `false` | Renders an optional back button |
| `onGoBack` | `() => void` | — | Callback for the back button |
| `showCTA` | `boolean` | `true` | Toggles the smart CTA button |
| `ctaText` | `string` | auto | Custom CTA label (requires `onCtaClick`) |
| `onCtaClick` | `() => void` | auto | Custom CTA handler |
| `ctaVariant` | `'primary' \| 'secondary'` | `'primary'` | Button style variant |

### Internal Helpers
- **`getDefaultContent()`** — Maps `type` to a Lucide icon, title, and description
- **`getSmartCTA()`** — Derives CTA text and navigation action from `type` + `window.location.pathname`, falling back to custom props if provided

## Usage Example

```typescript
// Basic usage with type defaults
<EmptyState type="vendors" />

// Search with custom title
<EmptyState
  type="search"
  title="No matching results"
  description="Try broadening your search terms."
/>

// Custom CTA and back button
<EmptyState
  type="posts"
  showCTA
  ctaText="Start Writing"
  onCtaClick={() => router.push('/blog/new')}
  ctaVariant="secondary"
  showBackButton
  onGoBack={() => router.back()}
  backButtonText="Return"
/>
```

> **Note:** Smart CTA routing is path-aware — for example, `type="vendors"` on `/vendors` renders **Reset Filters**, while on `/profile` it renders **Browse Vendors**.
