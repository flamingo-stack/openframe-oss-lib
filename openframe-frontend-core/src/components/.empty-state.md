A reusable React component that renders a contextual empty state UI for lists/search results, with smart CTA logic based on the current route and content type.

## Key Components

### `EmptyStateProps` (interface)
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'vendors' \| 'posts' \| 'search' \| 'generic'` | — | Drives default icon, title, and description |
| `title` | `string` | type-based | Override default heading text |
| `description` | `string` | type-based | Override default body text |
| `showCTA` | `boolean` | `true` | Show/hide the smart CTA button |
| `ctaText` | `string` | smart default | Custom CTA label |
| `onCtaClick` | `() => void` | smart default | Custom CTA handler |
| `ctaVariant` | `'primary' \| 'secondary'` | `'primary'` | CTA button style |
| `showBackButton` | `boolean` | `false` | Renders an optional back button |
| `onGoBack` | `() => void` | — | Handler for back button |

### `EmptyState` (component)
- **`getDefaultContent()`** — Returns a type-appropriate icon (`Package`, `FileText`, `Search`), title, and description.
- **`getSmartCTA()`** — Derives CTA label and navigation action from `window.location.pathname` and `type`. Custom `ctaText`+`onCtaClick` always take priority. Falls back to path-aware defaults (e.g. "Reset Filters" on `/vendors`, "Browse Vendors" from `/profile`).

## Usage Example

```typescript
// Basic usage — type drives all defaults
<EmptyState type="vendors" />

// Search result empty state with custom CTA
<EmptyState
  type="search"
  title="No results for 'firewall'"
  showCTA
  ctaText="Clear Search"
  onCtaClick={() => setQuery('')}
  ctaVariant="secondary"
/>

// With optional back navigation
<EmptyState
  type="posts"
  showBackButton
  onGoBack={() => router.back()}
  backButtonText="Return to Blog"
/>
```
