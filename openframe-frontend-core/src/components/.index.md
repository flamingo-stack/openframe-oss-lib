Central barrel export file for the Flamingo/OpenFrame shared component library, re-exporting all UI components, utilities, and feature modules from their respective subdirectories under a single entry point.

## Key Components

The barrel aggregates exports across several categories:

- **UI & Layout** — `ui`, `platform`, `navigation`, `footer`, `pagination`, `empty-state`, `content-loading-container`
- **Chat & AI** — `chat` (Mingo AI / Fae interfaces)
- **Vendor & MSP** — `vendor-display-button`, `vendor-icon`, `msp-display`, `vendor-tag`, `pricing-display`
- **Media & Images** — `hero-image-uploader`, `image-cropper`, `media-carousel`, `icons-block`
- **Shared Modules** — `onboarding`, `doc-search`, `product-release`, `dev-section`, `roadmap`, `delivery`, `legal-document`
- **Auth & Onboarding** — `auth-stub`, `shared/onboarding`
- **Date/Time** — `date-time-picker`
- **Loading & Skeletons** — `loading`, `dynamic-skeleton`, `vendor-page-skeleton`, `DetailPageSkeleton`
- **Filters & Pagination** — `unified-filter-logic`, `unified-pagination`, `persistent-filter-controls`, `persistent-pagination`

## Usage Example

```typescript
// Import any component directly from the shared package entry point
import {
  EmptyState,
  VendorIcon,
  MspDisplay,
  DetailPageSkeleton,
  UnifiedPagination,
  PricingDisplay,
} from '@flamingo/components'

export function VendorPage({ vendor, loading }) {
  if (loading) return <DetailPageSkeleton />

  return (
    <>
      <VendorIcon vendor={vendor} />
      <MspDisplay msp={vendor.msp} />
      <PricingDisplay tiers={vendor.pricing} />
    </>
  )
}
```

> **Note:** This file is marked `"use client"` — all exported components are client-side React components intended for Next.js App Router usage. Server components should import from their source paths directly if needed.
