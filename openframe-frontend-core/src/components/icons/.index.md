Barrel export file for the **deprecated** icons v1 module, re-exporting all icon components from their individual files into a single entry point.

> ⚠️ **Deprecated** — Migrate to `@flamingo-stack/openframe-frontend-core/components/icons-v2` which provides 1600+ icons, a consistent API, better TypeScript support, and smaller bundle size.

## Key Components

Exports are organized into the following categories:

| Category | Examples |
|---|---|
| Logo icons | `FlamingoLogo`, `OpenFrameLogo`, `OpenmspLogo` |
| Navigation & feature | `SearchIcon`, `FilterIcon`, `DashboardIcon`, `DevicesIcon` |
| UI state | `CheckCircleIcon`, `ErrorIcon`, `CopyIcon`, `SendIcon` |
| Brand & social | `GitHubIcon`, `SlackIcon`, `MicrosoftIcon`, `GoogleLogo` |
| Social platforms | `YouTubeIcon`, `InstagramIcon`, `TelegramIcon` |
| OS icons | `WindowsIcon`, `MacOSIcon`, `LinuxIcon` |
| Shell types | `PowerShellIcon`, `BashIcon`, `PythonIcon`, `DenoIcon` |
| OpenFrame AI agents | `NaturalLanguageIcon`, `SmartEscalationIcon`, `AuditLoggingIcon` |
| Trust page | `ShieldLockIcon`, `FolderShieldIcon`, `ShieldCheckIcon` |
| Review platforms | `G2Icon`, `CapterraIcon`, `TrustpilotIcon` |
| Utilities | `icon-utils` (wildcard), `device-type-icons` (wildcard) |

Also exports `XIcon` aliased as `IconsXIcon` to avoid naming conflicts with `ui/custom-icons`.

## Usage Example

```typescript
// ❌ Deprecated — avoid
import { CopyIcon, FlamingoLogo } from '@flamingo-stack/openframe-frontend-core/components/icons';

// ✅ Recommended — use icons-v2 instead
import { CopyIcon, FlamingoLogo } from '@flamingo-stack/openframe-frontend-core/components/icons-v2';
```
