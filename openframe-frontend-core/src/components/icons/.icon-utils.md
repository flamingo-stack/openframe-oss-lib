A utility function for dynamically rendering SVG icons based on string names, supporting both Lucide React icons and custom platform logos with fallback handling.

## Key Components

- **`renderSvgIcon`**: Main function that takes an icon name and optional SVG props, returning the corresponding React element
- **Icon mapping**: Supports Lucide icons (`megaphone`, `bell`, `info`, `star`, `rocket`, `package`, `globe`) and custom logos (`openframe-logo`, `openmsp-logo`, `flamingo-logo`, `tmcg-logo`, etc.)
- **Fallback behavior**: Defaults to Megaphone icon when an unknown name is provided
- **Props spreading**: Passes through any additional SVG props to the rendered icon

## Usage Example

```typescript
import { renderSvgIcon } from './icon-utils';

// Render a Lucide icon with custom props
const notificationIcon = renderSvgIcon('bell', { 
  size: 24, 
  className: 'text-blue-500' 
});

// Render a platform logo
const platformLogo = renderSvgIcon('openframe-logo', { 
  width: 32, 
  height: 32 
});

// Unknown icon falls back to megaphone
const fallbackIcon = renderSvgIcon('unknown-icon');

// Use in JSX
return (
  <div>
    {renderSvgIcon('star', { className: 'icon-star' })}
    {renderSvgIcon('flamingo-logo')}
  </div>
);
```
