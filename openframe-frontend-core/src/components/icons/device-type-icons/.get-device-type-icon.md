Returns a React icon element corresponding to a given device type, mapping device categories to visual icons from the `icons-v2-generated` library.

> ⚠️ **Deprecated** — This utility and its associated types are deprecated. Use icons directly from `icons-v2-generated` instead.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `DeviceType` | Type alias | Union of `'desktop' \| 'laptop' \| 'mobile' \| 'tablet' \| 'server'` |
| `DeviceTypeIconProps` | Interface | Optional `className`, `color`, and `size` props passed to the icon |
| `getDeviceTypeIcon` | Function | Returns a `React.ReactElement` icon based on device type |

**Icon Mapping:**

| Device Type | Icon Rendered |
|-------------|---------------|
| `desktop` | `MonitorIcon` |
| `laptop` | `LaptopIcon` |
| `server` | `HardDriveIcon` |
| `mobile`, `tablet`, `undefined` | `MonitorIcon` (fallback) |

## Usage Example

```typescript
import { getDeviceTypeIcon } from './get-device-type-icon'

// Render a laptop icon with custom styles
const icon = getDeviceTypeIcon('laptop', {
  className: 'device-icon',
  color: '#FFC008',
  size: 24,
})

// Render a server icon
const serverIcon = getDeviceTypeIcon('server', { size: 20 })

// Falls back to MonitorIcon for unknown/mobile/tablet types
const fallbackIcon = getDeviceTypeIcon('mobile')
```
