A utility function that returns the appropriate device icon component based on device type, providing consistent iconography for different device categories across the application.

## Key Components

- **`DeviceType`** - Type union defining supported device categories: desktop, laptop, mobile, tablet, server
- **`DeviceTypeIconProps`** - Interface for icon styling properties (className, color, size)
- **`getDeviceTypeIcon()`** - Main function that maps device types to their corresponding icon components

## Icon Mapping

The function maps device types to specific icon components:
- `desktop` → `MonitorIcon`
- `laptop` → `LaptopIcon` 
- `server` → `HardDriveIcon`
- `mobile`, `tablet`, and fallback → `MonitorIcon`

## Usage Example

```typescript
import { getDeviceTypeIcon } from './get-device-type-icon'

// Basic usage
const desktopIcon = getDeviceTypeIcon('desktop')

// With custom styling
const styledLaptopIcon = getDeviceTypeIcon('laptop', {
  className: 'device-icon',
  color: '#3b82f6',
  size: 24
})

// In a component
function DeviceList({ devices }) {
  return (
    <div>
      {devices.map(device => (
        <div key={device.id}>
          {getDeviceTypeIcon(device.type, { size: 20 })}
          <span>{device.name}</span>
        </div>
      ))}
    </div>
  )
}
```

The function handles undefined device types gracefully by falling back to the monitor icon, ensuring consistent UI rendering even with missing or invalid device type data.
