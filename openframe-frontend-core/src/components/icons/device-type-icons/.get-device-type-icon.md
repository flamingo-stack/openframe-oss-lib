A utility function that returns the appropriate React icon component based on device type, with support for customizable styling props.

## Key Components

- **`DeviceType`** - Type definition for supported device categories: `'desktop' | 'laptop' | 'mobile' | 'tablet' | 'server'`
- **`DeviceTypeIconProps`** - Interface defining optional styling properties (className, color, size)
- **`getDeviceTypeIcon()`** - Main function that maps device types to their corresponding icon components

## Usage Example

```typescript
import { getDeviceTypeIcon, DeviceType } from './get-device-type-icon'

// Basic usage with default fallback
const defaultIcon = getDeviceTypeIcon()

// Specific device type with custom styling
const laptopIcon = getDeviceTypeIcon('laptop', {
  className: 'device-icon',
  color: '#2563eb',
  size: 24
})

// Server icon with large size
const serverIcon = getDeviceTypeIcon('server', {
  size: 32,
  className: 'server-status-icon'
})

// Mobile/tablet devices use the generic DevicesIcon
const mobileIcon = getDeviceTypeIcon('mobile', { color: '#059669' })
```

The function provides a consistent API for rendering device-specific icons while handling fallbacks gracefully. Mobile and tablet types default to the generic `DevicesIcon`, while desktop, laptop, and server have dedicated icon components.
