A wrapper component that manages loading state transitions for content areas, showing skeleton placeholders while data loads and revealing actual content once ready.

## Key Components

**`ContentLoadingContainer`** ‚Äî Main component that overlays a skeleton component during loading while keeping the content area hidden (not unmounted), enabling smooth opacity transitions and preventing layout jumps.

**`useContentLoading`** ‚Äî Utility hook providing ARIA attributes and content-type-aware helpers for skeleton count and minimum height values.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isLoading` | `boolean` | ‚Äî | Controls skeleton/content visibility |
| `skeletonComponent` | `ReactNode` | ‚Äî | Placeholder UI shown during loading |
| `minHeight` | `string` | `"min-h-[300px] md:min-h-[800px]"` | Prevents layout jumps |
| `loadingOpacity` | `number` | `1` | Skeleton overlay opacity (0‚Äì1) |
| `transitionDuration` | `number` | `300` | Fade transition in milliseconds |

## Usage Example

```typescript
import { ContentLoadingContainer, useContentLoading } from "./content-loading-container"
import { CardSkeletonGrid } from "./card-skeleton-grid"

function VendorPage() {
  const { isLoading, vendors } = useVendors()
  const { getMinHeight, getSkeletonCount } = useContentLoading(isLoading)

  return (
    <ContentLoadingContainer
      isLoading={isLoading}
      skeletonComponent={
        <CardSkeletonGrid
          variant="vendor"
          count={getSkeletonCount("vendor")}
        />
      }
      minHeight={getMinHeight("vendor")}
      transitionDuration={400}
    >
      <VendorGrid vendors={vendors} />
    </ContentLoadingContainer>
  )
}
```

> **Note:** Content is hidden via `opacity-0` and `aria-hidden` during loading but remains mounted in the DOM, avoiding remount costs when transitioning to the loaded state.
