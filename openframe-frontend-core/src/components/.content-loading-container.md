A unified container component that manages content loading states by overlaying skeleton components while keeping layout dimensions stable and preventing UI jumps.

## Key Components

### `ContentLoadingContainer`
Main exported component that wraps content with a skeleton overlay during loading states.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isLoading` | `boolean` | — | Controls loading/loaded state |
| `children` | `ReactNode` | — | Content rendered when loaded |
| `skeletonComponent` | `ReactNode` | — | Skeleton shown during loading |
| `className` | `string` | — | Additional CSS classes |
| `minHeight` | `string` | `"min-h-[300px] md:min-h-[800px]"` | Prevents layout jumps |
| `loadingOpacity` | `number` | `1` | Overlay opacity (0–1) |
| `transitionDuration` | `number` | `300` | Fade transition in ms |

### `useContentLoading`
Helper hook that returns ARIA container props and utility functions for common content types.

**Returns:**

| Member | Description |
|--------|-------------|
| `containerProps` | `aria-busy` and `data-loading` attributes |
| `getSkeletonCount(type)` | Returns `12` for `vendor`, `6` for `blog` |
| `getMinHeight(type)` | Returns type-appropriate `min-h` Tailwind class |

## Usage Example

```typescript
import { ContentLoadingContainer, useContentLoading } from "./content-loading-container"
import { CardSkeletonGrid } from "./card-skeleton-grid"

function VendorPage() {
  const { isLoading, vendors } = useVendors()
  const { getMinHeight, getSkeletonCount } = useContentLoading(isLoading)

  return (
    <ContentLoadingContainer
      isLoading={isLoading}
      skeletonComponent={
        <CardSkeletonGrid variant="vendor" count={getSkeletonCount("vendor")} />
      }
      minHeight={getMinHeight("vendor")}
      transitionDuration={400}
    >
      <VendorGrid vendors={vendors} />
    </ContentLoadingContainer>
  )
}
```

> The skeleton overlay uses `absolute` positioning over the container, while actual content fades from `opacity-0` to visible — both transitions respect `transitionDuration` for a smooth UX. Full ARIA support (`aria-live`, `aria-busy`, `role="status"`) ensures screen readers announce loading state changes.
