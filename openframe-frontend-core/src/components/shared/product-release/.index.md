Barrel export file for the product release UI component library, re-exporting all public components, props types, and shared types from the product release feature module.

## Key Components

| Export | Source | Description |
|--------|--------|-------------|
| `ProductReleaseCard` | `./product-release-card` | Card component for displaying a single product release |
| `ProductReleaseCardSkeleton` | `./product-release-card-skeleton` | Loading skeleton for `ProductReleaseCard` |
| `ProductReleasesView` | `./product-releases-view` | List/grid view rendering multiple release cards |
| `ReleaseDetailPage` | `./release-detail-page` | Full detail page for a single release |
| `ReleaseDetailSkeleton` | `./release-detail-skeleton` | Loading skeleton for `ReleaseDetailPage` |
| `DeliveryResponse` | `../../../types/delivery` | Canonical delivery response type (re-sourced to preserve deep-import path resolution) |

### Notable Type Exports

- `ProductReleaseCardProps`, `ProductReleaseCardSkeletonProps`, `ProductReleasesViewProps`
- `ProductReleaseCardExtras` — additional data slots for release cards
- `ReleaseDetailPageProps`, `VideoDisplaySectionProps`, `MarkdownRendererProps`, `RoadmapItem`

> **Note:** `RoadmapSectionProps` and `DeliverySection Props` are intentionally **not** re-exported here to avoid TS2308 ambiguous re-export conflicts with concrete component prop types in `./shared`. Import them directly from the `release-detail-page` module if needed.

## Usage Example

```typescript
import {
  ProductReleasesView,
  ReleaseDetailPage,
  ReleaseDetailSkeleton,
  type ProductReleaseCardExtras,
  type DeliveryResponse,
} from '@flamingo-stack/openframe-frontend-core/components/shared/product-release'
```

For slot types not exposed here:

```typescript
import type { RoadmapSectionProps } from '@flamingo-stack/openframe-frontend-core/components/shared/product-release/release-detail-page'
```
