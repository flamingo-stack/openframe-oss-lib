Barrel (client-marked) export file for the product release UI component library, re-exporting all public components, props types, and associated types from the `shared/product-release` module.

## Key Components

| Export | Source |
|--------|--------|
| `ProductReleaseCard` / `ProductReleaseCardProps` | `./product-release-card` |
| `ProductReleaseCardSkeleton` / `ProductReleaseCardSkeletonProps` | `./product-release-card-skeleton` |
| `ProductReleasesView` / `ProductReleasesViewProps` / `ProductReleaseCardExtras` | `./product-releases-view` |
| `ReleaseDetailPage` + related prop/section types | `./release-detail-page` |
| `ReleaseDetailSkeleton` | `./release-detail-skeleton` |
| `DeliveryResponse` | `../../../types/delivery` |

## Notable Omissions

`RoadmapSectionProps` and `DeliverySectionProps` are **intentionally excluded** from this barrel to avoid TS2308 ambiguous re-export collisions with the concrete `<RoadmapGrid>` / `<DeliverySection>` components at the top-level barrel. Consumers needing these slot types must import directly:

```typescript
import type { RoadmapSectionProps } from '@flamingo-stack/openframe-frontend-core/components/shared/product-release/release-detail-page'
```

## Usage Example

```typescript
import {
  ProductReleaseCard,
  ProductReleasesView,
  ReleaseDetailPage,
  ReleaseDetailSkeleton,
  type ProductReleaseCardProps,
  type DeliveryResponse,
} from '@flamingo-stack/openframe-frontend-core/components/shared/product-release'

// Render a release card
const card: ProductReleaseCardProps = { /* ... */ }
<ProductReleaseCard {...card} />

// Render full detail page with loading fallback
<Suspense fallback={<ReleaseDetailSkeleton />}>
  <ReleaseDetailPage releaseId="v2.1.0" />
</Suspense>
```

> All exports are tagged `"use client"` — this module is safe only in React Client Component trees (Next.js App Router).
