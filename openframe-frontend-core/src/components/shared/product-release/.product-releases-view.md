Self-contained releases list surface that fetches, paginates, and renders `ProductReleaseCard` entries from a configurable API endpoint, reading search/status/page state from URL params written by the section chrome.

## Key Components

### `ProductReleasesView` (main export)
Primary list component. Reads URL search params, constructs the fetch URL, and manages loading/error/empty/paginated states internally via `useSelfFetch`. Accepts `initialData` for SSR hydration to skip the first client round-trip.

### `ReleaseRow` (internal)
Per-release row component that isolates hook calls (`useChatRuntime`, `useEntityCardLink`, `useRouter`) outside the parent `.map()` to satisfy Rules of Hooks. Resolves the detail href via `resolveContentHref` and handles modifier-click–aware navigation via `executeNavigationImperative`.

### `ProductReleaseCardExtras` (type export)
Subset of `ProductReleaseCardProps` that `buildCardProps` must return — excludes fields the view supplies directly (`title`, `summary`, `version`, `size`, `anchorProps`, `onClick`, `className`).

## Key Props

| Prop | Default | Purpose |
|------|---------|---------|
| `endpoint` | `/api/releases` | API route for the list |
| `initialData` | — | SSR hydration payload |
| `itemsPerPage` | `5` | Page size + skeleton slot count |
| `basePath` | `/releases` | Fallback detail-href prefix |
| `buildCardProps` | `buildProductReleaseCardProps` | Per-card prop builder |
| `searchParamKey` | `'search'` | URL param key for text filter |
| `statusParamKey` | `'release_status'` | URL param key for status filter |

## Usage Example

```typescript
// Minimal — zero config, SSR-hydrated first page
import { ProductReleasesView } from './product-releases-view'
import type { ProductReleaseListResponse } from '../../../types/product-release'

async function ReleasesPage() {
  const initialData: ProductReleaseListResponse = await fetchReleases({ limit: 5, offset: 0 })
  return <ProductReleasesView initialData={initialData} />
}

// Custom endpoint + card props override
<ProductReleasesView
  endpoint="/api/v2/product-releases"
  basePath="/changelog"
  itemsPerPage={10}
  buildCardProps={(release) => ({
    formattedDate: formatDate(release.released_at),
    type: release.type,
    status: release.status,
  })}
/>
```
