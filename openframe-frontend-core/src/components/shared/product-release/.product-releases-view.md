Self-contained releases list surface that fetches, paginates, and renders product release cards driven entirely by URL search params — no external state management required.

## Key Components

### `ProductReleasesView` (main export)
The primary list component. Reads `search`, `release_status`, and `page` URL params written by the section chrome, fetches the configured endpoint via `useSelfFetch`, and renders a stable fixed-slot grid of `ProductReleaseCard` entries with loading skeletons and pagination.

### `ReleaseRow` (internal)
Per-release child component that isolates per-item hooks (`useChatRuntime`, `useEntityCardLink`, `useRouter`) out of the parent `.map()` to satisfy Rules of Hooks. Resolves the detail href via `resolveContentHref` and handles soft/hard navigation via `executeNavigationImperative`.

### `ProductReleaseCardExtras` (exported type)
Subset of `ProductReleaseCardProps` — the caller-supplied fields per release, excluding nav/layout props that the view owns internally.

## Key Behaviors

- **SSR hydration** — pass `initialData` to skip the first client fetch
- **Stable layout** — always renders `itemsPerPage` slots; uses `visibility` toggling to prevent height shifts during loading/empty transitions
- **Filter reset** — clears `search`, `release_status`, and `page` params atomically via `router.replace`
- **Custom card builder** — defaults to `buildProductReleaseCardProps` (rich `lg` metadata); override via `buildCardProps` prop

## Usage Example

```typescript
import { ProductReleasesView } from './product-releases-view'

// Minimal — uses all defaults
<ProductReleasesView />

// With SSR hydration and custom endpoint
<ProductReleasesView
  endpoint="/api/v2/releases"
  initialData={serverFetchedData}
  itemsPerPage={10}
  basePath="/changelog"
  buildCardProps={(release) => ({
    formattedDate: formatDate(release.published_at),
    type: release.type,
    status: release.status,
  })}
/>
```
