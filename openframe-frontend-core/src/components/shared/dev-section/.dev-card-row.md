Shared layout component for `DevSectionPage` list rows (delivery, tickets, and future sections), providing a consistent two-column structure with fixed min-height blocks for alignment across grids.

## Key Components

**`DevCardRowContent`** — Primary layout component rendering a left column (title/subtitle/description) and a right column for caller-supplied badges via `rightBadges: ReactNode`. No interaction behavior baked in — the caller wraps it in a `<div>` or `<button>` as needed.

**`DevCardRowSkeleton`** — Single-row loading placeholder that mirrors the exact `min-height` values of `DevCardRowContent`, preventing layout reflow when real data loads.

**`DevCardRowSkeletonList`** — Renders `n` skeleton rows (default: 5) inside a bordered card container. Used directly by `delivery-table.tsx` and `tickets-list.tsx`.

### `DevCardRowContentProps`

| Prop | Type | Description |
|------|------|-------------|
| `title` | `string` | Row heading (h3, 2-line clamp) |
| `subtitle` | `string` | Uppercase single-line metadata |
| `description` | `string` | 3-line clamped body text |
| `emptyDescription` | `string?` | Fallback when description is empty |
| `rightBadges` | `ReactNode` | Caller-controlled stacked badge column |

## Usage Example

```typescript
// Static row (delivery surface)
<div>
  <DevCardRowContent
    title="feat: add OAuth2 support"
    subtitle="UPDATED today, #4271, Code review"
    description="Implements OAuth2 flow for the client portal login."
    emptyDescription="No delivery description available"
    rightBadges={
      <>
        <StatusBadge status="In Review" />
        <PriorityBadge level="High" />
      </>
    }
  />
</div>

// Clickable row (tickets surface)
<button onClick={handleOpen}>
  <DevCardRowContent
    title="VPN not connecting after update"
    subtitle="OPENED 2h ago, #8821"
    description="User reports VPN fails immediately after the latest client update."
    rightBadges={<SeverityBadge level="Critical" />}
  />
</button>

// Loading state
<DevCardRowSkeletonList rows={5} />
```

> **Note:** Ticket conversation rows use `<ChatMessageRow>` (`components/chat/chat-message-row.tsx`) — the same component as the OpenMSP Slack community feed — and are intentionally not handled here.
