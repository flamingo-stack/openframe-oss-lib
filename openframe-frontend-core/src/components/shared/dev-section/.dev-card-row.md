Shared layout component for `DevSectionPage` list rows (delivery, tickets, and future sections), providing a consistent two-column structure with a fixed-height left column (title/subtitle/description) and a caller-supplied right column for stacked badges.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `DevCardRowContentProps` | Interface | Props for the row content component |
| `DevCardRowContent` | Component | Renders the two-column row chrome — left: text hierarchy; right: badge slot |
| `DevCardRowSkeleton` | Component | Single animated skeleton row mirroring `DevCardRowContent` min-heights |
| `DevCardRowSkeletonList` | Component | Bordered card wrapping `n` skeleton rows (default: 5) for list loading states |

## Usage Example

```typescript
// Static row (delivery surface)
<div className="border-b border-ods-border p-4">
  <DevCardRowContent
    title="Deploy v2.4.1 to production"
    subtitle="UPDATED today, #4271, Code review"
    description="Bumps the API gateway and rotates the signing key."
    emptyDescription="No delivery notes provided."
    rightBadges={
      <>
        <StatusBadge status="in-progress" />
        <PriorityBadge priority="high" />
      </>
    }
  />
</div>

// Clickable row (tickets surface)
<button onClick={openTicket}>
  <DevCardRowContent
    title="VPN drops on Windows 11 24H2"
    subtitle="OPEN, #1042, Networking"
    description=""
    rightBadges={<SeverityBadge level="critical" />}
  />
</button>

// Loading state for a list shell
<DevCardRowSkeletonList rows={8} />
```

> **Note:** Ticket conversation rows are intentionally excluded — they use the shared `<ChatMessageRow>` component (`components/chat/chat-message-row.tsx`), the same component powering the OpenMSP community feed, keeping both surfaces pixel-identical.
