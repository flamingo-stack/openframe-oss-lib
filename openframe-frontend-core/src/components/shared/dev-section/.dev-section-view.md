Canonical layout shell for dev-center sections (Roadmap, Delivery, Releases), handling title rendering, inline search, filter pills, and URL parameter wiring in both hero (full-page) and compact (tabbed) display modes.

## Key Components

**`DevSectionView`** — Primary exported component that:
- Renders a **hero** (icon + `h1` + description) or **compact heading** (`h2`) based on the `hero` prop
- Wires a `SearchInput` to URL params, committing only on Enter (not on every keystroke)
- Wires a `StatusFilterComponent` to URL params via `handleFilterChange`
- Exposes a `preControls` slot rendered between the hero and the search/filter row
- Delegates all list rendering to `children`, which read URL params independently via `useSearchParams()`

**`DevSectionViewProps`** — Interface defining:
| Prop | Required | Description |
|---|---|---|
| `sectionKey` | ✅ | Looks up config in `OPENFRAME_DEV_SECTIONS` registry |
| `hero` | ❌ | Enables full-page hero mode with icon, title, description |
| `preControls` | ❌ | Slot rendered above search/filter controls |
| `children` | ✅ | List body consuming URL search/filter params |

## Usage Example

```typescript
// Full-page hero mode (e.g. /roadmap)
<DevSectionView
  sectionKey="roadmap"
  hero={{
    icon: <RoadmapIcon />,
    title: "Product Roadmap",
    description: "Upcoming features and improvements."
  }}
  preControls={<NewTicketForm />}
>
  <RoadmapList />
</DevSectionView>

// Compact tab mode (e.g. inside /roadmap-and-releases)
<DevSectionView sectionKey="releases">
  <ReleasesList />
</DevSectionView>
```

> Search state is lazy-initialised from the URL to avoid a stale-value flash on tab switches. Filter resets to the default value by deleting the param rather than writing the default string.
