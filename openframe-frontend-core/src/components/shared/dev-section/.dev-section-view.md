Canonical chrome component for dev-center sections (Roadmap, Delivery, Releases), supporting both tabbed compact mode and full-page hero mode with unified search and filter URL-param wiring.

## Key Components

**`DevSectionView`** — Main exported component that renders section chrome including title, search input, filter pills, and an optional hero block. Drives child list components via URL search params (`?<paramKey>=...`).

**`DevSectionViewProps`** — Interface defining:
- `sectionKey` — Looks up section config from `OPENFRAME_DEV_SECTIONS` registry
- `hero` — Optional; when provided renders full-page mode with icon, `<h1>`, and description
- `preControls` — Optional slot rendered between hero and search/filter controls
- `children` — List body that reads URL params written by this component

## URL Param Contract

Search commits to the URL **on Enter only** (not on keystroke). Filter changes are applied immediately. Both use `router.replace()` with `scroll: false` to avoid layout jumps. Children read params via `useSearchParams()`.

## Usage Example

```typescript
// Full-page hero mode
<DevSectionView
  sectionKey="roadmap"
  hero={{
    icon: <RoadmapIcon />,
    title: "Product Roadmap",
    description: "See what we're building next.",
  }}
  preControls={<NewTicketForm />}
>
  <RoadmapList />
</DevSectionView>

// Compact tab mode (no hero prop)
<DevSectionView sectionKey="releases">
  <ReleasesList />
</DevSectionView>
```
