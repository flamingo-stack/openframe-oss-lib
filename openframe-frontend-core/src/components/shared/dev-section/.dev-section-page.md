Full-page wrapper component for OpenFrame dev-center sections (`/roadmap`, `/bug-fixes-and-enhancements`, `/releases`), composing `PageShell` → `PageLayout` → `DevSectionView` with a configurable back-button, hero, and optional pre-controls slot.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `DevSectionPage` | Component | Main factory component for rendering any registered dev-center section page |
| `DevSectionPageProps` | Interface | Props contract for `DevSectionPage` |

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `sectionKey` | `OpenframeDevSectionKey` | required | Key into `OPENFRAME_DEV_SECTIONS` registry |
| `children` | `ReactNode` | required | Section-specific list body (e.g. `<RoadmapList />`) |
| `preControls` | `ReactNode` | — | Slot rendered between hero and search/filter controls |
| `backButton` | `{ label?, href? } \| false` | `{ label: 'Back to home', href: '/' }` | Back-button config; pass `false` to hide |
| `title` | `string` | `OPENFRAME_DEV_SECTIONS[key].hero.title` | Hero title override for non-OpenFrame embeds |
| `subtitle` | `string` | `OPENFRAME_DEV_SECTIONS[key].hero.description` | Hero subtitle override |

## Usage Example

```typescript
// pages/roadmap/page.tsx — minimal, uses all defaults
export default function RoadmapPage() {
  return (
    <DevSectionPage sectionKey="roadmap">
      <RoadmapList />
    </DevSectionPage>
  );
}

// Custom embed — overrides branding and back-button destination
export default function CustomRoadmapPage() {
  return (
    <DevSectionPage
      sectionKey="roadmap"
      title="Product Roadmap"
      subtitle="See what we're building next."
      backButton={{ label: 'Back to dashboard', href: '/dashboard' }}
      preControls={<NewTicketForm />}
    >
      <RoadmapList />
    </DevSectionPage>
  );
}
```

> **Extending:** Adding a new section requires only one entry in `OPENFRAME_DEV_SECTIONS` and a single page file mounting `DevSectionPage` with the new `sectionKey`. No changes to this component are needed.
