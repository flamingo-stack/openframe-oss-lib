Full-page wrapper component for dev-center sections (`/roadmap`, `/bug-fixes-and-enhancements`, `/releases`), composing `PageShell` → `PageLayout` → `DevSectionView` with a configurable hero, back-button, and optional pre-controls slot.

## Key Components

**`DevSectionPage`** — Default export component that renders the complete page chrome for an OpenFrame dev-center section.

**`DevSectionPageProps`** — Interface exposing:
- `sectionKey` — key from `OPENFRAME_DEV_SECTIONS` that drives the icon, title, and description defaults
- `children` — the page-specific list body (e.g. `<RoadmapList />`)
- `preControls` — optional slot rendered between the hero and search/filter controls
- `backButton` — back-button label/href override, or `false` to hide entirely (defaults to `{ label: 'Back to home', href: '/' }`)
- `title` / `subtitle` — override the section's default hero copy for non-OpenFrame embeds

## Usage Example

```typescript
// pages/roadmap/page.tsx — minimal usage
import { DevSectionPage } from '@/components/dev-section-page';
import { RoadmapList } from '@/components/roadmap-list';

export default function RoadmapPage() {
  return (
    <DevSectionPage sectionKey="roadmap">
      <RoadmapList />
    </DevSectionPage>
  );
}

// With overrides for a white-label embed
export default function CustomRoadmapPage() {
  return (
    <DevSectionPage
      sectionKey="roadmap"
      title="Product Roadmap"
      subtitle="See what we're building next."
      backButton={{ label: 'Back to dashboard', href: '/dashboard' }}
      preControls={<NewTicketForm />}
    >
      <RoadmapList />
    </DevSectionPage>
  );
}
```

## Notes

Adding a new section requires only one entry in `OPENFRAME_DEV_SECTIONS` plus a single-line page file — no changes to this component. The `PageLayout` back-button affordance is consumed directly from the design system, so lib-level updates propagate automatically.
