Shared roadmap list surface component that renders roadmap items either as a flat 2-column grid or grouped into collapsible per-quarter accordions, with centralized voting state shared across all quarter sections.

## Key Components

**`RoadmapGrid`** — Primary export. Accepts `items: RoadmapItem[]` and orchestrates both rendering modes and voting logic.

**`RoadmapGridSingle`** — Internal flat 2-col grid for a single set of items. Receives voting callbacks from the parent to avoid state races between quarters.

**Quarter helpers** — Pure utility functions for parsing (`parseQuarterString`), comparing (`compareQuarters`), and computing default expanded quarters (`computeDefaultExpandedQuarters`).

**`useRoadmapVoting`** — External hook managing vote state and toggle logic; instantiated once at the `RoadmapGrid` level.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `RoadmapItem[]` | — | Items to render |
| `groupByQuarter` | `boolean` | `false` | Enables per-quarter accordion grouping |
| `hasActiveFilters` | `boolean` | `false` | Expands all quarters when search/filters are active |
| `quartersToKeepClosed` | `number` | `2` | Past quarters within this window stay open by default |
| `showLeftMargin` | `boolean` | `true` | Toggles the `~120px` left margin for page alignment |
| `buildRefreshUrl` | `(taskId: string) => string` | `/api/roadmap/:id` | URL builder for post-vote item refresh |
| `onItemUpdate` | `(item: RoadmapItem) => void` | — | Callback to patch parent list after a vote |

## Usage Example

```typescript
// Full-page roadmap view with quarter grouping
<RoadmapGrid
  items={roadmapItems}
  groupByQuarter={true}
  hasActiveFilters={searchQuery.length > 0}
  quartersToKeepClosed={2}
  onItemUpdate={(updated) => setItems(prev =>
    prev.map(i => i.id === updated.id ? updated : i)
  )}
  votingOptions={{ endpoint: '/api/votes', storageKey: 'roadmap-votes' }}
/>

// Related-content rail (flat, no accordion)
<RoadmapGrid
  items={relatedItems}
  showLeftMargin={false}
  groupByQuarter={false}
/>
```

> **Hydration note:** `expandedQuarters` initializes as `[]` and is populated in a client-only `useEffect`, ensuring SSR markup matches the first client paint before accordion state is computed.
