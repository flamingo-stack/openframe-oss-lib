Shared roadmap list surface that renders roadmap items either as a flat 2-column grid or grouped into collapsible per-quarter accordions, with unified voting state across all sections.

## Key Components

- **`RoadmapGrid`** — Main export. Handles quarter bucketing, chronological sorting, accordion expand/collapse state (hydration-safe), and shared voting logic.
- **`RoadmapGridSingle`** — Internal flat 2-col grid for a single set of items; receives voting state from parent to prevent race conditions across quarters.
- **`computeDefaultExpandedQuarters`** — Pure helper that determines which quarters expand by default (current + future quarters, recent past within `quartersToKeepClosed` window).
- **`useRoadmapVoting`** — External hook providing `getVote` / `toggleVote`; instantiated once at `RoadmapGrid` level and shared downward.

## Usage Example

```typescript
// Full-page grouped mode (e.g. RoadmapView)
<RoadmapGrid
  items={roadmapItems}
  groupByQuarter={true}
  hasActiveFilters={searchQuery.length > 0}
  quartersToKeepClosed={2}
  onItemUpdate={(updated) => patchList(updated)}
  votingOptions={{ endpoint: '/api/roadmap/vote', storageKey: 'roadmap-votes' }}
  buildRefreshUrl={(taskId) => `/api/roadmap/${taskId}`}
/>

// Flat mode for related-content rails (existing callers unchanged)
<RoadmapGrid
  items={filteredItems}
  showLeftMargin={false}
  groupByQuarter={false}
/>
```

## Behavior Notes

| Prop | Default | Effect |
|---|---|---|
| `groupByQuarter` | `false` | `true` enables accordion quarter grouping |
| `hasActiveFilters` | `false` | `true` forces all quarters to expand |
| `quartersToKeepClosed` | `2` | Past quarters older than N quarters collapse by default |
| `showLeftMargin` | `true` | Pass `false` for embedded/rail contexts |
