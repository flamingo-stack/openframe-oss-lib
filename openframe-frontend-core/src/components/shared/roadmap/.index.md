Barrel export file for the shared roadmap surface module, carefully managing re-export paths to avoid TypeScript TS2308 ambiguous re-export conflicts.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `RoadmapGrid` / `RoadmapGridProps` | Component | Grid layout for displaying roadmap items |
| `RoadmapGridSkeleton` / `RoadmapGridSkeletonProps` | Component | Loading skeleton state for the roadmap grid |
| `useRoadmapVoting` / `VoteState` / `UseRoadmapVotingOptions` | Hook + Types | Voting logic and state management for roadmap items |
| `RoadmapView` / `RoadmapViewProps` | Component | Self-fetching roadmap surface — host only configures API routes |

## Usage Example

```typescript
import {
  RoadmapGrid,
  RoadmapGridSkeleton,
  RoadmapView,
  useRoadmapVoting,
  type VoteState,
  type UseRoadmapVotingOptions,
} from '@flamingo-stack/openframe-frontend-core/components/roadmap';

// Canonical `RoadmapItem` type must be imported separately
import type { RoadmapItem } from '@flamingo-stack/openframe-frontend-core/components/chat';
// or from:
// '@flamingo-stack/openframe-frontend-core/components'
```

## ⚠️ Intentional Omissions

**`RoadmapItem`** — Not re-exported here to prevent a third export path. Import from:
- `@flamingo-stack/openframe-frontend-core/components/chat` *(source of truth)*
- `@flamingo-stack/openframe-frontend-core/components`

**`VoteType`** — Not re-exported here to avoid TS2308 conflict with the same type already exported from `./chat`. Import the canonical `VoteType` from `@flamingo-stack/openframe-frontend-core/components/chat`.
