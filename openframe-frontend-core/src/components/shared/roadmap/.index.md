Barrel export file for the shared roadmap surface, re-exporting grid, skeleton, voting hook, and view components while deliberately excluding `RoadmapItem` and `VoteType` to avoid TypeScript TS2308 ambiguous re-export conflicts.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `RoadmapGrid` / `RoadmapGridProps` | Component | Grid layout for displaying roadmap items |
| `RoadmapGridSkeleton` / `RoadmapGridSkeletonProps` | Component | Loading skeleton for the roadmap grid |
| `useRoadmapVoting` / `VoteState` / `UseRoadmapVotingOptions` | Hook + Types | Voting logic and state management for roadmap items |
| `RoadmapView` / `RoadmapViewProps` | Component | Self-fetching roadmap surface; host only configures API routes |

## Intentional Omissions

- **`RoadmapItem`** — already exported via two canonical paths; a third would trigger TS2308
- **`VoteType`** — canonical export lives in `@flamingo-stack/openframe-frontend-core/components/chat`

## Usage Example

```typescript
// Import roadmap surface components
import {
  RoadmapGrid,
  RoadmapGridSkeleton,
  RoadmapView,
  useRoadmapVoting,
} from '@flamingo-stack/openframe-frontend-core/components/roadmap';

// Import excluded types from their canonical paths
import type { RoadmapItem } from '@flamingo-stack/openframe-frontend-core/components/chat';
import type { VoteType } from '@flamingo-stack/openframe-frontend-core/components/chat';
```

> **Note:** Never import `RoadmapItem` or `VoteType` from this barrel. Use the canonical paths above to avoid ambiguous re-export errors across the component barrel hierarchy.
