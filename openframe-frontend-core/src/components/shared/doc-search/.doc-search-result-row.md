Renders a single search result row for doc-search dropdowns, providing a consistent visual layout across all search surfaces (company hub data-room, onboarding-guide catalog, etc.).

## Key Components

- **`DocSearchResultRowEntry`** — Minimal result shape interface compatible with any `useDocSearch` hook result exposing `title?`, `path?`, and `metadata?`
- **`DocSearchResultRowProps`** — Props interface accepting a `result` entry and `isHighlighted` boolean
- **`DocSearchResultRow`** — Main component rendering the icon, title, and relative path for a single dropdown row

## Behavior

- Resolves the source icon via `resolveSourceIcon()` using `metadata.sourceRepo` and `metadata.documentType`, keeping icon appearance in sync with inline chat-card references
- Highlights the title in accent color (`text-ods-accent`) when `isHighlighted` is `true`
- Suppresses the path subtitle for group entries (`metadata.isGroup`) or flat paths without a `/`
- Formats the path display using `formatRelativePath()` for a cleaner breadcrumb-style label

## Usage Example

```typescript
import { DocSearchResultRow } from './doc-search-result-row'

const result = {
  title: 'Onboarding Guide',
  path: 'guides/onboarding/getting-started',
  metadata: {
    documentType: 'onboarding-guide',
    sourceRepo: 'internal-docs',
  },
}

<DocSearchResultRow result={result} isHighlighted={false} />
```
