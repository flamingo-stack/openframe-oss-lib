Renders a single search result row for doc-search dropdowns, providing a consistent visual layout across all search surfaces in the platform (company hub data-room, onboarding-guide catalog, etc.).

## Key Components

**Exports**
- `DocSearchResultRowEntry` — minimal result shape interface compatible with any `useDocSearch` hook result (`title?`, `path?`, `metadata?`)
- `DocSearchResultRowProps` — component props interface (`result`, `isHighlighted`)
- `DocSearchResultRow` — the row component itself

**Internal behavior**
- Resolves the source icon via `resolveSourceIcon()` using `metadata.sourceRepo` and `metadata.documentType`, guaranteeing icon parity with inline chat-card references
- Formats sub-path breadcrumbs via `formatRelativePath()`, hidden for group entries or flat paths
- Applies `text-ods-accent` on highlighted state, `text-ods-text-primary` otherwise

## Usage Example

```typescript
import { DocSearchResultRow } from './doc-search-result-row'

const result = {
  title: 'Onboarding Guide',
  path: 'guides/onboarding/index',
  metadata: {
    documentType: 'guide',
    sourceRepo: 'internal-docs',
    isGroup: false,
  },
}

<DocSearchResultRow
  result={result}
  isHighlighted={activeIndex === index}
/>
```

> **Note:** The `DocSearchResultRowEntry` interface is intentionally minimal — any `useDocSearch` hook result satisfies it as long as it exposes `title?`, `path?`, and `metadata?`, making this row drop-in compatible across all hub search surfaces without adaptation.
