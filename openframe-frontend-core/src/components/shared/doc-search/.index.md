Public API barrel file for the doc-search package, re-exporting all components, hooks, utilities, and types needed to embed documentation search functionality in any consumer application.

## Key Components

| Export | Source | Description |
|--------|--------|-------------|
| `DocSearchBar` / `DocSearchBarProps` | `./doc-search-bar` | Search input UI component |
| `DocSearchResultRow` / `DocSearchResultRowProps` / `DocSearchResultRowEntry` | `./doc-search-result-row` | Individual search result renderer |
| `useDocSearch` / `UseDocSearchConfig` | `./use-doc-search` | Hook encapsulating debounced fetch + result navigation logic |
| `resolveSearchResultAction` / `SearchResultAction` | `./resolve-search-result-action` | Determines the navigation action for a given result |
| `mapDocSearchResults` | `./map-doc-search-results` | Transforms raw API results into display-ready entries |
| `formatRelativePath` | `./format-relative-path` | Utility to normalize/display doc paths |
| `DocSearchResult` | `./types` | Shared result type |

## Usage Example

```typescript
import {
  DocSearchBar,
  DocSearchResultRow,
  useDocSearch,
  resolveSearchResultAction,
  type DocSearchResult,
} from '@flamingo/doc-search'

function SearchPanel() {
  const { query, setQuery, results, isLoading } = useDocSearch({
    debounceMs: 300,
    onNavigate: (path) => router.push(path),
  })

  return (
    <>
      <DocSearchBar value={query} onChange={setQuery} loading={isLoading} />
      {results.map((result: DocSearchResult) => (
        <DocSearchResultRow
          key={result.id}
          entry={result}
          action={resolveSearchResultAction(result)}
        />
      ))}
    </>
  )
}
```

> **Note:** `useDocSearch` was migrated from the hub's `hooks/use-docs.ts` so embedders can mount the search bar directly without re-implementing debounced fetch or result navigation logic.
