Public API barrel file for the `doc-search` module, re-exporting all components, hooks, utilities, and types needed to embed document search functionality in any consumer application.

## Key Components

| Export | Source | Description |
|--------|--------|-------------|
| `DocSearchBar` | `./doc-search-bar` | Search input UI component |
| `DocSearchResultRow` | `./doc-search-result-row` | Single search result display component |
| `formatRelativePath` | `./format-relative-path` | Utility to normalize doc paths for display |
| `useDocSearch` | `./use-doc-search` | Hook encapsulating debounced fetch + result navigation logic |
| `resolveSearchResultAction` | `./resolve-search-result-action` | Resolves the navigation action for a given result |
| `mapDocSearchResults` | `./map-doc-search-results` | Maps raw API results to typed `DocSearchResult` entries |
| `DocSearchResult` | `./types` | Core result type shared across the module |

## Usage Example

```typescript
import {
  DocSearchBar,
  DocSearchResultRow,
  useDocSearch,
  type UseDocSearchConfig,
  type DocSearchResult,
} from '@flamingo/doc-search'

const config: UseDocSearchConfig = {
  endpoint: '/api/docs/search',
  debounceMs: 300,
}

function DocSearch() {
  const { query, setQuery, results, navigate } = useDocSearch(config)

  return (
    <>
      <DocSearchBar value={query} onChange={setQuery} />
      {results.map((result: DocSearchResult) => (
        <DocSearchResultRow
          key={result.id}
          entry={result}
          onSelect={() => navigate(result)}
        />
      ))}
    </>
  )
}
```

> **Note:** `useDocSearch` was migrated from the hub's `hooks/use-docs.ts` so embedders can mount the search bar directly without re-implementing debounced fetch or result navigation logic.
