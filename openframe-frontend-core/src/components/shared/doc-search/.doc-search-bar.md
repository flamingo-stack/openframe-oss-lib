Presentation wrapper that connects `<SearchInput>` with `<DocSearchResultRow>` to create a consistent RAG-search dropdown used across doc-search surfaces (data-room sidebar, onboarding-guide catalog, etc.).

## Key Components

### `DocSearchBar` (default export)
Stateless component that delegates all rendering to `<SearchInput>` while defaulting the row renderer to `<DocSearchResultRow>`. Data-fetching stays hub-side via `useDocSearch` — this component only accepts its output as props.

### `DocSearchBarProps`
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `query` | `string` | — | Current search string |
| `onQueryChange` | `(value: string) => void` | — | Controlled input handler |
| `results` | `SearchResult[]` | — | Hook-fetched results |
| `isLoading` | `boolean` | — | Loading state for spinner |
| `onResultSelect` | `(result, modifiers?) => void` | — | Selection handler with modifier key support |
| `showDropdown` | `boolean?` | `undefined` | Force dropdown open state |
| `minQueryLength` | `number` | `2` | Minimum chars before search fires |
| `debounceMs` | `number` | `0` | Input debounce (consumers debounce in the hook) |
| `renderResult` | `(result, isHighlighted) => ReactNode` | `<DocSearchResultRow>` | Custom row renderer override |

## Usage Example

```typescript
// Minimal usage — hub-side hook feeds props directly
const { query, setQuery, results, isLoading, handleSelect } = useDocSearch()

return (
  <DocSearchBar
    placeholder="Search docs..."
    query={query}
    onQueryChange={setQuery}
    results={results}
    isLoading={isLoading}
    onResultSelect={handleSelect}
  />
)

// With custom row renderer override
<DocSearchBar
  placeholder="Find a guide..."
  query={query}
  onQueryChange={setQuery}
  results={results}
  isLoading={isLoading}
  onResultSelect={handleSelect}
  renderResult={(result, isHighlighted) => (
    <MyCustomRow result={result} active={isHighlighted} />
  )}
/>
```

> **Design note:** Modifier keys (`metaKey`, `ctrlKey`, `shiftKey`) passed to `onResultSelect` allow hub-side logic to short-circuit to `window.open()` on cmd/ctrl-click, preserving standard browser tab behavior.
