Wire shape for a single document search result returned by `/api/docs/search`.

## Key Components

### `DocSearchResult` interface

| Field | Type | Description |
|-------|------|-------------|
| `path` | `string` | Relative path to the document |
| `name` | `string` | Display name of the result |
| `type` | `'file' \| 'folder'` | Whether the result is a file or folder |
| `snippet` | `string` | Matched text excerpt |
| `matchType` | `'title' \| 'content'` | Where the match was found |
| `documentType` | `string?` | Optional content category |
| `externalUrl` | `string?` | Canonical viewer URL when hosted externally |
| `targetPlatform` | `string \| null?` | Platform owning `externalUrl`; used by `decideNewTab` to determine tab behavior |
| `sourceRepo` | `string?` | Repository the document originates from |
| `entityId` | `string?` | Primary key from `RagTableConfig`; enables chat panel drill-in via `entityIdFilter` for rows without a public viewer |

## Usage Example

```typescript
import type { DocSearchResult } from './types'

const result: DocSearchResult = {
  path: '/docs/onboarding/setup',
  name: 'Setup Guide',
  type: 'file',
  snippet: '...install dependencies before running...',
  matchType: 'content',
  documentType: 'guide',
  externalUrl: 'https://notion.so/workspace/setup',
  targetPlatform: 'notion',
}

// Entity row with no public viewer — open chat drill-in instead
const entityResult: DocSearchResult = {
  path: 'cap_table/row/42',
  name: 'Series A Round',
  type: 'file',
  snippet: 'Preferred shares: 1,200,000',
  matchType: 'content',
  entityId: '42',
}
```

> **Note:** This interface must remain field-for-field identical with `hub/lib/data/doc-search-utils.ts:DocSearchResult` — it is the shared contract between the hub's RAG layer and the consuming lib hook.
