Transforms RAG `/api/docs/search` wire results into `SearchResult` rows for the `<DocSearchBar>` dropdown, collapsing financial entity-table rows into single grouped entries.

## Key Components

**`SEARCH_GROUP_REPOS`** — Set of financial source repos (`financial-cap-table`, `financial-kpis`, `financial-pnl`, `financial-balance-sheet`, `financial-cash-flow`) whose rows are collapsed into a single grouped result.

**`ENTITY_LABELS`** — Display name map for known `sourceRepo` values (both grouped financial tables and individual content types like blog posts, webinars, etc.).

**`mapDocSearchResults(docs)`** — Pure transform function. Iterates `DocSearchResult[]`, groups financial table rows by `sourceRepo` (preserving the insertion order of the first occurrence), and returns a flat `SearchResult[]` where:
- Grouped repos → one row: `"Cap Table (12 records)"` with `metadata.isGroup = true` and a nested `items[]` array.
- All other docs → individual rows with `snippet`, `matchType`, `externalUrl`, and other metadata forwarded as-is.

## Usage Example

```typescript
import { mapDocSearchResults } from './map-doc-search-results'

// Raw results from GET /api/docs/search
const rawDocs: DocSearchResult[] = await fetchDocSearch(query)

// Ready for <DocSearchBar> dropdown
const rows: SearchResult[] = mapDocSearchResults(rawDocs)

// Grouped financial table example output:
// {
//   id: 'group-financial-cap-table',
//   title: 'Cap Table (5 records)',
//   metadata: { isGroup: true, items: [...] }
// }
```

> **Note:** This is a pure transform with no side effects. Callers that need telemetry or logging should wrap `mapDocSearchResults` rather than modifying it directly.
