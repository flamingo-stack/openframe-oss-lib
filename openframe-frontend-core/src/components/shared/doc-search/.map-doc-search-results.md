Maps RAG `/api/docs/search` API results into `SearchResult` rows for the `<DocSearchBar>` dropdown, collapsing multiple financial table records into single grouped entries to reduce visual noise.

## Key Components

**`SEARCH_GROUP_REPOS`** — Set of source repos whose results should be collapsed into a single grouped row (financial tables only: cap table, KPIs, P&L, balance sheets, cash flow).

**`ENTITY_LABELS`** — Display name map from `sourceRepo` slug to human-readable label (covers both grouped and individual entity types).

**`mapDocSearchResults(docs)`** — Pure transform function that:
1. Iterates docs, routing financial-table rows into `entityGroups` while preserving insertion order via an `order` array
2. Collapses grouped repos into a single `SearchResult` with a `"Cap Table (12 records)"` title, `isGroup: true`, and an `items` array of the individual rows
3. Passes non-grouped docs through as individual `SearchResult` rows with `matchType`, `documentType`, `externalUrl`, and other metadata spread conditionally

## Usage Example

```typescript
import { mapDocSearchResults } from './map-doc-search-results'
import type { DocSearchResult } from './types'

const rawResults: DocSearchResult[] = await fetch('/api/docs/search?q=revenue').then(r => r.json())

// Collapses e.g. 8 financial-pnl rows into one grouped entry;
// blog posts, markdown docs, etc. remain as individual rows.
const dropdownRows = mapDocSearchResults(rawResults)

// Grouped row example:
// {
//   id: 'group-financial-pnl',
//   title: 'Profit & Loss (8 records)',
//   metadata: { isGroup: true, items: [...] }
// }
```

> **Note:** This is a pure transform with no side effects. Callers requiring telemetry or navigation handling should wrap this function rather than expecting it internally.
