Debounced RAG search hook that fetches results from `/api/docs/search` and resolves result-click navigation across same-tab, new-tab, embed-mode, and AI-dispatch scenarios.

## Key Components

**`UseDocSearchConfig`** — Configuration interface with the following fields:

| Field | Required | Description |
|-------|----------|-------------|
| `source` | ✅ | Discriminator forwarded as `?source=` query param |
| `baseRoute` | ✅ | Route prefix used to detect in-page-swappable results |
| `onNavigate` | ✅ | Imperative fallback navigation (e.g. `router.push`) |
| `tableIds` | ❌ | Narrow search to specific RAG table IDs |
| `onInPageSwap` | ❌ | Hub callback for in-page doc-tree swaps; returning `true` suppresses router push |
| `searchEndpoint` | ❌ | Override for the search API path (default: `/api/docs/search`) |

**`useDocSearch(config)`** — The hook itself. Returns:

- `query` / `setQuery` — controlled input state
- `results` — mapped `SearchResult[]` from the API
- `isLoading` — true when debounce is pending or fetch is in-flight
- `handleResultSelect` — click/keyboard handler; resolves one of five action kinds: `navigate-same-tab`, `navigate-new-tab`, `ask-ai`, `route`, `noop`
- `keepDropdownOpen` — stays `true` after new-tab opens so users can pick another result

## Usage Example

```typescript
import { useDocSearch } from './use-doc-search'
import { useRouter } from 'next/navigation'

function MySearchBar() {
  const router = useRouter()

  const { query, setQuery, results, isLoading, handleResultSelect } =
    useDocSearch({
      source: 'openframe',
      baseRoute: '/onboarding-guides',
      tableIds: ['onboarding-guides'],
      onNavigate: (path) => router.push(path),
      onInPageSwap: (path) => {
        // return true if handled in-page, false to fall through
        return docNavigation.navigate(path)
      },
    })

  return (
    <SearchInput
      value={query}
      onChange={setQuery}
      results={results}
      loading={isLoading}
      onSelect={handleResultSelect}
    />
  )
}
```

> **Embed-mode behavior:** When `useChatRuntime()` returns `mode: 'embed'`, same-tab results are intercepted by `resolveExternalNavigation` and opened via the host app's navigation bridge rather than `router.push`.
