Debounced RAG search hook that fetches results from `/api/docs/search` and handles multi-mode navigation (same-tab, new-tab, in-page swap, AI dispatch, embed-mode).

## Key Components

**`UseDocSearchConfig`** — Configuration interface with the following options:

| Option | Required | Description |
|---|---|---|
| `source` | ✅ | Discriminator forwarded as `?source=` query param |
| `baseRoute` | ✅ | Route prefix used to detect in-page-swappable results |
| `onNavigate` | ✅ | Fallback navigation callback (e.g. `router.push`) |
| `tableIds` | ❌ | Narrow search to specific RAG table IDs |
| `onInPageSwap` | ❌ | Hub callback for soft doc-tree swaps; returns `true` if handled |
| `searchEndpoint` | ❌ | Defaults to `'/api/docs/search'` |

**`useDocSearch(config)`** — Returns:
- `query` / `setQuery` — controlled search input state
- `results` — mapped `SearchResult[]`
- `isLoading` — unified loading state (debounce lag + in-flight fetch)
- `handleResultSelect` — click/keyboard handler resolving one of five action kinds: `navigate-same-tab`, `navigate-new-tab`, `ask-ai`, `route`, `noop`
- `keepDropdownOpen` — `true` when a new-tab navigation occurred so the dropdown stays visible

## Usage Example

```typescript
// Hub usage — in-page swap wired to doc-tree navigation
const { query, setQuery, results, isLoading, handleResultSelect, keepDropdownOpen } =
  useDocSearch({
    source: 'openframe',
    baseRoute: '/onboarding-guides',
    onNavigate: (path) => router.push(path),
    tableIds: ['onboarding-guides'],
    onInPageSwap: (path) => docNavigation.navigate(path),
  })

// Bare embedder — minimal config, falls back to window.location
const search = useDocSearch({
  source: 'openframe',
  baseRoute: '/docs',
  onNavigate: (path) => window.location.assign(path),
  searchEndpoint: '/proxy/api/docs/search',
})
```

> **Note:** Modifier-key handling (`metaKey`, `ctrlKey`, `shiftKey`) forces new-tab behavior on `<div>`-based rows to match native `<a target="_blank">` semantics.
