Resolves what action to perform when a user selects a search result, returning one of five typed discriminated union actions for a single `switch` dispatch.

## Key Components

### `SearchResultAction` (union type)
Discriminated union with five variants:
- `navigate-same-tab` — external URL, open in current tab
- `navigate-new-tab` — external URL, open in new tab
- `ask-ai` — entity drill-in via AI chat using primary key metadata
- `route` — legacy path-based navigation fallback
- `noop` — nothing actionable found

### `resolveSearchResultAction(result, source, runtimeMode?)`
Pure function (no React, no side effects) that inspects a `SearchResult`'s metadata and applies a priority-ordered resolution chain:

1. `metadata.externalUrl` present → delegates to `decideNewTab()` to determine same-tab vs. new-tab based on `targetPlatform`
2. `metadata.id` + `metadata.sourceRepo` + `metadata.documentType` all present → returns `ask-ai` with a `ChatRef`
3. `result.path` present → returns `route` (legacy fallback)
4. Nothing matched → returns `noop`

## Usage Example

```typescript
import { resolveSearchResultAction } from './resolve-search-result-action'

const action = resolveSearchResultAction(searchResult, 'zabbix', 'host')

switch (action.kind) {
  case 'navigate-same-tab':
    router.push(action.href)
    break
  case 'navigate-new-tab':
    window.open(action.href, '_blank')
    break
  case 'ask-ai':
    chatPanel.open(action.detail)
    break
  case 'route':
    router.push(action.path)
    break
  case 'noop':
    break
}
```

> **Note:** The `source` parameter acts as `currentSource` in `decideNewTab`, preserving platform-vs-platform comparison behavior consistent with pre-migration hub logic.
