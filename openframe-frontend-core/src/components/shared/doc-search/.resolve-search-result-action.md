Resolves the appropriate navigation or interaction action when a user selects a search result, returning one of five typed discriminated union actions for a single `switch` dispatch.

## Key Components

### `SearchResultAction` (exported union type)

| Kind | Payload | Description |
|------|---------|-------------|
| `navigate-same-tab` | `href: string` | External URL, same tab |
| `navigate-new-tab` | `href: string` | External URL, new tab |
| `ask-ai` | `detail: { source, ref: ChatRef }` | Entity drill-in via AI chat |
| `route` | `path: string` | Legacy internal path navigation |
| `noop` | — | No actionable target found |

### `resolveSearchResultAction(result, source, runtimeMode?)`

Pure function — no React, no side effects. Evaluates `result.metadata` in priority order:

1. **`externalUrl`** present → delegates to `decideNewTab` using `targetPlatform` and `runtimeMode` to choose same-tab vs new-tab
2. **`id` + `sourceRepo` + `documentType`** all present → returns `ask-ai` with a `ChatRef` built from the row's primary key
3. **`result.path`** present → returns legacy `route` action
4. **Nothing actionable** → returns `noop`

## Usage Example

```typescript
import { resolveSearchResultAction } from './resolve-search-result-action'

const action = resolveSearchResultAction(searchResult, 'my-source', 'host')

switch (action.kind) {
  case 'navigate-same-tab':
    router.push(action.href)
    break
  case 'navigate-new-tab':
    window.open(action.href, '_blank')
    break
  case 'ask-ai':
    chatRef.current?.send(action.detail)
    break
  case 'route':
    navigate(action.path)
    break
  case 'noop':
    break
}
```
