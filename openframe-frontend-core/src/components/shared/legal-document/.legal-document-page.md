Reusable page component that renders markdown-backed legal documents (e.g. Privacy Policy, Terms of Service) with loading, error, and empty states.

## Key Components

**`LegalDocumentPage`** — Main page component that fetches and displays a legal document via `useLegalDocs`, handling all UI states.

**`LegalDocumentPageProps`** — Configuration interface controlling document type, display copy, API endpoint, and rendering behavior:
- `docType` — drives the default API endpoint `/api/legal/<docType>`
- `title` / `fallbackDescription` / `contactEmail` — per-document display strings
- `initialData` / `initialLastUpdatedLabel` — SSR hydration props
- `MarkdownRenderer` — injectable renderer (defaults to `SimpleMarkdownRenderer`)
- `backButton` — pass `false` to hide, or `{ label, href }` to customize

**`LegalDocumentMarkdownRendererProps`** — Interface for custom renderer components, accepting `content`, optional `sectionIds`, and `demoteMarkdownH1ToH2`.

## Usage Example

```typescript
// Privacy Policy page
<LegalDocumentPage
  docType="privacy"
  title="Privacy Policy"
  fallbackDescription="Our privacy policy and data protection practices."
  contactEmail="privacy@openframe.io"
  errorTitle="Unable to load privacy policy"
  errorContactPrompt="For privacy-related questions, please contact:"
  emptyStateMessage="Privacy policy content is not available at this time."
  initialData={ssrDocument}
  initialLastUpdatedLabel="January 1, 2025"
/>

// With custom renderer and hidden back button
<LegalDocumentPage
  docType="terms"
  title="Terms of Service"
  fallbackDescription="Our terms and conditions."
  contactEmail="legal@openframe.io"
  errorTitle="Unable to load terms of service"
  errorContactPrompt="For legal questions, please contact:"
  emptyStateMessage="Terms of service content is not available."
  MarkdownRenderer={RichMarkdownRenderer}
  backButton={false}
  apiEndpoint="/api/custom/terms"
/>
```
