React hook for fetching markdown-backed legal documents (privacy policy, terms of service, etc.) from a hub API endpoint, with optional SSR hydration support.

## Key Components

### Interfaces

| Interface | Description |
|-----------|-------------|
| `LegalDocument` | Shape of the fetched legal document, including content, sections, GitHub sync metadata, and meta stats |
| `UseLegalDocsOptions` | Optional `initialData` for SSR hydration and `apiEndpoint` override for reverse-proxy embedders |
| `UseLegalDocsReturn` | Hook return value: `data`, `isLoading`, `error`, and `refetch` |

### Hook

**`useLegalDocs(docType, options?)`** — fetches a legal document by type identifier.

- Default endpoint: `/api/legal/<docType>`
- Skips initial client fetch when `initialData` is provided (SSR/RSC hydration)
- Resets state on `docType` change to prevent stale content flicker during sequential fetches
- Guards against CodeQL `js/tainted-format-string` by keeping format strings constant in `console.error`

## Usage Example

```typescript
// Basic client-side fetch
const { data, isLoading, error, refetch } = useLegalDocs('privacy');

// With SSR-provided initial data (skips client fetch)
const { data } = useLegalDocs('terms', {
  initialData: serverFetchedDoc,
});

// Custom endpoint for reverse-proxy embedders
const { data, refetch } = useLegalDocs('privacy', {
  apiEndpoint: '/proxy/legal/privacy',
});

if (isLoading) return <Spinner />;
if (error) return <ErrorMessage message={error} />;
return <LegalPage title={data?.title} content={data?.content} />;
```
