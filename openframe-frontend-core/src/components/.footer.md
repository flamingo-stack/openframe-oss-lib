A highly configurable footer component that accepts comprehensive configuration from app-config.ts to render platform-aware footer layouts with logo, navigation sections, social links, and custom content areas.

## Key Components

- **FooterConfig** - Interface defining all footer configuration options including sections, social links, custom components, and layout controls
- **Footer** - Main component that accepts configuration and optional custom link renderer
- **UniversalFooter** - Internal component that renders the actual footer layout based on provided configuration
- **FooterLink/FooterSection** - Type definitions for navigation link structure

## Usage Example

```typescript
import { Footer } from './footer'

const footerConfig = {
  name: "MyApp",
  legalName: "MyApp Inc.",
  description: "Building the future of web applications",
  logo: <MyLogo />,
  sections: [
    {
      title: "Product",
      links: [
        { href: "/features", label: "Features" },
        { href: "/pricing", label: "Pricing" }
      ]
    },
    {
      title: "Company", 
      links: [
        { href: "/about", label: "About" },
        { href: "/careers", label: "Careers" }
      ]
    }
  ],
  social: {
    github: "https://github.com/myorg",
    twitter: "https://twitter.com/myapp"
  },
  moveDescriptionToRight: true,
  backgroundColor: "bg-ods-bg-card"
}

// Basic usage
<Footer config={footerConfig} />

// With custom link renderer
<Footer 
  config={footerConfig} 
  renderLink={(link) => <CustomLink href={link.href}>{link.label}</CustomLink>}
/>
```

The component supports flexible layouts with responsive grid columns, optional custom components, configurable social icons, and various content positioning options through the comprehensive configuration interface.
