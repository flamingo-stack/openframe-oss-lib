Displays a badge for shell/script types with an appropriate icon and label, supporting all Tactical RMM shell types (PowerShell, CMD, Bash, Python, Nushell, Deno, Shell).

## Key Components

- **`ShellTypeBadge`** — Main React component that renders an icon + label pair for a given shell type
- **`shellIconMap`** — Typed `Record<ShellType, ShellIconConfig>` mapping every `ShellType` value to its icon component and props
- **`ShellIconConfig`** — Interface defining the icon component type and its size/color/className props
- **`defaultIconConfig`** — Fallback config using `ShellIcon` for unknown/undefined shell types
- **`ShellTypeBadgeProps`** — Props interface exposing `shellType`, `className`, and `iconClassName`

## Usage Example

```typescript
import { ShellTypeBadge } from './ShellTypeBadge'
import { ShellTypeValues } from '../../types/shell.types'

// Basic usage
<ShellTypeBadge shellType={ShellTypeValues.POWERSHELL} />

// With custom wrapper and icon class overrides
<ShellTypeBadge
  shellType={ShellTypeValues.PYTHON}
  className="px-2 py-1 bg-surface rounded"
  iconClassName="opacity-75"
/>

// Rendering from a dynamic value (normalized to uppercase internally)
<ShellTypeBadge shellType={script.shell_type} />
```

## Notes

- `shellType` is normalized to uppercase internally, so lowercase string values from APIs are handled automatically
- Unknown shell types fall back to the generic `ShellIcon` via `defaultIconConfig`
- The `shellIconMap` uses `Record<ShellType, ...>` to enforce exhaustive coverage at compile time — adding a new `ShellType` value will surface a TypeScript error until the map is updated
