A flexible authentication stub that provides mock auth functionality for UI development while allowing integration with real authentication systems when available.

## Key Components

- **`AuthContextType`** - Interface defining the auth context structure with user and loading state
- **`AuthContext`** - React context for authentication state management
- **`setRealAuthHook()`** - Function to register a real authentication hook for production use
- **`useAuth()`** - Hook that returns real auth data when available, otherwise falls back to mock user
- **`AuthProvider`** - Context provider component that wraps the app with mock authentication state

## Usage Example

```typescript
// In a UI component library or development environment
import { AuthProvider, useAuth } from './auth-stub';

function App() {
  return (
    <AuthProvider>
      <Dashboard />
    </AuthProvider>
  );
}

function Dashboard() {
  const { user, isLoading } = useAuth();
  return <div>Welcome, {user?.name}!</div>;
}

// To integrate with real auth in production
import { setRealAuthHook } from './auth-stub';
import { useRealAuth } from './real-auth';

setRealAuthHook(useRealAuth);
```

This pattern is useful for component libraries or development environments where you need auth context but don't want to depend on a specific authentication implementation.
