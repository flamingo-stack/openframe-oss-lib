Accessible FAQ accordion component with animated expand/collapse transitions, supporting multiple simultaneously open items and keyboard navigation.

## Key Components

### Interfaces
- **`FaqItem`** — Shape of each FAQ entry with `id`, `question`, and `answer` fields
- **`FaqAccordionProps`** — Component props accepting an `items` array and optional `defaultOpenIds` for pre-expanded items

### Hooks
- **`useMeasuredHeight(isOpen)`** — Internal hook that reads `scrollHeight` outside the render cycle to compute accurate `max-height` values for CSS transitions, avoiding layout jank

### Component
- **`FaqAccordion`** — Renders a bordered card list of collapsible Q&A items. State is tracked via a `Set` to support multiple open items simultaneously. Each item includes full keyboard support (`Enter`/`Space`) and `aria-expanded` for accessibility.

## Usage Example

```typescript
import { FaqAccordion, FaqItem } from './faq-accordion'

const faqs: FaqItem[] = [
  { id: 1, question: 'What is Flamingo?', answer: 'An AI-powered MSP platform.' },
  { id: 2, question: 'What is OpenFrame?', answer: 'A unified IT operations interface.' },
]

// Render with the second item open by default
export default function SupportPage() {
  return (
    <FaqAccordion
      items={faqs}
      defaultOpenIds={[2]}
    />
  )
}
```

## Notes
- Animations use `max-height` + `opacity` CSS transitions (350ms `ease-in-out`) measured at runtime to handle variable-height answers
- Styling relies on ODS design tokens (`ods-border`, `ods-card`, `ods-text-primary`) for dark-mode consistency across the Flamingo UI
