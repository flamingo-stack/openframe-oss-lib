Accessible FAQ accordion component with animated expand/collapse transitions, supporting multiple simultaneously open items and keyboard navigation.

## Key Components

### Interfaces
- **`FaqItem`** — Shape of each FAQ entry: `id`, `question`, `answer`
- **`FaqAccordionProps`** — Accepts an `items` array and optional `defaultOpenIds` for pre-expanded entries

### Hook
- **`useMeasuredHeight(isOpen)`** — Internal hook that measures `scrollHeight` outside the render cycle to drive smooth `max-height` CSS transitions, returning a `ref` and `maxHeight` string

### Component
- **`FaqAccordion`** — Renders a bordered card list where each row toggles independently; uses a `Set` to track open IDs, supports multiple open items simultaneously

## Usage Example

```typescript
import { FaqAccordion, FaqItem } from './faq-accordion'

const faqs: FaqItem[] = [
  { id: 1, question: 'What is Flamingo?', answer: 'An AI-powered MSP platform.' },
  { id: 2, question: 'What is OpenFrame?', answer: 'A unified AI-driven IT support interface.' },
]

// Render with the second item open by default
<FaqAccordion items={faqs} defaultOpenIds={[2]} />
```

## Notes

- **Accessibility** — Each header uses `role="button"`, `tabIndex={0}`, `aria-expanded`, and handles `Enter`/`Space` keydown events
- **Animation** — `max-height` and `opacity` transition over `0.35s ease-in-out`; height is measured lazily only when an item opens
- **Styling** — Uses `ods-*` design tokens and `cn()` utility for conditional class merging
