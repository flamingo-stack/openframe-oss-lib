Provides the primary application shell layout, combining a navigation sidebar, responsive header, mobile burger menu, and a portal-ready drawer container into a single composable wrapper.

## Key Components

**`AppLayout`**
Main layout component that composes the full application chrome. Manages mobile menu open/close state and exposes a `div` ref as the drawer portal target via context.

**`useAppLayoutDrawerContainer`**
Custom hook that returns the `HTMLElement` portal target for `AppLayoutDrawer`. Returns `null` when called outside of `AppLayout`.

**`AppLayoutDrawerContainerContext`**
Internal React context holding the `HTMLElement | null` reference to the main-area container div. Drawers portaled into this element appear over main content only — sidebar and header remain visible and interactive.

## Props (`AppLayoutProps`)

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Page content rendered inside `<main>` |
| `sidebarConfig` | `NavigationSidebarConfig` | Navigation items/structure for sidebar and mobile menu |
| `headerProps` | `Omit<AppHeaderProps, ...>` | Passed through to `AppHeader` |
| `mobileBurgerMenuProps` | `Omit<MobileBurgerMenuProps, ...>` | Passed through to `MobileBurgerMenu` |
| `drawer` | `ReactNode` | Optional layout-level drawer slot (e.g. `AppLayoutDrawer`) |
| `disabled` | `boolean` | Freezes nav/header chrome; main content stays interactive |
| `loadingFallback` | `ReactNode` | Suspense fallback for page content |
| `mainClassName` | `string` | Extra classes on `<main>` |

## Usage Example

```typescript
import { AppLayout } from './app-layout'
import { AppLayoutDrawer } from './app-layout-drawer'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <AppLayout
      sidebarConfig={navigationConfig}
      headerProps={{ logo: <Logo />, actions: <UserMenu /> }}
      mobileBurgerMenuProps={{ footer: <SupportLink /> }}
      drawer={<AppLayoutDrawer />}
    >
      {children}
    </AppLayout>
  )
}

// Accessing the drawer portal target from a child component
import { useAppLayoutDrawerContainer } from './app-layout'

function MyDrawer() {
  const container = useAppLayoutDrawerContainer()
  // portal content into `container` so it overlays only the main area
}
```
