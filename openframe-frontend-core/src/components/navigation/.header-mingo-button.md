Accessible "Mingo AI" launcher button for `AppHeader`, toggling between a branded icon+wordmark view and a close (`X`) affordance based on active and display state.

## Key Components

### `HeaderMingoButton`
Main exported component. Renders a styled `<button>` that integrates into the `AppHeader` via the shared `divide-x` divider contract.

**Props (`HeaderMingoButtonProps`):**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isActive` | `boolean` | `false` | Reflects open drawer state; sets `aria-pressed` |
| `iconOnly` | `boolean` | `false` | Hides "Mingo AI" wordmark, collapses to icon-square |
| `className` | `string` | — | Additional Tailwind/CSS classes |
| `...props` | `ButtonHTMLAttributes` | — | Passed through to the underlying `<button>` |

**Rendering logic:**

- **Default:** `MingoIcon` (ODS cyan accents) + `"Mingo AI"` bold wordmark
- **`iconOnly + isActive`:** Swaps logo for `XmarkIcon`, sets `aria-label` to `"Close Mingo AI"`
- **`iconOnly` only:** Shows `MingoIcon` alone, no wordmark

## Usage Example

```typescript
import { HeaderMingoButton } from './header-mingo-button'

// Full button — drawer closed
<HeaderMingoButton onClick={openDrawer} />

// Full button — drawer open
<HeaderMingoButton isActive onClick={closeDrawer} />

// Narrow viewport (icon-only, drawer open → shows X)
<HeaderMingoButton
  isActive
  iconOnly
  onClick={closeDrawer}
  className="md:hidden"
/>
```
