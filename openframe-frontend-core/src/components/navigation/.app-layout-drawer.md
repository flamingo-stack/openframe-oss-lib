A layout-aware drawer component that renders **inside** the `AppLayout` content area (not as a viewport-level overlay), keeping the header and sidebar visible and interactive while the drawer is open.

## Key Components

| Export | Description |
|--------|-------------|
| `AppLayoutDrawerRoot` | Root wrapper; manages controlled/uncontrolled open state and exposes it via `DrawerOpenContext`. Forces `modal={false}` by default. |
| `AppLayoutDrawerTrigger` | Alias of `DialogPrimitive.Trigger` |
| `AppLayoutDrawerClose` | Alias of `DialogPrimitive.Close` |
| `appLayoutDrawerVariants` | CVA variants for the outer wrapper — handles side (left/right/top/bottom), flush mode, and slide animations using `absolute` positioning. |
| `appLayoutDrawerPanelVariants` | CVA variants for the inner panel — controls borders, rounding, padding, and flush breakpoint behavior. |
| `AppLayoutDrawerResizeHandle` | Drag/keyboard-accessible resize handle; supports pointer capture, directional arrow keys, and Home/End shortcuts. |
| `useContainedResizableSize` | Hook that tracks container dimensions via `ResizeObserver`, clamps drawer size to available space, and persists size to `localStorage`. |

## Usage Example

```typescript
import {
  AppLayoutDrawerRoot,
  AppLayoutDrawerTrigger,
  AppLayoutDrawerClose,
} from "./app-layout-drawer"

function MyPanel() {
  return (
    <AppLayoutDrawerRoot>
      <AppLayoutDrawerTrigger asChild>
        <button>Open Details</button>
      </AppLayoutDrawerTrigger>

      {/* Content portal targets AppLayout container, not document.body */}
      <AppLayoutDrawerContent side="right" resizable storageKey="my-drawer-width">
        <DrawerHeader>
          <DrawerTitle>Details</DrawerTitle>
        </DrawerHeader>
        <DrawerBody>Panel content here</DrawerBody>
        <DrawerFooter>
          <AppLayoutDrawerClose asChild>
            <button>Close</button>
          </AppLayoutDrawerClose>
        </DrawerFooter>
      </AppLayoutDrawerContent>
    </AppLayoutDrawerRoot>
  )
}
```

> **Key behavioral difference from standard `Drawer`:** Uses `absolute` positioning clipped to the layout container and `modal={false}` so header/sidebar remain interactive. A custom overlay div is rendered manually (since Radix's `DialogPrimitive.Overlay` returns `null` in non-modal mode) to prevent click-through flicker on trigger elements.
