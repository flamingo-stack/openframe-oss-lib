A layout-aware drawer component that renders **inside** the `AppLayout` container rather than as a viewport-level overlay, keeping the header and sidebar visible and interactive while the drawer is open.

## Key Components

| Export | Description |
|--------|-------------|
| `AppLayoutDrawerRoot` | Root component wrapping `DialogPrimitive.Root` with non-modal mode and internal open state tracking via `DrawerOpenContext` |
| `AppLayoutDrawerTrigger` | Alias for `DialogPrimitive.Trigger` |
| `AppLayoutDrawerClose` | Alias for `DialogPrimitive.Close` |
| `AppLayoutDrawerResizeHandle` | Drag/keyboard-accessible resize grip supporting all four drawer sides |
| `appLayoutDrawerVariants` | CVA variants for wrapper positioning (`side`, `flush`) |
| `appLayoutDrawerPanelVariants` | CVA variants for panel styling (`side`, `flush`) |
| `useContainedResizableSize` | Hook managing resizable panel size clamped to the container bounds, with optional `localStorage` persistence |
| `DrawerOpenContext` | Internal React context exposing open state to child components for animation synchronization |

Re-exports `DrawerBody`, `DrawerDescription`, `DrawerFooter`, `DrawerHeader`, `DrawerTitle` from the base `ui/drawer` for style consistency.

## Usage Example

```typescript
import {
  AppLayoutDrawerRoot,
  AppLayoutDrawerTrigger,
  AppLayoutDrawerClose,
} from "./app-layout-drawer"
import { DrawerHeader, DrawerTitle, DrawerBody } from "../ui/drawer"

function MyPanel() {
  return (
    <AppLayoutDrawerRoot>
      <AppLayoutDrawerTrigger asChild>
        <button>Open Details</button>
      </AppLayoutDrawerTrigger>

      {/* Content renders inside AppLayout container, not document.body */}
      <AppLayoutDrawerContent side="right" resizable storageKey="details-panel-size">
        <DrawerHeader>
          <DrawerTitle>Details</DrawerTitle>
        </DrawerHeader>
        <DrawerBody>Panel content here</DrawerBody>
        <AppLayoutDrawerClose asChild>
          <button>Close</button>
        </AppLayoutDrawerClose>
      </AppLayoutDrawerContent>
    </AppLayoutDrawerRoot>
  )
}
```

> **Key difference from standard `Drawer`:** Portals into the `AppLayout` container element (via `useAppLayoutDrawerContainer`), uses `position: absolute` instead of `fixed`, and runs in `modal={false}` mode — a custom overlay div is rendered manually since Radix suppresses its built-in overlay in non-modal mode.
