A loading skeleton component that displays placeholder elements while the actual header component is loading, maintaining the same layout structure as the real header.

## Key Components

- **HeaderSkeletonProps**: Interface defining optional `config` prop of type `HeaderConfig`
- **HeaderSkeleton**: Main component that renders animated placeholder elements based on header configuration
- **Conditional rendering logic**: Shows/hides skeleton elements based on config (navigation, actions, mobile menu, admin mode)
- **Layout sections**: Left (logo), center (navigation), and right (actions) with responsive behavior

## Usage Example

```typescript
import { HeaderSkeleton } from './components/header-skeleton'
import { HeaderConfig } from './types/navigation'

// Basic usage
function App() {
  return <HeaderSkeleton />
}

// With configuration
function AppWithConfig() {
  const headerConfig: HeaderConfig = {
    navigation: {
      items: [{ label: 'Home' }, { label: 'About' }],
      position: 'center'
    },
    actions: {
      right: [{ type: 'button', label: 'Sign In' }]
    },
    mobile: { enabled: true },
    className: 'admin-header'
  }

  return <HeaderSkeleton config={headerConfig} />
}
```

The skeleton automatically adapts its layout to match the configured header structure, showing appropriate placeholder elements with pulse animations.
