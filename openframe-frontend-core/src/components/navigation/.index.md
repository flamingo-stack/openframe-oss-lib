Barrel export file (marked `"use client"`) that consolidates all navigation-related components, hooks, and types for the Flamingo/OpenFrame UI navigation system into a single public API.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `Header` / `AppHeader` | Component | Primary and app-level header components |
| `ClientOnlyHeader` | Component | Header rendered only on the client side |
| `HeaderSkeleton` | Component | Loading placeholder for the header |
| `MobileNavPanel` / `MobileBurgerMenu` | Component | Mobile navigation controls |
| `SlidingSidebar` / `NavigationSidebar` | Component | Sidebar navigation variants |
| `StickySectionNav` | Component | Sticky in-page section navigation |
| `useSectionNavigation` | Hook | Controls active section tracking |
| `AppLayout` | Component | Root layout wrapper with drawer support |
| `useAppLayoutDrawerContainer` | Hook | Accesses the drawer container context |
| `AppLayoutDrawer` + sub-components | Components | Compound drawer component (trigger, close, content, header, body, footer) |
| `HeaderButton` / `HeaderMingoButton` | Component | Action buttons for the header bar |
| `HeaderGlobalSearch` | Component | Global search trigger in the header |
| `HeaderOrganizationFilter` | Component | Org-level filter control in the header |

## Usage Example

```typescript
import {
  AppLayout,
  AppHeader,
  NavigationSidebar,
  AppLayoutDrawer,
  AppLayoutDrawerTrigger,
  AppLayoutDrawerContent,
  useSectionNavigation,
} from '@/components/navigation'

export default function RootLayout({ children }) {
  return (
    <AppLayout>
      <AppHeader>
        <HeaderGlobalSearch />
        <HeaderOrganizationFilter />
        <HeaderMingoButton />
      </AppHeader>
      <NavigationSidebar />
      {children}
    </AppLayout>
  )
}
```

> All exports are client-only (`"use client"`). Avoid importing into Server Components directly — use dynamic imports or a server-side wrapper if needed.
