A footer UI component that renders a "Join Waitlist" button, routing navigation through the unified `ChatRuntimeContext` navigation system or falling back to the embed-shim router.

## Key Components

**`FooterWaitlistButtonProps`**
- `className?: string` — Optional CSS class for styling overrides.

**`FooterWaitlistButton`**
- Reads the host's `ChatRuntimeContext` via `useChatRuntime()` to access `runtime.navigation.navigate`.
- Uses `executeNavigationImperative` to route clicks through the unified navigation pipeline (handles cross-platform new-tab logic, embed-mode short-circuiting, same-URL re-scroll, etc.).
- Falls back to `router.push` (embed-shim) when no `ChatRuntimeContext` is mounted, keeping the component usable in third-party embed scenarios.
- Navigates to `/waitlist#top`, where `#top` maps to an explicit `<div id="top">` anchor for reliable cross-browser scroll behavior.

## Usage Example

```typescript
// Standard usage inside a footer layout
import { FooterWaitlistButton } from './footer-waitlist-button';

export function AppFooter() {
  return (
    <footer>
      <FooterWaitlistButton className="mt-4 w-full" />
    </footer>
  );
}
```

```typescript
// Without ChatRuntimeContext (third-party embed fallback)
// No additional setup needed — component auto-detects missing
// runtime and falls back to embed-shim router.push
import { FooterWaitlistButton } from './footer-waitlist-button';

export function EmbedFooter() {
  return <FooterWaitlistButton />;
}
```

> **Note:** For full unified navigation features (new-tab decisions, embed-mode handling), wrap the component tree with `ChatRuntimeContext` via `HubRuntimeProvider`. Without it, basic `router.push` navigation to `/waitlist#top` is used instead.
