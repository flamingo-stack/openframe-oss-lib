Renders the full Help Center surface for the `/tickets` page, managing ticket listing, optimistic UI, drawer state via URL params, and identity-gated access.

## Key Components

**`HelpCenterList`** — Public entry point. Reads URL params (`?search=`, `?status=`, `?page=`, `?ticket=`), resolves user identity via `useChatIdentity`, and either renders a loading skeleton, a sign-in prompt for anon users, or delegates to `HelpCenterListAuthed`.

**`HelpCenterListAuthed`** — Private authenticated sub-tree. Owns:
- `optimisticTickets` — local placeholder state (not in TanStack cache) so filter/page changes don't clear in-flight submissions
- `supportSystemDown` — surface-level flag for degraded-mode messaging
- Drawer open/close state derived from `?ticket=<external_id>` URL param (no separate state)
- `useTicketsList` — fetches paginated tickets; live-polls when a drawer is open (`TICKET_LIVE_POLL_MS`)
- `useTicketActions` — mutation handlers with `prependOptimistic` / `removeOptimistic` / `removeTicketFromCache` wired in
- `removeTicketFromCache` — patches all `['tickets']` query cache slots via `setQueriesData`, correctly handling the `TicketsCacheSlot` shape

## Usage Example

```typescript
// Drop-in on the /tickets route (Next.js App Router)
import { HelpCenterList } from '@/components/help-center/help-center-list'

export default function TicketsPage() {
  return <HelpCenterList />
}

// With a custom toast (e.g. in tests)
import { mockToast } from '../test-utils'

render(<HelpCenterList toast={mockToast} />)
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `toast` | `typeof defaultToast` | lib singleton | Toast override for testing |

## URL Params Consumed

| Param | Purpose |
|-------|---------|
| `?search=` | Full-text filter |
| `?status=` | Status filter (`all` / open / closed) |
| `?page=` | 1-based pagination |
| `?ticket=` | Deep-link — auto-opens matching drawer |
