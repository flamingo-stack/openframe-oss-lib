Expanded drawer view for a single support ticket, rendering the conversation timeline, metadata, and status-dependent actions (reply composer or reopen control).

## Key Components

**`TicketDetailDrawer`** — Root exported component. Accepts a ticket, busy/error state, and mutation callbacks; renders the full expanded panel including assignee header, optional ClickUp delivery card, conversation timeline, reply-failure banner, and action controls.

**`TicketTimelinePanel`** — Internal component that fetches and renders the chronological conversation. Uses `useTicketEngagements` with live polling (`TICKET_LIVE_POLL_MS`), `useStickToBottom` for auto-scroll, and splits legacy `ticket.body` multi-turn strings on the `---` separator. Suppresses the "Original message" body turn when a matching customer engagement already exists.

**`AssignedAgentRow`** — Displays the assigned support agent (or "Unassigned" fallback).

**`ReopenAction`** — Shown when `ticket.status === 'CLOSED'`; calls `onReopen` then `onActionCollapsed`.

**`ReplyFailureBanner`** — Persistent inline error banner above the composer for failed send attempts; distinct from transient toasts.

**`TicketRef`** — Exported type `{ id: string; external_id: string }` passed to all action callbacks.

## Usage Example

```typescript
<TicketDetailDrawer
  ticket={ticket}
  busy={isMutating}
  supportSystemDown={false}
  onSendMessage={async (ref, text, attachments) => {
    return await actions.sendMessage(ref, text, attachments)
  }}
  onClose={async (ref, resolution) => actions.closeTicket(ref, resolution)}
  onReopen={async (ref) => actions.reopenTicket(ref)}
  onActionCollapsed={() => setDrawerOpen(false)}
  replyError={replyError}
  onClearReplyError={() => actions.clearReplyError(ticket.external_id)}
/>
```

The parent owns ticket data and mutation state; this component manages only local UI state (composer text, attachment bag, close-confirm dialog).
