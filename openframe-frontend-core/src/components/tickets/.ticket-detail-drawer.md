Shared drawer component that renders the expanded detail view for a single support ticket, including metadata, conversation timeline, and status-dependent action controls (reply composer or reopen).

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `TicketDetailDrawer` | Component | Main drawer rendering metadata, timeline, and actions |
| `TicketTimelinePanel` | Component (internal) | Scrollable conversation feed with engagement polling |
| `AssignedAgentRow` | Component (internal) | Displays the currently assigned support agent |
| `ReopenAction` | Component (internal) | Reopen button shown when ticket status is `CLOSED` |
| `ReplyFailureBanner` | Component (internal) | Inline error banner persisted above the composer on send failure |
| `TicketRef` | Type | `{ id, external_id }` identity bundle passed to action callbacks |
| `TicketDetailDrawerProps` | Interface | Full props contract for the drawer |

## Usage Example

```typescript
import { TicketDetailDrawer } from './ticket-detail-drawer'

<TicketDetailDrawer
  ticket={ticket}
  busy={isMutating}
  supportSystemDown={false}
  onSendMessage={async (ref, text, attachments) => {
    return await actions.sendMessage(ref, text, attachments)
  }}
  onClose={async (ref, resolution) => {
    return await actions.closeTicket(ref, resolution)
  }}
  onReopen={async (ref) => {
    return await actions.reopenTicket(ref)
  }}
  onActionCollapsed={() => setDrawerOpen(false)}
  replyError={replyError}
  onClearReplyError={() => actions.clearReplyError(ticket.external_id)}
/>
```

## Notes

- **Live polling** — `TicketTimelinePanel` polls for new engagements every `TICKET_LIVE_POLL_MS` (8s) while mounted; polling stops automatically when the drawer closes.
- **Auto-scroll** — Uses `use-stick-to-bottom` to tail new messages, mirroring the behavior of `ChatMessageList`.
- **Optimistic tickets** — Engagement fetch is skipped until a real `external_id` is available.
- **Duplicate body suppression** — Prevents the original ticket body from rendering twice when the channel-first intake path also writes it as the first engagement.
