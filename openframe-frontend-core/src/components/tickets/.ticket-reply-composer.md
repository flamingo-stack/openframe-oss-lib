Reply composer for open tickets, combining a shared `ChatInput` with file attachment support and a confirmation dialog for closing tickets.

## Key Components

- **`TicketReplyComposer`** — Main exported component that renders the full reply UI
- **`TicketReplyComposerProps`** — Interface defining required props: `ticket`, `busy`, `supportSystemDown`, `onSendMessage`, `onClose`
- **`useChatAttachments`** — Hook managing staged file attachments (add, remove, clear, inflight tracking)
- **`ChatInput`** — Shared input component handling send semantics, max length, and disabled states
- **`AlertDialog`** — Confirmation dialog for closing a ticket with an optional resolution note

## Behavior Notes

- **Send resilience** — Draft text and staged files survive a failed send; `attachments.clear()` is only called on success
- **Attachment-only replies** — `allowEmptySend` enables sending when files are staged but no text is typed
- **Disabled states** — Input is blocked while `busy` OR uploads are inflight; the attach `+` button is hidden entirely when `supportSystemDown`
- **Close is non-destructive** — Styled as a low-emphasis text button (not red/destructive) since closing is reversible via Reopen
- **No drawer collapse** — `confirmClose` intentionally omits `onActionCollapsed` to preserve the ticket row in view after closure

## Usage Example

```typescript
<TicketReplyComposer
  ticket={ticket}
  busy={isSending}
  supportSystemDown={!isConnected}
  onSendMessage={async (ref, text, files) => {
    const ok = await sendReply(ref, text, files)
    return ok
  }}
  onClose={async (ref, resolution) => {
    await closeTicket(ref, resolution)
  }}
/>
```
