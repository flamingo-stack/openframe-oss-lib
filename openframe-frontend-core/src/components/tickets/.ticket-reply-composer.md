Reply composer component for open support tickets, combining a shared `ChatInput` with file attachment support and a close-ticket confirmation dialog.

## Key Components

- **`TicketReplyComposer`** — Main exported component that renders the full reply UI
- **`TicketReplyComposerProps`** — Interface defining `ticket`, `busy`, `supportSystemDown`, `onSendMessage`, and `onClose` props
- **`useChatAttachments`** — Hook managing staged file attachments (add, remove, clear, inflight state)
- **`ChatInput`** — Shared input with send button; blocks submission during uploads or while `busy`
- **`ChatAttachmentChipStrip`** — Displays staged file chips above the input
- **`ChatAttachmentAddButton`** — Attachment `+` button, hidden when `supportSystemDown`
- **`AlertDialog`** — Confirmation dialog for closing a ticket, with an optional resolution note textarea

## Behavior Notes

- **Draft preservation** — On a failed send, `handleSend` returns `false`; `ChatInput` retains the typed text and attachments are only cleared on success
- **Attachment-only replies** — `allowEmptySend` is set when ready files exist, enabling send without text
- **No drawer collapse** — `onClose` does not call `onActionCollapsed`; the ticket row stays mounted with its updated status badge
- **Close is non-destructive** — The confirm button uses accent styling (not red/destructive) since closing is reversible via Reopen

## Usage Example

```typescript
<TicketReplyComposer
  ticket={activeTicket}
  busy={isSending}
  supportSystemDown={!isConnected}
  onSendMessage={async (ref, text, files) => {
    const ok = await sendTicketReply(ref.id, text, files)
    return ok
  }}
  onClose={async (ref, resolution) => {
    await closeTicket(ref.id, resolution)
  }}
/>
```
