Fetches and caches the conversation timeline (notes + attachments) for a single HubSpot ticket, powering the drawer's timeline view with optional live polling.

## Key Components

**Interfaces**
- `TicketEngagementFile` — attachment metadata (id, name, url, mime, size)
- `TicketEngagement` — a single timeline entry with body, author info (`authorRole: 'customer' | 'support'`), timestamps, and attachments
- `UseTicketEngagementsReturn` — hook return shape (engagements array, loading/fetching/error states, refetch)

**Hook**
- `useTicketEngagements(externalTicketId, enabled, refetchInterval)` — wraps `@tanstack/react-query` to POST `/api/chat/agent/list-engagements`. Caching is intentionally disabled (`staleTime: 0`, `gcTime: 0`) so the drawer always reflects live HubSpot state.

**Notable behaviors**
- Skips fetching for optimistic tickets (`temp-*` prefix) and unauthenticated users (`authTier === 'anon'`)
- Solves a double-flash/blink bug by treating a ticket as "loading" from the very first render — even before `useChatIdentity` resolves — so the skeleton never replaces a momentarily rendered body
- Polling pauses automatically when the drawer unmounts or the tab is hidden

## Usage Example

```typescript
const { engagements, isLoading, error, refetch } = useTicketEngagements(
  ticket.externalId,
  true,
  TICKET_LIVE_POLL_MS, // e.g. 10_000 — poll every 10s while drawer is open
)

if (isLoading) return <TimelineSkeleton />
if (error) return <ErrorState onRetry={refetch} />

return <ConversationThread engagements={engagements} />
```
