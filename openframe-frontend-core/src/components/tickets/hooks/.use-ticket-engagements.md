Fetches and caches the conversation timeline (notes, messages, and attachments) for a single ticket, powering the drawer's engagement/timeline view with optional live polling.

## Key Components

**Interfaces**
- `TicketEngagementFile` — attachment metadata: `id`, `name`, `url`, `mime`, `size`
- `TicketEngagement` — single timeline entry with `body`, `authorRole` (`'customer' | 'support'`), resolved author fields (`authorName`, `authorEmail`, `authorAvatarUrl`), `createdAt`, and nested `attachments[]`
- `UseTicketEngagementsReturn` — hook return shape: `engagements[]`, `isLoading`, `isFetching`, `error`, `refetch`

**Hook**
- `useTicketEngagements(externalTicketId, enabled?, refetchInterval?)` — primary export; wraps `@tanstack/react-query` with zero stale/gc time (always-fresh HubSpot data), guards against optimistic `temp-*` ticket IDs, and skips the fetch until `useChatIdentity` resolves a non-anon session

## Usage Example

```typescript
// Basic — fetch once when drawer opens
const { engagements, isLoading, error } = useTicketEngagements(ticket.externalId)

// With live polling (e.g. 10-second cadence while drawer is open)
const TICKET_LIVE_POLL_MS = 10_000

const { engagements, isFetching, refetch } = useTicketEngagements(
  ticket.externalId,
  true,
  TICKET_LIVE_POLL_MS,
)

// Disabled for optimistic/unconfirmed tickets
const { isLoading } = useTicketEngagements(
  'temp-abc123', // detected via startsWith('temp-') — fetch is skipped
  true,
)
```

## Notes

- **Auth:** uses `embedAuthedFetch` (proxy credentials); server enforces ticket ownership via `ticketBelongsToCustomer` — cross-customer enumeration is blocked server-side
- **Loading state:** `isLoading` covers identity resolution lag to prevent skeleton flash on first render
- **Polling:** `refetchIntervalInBackground` defaults to `false` — polling pauses on hidden tabs; unmounting the drawer stops polling automatically
