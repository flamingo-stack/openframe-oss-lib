Fetches a customer-scoped, paginated ticket list from `POST /api/chat/agent/find-ticket` using TanStack Query, with support for free-text search, status filtering, and optional live-polling.

## Key Components

### `useTicketsList(filters)`
Main exported hook. Accepts a `UseTicketsListFilters` object and returns a `UseTicketsListReturn` with tickets, pagination metadata, and loading/error state.

### `UseTicketsListFilters`
| Field | Type | Description |
|---|---|---|
| `customerEmail` | `string` | Required. Drilled from parent to avoid identity-race conditions. |
| `search` | `string?` | Free-text FTS query. Empty string lists all tickets. |
| `status` | `string?` | `'open'` \| `'closed'` \| `'all'`. Empty/`'all'` removes the filter. |
| `page` | `number?` | 1-based. Defaults to `1`. |
| `pageSize` | `number?` | Server-capped at `100`. Defaults to `20`. |
| `refetchInterval` | `number \| false?` | Enables live-polling (ms). Pauses on hidden tabs. |

### `UseTicketsListReturn`
Returns `tickets`, `isLoading`, `isFetching`, `error`, `refetch`, `lastUpdatedAt`, `totalCount`, `page`, `pageSize`, and pre-computed `totalPages`.

## Key Behaviors

- **Cache strategy:** `staleTime: 0` / `gcTime: 0` — always fetches fresh on mount, focus, and navigation. Ticket data is expected to be live.
- **Cache key:** `['tickets', 'self', identity, search, status, page, pageSize]` — each filter+page combo gets its own slot without evicting others.
- **Loading truth:** Uses `data === undefined` (not TanStack's `isPending`) to avoid the EmptyState flash when switching filter slots with existing cached data.
- **Auth:** Relies on `embedAuthedFetch`; query stays disabled (`enabled = false`) when `customerEmail` is empty.

## Usage Example

```typescript
const { tickets, isLoading, totalCount, totalPages, page } = useTicketsList({
  customerEmail: identity.user.email,
  search: searchQuery,
  status: 'open',
  page: currentPage,
  pageSize: 20,
  refetchInterval: isDrawerOpen ? 5000 : false,
})
```
