Handles all ticket write operations for the Flamingo OpenFrame ticket UI, funneling five actions (create, update, reply, close, reopen) through a single `/api/chat/agent/ticket-action` endpoint with optimistic UI updates, per-row mutex protection, serialized mutation queuing, and mirror-sync retry logic.

## Key Components

### `useTicketActions(options: UseTicketActionsOptions): UseTicketActionsReturn`
Primary hook exporting five action methods and associated state selectors.

**Options:**
- `prependOptimistic` — Injects a placeholder row into the QueryClient cache before server roundtrip
- `removeOptimistic` — Removes placeholder on mirror-sync failure
- `removeTicketFromCache` — Drops a row without refetch on `TICKET_NOT_FOUND`
- `toast` — Toast callback (injected for test isolation)
- `onSupportSystemDown` — Called on 412 `HUBSPOT_DISCONNECTED`

**Returned interface (`UseTicketActionsReturn`):**
| Member | Description |
|---|---|
| `submitTicket` | Creates a new ticket with subject, content, and optional attachments |
| `sendMessage` | Sends a reply note (text + attachments merged into one HubSpot engagement) |
| `closeTicket` | Closes a ticket with an optional resolution string |
| `reopenTicket` | Reopens a closed ticket |
| `isSubmittingForm` | `true` while form-level POST is in flight |
| `isRowBusy(localId)` | Per-row mutex selector for disabling UI rows |
| `replyErrorFor(externalId)` | Returns the persisted reply-failure banner error for a ticket |
| `clearReplyError(externalId)` | Dismisses the reply-failure banner |

### Internal Mechanisms

- **`executeTicketAction`** — Core `POST` helper using `embedAuthedFetch` with bearer/act-as headers
- **`enqueue`** — Depth-1 serial mutation queue preventing server rate-limit stampedes
- **`watchMirrorSync`** — Schedules 3s/6s/12s backoff refetches when `mirror_synced === false`; aborted on unmount via `AbortController`
- **`REPLY_BANNER_CODES`** — Set of `TicketActionErrorCode` values that surface as inline drawer banners vs. toast-only errors

### Supporting Types

```typescript
interface TicketRef {
  id: string          // Local mirror UUID (React mutex + cache key)
  external_id: string // HubSpot numeric id (server/API calls)
}
```

## Usage Example

```typescript
const {
  submitTicket,
  sendMessage,
  closeTicket,
  reopenTicket,
  isSubmittingForm,
  isRowBusy,
  replyErrorFor,
  clearReplyError,
} = useTicketActions({
  prependOptimistic: (placeholder) => {
    queryClient.setQueryData(['tickets'], (prev: TicketsCacheSlot) => ({
      ...prev,
      tickets: [placeholder, ...(prev?.tickets ?? [])],
    }))
  },
  removeOptimistic: (id) => { /* remove row by id from cache */ },
  removeTicketFromCache: (id) => { /* drop row without refetch */ },
  toast,
  onSupportSystemDown: () => setSupportDown(true),
})

// Create a ticket
const ok = await submitTicket({
  subject: 'VPN not connecting',
  content: 'Unable to connect since this morning.',
  attachments: [],
})

// Reply with text and an attachment
await sendMessage(
  { id: localUUID, external_id: '12345' },
  'Please try restarting the client.',
  [screenshotAttachment],
)

// Close with resolution note
await closeTicket(
  { id: localUUID, external_id: '12345' },
  'Resolved by reinstalling the VPN client.',
)

// Check per-row loading state
const busy = isRowBusy(localUUID)

// Read/clear inline reply error banner
const error = replyErrorFor('12345')
clearReplyError('12345')
```
