Handles all ticket write operations (create, update, close, reopen, reply) through a single POST endpoint, with optimistic UI updates, per-row mutex guards, serialized mutation queuing, and mirror-sync retry logic.

## Key Components

### `useTicketActions(options)`
Main hook export. Accepts a `UseTicketActionsOptions` config object and returns `UseTicketActionsReturn`.

**Options:**
- `prependOptimistic` — Mutates QueryClient cache to show a placeholder row before server roundtrip
- `removeOptimistic` — Removes placeholder on failure or replacement
- `removeTicketFromCache` — Silent cache removal on `TICKET_NOT_FOUND`
- `toast` — Injected toast helper (test-friendly, no singleton import)
- `onSupportSystemDown` — Called on 412 `HUBSPOT_DISCONNECTED`

### Internal Mechanisms

| Mechanism | Purpose |
|-----------|---------|
| `formInFlightRef` + `isSubmittingForm` | Prevents duplicate form submits (sync ref + state mirror) |
| `busyRowsRef` + `busyRows` | Per-row `Set<ticket_id>` mutex for row-level disable |
| `mutationQueue` (`queueRef`) | Depth-1 serial queue — prevents server rate-limit stampede |
| `watcherControllersRef` | Tracks `AbortController` per placeholder for mirror-sync backoff |
| `replyErrorByTicket` | Persisted inline reply-failure banner state, keyed by `external_id` |

### `executeTicketAction(toolName, args)`
Internal helper that POSTs to `/api/chat/agent/ticket-action`. Used by all 5 public actions — collapses the chat agent's `propose → confirm-tool` two-step into a single roundtrip.

### Mirror-sync Retry
On `result.mirror_synced === false`, schedules refetches at **3s / 6s / 12s** (≤30s wall-clock). Drops the optimistic placeholder and surfaces a "Couldn't confirm — Reload" affordance if sync never resolves.

## Usage Example

```typescript
const {
  submitTicket,
  sendMessage,
  closeTicket,
  reopenTicket,
  isSubmittingForm,
  isRowBusy,
  replyErrorFor,
  clearReplyError,
} = useTicketActions({
  prependOptimistic: (placeholder) => {
    queryClient.setQueryData(['tickets'], (old: TicketsCacheSlot) => ({
      ...old,
      tickets: [placeholder, ...(old?.tickets ?? [])],
    }))
  },
  removeOptimistic: (id) => { /* remove by id from cache */ },
  removeTicketFromCache: (id) => { /* silent remove */ },
  toast,
  onSupportSystemDown: () => setSupportSystemDown(true),
})

// Create
await submitTicket({ subject: 'VPN down', content: 'Cannot connect' })

// Reply with attachment
await sendMessage(
  { id: 'uuid-local', external_id: '12345' },
  'Here is the fix',
  [attachment]
)

// Close with resolution note
await closeTicket(
  { id: 'uuid-local', external_id: '12345' },
  'Resolved by restarting the service'
)
```

> **Note:** `TicketRef` requires both `id` (local mirror UUID for React-side mutex) and `external_id` (HubSpot numeric ID for server calls). Passing the wrong ID to either side causes either a HubSpot 404 or a broken row-disable state.
