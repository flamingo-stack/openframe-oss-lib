Single expandable ticket row component for the Help Center list, rendering a summary button that toggles an inline `TicketDetailDrawer` with scroll management and optimistic-state handling.

## Key Components

- **`HelpCenterCard`** — Main export. Renders a full-width button row showing ticket title, subtitle, status/priority badges, and an expandable drawer beneath it.
- **`mapPriorityScheme`** — Local helper mapping `HIGH`/`URGENT`/`MEDIUM`/`LOW` priority strings to `StatusBadge` color schemes.
- **`STICKY_HEADER_OFFSET_PX`** — Single source of truth (`96px`) applied as both `scrollMarginTop` inline style (for browser/anchor-driven scroll) and `headerOffset` in the programmatic `scrollElementIntoView` call.

## Usage Example

```typescript
<HelpCenterCard
  ticket={ticket}
  expanded={expandedId === ticket.id}
  onToggle={(id) => setExpandedId(prev => prev === id ? null : id)}
  busy={isBusy}
  supportSystemDown={false}
  onSendMessage={handleSendMessage}
  onClose={handleClose}
  onReopen={handleReopen}
  onActionCollapsed={() => setExpandedId(null)}
  replyError={replyErrors[ticket.external_id]}
  onClearReplyError={() => clearReplyError(ticket.external_id)}
/>
```

## Behavioral Notes

- **Optimistic tickets** (pending server confirmation) render as non-expandable, reduced-opacity rows — `external_id` is unavailable so the drawer cannot fetch engagements.
- **Scroll-on-expand** uses a deferred `requestAnimationFrame` via `scrollElementIntoView` rather than native `scrollTo({behavior:'smooth'})`, avoiding browser scroll-anchoring cancellation that broke smooth scrolling after the first open.
- **Click isolation** — the drawer is a sibling of the toggle `<button>`, not nested inside it, so interactions within the drawer (composer, close dialog) do not bubble to the row toggle handler.
