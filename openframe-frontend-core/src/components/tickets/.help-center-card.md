Single ticket row component for the Help Center list that renders a summary button toggling an expandable `<TicketDetailDrawer />` beneath it.

## Key Components

**`HelpCenterCard`** — Main exported component. Renders a clickable summary row using `<DevCardRowContent>` (shared with the delivery list) with an optional `<TicketDetailDrawer>` sibling that expands below on toggle.

**`HelpCenterCardProps`** — Interface defining ticket data, expanded/busy state, event handlers (`onToggle`, `onSendMessage`, `onClose`, `onReopen`, `onActionCollapsed`), and optional reply-error props forwarded to the drawer.

**`mapPriorityScheme`** — Local helper mapping ticket priority strings (`HIGH`, `URGENT`, `MEDIUM`, `LOW`) to `StatusBadge` color schemes. Kept separate from the workflow-status `getStatusColorScheme` to avoid badge mis-rendering.

**`STICKY_HEADER_OFFSET_PX`** — Single constant (`96`) governing both `scrollMarginTop` inline style (CSS anchor/Tab-focus scroll) and the `headerOffset` passed to `scrollElementIntoView` (click-to-expand path). One source of truth for both scroll correction strategies.

## Notable Behaviors

- **Optimistic tickets** (pending `external_id`) render as non-expandable rows at 60% opacity — the drawer requires a real ID to fetch engagements.
- **Scroll-on-expand** uses a rAF-deferred `scrollElementIntoView` call in a `useEffect` keyed on `isExpanded`, intentionally not in the click handler. This avoids the browser scroll-anchoring bug that cancelled native smooth scrolls on every open after the first.
- **Click isolation** — the drawer is a sibling of the toggle `<button>`, not a child, so drawer interactions never bubble to the row's toggle handler.

## Usage Example

```typescript
<HelpCenterCard
  ticket={ticket}
  expanded={expandedId === ticket.id}
  onToggle={(id) => setExpandedId(prev => prev === id ? null : id)}
  busy={isBusy}
  supportSystemDown={false}
  onSendMessage={handleSendMessage}
  onClose={handleClose}
  onReopen={handleReopen}
  onActionCollapsed={() => setExpandedId(null)}
  replyError={replyErrors[ticket.external_id]}
  onClearReplyError={() => clearReplyError(ticket.external_id)}
/>
```
