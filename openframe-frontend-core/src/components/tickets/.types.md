Defines the shared type contract and UI constants for the ticket system, bridging the wire format from `POST /api/chat/agent/find-ticket` with client-side cache management, optimistic updates, and error handling.

## Key Components

### Interfaces
- **`TicketData`** — Full wire shape of a ticket row; mirrors `FindTicketResult` from `hubspot-tools.ts`. Includes HubSpot fields, ClickUp link, owner resolution, and customer metadata
- **`TicketAssignedOwner`** — Resolved agent profile (name, email, avatar) for drawer header attribution
- **`TicketClickupSummary`** — Compact linked ClickUp task projection; aligns with `DeliveryItem` for shared rendering via `DeliveryRow`
- **`OptimisticTicket`** — Extends `TicketData` with `_optimistic: true` for pre-server placeholder rows
- **`TicketsCacheSlot`** — Typed shape of a `['tickets', …]` TanStack Query cache entry; prevents the `t.map is not a function` class of regression
- **`MappedTicketActionError`** — Normalized error envelope with toast copy, cache-mutation hints, and retry metadata

### Types
- **`AnyTicket`** — Union of `TicketData | OptimisticTicket`
- **`TicketActionErrorCode`** — Discriminated string union of all server-side error codes routed through `mapTicketActionError`

### Functions
- **`isOptimistic(t)`** — Type guard narrowing `AnyTicket` to `OptimisticTicket`

### Constants
- **`TICKET_TEXT_MAX_CHARS`** — `5000` — Client-side character cap for ticket content
- **`TICKET_LIVE_POLL_MS`** — `8000` — Shared poll interval (ms) for both list and conversation refresh
- **`TOAST_COPY`** — Centralized, `as const` map of all user-visible toast strings

## Usage Example

```typescript
import {
  AnyTicket,
  isOptimistic,
  TicketsCacheSlot,
  TICKET_LIVE_POLL_MS,
  TICKET_TEXT_MAX_CHARS,
} from './types'

// Type guard for optimistic rows
function renderTicket(t: AnyTicket) {
  if (isOptimistic(t)) return <Skeleton />
  return <TicketDrawer ticket={t} />
}

// Safe cache mutation (avoids t.map regression)
queryClient.setQueriesData<TicketsCacheSlot>(['tickets'], (prev) => ({
  ...prev,
  tickets: prev?.tickets?.map((t) =>
    t.id === updatedId ? { ...t, status: 'CLOSED' } : t
  ),
}))

// Validate before submit
if (content.length > TICKET_TEXT_MAX_CHARS) {
  toast.error(`Max ${TICKET_TEXT_MAX_CHARS} characters`)
}
```
