A React component that embeds and manages Figma prototype viewers with responsive design, device detection, and section navigation capabilities.

## Key Components

**Main Component:**
- `FigmaPrototypeViewer` - Primary component for embedding Figma prototypes with adaptive scaling and device-specific rendering

**Core Interfaces:**
- `FigmaPrototypeViewerConfig` - Configuration interface supporting separate mobile/desktop files, sections, and dimensions
- `FigmaPrototypeSection` - Section definition with mobile/desktop-specific starting points
- `UnifiedState` - Comprehensive state management for view modes, scaling, and UI control

**Key Functions:**
- `getViewMode()` - Determines view mode based on screen size and touch capability
- `calculateScaling()` - Computes optimal iframe scaling and positioning
- `resolveFileKey()` - Resolves appropriate Figma file key from environment variables or config
- `generateEmbedUrl()` - Creates Figma embed URLs with proper parameters

**View Modes:**
- `DESKTOP` - Desktop view for non-touch devices â‰¥768px
- `MOBILE` - Mobile view for non-touch devices <768px  
- `MOBILE_TOUCH` - Touch-enabled mobile interface

## Usage Example

```typescript
import { FigmaPrototypeViewer } from './figma-prototype-viewer'

const prototypeConfig = {
  title: "Mobile App Prototype",
  mobileFileKey: "mobile-figma-file-key",
  desktopFileKey: "desktop-figma-file-key",
  sections: [
    {
      id: "onboarding",
      number: "01",
      title: "Onboarding Flow",
      startingNodeId: "123:456",
      mobileStartingNodeId: "789:012"
    }
  ],
  mobileContentDimensions: { width: 375, height: 600 },
  desktopContentDimensions: { width: 1024, height: 768 }
}

<FigmaPrototypeViewer 
  config={prototypeConfig}
  activeSection="onboarding"
  onSectionClick={(id) => console.log('Section changed:', id)}
/>
```
