A React component for displaying organization logos with automatic fallback to initials, built to match VendorIcon styling exactly while supporting both pre-fetched blob URLs and on-demand authenticated image fetching.

## Key Components

- **OrganizationIcon** - Main component with dual-mode image handling (blob URLs vs API fetching)
- **OrganizationIconProps** - Comprehensive interface with size variants, background styles, and authentication config
- **sizeClasses** - Tailwind size mappings (xs: 24px â†’ xl: 64px) matching VendorIcon exactly
- **imageSizeMap** - Internal Next.js Image dimensions for each size variant
- **backgroundClasses** - Three background style variants (dark, light, white) with ODS theming

## Usage Example

```typescript
import { OrganizationIcon } from './organization-icon'

// With pre-fetched blob URL (for batch loading)
<OrganizationIcon
  imageUrl={batchImages[org.imageUrl]}
  organizationName="Acme Corp"
  size="md"
  preFetched={true}
/>

// With automatic authenticated fetching
<OrganizationIcon
  imageUrl="/api/organizations/123/image"
  organizationName="Tech Solutions"
  size="xl"
  refreshKey={organization.imageVersion}
  config={{ retryCount: 2, timeout: 5000 }}
/>

// Inline without background
<OrganizationIcon
  imageUrl={org.imageUrl}
  organizationName="Startup Inc"
  size="sm"
  showBackground={false}
  backgroundStyle="white"
/>
```

The component automatically detects blob URLs vs API paths, provides loading states, and falls back to 2-letter initials when images fail to load, ensuring zero layout shifts and consistent visual presentation.
