Central barrel export file for the OpenFrame UI component library, re-exporting all feature components, providers, hooks, and utilities as a unified public API.

## Key Components

**Theme System**
- `DynamicThemeProvider` / `useDynamicTheme` — runtime theme switching
- `ThemeProvider` / `useTheme` / `useThemeToggle` — canonical ODS theme layer (wraps `next-themes`), headless by design
- `THEME_STORAGE_KEY`, `THEME_ATTRIBUTE`, `DEFAULT_THEME` — theme constants
- `Theme`, `ThemeProviderProps`, `UseThemeToggleResult` — TypeScript types

**Video Primitives**
- `video` — unified `<Video>` component replacing legacy `<VideoPlayer>` + `<YouTubeEmbed>`
- `entity-video-section`, `video-bites-display` — public detail-page wrappers
- `video-ratio-tabs` — aspect-ratio grouping shared by bites grid and admin editors
- `use-video-warmup`, `captions-url`, `video-source-selector`, `transcript-summary-editor`

**Feature Components (selected)**
- `command-box`, `filters-dropdown`, `tags-selector`, `select-button`
- `changelog-manager`, `changelog-sections-manager`
- `ai-enrich`, `policy-configuration-panel`
- `board`, `ticket-status-config-list`, `notifications`
- `error-boundary`, `loading-provider`, `seo-editor-preview`

## Usage Example

```typescript
import {
  ThemeProvider,
  useThemeToggle,
  Video,
  CommandBox,
  ErrorBoundary,
  AiEnrich,
} from '@openframe/ui/features'

function App() {
  return (
    <ThemeProvider>
      <ErrorBoundary>
        <CommandBox />
        <Video src="https://..." />
      </ErrorBoundary>
    </ThemeProvider>
  )
}

// Headless theme toggle — bring your own button
function ThemeToggle() {
  const { toggle, theme } = useThemeToggle()
  return <Button onClick={toggle}>Mode: {theme}</Button>
}
```
