Builds the captions API URL for a video entity, returning the HTTPS endpoint path that serves VTT content for iOS native fullscreen subtitles, or `undefined` if no subtitle content exists.

## Key Components

### `getCaptionsUrl(entityType, entityId, srtContent?)`

Constructs a cache-busted URL pointing to `/api/captions/[entityType]/[entityId]`.

| Parameter | Type | Description |
|-----------|------|-------------|
| `entityType` | `string` | Entity type discriminator matching the reverse-proxied route |
| `entityId` | `string \| number` | Unique identifier for the entity |
| `srtContent` | `string \| null \| undefined` | Raw SRT subtitle content; returns `undefined` if falsy |

**Cache-busting strategy:** The `?v=` hash is derived from `srtContent.length` + the first 8 non-whitespace characters, ensuring iOS Safari re-fetches the `<track>` src when subtitles are regenerated (Safari aggressively caches these URLs regardless of `Cache-Control`).

## Usage Example

```typescript
import { getCaptionsUrl } from './captions-url'

// Returns URL with cache-busting hash
const url = getCaptionsUrl('video', 42, '1\n00:00:01...')
// → '/api/captions/video/42?v=15-100000'

// Returns undefined when no subtitles available
const noUrl = getCaptionsUrl('video', 42, null)
// → undefined

// Use with a <track> element
const trackSrc = getCaptionsUrl('episode', episode.id, episode.srt_content)
if (trackSrc) {
  // Safe to attach as <track src={trackSrc} kind="subtitles" />
}
```

> **Note:** `entityType` is typed as `string` (widened from the hub's `VideoEnabledEntityType` enum) so embedders can pass any entity-type discriminator their reverse-proxied `/api/captions/...` route expects.
