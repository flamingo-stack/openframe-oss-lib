Builds the captions API URL for a video entity, returning the HTTPS endpoint path to `/api/captions/[entityType]/[entityId]` with a cache-busting hash. Returns `undefined` if the entity has no SRT content.

## Key Components

- **`getCaptionsUrl(entityType, entityId, srtContent?)`** — Constructs a versioned captions URL for VTT subtitle delivery. Accepts any string entity type discriminator (widened from the hub's `VideoEnabledEntityType` enum), making it usable across reverse-proxied embedder routes.

The cache-busting `?v=` hash is derived from the `srtContent` length and first 8 non-whitespace characters, ensuring iOS Safari re-fetches VTT files when subtitles are regenerated (Safari aggressively caches `<track>` src URLs regardless of `Cache-Control` headers).

## Usage Example

```typescript
import { getCaptionsUrl } from './captions-url'

// Standard usage
const url = getCaptionsUrl('video', 42, srtContent)
// → "/api/captions/video/42?v=1024-1BrianWas"

// Returns undefined when no subtitles exist
const noUrl = getCaptionsUrl('video', 42, null)
// → undefined

// Custom entity type for embedder routes
const embedUrl = getCaptionsUrl('lesson', 'abc-123', srtContent)
// → "/api/captions/lesson/abc-123?v=512-1BrianWas"
```

> **Note:** Ported from hub `lib/utils/captions-url.ts`. The entity type is intentionally typed as `string` rather than a closed enum so downstream embedders can pass whatever discriminator their proxied `/api/captions/...` route expects.
