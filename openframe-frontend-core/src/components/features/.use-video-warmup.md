Provides two React hooks for warming network connections to video assets, reducing click-to-first-frame latency by preconnecting to video origins and conditionally preloading video bytes based on viewport proximity.

## Key Components

### `useVideoOriginPreconnect(options?)`
Preconnect-only hook that fires TCP/TLS handshakes to Mux stream, Mux image, and Supabase storage origins via `ReactDOM.preconnect`. Use this when you cannot attach a `ref` to the video container (e.g. a sibling-rendered player).

### `useVideoWarmup<T>(options?)`
Full warmup hook that composes `useVideoOriginPreconnect` with an IntersectionObserver-gated `<link rel="preload" as="video">` injection. The preload fires only when all three conditions are met:
- The container scrolls within `nearMargin` of the viewport
- `navigator.connection?.saveData !== true`
- The video URL resolves to the Supabase storage origin

Returns `{ ref, isNear }` — attach `ref` to the video container element.

### Re-exports
`MUX_STREAM_ORIGIN` and `MUX_IMAGE_ORIGIN` — re-exported from the server-safe `mux-origins.ts` module to avoid binding these constants to a `'use client'` boundary.

## Usage Example

```typescript
// Full warmup — owns the video container
function VideoCard({ videoUrl }: { videoUrl: string }) {
  const { ref, isNear } = useVideoWarmup<HTMLDivElement>({
    videoUrl,
    nearMargin: '800px',
  })

  return (
    <div ref={ref}>
      {isNear && <MuxPlayer src={videoUrl} />}
    </div>
  )
}

// Preconnect-only — no container ref available
function ReleaseDetailPage() {
  useVideoOriginPreconnect()
  return <PlayerSiblingComponent />
}
```

**Origin resolution:** Both hooks read `supabaseStorageOrigin` from `ChatRuntime.endpoints` automatically when `HubRuntimeProvider` is mounted. Pass `supabaseStorageOrigin` explicitly to override.
