Provides network warming for video content by preconnecting to video-bearing origins and conditionally preloading video bytes when a container nears the viewport.

## Key Components

### `useVideoOriginPreconnect(options?)`
Fires preconnect hints to Mux stream/image CDN origins and Supabase Storage without any IntersectionObserver setup. Use this when no container `ref` is available (e.g. a page that delegates player rendering to a sibling component).

- **`supabaseStorageOrigin?`** — overrides the value auto-read from `ChatRuntime.endpoints.supabaseStorageOrigin`

### `useVideoWarmup<T>(options?)`
Full warmup hook. Composes `useVideoOriginPreconnect` with an IO-gated preload step that injects a `<link rel="preload" as="video">` when:
- The container ref enters within `nearMargin` of the viewport
- `navigator.connection?.saveData` is not `true`
- `videoUrl` resolves to the Supabase Storage origin (Mux HLS and YouTube are intentionally excluded)

**Options:**

| Option | Default | Description |
|---|---|---|
| `videoUrl` | `undefined` | Effective video URL; only Supabase-hosted URLs are preloaded |
| `supabaseStorageOrigin` | runtime fallback | Deployment-specific Supabase storage origin |
| `nearMargin` | `'1000px'` | IO root margin for the preload gate |

**Returns:** `{ ref, isNear }` — attach `ref` to the video container element.

### Re-exports
`MUX_STREAM_ORIGIN`, `MUX_IMAGE_ORIGIN` — re-exported from the server-safe `mux-origins.ts` module to avoid binding those constants to this `'use client'` file.

## Usage Example

```typescript
// Full warmup — owns the video container
const { ref } = useVideoWarmup({
  videoUrl: entity.videoUrl,
  nearMargin: '800px',
})
return <div ref={ref}><MuxPlayer /></div>

// Preconnect-only — no container ref available
useVideoOriginPreconnect()
// supabaseStorageOrigin auto-resolved from ChatRuntime when
// HubRuntimeProvider is mounted in the tree
```
