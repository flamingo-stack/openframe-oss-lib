Defines the canonical Mux CDN origin URL constants, isolated in a server-safe module to prevent Next.js client-reference poisoning when imported by server-side code.

## Key Components

| Export | Value | Description |
|--------|-------|-------------|
| `MUX_STREAM_ORIGIN` | `https://stream.mux.com` | HLS playback base URL for `.m3u8` manifests, segments, and MP4 renditions |
| `MUX_IMAGE_ORIGIN` | `https://image.mux.com` | Server-generated thumbnail base URL |

## Usage Example

```typescript
import { MUX_STREAM_ORIGIN, MUX_IMAGE_ORIGIN } from '@/lib/config/mux-origins'

// Build a playback URL (server or client safe)
const playbackUrl = `${MUX_STREAM_ORIGIN}/${playbackId}.m3u8`

// Build a thumbnail URL
const thumbnailUrl = `${MUX_IMAGE_ORIGIN}/${playbackId}/thumbnail.jpg`

// Safe for hostname extraction in server-side code
const streamHost = new URL(MUX_STREAM_ORIGIN).hostname // 'stream.mux.com'
```

## Why a Separate Module

Previously co-located with `use-video-warmup.ts` (`'use client'`), server-side imports caused a Vercel build crash — Next.js replaced the constants with client-function stubs, making `new URL(MUX_STREAM_ORIGIN)` throw `TypeError: Invalid URL`. Splitting into this non-client module restores server-safe imports while `use-video-warmup.ts` re-exports these for backward compatibility with existing client callers.
