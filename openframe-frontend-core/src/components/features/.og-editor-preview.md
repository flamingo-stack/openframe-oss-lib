Reusable SEO & Open Graph editor panel that combines meta fields, OG image upload, and a live preview into a single composable component.

## Key Components

**`OGEditorPreview`** — Main component rendering a card layout with:
- SEO Title + Keywords (side-by-side grid)
- SEO Description + OG Image upload (side-by-side grid)
- OG Preview section (injected via render prop)

**`OGEditorPreviewProps`** — Interface defining:
- SEO field values (`seoTitle`, `seoDescription`, `seoKeywords`, `ogImageUrl`)
- Auto-populate sources (`title`, `summary`, `featuredImage`, `categories`)
- Change handlers for each SEO field
- `OGImageUploadComponent` / `OGPreviewComponent` — render props for parent-controlled upload/preview logic

## Usage Example

```typescript
import { OGEditorPreview } from '@/components/og-editor-preview';

<OGEditorPreview
  seoTitle={post.seoTitle}
  seoDescription={post.seoDescription}
  seoKeywords={post.seoKeywords}
  ogImageUrl={post.ogImageUrl}
  title={post.title}
  summary={post.summary}
  featuredImage={post.featuredImage}
  onSeoTitleChange={(val) => updatePost({ seoTitle: val })}
  onSeoDescriptionChange={(val) => updatePost({ seoDescription: val })}
  onSeoKeywordsChange={(val) => updatePost({ seoKeywords: val })}
  onOgImageUrlChange={(val) => updatePost({ ogImageUrl: val })}
  OGImageUploadComponent={<OGImageUploader postId={post.id} />}
  OGPreviewComponent={<OGPreview title={post.seoTitle} image={post.ogImageUrl} />}
  disabled={isSaving}
/>
```

## Notes

- Upload and preview logic are intentionally **decoupled** via render props, keeping this component reusable across blog posts, case studies, and product releases.
- `disabled` prop propagates to all form inputs for save/loading states.
- Responsive grid collapses to single column on smaller screens (`lg:grid-cols-2`).
