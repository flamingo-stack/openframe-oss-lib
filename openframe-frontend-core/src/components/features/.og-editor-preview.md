Unified SEO and Open Graph editor component that combines meta title, description, keywords, OG image upload, and social preview into a single reusable panel for content editors.

## Key Components

**`OGEditorPreview`** — Main exported component rendering a two-column grid layout with:
- SEO Title + Keywords (row 1)
- SEO Description + OG Image uploader (row 2)
- OG social preview panel (row 3)

**`OGEditorPreviewProps`** — Interface defining all required and optional props including SEO field values, change handlers, and injectable sub-components.

## Usage Example

```typescript
import { OGEditorPreview } from './og-editor-preview';

<OGEditorPreview
  seoTitle={post.seoTitle}
  seoDescription={post.seoDescription}
  seoKeywords={post.seoKeywords}
  ogImageUrl={post.ogImageUrl}
  title={post.title}
  summary={post.summary}
  featuredImage={post.featuredImage}
  categories={post.categories}
  onSeoTitleChange={(val) => updateField('seoTitle', val)}
  onSeoDescriptionChange={(val) => updateField('seoDescription', val)}
  onSeoKeywordsChange={(val) => updateField('seoKeywords', val)}
  onOgImageUrlChange={(val) => updateField('ogImageUrl', val)}
  OGImageUploadComponent={<OGImageUploader />}
  OGPreviewComponent={<OGPreview title={post.seoTitle} />}
  disabled={isSaving}
/>
```

## Notes

- Designed for use across **blog posts**, **case studies**, and **product releases**
- `OGImageUploadComponent` and `OGPreviewComponent` are injected as `React.ReactNode`, keeping upload/preview logic decoupled from this component
- `disabled` prop propagates to all form inputs for loading/saving states
- Several props on the interface (`ogImageUrl`, `title`, `summary`, `featuredImage`, `categories`, `onOgImageUrlChange`) are declared but not consumed by the component body — intended as data sources for parent-supplied child components
