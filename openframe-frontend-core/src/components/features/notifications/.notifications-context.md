React context provider for managing in-app notification state, including read/unread tracking, popup visibility, drawer open state, and optional server-side persistence callbacks.

## Key Components

**`NotificationsProvider`** — Top-level provider wrapping the app (or a subtree). Manages notification state via a `useReducer` and exposes all actions through context.

**`useNotifications()`** — Hook for consuming the context; throws if used outside the provider.

**`useOptionalNotifications()`** — Safe variant that returns `null` instead of throwing when used outside the provider.

**`NotificationsActions`** — Optional side-effect callbacks (`onMarkRead`, `onMarkAllRead`, `onRemove`, `onMarkSettled`) for persisting changes server-side after local state updates.

**`reducer`** — Pure reducer handling `add`, `upsert`, `set`, `markRead`, `markAllRead`, `markSettled`, `remove`, and `clear` actions. The `upsert` action preserves existing item position; `add` always promotes to the top.

## Usage Example

```typescript
// 1. Wrap your app (or feature subtree)
import { NotificationsProvider, useNotifications } from './notifications-context'

function App() {
  return (
    <NotificationsProvider
      maxNotifications={100}
      defaultShowPopups={true}
      actions={{
        onMarkRead: async (id) => await api.markNotificationRead(id),
        onRemove: async (id) => await api.deleteNotification(id),
      }}
      hasMore={hasNextPage}
      isLoadingMore={isFetching}
      onLoadMore={fetchNextPage}
    >
      <Shell />
    </NotificationsProvider>
  )
}

// 2. Consume in any child component
function NotificationBell() {
  const { unreadCount, addNotification, markAllRead, open } = useNotifications()

  const notify = () =>
    addNotification({
      title: 'Ticket updated',
      description: 'Ticket #1042 was assigned to you.',
      type: 'info',
    })

  return (
    <button onClick={open}>
      🔔 {unreadCount > 0 && <span>{unreadCount}</span>}
    </button>
  )
}
```

## Notes

- Notifications are capped at `maxNotifications` (default `50`); oldest entries are dropped on overflow.
- `addNotification` promotes duplicates to the top; `upsertNotification` updates in-place.
- `actions` callbacks are stored in a ref, so they can be updated without re-subscribing consumers.
- IDs are auto-generated via `crypto.randomUUID()` with a `Date.now()` fallback.
