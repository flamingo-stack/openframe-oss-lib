React context provider for managing in-app notification state, including read/unread tracking, drawer open state, popup visibility, and optional server-side persistence callbacks.

## Key Components

**`NotificationsProvider`** — Top-level provider wrapping the app or a subtree. Manages a `useReducer`-based notification list with a configurable cap (`maxNotifications`, default `50`), drawer open state, and popup visibility.

**`useNotifications()`** — Hook that returns the full context value; throws if used outside the provider.

**`useOptionalNotifications()`** — Safe version that returns `null` instead of throwing when used outside the provider.

**`NotificationsActions`** — Optional side-effect callbacks (`onMarkRead`, `onMarkAllRead`, `onRemove`, `onMarkSettled`) fired after local state updates for server-side persistence.

**Reducer actions** — `add`, `upsert`, `set`, `markRead`, `markAllRead`, `markSettled`, `remove`, `clear`

- `add` prepends and deduplicates by `id`, respects `maxNotifications`
- `upsert` merges in-place if the `id` already exists, otherwise prepends
- `set` syncs a full list while preserving existing `read`/`settled` state

## Usage Example

```typescript
// Wrap your app
<NotificationsProvider
  initialNotifications={serverNotifications}
  maxNotifications={100}
  actions={{
    onMarkRead: async (id) => await api.markNotificationRead(id),
    onMarkAllRead: async () => await api.markAllRead(),
    onRemove: async (id) => await api.deleteNotification(id),
  }}
  hasMore={pagination.hasMore}
  isLoadingMore={pagination.isLoading}
  onLoadMore={pagination.loadNextPage}
>
  <App />
</NotificationsProvider>

// Consume in any child component
function NotificationBell() {
  const { unreadCount, toggle, addNotification } = useNotifications()

  const handleNew = () => {
    addNotification({
      title: 'Ticket updated',
      message: 'INC-1042 was assigned to you',
      type: 'info',
    })
  }

  return <button onClick={toggle}>🔔 {unreadCount}</button>
}
```
