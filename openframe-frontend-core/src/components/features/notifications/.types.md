Defines the core TypeScript types and interfaces for the notification system used across the Flamingo/OpenFrame platform.

## Key Components

- **`NotificationVariant`** — Visual style variants: `'default'`, `'success'`, `'warning'`, `'error'`, `'info'`
- **`NotificationSeverity`** — Severity levels for triage/filtering: `'INFO'`, `'WARNING'`, `'DANGER'`
- **`Notification`** — Full notification object interface including display, state, and behavioral fields
- **`AddNotificationInput`** — Input type for creating notifications; makes `id`, `createdAt`, and `read` optional while omitting `settled`

## Usage Example

```typescript
import type { Notification, AddNotificationInput } from './types'

// Creating a new notification (input shape)
const input: AddNotificationInput = {
  variant: 'success',
  title: 'Ticket Resolved',
  description: 'Ticket #1042 has been closed.',
  severity: 'INFO',
  category: 'tickets',
  onClick: () => console.log('Notification clicked'),
}

// Full notification (as stored in state)
const notification: Notification = {
  id: 'notif-abc123',
  createdAt: Date.now(),
  read: false,
  settled: false,
  ...input,
}
```

## Notes

- `title` and `description` accept `ReactNode`, allowing rich JSX content inside notifications
- `meta` is a flexible `Record<string, unknown>` for attaching arbitrary context (e.g., ticket IDs, device references)
- `AddNotificationInput` derives from `Notification` via `Omit` + optional overrides, ensuring type consistency between creation and storage
