Type definitions for the notification system, defining the shape of notification objects and related input types used across the UI.

## Key Components

- **`NotificationVariant`** — Union type for visual styling: `'default' | 'success' | 'warning' | 'error' | 'info'`
- **`NotificationSeverity`** — Union type for severity levels: `'INFO' | 'WARNING' | 'DANGER'`
- **`Notification`** — Core interface representing a full notification record with display, state, and interaction fields
- **`AddNotificationInput`** — Input type for creating notifications, omitting server-managed fields (`id`, `createdAt`, `settled`) and making `id`, `createdAt`, and `read` optional

## Notification Interface Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | ✅ | Unique identifier |
| `title` | `ReactNode` | ✅ | Primary display content |
| `createdAt` | `number` | ✅ | Unix timestamp |
| `variant` | `NotificationVariant` | ❌ | Visual style |
| `description` | `ReactNode` | ❌ | Secondary content |
| `read` | `boolean` | ❌ | Read state |
| `settled` | `boolean` | ❌ | Dismissed/resolved state |
| `severity` | `NotificationSeverity` | ❌ | Alert severity level |
| `category` | `string` | ❌ | Grouping label |
| `meta` | `Record<string, unknown>` | ❌ | Arbitrary metadata |
| `onClick` | `() => void` | ❌ | Click handler |

## Usage Example

```typescript
import type { Notification, AddNotificationInput } from './types'

// Creating a new notification
const input: AddNotificationInput = {
  title: 'Ticket Updated',
  description: 'Ticket #1042 has been resolved.',
  variant: 'success',
  severity: 'INFO',
  category: 'tickets',
  onClick: () => router.push('/tickets/1042'),
}

// Full notification after system assignment
const notification: Notification = {
  ...input,
  id: 'notif_abc123',
  createdAt: Date.now(),
  read: false,
  settled: false,
}
```
