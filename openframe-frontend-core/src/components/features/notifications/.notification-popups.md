Renders a fixed-position overlay of transient notification toasts, consuming live (unread, unsettled, within `liveDurationMs`) notifications from the notifications context.

## Key Components

**`NotificationPopupsPosition`** — Union type for the four supported corner positions: `'top-right'`, `'top-left'`, `'bottom-right'`, `'bottom-left'`.

**`NotificationPopupsProps`** — Configuration interface:
| Prop | Default | Description |
|---|---|---|
| `liveDurationMs` | `4000` | How long (ms) a notification stays visible |
| `maxVisible` | `4` | Maximum simultaneous toasts |
| `position` | `'top-right'` | Corner anchor for the popup stack |
| `hideWhenDrawerOpen` | `true` | Hides popups when the notification drawer is open |

**`NotificationPopups`** — Main export. Reads from `useOptionalNotifications` context; returns `null` if context is absent, `showPopups` is false, or no live notifications exist.

**Activation behavior** — Clicking or pressing `Enter`/`Space` on a toast:
- If `notification.onClick` exists → calls it, marks the notification read and settled
- Otherwise → opens the notification drawer, marks settled only

Nested `<button>` elements (dismiss/action controls) handle their own events independently via `closest('button')` guard.

## Usage Example

```typescript
import { NotificationPopups } from './notification-popups'
import { NotificationsProvider } from './notifications-context'

export function AppShell() {
  return (
    <NotificationsProvider>
      <main>{/* app content */}</main>

      {/* Toasts in bottom-right, hidden after 5s, max 3 visible */}
      <NotificationPopups
        position="bottom-right"
        liveDurationMs={5000}
        maxVisible={3}
        hideWhenDrawerOpen={true}
      />
    </NotificationsProvider>
  )
}
```
