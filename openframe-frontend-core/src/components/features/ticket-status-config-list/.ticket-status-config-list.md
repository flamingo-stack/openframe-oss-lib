Reusable drag-and-drop sortable list component for ticket status configuration, built on `@dnd-kit/core` with pointer and keyboard sensor support.

## Key Components

- **`TicketStatusConfigList<T>`** — Generic sortable list component that wraps `DndContext` and `SortableContext`, accepting any items with an `id` string field
- **`SortableRow`** — Internal render-prop component that applies drag transform/transition styles via `useSortable` and exposes drag handle props to the parent
- **`SortableRowRenderArgs`** — Interface providing `dragHandleProps`, `dragHandleAttributes`, and `isDragging` to the consumer's render function
- **`TicketStatusConfigListProps<T>`** — Props interface with `items`, `onReorder` callback (old/new index), `renderRow` render prop, and optional `className`

## Usage Example

```typescript
import {
  TicketStatusConfigList,
  type SortableRowRenderArgs,
} from './ticket-status-config-list'

const statuses = [
  { id: '1', label: 'Open' },
  { id: '2', label: 'In Progress' },
  { id: '3', label: 'Closed' },
]

function StatusSettings() {
  const [items, setItems] = React.useState(statuses)

  const handleReorder = (oldIndex: number, newIndex: number) => {
    const reordered = [...items]
    const [moved] = reordered.splice(oldIndex, 1)
    reordered.splice(newIndex, 0, moved)
    setItems(reordered)
  }

  return (
    <TicketStatusConfigList
      items={items}
      onReorder={handleReorder}
      renderRow={(item, { dragHandleProps, dragHandleAttributes, isDragging }: SortableRowRenderArgs) => (
        <div style={{ opacity: isDragging ? 0.5 : 1 }}>
          <span {...dragHandleProps} {...dragHandleAttributes}>⠿</span>
          {item.label}
        </div>
      )}
    />
  )
}
```

> Drag activation requires a minimum pointer movement of 4px, preventing accidental reorders on click. Keyboard reordering is also fully supported via `sortableKeyboardCoordinates`.
