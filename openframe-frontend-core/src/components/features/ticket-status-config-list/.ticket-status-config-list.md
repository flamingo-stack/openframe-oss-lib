A reusable drag-and-drop sortable list component for ticket status configuration, built on `@dnd-kit/core` with keyboard and pointer sensor support.

## Key Components

- **`TicketStatusConfigList<T>`** — Generic sortable list container that manages DnD context, sensors, and reorder callbacks. Accepts any item type with an `id: string` field.
- **`SortableRow`** — Internal wrapper that applies DnD transform/transition styles and exposes drag handle props via a render-prop pattern.
- **`TicketStatusConfigListProps<T>`** — Props interface accepting `items`, `onReorder(oldIndex, newIndex)`, a `renderRow` render-prop, and an optional `className`.
- **`SortableRowRenderArgs`** — Render-prop argument type exposing `dragHandleProps`, `dragHandleAttributes`, and `isDragging` to the consumer.

## Usage Example

```typescript
import {
  TicketStatusConfigList,
  type SortableRowRenderArgs,
} from './ticket-status-config-list'

const statuses = [
  { id: '1', label: 'Open' },
  { id: '2', label: 'In Progress' },
  { id: '3', label: 'Closed' },
]

function StatusSettings() {
  const [items, setItems] = React.useState(statuses)

  const handleReorder = (oldIndex: number, newIndex: number) => {
    setItems(prev => {
      const next = [...prev]
      const [moved] = next.splice(oldIndex, 1)
      next.splice(newIndex, 0, moved)
      return next
    })
  }

  return (
    <TicketStatusConfigList
      items={items}
      onReorder={handleReorder}
      renderRow={(item, { dragHandleProps, dragHandleAttributes, isDragging }: SortableRowRenderArgs) => (
        <div style={{ opacity: isDragging ? 0.5 : 1 }}>
          <span {...dragHandleAttributes} {...dragHandleProps}>⠿</span>
          {item.label}
        </div>
      )}
    />
  )
}
```

The `onReorder` callback receives array indices rather than IDs, so the parent is responsible for applying the reorder logic to its own state.
