A reusable authentication provider button component that supports multiple OAuth providers (Microsoft, Google, Slack, GitHub) with built-in loading states and error handling.

## Key Components

- **ProviderButton**: Main component that renders provider-specific sign-in buttons
- **ProviderButtonProps**: Interface defining component props including provider type, click handler, and optional disabled/loading states
- **PROVIDER_CONFIG**: Configuration object mapping provider types to display names and icons
- **Internal loading state management**: Handles loading state when not externally managed

## Usage Example

```typescript
import { ProviderButton } from './provider-button';

// Basic usage
<ProviderButton
  provider="google"
  onClick={async () => {
    await signInWithProvider('google');
  }}
/>

// With external loading state
<ProviderButton
  provider="github"
  onClick={handleGitHubSignIn}
  loading={isAuthenticating}
  disabled={!isFormValid}
/>

// Multiple providers
{['google', 'microsoft', 'github'].map(provider => (
  <ProviderButton
    key={provider}
    provider={provider}
    onClick={() => handleProviderAuth(provider)}
  />
))}
```

The component automatically handles loading states, prevents event bubbling, displays appropriate icons, and provides accessibility features through ARIA labels.
