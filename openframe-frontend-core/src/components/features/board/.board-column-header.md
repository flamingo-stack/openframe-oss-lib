Renders the header for a Kanban board column, supporting both expanded and collapsed states with status tag, ticket count, and optional action buttons.

## Key Components

**`BoardColumnHeaderProps`** — Interface defining the component's props:
- `column` — `BoardColumnDef` containing label, status key, color, and ticket data
- `collapsed` — toggles between vertical (collapsed) and horizontal (expanded) layouts
- `onToggleCollapse` — required callback to expand/collapse the column
- `onAddTicket` — optional callback; renders a `+` button when provided
- `onArchive` — optional callback; renders an archive button when provided

**`BoardColumnHeader`** — Main exported component with two render paths:
- **Collapsed:** vertical layout with expand arrow, rotated status tag, and ticket count
- **Expanded:** horizontal layout with status tag, count, collapse arrow, and optional archive/add-ticket actions

**Status resolution logic** — Uses `resolveTicketStatus` to determine if the column maps to a known canonical status (e.g. `ACTIVE`, `RESOLVED`). Known statuses use the built-in tag variant; unrecognized IDs (custom UUIDs) fall back to the column's `color` prop.

## Usage Example

```typescript
<BoardColumnHeader
  column={{
    id: 'in-progress',
    label: 'In Progress',
    tickets: [],
    total: 12,
    color: '#FFC008',
  }}
  collapsed={false}
  onToggleCollapse={() => setCollapsed(true)}
  onAddTicket={() => openNewTicketModal('in-progress')}
  onArchive={() => archiveResolved('in-progress')}
/>
```
