Core type definitions and utility functions for the ticket board (kanban-style) feature, including column definitions, ticket shapes, drag-and-drop change descriptors, and helpers for building columns from ticket statuses.

## Key Components

### Types & Interfaces

| Export | Description |
|--------|-------------|
| `BoardPriority` | Union type: `'low' \| 'medium' \| 'high' \| 'urgent'` |
| `BoardTicketAssignee` | Assignee shape with optional avatar, initials, and name |
| `BoardTicket` | Core ticket shape used on the board |
| `BoardColumnDef` | Full column definition including display, loading state, and DnD constraints |
| `BoardChange` | Describes a ticket move event between columns |

### Functions

| Export | Description |
|--------|-------------|
| `columnFromTicketStatus` | Builds a `BoardColumnDef` from a canonical `TicketStatus` with optional overrides |
| `groupTicketsByStatus` | Groups a flat ticket array into ordered `BoardColumnDef[]` by status |

## Usage Example

```typescript
import { columnFromTicketStatus, groupTicketsByStatus } from './types'

// Build a single column with overrides
const activeColumn = columnFromTicketStatus('ACTIVE', myTickets, {
  label: 'In Progress',
  dropDisabled: false,
  system: true,
})

// Group all tickets into ordered board columns
const columns = groupTicketsByStatus(allTickets, [
  'ACTIVE',
  'AI_ASSISTANCE',
  'TECH_REQUIRED',
  'ON_HOLD',
  'RESOLVED',
])
```

## Notes

- `BoardColumnDef.statusKey` allows custom-status columns (UUID `id`) to still resolve the correct `TicketStatusTag` variant for header styling.
- `canonicalize()` (internal) normalises status strings — mapping legacy `ACTION_REQUIRED` → `TECH_REQUIRED` and uppercasing/underscoring free-text values before bucketing.
- `STATUS_DEFAULTS` defines the canonical label and hex color for each system status used as fallbacks in `columnFromTicketStatus`.
