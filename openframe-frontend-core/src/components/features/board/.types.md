Core type definitions and utilities for the ticket board's column/card data model, including interfaces for tickets, columns, and drag-drop changes, plus helpers to build and group columns from ticket statuses.

## Key Components

**Types**
- `BoardPriority` — Union type for ticket priority levels (`low | medium | high | urgent`)
- `BoardTicketAssignee` — Assignee shape with optional avatar/initials
- `BoardTicket` — Card data: id, title, number, status, and optional metadata
- `BoardColumnDef` — Column definition including tickets, display config, pagination state, and drag-drop rules
- `BoardChange` — Describes a ticket move: source/target columns and surrounding ticket ids for ordering

**Functions**
- `columnFromTicketStatus()` — Creates a `BoardColumnDef` from a canonical `TicketStatus`, applying `STATUS_DEFAULTS` with optional overrides
- `groupTicketsByStatus()` — Buckets a flat ticket array into ordered columns for a given status list
- `canonicalize()` *(internal)* — Normalizes raw status strings to `TicketStatus` keys (e.g. maps `ACTION_REQUIRED → TECH_REQUIRED`)

## Usage Example

```typescript
import { groupTicketsByStatus, columnFromTicketStatus } from './types'

// Build columns from a flat ticket list
const columns = groupTicketsByStatus(tickets, ['ACTIVE', 'ON_HOLD', 'RESOLVED'])

// Build a single column with custom label and pagination state
const activeColumn = columnFromTicketStatus('ACTIVE', activeTickets, {
  label: 'In Progress',
  total: 42,
  hasMore: true,
  isLoadingMore: false,
})
```
