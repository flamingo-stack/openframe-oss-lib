Kanban-style drag-and-drop board component that renders columns of tickets with full DnD support via `@dnd-kit/core`, including cross-column moves, drop restrictions, and a custom horizontal scrollbar.

## Key Components

- **`Board`** — Main exported component; manages drag state, column items, and fires `onChange` with move metadata.
- **`collisionDetection`** — Custom collision strategy: prefers ticket hits → column hits → closest corners fallback.
- **`handleDragStart/Over/End/Cancel`** — DnD lifecycle handlers that optimistically mutate local `items` state and reconcile with `columns` prop on end/cancel.
- **`locate`** — Utility to find a ticket across all columns, returning `{ ticket, columnId, index }`.
- **`findIndexInColumn`** — Utility to find a ticket's index within a specific column.
- **`useBoardCollapse`** — Hook managing per-column collapsed state, optionally persisted via `collapseStorageKey`.
- **`useHorizontalScrollbar`** — Hook providing a custom scrollbar track/thumb with pointer drag support.

## Usage Example

```typescript
import { Board } from './board'
import type { BoardColumnDef, BoardChange } from './types'

const columns: BoardColumnDef[] = [
  { id: 'todo',       label: 'To Do',      tickets: [...] },
  { id: 'in-progress',label: 'In Progress', tickets: [...] },
  { id: 'done',       label: 'Done',        tickets: [...], dropDisabled: false },
]

function TicketBoard() {
  const handleChange = (change: BoardChange) => {
    console.log(`Moved ${change.ticketId} from ${change.fromColumnId} to ${change.toColumnId}`)
    // Persist reorder/move to backend here
  }

  return (
    <Board
      columns={columns}
      onChange={handleChange}
      onAddTicket={(colId) => openNewTicketModal(colId)}
      onArchiveColumn={(colId) => archiveColumn(colId)}
      getTicketHref={(id) => `/tickets/${id}`}
      collapseStorageKey="board-collapse-state"
      renderAssignSlot={(ticket) => <AssigneePicker ticket={ticket} />}
    />
  )
}
```

> **Note:** `onChange` fires only on confirmed drops. Optimistic column state is reset to the `columns` prop on drag cancel or invalid drops. Cross-column moves respect `dropDisabled` and `allowedFromColumns` per column definition.
