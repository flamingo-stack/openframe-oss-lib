Provides the core layout primitives for Flamingo's chat UI shell, composing a full-screen chat view with a header, scrollable content area, and footer — used by both the Fae client assistant and Mingo AI technician interfaces.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `ChatContainer` | `forwardRef<div>` | Root full-screen flex wrapper with ODS background/text tokens |
| `ChatHeader` | `forwardRef<div>` | Avatar, name, connection status, ticket info, and action buttons |
| `ChatContent` | `forwardRef<div>` | Flex-grow scrollable content column |
| `ChatFooter` | `forwardRef<div>` | Sticky bottom wrapper with optional narrow-column constraint |
| `ConnectionIndicator` | `FC` | Pulsing dot using ODS attention tokens (`connected` / `connecting` / `disconnected`) |

### `ChatHeader` Notable Props

| Prop | Type | Default | Notes |
|------|------|---------|-------|
| `connectionStatus` | `'connected' \| 'connecting' \| 'disconnected'` | `'disconnected'` | Drives `ConnectionIndicator` dot color |
| `serverUrl` | `string \| null` | `null` | Shows URL + dot when provided |
| `ticketInfo` | `object` | — | Renders a divider row with title, meta, and `TicketStatusTag` |
| `fullWidth` | `boolean` | `false` | Drops `max-w-ods-content-narrow` for side-panel use |
| `bare` | `boolean` | `false` | Removes card border/background |
| `onBack` | `() => void` | — | Renders a back chevron button anchored outside the card |

## Usage Example

```typescript
import {
  ChatContainer,
  ChatHeader,
  ChatContent,
  ChatFooter,
} from "@/components/chat/chat-container"

export function MyChatView() {
  return (
    <ChatContainer>
      <ChatHeader
        userName="Fae"
        userTitle="Your Personal Assistant"
        connectionStatus="connected"
        serverUrl="mcp.flamingo.run"
        onNewChat={() => startNewSession()}
        onClose={() => closePanel()}
        fullWidth
        ticketInfo={{
          title: "VPN not connecting",
          meta: "TKT-1042 · John Smith",
          status: "in_progress",
        }}
      />
      <ChatContent>
        {/* message bubbles */}
      </ChatContent>
      <ChatFooter fullWidth>
        {/* input composer */}
      </ChatFooter>
    </ChatContainer>
  )
}
```

> `fullWidth` should be set consistently on both `ChatHeader` and `ChatFooter` when embedding the chat shell inside a side panel, otherwise the header and input composer will misalign at narrower widths.
