Loading state skeleton components for chat message UI, mimicking the layout of `ChatMessageEnhanced` and `ChatMessageList` during data fetch.

## Key Components

### `ChatMessageSkeleton`
Renders a single animated placeholder message row with:
- Optional inline avatar (assistant only, matches 2025 chat pattern)
- Name + timestamp header row
- Three body lines with percentage widths for fluid panel reflow

**Props:**
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | — | Additional classes on the row wrapper |
| `showAvatar` | `boolean` | `true` | Show the circular avatar placeholder |
| `isUser` | `boolean` | `false` | Suppresses avatar for user-side messages |

### `ChatMessageListSkeleton`
Renders a scrollable column of `ChatMessageSkeleton` rows, matching `ChatMessageList`'s layout including `fullWidth` mode.

**Props:**
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `messageCount` | `number` | `6` | Number of skeleton rows to render |
| `showAvatars` | `boolean` | `true` | Passed through to each row |
| `fullWidth` | `boolean` | `false` | Drops `max-w-ods-content-narrow` to fill panel |
| `contentClassName` | `string` | — | Extra classes on the inner content column |

## Usage Example

```typescript
// Full list skeleton while messages load
<ChatMessageListSkeleton
  messageCount={8}
  fullWidth={true}
  showAvatars={true}
/>

// Single row (e.g. streaming response placeholder)
<ChatMessageSkeleton
  isUser={false}
  showAvatar={true}
  className="opacity-60"
/>
```

> Messages alternate user/assistant at every third index (`index % 3 === 0`), producing a realistic mixed-sender skeleton. The list is bottom-anchored via a `flex-1` spacer to mirror how real chat lists grow upward.
