A chat-safe info card component (`MingoInfoCard`) that renders entity metadata — avatar/icon, title, description, status pill, and overflow menu — using only phrasing-content elements (`<span>`, `<a>`, `<button>`, `<img>`) so it can be embedded inline inside `<p>` tags in chat message streams.

## Key Components

- **`MingoInfoCard`** — Root component. Supports three content-region modes: plain (non-interactive), anchor (`<a>` via `anchorProps`), or button (`onClick`). The overflow menu renders as an independent sibling, never nested inside the anchor/button.
- **`CardMedia`** — Internal sub-component rendering a 40px leading visual: an avatar image with initials fallback on error, or a boxed icon glyph.
- **`MingoInfoCardProps`** — Full prop interface including `title`, `description`, `imageSrc`, `icon`, `status`, `menuGroups`, and interaction props.
- **`MingoInfoCardStatus`** — Status pill configuration (`label` + `variant`) reusing the shared `Tag` colour palette.
- **`MingoInfoCardAnchorProps`** — Anchor bundle (`href`, `target`, `rel`, `onClick`) for navigation-style cards.

## Usage Example

```typescript
import { MingoInfoCard } from './mingo-info-card'
import { ServerIcon } from '../icons-v2-generated'

// Anchor card with avatar, status pill, and overflow menu
<MingoInfoCard
  title="web-server-01"
  description="Ubuntu 22.04 LTS"
  imageSrc="/assets/servers/ws01.png"
  imageAlt="Web Server 01"
  status={{ label: 'Online', variant: 'success' }}
  anchorProps={{ href: '/devices/ws01', target: '_blank' }}
  menuGroups={[
    {
      items: [
        { label: 'Connect', onClick: () => connect('ws01') },
        { label: 'Restart', onClick: () => restart('ws01') },
      ],
    },
  ]}
/>

// Icon-only card with click handler
<MingoInfoCard
  title="DNS Service"
  description="Port 53"
  icon={<ServerIcon size={24} />}
  status={{ label: 'Degraded', variant: 'warning' }}
  onClick={() => openServiceDetails('dns')}
/>
```
