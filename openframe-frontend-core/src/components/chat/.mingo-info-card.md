A reusable info card component for the Mingo AI interface that displays an entity (user, device, service, etc.) with a leading avatar/icon, title, description, optional status pill, and overflow actions menu.

## Key Components

**`MingoInfoCard`** — Primary export. Renders a horizontally-laid-out card with:
- Leading 40px media (image with initials fallback, or boxed icon)
- Title + optional description text column
- Optional coloured status `Tag` pill
- Optional `ActionsMenuDropdown` triggered by a `⋯` button

**`CardMedia`** *(internal)* — Handles the leading avatar slot. Shows an image with an initials fallback on load error, or a boxed glyph icon.

**`MingoInfoCardStatus`** — Interface for the status pill (`label` + `variant`).

**`MingoInfoCardAnchorProps`** — Interface for turning the content region into an `<a>` tag (`href`, `target`, `rel`, `onClick`).

> **Note:** All elements use `<span>`/`<a>`/`<button>` (no `<div>`) so the card renders safely inside chat `<p>` elements via the `[card://]` inline replacement pattern.

## Usage Example

```typescript
import { MingoInfoCard } from './mingo-info-card'
import { ServerIcon } from '../icons-v2-generated'

// Basic card with icon and status
<MingoInfoCard
  title="web-server-01"
  description="192.168.1.10 — Ubuntu 22.04"
  icon={<ServerIcon size={24} />}
  status={{ label: 'Online', variant: 'success' }}
  menuGroups={[
    {
      items: [
        { label: 'View details', onClick: () => navigate('/device/1') },
        { label: 'Restart', onClick: handleRestart },
      ],
    },
  ]}
/>

// Navigable card with avatar image
<MingoInfoCard
  title="Jane Smith"
  description="jane@company.com"
  imageSrc="/avatars/jane.jpg"
  imageAlt="Jane Smith"
  status={{ label: 'Away', variant: 'warning' }}
  anchorProps={{ href: '/users/jane', target: '_blank' }}
/>
```
