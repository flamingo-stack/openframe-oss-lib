Reusable Slack-style message row component used as the single source of truth for chat message rendering across both the OpenMSP community feed and ticket conversation surfaces, ensuring pixel-identical layout on all surfaces.

## Key Components

### `ChatMessageRow`
The primary message row component rendering avatar, display name, relative timestamp, body text, and an optional footer slot.

**Props (`ChatMessageRowProps`):**

| Prop | Type | Description |
|------|------|-------------|
| `displayName` | `string` | Bold name shown in the message header |
| `avatarUrl` | `string \| null` | Avatar image URL; proxied and falls back to initials |
| `timeLabel` | `string \| null` | Pre-formatted relative time (e.g. `"2h ago"`) |
| `body` | `string` | Message body text; hidden if empty with no footer |
| `footer` | `ReactNode` | Per-surface slot — reply badge or attachment list |

### `ChatMessageRowSkeleton`
Loading placeholder with structural 1:1 parity to `ChatMessageRow`, preventing layout reflow on the loading-to-loaded transition. Uses `bg-ods-border` with `animate-pulse` for shimmer bars.

## Usage Example

```typescript
// Slack community feed
<ChatMessageRow
  displayName="Jane Doe"
  avatarUrl="https://cdn.example.com/avatars/jane.png"
  timeLabel="2h ago"
  body="Has anyone seen this BSOD on Windows 11 24H2?"
  footer={<ReplyBadge count={3} />}
/>

// Ticket conversation feed
<ChatMessageRow
  displayName="John Smith"
  avatarUrl={null}            // renders initials "JS"
  timeLabel={formatRelativeTime(message.createdAt)}
  body={message.text}
  footer={<TicketAttachmentsList attachments={message.attachments} />}
/>

// Loading state
<ChatMessageRowSkeleton />
```

> **Design contract:** All color values use ODS theme tokens (`text-ods-text-primary`, `text-ods-text-secondary`). Typography utilities (`text-[Npx]`, `font-body`, `leading-*`) are locked to match Slack's layout exactly and must not be altered independently per surface.
