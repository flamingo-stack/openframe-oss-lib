Pure presentation component for rendering blog post cards in two densities: a full vertical card (`default`) and a compact horizontal card (`sm`) for inline chat rendering.

## Key Components

### `BlogCard`
Main component rendering a blog post card. Handles image fallback chain (`featured_image` → `placeholderUrl` → background), link targeting via `useEntityCardLink`, and placeholder generation via `useEntityCardPlaceholder`.

### `BlogCardSkeleton`
Loading placeholder matching the card's layout for both `default` and `sm` sizes, using `animate-pulse` for visual feedback.

## Props (`BlogCardProps`)

| Prop | Type | Description |
|------|------|-------------|
| `post` | `BlogPostSummary` | Blog post data |
| `href` | `string` | Resolved detail URL from caller |
| `target` | `'_blank'` | Opens in new tab (set by chat dispatch) |
| `targetPlatform` | `string \| null` | Platform owning `href` |
| `placeholderUrl` | `string \| null` | Fallback image URL |
| `size` | `'default' \| 'sm'` | Card density |
| `hasEmbeddedVideo` | `boolean` | Shows "Video" badge in compact mode |
| `priority` | `boolean` | Next.js image priority hint |

## Usage Example

```typescript
import { BlogCard, BlogCardSkeleton } from './blog-card'

// Default full card
<BlogCard
  post={post}
  href={buildContentURL(post.slug)}
  placeholderUrl={useOgPlaceholder(post.title)}
  size="default"
  priority={isAboveFold}
/>

// Compact inline card (e.g. inside chat)
<BlogCard
  post={post}
  href={detailUrl}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
  hasEmbeddedVideo={post.has_video}
/>

// Skeleton while loading
<BlogCardSkeleton size="default" />
```

> **Note:** The card contains no click logic — callers are responsible for wrapping with their own anchor or passing a resolved `href`. The component is a pure presentation layer.
