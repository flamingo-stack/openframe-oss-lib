Pure presentation component for rendering blog post cards in two density modes: full vertical (`default`) and compact horizontal (`sm`) for chat-inline contexts.

## Key Components

### `BlogCard`
Main card component. Renders a fully clickable card with cover image, title, summary, and author/date footer. Delegates link behavior to `useEntityCardLink` and image fallback to `useEntityCardPlaceholder`.

### `BlogCardSkeleton`
Loading placeholder that mirrors the layout of each size variant using animated pulse styles.

### `BlogCardProps`
| Prop | Type | Description |
|------|------|-------------|
| `post` | `BlogPostSummary` | Post data to render |
| `href` | `string` | Resolved detail URL (caller-provided) |
| `size` | `'default' \| 'sm'` | Card density mode |
| `placeholderUrl` | `string \| null` | Fallback image (e.g. from `useOgPlaceholder`) |
| `hasEmbeddedVideo` | `boolean` | Shows a "Video" badge in `sm` mode |
| `priority` | `boolean` | Passes LCP priority hint to the cover image |
| `target` / `rel` | string | Tab behavior, set by chat dispatch |

## Usage Example

```typescript
import { BlogCard, BlogCardSkeleton } from './blog-card'

// Full card (default density)
<BlogCard
  post={post}
  href={buildContentURL(post.slug)}
  placeholderUrl={useOgPlaceholder(post.title)}
  size="default"
  priority={isAboveFold}
/>

// Compact card for chat inline rendering
<BlogCard
  post={post}
  href={detailUrl}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
  hasEmbeddedVideo={true}
/>

// Loading state
<BlogCardSkeleton size="default" />
```

> **No click logic lives inside this component.** Callers are responsible for wrapping with an anchor or passing a resolved `href`. The image fallback chain is: `post.featured_image` → `placeholderUrl` → `bg-ods-bg` background slot.
