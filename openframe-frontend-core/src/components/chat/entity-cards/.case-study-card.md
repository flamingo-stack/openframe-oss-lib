Pure presentation component for rendering MSP case study cards in two density modes (`default` for vertical detail layouts and `sm` for compact horizontal chat-inline display).

## Key Components

- **`CaseStudyCard`** — Main card component. Renders a full vertical card (`default`) or compact horizontal card (`sm`). Delegates link behavior to `useEntityCardLink` and image placeholder generation to `useEntityCardPlaceholder`. Contains no internal click logic — callers provide `href`.
- **`CaseStudyCardSkeleton`** — Matching loading skeleton for both size variants, preserving layout dimensions (OG 1200×630 aspect ratio for `default`, compact slot for `sm`).
- **`CaseStudyCardProps`** — Interface defining `study`, `href`, optional `target`/`rel`, `placeholderUrl`, `size`, and `className`.

**Image fallback chain:** `study.featured_image` → `placeholderUrl` (caller-provided OG placeholder) → `bg-ods-bg` background.

## Usage Example

```typescript
import { CaseStudyCard, CaseStudyCardSkeleton } from './case-study-card'
import { useOgPlaceholder } from '../hooks/use-og-placeholder'

// Default full card
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  placeholderUrl={useOgPlaceholder(caseStudy.title)}
  size="default"
/>

// Compact variant for chat inline rendering
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>

// Loading state
<CaseStudyCardSkeleton size="default" />
```
