Pure presentation component for displaying MSP case studies in two density modes — `default` (vertical detail card) and `sm` (compact horizontal for chat-inline contexts).

## Key Components

**`CaseStudyCard`** — Main card component. Renders a fully linked card with cover image, title, and author/MSP attribution. Delegates navigation logic to `useEntityCardLink` and image fallback resolution to `useEntityCardPlaceholder`. No click logic is owned internally; callers supply `href`.

**`CaseStudyCardSkeleton`** — Loading placeholder matching the real card's layout dimensions exactly (including the `1200/630` aspect ratio for the image slot), available in both `default` and `sm` sizes.

**`hideOnError`** — Internal handler that hides broken `<img>` elements by setting `display: none`.

## Props (`CaseStudyCardProps`)

| Prop | Type | Description |
|------|------|-------------|
| `study` | `CaseStudy` | Case study data object |
| `href` | `string` | Resolved detail URL (caller's responsibility) |
| `target` | `'_blank'` | Optional new-tab flag (chat dispatch sets via `computeIsNewTab`) |
| `placeholderUrl` | `string \| null` | OG placeholder when `featured_image` is absent |
| `size` | `'default' \| 'sm'` | Card density mode |

**Image fallback chain:** `study.featured_image` → `placeholderUrl` → `bg-ods-bg` background.

## Usage Example

```typescript
import { CaseStudyCard, CaseStudyCardSkeleton } from './case-study-card'

// Loading state
<CaseStudyCardSkeleton size="default" />

// Rendered card (default density)
<CaseStudyCard
  study={caseStudy}
  href={`/case-studies/${caseStudy.slug}`}
  placeholderUrl={useOgPlaceholder(caseStudy.title)}
  size="default"
/>

// Compact inline variant (e.g. inside chat response)
<CaseStudyCard
  study={caseStudy}
  href={resolvedUrl}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>
```
