Pure presentation component for displaying a `CustomerInterview` entity as a card, supporting two density modes: `default` (full vertical layout) and `sm` (compact horizontal for chat-inline use).

## Key Components

### `CustomerInterviewCard`
Main card component. Renders a clickable `<a>` wrapper with no internal click logic — callers provide the resolved `href`. Supports:
- `default` size: thumbnail (1200×630 aspect), title, summary, and author/MSP avatar row
- `sm` size: compact horizontal layout using shared `COMPACT_CARD_*` class constants
- Video play overlay when `interview.main_video_url` is present
- Fallback placeholder image via `useEntityCardPlaceholder`
- New-tab behavior resolved via `useEntityCardLink`

### `CustomerInterviewCardSkeleton`
Animated loading placeholder matching the layout of each size variant.

## Usage Example

```typescript
import { CustomerInterviewCard, CustomerInterviewCardSkeleton } from './customer-interview-card'

// Default card (e.g. in a grid)
<CustomerInterviewCard
  interview={interview}
  href={`/interviews/${interview.slug}`}
  size="default"
/>

// Compact card for chat inline rendering
<CustomerInterviewCard
  interview={interview}
  href={`/interviews/${interview.slug}`}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
  placeholderUrl={ogImageUrl}
/>

// Loading state
<CustomerInterviewCardSkeleton size="sm" />
```

## Props (`CustomerInterviewCardProps`)

| Prop | Type | Description |
|------|------|-------------|
| `interview` | `CustomerInterview` | Entity data to render |
| `href` | `string` | Resolved detail URL (required) |
| `target` | `'_blank'` | Opens in new tab when set |
| `rel` | `'noopener noreferrer'` | Link rel attribute |
| `targetPlatform` | `string \| null` | Passed to `useEntityCardLink` for tab resolution |
| `placeholderUrl` | `string \| null` | OG image fallback when `featured_image` is absent |
| `size` | `'default' \| 'sm'` | Card density; defaults to `'default'` |
| `className` | `string` | Additional CSS classes |
