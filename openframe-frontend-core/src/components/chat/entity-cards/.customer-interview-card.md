Pure presentation component for rendering customer interview cards in two density modes (`default` and `sm`), with no embedded click logic — callers supply the resolved `href`.

## Key Components

### `CustomerInterviewCard`
Main card component. Renders a full-size (`default`) or compact horizontal (`sm`) card for a `CustomerInterview` entity. Delegates link behavior to `useEntityCardLink` and thumbnail fallback to `useEntityCardPlaceholder`.

### `CustomerInterviewCardSkeleton`
Loading placeholder matching the card layout for both size variants. Uses `animate-pulse` with `bg-ods-bg` shimmer slots.

### `CustomerInterviewCardProps`
```typescript
interface CustomerInterviewCardProps {
  interview: CustomerInterview   // Data model
  href: string                   // Resolved detail URL (caller-supplied)
  target?: '_blank'              // New-tab behavior
  rel?: 'noopener noreferrer'
  targetPlatform?: string | null
  placeholderUrl?: string | null // OG fallback when featured_image is absent
  size?: 'default' | 'sm'       // 'sm' = compact horizontal (chat-inline)
  className?: string
}
```

## Usage Example

```typescript
// Default (full card)
<CustomerInterviewCard
  interview={interviewData}
  href="/interviews/acme-msp"
  size="default"
  placeholderUrl="https://cdn.example.com/og-placeholder.jpg"
/>

// Compact — chat-inline
<CustomerInterviewCard
  interview={interviewData}
  href="https://external.com/interview/123"
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>

// Skeleton while loading
<CustomerInterviewCardSkeleton size="default" />
```

## Behavior Notes

- **Images:** `onError` hides broken thumbnails silently via inline style
- **Video indicator:** Overlays a `Video` icon when `main_video_url` is present; full-size variant renders an amber play button
- **Author row:** Shows user avatar with MSP icon badge; falls back to initials if no `avatar_url`
- **`sm` layout:** Uses shared `COMPACT_CARD_*` class constants from `compact-card-classes` for consistency across entity card types
