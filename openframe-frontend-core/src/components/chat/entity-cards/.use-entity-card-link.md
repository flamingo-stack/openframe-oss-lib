Resolves `target` and `rel` anchor attributes for entity cards by delegating the same-tab vs. new-tab decision to `ChatRuntime.navigation.decideNewTab`, eliminating the need for hub-side wrapper components.

## Key Components

### `useEntityCardLink(args): EntityCardLinkProps`
Main hook that computes safe anchor attributes for any entity card link.

**Priority order:**
1. **Explicit props win** — if `target` or `rel` are passed, runtime logic is skipped. A bare `target="_blank"` without `rel` is auto-paired with `"noopener noreferrer"` to prevent tabnabbing.
2. **Runtime decision** — delegates to `computeIsNewTab(runtime, href, targetPlatform)` for consistent behavior between rendered anchor attributes and the click handler (`executeNavigation`).
3. **No runtime fallback** — returns `{ target: undefined, rel: undefined }` (same-tab), matching the embed-shim contract.

### `UseEntityCardLinkArgs`
| Prop | Type | Description |
|---|---|---|
| `href` | `string` | Destination URL |
| `targetPlatform` | `string \| null` | Platform context hint for cross-platform topology |
| `target` | `'_blank'` | Explicit override; skips runtime |
| `rel` | `'noopener noreferrer'` | Explicit override; skips runtime |

### `EntityCardLinkProps`
Return shape: `{ target: '_blank' | undefined, rel: 'noopener noreferrer' | undefined }`.

## Usage Example

```typescript
import { useEntityCardLink } from './use-entity-card-link'

function BlogCard({ href, platform }: Props) {
  const { target, rel } = useEntityCardLink({
    href,
    targetPlatform: platform,
  })

  return <a href={href} target={target} rel={rel}>{children}</a>
}

// Explicit override (chat dispatcher pre-resolved path)
const attrs = useEntityCardLink({
  href,
  target: '_blank', // rel auto-paired as 'noopener noreferrer'
})
```
