Resolves `target` and `rel` link attributes for entity cards by delegating the same-tab vs. new-tab decision to the `ChatRuntime` navigation callback, eliminating the need for hub-side wrapper components.

## Key Components

### `useEntityCardLink(args)`
Main hook that returns computed anchor attributes for entity card links.

- **Explicit override priority** — if `target` or `rel` props are passed, they win immediately; `rel` is auto-paired with `noopener noreferrer` when `target="_blank"` is provided alone to prevent tabnabbing.
- **Runtime delegation** — when no overrides exist, calls `computeIsNewTab(runtime, href, targetPlatform)` for a unified decision consistent with the click handler (`executeNavigation`).
- **No-runtime fallback** — returns `{ target: undefined, rel: undefined }` (same-tab), matching the embed-shim documented behavior.

### `UseEntityCardLinkArgs`
Input interface with `href`, optional `targetPlatform`, and optional explicit `target`/`rel` overrides.

### `EntityCardLinkProps`
Output interface — `target: '_blank' | undefined` and `rel: 'noopener noreferrer' | undefined`.

## Usage Example

```typescript
import { useEntityCardLink } from './use-entity-card-link'

function BlogCard({ href, targetPlatform }) {
  const { target, rel } = useEntityCardLink({ href, targetPlatform })

  return (
    <a href={href} target={target} rel={rel}>
      Read more
    </a>
  )
}

// With explicit override (e.g. from chat dispatcher):
const { target, rel } = useEntityCardLink({
  href: 'https://example.com',
  target: '_blank',
  // rel auto-resolves to 'noopener noreferrer'
})
```
