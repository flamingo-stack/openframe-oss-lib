Brief description of the file's purpose: Renders onboarding guide entities in three visual variants (`catalog`, `default`, `sm`) as interactive link cards with runtime-derived tab behavior and placeholder image resolution.

## Key Components

### `OnboardingGuideCard`
Main presentational component. Accepts an `OnboardingGuide` entity and renders one of three layouts:
- **`catalog`** — Rich hero card with cover image, video duration badge, title, summary, and author/section metadata grid. Used on public catalog pages.
- **`default`** — Horizontal step-numbered card for "More in section" rails.
- **`sm`** — Compact horizontal card optimized for inline chat rendering.

Link `target`/`rel` are resolved via `useEntityCardLink` (hub-wired `ChatRuntime.navigation.decideNewTab`); placeholder images via `useEntityCardPlaceholder` (`ChatRuntime.resolvePlaceholderUrl`). Explicit `target`, `rel`, and `placeholderUrl` props always take precedence.

### `OnboardingGuideCardSkeleton`
Loading skeleton matching each of the three size variants. Used for SSR/streaming placeholders.

### `OnboardingGuideCardProps`
TypeScript interface defining all accepted props.

## Usage Example

```typescript
import { OnboardingGuideCard, OnboardingGuideCardSkeleton } from './onboarding-guide-card'

// Catalog variant (public listing page)
<OnboardingGuideCard
  guide={guideEntity}
  href="/guides/getting-started"
  size="catalog"
/>

// Compact inline variant (chat context, pre-resolved link attrs)
<OnboardingGuideCard
  guide={guideEntity}
  href="/guides/getting-started"
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
  placeholderUrl="https://cdn.example.com/placeholder.jpg"
/>

// Loading state
<OnboardingGuideCardSkeleton size="catalog" />
```
