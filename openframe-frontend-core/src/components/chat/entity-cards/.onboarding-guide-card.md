Brief description: Presentational card component for displaying onboarding guide entries in three layout variants (`catalog`, `default`, `sm`), with runtime-derived link attributes and placeholder image resolution via hub-wired context hooks.

## Key Components

### `OnboardingGuideCard`
Main card component rendering one of three variants based on the `size` prop:
- **`catalog`** — Rich detail card with hero image, author grid, step/section metadata, and video duration badge
- **`default`** — Horizontal step-numbered card for "More in section" rails
- **`sm`** — Compact inline card for chat-embedded rendering

Uses `useEntityCardLink` to derive `target`/`rel` from `ChatRuntime.navigation.decideNewTab` and `useEntityCardPlaceholder` for OG placeholder resolution. Explicit prop values always take precedence over runtime-derived ones.

### `OnboardingGuideCardSkeleton`
Loading skeleton component matching each of the three size variants with pulse animation placeholders.

### `OnboardingGuideCardProps`
TypeScript interface for all card props, including the `OnboardingGuide` entity, resolved `href`, optional link behavior overrides, and display variant.

## Usage Example

```typescript
// Catalog variant (public guide listing page)
<OnboardingGuideCard
  guide={guide}
  href={`/guides/${guide.slug}`}
  size="catalog"
  targetPlatform="hub"
/>

// Compact variant (chat-inline rendering with pre-resolved props)
<OnboardingGuideCard
  guide={guide}
  href={`/guides/${guide.slug}`}
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
  placeholderUrl="https://cdn.example.com/placeholder.jpg"
/>

// Loading state
<OnboardingGuideCardSkeleton size="catalog" />
```

> **Note:** When no runtime is mounted (e.g. tests or SSR), the card defaults to same-tab navigation and omits placeholder images. Pass explicit `target`, `rel`, and `placeholderUrl` props to override.
