Renders the correct inline entity card component for a given `ChatRef` type, dispatching to specialized card components based on document/entity type within the chat interface.

## Key Components

### Types & Interfaces

- **`ChatCardDispatchExtras`** — Host-supplied builders for hub-side derived props (program configs, product release card props, OG placeholder URLs) that cannot be computed in lib alone
- **`ChatCardRenderOptions`** — Per-card render options including `baseRoute`, `isNewTab`, `discuss`, and `extras`
- **`CardDiscussAction`** *(partial)* — Host-provided "Ask Mingo"/"Display" affordance for the card's context menu

### Internal Components

- **`GitHubChatCard`** — Renders a `MingoInfoCard` for GitHub activity refs (`pull_request`, `pr_review`, `commit`), parsing title/review state and formatting the activity ID + date description
- **`githubKindIcon(kind)`** — Returns the leading icon glyph for a given `GitHubActivityKind`
- **`reviewStateToVariant(state)`** — Maps a `PrReviewState` to a `MingoInfoCardStatus` color variant (`success`, `error`, `warning`, `grey`)

### Public Entry Points

- **`renderChatInlineEntityCard(ref, opts)`** — Top-level dispatch; hoists video refs to block-card layout, otherwise renders the compact inline card
- **`ChatCardLoader`** — Fetch + skeleton + render component for fetch-mode entries; computes `isNewTab` once via `computeIsNewTab` and threads it through render options

## Usage Example

```typescript
// Inside the chat shell's message renderer
import { renderChatInlineEntityCard } from './dispatch'

const card = renderChatInlineEntityCard(chatRef, {
  isNewTab: false,
  extras: {
    programConfigs: { podcast: myPodcastConfig },
    buildProductReleaseCardProps: (release) => ({ ... }),
    buildOgPlaceholderUrl: (title) => `/og?title=${encodeURIComponent(title)}`,
  },
})
```

> **Design note:** All hub-specific utilities (`useNavLink`, `buildContentURL`, `currentPlatform`) are intentionally excluded. URLs arrive pre-composed via `ref.url` from the server RAG mapper, and click routing flows through `handleChatNavClick` / `NavLinkAnchorViaRuntime`.
