Unified presentation card for GitHub activity items (commits, pull requests, and PR reviews), supporting both row and compact display variants. Pure presentational component with no internal data fetching or navigation logic.

## Key Components

### `GitHubActivityCard`
Main card component accepting a `GitHubActivityItem` and rendering in two layouts:
- **`variant="row"`** — horizontal list row for employee detail views
- **`variant="compact"`** — inline card for chat `[card://github_commit:<sha>]` markers

Navigation is handled externally via the `anchorProps` prop bundle (typically composed by a `useNavLink` hook in the consumer).

### `GitHubActivityCardSkeleton`
Loading placeholder matching both `row` and `compact` variants with animated pulse.

### Exported Helpers

| Export | Description |
|--------|-------------|
| `kindLabel(kind)` | Maps `GitHubActivityKind` → display string (`"Commit"`, `"PR"`, `"Review"`) |
| `formatActivityId(id, kind)` | Formats SHA to 7-char short hash or PR number to `#123` |
| `parseGithubTitle(title, kind)` | Strips prefix tags (e.g. `[Review:APPROVED]`), extracts `PrReviewState` |
| `reviewStateLabel(state)` | Maps `PrReviewState` → short label for shared use with `MingoInfoCard` |

## Usage Example

```typescript
import { GitHubActivityCard } from './github-activity-card'
import type { GitHubActivityItem } from '../types/entities/github-activity'

const item: GitHubActivityItem = {
  kind: 'pull_request',
  id: '42',
  title: 'Fix memory leak in worker pool',
  url: 'https://github.com/acme/core/pull/42',
  dateUpdated: '2024-06-01T12:00:00Z',
}

// Compact (chat inline card) — external link fallback
<GitHubActivityCard item={item} variant="compact" />

// Row with caller-provided navigation
<GitHubActivityCard
  item={item}
  variant="row"
  anchorProps={{
    href: '/internal/activity/42',
    onClick: (e) => handleNav(e),
  }}
/>
```
