Unified presentation card for GitHub activity items (commits, pull requests, and PR reviews), supporting both `row` and `compact` display variants with no internal data fetching.

## Key Components

**`GitHubActivityCard`** — Main card component rendering GitHub activity in two variants:
- `row` — Horizontal list row for activity feeds (employee detail views)
- `compact` — Inline card for chat markers like `[card://github_commit:<sha>]`

**`GitHubActivityCardSkeleton`** — Loading placeholder matching both `row` and `compact` layouts.

**Exported Utilities**

| Function | Purpose |
|---|---|
| `kindLabel(kind)` | Maps `GitHubActivityKind` → display string (`"Commit"`, `"PR"`, `"Review"`) |
| `formatActivityId(id, kind)` | Shortens SHAs to 7 chars; prefixes PR/review IDs with `#` |
| `parseGithubTitle(title, kind)` | Extracts `[review:STATE]`/`[PR #N]` prefixes; returns clean display text + `PrReviewState` |
| `reviewStateLabel(state)` | Maps `PrReviewState` → human label (`"Approved"`, `"Changes"`, etc.) |

**`ReviewStateBadge`** (internal) — Color-coded pill for PR review states (`APPROVED`, `CHANGES_REQUESTED`, `COMMENTED`, `DISMISSED`, `PENDING`).

## Usage Example

```typescript
import { GitHubActivityCard } from './github-activity-card'

// Compact variant (chat inline card)
<GitHubActivityCard
  item={{
    kind: 'pull_request',
    id: '42',
    title: 'Fix memory leak in worker pool',
    url: 'https://github.com/org/repo/pull/42',
    repo: 'org/repo',
    dateUpdated: '2025-01-15T10:00:00Z',
  }}
  variant="compact"
  anchorProps={{
    href: '/internal/github/pr/42',
    onClick: (e) => handleNav(e),
  }}
/>

// Row variant (activity list)
<GitHubActivityCard
  item={activityItem}
  variant="row"
/>

// Skeleton while loading
<GitHubActivityCardSkeleton variant="compact" />
```
