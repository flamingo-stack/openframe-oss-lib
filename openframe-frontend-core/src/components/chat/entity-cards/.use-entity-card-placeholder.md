Resolves Open Graph placeholder URLs for entity cards, consolidating placeholder logic inside the card rather than requiring callers to pre-compute it.

## Key Components

**`UseEntityCardPlaceholderArgs`** — Input interface:
- `title` — Entity title used as the placeholder label
- `placeholderUrl` — Explicit override; bypasses runtime resolution entirely
- `siteName` — Optional site name displayed beneath the title
- `aspect` — Output aspect ratio: `'wide'` (default, catalog cards) or `'square'` (chat-inline cards)

**`useEntityCardPlaceholder(args)`** — Main hook. Reads `resolvePlaceholderUrl` from `ChatRuntime` context and delegates to `useOgPlaceholder`. Resolution priority:
1. Explicit `placeholderUrl` prop (including `null`) — always wins
2. Runtime-derived value via `ChatRuntime.resolvePlaceholderUrl`
3. `null` fallback when no runtime callback is wired (card renders empty-state)

## Usage Example

```typescript
import { useEntityCardPlaceholder } from './use-entity-card-placeholder'

// Runtime-resolved (typical card usage)
function CatalogCard({ title, siteName }: Props) {
  const placeholder = useEntityCardPlaceholder({ title, siteName, aspect: 'wide' })
  return <img src={placeholder ?? '/fallback.png'} />
}

// Explicit override (pre-resolved in chat dispatch or tests)
function CatalogCardWithPreset({ title, url }: Props) {
  const placeholder = useEntityCardPlaceholder({
    title,
    placeholderUrl: url, // runtime resolver is skipped
  })
  return <img src={placeholder ?? '/fallback.png'} />
}
```

> **Backwards compatibility:** passing an explicit `placeholderUrl` (or `null`) always takes precedence, so existing callers that pre-resolve the URL are unaffected by this hook.
