A chat UI row component that renders a single support ticket as a clickable button, displaying the ticket title, metadata subtitle, status badge, and an optional linked-work chip.

## Key Components

### `ChatTicketItemData`
Data interface for ticket row content:
- `id`, `title`, `ticketNumber` — core identifiers
- `status` — accepts any known format (`ACTIVE`, `active`, `ACTION_REQUIRED`, etc.)
- `statusLabel?` — overrides the badge display text while preserving variant/icon from canonical status
- `statusKind?` / `statusColor?` — drives custom-status lifecycle styling
- `linkedTaskLabel?` — when set, renders a wrench icon chip showing linked delivery task progress
- `category?`, `timeAgo?` — appended to the subtitle dot-separated string

### `ChatTicketItemProps`
Extends `ButtonHTMLAttributes<HTMLButtonElement>` (minus `onClick`), adding:
- `ticket: ChatTicketItemData`
- `onClick?: (ticketId: string) => void` — receives the ticket's `id` string

### `ChatTicketItem`
A `forwardRef` button component rendering:
- **Title** — truncated, dimmed when `status` resolves to `RESOLVED`
- **Subtitle** — `ticketNumber • category • timeAgo` joined with bullet separator
- **Linked-work chip** — `Tag` with `WrenchIcon`, hidden on `xs` screens (`sm:inline-flex`)
- **Status tag** — `TicketStatusTag` resolved via `resolveStatusTagProps`
- **Chevron button** — right-side navigation affordance

## Usage Example

```typescript
import { ChatTicketItem, ChatTicketItemData } from './chat-ticket-item'

const ticket: ChatTicketItemData = {
  id: 'tkt_01',
  title: 'VPN not connecting after update',
  ticketNumber: '#1042',
  status: 'ACTION_REQUIRED',
  statusLabel: 'Waiting on contact',
  statusKind: 'ACTIVE',
  category: 'Network',
  timeAgo: '2h ago',
  linkedTaskLabel: 'Waiting on version release',
}

<ChatTicketItem
  ticket={ticket}
  onClick={(id) => router.push(`/tickets/${id}`)}
/>
```
