A clickable ticket row component for chat interfaces, displaying ticket title, metadata subtitle, status badge, optional linked-work chip, and a chevron affordance.

## Key Components

### Interfaces

- **`ChatTicketItemData`** — Data model for a ticket row:
  - `id`, `title`, `ticketNumber` — core identifiers
  - `status` — accepts any known format (`ACTIVE`, `active`, `ACTION_REQUIRED`, etc.)
  - `statusLabel?` — overrides the badge text while keeping variant/icon from canonical status
  - `statusKind?` / `statusColor?` — drives custom-status (lifecycle) styling
  - `category?`, `timeAgo?` — optional subtitle segments
  - `linkedTaskLabel?` — when set, renders a wrench-icon "Linked work" chip

- **`ChatTicketItemProps`** — Extends `ButtonHTMLAttributes` (excluding `onClick`), adding:
  - `ticket: ChatTicketItemData`
  - `onClick?: (ticketId: string) => void`

### Component

- **`ChatTicketItem`** — `forwardRef` button component rendering a full-width `h-20` row with:
  - Left: title (dimmed when resolved) + dot-separated subtitle (`ticketNumber • category • timeAgo`)
  - Center-right: optional `Tag` chip with `WrenchIcon` (hidden on small screens)
  - Right: `TicketStatusTag` + chevron icon button

## Usage Example

```typescript
import { ChatTicketItem, ChatTicketItemData } from './chat-ticket-item'

const ticket: ChatTicketItemData = {
  id: 'TKT-001',
  title: 'VPN not connecting after Windows update',
  ticketNumber: '#1042',
  status: 'ACTION_REQUIRED',
  category: 'Network',
  timeAgo: '2h ago',
  linkedTaskLabel: 'Waiting on version release',
}

<ChatTicketItem
  ticket={ticket}
  onClick={(id) => router.push(`/tickets/${id}`)}
/>
```
