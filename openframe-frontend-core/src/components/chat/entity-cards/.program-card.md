Pure presentation card component for displaying program content (podcasts, webinars, events) in two density modes: `default` (vertical detail) and `sm` (compact horizontal for chat-inline use).

## Key Components

### Exports

- **`ProgramCard<T>`** — Generic card component accepting any `BaseProgramItem`. Renders full-detail or compact layout based on `size` prop. Contains no click logic; callers provide `href`.
- **`ProgramCardSkeleton`** — Animated loading placeholder matching both `default` and `sm` card layouts.

### Internal Helpers

- **`getHosts()`** — Normalizes `ProgramHost[]` (or JSON string fallback) into `{ name, avatar }` pairs.
- **`MediaGallery`** — Horizontal scrollable thumbnail strip with lightbox support via `ImageGalleryModal`.

### Props (`ProgramCardProps<T>`)

| Prop | Type | Description |
|------|------|-------------|
| `config` | `ProgramConfig<T>` | Program type (`podcast`, `webinar`, `event`) and labels |
| `item` | `T extends BaseProgramItem` | The program item data |
| `href` | `string` | Resolved detail URL (caller-provided) |
| `size` | `'default' \| 'sm'` | Card density; defaults to `'default'` |
| `target` | `'_blank'` | Opens in new tab (set by chat dispatch) |
| `media` | `ProgramMedia[]` | Optional image gallery items |
| `renderMeta` | `(item: T) => ReactNode` | Override default meta rendering |

## Usage Example

```typescript
import { ProgramCard, ProgramCardSkeleton } from './program-card'

// Loading state
<ProgramCardSkeleton size="sm" />

// Compact inline (chat)
<ProgramCard
  config={{ type: 'podcast', labels: { singular: 'Episode' } }}
  item={podcastEpisode}
  href="/podcasts/episode-42"
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>

// Full detail card
<ProgramCard
  config={{ type: 'webinar', labels: { singular: 'Webinar' } }}
  item={webinarItem}
  media={mediaAssets}
  href="/webinars/intro-to-openframe"
  size="default"
/>
```
