Generic presentation card for podcasts, webinars, and events. Supports two render densities (`default` for vertical detail, `sm` for compact horizontal/chat-inline) and delegates all navigation logic to the caller via `href`.

## Key Components

- **`ProgramCard<T>`** — Main generic card component; renders either full-detail or compact layout based on `size` prop
- **`ProgramCardSkeleton`** — Animated loading placeholder matching both `default` and `sm` densities
- **`MediaGallery`** — Internal scrollable thumbnail strip with lightbox modal for image media
- **`getHosts()`** — Utility that normalizes `ProgramHost[]` (or JSON string fallback) into a consistent `{ name, avatar }` array

## Usage Example

```typescript
import { ProgramCard, ProgramCardSkeleton } from './program-card'
import type { PodcastItem } from '../types/entities/program-types'

// Full-detail card (default size)
<ProgramCard<PodcastItem>
  config={{ type: 'podcast', labels: { singular: 'Episode' } }}
  item={podcastItem}
  media={mediaItems}
  href="/podcasts/episode-42"
  size="default"
/>

// Compact inline card for chat dispatch
<ProgramCard<PodcastItem>
  config={{ type: 'podcast' }}
  item={podcastItem}
  href="https://external-site.com/episode"
  size="sm"
  target="_blank"
  rel="noopener noreferrer"
  targetPlatform="web"
/>

// Loading state
<ProgramCardSkeleton size="sm" />
```

## Notes

- **No click logic** — the card is a pure presenter; callers supply `href`, `target`, and `rel`
- `renderMeta` prop overrides the default type-specific metadata (duration for podcasts, location for events, time range for webinars)
- `placeholderUrl` feeds `useEntityCardPlaceholder` for OG-based cover fallback when `cover_url` is absent
- `wholeCardClickable` flag is accepted but delegated — wrap the component in your own `<a>` for full-card interaction
