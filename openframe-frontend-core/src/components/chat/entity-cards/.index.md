Barrel export module that re-exports all chat-surface entity card components, types, and utilities from a single import path. Cards in this directory are pure presentation components — they accept pre-resolved props and contain no internal routing logic.

## Key Components

**Supporting Primitives**
- `EntityAuthorCard` — author metadata display with `EMPTY_AUTHOR_PLACEHOLDER`
- `BlogImagePlaceholder`, `AdminContentCard` — shared UI primitives

**Flat / Inline Cards**
- `BlockCard`, `ChatTicketItem`, `ChatVideoEntityCard` — lightweight inline chat cards
- `ProductReleaseCard` + `ProductReleaseCardSkeleton` — release cards with size variants

**Card-Prop Builders**
- `defaultBuildProductReleaseCardProps` — small chat default builder
- `buildProductReleaseCardProps` — rich large builder with full metadata grid

**Pure-Presentation Cards** (each ships a `*Skeleton` loading variant)
- `BlogCard`, `CaseStudyCard`, `CustomerInterviewCard`, `InvestorUpdateCard`
- `OnboardingGuideCard`, `RoadmapCard`, `GitHubActivityCard`, `SlackMessageCard`
- `HubspotTicketCard`, `DataRoomDocCard`, `ProgramCard`, `CampaignCardAdmin`
- `GenericEntityCard` — fallback card for unmapped entity types

**Dispatch Layer**
- `renderChatInlineEntityCard` — function that resolves the correct card for a given entity type
- `ChatCardLoader` — component wrapper that drives the dispatch switch via `CHAT_CARD_REGISTRY`

## Usage Example

```typescript
// Single import for any card in the surface
import {
  BlogCard,
  BlogCardSkeleton,
  buildProductReleaseCardProps,
  ChatCardLoader,
  type BlogCardProps,
} from '@/components/entity-cards'

// Pure presentation — consumer provides href, no internal click logic
function MyBlogCard({ post, href }: { post: Post; href: string }) {
  return (
    <NavLinkAnchor href={href}>
      <BlogCard title={post.title} imageUrl={post.coverUrl} />
    </NavLinkAnchor>
  )
}

// Dispatch layer for chat surfaces
function ChatCard({ entity }: { entity: ResolvedEntity }) {
  return <ChatCardLoader entity={entity} />
}
```

> **Design contract:** All cards accept a pre-resolved `href` and write no internal navigation. Routing is the consumer's responsibility — use `<NavLinkAnchor>` (hub) or `<NavLinkAnchorViaRuntime>` (lib embedders).
