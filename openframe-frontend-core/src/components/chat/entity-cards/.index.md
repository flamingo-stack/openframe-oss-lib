Barrel export file for all chat-surface entity cards, consolidating every pure-presentation card component into a single importable path.

## Key Components

### Supporting Primitives
- **`EntityAuthorCard`** — Author metadata display with `EMPTY_AUTHOR_PLACEHOLDER` fallback
- **`BlogImagePlaceholder`** — Placeholder image for blog content
- **`AdminContentCard`** — Admin-facing content card

### Flat Cards
- **`BlockCard`** — Generic block-level card
- **`ChatTicketItem`** — Ticket entry for chat surfaces
- **`ChatVideoEntityCard`** — Video entity card for chat
- **`ProductReleaseCard`** / **`ProductReleaseCardSkeleton`** — Release notes card with size variants

### Dispatch
- **`renderChatInlineEntityCard`** — Switch-based renderer over all registered card types
- **`ChatCardLoader`** — Async card loader with registry lookup via `CHAT_CARD_REGISTRY`

### Card-Prop Builders
- **`defaultBuildProductReleaseCardProps`** — Small chat default builder
- **`buildProductReleaseCardProps`** — Rich large-format builder with full metadata grid

## Usage Example

```typescript
import {
  BlogCard,
  BlogCardSkeleton,
  renderChatInlineEntityCard,
  type BlogCardProps,
  type ChatCardRenderOptions,
} from '@/components/entity-cards'

// Pure-presentation usage — consumer owns the anchor
function BlogCardWrapper({ post, href }: { post: BlogPost; href: string }) {
  return (
    <NavLinkAnchor href={href}>
      <BlogCard {...post} />
    </NavLinkAnchor>
  )
}

// Dispatch-based rendering for chat surfaces
const card = renderChatInlineEntityCard(entityType, data, options)
```

> **Design contract:** All cards are pure-presentation — they accept a pre-resolved `href` and entity data as props and contain no internal navigation logic. Consumers are responsible for wrapping cards with their own anchor component (`NavLinkAnchor` or `NavLinkAnchorViaRuntime`).
