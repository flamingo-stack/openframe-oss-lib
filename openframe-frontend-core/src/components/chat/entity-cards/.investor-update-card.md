Pure presentation component for displaying investor update entities in two density modes (`default` and `sm`), with no built-in click logic — callers supply the resolved `href`.

## Key Components

### `InvestorUpdateCard`
Main card component accepting an `InvestorUpdate` entity and rendering either a full `AdminContentCard` layout (`default`) or a compact horizontal layout (`sm`). Link behavior (same-tab vs. new-tab) is resolved via `useEntityCardLink`; placeholder images are generated via `useEntityCardPlaceholder`.

### `InvestorUpdateCardSkeleton`
Loading state placeholder matching the visual footprint of each density variant — animated pulse blocks for image and text slots.

### `InvestorUpdateCardProps`
```typescript
interface InvestorUpdateCardProps {
  update: InvestorUpdate
  href: string
  target?: '_blank'
  rel?: 'noopener noreferrer'
  targetPlatform?: string | null
  placeholderUrl?: string | null
  size?: 'default' | 'sm'
  className?: string
}
```

## Usage Example

```typescript
import { InvestorUpdateCard, InvestorUpdateCardSkeleton } from './investor-update-card'

// Default (full card)
<InvestorUpdateCard
  update={investorUpdate}
  href={`/investor-updates/${investorUpdate.id}`}
  size="default"
/>

// Compact horizontal (e.g. feed lists, chat dispatch)
<InvestorUpdateCard
  update={investorUpdate}
  href={`/investor-updates/${investorUpdate.id}`}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>

// Loading state
<InvestorUpdateCardSkeleton size="sm" />
```

**Key behaviors:**
- Falls back to a generated placeholder image when `update.featured_image` is absent
- Truncates `strategic_update` / `content` preview to 120 characters
- Renders `#update_number` badge when present
- `sm` variant displays period range via `formatInvestorUpdatePeriod`; shows a `<Calendar>` icon fallback when no cover image is available
