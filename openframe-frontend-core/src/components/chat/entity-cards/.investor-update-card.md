Pure presentation component for displaying investor updates in two density modes (`default` and `sm`), with no built-in click logic — navigation is handled by the caller via `href`.

## Key Components

### `InvestorUpdateCard`
Main card component rendering an `InvestorUpdate` entity. Delegates link behavior to `useEntityCardLink` and image fallbacks to `useEntityCardPlaceholder`.

- **`default` size** — full card using `AdminContentCard` with cover image, title, summary, update number badge, and period meta.
- **`sm` size** — compact horizontal layout using shared `COMPACT_CARD_*` class tokens with a square thumbnail, title row, period subtitle, and truncated summary.

### `InvestorUpdateCardSkeleton`
Loading placeholder matching the visual footprint of both `default` and `sm` sizes using animated pulse styles.

## Props (`InvestorUpdateCardProps`)

| Prop | Type | Description |
|------|------|-------------|
| `update` | `InvestorUpdate` | Entity data |
| `href` | `string` | Resolved detail URL (caller-provided) |
| `target` | `'_blank'` | Opens in new tab when set |
| `rel` | `string` | Link rel attribute |
| `targetPlatform` | `string \| null` | Platform hint for link resolution |
| `placeholderUrl` | `string \| null` | OG fallback when `featured_image` is missing |
| `size` | `'default' \| 'sm'` | Card density (defaults to `'default'`) |
| `className` | `string` | Additional CSS classes |

## Usage Example

```typescript
import { InvestorUpdateCard, InvestorUpdateCardSkeleton } from './investor-update-card'

// Default full card
<InvestorUpdateCard
  update={investorUpdate}
  href={`/updates/${investorUpdate.id}`}
  size="default"
/>

// Compact card opened in new tab (e.g., chat dispatch)
<InvestorUpdateCard
  update={investorUpdate}
  href={`/updates/${investorUpdate.id}`}
  target="_blank"
  rel="noopener noreferrer"
  size="sm"
/>

// Loading state
<InvestorUpdateCardSkeleton size="sm" />
```
