Provides default prop-builder functions for `<ProductReleaseCard>`, covering both the compact `sm` variant (used in chat dispatch) and the full `lg` variant (used in `<ProductReleasesView>`).

## Key Components

### Interfaces

- **`ReleaseLike`** — Minimal shape for `sm` builder input; covers image/video URL fields and release date.
- **`RichReleaseLike`** — Extended shape for `lg` builder input; adds `release_type`, `release_status`, `view_count`, `author`, and changelog arrays.
- **`ProductReleaseCardDerivedProps`** — Full derived props contract for the `lg` card variant.

### Functions

- **`pickCover(item)`** *(internal)* — Resolves the best cover image URL from a `ReleaseLike` row; prefers video thumbnails when a video URL is present, falls back to `featured_image` / `og_image_url`.
- **`defaultBuildProductReleaseCardProps(item)`** — Lightweight `sm`-only builder. Returns `coverImage`, `hasVideoCover`, and `formattedDate`. Used as the fallback when no custom builder is passed via `opts.extras`.
- **`buildProductReleaseCardProps(item)`** — Rich `lg` builder and default for `<ProductReleasesView>`. Populates the full metadata grid including type, status, badge color, author, view count, and changelog counts.

## Usage Example

```typescript
// sm card default (chat dispatch fallback)
import { defaultBuildProductReleaseCardProps } from './product-release-card-defaults'

const smProps = defaultBuildProductReleaseCardProps(apiRow)
// { coverImage: '...', hasVideoCover: false, formattedDate: 'Jan 15, 2025' }

// lg card (ProductReleasesView default)
import { buildProductReleaseCardProps } from './product-release-card-defaults'

const lgProps = buildProductReleaseCardProps(apiRow)
// { coverImage, hasVideoCover, formattedDate, releaseType: 'minor',
//   releaseStatus: 'stable', releaseTypeBadgeColor, viewCount: 42,
//   author: { full_name, avatar_url, job_title },
//   changelogCounts: { features: 3, fixes: 1, improvements: 2, breaking: 0 } }
```

> Hub-side embedders override the default by passing a custom `buildCardProps` via `extras`. The `sm` chat card always uses `defaultBuildProductReleaseCardProps` regardless, since it only reads the three `sm`-relevant fields.
