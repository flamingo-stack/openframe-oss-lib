Provides default prop-builder functions for `<ProductReleaseCard>`, covering both the compact `sm` chat variant (3 fields) and the full `lg` catalog variant (complete metadata grid).

## Key Components

**`defaultBuildProductReleaseCardProps(item)`**
Lightweight builder for the `sm` card variant. Derives `coverImage`, `hasVideoCover`, and `formattedDate` from raw list-API row data. Used as the fallback when embedders don't supply a custom builder via `opts.extras.buildProductReleaseCardProps`.

**`buildProductReleaseCardProps(item)`**
Rich builder for the `lg` card variant. Populates the full `ProductReleaseCardDerivedProps` bundle including release type/status, badge color, view count, author, and changelog counts (`features`, `fixes`, `improvements`, `breaking`). This is the **default** builder for `<ProductReleasesView>`.

**`pickCover(item)`** *(internal)*
Selects the best cover image from a row's candidate fields. Prefers video thumbnails when a video URL is present, falling back to `featured_image` → `og_image_url`.

**`ProductReleaseCardDerivedProps`** *(exported interface)*
Full type contract for the `lg` builder's return value.

## Usage Example

```typescript
import {
  defaultBuildProductReleaseCardProps,
  buildProductReleaseCardProps,
} from './product-release-card-defaults'

// sm chat card — minimal fields only
const smProps = defaultBuildProductReleaseCardProps(apiRow)
// { coverImage, hasVideoCover, formattedDate }

// lg catalog card — full metadata grid
const lgProps = buildProductReleaseCardProps(apiRow)
// { coverImage, hasVideoCover, formattedDate, releaseType,
//   releaseStatus, releaseTypeBadgeColor, viewCount,
//   author, changelogCounts }

// Pass a custom builder via extras to override the default
<ProductReleasesView
  buildCardProps={buildProductReleaseCardProps}
/>
```

> **TZ safety note:** `formattedDate` uses `formatReleaseDate` in both builders — it splits the `YYYY-MM-DD` string before constructing a `Date`, avoiding the off-by-one-day shift that `new Date(input).toLocaleDateString` produces for date-only inputs west of UTC.
