A React component that renders interactive approval request messages with command display and approval/rejection functionality for the Flamingo MSP platform.

## Key Components

- **ApprovalRequestMessage** - Main forwardRef component that displays approval requests with command text, optional explanations, and interactive buttons
- **Command Display** - Scrollable code block showing the command to be approved with optional icon
- **Status Handling** - Shows different UI states for pending, approved, and rejected requests
- **Action Buttons** - Approve/Reject buttons with loading states and async handling
- **StatusTag Integration** - Visual status indicators for completed requests

## Usage Example

```typescript
import { ApprovalRequestMessage } from './approval-request-message'
import { Terminal } from 'lucide-react'

function MyComponent() {
  const handleApprove = async (requestId: string) => {
    // Process approval
    await api.approveRequest(requestId)
  }

  const handleReject = async (requestId: string) => {
    // Process rejection  
    await api.rejectRequest(requestId)
  }

  return (
    <ApprovalRequestMessage
      data={{
        requestId: "req_123",
        command: "sudo apt update && sudo apt upgrade -y",
        explanation: "System update required for security patches",
        icon: <Terminal className="w-4 h-4" />
      }}
      status="pending"
      onApprove={handleApprove}
      onReject={handleReject}
    />
  )
}
```

The component automatically handles processing states, displays appropriate status indicators, and provides a clean interface for MSP technicians to review and approve system commands through the OpenFrame platform.
