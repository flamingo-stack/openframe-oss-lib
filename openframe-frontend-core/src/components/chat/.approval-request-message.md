A React component for displaying command approval requests in MSP platforms, featuring interactive approve/reject buttons and status indicators for AI-driven operations.

## Key Components

- **ApprovalRequestMessage** - Main component with pending/completed states
- **ApprovalRequestData** - Interface defining command, explanation, icon, and request ID
- **ApprovalRequestMessageProps** - Component props with approval/rejection callbacks
- **Status States** - Pending (interactive), approved/rejected (read-only with status tags)

## Usage Example

```typescript
import { ApprovalRequestMessage } from "./approval-request-message"
import { Terminal } from "lucide-react"

const MyComponent = () => {
  const handleApprove = (requestId?: string) => {
    console.log("Approved request:", requestId)
    // Execute the approved command
  }

  const handleReject = (requestId?: string) => {
    console.log("Rejected request:", requestId)
    // Log rejection or provide alternative
  }

  return (
    <ApprovalRequestMessage
      data={{
        command: "sudo systemctl restart nginx",
        explanation: "Restart web server to apply configuration changes",
        icon: <Terminal className="w-4 h-4" />,
        requestId: "req-123"
      }}
      status="pending"
      onApprove={handleApprove}
      onReject={handleReject}
    />
  )
}
```

The component automatically switches to a read-only status display when approved or rejected, showing appropriate success/error indicators with StatusTag components.
