Mobile-specific header component for the chat panel, implementing a distinct layout from the desktop variant with a back chevron, truncated `h2` title, and a contextual `⋯` actions menu.

## Key Components

**`ChatPanelHeaderMobile`** — Primary export. Renders a card-style header (`ods-card` surface with bottom border) optimized for mobile drawer navigation.

**`ChatPanelHeaderMobileProps`** — Extends `ChatPanelHeaderProps` with an optional `className` override.

**`menuItems`** — Dynamically computed array of `ActionsMenuItem` entries, conditionally including:
- `open-archive` — shown when not in back-navigation mode and `onOpenArchive` is provided
- `unarchive` — shown in archived view when `onRestore` is provided
- `rename` / `archive` — shown in active chat view when respective handlers exist

The `⋯` trigger (`ActionsMenuDropdown`) is hidden entirely when no items apply.

## Usage Example

```typescript
<ChatPanelHeaderMobile
  title="Support Ticket #1042"
  showBack={true}
  backAriaLabel="Back to chats"
  onBack={() => router.back()}
  onRename={handleRename}
  onArchive={handleArchive}
  onOpenArchive={handleOpenArchive}
  className="custom-header"
/>
```

> **Note:** `onClose` is accepted via the base `ChatPanelHeaderProps` contract for API parity but is intentionally unused — mobile drawer dismissal is handled externally via the backdrop or back chevron. The orchestrator renders this component on mobile and the desktop `ChatPanelHeader` at `md+` breakpoint.
