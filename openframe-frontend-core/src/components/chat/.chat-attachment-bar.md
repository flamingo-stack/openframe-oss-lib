Renders the attachment UI layer for the chat compose box: a horizontal chip strip (above the textarea) showing staged files with progress/status, and an icon-only `+` button (below the textarea) for picking new files.

## Key Components

### Exports
- **`ChatAttachmentChipStrip`** — Renders a horizontal row of file chips above the textarea. Returns `null` when no attachments are staged.
- **`ChatAttachmentAddButton`** — Borderless `+` icon button rendered inline in the bottom-controls row. Renders an invisible same-size `<span>` placeholder when `attachmentsEnabled` is `false` to prevent layout shift on async identity resolution.
- **`StagedAttachment`** — Interface mirroring the hub's `useChatAttachments` output (`id`, `file`, `status`, `progress`, `storagePath`, `viewToken`, `errorMessage`).
- **`ChatAttachmentSize`** — `'default' | 'compact'` density token for narrow surfaces (e.g. ticket-drawer reply composer).
- **`CHAT_ATTACHMENT_MIME_TYPES`** — Curated allow-list (images + video); excludes `image/svg+xml` for XSS safety.
- **`CHAT_ATTACHMENT_CONCURRENT_UPLOADS_PER_USER`** — Client-side cap of 5 concurrent uploads.

### Internal
- **`AttachmentChip`** — Single file pill with blob-URL thumbnail preview (images), extension initials fallback, upload progress bar overlay, and `×` remove button.

## Usage Example

```typescript
import {
  ChatAttachmentChipStrip,
  ChatAttachmentAddButton,
} from './chat-attachment-bar'

// Above textarea:
<ChatAttachmentChipStrip
  attachments={stagedAttachments}
  onRemove={(id) => removeAttachment(id)}
  disabled={isStreaming}
  size="compact"
/>

// Inside bottom-controls row:
<ChatAttachmentAddButton
  attachmentsEnabled={identity.attachmentsEnabled}
  attachmentsCount={stagedAttachments.length}
  onAddFiles={(files) => addFiles(files)}
  disabled={isStreaming}
  size="compact"
/>
```
