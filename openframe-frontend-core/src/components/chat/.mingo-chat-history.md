Scrollable chat history list component for the Mingo AI interface, displaying dialogs grouped by date (Today / Yesterday / Older) with unread badges, row actions (rename/archive), infinite scroll, and a matching skeleton loader.

## Key Components

| Export | Description |
|--------|-------------|
| `MingoChatHistory` | Main component rendering grouped dialog rows with scroll fades and infinite-scroll sentinel |
| `MingoChatHistorySkeleton` | Pulse-animated loading placeholder matching the grouped layout to prevent layout flicker |
| `MingoChatHistoryProps` | Full prop interface including pagination, active state, and action callbacks |
| `MingoChatHistoryRow` | Internal row component with unread badge, title truncation, and `⋯` actions menu |
| `groupDialogs` | Date-bucketing utility splitting dialogs into Today / Yesterday / Older groups |

## Usage Example

```typescript
import {
  MingoChatHistory,
  MingoChatHistorySkeleton,
} from '@/components/mingo/mingo-chat-history'
import type { DialogItem } from '@/components/mingo/types/component.types'

const dialogs: DialogItem[] = [
  { id: '1', title: 'Printer offline - Site A', timestamp: new Date(), unreadMessagesCount: 3 },
  { id: '2', title: 'VPN setup request', timestamp: new Date(Date.now() - 86400000) },
]

// Loading state
if (isLoading) return <MingoChatHistorySkeleton />

// Loaded state
return (
  <MingoChatHistory
    dialogs={dialogs}
    activeDialogId="1"
    onSelectDialog={(id) => router.push(`/mingo/${id}`)}
    onRequestRename={(dialog) => openRenameModal(dialog)}
    onRequestArchive={(dialog) => openArchiveModal(dialog)}
    hasMore={pagination.hasNextPage}
    isLoadingMore={pagination.isFetchingNextPage}
    onLoadMore={pagination.fetchNextPage}
  />
)
```

## Notes

- Dialogs are expected **pre-sorted newest-first** by the host; the component only groups, never re-sorts.
- The `⋯` actions menu is hidden on mobile (touch has no hover) and stays visible while the Radix dropdown is open via local `menuOpen` state.
- Unread counts above 99 render as `99+`.
- Timestamp-less dialogs fall into the **Older** bucket.
