Scrollable chat history panel for the Mingo AI interface that renders past dialogs grouped by recency (Today / Yesterday / Older) with unread badges, rename/archive actions, infinite scroll, and a skeleton loading state.

## Key Components

- **`MingoChatHistory`** — Main export. Renders grouped dialog rows with scroll-fade affordances and an IntersectionObserver-based infinite-scroll sentinel.
- **`MingoChatHistorySkeleton`** — Pulse-animated loading placeholder that mirrors the grouped row layout to prevent layout flicker on first load.
- **`MingoChatHistoryRow`** *(internal)* — Individual dialog row with unread badge, truncated title, and a `⋯` actions dropdown (Rename / Archive).
- **`groupDialogs`** *(internal)* — Partitions dialogs into Today / Yesterday / Older buckets based on wall-clock time.
- **`MingoChatHistoryProps`** — Full prop interface including cursor-pagination controls (`hasMore`, `isLoadingMore`, `onLoadMore`).

## Usage Example

```typescript
import {
  MingoChatHistory,
  MingoChatHistorySkeleton,
} from './mingo-chat-history'

// Show skeleton while first page loads
if (isLoading) return <MingoChatHistorySkeleton />

// Render history once data arrives
return (
  <MingoChatHistory
    dialogs={dialogs}
    activeDialogId={currentDialogId}
    onSelectDialog={(id) => router.push(`/mingo/${id}`)}
    onRequestRename={(dialog) => openRenameModal(dialog)}
    onRequestArchive={(dialog) => openArchiveModal(dialog)}
    hasMore={pagination.hasNextPage}
    isLoadingMore={isFetchingMore}
    onLoadMore={fetchNextPage}
  />
)
```

> **Note:** Dialogs should be pre-sorted newest-first by the host. The component only handles grouping, not sorting. The `onRequestRename` / `onRequestArchive` callbacks are **request signals** — the host is responsible for opening the appropriate confirmation/rename modal.
