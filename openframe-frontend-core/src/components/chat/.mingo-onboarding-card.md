A reusable onboarding/slash-command row card component for Mingo's chat empty-state UI, rendering as either an interactive `<button>` or a static `<div>` depending on whether actions or an `onClick` handler are provided.

## Key Components

### `MingoOnboardingCard`
The primary export. Renders a single card row with:
- **Icon slot** — optional 16×16 leading icon using `currentColor` (inherits `ods-text-secondary`)
- **Title row** — truncating title + optional right-aligned slash-command label
- **Description** — optional secondary-color paragraph below the title row
- **Actions row** — optional outline chip buttons (e.g. `Recent`, `Search`, `Find`); when present, the card itself becomes non-interactive

### Interactivity Logic
| Condition | Root element |
|-----------|-------------|
| `actions` provided (non-empty) | `<div>` — each action owns its click |
| `onClick` provided, no `actions` | `<button>` with hover/focus styles |
| Neither | `<div>` |

### Interfaces
- **`MingoOnboardingCardProps`** — full card configuration
- **`MingoOnboardingCardAction`** — shape for each action button (`id`, `label`, `onClick`)

## Usage Example

```typescript
// Interactive card (no actions)
<MingoOnboardingCard
  icon={<RoadmapIcon />}
  title="Roadmap"
  slashCommand="/roadmap"
  description="View and manage your product roadmap."
  onClick={() => insertCommand('/roadmap')}
/>

// Static card with action buttons
<MingoOnboardingCard
  icon={<SearchIcon />}
  title="Tickets"
  slashCommand="/tickets"
  description="Browse open tickets."
  actions={[
    { id: 'recent', label: 'Recent', onClick: (e) => handleRecent(e) },
    { id: 'search', label: 'Search', onClick: (e) => handleSearch(e) },
  ]}
/>
```

> **Layout note:** Stack multiple cards inside a `rounded-md` container — the `last:border-b-0` rule on `border-b` ensures the bottom card's divider is hidden, keeping the visual frame clean.
