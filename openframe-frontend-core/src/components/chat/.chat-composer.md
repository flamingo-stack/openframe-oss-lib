Composes the chat panel footer by combining the message input (or archived placeholder) with the model/usage row and optional attachment controls.

## Key Components

- **`ChatComposer`** ‚Äî Primary export; a layout wrapper that renders either a live `ChatInput` or a read-only archived state banner, followed by the model display row
- **`ChatComposerProps`** ‚Äî Interface defining all props including `archived`, `inputRef`, `onSend`, `onStop`, `sending`, `model`, and optional attachment controls

## Usage Example

```typescript
import { ChatComposer } from './chat-composer'
import { useRef } from 'react'
import type { ChatInputRef } from './types/component.types'

function ChatPanel() {
  const inputRef = useRef<ChatInputRef>(null)

  return (
    <ChatComposer
      inputRef={inputRef}
      onSend={(text) => console.log('Sending:', text)}
      onStop={() => console.log('Stopped')}
      sending={false}
      placeholder="Ask Mingo anything..."
      autoFocus
      showAttachmentButton
      attachmentsCount={2}
      onAddFiles={(files) => handleFiles(files)}
      model={{
        modelName: 'gpt-4o',
        usage: { tokensUsed: 1200, tokensLimit: 8000 },
      }}
    />
  )
}

// Archived chat ‚Äî shows unarchive prompt instead of input
<ChatComposer
  archived
  inputRef={inputRef}
  onSend={noop}
  onStop={noop}
  sending={false}
  placeholder=""
  model={modelProps}
/>
```

## Notes

- `showAttachmentButton` is **Guide-only** ‚Äî Mingo's NATS agent backend does not accept attachments, so omitting this prop entirely collapses the layout slot rather than leaving an invisible gap
- iOS safe-area inset is handled via `max(var(--spacing-system-xxs), env(safe-area-inset-bottom))` to clear the home bar
- Inherits panel surface background (no own `bg-*` class) for seamless blending
