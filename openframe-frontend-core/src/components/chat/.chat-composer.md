Composes the chat panel footer by combining the message input (or archived placeholder) with the model/usage display row and an optional file attachment button.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `ChatComposer` | `React.FC` | Main composer component rendering the full footer region |
| `ChatComposerProps` | `interface` | Props contract for the composer |

**Internal dependencies:** `ChatFooter`, `ChatInput`, `ChatAttachmentAddButton`, `ModelDisplay`, `BoxArchiveIcon`

## Usage Example

```typescript
import { ChatComposer } from './chat-composer'
import { useRef } from 'react'
import type { ChatInputRef } from './types/component.types'

function ChatPanel() {
  const inputRef = useRef<ChatInputRef>(null)

  return (
    <ChatComposer
      inputRef={inputRef}
      onSend={(text) => console.log('Send:', text)}
      onStop={() => console.log('Stop')}
      sending={false}
      placeholder="Type a message…"
      autoFocus
      model={{
        modelName: 'gpt-4o',
        tokensUsed: 1200,
        tokenLimit: 8000,
      }}
      // Guide mode: enable file attachments
      showAttachmentButton
      attachmentsCount={2}
      onAddFiles={(files) => handleFiles(files)}
    />
  )
}

// Archived (read-only) state — renders the unarchive prompt instead of input
<ChatComposer
  archived
  inputRef={inputRef}
  onSend={() => {}}
  onStop={() => {}}
  sending={false}
  placeholder=""
  model={modelProps}
/>
```

## Behavior Notes

- **`archived`** swaps the `ChatInput` for a read-only banner prompting the user to unarchive — no input is rendered
- **`showAttachmentButton`** is Guide-only; omitting it (or `onAddFiles`) collapses the attachment slot entirely to avoid a 28 px ghost gap
- Bottom padding respects the iOS safe-area inset via `max(xxs, env(safe-area-inset-bottom))`
