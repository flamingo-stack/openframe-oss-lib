Central barrel export file for the `chat` UI library, re-exporting all public components, hooks, types, and utilities from the chat module as a single entry point.

## Key Components

**Chat UI Components**
- `ChatContainer`, `ChatInput`, `ChatComposer` — Core chat layout and input primitives
- `ChatMessageEnhanced`, `ChatMessageList`, `ChatMessageRow` — Message rendering pipeline
- `ChatPanelHeader`, `ChatHeaderIconButton`, `ChatSidebar` — Chat chrome and navigation
- `ChatTypingIndicator`, `ToolExecutionDisplay`, `ToolCallBlocks` — Real-time feedback components

**Approval & Display**
- `ApprovalRequestMessage`, `ApprovalBatchMessage` — AI action approval flows
- `ThinkingDisplay`, `ContextCompactionDisplay`, `ErrorMessageDisplay` — Status/state displays

**Mingo AI Specific**
- `MingoOnboardingCard`, `MingoWelcome`, `MingoChatHistory`, `MingoChatModals` — Mingo AI onboarding and session management
- `GuideModeBanner`, `GuideWelcome` — Guide mode UI

**Attachments & Suggestions**
- `ChatAttachmentAddButton`, `ChatAttachmentChipStrip` — File attachment handling with `CHAT_ATTACHMENT_MIME_TYPES` and `CHAT_ATTACHMENT_CONCURRENT_UPLOADS_PER_USER` constants
- `SlashCommandSuggestions` — Slash command autocomplete

**Orchestration**
- `EmbeddableChat` — Full chat orchestrator; mount once at the root inside a `<ChatRuntimeContext.Provider>`

**Supporting Exports**
- `ChatRef` — Ref type for programmatic chat control
- `remarkCardLinks` — Remark plugin for card-style link rendering
- `NavLinkAnchorViaRuntime`, `SourceActionButton` — Shared UI primitives migrated from hub

## Usage Example

```typescript
import {
  EmbeddableChat,
  ChatInput,
  ChatMessageList,
  ChatAttachmentAddButton,
  CHAT_ATTACHMENT_MIME_TYPES,
  type ChatRef,
} from '@flamingo/chat'

// Mount the full chat orchestrator
export function AppRoot() {
  const chatRef = useRef<ChatRef>(null)

  return (
    <ChatRuntimeContext.Provider value={runtime}>
      <EmbeddableChat ref={chatRef} agentId="mingo" />
    </ChatRuntimeContext.Provider>
  )
}
```
