Barrel (index) file that re-exports all public-facing components, hooks, types, utilities, and constants from the chat feature module, enabling consumers to import from a single entry point.

## Key Components

**Chat UI Components**
- `ChatContainer`, `ChatComposer`, `ChatInput`, `ChatPanelHeader` — core layout and input primitives
- `ChatMessageEnhanced`, `ChatMessageList`, `ChatMessageRow` — message rendering
- `ChatTypingIndicator`, `ThinkingDisplay`, `ContextCompactionDisplay` — real-time state indicators
- `ToolExecutionDisplay`, `ToolCallBlocks` — AI tool call visualization
- `ApprovalRequestMessage`, `ApprovalBatchMessage` — human-in-the-loop approval flows
- `ErrorMessageDisplay`, `ExpandChevron` — shared UI primitives

**Onboarding & Navigation**
- `MingoOnboardingCard`, `MingoWelcome`, `GuideWelcome`, `GuideMo deBanner` — first-run experiences
- `MingoChatHistory`, `MingoChatModals`, `ChatSidebar` — navigation and history

**Entity Cards & Attachments**
- `entity-cards` — 12+ refactored entity card components
- `ChatAttachmentAddButton`, `ChatAttachmentChipStrip` — file attachment bar with MIME type constants

**Supporting Exports**
- `EmbeddableChat` — portable chat orchestrator, equivalent to `<GlobalAskAI>`, mounted once at the root under `<ChatRuntimeContext.Provider>`
- `ChatRef` — ref type for imperative chat control
- `remarkCardLinks` — remark plugin for card-style link rendering
- `NavLinkAnchorViaRuntime`, `SourceActionButton` — migrated shared UI primitives
- `hooks`, `utils`, `types` — full re-export of chat hooks, utilities, and type definitions

## Usage Example

```typescript
import {
  EmbeddableChat,
  ChatContainer,
  ChatInput,
  ChatMessageList,
  type ChatRef,
  CHAT_ATTACHMENT_MIME_TYPES,
} from '@flamingo/chat'

function AppRoot() {
  const chatRef = useRef<ChatRef>(null)

  return (
    <ChatRuntimeContext.Provider value={runtime}>
      <EmbeddableChat ref={chatRef} />
    </ChatRuntimeContext.Provider>
  )
}
```
