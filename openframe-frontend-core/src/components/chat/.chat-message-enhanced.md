Renders an enhanced chat message bubble with support for multi-segment content, inline/block entity card resolution, tool execution displays, approval flows, thinking indicators, and rich markdown — all composed from host-provided render props to keep the library data-agnostic.

## Key Components

**`ChatMessageEnhanced`** — Main `forwardRef` component accepting:
- `role` — `'user' | 'error' | 'assistant'` controls layout and avatar type
- `content` — `MessageContent` (string or `MessageSegment[]`); normalized via `normalizeContent()`
- `chatRefs` — host-provided `Record<string, ChatRef>` for resolving `[card://type:id]` markers
- `renderEntityCard` — host callback returning inline or `<BlockCard>`-wrapped JSX per ref
- `NavLinkAnchor` — host navigation component injected into markdown link overrides
- `assistantType` — `'mingo' | 'fae'` controls avatar/author styling

**`normalizeContent()`** — Converts string content to a single `MessageSegment[]` for uniform rendering.

**`CARD_MARKER_REGEX`** — Shared regex kept in lockstep with `remarkCardLinks` to consistently detect `[card://type:id]` patterns across pre-scan and remark transforms.

**`renderingPlan` (useMemo)** — Builds per-segment split plans: text segments containing block-level card markers are split into `{ kind: 'text' }` and `{ kind: 'block' }` parts so block cards render at marker position during streaming, not appended at segment end.

**`cardComponentOverrides`** — Overrides the markdown `<a>` element to intercept `card://` hrefs and render pre-computed inline nodes or graceful fallbacks (real title, raw `cardId`, or dimmed placeholder).

## Usage Example

```typescript
<ChatMessageEnhanced
  role="assistant"
  assistantType="mingo"
  content={[
    { type: 'text', text: 'Here is the ticket [card://ticket:abc-123]' },
  ]}
  chatRefs={{ 'ticket:abc-123': { type: 'ticket', id: 'abc-123', title: 'Fix login bug', url: '/tickets/abc-123' } }}
  renderEntityCard={(ref) => <TicketPill ref={ref} />}
  NavLinkAnchor={AppNavLink}
/>
```
