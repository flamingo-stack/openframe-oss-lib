Renders individual chat messages with support for multiple content types including markdown text, tool executions, approval requests, thinking displays, and inline entity cards with `[card://type:id]` marker resolution.

## Key Components

### `ChatMessageEnhanced` (default export)
A `forwardRef` component that renders a single chat message. Handles role-based styling (`user`, `error`, assistant variants), avatar display, and multi-segment content rendering.

### `normalizeContent(content)`
Converts `MessageContent` (string or `MessageSegment[]`) into a normalized `MessageSegment[]` array for uniform rendering.

### `CARD_MARKER_REGEX`
Regex kept in lockstep with `remarkCardLinks` to detect `[card://type:id]` markers in text segments. Both must update together if the grammar changes.

### `renderingPlan` (internal `useMemo`)
Builds a per-message split plan for `[card://]` markers. Block-bearing markers split their containing text segment so block payloads (e.g. video players via `<BlockCard>`) render at the exact marker position in streamed text rather than drifting to the end.

### `cardComponentOverrides` (internal `useMemo`)
Overrides the `<a>` element in `SimpleMarkdownRenderer` to intercept `card://` hrefs emitted by `remarkCardLinks`, resolving them to inline entity card nodes or structured fallbacks.

## Usage Example

```typescript
<ChatMessageEnhanced
  role="assistant"
  assistantType="mingo"
  content={[
    { type: 'text', text: 'Here is the ticket [card://ticket:123]' },
    { type: 'tool_execution', toolName: 'search', result: '...' },
  ]}
  chatRefs={{
    'ticket:123': { type: 'ticket', id: '123', title: 'Login bug', url: '/tickets/123' }
  }}
  renderEntityCard={(ref) => <MyEntityCard ref={ref} />}
  NavLinkAnchor={MyNavLink}
  showAvatar={true}
/>
```

## Segment Types Rendered

| Segment Type | Component |
|---|---|
| `text` | `SimpleMarkdownRenderer` + card overrides |
| `tool_execution` | `ToolExecutionDisplay` |
| `approval_request` | `ApprovalRequestMessage` |
| `approval_batch` | `ApprovalBatchMessage` |
| `error` | `ErrorMessageDisplay` |
| `context_compaction` | `ContextCompactionDisplay` |
| `thinking` | `ThinkingDisplay` |
