Re-exports blog entity types from the canonical `src/types/blog.ts` lib, providing a single import path for the chat card entity barrel.

## Key Components

| Export | Description |
|--------|-------------|
| `BlogPost` | Full blog post row shape |
| `BlogPostSummary` | Catalog projection for listing/cards |
| `BlogPostPlatform` | Platform identifier type |
| `BlogPostCategory` | Post-level category association |
| `BlogPostTag` | Post-level tag association |
| `BlogCategory` | Blog category entity |
| `BlogTag` | Blog tag entity |
| `BlogAuthor` | Author entity shape |
| `BlogMediaAsset` | Media/image asset type |
| `BlogStatus` | Post status enum/union |
| `BlogPagination` | Pagination metadata shape |
| `BlogSearchParams` | Search/filter query params |

## Usage Example

```typescript
// Import blog types from the entities barrel (single path)
import type {
  BlogPost,
  BlogPostSummary,
  BlogAuthor,
  BlogPagination,
} from '@/entities/blog';

// Full post row (e.g. detail page)
function renderPost(post: BlogPost) {
  return `${post.title} by ${post.author.name}`;
}

// Summary projection (e.g. catalog card)
function renderCard(summary: BlogPostSummary) {
  return `${summary.title} — ${summary.category}`;
}
```

> **Note:** Do not define competing blog shapes in the hub layer. Always consume from this barrel to ensure a single source of truth via `@/types/blog`.
