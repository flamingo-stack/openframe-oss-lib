Barrel export file that centralises all **chat entity row types** — the canonical wire shapes consumed by chat cards across the Flamingo/OpenFrame platform.

## Key Components

Re-exports types from the following entity modules:

| Module | Entity |
|---|---|
| `blog` | Blog post wire shape |
| `case-study` | Case study wire shape |
| `content-ref` | Generic content reference |
| `customer-interview` | Customer interview wire shape |
| `data-room-doc` | Data room document wire shape |
| `github-activity` | Commit / PR / review wire shape |
| `hubspot-ticket` | HubSpot ticket wire shape |
| `investor-update` | Investor update wire shape |
| `onboarding-guide` | Onboarding guide wire shape |
| `program-types` | Program item wire shapes |
| `roadmap-item` | Roadmap item wire shape |
| `slack-message` | Slack message wire shape |

  // consume canonical wire shape — no hub-tree imports needed

## Architecture Notes

- **Types only** — pure wire shapes and formatters that operate on them live here; no DAL functions, RAG mappers, or sync engine logic.
- **Hub isolation** — server-side utilities that *produce* these shapes remain in the hub; this package is the single source of truth for *consuming* them.
- **Chat card contract** — any chat card component should import entity types from this barrel rather than reaching into hub internals.
