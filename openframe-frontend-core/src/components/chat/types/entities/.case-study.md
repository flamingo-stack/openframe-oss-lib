Re-exports `CaseStudy` entity types from the canonical lib definition (`src/types/case-study.ts`) into the chat card consumer barrel — no local type definitions are introduced.

## Key Components

| Export | Description |
|--------|-------------|
| `CaseStudy` | Core case study entity shape |
| `CaseStudyFilters` | Filter parameters for querying case studies |
| `CaseStudyListResponse` | Paginated/list response wrapper for case study results |

## Usage Example

```typescript
import type { CaseStudy, CaseStudyFilters, CaseStudyListResponse } from '@/entities/case-study';

const filters: CaseStudyFilters = { tag: 'networking' };

const response: CaseStudyListResponse = await fetchCaseStudies(filters);

response.items.forEach((study: CaseStudy) => {
  console.log(study.title);
});
```

> All types originate from `@/types/case-study` — this barrel acts as a stable re-export boundary so chat card consumers are decoupled from the underlying lib path.
