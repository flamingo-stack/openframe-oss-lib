Defines comprehensive network communication types for NATS messaging, WebSocket connections, and API responses. Includes configuration constants, chunk data structures, and error handling types for real-time messaging systems.

## Key Components

**Core Types:**
- `NatsMessageType` - Message classification ('message' | 'admin-message')
- `NatsConnectionStatus` - Connection state management
- `ChunkData` - Streaming data chunks with tool integration support
- `WebSocketMessage` & `WebSocketConfig` - WebSocket communication setup

**Configuration:**
- `NETWORK_CONFIG` - Network timing and limits constants
- Connection timeouts, reconnect intervals, and message limits

**Response Types:**
- `NetworkResponse<T>` - Standardized API response wrapper
- `PaginatedResponse<T>` - Paginated data responses
- `NetworkError` - Structured error information

## Usage Example

```typescript
import { ChunkData, NETWORK_CONFIG, WebSocketConfig } from './network.types'

// Configure WebSocket connection
const wsConfig: WebSocketConfig = {
  url: 'ws://localhost:8080',
  reconnect: true,
  reconnectInterval: NETWORK_CONFIG.RECONNECT_TIME_WAIT_MS,
  maxReconnectAttempts: 5,
  onMessage: (event) => {
    const chunk: ChunkData = JSON.parse(event.data)
    console.log('Received chunk:', chunk.type, chunk.text)
  }
}

// Handle streaming responses
const processChunk = (chunk: ChunkData) => {
  if (chunk.type === 'tool_call' && chunk.toolFunction) {
    return executeFunction(chunk.toolFunction, chunk.parameters)
  }
  return chunk.text
}
```
