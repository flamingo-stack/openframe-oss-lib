Core TypeScript type definitions for Flamingo's AI-powered chat system, providing strongly-typed constants and interfaces for chat operations across MSP platform interactions.

## Key Components

- **CHAT_TYPE** - Defines client and admin chat channel types
- **OWNER_TYPE** - Specifies message ownership (CLIENT, ADMIN, ASSISTANT)
- **MESSAGE_ROLE** - Standard chat roles (user, assistant, system, error)
- **ASSISTANT_TYPE** - Flamingo AI assistants (Fae for clients, Mingo for technicians)
- **APPROVAL_STATUS** - Chat message approval workflow states
- **CONNECTION_STATUS** - WebSocket connection state management
- **MessageOwner** - Interface for tracking message attribution and metadata

## Usage Example

```typescript
import { 
  CHAT_TYPE, 
  MESSAGE_ROLE, 
  ASSISTANT_TYPE, 
  MessageOwner,
  ConnectionStatus 
} from './chat.types'

// Create a client chat message
const clientMessage = {
  role: MESSAGE_ROLE.USER,
  content: "Help with server issues",
  chatType: CHAT_TYPE.CLIENT
}

// Define assistant response owner
const faeOwner: MessageOwner = {
  type: OWNER_TYPE.ASSISTANT,
  machineId: 'fae-01',
  model: ASSISTANT_TYPE.FAE
}

// Handle connection states
const handleConnectionChange = (status: ConnectionStatus) => {
  if (status === CONNECTION_STATUS.CONNECTED) {
    console.log('Chat service ready')
  }
}
```

This type system ensures type safety across Flamingo's chat components while supporting both client-facing (Fae) and technician-focused (Mingo) AI interactions within the unified OpenFrame platform.
