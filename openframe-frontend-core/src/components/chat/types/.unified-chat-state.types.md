Defines the unified contract (`UnifiedChatState`) that both SSE (Guide mode) and NATS (Mingo mode) transport hooks must satisfy, enabling consumers to work against a single interface regardless of the underlying transport.

## Key Components

| Export | Kind | Description |
|--------|------|-------------|
| `UnifiedChatState` | `interface` | Primary return shape all transport hooks must implement |
| `UnifiedChatMessage` | `interface` | Generic message shape shared across Guide and Mingo modes |
| `UnifiedSendMessageOptions` | `interface` | Options passed to `sendMessage` (attachments, hidden flag) |
| `UnifiedUsageBreakdown` | `interface` | SSE-only per-call token telemetry (Haiku pipeline costs) |
| `DialogTokenUsage` | `interface` | Mingo backend token usage snapshot per active dialog |
| `ChatConnectionState` | `type` | Transport-level connection status: `'connected' \| 'connecting' \| 'disconnected'` |
| `StreamingPhase` | `type` | Granular streaming status: `'idle' \| 'thinking' \| 'streaming'` |

## Usage Example

```typescript
import type {
  UnifiedChatState,
  UnifiedChatMessage,
  StreamingPhase,
} from './unified-chat-state.types'

// Consuming the unified contract — works for both SSE and NATS transports
function ChatShell({ chat }: { chat: UnifiedChatState }) {
  const { messages, isLoading, streamingPhase, sendMessage } = chat

  const statusLabel: Record<StreamingPhase, string> = {
    idle: '',
    thinking: 'Thinking...',
    streaming: 'Streaming...',
  }

  return {
    status: statusLabel[streamingPhase],
    messageCount: messages.length,
    // SSE-only fields gracefully degrade to null in Mingo mode
    modelLabel: chat.currentModelLabel ?? 'Unknown model',
    cacheHit: chat.currentCacheHitRatePct ?? 0,
    onSubmit: (text: string) =>
      sendMessage(text, { hidden: false, attachments: [] }),
  }
}
```

> **Transport transparency:** Fields exclusive to one transport (e.g. `currentProvider`, `currentUsageBreakdown` for SSE; `dialogs`, `activeDialogId` for Mingo) are typed as `null` or `undefined` in the off-mode — consumers never need to branch on `mode` directly.
