Defines the shared TypeScript contract that both SSE (Guide mode) and NATS (Mingo mode) transport hooks must satisfy, ensuring consumers never need to branch on transport implementation details.

## Key Components

### Interfaces

- **`DialogTokenUsage`** — Token telemetry snapshot from the Mingo/NATS backend (`inputTokensSize`, `outputTokensSize`, `totalTokensSize`, `contextSize`)
- **`UnifiedUsageBreakdown`** — SSE-only per-call token breakdown for the Haiku rewriter/classifier/summarizer pipeline and routed-answer metadata
- **`UnifiedChatMessage`** — Transport-agnostic message shape with optional SSE-only fields (`sources`, `chatRefs`) and common fields (`segments`, `scrollAnchor`, `hidden`)
- **`UnifiedSendMessageOptions`** — Options for `sendMessage`, including `hidden` (background context turns) and `attachments` (SSE-only image uploads)
- **`UnifiedChatState`** — The primary return contract for `useChat({ mode })`, covering the message thread, streaming state, actions, per-turn LLM metadata, and dialog management

### Types

- **`ChatConnectionState`** — `'connected' | 'connecting' | 'disconnected'`
- **`StreamingPhase`** — `'idle' | 'thinking' | 'streaming'`

## Usage Example

```typescript
import type {
  UnifiedChatState,
  UnifiedChatMessage,
  StreamingPhase,
} from './unified-chat-state.types'

// Consuming the contract in a transport-agnostic component
function ChatShell({ chat }: { chat: UnifiedChatState }) {
  const phase: StreamingPhase = chat.streamingPhase

  const handleSend = async (text: string) => {
    await chat.sendMessage(text, {
      hidden: false,
      attachments: [],
    })
  }

  return {
    messages: chat.messages,          // UnifiedChatMessage[]
    isLoading: chat.isLoading,        // boolean
    phase,                            // 'idle' | 'thinking' | 'streaming'
    provider: chat.currentProvider,   // null in Mingo mode
    dialogs: chat.dialogs,            // DialogItem[]
  }
}
```

> **Note:** SSE-only fields (`currentProvider`, `currentModelLabel`, `currentUsageBreakdown`, etc.) are always `null` in Mingo/NATS mode. Guard with null checks before rendering LLM metadata UI.
