Defines comprehensive TypeScript types for Flamingo's chat messaging system, covering message content, tool executions, approval workflows, and message processing pipeline.

## Key Components

- **MESSAGE_TYPE** - Enum-like constant defining all supported message types (TEXT, EXECUTING_TOOL, APPROVAL_REQUEST, etc.)
- **MessageSegment** - Union type for structured message content (text, tool execution, approval requests)
- **MessageData** - GraphQL/API message data interfaces with type-specific fields
- **HistoricalMessage** - Raw message format from database/API
- **ProcessedMessage/Message** - Normalized message format for UI rendering
- **Tool & Approval Types** - Specialized interfaces for tool execution and approval workflows

## Usage Example

```typescript
import type { Message, MessageType, MessageSegment } from './message.types'

// Creating a text message
const textMessage: Message = {
  id: 'msg-123',
  role: 'user',
  content: 'Hello, how can I help?',
  timestamp: new Date()
}

// Creating a structured message with tool execution
const toolMessage: Message = {
  id: 'msg-456', 
  role: 'assistant',
  content: [
    { type: 'text', text: 'Running deployment...' },
    {
      type: 'tool_execution',
      data: {
        type: 'EXECUTING_TOOL',
        integratedToolType: 'kubernetes',
        toolFunction: 'deployPod',
        parameters: { namespace: 'default' }
      }
    }
  ],
  assistantType: 'mingo'
}

// Type-safe message type checking
if (messageData.type === MESSAGE_TYPE.APPROVAL_REQUEST) {
  // TypeScript knows this is ApprovalRequestMessageData
  console.log(messageData.command)
}
```
