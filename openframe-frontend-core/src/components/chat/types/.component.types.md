Defines all TypeScript prop and configuration types for the chat UI component library, covering container, header, message, input, and slash-command interfaces used across the Flamingo/OpenFrame chat surface.

## Key Components

### Core Prop Interfaces
- **`ChatContainerProps`** — Base div wrapper props for the chat layout root
- **`ChatHeaderProps`** — Header configuration including user info, connection status, ticket metadata, and layout flags (`fullWidth`, `bare`)
- **`ChatMessageEnhancedProps`** — Rich message props supporting multi-segment content, approval callbacks, inline entity card rendering, and custom anchor routing
- **`ChatMessageListProps`** — Message list with infinite scroll, avatar control, approval delegation, and entity card/anchor forwarding
- **`ChatMessageListRef`** — Imperative scroll control handle (`scrollToBottom`, `getScrollPosition`, `setScrollPosition`)

### Slash Command Types
- **`SlashCommandSummary`** — Server-resolved command descriptor with actions, icons, labels, and display ordering
- **`SlashCommandSummaryAction`** — Single action affordance (`browse` | `search` | `find` | `display`) with resolved label
- **`SlashCommandsProp`** — Consumer-provided `fetchCommands` callback and optional `resolveSourceIcon` resolver
- **`SlashCommandSourceMeta`** — Icon + label pair returned by the source icon resolver

### Extensibility Types
- **`NavLinkAnchorComponent`** — Host-supplied anchor component for markdown link routing (soft nav, new-tab, modifier-click handling)
- **`ChatHeaderTicketInfo`** — Ticket context metadata including lifecycle status name, color, and kind

## Usage Example

```typescript
import type {
  ChatHeaderProps,
  ChatMessageListProps,
  SlashCommandsProp,
  NavLinkAnchorComponent,
} from './component.types'

const MyAnchor: NavLinkAnchorComponent = ({ href, className, children }) => (
  <a href={href} className={className}>{children}</a>
)

const headerProps: ChatHeaderProps = {
  userName: 'Mingo AI',
  connectionStatus: 'connected',
  fullWidth: true,
  bare: false,
  ticketInfo: {
    title: 'Printer offline – HQ',
    statusKind: 'in-progress',
    statusName: 'In Progress',
    statusColor: '#FFC008',
  },
}

const slashCommands: SlashCommandsProp = {
  fetchCommands: async (prefix, signal) => {
    const res = await fetch(`/api/docs/commands?prefix=${prefix}`, { signal })
    return res.json()
  },
  resolveSourceIcon: (sourceId) => {
    if (sourceId === 'tickets') return { Icon: TicketIcon, label: 'Tickets' }
    return undefined
  },
}
```
