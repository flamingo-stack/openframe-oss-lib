Comprehensive TypeScript definitions for API interactions and React hooks in a chat system with real-time messaging capabilities. Includes types for NATS subscriptions, chunk processing, dialog management, and approval workflows.

## Key Components

**Hook Options:**
- `UseChunkCatchupOptions` - Configuration for catching up on missed message chunks
- `UseNatsDialogSubscriptionOptions` - NATS WebSocket subscription configuration with client settings
- `UseRealtimeChunkProcessorOptions` - Real-time message processing with approval filtering

**Hook Return Types:**
- `UseChunkCatchupReturn` - Chunk catchup functionality with buffering controls
- `UseNatsDialogSubscriptionReturn` - Connection status for NATS subscriptions
- `UseRealtimeChunkProcessorReturn` - Message segment processing and approval management

**API Request/Response Types:**
- Chat API endpoints (`ChatAPIRequest`, `ChatAPIResponse`)
- Dialog management (`DialogCreateRequest`, `DialogListResponse`)
- Approval workflows (`ApprovalRequest`, `ApprovalResponse`)
- Settings management (`ChatSettings`, `UpdateSettingsRequest`)

**Callback Interfaces:**
- `RealtimeChunkCallbacks` - Comprehensive event handlers for streaming, metadata, approvals, and escalations

## Usage Example

```typescript
// Configure real-time chunk processing
const processorOptions: UseRealtimeChunkProcessorOptions = {
  callbacks: {
    onStreamStart: () => console.log('Stream started'),
    onSegmentsUpdate: (segments) => updateUI(segments),
    onApprove: async (requestId) => await approveRequest(requestId),
    onEscalatedApproval: (id, data) => handleEscalation(id, data)
  },
  displayApprovalTypes: ['CLIENT', 'ADMIN'],
  initialState: {
    existingSegments: previousSegments,
    pendingApprovals: new Map()
  }
};

// Dialog creation request
const createDialog: DialogCreateRequest = {
  name: 'Support Session',
  metadata: { customer: 'user123', priority: 'high' }
};
```
