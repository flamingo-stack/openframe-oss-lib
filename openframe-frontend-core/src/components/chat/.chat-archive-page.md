Renders the Chat Archive page within the chat panel, displaying a header with back/close navigation and a date-grouped list of archived dialogs powered by `MingoChatHistory`.

## Key Components

**`ChatArchivePage`** — Primary exported component that composes:
- `ChatPanelHeaderMobile` — Mobile header (back + close) hidden on `md+`
- Desktop header bar — Back chevron + title + close button (visible `md+`)
- `MingoChatHistory` — Reused dialog list (pagination-aware, no row action menus)
- Loading skeleton — 6 animated pulse rows matching dialog-item height
- Empty state — Centered `BoxArchiveIcon` with instructional copy

**`ChatArchivePageProps`** — Interface defining:

| Prop | Type | Description |
|------|------|-------------|
| `dialogs` | `ReadonlyArray<DialogItem>` | Archived dialogs (pre-grouped by date) |
| `onSelectDialog` | `(id: string) => void` | Opens a selected archived dialog |
| `onBack` | `() => void` | Returns to previous list view |
| `onClose` | `() => void` | Closes the entire chat panel |
| `isLoading` | `boolean?` | Drives skeleton/spinner display |
| `hasMore` | `boolean?` | Indicates additional pages available |
| `onLoadMore` | `() => void?` | Fetches next page of dialogs |

## Usage Example

```typescript
<ChatArchivePage
  dialogs={archivedDialogs}
  onSelectDialog={(id) => openDialog(id)}
  onBack={() => setView('list')}
  onClose={() => setChatOpen(false)}
  isLoading={isFetching}
  hasMore={pagination.hasNextPage}
  onLoadMore={fetchNextPage}
/>
```

The component renders three content states: a paginated dialog list when `dialogs.length > 0`, a pulse skeleton during initial load (`isLoading && dialogs.length === 0`), and an empty state when no archived chats exist.
