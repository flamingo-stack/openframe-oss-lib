A chat-aware `<a>` wrapper that routes link clicks through the active `ChatRuntime`, handling new-tab decisions, panel state, and navigation — for use exclusively within the chat component tree.

## Key Components

### `NavLinkAnchorViaRuntimeProps`
| Prop | Type | Description |
|------|------|-------------|
| `href` | `string` | Raw href to resolve and navigate to |
| `path` | `string \| null` | Optional internal path hint for same-tab routing |
| `targetPlatform` | `string \| null` | Platform context for new-tab resolution |
| `className` | `string` | Optional CSS class |
| `children` | `ReactNode` | Anchor content (optional per `NavLinkAnchorComponent` contract) |

### `NavLinkAnchorViaRuntime`
The primary export. On each click it:
1. Resolves the final href via `resolveHrefForRuntime`
2. Determines new-tab vs. same-tab via `computeIsNewTab` (embed-mode short-circuit → `runtime.navigation.decideNewTab` → lib fallback)
3. Delegates to `handleChatNavClick` for actual navigation
4. Closes the chat panel via `ChatPanelContext` on same-tab navigations; leaves it open for new-tab navigations

> **⚠️ Chat-tree only:** Uses `useRequiredChatRuntime`, which **throws** if mounted outside a `<ChatRuntimeContext.Provider>`. Use the hub's `useNavLink`-based anchor for surfaces outside the chat tree (header chrome, marketing pages, etc.).

## Usage Example

```typescript
// Inside a chat markdown renderer or source-chip component
import { NavLinkAnchorViaRuntime } from './nav-link-anchor-via-runtime'

function ChatMarkdownLink({ href, children }) {
  return (
    <NavLinkAnchorViaRuntime
      href={href}
      className="chat-link"
    >
      {children}
    </NavLinkAnchorViaRuntime>
  )
}
```
