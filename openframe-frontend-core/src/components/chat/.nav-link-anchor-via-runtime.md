Chat-tree `<a>` wrapper that routes link clicks through the active `ChatRuntime`, applying new-tab logic, relay attributes, and panel close behavior for same-tab navigation.

## Key Components

### `NavLinkAnchorViaRuntimeProps`
| Prop | Type | Description |
|------|------|-------------|
| `href` | `string` | Raw href, resolved against the runtime before use |
| `path` | `string \| null` | Optional internal path for same-tab routing |
| `targetPlatform` | `string \| null` | Hints the new-tab decision logic |
| `className` | `string` | Forwarded to the `<a>` tag |
| `children` | `ReactNode` | Optional — empty anchors are valid |

### `NavLinkAnchorViaRuntime`
The exported component. On each render it:
1. Reads the `ChatRuntime` via `useRequiredChatRuntime` — **throws** if mounted outside a `<ChatRuntimeContext.Provider>`
2. Resolves the final href via `resolveHrefForRuntime`
3. Determines new-tab behavior via `computeIsNewTab` (embed-mode short-circuit → `runtime.navigation.decideNewTab` → lib fallback)
4. On click, delegates to `handleChatNavClick`; if handled in the same tab, calls `panel.closeChat()`

## Usage Example

```typescript
// Inside a chat markdown body or source-chip drill-in
<NavLinkAnchorViaRuntime
  href="https://docs.example.com/guide"
  path="/guide"
  targetPlatform="web"
  className="chat-link"
>
  View Documentation
</NavLinkAnchorViaRuntime>
```

> **Chat-only contract:** This component is intentionally restricted to the chat tree. For navigation outside the chat context (header chrome, marketing pages), use the hub's `useNavLink`-based anchor instead.
