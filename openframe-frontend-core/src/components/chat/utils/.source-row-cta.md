Resolves the click-target contract (icon, href, platform, and Ask drill-in eligibility) for a single RAG source row, used uniformly across chat source chips, inline cards, and search results.

## Key Components

### Types

| Type | Description |
|------|-------------|
| `SourceRowInput` | Shape of a RAG row — `sourceRepo`, `documentType`, `id`, `title`, `externalUrl`, `path`, etc. |
| `SourceRowContext` | Host-injected surface config — `baseRoute`, `chipBasePlatform`, `currentPlatform`, `composeContentUrl`, `docPlatformTargets`, `tableIdForDocumentType` |
| `SourceRowCTA` | Resolved contract — `icon`, `iconLabel`, `href`, `targetPlatform`, `askable`, `chatRef` |

### Functions

| Function | Description |
|----------|-------------|
| `resolveSourceRowCTA(row, ctx)` | Pure function; resolves the full CTA contract for a row given host context |
| `sourceRowCtxFromRuntime(runtime, surface)` | Factory that builds a `SourceRowContext` from a `ChatRuntime` and surface config |
| `sanitizePath(path)` | Strips unsafe characters from path segments to prevent traversal or injection |

### Constants

| Export | Description |
|--------|-------------|
| `DOC_TABLE_TYPES` | `['markdown', 'data_room_doc']` — doc-table types eligible for path-based navigation fallback |

## Usage Example

```typescript
import { resolveSourceRowCTA, sourceRowCtxFromRuntime } from './source-row-cta'

// Build context once per surface render from the chat runtime
const ctx = sourceRowCtxFromRuntime(runtime, {
  baseRoute: '/knowledge-base',
  chipBasePlatform: 'flamingo',
})

// Resolve CTA for each row (chip, inline card, search result)
const cta = resolveSourceRowCTA(
  {
    sourceRepo: 'blog-posts',
    documentType: 'markdown',
    id: 'row-123',
    title: 'Getting Started Guide',
    path: 'docs/getting-started',
  },
  ctx,
)

// cta.icon     → React component for the row's source
// cta.href     → safe destination URL (or null → Ask-only)
// cta.askable  → true if Ask drill-in can fire
// cta.chatRef  → pre-built ChatRef for the Ask handler
```

**URL priority order for doc rows (no `externalUrl`):**
1. `ctx.docPlatformTargets[documentType]` — per-type cross-platform target
2. `ctx.chipBasePlatform` — legacy single-platform fallback
3. `ctx.baseRoute` — in-app relative navigation
4. `null` — Ask-only
