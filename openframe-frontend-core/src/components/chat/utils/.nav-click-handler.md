Thin compatibility wrapper that adapts the legacy `handleChatNavClick` call signature used by chat-rendered nav elements (chip blocks, `ChatCardNavWrap`, `NavLinkAnchorViaRuntime`) to the unified `executeNavigation` primitive.

## Key Components

- **`ChatNavClickInput`** — Input interface defining `href` (required), `path`, and `targetPlatform` (both optional).
- **`handleChatNavClick`** — Deprecated forwarding function that maps the existing 4-argument call shape onto `executeNavigation`. Returns a `boolean` indicating whether the event was handled.

## Usage Example

```typescript
import { handleChatNavClick } from './nav-click-handler'
import { useRuntime } from '../../../contexts/chat-runtime-context'
import { useRouter } from 'next/navigation'

// Inside a chat-rendered nav component:
const runtime = useRuntime()
const router = useRouter()

<a
  href={href}
  onClick={(e) =>
    handleChatNavClick(
      e,
      runtime,
      { href, path: '/dashboard', targetPlatform: 'web' },
      router.push,
    )
  }
>
  Go to Dashboard
</a>
```

> **Deprecated:** New code should call `executeNavigation` from `execute-navigation.ts` directly. This file exists solely to preserve backward compatibility for existing chat callers without requiring call-site changes.
