Thin compatibility wrapper that forwards chat navigation clicks to the unified `executeNavigation` primitive, preserving the existing call signature used by chip blocks, `<ChatCardNavWrap>`, and `<NavLinkAnchorViaRuntime>`.

## Key Components

### `ChatNavClickInput`
Input shape describing the navigation target:
- `href` — raw anchor href
- `path` — optional resolved internal path
- `targetPlatform` — optional platform hint for routing decisions

### `handleChatNavClick(e, runtime, input, fallbackNavigate?)`
Delegates entirely to `executeNavigation`. Returns `boolean` indicating whether the event was handled. The `fallbackNavigate` parameter accepts `useRouter().push` for same-origin soft navigation when the runtime does not wire `navigation.navigate`.

## Usage Example

```typescript
import { handleChatNavClick } from './nav-click-handler'
import { useRouter } from 'next/navigation'

const router = useRouter()

function onAnchorClick(e: React.MouseEvent<HTMLAnchorElement>) {
  handleChatNavClick(
    e,
    runtime,
    { href: '/dashboard', path: '/dashboard', targetPlatform: null },
    router.push,
  )
}
```

## Notes

> **Deprecated.** New code should call `executeNavigation` directly from `execute-navigation.ts`. This file exists solely for backward compatibility — the full decision tree (empty-href no-op, modifier/non-primary passthrough, new-tab via `openExternal`/`window.open`, same-tab via `navigate` → fallback → `window.location`) lives in `executeNavigation`.
