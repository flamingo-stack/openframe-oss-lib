Thin compatibility shim that re-exports the canonical embed proxy authentication utilities under legacy `chat-`-prefixed names, preserving backward compatibility for existing chat-namespace callers.

## Key Components

| Export | Original Name | Description |
|--------|--------------|-------------|
| `ChatProxyAuth` | `EmbedProxyAuth` | Type alias for the proxy auth object |
| `getChatProxyAuth` | `getEmbedProxyAuth` | Retrieves current proxy auth from storage |
| `setChatProxyAuth` | `setEmbedProxyAuth` | Persists proxy auth to storage |
| `clearChatProxyAuth` | `clearEmbedProxyAuth` | Clears proxy auth from storage |
| `getPersistedProxyEmail` | *(unchanged)* | Reads the persisted proxy email |
| `applyProxyAuth` | *(unchanged)* | Applies proxy auth headers to requests |

> **Deprecated:** All exports here are forwarded from `@flamingo-stack/openframe-frontend-core/utils/embed-proxy-auth-storage`. New code should import from that module directly. Wire headers (`X-Chat-Act-As`, etc.) and the storage key (`chat.proxy-auth.v1`) remain unchanged.

## Usage Example

**Legacy callers (still valid, but deprecated):**

```typescript
import { getChatProxyAuth, setChatProxyAuth, clearChatProxyAuth } from './chat-proxy-auth-storage'

const auth = getChatProxyAuth()
setChatProxyAuth({ token: 'abc123', email: 'tech@msp.com' })
clearChatProxyAuth()
```

**New code should import from the canonical path:**

```typescript
import {
  getEmbedProxyAuth,
  setEmbedProxyAuth,
  clearEmbedProxyAuth,
  applyProxyAuth,
} from '@flamingo-stack/openframe-frontend-core/utils'
```
