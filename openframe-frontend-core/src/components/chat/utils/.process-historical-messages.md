Converts raw `HistoricalMessage` arrays from GraphQL/API responses into display-ready `ProcessedMessage` objects, handling message grouping, assistant segment accumulation, and approval request lifecycle tracking.

## Key Components

### `processHistoricalMessages(messages, options)`
Main export. Iterates over historical messages, groups consecutive assistant messages into batched segments via `MessageSegmentAccumulator`, and flushes them at user-message boundaries or end-of-array. Returns both the processed message list and a map of escalated approvals filtered out by `displayApprovalTypes`.

### `ProcessHistoricalMessagesResult`
Return type containing:
- `messages` — `ProcessedMessage[]` ready for rendering
- `escalatedApprovals` — `Map` of approval requests excluded from the current chat view (e.g. admin-only approvals surfaced to a different UI layer)

### `processMessageData(data, accumulator, approvalStatuses, options, escalatedApprovals)`
Private dispatcher that routes each `MessageData` item by type:
- `TEXT` / `THINKING` → appends to accumulator
- `EXECUTING_TOOL` / `EXECUTED_TOOL` → adds tool execution segments
- `APPROVAL_REQUEST` → resolves batch vs. single, pending vs. resolved, and honors `batchApprovalsEnabled` flag
- `APPROVAL_RESULT` → materializes deferred approvals stored in `escalatedApprovals`

### Helper Functions
- `getOwnerDisplayName(owner)` — resolves display name for admin or client owners
- `getOwnerAvatar(owner)` — extracts optional avatar URL from owner profile
- `pushStandaloneMessages(...)` — handles `SYSTEM` type messages as standalone entries

## Usage Example

```typescript
import { processHistoricalMessages } from './process-historical-messages'

const { messages, escalatedApprovals } = processHistoricalMessages(rawMessages, {
  assistantName: 'Fae',
  assistantType: 'fae',
  assistantAvatar: '/avatars/fae.png',
  approvalStatuses: { 'req-123': 'approved' },
  displayApprovalTypes: ['CLIENT'],
  batchApprovalsEnabled: true,
  onApprove: (id) => handleApprove(id),
  onReject: (id) => handleReject(id),
})

// escalatedApprovals contains admin-level requests excluded from CLIENT view
console.log(escalatedApprovals.get('req-456')?.approvalType) // 'ADMIN'
```
