Converts raw GraphQL/API historical messages into structured, display-ready `ProcessedMessage` objects, handling grouping of assistant segments, approval workflows, tool executions, and user/admin attribution.

## Key Components

### `processHistoricalMessages(messages, options)`
Main export. Iterates over `HistoricalMessage[]` and produces a `ProcessHistoricalMessagesResult` containing:
- **`messages`** — ordered array of `ProcessedMessage` objects ready for rendering
- **`escalatedApprovals`** — `Map` of approval requests filtered out by `displayApprovalTypes` (e.g. admin-tier approvals hidden from client chat)

### `ProcessHistoricalMessagesResult`
Exported interface describing the return shape of `processHistoricalMessages`.

### `processMessageData` *(internal)*
Dispatches individual `MessageData` items to the appropriate `MessageSegmentAccumulator` method based on `MESSAGE_TYPE` (`TEXT`, `THINKING`, `EXECUTING_TOOL`, `EXECUTED_TOOL`, `APPROVAL_REQUEST`, `APPROVAL_RESULT`).

### `flushAssistantMessage` *(internal closure)*
Coalesces consecutive assistant `HistoricalMessage` rows into a single `ProcessedMessage` with grouped segments, using the last message ID in the group for stable React keys.

### Helper functions
| Function | Purpose |
|---|---|
| `getOwnerDisplayName` | Resolves admin full name or falls back to `"Admin"` / `"You"` |
| `getOwnerAvatar` | Extracts optional profile image URL from the message owner |
| `pushStandaloneMessages` | Emits `SYSTEM`-type messages as standalone user-role entries |

## Usage Example

```typescript
import { processHistoricalMessages } from './process-historical-messages'

const { messages, escalatedApprovals } = processHistoricalMessages(
  historicalMessages,
  {
    assistantName: 'Fae',
    assistantType: 'fae',
    assistantAvatar: '/avatars/fae.png',
    approvalStatuses: { 'req-abc123': 'approved' },
    displayApprovalTypes: ['CLIENT'],
    batchApprovalsEnabled: true,
    onApprove: (id) => handleApprove(id),
    onReject: (id) => handleReject(id),
  }
)

// escalatedApprovals holds ADMIN-tier requests the client UI should not render
escalatedApprovals.forEach((data, requestId) => {
  console.log(`Escalated: ${requestId} — ${data.command}`)
})
```

> **Note:** `batchApprovalsEnabled` defaults to `true`. Pass `false` to unfold batch approval cards into individual legacy approval UI cards for consumers that have not yet adopted the batch approval flow.
