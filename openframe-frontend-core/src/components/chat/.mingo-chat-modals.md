Modal components for chat management actions (rename, archive, unarchive) used within the Mingo AI chat interface.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `RenameChatModal` | Component | Input modal to rename an existing chat; reseeds on each open |
| `ArchiveChatModal` | Component | Destructive confirmation modal to hide a chat from current view |
| `UnarchiveChatModal` | Component | Confirmation modal to restore an archived chat |
| `ChatDialogModals` | Component | Composite wrapper rendering all three modals from a single prop shape |
| `RenameChatModalProps` | Interface | Props for `RenameChatModal` |
| `ArchiveChatModalProps` | Interface | Props for `ArchiveChatModal` |
| `UnarchiveChatModalProps` | Interface | Props for `UnarchiveChatModal` |
| `ChatDialogModalsProps` | Interface | Props for `ChatDialogModals` |

## Usage Example

```typescript
// Pair ChatDialogModals with useChatDialogManager for full dialog state management
const {
  renameTarget, setRenameTarget, onConfirmRename,
  archiveTarget, setArchiveTarget, onConfirmArchive,
  restoreTarget, setRestoreTarget, onConfirmRestore,
} = useChatDialogManager()

return (
  <ChatDialogModals
    renameTarget={renameTarget}
    setRenameTarget={setRenameTarget}
    onConfirmRename={onConfirmRename}
    archiveTarget={archiveTarget}
    setArchiveTarget={setArchiveTarget}
    onConfirmArchive={onConfirmArchive}
    restoreTarget={restoreTarget}
    setRestoreTarget={setRestoreTarget}
    onConfirmRestore={onConfirmRestore}
  />
)
```

## Notes

- All modals share consistent ODS-token button styling (`footerBtn`, `cancelBtn`, `saveBtn`, `dangerBtn`) matching the Figma `button-full` spec
- `RenameChatModal` auto-focuses the input and supports `Enter` to save; the Save button is disabled when the trimmed value is empty
- `ArchiveChatModal` uses the `dangerBtn` (error/red) style to signal a destructive action
