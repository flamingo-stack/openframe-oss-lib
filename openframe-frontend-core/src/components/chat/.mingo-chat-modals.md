A collection of modal dialog components for managing Mingo chat conversations, providing UI for renaming, archiving, and unarchiving chat dialogs.

## Key Components

- **`RenameChatModal`** — Input modal to rename a chat; auto-seeds from `initialName` on open, fires `onSave` with the trimmed name on submit
- **`ArchiveChatModal`** — Confirmation modal for archiving (hiding) a chat from the active list
- **`UnarchiveChatModal`** — Confirmation modal for restoring an archived chat back to the active list
- **`ChatDialogModals`** — Composite wrapper that renders all three modals together, driven by nullable `DialogItem` targets; designed to pair with `useChatDialogManager`

## Usage Example

```typescript
// Standalone usage
<RenameChatModal
  isOpen={isOpen}
  initialName="My Chat"
  onClose={() => setIsOpen(false)}
  onSave={(newName) => handleRename(newName)}
/>

// Composite usage (recommended — pairs with useChatDialogManager)
<ChatDialogModals
  renameTarget={renameTarget}
  setRenameTarget={setRenameTarget}
  onConfirmRename={(name) => handleRename(name)}
  archiveTarget={archiveTarget}
  setArchiveTarget={setArchiveTarget}
  onConfirmArchive={handleArchive}
  restoreTarget={restoreTarget}
  setRestoreTarget={setRestoreTarget}
  onConfirmRestore={handleRestore}
/>
```

## Notes

- All modals are capped at `max-w-[600px]` on medium+ screens via `ModalV2`
- Button styles (`cancelBtn`, `saveBtn`, `dangerBtn`) follow the Figma `button-full` spec — full-width, DM Sans Bold 18px, `rounded-md`
- `RenameChatModal` supports `Enter` key submission and disables Save when the trimmed input is empty
