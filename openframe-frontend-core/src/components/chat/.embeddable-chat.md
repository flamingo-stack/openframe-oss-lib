`EmbeddableChat` is a lib-portable port of the hub's `GlobalAskAI` component, designed to work across hub and embedded contexts by routing all navigation and identity decisions through `useRequiredChatRuntime()`. It supports dual-mode (Guide/Mingo) chat, controlled/uncontrolled open state, configurable shell wrappers, and extensible entity card dispatch.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `EmbeddableChat` | Component | Main chat panel component |
| `EmbeddableChatProps` | Interface | Full prop surface for the component |
| `ChatCardDispatchExtras` | Type (re-export) | Optional builders for hub-land card types (programs, product releases) |

### Notable Props

| Prop | Type | Description |
|------|------|-------------|
| `modes` | `UseUnifiedChatModes` | Per-mode transport config (Guide=SSE, Mingo=NATS) |
| `mingoState` | `UnifiedChatState` | Host-injected Mingo state (bypasses internal NATS adapter) |
| `shell` | `'drawer' \| 'none'` | Wraps in Radix Drawer or renders bare body for custom containers |
| `extras` | `ChatCardDispatchExtras` | Builders for chat-card types whose props live in hub-land |
| `activeMode` / `onActiveModeChange` | controlled pair | Controlled mode toggle between `'guide'` and `'mingo'` |
| `mingoWelcome` | `Partial<MingoWelcomeProps>` | Content overrides for the Mingo empty state |
| `guideWelcome` | `Partial<GuideWelcomeProps>` | Content overrides for the Guide empty state |

## Usage Example

```typescript
import { EmbeddableChat } from '@flamingo/chat-lib'

// Minimal uncontrolled usage — runtime wired at root via HubRuntimeProvider
function MyLayout() {
  return (
    <HubRuntimeProvider source="openframe" user={currentUser}>
      <EmbeddableChat
        showInternalTrigger
        modes={{
          guide: { tableIdForDocumentType: (type) => ragTableMap[type] ?? null },
          mingo: { natsConfig },
        }}
        defaultActiveMode="mingo"
        shell="drawer"
      />
    </HubRuntimeProvider>
  )
}

// Controlled open state with host-injected Mingo state (no internal NATS adapter)
function EmbeddedPanel({ chatState, open, onOpenChange }) {
  return (
    <EmbeddableChat
      open={open}
      onOpenChange={onOpenChange}
      shell="none"
      mingoState={chatState}
      mingoDialogCapabilities={{
        canRename: true,
        canArchive: true,
        fetchArchivedDialogs: api.fetchArchived,
        unarchiveDialog: api.unarchive,
      }}
    />
  )
}
```

> **Note:** When `shell="none"`, the internal trigger and iOS scroll-lock are suppressed. The consumer owns mount/unmount and must drive open/close via the `open`/`onOpenChange` props.
