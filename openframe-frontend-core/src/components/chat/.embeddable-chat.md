Brief description of the component's purpose and public API for the `EmbeddableChat` component — a library-portable, host-agnostic port of the hub's `<GlobalAskAI>` chat panel that delegates all navigation, identity, and source decisions through a runtime context.

## Key Components

### `EmbeddableChat` (default export)
The main chat panel component supporting two modes:
- **Guide mode** — SSE-based RAG retrieval chat
- **Mingo mode** — NATS-based AI agent chat

### `EmbeddableChatProps`
Full prop bundle controlling the chat panel behavior:

| Prop | Type | Description |
|------|------|-------------|
| `open` / `onOpenChange` | `boolean` / `fn` | Controlled open state |
| `shell` | `'drawer' \| 'none'` | Layout wrapper strategy |
| `modes` | `UseUnifiedChatModes` | Per-mode transport config |
| `mingoState` | `UnifiedChatState` | Host-injected Mingo state |
| `extras` | `ChatCardDispatchExtras` | Host-supplied card builders |
| `activeMode` / `onActiveModeChange` | `ChatMode` / `fn` | Controlled mode toggle |
| `mingoWelcome` | `Partial<MingoWelcomeProps>` | Mingo empty-state overrides |
| `guideWelcome` | `Partial<GuideWelcomeProps>` | Guide empty-state overrides |

### Key Hooks Used
- `useRequiredChatRuntime()` — reads user identity, source, and navigation from the host-provided runtime
- `useUnifiedChat()` — orchestrates Guide + Mingo mode state
- `useChatAttachments()` — manages file/entity attachment strips
- `useSlashCommandRegistry()` — slash command lookup and dispatch

## Usage Example

```typescript
// Host wires the runtime once at root
import { HubRuntimeProvider } from './hub-runtime-provider'
import { EmbeddableChat } from '@flamingo/chat-lib'

function App() {
  return (
    <HubRuntimeProvider>
      <EmbeddableChat
        shell="drawer"
        modes={{
          guide: { tableIdForDocumentType: (type) => tableMap[type] },
          mingo: { agentId: 'mingo-v2' },
        }}
        mingoWelcome={{
          title: 'Hi, I am Mingo',
          subtitle: 'Your AI IT assistant',
        }}
        extras={{
          buildProgramCardProps: (id) => programCardPropsMap[id],
        }}
      />
    </HubRuntimeProvider>
  )
}
```

## Architecture Notes

- **No hub imports** — all hub-specific concerns (`useAuth`, `currentPlatform`, `NavLink`, etc.) are replaced by `useRequiredChatRuntime()`
- **Shell modes** — `'drawer'` wraps in a Radix `<Drawer>` with iOS scroll-lock; `'none'` renders the bare chat body for embedding inside host containers
- **Injected state** — when `mingoState` is provided, no internal NATS subscription is opened; the host owns the data lifecycle across panel unmounts
