Chat input component for AI chat interfaces that handles message composition, submission, slash command autocomplete, and exposes an imperative ref API for programmatic control.

## Key Components

### `ChatInput` (default export)
A `forwardRef` component wrapping a resizable `Textarea` with:
- **Send/Stop controls** — toggles between a Send button and Stop button based on `sending`/`awaitingResponse` state
- **Slash command autocomplete** — debounced (150ms) dropdown triggered by `/` prefix, with keyboard navigation (↑/↓/Tab/Enter/Escape)
- **Smart submit** — clears draft only on successful send (`onSend` return value gating); supports `allowEmptySend` for attachment surfaces
- **`preventScroll` focus** — all focus paths use `{ preventScroll: true }` to avoid viewport jumps

### `ChatInputRef` (imperative handle)
Exposed methods for parent control:

| Method | Description |
|--------|-------------|
| `focus()` | Focus the textarea |
| `blur()` | Blur the textarea |
| `clear()` | Clear the draft |
| `setValue(text)` | Pre-fill and focus with cursor at end |
| `setValueAndCursor(text, offset)` | Pre-fill with clamped cursor position |
| `submit(text)` | Programmatically fire a message |
| `getValue()` | Read current draft value |

### `SLASH_INPUT_TRIGGER`
Regex `/^\/([a-z][a-z0-9-]*)?$/` — shared constraint kept in sync with `lib/config/slash-commands-config.ts` and the chat-route slash dispatch parser.

## Usage Example

```typescript
const inputRef = useRef<ChatInputRef>(null)

// Programmatically inject a slash command
inputRef.current?.setValue('/restart ')

// Submit on behalf of the user
inputRef.current?.submit('/status check')

<ChatInput
  ref={inputRef}
  onSend={async (message) => {
    const ok = await sendMessage(message)
    return ok ? undefined : false // returning false keeps the draft
  }}
  onStop={() => abortController.abort()}
  sending={isSending}
  awaitingResponse={isWaiting}
  slashCommands={{ fetchCommands: fetchSlashCommands }}
  autoFocus
/>
```
