Responsive chat panel top-navigation bar that renders different layouts for mobile and desktop, surfacing context-aware actions (back navigation, rename, archive, restore, close) based on the current view state.

## Key Components

**`ChatPanelHeader`** — Primary export. Renders a `ChatPanelHeaderMobile` for viewports below `md`, and a fixed-height (`h-14`) desktop bar at `md+`. Right-hand actions are conditionally mounted based on props:

| Condition | Action rendered |
|---|---|
| `isArchivedView && onRestore` | Restore (refresh) button |
| `showBack && !isArchivedView && menuItems.length > 0` | `⋯` dropdown (rename / archive) |
| `!showBack && onOpenArchive` | Clock / archive-entry button |
| Always | Close (`✕`) button |

**`COMPACT_HEADER_BUTTON`** — Exported Tailwind class string for the borderless 32px back-chevron button used in the desktop layout.

**`ChatPanelHeaderProps`** — Full prop interface covering all view states (list, active conversation, archived conversation, guide).

## Usage Example

```typescript
<ChatPanelHeader
  showBack={true}
  title="Support ticket #1042"
  backAriaLabel="Back to chats"
  isArchivedView={false}
  onBack={() => setView('list')}
  onClose={() => setPanelOpen(false)}
  onRename={() => openRenameDialog(chatId)}
  onArchive={() => archiveChat(chatId)}
/>

// List view (no back chevron, shows archive entry button)
<ChatPanelHeader
  title="Mingo AI"
  onBack={() => {}}
  onClose={() => setPanelOpen(false)}
  onOpenArchive={() => setView('archive')}
/>
```
