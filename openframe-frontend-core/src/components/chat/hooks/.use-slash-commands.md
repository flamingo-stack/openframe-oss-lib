Provides slash-command autocomplete functionality for the OpenFrame chat interface, including a stateless fetch helper, a debounced React hook, and a session-cached registry hook.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `fetchSlashCommands` | `async function` | Stateless fetch wrapper; resolves commands from a caller-supplied URL, returns empty array on non-2xx responses |
| `useSlashCommands` | React hook | Debounced (150ms), cancellable hook returning `{ commands, loading }` for live prefix-based autocomplete |
| `useSlashCommandRegistry` | React hook | Full registry fetch via `react-query`; session-cached (`staleTime`/`gcTime: Infinity`), deduplicated across consumers |
| `SlashCommandSummary` | Re-exported type | Canonical type from `component.types`; shared by the dropdown UI and chat runtime |
| `SlashCommandSummaryAction` | Re-exported type | Action descriptor attached to each command |
| `SlashCommandActionId` | Re-exported type | Identifier type for command actions |

## Usage Example

```typescript
import {
  useSlashCommands,
  useSlashCommandRegistry,
  fetchSlashCommands,
} from "@flamingo-stack/openframe-frontend-core/components/chat/hooks";

// Live autocomplete as user types (debounced, cancellable)
function SlashMenu({ prefix }: { prefix: string | null }) {
  const { commandsUrl } = useChatRuntime().endpoints;
  const { commands, loading } = useSlashCommands(prefix, commandsUrl);

  return loading ? <Spinner /> : <CommandList items={commands} />;
}

// Session-wide registry — one network request, shared across consumers
function ChatPanel({ mode }: { mode: "guide" | "mingo" }) {
  const { commandsUrl } = useChatRuntime().endpoints;
  const { commands, loaded } = useSlashCommandRegistry(commandsUrl, {
    enabled: mode === "guide",
  });

  return loaded ? <OnboardingCard commands={commands} /> : null;
}

// Stateless imperative fetch (e.g. SSE adapter, server actions)
const results = await fetchSlashCommands("dep", undefined, commandsUrl);
```

> **Auth note:** All fetches ride on `embedAuthedFetch`, inheriting bearer/act-as credentials and 401 self-healing automatically — no host-side `window.fetch` patching required.
