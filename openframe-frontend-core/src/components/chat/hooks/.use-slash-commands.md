Provides slash-command autocomplete logic for the OpenFrame chat UI — a stateless fetch helper, a debounced React hook, and a session-cached registry hook.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `fetchSlashCommands` | `async function` | Stateless GET helper; resolves `commandsUrl` against `window.location.origin`, appends `?q=prefix`, returns `SlashCommandSummary[]` or `[]` on any non-2xx |
| `useSlashCommands` | React hook | Debounced (150ms), cancellable hook wrapping `fetchSlashCommands`; returns `{ commands, loading }` |
| `useSlashCommandRegistry` | React hook | Full registry fetch (empty prefix) backed by react-query; dedupes to a single network request per `commandsUrl`, cached for the session lifetime |
| `SlashCommandSummary` | re-exported type | Canonical type from `../types/component.types`; shared by autocomplete UI and chat runtime |
| `SlashCommandActionId` | re-exported type | Action ID type for slash command entries |
| `SlashCommandSummaryAction` | re-exported type | Action descriptor attached to each summary |

## Usage Example

```typescript
// One-shot fetch (e.g. SSE adapter)
const commands = await fetchSlashCommands("dep", abortSignal, commandsUrl);

// Debounced autocomplete dropdown
function SlashMenu({ prefix }: { prefix: string | null }) {
  const { commandsUrl } = useChatRuntime().endpoints;
  const { commands, loading } = useSlashCommands(prefix, commandsUrl);
  return loading ? <Spinner /> : <CommandList items={commands} />;
}

// Session-cached registry (shared across onboarding card + SSE adapter)
function ChatPanel() {
  const { commandsUrl } = useChatRuntime().endpoints;
  const { commands, loaded } = useSlashCommandRegistry(commandsUrl, {
    enabled: activeMode === "guide",
  });
  // Both consumers read from the same react-query cache entry —
  // only one GET /commands fires regardless of how many components mount.
}
```

## Notes

- **Auth**: delegates to `embedAuthedFetch` — bearer/act-as tokens, proxy impersonation, and 401 self-heal are all handled transparently; no host-side `fetch` patch needed.
- **Error handling**: non-2xx responses (401, 403, 429) silently return `[]` — the autocomplete shows no suggestions rather than surfacing error toasts.
- **`useSlashCommandRegistry`** uses `staleTime: Infinity` + `gcTime: Infinity` so remounting the chat drawer never triggers a refetch within the same session.
