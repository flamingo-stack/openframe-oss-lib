Unified entry-point hook that routes between SSE/Guide (`useSseChatAdapter`) and NATS/Mingo (`useNatsChatAdapter`) transports based on the `activeMode` flag, always calling both hooks to satisfy React rules while keeping the inactive adapter idle.

## Key Components

- **`ChatMode`** — `'guide' | 'mingo'` discriminator selecting the active transport
- **`UseUnifiedChatModes`** — per-mode config slots; each is optional so single-mode consumers only populate what they need
- **`UseUnifiedChatOptions`** — full hook config: `modes`, `activeMode`, and optional `mingoStateOverride` for host-managed Mingo state that survives panel unmounts
- **`useUnifiedChat(options)`** — the exported hook; returns a stable `UnifiedChatState` with forwarded actions for messaging, dialog management, approvals, and token usage

## Usage Example

```typescript
// Guide-only consumer
const chatState = useUnifiedChat({
  activeMode: 'guide',
  modes: {
    guide: { /* UseSseChatAdapterOptions */ },
  },
})

// Dual-mode consumer (user toggles between guide and mingo)
const chatState = useUnifiedChat({
  activeMode: currentMode, // 'guide' | 'mingo'
  modes: {
    guide: {},
    mingo: {
      dialogId: activeDialogId,
      getNatsWsUrl: (token) => `wss://nats.example.com?token=${token}`,
      publishUserMessage: async (msg) => await api.publish(msg),
    },
  },
})

// Host-owned Mingo state (persists across panel unmounts)
const chatState = useUnifiedChat({
  activeMode: 'mingo',
  modes: { guide: {} }, // no modes.mingo — internal NATS adapter stays idle
  mingoStateOverride: storeBackedMingoState,
})
```

## Notes

- Inactive adapter retains its local message buffer — flipping `activeMode` restores the user to exactly where they left off with no data loss
- `mingoStateOverride` is the seam for hosts that own their Mingo state in an external store or cache
- All returned action callbacks are individually `useCallback`-stabilised; the final object is `useMemo`-ed to prevent spurious downstream re-renders
