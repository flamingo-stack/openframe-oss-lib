A unified React hook that acts as the single public entry point for Flamingo's chat surface, dispatching between SSE/Guide and NATS/Mingo transport adapters based on the active mode while preserving message history across mode switches.

## Key Components

### Types

- **`ChatMode`** — Discriminator union (`'guide' | 'mingo'`) controlling which transport adapter is live
- **`UseUnifiedChatModes`** — Per-mode config container; each slot is optional, allowing single-mode consumers
- **`UseUnifiedChatOptions`** — Hook input shape including `modes`, `activeMode`, and the optional `mingoStateOverride` escape hatch

### Constants / Helpers

- **`EMPTY_SSE_OPTIONS`** — Stable fallback config for the SSE adapter when guide mode is not configured
- **`createDisabledNatsConfig()`** — Produces a throw-on-call NATS config so the inactive adapter satisfies React's rules of hooks without opening any subscriptions

### Hook

- **`useUnifiedChat(options)`** — Main export. Always calls both `useSseChatAdapter` and `useNatsChatAdapter`; only the active one performs real network work. Forwards all `UnifiedChatState` actions as stable `useCallback` wrappers and returns a memoized state object.

## Usage Example

```typescript
// Single-mode consumer (Guide only)
const chatState = useUnifiedChat({
  activeMode: 'guide',
  modes: {
    guide: { /* UseSseChatAdapterOptions */ },
  },
})

// Dual-mode consumer with host-owned Mingo state
const chatState = useUnifiedChat({
  activeMode: 'mingo',
  modes: {
    guide: {},
    // omit modes.mingo — host owns the NATS subscription
  },
  mingoStateOverride: hostStoreChatState, // survives panel unmount
})

chatState.sendMessage('Hello, Mingo!') // routes to active adapter
chatState.selectDialog('dialog-123')
chatState.startNewDialog()
```

> **Note:** `mingoStateOverride` is the recommended seam for hosts that need chat state to survive panel unmounts — the internal NATS adapter stays idle and the host's store-backed state is passed through verbatim.
