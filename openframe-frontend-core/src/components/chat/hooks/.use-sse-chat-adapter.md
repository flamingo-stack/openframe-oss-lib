Adapter hook that bridges the unified chat surface to an SSE (Server-Sent Events) transport layer for Guide mode, handling message streaming, source tracking, metadata extraction, and turn lifecycle management.

## Key Components

**`useSseChatAdapter(options?, runtimeOptions?)`**
Main hook returning a `UnifiedChatState` object. Wraps `useChat` with SSE-specific streaming logic, slash-command hydration, and per-turn metadata tracking.

**`createDocStreamFn(...)`**
Factory that produces a per-call async generator consuming the SSE stream. Handles message history filtering, approval-action routing, and incremental segment yielding. Decoder/buffer are scoped per-call to prevent cross-stream contamination on rapid send-stop-send sequences.

**Types**
| Type | Purpose |
|------|---------|
| `DocChatMessage` | Full message shape including segments, sources, refs, and scroll anchors |
| `ChatSource` | RAG source metadata with grouped-chip support |
| `ChatTurnMeta` | Per-turn provider/model/token/routing metadata extracted from stream frames |
| `UseSseChatAdapterOptions` | DI slot for `tableIdForDocumentType` override |
| `StreamingPhase` | Re-exported from `unified-chat-state.types` for backward-compatible imports |

**Key behavioral contracts**
- `source` is read from runtime, never sent on the wire (server resolves via `currentPlatform()`)
- Falls back to `defaultTableIdForDocumentType` when no override is injected
- Navigation is delegated to `handleChatNavClick`, not internal router calls
- Hidden messages are filtered from the API history before each send

## Usage Example

```typescript
import { useSseChatAdapter } from './use-sse-chat-adapter'

function GuidePanel() {
  const chat = useSseChatAdapter(
    { tableIdForDocumentType: myTenantLookup },
    { active: true }
  )

  return (
    <ChatSurface
      messages={chat.messages}
      onSend={(text) => chat.sendMessage(text)}
      streamingPhase={chat.streamingPhase}
    />
  )
}
```

> **Backward compatibility:** The legacy name `useEmbeddedChat` is re-exported as an alias from `./use-embedded-chat`. Internal code should import `useSseChatAdapter` directly.
