Adapter hook that wires the SSE (Server-Sent Events) / Guide-mode transport to the unified chat surface, implementing `UnifiedChatState` for streaming AI responses over HTTP.

## Key Components

- **`useSseChatAdapter`** — Primary hook. Consumes `ChatRuntimeContext`, manages message history in localStorage, and drives an SSE stream via a generated `streamFn` passed to `useChat`.
- **`createDocStreamFn`** — Factory that returns a per-call async generator. Builds the API request body, opens the SSE connection via `chatAuthedFetch`, and yields typed `MessageSegment` values (text deltas, thinking blocks, metadata frames).
- **`ChatTurnMeta`** — Per-turn metadata interface capturing provider, model label, token counts, cache hit rate, routing decision, and scroll anchor extracted from the trailing SSE metadata frame.
- **`DocChatMessage`** / **`ChatSource`** — Public message and source-reference shapes surfaced to rendering components.
- **`UseSseChatAdapterOptions`** — DI bag; accepts an optional `tableIdForDocumentType` override (defaults to the lib-baked `defaultTableIdForDocumentType` map).
- **`UseSseChatAdapterRuntimeOptions`** — Runtime gate; `active: false` suppresses the background slash-command prefetch when Guide mode is configured but inactive.

## Usage Example

```typescript
import { useSseChatAdapter } from '@/lib/chat/hooks/use-sse-chat-adapter'

function GuideChat() {
  const {
    messages,
    streamingPhase,
    sendMessage,
    stopStreaming,
    resetMessages,
  } = useSseChatAdapter(
    // Optional: override documentType → tableId lookup for tenant-specific types
    { tableIdForDocumentType: (docType) => myTenantRegistry[docType] ?? null },
    { active: true },
  )

  return (
    <ChatUI
      messages={messages}
      isStreaming={streamingPhase !== 'idle'}
      onSend={(text) => sendMessage(text)}
      onStop={stopStreaming}
    />
  )
}
```

> **Note:** `source` is resolved server-side via `currentPlatform()` and is never sent on the wire. Navigation from rendered cards/chips goes through `handleChatNavClick`, not this hook's internal routing.
