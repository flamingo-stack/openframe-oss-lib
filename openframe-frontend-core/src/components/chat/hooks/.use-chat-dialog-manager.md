Manages the chat panel's dialog-history concerns — archive browsing, read-only archived conversation mode, and confirmation modal state — keeping `EmbeddableChat` as a thin orchestration layer.

## Key Components

**`UseChatDialogManagerArgs`** — Input interface wiring the host adapter's dialogs, selection callbacks, and optional archive/restore capabilities.

**`useChatDialogManager(args)`** — Core hook returning flat state and handlers grouped into four concerns:

| Group | Returned Fields |
|---|---|
| Archive page | `archiveOpen`, `archivedDialogs`, `archivedCursor`, `archivedLoading`, `openArchive`, `closeArchive`, `loadArchivedPage`, `handleArchivedSelect` |
| Current-chats open | `handleSelectDialog`, `isOpeningDialog` |
| Archived conversation | `isViewingArchived`, `handleBack`, `activeDialog` |
| Action modals | `renameTarget`, `archiveTarget`, `restoreTarget`, `setRename/Archive/RestoreTarget`, `handleConfirmRename/Archive/Restore` |

## Usage Example

```typescript
const manager = useChatDialogManager({
  dialogs,
  activeDialogId,
  selectDialog,
  clearMessages,
  renameDialog,
  archiveDialog,
  fetchArchivedDialogs,
  unarchiveDialog,
})

// Open the archive page
manager.openArchive()

// Select a dialog from the current-chats list (instant panel flip)
manager.handleSelectDialog('dialog-123')

// Select from the archive (enters read-only mode)
manager.handleArchivedSelect('dialog-456')

// Trigger rename modal
manager.setRenameTarget(someDialogItem)

// Confirm archival — retries on failure, exits read-only on success
await manager.handleConfirmArchive()
```

## Notes

- `isOpeningDialog` sets synchronously on selection so the conversation surface flips immediately, matching the latency of archived chat opens.
- `handleBack` from an archived conversation returns to the archive page rather than the chat list.
- Failed archive/restore operations keep modals open for user retry without corrupting local state.
- `activeDialog` resolves from both the live `dialogs` list and the locally-cached `archivedDialogs`, ensuring header title and action buttons work for archived conversations.
