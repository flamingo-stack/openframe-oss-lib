React hook that manages a NATS JetStream ephemeral ordered consumer subscription for a chat dialog stream, handling connection lifecycle, reconnection with sequence-safe resume, and cleanup.

## Key Components

**`useJetStreamDialogSubscription(options)`**
Main exported hook that orchestrates two layered effects:

- **Connection effect** — acquires/releases a shared NATS WebSocket client via `acquireSharedClient`/`releaseSharedClient`, drives `startConnectionLifecycle` for reconnection backoff, and updates `isConnected` state. Only retries on `closed`/`disconnected` status to avoid looping on JetStream protocol errors (`-ERR`).
- **Subscription effect** — creates an ephemeral `OrderedConsumer` via `client.subscribeJetStreamOrdered` whenever `isConnected && dialogId` is truthy. On reconnect, resumes from `highestStreamSeq + 1` to prevent replays or gaps. Resets sequence tracking on `dialogId` change.

**Ref-stabilized callbacks** — `onEvent`, `onConnect`, `onDisconnect`, `onSubscribed`, `onBeforeReconnect`, and `getNatsWsUrl` are all kept in refs so they never trigger effect re-runs when their identity changes (e.g., after token rotations).

**Returns** `{ isConnected, isSubscribed, reconnectionCount, currentStreamSeq }`

## Usage Example

```typescript
const { isConnected, isSubscribed, currentStreamSeq } = useJetStreamDialogSubscription({
  enabled: true,
  dialogId: 'dialog-abc123',
  topic: 'messages',
  streamName: 'CHAT_CHUNKS',        // optional, defaults to 'CHAT_CHUNKS'
  optStartSeq: 42,                  // resume from seq 43; null = live-tail (DeliverPolicy.New)
  inactiveThresholdMs: 3 * 60_000,  // optional, defaults to 5 minutes
  getNatsWsUrl: () => buildNatsUrl(authToken),
  onEvent: (payload, topic) => {
    console.log('chunk received on', topic, payload)
  },
  onConnect: () => console.log('NATS connected'),
  onDisconnect: () => console.log('NATS disconnected'),
  onSubscribed: () => console.log('consumer active'),
  onBeforeReconnect: () => refreshAuthToken(),
})
```

> **Delivery policy:** Pass `optStartSeq` (number) to replay from a known offset; omit or pass `null` to receive only new messages. On reconnect, the hook automatically advances to `highestStreamSeq + 1` regardless of the original `optStartSeq`.
