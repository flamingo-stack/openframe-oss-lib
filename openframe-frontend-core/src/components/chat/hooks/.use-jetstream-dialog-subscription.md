React hook for subscribing to a NATS JetStream chat dialog stream via an ephemeral `OrderedConsumer`, with automatic reconnection and sequence tracking.

## Key Components

- **`useJetStreamDialogSubscription`** — Primary export. Manages the full lifecycle of a NATS WebSocket connection and JetStream consumer for a given `dialogId`.
- **Connection lifecycle** — Delegates to `startConnectionLifecycle` from the shared NATS client pool (`acquireClient` / `releaseClient`). Reconnects only on `closed` or `disconnected` states; protocol-level `-ERR` responses are surfaced separately.
- **Sequence tracking** — Tracks the highest observed `streamSequence` in `highestStreamSeqRef`. On reconnect, resumes at `highestStreamSeq + 1` to avoid replaying or skipping chunks.
- **`DeliverPolicy`** — Uses `byStartSequence` when `optStartSeq` or a resume point is known, otherwise falls back to `new` (live-tail).
- **Stable callback refs** — All callbacks (`onEvent`, `onConnect`, `onDisconnect`, `onSubscribed`, `onBeforeReconnect`) and options (`getNatsWsUrl`, `reconnectionBackoff`, `inactiveThresholdMs`) are stored in refs to prevent spurious effect re-runs on inline function identity changes.
- **URL-based reconnect guard** — Resolves `getNatsWsUrl()` synchronously each render so the connection effect depends on the resolved URL string, not the callback identity, avoiding unnecessary teardowns during token rotations.

## Usage Example

```typescript
const { isConnected, isSubscribed, reconnectionCount, currentStreamSeq } =
  useJetStreamDialogSubscription({
    enabled: !!dialogId,
    dialogId: 'dialog-abc-123',
    topic: 'stream',
    optStartSeq: lastKnownSeq ?? null,
    streamName: 'CHAT_CHUNKS',              // optional, default
    inactiveThresholdMs: 5 * 60_000,        // optional, default
    getNatsWsUrl: () => buildWsUrl(token),
    onEvent: (payload, topic) => {
      appendChunk(payload)
    },
    onConnect: () => console.log('NATS connected'),
    onDisconnect: () => console.log('NATS disconnected'),
    onSubscribed: () => console.log('Consumer ready'),
  })
```

## Return Values

| Field | Type | Description |
|---|---|---|
| `isConnected` | `boolean` | WebSocket/NATS connection is live |
| `isSubscribed` | `boolean` | JetStream consumer is active |
| `reconnectionCount` | `number` | Increments on each successful reconnect |
| `currentStreamSeq` | `number \| null` | Highest stream sequence received so far |
