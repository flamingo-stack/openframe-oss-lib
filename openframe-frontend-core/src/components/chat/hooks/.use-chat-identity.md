Resolves and surfaces the current chat session's identity capability bag (`authTier`, `source`, `attachmentsEnabled`, `user`) so client components can gate UI elements without initiating a chat message first.

## Key Components

### Interfaces

- **`ChatIdentityResponse`** — Wire-shape mirroring the hub's `/api/auth/identity` route response; includes `authTier`, `source`, `attachmentsEnabled`, and optional `user` fields (`name`, `email`, `firstName`, `lastName`, `avatarUrl`)
- **`ChatIdentitySurface`** — Extends `ChatIdentityResponse` with `isLoading: boolean`; never throws — errors and loading states fall back to anonymous defaults

### Functions & Components

- **`useResolveChatIdentity(enabled)`** — Internal `useEffect` + `useState` resolver; skips the fetch when `enabled = false` (provider present above); refetches automatically on proxy credential rotation via `proxyEmail` dep
- **`ChatIdentityProvider`** — Optional context provider; resolves identity **once** for its entire subtree; ideal for independent React subtrees (e.g. a persistent chat widget + a tickets router outlet) that cannot share identity via props
- **`useChatIdentity()`** — Public hook; reads from `ChatIdentityProvider` when present, otherwise self-fetches; always returns a `ChatIdentitySurface`, never suspends

## Usage Example

### Standalone hook (default, no provider)

```typescript
import { useChatIdentity } from './use-chat-identity'

function AttachmentButton() {
  const { isLoading, attachmentsEnabled, user } = useChatIdentity()

  if (isLoading) return <Skeleton />
  if (!attachmentsEnabled) return null

  return <button>Attach file ({user?.firstName ?? 'you'})</button>
}
```

### Shared provider for independent subtrees

```typescript
import { ChatIdentityProvider, useChatIdentity } from './use-chat-identity'

// Mount once inside your ChatRuntimeContext.Provider
function EmbedRoot({ children }: { children: ReactNode }) {
  return (
    <ChatRuntimeContext.Provider value={runtime}>
      <ChatIdentityProvider>
        {children}
      </ChatIdentityProvider>
    </ChatRuntimeContext.Provider>
  )
}

// Both components share one /identity fetch
function ChatWidget() {
  const { user } = useChatIdentity() // reads provider
}

function TicketsPage() {
  const { authTier } = useChatIdentity() // same resolution, no duplicate request
}
```

> **No client-side cache by design.** Identity is always fetched fresh to prevent stale-identity flashes across credential rotations. The `ChatIdentityProvider` lifts the resolver to one shared fetch per subtree without persisting state beyond the component lifecycle.
