React hook for managing NATS WebSocket subscriptions scoped to a dialog/chat session, handling connection lifecycle, auto-reconnection, and multi-topic message routing.

## Key Components

### `useNatsDialogSubscription(options)`
Primary hook that manages the full NATS connection and subscription lifecycle for a dialog context.

**Parameters** (`UseNatsDialogSubscriptionOptions`):
- `enabled` — toggles connection/subscription on or off
- `dialogId` — target dialog identifier; subscriptions use the subject pattern `chat.<dialogId>.<topic>`
- `topics` — array of `NatsMessageType` values to subscribe to (default: `['message']`)
- `onEvent(payload, messageType)` — callback fired on each incoming message
- `onConnect / onDisconnect / onSubscribed / onBeforeReconnect` — lifecycle callbacks
- `getNatsWsUrl` — factory returning the resolved WebSocket URL
- `clientConfig` — NATS client credentials (`name`, `user`, `pass`)
- `reconnectionBackoff` — backoff strategy for reconnection attempts

**Returns** (`UseNatsDialogSubscriptionReturn`):
- `isConnected` — current WebSocket connection state
- `isSubscribed` — whether topic subscriptions are active
- `reconnectionCount` — number of reconnections since mount

### `buildNatsWsUrl(apiBaseUrl, options?)`
Utility that constructs a NATS WebSocket URL from a REST API base URL, handling protocol conversion (`https:` → `wss:`) and optional bearer token injection via query param.

## Usage Example

```typescript
import {
  useNatsDialogSubscription,
  buildNatsWsUrl,
} from './use-nats-dialog-subscription'

function ChatWindow({ dialogId, token }: { dialogId: string; token: string }) {
  const { isConnected, isSubscribed, reconnectionCount } =
    useNatsDialogSubscription({
      enabled: true,
      dialogId,
      topics: ['message', 'typing'],
      getNatsWsUrl: () =>
        buildNatsWsUrl('https://api.example.com', {
          token,
          includeAuthParam: true,
        }),
      onEvent: (payload, messageType) => {
        console.log(`[${messageType}]`, payload)
      },
      onConnect: () => console.log('NATS connected'),
      onDisconnect: () => console.log('NATS disconnected'),
    })

  return <div>{isSubscribed ? 'Listening...' : 'Connecting...'}</div>
}
```

> **Note:** The hook resolves `getNatsWsUrl()` synchronously on each render and uses the resulting URL string as an effect dependency — preventing unnecessary reconnections caused by callback identity changes (e.g. during silent token rotation).
