React hook for managing NATS WebSocket subscriptions scoped to a specific dialog, handling connection lifecycle, automatic reconnection, and multi-topic message routing.

## Key Components

### `useNatsDialogSubscription(options)`
Primary hook that manages the full NATS connection and subscription lifecycle for a dialog.

**Parameters (`UseNatsDialogSubscriptionOptions`):**
- `enabled` — toggles the connection on/off
- `dialogId` — the dialog to subscribe to (maps to `chat.<dialogId>.<topic>`)
- `topics` — array of `NatsMessageType` topics (default: `['message']`)
- `onEvent` — callback fired with parsed message payload and topic type
- `onConnect` / `onDisconnect` / `onSubscribed` / `onBeforeReconnect` — lifecycle callbacks
- `getNatsWsUrl` — function returning the WebSocket URL (resolved each render to avoid unnecessary reconnects on callback identity changes)
- `clientConfig` — NATS client credentials (`name`, `user`, `pass`)
- `reconnectionBackoff` — custom backoff strategy

**Returns (`UseNatsDialogSubscriptionReturn`):**
- `isConnected` — current WebSocket connection state
- `isSubscribed` — whether topic subscriptions are active
- `reconnectionCount` — number of reconnections since mount

### `buildNatsWsUrl(apiBaseUrl, options?)`
Utility to construct a NATS WebSocket URL from an HTTP/HTTPS base URL.

**Parameters:**
- `source: 'dashboard'` routes to `/ws/nats-api`, otherwise `/ws/nats`
- `includeAuthParam` + `token` — appends `?authorization=<token>` to the URL

## Usage Example

```typescript
import {
  useNatsDialogSubscription,
  buildNatsWsUrl,
} from './use-nats-dialog-subscription'

function DialogListener({ dialogId, token }: { dialogId: string; token: string }) {
  const { isConnected, isSubscribed, reconnectionCount } = useNatsDialogSubscription({
    enabled: true,
    dialogId,
    topics: ['message', 'typing'],
    getNatsWsUrl: () =>
      buildNatsWsUrl('https://api.example.com', {
        token,
        includeAuthParam: true,
        source: 'dashboard',
      }),
    onEvent: (payload, topic) => {
      console.log(`[${topic}]`, payload)
    },
    onConnect: () => console.log('NATS connected'),
    onDisconnect: () => console.log('NATS disconnected'),
  })

  return (
    <span>
      {isConnected ? '🟢' : '🔴'} Subscribed: {String(isSubscribed)} | Reconnects: {reconnectionCount}
    </span>
  )
}
```

> **Note:** The `getNatsWsUrl` function is resolved on every render and compared by URL string value — not by callback identity — to prevent spurious reconnects during token rotation.
