Skeleton loading components for the page layout, providing animated placeholder UI that mirrors the structure of `AnnouncementBar`, `Header`, `Hero`, `Sidebar`, `Article`, and `Vendor Detail` sections during data fetching.

## Key Components

| Export | Description |
|--------|-------------|
| `AnnouncementBarSkeleton` | Animated placeholder matching the `AnnouncementBar` component (logo, text, close button) |
| `HeaderSkeleton` | Sticky header skeleton with logo, nav links, and action buttons (responsive) |
| `HeroSkeleton` | Full-width hero section with title, subtitle, and CTA button placeholders |
| `SearchContainerSkeleton` | Search input + optional filter chip row |
| `CategorySidebarSkeleton` | 11-item nav skeleton matching `MultiLevelNavigation` (folder/file icons, README badges, chevrons) |
| `BreadcrumbSkeleton` | Horizontal breadcrumb trail with separator dots |
| `ResultsHeaderSkeleton` | Result count / sort bar placeholder |
| `TwoColumnLayoutSkeleton` | Composable two-column grid (sidebar + main); supports `left`/`right` sidebar position |
| `ArticleLayoutSkeleton` | Full article layout: tags, title, metadata, featured image, 12-paragraph body |
| `VendorDetailLayoutSkeleton` | Complete vendor detail page structure with breadcrumb, hero, and content areas |

## Usage Example

```typescript
import {
  HeaderSkeleton,
  TwoColumnLayoutSkeleton,
  CategorySidebarSkeleton,
  ArticleLayoutSkeleton,
} from "@/components/skeletons/page-layout-skeleton"

// Simple page shell while data loads
function LoadingPage() {
  return (
    <>
      <HeaderSkeleton />
      <TwoColumnLayoutSkeleton
        sidebarPosition="left"
        sidebarContent={<CategorySidebarSkeleton />}
      />
    </>
  )
}

// Article page loading state
function LoadingArticle() {
  return <ArticleLayoutSkeleton className="px-4" />
}

// Search page with filters disabled
function LoadingSearch() {
  return <SearchContainerSkeleton showFilters={false} className="my-6" />
}
```

All components accept an optional `className` prop for layout overrides and internally use `UnifiedSkeleton`, `TextSkeleton`, `MediaSkeleton`, and `InteractiveSkeleton` primitives for consistent pulse animations.
