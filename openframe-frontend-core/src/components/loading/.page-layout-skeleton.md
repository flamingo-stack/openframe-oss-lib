Skeleton loading UI components for a full page layout, providing placeholder animations that mirror the structure of announcement bars, headers, heroes, sidebars, article layouts, and vendor detail pages.

## Key Components

| Export | Description |
|--------|-------------|
| `AnnouncementBarSkeleton` | Pulsing banner matching the `AnnouncementBar` structure with logo, text, and close button placeholders |
| `HeaderSkeleton` | Sticky header skeleton with logo, desktop nav buttons, and responsive mobile/desktop action slots |
| `HeroSkeleton` | Centered hero section with title, subtitle, and CTA button placeholders |
| `SearchContainerSkeleton` | Search input + button row with optional filter chip row |
| `CategorySidebarSkeleton` | Sidebar mimicking `MultiLevelNavigation` with folder/file icons, README badges, and chevrons |
| `BreadcrumbSkeleton` | Inline breadcrumb trail with separator dots |
| `ResultsHeaderSkeleton` | Results count / sort header placeholder |
| `TwoColumnLayoutSkeleton` | Configurable two-column grid accepting custom `sidebarContent`, `mainContent`, and `sidebarPosition` |
| `ArticleLayoutSkeleton` | Full article skeleton: tags, title, metadata, featured image, and 12 paragraph blocks |
| `VendorDetailLayoutSkeleton` | Complete vendor detail page structure with breadcrumb, hero, and responsive left/right panels |

## Usage Example

```typescript
import {
  HeaderSkeleton,
  TwoColumnLayoutSkeleton,
  CategorySidebarSkeleton,
  ArticleLayoutSkeleton,
} from "@/components/skeletons/page-layout-skeleton"

// Wrap async page content
export default function DocsPage() {
  return (
    <Suspense fallback={
      <>
        <HeaderSkeleton />
        <TwoColumnLayoutSkeleton
          sidebarPosition="left"
          sidebarContent={<CategorySidebarSkeleton />}
        />
      </>
    }>
      <DocsContent />
    </Suspense>
  )
}

// Article route fallback
export function ArticleLoading() {
  return <ArticleLayoutSkeleton className="px-4 py-8" />
}
```

All components accept an optional `className` prop and use `cn()` for Tailwind class merging. Animations rely on `animate-pulse` and the shared `UnifiedSkeleton`, `TextSkeleton`, `MediaSkeleton`, and `InteractiveSkeleton` primitives.
