A loading skeleton placeholder component for the Margin Report page, rendering animated grey block placeholders that mirror the full report layout while data is being fetched.

## Key Components

### `MarginReportSkeleton`
Main export. Renders a full-page skeleton UI with optional pulse animation and overlay support.

**Props (`MarginReportSkeletonProps`):**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `animate` | `boolean` | `true` | Enables CSS `animate-pulse` on the skeleton blocks |
| `description` | `React.ReactNode` | — | Optional heading text shown inside the overlay |
| `overlayContent` | `React.ReactNode` | — | Optional node (button, spinner, etc.) centered over the skeleton via a fixed overlay |

**Skeleton sections rendered (in order):**
- Page header (title + subtitle bars)
- 3-column summary cards
- MSP Profile + Report Info cards (2-column)
- Commercial Stack & Open-Source Stack vendor lists (5 rows each)
- Strategic Recommendations header + 3-card grid
- OpenFrame value section (header + 3-card grid)

## Usage Example

```typescript
import { MarginReportSkeleton } from './margin-report-skeleton';

// Basic loading state
<MarginReportSkeleton />

// With overlay CTA while report is being generated
<MarginReportSkeleton
  animate={true}
  description="Your margin report is being generated…"
  overlayContent={
    <button onClick={handleGenerateReport}>
      Generate Report
    </button>
  }
/>

// Static snapshot (no animation)
<MarginReportSkeleton animate={false} />
```

> **Note:** The overlay (`overlayContent`) uses `position: fixed` centering so it stays visible regardless of scroll position. The backdrop uses `pointer-events-none` while the inner content restores `pointer-events-auto`, keeping the rest of the page non-interactive.
