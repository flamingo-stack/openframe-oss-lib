A loading skeleton component that provides placeholder UI while organization card data is being fetched, maintaining exact layout dimensions to prevent visual jumps during loading transitions.

## Key Components

- **`OrganizationCardSkeleton`** - Main skeleton component matching OrganizationCard layout with configurable sections
- **`OrganizationCardSkeletonGrid`** - Grid wrapper for displaying multiple skeleton cards in responsive layout
- **`OrganizationCardSkeletonProps`** - TypeScript interface defining component props including visibility toggles

## Usage Example

```typescript
// Basic organization card skeleton
<OrganizationCardSkeleton />

// Skeleton without footer stats
<OrganizationCardSkeleton 
  showFooter={false}
  className="shadow-lg"
/>

// Grid of 6 skeleton cards
<OrganizationCardSkeletonGrid 
  count={6}
  showDescription={false}
  containerClassName="bg-white border-gray-200"
/>

// Custom styled skeleton
<OrganizationCardSkeleton
  containerClassName="bg-blue-50 border-blue-200"
  showFooter={true}
  showDescription={true}
/>
```

The skeleton maintains a 60px organization logo, title/subtitle text areas, optional description section (48px height), and footer with stats/badges. The grid component uses responsive breakpoints: 1 column on mobile, 2 on tablet (md), and 3 on desktop (xl).
