Loading skeleton component that renders a placeholder layout matching the `OrganizationCard` component's exact dimensions and structure, preventing layout shifts during data fetching.

## Key Components

### `OrganizationCardSkeleton`
Single card skeleton with three optional sections:
- **Header** — 60×60px org logo (`OrganizationIconSkeleton`) + title + subtitle text placeholders
- **Description** — Fixed 48px height, 2-line body text placeholders (toggled via `showDescription`)
- **Footer** — Icon+caption stat pairs and a tag/badge placeholder (toggled via `showFooter`)

### `OrganizationCardSkeletonGrid`
Renders `count` skeleton cards in a responsive grid (1 → 2 → 3 columns across mobile/tablet/desktop).

### `OrganizationCardSkeletonProps`
| Prop | Type | Default | Description |
|---|---|---|---|
| `className` | `string` | — | Extra classes on the card wrapper |
| `containerClassName` | `string` | — | Overrides card background/border classes |
| `showFooter` | `boolean` | `true` | Renders the stats/badge footer row |
| `showDescription` | `boolean` | `true` | Renders the description block |

## Usage Example

```typescript
// Single skeleton card
<OrganizationCardSkeleton
  showFooter={true}
  showDescription={false}
  containerClassName="bg-white border border-gray-200"
/>

// Full grid while loading
<OrganizationCardSkeletonGrid
  count={9}
  showFooter={true}
  showDescription={true}
/>
```
