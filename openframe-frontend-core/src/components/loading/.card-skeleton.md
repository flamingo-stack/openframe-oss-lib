A comprehensive card skeleton component system that provides consistent loading states for different card types across the application, including vendor cards, blog posts, categories, and alternative comparisons.

## Key Components

- **CardSkeleton** - Main skeleton component with variant support ('vendor', 'blog', 'category', 'alternative')
- **CardSkeletonGrid** - Grid wrapper for displaying multiple skeleton cards with responsive layouts
- **VendorCardContent** - Matches vendor card structure with logo, title, description, and stats
- **BlogCardContent** - Blog post skeleton with image, title, chips, and metadata
- **CategoryCardContent** - Category card with icon grid and content area
- **AlternativeCardContent** - Simplified layout for comparison/alternative cards

## Usage Example

```typescript
import { CardSkeleton, CardSkeletonGrid } from './card-skeleton'

// Single vendor card skeleton
<CardSkeleton 
  variant="vendor"
  showActions={true}
  showMetadata={true}
  containerClassName="custom-bg"
/>

// Blog posts grid while loading
<CardSkeletonGrid 
  count={6}
  variant="blog"
  className="custom-grid-spacing"
/>

// Category cards without actions
<CardSkeleton 
  variant="category"
  showActions={false}
/>

// Alternative vendor comparison
<CardSkeleton 
  variant="alternative"
  showMetadata={false}
/>
```

The component uses unified skeleton primitives and maintains exact structural parity with actual card components for seamless loading transitions.
