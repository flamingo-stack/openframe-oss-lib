Reusable skeleton loading components for card UI patterns, providing animated placeholder states for vendor, blog, category, and alternative card layouts during data fetching.

## Key Components

**`CardSkeleton`** — Primary export. Renders a single card skeleton matching one of four layout variants (`vendor`, `blog`, `category`, `alternative`). Accepts optional `showActions` and `showMetadata` flags to toggle sub-sections.

**`CardSkeletonGrid`** — Renders a responsive grid of `CardSkeleton` instances. Grid column layout auto-adjusts per variant (e.g. 3-col for vendor, 2-col for blog, 4-col for category).

**Internal content components** (not exported):
- `VendorCardContent` — Logo, title, category, description, stats, and tag badge
- `BlogCardContent` — OG image, title chips, description, author meta, and read-time footer
- `CategoryCardContent` — Icon grid, heading, description, and action button
- `AlternativeCardContent` — Avatar, title, description, and optional action button

All internals use primitives from `unified-skeleton` (`TextSkeleton`, `MediaSkeleton`, `InteractiveSkeleton`).

## Usage Example

```typescript
import { CardSkeleton, CardSkeletonGrid } from "./card-skeleton"

// Single card skeleton
<CardSkeleton variant="vendor" showActions showMetadata />

// Blog card without footer metadata
<CardSkeleton
  variant="blog"
  showMetadata={false}
  className="my-custom-class"
/>

// Grid of 9 category skeletons
<CardSkeletonGrid
  variant="category"
  count={9}
  containerClassName="bg-white border border-gray-200"
/>

// Vendor comparison list (single column)
<CardSkeletonGrid variant="alternative" count={3} />
```

> Both components forward `className` and `containerClassName` for layout/theme overrides without breaking internal structure. The `role="status"` and `aria-label` attributes are set automatically for accessibility.
