Reusable skeleton loading components for card UI elements, providing animated placeholder states that match the structure of vendor, blog, category, and alternative card layouts.

## Key Components

**`CardSkeleton`** — Main export. Renders a single card skeleton with configurable variant and display options. Accepts `role="status"` and `aria-label` for accessibility.

**`CardSkeletonGrid`** — Renders a responsive grid of `CardSkeleton` instances. Grid columns adapt automatically per variant.

**Internal Content Components** (not exported):

| Component | Mirrors |
|---|---|
| `VendorCardContent` | Vendor cards with logo, title, stats, and tag |
| `BlogCardContent` | Blog cards with OG image, chips, description, author, and date |
| `CategoryCardContent` | Category cards with icon grid and action button |
| `AlternativeCardContent` | Comparison list cards with avatar and description |

## Props

```typescript
interface CardSkeletonProps {
  variant?: 'vendor' | 'blog' | 'category' | 'alternative' // default: 'vendor'
  showActions?: boolean    // default: true
  showMetadata?: boolean   // default: true
  className?: string
  containerClassName?: string
}
```

## Usage Example

```typescript
// Single card skeleton
<CardSkeleton variant="blog" showMetadata={true} />

// Grid of vendor card skeletons
<CardSkeletonGrid
  count={6}
  variant="vendor"
  containerClassName="bg-white border border-gray-200"
/>

// Category grid (4-column on large screens)
<CardSkeletonGrid count={4} variant="category" />

// Alternative cards (single column, no actions)
<CardSkeletonGrid count={3} variant="alternative" showActions={false} />
```

> Uses `UnifiedSkeleton` primitives (`TextSkeleton`, `MediaSkeleton`, `InteractiveSkeleton`) to ensure consistent pulse animations across all card types.
