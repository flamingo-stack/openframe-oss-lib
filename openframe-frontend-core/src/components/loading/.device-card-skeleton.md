Loading skeleton component that mimics the exact layout of `DeviceCard` during data fetching, preventing layout shifts with matching dimensions and animated placeholders.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `DeviceCardSkeleton` | Component | Single card skeleton matching DeviceCard's 3-row layout |
| `DeviceCardSkeletonGrid` | Component | Responsive grid of `DeviceCardSkeleton` instances |
| `DeviceCardSkeletonProps` | Interface | Props for `DeviceCardSkeleton` (`className?`) |

**Layout rows matched by `DeviceCardSkeleton`:**
- **Row 1** — Device icon + device name + more button
- **Row 2** — OS badge + organization icon + organization name
- **Row 3** — Status badge + last seen timestamp

**`DeviceCardSkeletonGrid` responsive columns:**
- Mobile: 1 col · Tablet (`md`): 2 cols · Desktop (`lg`/`xl`): 4 cols

## Usage Example

```typescript
// Single skeleton (e.g., while one card is loading)
import { DeviceCardSkeleton } from "./device-card-skeleton"

<DeviceCardSkeleton className="my-custom-class" />

// Grid skeleton (e.g., full page loading state)
import { DeviceCardSkeletonGrid } from "./device-card-skeleton"

// Default: renders 12 skeleton cards
<DeviceCardSkeletonGrid />

// Custom count
<DeviceCardSkeletonGrid count={8} className="p-4" />
```

Both components include `role="status"` and `aria-label` for accessible loading announcements, and use `animate-pulse` on placeholder elements for visual feedback.
