Loading skeleton component that provides placeholder UI while device card data is being fetched, preventing layout shifts by matching the exact structure and dimensions of the actual DeviceCard component.

## Key Components

- **`DeviceCardSkeleton`** - Individual skeleton card matching DeviceCard layout with three rows (device info, OS/organization, status)
- **`DeviceCardSkeletonGrid`** - Grid container for multiple skeleton cards with responsive columns
- **`DeviceCardSkeletonProps`** - Props interface for customizing skeleton appearance
- **`OrganizationIconSkeleton`** - Imported skeleton for organization icons

## Usage Example

```typescript
import { DeviceCardSkeleton, DeviceCardSkeletonGrid } from './device-card-skeleton'

// Single skeleton card
function LoadingSingleDevice() {
  return <DeviceCardSkeleton className="max-w-md" />
}

// Grid of skeleton cards
function LoadingDeviceList() {
  return (
    <DeviceCardSkeletonGrid 
      count={8} 
      className="p-4" 
    />
  )
}

// Conditional loading state
function DeviceList({ isLoading, devices }) {
  if (isLoading) {
    return <DeviceCardSkeletonGrid count={12} />
  }
  
  return <DevicesGrid devices={devices} />
}
```

The skeleton uses CSS animations and semantic HTML with proper ARIA labels for accessibility during loading states.
