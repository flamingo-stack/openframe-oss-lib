Loading skeleton component that renders a placeholder matching the exact layout of `DeviceCard` to prevent layout shifts during data fetching.

## Key Components

### `DeviceCardSkeleton`
Single card skeleton with three rows mirroring the real `DeviceCard` structure:
- **Row 1:** Device icon + device name + more button
- **Row 2:** OS badge + organization icon + organization name
- **Row 3:** Status badge + last seen timestamp

### `DeviceCardSkeletonGrid`
Responsive grid of `DeviceCardSkeleton` instances with the same column breakpoints as `DevicesGrid` (1 → 2 → 4 columns across mobile/tablet/desktop).

### `DeviceCardSkeletonProps`
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | — | Additional CSS classes |

### `DeviceCardSkeletonGrid` Props
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `count` | `number` | `12` | Number of skeleton cards to render |
| `className` | `string` | — | Additional CSS classes |

## Usage Example

```typescript
// Single card skeleton while loading device details
import { DeviceCardSkeleton } from "./device-card-skeleton"

function DeviceCardLoader({ isLoading, device }) {
  if (isLoading) return <DeviceCardSkeleton />
  return <DeviceCard device={device} />
}

// Full grid skeleton while fetching device list
import { DeviceCardSkeletonGrid } from "./device-card-skeleton"

function DevicesPage({ isLoading, devices }) {
  if (isLoading) return <DeviceCardSkeletonGrid count={8} />
  return <DevicesGrid devices={devices} />
}
```

> Both components include `role="status"` and `aria-label` attributes for accessibility, ensuring screen readers announce the loading state correctly.
