A React component that renders device information in a structured card layout with interactive actions and customizable visual elements.

## Key Components

- **Device Interface**: Comprehensive device data model supporting various device types, OS platforms, and metadata
- **ActionButton Interface**: Configuration for custom action buttons with labels and click handlers
- **DeviceCardProps Interface**: Props interface extending HTML div attributes with device data and action configuration
- **DeviceCard Component**: Main card component with four distinct content rows (header, OS/organization, actions, status/tags)
- **formatLastSeen Function**: Internal utility for formatting date/time strings in YYYY/MM/DD, HH:MM format

## Usage Example

```typescript
import { DeviceCard, Device } from './device-card'

const device: Device = {
  id: '123',
  name: 'MacBook Pro',
  type: 'laptop',
  operatingSystem: 'darwin',
  organization: 'Acme Corp',
  status: 'active',
  lastSeen: new Date(),
  tags: ['development', 'remote']
}

<DeviceCard
  device={device}
  actions={{
    moreButton: { visible: true, onClick: () => console.log('More clicked') },
    detailsButton: { visible: true, component: <Button>Details</Button> },
    customActions: [
      { label: 'Connect', onClick: () => connect(device.id) },
      { label: 'Restart', onClick: () => restart(device.id) }
    ]
  }}
  statusBadgeComponent={<StatusBadge status={device.status} />}
  onDeviceClick={(device) => navigate(`/devices/${device.id}`)}
/>
```

The component uses OpenFrame's design system (ODS) classes and supports both legacy OS identifiers and the newer OSPlatformId system for cross-compatibility.
