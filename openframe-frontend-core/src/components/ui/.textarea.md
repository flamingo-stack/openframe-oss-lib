A client-side textarea component that extends the native HTML `<textarea>` with ODS design system styling, optional label/error display, and support for an end icon (static or interactive button).

## Key Components

**`Textarea`** — The primary export; a `React.forwardRef` component with two layout modes:
- **Without `endIcon`**: Renders a styled `<textarea>` directly inside `FieldWrapper` with resizable height and full border/hover/focus/error states.
- **With `endIcon`**: Lifts border and background onto a `<label>` wrapper (mirroring the `Input` component layout), with the textarea as a transparent child and the icon anchored to the bottom-right. Uses CSS `field-sizing: content` for auto-grow behavior.

**`FieldWrapper`** — Wraps both layouts to render the optional `label` above and `error` message below.

**`TextareaProps`** — Interface extending `React.TextareaHTMLAttributes<HTMLTextAreaElement>`:

| Prop | Type | Description |
|------|------|-------------|
| `invalid` | `boolean` | Forces red error border/ring |
| `label` | `string` | Label rendered above the field |
| `error` | `string` | Error message rendered below the field |
| `endIcon` | `React.ReactNode` | Icon rendered at the right edge |
| `endIconAsButton` | `boolean` | Wraps `endIcon` in a styled `<button>` |
| `endIconButtonProps` | `ButtonHTMLAttributes` | Extra props for the end-icon button |

## Usage Example

```typescript
// Basic textarea with label and error
<Textarea
  label="Message"
  placeholder="Type here..."
  error="This field is required"
/>

// Chat-style textarea with interactive send button
import { SendIcon } from "lucide-react"

<Textarea
  label="Message"
  placeholder="Ask anything..."
  endIcon={<SendIcon />}
  endIconAsButton
  endIconButtonProps={{
    "aria-label": "Send message",
    onClick: handleSend,
    disabled: isLoading,
  }}
/>

// Disabled state
<Textarea
  label="Read-only notes"
  value="Cannot edit this"
  disabled
/>
```
