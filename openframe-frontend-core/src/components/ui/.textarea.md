A client-side textarea component that extends the native HTML `<textarea>` with design system styling, optional label/error display, and support for an inline end icon (as a decorative span or interactive button).

## Key Components

**`Textarea`** — `React.forwardRef` component with two internal layouts:
- **Plain layout** (no `endIcon`): border and background live directly on the `<textarea>` element.
- **Wrapper layout** (with `endIcon`): border and background are lifted onto a `<label>` wrapper so the icon renders visually inside the field. Auto-grows via CSS `field-sizing: content` (Chrome 123+, FF 124+, Safari 16.4+), falls back to `rows={1}`.

**`FieldWrapper`** — imported wrapper that renders the `label` and `error` message around the field.

### Props (`TextareaProps`)

| Prop | Type | Description |
|------|------|-------------|
| `invalid` | `boolean` | Renders red error border and ring |
| `label` | `string` | Label text above the textarea |
| `error` | `string` | Error message below the textarea |
| `endIcon` | `ReactNode` | Element rendered at the right edge |
| `endIconAsButton` | `boolean` | Wraps `endIcon` in a styled `<button>` |
| `endIconButtonProps` | `ButtonHTMLAttributes` | Extra props for the end-icon button |

## Usage Example

```typescript
import { Textarea } from './textarea'
import { SendIcon } from 'lucide-react'

// Basic textarea with label and validation
<Textarea
  label="Message"
  placeholder="Type your message..."
  error="Message is required"
/>

// Auto-growing textarea with a submit button icon
<Textarea
  label="Reply"
  placeholder="Write a reply..."
  endIcon={<SendIcon />}
  endIconAsButton
  endIconButtonProps={{
    'aria-label': 'Send reply',
    onClick: handleSend,
    disabled: isSending,
  }}
/>
```
