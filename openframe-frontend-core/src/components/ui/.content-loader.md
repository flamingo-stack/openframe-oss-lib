A unified content loading component that provides skeleton screens for different content types using ODS design tokens. Replaces duplicate loading patterns across the application with consistent, themeable skeletons.

## Key Components

- **`ContentLoader`** - Main component with configurable variants (card, form, detail, list)
- **`ContentLoaderProps`** - TypeScript interface defining component props
- **Variant-specific exports** - `CardLoader`, `FormLoader`, `DetailLoader`, `ListLoader` for convenience

## Usage Example

```typescript
import { ContentLoader, CardLoader, FormLoader } from './content-loader'

// Generic content loader with variant
<ContentLoader 
  variant="card" 
  items={6} 
  showTitle={true}
  className="my-custom-class" 
/>

// Specific loader variants
<CardLoader items={4} showTitle={false} />
<FormLoader items={3} />
<DetailLoader />
<ListLoader items={8} containerClassName="max-w-4xl" />

// Loading state in a component
{isLoading ? (
  <ContentLoader variant="list" items={5} />
) : (
  <DataTable data={data} />
)}
```

The component automatically handles animation, spacing, and theming using CSS classes that follow the ODS design system. Each variant renders appropriate skeleton layouts optimized for different content structures.
