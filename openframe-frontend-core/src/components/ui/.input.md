A client-side React input component built with `forwardRef`, supporting labels, adornments, validation states, loading indicators, and a specialized range slider variant.

## Key Components

- **`Input`** — The main exported component extending `React.InputHTMLAttributes<HTMLInputElement>`
- **`InputProps`** — Interface adding `invalid`, `startAdornment`, `endAdornment`, `label`, `error`, `errorVariant`, and `loading` to standard input props
- **`invalidBorderClasses`** — Tailwind class map for `"error"` (red) and `"warning"` (yellow) border states
- **`FieldWrapper`** — External wrapper component that renders the label and error message

## Behavior Notes

- **Range type** — Renders a clean styled slider (no border/adornment wrapper); still supports `label` via `FieldWrapper`
- **Loading state** — Shows a `Loader2` spinner as the end adornment; suppresses `endAdornment` while active
- **Invalid state** — Triggered by either `invalid={true}` or a non-empty `error` string
- **Autofill override** — Webkit autofill background is neutralized via CSS custom properties

## Usage Example

```typescript
import { Input } from "@/components/ui/input";
import { Search } from "lucide-react";

// Basic labeled input with validation
<Input
  label="Email"
  type="email"
  placeholder="you@example.com"
  error="Invalid email address"
  errorVariant="error"
/>

// Input with adornments and loading state
<Input
  label="Search"
  startAdornment={<Search />}
  loading={isFetching}
  placeholder="Search tickets..."
/>

// Range slider
<Input
  type="range"
  label="Volume"
  min={0}
  max={100}
  defaultValue={50}
/>
```
