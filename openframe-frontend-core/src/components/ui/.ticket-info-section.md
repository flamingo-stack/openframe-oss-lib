A collapsible ticket information panel that displays key ticket metadata (organization, assignee, device, status) in a header row, with an expandable section showing description, attachments, tags, and notes.

## Key Components

- **`TicketInfoSection`** — Main export. Renders the full ticket info panel with collapsed/expanded states.
- **`InfoCell`** — Internal helper component displaying a value/label pair with optional icon and click handler.
- **`TicketInfoSectionProps`** — Full prop interface covering both collapsed header fields and expanded detail fields.

## Usage Example

```typescript
import { TicketInfoSection } from "./ticket-info-section"

<TicketInfoSection
  organization={{ name: "Acme Corp", imageSrc: "/logos/acme.png" }}
  device={{ name: "MacBook Pro", onClick: () => router.push("/devices/123") }}
  status="open"
  statusLabel="In Progress"
  statusColor="#FFC008"
  statusOptions={[
    { id: "open", label: "Open" },
    { id: "closed", label: "Closed" },
  ]}
  onStatusSelect={(id) => updateTicketStatus(id)}
  expanded={isExpanded}
  onExpand={() => setIsExpanded(!isExpanded)}
  assigned={{
    currentAssignee: { id: "u1", name: "Jane Doe", avatarSrc: "/avatars/jane.png" },
    options: assigneeOptions,
    onAssign: (userId) => reassignTicket(userId),
  }}
  createdAt="Jan 15, 2025"
  description="## Issue\nUser cannot log in after password reset."
  attachments={[{ name: "screenshot.png", url: "/files/screenshot.png" }]}
  tags={["urgent", "auth"]}
  notes={ticketNotes}
  onAddNote={(text) => addNote(ticketId, text)}
  onEditNote={(id, text) => editNote(id, text)}
  onDeleteNote={(id) => deleteNote(id)}
/>
```

**Collapsed state** shows only the header row (org, assignee, device, status + expand button).
**Expanded state** additionally renders `createdAt`, markdown `description`, `attachments`, `tags`, and the full notes section with add/edit/delete support.
