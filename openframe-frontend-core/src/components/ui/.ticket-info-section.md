Collapsible ticket information panel component that displays and manages ticket metadata (organization, assignee, device, status) in a compact header row with an optional expanded view for additional details like description, attachments, tags, and notes.

## Key Components

- **`TicketInfoSection`** — Main export; renders the full panel with header and optional expanded content
- **`InfoCell`** — Internal helper; renders a labeled value with optional icon and clickable behavior
- **`TicketInfoSectionProps`** — Full prop interface for the component
- **`TicketAssigneeOption`** — Re-exported type from `AssigneeDropdown`

## Usage Example

```typescript
import { TicketInfoSection } from "./ticket-info-section"

<TicketInfoSection
  organization={{ name: "Acme Corp", imageSrc: "/acme-logo.png" }}
  device={{ name: "DESKTOP-001", onClick: () => navigate("/devices/1") }}
  status="open"
  statusLabel="Open"
  statusColor="#4ECDC4"
  statusOptions={[
    { id: "open", label: "Open" },
    { id: "closed", label: "Closed" },
  ]}
  onStatusSelect={(id) => updateTicketStatus(id)}
  expanded={isExpanded}
  onExpand={() => setIsExpanded(!isExpanded)}
  assigned={{
    currentAssignee: { id: "u1", name: "Jane Doe" },
    options: assigneeList,
    onAssign: (userId) => reassignTicket(userId),
  }}
  createdAt="Jan 15, 2025"
  description="## Issue\nUser cannot log in after password reset."
  attachments={[{ id: "a1", name: "screenshot.png", url: "/files/screenshot.png" }]}
  tags={["vpn", "urgent"]}
  notes={notesList}
  onAddNote={(text) => addNote(text)}
  onEditNote={(id, text) => editNote(id, text)}
  onDeleteNote={(id) => deleteNote(id)}
/>
```

## Behavior Notes

- The header row is always visible; expanded content renders conditionally via the `expanded` prop
- Status tag becomes an interactive dropdown when `statusOptions` is provided
- Assignee section falls back to an "Unassigned" placeholder when `assigned` is not provided
- Description renders via `SimpleMarkdownRenderer` supporting markdown/HTML content
