A client-side React component that renders Markdown content safely with LLM/chat output in mind, combining `react-markdown`, GFM, syntax highlighting, and a layered XSS sanitization pipeline.

## Key Components

### `rehypeStripUnsafe()`
A custom rehype plugin (runs post `rehype-raw`) that strips dangerous HAST nodes and attributes:
- Drops `<script>`, `<style>`, `<noscript>`, `<object>`, and related elements
- Removes `on*` event handler attributes
- Blocks `javascript:` and `data:` URI schemes on URL-bearing attributes (`href`, `src`, `srcset`, `formaction`, etc.)
- Handles multi-candidate `srcset` strings via `srcsetHasUnsafeCandidate()`
- Neutralizes root-level strip elements in-place

### `cardAwareUrlTransform(url, key)`
Extends `react-markdown`'s default URL transformer to allow the internal `card://` scheme (used for inline chat-card markers) while keeping all other dangerous schemes (`javascript:`, `data:`, etc.) blocked.

### `escapeUnknownHtmlTags(text)`
Pre-processes raw markdown text to escape non-allowlisted HTML tags before they reach `rehype-raw`, preventing LLM-emitted pseudo-tags (`<their>`, `<ticket>`) from reaching React as unrecognized custom elements. Uses a ReDoS-hardened, bounded regex (`TAG_LIKE_REGEX`) with hard limits on tag name length (63 chars) and attribute length (4096 chars).

### `SAFE_HTML_TAGS`
An allowlist of permitted HTML5 tags forwarded as-is through the pipeline (block/inline text, media, forms).

## Usage Example

```typescript
import { escapeUnknownHtmlTags } from './simple-markdown-renderer';

// Pre-process LLM output before rendering
const safeMarkdown = escapeUnknownHtmlTags(llmOutput);

// The component internally applies:
// 1. escapeUnknownHtmlTags (text pre-pass)
// 2. rehype-raw (parse inline HTML)
// 3. rehypeStripUnsafe (HAST sanitizer)
// 4. rehype-highlight (syntax coloring)
```

> **Security note:** `rehypeStripUnsafe` is the authoritative security boundary. `escapeUnknownHtmlTags` is a best-effort pre-pass for React compatibility, not a security control.
