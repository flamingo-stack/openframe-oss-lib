A client-side React component that renders Markdown content safely, combining `react-markdown` with GFM/breaks plugins, custom XSS sanitization, and LLM-output normalization for use in AI chat interfaces.

## Key Components

### `rehypeStripUnsafe()`
A custom rehype plugin that runs post-`rehype-raw` to eliminate XSS vectors from parsed HTML:
- Strips dangerous elements (`<script>`, `<style>`, `<noscript>`, `<object>`, etc.)
- Removes `on*` event handler attributes from any element
- Blocks `javascript:` and `data:` URI schemes on URL-bearing attributes (`href`, `src`, `srcset`, `formaction`, etc.)
- Handles multi-candidate `srcset` values via `srcsetHasUnsafeCandidate()`

### `srcsetHasUnsafeCandidate(srcset: string): boolean`
Splits `srcset` attribute strings by comma and inspects each URL candidate individually, preventing bypass via a safe first candidate followed by a malicious second.

### `cardAwareUrlTransform(url: string, key: string): string`
Extends `react-markdown`'s default URL sanitizer to allow the internal `card://` scheme (used for inline chat-card markers) on `href` attributes only. All other URL schemes still pass through the default sanitizer.

### `escapeUnknownHtmlTags(text: string): string`
Pre-processes raw markdown text to escape HTML-style tags not in the `SAFE_HTML_TAGS` allowlist, preventing LLM-emitted pseudo-XML (e.g., `<ticket>`, `<their>`) from reaching React as unrecognized custom elements. Skips fenced and inline code spans.

### `SAFE_HTML_TAGS`
A `Set<string>` of all permitted HTML5 tag names forwarded as-is to React. Covers standard block/inline, media, and form elements.

## Usage Example

```typescript
import { SimpleMarkdownRenderer } from './simple-markdown-renderer';

// Basic usage in a chat message component
function ChatMessage({ content }: { content: string }) {
  return (
    <SimpleMarkdownRenderer
      content={content}
      className="prose dark:prose-invert"
    />
  );
}

// LLM output with raw HTML is sanitized automatically:
const llmOutput = `
## Summary

Click <a href="javascript:alert(1)">here</a> — blocked by sanitizer.

The <ticket> element is stripped as unknown tag.

\`\`\`typescript
const x: Array<string> = [];
\`\`\`
`;
```

> **Security model:** `escapeUnknownHtmlTags` handles accidental LLM tag emissions at the text level; `rehypeStripUnsafe` is the authoritative XSS boundary at the HAST level — safe even if the text pre-pass is bypassed.
