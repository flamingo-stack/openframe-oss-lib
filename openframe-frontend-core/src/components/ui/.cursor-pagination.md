React components for cursor-based pagination navigation, supporting both full-featured and minimal icon-only variants.

## Key Components

### `CursorPagination`
Full-featured pagination bar with optional info text, reset/previous/next navigation, and compact mode support.

### `CursorPaginationSimple`
Minimal icon-only pagination using transparent icon buttons — ideal for tight layouts like table toolbars.

### `CursorPaginationProps`
Full interface controlling cursor values, page state, counts, labels, and callbacks.

## Usage Example

```typescript
// Full pagination bar with count info and reset
<CursorPagination
  hasNextPage={true}
  hasPreviousPage={false}
  isFirstPage={true}
  endCursor="cursor_abc123"
  currentCount={25}
  totalCount={200}
  itemName="tickets"
  onNext={(cursor) => fetchPage({ after: cursor })}
  onReset={() => fetchPage({})}
  resetButtonIcon="home"
  showInfo={true}
  compact={false}
/>

// Compact mode — labels hidden on mobile, visible md+
<CursorPagination
  hasNextPage={true}
  endCursor="cursor_xyz"
  onNext={(cursor) => loadNext(cursor)}
  compact={true}
/>

// Minimal icon-only variant for table headers
<CursorPaginationSimple
  hasNextPage={true}
  hasPreviousPage={false}
  isFirstPage={true}
  onNext={() => fetchNext()}
  onReset={() => resetPagination()}
  loading={isFetching}
/>
```

## Notes

- When `onReset` is provided, it replaces the **Previous** button with a **First**/**Home** button, disabled when already on the first page.
- `getDisplayText()` renders three display modes: exact count (`25 of 200`), unbounded (`25+`), or final page count (`25 items`).
- `CursorPaginationSimple` passes an empty string to `onNext`/`onPrevious` — callers must manage cursor state externally.
- Both components accept a `loading` prop that disables all buttons during data fetching.
