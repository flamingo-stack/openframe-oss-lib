A React component that renders contextually appropriate file and folder icons based on file types and extensions using Lucide React icons.

## Key Components

- **`FileIcon`** - Main component that displays file/folder icons with size and styling options
- **`getFileIcon`** - Utility function that maps file extensions to appropriate Lucide icons
- **`sizeMap`** - Configuration object defining icon sizes (sm, md, lg)
- **Icon categories** - Document, image, video, audio, code, and archive file types supported

## Usage Example

```typescript
import { FileIcon } from './file-icon'

// Folder icon
<FileIcon type="folder" />

// File icons based on extension
<FileIcon extension="pdf" size="lg" />
<FileIcon extension="js" className="text-blue-500" />
<FileIcon extension="mp4" size="sm" />

// Generic file icon (no extension)
<FileIcon />
```

The component automatically applies appropriate colors (accent for folders, secondary text for files) and supports custom styling through the `className` prop. File types are intelligently categorized - for example, `.js`, `.ts`, `.py` files use the code icon, while `.mp4`, `.avi` files use the video icon.
