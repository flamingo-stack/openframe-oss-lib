A loading placeholder component that displays skeleton screens while file manager content loads, mimicking the layout and structure of a complete file management interface.

## Key Components

- **FileManagerSkeleton** - Main component that renders animated skeleton placeholders
- **FileManagerSkeletonProps** - Interface defining customization options:
  - `rows` - Number of file rows to display (default: 8)
  - `showSearch` - Whether to show search bar skeleton (default: true)
  - `showActions` - Whether to show action buttons skeleton (default: true)

The component renders skeleton placeholders for:
- Breadcrumb navigation with separator icons
- Action button toolbar (upload, create folder, etc.)
- Search bar
- Table header with column labels (NAME, SIZE, EDITED)
- File/folder rows with checkboxes, icons, names, metadata, and action buttons

## Usage Example

```typescript
import { FileManagerSkeleton } from './components/ui/file-manager-skeleton'

// Basic usage with defaults
function LoadingFileManager() {
  return <FileManagerSkeleton />
}

// Customized skeleton
function CustomLoadingFileManager() {
  return (
    <FileManagerSkeleton 
      rows={12}
      showSearch={false}
      showActions={true}
    />
  )
}

// Conditional loading state
function FileManagerView({ isLoading, files }) {
  if (isLoading) {
    return <FileManagerSkeleton rows={files?.length || 8} />
  }
  
  return <FileManager files={files} />
}
```
