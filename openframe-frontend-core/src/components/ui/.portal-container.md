Provides a React context and hook for controlling where Radix UI overlay portals (dropdowns, tooltips, etc.) render in the DOM, enabling stacking context isolation within nested surfaces like drawers or modals.

## Key Components

- **`PortalContainerContext`** — React context holding an `HTMLElement | null` value; `null` defaults to Radix's standard `document.body` portal behavior
- **`usePortalContainer()`** — Convenience hook that reads the active portal container from context

## Usage Example

```typescript
// Wrap a high-z surface (e.g. chat drawer) with a ref-bound provider
// so all Radix overlays inside portal into this node instead of document.body

function ChatDrawer() {
  const [portalNode, setPortalNode] = React.useState<HTMLElement | null>(null)

  return (
    <div ref={setPortalNode} className="drawer">
      <PortalContainerContext.Provider value={portalNode}>
        <DrawerContent />
      </PortalContainerContext.Provider>
    </div>
  )
}

// Inside any child component, pass the container to a Radix portal prop
function DrawerContent() {
  const portalContainer = usePortalContainer()

  return (
    <DropdownMenu.Portal container={portalContainer}>
      <DropdownMenu.Content>...</DropdownMenu.Content>
    </DropdownMenu.Portal>
  )
}
```

## Notes

- Radix uses `position: fixed` for portaled content, so it still escapes `overflow: hidden` clipping regardless of the portal target
- When no `PortalContainerContext.Provider` is present, `usePortalContainer()` returns `null` and Radix falls back to `document.body` — safe to use unconditionally
