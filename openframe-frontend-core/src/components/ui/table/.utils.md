Utility functions for managing responsive table column visibility using Tailwind CSS breakpoints.

## Key Components

- **`getHideClasses()`** - Generates Tailwind CSS classes to hide columns at specific breakpoints
- **`isHiddenOnMobile()`** - Determines if a column should be hidden on mobile devices

## Usage Example

```typescript
import { getHideClasses, isHiddenOnMobile } from './utils'
import type { TableColumn } from './types'

// Generate hide classes for responsive design
const hideClasses = getHideClasses('md') // Returns: 'hidden md:flex'
const multiBreakpointClasses = getHideClasses(['sm', 'md']) // Returns: 'sm:hidden md:hidden lg:flex'

// Check if column should be hidden on mobile
const column: TableColumn<User> = {
  key: 'email',
  title: 'Email',
  hideAt: ['sm', 'md']
}

const shouldHideOnMobile = isHiddenOnMobile(column) // Returns: true

// Apply classes to table cells
<td className={getHideClasses(column.hideAt)}>
  {row.email}
</td>
```

The `getHideClasses()` function handles both single breakpoints and arrays, automatically calculating the appropriate show/hide pattern. For single breakpoints, it hides below and shows at/above the specified breakpoint. For multiple breakpoints, it hides at each specified breakpoint and shows after the largest one. The `isHiddenOnMobile()` function specifically checks for `sm` and `md` breakpoints to filter mobile-hidden columns.
