A comprehensive TypeScript type definition file for a flexible table component system, providing interfaces for data display, pagination, filtering, and user interactions.

## Key Components

**Core Types:**
- `TableColumn<T>` - Column configuration with sorting, filtering, custom rendering, and responsive visibility
- `TableProps<T>` - Main table component props with data, styling, and interaction handlers
- `CursorPagination` & `PagePagination` - Two pagination strategies for different data loading patterns
- `TableFilters` - Filter state management with column-based filter values

**Action Types:**
- `BulkAction<T>` - Actions that operate on multiple selected rows
- `RowAction<T>` - Actions for individual table rows
- `FilterOption` & `FilterSection` - Filter UI configuration

**Component Props:**
- `TableHeaderProps`, `TableRowProps`, `TableCellProps` - Individual table element configurations
- `TableCardSkeletonProps` - Loading state skeleton display
- `TableEmptyStateProps` - Empty state with optional call-to-action

## Usage Example

```typescript
interface User {
  id: string
  name: string
  email: string
  status: 'active' | 'inactive'
}

const userColumns: TableColumn<User>[] = [
  {
    key: 'name',
    label: 'Name',
    sortable: true,
    width: 'w-48'
  },
  {
    key: 'email',
    label: 'Email',
    hideAt: 'sm',
    renderCell: (user) => <a href={`mailto:${user.email}`}>{user.email}</a>
  },
  {
    key: 'status',
    label: 'Status',
    filterable: true,
    filterOptions: [
      { id: 'active', label: 'Active', value: 'active' },
      { id: 'inactive', label: 'Inactive', value: 'inactive' }
    ]
  }
]

const tableProps: TableProps<User> = {
  data: users,
  columns: userColumns,
  rowKey: 'id',
  selectable: true,
  cursorPagination: {
    hasNextPage: true,
    onNext: (cursor) => loadMore(cursor)
  }
}
```
