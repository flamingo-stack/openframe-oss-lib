Renders a styled ticket status badge (`Tag`) with support for any status string format, custom hex colors, and an optional inline status-change dropdown.

## Key Components

### Types & Interfaces
- **`TicketStatus`** — Canonical union type: `'ACTIVE' | 'AI_ASSISTANCE' | 'TECH_REQUIRED' | 'ON_HOLD' | 'RESOLVED' | 'ARCHIVED'`
- **`TicketStatusKind`** — Backend status-definition kinds including `'CUSTOM'`
- **`TicketStatusInput`** — Input shape combining legacy `status`, `statusKind`, `statusName`, and `statusColor`
- **`TicketStatusTagProps`** — Full component props including `color`, `options`, and `onSelect`

### Utility Functions
- **`resolveTicketStatus(status)`** — Normalizes any alias (backend, chat, HubSpot) to a canonical `TicketStatus` or `null`
- **`getTicketStatusConfig(status)`** — Returns `{ label, variant, icon }` for a status string
- **`getTicketStatusTag(status)`** — Drop-in utility returning label/variant/icon for use with `<Tag>`
- **`kindToCanonicalStatus(kind)`** — Maps backend `TicketStatusKind` to a canonical status
- **`usesCanonicalStatusStyle(kind)`** — Returns `true` when the kind uses system styling over a custom color
- **`resolveStatusTagProps(input)`** — Maps `TicketStatusInput` fields to `TicketStatusTag` props

### Component
- **`TicketStatusTag`** — Primary component; renders a `<Tag>` or a dropdown-enabled status-changer button

## Usage Example

```typescript
// Basic status badge
<TicketStatusTag status="ACTIVE" />

// With custom hex color and label override
<TicketStatusTag status="TECH_REQUIRED" color="#7C3AED" label="Escalated" />

// Inline status-change dropdown
<TicketStatusTag
  status="ON_HOLD"
  options={[
    { id: 'status_1', name: 'Active', color: '#22c55e' },
    { id: 'status_2', name: 'Resolved', color: '#6b7280' },
  ]}
  onSelect={(id) => updateTicketStatus(id)}
  isPending={isMutating}
/>

// From backend lifecycle fields
const tagProps = resolveStatusTagProps({
  statusKind: 'CUSTOM',
  statusName: 'Awaiting Parts',
  statusColor: '#f59e0b',
})
<TicketStatusTag {...tagProps} />
```
