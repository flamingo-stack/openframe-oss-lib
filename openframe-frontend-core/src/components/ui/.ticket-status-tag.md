Renders a styled ticket status badge (`Tag`) that normalizes any status string format (backend enum, chat lowercase, HubSpot canonical, dialog-layer alias) to a canonical display with the correct color variant, icon, and optional inline status-change dropdown.

## Key Components

### Types & Exports
- **`TicketStatus`** — Canonical union type: `'ACTIVE' | 'AI_ASSISTANCE' | 'TECH_REQUIRED' | 'ON_HOLD' | 'RESOLVED' | 'ARCHIVED'`
- **`TicketStatusKind`** — Backend status-definition kinds including `'CUSTOM'`
- **`TicketStatusInput`** — Unified input shape combining legacy `status`, `statusKind`, `statusName`, and `statusColor`
- **`TicketStatusTagOption`** — Selectable option shape for the inline dropdown (`id`, `name`, `color`)
- **`TicketStatusTagProps`** — Full component prop interface

### Utility Functions
- **`resolveTicketStatus(status)`** — Resolves any string alias to a canonical `TicketStatus` or `null`
- **`getTicketStatusConfig(status)`** — Returns `{ label, variant, icon }` for any known status format
- **`getTicketStatusTag(status)`** — Drop-in utility returning `{ label, variant, icon }` for use with `<Tag>`
- **`kindToCanonicalStatus(kind)`** — Maps backend kind to canonical status; returns `undefined` for `CUSTOM`
- **`usesCanonicalStatusStyle(kind)`** — Returns `true` for kinds that use system styling (not custom color)
- **`resolveStatusTagProps(input)`** — Maps `TicketStatusInput` fields to `TicketStatusTag` props

### Component
- **`TicketStatusTag`** — Main component; renders a `Tag` or an `ActionsMenuDropdown`-wrapped button with chevron when `options` + `onSelect` are provided

## Usage Example

```typescript
// Basic status tag
<TicketStatusTag status="ACTIVE" />

// With legacy alias formats
<TicketStatusTag status="action_required" />  // → "Tech Required" warning tag
<TicketStatusTag status="OPEN" />              // → "Active" success tag

// Custom color override (e.g. HubSpot pipeline stage)
<TicketStatusTag
  status="OPEN"
  label="Waiting on Contact"
  color="#7C3AED"
/>

// Inline status changer dropdown
<TicketStatusTag
  status={ticket.status}
  options={statusOptions}
  onSelect={(id) => updateStatus(ticket.id, id)}
  isPending={isUpdating}
/>

// From lifecycle (custom-status) backend fields
const props = resolveStatusTagProps({
  statusKind: ticket.statusKind,
  statusName: ticket.statusName,
  statusColor: ticket.statusColor,
})
<TicketStatusTag {...props} />
```
