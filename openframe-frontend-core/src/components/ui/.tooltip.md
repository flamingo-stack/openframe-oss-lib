Accessible tooltip component built on Radix UI primitives, with portal-aware rendering that inherits the correct stacking context from parent containers like drawers.

## Key Components

| Export | Type | Description |
|--------|------|-------------|
| `TooltipProvider` | Component | Radix `Provider` wrapper; wrap your app or section to enable tooltip functionality |
| `Tooltip` | Component | Radix `Root`; manages open/close state |
| `TooltipTrigger` | Component | Radix `Trigger`; the element that activates the tooltip on hover/focus |
| `TooltipContent` | Component | Custom `forwardRef` content panel; portals into the active container via `usePortalContainer`, falls back to `document.body` |

## Key Behaviors

- **Portal container awareness** — uses `usePortalContainer()` to render inside the nearest active container (e.g. a drawer), preventing z-index conflicts
- **Animated transitions** — enter/exit animations via Tailwind `animate-in`/`animate-out` with directional slide offsets per `data-[side=*]`
- **Default `sideOffset`** — set to `4px` for consistent spacing out of the box

## Usage Example

```typescript
import {
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip"

export function SaveButton() {
  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <button>💾</button>
        </TooltipTrigger>
        <TooltipContent side="top">
          Save changes
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}
```

> `TooltipProvider` should wrap the root of your app or layout once, rather than each individual tooltip instance.
