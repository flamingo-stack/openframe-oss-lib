Radix UI-based tooltip component with portal support for correct stacking context inside drawers and overlays.

## Key Components

| Export | Description |
|--------|-------------|
| `TooltipProvider` | Wraps the app/section to enable tooltip functionality (re-export of `TooltipPrimitive.Provider`) |
| `Tooltip` | Root tooltip component managing open/close state |
| `TooltipTrigger` | Element that activates the tooltip on hover/focus |
| `TooltipContent` | Styled tooltip bubble with animated enter/exit transitions, portaled into the active stacking context |

## Usage Example

```typescript
import {
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip"

export function ActionButton() {
  return (
    <TooltipProvider>
      <Tooltip>
        <TooltipTrigger asChild>
          <button>Hover me</button>
        </TooltipTrigger>
        <TooltipContent>
          <p>Helpful description</p>
        </TooltipContent>
      </Tooltip>
    </TooltipProvider>
  )
}
```

## Notes

- `TooltipContent` uses `usePortalContainer` to portal into the nearest active container (e.g. a drawer), falling back to `document.body` — preventing z-index clipping issues in nested overlays.
- Default `sideOffset` is `4px`.
- Animations are data-attribute driven (`data-[state]`, `data-[side]`) supporting all four placement directions.
- Styling uses ODS design tokens (`ods-card`, `ods-border`, `ods-text-primary`) for consistent theming.
