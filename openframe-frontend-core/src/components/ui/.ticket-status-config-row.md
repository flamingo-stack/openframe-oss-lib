A configurable row component for managing individual ticket status entries in a settings/configuration UI, supporting both system-defined (read-only) and custom (editable, draggable) status variants.

## Key Components

### `TicketStatusConfigRow`
Main exported component rendering a full-width row with:
- **Drag handle** — active grab button for custom rows; disabled ghost icon for system rows
- **Name input** — editable for custom rows, read-only for system rows
- **Color controls** — `ColorPresetSelect` + `ColorPickerInput` for custom rows only
- **Live preview** — `TicketStatusTag` reflecting current name/color in real time
- **Action slot** — info tooltip icon (system) or delete button with optional disabled-reason tooltip (custom)

### `TicketStatusConfigRowProps`
Full interface controlling row behavior. Key props:

| Prop | Type | Purpose |
|------|------|---------|
| `variant` | `'system' \| 'custom'` | Switches between read-only and editable modes |
| `statusKey` | `string` | Canonical key forwarded to `TicketStatusTag` |
| `onColorChange` | `(next: { color: string; preset?: string }) => void` | Color/preset change callback |
| `dragHandleProps` / `dragHandleAttributes` | dnd-kit types | Wired to a `@dnd-kit` sortable context |
| `deleteDisabledReason` | `string` | Tooltip shown when delete is blocked |

### `WithLeftTooltip` (internal)
Utility wrapper that conditionally wraps children in a left-aligned `TooltipProvider`; renders children directly when no content is provided.

## Usage Example

```typescript
<TicketStatusConfigRow
  variant="custom"
  statusKey="IN_PROGRESS"
  name={statusName}
  onNameChange={setStatusName}
  color="#4ECDC4"
  presetKey={undefined}
  onColorChange={({ color, preset }) => handleColorUpdate(color, preset)}
  onDelete={handleDelete}
  deleteDisabled={isLastStatus}
  deleteDisabledReason="At least one custom status is required"
  dragHandleProps={listeners}
  dragHandleAttributes={attributes}
  isDragging={isDragging}
/>
```
