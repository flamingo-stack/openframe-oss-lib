A React component for testing the adaptive `ods-current` color token across different platform types (OpenFrame, Flamingo, TMCG). It demonstrates how the token renders different colors based on the active platform context.

## Key Components

- **OdsCurrentTest**: Main test component that renders multiple visual tests
- **Platform-specific color mapping**: Yellow for OpenFrame, Pink for Flamingo/TMCG, White for default
- **Test scenarios**: Text color, background color, border color, SVG fill, and direct CSS variable usage

## Usage Example

```typescript
import { OdsCurrentTest } from './ods-current-test'

// Render the test component to verify ods-current token behavior
function App() {
  return (
    <div data-app-type="openframe"> {/* or "flamingo", "tmcg" */}
      <OdsCurrentTest />
    </div>
  )
}

// The component will automatically adapt colors based on the data-app-type:
// - OpenFrame: #ffc008 (yellow)
// - Flamingo/TMCG: #f357bb (pink) 
// - Default: #fafafa (white)
```

The component provides comprehensive visual testing for the `ods-current` token across different CSS properties (text, background, border, fill) and includes reference colors for manual verification. It's designed to work with a CSS-in-JS or CSS custom properties system that responds to platform context.
