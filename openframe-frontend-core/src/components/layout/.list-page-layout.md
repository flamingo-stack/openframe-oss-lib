A standardized React layout component that provides consistent structure and spacing for all list-based pages throughout the OpenFrame application, featuring integrated search functionality, header actions, and error handling.

## Key Components

- **`ListPageLayout`** - Main layout component with title, search bar, and content areas
- **`ListPageLayoutProps`** - TypeScript interface defining all component props including styling options
- Built-in integration with `ListPageContainer`, `SearchBar`, and `PageError` components

## Usage Example

```typescript
import { ListPageLayout } from './list-page-layout'
import { Button, Table } from '../ui'

function DevicesPage() {
  const [searchTerm, setSearchTerm] = useState('')
  const [error, setError] = useState<string | null>(null)
  
  return (
    <ListPageLayout
      title="Devices"
      headerActions={
        <div className="flex gap-2">
          <Button onClick={handleRefresh}>Refresh</Button>
          <Button variant="primary">Add Device</Button>
        </div>
      }
      searchPlaceholder="Search devices..."
      searchValue={searchTerm}
      onSearch={setSearchTerm}
      error={error}
      padding="md"
      background="card"
    >
      <Table data={filteredDevices} columns={deviceColumns} />
    </ListPageLayout>
  )
}
```

The component provides a three-section layout: title bar with actions, full-width search bar, and main content area. It includes error state handling, responsive design, and customizable styling options for consistent UI patterns across device management, logs, scripts, and policy pages.
