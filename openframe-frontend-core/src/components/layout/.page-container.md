A React client component that provides a unified page container with both legacy layout support and modern UI patterns for consistent page structure across the application.

## Key Components

- **PageContainer**: Main component that automatically detects and renders either legacy or advanced page layouts
- **LegacyPageContainerProps**: Interface for backward-compatible layout patterns with background and padding controls
- **AdvancedPageContainerProps**: Interface for modern UI patterns with built-in header, actions, and responsive behavior
- **isAdvancedProps**: Type guard function to differentiate between legacy and advanced usage patterns
- **renderLegacyPageContainer**: Implementation for simple layout containers with customizable backgrounds
- **renderAdvancedPageContainer**: Implementation for feature-rich pages with titles, actions, and variant-specific layouts

## Usage Example

```typescript
// Legacy usage (backward compatible)
<PageContainer 
  backgroundClassName="bg-gray-100"
  maxWidth="max-w-6xl"
>
  <h1>Your Content</h1>
</PageContainer>

// Advanced usage (modern UI patterns)
<PageContainer 
  variant="detail"
  title="User Profile"
  subtitle="Manage user information and settings"
  backButton={{ onClick: () => router.back() }}
  actions={[
    { label: 'Edit', onClick: handleEdit },
    { label: 'Delete', onClick: handleDelete, variant: 'destructive' }
  ]}
  padding="md"
  background="card"
>
  <UserProfileForm />
</PageContainer>

// List page with icon-based actions
<PageContainer
  variant="list" 
  title="Team Members"
  actions={[
    { icon: <Plus />, onClick: handleAdd, tooltip: 'Add Member' },
    { icon: <Filter />, onClick: showFilters, tooltip: 'Filter' }
  ]}
>
  <TeamMembersList />
</PageContainer>
```

The component automatically adapts action button styles based on the variant: list pages use collapsible icon buttons, while detail/form pages use fixed bottom primary buttons on mobile devices.
