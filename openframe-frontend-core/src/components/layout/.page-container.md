A unified page container component that supports both legacy layout patterns and advanced UI patterns with responsive headers, actions, and multiple variants.

## Key Components

- **PageContainer** - Main component with dual interface support
- **LegacyPageContainerProps** - Backward-compatible layout interface
- **AdvancedPageContainerProps** - Feature-rich UI interface with variants
- **Type guards** - `isAdvancedProps()` to determine interface usage
- **Render functions** - Separate implementations for legacy and advanced modes

## Usage Example

**Legacy Usage (Layout Mode):**
```typescript
<PageContainer 
  backgroundClassName="bg-gray-100"
  maxWidth="max-w-6xl"
  contentPadding="p-6"
>
  <h1>Your Content</h1>
</PageContainer>
```

**Advanced Usage (UI Mode):**
```typescript
<PageContainer 
  variant="detail"
  title="User Details"
  subtitle="Manage user account settings"
  backButton={{ 
    label: "Back to Users", 
    onClick: () => router.back() 
  }}
  actions={[
    { label: "Edit", onClick: handleEdit },
    { label: "Delete", onClick: handleDelete, variant: "destructive" }
  ]}
>
  <UserForm />
</PageContainer>
```

**Available Variants:**
- `list` - For data tables with icon-button actions
- `detail` - For detail views with primary-button actions
- `form` - For forms with larger spacing
- `content` - Default content layout

The component automatically handles responsive layouts, mobile-fixed action buttons, and proper spacing based on the selected variant.
