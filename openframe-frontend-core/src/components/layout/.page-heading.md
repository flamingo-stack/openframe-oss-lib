Renders a consistent page-level heading (`<h1>` or `<h2>`) with an optional description paragraph, acting as the single source of truth for the ODS `text-h1` typography token across all pages.

## Key Components

- **`PAGE_HEADING_CLASS`** — Exported constant (`'text-h1 text-ods-text-primary'`) for the rare cases where the heading markup must be applied without the component.
- **`PageHeadingProps`** — Interface defining `children`, `as`, `description`, `className`, and `descriptionClassName`.
- **`PageHeading`** — Primary component that renders the heading tag and an optional `<p>` description, safely excluding falsy ReactNode values (`false`, `''`, `null`) to prevent phantom vertical gaps.

## Usage Example

```typescript
// Standard h1 with description
<PageHeading description="Manage and monitor all connected devices.">
  Devices
</PageHeading>

// h2 variant when an h1 already exists above (e.g. hero section)
<PageHeading as="h2" className="mt-8">
  Related Items
</PageHeading>

// With accent span and custom description class
<PageHeading
  description={<>New in <span className="text-amber-400">v2.0</span></>}
  descriptionClassName="max-w-[480px]"
>
  What&apos;s New
</PageHeading>

// Using the exported class directly (non-h1 edge case)
<span className={PAGE_HEADING_CLASS}>Custom Heading</span>
```

## Notes

- Always prefer `<PageHeading>` over hardcoding `text-h1` — the component is the single source of truth for page-title sizing.
- Default tag is `h1`; use `as="h2"` only when a semantic `h1` already exists higher in the page (e.g. a hero component).
- Layout concerns (margins, containers) belong to the caller; this component owns only the heading and description styles.
