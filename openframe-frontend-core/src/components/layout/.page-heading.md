Renders a consistent page-level heading (`<h1>` or `<h2>`) with an optional description paragraph, serving as the single source of truth for the ODS `text-h1` typographic token across all pages.

## Key Components

- **`PAGE_HEADING_CLASS`** — Exported constant (`'text-h1 text-ods-text-primary'`) for rare cases where the heading element cannot be used directly
- **`PageHeadingProps`** — Interface defining `children`, `as`, `description`, `className`, and `descriptionClassName`
- **`PageHeading`** — Primary component; renders the heading tag and an optional `<p>` description, safely excluding falsy `ReactNode` values (empty string, `false`) to prevent phantom spacing

## Usage Example

```typescript
// Standard h1 with description
<PageHeading description="Manage all your connected devices from one place.">
  Devices
</PageHeading>

// Pass 'h2' when a hero h1 already exists above
<PageHeading as="h2" className="mt-8">
  Recent Activity
</PageHeading>

// With an accent span in the heading
<PageHeading description={<>Learn more at <a href="/docs">docs</a></>}>
  Welcome to <span className="text-amber-400">OpenFrame</span>
</PageHeading>

// Using PAGE_HEADING_CLASS directly (non-component caller)
<div className={PAGE_HEADING_CLASS}>Custom Heading</div>
```

## Notes

- **Do not** hardcode `text-h1` sizes or letter-spacing elsewhere — always use `<PageHeading>` or `PAGE_HEADING_CLASS`
- Layout concerns (margins, container widths) belong to the caller; this component owns only the heading and description styling
- The `description` guard handles `cond && 'text'` patterns that may evaluate to `false`, preventing empty `<p>` elements from adding unwanted vertical gap
