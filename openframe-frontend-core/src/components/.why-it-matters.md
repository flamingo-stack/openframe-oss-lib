A React client component that renders a "Why It Matters" section displaying four numbered value propositions for the OpenFrame/Flamingo platform in a styled card list.

## Key Components

- **`WhyItMattersItem`** — List item subcomponent rendering a numbered entry with title and description. Accepts `number`, `title`, `description`, and optional `isLast` (suppresses bottom border on final item).
- **`WhyItMatters`** — Default export. Renders a full-width section with a heading and an ordered list of four hardcoded value propositions: *Cut Costs*, *Stay in Control*, *Build What You Need*, and *Scale Freely*.

## Usage Example

```typescript
import WhyItMatters from '@/components/why-it-matters';

// Drop into any page layout — no props required
export default function LandingPage() {
  return (
    <main>
      <WhyItMatters />
    </main>
  );
}
```

To extend or modify the value propositions, update the `items` array inside `WhyItMatters`:

```typescript
const items = [
  {
    number: "1.",
    title: "Custom Reason",
    description: "Your description here",
  },
  // ...
];
```

## Notes

- Marked `"use client"` — renders on the client side (Next.js App Router).
- Styled with Tailwind CSS using design system tokens (`ods-bg`, `ods-card`, `ods-accent`, `ods-text-primary`, `ods-border`).
- Responsive layout: stacks vertically on mobile (`flex-col`), switches to horizontal on `md+` (`flex-row`).
- Content is hardcoded; no external data fetching or props needed at the section level.
