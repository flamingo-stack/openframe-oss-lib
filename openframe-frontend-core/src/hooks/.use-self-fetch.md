A client-side data-fetching hook that GETs a URL into component state using plain `fetch` + `useEffect`, with no external query-client dependency. Designed as the single shared primitive for self-fetching content views (`ProductReleasesView`, `RoadmapView`, onboarding catalog/detail).

## Key Components

### `UseSelfFetchResult<T>` (interface)

| Field | Type | Description |
|-------|------|-------------|
| `data` | `T \| null` | Fetched (or hydrated) payload |
| `setData` | `Dispatch<SetStateAction<T \| null>>` | Imperative patch for optimistic updates |
| `isLoading` | `boolean` | True while a fetch is in-flight |
| `error` | `boolean` | True after a non-abort failure |
| `reload` | `() => void` | Forces a retry of the current URL |

### `useSelfFetch<T>(url, options?)` (hook)

**Parameters:**
- `url: string | null` — `null` disables fetching entirely (SSR/controlled mode)
- `options.initialData?: T` — Hydrates state immediately and skips the first fetch; subsequent URL changes still re-fetch

**Behaviour highlights:**
- `url` is the cache key — all query params must be folded into it
- Stale-response guard (`cancelled` flag + `AbortController`) prevents out-of-order writes
- `dataUrlRef` comparison skips re-fetching SSR-hydrated data on mount, and survives React 18 StrictMode's double-invoke
- `reload()` nulls `dataUrlRef` then bumps an internal `reloadKey` to force a retry of the same URL

## Usage Example

```typescript
// Self-fetch mode (no SSR data)
const { data, isLoading, error, reload } = useSelfFetch<Product[]>(
  '/api/products?page=1'
)

// SSR-hydrated mode (skips first fetch, re-fetches on url change)
const { data, setData } = useSelfFetch<RoadmapItem[]>(
  `/api/roadmap?slug=${slug}`,
  { initialData: serverData }
)

// Optimistic update
setData((prev) => prev ? [...prev, newItem] : [newItem])

// Disabled / controlled mode
const { data } = useSelfFetch<Config>(null) // never fetches
```
