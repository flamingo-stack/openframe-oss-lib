A lightweight client-side data-fetching hook that GETs a URL into component state using plain `fetch` + `useEffect`, with no external dependencies like react-query.

## Key Components

### `UseSelfFetchResult<T>` (interface)

| Field | Type | Description |
|-------|------|-------------|
| `data` | `T \| null` | Fetched (or hydrated) data |
| `setData` | `Dispatch<SetStateAction<T \| null>>` | Optimistic / imperative patch |
| `isLoading` | `boolean` | `true` while a fetch is in-flight |
| `error` | `boolean` | `true` after a non-abort fetch failure |
| `reload` | `() => void` | Clears stale-url guard and retries |

### `useSelfFetch<T>(url, options?)` (hook)

| Parameter | Type | Description |
|-----------|------|-------------|
| `url` | `string \| null` | `null` disables fetching entirely |
| `options.initialData` | `T` (optional) | SSR-hydrated seed; skips first fetch |

**Key behaviours:**

- `url = null` → disabled/SSR mode; returns `initialData ?? null`, never fetches
- `initialData` provided → hydrates immediately, skips the first fetch to avoid re-fetching server-rendered data
- Stale-URL guard (`cancelled` + `AbortController`) prevents out-of-order responses from overwriting newer state
- `reload()` nulls the held-URL ref and bumps an internal key, forcing a retry of the same URL
- React 18 StrictMode safe — aborted double-invoke requests are silently ignored

## Usage Example

```typescript
// Self-fetch mode (no initial data)
const { data, isLoading, error, reload } = useSelfFetch<ReleaseList>(
  '/api/releases?page=1',
)

// SSR hydration mode (skip first fetch)
const { data, setData } = useSelfFetch<RoadmapData>(
  '/api/roadmap',
  { initialData: serverSideProps.roadmap },
)

// Optimistic update
setData(prev => ({ ...prev, votes: (prev?.votes ?? 0) + 1 }))

// Disabled / controlled mode
const { data } = useSelfFetch<CatalogItem>(null) // never fetches
```

> **Design note:** Intentionally avoids `react-query` so embedders don't need a `QueryClient` provider. The URL string acts as the full cache key — all query params should be folded into it.
