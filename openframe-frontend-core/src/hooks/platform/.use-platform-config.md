A React hook for managing platform configuration data with built-in caching and multiple output formats for different UI components.

## Key Components

- **`usePlatformConfig()`** - Main hook that fetches and caches platform configurations
- **`usePlatformByValue(value)`** - Helper hook to find a specific platform by its value
- **`useValidatePlatform(value)`** - Helper hook to validate if a platform value exists
- **`UsePlatformConfigResult`** - Interface defining the hook's return structure

## Usage Example

```typescript
import { usePlatformConfig, usePlatformByValue } from './use-platform-config';

function PlatformSelector() {
  const { platformOptions, selectableOptions, isLoading, error } = usePlatformConfig();
  
  if (isLoading) return <div>Loading platforms...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <select>
      {platformOptions.map(option => (
        <option key={option.value} value={option.value}>
          {option.label}
        </option>
      ))}
    </select>
  );
}

function PlatformDetails({ platformValue }: { platformValue: string }) {
  const platform = usePlatformByValue(platformValue);
  
  return platform ? (
    <div>{platform.label} - {platform.description}</div>
  ) : (
    <div>Platform not found</div>
  );
}
```

The hook implements session-level caching to minimize API calls and provides three data formats: full configs, simple dropdown options, and rich selectable options with visual elements.
