A comprehensive React hook that manages cursor-based pagination state including URL synchronization, debounced search, and page tracking. This hook eliminates 60-80 lines of boilerplate code from paginated components by centralizing common pagination logic.

## Key Components

- **`useCursorPaginationState`** - Main hook that combines URL state management, search debouncing, and pagination tracking
- **`UseCursorPaginationStateOptions`** - Configuration interface for customizing URL params, debounce timing, and callbacks
- **`CursorPaginationStateReturn`** - Return interface providing search state, pagination handlers, and URL param access

## Usage Example

```typescript
import { useCursorPaginationState } from './use-cursor-pagination-state'

function DialogsList() {
  const {
    searchInput,
    setSearchInput,
    hasLoadedBeyondFirst,
    handleNextPage,
    handleResetToFirstPage,
    handleFilterChange
  } = useCursorPaginationState({
    paramPrefix: 'dialogs', // Creates dialogsSearch, dialogsCursor URL params
    debounceMs: 500,
    filterSchema: {
      status: { type: 'array', default: [] },
      priority: { type: 'string', default: '' }
    },
    onInitialLoad: async (search, cursor) => {
      await fetchDialogs({ search, cursor, reset: true })
    },
    onSearchChange: async (search) => {
      await fetchDialogs({ search, reset: true })
    },
    onFiltersChange: async (filters) => {
      await fetchDialogs({ filters, reset: true })
    }
  })

  // Component renders search input, filters, and paginated results
  return (
    <div>
      <input 
        value={searchInput} 
        onChange={(e) => setSearchInput(e.target.value)}
        placeholder="Search dialogs..."
      />
      {/* Table with pagination controls using handleNextPage/handleResetToFirstPage */}
    </div>
  )
}
```

The hook automatically handles URL parameter synchronization, prevents infinite loops during state updates, and provides clean separation between initial loads, search changes, and filter updates.
