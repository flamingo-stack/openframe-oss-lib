A React hook for managing URL state parameters in REST API applications with automatic type coercion and URL synchronization. Provides the same URL state management as GraphQL-based hooks but with manual schema definition.

## Key Components

**Main Hook:**
- `useApiParams<TParams>()` - Core hook for URL parameter management

**Types:**
- `ParamSchema` - Schema definition interface for parameter types and defaults
- `UseApiParamsReturn<TParams>` - Hook return type with typed parameter access
- `UseApiParamsOptions` - Configuration options interface

**Helper:**
- `createSearchParams()` - Utility to convert object to URLSearchParams

## Usage Example

```typescript
// Define parameter schema
const schema = {
  search: { type: 'string', default: '' },
  page: { type: 'number', default: 1 },
  tags: { type: 'array', default: [] },
  active: { type: 'boolean', default: true }
}

// Use in component
const { params, urlSearchParams, setParam, setParams } = useApiParams(schema)

// Make API request
const response = await fetch(`/api/items?${urlSearchParams}`)

// Update parameters
setParam('search', 'laptop')
setParams({ page: 2, tags: ['electronics', 'sale'] })

// Access typed parameters
console.log(params.search) // string
console.log(params.page)   // number
console.log(params.tags)   // array
```

The hook automatically syncs parameter changes to the URL and provides type-safe access to parsed values.
