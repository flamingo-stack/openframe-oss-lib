A React hook that automatically generates URL state management from GraphQL queries by parsing query ASTs and syncing parameters with the URL.

## Key Components

- **`useQueryParams`** - Main hook that converts GraphQL query variables to URL parameters
- **`UseQueryParamsOptions`** - Configuration interface for customizing behavior
- **`UseQueryParamsReturn`** - Hook return type with variables, setters, and state
- **`applyParamMapping`** - Helper to customize URL parameter names

## Usage Example

```typescript
import { gql, useQuery } from '@apollo/client'
import { useQueryParams } from './use-query-params'

const LOGS_QUERY = gql`
  query GetLogs($search: String, $filter: LogFilterInput) {
    logs(search: $search, filter: $filter) {
      id
      message
      severity
    }
  }
`

function LogsPage() {
  // Hook automatically parses query and manages URL state
  const { variables, setParam, setParams, isReady } = useQueryParams(LOGS_QUERY, {
    defaultValues: { search: '', filter: { severity: [] } },
    debug: true
  })

  const { data, loading } = useQuery(LOGS_QUERY, { variables })

  if (!isReady) return <div>Loading schema...</div>

  return (
    <div>
      <input 
        value={variables.search || ''} 
        onChange={(e) => setParam('search', e.target.value)}
        placeholder="Search logs..."
      />
      
      <select 
        onChange={(e) => setParam('filter.severity', [e.target.value])}
      >
        <option value="info">Info</option>
        <option value="warning">Warning</option>
        <option value="error">Error</option>
      </select>
      
      {/* URL automatically updates: /logs?search=error&filter.severity=critical */}
      {data?.logs?.map(log => <div key={log.id}>{log.message}</div>)}
    </div>
  )
}
```

The hook performs GraphQL schema introspection, flattens nested input types to URL parameters, and provides bidirectional sync between URL state and GraphQL variables.
