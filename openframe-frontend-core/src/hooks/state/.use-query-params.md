A Next.js React hook that automatically generates URL state management from GraphQL queries by parsing the query AST, flattening nested input types, and maintaining bidirectional synchronization between URL parameters and GraphQL variables.

## Key Components

- **`useQueryParams`** - Main hook that takes a GraphQL DocumentNode and returns URL state management
- **`UseQueryParamsOptions`** - Configuration interface for default values, introspection settings, and custom mappings
- **`UseQueryParamsReturn`** - Return type providing variables, parameter setters, and loading states
- **`applyParamMapping`** - Internal utility for custom parameter name transformations

## Usage Example

```typescript
import { gql } from '@apollo/client'
import { useQuery } from '@apollo/client'
import { useQueryParams } from './use-query-params'

const LOGS_QUERY = gql`
  query GetLogs($search: String, $filter: LogFilterInput) {
    logs(search: $search, filter: $filter) {
      id
      message
      severity
    }
  }
`

function LogsPage() {
  const { variables, setParam, isReady } = useQueryParams(LOGS_QUERY, {
    defaultValues: { search: '', filter: { severity: [] } }
  })
  
  const { data, loading } = useQuery(LOGS_QUERY, { 
    variables,
    skip: !isReady 
  })

  return (
    <div>
      <input 
        value={variables.search || ''} 
        onChange={(e) => setParam('search', e.target.value)}
      />
      {/* URL automatically updates: /logs?search=error&severity=critical */}
    </div>
  )
}
```
