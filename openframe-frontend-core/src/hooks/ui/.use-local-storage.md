A React hook that provides localStorage persistence with automatic synchronization across browser tabs and components, including proper SSR handling and error recovery.

## Key Components

- **`useLocalStorage<T>`** - Main hook that manages localStorage state synchronization
- **Storage Event Listeners** - Handles cross-tab synchronization via `storage` and custom `localStorageUpdate` events
- **Initialization Guards** - Prevents hydration mismatches and duplicate localStorage reads
- **Error Handling** - Graceful fallbacks for localStorage access failures

## Usage Example

```typescript
import { useLocalStorage } from './use-local-storage'

function UserPreferences() {
  // Basic usage with primitive values
  const [theme, setTheme] = useLocalStorage('theme', 'light')
  const [fontSize, setFontSize] = useLocalStorage('fontSize', 16)
  
  // Usage with complex objects
  const [userSettings, setUserSettings] = useLocalStorage('userSettings', {
    notifications: true,
    autoSave: false
  })

  return (
    <div>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        Toggle Theme: {theme}
      </button>
      
      <button onClick={() => setFontSize(size => size + 2)}>
        Increase Font: {fontSize}px
      </button>
      
      <button onClick={() => setUserSettings(prev => ({
        ...prev,
        notifications: !prev.notifications
      }))}>
        Notifications: {userSettings.notifications ? 'On' : 'Off'}
      </button>
    </div>
  )
}
```

The hook automatically syncs changes across browser tabs and handles SSR scenarios by deferring localStorage access until after client-side hydration.
