A React hook that provides callback memoization with deep dependency comparison, offering more precise control over when callbacks are recreated compared to the standard `useCallback`.

## Key Components

- **`useMemoizedCallback<T>`** - Main hook function that memoizes callbacks based on dependency changes
- **Generic type `T`** - Extends function type to preserve callback signature and type safety
- **Dependency tracking** - Uses `useRef` to store and compare previous dependencies with `Object.is()`

## Usage Example

```typescript
import { useMemoizedCallback } from './use-memoized-callback'

function MyComponent({ userId, filter }) {
  // Callback will only recreate when userId or filter actually changes
  const handleSearch = useMemoizedCallback(
    (query: string) => {
      searchAPI({ userId, filter, query })
    },
    [userId, filter]
  )

  // Compare with regular useCallback - may recreate unnecessarily
  const regularCallback = useCallback(
    (query: string) => searchAPI({ userId, filter, query }),
    [userId, filter]
  )

  return <SearchInput onSearch={handleSearch} />
}
```

The hook prevents unnecessary callback recreation by using `Object.is()` for precise dependency comparison, making it ideal for optimizing expensive operations or preventing unwanted re-renders in child components.
