A React hook for detecting CSS media query matches with SSR-safe hydration handling. Returns `undefined` during server-side rendering and initial client-side render to prevent hydration mismatches.

## Key Components

- **`useMediaQuery(query: string)`** - Core hook that accepts any CSS media query string and returns match status
- **`breakpoints`** - Predefined Tailwind CSS-compatible breakpoint definitions (sm, md, lg, xl, 2xl)
- **Convenience hooks** - `useSmUp()`, `useMdUp()`, `useLgUp()`, `useXlUp()`, `use2xlUp()` for common responsive breakpoints

## Usage Example

```typescript
import { useMediaQuery, useMdUp, breakpoints } from './use-media-query'

function ResponsiveComponent() {
  // Custom media query
  const isDarkMode = useMediaQuery('(prefers-color-scheme: dark)')
  
  // Predefined breakpoint
  const isMdUp = useMdUp()
  
  // Custom breakpoint
  const isWide = useMediaQuery('(min-width: 1400px)')

  // Handle undefined state during SSR/hydration
  if (isDarkMode === undefined || isMdUp === undefined) {
    return <div>Loading...</div>
  }

  return (
    <div>
      <p>Dark mode: {isDarkMode ? 'Yes' : 'No'}</p>
      <p>Medium screen or larger: {isMdUp ? 'Yes' : 'No'}</p>
      <p>Wide screen: {isWide ? 'Yes' : 'No'}</p>
    </div>
  )
}
```

The hook uses `useLayoutEffect` to avoid visual flicker and properly cleans up event listeners to prevent memory leaks.
