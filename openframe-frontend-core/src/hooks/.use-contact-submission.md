A React hook that manages contact form submissions with loading states, toast notifications, and automatic redirect handling.

## Key Components

- **`useContactSubmission`** - Main hook that provides contact form submission functionality
- **`ContactSubmissionOptions`** - Configuration interface for customizing behavior (userId, redirect URL, success message, callback)
- **`ContactFormData`** - Type definition for contact form fields (name, email, company size, referral source, help category, message)
- **`submit`** - Async function that handles form submission to `/api/contact` endpoint
- **State management** - Tracks `isSubmitting`, `isSuccess` states and handles automatic redirects

## Usage Example

```typescript
import { useContactSubmission } from './use-contact-submission';

function ContactForm() {
  const { submit, isSubmitting, isSuccess } = useContactSubmission({
    userId: 'user-123',
    successRedirectUrl: '/thank-you',
    successToastMessage: 'We will get back to you within 24 hours.',
    onSuccess: () => console.log('Form submitted successfully!')
  });

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    
    const formData = {
      name: 'John Doe',
      email: 'john@example.com',
      companySize: '1-10',
      referralSource: 'Google',
      helpCategory: 'Support',
      message: 'Need help with implementation'
    };

    try {
      await submit(formData);
    } catch (error) {
      // Handle error if needed
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Sending...' : 'Send Message'}
      </button>
    </form>
  );
}
```

The hook automatically handles success toast notifications, optional redirects after a 1.5-second delay, and follows the same pattern as `useWaitlistRegistration` for consistency across the application.
