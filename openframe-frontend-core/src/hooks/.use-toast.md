A custom toast notification wrapper built on top of the Sonner library, providing a simplified interface for displaying various types of toast messages with consistent styling and behavior.

## Key Components

- **`ToastOptions`** - Interface defining toast configuration options including title, description, variant, and duration
- **`toast`** - Main function that accepts either a string message or options object to display notifications
- **`useToast`** - React hook that returns the toast function along with direct access to underlying Sonner methods

## Usage Example

```typescript
import { toast, useToast } from './use-toast'

// Simple string message
toast('Operation completed successfully')

// With options object
toast({
  title: 'Success!',
  description: 'Your changes have been saved',
  variant: 'success',
  duration: 3000
})

// In a React component
function MyComponent() {
  const { toast } = useToast()

  const handleSave = async () => {
    try {
      await saveData()
      toast({
        title: 'Data saved',
        variant: 'success'
      })
    } catch (error) {
      toast({
        title: 'Error saving data',
        description: error.message,
        variant: 'destructive'
      })
    }
  }

  return <button onClick={handleSave}>Save</button>
}
```

The wrapper supports five variants: `default`, `success`, `destructive`, `info`, and `warning`, each mapping to appropriate Sonner toast methods for consistent visual feedback.
