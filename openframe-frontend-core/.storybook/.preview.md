Storybook preview configuration for a Next.js app, defining global parameters and importing required CSS/font stylesheets.

## Key Components

- **`preview`** — Default export; a `Preview` object configuring Storybook's global behavior with the following parameters:
  - `controls.matchers` — Auto-maps control types: regex for `color`/`background` fields and `Date` fields.
  - `nextjs.appDirectory` — Enables App Router compatibility (`true`).
- **CSS Imports** — Loads `storybook-fonts.css` (direct font import, since `next/font/google` is unavailable in Storybook) and `index.css` (global styles).

## Usage Example

```typescript
// .storybook/preview.ts
import type { Preview } from '@storybook/nextjs-vite';
import '../src/styles/storybook-fonts.css';
import '../src/styles/index.css';

const preview: Preview = {
  parameters: {
    controls: {
      matchers: {
        color: /(background|color)$/i,
        date: /Date$/i,
      },
    },
    nextjs: {
      appDirectory: true, // Required for Next.js App Router support
    },
  },
};

export default preview;
```

> **Note:** The `storybook-fonts.css` file is a Storybook-specific workaround — Next.js apps typically load fonts via `next/font/google` in `fonts.ts`, which is not available in the Storybook runtime. Any font changes in `fonts.ts` must be mirrored in `storybook-fonts.css`.
