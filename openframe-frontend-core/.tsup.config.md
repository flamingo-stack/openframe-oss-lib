Configures the dual-build pipeline for the component library using `tsup`, splitting output into server-safe and client-side bundles to correctly handle Next.js `"use client"` boundaries.

## Key Components

**Server/Universal Build**
- Compiles pure types, configs, schemas, fonts, utilities, and server-safe constants (e.g. Mux CDN origins, Zod contact schema)
- `treeshake: true` — enables dead code elimination
- No `"use client"` banner — safe for server-side imports

**Client Build**
- Compiles React components, hooks, contexts, NATS bindings, and embed shims
- `banner: { js: '"use client";' }` — prepends `"use client"` to every output file, establishing a Next.js Client Component boundary
- `splitting: true` — enables code splitting for the larger component surface; `treeshake` omitted (incompatible with `banner`)

**Embed Shims**
- Individual entries per Next.js adapter (`next-image`, `next-link`, `next-dynamic`, `next-navigation`) allowing deep imports without loading the full barrel

**Shared Settings (both builds)**
- Output formats: `esm` + `cjs`
- `dts: false` — declarations are generated separately via `tsc` to produce `.d.ts.map` files
- `sourcemap: true`
- Externals: `react`, `react-dom`, `next`, `@tanstack/react-query`

## Usage Example

```bash
# Declarations generated by tsc (see tsconfig.declarations.json)
# dist/ is cleaned by the build script before tsup runs
tsup
```

```typescript
// Deep import a server-safe schema (no client boundary pollution)
import { contactSchema } from '@flamingo/ui/schemas/contact-schema'

// Deep import a single embed shim without loading the full barrel
import { NextImageShim } from '@flamingo/ui/embed-shims/next-image'
```
