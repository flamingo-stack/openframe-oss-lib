Configures the dual-build pipeline for the shared UI library using `tsup`, separating server-safe and client-side entries to prevent Next.js SSR boundary violations.

## Key Components

**Server/Universal Build**
- Compiles pure types, configs, Zod schemas, Tailwind config, assets, fonts, and server-safe utilities
- Enables `treeshake` to reduce bundle size; omits `"use client"` banner
- Exports `mux-origins` as a dedicated server-safe subpath to avoid client-reference poisoning in server modules

**Client Build**
- Compiles all React components, hooks, contexts, NATS bindings, and embed shims
- Injects `"use client";` via `banner` so Next.js treats each output file as a Client Component boundary
- Disables `treeshake` (incompatible with `banner`/rollup interaction); enables `splitting` for code-splitting

**Shared Settings (both builds)**
- Output formats: `esm` and `cjs`
- `dts: false` — declarations are generated separately by `tsc` via `tsconfig.declarations.json` to produce `.d.ts.map` files for source navigation
- `sourcemap: true`
- Externals: `react`, `react-dom`, `next`, `@tanstack/react-query`
- `dist/` is cleaned by the build script before `tsup` runs (`clean: true` not needed)

## Usage Example

```bash
# Run both builds sequentially (as called by the build script)
tsup
```

```typescript
// Server-side import — safe, no React boundary
import { contactSchema } from 'your-lib/schemas/contact-schema'
import { MUX_ORIGINS } from 'your-lib/components/features/mux-origins'

// Client-side import — hits the "use client" bundle
import { Button } from 'your-lib/components/ui/index'
import { useChat } from 'your-lib/hooks/index'
```
