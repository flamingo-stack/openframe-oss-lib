Configuration file for tsup bundler that builds a React component library with multiple entry points for modular imports.

## Key Components

- **Multiple Entry Points**: Defines 14 separate entry points for different modules including components, hooks, utils, types, and assets
- **Dual Format Output**: Generates both ESM and CommonJS builds for maximum compatibility
- **TypeScript Support**: Enables declaration file generation with `dts: true`
- **External Dependencies**: Excludes React, Next.js, and TanStack Query from the bundle
- **Build Optimization**: Enables tree-shaking and source maps while disabling code splitting

## Usage Example

This configuration enables consumers to import specific parts of the library:

```typescript
// Import the main entry point
import { Button } from 'your-library'

// Import specific UI components
import { FileManager } from 'your-library/components/ui/file-manager'

// Import utilities separately
import { formatDate } from 'your-library/utils'

// Import type definitions
import type { NavigationItem } from 'your-library/types/navigation'
```

The build process generates both ES modules and CommonJS formats, allowing usage in various environments while maintaining TypeScript support through generated declaration files.
