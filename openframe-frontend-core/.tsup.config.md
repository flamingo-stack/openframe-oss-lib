Configures the dual-build pipeline for the shared UI library, splitting outputs into server-safe and client-side bundles to prevent React context errors in Next.js server environments.

## Key Components

**Build 1 — Server/Universal Entries**
Compiles pure types, configs, NATS utilities, Zod schemas, assets, fonts, and Tailwind config. Targets both `esm` and `cjs` formats with `treeshake: true` and no `"use client"` banner.

| Entry | Source |
|---|---|
| `nats/index` | NATS messaging utilities |
| `types/*` | Shared TypeScript types (navigation, announcements) |
| `schemas/contact-schema` | Zod validation schema (server + client shared) |
| `utils/index` | Server-safe utility functions |
| `assets/index`, `fonts`, `tailwind.config` | Static assets and styling config |

**Build 2 — Client Entries**
Compiles all React components, hooks, contexts, and embed shims. Injects `"use client";` as a banner on every output file so Next.js treats them as Client Component boundaries. Uses `splitting: true` instead of `treeshake` (incompatible with `banner`).

| Entry Group | Contents |
|---|---|
| `components/*` | UI, chat, tickets, contact, features, toast, icons, navigation |
| `hooks/index` | React hooks |
| `contexts/index` | React context providers |
| `embed-shims/*` | Per-entry wrappers for `next/dynamic`, `next/link`, `next/image`, `next/navigation` |

## Key Design Decisions

- **`dts: false`** on both builds — declarations are generated separately via `tsc` + `tsconfig.declarations.json` to produce `.d.ts.map` files for source navigation
- **`treeshake` vs `banner`** are mutually exclusive — Rollup strips banners during tree-shaking, so each build uses only one
- **`dist/` cleanup** is handled by the build script (`rm -rf dist`) rather than `clean: true`
- **Embed shims as individual entries** — allows deep imports (`embed-shims/next-image`) without loading the full barrel

## Usage Example

```bash
# Build both server and client bundles
rm -rf dist && tsup && tsc -p tsconfig.declarations.json
```

```typescript
// Server-safe import (no "use client" boundary crossed)
import { contactSchema } from '@your-lib/schemas/contact-schema'

// Client component import (banner ensures Next.js treats as client boundary)
import { Button } from '@your-lib/components/ui'

// Deep embed-shim import without pulling full barrel
import { NextImage } from '@your-lib/embed-shims/next-image'
```
