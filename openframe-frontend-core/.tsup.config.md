This tsup configuration file defines the build settings for a comprehensive React component library with multiple entry points and modern JavaScript tooling.

## Key Components

- **Entry Points**: Defines 15 separate entry points covering components, hooks, utilities, types, and assets
- **Build Formats**: Outputs both ESM and CommonJS formats for maximum compatibility
- **TypeScript Support**: Generates declaration files (`.d.ts`) for full TypeScript support
- **External Dependencies**: Excludes React ecosystem packages from the bundle
- **Build Optimizations**: Enables tree-shaking, source maps, and clean builds

## Usage Example

```typescript
// This configuration enables multiple import paths:

// Main library entry
import { SomeComponent } from 'your-library'

// Specific component categories
import { Button } from 'your-library/components/ui'
import { FileManager } from 'your-library/components/ui/file-manager'
import { useCustomHook } from 'your-library/hooks'

// Utilities and types
import { formatDate } from 'your-library/utils'
import type { NavigationType } from 'your-library/types/navigation'

// Assets and configuration
import { icons } from 'your-library/assets'
import tailwindConfig from 'your-library/tailwind.config'
```

This setup allows consumers to import only what they need, reducing bundle size through selective imports while maintaining full TypeScript support and compatibility across different module systems.
