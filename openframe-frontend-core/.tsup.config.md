Configuration file for tsup bundler that defines build settings for a React component library with granular entry points and dual-format output.

## Key Components

- **Entry Points**: Multiple entry configurations for different library modules (components, hooks, utils, types, etc.)
- **Build Formats**: Dual ESM and CommonJS output formats
- **TypeScript Support**: Automatic TypeScript declaration file generation
- **External Dependencies**: React, Next.js, and TanStack Query marked as externals
- **Optimization**: Tree-shaking enabled with source maps and clean builds

## Usage Example

```typescript
// Install and run tsup with this config
npm install tsup

// Build the library
npx tsup

// The config will generate multiple entry points:
// - dist/index.js (main entry)
// - dist/components/index.js
// - dist/hooks/index.js
// - dist/utils/index.js
// etc.

// Consumers can import specific modules:
import { Button } from 'your-library/components'
import { useCustomHook } from 'your-library/hooks'
import { utilityFunction } from 'your-library/utils'
```

This configuration enables selective imports and optimal tree-shaking for library consumers while maintaining compatibility with both ESM and CommonJS environments.
