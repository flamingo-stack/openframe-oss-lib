Custom exception thrown when a user attempts to delete their own account, which is a restricted operation in the OpenFrame platform.

## Key Components

| Element | Description |
|---|---|
| `UserSelfDeleteNotAllowedException` | Extends `ConflictException` to signal a self-deletion conflict |
| `ErrorCode.USER_SELF_DELETE_NOT_ALLOWED` | Standardized error code passed to the parent exception |
| Constructor | Accepts a descriptive `message` string forwarded to `ConflictException` |

## Usage Example

```java
// Thrown in user service when the requesting user matches the target user ID
if (requestingUserId.equals(targetUserId)) {
    throw new UserSelfDeleteNotAllowedException(
        "User cannot delete their own account."
    );
}
```

The exception propagates as an HTTP `409 Conflict` response, consistent with the `ConflictException` base class behavior across the OpenFrame API.
