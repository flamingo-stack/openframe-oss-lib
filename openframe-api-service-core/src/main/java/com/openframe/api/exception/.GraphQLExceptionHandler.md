A custom GraphQL exception handler that provides structured error handling and user-friendly error messages for different types of exceptions that can occur during GraphQL data fetching operations.

## Key Components

- **GraphQLExceptionHandler** - Spring component that extends `SimpleDataFetcherExceptionHandler` to customize error handling behavior
- **handleException()** - Main method that processes exceptions and converts them into structured GraphQL errors with appropriate error codes and messages

## Exception Types Handled

The handler categorizes and handles the following exception types:

- **PinotQueryException** - Query failures from Pinot database operations (`PINOT_QUERY_ERROR`)
- **DataAccessException** - General database operation failures (`DATABASE_ERROR`) 
- **IllegalArgumentException** - Input validation errors (`VALIDATION_ERROR`)
- **RuntimeException** - Unexpected runtime errors (`INTERNAL_ERROR`)
- **Other exceptions** - All other throwables (`UNKNOWN_ERROR`)

## Usage Example

This handler is automatically registered as a Spring component and used by the GraphQL execution engine:

```java
// The handler automatically processes exceptions thrown by resolvers
@QueryMapping
public User getUser(@Argument String userId) {
    if (userId == null || userId.isEmpty()) {
        throw new IllegalArgumentException("User ID is required");
        // Results in VALIDATION_ERROR with the exception message
    }
    
    return userService.findById(userId);
    // Any DataAccessException becomes DATABASE_ERROR
    // Any PinotQueryException becomes PINOT_QUERY_ERROR
}
```

Each error includes structured extensions with an error code and timestamp for better client-side error handling and debugging.
