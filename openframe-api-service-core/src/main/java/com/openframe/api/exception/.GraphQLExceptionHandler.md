A Spring component that provides centralized exception handling for GraphQL data fetchers, converting various types of exceptions into user-friendly GraphQL errors with structured error codes and metadata.

## Key Components

- **GraphQLExceptionHandler**: Main exception handler extending `SimpleDataFetcherExceptionHandler`
- **handleException()**: Core method that processes exceptions and returns structured GraphQL errors
- **Exception Types Handled**:
  - `PinotQueryException` → PINOT_QUERY_ERROR
  - `DataAccessException` → DATABASE_ERROR  
  - `IllegalArgumentException` → VALIDATION_ERROR
  - `RuntimeException` → INTERNAL_ERROR
  - Other exceptions → UNKNOWN_ERROR

## Usage Example

The handler automatically processes exceptions thrown by GraphQL data fetchers:

```java
@Component
public class UserDataFetcher implements DataFetcher<User> {
    
    @Override
    public User get(DataFetchingEnvironment env) throws Exception {
        // If this throws a PinotQueryException, the handler converts it to:
        // {
        //   "message": "Query failed. Please try again later.",
        //   "extensions": {
        //     "code": "PINOT_QUERY_ERROR",
        //     "timestamp": 1699123456789
        //   }
        // }
        return userService.findById(id);
    }
}
```

Each error includes a user-friendly message, specific error code for client handling, and timestamp for debugging purposes.
