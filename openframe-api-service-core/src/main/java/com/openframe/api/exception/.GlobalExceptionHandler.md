Centralized exception handling component that provides consistent error responses and logging across the OpenFrame API using Spring Boot's `@RestControllerAdvice`.

## Key Components

- **Global Exception Handlers**: Methods annotated with `@ExceptionHandler` to catch specific exception types
- **HTTP Status Mapping**: Each handler maps exceptions to appropriate HTTP status codes using `@ResponseStatus`
- **ErrorResponse**: Standardized error response format with error codes and messages
- **Comprehensive Logging**: SLF4J logging for all exceptions with appropriate log levels
- **Validation Support**: Specialized handling for bean validation and constraint violations

## Usage Example

The handler automatically intercepts exceptions thrown by any REST controller:

```java
@RestController
public class UserController {
    
    @GetMapping("/users/{id}")
    public User getUser(@PathVariable Long id) {
        // This exception will be caught by GlobalExceptionHandler
        throw new UserNotFoundException("User with ID " + id + " not found");
        // Results in HTTP 404 with ErrorResponse body
    }
    
    @PostMapping("/users")
    public User createUser(@Valid @RequestBody CreateUserRequest request) {
        // Validation failures automatically handled
        // Returns HTTP 400 with detailed validation messages
    }
}
```

The handler supports various exception categories including authentication errors (401), authorization failures (403), validation errors (400), not found exceptions (404), and server errors (500), ensuring consistent error responses throughout the application.
