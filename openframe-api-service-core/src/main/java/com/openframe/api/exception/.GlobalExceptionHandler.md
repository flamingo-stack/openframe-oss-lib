Centralized exception handling class for the OpenFrame API that provides consistent error responses and logging across all REST endpoints using Spring's `@RestControllerAdvice` annotation.

## Key Components

**Exception Handlers:**
- **Authentication/Authorization**: `AccessDeniedException`, `BadCredentialsException` → 401/403 responses
- **Validation**: `MethodArgumentNotValidException`, `ConstraintViolationException` → 400 responses with field details
- **Resource Not Found**: `UserNotFoundException`, `DeviceNotFoundException`, `ApiKeyNotFoundException` → 404 responses
- **Business Logic**: `IllegalStateException`, `UserSelfDeleteNotAllowedException` → 409/403 responses
- **HTTP Protocol**: `HttpRequestMethodNotSupportedException`, `HttpMediaTypeNotSupportedException` → 405/415 responses
- **Generic**: `Exception` → 500 response with fallback handling

**Features:**
- Structured error logging with appropriate log levels
- Consistent `ErrorResponse` format for all exceptions
- Validation error message aggregation

## Usage Example

```java
// Custom exception handling
@RestController
public class UserController {
    
    @PostMapping("/users")
    public User createUser(@Valid @RequestBody UserRequest request) {
        // Validation errors automatically handled by GlobalExceptionHandler
        // Returns 400 with field-specific error messages
        return userService.createUser(request);
    }
    
    @GetMapping("/users/{id}")
    public User getUser(@PathVariable Long id) {
        // UserNotFoundException automatically returns 404
        // with {"errorCode": "USER_NOT_FOUND", "message": "..."}
        return userService.findById(id);
    }
}
```
