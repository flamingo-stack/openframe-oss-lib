REST controller that provides administrative endpoints for forcefully managing OpenFrame client and tool agent operations across the system.

## Key Components

**Main Service Dependencies:**
- `ForceToolInstallationService` - Handles forced tool installations and reinstallations
- `ForceClientUpdateService` - Manages forced client updates
- `ForceToolAgentUpdateService` - Controls forced tool agent updates

**Core Endpoints:**
- `POST /force/tool-agent/install` - Force install tool agent on specific machines
- `POST /force/client/update` - Force update specific clients
- `POST /force/tool-agent/update` - Force update specific tool agents
- `POST /force/**/all` - Batch operations for all instances
- `POST /force/tool-agent/reinstall` - Force reinstallation of tool agents

## Usage Example

```java
@Autowired
private ForceAgentController forceController;

// Force install tool agent on specific machines
ForceToolInstallationRequest installRequest = new ForceToolInstallationRequest();
installRequest.setToolAgentId("agent-123");
installRequest.setMachineIds(Arrays.asList("machine-1", "machine-2"));
ForceToolAgentInstallationResponse response = forceController.forceToolInstallation(installRequest);

// Force update all clients
ForceClientUpdateResponse updateResponse = forceController.forceClientUpdateAll();

// Force reinstall tool agent on all machines
ForceToolInstallationAllRequest reinstallRequest = new ForceToolInstallationAllRequest();
reinstallRequest.setToolAgentId("agent-456");
ForceToolAgentInstallationResponse reinstallResponse = forceController.forceToolReinstallationAll(reinstallRequest);
```

The controller uses the `/force` base path to clearly indicate these are administrative force operations that bypass normal update/installation flows.
