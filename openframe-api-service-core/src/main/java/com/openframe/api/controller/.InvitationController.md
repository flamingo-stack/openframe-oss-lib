REST controller that manages user invitation operations including creation, listing, and revocation. Provides endpoints for handling invitation lifecycle management with pagination support.

## Key Components

- **InvitationController** - Main REST controller class mapped to `/invitations`
- **createInvitation()** - POST endpoint to create new invitations
- **listInvitations()** - GET endpoint to retrieve paginated invitation lists
- **revokeInvitation()** - DELETE endpoint to revoke specific invitations by ID
- **InvitationService** - Injected service handling business logic

## Usage Example

```java
// Create a new invitation
POST /invitations
Content-Type: application/json

{
  "email": "user@example.com",
  "role": "USER"
}

// List invitations with pagination
GET /invitations?page=0&size=10

// Revoke an invitation
DELETE /invitations/invitation-id-123
```

The controller uses standard HTTP status codes (201 for creation, 200 for retrieval, 204 for deletion) and includes request validation through `@Valid` annotation. Pagination parameters default to page 0 with 20 items per page if not specified.
