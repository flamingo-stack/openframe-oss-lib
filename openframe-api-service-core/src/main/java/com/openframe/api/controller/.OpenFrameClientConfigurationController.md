REST controller that provides client configuration data for OpenFrame applications through a simple GET endpoint.

## Key Components

- **OpenFrameClientConfigurationController** - Main controller class exposing client configuration endpoints
- **getClientConfiguration()** - GET endpoint method that retrieves current client configuration
- **OpenFrameClientConfigurationQueryService** - Injected service handling configuration retrieval logic
- **ClientConfigurationResponse** - DTO containing configuration data returned to clients

## Usage Example

```java
// HTTP GET request to retrieve client configuration
GET /openframe-client/configuration

// Response format (example)
{
  "theme": "dark",
  "apiVersion": "v1.2.0",
  "features": {
    "enableNotifications": true,
    "maxFileSize": 10485760
  }
}

// Using in a Spring Boot application
@Autowired
private OpenFrameClientConfigurationQueryService configService;

public void setupClient() {
    ClientConfigurationResponse config = configService.get();
    // Configure client based on response
}
```

The controller follows Spring Boot REST conventions with proper dependency injection and logging capabilities. It serves as a lightweight endpoint for clients to fetch their configuration settings dynamically.
