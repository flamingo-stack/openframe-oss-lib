REST controller that provides client configuration data for OpenFrame applications through a single GET endpoint.

## Key Components

**OpenFrameClientConfigurationController**
- `@RestController` exposing configuration endpoints at `/openframe-client/configuration`
- `clientConfigurationQueryService` - Service dependency for retrieving client configuration
- `getClientConfiguration()` - GET endpoint returning `ClientConfigurationResponse`

## Usage Example

```java
// The controller provides a simple GET endpoint for client configuration
@GetMapping("/openframe-client/configuration")
public ClientConfigurationResponse getConfiguration() {
    // Returns configuration data via the injected query service
    return clientConfigurationQueryService.get();
}

// Example API call:
// GET /openframe-client/configuration
// Returns: ClientConfigurationResponse with application settings
```

The controller follows Spring Boot REST conventions with dependency injection via `@RequiredArgsConstructor` and logging support through `@Slf4j`. It serves as a lightweight facade over the configuration query service, providing client applications with necessary configuration data through a standardized REST interface.
