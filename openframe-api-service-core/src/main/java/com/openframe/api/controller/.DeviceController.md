REST controller that handles device status updates through internal API endpoints.

## Key Components

- **DeviceController**: Main REST controller class annotated with `@RestController`
- **updateDeviceStatus()**: PATCH endpoint that updates device status by machine ID
- **DeviceService**: Injected service dependency for business logic
- **UpdateDeviceStatusRequest**: DTO for request body containing status information

## Usage Example

```java
// PATCH request to update device status
// Endpoint: PATCH /devices/{machineId}
// Request body: {"status": "ONLINE"}

// Example using RestTemplate
RestTemplate restTemplate = new RestTemplate();
UpdateDeviceStatusRequest request = new UpdateDeviceStatusRequest("ONLINE");

restTemplate.patchForObject(
    "http://localhost:8080/devices/device-123", 
    request, 
    Void.class
);

// Example request body JSON
{
  "status": "OFFLINE"
}
```

The controller uses Spring Boot annotations for dependency injection (`@RequiredArgsConstructor`) and logging (`@Slf4j`). The endpoint returns HTTP 204 No Content on successful updates and logs all status change operations for audit purposes.
