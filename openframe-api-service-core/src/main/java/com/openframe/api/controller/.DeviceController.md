REST controller for internal device management operations, providing endpoints to update device status information.

## Key Components

- **DeviceController**: Main REST controller class handling device-related HTTP requests
- **updateDeviceStatus()**: PATCH endpoint for updating device status by machine ID
- **DeviceService**: Injected service dependency for business logic execution
- **UpdateDeviceStatusRequest**: DTO for status update request payload

## Usage Example

```java
// PATCH request to update device status
// URL: PATCH /devices/{machineId}
// Request Body: {"status": "ACTIVE"}

@Autowired
private DeviceService deviceService;

// The controller automatically handles:
// 1. Path variable extraction (machineId)
// 2. Request body deserialization (UpdateDeviceStatusRequest)
// 3. Service method invocation
// 4. HTTP 204 No Content response

// Example service call flow:
deviceService.updateStatusByMachineId("device-123", DeviceStatus.ACTIVE);
```

The controller uses Spring Boot annotations for automatic request mapping, logging via Slf4j, and dependency injection through Lombok's `@RequiredArgsConstructor`. Returns HTTP 204 (No Content) on successful status updates.
