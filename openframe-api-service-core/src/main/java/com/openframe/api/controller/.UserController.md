REST controller that handles user management operations including listing and deletion with pagination support.

## Key Components

- **UserController** - Main REST controller class mapped to `/users` endpoint
- **listUsers()** - GET endpoint for paginated user retrieval with configurable page size
- **deleteUser()** - DELETE endpoint for soft-deleting users by ID with authentication
- **UserService** - Injected service dependency for business logic
- **AuthPrincipal** - Custom authentication principal for accessing current user context

## Usage Example

```java
// GET /users - List users with default pagination
// Returns: UserPageResponse with users (page=0, size=20)

// GET /users?page=1&size=10 - Custom pagination
// Returns: UserPageResponse with 10 users from page 1

// DELETE /users/123 - Soft delete user
// Requires: Valid JWT token with AuthPrincipal
// Returns: 204 No Content status

@Autowired
private UserService userService;

// Service layer handles the actual business logic
public UserPageResponse listUsers(int page, int size) {
    return userService.listUsers(page, size);
}

public void deleteUser(String userId, String principalId) {
    userService.softDeleteUser(userId, principalId);
}
```

The controller uses constructor injection with Lombok's `@RequiredArgsConstructor` and Spring Security's `@AuthenticationPrincipal` for secure user operations. Soft deletion tracks which authenticated user performed the action.
