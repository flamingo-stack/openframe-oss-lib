REST controller that manages Single Sign-On (SSO) provider configurations, providing endpoints for both admin configuration and client-side provider discovery.

## Key Components

**Main Class:**
- `SSOConfigController` - REST controller with `/sso` base mapping

**Primary Endpoints:**
- `GET /sso/providers` - Returns enabled SSO providers for login components
- `GET /sso/providers/available` - Returns all available provider implementations
- `GET /sso/{provider}` - Retrieves full configuration for a specific provider
- `POST/PUT /sso/{provider}` - Creates or updates provider configuration
- `PATCH /sso/{provider}/toggle` - Enables/disables a provider
- `DELETE /sso/{provider}` - Removes provider configuration

**Dependencies:**
- `SSOConfigService` - Business logic service
- Various DTO classes for request/response handling

## Usage Example

```java
// Get enabled providers for login UI
@Autowired
private SSOConfigController controller;

// Retrieve providers available to users
List<SSOConfigStatusResponse> enabledProviders = controller.getEnabledProviders();

// Configure a new OAuth provider
SSOConfigRequest request = new SSOConfigRequest();
request.setClientId("your-client-id");
request.setClientSecret("your-secret");
request.setEnabled(true);

SSOConfigResponse response = controller.saveConfig("google", request);

// Toggle provider availability
controller.toggleEnabled("google", false);
```

The controller follows RESTful conventions and separates admin configuration endpoints from client-facing provider discovery endpoints.
