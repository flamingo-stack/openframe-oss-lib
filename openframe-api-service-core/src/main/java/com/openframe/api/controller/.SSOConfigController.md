A REST controller that manages Single Sign-On (SSO) configuration for different authentication providers. It provides endpoints for both end-user authentication components and admin configuration management.

## Key Components

- **`getEnabledProviders()`** - Returns active SSO providers for login UI components
- **`getAvailableProviders()`** - Lists all supported SSO provider types for admin dropdowns
- **`getConfig()`** - Retrieves full configuration details for a specific provider
- **`saveConfig()`** - Creates new SSO provider configuration
- **`updateConfig()`** - Updates existing SSO provider configuration
- **`toggleEnabled()`** - Enables or disables a provider without full configuration changes
- **`deleteConfig()`** - Removes SSO provider configuration

## Usage Example

```java
// Configure Google SSO provider
SSOConfigRequest googleConfig = SSOConfigRequest.builder()
    .clientId("google-client-id")
    .clientSecret("google-client-secret")
    .enabled(true)
    .build();

// Create configuration
POST /sso/google
Content-Type: application/json
{
  "clientId": "google-client-id",
  "clientSecret": "google-client-secret",
  "enabled": true
}

// Get enabled providers for login page
GET /sso/providers
// Returns: [{"provider": "google", "clientId": "google-client-id", "enabled": true}]

// Toggle provider on/off
PATCH /sso/google/toggle?enabled=false
```

The controller separates public provider information (used by login components) from sensitive configuration details (used by admin interfaces), ensuring secure SSO management across the application.
