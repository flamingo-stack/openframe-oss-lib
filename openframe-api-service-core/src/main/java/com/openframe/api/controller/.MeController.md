REST controller that provides authenticated user information through a `/me` endpoint, returning the current user's profile data or an unauthorized response.

## Key Components

- **MeController**: Main REST controller class
- **getCurrentUser()**: GET endpoint method that retrieves authenticated user details
- **AuthPrincipal**: Custom authentication principal containing user information
- **ResponseEntity**: Returns JSON responses with user data or error messages

## Usage Example

```java
// Making a request to get current user info
@Autowired
private TestRestTemplate restTemplate;

// Authenticated request
HttpHeaders headers = new HttpHeaders();
headers.setBearerAuth(jwtToken);
HttpEntity<?> entity = new HttpEntity<>(headers);

ResponseEntity<Map> response = restTemplate.exchange(
    "/me", 
    HttpMethod.GET, 
    entity, 
    Map.class
);

// Successful response structure:
{
    "authenticated": true,
    "user": {
        "id": "user123",
        "email": "user@example.com",
        "displayName": "John Doe",
        "roles": ["USER", "ADMIN"],
        "tenantId": "tenant456"
    }
}

// Unauthorized response:
{
    "error": "unauthorized",
    "error_description": "No authenticated user found"
}
```

The endpoint automatically extracts the authenticated user from the security context and returns their profile information, making it useful for client applications to verify authentication status and retrieve user details.
