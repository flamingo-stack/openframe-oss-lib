REST controller that provides user authentication information and current user details through a single endpoint.

## Key Components

- **MeController** - Spring Boot REST controller class
- **getCurrentUser()** - GET endpoint method that returns authenticated user information
- **AuthPrincipal** - Custom authentication principal containing user details
- **ResponseEntity** - HTTP response wrapper with proper status codes

## Usage Example

The controller exposes a single endpoint for retrieving current user information:

```java
// GET /me endpoint usage
// When authenticated, returns:
{
  "authenticated": true,
  "user": {
    "id": "user123",
    "email": "user@example.com", 
    "displayName": "John Doe",
    "roles": ["USER", "ADMIN"],
    "tenantId": "tenant456"
  }
}

// When not authenticated, returns 401:
{
  "error": "unauthorized",
  "error_description": "No authenticated user found"
}
```

The endpoint automatically extracts the authenticated user from the security context using Spring's `@AuthenticationPrincipal` annotation. It handles both authenticated and unauthenticated scenarios, returning appropriate HTTP status codes and error messages. This is commonly used by frontend applications to determine the current user's authentication state and profile information.
