A Spring REST controller that provides endpoints for managing user API keys, including creation, retrieval, updating, deletion, and regeneration operations.

## Key Components

- **ApiKeyController**: Main REST controller handling API key management endpoints
- **ApiKeyService**: Injected service handling business logic for API key operations
- **AuthPrincipal**: Security principal used to identify the authenticated user
- **DTOs**: Request/response objects including `CreateApiKeyRequest`, `UpdateApiKeyRequest`, `ApiKeyResponse`, and `CreateApiKeyResponse`

## Usage Example

```java
// The controller provides these REST endpoints:

// GET /api-keys - List all API keys for authenticated user
// POST /api-keys - Create new API key
{
    "name": "Production API Key",
    "description": "Key for production environment"
}

// GET /api-keys/{keyId} - Get specific API key
// PUT /api-keys/{keyId} - Update API key
{
    "name": "Updated API Key Name",
    "description": "Updated description"
}

// DELETE /api-keys/{keyId} - Delete API key (returns 204 No Content)
// POST /api-keys/{keyId}/regenerate - Generate new secret for existing key
```

All endpoints automatically use the authenticated user's ID from the security context, ensuring users can only manage their own API keys.
