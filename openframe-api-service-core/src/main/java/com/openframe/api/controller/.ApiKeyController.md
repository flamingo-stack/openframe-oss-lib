REST controller for managing user API keys in the OpenFrame application. Provides endpoints for creating, retrieving, updating, deleting, and regenerating API keys with user authentication.

## Key Components

- **ApiKeyController** - Main REST controller class handling API key operations
- **ApiKeyService** - Injected service for business logic operations
- **Authentication Integration** - Uses `@AuthenticationPrincipal` for user context
- **Request/Response DTOs** - Structured data transfer objects for API operations
- **Validation** - Input validation using `@Valid` annotations

## Usage Example

```java
// The controller automatically handles these HTTP requests:

// GET /api-keys - List all API keys for authenticated user
// Response: List<ApiKeyResponse>

// POST /api-keys - Create new API key
// Request body: {"name": "My API Key", "description": "For mobile app"}
// Response: CreateApiKeyResponse with generated key

// GET /api-keys/{keyId} - Get specific API key details
// Response: ApiKeyResponse

// PUT /api-keys/{keyId} - Update API key metadata  
// Request body: {"name": "Updated Name", "description": "New description"}
// Response: ApiKeyResponse

// DELETE /api-keys/{keyId} - Delete API key (204 No Content)

// POST /api-keys/{keyId}/regenerate - Generate new key value
// Response: CreateApiKeyResponse with new key
```

All endpoints require authentication and automatically associate operations with the authenticated user's ID. The controller includes comprehensive logging for debugging and audit purposes.
