Data Transfer Object (DTO) for configuring Single Sign-On (SSO) authentication settings, including client credentials, user provisioning rules, and domain restrictions.

## Key Components

- **`clientId`** - Required OAuth client identifier with validation
- **`clientSecret`** - Required OAuth client secret with validation  
- **`autoProvisionUsers`** - Boolean flag controlling automatic user creation
- **`msTenantId`** - Optional Microsoft tenant identifier for Azure AD integration
- **`allowedDomains`** - List of email domains permitted for auto-provisioning

## Usage Example

```java
// Configure SSO with auto-provisioning enabled
SSOConfigRequest ssoConfig = new SSOConfigRequest();
ssoConfig.setClientId("your-oauth-client-id");
ssoConfig.setClientSecret("your-oauth-client-secret");
ssoConfig.setAutoProvisionUsers(true);
ssoConfig.setAllowedDomains(List.of("company.com", "subsidiary.org"));

// For Microsoft Azure AD integration
ssoConfig.setMsTenantId("12345678-1234-1234-1234-123456789012");

// Disable auto-provisioning for existing users only
SSOConfigRequest restrictiveConfig = new SSOConfigRequest();
restrictiveConfig.setClientId("client-id");
restrictiveConfig.setClientSecret("client-secret");
restrictiveConfig.setAutoProvisionUsers(false);
```

The class uses Jakarta validation annotations to ensure required fields are provided and Lombok's `@Data` annotation for automatic getter/setter generation.
