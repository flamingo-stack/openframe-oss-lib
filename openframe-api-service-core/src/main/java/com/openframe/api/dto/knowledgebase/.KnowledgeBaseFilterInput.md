A DTO (Data Transfer Object) used to encapsulate filter criteria when querying knowledge base items in the OpenFrame API.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `parentId` | `String` | Filters items by their parent node ID |
| `type` | `KnowledgeBaseItemType` | Filters by item type (e.g., folder, article) |
| `tagIds` | `List<String>` | Filters items matching one or more tag IDs |

## Usage Example

```java
// Filter knowledge base articles under a specific parent with tags
KnowledgeBaseFilterInput filter = KnowledgeBaseFilterInput.builder()
    .parentId("folder-123")
    .type(KnowledgeBaseItemType.ARTICLE)
    .tagIds(List.of("tag-001", "tag-002"))
    .build();

// Or using no-args constructor
KnowledgeBaseFilterInput emptyFilter = new KnowledgeBaseFilterInput();
emptyFilter.setType(KnowledgeBaseItemType.FOLDER);
```

Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate getters, setters, builder pattern, and constructors, keeping the class concise.
