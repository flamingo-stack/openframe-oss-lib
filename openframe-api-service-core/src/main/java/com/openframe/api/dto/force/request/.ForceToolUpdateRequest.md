A request DTO for triggering forced tool updates across multiple machines in the OpenFrame system.

## Key Components

- **`machineIds`** - List of machine identifiers that should receive the tool update
- **`toolAgentId`** - Identifier of the specific tool agent to be updated
- **`@Data`** - Lombok annotation providing automatic getters, setters, equals, hashCode, and toString methods

## Usage Example

```java
// Create a force tool update request
ForceToolUpdateRequest request = new ForceToolUpdateRequest();
request.setMachineIds(List.of("machine-001", "machine-002", "machine-003"));
request.setToolAgentId("docker-agent-v2.1");

// Send via API client
apiClient.forceToolUpdate(request);

// Or build with constructor if available
ForceToolUpdateRequest request = new ForceToolUpdateRequest();
request.setMachineIds(Arrays.asList("prod-server-01", "prod-server-02"));
request.setToolAgentId("monitoring-agent-v1.5");
```

This DTO is typically used when administrators need to push critical tool updates to specific machines immediately, bypassing normal update schedules or when certain machines have failed to receive updates through regular channels.
