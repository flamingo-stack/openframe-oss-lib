A data transfer object (DTO) representing OIDC (OpenID Connect) UserInfo endpoint response data, supporting both successful user profile information and error responses.

## Key Components

- **User Identity Fields**: `sub` (subject identifier), `name`, `email` for basic user information
- **Name Components**: `givenName` and `familyName` with proper JSON property mapping
- **Verification Status**: `emailVerified` boolean flag for email validation status
- **Error Handling**: `error` and `errorDescription` fields for API error responses
- **Jackson Annotations**: `@JsonProperty` for snake_case JSON mapping and `@JsonInclude` to exclude null values
- **Lombok Integration**: `@Data` and `@Builder` for automatic getters, setters, and builder pattern

## Usage Example

```java
// Building a successful UserInfo response
UserInfo userInfo = UserInfo.builder()
    .sub("1234567890")
    .name("John Doe")
    .email("john.doe@example.com")
    .givenName("John")
    .familyName("Doe")
    .emailVerified(true)
    .build();

// Building an error response
UserInfo errorResponse = UserInfo.builder()
    .error("invalid_token")
    .errorDescription("The access token expired")
    .build();

// JSON serialization will map to OIDC standard format:
// {
//   "sub": "1234567890",
//   "given_name": "John",
//   "family_name": "Doe",
//   "email_verified": true
// }
```
