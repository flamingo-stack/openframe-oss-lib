A response DTO that encapsulates API key creation results, providing both the API key metadata and the complete key value needed for client authentication.

## Key Components

- **CreateApiKeyResponse** - Main record class annotated with `@Builder` for flexible object construction
- **apiKey** - `ApiKeyResponse` object containing API key metadata (ID, name, creation date, etc.)
- **fullKey** - `String` containing the complete API key value that clients use for authentication

## Usage Example

```java
// Creating a response after successful API key generation
CreateApiKeyResponse response = CreateApiKeyResponse.builder()
    .apiKey(ApiKeyResponse.builder()
        .id("key-123")
        .name("My API Key")
        .createdAt(Instant.now())
        .build())
    .fullKey("ak_live_1234567890abcdef")
    .build();

// Accessing the components
String keyForClient = response.fullKey(); // "ak_live_1234567890abcdef"
ApiKeyResponse metadata = response.apiKey(); // Contains ID, name, etc.

// In a REST controller
@PostMapping("/api-keys")
public ResponseEntity<CreateApiKeyResponse> createApiKey(@RequestBody CreateApiKeyRequest request) {
    CreateApiKeyResponse response = apiKeyService.createApiKey(request);
    return ResponseEntity.ok(response);
}
```

The `fullKey` field is typically only returned once during creation for security purposes, while `apiKey` contains the persistent metadata stored in the system.
