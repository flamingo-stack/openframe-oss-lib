A data transfer object that represents the response containing Single Sign-On (SSO) configuration status information, typically used in API responses to inform clients about current SSO settings.

## Key Components

- **enabled** - Boolean flag indicating whether SSO is currently enabled
- **provider** - String identifying the SSO provider (e.g., "okta", "azure", "google")
- **clientId** - String containing the client identifier for the SSO configuration
- **@Data** - Lombok annotation providing getters, setters, toString, equals, and hashCode methods
- **@Builder** - Lombok annotation enabling the builder pattern for object construction

## Usage Example

```java
// Creating an SSO config status response
SSOConfigStatusResponse response = SSOConfigStatusResponse.builder()
    .enabled(true)
    .provider("okta")
    .clientId("abc123-client-id")
    .build();

// Using in a REST controller
@GetMapping("/api/sso/status")
public SSOConfigStatusResponse getSSOStatus() {
    return SSOConfigStatusResponse.builder()
        .enabled(ssoService.isEnabled())
        .provider(ssoService.getProviderName())
        .clientId(ssoService.getClientId())
        .build();
}

// Accessing properties
if (response.isEnabled()) {
    String provider = response.getProvider();
    // Process SSO configuration
}
```
