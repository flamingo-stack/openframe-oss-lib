A data transfer object that represents the response structure for Single Sign-On (SSO) configuration details, containing provider settings, authentication parameters, and user provisioning options.

## Key Components

- **@Data/@Builder**: Lombok annotations for automatic getter/setter generation and builder pattern implementation
- **provider**: String identifying the SSO provider (e.g., "microsoft", "google", "okta")
- **clientId/clientSecret**: OAuth 2.0 client credentials for authentication
- **autoProvisionUsers**: Boolean flag controlling automatic user account creation
- **msTenantId**: Microsoft-specific tenant identifier for Azure AD integration
- **enabled**: Global toggle for the SSO configuration
- **allowedDomains**: List of email domains permitted for SSO authentication

## Usage Example

```java
// Building an SSO configuration response
SSOConfigResponse config = SSOConfigResponse.builder()
    .id("sso-001")
    .provider("microsoft")
    .clientId("your-client-id")
    .clientSecret("your-client-secret")
    .autoProvisionUsers(true)
    .msTenantId("tenant-uuid")
    .enabled(true)
    .allowedDomains(List.of("company.com", "subsidiary.com"))
    .build();

// Accessing configuration properties
if (config.isEnabled() && config.getAllowedDomains().contains("user@company.com")) {
    // Process SSO authentication
    String providerId = config.getProvider();
    boolean shouldProvision = config.isAutoProvisionUsers();
}
```
