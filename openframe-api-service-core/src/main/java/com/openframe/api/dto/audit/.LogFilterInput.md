A GraphQL input filter DTO for audit log operations, designed to provide comprehensive filtering capabilities for log queries with optimized schema compatibility.

## Key Components

- **Date Range Filtering**: `startDate` and `endDate` fields for temporal filtering using `LocalDate`
- **Event Classification**: `eventTypes` and `toolTypes` lists for filtering by specific log categories
- **Severity Filtering**: `severities` list for filtering logs by importance/criticality levels
- **Organization Scope**: `organizationIds` list for multi-tenant filtering capabilities
- **Device Targeting**: `deviceId` field for device-specific log retrieval
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic boilerplate generation

## Usage Example

```java
// Build a comprehensive log filter
LogFilterInput filter = LogFilterInput.builder()
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.now())
    .eventTypes(List.of("LOGIN", "ERROR", "CONFIGURATION_CHANGE"))
    .severities(List.of("HIGH", "CRITICAL"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use in GraphQL query service
List<AuditLog> filteredLogs = logService.findLogs(filter);

// Create minimal filter for specific device
LogFilterInput deviceFilter = LogFilterInput.builder()
    .deviceId("device-001")
    .startDate(LocalDate.now().minusDays(7))
    .build();
```
