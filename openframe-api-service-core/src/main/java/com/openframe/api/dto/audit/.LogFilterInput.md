A GraphQL input DTO class for filtering audit log queries with date ranges, event types, and organizational parameters.

## Key Components

- **LogFilterInput** - Main filter class with Lombok annotations for boilerplate code generation
- **Date Filtering** - `startDate` and `endDate` fields using `LocalDate` for time-based queries
- **Multi-value Filters** - List-based filters for `eventTypes`, `toolTypes`, `severities`, and `organizationIds`
- **Device Filtering** - Single `deviceId` field for device-specific log queries

## Usage Example

```java
// Create a filter for error logs from last week
LogFilterInput filter = LogFilterInput.builder()
    .startDate(LocalDate.now().minusWeeks(1))
    .endDate(LocalDate.now())
    .severities(List.of("ERROR", "CRITICAL"))
    .eventTypes(List.of("LOGIN_FAILED", "SYSTEM_ERROR"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Use in GraphQL query or service layer
List<AuditLog> logs = auditService.findLogs(filter);
```

The class leverages Lombok annotations to automatically generate constructors, getters, setters, and builder pattern methods, making it ideal for GraphQL input parameter binding and flexible query construction.
