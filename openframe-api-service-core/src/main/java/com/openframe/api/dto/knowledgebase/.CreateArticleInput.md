Data Transfer Object (DTO) for capturing input when creating a new knowledge base article, including content, metadata, status, and associated entity references.

## Key Components

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `name` | `String` | `@NotBlank` | Article title (required) |
| `parentId` | `String` | — | Parent article/folder ID for hierarchy |
| `content` | `String` | — | Full article body |
| `summary` | `String` | — | Brief article description |
| `status` | `KnowledgeBaseArticleStatus` | — | Publication status (e.g., draft, published) |
| `tagIds` | `List<String>` | `@Size(max=50)` | Associated tag identifiers |
| `assignedOrganizationIds` | `List<String>` | `@Size(max=50)` | Linked organization IDs |
| `assignedDeviceIds` | `List<String>` | `@Size(max=50)` | Linked device IDs |
| `assignedTicketIds` | `List<String>` | `@Size(max=50)` | Linked ticket IDs |
| `assignedKnowledgeArticleIds` | `List<String>` | `@Size(max=50)` | Related knowledge article IDs |

## Usage Example

```java
CreateArticleInput input = CreateArticleInput.builder()
    .name("VPN Setup Guide")
    .content("## Overview\nFollow these steps to configure VPN...")
    .summary("Step-by-step VPN configuration for end users")
    .status(KnowledgeBaseArticleStatus.PUBLISHED)
    .parentId("folder-abc123")
    .tagIds(List.of("tag-vpn", "tag-network"))
    .assignedOrganizationIds(List.of("org-001", "org-002"))
    .assignedTicketIds(List.of("ticket-789"))
    .build();
```

> **Validation:** `name` is mandatory (`@NotBlank`). All list fields are capped at **50 entries** each to prevent oversized payloads. The `status` field accepts values from the `KnowledgeBaseArticleStatus` enum, allowing control over article visibility at creation time.
