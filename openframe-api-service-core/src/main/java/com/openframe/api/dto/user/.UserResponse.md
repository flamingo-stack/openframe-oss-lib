Data Transfer Object (DTO) that represents a user response containing comprehensive user information including profile details, roles, status, and timestamps.

## Key Components

- **User Identification**: `id`, `email` fields for unique user identification
- **Profile Information**: `firstName`, `lastName`, `image` fields for user profile data
- **Authorization**: `roles` list containing user permission roles
- **Status Tracking**: `status` field indicating current user state
- **Audit Fields**: `createdAt`, `updatedAt` timestamps for record tracking
- **Lombok Annotations**: `@Data` and `@Builder` for automatic getters, setters, and builder pattern

## Usage Example

```java
// Building a user response
UserResponse user = UserResponse.builder()
    .id("user-123")
    .email("john.doe@example.com")
    .firstName("John")
    .lastName("Doe")
    .roles(List.of("USER", "ADMIN"))
    .status("ACTIVE")
    .image(ImageDto.builder().url("profile.jpg").build())
    .createdAt(LocalDateTime.now())
    .updatedAt(LocalDateTime.now())
    .build();

// Accessing user data
String fullName = user.getFirstName() + " " + user.getLastName();
boolean isAdmin = user.getRoles().contains("ADMIN");
```

This response DTO is typically used in REST API endpoints to return user information to clients, providing a clean separation between internal entity models and external API contracts.
