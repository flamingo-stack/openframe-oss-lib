A generic GraphQL-style connection wrapper that pairs a list of edges with pagination metadata, following the Relay connection specification pattern.

## Key Components

| Component | Type | Description |
|-----------|------|-------------|
| `edges` | `List<T>` | Collection of edge objects extending `GenericEdge` |
| `pageInfo` | `PageInfo` | Pagination metadata (cursors, hasNextPage, etc.) |

The class is parameterized with `T extends GenericEdge`, enforcing that all edge types conform to the base edge contract.

**Lombok annotations used:**
- `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString`
- `@SuperBuilder` — enables builder pattern with inheritance support
- `@NoArgsConstructor` / `@AllArgsConstructor` — generates both default and full constructors

## Usage Example

```java
// Build a paginated connection of ticket edges
GenericConnection<TicketEdge> connection = GenericConnection.<TicketEdge>builder()
    .edges(List.of(
        new TicketEdge("cursor-1", ticket1),
        new TicketEdge("cursor-2", ticket2)
    ))
    .pageInfo(PageInfo.builder()
        .hasNextPage(true)
        .endCursor("cursor-2")
        .build())
    .build();

// Access edges and page info
List<TicketEdge> edges = connection.getEdges();
PageInfo pageInfo = connection.getPageInfo();
boolean hasMore = pageInfo.isHasNextPage();
```

> **Note:** `@SuperBuilder` is used instead of `@Builder` to allow subclasses of `GenericConnection` to inherit and extend the builder pattern correctly.
