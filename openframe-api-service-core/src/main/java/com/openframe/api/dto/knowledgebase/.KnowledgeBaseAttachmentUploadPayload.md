DTO representing the response payload for a knowledge base attachment upload operation, encapsulating either a successful upload result or a list of user-facing errors.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `attachment` | `KnowledgeBaseItemAttachment` | The created attachment document returned on success |
| `uploadUrl` | `String` | Pre-signed or generated URL for the client to upload the file |
| `userErrors` | `List<MutationError>` | List of user-facing errors; defaults to empty list |

### Static Factory Methods

- **`success(attachment, uploadUrl)`** — Builds a payload with the attachment and upload URL, and no errors
- **`error(message)`** — Builds a payload with a single `MutationError` and null attachment/URL fields

## Usage Example

```java
// Success case
KnowledgeBaseItemAttachment attachment = attachmentService.create(metadata);
String uploadUrl = storageService.generateUploadUrl(attachment.getId());

KnowledgeBaseAttachmentUploadPayload payload =
    KnowledgeBaseAttachmentUploadPayload.success(attachment, uploadUrl);

// Error case
KnowledgeBaseAttachmentUploadPayload payload =
    KnowledgeBaseAttachmentUploadPayload.error("File type not supported.");

// Checking result
if (payload.getUserErrors().isEmpty()) {
    // proceed with uploadUrl
} else {
    payload.getUserErrors().forEach(e -> log.warn(e.getMessage()));
}
```

> This follows the GraphQL mutation payload pattern — callers should always check `userErrors` before consuming `attachment` or `uploadUrl`.
