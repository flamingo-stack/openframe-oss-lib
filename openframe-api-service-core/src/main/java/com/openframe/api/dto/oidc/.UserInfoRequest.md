A data transfer object (DTO) representing a request for user information in OIDC authentication flows.

## Key Components

- **UserInfoRequest**: A builder-pattern enabled DTO class with Lombok annotations
- **userId**: String identifier for the user
- **email**: User's email address
- **firstName**: User's first name
- **lastName**: User's last name

## Usage Example

```java
// Create a user info request using the builder pattern
UserInfoRequest request = UserInfoRequest.builder()
    .userId("user123")
    .email("john.doe@example.com")
    .firstName("John")
    .lastName("Doe")
    .build();

// Access user information
String userEmail = request.getEmail();
String fullName = request.getFirstName() + " " + request.getLastName();

// All fields are mutable due to @Data annotation
request.setEmail("newemail@example.com");
```

This DTO is typically used when requesting or transferring user profile information in OpenID Connect authentication workflows, providing a clean structure for user data with automatic getter/setter generation via Lombok's `@Data` annotation.
