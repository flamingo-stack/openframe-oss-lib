A data transfer object (DTO) that encapsulates filtering criteria for device queries, allowing clients to specify multiple filter parameters to narrow down device search results.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter devices by their operational status
- **deviceTypes** - List of `DeviceType` enums to filter by device categories
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization identifiers to scope devices by organization
- **tagNames** - List of tag name strings to filter devices by associated tags

## Usage Example

```java
// Create a comprehensive device filter
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ONLINE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.TABLET, DeviceType.DISPLAY))
    .osTypes(List.of("Android", "Linux"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("lobby", "conference-room"))
    .build();

// Use in device service queries
List<Device> filteredDevices = deviceService.findDevices(filter);

// Create status-only filter
DeviceFilterInput statusFilter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.OFFLINE))
    .build();
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation, making it ideal for flexible API queries and service layer filtering operations.
