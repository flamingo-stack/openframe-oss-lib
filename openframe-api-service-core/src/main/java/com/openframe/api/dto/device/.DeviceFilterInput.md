A DTO (Data Transfer Object) used to encapsulate filter criteria when querying devices in the OpenFrame API.

## Key Components

| Field | Type | Description |
|---|---|---|
| `statuses` | `List<DeviceStatus>` | Filter by one or more device status values |
| `deviceTypes` | `List<DeviceType>` | Filter by device type (e.g., workstation, server) |
| `osTypes` | `List<String>` | Filter by operating system type |
| `organizationIds` | `List<String>` | Scope results to specific organizations |
| `tagKeys` | `List<String>` | Filter by tag key names |
| `tagValues` | `List<String>` | Filter by tag value names |

Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) handle boilerplate for getters/setters, builder pattern, and constructors.

## Usage Example

```java
// Build a filter for active Windows workstations in a specific org
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.WORKSTATION))
    .osTypes(List.of("Windows"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagKeys(List.of("environment"))
    .tagValues(List.of("production"))
    .build();

// Pass to a device query service
List<Device> devices = deviceService.findDevices(filter);
```

> All fields are optional lists — pass only the criteria relevant to your query. Null fields are typically ignored by the underlying repository layer.
