A data transfer object (DTO) that encapsulates filtering criteria for device queries, allowing clients to specify multiple filter parameters to narrow down device search results.

## Key Components

- **statuses** - List of `DeviceStatus` enums to filter by device operational status
- **deviceTypes** - List of `DeviceType` enums to filter by hardware/device category
- **osTypes** - List of operating system types as strings
- **organizationIds** - List of organization identifiers for multi-tenant filtering
- **tagNames** - List of tag names for label-based filtering

## Usage Example

```java
// Create a filter for active Android devices in specific organizations
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.IDLE))
    .deviceTypes(List.of(DeviceType.MOBILE, DeviceType.TABLET))
    .osTypes(List.of("Android", "iOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "customer-facing"))
    .build();

// Use with device service
List<Device> filteredDevices = deviceService.findDevices(filter);
```
