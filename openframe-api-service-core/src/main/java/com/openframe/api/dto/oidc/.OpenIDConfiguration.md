A data transfer object that represents the OpenID Connect configuration metadata, containing endpoints and supported capabilities of an OIDC provider.

## Key Components

- **OpenIDConfiguration**: Main DTO class with builder pattern support
- **Core Endpoints**: `issuer`, `authorizationEndpoint`, `tokenEndpoint`, `userinfoEndpoint`, `jwksUri`
- **Capability Lists**: Various supported features like response types, subject types, signing algorithms, scopes, authentication methods, and claims
- **JSON Mapping**: Uses Jackson annotations for proper JSON serialization/deserialization

## Usage Example

```java
// Building a configuration object
OpenIDConfiguration config = OpenIDConfiguration.builder()
    .issuer("https://auth.example.com")
    .authorizationEndpoint("https://auth.example.com/oauth/authorize")
    .tokenEndpoint("https://auth.example.com/oauth/token")
    .userinfoEndpoint("https://auth.example.com/userinfo")
    .jwksUri("https://auth.example.com/.well-known/jwks.json")
    .responseTypesSupported(List.of("code", "token"))
    .subjectTypesSupported(List.of("public"))
    .scopesSupported(List.of("openid", "profile", "email"))
    .build();

// Accessing configuration data
String issuer = config.getIssuer();
List<String> scopes = config.getScopesSupported();
```

This class is typically used to parse and represent the `.well-known/openid_configuration` endpoint response from OIDC providers.
