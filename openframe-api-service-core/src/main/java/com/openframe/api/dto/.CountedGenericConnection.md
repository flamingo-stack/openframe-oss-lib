A specialized GraphQL connection type that extends `GenericConnection` to include filtered result counts, providing additional metadata for paginated queries with filtering applied.

## Key Components

- **CountedGenericConnection<T>**: Generic class extending `GenericConnection` for paginated results with count information
- **filteredCount**: Integer field tracking the total number of items matching the applied filters
- **Type Parameter T**: Bounded to classes extending `GenericEdge` for type-safe edge handling
- **Lombok Annotations**: Provides automatic generation of constructors, builders, and data methods

## Usage Example

```java
// Creating a counted connection for user edges
CountedGenericConnection<UserEdge> userConnection = CountedGenericConnection.<UserEdge>builder()
    .edges(userEdges)
    .pageInfo(pageInfo)
    .filteredCount(150)  // Total users matching filter criteria
    .build();

// Accessing filtered count for UI display
int totalFiltered = userConnection.getFilteredCount();
System.out.println("Showing " + userConnection.getEdges().size() + 
                   " of " + totalFiltered + " filtered results");

// Use in GraphQL resolver
@GraphQLQuery
public CountedGenericConnection<ProductEdge> getProducts(
    @GraphQLArgument("filter") ProductFilter filter,
    @GraphQLArgument("first") int first) {
    
    List<ProductEdge> edges = productService.getFilteredProducts(filter, first);
    int totalCount = productService.countFilteredProducts(filter);
    
    return CountedGenericConnection.<ProductEdge>builder()
        .edges(edges)
        .pageInfo(buildPageInfo(edges, first))
        .filteredCount(totalCount)
        .build();
}
```
