This class extends GraphQL connection functionality by adding a filtered count field to track the number of items that match filter criteria, useful for pagination with filtering scenarios.

## Key Components

- **CountedGenericConnection<T>**: Generic connection class that extends `GenericConnection<T>` where T must extend `GenericEdge`
- **filteredCount**: Integer field representing the count of items after filtering is applied
- **Lombok annotations**: Uses `@Data`, `@SuperBuilder`, `@NoArgsConstructor`, and `@AllArgsConstructor` for boilerplate code generation

## Usage Example

```java
// Create a counted connection for user edges
CountedGenericConnection<UserEdge> userConnection = CountedGenericConnection.<UserEdge>builder()
    .edges(userEdges)
    .pageInfo(pageInfo)
    .filteredCount(42)
    .build();

// Access the filtered count
int totalFiltered = userConnection.getFilteredCount();

// Use in GraphQL responses where you need both pagination and count info
public CountedGenericConnection<ProductEdge> getFilteredProducts(String filter, int first) {
    List<ProductEdge> edges = productService.getFilteredProducts(filter, first);
    int filteredCount = productService.getFilteredProductCount(filter);
    
    return CountedGenericConnection.<ProductEdge>builder()
        .edges(edges)
        .filteredCount(filteredCount)
        .build();
}
```
