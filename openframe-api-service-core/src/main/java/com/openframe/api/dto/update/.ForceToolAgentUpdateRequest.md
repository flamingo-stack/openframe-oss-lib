A request DTO for forcing tool agent updates across specified machines in the OpenFrame system.

## Key Components

- **machineIds** - List of machine identifiers that should receive the tool agent update
- **toolAgentId** - Identifier of the specific tool agent to be updated
- **@Data** - Lombok annotation providing automatic getter/setter generation

## Usage Example

```java
// Create request to update tool agent across multiple machines
ForceToolAgentUpdateRequest request = new ForceToolAgentUpdateRequest();
request.setMachineIds(Arrays.asList("machine-001", "machine-002", "machine-003"));
request.setToolAgentId("agent-v2.1.0");

// Send update request through API
updateService.forceToolAgentUpdate(request);
```

```java
// Builder pattern usage (if using Lombok's @Builder)
ForceToolAgentUpdateRequest request = ForceToolAgentUpdateRequest.builder()
    .machineIds(List.of("prod-server-01", "prod-server-02"))
    .toolAgentId("security-agent-1.5")
    .build();
```

This DTO is typically used in administrative operations where tool agents need to be forcibly updated across a fleet of machines, bypassing normal update schedules or mechanisms.
