A data transfer object (DTO) that defines filter criteria for querying and filtering tool entities in the API layer.

## Key Components

- **enabled**: Boolean flag to filter tools by their enabled/disabled status
- **type**: String field to filter tools by their specific type
- **category**: String field to filter tools by general category
- **platformCategory**: String field to filter tools by platform-specific category

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder pattern implementation.

## Usage Example

```java
// Create filter for enabled tools of a specific type
ToolFilterInput filter = ToolFilterInput.builder()
    .enabled(true)
    .type("deployment")
    .category("devops")
    .platformCategory("kubernetes")
    .build();

// Create filter with only category specified
ToolFilterInput categoryFilter = ToolFilterInput.builder()
    .category("monitoring")
    .build();

// Use default constructor and setters
ToolFilterInput manualFilter = new ToolFilterInput();
manualFilter.setEnabled(false);
manualFilter.setType("analytics");
```

This DTO is typically used as input for repository queries or service layer methods that need to filter tool collections based on multiple optional criteria.
