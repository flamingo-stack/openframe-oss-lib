A data transfer object (DTO) that encapsulates filter criteria for querying tools in the OpenFrame API. This class uses Lombok annotations to automatically generate boilerplate code for a clean, immutable-style data structure.

## Key Components

- **enabled** - Boolean flag to filter tools by their enabled/disabled status
- **type** - String field to filter tools by their specific type
- **category** - String field to filter tools by general category
- **platformCategory** - String field to filter tools by platform-specific category

## Usage Example

```java
// Create a filter to find enabled web development tools
ToolFilterInput filter = ToolFilterInput.builder()
    .enabled(true)
    .type("web")
    .category("development")
    .platformCategory("frontend")
    .build();

// Create a simple filter for disabled tools only
ToolFilterInput disabledFilter = ToolFilterInput.builder()
    .enabled(false)
    .build();

// Use with default constructor and setters
ToolFilterInput filter2 = new ToolFilterInput();
filter2.setEnabled(true);
filter2.setCategory("productivity");
```

The class leverages Lombok's `@Data` annotation to automatically generate getters, setters, `equals()`, `hashCode()`, and `toString()` methods, while `@Builder` provides a fluent API for object construction. All filter fields are optional, allowing for flexible query composition.
