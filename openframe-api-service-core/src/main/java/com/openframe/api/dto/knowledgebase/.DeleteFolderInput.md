Data Transfer Object (DTO) for capturing the input parameters required to delete a knowledge base folder, including how to handle its child items.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | (**Required**) The unique identifier of the folder to delete |
| `childrenAction` | `FolderChildrenAction` | Enum defining what to do with child items (e.g., delete or move) |
| `moveTargetFolderId` | `String` | Target folder ID when children are moved instead of deleted |

## Usage Example

```java
// Delete folder and move its children to another folder
DeleteFolderInput input = DeleteFolderInput.builder()
    .id("folder-123")
    .childrenAction(FolderChildrenAction.MOVE)
    .moveTargetFolderId("folder-456")
    .build();

// Delete folder and remove all children
DeleteFolderInput input = DeleteFolderInput.builder()
    .id("folder-123")
    .childrenAction(FolderChildrenAction.DELETE)
    .build();
```

## Notes

- `id` is validated with `@NotBlank` — an empty or null folder ID will fail validation
- `moveTargetFolderId` is only required when `childrenAction` is set to a move operation; it is ignored otherwise
- Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate boilerplate such as getters, setters, and constructors
