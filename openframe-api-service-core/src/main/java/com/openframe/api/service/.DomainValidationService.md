A Spring service that provides domain validation functionality, including checks for existing domains in the system and filtering out generic public email providers.

## Key Components

- **DomainValidationService**: Main service class with domain validation logic
- **GENERIC_PUBLIC_DOMAINS**: Static set containing 60+ popular public email domains (Gmail, Yahoo, Outlook, etc.)
- **validateExists()**: Validates that domains don't already exist in the system
- **validateGenericPublicDomain()**: Prevents registration of generic public domains
- **isGenericPublicDomain()**: Utility method to check if a domain is a public provider

## Usage Example

```java
@Autowired
private DomainValidationService domainValidationService;

// Check if domains already exist in system
List<String> newDomains = List.of("mycompany.com", "example.org");
domainValidationService.validateExists(newDomains);

// Prevent generic domains from being registered
List<String> domains = List.of("gmail.com", "mycompany.com");
domainValidationService.validateGenericPublicDomain(domains); // Throws exception

// Check individual domain
boolean isPublic = domainValidationService.isGenericPublicDomain("outlook.com"); // true
```

The service supports international domains including Chinese (qq.com), Korean (naver.com), German (web.de), and privacy-focused providers (protonmail.com). Both validation methods throw `IllegalArgumentException` when validation fails, making them suitable for use in business logic where domain restrictions are enforced.
