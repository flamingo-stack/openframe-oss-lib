Default implementation of the UserProcessor interface that provides basic logging functionality for user-related operations. This component serves as a fallback processor when no custom implementation is available.

## Key Components

- **DefaultUserProcessor** - Primary component implementing UserProcessor interface
- **postProcessUserDeleted()** - Handles post-processing after user deletion
- **postProcessUserGet()** - Handles post-processing after user retrieval
- **@ConditionalOnMissingBean** - Ensures this bean is only created when no other UserProcessor exists

## Usage Example

This processor is automatically registered as a Spring bean and used by the system:

```java
// The processor is automatically injected and used by user services
@Service
public class UserService {
    
    @Autowired
    private UserProcessor userProcessor; // Will be DefaultUserProcessor if no custom impl
    
    public void deleteUser(String userId) {
        User user = findUserById(userId);
        // ... deletion logic
        userProcessor.postProcessUserDeleted(user); // Logs: "User deleted: {userId}"
    }
    
    public UserPageResponse getUsers() {
        UserPageResponse response = fetchUsers();
        userProcessor.postProcessUserGet(response); // Logs: "Users fetched. Count: {size}"
        return response;
    }
}
```

The component uses conditional bean creation to allow custom processors to override the default behavior while ensuring a fallback implementation is always available.
