Default implementation of the UserProcessor interface that provides basic logging functionality for user-related operations in the OpenFrame API service. This component serves as a fallback processor when no custom implementation is available.

## Key Components

- **DefaultUserProcessor**: Default implementation of UserProcessor interface
- **postProcessUserDeleted()**: Logs user deletion events with user ID
- **postProcessUserGet()**: Logs user fetch operations with item count
- **postProcessUserUpdated()**: Logs user update events with user ID

## Usage Example

```java
// This component is automatically registered when no other UserProcessor exists
@Autowired
private UserProcessor userProcessor;

// The processor methods are called automatically by the service layer
public void deleteUser(String userId) {
    User user = userRepository.findById(userId);
    userRepository.delete(user);
    // DefaultUserProcessor.postProcessUserDeleted() is called automatically
    // Output: "User deleted: {userId}"
}

public UserPageResponse getUsers(Pageable pageable) {
    UserPageResponse response = userService.getUsers(pageable);
    // DefaultUserProcessor.postProcessUserGet() is called automatically
    // Output: "Users fetched. Count: {itemCount}"
    return response;
}
```

The `@ConditionalOnMissingBean` annotation ensures this default implementation is only used when no other UserProcessor bean is defined, allowing for easy customization in different environments or modules.
