A default implementation of the InvitationProcessor interface that provides basic logging functionality for invitation lifecycle events. This component serves as a fallback when no custom invitation processor is configured.

## Key Components

- **DefaultInvitationProcessor** - Default Spring component implementing InvitationProcessor
- **postProcessInvitationCreated()** - Logs invitation creation events at debug level
- **postProcessInvitationRevoked()** - Logs invitation revocation events at debug level
- **@ConditionalOnMissingBean** - Ensures this bean is only created when no other InvitationProcessor exists

## Usage Example

This component is automatically registered by Spring Boot when no custom InvitationProcessor is provided:

```java
// Custom implementation to override the default
@Component
public class CustomInvitationProcessor implements InvitationProcessor {
    
    @Override
    public void postProcessInvitationCreated(Invitation invitation) {
        // Send email notification
        emailService.sendInvitationEmail(invitation);
    }
    
    @Override
    public void postProcessInvitationRevoked(Invitation invitation) {
        // Send revocation notification
        emailService.sendRevocationEmail(invitation);
    }
}
```

The default processor will be automatically disabled when a custom implementation is present, thanks to the `@ConditionalOnMissingBean` annotation. It provides minimal overhead with debug-level logging for monitoring invitation activities.
