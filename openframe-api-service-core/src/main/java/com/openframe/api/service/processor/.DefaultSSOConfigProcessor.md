Default implementation of the SSOConfigProcessor interface that provides basic logging functionality for SSO configuration lifecycle events. This component serves as a fallback processor when no custom implementation is available.

## Key Components

- **DefaultSSOConfigProcessor**: Main component implementing `SSOConfigProcessor` interface
- **postProcessConfigSaved()**: Handles post-processing after SSO config is saved
- **postProcessConfigDeleted()**: Handles post-processing after SSO config is deleted  
- **postProcessConfigToggled()**: Handles post-processing after SSO config is enabled/disabled

## Usage Example

This component is automatically registered by Spring when no other `SSOConfigProcessor` implementation exists:

```java
// The component is auto-configured, but you can inject it if needed
@Autowired
private SSOConfigProcessor ssoProcessor;

// Example SSO config operations that trigger the processor
SSOConfig config = new SSOConfig();
config.setProvider("google");
config.setEnabled(true);

// These operations would trigger the corresponding processor methods
ssoConfigService.save(config);     // -> postProcessConfigSaved()
ssoConfigService.toggle(config);   // -> postProcessConfigToggled()  
ssoConfigService.delete(config);   // -> postProcessConfigDeleted()
```

To override with custom logic, simply create another `SSOConfigProcessor` bean - this default implementation will be automatically disabled due to the `@ConditionalOnMissingBean` annotation.
