Default implementation of the SSOConfigProcessor interface that provides basic logging functionality for SSO configuration events. This component serves as a fallback processor that will be automatically registered when no custom implementation is available.

## Key Components

- **DefaultSSOConfigProcessor** - Spring component implementing SSOConfigProcessor interface
- **@ConditionalOnMissingBean** - Ensures this bean is only created when no other SSOConfigProcessor exists
- **postProcessConfigSaved()** - Logs when SSO configurations are saved
- **postProcessConfigDeleted()** - Logs when SSO configurations are deleted  
- **postProcessConfigToggled()** - Logs when SSO configurations are enabled/disabled

## Usage Example

This component is automatically registered by Spring Boot when no custom SSOConfigProcessor is provided:

```java
// Default processor will be used automatically
@Autowired
private SSOConfigProcessor processor;

// When SSO config operations occur, logging happens automatically
SSOConfig config = new SSOConfig();
config.setProvider("google");
config.setEnabled(true);

// This will trigger postProcessConfigSaved() logging
processor.postProcessConfigSaved(config);
```

To override with a custom implementation:

```java
@Component
public class CustomSSOConfigProcessor implements SSOConfigProcessor {
    
    @Override
    public void postProcessConfigSaved(SSOConfig config) {
        // Custom logic here - this will replace DefaultSSOConfigProcessor
        sendNotificationToAdmins(config);
    }
    
    // Other methods...
}
```
