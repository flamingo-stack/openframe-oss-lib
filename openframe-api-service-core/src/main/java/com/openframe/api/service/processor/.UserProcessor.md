This interface defines processing hooks for user operations in the API service, designed to handle post-operation tasks like event publishing.

## Key Components

- **`postProcessUserDeleted(User user)`** - Processes actions after a user has been soft deleted
- **`postProcessUserGet(UserPageResponse response)`** - Processes actions after user data retrieval

## Usage Example

```java
@Component
public class KafkaUserProcessor implements UserProcessor {
    
    @Autowired
    private KafkaTemplate<String, Object> kafkaTemplate;
    
    @Override
    public void postProcessUserDeleted(User user) {
        // Publish user deletion event to Kafka
        kafkaTemplate.send("user-deleted-topic", user.getId(), user);
        logger.info("Published user deletion event for user: {}", user.getId());
    }
    
    @Override
    public void postProcessUserGet(UserPageResponse response) {
        // Log user access or update metrics
        logger.debug("User data retrieved: {} users", response.getUsers().size());
    }
}
```

The interface follows the processor pattern, allowing for clean separation of business logic from side effects like event publishing or logging.
