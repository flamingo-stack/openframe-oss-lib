A service processor interface that provides hooks for handling user lifecycle events in the OpenFrame API, designed to support future event publishing capabilities.

## Key Components

- **postProcessUserDeleted()** - Handles post-processing after user soft deletion
- **postProcessUserGet()** - Processes user retrieval operations with page response data
- **postProcessUserUpdated()** - Manages post-processing for user updates
- **User** - Domain object representing user entities
- **UserPageResponse** - DTO for paginated user response data

## Usage Example

```java
@Component
public class UserEventProcessor implements UserProcessor {
    
    @Override
    public void postProcessUserDeleted(User user) {
        // Log deletion event
        log.info("User {} has been soft deleted", user.getId());
        // Future: Publish to Kafka topic
        // kafkaTemplate.send("user.deleted", user);
    }
    
    @Override
    public void postProcessUserGet(UserPageResponse response) {
        // Track user access patterns
        analyticsService.recordUserAccess(response);
    }
    
    @Override
    public void postProcessUserUpdated(User user) {
        // Invalidate cache and notify systems
        cacheService.evictUser(user.getId());
        // Future: Publish update event
        // eventPublisher.publishUserUpdated(user);
    }
}
```

This interface follows the processor pattern to decouple user operations from side effects, enabling clean separation of concerns and future scalability through event-driven architecture.
