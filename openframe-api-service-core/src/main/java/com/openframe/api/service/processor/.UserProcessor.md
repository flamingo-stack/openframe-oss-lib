Interface defining post-processing hooks for user operations in the OpenFrame API service, enabling extensible handling of user lifecycle events such as deletion, retrieval, and updates.

## Key Components

| Method | Description |
|--------|-------------|
| `postProcessUserDeleted(User user)` | Triggered after a user is soft-deleted (status: `DELETED`) |
| `postProcessUserGet(UserPageResponse response)` | Triggered after a paginated list of users is retrieved |
| `postProcessUserGet(UserResponse response)` | Triggered after a single user is retrieved |
| `postProcessUserUpdated(User user)` | Triggered after a user record is updated |

## Usage Example

```java
@Component
public class KafkaUserProcessor implements UserProcessor {

    private final KafkaTemplate<String, Object> kafkaTemplate;

    @Override
    public void postProcessUserDeleted(User user) {
        kafkaTemplate.send("user-events", new UserDeletedEvent(user.getId()));
    }

    @Override
    public void postProcessUserGet(UserPageResponse response) {
        // e.g., log access audit trail
    }

    @Override
    public void postProcessUserGet(UserResponse response) {
        // e.g., track single user access metrics
    }

    @Override
    public void postProcessUserUpdated(User user) {
        kafkaTemplate.send("user-events", new UserUpdatedEvent(user.getId()));
    }
}
```

## Notes

- Designed for extensibility — implementations can range from no-ops to full event publishing (e.g., Kafka)
- Follows the **post-processing pattern**, decoupling business logic from side effects
- Overloaded `postProcessUserGet` handles both single-user and paginated responses independently
