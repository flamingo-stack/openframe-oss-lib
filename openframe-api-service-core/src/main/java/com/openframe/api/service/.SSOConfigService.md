Service class responsible for managing Single Sign-On (SSO) provider configurations, including CRUD operations, provider status management, and domain validation for auto-provisioning.

## Key Components

- **`getEnabledProviders()`** - Returns list of enabled SSO providers for login components
- **`getAvailableProviders()`** - Retrieves all available SSO provider configurations
- **`getConfig(String provider)`** - Fetches complete SSO configuration for admin forms with decrypted secrets
- **`upsertConfig(String provider, SSOConfigRequest request)`** - Creates or updates SSO configuration with validation
- **`deleteConfig(String provider)`** - Removes SSO configuration for specified provider
- **`toggleEnabled(String provider, boolean enabled)`** - Enables/disables SSO provider without modifying configuration
- **`validateAutoProvision()`** - Validates auto-provisioning settings and domain requirements

## Usage Example

```java
@RestController
@RequiredArgsConstructor
public class SSOController {
    private final SSOConfigService ssoConfigService;

    @GetMapping("/sso/providers/enabled")
    public List<SSOConfigStatusResponse> getEnabledProviders() {
        return ssoConfigService.getEnabledProviders();
    }

    @PostMapping("/admin/sso/{provider}")
    public SSOConfigResponse configureProvider(
            @PathVariable String provider,
            @Valid @RequestBody SSOConfigRequest request) {
        return ssoConfigService.upsertConfig(provider, request);
    }

    @PatchMapping("/admin/sso/{provider}/toggle")
    public void toggleProvider(@PathVariable String provider, 
                              @RequestParam boolean enabled) {
        ssoConfigService.toggleEnabled(provider, enabled);
    }
}
```

The service integrates encryption for client secrets, domain validation for security, and post-processing workflows for configuration changes. It provides comprehensive SSO management with special handling for Microsoft tenant configurations and auto-provisioning validation.
