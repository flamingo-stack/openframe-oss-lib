Service class that manages Single Sign-On (SSO) provider configurations, handling CRUD operations for SSO settings including client credentials, domain validation, and auto-provisioning features.

## Key Components

- **getEnabledProviders()** - Returns list of active SSO providers for login components
- **getAvailableProviders()** - Retrieves all available SSO provider options
- **getConfig(String provider)** - Fetches complete SSO configuration with decrypted secrets for admin forms
- **upsertConfig(String provider, SSOConfigRequest request)** - Creates or updates SSO configuration
- **deleteConfig(String provider)** - Removes SSO configuration for specified provider
- **toggleEnabled(String provider, boolean enabled)** - Enables/disables SSO provider
- **validateAutoProvision()** - Validates auto-provisioning settings and domain requirements

## Usage Example

```java
@Autowired
private SSOConfigService ssoConfigService;

// Get enabled providers for login page
List<SSOConfigStatusResponse> enabledProviders = ssoConfigService.getEnabledProviders();

// Configure Microsoft SSO with auto-provisioning
SSOConfigRequest request = SSOConfigRequest.builder()
    .clientId("your-client-id")
    .clientSecret("your-client-secret")
    .msTenantId("your-tenant-id")
    .autoProvisionUsers(true)
    .allowedDomains(List.of("company.com"))
    .build();

SSOConfigResponse config = ssoConfigService.upsertConfig("microsoft", request);

// Toggle provider status
ssoConfigService.toggleEnabled("microsoft", true);
```
