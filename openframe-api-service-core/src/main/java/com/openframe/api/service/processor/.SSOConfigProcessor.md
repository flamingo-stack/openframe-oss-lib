Processor interface for SSO configuration operations in API service, providing hooks for post-processing SSO config lifecycle events with default no-op implementations that can be extended for event publishing.

## Key Components

- **`postProcessConfigSaved(SSOConfig config)`** - Hook called after SSO configuration is created or updated
- **`postProcessConfigDeleted(SSOConfig config)`** - Hook called after SSO configuration is deleted
- **`postProcessConfigToggled(SSOConfig config)`** - Hook called after SSO configuration is enabled/disabled

All methods provide default empty implementations, allowing selective override of only needed operations.

## Usage Example

```java
@Component
public class KafkaSSOConfigProcessor implements SSOConfigProcessor {

    @Autowired
    private KafkaTemplate<String, Object> kafkaTemplate;

    @Override
    public void postProcessConfigSaved(SSOConfig config) {
        // Publish SSO config saved event to Kafka
        kafkaTemplate.send("sso-config-events", 
            new SSOConfigEvent("SAVED", config.getId(), config.getDomain()));
    }

    @Override
    public void postProcessConfigDeleted(SSOConfig config) {
        // Publish SSO config deleted event to Kafka
        kafkaTemplate.send("sso-config-events", 
            new SSOConfigEvent("DELETED", config.getId(), config.getDomain()));
    }
}
```

The interface follows the processor pattern, enabling loose coupling between SSO configuration operations and downstream processing like event publishing, logging, or cache invalidation.
