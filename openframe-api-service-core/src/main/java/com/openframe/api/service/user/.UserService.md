Manages user lifecycle operations including retrieval, updates, and soft deletion, with post-processing hooks applied via `UserProcessor` after each operation.

## Key Components

| Method | Description |
|---|---|
| `getUserByEmail(String email)` | Looks up a user by email, returning an `Optional<User>` |
| `existsActiveUserByEmail(String email)` | Checks whether an active user exists for a given email |
| `listUsers(int page, int size)` | Returns a paginated list of users mapped to `UserPageResponse` |
| `getUserById(String id)` | Fetches a single user by ID, throws `IllegalArgumentException` if not found |
| `getUsersByIds(Collection<String> ids)` | Batch-fetches users by a collection of IDs |
| `updateUser(String id, UpdateUserRequest request)` | Partially updates `firstName` and/or `lastName` fields |
| `softDeleteUser(String id, String requesterUserId)` | Marks a user as `DELETED` with self-delete and owner-delete guards |

## Usage Example

```java
// Paginate users
UserPageResponse page = userService.listUsers(0, 20);

// Fetch a specific user
UserResponse user = userService.getUserById("user-abc123");

// Update user profile
UpdateUserRequest req = new UpdateUserRequest();
req.setFirstName("Jane");
UserResponse updated = userService.updateUser("user-abc123", req);

// Soft delete (requester cannot delete themselves or an OWNER)
userService.softDeleteUser("user-xyz789", "requester-user-id");
```

## Guard Conditions in `softDeleteUser`

- **Self-delete**: Throws `UserSelfDeleteNotAllowedException` if the requester targets their own account
- **Owner protection**: Throws `OperationNotAllowedException` if the target user holds the `OWNER` role
- **Idempotency**: Skips persistence if the user is already in `DELETED` status
