A Spring service class that handles user management operations including retrieval, listing, and soft deletion with business rule validation.

## Key Components

- **getUserByEmail()** - Retrieves a user by email address
- **existsActiveUserByEmail()** - Checks if an active user exists with the given email
- **listUsers()** - Returns paginated user list with post-processing
- **softDeleteUser()** - Performs soft deletion with validation rules

## Usage Example

```java
@Autowired
private UserService userService;

// Check if user exists
boolean userExists = userService.existsActiveUserByEmail("user@example.com");

// Get paginated users
UserPageResponse users = userService.listUsers(0, 10);

// Soft delete user with validation
try {
    userService.softDeleteUser("userId123", "requesterId456");
} catch (UserSelfDeleteNotAllowedException e) {
    // Handle self-deletion attempt
} catch (OperationNotAllowedException e) {
    // Handle owner deletion attempt
}

// Find user by email
Optional<User> user = userService.getUserByEmail("user@example.com");
if (user.isPresent()) {
    // Process user
}
```

The service enforces business rules preventing self-deletion and owner account deletion, while supporting post-processing hooks through the UserProcessor component.
