A Spring service that handles forced updates of tool agents across machines in the OpenFrame system. It processes update requests for specific machines or all machines, publishing updates and tracking their status.

## Key Components

- **process()** - Processes tool agent updates for a specific list of machines
- **processAll()** - Processes tool agent updates for all machines in the system
- **processMachines()** - Internal method that maps machine processing operations
- **processMachine()** - Handles individual machine update processing and error handling
- **validateToolAgentId()** / **validateMachineIds()** - Input validation methods
- **buildResponseItem()** - Creates response items with machine ID, tool agent ID, and status

## Usage Example

```java
@Autowired
private ForceToolAgentUpdateService forceUpdateService;

// Update specific machines
ForceToolAgentUpdateRequest request = new ForceToolAgentUpdateRequest();
request.setToolAgentId("agent-123");
request.setMachineIds(Arrays.asList("machine-001", "machine-002"));

ForceToolAgentUpdateResponse response = forceUpdateService.process(request);

// Update all machines
ForceToolAgentUpdateResponse allResponse = forceUpdateService.processAll("agent-123");

// Check results
response.getItems().forEach(item -> {
    if (item.getStatus() == ForceAgentStatus.PROCESSED) {
        log.info("Successfully updated machine: {}", item.getMachineId());
    } else {
        log.error("Failed to update machine: {}", item.getMachineId());
    }
});
```

The service validates inputs, retrieves tool agent configurations, publishes updates via `ToolAgentUpdateUpdatePublisher`, and returns detailed status information for each processed machine. Failed updates are logged but don't stop processing of remaining machines.
