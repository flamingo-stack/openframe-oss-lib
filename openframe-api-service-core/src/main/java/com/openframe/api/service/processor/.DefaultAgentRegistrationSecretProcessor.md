Default fallback implementation of `AgentRegistrationSecretProcessor` that provides no-op lifecycle hooks for agent registration secret events, used automatically in OSS deployments when no custom processor bean is present.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `postProcessSecretGenerated` | Method | No-op hook called after a new agent registration secret is generated; logs a debug message |
| `postProcessSecretDeactivated` | Method | No-op hook called after an agent registration secret is deactivated; logs a debug message |

## Conditional Registration

This component uses `@ConditionalOnMissingBean` to register only when no other `AgentRegistrationSecretProcessor` implementation is present in the Spring context. Enterprise or custom deployments can override behavior by providing their own bean — this default will step aside automatically.

## Usage Example

```java
// OSS / default deployment — no custom processor needed.
// Spring auto-registers DefaultAgentRegistrationSecretProcessor.

// To override in a custom deployment, provide your own implementation:
@Component
public class CustomAgentRegistrationSecretProcessor
        implements AgentRegistrationSecretProcessor {

    @Override
    public void postProcessSecretGenerated(
            AgentRegistrationSecret secret, String decryptedKey) {
        // e.g. push secret metadata to an external vault
        vaultService.store(secret.getId(), decryptedKey);
    }

    @Override
    public void postProcessSecretDeactivated(AgentRegistrationSecret secret) {
        // e.g. revoke the secret in the external vault
        vaultService.revoke(secret.getId());
    }
}
```

> When a custom `AgentRegistrationSecretProcessor` bean is detected, this default implementation is skipped entirely — no additional configuration required.
