Implements a DGS `BatchLoader` that resolves multiple `Ticket` objects in a single database query, reducing N+1 query issues during GraphQL execution.

## Key Components

| Component | Description |
|---|---|
| `@DgsDataLoader(name = "ticketDataLoader")` | Registers this loader with the Netflix DGS framework under the name `ticketDataLoader` |
| `TicketRepository` | Injected repository used to fetch tickets via `findAllById` |
| `load(List<String> ticketIds)` | Core batch method; deduplicates IDs, fetches all tickets in one query, and maps results back to the original ID order |

## Usage Example

```java
// In a DGS data fetcher, inject the DataLoaderRegistry and retrieve the loader
@DgsData(parentType = "AssignableTarget", field = "ticket")
public CompletableFuture<Ticket> ticket(
        DataFetchingEnvironment env) {

    DataLoader<String, Ticket> loader =
            env.getDataLoader("ticketDataLoader");

    String ticketId = ((AssignableTarget) env.getSource()).getTicketId();
    return loader.load(ticketId);
}
```

## Behavior Notes

- **Null-safe**: `null` IDs in the input list are filtered before the DB query and mapped back to `null` in the result list, preserving positional alignment required by the `BatchLoader` contract.
- **Deduplication**: Uses a `Set` to avoid redundant DB lookups for repeated IDs.
- **Async**: Wraps execution in `CompletableFuture.supplyAsync` to avoid blocking the GraphQL execution thread.
