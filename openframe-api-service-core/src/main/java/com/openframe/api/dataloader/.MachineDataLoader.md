Implements a DGS `BatchLoader` that resolves multiple `Machine` documents in a single database query, used primarily for `AssignableTarget` polymorphic resolution when the target type is `DEVICE`.

## Key Components

| Component | Description |
|-----------|-------------|
| `@DgsDataLoader(name = "machineDataLoader")` | Registers this loader with the Netflix DGS framework under the name `machineDataLoader` |
| `BatchLoader<String, Machine>` | Accepts a list of `machineId` strings and returns the corresponding `Machine` documents in matching order |
| `MachineRepository` | MongoDB (or JPA) repository injected via Lombok `@RequiredArgsConstructor` |
| `load()` | Core batch method — deduplicates IDs, fetches all machines in one query, then maps results back to the original order preserving `null` for missing/null IDs |

## Behavior Notes

- **Null safety**: Filters out `null` IDs before querying, then re-inserts `null` at the corresponding positions in the result list — required by the `BatchLoader` contract.
- **Deduplication**: Uses a `Set` to avoid redundant DB lookups for repeated IDs.
- **Async execution**: Wraps work in `CompletableFuture.supplyAsync()` to avoid blocking the GraphQL request thread.

## Usage Example

```java
// Injected into a DGS data fetcher via DgsDataFetchingEnvironment
@DgsData(parentType = "AssignableTarget", field = "machine")
public CompletableFuture<Machine> getMachine(DgsDataFetchingEnvironment dfe) {
    DataLoader<String, Machine> loader =
        dfe.getDataLoader("machineDataLoader");

    String machineId = ((AssignableTarget) dfe.getSource()).getMachineId();
    return loader.load(machineId); // Batched across all sibling resolvers
}
```
