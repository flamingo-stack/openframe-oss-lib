Batch DataLoader for `KnowledgeBaseItem` entities, resolving multiple IDs in a single async database query to avoid N+1 problems in GraphQL.

## Key Components

| Component | Description |
|-----------|-------------|
| `@DgsDataLoader(name = "knowledgeBaseItemDataLoader")` | Registers this loader with the Netflix DGS framework under the key `knowledgeBaseItemDataLoader` |
| `BatchLoader<String, KnowledgeBaseItem>` | Accepts a list of string IDs and returns a corresponding ordered list of `KnowledgeBaseItem` objects |
| `load(List<String> itemIds)` | Core batch method — filters nulls, fetches all items in one `findByIdIn` call, then maps results back to the original ID order |
| `KnowledgeBaseItemRepository` | MongoDB/Spring Data repository used to perform the bulk lookup |

## Usage Example

```java
// In a DGS data fetcher, inject the DataLoader and use it for KNOWLEDGE_ARTICLE targets
@DgsComponent
public class AssignableTargetDataFetcher {

    @DgsData(parentType = "AssignableTarget", field = "knowledgeArticle")
    public CompletableFuture<KnowledgeBaseItem> knowledgeArticle(
            DgsDataFetchingEnvironment env) {

        String itemId = env.getSource(); // ID from parent resolver
        DataLoader<String, KnowledgeBaseItem> loader =
            env.getDataLoader("knowledgeBaseItemDataLoader");

        return loader.load(itemId); // Batched automatically across all concurrent resolvers
    }
}
```

## Behavior Notes

- **Null safety**: null IDs in the input list are preserved as `null` in the output list, maintaining positional alignment required by the `BatchLoader` contract.
- **Order preservation**: results are mapped back to the original `itemIds` order, which is required by the DataLoader specification.
- **Empty guard**: if all IDs are null, skips the DB call entirely and returns a null-filled list.
