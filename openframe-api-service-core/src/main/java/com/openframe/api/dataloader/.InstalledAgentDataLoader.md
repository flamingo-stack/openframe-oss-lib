A GraphQL DataLoader implementation that efficiently batches requests for installed agents by machine IDs to prevent N+1 query problems.

## Key Components

- **InstalledAgentDataLoader** - Main DataLoader class implementing `BatchLoader<String, List<InstalledAgent>>`
- **@DgsDataLoader** - Netflix DGS annotation registering this as "installedAgentDataLoader"
- **InstalledAgentService** - Injected service handling the actual data retrieval logic
- **load()** - Batching method that accepts multiple machine IDs and returns installed agents asynchronously

## Usage Example

```java
// In a GraphQL resolver
@DgsComponent
public class MachineDataFetcher {
    
    @DgsDataFetchingEnvironment
    public CompletableFuture<List<InstalledAgent>> installedAgents(
            DgsDataFetchingEnvironment dfe, 
            DataLoader<String, List<InstalledAgent>> dataLoader) {
        
        String machineId = dfe.getSource();
        return dataLoader.load(machineId);
    }
}

// DataLoader automatically batches multiple calls:
// - Individual calls: load("machine1"), load("machine2"), load("machine3")
// - Batched execution: load(["machine1", "machine2", "machine3"])
```

The DataLoader pattern prevents N+1 queries by collecting individual load requests during a single GraphQL execution cycle, then batching them into a single service call. This significantly improves performance when fetching installed agents for multiple machines simultaneously.
