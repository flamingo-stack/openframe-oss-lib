A DGS (Domain Graph Service) batch data loader that efficiently resolves `Tag` lists for multiple knowledge base items in a single batched GraphQL query.

## Key Components

| Component | Description |
|-----------|-------------|
| `@DgsDataLoader(name = "knowledgeBaseTagDataLoader")` | Registers this class as a named DGS data loader for use in GraphQL resolvers |
| `BatchLoader<String, List<Tag>>` | Accepts a list of item IDs (`String`) and returns a list of `Tag` collections per item |
| `KnowledgeBaseTagService` | Injected service that performs the underlying tag lookup by item IDs |
| `load(List<String> itemIds)` | Batches all pending item ID requests into a single async service call |

## Usage Example

Inject and use the data loader within a DGS data fetcher to avoid N+1 query problems:

```java
@DgsComponent
public class KnowledgeBaseDataFetcher {

    @DgsData(parentType = "KnowledgeBaseItem", field = "tags")
    public CompletableFuture<List<Tag>> getTags(
            DgsDataFetchingEnvironment dfe,
            DataFetchingEnvironment env) {

        DataLoader<String, List<Tag>> loader =
            dfe.getDataLoader("knowledgeBaseTagDataLoader");

        String itemId = dfe.<KnowledgeBaseItem>getSource().getId();
        return loader.load(itemId);
    }
}
```

> **Note:** The `BatchLoader` pattern consolidates all individual `load()` calls made during a single GraphQL request into one `getTagsByItemIds()` call, significantly reducing database round-trips when resolving tags across multiple knowledge base items.
