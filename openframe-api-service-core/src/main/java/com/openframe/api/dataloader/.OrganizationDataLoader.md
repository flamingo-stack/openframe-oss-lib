A GraphQL DataLoader implementation that efficiently batch loads Organization entities by their IDs, preventing N+1 query problems in GraphQL resolvers.

## Key Components

- **OrganizationDataLoader** - Main class implementing `BatchLoader<String, Organization>`
- **load()** - Batch loading method that accepts organization IDs and returns organizations
- **organizationRepository** - Injected repository for database access
- **@DgsDataLoader** - Netflix DGS annotation registering this as "organizationDataLoader"

## Usage Example

```java
// In a GraphQL resolver
@DgsComponent
public class MachineResolver {
    
    @DgsData(parentType = "Machine", field = "organization")
    public CompletableFuture<Organization> organization(
            DgsDataFetchingEnvironment dfe) {
        
        DataLoader<String, Organization> loader = 
            dfe.getDataLoader("organizationDataLoader");
        
        Machine machine = dfe.getSource();
        return loader.load(machine.getOrganizationId());
    }
}

// The DataLoader automatically batches multiple organization ID requests
// and filters out soft-deleted organizations, maintaining input order
```

The DataLoader handles null IDs gracefully, filters out soft-deleted organizations, and maintains the order of requested IDs in the response.
