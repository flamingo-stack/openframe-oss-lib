Batch DataLoader for resolving `Organization` objects by ID in a single database query, preventing N+1 query problems in GraphQL resolvers.

## Key Components

| Component | Description |
|-----------|-------------|
| `@DgsDataLoader(name = "organizationDataLoader")` | Registers this loader with the Netflix DGS framework |
| `BatchLoader<String, Organization>` | Processes a list of organization IDs and returns corresponding `Organization` objects |
| `load(List<String> organizationIds)` | Core batch method — deduplicates IDs, queries the repository once, filters to `ACTIVE` only, and returns results in input order |
| `OrganizationRepository` | Injected repository used for the batched `findByOrganizationIdIn` query |

## Key Behaviors

- **Null safety** — filters out null IDs before querying; maps nulls back in the result list
- **Deduplication** — uses a `Set` to avoid redundant DB lookups
- **Status filtering** — excludes soft-deleted organizations by only retaining `ACTIVE` status entries
- **Order preservation** — output list aligns 1:1 with the input ID list, as required by the `BatchLoader` contract

## Usage Example

```java
// In a DGS data fetcher, inject the DataLoader via context
@DgsData(parentType = "Machine", field = "organization")
public CompletableFuture<Organization> getOrganization(
        DataFetchingEnvironment env) {

    DataLoader<String, Organization> loader =
            env.getDataLoader("organizationDataLoader");

    String orgId = ((Machine) env.getSource()).getOrganizationId();
    return loader.load(orgId);
}
```

> When multiple machines reference the same organization IDs within a single GraphQL request, this loader batches all `load()` calls into one `findByOrganizationIdIn` query, significantly reducing database round-trips.
