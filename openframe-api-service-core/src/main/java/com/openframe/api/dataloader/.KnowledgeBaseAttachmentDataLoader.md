Registers a DGS batch data loader that efficiently fetches attachments for multiple Knowledge Base items in a single asynchronous call, avoiding N+1 query problems in GraphQL resolvers.

## Key Components

| Element | Description |
|---|---|
| `@DgsDataLoader("knowledgeBaseAttachmentDataLoader")` | Registers this class as a named DGS batch loader, referenced by GraphQL data fetchers |
| `BatchLoader<String, List<KnowledgeBaseItemAttachment>>` | Maps each `itemId` (String) to its list of attachments |
| `load(List<String> itemIds)` | Batch entry point — delegates to `KnowledgeBaseAttachmentService` asynchronously |
| `KnowledgeBaseAttachmentService` | Service responsible for querying attachments by article IDs |

## Usage Example

```java
// In a DGS data fetcher, inject the DataLoaderRegistry and load attachments per item
@DgsData(parentType = "KnowledgeBaseItem", field = "attachments")
public CompletableFuture<List<KnowledgeBaseItemAttachment>> attachments(
        DgsDataFetchingEnvironment dfe) {

    DataLoader<String, List<KnowledgeBaseItemAttachment>> loader =
            dfe.getDataLoader("knowledgeBaseAttachmentDataLoader");

    KnowledgeBaseItem item = dfe.getSource();
    return loader.load(item.getId());
}
```

> Calls to `loader.load(id)` across multiple sibling resolvers are automatically batched into a single `load(List<String> itemIds)` invocation, reducing database round-trips.
