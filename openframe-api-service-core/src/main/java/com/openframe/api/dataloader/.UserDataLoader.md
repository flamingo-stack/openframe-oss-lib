Batch DataLoader that resolves `UserResponse` objects by ID in a single async operation, used to avoid N+1 query problems when loading `KnowledgeBaseItem.author` fields in GraphQL.

## Key Components

| Component | Description |
|-----------|-------------|
| `@DgsDataLoader(name = "userDataLoader")` | Registers this loader with the DGS framework under the key `"userDataLoader"` |
| `BatchLoader<String, UserResponse>` | Accepts a list of user ID strings and returns a positionally-matched list of `UserResponse` objects |
| `load(List<String> userIds)` | Core batch method — deduplicates IDs, fetches via `UserService`, and maps results back to the original order |
| `UserService.getUsersByIds()` | Underlying service call; routes through any registered `UserProcessor` (e.g. SaaS enrichment) to attach image data |

## Usage Example

```java
// In a DGS field resolver for KnowledgeBaseItem.author
@DgsData(parentType = "KnowledgeBaseItem", field = "author")
public CompletableFuture<UserResponse> getAuthor(
        DgsDataFetchingEnvironment dfe,
        DataFetchingEnvironment env) {

    String authorId = ((KnowledgeBaseItem) env.getSource()).getAuthorId();

    DataLoader<String, UserResponse> loader =
            dfe.getDataLoader("userDataLoader");

    return loader.load(authorId);
}
```

## Behavior Notes

- **Null-safe**: null IDs are filtered before the service call; the result list preserves `null` at those positions to maintain index alignment required by the `BatchLoader` contract.
- **Deduplication**: IDs are collected into a `Set` before querying, preventing duplicate DB lookups within the same batch window.
- **Enrichment**: Routing through `UserService` ensures any registered `UserProcessor` (e.g. avatar/image injection in a SaaS deployment) is applied consistently.
