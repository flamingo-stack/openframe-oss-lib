A GraphQL DataLoader implementation that efficiently batches and caches tool connection lookups for multiple machines, preventing N+1 query problems when resolving tool connections in GraphQL queries.

## Key Components

- **@DgsDataLoader Annotation**: Registers this class as a Netflix DGS DataLoader with name "toolConnectionDataLoader"
- **BatchLoader Interface**: Implements `BatchLoader<String, List<ToolConnection>>` to batch machine ID lookups
- **ToolConnectionService**: Injected service dependency that handles the actual data retrieval
- **load() Method**: Asynchronous batch loading method that takes machine IDs and returns tool connections

## Usage Example

This DataLoader is automatically registered with the GraphQL context and can be used in field resolvers:

```java
@DgsComponent
public class MachineDataFetcher {
    
    @DgsData(parentType = "Machine", field = "toolConnections")
    public CompletableFuture<List<ToolConnection>> getToolConnections(
            @DgsDataFetchingEnvironment DgsDataFetchingEnvironment dfe) {
        
        Machine machine = dfe.getSource();
        DataLoader<String, List<ToolConnection>> loader = 
            dfe.getDataLoader("toolConnectionDataLoader");
        
        return loader.load(machine.getId());
    }
}
```

The DataLoader batches multiple machine ID requests into a single service call, improving performance when loading tool connections for multiple machines in a single GraphQL query. The `@RequiredArgsConstructor` annotation generates a constructor that injects the required `ToolConnectionService` dependency.
