A GraphQL DataLoader implementation that efficiently batches and loads tags associated with multiple machines to prevent N+1 query problems.

## Key Components

- **TagDataLoader**: Main DataLoader class implementing `BatchLoader<String, List<Tag>>`
- **@DgsDataLoader**: Netflix DGS annotation registering this as "tagDataLoader"
- **TagService**: Injected service handling the actual tag retrieval logic
- **load()**: Asynchronous method that batches machine IDs and returns their associated tags

## Usage Example

```java
// In a GraphQL resolver
@DgsComponent
public class MachineResolver {
    
    @DgsData(parentType = "Machine")
    public CompletableFuture<List<Tag>> tags(DgsDataFetchingEnvironment dfe) {
        DataLoader<String, List<Tag>> tagLoader = dfe.getDataLoader("tagDataLoader");
        String machineId = dfe.getSource().getId();
        return tagLoader.load(machineId);
    }
}

// DataLoader automatically batches multiple requests:
// Instead of N individual calls to TagService.getTagsForMachine(id)
// Makes 1 batched call to TagService.getTagsForMachines(List<String> ids)
```

The DataLoader collects multiple individual machine ID requests within a single execution cycle and batches them into one service call, significantly improving performance when fetching tags for multiple machines simultaneously.
