A GraphQL DataLoader implementation that efficiently batches and loads tags for multiple machines to prevent N+1 query problems.

## Key Components

- **@DgsDataLoader**: Netflix DGS annotation registering this as a GraphQL data loader with name "tagDataLoader"
- **BatchLoader Interface**: Implements DataLoader pattern for batching multiple tag requests into single operations
- **TagService Integration**: Uses injected TagService to fetch tags for multiple machine IDs in one call
- **Asynchronous Processing**: Returns CompletableFuture for non-blocking tag retrieval

## Usage Example

```java
// In a GraphQL resolver
@DgsComponent
public class MachineResolver {
    
    @DgsData(parentType = "Machine", field = "tags")
    public CompletableFuture<List<Tag>> getTags(DgsDataFetchingEnvironment dfe) {
        DataLoader<String, List<Tag>> loader = dfe.getDataLoader("tagDataLoader");
        Machine machine = dfe.getSource();
        return loader.load(machine.getId());
    }
}

// The DataLoader will automatically batch multiple requests:
// Instead of: N separate calls to tagService.getTagsForMachine(id)
// Executes: 1 call to tagService.getTagsForMachines(List<String> ids)
```

This implementation optimizes database queries by collecting multiple tag requests during GraphQL execution and resolving them in a single batch operation, significantly improving performance when fetching tags for multiple machines.
