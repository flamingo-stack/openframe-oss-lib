A DGS batch data loader that resolves `Tag` lists for multiple machine IDs in a single async operation, reducing N+1 query issues in GraphQL resolvers.

## Key Components

| Element | Description |
|---|---|
| `@DgsDataLoader(name = "tagDataLoader")` | Registers this class as a named DGS data loader, available for injection in GraphQL fetchers |
| `BatchLoader<String, List<Tag>>` | Batches requests keyed by `machineId` (String), returning a `List<Tag>` per machine |
| `load(List<String> machineIds)` | Collects all machine ID keys from a single GraphQL execution and delegates to `TagService` in one call |
| `TagService` | Injected service responsible for the underlying tag query logic |

## Usage Example

```java
// In a DGS data fetcher, inject and use the loader by name
@DgsComponent
public class MachineDataFetcher {

    @DgsData(parentType = "Machine", field = "tags")
    public CompletableFuture<List<Tag>> tags(
            DgsDataFetchingEnvironment dfe,
            DataFetchingEnvironment env) {

        DataLoader<String, List<Tag>> tagLoader =
                dfe.getDataLoader("tagDataLoader");

        String machineId = dfe.<Machine>getSource().getId();
        return tagLoader.load(machineId);
    }
}
```

> **Note:** The loader batches all `machineId` keys accumulated during a single GraphQL request tick, then calls `tagService.getTagsForMachines(machineIds)` once — replacing N individual tag lookups with a single bulk fetch.
