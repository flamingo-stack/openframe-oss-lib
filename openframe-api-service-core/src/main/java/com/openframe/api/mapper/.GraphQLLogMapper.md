Maps audit log GraphQL inputs and query results to domain-level filter criteria, pagination parameters, and cursor-paginated connection responses.

## Key Components

| Method | Description |
|--------|-------------|
| `toLogFilterCriteria(LogFilterInput)` | Converts a `LogFilterInput` GraphQL DTO into a `LogFilterCriteria` domain object; returns an empty criteria if input is `null` |
| `toCursorPaginationCriteria(ConnectionArgs)` | Delegates to `CursorPaginationCriteria.fromConnectionArgs` to build pagination settings from GraphQL connection arguments |
| `toLogConnection(GenericQueryResult<LogEvent>)` | Transforms a paginated query result into a `GenericConnection` of `LogEvent` edges with encoded cursors and page info |
| `createLogCursor(LogEvent)` *(private)* | Builds a stable cursor string from the event's epoch-millisecond timestamp and `toolEventId`, separated by `_` |

## Usage Example

```java
@Autowired
private GraphQLLogMapper logMapper;

// Convert filter input from GraphQL resolver
LogFilterCriteria criteria = logMapper.toLogFilterCriteria(filterInput);

// Build pagination settings
CursorPaginationCriteria pagination = logMapper.toCursorPaginationCriteria(connectionArgs);

// Query your service
GenericQueryResult<LogEvent> result = logService.query(criteria, pagination);

// Map to GraphQL-ready connection response
GenericConnection<GenericEdge<LogEvent>> connection = logMapper.toLogConnection(result);
```

> **Cursor format:** `{epochMillis}_{toolEventId}` — encoded via `CursorCodec.encode` to produce an opaque, Base64-safe cursor string safe for use in GraphQL pagination.
