Maps GraphQL input types to internal domain commands and query criteria for the Knowledge Base module, handling Relay global ID decoding and cursor-based pagination translation.

## Key Components

| Method | Description |
|--------|-------------|
| `toFilterCriteria` | Converts `KnowledgeBaseFilterInput` to `KnowledgeBaseFilterCriteria`, decoding Relay global IDs for parent and tag filters |
| `toCursorPaginationCriteria` | Translates `ConnectionArgs` into `CursorPaginationCriteria` for paginated queries |
| `toItemConnection` | Wraps a `CountedGenericQueryResult` into a Relay-compatible `CountedGenericConnection` with encoded cursors |
| `toCreateCommand` | Maps `CreateArticleInput` to `CreateArticleCommand`, decoding all relational IDs (tags, organizations, devices, tickets, related articles) |
| `toUpdateCommand` | Maps `UpdateArticleInput` to `UpdateArticleCommand` with decoded IDs |
| `decodeId` / `decodeIds` | Private helpers that decode Relay global IDs using `graphql-java`'s `Relay` utility |

## Usage Example

```java
@Service
public class KnowledgeBaseGraphQLService {

    private final GraphQLKnowledgeBaseMapper mapper;

    // Convert GraphQL input to domain filter criteria
    public CountedGenericConnection<GenericEdge<KnowledgeBaseItem>> listArticles(
            KnowledgeBaseFilterInput filterInput,
            ConnectionArgs connectionArgs) {

        KnowledgeBaseFilterCriteria criteria = mapper.toFilterCriteria(filterInput);
        CursorPaginationCriteria pagination = mapper.toCursorPaginationCriteria(connectionArgs);

        CountedGenericQueryResult<KnowledgeBaseItem> result =
            knowledgeBaseService.findAll(criteria, pagination);

        return mapper.toItemConnection(result);
    }

    // Convert GraphQL mutation input to a create command
    public KnowledgeBaseItem createArticle(CreateArticleInput input) {
        CreateArticleCommand command = mapper.toCreateCommand(input);
        return knowledgeBaseService.create(command);
    }
}
```

> All ID fields are treated as Relay global IDs and decoded to internal IDs before being passed downstream, ensuring the GraphQL layer remains decoupled from persistence-layer identifiers.
