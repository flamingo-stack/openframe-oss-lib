Maps notification domain objects to GraphQL-compatible DTOs, handling cursor pagination and connection structures for notification queries.

## Key Components

| Method | Description |
|--------|-------------|
| `toCursorPaginationCriteria(ConnectionArgs)` | Converts GraphQL connection arguments into cursor-based pagination criteria |
| `toView(Notification, boolean)` | Maps a `Notification` document to a `NotificationView` DTO, including read state |
| `toConnection(GenericQueryResult<NotificationView>)` | Wraps a paginated result into a Relay-style `GenericConnection` with encoded cursors and page info |

## Usage Example

```java
@Component
public class NotificationResolver {

    private final GraphQLNotificationMapper mapper;

    public GenericConnection<GenericEdge<NotificationView>> getNotifications(
            ConnectionArgs args, String userId) {

        // Convert GraphQL args to pagination criteria
        CursorPaginationCriteria criteria = mapper.toCursorPaginationCriteria(args);

        // Fetch paginated notifications from service
        GenericQueryResult<NotificationView> result = notificationService
                .findByUser(userId, criteria);

        // Return Relay-style connection
        return mapper.toConnection(result);
    }

    public NotificationView getSingleNotification(Notification notification) {
        return mapper.toView(notification, /* read= */ true);
    }
}
```

> **Note:** Cursors are base64-encoded via `CursorCodec.encode()` using the notification's ID, enabling stateless pagination across requests.
