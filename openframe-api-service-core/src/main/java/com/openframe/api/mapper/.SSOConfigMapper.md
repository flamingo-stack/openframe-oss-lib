A Spring component that handles mapping between SSO configuration DTOs and entities, with built-in encryption support for sensitive data like client secrets.

## Key Components

- **`toEntity()`** - Converts `SSOConfigRequest` to `SSOConfig` entity with encrypted client secret
- **`updateEntity()`** - Updates existing `SSOConfig` entity from request data (excludes provider and enabled fields)
- **`toResponse()`** - Converts `SSOConfig` entity to `SSOConfigResponse` DTO with decrypted client secret

## Usage Example

```java
@Service
public class SSOConfigService {
    
    @Autowired
    private SSOConfigMapper mapper;
    
    @Autowired
    private EncryptionService encryptionService;
    
    public SSOConfig createConfig(String provider, SSOConfigRequest request) {
        // Create new SSO configuration with encrypted secrets
        return mapper.toEntity(provider, request, encryptionService);
    }
    
    public void updateConfig(SSOConfig existing, SSOConfigRequest request) {
        // Update existing configuration while preserving provider and enabled status
        mapper.updateEntity(existing, request, encryptionService);
    }
    
    public SSOConfigResponse getConfigResponse(SSOConfig entity) {
        // Convert to response DTO with decrypted client secret
        String decryptedSecret = encryptionService.decryptClientSecret(entity.getClientSecret());
        return mapper.toResponse(entity, decryptedSecret);
    }
}
```

The mapper ensures client secrets are always encrypted when persisted and handles the Microsoft tenant ID and domain restrictions for SSO providers.
