A Spring component that handles mapping between SSO configuration DTOs and domain entities, with built-in client secret encryption support.

## Key Components

- **`toEntity(String, SSOConfigRequest, EncryptionService)`** - Creates a new SSOConfig entity from request data with encrypted client secret
- **`updateEntity(SSOConfig, SSOConfigRequest, EncryptionService)`** - Updates an existing SSOConfig entity with new request data
- **`toResponse(SSOConfig, String)`** - Converts SSOConfig entity to response DTO with decrypted client secret

## Usage Example

```java
@Service
public class SSOConfigService {
    
    @Autowired
    private SSOConfigMapper mapper;
    
    @Autowired 
    private EncryptionService encryptionService;
    
    public SSOConfig createConfig(String provider, SSOConfigRequest request) {
        // Create new SSO configuration
        return mapper.toEntity(provider, request, encryptionService);
    }
    
    public void updateConfig(SSOConfig existing, SSOConfigRequest request) {
        // Update existing configuration
        mapper.updateEntity(existing, request, encryptionService);
    }
    
    public SSOConfigResponse getConfigResponse(SSOConfig entity) {
        String decryptedSecret = encryptionService.decrypt(entity.getClientSecret());
        return mapper.toResponse(entity, decryptedSecret);
    }
}
```

The mapper automatically encrypts client secrets when creating/updating entities and requires decrypted secrets for response mapping.
