Maps GraphQL device query inputs to internal service criteria and transforms paginated `Machine` query results into GraphQL-compatible connection responses.

## Key Components

| Method | Description |
|--------|-------------|
| `toDeviceFilterCriteria(DeviceFilterInput)` | Converts a GraphQL filter input (statuses, device types, OS types, organization IDs, tags) into a `DeviceFilterCriteria` domain object; returns an empty criteria if input is `null` |
| `toCursorPaginationCriteria(ConnectionArgs)` | Delegates to `CursorPaginationCriteria.fromConnectionArgs()` to build cursor-based pagination parameters from GraphQL connection arguments |
| `toDeviceConnection(CountedGenericQueryResult<Machine>)` | Transforms a paginated query result into a `CountedGenericConnection`, encoding each `Machine` ID as a Base64 cursor and attaching page info and filtered count |

## Usage Example

```java
@Component
public class DeviceQueryResolver {

    private final GraphQLDeviceMapper mapper;
    private final DeviceService deviceService;

    public CountedGenericConnection<GenericEdge<Machine>> devices(
            DeviceFilterInput filter,
            ConnectionArgs args) {

        DeviceFilterCriteria criteria = mapper.toDeviceFilterCriteria(filter);
        CursorPaginationCriteria pagination = mapper.toCursorPaginationCriteria(args);

        CountedGenericQueryResult<Machine> result =
                deviceService.findDevices(criteria, pagination);

        return mapper.toDeviceConnection(result);
    }
}
```

> **Note:** Cursor values are Base64-encoded `Machine` IDs via `CursorCodec.encode()`, enabling stable, opaque pagination cursors across pages.
