Maps GraphQL event-related inputs and query results to internal DTOs, handling filter criteria, cursor pagination, and connection-style responses for the OpenFrame API.

## Key Components

| Method | Description |
|--------|-------------|
| `toEventFilterCriteria(EventFilterInput)` | Converts a GraphQL `EventFilterInput` into an `EventFilterCriteria` domain object; returns an empty criteria if input is `null` |
| `toCursorPaginationCriteria(ConnectionArgs)` | Delegates to `CursorPaginationCriteria.fromConnectionArgs()` to build pagination settings from GraphQL connection arguments |
| `toEventConnection(GenericQueryResult<Event>)` | Transforms a paginated query result into a Relay-style `GenericConnection`, encoding each event's ID as a Base64 cursor |

## Usage Example

```java
@Autowired
private GraphQLEventMapper eventMapper;

// Convert filter input
EventFilterInput input = new EventFilterInput();
input.setUserIds(List.of("user-1", "user-2"));
input.setEventTypes(List.of("LOGIN", "LOGOUT"));
input.setStartDate(LocalDate.now().minusDays(7));
EventFilterCriteria criteria = eventMapper.toEventFilterCriteria(input);

// Convert pagination args
ConnectionArgs args = new ConnectionArgs(10, null, null, null);
CursorPaginationCriteria pagination = eventMapper.toCursorPaginationCriteria(args);

// Map query result to GraphQL connection
GenericQueryResult<Event> queryResult = eventService.findEvents(criteria, pagination);
GenericConnection<GenericEdge<Event>> connection = eventMapper.toEventConnection(queryResult);
```

> **Note:** Null-safe by design — both `toEventFilterCriteria` and `toEventConnection` return safe empty defaults when given a `null` input, preventing NullPointerExceptions in resolver chains.
