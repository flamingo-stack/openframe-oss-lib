Converts GraphQL organization-related inputs into internal DTOs and maps paginated query results into GraphQL-compatible connection responses.

## Key Components

| Method | Description |
|--------|-------------|
| `toFilterOptions(OrganizationFilterInput)` | Maps GraphQL filter input fields (category, employee range, contract status, etc.) to `OrganizationFilterOptions` |
| `toCursorPaginationCriteria(ConnectionArgs)` | Delegates pagination args to `CursorPaginationCriteria.fromConnectionArgs()` |
| `toOrganizationConnection(CountedGenericQueryResult)` | Transforms a paginated query result into a `CountedGenericConnection` with cursor-encoded edges |

## Usage Example

```java
@Component
public class OrganizationResolver {

    private final GraphQLOrganizationMapper mapper;
    private final OrganizationService service;

    public CountedGenericConnection<GenericEdge<Organization>> organizations(
            OrganizationFilterInput filter,
            ConnectionArgs args) {

        OrganizationFilterOptions options = mapper.toFilterOptions(filter);
        CursorPaginationCriteria pagination = mapper.toCursorPaginationCriteria(args);

        CountedGenericQueryResult<Organization> result =
                service.findOrganizations(options, pagination);

        return mapper.toOrganizationConnection(result);
    }
}
```

## Notes

- Returns `null` from `toFilterOptions()` when input is `null`, allowing optional filter arguments in GraphQL queries
- Each edge cursor is Base64-encoded via `CursorCodec.encode()` using the organization's ID, enabling relay-style cursor pagination
- Registered as a Spring `@Component` for dependency injection across resolvers
