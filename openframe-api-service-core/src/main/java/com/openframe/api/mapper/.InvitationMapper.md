A Spring component that handles mapping between invitation DTOs and data entities, with built-in expiration logic and role conversion capabilities.

## Key Components

- **`toEntity(CreateInvitationRequest)`** - Converts creation requests to Invitation entities with auto-generated UUID, normalized email, default admin role, and TTL-based expiration
- **`toResponse(Invitation)`** - Converts entities to response DTOs with real-time expiration status checking
- **`@Value("${openframe.invitations.ttl-hours:24h}")`** - Configurable invitation time-to-live with 24-hour default
- **Role Mapping** - Bidirectional conversion between DTO and entity role enums
- **Status Logic** - Automatically marks pending invitations as expired when past expiration time

## Usage Example

```java
@Autowired
private InvitationMapper invitationMapper;

// Create invitation entity from request
CreateInvitationRequest request = CreateInvitationRequest.builder()
    .email("user@example.com")
    .roles(List.of(Role.USER, Role.MODERATOR))
    .build();

Invitation entity = invitationMapper.toEntity(request);
// Results in: normalized email, UUID, 24h expiration, PENDING status

// Convert to response with expiration check
InvitationResponse response = invitationMapper.toResponse(entity);
// Automatically shows EXPIRED status if past expiration time
```

The mapper ensures data consistency by normalizing emails to lowercase, providing sensible defaults (admin role, pending status), and performing real-time expiration validation during response conversion.
