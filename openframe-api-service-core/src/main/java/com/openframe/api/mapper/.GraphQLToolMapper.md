Converts GraphQL `ToolFilterInput` DTOs into internal `ToolFilterCriteria` domain objects, decoupling the GraphQL API layer from the service layer.

## Key Components

| Element | Description |
|---|---|
| `GraphQLToolMapper` | Spring-managed mapper component for tool filter translation |
| `toToolFilterCriteria(ToolFilterInput)` | Maps a GraphQL filter input to an internal criteria object; returns an empty criteria if input is `null` |

## Usage Example

```java
@Service
public class ToolService {

    private final GraphQLToolMapper toolMapper;

    public List<Tool> getTools(ToolFilterInput filterInput) {
        ToolFilterCriteria criteria = toolMapper.toToolFilterCriteria(filterInput);
        return toolRepository.findByCriteria(criteria);
    }
}
```

**Null-safe behavior** — passing `null` returns an empty `ToolFilterCriteria` (all fields unset), avoiding `NullPointerException` downstream:

```java
// Both are safe
ToolFilterCriteria criteria1 = toolMapper.toToolFilterCriteria(null);       // empty criteria
ToolFilterCriteria criteria2 = toolMapper.toToolFilterCriteria(filterInput); // mapped criteria
```

**Mapped fields:**

| `ToolFilterInput` field | `ToolFilterCriteria` field |
|---|---|
| `enabled` | `enabled` |
| `type` | `type` |
| `category` | `category` |
| `platformCategory` | `platformCategory` |
