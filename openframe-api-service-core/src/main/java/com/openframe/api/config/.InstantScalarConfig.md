Custom GraphQL scalar type configuration that enables the use of Java `Instant` objects in GraphQL schemas by handling serialization and parsing between `Instant` and ISO-8601 string representations.

## Key Components

- **@DgsScalar(name = "Instant")** - Registers this as a custom GraphQL scalar type named "Instant"
- **Coercing<Instant, String>** - Implementation that converts between Java `Instant` and GraphQL string values
- **serialize()** - Converts `Instant` objects to ISO-8601 strings for GraphQL responses
- **parseValue()** - Parses string input variables into `Instant` objects
- **parseLiteral()** - Parses string literals from GraphQL queries into `Instant` objects

## Usage Example

With this scalar configured, you can use `Instant` types directly in your GraphQL schema and resolvers:

```java
// GraphQL Schema
type Event {
    id: ID!
    name: String!
    createdAt: Instant!  # Uses the custom scalar
    updatedAt: Instant!
}

// Resolver method
@DgsQuery
public Event getEvent(@InputArgument String id) {
    return Event.builder()
        .id(id)
        .name("Sample Event")
        .createdAt(Instant.now())  # Automatically serialized to ISO-8601
        .updatedAt(Instant.parse("2024-01-15T10:30:00Z"))
        .build();
}

// GraphQL Query
query {
    event(id: "123") {
        createdAt  # Returns: "2024-01-15T10:30:00Z"
    }
}
```

The scalar handles all conversions automatically, supporting ISO-8601 formatted strings in queries and returning properly formatted timestamp strings in responses.
