Configuration class that manages Single Sign-On (SSO) provider settings for the OpenFrame API, automatically binding SSO configuration properties from application configuration files.

## Key Components

- **SSOProperties** - Spring Boot configuration properties class
- **providers** - List of `SSOProviderInfo` objects containing SSO provider configurations
- **@ConfigurationProperties** - Binds properties with "sso" prefix from configuration files
- **@Configuration** - Marks class as Spring configuration component
- **@Data** - Lombok annotation generating getters, setters, and other utility methods

## Usage Example

**Application Configuration (application.yml):**
```yaml
sso:
  providers:
    - name: "Google"
      clientId: "google-client-id"
      clientSecret: "google-secret"
      redirectUri: "http://localhost:8080/auth/google/callback"
    - name: "GitHub"
      clientId: "github-client-id"
      clientSecret: "github-secret"
      redirectUri: "http://localhost:8080/auth/github/callback"
```

**Injecting Configuration:**
```java
@Service
public class SSOService {
    private final SSOProperties ssoProperties;
    
    public SSOService(SSOProperties ssoProperties) {
        this.ssoProperties = ssoProperties;
    }
    
    public List<SSOProviderInfo> getAvailableProviders() {
        return ssoProperties.getProviders();
    }
}
```

The configuration automatically validates and populates the providers list at application startup, making SSO provider information available throughout the application.
