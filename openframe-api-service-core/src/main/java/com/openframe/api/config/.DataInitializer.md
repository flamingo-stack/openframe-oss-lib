Spring Boot configuration class that initializes OAuth client data on application startup by reading client credentials from environment properties and ensuring the default OAuth client exists in the database.

## Key Components

- **`DataInitializer`** - Configuration class that sets up data initialization beans
- **`initOAuthClients()`** - CommandLineRunner bean that creates or updates the default OAuth client
- **Environment integration** - Reads OAuth client credentials from application properties
- **Repository operations** - Uses `OAuthClientRepository` to persist client data

## Usage Example

Configure the required properties in your `application.yml`:

```java
oauth:
  client:
    default:
      id: "my-client-id"
      secret: "my-client-secret"
```

The initializer automatically runs on startup and will:
- Create a new OAuth client if none exists with the specified ID
- Update the client secret if it has changed
- Set default grant types (`password`, `refresh_token`) and scopes (`read`, `write`)

```java
// The OAuth client is automatically available after startup
@Autowired
private OAuthClientRepository clientRepository;

public void someMethod() {
    Optional<OAuthClient> client = clientRepository.findByClientId("my-client-id");
    // Client is guaranteed to exist after initialization
}
```

This ensures your application always has the required OAuth client configuration available for authentication flows.
