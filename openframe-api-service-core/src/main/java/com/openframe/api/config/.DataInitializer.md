Spring Boot configuration class that automatically initializes OAuth client data on application startup by reading credentials from environment properties.

## Key Components

- **`DataInitializer`** - Main configuration class that sets up data initialization logic
- **`initOAuthClients()`** - CommandLineRunner bean that creates or updates OAuth client configurations
- **Environment property integration** - Reads OAuth client credentials from `oauth.client.default.id` and `oauth.client.default.secret` properties
- **Conditional logic** - Updates existing clients if secrets change, or creates new clients with default grant types and scopes

## Usage Example

```java
// Application properties configuration
oauth.client.default.id=my-client-app
oauth.client.default.secret=super-secret-key

// The DataInitializer will automatically run on startup and:
// 1. Check if client exists in database
// 2. Create new client with default scopes ["read", "write"] 
// 3. Set grant types to ["password", "refresh_token"]
// 4. Update existing client secret if it has changed

// No manual intervention required - runs automatically via CommandLineRunner
```

This class ensures OAuth clients are properly configured in the database before the application begins processing requests, making it essential for OAuth-secured applications that need consistent client setup across deployments.
