A Spring Boot configuration class that automatically initializes OAuth client data on application startup by reading configuration properties and ensuring a default OAuth client exists in the database.

## Key Components

- **DataInitializer**: Configuration class that provides application startup initialization
- **initOAuthClients()**: Bean factory method that returns a CommandLineRunner for OAuth client setup
- **Environment injection**: Uses Spring's Environment to read OAuth configuration properties
- **OAuthClientRepository**: Repository dependency for database operations on OAuth clients

## Usage Example

```java
// Application properties configuration
// oauth.client.default.id=my-client-id
// oauth.client.default.secret=my-client-secret

// The DataInitializer automatically runs on startup and:
// 1. Reads the configured client ID and secret from properties
// 2. Checks if an OAuth client with that ID already exists
// 3. Creates a new client if none exists, or updates the secret if it changed
// 4. Sets default grant types (password, refresh_token) and scopes (read, write)

@Autowired
private OAuthClientRepository clientRepository;

// After startup, you can verify the client was created:
Optional<OAuthClient> client = clientRepository.findByClientId("my-client-id");
```

The class uses a fail-safe approach with `ifPresentOrElse()` to handle both new client creation and existing client updates, ensuring the OAuth configuration stays synchronized with the application properties on every startup.
