Spring Security configuration that provides minimal OAuth2 resource server setup for JWT token processing while delegating primary authentication to an upstream gateway.

## Key Components

- **`jwtProviderCache`** - Caffeine cache bean for storing JWT authentication providers by issuer to improve performance
- **`securityFilterChain`** - Main security configuration with OAuth2 resource server and issuer-based authentication resolution
- **Configuration properties** - Cache settings for expiration, refresh intervals, and maximum size
- **`JwtIssuerAuthenticationManagerResolver`** - Resolves authentication managers based on JWT issuer claims

## Usage Example

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public ResponseEntity<UserProfile> getProfile(
        @AuthenticationPrincipal JwtAuthenticationToken token) {
        
        String userId = token.getToken().getClaimAsString("sub");
        String issuer = token.getToken().getIssuer().toString();
        
        return ResponseEntity.ok(userService.getProfile(userId));
    }
}
```

Configure cache properties in `application.yml`:
```yaml
openframe:
  security:
    jwt:
      cache:
        expire-after: PT1H
        refresh-after: PT30M
        maximum-size: 100
```

The configuration permits all requests since the gateway handles authorization, but enables `@AuthenticationPrincipal` access to JWT claims in controllers.
