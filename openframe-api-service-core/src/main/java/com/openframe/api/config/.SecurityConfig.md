Spring Security configuration for the OpenFrame API service that provides minimal security setup with JWT support and caching for performance optimization.

## Key Components

- **`jwtProviderCache()`** - Creates a Caffeine cache for JWT authentication providers to improve performance by caching JWT decoders per issuer
- **`securityFilterChain()`** - Configures the main security filter chain with OAuth2 resource server support and permits all requests
- **`JwtIssuerAuthenticationManagerResolver`** - Resolves JWT authentication managers based on token issuers using the cached providers

## Usage Example

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public ResponseEntity<UserProfile> getProfile(
            @AuthenticationPrincipal JwtAuthenticationToken token) {
        // JWT token is automatically resolved and cached
        String userId = token.getToken().getSubject();
        return ResponseEntity.ok(getUserProfile(userId));
    }
}

// Configuration properties
# application.yml
openframe:
  security:
    jwt:
      cache:
        expire-after: PT1H
        refresh-after: PT30M
        maximum-size: 100
```

**Note**: This configuration assumes the API Gateway handles primary authentication and authorization. The service only enables OAuth2 resource server capabilities for `@AuthenticationPrincipal` support while permitting all requests through.
