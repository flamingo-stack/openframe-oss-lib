Spring Web MVC configuration class that enables custom authentication argument resolution by registering the `AuthPrincipalArgumentResolver` to support `@AuthenticationPrincipal` annotations in controller methods.

## Key Components

- **AuthenticationConfig**: Main configuration class implementing `WebMvcConfigurer`
- **addArgumentResolvers()**: Overridden method that registers custom argument resolvers
- **AuthPrincipalArgumentResolver**: Custom resolver for handling authentication principal injection

## Usage Example

With this configuration, you can inject authenticated user principals directly into controller methods:

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public UserProfile getProfile(@AuthenticationPrincipal AuthPrincipal principal) {
        // The AuthPrincipal is automatically resolved from the security context
        return userService.getProfile(principal.getUserId());
    }
    
    @PostMapping("/settings")
    public void updateSettings(@AuthenticationPrincipal AuthPrincipal principal, 
                              @RequestBody Settings settings) {
        settingsService.update(principal.getUserId(), settings);
    }
}
```

The configuration automatically handles the conversion from Spring Security's authentication context to your custom `AuthPrincipal` type, eliminating the need for manual principal extraction in each controller method.
