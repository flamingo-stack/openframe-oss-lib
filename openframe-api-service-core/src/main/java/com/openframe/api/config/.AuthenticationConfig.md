Spring MVC configuration class that registers a custom argument resolver for handling authenticated principals in REST endpoints.

## Key Components

- **AuthenticationConfig**: Spring configuration class implementing `WebMvcConfigurer`
- **addArgumentResolvers()**: Registers the `AuthPrincipalArgumentResolver` to enable automatic injection of authentication principals into controller methods

## Usage Example

With this configuration, you can inject authenticated user principals directly into your controller methods:

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public UserProfile getProfile(@AuthenticationPrincipal AuthPrincipal principal) {
        // The AuthPrincipal is automatically resolved from the security context
        return userService.getProfile(principal.getUserId());
    }
    
    @PostMapping("/update")
    public ResponseEntity<Void> updateProfile(
            @AuthenticationPrincipal AuthPrincipal principal,
            @RequestBody UpdateRequest request) {
        userService.updateProfile(principal.getUserId(), request);
        return ResponseEntity.ok().build();
    }
}
```

This eliminates the need to manually extract authentication information from the SecurityContext in each controller method.
