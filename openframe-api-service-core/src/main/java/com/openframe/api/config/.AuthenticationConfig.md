Spring MVC configuration that registers a custom argument resolver to enable automatic injection of authenticated user principals into controller method parameters.

## Key Components

**Main Class:**
- `AuthenticationConfig` - Spring configuration class that implements `WebMvcConfigurer`

**Key Methods:**
- `addArgumentResolvers(List<HandlerMethodArgumentResolver>)` - Registers the custom `AuthPrincipalArgumentResolver`

**Dependencies:**
- `AuthPrincipalArgumentResolver` - Custom resolver for handling `@AuthenticationPrincipal` annotations
- Spring Web MVC framework components

## Usage Example

With this configuration, controllers can automatically receive the authenticated user principal:

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public UserProfile getProfile(@AuthenticationPrincipal AuthPrincipal principal) {
        // AuthPrincipal is automatically injected by AuthPrincipalArgumentResolver
        return userService.getProfile(principal.getUserId());
    }
    
    @PostMapping("/settings")
    public ResponseEntity<Void> updateSettings(
        @AuthenticationPrincipal AuthPrincipal principal,
        @RequestBody SettingsRequest request) {
        
        settingsService.update(principal.getUserId(), request);
        return ResponseEntity.ok().build();
    }
}
```

The resolver eliminates the need to manually extract authentication information from the security context in each controller method.
