A GraphQL custom scalar configuration that handles date values in yyyy-MM-dd format, providing automatic conversion between Java LocalDate objects and GraphQL Date strings with validation.

## Key Components

- **@DgsScalar(name = "Date")** - Registers this as a custom GraphQL scalar type named "Date"
- **serialize()** - Converts LocalDate objects to yyyy-MM-dd formatted strings for GraphQL responses
- **parseValue()** - Parses string variables from GraphQL requests into LocalDate objects
- **parseLiteral()** - Parses literal date values directly from GraphQL query text
- **DATE_FORMATTER** - Static formatter using "yyyy-MM-dd" pattern for consistent date handling

## Usage Example

Once configured, you can use the Date scalar in your GraphQL schema and resolvers:

```java
// GraphQL Schema Definition
scalar Date

type User {
    id: ID!
    name: String!
    birthDate: Date
    createdAt: Date
}

// In your DGS resolver
@DgsQuery
public User getUser(@InputArgument String id) {
    return User.builder()
        .id(id)
        .name("John Doe")
        .birthDate(LocalDate.of(1990, 5, 15)) // Will serialize to "1990-05-15"
        .createdAt(LocalDate.now())
        .build();
}

// GraphQL Query
query {
    user(id: "123") {
        name
        birthDate  # Returns "1990-05-15"
    }
}
```

The scalar automatically validates date format and throws appropriate exceptions for invalid inputs, ensuring type safety across your GraphQL API.
