A custom GraphQL scalar type configuration that handles date values in yyyy-MM-dd format, providing automatic conversion between `LocalDate` objects and string representations.

## Key Components

- **@DgsScalar(name = "Date")** - Registers this as a custom "Date" scalar type in the GraphQL schema
- **serialize()** - Converts `LocalDate` objects to yyyy-MM-dd string format for GraphQL responses
- **parseValue()** - Parses string date inputs from variables into `LocalDate` objects
- **parseLiteral()** - Parses date literals directly written in GraphQL queries
- **DATE_FORMATTER** - Static formatter using yyyy-MM-dd pattern for consistent date handling

## Usage Example

Once registered, you can use the Date scalar in your GraphQL schema and resolvers:

```java
// In your GraphQL schema (.graphqls file)
type User {
    id: ID!
    name: String!
    birthDate: Date
}

// In your DGS resolver
@DgsQuery
public User getUser(@InputArgument String id) {
    return User.builder()
        .id(id)
        .name("John Doe")
        .birthDate(LocalDate.of(1990, 5, 15)) // Will serialize to "1990-05-15"
        .build();
}
```

GraphQL queries can then use date strings directly:
```graphql
query {
    user(id: "123") {
        name
        birthDate  # Returns "1990-05-15"
    }
}
```
