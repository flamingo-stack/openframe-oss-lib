GraphQL data fetcher for device-related queries in the OpenFrame API, handling device listing, filtering, pagination, and lazy-loading of related entities via DataLoaders.

## Key Components

| Method | Type | Description |
|--------|------|-------------|
| `deviceFilters` | `@DgsQuery` | Returns available filter options for the device list based on a `DeviceFilterInput` |
| `devices` | `@DgsQuery` | Paginated device list with cursor-based pagination, filtering, search, and sort support |
| `deviceById` | `@DgsQuery` | Fetches a single device by its Relay global ID (decoded to `machineId`) |
| `device` | `@DgsQuery` | Fetches a single device directly by `machineId` |
| `machineNodeId` | `@DgsData` | Resolves the Relay-compliant global `id` field on `Machine` |
| `toolConnectionNodeId` | `@DgsData` | Resolves the global `id` field on `ToolConnection` |
| `installedAgentNodeId` | `@DgsData` | Resolves the global `id` field on `InstalledAgent` |
| `tags` | `@DgsData` | Batch-loads tags for a `Machine` via `tagDataLoader` |
| `toolConnections` | `@DgsData` | Batch-loads tool connections for a `Machine` via `toolConnectionDataLoader` |
| `installedAgents` | `@DgsData` | Batch-loads installed agents for a `Machine` via `installedAgentDataLoader` |
| `organization` | `@DgsData` | Batch-loads the parent organization for a `Machine` via `organizationDataLoader` |

## Usage Example

```java
// GraphQL query example resolved by this fetcher
/*
query {
  devices(
    filter: { organizationId: "org-123" }
    first: 10
    after: "cursor=="
    search: "workstation"
    sort: { field: "hostname", direction: ASC }
  ) {
    totalCount
    edges {
      node {
        id
        machineId
        tags { name }
        organization { name }
        installedAgents { agentType }
      }
    }
  }
}
*/

// Fetching a device by Relay global ID
/*
query {
  deviceById(id: "TWFjaGluZTptYWNoaW5lLTEyMw==") {
    id
    machineId
    toolConnections { id }
  }
}
*/
```

> **Note:** Related fields (`tags`, `toolConnections`, `installedAgents`, `organization`) are resolved asynchronously using the DataLoader pattern to prevent N+1 query issues. All ID fields on `Machine`, `ToolConnection`, and `InstalledAgent` are encoded as Relay global IDs.
