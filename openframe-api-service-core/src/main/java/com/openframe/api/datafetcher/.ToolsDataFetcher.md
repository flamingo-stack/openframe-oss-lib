Brief description of the file's purpose in 1-2 sentences:

GraphQL data fetcher for integrated tool queries in the OpenFrame platform, handling tool listing, filtering, searching, and Relay-compliant global ID resolution for `IntegratedTool` nodes.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `RELAY` | `Relay` (static) | Relay spec encoder for generating global node IDs |
| `toolService` | `ToolService` | Business logic layer for tool queries and filter retrieval |
| `toolMapper` | `GraphQLToolMapper` | Maps GraphQL input DTOs to internal filter criteria |
| `toolNodeId` | `@DgsData` resolver | Encodes `IntegratedTool.id` as a Relay global ID |
| `integratedTools` | `@DgsQuery` | Returns a paginated/filtered `ToolList` based on filter, search, and sort inputs |
| `toolFilters` | `@DgsQuery` | Returns all available filter options for the tool catalog |

## Usage Example

```java
// GraphQL query resolved by integratedTools()
// query {
//   integratedTools(
//     filter: { category: "RMM" },
//     search: "NinjaRMM",
//     sort: { field: "name", direction: ASC }
//   ) {
//     edges { node { id name category } }
//   }
// }

// Internally, the fetcher delegates like this:
ToolFilterCriteria criteria = toolMapper.toToolFilterCriteria(filterInput);
ToolList result = toolService.queryTools(criteria, "NinjaRMM", sortInput);

// toolNodeId encodes the Relay global ID:
// "IntegratedTool:abc123" → base64 → "SW50ZWdyYXRlZFRvb2w6YWJjMTIz"
String globalId = RELAY.toGlobalId("IntegratedTool", tool.getId());
```

> **Note:** All query inputs are validated via `@Valid` and Spring's `@Validated`, ensuring malformed filter or sort inputs are rejected before reaching the service layer.
