Resolves concrete GraphQL types for the `NotificationContext` union/interface by mapping discriminator values to their corresponding GraphQL type names at runtime.

## Key Components

| Element | Description |
|---|---|
| `@DgsComponent` | Registers this class as a DGS (Domain Graph Service) component |
| `@DgsTypeResolver(name = "NotificationContext")` | Binds the resolver to the `NotificationContext` GraphQL type |
| `DEFAULT_TYPE` | Fallback type (`"GenericContext"`) used when no discriminator match is found |
| `typeNamesByDiscriminator` | Immutable map built from injected `NotificationContextDescriptor` instances, keyed by discriminator string |
| `resolveType(NotificationContext)` | Core resolution method — returns the matching GraphQL type name or falls back to `DEFAULT_TYPE` |

## Usage Example

```java
// NotificationContextDescriptor implementations are registered as Spring beans
// and injected automatically into the resolver constructor.

@Component
public class AlertContextDescriptor implements NotificationContextDescriptor {

    @Override
    public String type() {
        return "ALERT"; // discriminator stored on the document
    }

    @Override
    public String graphqlTypeName() {
        return "AlertContext"; // concrete GraphQL type name
    }
}

// At runtime, when GraphQL resolves a NotificationContext field:
// source.getType() == "ALERT"  →  resolves to "AlertContext"
// source.getType() == "UNKNOWN" →  resolves to "GenericContext" (default)
```

## Notes

- Duplicate discriminators are handled gracefully — the first registered descriptor wins (`(left, right) -> left`).
- Adding a new context type requires only a new `NotificationContextDescriptor` bean; no changes to this resolver are needed.
