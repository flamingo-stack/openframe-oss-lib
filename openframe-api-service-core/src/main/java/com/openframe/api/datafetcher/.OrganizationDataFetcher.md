GraphQL DataFetcher for resolving Organization-related queries, including paginated list retrieval and individual lookups by global Relay ID or native organization ID.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `organizationNodeId` | `@DgsData` field resolver | Converts the internal `organizationId` to a Relay-compliant global ID on the `Organization.id` field |
| `organizations` | `@DgsQuery` | Paginated, filterable, sortable list of organizations using Relay cursor-based pagination |
| `organization` | `@DgsQuery` | Fetches a single organization by its Relay global ID |
| `organizationByOrganizationId` | `@DgsQuery` | Fetches a single organization by its native `organizationId` |
| `OrganizationQueryService` | Service dependency | Handles complex filtering, pagination, and search logic |
| `OrganizationService` | Service dependency | Handles direct document-level lookups |
| `GraphQLOrganizationMapper` | Mapper dependency | Transforms DTOs into GraphQL-compatible connection/pagination structures |

## Usage Example

```java
// GraphQL query — paginated list with filter and sort
query {
  organizations(
    first: 10,
    after: "cursor==",
    search: "Acme",
    filter: { active: true },
    sort: { field: "name", direction: ASC }
  ) {
    totalCount
    edges {
      node {
        id
        name
      }
    }
  }
}

// GraphQL query — single organization by Relay global ID
query {
  organization(id: "T3JnYW5pemF0aW9uOjEyMw==") {
    id
    name
  }
}

// GraphQL query — single organization by native ID
query {
  organizationByOrganizationId(organizationId: "org-123") {
    id
    name
  }
}
```

> **Note:** The `id` field on `Organization` is automatically encoded as a Relay global ID (e.g., `Organization:org-123` → base64) via the `organizationNodeId` resolver. Pass this encoded value to the `organization(id:)` query.
