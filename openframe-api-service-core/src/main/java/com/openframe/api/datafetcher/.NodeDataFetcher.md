GraphQL Relay node resolver that implements the `node` and `nodes` queries, enabling global object identification by decoding Relay global IDs and dispatching lookups to the appropriate service.

## Key Components

| Element | Description |
|---|---|
| `node(@InputArgument String id)` | Resolves a single Relay global ID to its corresponding domain object |
| `nodes(@InputArgument List<String> ids)` | Batch-resolves multiple Relay global IDs, returning `null` for any that fail |
| `resolveNode(ResolvedGlobalId)` | Internal dispatcher that decodes the `NodeType` and delegates to the correct service |
| `NodeType` enum | Maps type names to service calls (`MACHINE`, `ORGANIZATION`, `EVENT`, `INTEGRATED_TOOL`, `TAG`, `TOOL_CONNECTION`, `INSTALLED_AGENT`, `TENANT`) |
| `TenantRepository` | Optionally injected — `TENANT` resolution is skipped gracefully if unavailable |

## Usage Example

```java
// GraphQL query — single node
query {
  node(id: "TWFjaGluZToxMjM=") {
    ... on Machine {
      id
      hostname
    }
  }
}

// GraphQL query — batch nodes
query {
  nodes(ids: ["TWFjaGluZToxMjM=", "T3JnYW5pemF0aW9uOjQ1Ng=="]) {
    ... on Machine { hostname }
    ... on Organization { name }
  }
}
```

```java
// The fetcher decodes the base64 global ID internally:
// "TWFjaGluZToxMjM=" → type=MACHINE, id="123"
// then routes to: deviceService.findByMachineId("123")
```

## Notes

- Failed individual lookups in `nodes()` are caught and returned as `null` rather than failing the entire batch.
- `TenantRepository` is `@Autowired(required = false)` — multi-tenant deployments may not have this bean present; the resolver returns `null` safely instead of throwing.
- Unsupported `NodeType` values throw `IllegalArgumentException` immediately to surface schema/ID mismatches early.
