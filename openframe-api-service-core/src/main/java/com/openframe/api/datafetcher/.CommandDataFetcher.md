GraphQL mutation resolver for RMM (Remote Monitoring & Management) ad-hoc command dispatch operations. Bridges GraphQL API calls to the `CommandDispatchService` for executing and cancelling remote commands on managed devices.

## Key Components

| Element | Description |
|---|---|
| `runCommand` | Mutation that dispatches an ad-hoc command to a remote device via `RunCommandInput` |
| `cancelExecution` | Mutation that cancels an in-progress command dispatch via `CancelExecutionInput` |
| `CommandDispatchService` | Injected service handling the underlying RMM command execution logic |

## Usage Example

```java
// GraphQL mutation — run a command
mutation {
  runCommand(input: {
    deviceId: "dev-123",
    command: "Get-Service | Where-Object { $_.Status -eq 'Running' }"
  }) {
    dispatchId
    status
  }
}

// GraphQL mutation — cancel an in-progress execution
mutation {
  cancelExecution(input: {
    dispatchId: "dispatch-abc-456"
  }) {
    success
    message
  }
}
```

```java
// Service delegation (internal)
@DgsMutation
public CommandDispatchResponse runCommand(
    @InputArgument @Valid RunCommandInput input) {
    return commandDispatchService.runCommand(input);
}
```

> **Validation:** Both mutations use `@Valid` + `@Validated` to enforce bean validation constraints on input DTOs before reaching the service layer, rejecting malformed requests early.
