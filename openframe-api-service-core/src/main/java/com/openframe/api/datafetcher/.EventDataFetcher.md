GraphQL data fetcher for event-related operations in the OpenFrame API, handling queries and mutations for `Event` entities with Relay-compliant cursor-based pagination and global ID encoding.

## Key Components

| Component | Description |
|-----------|-------------|
| `eventNodeId` | Encodes raw MongoDB IDs into Relay-compliant global IDs for the `Event` type |
| `events` | Paginated query supporting cursor-based navigation (`first`/`after`/`last`/`before`), filtering, search, and sorting |
| `eventById` | Fetches a single event by its Relay global ID, decoding it to a raw ID before lookup |
| `eventFilters` | Returns available filter metadata for the event list |
| `createEvent` | Mutation to create a new event with auto-assigned `Instant.now()` timestamp |
| `updateEvent` | Mutation to update an existing event by global ID |

## Usage Example

```java
// Query: Paginated events with filter and sort
query {
  events(
    first: 10,
    after: "cursor==",
    filter: { type: "TICKET_CREATED" },
    sort: { field: "timestamp", direction: DESC }
  ) {
    edges {
      node {
        id
        type
        userId
        timestamp
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

// Mutation: Create a new event
mutation {
  createEvent(input: {
    userId: "user-123",
    type: "TICKET_UPDATED",
    data: { ticketId: "ticket-456" }
  }) {
    id
    timestamp
  }
}
```

## Dependencies

- **`EventService`** — business logic layer for querying, creating, and updating events
- **`GraphQLEventMapper`** — maps between GraphQL input DTOs and domain criteria/connection objects
- **`Relay`** — handles global ID encoding/decoding for Relay specification compliance
