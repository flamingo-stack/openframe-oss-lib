GraphQL DataFetcher component that handles event-related queries and mutations using Netflix DGS framework. Provides CRUD operations for events with cursor-based pagination, filtering, and search capabilities.

## Key Components

**Main Class:**
- `EventDataFetcher` - DGS component implementing GraphQL resolvers for event operations

**Query Methods:**
- `events()` - Fetches events with filtering, pagination, search, and sorting
- `eventById()` - Retrieves a single event by ID
- `eventFilters()` - Returns available filter options for events

**Mutation Methods:**
- `createEvent()` - Creates a new event with timestamp
- `updateEvent()` - Updates an existing event by ID

**Dependencies:**
- `EventService` - Business logic for event operations
- `GraphQLEventMapper` - Maps between DTOs and domain objects

## Usage Example

```java
// Query events with pagination and filtering
@DgsQuery
public GenericConnection<GenericEdge<Event>> events(
    @InputArgument @Valid EventFilterInput filter,
    @InputArgument @Valid CursorPaginationInput pagination,
    @InputArgument String search,
    @InputArgument @Valid SortInput sort
) {
    // Implementation handles mapping and service calls
}

// Create a new event
@DgsMutation
public Event createEvent(@InputArgument @Valid CreateEventInput input) {
    Event event = Event.builder()
        .userId(input.getUserId())
        .type(input.getType())
        .payload(input.getData())
        .timestamp(Instant.now())
        .build();
    
    return eventService.createEvent(event);
}
```

The component uses validation annotations (`@Valid`, `@NotBlank`) and comprehensive logging for debugging and monitoring GraphQL operations.
