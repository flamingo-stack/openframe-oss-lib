GraphQL data fetcher component that provides query endpoints for retrieving log/audit data with filtering, pagination, and detail lookup capabilities. Part of the OpenFrame API's audit trail system using Netflix DGS framework.

## Key Components

**Main Class:**
- `LogDataFetcher` - DGS component handling GraphQL log queries

**Query Methods:**
- `logFilters()` - Retrieves available filter options for log queries
- `logs()` - Fetches paginated log events with filtering, search, and sorting
- `logDetails()` - Gets detailed information for a specific log event by composite key

**Dependencies:**
- `LogService` - Business logic for log operations
- `GraphQLLogMapper` - Converts between GraphQL DTOs and service objects

## Usage Example

```java
// GraphQL query to fetch filtered logs with pagination
query GetLogs($filter: LogFilterInput, $pagination: CursorPaginationInput) {
  logs(filter: $filter, pagination: $pagination, search: "error", sort: {field: "timestamp", direction: DESC}) {
    edges {
      node {
        eventType
        timestamp
        toolType
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

// Query for specific log details
query GetLogDetails {
  logDetails(
    ingestDay: "2024-01-15"
    toolType: "SYSTEM"
    eventType: "ERROR"
    timestamp: "2024-01-15T10:30:00Z"
    toolEventId: "evt_12345"
  ) {
    message
    severity
    metadata
  }
}
```

The component implements cursor-based pagination for efficient log browsing and uses composite keys (ingest day, tool type, event type, timestamp, tool event ID) for precise log detail retrieval.
