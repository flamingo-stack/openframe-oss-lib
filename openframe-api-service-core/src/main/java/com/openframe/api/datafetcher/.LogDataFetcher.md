A DGS (Domain Graph Service) GraphQL data fetcher that handles audit log queries, providing paginated log event listings, filter options, and detailed log record lookups backed by Cassandra.

## Key Components

| Component | Description |
|-----------|-------------|
| `logEventId` | Resolves Relay global IDs for `LogEvent` nodes |
| `logDetailsId` | Resolves Relay global IDs for `LogDetails` nodes |
| `logFilters` | Returns available filter options for the audit log UI |
| `logs` | Cursor-paginated query returning a `GenericConnection<LogEvent>` with sort and search support |
| `logDetails` | Fetches a single detailed log record by composite key |

**Conditional activation:** The component only loads when `spring.data.cassandra.enabled=true`.

## Usage Example

```java
// GraphQL query — paginated logs with filters
query {
  logs(
    filter: { toolType: "PSA", eventType: "TICKET_CREATED" },
    first: 20,
    after: "cursor==",
    sort: { field: "timestamp", direction: DESC },
    search: "john.doe"
  ) {
    edges {
      node {
        id
        timestamp
        eventType
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

// GraphQL query — fetch log detail by composite key
query {
  logDetails(
    ingestDay: "2024-06-01",
    toolType: "PSA",
    eventType: "TICKET_CREATED",
    timestamp: "2024-06-01T12:00:00Z",
    toolEventId: "evt-abc123"
  ) {
    id
    payload
  }
}
```

> **Note:** All IDs returned follow the [Relay Global Object Identification spec](https://relay.dev/graphql/objectidentification.htm), encoded as `LogEvent:<raw-id>` via `graphql.relay.Relay`.
