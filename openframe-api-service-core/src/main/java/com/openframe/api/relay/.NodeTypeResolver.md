Resolves concrete Java types to their GraphQL type names for the `Node` interface in the DGS (Domain Graph Service) framework, enabling Relay-style global object identification.

## Key Components

| Element | Description |
|---|---|
| `@DgsComponent` | Registers this class as a DGS-managed component |
| `@DgsTypeResolver(name = "Node")` | Binds the resolver to the GraphQL `Node` interface |
| `resolveNode(Object node)` | Maps domain objects to their GraphQL type name strings |

**Supported Node types:**

| Domain Class | GraphQL Type Name |
|---|---|
| `Machine` | `"Machine"` |
| `Organization` | `"Organization"` |
| `Event` | `"Event"` |
| `IntegratedTool` | `"IntegratedTool"` |
| `Tenant` | `"Tenant"` |
| `ItemAssignment` | `"ItemAssignment"` |
| `Ticket` | `"Ticket"` |
| `KnowledgeBaseItem` | `"KnowledgeBaseItem"` |
| `User` | `"User"` |

## Usage Example

This resolver is invoked automatically by the DGS framework when a GraphQL query uses the `Node` interface (e.g., Relay's `node(id: ID!)` query). No manual invocation is required.

```java
// GraphQL query that triggers this resolver:
// query { node(id: "Ticket:abc123") { ... on Ticket { title } } }

// When DGS fetches a Ticket object and needs to resolve its __typename:
NodeTypeResolver resolver = new NodeTypeResolver();
String typeName = resolver.resolveNode(new Ticket()); // returns "Ticket"

// Unknown types throw immediately:
resolver.resolveNode(new UnknownEntity()); 
// throws IllegalArgumentException: "Unknown Node type: com.openframe...UnknownEntity"
```

> **Note:** When adding new domain types that implement the `Node` interface, a corresponding `instanceof` branch must be added to `resolveNode` — otherwise an `IllegalArgumentException` will be thrown at runtime.
