Minimal Spring Boot configuration class used as the test application context for service-layer integration tests, wiring only the beans required for notification-related functionality.

## Key Components

| Element | Description |
|---|---|
| `@SpringBootConfiguration` | Marks this as a lightweight Spring Boot configuration (no full auto-configuration scan) |
| `@ImportAutoConfiguration` | Selectively imports MongoDB, Jackson, and repository auto-configurations |
| `@EnableMongoAuditing` | Activates MongoDB auditing (e.g., `@CreatedDate`, `@LastModifiedDate`) |
| `@EnableMongoRepositories` | Scans for Mongo repositories starting from `NotificationRepository` |
| `@Import` | Manually registers all required notification service beans |

**Imported Beans:**

- `CustomNotificationRepositoryImpl` / `CustomNotificationReadStateRepositoryImpl` — custom repository implementations
- `NotificationBroadcaster` — NATS-based event broadcaster
- `NotificationContextDescriptorRegistry` — notification type registry
- `NotificationReadStateService` / `NotificationService` — core service layer
- `GraphQLNotificationMapper` — maps notifications to GraphQL response types
- `NotificationContextMongoConfig` — MongoDB context configuration

## Usage Example

```java
@SpringBootTest(classes = ServiceIntegrationTestApplication.class)
@AutoConfigureTestDatabase
class NotificationServiceIntegrationTest {

    @Autowired
    private NotificationService notificationService;

    @Test
    void shouldPersistAndRetrieveNotification() {
        // arrange
        var notification = buildTestNotification();

        // act
        notificationService.save(notification);

        // assert
        assertThat(notificationService.findById(notification.getId()))
            .isPresent();
    }
}
```

> This class is **test-scope only**. It deliberately avoids a full `@SpringBootApplication` context to keep integration tests fast and focused on the notification domain without loading unrelated beans.
