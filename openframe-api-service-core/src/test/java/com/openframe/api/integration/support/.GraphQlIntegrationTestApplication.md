Minimal Spring Boot test application context used to bootstrap GraphQL integration tests for the notification subsystem.

## Key Components

| Element | Description |
|---|---|
| `@SpringBootConfiguration` | Marks this as a Spring Boot configuration entry point for the test context |
| `@EnableAutoConfiguration` (exclude) | Strips out unneeded auto-configurations (Security, Cassandra, Kafka, Redis) to keep the test context lightweight |
| `@EnableMongoRepositories` | Activates MongoDB repository support scoped to `NotificationRepository` |
| `@EnableMongoAuditing` | Enables MongoDB auditing (created/modified timestamps) within the test context |
| `@EnableMethodSecurity` | Enables method-level security annotations during tests |
| `@Import` | Manually wires the exact beans needed: data fetchers, mappers, services, repositories, scalar configs, and the `TestApprovalContextDescriptor` test stub |

## Usage Example

```java
@SpringBootTest
@ContextConfiguration(classes = GraphQlIntegrationTestApplication.class)
class NotificationDataFetcherIntegrationTest {

    @Autowired
    private NotificationDataFetcher notificationDataFetcher;

    @Test
    void shouldFetchNotificationsForUser() {
        // Integration test against the real GraphQL data fetcher
        // backed by an embedded/test MongoDB instance
    }
}
```

## Notes

- This class contains **no application logic** — it is a pure configuration shell for integration tests.
- The excluded auto-configurations (Cassandra, Kafka, Redis, OAuth2) ensure only MongoDB and GraphQL-related beans are loaded, keeping test startup fast.
- `TestApprovalContextDescriptor` is a test-specific stub registered in place of production notification context descriptors.
