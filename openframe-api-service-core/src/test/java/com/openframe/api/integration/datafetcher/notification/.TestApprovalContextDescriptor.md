A Spring component that registers the `TEST_APPROVAL_REQUEST` notification type by mapping it to its corresponding `TestApprovalContext` class via the `NotificationContextDescriptor` contract.

## Key Components

| Member | Description |
|--------|-------------|
| `TYPE` | Constant string `"TEST_APPROVAL_REQUEST"` identifying this notification type |
| `type()` | Returns the notification type identifier used for routing/deserialization |
| `contextClass()` | Returns `TestApprovalContext.class` as the concrete context payload type |

## Usage Example

```java
// Resolved automatically by Spring; used by the notification framework
// to deserialize incoming notification payloads to the correct context type.

@Autowired
List<NotificationContextDescriptor> descriptors;

NotificationContextDescriptor descriptor = descriptors.stream()
    .filter(d -> d.type().equals(TestApprovalContextDescriptor.TYPE))
    .findFirst()
    .orElseThrow();

Class<? extends NotificationContext> ctxClass = descriptor.contextClass();
// ctxClass == TestApprovalContext.class
```

> **Note:** This descriptor is part of the approval workflow notification pipeline. Adding a new notification type requires implementing both a `NotificationContextDescriptor` (like this class) and a matching `NotificationContext` subclass (`TestApprovalContext`).
