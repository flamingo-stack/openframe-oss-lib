Integration test suite that verifies the `NotificationDataFetcher` GraphQL layer end-to-end against a real MongoDB instance, covering queries, mutations, and JWT-based principal resolution for both admin users and machine agents.

## Key Components

- **`admin_lists_own_rows`** — Confirms an ADMIN JWT principal retrieves only their own `USER`-addressed read-state rows via the `notifications` query.
- **`agent_lists_machine_rows`** — Confirms an AGENT JWT principal with a `machine_id` claim retrieves `MACHINE`-addressed rows without requiring an explicit schema argument.
- **`has_unread`** — Verifies `hasUnreadNotifications` returns `true` for unread rows and flips to `false` after `markRead` is called.
- **`mark_as_read`** — Exercises the `markNotificationAsRead` mutation using a Relay global ID and asserts the mutation returns `true`.
- **`mark_all_as_read`** — Exercises `markAllNotificationsAsRead` and asserts the returned count equals the number of unread rows.
- **`delete_notification`** — Exercises `deleteNotification` with a Relay global ID and asserts soft-deletion (returns `true`).
- **`delete_all_read`** — Exercises `deleteAllReadNotifications` and asserts only READ rows are transitioned to DELETED.
- **`admin_lists_oldest_first_with_sort_asc`** — Verifies `sort: { direction: ASC }` returns edges oldest-first.
- **`admin_lists_newest_first_by_default`** — Verifies default sort order is newest-first (DESC) when no `sort` argument is provided.

## Usage Example

```java
// Run integration tests with the required system property
// mvn verify -Dintegration.tests=true

// Typical test pattern used throughout the suite
loginAsAdmin(ALICE);
Notification n = mongoTemplate.save(NotificationFixtures.basic("welcome"));
readStateService.createForAudience(
    n.getId(), NotificationCategory.TICKETS, "title",
    RecipientType.USER, Set.of(ALICE)
);

ExecutionResult res = queryExecutor.execute("""
    query { notifications(first: 10) { edges { node { id title read } } } }
""");
assertThat(res.getErrors()).isEmpty();
assertThat(edges(res)).hasSize(1);
```

> **Note:** Tests are gated behind `-Dintegration.tests=true` and require a running MongoDB instance provided by `BaseMongoIntegrationTest`. Spring Cloud Stream auto-configurations are excluded to keep the test context lightweight.
