Abstract base class that bootstraps a shared Testcontainers MongoDB instance for Spring Boot integration tests using MongoDB 7.

## Key Components

- **`MONGO`** — Static `MongoDBContainer` field running `mongo:7` via Docker, shared across all subclass test suites.
- **`mongoProperties()`** — `@DynamicPropertySource` method that lazily starts the container (if not already running) and wires the following Spring properties:
  - `spring.data.mongodb.uri` — Points to the containerized MongoDB with `/test` database and `directConnection=true`.
  - `spring.data.mongodb.auto-index-creation` — Enabled to reflect production index behavior in tests.

## Usage Example

Extend this class in any Spring Boot integration test that requires a real MongoDB connection:

```java
@SpringBootTest
@AutoConfigureMockMvc
class DeviceRepositoryIntegrationTest extends BaseMongoIntegrationTest {

    @Autowired
    private DeviceRepository deviceRepository;

    @Test
    void shouldPersistAndRetrieveDevice() {
        Device device = new Device("device-001", "Workstation");
        deviceRepository.save(device);

        Optional<Device> found = deviceRepository.findById("device-001");
        assertThat(found).isPresent();
        assertThat(found.get().getName()).isEqualTo("Workstation");
    }
}
```

> The container is started lazily on first test run and reused across all subclasses within the same JVM process, keeping test suite startup time low. No manual `@BeforeAll` container lifecycle management is required in subclasses.
