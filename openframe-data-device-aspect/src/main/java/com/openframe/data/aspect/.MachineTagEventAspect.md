AOP aspect that intercepts Spring Data repository save and delete operations for `Machine`, `Tag`, and `TagAssignment` entities, delegating change events to `MachineTagEventService` for downstream processing (e.g., Pinot re-sync).

## Key Components

| Method | Advice Type | Intercepted Operation |
|---|---|---|
| `afterMachineSave` | `@AfterReturning` | `MachineRepository.save()` |
| `afterMachineSaveAll` | `@AfterReturning` | `MachineRepository.saveAll()` |
| `afterTagAssignmentSave` | `@AfterReturning` | `TagAssignmentRepository.save()` |
| `afterTagAssignmentSaveAll` | `@AfterReturning` | `TagAssignmentRepository.saveAll()` |
| `aroundTagAssignmentDelete` | `@Around` | `TagAssignmentRepository.deleteByEntityIdAndTagIdAndEntityType()` |
| `aroundTagAssignmentDeleteByTagId` | `@Around` | `TagAssignmentRepository.deleteByTagId()` |
| `aroundTagSave` | `@Around` | `TagRepository.save()` |
| `aroundTagSaveAll` | `@Around` | `TagRepository.saveAll()` |

**Activation:** Conditionally enabled via the property `openframe.device.aspect.enabled=true` (defaults to `true` when the property is absent).

**Advice strategy:** Delete operations use `@Around` to capture entity state _before_ deletion occurs; save operations use `@AfterReturning` (or `@Around` for `Tag`) to process state after a successful persist.

## Usage Example

```java
// application.properties — enable or disable the aspect
openframe.device.aspect.enabled=true
```

```java
// No direct invocation needed — the aspect activates transparently on repository calls:
machineRepository.save(machine);
// → triggers afterMachineSave → machineTagEventService.processMachineSave(machine)

tagAssignmentRepository.deleteByEntityIdAndTagIdAndEntityType(entityId, tagId, entityType);
// → triggers aroundTagAssignmentDelete (pre-delete sync) → proceeds with delete

tagRepository.save(tag);
// → triggers aroundTagSave → machineTagEventService.processTagSave(tag) after successful persist
```
