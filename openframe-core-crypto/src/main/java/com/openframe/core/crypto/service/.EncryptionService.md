A Spring `@Service` that provides AES-256-CBC + HMAC symmetric encryption/decryption for sensitive data, with dedicated helpers for OAuth client secrets, backed by Spring Security Crypto.

## Key Components

| Method | Description |
|---|---|
| `EncryptionService(password, salt)` | Constructor — validates minimum key/salt lengths (32 / 16 chars) and initializes `TextEncryptor` |
| `encrypt(plainText)` | Encrypts arbitrary plain text; throws `EncryptionException` on failure |
| `decrypt(encryptedText)` | Decrypts previously encrypted text; throws `EncryptionException` on failure |
| `encryptClientSecret(clientSecret)` | Thin wrapper around `encrypt()` scoped to OAuth client secrets |
| `decryptClientSecret(encryptedClientSecret)` | Thin wrapper around `decrypt()` scoped to OAuth client secrets |

## Configuration

The service requires two properties in your application configuration:

```yaml
security:
  encryption:
    password: "your-minimum-32-character-password-here"
    salt: "your16charhexsalt"   # minimum 16 hex-encoded characters
```

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class OAuthConfigService {

    private final EncryptionService encryptionService;

    // Store an OAuth integration
    public void saveOAuthConfig(OAuthConfig config) {
        String encrypted = encryptionService.encryptClientSecret(config.getClientSecret());
        config.setClientSecret(encrypted);
        oauthConfigRepository.save(config);
    }

    // Retrieve and use an OAuth integration
    public OAuthConfig loadOAuthConfig(String id) {
        OAuthConfig config = oauthConfigRepository.findById(id);
        String plainSecret = encryptionService.decryptClientSecret(config.getClientSecret());
        config.setClientSecret(plainSecret);
        return config;
    }
}
```

> **Note:** The `TextEncryptor` produces a different ciphertext on each call (random IV) — store the full encrypted string and always decrypt via `decryptClientSecret()` rather than comparing ciphertext directly.
