Initializes and deploys Apache Pinot schemas and table configurations to the Pinot controller on application startup, with retry logic and placeholder resolution support.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `PINOT_CONFIGS` | `List<PinotConfig>` | Static list of Pinot configurations to deploy (`devices`, `logs`) |
| `init()` | `@EventListener` method | Triggers deployment on `ApplicationReadyEvent`; collects and reports failures |
| `deployPinotConfig()` | Private method | Orchestrates schema + table config deployment for a single `PinotConfig` entry |
| `deploySchema()` | Private method | POSTs a schema JSON to `/schemas` on the Pinot controller |
| `deployTableConfig()` | Private method | PUTs (update) or POSTs (create) a table config to `/tables`, auto-detecting `_REALTIME`/`_OFFLINE` suffix |
| `deployWithRetry()` | Private method | Wraps a deployment `Runnable` with configurable retry logic on `ResourceAccessException` |
| `loadResource()` | Private method | Reads JSON config files from `classpath:pinot/config/` |
| `resolvePlaceholders()` | Private method | Resolves Spring `Environment` property placeholders inside config file content |

## Configuration Properties

| Property | Default | Description |
|----------|---------|-------------|
| `pinot.controller.url` | — | Pinot controller host:port |
| `pinot.config.enabled` | `true` | Toggle deployment on/off |
| `pinot.config.retry.max-attempts` | `5` | Max retry attempts per deployment |
| `pinot.config.retry.delay-ms` | `5000` | Delay between retries (ms) |

## Usage Example

```java
# application.properties
pinot.controller.url=localhost:9000
pinot.config.enabled=true
pinot.config.retry.max-attempts=5
pinot.config.retry.delay-ms=5000
```

```text
classpath:
  pinot/config/
    schema-devices.json
    table-config-devices.json
    schema-logs.json
    table-config-logs-realtime.json
```

On startup, the initializer automatically:
1. Loads and resolves placeholders in each JSON config file
2. Deploys the schema via `POST /schemas`
3. Upserts the table config via `PUT /tables/{name}`, falling back to `POST /tables` on 404
