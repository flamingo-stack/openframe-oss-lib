Unit test suite for `NotificationBroadcaster`, verifying persistence, read-state creation, NATS publishing fan-out, and error-recovery behavior across admin, machine, and mixed audience scenarios.

## Key Components

| Test Method | What It Validates |
|---|---|
| `admin_only_command_fans_out_to_admin_path` | Admin-only commands create `USER` read states and call `publishToUser` per recipient; no machine-side calls |
| `machine_only_command_fans_out_to_machine_path` | Machine-only commands create `MACHINE` read states and call `publishToMachine`; no admin-side calls |
| `mixed_audience_fans_out_to_both_paths` | Mixed audiences trigger both `createForAudience` invocations and both publish loops |
| `no_publisher_persists_without_nats` | When NATS publisher is absent (`Optional.empty()`), notification is still persisted and read states created |
| `persisted_notification_carries_command_fields` | Saved document carries `title`, `description`, `severity`, and `context` verbatim from the command |
| `broadcast_returns_persisted_notification` | Returned `Notification` has the repository-assigned ID, not a transient builder object |
| `category_resolved_from_registry_and_denormalized_into_read_state` | Category is resolved via `NotificationContextDescriptorRegistry` by context type before persisting read state |
| `create_for_audience_failure_triggers_orphan_cleanup_and_skips_nats` | On `createForAudience` failure, the orphan notification document is deleted and NATS publish is skipped |
| `nats_publish_failure_for_one_recipient_does_not_skip_others` | A NATS exception for one recipient does not abort delivery to remaining recipients |

## Usage Example

```java
// Typical test setup pattern used in this suite
@BeforeEach
void setUp() {
    notificationRepository = mock(NotificationRepository.class);
    readStateService       = mock(NotificationReadStateService.class);
    descriptorRegistry     = mock(NotificationContextDescriptorRegistry.class);
    natsPublisher          = mock(NotificationNatsPublisher.class);

    // Simulate repository assigning a persistent ID on save
    when(notificationRepository.save(any(Notification.class))).thenAnswer(inv -> {
        Notification arg = inv.getArgument(0);
        arg.setId("notif-id-1");
        return arg;
    });

    when(descriptorRegistry.categoryOf(anyString()))
        .thenReturn(NotificationCategory.MINGO);

    broadcaster = newBroadcaster(Optional.of(natsPublisher), true);
}

// Broadcasting to a mixed audience
@Test
void mixed_audience_fans_out_to_both_paths() {
    NotificationCommand cmd = NotificationCommand.builder()
        .title("Ticket status changed")
        .severity(NotificationSeverity.INFO)
        .context(genericContext("TICKET_STATUS_CHANGED"))
        .adminAudience(Set.of("admin-1"))
        .machineAudience(Set.of("m-1"))
        .build();

    broadcaster.broadcast(cmd);

    // One read-state row per audience type
    verify(readStateService).createForAudience(
        eq("notif-id-1"), eq(NotificationCategory.TICKETS),
        anyString(), eq(RecipientType.USER), eq(Set.of("admin-1")));
    verify(readStateService).createForAudience(
        eq("notif-id-1"), eq(NotificationCategory.TICKETS),
        anyString(), eq(RecipientType.MACHINE), eq(Set.of("m-1")));

    // NATS fan-out per recipient
    verify(natsPublisher).publishToUser(eq("admin-1"), any(Notification.class));
    verify(natsPublisher).publishToMachine(eq("m-1"), any(Notification.class));
}
```
