Unit test suite for `CommandNatsPublisher`, verifying NATS subject routing, payload validation, transport selection, and error propagation for RMM command and cancel message publishing.

## Key Components

| Test Method | What It Verifies |
|---|---|
| `publishCommand_routesToMachineSubjectWithUnchangedPayload` | Subject resolves to `machine.<machineId>.command-execution` and payload is forwarded unchanged |
| `publishCommand_usesCoreNatsNotJetStream` | Core NATS `publish()` is used — never `publishPersistent()` |
| `publishCommand_rejectsBlankMachineId` | Blank/whitespace `machineId` throws `IllegalArgumentException` before any broker call |
| `publishCommand_rejectsNullMessage` | Null `CommandMessage` throws `IllegalArgumentException` |
| `publishCommand_rejectsBlankExecutionId` | Blank `executionId` throws `IllegalArgumentException` |
| `publishCommand_propagatesTransportFailure` | `NatsException` from the broker propagates to the caller |
| `publishCancel_routesToCancelSubject` | Subject resolves to `machine.<machineId>.command-cancel` |
| `publishCancel_usesCoreNatsNotJetStream` | Cancel messages also use core NATS, not JetStream |
| `publishCancel_rejectsBlankMachineId` | Blank `machineId` rejected before broker call |
| `publishCancel_rejectsMissingExecutionId` | Both null message and blank `executionId` rejected |

## Usage Example

```java
// Test setup pattern used across all cases
@BeforeEach
void setUp() {
    messagePublisher = mock(NatsMessagePublisher.class);
    publisher = new CommandNatsPublisher(messagePublisher);
}

// Verifying subject routing with ArgumentCaptor
CommandMessage message = CommandMessage.builder()
        .executionId("exec-1")
        .code("df -h")
        .build();

publisher.publishCommand("machine-42", message);

ArgumentCaptor<String> subject = ArgumentCaptor.forClass(String.class);
verify(messagePublisher).publish(subject.capture(), any());
assertThat(subject.getValue()).isEqualTo("machine.machine-42.command-execution");
```

## Design Rationale

- **Core NATS over JetStream** — Commands and cancels are intentionally non-durable. Re-delivering a cancel after a broker restart could target an unrelated subsequent execution.
- **Pre-flight validation** — All guard checks run before any broker interaction, ensuring `NatsMessagePublisher` is never called with a malformed subject or incomplete correlation ID.
