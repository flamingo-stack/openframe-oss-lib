Unit test suite for `CommandNatsPublisher`, verifying NATS subject routing, payload integrity, input validation, and transport-layer behavior for RMM command dispatch and cancellation.

## Key Components

| Test Method | Covers |
|---|---|
| `publishCommand_routesToMachineSubjectWithUnchangedPayload` | Subject format `machine.<machineId>.command-execution` and payload pass-through |
| `publishCommand_usesCoreNatsNotJetStream` | Enforces core NATS (not JetStream/persistent) for ad-hoc commands |
| `publishCommand_rejectsBlankMachineId` | Guards against malformed subjects like `machine..command-execution` |
| `publishCommand_rejectsNullMessage` / `_rejectsBlankExecutionId` | Input validation before broker interaction |
| `publishCommand_propagatesTransportFailure` | `NatsException` surfaces to the GraphQL mutation layer |
| `publishCancel_routesToCancelSubject` | Distinct subject `machine.<machineId>.command-cancel` for agent prioritization |
| `publishCancel_usesCoreNatsNotJetStream` | Prevents re-delivery of cancel signals after broker restarts |
| `publishCancel_rejectsMissingExecutionId` | Null message and blank `executionId` both rejected |

## Usage Example

```java
// Wiring the publisher under test (mirrors setUp())
NatsMessagePublisher messagePublisher = mock(NatsMessagePublisher.class);
CommandNatsPublisher publisher = new CommandNatsPublisher(messagePublisher);

// Asserting subject routing with ArgumentCaptor
ArgumentCaptor<String> subject = ArgumentCaptor.forClass(String.class);
ArgumentCaptor<CommandMessage> body = ArgumentCaptor.forClass(CommandMessage.class);

publisher.publishCommand("machine-42", CommandMessage.builder()
        .executionId("exec-1")
        .code("df -h")
        .build());

verify(messagePublisher).publish(subject.capture(), body.capture());
assertThat(subject.getValue()).isEqualTo("machine.machine-42.command-execution");

// Asserting validation guard — no broker calls on bad input
assertThatThrownBy(() -> publisher.publishCommand("   ", message))
        .isInstanceOf(IllegalArgumentException.class)
        .hasMessageContaining("machineId");
verifyNoInteractions(messagePublisher);
```

> **Design intent:** Both `publishCommand` and `publishCancel` intentionally use core NATS (not JetStream) — commands must not be durable or replayable, and cancel signals must not re-fire against a later unrelated execution after a broker restart.
