Abstract base class for integration tests that provisions containerized MongoDB and NATS infrastructure using Testcontainers, and dynamically registers their connection properties with the Spring test context.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `MONGO` | `MongoDBContainer` | MongoDB 7 container shared across tests |
| `NATS` | `GenericContainer` | NATS 2.10 container with JetStream (`-js`) enabled, exposed on port `4222` |
| `infrastructureProperties()` | `@DynamicPropertySource` | Lazily starts containers and registers Spring properties for MongoDB URI, auto-index creation, Spring Cloud Config/Stream, and NATS server addresses |
| `natsUri()` | `protected static String` | Lazily starts the NATS container and returns its `nats://host:port` connection string |

## Usage Example

Extend this class in any Spring integration test that requires MongoDB and NATS:

```java
@SpringBootTest
@ActiveProfiles("test")
class MyServiceIntegrationTest extends BaseIntegrationTest {

    @Autowired
    private MyService myService;

    @Test
    void shouldPublishAndConsumeMessage() {
        // MongoDB and NATS containers are already running and
        // their properties are injected into the Spring context
        myService.processEvent("test-payload");

        // assert side effects in MongoDB or NATS subscriptions
    }
}
```

> **Note:** Containers are started lazily on first access and shared across the test suite via `static` fields, minimizing container startup overhead. NATS waits for the `Server is ready` log line before tests proceed, ensuring a healthy connection.
