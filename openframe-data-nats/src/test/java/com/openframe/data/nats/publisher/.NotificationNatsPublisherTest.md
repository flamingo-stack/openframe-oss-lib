Unit test suite for `NotificationNatsPublisher`, verifying correct NATS subject routing, message payload integrity, and guard-rail validations before any broker interaction occurs.

## Key Components

| Component | Role |
|-----------|------|
| `NotificationNatsPublisher` | Subject under test — wraps `NatsMessagePublisher` to route notifications |
| `NatsMessagePublisher` | Mocked broker dependency injected via constructor |
| `ArgumentCaptor<String>` | Captures the NATS subject string for assertion |
| `ArgumentCaptor<NotificationMessage>` | Captures the serialized message payload for assertion |
| `persistedNotification()` | Shared factory method producing a valid, ID-bearing `Notification` |

## Key Scenarios

| Test | Behaviour Verified |
|------|--------------------|
| `publish_to_user_routes_to_user_subject` | Subject formatted as `user.<userId>.notification`; `id` and `title` preserved |
| `publish_to_machine_routes_to_machine_subject` | Subject formatted as `machine.<machineId>.notification` |
| `broker_failure_swallowed` | `NatsException` from broker does not propagate — MongoDB is source of truth |
| `blank_user_id_rejected` | Blank/whitespace `userId` raises `IllegalArgumentException` before broker call |
| `blank_machine_id_rejected` | Empty `machineId` raises `IllegalArgumentException` before broker call |
| `unpersisted_notification_throws_before_broker` | Notification without an `id` raises `IllegalArgumentException` — caller must `save()` first |

## Usage Example

```java
// Wiring under test (mirrors production constructor injection)
NatsMessagePublisher messagePublisher = mock(NatsMessagePublisher.class);
NotificationNatsPublisher publisher = new NotificationNatsPublisher(messagePublisher);

// Assert subject routing
Notification notification = Notification.builder()
        .id("notif-123")
        .title("Hello")
        .createdAt(Instant.now())
        .context(GenericContext.builder().type("welcome").build())
        .build();

publisher.publishToUser("user-42", notification);

ArgumentCaptor<String> subject = ArgumentCaptor.forClass(String.class);
verify(messagePublisher).publish(subject.capture(), any());
assertThat(subject.getValue()).isEqualTo("user.user-42.notification");
```

> **Design note:** Broker failures are intentionally swallowed — NATS delivery is best-effort and MongoDB remains the authoritative store. All guard-rail checks (`blank id`, `unpersisted notification`) fire **before** any broker interaction, keeping `verifyNoInteractions(messagePublisher)` as a clean assertion pattern.
