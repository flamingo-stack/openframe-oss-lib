A Spring service that provides NATS message publishing capabilities via two delivery strategies: fire-and-forget via Spring Cloud Stream and persistent JetStream publishing.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `publish()` | Method | Sends a message to a NATS subject using `StreamBridge` (Spring Cloud Stream); throws `NatsException` if delivery fails |
| `publishPersistent()` | Method | Publishes a durable message via NATS JetStream, serializing the payload to JSON bytes; returns a `PublishAck` confirming server receipt |
| `@ConditionalOnProperty` | Annotation | Bean is only registered when `spring.cloud.stream.enabled` is `true` |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class TicketEventService {

    private final NatsMessagePublisher publisher;

    // Fire-and-forget via Spring Cloud Stream
    public void notifyTicketCreated(TicketEvent event) {
        publisher.publish("tickets.created", event);
    }

    // Persistent delivery with acknowledgement via JetStream
    public void persistTicketUpdate(TicketEvent event) {
        PublishAck ack = publisher.publishPersistent("tickets.updated", event);
        log.info("Message persisted at sequence: {}", ack.getSeqno());
    }
}
```

> **When to use which method:**
> Use `publish()` for lightweight, ephemeral events where guaranteed delivery is not critical. Use `publishPersistent()` when the message must survive broker restarts and delivery confirmation is required (e.g., billing events, audit logs).
