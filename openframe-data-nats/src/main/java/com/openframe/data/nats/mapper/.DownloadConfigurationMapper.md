Maps document-layer `DownloadConfiguration` objects to their NATS messaging model equivalents, resolving download links and translating installation types in the process.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `downloadConfigurationResolver` | `DownloadConfigurationLinkResolver` | Resolves the versioned download URL for each configuration entry |
| `map()` | Public method | Converts a list of document-layer configs to NATS model configs for a given version |
| `mapToNatsDownloadConfiguration()` | Private method | Handles field-by-field mapping for a single configuration entry |
| `mapInstallationType()` | Private method | Translates `InstallationType` enum values; defaults to `STANDARD` when `null` |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class ClientConfigurationService {

    private final DownloadConfigurationMapper downloadConfigurationMapper;

    public List<DownloadConfiguration> buildNatsPayload(
            List<com.openframe.data.document.clientconfiguration.DownloadConfiguration> rawConfigs,
            String version
    ) {
        // Returns null if rawConfigs is null; empty list if rawConfigs is empty
        return downloadConfigurationMapper.map(rawConfigs, version);
    }
}
```

> **Note:** The `map()` method returns `null` when the input list is `null`. Callers should guard against this if a non-null result is required. Supported `InstallationType` values are `STANDARD`, `SERVICE`, and `GUI_APP`; any unmapped or `null` value falls back to `STANDARD`.
