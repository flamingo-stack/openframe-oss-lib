Maps domain `LocalFilenameConfiguration` objects to their NATS messaging counterparts (`ToolInstallationMessage.LocalFilenameConfiguration`), enabling tool installation events to be published over NATS with OS-specific filename metadata.

## Key Components

| Element | Description |
|---|---|
| `map(List<...>)` | Converts a list of document-layer `LocalFilenameConfiguration` objects to NATS message model equivalents; returns `null` if input is `null` |
| `mapToNatsLocalFilenameConfiguration(...)` | Private helper that transfers `filename` and `os` fields from the document model to the NATS event model |

## Usage Example

```java
@Service
@RequiredArgsConstructor
public class ToolInstallationEventService {

    private final LocalFilenameConfigurationMapper localFilenameConfigurationMapper;

    public ToolInstallationMessage buildMessage(ToolAgent toolAgent) {
        ToolInstallationMessage message = new ToolInstallationMessage();

        List<ToolInstallationMessage.LocalFilenameConfiguration> natsConfigs =
                localFilenameConfigurationMapper.map(toolAgent.getLocalFilenameConfigurations());

        message.setLocalFilenameConfigurations(natsConfigs);
        return message;
    }
}
```

> **Note:** The mapper handles `null` input gracefully by returning `null`, so callers do not need to perform null checks before invoking `map()`. Only `filename` and `os` fields are mapped — extend `mapToNatsLocalFilenameConfiguration` if additional fields are added to either model.
