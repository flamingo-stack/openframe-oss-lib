Publishes tool agent update messages to NATS persistent subjects, handling publish failures and optimistic locking conflicts via a fallback scheduler retry mechanism.

## Key Components

| Member | Description |
|--------|-------------|
| `TOPIC_NAME_TEMPLATE` | Subject pattern `machine.all.tool.<id>.update` used to route updates per tool agent |
| `publish()` | High-level method: sends the update, marks the agent as published/non-published, and handles concurrent write conflicts |
| `send()` | Low-level method: builds the topic name and message, then delegates to `NatsMessagePublisher.publishPersistent()` |
| `buildMessage()` | Maps an `IntegratedToolAgent` to a `ToolAgentUpdateMessage`, including download configurations and assets |
| `mapAssets()` | Converts a list of `ToolAgentAsset` to `AssetUpdate` DTOs with version-resolved download configs |

**Conditional activation:** The service is only instantiated when `spring.cloud.stream.enabled` is set.

## Usage Example

```java
// Inject and call publish() to send a tool agent update over NATS
@Service
@RequiredArgsConstructor
public class ToolAgentUpdateScheduler {

    private final ToolAgentUpdateUpdatePublisher publisher;
    private final IntegratedToolAgentService toolAgentService;

    @Scheduled(fixedDelay = 30_000)
    public void retryUnpublished() {
        List<IntegratedToolAgent> pending = toolAgentService.findNonPublished();
        for (IntegratedToolAgent agent : pending) {
            // Sends to: machine.all.tool.<agentId>.update
            // Marks agent as published on success,
            // or non-published on failure for next retry cycle
            publisher.publish(agent);
        }
    }
}
```

> **Concurrency note:** `OptimisticLockingFailureException` is caught on both the success and failure marking paths — a concurrent writer winning the race is treated as a safe skip rather than an error.
