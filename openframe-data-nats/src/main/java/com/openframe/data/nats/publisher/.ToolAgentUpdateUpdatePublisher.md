Publishes tool agent update messages to NATS persistent streams when an `IntegratedToolAgent` is updated, handling optimistic locking conflicts and marking publish state accordingly.

## Key Components

| Element | Description |
|---|---|
| `TOPIC_NAME_TEMPLATE` | Topic pattern `machine.all.tool.<toolAgentId>.update` |
| `publish()` | Sends the update and marks the agent as published; marks as non-published on failure |
| `send()` | Builds and dispatches the `ToolAgentUpdateMessage` to the NATS persistent stream |
| `buildMessage()` | Constructs the full `ToolAgentUpdateMessage` including version, session type, download configs, and assets |
| `mapAssets()` | Maps `ToolAgentAsset` list to `AssetUpdate` DTOs |
| `@ConditionalOnProperty` | Only active when `spring.cloud.stream.enabled` is set |

## Usage Example

```java
// Triggered when a tool agent update needs to be broadcast
IntegratedToolAgent toolAgent = integratedToolAgentService.findById("my-tool");

// Full publish with state tracking (marks as published or non-published)
toolAgentUpdateUpdatePublisher.publish(toolAgent);

// Fire-and-forget send without state management
toolAgentUpdateUpdatePublisher.send(toolAgent);
```

> **Note:** `publish()` handles `OptimisticLockingFailureException` on both the failure-mark and success-mark paths, safely skipping the state update if a concurrent writer is detected. Failed publishes are left in a non-published state for retry by a scheduled job.
