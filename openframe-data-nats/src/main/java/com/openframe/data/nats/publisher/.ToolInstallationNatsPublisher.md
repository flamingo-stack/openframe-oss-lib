Publishes tool installation messages to NATS for a specific machine, triggering agent deployment or reinstallation workflows.

## Key Components

- **`TOPIC_NAME_TEMPLATE`** ‚Äî Topic pattern `machine.{machineId}.tool-installation` used to route messages per machine
- **`publish(machineId, toolAgent, tool)`** ‚Äî Publishes a standard installation message (reinstall defaults to `false`)
- **`publish(machineId, toolAgent, tool, reinstall)`** ‚Äî Publishes an installation message with explicit reinstall flag via persistent NATS delivery
- **`buildMessage(...)`** ‚Äî Assembles a `ToolInstallationMessage` from `IntegratedToolAgent` and `IntegratedTool`, mapping download configs, assets, and command args
- **`mapAssets(...)`** ‚Äî Converts a list of `ToolAgentAsset` into `ToolInstallationMessage.Asset` objects including filenames, download configs, and source
- **`mapAssetSource(...)`** ‚Äî Translates `ToolAgentAssetSource` enum (`ARTIFACTORY`, `TOOL_API`, `GITHUB`) to the message-layer equivalent

> **Note:** Active only when `spring.cloud.stream.enabled` property is set (`@ConditionalOnProperty`).

## Usage Example

```java
// Standard installation
toolInstallationNatsPublisher.publish(machineId, toolAgent, tool);

// Force reinstall
toolInstallationNatsPublisher.publish(machineId, toolAgent, tool, true);
```

The publisher delegates to `NatsMessagePublisher.publishPersistent(...)`, ensuring the message is durably stored in the NATS JetStream layer until the target machine acknowledges receipt.
