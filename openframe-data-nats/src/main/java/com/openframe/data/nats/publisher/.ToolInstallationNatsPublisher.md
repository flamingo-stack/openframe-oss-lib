Publishes tool installation messages to NATS subjects targeting specific machines, enabling remote tool agent deployment and reinstallation via persistent messaging.

## Key Components

- **`TOPIC_NAME_TEMPLATE`** — Subject pattern `machine.{machineId}.tool-installation` used to route messages to the correct machine
- **`publish(machineId, toolAgent, tool)`** — Convenience overload that triggers a standard (non-reinstall) installation message
- **`publish(machineId, toolAgent, tool, reinstall)`** — Core publish method; builds the topic and message, then delegates to `NatsMessagePublisher.publishPersistent()`
- **`buildMessage(...)`** — Assembles a `ToolInstallationMessage` from the `IntegratedToolAgent` and `IntegratedTool`, mapping download configs, assets, and all command args
- **`mapAssets(...)`** — Converts a list of `ToolAgentAsset` domain objects to `ToolInstallationMessage.Asset` message models
- **`mapAssetSource(...)`** — Translates `ToolAgentAssetSource` enum values (`ARTIFACTORY`, `TOOL_API`, `GITHUB`) to their message-layer counterparts

## Usage Example

```java
// Standard installation
toolInstallationNatsPublisher.publish(
    "machine-abc123",
    integratedToolAgent,
    integratedTool
);

// Force reinstall
toolInstallationNatsPublisher.publish(
    "machine-abc123",
    integratedToolAgent,
    integratedTool,
    true
);
```

> **Note:** This component is conditionally activated by the `spring.cloud.stream.enabled` property. It will not be registered as a Spring bean unless that property is set to `true`.
