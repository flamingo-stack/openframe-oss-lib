Orchestrates the full lifecycle of a notification broadcast: persisting the notification document, creating per-recipient read states, and publishing real-time events via NATS.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `broadcast(NotificationCommand)` | `Notification` | Main entry point — persists, hydrates read states, and publishes to all recipients |
| `publishSafely(...)` | `void` | Wraps individual NATS publishes in a non-fatal try/catch; failed deliveries fall back to GraphQL catch-up |
| `notificationsEnabled` | `boolean` | Feature flag (`openframe.features.notifications.enabled`); short-circuits the entire flow when `false` |
| `natsPublisher` | `Optional<NotificationNatsPublisher>` | Optional NATS transport; absent in environments where NATS is not configured |
| `readStateService` | `NotificationReadStateService` | Creates per-recipient read-state records for both `USER` and `MACHINE` recipient types |

## Error Handling Strategy

```text
persist notification
  └─► createForAudience (USER + MACHINE)
        └─► RuntimeException → delete orphaned doc → rethrow (caller retries)
publish via NATS (per recipient)
        └─► RuntimeException → warn + skip (GraphQL catch-up covers gap)
```

Persistence failures are hard (rollback + rethrow); NATS delivery failures are soft (warn-only).

## Usage Example

```java
NotificationCommand command = NotificationCommand.builder()
        .severity(Severity.WARNING)
        .title("Disk usage critical")
        .description("Volume /data is at 95% capacity")
        .context(NotificationContext.of(ContextType.MACHINE_HEALTH))
        .adminAudience(Set.of("admin-001", "admin-002"))
        .machineAudience(Set.of("machine-abc"))
        .build();

Notification result = notificationBroadcaster.broadcast(command);
// result is null when notificationsEnabled=false
```
