Represents the configuration model for an agent download, encapsulating metadata required to identify, retrieve, and install an agent package for a specific operating system.

## Key Components

| Field | Type | Description |
|---|---|---|
| `os` | `String` | Target operating system for the download |
| `fileName` | `String` | Name of the file to download |
| `targetFileName` | `String` | Mapped from JSON as `agentFileName`; the local target file name |
| `link` | `String` | Download URL for the agent package |
| `installationType` | `InstallationType` | Installation mode; defaults to `STANDARD` |
| `bundleId` | `String` | Bundle identifier for the agent package |
| `serviceName` | `String` | Name of the service associated with the download |

## Usage Example

```java
DownloadConfiguration config = new DownloadConfiguration();
config.setOs("windows");
config.setFileName("openframe-agent.exe");
config.setTargetFileName("agent.exe");
config.setLink("https://downloads.openframe.ai/agent/windows/openframe-agent.exe");
config.setInstallationType(InstallationType.STANDARD);
config.setBundleId("com.openframe.agent");
config.setServiceName("openframe-agent-svc");
```

Or deserialized from a NATS message payload:

```java
DownloadConfiguration config = objectMapper.readValue(payload, DownloadConfiguration.class);
// JSON field "agentFileName" maps to targetFileName via @JsonProperty
```

## Notes

- Annotated with Lombok `@Data` — generates getters, setters, `equals`, `hashCode`, and `toString` automatically.
- The `@JsonProperty("agentFileName")` annotation ensures backward-compatible JSON deserialization when the wire format differs from the field name.
- Lives in the `com.openframe.data.nats.model` package, indicating use within the OpenFrame NATS messaging layer.
