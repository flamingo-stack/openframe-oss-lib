Publishes notification events to NATS subjects for user and machine targets, wrapping a `NatsMessagePublisher` with topic routing logic and input validation.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `USER_TOPIC_TEMPLATE` | Constant | Topic pattern `user.{userId}.notification` for user-scoped subjects |
| `MACHINE_TOPIC_TEMPLATE` | Constant | Topic pattern `machine.{machineId}.notification` for machine-scoped subjects |
| `publishToUser()` | Method | Validates `userId` and publishes a notification to the user-specific NATS subject |
| `publishToMachine()` | Method | Validates `machineId` and publishes a notification to the machine-specific NATS subject |
| `publish()` | Private Method | Validates the notification has been persisted (non-null ID), then delegates to `NatsMessagePublisher`; logs a warning on `NatsException` instead of propagating |
| `buildMessage()` | Private Method | Maps a `Notification` document to a `NotificationMessage` NATS payload |

> **Activation:** This component is only registered when `spring.cloud.stream.enabled=true` is set in the application properties (`@ConditionalOnProperty`).

## Usage Example

```java
// Inject the publisher
@RequiredArgsConstructor
public class AlertService {

    private final NotificationNatsPublisher notificationNatsPublisher;

    public void notifyUser(String userId, Notification savedNotification) {
        // Notification must already be persisted (non-null ID)
        notificationNatsPublisher.publishToUser(userId, savedNotification);
        // Publishes to: user.{userId}.notification
    }

    public void notifyMachine(String machineId, Notification savedNotification) {
        notificationNatsPublisher.publishToMachine(machineId, savedNotification);
        // Publishes to: machine.{machineId}.notification
    }
}
```

> **Error Handling:** Blank IDs throw `IllegalArgumentException` eagerly. Unpersisted notifications (null ID) also throw. NATS transport failures are caught and logged as warnings — callers are not interrupted by publish failures.
