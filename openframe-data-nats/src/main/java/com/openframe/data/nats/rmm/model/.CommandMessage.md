NATS wire payload model for dispatching a single ad-hoc command to an OpenFrame agent, published on the `machine.{machineId}.command-execution` subject.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `executionId` | `String` | Unique identifier for the command execution (e.g., ULID) |
| `code` | `String` | The command or script body to execute |
| `shell` | `ScriptShell` | Target shell interpreter (e.g., `BASH`, `POWERSHELL`) |
| `privilegeLevel` | `PrivilegeLevel` | Execution privilege context (e.g., `ADMIN`, `USER`) |
| `timeout` | `Integer` | Maximum execution time in seconds; omitted if `null` |

Null fields are excluded from serialization via `@JsonInclude(NON_NULL)`. Lombok `@Builder` and `@Data` provide fluent construction and accessors.

## Usage Example

```java
CommandMessage message = CommandMessage.builder()
    .executionId("01HXYZ...")
    .code("echo hello")
    .shell(ScriptShell.BASH)
    .privilegeLevel(PrivilegeLevel.ADMIN)
    .timeout(30)
    .build();

// Publish over NATS
String subject = "machine." + machineId + ".command-execution";
natsConnection.publish(subject, objectMapper.writeValueAsBytes(message));
```
