NATS wire payload model for dispatching a single ad-hoc command to an OpenFrame agent, published on the `machine.{machineId}.command-execution` subject.

## Key Components

| Field | Type | Description |
|---|---|---|
| `executionId` | `String` | Unique identifier for this command execution |
| `code` | `String` | The command/script content to execute |
| `shell` | `ScriptShell` | Target shell interpreter (e.g., `BASH`, `POWERSHELL`) |
| `privilegeLevel` | `PrivilegeLevel` | Execution privilege level (e.g., `ADMIN`, `USER`) |
| `timeout` | `Integer` | Maximum execution time in seconds; `null` fields are omitted via `@JsonInclude` |

## Usage Example

```java
CommandMessage message = CommandMessage.builder()
    .executionId("01HXYZ...")
    .code("echo hello")
    .shell(ScriptShell.BASH)
    .privilegeLevel(PrivilegeLevel.ADMIN)
    .timeout(30)
    .build();

// Publish to agent over NATS
String subject = "machine." + machineId + ".command-execution";
natsConnection.publish(subject, objectMapper.writeValueAsBytes(message));
```

> **Note:** `@JsonInclude(NON_NULL)` ensures unset optional fields (e.g., `timeout`) are excluded from the serialized payload, keeping the wire format clean for the receiving agent.
