Immutable command object representing a notification dispatch request, validated at construction time using the Builder pattern.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `title` | `String` | Required non-blank notification title |
| `description` | `String` | Optional notification body text |
| `severity` | `NotificationSeverity` | Required severity level (e.g. INFO, WARNING, CRITICAL) |
| `context` | `NotificationContext` | Required context with a non-blank `type` field |
| `adminAudience` | `Set<String>` | Target admin recipient IDs |
| `machineAudience` | `Set<String>` | Target machine/device recipient IDs |
| `sanitizeAudience()` | `private static` | Null-safe audience validator — rejects blank entries, returns immutable copy |

## Validation Rules

All constraints are enforced in the constructor, failing fast with `IllegalArgumentException` or `NullPointerException`:

- `title` must not be blank
- `severity` and `context` must not be null
- `context.type` must not be blank
- At least one of `adminAudience` or `machineAudience` must be non-empty
- Neither audience set may contain blank string entries

## Usage Example

```java
NotificationCommand command = NotificationCommand.builder()
    .title("Disk Usage Critical")
    .description("Server disk usage has exceeded 90%")
    .severity(NotificationSeverity.CRITICAL)
    .context(NotificationContext.builder()
        .type("infrastructure")
        .build())
    .adminAudience(Set.of("admin-001", "admin-002"))
    .machineAudience(Set.of("server-nyc-01"))
    .build();

// Access fields via Lombok-generated getters
String title = command.getTitle();
Set<String> admins = command.getAdminAudience();
```

> Both `adminAudience` and `machineAudience` are stored as immutable sets via `Set.copyOf()`, preventing post-construction mutation.
