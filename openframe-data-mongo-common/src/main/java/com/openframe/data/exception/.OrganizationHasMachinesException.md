Exception thrown when an archive operation is attempted on an organization that still has active machines/devices associated with it.

## Key Components

- **`OrganizationHasMachinesException`** — Extends `ConflictException`, representing an HTTP 409 Conflict scenario
- **`ErrorCode.ORGANIZATION_HAS_MACHINES`** — Standardized error code used for client-side error identification
- **Constructor** — Accepts `organizationId` as context, paired with a fixed user-friendly message guiding resolution

## Usage Example

```java
// Thrown during organization archive validation
if (!machineRepository.findActiveByOrganizationId(organizationId).isEmpty()) {
    throw new OrganizationHasMachinesException(organizationId);
}
```

The resulting error payload will carry the `ORGANIZATION_HAS_MACHINES` error code and the message:

```text
This organization still has active devices. To archive it, you'll need to delete or archive all devices first.
```

> **Note:** Callers should handle this exception by prompting users to delete or archive all associated devices before retrying the organization archive operation.
