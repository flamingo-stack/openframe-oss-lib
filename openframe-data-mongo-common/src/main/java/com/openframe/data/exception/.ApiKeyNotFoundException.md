Custom exception for handling missing API key lookups, extending `NotFoundException` with the `API_KEY_NOT_FOUND` error code.

## Key Components

- **`ApiKeyNotFoundException(String keyId)`** — Thrown when an API key cannot be located by its ID alone.
- **`ApiKeyNotFoundException(String keyId, String userId)`** — Thrown when an API key is not found for a specific user, providing both identifiers in the message.
- **`ApiKeyNotFoundException(String message, Throwable cause)`** — Wraps an underlying cause with a custom message, useful for chained exception handling.

## Usage Example

```java
// Lookup by key ID only
ApiKey key = apiKeyRepository.findById(keyId)
    .orElseThrow(() -> new ApiKeyNotFoundException(keyId));

// Lookup scoped to a specific user
ApiKey key = apiKeyRepository.findByIdAndUserId(keyId, userId)
    .orElseThrow(() -> new ApiKeyNotFoundException(keyId, userId));

// Wrapping a lower-level exception
try {
    apiKeyStore.fetch(keyId);
} catch (StorageException e) {
    throw new ApiKeyNotFoundException("Failed to retrieve API key: " + keyId, e);
}
```

All constructors delegate to `NotFoundException` with `ErrorCode.API_KEY_NOT_FOUND`, ensuring consistent error classification and HTTP response mapping across the OpenFrame platform.
