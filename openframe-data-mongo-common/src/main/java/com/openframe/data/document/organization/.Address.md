Represents a physical address as an embedded MongoDB document within the organization domain — not stored as a separate collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `street1` | `String` | Primary street address line |
| `street2` | `String` | Secondary address line (optional) |
| `city` | `String` | City name |
| `state` | `String` | State or province |
| `postalCode` | `String` | ZIP or postal code |
| `country` | `String` | Country name |

Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) provide getters, setters, builder pattern, and constructors automatically.

## Usage Example

```java
// Builder pattern
Address address = Address.builder()
    .street1("123 Main St")
    .street2("Suite 400")
    .city("Austin")
    .state("TX")
    .postalCode("78701")
    .country("US")
    .build();

// Embedded within an organization document
Organization org = Organization.builder()
    .name("Acme Corp")
    .address(address)
    .build();
```

> **Embedded Document:** `Address` is designed to be nested inside parent documents (e.g., `Organization`) rather than persisted as a standalone MongoDB collection. No `@Document` annotation is present by design.
