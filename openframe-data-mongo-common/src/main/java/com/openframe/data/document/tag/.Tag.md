MongoDB document entity representing a tenant-scoped tag definition, used to categorize and label entities (e.g., devices, organizations) within a tenant's namespace.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier |
| `tenantId` | `String` | Tenant scope identifier (via `TenantScoped`) |
| `key` | `String` | Tag key identifier (e.g., `"site"`, `"org_type"`) — unique per `entityType` per tenant |
| `values` | `List<String>` | Predefined allowed values; `null` = free-text, non-empty = dropdown/checkbox UI |
| `entityType` | `TagEntityType` | The entity category this tag applies to |
| `color` | `String` | Optional display color |
| `createdAt` / `createdBy` | `Instant` / `String` | Audit fields |

**Compound Index:** `tenantId + key + entityType` enforces uniqueness — no duplicate tag keys for the same entity type within a tenant.

## Usage Example

```java
// Create a site tag with predefined location values
Tag siteTag = Tag.builder()
    .tenantId("tenant-123")
    .key("site")
    .description("Office location")
    .color("#FFC008")
    .values(List.of("CHICAGO", "DENVER", "NEW_YORK"))
    .entityType(TagEntityType.DEVICE)
    .createdAt(Instant.now())
    .createdBy("admin-user-id")
    .build();

// Create a free-text label tag (no predefined values)
Tag labelTag = Tag.builder()
    .tenantId("tenant-123")
    .key("notes")
    .values(null) // free-text input in UI
    .entityType(TagEntityType.ORGANIZATION)
    .build();
```

> **`values` field behavior:** `null` → free-text input · non-empty list → dropdown/checkbox UI · empty list → values explicitly cleared.
