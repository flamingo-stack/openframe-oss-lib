Base MongoDB document entity representing a tenant in the OpenFrame multi-tenant architecture, mapped to the `tenants` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB primary key (`@Id`) |
| `name` | `String` | Organization display name, indexed for query performance |
| `domain` | `String` | Unique subdomain (e.g. `company.openframe.io`), used for subdomain-based routing |

**Annotations:**
- `@Document(collection = "tenants")` — Maps to the MongoDB `tenants` collection
- `@SuperBuilder(toBuilder = true)` — Enables builder pattern with inheritance support via Lombok
- `@Indexed(unique = true)` on `domain` — Enforces uniqueness at the database level

## Usage Example

```java
// Create a new tenant using the builder
CoreTenant tenant = CoreTenant.builder()
    .name("Acme IT")
    .domain("acme.openframe.io")
    .build();

// Copy and modify an existing tenant
CoreTenant updated = tenant.toBuilder()
    .name("Acme IT Services")
    .build();
```

> **Note:** `CoreTenant` is designed as a base class. Use `@SuperBuilder` in subclasses to extend it with additional fields (e.g. billing info, feature flags) while preserving builder inheritance.
