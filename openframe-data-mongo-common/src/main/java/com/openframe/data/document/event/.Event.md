A MongoDB document model representing a tenant-scoped event record, used to store system or user-triggered events with payload data and metadata.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Indexed tenant reference for multi-tenancy scoping |
| `type` | `String` | Event type/category identifier |
| `payload` | `String` | Serialized event data (typically JSON) |
| `timestamp` | `Instant` | UTC timestamp of when the event occurred |
| `userId` | `String` | Reference to the user who triggered the event |

Implements `TenantScoped` to enforce multi-tenant data isolation, with `tenantId` indexed for efficient per-tenant queries.

## Usage Example

```java
// Create a new event using the Lombok builder
Event event = Event.builder()
    .tenantId("tenant-abc")
    .type("TICKET_CREATED")
    .payload("{\"ticketId\": \"T-001\", \"priority\": \"HIGH\"}")
    .timestamp(Instant.now())
    .userId("user-xyz")
    .build();

// Persist via a MongoRepository
eventRepository.save(event);

// Query all events for a tenant
List<Event> tenantEvents = eventRepository.findByTenantId("tenant-abc");
```

> **Note:** The `tenantId` field is indexed (`@Indexed`) to optimize multi-tenant queries. Always populate this field before persisting to ensure proper data isolation across tenants.
