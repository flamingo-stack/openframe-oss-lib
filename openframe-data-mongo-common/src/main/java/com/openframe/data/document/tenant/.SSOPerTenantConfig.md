Extends `SSOConfig` to store per-tenant SSO configuration with audit timestamps managed by Spring Data.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `createdAt` | `LocalDateTime` | Auto-populated by `@CreatedDate` on document creation |
| `updatedAt` | `LocalDateTime` | Auto-updated by `@LastModifiedDate` on each modification |

Inherits from `SSOConfig`: provider settings, client credentials, and the `enabled` flag.

## Usage Example

```java
SSOPerTenantConfig config = new SSOPerTenantConfig();
config.setProvider("okta");
config.setClientId("client-abc");
config.setClientSecret("secret-xyz");
config.setEnabled(true);
// createdAt and updatedAt are managed automatically by Spring Data auditing
tenantDocument.setSsoConfig(config);
mongoTemplate.save(tenantDocument);
```

> Requires Spring Data auditing enabled (`@EnableMongoAuditing`) for `@CreatedDate` and `@LastModifiedDate` to populate automatically.
