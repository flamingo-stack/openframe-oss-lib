Tenant-specific SSO configuration document that extends `SSOConfig` with tenant linkage and audit timestamps for MongoDB persistence.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `tenantId` | `String` | Uniquely indexed tenant identifier (sparse index) |
| `createdAt` | `LocalDateTime` | Auto-populated on document creation via `@CreatedDate` |
| `updatedAt` | `LocalDateTime` | Auto-updated on every save via `@LastModifiedDate` |

Inherits from `SSOConfig`: provider type, client credentials, and enabled flag.

## Usage Example

```java
// Create a new SSO config for a tenant
SSOPerTenantConfig config = new SSOPerTenantConfig();
config.setTenantId("tenant-abc-123");
config.setProvider("OKTA");
config.setClientId("okta-client-id");
config.setClientSecret("okta-client-secret");
config.setEnabled(true);

// Persist via repository — createdAt/updatedAt are managed automatically
ssoPerTenantConfigRepository.save(config);

// Lookup by tenant
Optional<SSOPerTenantConfig> found = ssoPerTenantConfigRepository
    .findByTenantId("tenant-abc-123");
```

> **Note:** The `tenantId` field uses a sparse unique index, meaning only documents where `tenantId` is present are indexed — allowing multiple documents without a `tenantId` to coexist in MongoDB without constraint violations.
