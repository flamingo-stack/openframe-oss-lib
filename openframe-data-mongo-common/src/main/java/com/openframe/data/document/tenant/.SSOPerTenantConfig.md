Extends `SSOConfig` to provide a tenant-scoped SSO configuration document with automatic audit timestamps managed by Spring Data.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `createdAt` | `LocalDateTime` | Auto-populated on document creation via `@CreatedDate` |
| `updatedAt` | `LocalDateTime` | Auto-updated on every save via `@LastModifiedDate` |

Inherits from `SSOConfig`: provider, client credentials, and enabled flag.

## Usage Example

```java
// Build a tenant SSO config (inherits SSOConfig fields)
SSOPerTenantConfig config = new SSOPerTenantConfig();
config.setProvider("okta");
config.setClientId("tenant-client-id");
config.setClientSecret("tenant-client-secret");
config.setEnabled(true);
// createdAt / updatedAt are managed automatically by Spring Data auditing

// Embed within a Tenant document
tenant.setSsoConfig(config);
tenantRepository.save(tenant);
```

> Spring Data auditing must be enabled (e.g. `@EnableMongoAuditing`) for `createdAt` and `updatedAt` to populate automatically.
