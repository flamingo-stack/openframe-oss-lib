Enum defining the categorized URL endpoint types that can be associated with a tool integration in the OpenFrame platform.

## Key Components

| Value | Description |
|---|---|
| `DASHBOARD` | Web UI dashboard endpoint |
| `API` | REST/HTTP API endpoint |
| `WS` | WebSocket endpoint |
| `NATS` | NATS messaging endpoint |
| `DATABASE` | Database connection endpoint |
| `CACHE` | Cache service endpoint |
| `BROKER` | Message broker endpoint |
| `COORDINATOR` | Coordination service endpoint |
| `CONTROLLER` | Controller service endpoint |
| `SERVER` | Generic server endpoint |
| `METRICS` | Metrics/monitoring endpoint |
| `LOGS` | Log aggregation endpoint |
| `INTERNAL` | Internal service endpoint |
| `MINION` | Minion/agent endpoint |

## Usage Example

```java
ToolUrl toolUrl = new ToolUrl();
toolUrl.setType(ToolUrlType.API);
toolUrl.setUrl("https://my-tool.internal/api");

// Filter URLs by type
List<ToolUrl> apiUrls = tool.getUrls().stream()
    .filter(url -> url.getType() == ToolUrlType.API)
    .toList();

// Switch on type
switch (toolUrl.getType()) {
    case DASHBOARD -> openInBrowser(toolUrl.getUrl());
    case API       -> registerApiClient(toolUrl.getUrl());
    case METRICS   -> configureMonitoring(toolUrl.getUrl());
    default        -> log.info("Registered URL: {}", toolUrl.getUrl());
}
```
