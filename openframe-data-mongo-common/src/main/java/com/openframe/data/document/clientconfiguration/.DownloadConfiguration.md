A simple data model representing the configuration required to download a client-side installation package, including OS targeting, file naming, and installation metadata.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `os` | `String` | Target operating system identifier |
| `linkTemplate` | `String` | URL template used to construct the download link |
| `fileName` | `String` | Original name of the file to download |
| `targetFileName` | `String` | Desired name of the file after download |
| `installationType` | `InstallationType` | Installation mode; defaults to `STANDARD` |
| `bundleId` | `String` | Bundle identifier for the package |
| `serviceName` | `String` | Associated service name for the installation |

## Usage Example

```java
DownloadConfiguration config = new DownloadConfiguration();
config.setOs("windows");
config.setLinkTemplate("https://downloads.example.com/{version}/installer.exe");
config.setFileName("installer.exe");
config.setTargetFileName("client-setup.exe");
config.setInstallationType(InstallationType.STANDARD);
config.setBundleId("com.openframe.client");
config.setServiceName("OpenFrameAgent");
```
