MongoDB document entity representing the assignment of a tag to a specific entity within a tenant, stored in the `tag_assignments` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant scope identifier (via `TenantScoped`) |
| `entityId` | `String` | ID of the entity receiving the tag |
| `tagId` | `String` | Reference to the assigned tag |
| `entityType` | `TagEntityType` | Enum classifying the entity (e.g., device, site) |
| `values` | `List<String>` | Per-entity values for key-value tags; empty for label-only tags |
| `taggedAt` | `Instant` | Timestamp of assignment |
| `taggedBy` | `String` | User or system that performed the assignment |

A **unique compound index** (`tenant_entity_tag_idx`) enforces that the combination of `tenantId`, `entityId`, `tagId`, and `entityType` is unique, preventing duplicate tag assignments.

## Usage Example

```java
TagAssignment assignment = TagAssignment.builder()
    .tenantId("tenant-001")
    .entityId("device-abc")
    .tagId("tag-site")
    .entityType(TagEntityType.DEVICE)
    .values(List.of("site1", "site2"))
    .taggedAt(Instant.now())
    .taggedBy("admin-user")
    .build();

tagAssignmentRepository.save(assignment);
```

> For simple label tags (no associated values), set `values` to an empty list or omit it via the builder — the field defaults to an empty `List<String>`.
