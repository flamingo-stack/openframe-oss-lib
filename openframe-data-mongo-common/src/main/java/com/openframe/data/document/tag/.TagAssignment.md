MongoDB document entity representing the assignment of a tag to a specific entity (device, asset, etc.) within a tenant's scope, stored in the `tag_assignments` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant scope identifier (implements `TenantScoped`) |
| `entityId` | `String` | ID of the target entity receiving the tag |
| `tagId` | `String` | Reference to the assigned tag |
| `entityType` | `TagEntityType` | Enum discriminating the entity kind |
| `values` | `List<String>` | Per-entity values for key-based tags (empty for label-only tags) |
| `taggedAt` | `Instant` | Timestamp of assignment |
| `taggedBy` | `String` | User or system actor that created the assignment |

**Index:** A unique compound index (`tenant_entity_tag_idx`) enforces that the combination of `tenantId + entityId + tagId + entityType` is unique, preventing duplicate tag assignments per entity per tenant.

## Usage Example

```java
TagAssignment assignment = TagAssignment.builder()
    .tenantId("tenant-001")
    .entityId("device-abc")
    .tagId("tag-site")
    .entityType(TagEntityType.DEVICE)
    .values(List.of("site1", "site2"))
    .taggedAt(Instant.now())
    .taggedBy("admin@example.com")
    .build();

tagAssignmentRepository.save(assignment);
```

> **Note:** `values` should be an empty list for simple label tags (e.g., `"production"`, `"critical"`) and populated only when the tag represents a key-value pair (e.g., `site → ["hq", "branch"]`).
