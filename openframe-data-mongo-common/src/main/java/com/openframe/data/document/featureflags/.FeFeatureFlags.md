MongoDB document entity for storing per-cluster frontend feature flag overrides. Maps cluster IDs to boolean flag values, with precedence over YAML-configured defaults in `openframe.fe-feature-flag.*`.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `clusterId` | `String` | MongoDB `_id` — one document per cluster (`openframe.cluster-id`) |
| `flags` | `Map<String, Boolean>` | Named feature flag overrides; missing keys fall back to YAML defaults |
| `createdAt` | `LocalDateTime` | Auto-populated on insert via `@CreatedDate` |
| `updatedAt` | `LocalDateTime` | Auto-updated on save via `@LastModifiedDate` |

- **Collection:** `fe_feature_flags`
- **Annotations:** Lombok `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`

## Usage Example

```java
// Create a new feature flags document for a cluster
FeFeatureFlags featureFlags = FeFeatureFlags.builder()
    .clusterId("cluster-prod-01")
    .flags(new LinkedHashMap<>(Map.of(
        "newDashboardEnabled", true,
        "betaReportsEnabled",  false
    )))
    .build();

// Access a specific flag override
Boolean isDashboardEnabled = featureFlags.getFlags().get("newDashboardEnabled");

// Add or update a flag at runtime
featureFlags.getFlags().put("experimentalSearch", true);
```

> **Override precedence:** Any flag key present in `flags` takes priority over `openframe.fe-feature-flag.*` YAML configuration. Flags absent from the map fall back to the YAML default, enabling partial overrides without redeployment.
