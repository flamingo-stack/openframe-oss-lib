Defines the publication lifecycle states for Knowledge Base articles within the OpenFrame platform, enforcing a soft-delete pattern where articles progress through draft, published, and archived states rather than being permanently removed.

## Key Components

| Value | Description |
|-------|-------------|
| `DRAFT` | Article is being authored and is not yet visible to end users |
| `PUBLISHED` | Article is live and accessible to clients/technicians |
| `ARCHIVED` | Soft-delete terminal state — article is hidden but retained in the data store |

## Usage Example

```java
KnowledgeBaseArticle article = new KnowledgeBaseArticle();

// Initial creation
article.setStatus(KnowledgeBaseArticleStatus.DRAFT);

// Make article visible
article.setStatus(KnowledgeBaseArticleStatus.PUBLISHED);

// Soft-delete (never hard-delete articles)
article.setStatus(KnowledgeBaseArticleStatus.ARCHIVED);

// Lifecycle check
if (article.getStatus() == KnowledgeBaseArticleStatus.ARCHIVED) {
    // Exclude from search results
}
```

## Notes

- **Folders are excluded** — this enum applies only to articles; KB folders use hard-delete.
- **No hard-delete** — `ARCHIVED` is the permanent terminal state, preserving article history and audit trails.
- Filtering queries should exclude `ARCHIVED` articles from client-facing surfaces (Fae) and default technician views (Mingo AI).
