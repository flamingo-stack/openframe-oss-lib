A generic, flexible implementation of `NotificationContext` that carries arbitrary notification data using a type identifier and a string-encoded payload.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `type` | `String` | Identifies the kind of notification or payload format |
| `payload` | `String` | The raw notification content (e.g., JSON, plain text) |

Inherits all fields and behavior from `NotificationContext` via `@EqualsAndHashCode(callSuper = true)` and `@ToString(callSuper = true)`.

## Usage Example

```java
// Builder pattern (recommended)
GenericContext context = GenericContext.builder()
    .type("ALERT")
    .payload("{\"message\": \"Disk usage above 90%\"}")
    .build();

// All-args constructor
GenericContext context = new GenericContext("ALERT", "{\"message\": \"Disk usage above 90%\"}");

// No-args constructor with setters
GenericContext context = new GenericContext();
context.setType("TICKET_UPDATE");
context.setPayload("{\"ticketId\": \"TKT-001\", \"status\": \"resolved\"}");
```

> **Note:** `payload` is intentionally untyped (`String`) to support any serialization format. Callers are responsible for serializing and deserializing the payload content.
