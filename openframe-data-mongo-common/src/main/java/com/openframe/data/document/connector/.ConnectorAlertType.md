Enumeration defining the types of alerts that can be triggered by a connector when critical failure conditions occur during execution.

## Key Components

| Constant | Description |
|---|---|
| `MAX_RETRIES_EXCEEDED` | Triggered when a connector exhausts its maximum number of retry attempts |
| `NON_RECOVERABLE_ERROR` | Triggered when a connector encounters a fatal error that cannot be retried or recovered from |

## Usage Example

```java
ConnectorAlertType alertType = ConnectorAlertType.MAX_RETRIES_EXCEEDED;

switch (alertType) {
    case MAX_RETRIES_EXCEEDED:
        notifyAdmin("Connector failed after maximum retries.");
        break;
    case NON_RECOVERABLE_ERROR:
        notifyAdmin("Connector encountered a fatal error.");
        break;
}
```
