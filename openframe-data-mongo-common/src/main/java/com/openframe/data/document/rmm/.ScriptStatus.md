Enum defining the three lifecycle states for a `Script` document in the RMM (Remote Monitoring & Management) module, supporting soft-delete semantics.

## Key Components

| Value | Description |
|-------|-------------|
| `ACTIVE` | Script is live and visible through standard API surfaces |
| `ARCHIVED` | Script is inactive but retained for reference |
| `DELETED` | Soft-deleted; hidden from default API surfaces but preserved in MongoDB for historic execution record integrity |

## Usage Example

```java
// Check if a script is available for execution
if (script.getStatus() == ScriptStatus.ACTIVE) {
    executor.run(script);
}

// Soft-delete a script
script.setStatus(ScriptStatus.DELETED);
mongoRepository.save(script); // Document remains; name stays occupied on (tenantId, name) index

// Filter scripts in repository queries
List<Script> visible = scriptRepository
    .findByTenantIdAndStatusNot(tenantId, ScriptStatus.DELETED);
```

## Important Notes

- **Soft-delete caveat:** `DELETED` documents still occupy the compound unique index on `(tenantId, name)`. A deleted script's name **cannot be reused** without a hard-delete via admin tooling.
- Mirrors the `OrganizationStatus` pattern used across the OpenFrame data model for consistency.
- Historic execution records safely reference `DELETED` scripts since the document is never physically removed from MongoDB.
