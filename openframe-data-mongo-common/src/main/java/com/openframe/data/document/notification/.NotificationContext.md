Abstract base class for notification context objects, enabling polymorphic JSON deserialization based on a `type` discriminator field.

## Key Components

- **`NotificationContext`** — Abstract base class that all notification context types must extend
- **`getType()`** — Abstract method subclasses must implement to return the context's type identifier
- **`@JsonTypeInfo`** — Configures Jackson to deserialize concrete subtypes using the `type` property as a discriminator, falling back to `GenericContext` when no matching type is found
- **`@SuperBuilder`** — Enables Lombok builder inheritance across subclass hierarchies
- **`@NoArgsConstructor`** — Generates a no-arg constructor required for Jackson deserialization

## Usage Example

```java
// Define a concrete subclass
@JsonTypeName("ticket_update")
@SuperBuilder
@NoArgsConstructor
public class TicketUpdateContext extends NotificationContext {

    private String ticketId;
    private String status;

    @Override
    public String getType() {
        return "ticket_update";
    }
}

// Jackson deserializes the correct subtype automatically
String json = """
    {
        "type": "ticket_update",
        "ticketId": "TKT-123",
        "status": "resolved"
    }
    """;

NotificationContext ctx = objectMapper.readValue(json, NotificationContext.class);
// ctx is an instance of TicketUpdateContext
```

> Unknown or unregistered `type` values fall back to `GenericContext`, preventing deserialization failures for forward-compatibility.
