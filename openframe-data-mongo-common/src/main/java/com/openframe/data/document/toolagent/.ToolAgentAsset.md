A data model representing a downloadable or deployable asset associated with a tool agent, including its source, versioning, download configurations, and file system placement details.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier for the asset |
| `version` | `String` | Version string of the asset |
| `downloadConfigurations` | `List<DownloadConfiguration>` | List of download configurations defining how the asset is retrieved |
| `source` | `ToolAgentAssetSource` | Enum or object describing the asset's origin source |
| `localFilenameConfiguration` | `List<LocalFilenameConfiguration>` | Rules for naming the asset on the local filesystem |
| `path` | `String` | Target file system path where the asset will be placed |
| `executable` | `boolean` | Whether the asset should be marked as executable after download |

## Usage Example

```java
ToolAgentAsset asset = new ToolAgentAsset();
asset.setId("nmap-tool");
asset.setVersion("7.94");
asset.setSource(ToolAgentAssetSource.GITHUB_RELEASE);
asset.setPath("/usr/local/bin/nmap");
asset.setExecutable(true);

DownloadConfiguration downloadConfig = new DownloadConfiguration();
// configure download URL, headers, etc.
asset.setDownloadConfigurations(List.of(downloadConfig));

LocalFilenameConfiguration filenameConfig = new LocalFilenameConfiguration();
// configure local filename rules
asset.setLocalFilenameConfiguration(List.of(filenameConfig));
```

> Lombok `@Getter` and `@Setter` annotations auto-generate all accessors, keeping the model clean and boilerplate-free. This class is typically deserialized from a tool agent document stored in the OpenFrame data layer.
