MongoDB document entity that tracks the read/unread state of notifications for individual recipients, stored in the `notification_read_states` collection.

## Key Components

| Element | Description |
|---|---|
| `@Document` | Maps to the `notification_read_states` MongoDB collection |
| `recipientId` | Identifier of the notification recipient |
| `recipientType` | Enum (`RecipientType`) indicating recipient kind (e.g., user, client) |
| `notificationId` | Reference to the associated notification document |
| `status` | Enum (`ReadStatus`) representing read or unread state |
| `category` | Enum (`NotificationCategory`) for grouping notifications by type |
| `title` | Notification title snapshot stored on the read state record |
| `readAt` | Timestamp (`Instant`) recording when the notification was read |

### Compound Indexes

| Index Name | Fields | Purpose |
|---|---|---|
| `recipient_notification_unique` | `recipientId`, `recipientType`, `notificationId` | Enforces one read-state record per recipient per notification |
| `recipient_status` | `recipientId`, `recipientType`, `status` | Optimises queries for all unread/read notifications of a recipient |
| `recipient_category_status` | `recipientId`, `recipientType`, `category`, `status` | Optimises filtered queries by category and status |

## Usage Example

```java
// Build a read state when a recipient opens a notification
NotificationReadState readState = NotificationReadState.builder()
        .recipientId("user-123")
        .recipientType(RecipientType.USER)
        .notificationId("notif-456")
        .status(ReadStatus.READ)
        .category(NotificationCategory.ALERT)
        .title("Ticket #42 Updated")
        .readAt(Instant.now())
        .build();

notificationReadStateRepository.save(readState);
```
