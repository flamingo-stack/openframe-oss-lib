Defines the contract for describing a notification context type, linking a string type identifier to its corresponding `NotificationContext` implementation class.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `type()` | `String` | Unique string identifier for the notification context type |
| `contextClass()` | `Class<? extends NotificationContext>` | The concrete `NotificationContext` class associated with this descriptor |
| `graphqlTypeName()` | `default String` | Returns the simple class name for GraphQL type resolution; defaults to `contextClass().getSimpleName()` |
| `category()` | `default NotificationCategory` | Classifies the notification; defaults to `NotificationCategory.GENERIC` |

## Usage Example

```java
public class TicketNotificationContextDescriptor implements NotificationContextDescriptor {

    @Override
    public String type() {
        return "TICKET_UPDATE";
    }

    @Override
    public Class<? extends NotificationContext> contextClass() {
        return TicketNotificationContext.class;
    }

    @Override
    public NotificationCategory category() {
        return NotificationCategory.TICKET;
    }

    // graphqlTypeName() defaults to "TicketNotificationContext"
}
```

Implementations are registered so the notification system can dynamically resolve the correct context class and GraphQL type from a raw type string, enabling extensible, type-safe notification handling across the OpenFrame platform.
