A data model representing a single compliance requirement check result for a device, capturing the compliance standard, control identifier, pass/fail status, and last evaluation timestamp.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `standard` | `String` | Compliance framework name (e.g., `"NIST"`, `"ISO27001"`) |
| `control` | `String` | Specific control identifier within the standard |
| `compliant` | `boolean` | Whether the device satisfies the control |
| `details` | `String` | Human-readable explanation of the compliance result |
| `lastCheck` | `Instant` | UTC timestamp of the most recent evaluation |

Uses Lombok `@Data` to auto-generate getters, setters, `equals`, `hashCode`, and `toString`.

## Usage Example

```java
ComplianceRequirement req = new ComplianceRequirement();
req.setStandard("NIST");
req.setControl("AC-2");
req.setCompliant(false);
req.setDetails("Account management policy not enforced — local admin accounts detected.");
req.setLastCheck(Instant.now());

if (!req.isCompliant()) {
    System.out.println("Control " + req.getControl()
        + " failed under " + req.getStandard()
        + ": " + req.getDetails());
}
```

## Notes

- Intended to be embedded as a list within a parent device document (e.g., `List<ComplianceRequirement>`) stored in a NoSQL/document store.
- `lastCheck` uses `Instant` for timezone-neutral UTC storage — convert to `ZonedDateTime` when displaying to end users.
- `standard` values should be normalized (e.g., always uppercase) to ensure consistent querying across the Flamingo/OpenFrame platform.
