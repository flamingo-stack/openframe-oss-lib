Enum representing the possible lifecycle states of a support ticket, distinguishing between system-managed statuses and custom user-defined ones.

## Key Components

| Member | Description |
|--------|-------------|
| `AI_ASSISTANCE` | Ticket is currently being handled by AI automation |
| `TECH_REQUIRED` | Ticket requires human technician intervention |
| `RESOLVED` | Ticket has been successfully resolved |
| `ARCHIVED` | Ticket has been archived |
| `CUSTOM` | User-defined status outside the standard workflow |
| `isSystem()` | Returns `true` for all statuses except `CUSTOM` |

## Usage Example

```java
TicketStatusKind status = ticket.getStatusKind();

if (status.isSystem()) {
    // Handle AI_ASSISTANCE, TECH_REQUIRED, RESOLVED, or ARCHIVED
    applySystemWorkflow(status);
} else {
    // Handle custom org-defined status
    applyCustomWorkflow(ticket);
}

// Direct comparison
if (status == TicketStatusKind.TECH_REQUIRED) {
    notifyOnCallTechnician(ticket);
}
```

> **Note:** Only `CUSTOM` returns `false` from `isSystem()`, making it straightforward to branch logic between platform-managed and organization-defined ticket states.
