Enum representing the possible status categories for a ticket, distinguishing between system-managed statuses and custom user-defined ones.

## Key Components

| Member | Description |
|--------|-------------|
| `AI_ASSISTANCE` | Ticket is being handled by AI automation (Mingo AI / Fae) |
| `TECH_REQUIRED` | Ticket requires human technician intervention |
| `RESOLVED` | Ticket has been closed/resolved |
| `ARCHIVED` | Ticket has been archived |
| `CUSTOM` | User-defined status outside the standard system statuses |
| `isSystem()` | Returns `true` for all values except `CUSTOM` |

## Usage Example

```java
TicketStatusKind status = ticket.getStatusKind();

if (status.isSystem()) {
    // Handle built-in lifecycle status (AI_ASSISTANCE, TECH_REQUIRED, RESOLVED, ARCHIVED)
    applySystemWorkflow(status);
} else {
    // Handle custom MSP-defined status
    applyCustomWorkflow(ticket);
}

// Direct comparison
if (status == TicketStatusKind.TECH_REQUIRED) {
    notifyOnCallTechnician(ticket);
}
```

> `isSystem()` is the canonical way to distinguish platform-managed statuses from custom ones — avoid hardcoding checks against individual enum values where possible.
