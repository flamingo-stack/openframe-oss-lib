MongoDB document entity representing a tenant-scoped integrated tool agent configuration, storing deployment metadata, command arguments, and lifecycle state for MSP tool agents within the OpenFrame platform.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant isolation field (implements `TenantScoped`) |
| `key` | `String` | Unique agent key per tenant (compound index with `tenantId`) |
| `documentVersion` | `Long` | Optimistic locking version (`@Version`) |
| `toolId` | `String` | Reference to the associated tool |
| `releaseVersion` | `boolean` | Flags whether this is a release-grade version |
| `sessionType` | `SessionType` | Defines agent session behavior |
| `downloadConfigurations` | `List<DownloadConfiguration>` | Asset download configurations |
| `assets` | `List<ToolAgentAsset>` | Agent binary/file assets |
| `installationCommandArgs` | `List<String>` | CLI args for installation |
| `runCommandArgs` | `List<String>` | CLI args for agent execution |
| `agentToolIdCommandArgs` | `List<String>` | CLI args for tool ID resolution |
| `uninstallationCommandArgs` | `List<String>` | CLI args for removal |
| `status` | `ToolAgentStatus` | Current operational status |
| `publishState` | `PublishState` | Publication lifecycle state |

## Usage Example

```java
IntegratedToolAgent agent = new IntegratedToolAgent();
agent.setTenantId("tenant-123");
agent.setKey("connectwise-rmm-agent");
agent.setToolId("tool-456");
agent.setVersion("2.1.0");
agent.setReleaseVersion(true);
agent.setSessionType(SessionType.PERSISTENT);
agent.setInstallationCommandArgs(List.of("--silent", "--accept-eula"));
agent.setRunCommandArgs(List.of("--mode", "service"));
agent.setUninstallationCommandArgs(List.of("--silent", "--remove-all"));
agent.setStatus(ToolAgentStatus.ACTIVE);
agent.setPublishState(PublishState.PUBLISHED);
```

> The compound index `(tenantId, key)` enforces uniqueness per tenant, ensuring no two agents share the same key within a tenant scope.
