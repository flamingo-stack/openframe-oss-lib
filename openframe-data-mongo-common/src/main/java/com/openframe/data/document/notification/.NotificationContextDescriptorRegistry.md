A Spring `@Component` that acts as a registry for `NotificationContextDescriptor` instances, providing lookup of notification categories by type string.

## Key Components

| Element | Description |
|---|---|
| `byType` | Immutable map of notification type strings to their descriptors, built at construction time |
| `NotificationContextDescriptorRegistry(List<...>)` | Constructor that auto-wires all `NotificationContextDescriptor` beans and indexes them by their `type()` value |
| `categoryOf(String type)` | Returns the `NotificationCategory` for a given type, falling back to `NotificationCategory.GENERIC` if the type is unregistered |

## Usage Example

```java
@Service
public class NotificationService {

    private final NotificationContextDescriptorRegistry registry;

    public NotificationService(NotificationContextDescriptorRegistry registry) {
        this.registry = registry;
    }

    public void handleNotification(String type, Object payload) {
        NotificationCategory category = registry.categoryOf(type);

        switch (category) {
            case ALERT   -> processAlert(payload);
            case GENERIC -> processGeneric(payload);
            // ...
        }
    }
}
```

Any `NotificationContextDescriptor` bean registered in the Spring context is automatically picked up — no manual wiring needed. Unknown or unregistered type strings gracefully return `NotificationCategory.GENERIC` instead of throwing.
