A filter data transfer object used to query tickets with flexible, multi-criteria filtering. Supports both legacy status-based filtering and the newer lifecycle-aware status system via a feature flag transition pattern.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `statuses` | `List<TicketStatus>` | **Legacy** — enum-based status filter (pre-lifecycle rollout) |
| `statusIds` | `List<String>` | **Lifecycle** — filter by specific status IDs |
| `statusKinds` | `List<TicketStatusKind>` | **Lifecycle** — filter by status kind categories |
| `organizationIds` | `List<String>` | Scope results to specific organizations |
| `assigneeIds` | `List<String>` | Filter by assigned technician(s) |
| `labelIds` | `List<String>` | Filter by ticket labels |
| `deviceIds` | `List<String>` | Filter by associated device(s) |
| `creationSources` | `List<TicketCreationSource>` | Filter by how the ticket was created |
| `createdAtFrom` / `createdAtTo` | `Instant` | Date range filter for ticket creation time |

> **Note:** `statuses` is marked for removal after the lifecycle feature flag rollout completes. New code should prefer `statusIds` and `statusKinds`.

## Usage Example

```java
// Legacy usage (lifecycle flag OFF)
TicketQueryFilter legacyFilter = TicketQueryFilter.builder()
    .statuses(List.of(TicketStatus.OPEN, TicketStatus.IN_PROGRESS))
    .organizationIds(List.of("org-123"))
    .build();

// Lifecycle usage (lifecycle flag ON)
TicketQueryFilter lifecycleFilter = TicketQueryFilter.builder()
    .statusKinds(List.of(TicketStatusKind.OPEN))
    .assigneeIds(List.of("tech-456"))
    .createdAtFrom(Instant.parse("2024-01-01T00:00:00Z"))
    .createdAtTo(Instant.now())
    .creationSources(List.of(TicketCreationSource.EMAIL))
    .build();
```
