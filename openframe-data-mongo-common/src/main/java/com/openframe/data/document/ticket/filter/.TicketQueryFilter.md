A filter object used to construct scoped ticket queries, supporting both legacy status-based filtering and the newer lifecycle-aware status model.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `statuses` | `List<TicketStatus>` | **Legacy** — filter by enum statuses (pre-lifecycle rollout) |
| `statusIds` | `List<String>` | **Lifecycle** — filter by status document IDs |
| `statusKinds` | `List<TicketStatusKind>` | **Lifecycle** — filter by status kind categories |
| `organizationIds` | `List<String>` | Scope tickets to specific organizations |
| `assigneeIds` | `List<String>` | Filter by assigned technician IDs |
| `labelIds` | `List<String>` | Filter by label IDs |
| `deviceIds` | `List<String>` | Filter by associated device IDs |
| `creationSources` | `List<TicketCreationSource>` | Filter by how the ticket was created |
| `createdAtFrom` / `createdAtTo` | `Instant` | Date range filter on creation timestamp |

> **Note:** `statuses` is marked for removal after the lifecycle feature flag rollout is complete. New code should prefer `statusIds` and `statusKinds`.

## Usage Example

```java
// Legacy usage (lifecycle flag OFF)
TicketQueryFilter legacyFilter = TicketQueryFilter.builder()
    .statuses(List.of(TicketStatus.OPEN, TicketStatus.IN_PROGRESS))
    .organizationIds(List.of("org-123"))
    .build();

// Lifecycle usage (lifecycle flag ON)
TicketQueryFilter lifecycleFilter = TicketQueryFilter.builder()
    .statusKinds(List.of(TicketStatusKind.ACTIVE))
    .assigneeIds(List.of("tech-456"))
    .createdAtFrom(Instant.parse("2024-01-01T00:00:00Z"))
    .createdAtTo(Instant.now())
    .build();
```
