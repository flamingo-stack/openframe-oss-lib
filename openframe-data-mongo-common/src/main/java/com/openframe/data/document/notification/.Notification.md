MongoDB document entity representing a system notification, storing metadata such as severity, title, description, and contextual information for the OpenFrame notification system.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `severity` | `NotificationSeverity` | Notification level, defaults to `INFO` |
| `title` | `String` | Short notification heading |
| `description` | `String` | Full notification message body |
| `createdAt` | `Instant` | Auto-populated timestamp via `@CreatedDate` |
| `context` | `NotificationContext` | Additional contextual metadata for the notification |

## Usage Example

```java
// Build a warning notification with context
Notification notification = Notification.builder()
    .severity(NotificationSeverity.WARNING)
    .title("High CPU Usage Detected")
    .description("Server node-01 exceeded 90% CPU threshold for 5 minutes.")
    .context(NotificationContext.builder()
        .resourceId("node-01")
        .resourceType("SERVER")
        .build())
    .build();

// Save to MongoDB
notificationRepository.save(notification);

// Build a default INFO notification (severity defaults to INFO)
Notification info = Notification.builder()
    .title("Backup Completed")
    .description("Nightly backup finished successfully.")
    .build();
```

## Notes

- Persisted to the `notifications` MongoDB collection via `@Document`
- Uses Lombok `@Builder`, `@Data`, `@NoArgsConstructor`, and `@AllArgsConstructor` to reduce boilerplate
- `createdAt` is automatically set by Spring Data MongoDB's auditing — ensure `@EnableMongoAuditing` is active in your configuration
- Default severity (`INFO`) is enforced via `@Builder.Default` to prevent null values on builder construction
