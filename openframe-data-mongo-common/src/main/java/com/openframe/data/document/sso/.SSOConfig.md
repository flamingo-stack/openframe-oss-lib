MongoDB document entity representing a tenant-scoped SSO (Single Sign-On) configuration, storing provider credentials and provisioning rules for authentication integrations.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier |
| `tenantId` | `String` | Indexed tenant scope reference |
| `provider` | `String` | SSO provider name (e.g., `microsoft`, `google`) |
| `clientId` / `clientSecret` | `String` | OAuth2/OIDC application credentials |
| `enabled` | `boolean` | Toggles the SSO configuration on/off |
| `autoProvisionUsers` | `boolean` | Auto-creates users on first SSO login if `true` |
| `msTenantId` | `String` | Microsoft Entra tenant ID for Office 365 OIDC |
| `allowedDomains` | `List<String>` | Email domain whitelist for auto-provisioning; unrestricted if empty |

**Implements:** `TenantScoped` — enforces multi-tenant data isolation via `tenantId`.

**Collection:** `sso_configs`

## Usage Example

```java
// Create an Office 365 SSO config with domain-restricted auto-provisioning
SSOConfig config = new SSOConfig();
config.setTenantId("tenant-abc");
config.setProvider("microsoft");
config.setClientId("your-client-id");
config.setClientSecret("your-client-secret");
config.setMsTenantId("your-ms-tenant-id");
config.setEnabled(true);
config.setAutoProvisionUsers(true);
config.setAllowedDomains(List.of("acme.com", "acme.org"));

// Save via repository
ssoConfigRepository.save(config);
```

> **Security note:** `clientSecret` is stored as a raw string — ensure field-level encryption or a secrets manager is applied at the infrastructure level before persisting SSO credentials.
