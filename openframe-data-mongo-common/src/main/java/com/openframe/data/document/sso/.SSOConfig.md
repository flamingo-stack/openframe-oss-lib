MongoDB document model for storing Single Sign-On (SSO) provider configurations, mapped to the `sso_configs` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document ID (`@Id`) |
| `provider` | `String` | SSO provider identifier (e.g., `google`, `microsoft`) |
| `clientId` | `String` | OAuth2/OIDC client ID |
| `clientSecret` | `String` | OAuth2/OIDC client secret |
| `enabled` | `boolean` | Toggles this SSO configuration on/off |
| `autoProvisionUsers` | `boolean` | Whether new users are auto-created on first SSO login |
| `msTenantId` | `String` | Microsoft Entra tenant ID for Office 365 OIDC (optional) |
| `allowedDomains` | `List<String>` | Email domain whitelist for auto-provisioning; allows all domains if empty or null |

## Usage Example

```java
// Create and persist a Microsoft SSO configuration
SSOConfig config = new SSOConfig();
config.setProvider("microsoft");
config.setClientId("your-client-id");
config.setClientSecret("your-client-secret");
config.setEnabled(true);
config.setAutoProvisionUsers(true);
config.setMsTenantId("your-tenant-id");
config.setAllowedDomains(List.of("flamingo.run", "openframe.ai"));

mongoTemplate.save(config); // persists to "sso_configs" collection
```

## Notes

- `autoProvisionUsers: false` restricts SSO login to pre-existing users only — useful for tightly controlled environments.
- `allowedDomains` acts as a guard for auto-provisioning; leave empty to allow any verified email domain.
- `msTenantId` is optional — omit to use Microsoft's common (`/common`) endpoint, which supports multi-tenant scenarios.
- Uses Lombok `@Data` for automatic generation of getters, setters, `equals`, `hashCode`, and `toString`.
