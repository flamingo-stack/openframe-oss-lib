MongoDB document entity representing a tenant-scoped SSO (Single Sign-On) configuration, storing provider credentials and provisioning rules for a given tenant.

## Key Components

| Field | Type | Description |
|---|---|---|
| `id` | `String` | MongoDB document ID (`@Id`) |
| `tenantId` | `String` | Indexed tenant identifier (implements `TenantScoped`) |
| `provider` | `String` | SSO provider name (e.g., `microsoft`, `google`) |
| `clientId` / `clientSecret` | `String` | OAuth2/OIDC application credentials |
| `enabled` | `boolean` | Toggles SSO provider for the tenant |
| `autoProvisionUsers` | `boolean` | Allows auto-creation of new users on first SSO login |
| `msTenantId` | `String` | Microsoft Entra tenant ID for Office 365 OIDC |
| `allowedDomains` | `List<String>` | Email domain whitelist for auto-provisioning; unrestricted if empty/null |

## Usage Example

```java
SSOConfig config = new SSOConfig();
config.setTenantId("tenant-abc");
config.setProvider("microsoft");
config.setClientId("app-client-id");
config.setClientSecret("app-client-secret");
config.setMsTenantId("ms-entra-tenant-id");
config.setEnabled(true);
config.setAutoProvisionUsers(true);
config.setAllowedDomains(List.of("acmecorp.com", "acmecorp.io"));

ssoConfigRepository.save(config);
```

> **Note:** When `autoProvisionUsers` is `false`, only users already present in the system can authenticate via the configured provider. The `allowedDomains` restriction only applies when auto-provisioning is enabled.
