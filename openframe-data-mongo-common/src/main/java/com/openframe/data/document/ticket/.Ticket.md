MongoDB document entity representing a support ticket in the OpenFrame PSA/ticketing system. It stores all ticket metadata including status lifecycle, ownership, device/organization associations, and timestamps.

## Key Components

| Element | Type | Description |
|---|---|---|
| `Ticket` | `@Document` | Root entity mapped to the `tickets` MongoDB collection |
| `tenantId` / `ticketNumber` | Fields | Compound unique index ensures ticket numbers are scoped per tenant |
| `status` | `TicketStatus` | **Legacy** — being replaced by `statusId` + `statusKind` |
| `statusKind` | `TicketStatusKind` | New lifecycle-aware status classification |
| `statusId` | `String` | Reference to a named status definition |
| `owner` | `TicketOwner` | Embedded ownership metadata |
| `creationSource` | `TicketCreationSource` | Origin of the ticket (e.g., manual, AI, client portal) |
| `isAiDisabled()` | Method | Returns `true` when the ticket status does not permit AI assistance |
| `TenantScoped` | Interface | Enforces multi-tenant data isolation |

> **Migration note:** The legacy `status` field and its compound indexes (`status_order`, `status_kind`) are pending removal once all reads/writes migrate to `statusKind`/`statusId`.

## Usage Example

```java
Ticket ticket = Ticket.builder()
    .tenantId("tenant-abc")
    .ticketNumber(1042)
    .title("Printer offline - Floor 3")
    .description("HP LaserJet not responding since 09:00.")
    .statusKind(TicketStatusKind.AI_ASSISTANCE)
    .statusId("status-open-id")
    .organizationId("org-xyz")
    .organizationName("Acme Corp")
    .assignedTo("tech-user-id")
    .assignedName("Jane Smith")
    .creationSource(TicketCreationSource.CLIENT_PORTAL)
    .build();

// Check if Mingo AI can assist on this ticket
if (!ticket.isAiDisabled()) {
    mingoService.triggerAssistance(ticket);
}
```
