MongoDB document entity representing a support ticket in the OpenFrame PSA/ticketing system. It stores all ticket metadata including ownership, status lifecycle, assignment, and device/organization associations.

## Key Components

| Field / Method | Type | Description |
|---|---|---|
| `id` | `String` | MongoDB document ID |
| `tenantId` | `String` | Tenant scope (implements `TenantScoped`) |
| `ticketNumber` | `Integer` | Human-readable ticket number (unique per tenant) |
| `status` | `TicketStatus` | **Legacy** — being replaced by `statusKind`/`statusId` |
| `statusId` | `String` | New lifecycle status reference |
| `statusKind` | `TicketStatusKind` | Enum-based status category for lifecycle routing |
| `creationSource` | `TicketCreationSource` | How the ticket was created (e.g., AI, manual) |
| `owner` | `TicketOwner` | Ownership metadata |
| `deviceId` / `organizationId` / `assignedTo` | `String` | Indexed foreign keys for query performance |
| `isAiDisabled()` | `boolean` | Returns `true` if the ticket's status excludes AI assistance |

**Compound Indexes:**
- `tenant_ticketNumber_idx` — unique constraint per tenant
- `status_id_order` — supports paginated queries by lifecycle status
- `status_order` — **legacy**, scheduled for removal post lifecycle rollout

## Usage Example

```java
Ticket ticket = Ticket.builder()
    .tenantId("tenant-abc")
    .ticketNumber(1042)
    .title("Workstation not booting")
    .description("User reports black screen on startup.")
    .statusKind(TicketStatusKind.AI_ASSISTANCE)
    .statusId("status-open-id")
    .creationSource(TicketCreationSource.PORTAL)
    .organizationId("org-xyz")
    .assignedTo("tech-user-id")
    .build();

// Check if AI workflows are suppressed for this ticket
if (ticket.isAiDisabled()) {
    // skip Mingo AI processing
}
```

> **Migration note:** The `status` field and `status_order` index are deprecated. All new reads and writes should use `statusKind` and `statusId`. See `TODO(lifecycle-rollout)` comments for removal tracking.
