MongoDB document entity representing a PSA/ticketing system ticket, storing all ticket metadata including status, ownership, device association, and organizational context.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document ID (`@Id`) |
| `ticketNumber` | `Integer` | Auto-incremented, tenant-unique human-readable number |
| `title` / `description` | `String` | Ticket summary and rich-text HTML body |
| `status` | `TicketStatus` | Indexed ticket lifecycle state |
| `creationSource` | `TicketCreationSource` | Origin of ticket (e.g., manual, AI, email) |
| `owner` | `TicketOwner` | Embedded ownership metadata |
| `deviceId` / `deviceHostname` | `String` | Associated machine |
| `organizationId` / `organizationName` | `String` | Client organization context |
| `assignedTo` / `assignedName` | `String` | Assigned technician reference |
| `order` | `String` | Indexed sort/ordering field |
| `createdAt` / `updatedAt` / `resolvedAt` | `Instant` | Lifecycle timestamps |

**Compound Indexes:**

```text
status_created       → { status: 1, createdAt: -1 }
assignee_status      → { assignedTo: 1, status: 1 }
organization_status  → { organizationId: 1, status: 1 }
device_status        → { deviceId: 1, status: 1 }
```

> Tags are managed via a shared `TagAssignment` collection. Attachments live in the `TicketAttachment` collection — not embedded here.

## Usage Example

```java
Ticket ticket = Ticket.builder()
    .ticketNumber(1042)
    .title("VPN not connecting after Windows update")
    .description("<p>User reports VPN fails post-KB5034441 patch.</p>")
    .status(TicketStatus.OPEN)
    .creationSource(TicketCreationSource.MANUAL)
    .deviceId("dev-abc123")
    .deviceHostname("ws-john-doe")
    .organizationId("org-xyz")
    .organizationName("Acme Corp")
    .assignedTo("user-789")
    .assignedName("Jane Smith")
    .build();

ticketRepository.save(ticket);
```
