Defines a MongoDB document model representing a tenant-scoped ticket status definition, stored in the `ticket_statuses` collection.

## Key Components

- **`TicketStatusDefinition`** — Main document entity implementing `TenantScoped`, mapping ticket status records per tenant
- **Fields:**
  - `id` — MongoDB document identifier
  - `tenantId` — Tenant isolation key (from `TenantScoped`)
  - `kind` — Enum (`TicketStatusKind`) classifying the status (e.g., `AI_ASSISTANCE`, `TECH_REQUIRED`, `RESOLVED`, `ARCHIVED`)
  - `name` / `color` / `position` — Display metadata for UI rendering
  - `createdAt` / `updatedAt` — Auto-managed audit timestamps
- **Compound Index (`tenant_kind_unique`)** — Enforces uniqueness on `(tenantId, kind)` pairs, but only for the four built-in system kinds, allowing unlimited custom statuses per tenant

## Usage Example

```java
TicketStatusDefinition status = TicketStatusDefinition.builder()
    .tenantId("tenant-abc")
    .kind(TicketStatusKind.RESOLVED)
    .name("Resolved")
    .color("#4ECDC4")
    .position("4")
    .build();

mongoTemplate.save(status);
```

> The partial filter on the compound index means only system-defined status kinds (`AI_ASSISTANCE`, `TECH_REQUIRED`, `RESOLVED`, `ARCHIVED`) are constrained to one per tenant — custom statuses bypass the uniqueness constraint.
