A simple filter model used to query events by user, type, and date range within the OpenFrame data layer.

## Key Components

| Field | Type | Description |
|---|---|---|
| `userIds` | `List<String>` | Filter events by one or more user IDs |
| `eventTypes` | `List<String>` | Filter events by one or more event type identifiers |
| `startDate` | `LocalDate` | Inclusive start date for the query range |
| `endDate` | `LocalDate` | Inclusive end date for the query range |

## Usage Example

```java
// Build a filter for specific users and event types within a date range
EventQueryFilter filter = EventQueryFilter.builder()
    .userIds(List.of("user-123", "user-456"))
    .eventTypes(List.of("LOGIN", "TICKET_CREATED"))
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.of(2024, 3, 31))
    .build();

// Or construct with no-args constructor and setters
EventQueryFilter emptyFilter = new EventQueryFilter();
emptyFilter.setUserIds(List.of("user-789"));
emptyFilter.setStartDate(LocalDate.now().minusDays(7));
```

> All fields are optional. Passing `null` for any field typically means no restriction is applied for that criteria — validate this behaviour against the consuming repository or query service.
