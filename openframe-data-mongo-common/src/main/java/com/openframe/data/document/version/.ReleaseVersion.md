MongoDB document entity representing a versioned release record scoped to a specific tenant, stored in the `release_versions` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant scope identifier, indexed for query performance |
| `documentVersion` | `Long` | Optimistic locking version via Spring's `@Version` |
| `version` | `String` | Release version label (e.g. `1.0.0`), indexed |
| `createdAt` | `Instant` | Auto-populated creation timestamp |
| `updatedAt` | `Instant` | Auto-populated last modification timestamp |

## Key Behaviors

- Implements `TenantScoped` to enforce multi-tenant data isolation
- Uses Spring Data MongoDB's optimistic locking (`@Version`) to prevent concurrent write conflicts
- Indexed fields (`tenantId`, `version`) optimize tenant-filtered version lookups

## Usage Example

```java
// Create a new release version for a tenant
ReleaseVersion release = new ReleaseVersion();
release.setTenantId("tenant-abc");
release.setVersion("2.1.0");

// Persist via a Spring Data repository
releaseVersionRepository.save(release);

// Query by tenant and version
Optional<ReleaseVersion> found = releaseVersionRepository
    .findByTenantIdAndVersion("tenant-abc", "2.1.0");
```
