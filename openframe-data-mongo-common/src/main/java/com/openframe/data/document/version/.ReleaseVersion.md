MongoDB document entity representing a versioned release record scoped to a specific tenant, stored in the `release_versions` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant isolator, indexed for fast lookup |
| `documentVersion` | `Long` | Optimistic locking version (`@Version`) |
| `version` | `String` | Release version string, indexed |
| `createdAt` | `Instant` | Auto-populated creation timestamp |
| `updatedAt` | `Instant` | Auto-populated last-modified timestamp |

- Implements `TenantScoped` to enforce multi-tenant data isolation
- Uses Spring Data MongoDB auditing (`@CreatedDate`, `@LastModifiedDate`)
- Optimistic concurrency control via Spring's `@Version`

## Usage Example

```java
// Creating and persisting a new release version
ReleaseVersion release = new ReleaseVersion();
release.setTenantId("tenant-abc");
release.setVersion("1.4.2");

releaseVersionRepository.save(release);

// Querying by tenant and version string
Optional<ReleaseVersion> found = releaseVersionRepository
    .findByTenantIdAndVersion("tenant-abc", "1.4.2");
```

> **Note:** The `documentVersion` field is managed automatically by Spring Data — do not set it manually. Concurrent modifications to the same document will throw an `OptimisticLockingFailureException`.
