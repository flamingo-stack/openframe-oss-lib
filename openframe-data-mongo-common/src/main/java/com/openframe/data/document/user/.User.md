MongoDB document entity representing a platform user within a tenant, mapped to the `users` collection with multi-tenant isolation via `TenantScoped`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document `_id` |
| `tenantId` | `String` | Indexed tenant isolator (implements `TenantScoped`) |
| `email` | `String` | Indexed, normalized to lowercase on set |
| `firstName` / `lastName` | `String` | Display name fields |
| `roles` | `List<UserRole>` | Assigned roles, defaults to empty list |
| `emailVerified` | `boolean` | Email verification flag; auto-set `true` for SSO logins |
| `status` | `UserStatus` | Indexed lifecycle status, defaults to `ACTIVE` |
| `createdAt` / `updatedAt` | `LocalDateTime` | Audit timestamps via Spring Data `@CreatedDate` / `@LastModifiedDate` |

**`setEmail(String)`** — Custom setter that trims and lowercases the email using `Locale.ROOT` before persisting, preventing case-sensitive duplicates.

## Usage Example

```java
// Build a new user with the Lombok builder
User user = User.builder()
    .tenantId("tenant-abc")
    .email("  Alice@Example.com  ")   // normalized to "alice@example.com"
    .firstName("Alice")
    .lastName("Smith")
    .roles(List.of(UserRole.ADMIN))
    .emailVerified(true)
    .status(UserStatus.ACTIVE)
    .build();

// Retrieve tenant-scoped users via repository
List<User> tenantUsers = userRepository.findByTenantId("tenant-abc");
```

> **Note:** Always use `setEmail()` (or the builder's `.email()`) rather than direct field injection to guarantee email normalization before storage.
