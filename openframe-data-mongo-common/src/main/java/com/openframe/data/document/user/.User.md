MongoDB document entity representing a user account, mapped to the `users` collection with auditing, role management, and normalized email handling.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB `@Id` primary key |
| `email` | `String` | Indexed, auto-normalized to lowercase on set |
| `firstName` / `lastName` | `String` | User display name fields |
| `roles` | `List<UserRole>` | Assigned roles, defaults to empty list |
| `emailVerified` | `boolean` | Email verification flag; `true` by default for SSO users |
| `status` | `UserStatus` | Indexed account status, defaults to `ACTIVE` |
| `createdAt` / `updatedAt` | `LocalDateTime` | Spring Data audit timestamps |

## Key Behaviors

- **Email normalization** — `setEmail()` trims whitespace and lowercases the value using `Locale.ROOT`, ensuring consistent storage and lookups.
- **Lombok `@SuperBuilder`** — Supports builder inheritance for subclasses; `@Builder.Default` ensures `roles` and `status` have safe defaults even when using the builder pattern.
- **Spring Data auditing** — `@CreatedDate` and `@LastModifiedDate` are populated automatically when auditing is enabled in the application context.

## Usage Example

```java
// Build a new user
User user = User.builder()
    .email("  Alice@Example.com  ")  // normalized to "alice@example.com"
    .firstName("Alice")
    .lastName("Smith")
    .roles(List.of(UserRole.TECHNICIAN))
    .emailVerified(true)
    .build();

// Status defaults to ACTIVE
assert user.getStatus() == UserStatus.ACTIVE;

// Save via repository
userRepository.save(user);
```
