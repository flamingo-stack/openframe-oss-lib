MongoDB document entity representing a platform user within a tenant, handling identity, roles, status, and audit timestamps with automatic email normalization.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB `@Id` primary key |
| `tenantId` | `String` | Indexed foreign key for multi-tenant scoping via `TenantScoped` |
| `email` | `String` | Indexed, auto-normalized to lowercase/trimmed on set |
| `firstName` / `lastName` | `String` | User display name fields |
| `roles` | `List<UserRole>` | Assigned roles, defaults to empty list |
| `emailVerified` | `boolean` | Defaults to `false`; set to `true` on SSO first login |
| `status` | `UserStatus` | Indexed lifecycle status, defaults to `ACTIVE` |
| `createdAt` / `updatedAt` | `LocalDateTime` | Audit timestamps managed by Spring Data |

## Usage Example

```java
// Build a new user with the Lombok builder
User user = User.builder()
    .tenantId("tenant-abc")
    .firstName("Jane")
    .lastName("Doe")
    .roles(List.of(UserRole.ADMIN))
    .build();

// Email is normalized automatically via the custom setter
user.setEmail("  Jane.Doe@Example.com  ");
// user.getEmail() → "jane.doe@example.com"

// Persist via Spring Data MongoDB repository
userRepository.save(user);

// Retrieve all active users for a tenant
List<User> activeUsers = userRepository
    .findByTenantIdAndStatus("tenant-abc", UserStatus.ACTIVE);
```

> **Note:** `setEmail` always trims whitespace and lowercases the input using `Locale.ROOT`, ensuring consistent lookups regardless of how the email was originally provided.
