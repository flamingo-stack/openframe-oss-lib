Embedded document model representing an organization's contact details, including contact persons and physical/mailing addresses within the OpenFrame data layer.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `contacts` | `List<ContactPerson>` | List of contact persons associated with the organization (defaults to empty list) |
| `physicalAddress` | `Address` | Physical location of the organization |
| `mailingAddress` | `Address` | Mailing address of the organization |
| `mailingAddressSameAsPhysical` | `Boolean` | Flag indicating if mailing address mirrors the physical address (defaults to `false`) |

## Usage Example

```java
// Build with separate addresses
ContactInformation contactInfo = ContactInformation.builder()
    .contacts(List.of(primaryContact, billingContact))
    .physicalAddress(Address.builder()
        .street("123 Main St")
        .city("Austin")
        .state("TX")
        .build())
    .mailingAddress(Address.builder()
        .street("PO Box 456")
        .city("Austin")
        .state("TX")
        .build())
    .mailingAddressSameAsPhysical(false)
    .build();

// Build with same physical and mailing address
ContactInformation sameAddress = ContactInformation.builder()
    .contacts(List.of(primaryContact))
    .physicalAddress(officeAddress)
    .mailingAddressSameAsPhysical(true)
    .build();
```

> **Note:** `ContactInformation` is an embedded document and is not stored as a separate MongoDB collection. It is intended to be nested within an organization document.
