Represents a MongoDB document for internal technician notes on a support ticket, stored in the `ticket_notes` collection. Notes are separate from ticket dialog/messages and support plain text or markdown content.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier |
| `tenantId` | `String` | Tenant scope identifier (indexed) |
| `ticketId` | `String` | Reference to the parent ticket (indexed) |
| `content` | `String` | Note body — plain text or markdown |
| `authorId` | `String` | User ID of the technician; name resolved via DataLoader |
| `createdAt` | `Instant` | Auto-set on creation (`@CreatedDate`) |
| `updatedAt` | `Instant` | Auto-set on modification (`@LastModifiedDate`) |

**Indexes:**
- Compound index on `(ticketId ASC, createdAt DESC)` for efficient chronological note retrieval per ticket
- Single-field indexes on `tenantId` and `ticketId`

## Usage Example

```java
TicketNote note = TicketNote.builder()
    .tenantId("tenant-123")
    .ticketId("ticket-456")
    .authorId("user-789")
    .content("Checked device logs — issue stems from a misconfigured DNS entry.")
    .build();

ticketNoteRepository.save(note);

// Fetch notes for a ticket (resolved via compound index)
List<TicketNote> notes = ticketNoteRepository
    .findByTicketIdOrderByCreatedAtDesc("ticket-456");
```

> Notes support edit and delete operations. The author's display name and avatar are not stored directly — they are resolved at query time via a DataLoader using `authorId`.
