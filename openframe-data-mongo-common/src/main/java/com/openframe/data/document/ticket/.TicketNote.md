Represents a MongoDB document for internal technician notes attached to a support ticket, stored separately from client-facing dialog/messages.

## Key Components

| Field | Type | Description |
|---|---|---|
| `id` | `String` | MongoDB document ID |
| `tenantId` | `String` | Tenant scope identifier (indexed) |
| `ticketId` | `String` | Parent ticket reference (indexed) |
| `content` | `String` | Note body — plain text or Markdown |
| `authorId` | `String` | Technician user ID; name resolved via DataLoader |
| `createdAt` | `Instant` | Auto-set on creation (`@CreatedDate`) |
| `updatedAt` | `Instant` | Auto-updated on modification (`@LastModifiedDate`) |

**Indexes:** Single indexes on `tenantId` and `ticketId`; compound index `ticket_created` on `{ticketId: 1, createdAt: -1}` for efficient per-ticket note listing in reverse-chronological order.

## Usage Example

```java
TicketNote note = TicketNote.builder()
    .tenantId("tenant-abc")
    .ticketId("ticket-123")
    .authorId("user-456")
    .content("Checked device logs — issue reproduced after reboot.")
    .build();

ticketNoteRepository.save(note);
```

> Notes support edit and delete operations. Author display name and avatar are resolved at query time via DataLoader rather than being stored on the document, keeping the record lightweight and consistent with user profile changes.
