Represents an internal technician note attached to a support ticket, stored as a MongoDB document separate from client-facing messages or dialog threads.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `ticketId` | `String` | Reference to the parent ticket (indexed) |
| `content` | `String` | Note body — plain text or Markdown |
| `authorId` | `String` | User ID of the technician who wrote the note; display name resolved via DataLoader |
| `createdAt` | `Instant` | Auto-populated creation timestamp (`@CreatedDate`) |
| `updatedAt` | `Instant` | Auto-populated last-modified timestamp (`@LastModifiedDate`) |

**Collection:** `ticket_notes`
**Index:** Compound index on `{ ticketId: 1, createdAt: -1 }` for efficient chronological retrieval per ticket.

## Usage Example

```java
// Create a new internal note for a ticket
TicketNote note = TicketNote.builder()
    .ticketId("ticket-abc123")
    .authorId("tech-user-456")
    .content("Checked device logs — confirmed network adapter failure. Ordered replacement part.")
    .build();

ticketNoteRepository.save(note);

// Fetch all notes for a ticket ordered by creation date (descending)
List<TicketNote> notes = ticketNoteRepository
    .findByTicketIdOrderByCreatedAtDesc("ticket-abc123");
```

> **Note:** `authorId` stores only the technician's User ID. The display name and avatar shown in the UI are resolved at query time using a DataLoader to avoid denormalized data. Notes support edit and delete operations via `updatedAt` tracking.
