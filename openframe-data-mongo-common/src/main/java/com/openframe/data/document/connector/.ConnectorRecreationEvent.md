Represents a single connector recreation event persisted to MongoDB, used by the Debezium connector recovery rate limiter to enforce a rolling-window cap on recreate-under-new-version operations across all management replicas.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `baseName` | `String` | Debezium connector base name being tracked |
| `createdAt` | `Instant` | Timestamp of the recreation event |

- **Collection:** `connector_recreation_events`
- **Lombok:** `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` — full boilerplate generation
- **Lifecycle note:** Old events are purged on each write by the rate limiter; no MongoDB TTL index is required

## Usage Example

```java
// Record a new recreation event
ConnectorRecreationEvent event = ConnectorRecreationEvent.builder()
    .id(UUID.randomUUID().toString())
    .baseName("customers-connector")
    .createdAt(Instant.now())
    .build();

mongoTemplate.save(event);

// Query recent events within the rolling window
Instant windowStart = Instant.now().minus(Duration.ofMinutes(10));
List<ConnectorRecreationEvent> recentEvents = mongoTemplate.find(
    Query.query(
        Criteria.where("baseName").is("customers-connector")
                .and("createdAt").gte(windowStart)
    ),
    ConnectorRecreationEvent.class
);
```
