Enum defining the two possible owner (reporter) types for a ticket, used to distinguish whether a ticket was created by an end-user client (via Fae) or an MSP platform administrator.

## Key Components

| Value | Description |
|-------|-------------|
| `CLIENT` | End-user device reporter, typically interacting through the Fae client interface |
| `ADMIN` | MSP platform user or administrator creating the ticket directly |

## Usage Example

```java
Ticket ticket = new Ticket();
ticket.setOwnerType(TicketOwnerType.CLIENT);

if (ticket.getOwnerType() == TicketOwnerType.ADMIN) {
    // Apply admin-specific ticket handling logic
} else {
    // Handle client-submitted ticket from Fae
}
```

## Notes

- Intentionally kept separate from any general `OwnerType` enum to allow independent feature flag control per owner category.
- `CLIENT` maps to tickets originating from the Fae end-user AI interface.
- `ADMIN` maps to tickets created by MSP technicians or administrators within the OpenFrame platform.
