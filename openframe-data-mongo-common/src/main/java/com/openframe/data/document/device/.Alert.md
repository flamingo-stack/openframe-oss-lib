A simple data model representing a device alert with severity classification, message content, and resolution status.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | Unique identifier for the alert |
| `severity` | `String` | Alert level: `HIGH`, `MEDIUM`, or `LOW` |
| `message` | `String` | Human-readable alert description |
| `timestamp` | `Instant` | UTC timestamp of when the alert was created |
| `resolved` | `boolean` | Indicates whether the alert has been addressed |

> Uses Lombok `@Data` to auto-generate getters, setters, `equals`, `hashCode`, and `toString`.

## Usage Example

```java
Alert alert = new Alert();
alert.setId("alert-001");
alert.setSeverity("HIGH");
alert.setMessage("CPU usage exceeded 95% threshold");
alert.setTimestamp(Instant.now());
alert.setResolved(false);

// Check if action is needed
if (!alert.isResolved() && alert.getSeverity().equals("HIGH")) {
    notifyTechnician(alert);
}
```
