MongoDB document entity representing an integrated third-party tool within a tenant's OpenFrame environment, storing connection, configuration, and monitoring metadata.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document ID (`@Id`) |
| `tenantId` | `String` | Tenant scoping identifier (via `TenantScoped`) |
| `key` | `String` | Unique tool identifier per tenant |
| `toolUrls` | `List<ToolUrl>` | Endpoint URLs associated with the tool |
| `credentials` | `ToolCredentials` | Authentication/connection credentials |
| `layer` / `layerOrder` / `layerColor` | `String/Integer` | UI layer positioning and styling |
| `metricsPath` | `String` | Prometheus/metrics scrape path |
| `healthCheckEndpoint` | `String` | Health probe URL |
| `debeziumConnectors` | `Object[]` | CDC connector configurations |

**Indexes:** Compound unique index on `(tenantId, key)` ensures no duplicate tool keys per tenant.

## Usage Example

```java
IntegratedTool tool = IntegratedTool.builder()
    .tenantId("tenant-abc")
    .key("zabbix")
    .name("Zabbix Monitoring")
    .type("monitoring")
    .toolType("zabbix")
    .category("infrastructure")
    .platformCategory("observability")
    .enabled(true)
    .metricsPath("/metrics")
    .healthCheckEndpoint("/api/v1/ping")
    .healthCheckInterval(30)
    .connectionTimeout(5000)
    .readTimeout(10000)
    .layer("monitoring")
    .layerOrder(1)
    .layerColor("#FFC008")
    .build();

// Persist via repository
integratedToolRepository.save(tool);

// Query by tenant and key
Optional<IntegratedTool> found = integratedToolRepository
    .findByTenantIdAndKey("tenant-abc", "zabbix");
```
