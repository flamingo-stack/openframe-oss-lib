MongoDB document entity representing an integrated third-party tool in the OpenFrame platform, storing configuration, connectivity, and monitoring metadata for MSP tool integrations.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `name` / `description` / `icon` | `String` | Display metadata for the tool |
| `toolUrls` | `List<ToolUrl>` | Endpoint URLs associated with the tool |
| `type` / `toolType` / `category` / `platformCategory` | `String` | Classification fields for tool categorization |
| `enabled` | `boolean` | Toggle for activating/deactivating the integration |
| `credentials` | `ToolCredentials` | Authentication/credential configuration |
| `layer` / `layerOrder` / `layerColor` | `String` / `Integer` | UI layer stack positioning and styling |
| `metricsPath` / `healthCheckEndpoint` | `String` | Monitoring and observability configuration |
| `healthCheckInterval` / `connectionTimeout` / `readTimeout` | `Integer` | Timing parameters for health checks and connections |
| `allowedEndpoints` | `String[]` | Allowlisted endpoint paths for the tool |
| `debeziumConnectors` | `Object[]` | CDC (Change Data Capture) connector configurations |

## Usage Example

```java
IntegratedTool tool = IntegratedTool.builder()
    .name("Zabbix")
    .description("Network monitoring platform")
    .icon("zabbix-icon.svg")
    .type("monitoring")
    .toolType("open-source")
    .category("infrastructure")
    .platformCategory("observability")
    .enabled(true)
    .layer("network")
    .layerOrder(2)
    .layerColor("#FFC008")
    .healthCheckEndpoint("/api/v1/health")
    .healthCheckInterval(30)
    .connectionTimeout(5000)
    .readTimeout(3000)
    .metricsPath("/metrics")
    .credentials(toolCredentials)
    .toolUrls(List.of(new ToolUrl("primary", "https://zabbix.internal")))
    .build();
```

> **Note:** This entity is persisted to the `integrated_tools` MongoDB collection and is managed by the OpenFrame platform layer. Lombok annotations (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate boilerplate code.
