MongoDB document entity representing file attachment metadata for tickets, storing references to externally hosted files (S3/MinIO) rather than the binary content itself.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant isolator, indexed for multi-tenancy scoping |
| `ticketId` | `String` | Reference to the parent ticket, indexed for lookups |
| `fileName` | `String` | Original name of the uploaded file |
| `contentType` | `String` | MIME type (e.g., `image/png`, `application/pdf`) |
| `fileSize` | `Long` | File size in bytes |
| `storagePath` | `String` | External storage key (S3 object key or MinIO path) |
| `uploadedAt` | `Instant` | Auto-set creation timestamp via `@CreatedDate` |
| `uploadedBy` | `String` | Identifier of the user who uploaded the file |

**Implements:** `TenantScoped` — enforces tenant isolation across queries.
**Collection:** `ticket_attachments`

## Usage Example

```java
// Create and persist an attachment record after uploading to S3/MinIO
TicketAttachment attachment = TicketAttachment.builder()
    .tenantId("tenant-abc")
    .ticketId("ticket-001")
    .fileName("screenshot.png")
    .contentType("image/png")
    .fileSize(204800L)
    .storagePath("tenants/tenant-abc/tickets/ticket-001/screenshot.png")
    .uploadedBy("user-42")
    .build();

ticketAttachmentRepository.save(attachment);

// Fetch all attachments for a ticket
List<TicketAttachment> attachments = ticketAttachmentRepository
    .findByTenantIdAndTicketId("tenant-abc", "ticket-001");
```

> **Note:** This entity holds only metadata. The actual file binary is stored and retrieved separately via the configured S3/MinIO storage backend using `storagePath` as the object key.
