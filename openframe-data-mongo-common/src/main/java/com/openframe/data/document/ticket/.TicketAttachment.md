MongoDB document entity that stores file attachment metadata for tickets, with the actual file content held in external object storage (S3/MinIO).

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Tenant scope index for multi-tenancy isolation |
| `ticketId` | `String` | Indexed reference to the parent ticket |
| `fileName` | `String` | Original file name |
| `contentType` | `String` | MIME type of the uploaded file |
| `fileSize` | `Long` | File size in bytes |
| `storagePath` | `String` | S3 key or MinIO path to retrieve the actual file |
| `uploadedAt` | `Instant` | Auto-populated timestamp via `@CreatedDate` |
| `uploadedBy` | `String` | Identifier of the user who uploaded the file |

Implements `TenantScoped` to enforce tenant isolation across all queries.

## Usage Example

```java
TicketAttachment attachment = TicketAttachment.builder()
    .tenantId("tenant-abc")
    .ticketId("ticket-001")
    .fileName("network-diagram.png")
    .contentType("image/png")
    .fileSize(204800L)
    .storagePath("tenants/tenant-abc/tickets/ticket-001/network-diagram.png")
    .uploadedBy("user-xyz")
    .build();

ticketAttachmentRepository.save(attachment);
```

> **Note:** `storagePath` is a reference key only — file retrieval requires a separate call to your configured S3/MinIO client using this path.
