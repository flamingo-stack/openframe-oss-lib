MongoDB document entity representing file attachment metadata linked to a support ticket. Files are stored externally (S3/MinIO) while this document captures storage references and upload details in the `ticket_attachments` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `ticketId` | `String` | Indexed reference to the parent ticket |
| `fileName` | `String` | Original file name |
| `contentType` | `String` | MIME type of the file |
| `fileSize` | `Long` | File size in bytes |
| `storagePath` | `String` | External storage key (S3/MinIO path) |
| `uploadedAt` | `Instant` | Auto-populated creation timestamp (`@CreatedDate`) |
| `uploadedBy` | `String` | Identifier of the user who uploaded the file |

## Usage Example

```java
// Build and persist a new attachment record after uploading to MinIO
TicketAttachment attachment = TicketAttachment.builder()
    .ticketId("ticket-abc123")
    .fileName("network-diagram.png")
    .contentType("image/png")
    .fileSize(204800L)
    .storagePath("attachments/2024/01/ticket-abc123/network-diagram.png")
    .uploadedBy("tech-user-42")
    .build();

ticketAttachmentRepository.save(attachment);

// Query all attachments for a ticket
List<TicketAttachment> attachments = ticketAttachmentRepository
    .findByTicketId("ticket-abc123");
```

> **Note:** This entity stores only metadata. The actual file retrieval requires a presigned URL or direct access to the external storage using `storagePath` as the object key.
