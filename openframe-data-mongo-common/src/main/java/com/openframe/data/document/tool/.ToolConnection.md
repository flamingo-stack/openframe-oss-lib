MongoDB document entity representing a connection between a machine and a specific MSP tool within a tenant's scope, persisted in the `tool_connections` collection.

## Key Components

- **`@Document(collection = "tool_connections")`** — Maps this entity to the MongoDB `tool_connections` collection
- **`@CompoundIndex`** — Enforces a unique constraint on `(tenantId, machineId, toolType)`, preventing duplicate tool connections per machine per tenant
- **`TenantScoped`** — Implements multi-tenancy interface, ensuring all queries are scoped to a specific tenant
- **`toolType`** — Enum (`ToolType`) identifying which MSP tool is connected (e.g., RMM, PSA)
- **`status`** — Enum (`ConnectionStatus`) tracking the current connection state
- **`metadata`** — Flexible JSON string for storing tool-specific configuration or data
- **Timestamps** — `connectedAt`, `lastSyncAt`, `disconnectedAt` track the full connection lifecycle

## Usage Example

```java
ToolConnection connection = new ToolConnection();
connection.setTenantId("tenant-abc");
connection.setMachineId("machine-xyz");
connection.setToolType(ToolType.RMM);
connection.setAgentToolId("agent-001");
connection.setStatus(ConnectionStatus.CONNECTED);
connection.setMetadata("{\"version\": \"6.0\", \"agentPort\": 9090}");
connection.setConnectedAt(Instant.now());

toolConnectionRepository.save(connection);
```

> **Note:** The unique compound index ensures only one active connection record exists per `(tenantId, machineId, toolType)` combination, preventing duplicate registrations across OpenFrame's multi-tenant architecture.
