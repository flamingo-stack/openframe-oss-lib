MongoDB document entity representing a connection between a machine and an external tool within a tenant's scope, persisted in the `tool_connections` collection.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier |
| `tenantId` | `String` | Tenant scope identifier (via `TenantScoped`) |
| `machineId` | `String` | Target machine for the tool connection |
| `toolType` | `ToolType` | Enum defining the type of tool (e.g., RMM, PSA) |
| `agentToolId` | `String` | External reference ID from the connected tool |
| `status` | `ConnectionStatus` | Current connection state |
| `metadata` | `String` | JSON blob for tool-specific configuration data |
| `connectedAt` | `Instant` | Timestamp when connection was established |
| `lastSyncAt` | `Instant` | Timestamp of the most recent sync |
| `disconnectedAt` | `Instant` | Timestamp when connection was terminated |

A **unique compound index** (`tenant_machine_tool_idx`) enforces that each machine can have at most one connection per tool type within a tenant.

## Usage Example

```java
ToolConnection connection = new ToolConnection();
connection.setTenantId("tenant-abc");
connection.setMachineId("machine-xyz");
connection.setToolType(ToolType.RMM);
connection.setAgentToolId("ext-agent-001");
connection.setStatus(ConnectionStatus.CONNECTED);
connection.setMetadata("{\"agentVersion\": \"5.2\", \"platform\": \"windows\"}");
connection.setConnectedAt(Instant.now());

// Persisted to 'tool_connections' collection
toolConnectionRepository.save(connection);
```

> The `metadata` field stores tool-specific data as a raw JSON string, allowing flexible schema per `ToolType` without requiring additional document types.
