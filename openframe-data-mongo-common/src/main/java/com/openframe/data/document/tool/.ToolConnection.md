Represents a persistent MongoDB document that tracks the connection state between a machine and a specific tool integration within the OpenFrame platform.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `machineId` | `String` | References the machine this connection belongs to |
| `toolType` | `ToolType` | Enum identifying the tool integration type |
| `agentToolId` | `String` | External identifier for the tool on the agent side |
| `status` | `ConnectionStatus` | Current lifecycle state of the connection |
| `metadata` | `String` | JSON string for tool-specific configuration/data |
| `connectedAt` | `Instant` | Timestamp when the connection was established |
| `lastSyncAt` | `Instant` | Timestamp of the most recent synchronization |
| `disconnectedAt` | `Instant` | Timestamp when the connection was terminated |

**Index:** A unique compound index (`machine_tool_idx`) on `machineId` + `toolType` enforces that each machine can have only one active connection per tool type.

## Usage Example

```java
ToolConnection connection = new ToolConnection();
connection.setMachineId("machine-abc-123");
connection.setToolType(ToolType.RMMAGENT);
connection.setAgentToolId("agent-tool-xyz");
connection.setStatus(ConnectionStatus.CONNECTED);
connection.setMetadata("{\"endpoint\": \"https://rmm.example.com\"}");
connection.setConnectedAt(Instant.now());

// Persisted to the "tool_connections" collection
toolConnectionRepository.save(connection);
```

## Notes

- The unique compound index prevents duplicate tool registrations per machine at the database level
- `metadata` is stored as a raw JSON string, allowing flexible tool-specific payloads without schema changes
- Lifecycle timestamps (`connectedAt`, `lastSyncAt`, `disconnectedAt`) enable connection auditing and health monitoring
