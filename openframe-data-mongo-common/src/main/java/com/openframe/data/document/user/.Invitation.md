MongoDB document entity representing a user invitation within a tenant, tracking email, roles, expiration, and lifecycle status.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Indexed tenant reference (implements `TenantScoped`) |
| `email` | `String` | Indexed invitee email, normalized on set |
| `roles` | `List<UserRole>` | Roles to assign upon acceptance (defaults to empty list) |
| `expiresAt` | `Instant` | Invitation expiry timestamp |
| `status` | `InvitationStatus` | Lifecycle state, defaults to `PENDING` |
| `createdAt` / `updatedAt` | `Instant` | Audit timestamps managed by Spring Data |

## Usage Example

```java
// Create a new invitation
Invitation invitation = Invitation.builder()
    .tenantId("tenant-123")
    .email("  User@Example.com  ") // normalized to "user@example.com"
    .roles(List.of(UserRole.TECHNICIAN))
    .expiresAt(Instant.now().plus(7, ChronoUnit.DAYS))
    .build();

// Email is always stored trimmed and lowercased
invitation.setEmail("  Admin@Company.COM  ");
System.out.println(invitation.getEmail()); // "admin@company.com"

// Check invitation state
if (invitation.getStatus() == InvitationStatus.PENDING) {
    // process acceptance
}
```

> **Note:** Email normalization (trim + lowercase) is enforced in the setter, ensuring consistent lookups regardless of how the address was submitted.
