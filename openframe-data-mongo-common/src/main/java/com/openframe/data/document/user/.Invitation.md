MongoDB document entity representing a user invitation record, stored in the `invitations` collection. Handles email normalization and tracks invitation lifecycle state.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `email` | `String` | Indexed, normalized email address of the invitee |
| `roles` | `List<UserRole>` | Roles to assign upon invitation acceptance (defaults to empty list) |
| `expiresAt` | `Instant` | Invitation expiration timestamp |
| `status` | `InvitationStatus` | Current state, defaults to `PENDING` |
| `createdAt` | `Instant` | Auto-populated creation timestamp (`@CreatedDate`) |
| `updatedAt` | `Instant` | Auto-populated last modified timestamp (`@LastModifiedDate`) |

**`setEmail(String email)`** — Custom setter that trims whitespace and lowercases the email using `Locale.ROOT` before persistence, preventing duplicate entries from case or spacing variations.

## Usage Example

```java
// Build a new invitation
Invitation invitation = Invitation.builder()
    .email("  John.Doe@Example.com  ")  // will be normalized to "john.doe@example.com"
    .roles(List.of(UserRole.TECHNICIAN))
    .expiresAt(Instant.now().plus(7, ChronoUnit.DAYS))
    .build();

// Status defaults to PENDING
assert invitation.getStatus() == InvitationStatus.PENDING;

// Update status after acceptance
invitation.setStatus(InvitationStatus.ACCEPTED);
```

> **Note:** Email normalization is applied automatically via the custom `setEmail` setter — both direct assignment and builder usage will result in a trimmed, lowercase value stored in MongoDB.
