MongoDB document entity representing a user invitation within a tenant, storing invitation state, target email, assigned roles, and lifecycle timestamps.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Indexed tenant reference (implements `TenantScoped`) |
| `email` | `String` | Indexed, normalized invitee email address |
| `roles` | `List<UserRole>` | Roles to assign upon acceptance (defaults to empty list) |
| `expiresAt` | `Instant` | Invitation expiry timestamp |
| `status` | `InvitationStatus` | Current state, defaults to `PENDING` |
| `createdAt` / `updatedAt` | `Instant` | Auto-managed audit timestamps |

**`setEmail(String email)`** — Custom setter that normalizes input via `trim().toLowerCase()` before persisting, ensuring consistent email lookups.

## Usage Example

```java
// Create a new invitation
Invitation invitation = Invitation.builder()
    .tenantId("tenant-123")
    .email("  John.Doe@Example.com  ") // normalized to "john.doe@example.com"
    .roles(List.of(UserRole.TECHNICIAN))
    .expiresAt(Instant.now().plus(7, ChronoUnit.DAYS))
    .build();

// Check status
if (invitation.getStatus() == InvitationStatus.PENDING) {
    invitation.setStatus(InvitationStatus.ACCEPTED);
}
```

> **Note:** The `@Indexed` annotations on `tenantId` and `email` ensure efficient multi-tenant queries. Always query invitations scoped by `tenantId` to respect tenant isolation.
