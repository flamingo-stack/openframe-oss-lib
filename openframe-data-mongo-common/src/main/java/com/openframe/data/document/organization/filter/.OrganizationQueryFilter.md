Data transfer object for specifying filter criteria when querying organization documents in MongoDB.

## Key Components

| Field | Type | Description |
|---|---|---|
| `category` | `String` | Filter by organization category |
| `minEmployees` | `Integer` | Lower bound for employee count |
| `maxEmployees` | `Integer` | Upper bound for employee count |
| `hasActiveContract` | `Boolean` | Filter by active contract presence |
| `status` | `String` | Filter by organization status |

## Usage Example

```java
// Build a filter for mid-sized organizations with active contracts
OrganizationQueryFilter filter = OrganizationQueryFilter.builder()
    .category("MSP")
    .minEmployees(10)
    .maxEmployees(500)
    .hasActiveContract(true)
    .status("ACTIVE")
    .build();

// Use with a repository or query builder
List<Organization> results = organizationRepository.findByFilter(filter);
```

All fields are optional — unset fields are ignored when constructing the underlying MongoDB query, allowing partial or open-ended filtering with any combination of criteria.
