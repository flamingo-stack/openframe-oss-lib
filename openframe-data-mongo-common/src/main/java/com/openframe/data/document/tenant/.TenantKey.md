MongoDB document entity representing a cryptographic key pair associated with a specific tenant, stored in the `tenant_keys` collection.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Reference to the owning tenant |
| `keyId` | `String` | Unique identifier for this key pair |
| `publicPem` | `String` | PEM-encoded public key |
| `privateEncrypted` | `String` | Encrypted private key (stored securely) |
| `active` | `boolean` | Whether this key is currently in use |
| `createdAt` | `Instant` | Auto-populated timestamp on creation |
| `rotatedAt` | `Instant` | Auto-updated timestamp on modification (tracks key rotation) |

## Usage Example

```java
// Creating a new tenant key
TenantKey tenantKey = new TenantKey();
tenantKey.setTenantId("tenant-abc123");
tenantKey.setKeyId("key-xyz789");
tenantKey.setPublicPem("-----BEGIN PUBLIC KEY-----\n...");
tenantKey.setPrivateEncrypted("enc:base64encodedEncryptedPrivateKey==");
tenantKey.setActive(true);

// Persisting via Spring Data MongoDB repository
tenantKeyRepository.save(tenantKey);

// Querying active keys for a tenant
List<TenantKey> activeKeys = tenantKeyRepository
    .findByTenantIdAndActiveTrue("tenant-abc123");
```

## Notes

- Private keys are **never stored in plaintext** — `privateEncrypted` should always contain an encrypted payload.
- `rotatedAt` serves as a key rotation audit trail, automatically updated by Spring Data's `@LastModifiedDate`.
- Only one key should typically be `active = true` per tenant at a time to avoid authentication conflicts.
