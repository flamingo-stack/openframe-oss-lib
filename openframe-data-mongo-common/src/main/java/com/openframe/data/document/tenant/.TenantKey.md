A MongoDB document entity representing a cryptographic key pair scoped to a specific tenant, stored in the `tenant_keys` collection.

## Key Components

| Field | Type | Description |
|---|---|---|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Indexed reference to the owning tenant |
| `keyId` | `String` | Logical identifier for the key pair |
| `publicPem` | `String` | PEM-encoded public key |
| `privateEncrypted` | `String` | Encrypted private key material |
| `active` | `boolean` | Indicates if this key is currently in use |
| `createdAt` | `Instant` | Auto-populated creation timestamp |
| `rotatedAt` | `Instant` | Auto-updated timestamp on key rotation |

Implements `TenantScoped` to enforce multi-tenancy isolation across the platform.

## Usage Example

```java
// Creating and persisting a new tenant key
TenantKey key = new TenantKey();
key.setTenantId("tenant-abc123");
key.setKeyId("signing-key-v1");
key.setPublicPem("-----BEGIN PUBLIC KEY-----\n...");
key.setPrivateEncrypted("enc::AES256:...");
key.setActive(true);

tenantKeyRepository.save(key);

// Querying active keys for a tenant
List<TenantKey> activeKeys = tenantKeyRepository
    .findByTenantIdAndActiveTrue("tenant-abc123");
```

> **Security Note:** The `privateEncrypted` field should never be stored or transmitted in plaintext. Always apply tenant-specific encryption before persisting and decrypt only in trusted service boundaries.
