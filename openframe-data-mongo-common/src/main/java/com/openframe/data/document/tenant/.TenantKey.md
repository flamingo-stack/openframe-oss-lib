Represents a tenant-scoped cryptographic key pair stored in MongoDB, used to manage public/private key material per tenant in the OpenFrame platform.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `id` | `String` | MongoDB document identifier (`@Id`) |
| `tenantId` | `String` | Indexed tenant reference implementing `TenantScoped` |
| `keyId` | `String` | Logical identifier for the key pair |
| `publicPem` | `String` | PEM-encoded public key |
| `privateEncrypted` | `String` | Encrypted private key material |
| `active` | `boolean` | Indicates if this key is currently in use |
| `createdAt` | `Instant` | Auto-populated creation timestamp |
| `rotatedAt` | `Instant` | Auto-updated on each document modification (tracks rotation) |

## Usage Example

```java
// Creating a new tenant key document
TenantKey tenantKey = new TenantKey();
tenantKey.setTenantId("tenant-abc123");
tenantKey.setKeyId("key-v1");
tenantKey.setPublicPem("-----BEGIN PUBLIC KEY-----\n...");
tenantKey.setPrivateEncrypted("enc:base64==...");
tenantKey.setActive(true);

// Persisting via Spring Data MongoDB repository
tenantKeyRepository.save(tenantKey);

// Querying active key for a tenant
Optional<TenantKey> activeKey = tenantKeyRepository
    .findByTenantIdAndActiveTrue("tenant-abc123");
```

## Notes

- Collection: `tenant_keys`
- `privateEncrypted` should **never** store raw private key material — always store an encrypted ciphertext
- `rotatedAt` doubles as a key rotation timestamp via `@LastModifiedDate`
- Implements `TenantScoped` to enforce multi-tenant data isolation patterns across the platform
