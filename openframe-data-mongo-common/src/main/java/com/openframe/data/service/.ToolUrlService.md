Service responsible for resolving URLs from integrated tool configurations by filtering against a specified `ToolUrlType`.

## Key Components

| Component | Description |
|-----------|-------------|
| `ToolUrlService` | Spring `@Service` bean for URL lookup operations |
| `getUrlByToolType(IntegratedTool, ToolUrlType)` | Finds the first `ToolUrl` matching the given `ToolUrlType` from an integrated tool's URL list |

## Usage Example

```java
@Autowired
private ToolUrlService toolUrlService;

// Retrieve the API base URL for a specific integrated tool
Optional<ToolUrl> apiUrl = toolUrlService.getUrlByToolType(integratedTool, ToolUrlType.API);

apiUrl.ifPresent(url -> {
    String endpoint = url.getValue();
    // Use endpoint to make API calls
});
```

## Behavior Notes

- Returns an `Optional<ToolUrl>` — callers should handle the empty case when no URL matches the given type.
- Uses stream `findFirst()`, so if multiple URLs share the same `ToolUrlType`, only the first registered entry is returned.
- Operates on the in-memory URL list from `IntegratedTool`; no database calls are made within this service.
