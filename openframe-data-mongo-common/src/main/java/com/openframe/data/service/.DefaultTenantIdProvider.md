Fallback implementation of `TenantIdProvider` that resolves the tenant identifier from the `TENANT_ID` environment variable, defaulting to `"oss"` when the variable is not set.

## Key Components

| Element | Description |
|---|---|
| `DefaultTenantIdProvider` | Spring `@Service` implementing `TenantIdProvider` |
| `getTenantId()` | Returns the resolved tenant ID string |
| `@ConditionalOnMissingBean` | Only registers this bean if no other `TenantIdProvider` is present in the context |
| `TENANT_ID` env var | Configures the tenant ID; defaults to `"oss"` if absent |

## Usage Example

```java
// Automatic fallback — no custom provider registered
// application resolves tenant ID from environment:
// export TENANT_ID=acme-corp

@Autowired
private TenantIdProvider tenantIdProvider;

String tenantId = tenantIdProvider.getTenantId(); // returns "acme-corp" or "oss"
```

To override this default, define a custom bean implementing `TenantIdProvider`:

```java
@Service
public class CustomTenantIdProvider implements TenantIdProvider {
    @Override
    public String getTenantId() {
        return SecurityContextHolder.getContext()
                                   .getAuthentication()
                                   .getTenantId();
    }
}
// DefaultTenantIdProvider is skipped automatically
```

> **OSS default:** When no `TENANT_ID` is configured, the platform operates under the `"oss"` tenant, suitable for self-hosted / open-source deployments via [OpenFrame](https://openframe.ai).
