Fallback implementation of `TenantIdProvider` that resolves the tenant ID from the `TENANT_ID` environment variable, defaulting to `"oss"` when not set.

## Key Components

| Element | Description |
|---------|-------------|
| `DefaultTenantIdProvider` | Spring `@Service` implementing `TenantIdProvider`; active only when no other `TenantIdProvider` bean is registered |
| `@ConditionalOnMissingBean` | Ensures this bean is skipped if a custom `TenantIdProvider` is already defined in the application context |
| `tenantId` | Injected from `TENANT_ID` env var or Spring property; falls back to `"oss"` (open-source default) |
| `getTenantId()` | Returns the resolved tenant identifier string |

## Usage Example

```java
// Auto-registered by Spring Boot when no custom TenantIdProvider exists.
// Override by defining your own bean:

@Service
public class MyTenantIdProvider implements TenantIdProvider {
    @Override
    public String getTenantId() {
        return "acme-corp"; // custom tenant resolution logic
    }
}

// If no override is present, the default resolves via:
// TENANT_ID=my-tenant  ->  getTenantId() returns "my-tenant"
// (unset)              ->  getTenantId() returns "oss"
```

> **Note:** Set the `TENANT_ID` environment variable at deployment time to configure the active tenant for the service. In multi-tenant or cloud-managed deployments, provide a custom `TenantIdProvider` bean to replace this default entirely.
