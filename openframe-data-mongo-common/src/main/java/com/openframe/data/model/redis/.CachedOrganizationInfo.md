Lightweight DTO for caching essential organization information in Redis, designed to minimize serialization overhead and cache storage size.

## Key Components

| Element | Description |
|---|---|
| `organizationId` | Unique identifier for the organization |
| `name` | Display name of the organization |
| `serialVersionUID` | Explicit serial version for stable Java serialization |

**Lombok Annotations:**
- `@Data` ‚Äî Generates getters, setters, `equals`, `hashCode`, and `toString`
- `@NoArgsConstructor` ‚Äî Default constructor required for deserialization
- `@AllArgsConstructor` ‚Äî Convenience constructor for quick instantiation

## Usage Example

```java
// Store organization info in Redis cache
CachedOrganizationInfo orgInfo = new CachedOrganizationInfo(
    "org-123",
    "Acme IT Services"
);
redisTemplate.opsForValue().set("org:org-123", orgInfo, 1, TimeUnit.HOURS);

// Retrieve from cache
CachedOrganizationInfo cached = (CachedOrganizationInfo) redisTemplate
    .opsForValue()
    .get("org:org-123");

String orgName = cached.getName(); // "Acme IT Services"
```

## Notes

- Implements `Serializable` for compatibility with Redis serialization strategies
- Intentionally minimal ‚Äî only includes fields necessary for common lookup operations, avoiding the overhead of serializing full organization entity graphs
- Should be kept in sync with the canonical organization entity when new cacheable fields are required
