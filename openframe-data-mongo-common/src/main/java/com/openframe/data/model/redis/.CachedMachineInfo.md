Lightweight DTO for caching essential machine information in Redis, designed to minimize serialization overhead and reduce cache footprint.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `machineId` | `String` | Unique identifier for the machine |
| `hostname` | `String` | Machine hostname |
| `organizationId` | `String` | Associated organization identifier |
| `serialVersionUID` | `long` | Ensures stable Java serialization across versions |

Lombok annotations (`@Data`, `@NoArgsConstructor`, `@AllArgsConstructor`) auto-generate getters, setters, constructors, `equals`, `hashCode`, and `toString`.

## Usage Example

```java
// Construct via all-args constructor
CachedMachineInfo machineInfo = new CachedMachineInfo(
    "machine-uuid-123",
    "workstation-01.corp.local",
    "org-456"
);

// Store in Redis (e.g., via RedisTemplate)
redisTemplate.opsForValue().set(
    "machine:" + machineInfo.getMachineId(),
    machineInfo,
    Duration.ofMinutes(30)
);

// Retrieve and access fields
CachedMachineInfo cached = (CachedMachineInfo) redisTemplate
    .opsForValue()
    .get("machine:machine-uuid-123");

String orgId = cached.getOrganizationId();
```

> **Design note:** This DTO intentionally exposes only three fields rather than mirroring the full machine entity. This keeps the Redis cache lean and avoids serialization failures caused by complex nested objects or JPA-managed relationships.
