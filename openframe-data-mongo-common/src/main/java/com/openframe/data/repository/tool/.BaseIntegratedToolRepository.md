Generic repository interface for querying integrated tool records, supporting both blocking (`Optional`) and reactive (`Mono`) implementations through parameterized return types.

## Key Components

| Element | Description |
|---------|-------------|
| `BaseIntegratedToolRepository<T, B, ID>` | Generic interface parameterized by return wrapper type, boolean type, and ID type |
| `findByType(String type)` | Looks up an integrated tool by its type identifier |
| `findByKey(String key)` | Looks up an integrated tool by its human-readable key (e.g. `"fleetmdm-server"`); tenant context is auto-injected via `TenantAwareMongoTemplate` |

## Usage Example

```java
// Blocking (non-reactive) implementation
public interface IntegratedToolRepository
    extends BaseIntegratedToolRepository<Optional<IntegratedTool>, Boolean, String> {
    // findByType and findByKey return Optional<IntegratedTool>
}

// Reactive implementation
public interface ReactiveIntegratedToolRepository
    extends BaseIntegratedToolRepository<Mono<IntegratedTool>, Mono<Boolean>, String> {
    // findByType and findByKey return Mono<IntegratedTool>
}

// Service usage (blocking)
Optional<IntegratedTool> tool = repository.findByKey("fleetmdm-server");
tool.ifPresent(t -> log.info("Found tool: {}", t.getType()));

// Service usage (reactive)
reactiveRepository.findByType("mdm")
    .doOnNext(t -> log.info("Found tool: {}", t.getKey()))
    .subscribe();
```

> **Note:** Tenant isolation is handled automatically — `TenantAwareMongoTemplate` injects the current tenant's ID at query time, so no manual tenant filtering is required in service-layer code.
