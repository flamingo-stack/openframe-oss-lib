Generic base repository interface for integrated tool persistence operations, designed to support both reactive and blocking implementations through type parameters.

## Key Components

| Component | Description |
|-----------|-------------|
| `BaseIntegratedToolRepository<T, B, ID>` | Generic interface parameterized for blocking or reactive return types |
| `T` | Return type wrapper — `Optional<IntegratedTool>` (blocking) or `Mono<IntegratedTool>` (reactive) |
| `B` | Boolean return type — `boolean` (blocking) or `Mono<Boolean>` (reactive) |
| `ID` | Identifier type, typically `String` |
| `findByType(String type)` | Locates an integrated tool record by its tool type string |

## Usage Example

**Blocking implementation:**

```java
public interface IntegratedToolRepository
    extends BaseIntegratedToolRepository<Optional<IntegratedTool>, Boolean, String> {

    @Override
    Optional<IntegratedTool> findByType(String type);
}
```

**Reactive implementation:**

```java
public interface ReactiveIntegratedToolRepository
    extends BaseIntegratedToolRepository<Mono<IntegratedTool>, Mono<Boolean>, String> {

    @Override
    Mono<IntegratedTool> findByType(String type);
}
```

**Usage in a service:**

```java
@Service
public class ToolService {
    private final IntegratedToolRepository repository;

    public Optional<IntegratedTool> getByType(String type) {
        return repository.findByType(type); // e.g., "connectwise", "halo"
    }
}
```

> **Design note:** The `B` type parameter is reserved for future boolean-returning operations (e.g., `existsByType`), keeping the interface extensible without breaking existing implementations.
