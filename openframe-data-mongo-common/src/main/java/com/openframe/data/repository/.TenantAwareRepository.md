Composed annotation that marks Spring Data repository interfaces as tenant-scoped, combining `@Repository` registration with a filter marker that enables shared-service exclusion.

## Key Components

- **`@TenantAwareRepository`** — Meta-annotation applied at the type level (`ElementType.TYPE`) with runtime retention. Inherits `@Repository` for Spring component registration and serves as a filter target to exclude tenant-scoped repositories from shared-service contexts.

## Usage Example

**Annotating a tenant-aware repository:**

```java
@TenantAwareRepository
public interface DeviceRepository extends MongoRepository<Device, String> {
    List<Device> findByTenantId(String tenantId);
}
```

**Excluding tenant repositories in a shared service configuration:**

```java
@Configuration
@EnableMongoRepositories(
    excludeFilters = @ComponentScan.Filter(
        type = FilterType.ANNOTATION,
        classes = TenantAwareRepository.class
    )
)
public class SharedServiceMongoConfig { }
```

## Notes

- Only valid in contexts where `TenantAwareMongoTemplate` is available (`tenant-isolation.enabled=true`)
- Repositories without this annotation are loaded in all contexts (shared + tenant)
- Repositories **with** this annotation are silently excluded from shared-service bootstrap, preventing missing-bean errors at startup
