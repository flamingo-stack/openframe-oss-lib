Composed annotation that marks Spring Data repository interfaces as tenant-scoped, combining `@Repository` registration with a marker for automatic exclusion from shared-service component scanning.

## Key Components

- **`@TenantAwareRepository`** — Meta-annotation applied at the type level (`ElementType.TYPE`), retained at runtime (`RetentionPolicy.RUNTIME`), and composed with Spring's `@Repository` stereotype.

## Usage Example

**Annotating a tenant-scoped repository:**

```java
@TenantAwareRepository
public interface DeviceRepository extends MongoRepository<Device, String> {
    List<Device> findByTenantId(String tenantId);
}
```

**Excluding tenant repositories in shared-service configuration:**

```java
@EnableMongoRepositories(
    excludeFilters = @ComponentScan.Filter(
        type = FilterType.ANNOTATION,
        classes = TenantAwareRepository.class
    )
)
@Configuration
public class SharedServiceMongoConfig { }
```

## Behavior Notes

- Repositories annotated with `@TenantAwareRepository` are **only valid** when `TenantAwareMongoTemplate` is available (i.e., `tenant-isolation.enabled=true`).
- Shared services **must** exclude these repositories via `@EnableMongoRepositories(excludeFilters = ...)` to avoid autowiring failures when the tenant-scoped `MongoTemplate` bean is absent.
- Acts as a purely declarative marker — no runtime processing logic is embedded in the annotation itself.
