Generic base interface for tenant repository operations, providing a technology-agnostic contract that supports both blocking and reactive implementations.

## Key Components

| Member | Description |
|---|---|
| `BaseTenantRepository<T, B, ID>` | Generic interface with three type parameters for flexible implementation |
| `findByDomain(String domain)` | Locates a tenant by domain, returning `T` (e.g., `Optional<Tenant>` or `Mono<Tenant>`) |
| `existsByDomain(String domain)` | Checks domain existence, returning `B` (e.g., `boolean` or `Mono<Boolean>`) |

**Type Parameters:**

| Parameter | Blocking | Reactive |
|---|---|---|
| `T` | `Optional<Tenant>` | `Mono<Tenant>` |
| `B` | `boolean` | `Mono<Boolean>` |
| `ID` | `String` | `String` |

## Usage Example

```java
// Blocking implementation
public interface TenantRepository
    extends BaseTenantRepository<Optional<Tenant>, Boolean, String> {
    // JPA / standard blocking repository
}

// Reactive implementation
public interface ReactiveTenantRepository
    extends BaseTenantRepository<Mono<Tenant>, Mono<Boolean>, String> {
    // R2DBC / reactive repository
}

// Usage in a service
Optional<Tenant> tenant = tenantRepository.findByDomain("acme.flamingo.run");
boolean exists = tenantRepository.existsByDomain("acme.flamingo.run");

// Reactive usage
Mono<Tenant> tenantMono = reactiveTenantRepository.findByDomain("acme.flamingo.run");
Mono<Boolean> existsMono = reactiveTenantRepository.existsByDomain("acme.flamingo.run");
```

> This interface enforces a consistent tenant lookup contract across the OpenFrame data layer, decoupling service logic from the underlying persistence strategy (blocking vs. reactive).
