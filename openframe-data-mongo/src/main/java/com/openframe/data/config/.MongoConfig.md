Spring Data MongoDB configuration class that provides conditional setup for both traditional blocking and reactive MongoDB repositories with custom converter configuration.

## Key Components

- **MongoConfiguration**: Inner configuration class that enables standard MongoDB repositories when the `spring.data.mongodb.enabled` property is set to true
- **ReactiveMongoConfiguration**: Inner configuration class that enables reactive MongoDB repositories for reactive web applications
- **mappingMongoConverter**: Custom bean that configures MongoDB document conversion with dot key replacement for handling nested field names

## Usage Example

```java
// Application properties to enable MongoDB
spring.data.mongodb.enabled=true
spring.data.mongodb.uri=mongodb://localhost:27017/openframe

// Repository usage (automatically enabled by configuration)
@Repository
public interface UserRepository extends MongoRepository<User, String> {
    List<User> findByEmail(String email);
}

// Reactive repository usage (enabled for reactive applications)
@Repository
public interface ReactiveUserRepository extends ReactiveMongoRepository<User, String> {
    Flux<User> findByEmail(String email);
}

// Documents with dot-containing keys are handled automatically
@Document
public class Config {
    private Map<String, Object> settings; // Keys like "api.key" become "api__dot__key"
}
```

The configuration automatically detects the application type and enables the appropriate MongoDB repository support, with conditional activation based on properties and application context.
