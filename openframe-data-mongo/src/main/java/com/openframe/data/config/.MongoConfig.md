Configuration class that sets up MongoDB integration for both traditional and reactive Spring Boot applications with conditional activation based on properties.

## Key Components

- **MongoConfiguration**: Inner configuration class that enables traditional MongoDB repositories when `spring.data.mongodb.enabled=true`
- **ReactiveMongoConfiguration**: Inner configuration class that enables reactive MongoDB repositories for WebFlux applications
- **mappingMongoConverter()**: Bean that creates a custom MongoDB converter with dot replacement for map keys
- **@EnableMongoAuditing**: Enables automatic auditing features for MongoDB entities
- **@EnableMongoRepositories**: Scans for traditional MongoDB repositories in `com.openframe.data.repository`
- **@EnableReactiveMongoRepositories**: Scans for reactive MongoDB repositories in `com.openframe.data.reactive.repository`

## Usage Example

```java
// Application properties to enable MongoDB
spring.data.mongodb.enabled=true
spring.data.mongodb.uri=mongodb://localhost:27017/mydb

// The configuration automatically enables repositories
@Repository
public interface UserRepository extends MongoRepository<User, String> {
    List<User> findByEmail(String email);
}

// For reactive applications
@Repository
public interface ReactiveUserRepository extends ReactiveMongoRepository<User, String> {
    Flux<User> findByEmail(String email);
}
```

The configuration handles dot character replacement in map keys (converts dots to `__dot__`) to comply with MongoDB field naming restrictions.
