Configuration class that automatically creates MongoDB indexes for the application_events collection during Spring application startup to optimize query performance.

## Key Components

- **MongoIndexConfig**: Spring configuration class that manages MongoDB index creation
- **initIndexes()**: Method annotated with `@PostConstruct` that runs after bean initialization to create compound indexes
- **mongoTemplate**: Injected MongoTemplate for performing index operations

## Usage Example

The indexes are created automatically when the Spring application starts. The configuration creates two compound indexes:

```java
// Index 1: Optimizes queries filtering by userId and sorting by timestamp
db.application_events.find({"userId": "user123"}).sort({"timestamp": -1})

// Index 2: Optimizes queries filtering by event type and metadata tags
db.application_events.find({"type": "LOGIN", "metadata.tags": "security"})
```

To add custom indexes, extend the `initIndexes()` method:

```java
@PostConstruct
public void initIndexes() {
    // Existing indexes...
    
    mongoTemplate.indexOps("application_events")
        .ensureIndex(new Index().on("status", Sort.Direction.ASC)
                              .sparse());
}
```

The `ensureIndex()` method is idempotent - it only creates indexes that don't already exist, making it safe for application restarts.
