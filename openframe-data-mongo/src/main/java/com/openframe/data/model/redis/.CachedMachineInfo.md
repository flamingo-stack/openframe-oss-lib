A lightweight Data Transfer Object (DTO) designed specifically for caching machine information in Redis, containing only essential fields to optimize serialization and reduce cache size.

## Key Components

- **CachedMachineInfo** - Main DTO class implementing `Serializable` for Redis caching
- **machineId** - Unique identifier for the machine
- **hostname** - Network hostname of the machine
- **organizationId** - Organization context for multi-tenant environments
- **Lombok annotations** - `@Data`, `@NoArgsConstructor`, `@AllArgsConstructor` for boilerplate code generation

## Usage Example

```java
// Creating a cached machine info object
CachedMachineInfo machineInfo = new CachedMachineInfo(
    "machine-001",
    "web-server-1.example.com", 
    "org-123"
);

// Or using the no-args constructor
CachedMachineInfo machineInfo = new CachedMachineInfo();
machineInfo.setMachineId("machine-001");
machineInfo.setHostname("web-server-1.example.com");
machineInfo.setOrganizationId("org-123");

// This object can be directly serialized to Redis
redisTemplate.opsForValue().set("machine:" + machineInfo.getMachineId(), machineInfo);
```

This class is optimized for Redis caching scenarios where you need quick access to basic machine information without the overhead of full machine objects. The `Serializable` implementation ensures compatibility with Redis serialization mechanisms.
