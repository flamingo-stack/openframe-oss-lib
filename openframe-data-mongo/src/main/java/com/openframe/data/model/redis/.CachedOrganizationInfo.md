A lightweight data transfer object optimized for Redis caching that stores essential organization information while minimizing serialization overhead and cache memory usage.

## Key Components

- **organizationId**: Unique identifier for the organization
- **name**: Organization display name
- **Serializable interface**: Enables Redis serialization/deserialization
- **Lombok annotations**: Auto-generates constructors, getters, setters, and utility methods

## Usage Example

```java
// Create cached organization info
CachedOrganizationInfo orgInfo = new CachedOrganizationInfo(
    "org-123", 
    "Acme Corporation"
);

// Store in Redis cache
redisTemplate.opsForValue().set("org:org-123", orgInfo);

// Retrieve from Redis cache
CachedOrganizationInfo cached = (CachedOrganizationInfo) 
    redisTemplate.opsForValue().get("org:org-123");

// Access organization data
String orgName = cached.getName();
String orgId = cached.getOrganizationId();
```

This class is designed specifically for caching scenarios where only core organization data is needed, reducing both network overhead and Redis memory consumption compared to caching full organization entities.
