A custom runtime exception thrown when an API key lookup fails in the OpenFrame data access layer. This exception provides specific error messaging for different API key lookup scenarios.

## Key Components

- **ApiKeyNotFoundException** - Extends `RuntimeException` for unchecked exception handling
- **Constructor overloads** - Three constructors for different error contexts:
  - `keyId` only - Basic key not found scenario
  - `keyId` and `userId` - User-specific key lookup failure
  - `message` and `cause` - General exception with underlying cause

## Usage Example

```java
// Basic key not found
throw new ApiKeyNotFoundException("api-key-123");

// User-specific key not found
throw new ApiKeyNotFoundException("api-key-123", "user-456");

// With underlying cause
try {
    // database lookup
} catch (SQLException e) {
    throw new ApiKeyNotFoundException("Database error during key lookup", e);
}

// Catching the exception
try {
    apiKeyService.validateKey(keyId);
} catch (ApiKeyNotFoundException e) {
    logger.error("API key validation failed: {}", e.getMessage());
    return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
}
```

This exception is typically thrown by API key validation services or data access objects when authentication credentials cannot be located in the system.
