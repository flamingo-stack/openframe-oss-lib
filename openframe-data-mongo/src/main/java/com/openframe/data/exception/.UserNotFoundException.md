A custom runtime exception that is thrown when a user lookup operation fails to find a user with the specified identifier. This exception provides a clear, descriptive error message for user-related data access failures.

## Key Components

- **UserNotFoundException** - Custom runtime exception class extending `RuntimeException`
- **Constructor** - Accepts a `userId` parameter and formats a descriptive error message

## Usage Example

```java
// Service method that throws the exception
public User findUserById(String userId) {
    User user = userRepository.findById(userId);
    if (user == null) {
        throw new UserNotFoundException(userId);
    }
    return user;
}

// Handling the exception
try {
    User user = userService.findUserById("user123");
} catch (UserNotFoundException e) {
    logger.error("Failed to find user: " + e.getMessage());
    // Handle user not found scenario
}
```

This exception is typically used in service layers, repositories, or data access components when user queries return no results, providing a more specific alternative to generic exceptions.
