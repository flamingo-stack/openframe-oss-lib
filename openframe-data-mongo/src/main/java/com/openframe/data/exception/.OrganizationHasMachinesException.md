A custom runtime exception that prevents deletion of organizations when they still have associated machines, ensuring referential integrity in the system.

## Key Components

- **OrganizationHasMachinesException** - Custom runtime exception class
- **Constructor (organizationId)** - Creates exception with basic message including organization ID
- **Constructor (organizationId, machineCount)** - Creates exception with detailed message including machine count

## Usage Example

```java
// Basic usage when machine count is unknown
if (organizationService.hasMachines(organizationId)) {
    throw new OrganizationHasMachinesException(organizationId);
}

// Detailed usage with specific machine count
long machineCount = machineRepository.countByOrganizationId(organizationId);
if (machineCount > 0) {
    throw new OrganizationHasMachinesException(organizationId, machineCount);
}

// Exception handling in service layer
try {
    organizationService.deleteOrganization(orgId);
} catch (OrganizationHasMachinesException e) {
    logger.warn("Deletion blocked: {}", e.getMessage());
    return ResponseEntity.badRequest().body(e.getMessage());
}
```

This exception follows the fail-fast principle by immediately preventing invalid operations, making it clear to developers and users why an organization deletion was rejected.
