A custom exception thrown to prevent deletion of organizations that have associated machines, ensuring referential integrity in the system.

## Key Components

- **OrganizationHasMachinesException** - Main exception class extending RuntimeException
- **Constructor with organizationId** - Creates exception with basic organization identifier
- **Constructor with organizationId and machineCount** - Creates exception with detailed machine count information

## Usage Example

```java
// Basic usage when checking for associated machines
if (hasAssociatedMachines(organizationId)) {
    throw new OrganizationHasMachinesException(organizationId);
}

// Usage with specific machine count for detailed error reporting
long machineCount = getMachineCount(organizationId);
if (machineCount > 0) {
    throw new OrganizationHasMachinesException(organizationId, machineCount);
}

// Handling the exception
try {
    organizationService.deleteOrganization("org-123");
} catch (OrganizationHasMachinesException e) {
    logger.error("Cannot delete organization: " + e.getMessage());
    return ResponseEntity.badRequest().body(e.getMessage());
}
```

This exception helps maintain data consistency by preventing cascading deletion issues and provides clear error messages for both logging and user feedback.
