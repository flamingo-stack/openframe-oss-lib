A Spring service that manages OpenFrame client configuration data with CRUD operations and default configuration handling.

## Key Components

- **DEFAULT_ID**: Constant string "default" used as the standard configuration identifier
- **get()**: Retrieves the default client configuration, throwing an exception if not found
- **findById()**: Optional-based lookup for configurations by ID
- **save()**: Persists configuration changes to the repository

## Usage Example

```java
@Autowired
private OpenFrameClientConfigurationService configService;

// Get the default configuration (throws exception if missing)
OpenFrameClientConfiguration defaultConfig = configService.get();

// Safely check for a specific configuration
Optional<OpenFrameClientConfiguration> config = configService.findById("tenant-123");
if (config.isPresent()) {
    // Process configuration
    processConfig(config.get());
}

// Create or update configuration
OpenFrameClientConfiguration newConfig = new OpenFrameClientConfiguration();
newConfig.setId("tenant-456");
configService.save(newConfig);
```

The service acts as a business logic layer between controllers and the MongoDB repository, providing a clean API for managing client configurations across different tenants or deployment contexts.
