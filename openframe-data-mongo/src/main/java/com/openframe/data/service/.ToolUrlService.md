Service class that provides utility methods for retrieving specific URLs from integrated tools based on URL type filtering.

## Key Components

- **getUrlByToolType()** - Finds the first `ToolUrl` matching a specific `ToolUrlType` within an `IntegratedTool`

## Usage Example

```java
@Autowired
private ToolUrlService toolUrlService;

public void processToolUrl(IntegratedTool tool) {
    // Get the primary URL for a tool
    Optional<ToolUrl> primaryUrl = toolUrlService.getUrlByToolType(tool, ToolUrlType.PRIMARY);
    
    if (primaryUrl.isPresent()) {
        String url = primaryUrl.get().getUrl();
        // Process the URL
    }
    
    // Get documentation URL
    Optional<ToolUrl> docsUrl = toolUrlService.getUrlByToolType(tool, ToolUrlType.DOCUMENTATION);
    docsUrl.ifPresent(url -> {
        // Handle documentation URL
        String docLink = url.getUrl();
    });
}
```

The service acts as a convenient utility for filtering tool URLs by type, returning the first match or an empty Optional if no URL of the specified type exists. This is useful for retrieving specific types of URLs (like primary, documentation, or API endpoints) from tools that may have multiple associated URLs.
