A Spring service that manages integrated tools through CRUD operations and repository interactions. This service is conditionally enabled based on configuration properties and web application type.

## Key Components

**Main Class:**
- `IntegratedToolService` - Core service class with conditional activation

**Repository Dependencies:**
- `IntegratedToolRepository` - Data access layer for IntegratedTool entities

**Core Methods:**
- `getAllTools()` - Retrieves all available tools
- `getTool(String toolType)` - Finds tool by type identifier
- `getToolById(String toolId)` - Finds tool by unique ID
- `saveTool(IntegratedTool tool)` - Persists or updates a tool

**Conditional Activation:**
- Enabled when `openframe.integration.tool.enabled=true`
- Only active in servlet-based web applications

## Usage Example

```java
@Autowired
private IntegratedToolService toolService;

// Get all available tools
List<IntegratedTool> allTools = toolService.getAllTools();

// Find specific tool by type
Optional<IntegratedTool> analyticsTool = toolService.getTool("analytics");

// Find tool by ID
Optional<IntegratedTool> tool = toolService.getToolById("tool-123");

// Save or update a tool
IntegratedTool newTool = new IntegratedTool();
newTool.setType("monitoring");
IntegratedTool saved = toolService.saveTool(newTool);
```

The service provides a clean abstraction layer for tool management operations with built-in logging and conditional deployment support.
