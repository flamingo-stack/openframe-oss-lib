A MongoDB document entity representing an organization/company in the system with business information, contract details, and soft delete capabilities.

## Key Components

- **Entity Fields**: Core organization data including `name`, `organizationId` (unique UUID), `category`, `numberOfEmployees`, `websiteUrl`, and financial information like `monthlyRevenue`
- **Contract Management**: `contractStartDate`, `contractEndDate` fields with `isContractActive()` method to check current contract status
- **Contact Information**: Embedded `ContactInformation` object for addresses and contacts
- **Soft Delete**: `deleted` flag and `deletedAt` timestamp with `isDeleted()` helper method
- **Auditing**: Automatic `createdAt` and `updatedAt` timestamps via Spring Data annotations
- **Indexing**: Strategic MongoDB indexes on `name`, `organizationId` (unique), `isDefault`, and `deleted` fields

## Usage Example

```java
// Create new organization
Organization org = Organization.builder()
    .name("Acme Corp")
    .organizationId(UUID.randomUUID().toString())
    .category("Technology")
    .numberOfEmployees(50)
    .monthlyRevenue(new BigDecimal("100000"))
    .contractStartDate(LocalDate.now())
    .contractEndDate(LocalDate.now().plusYears(1))
    .isDefault(false)
    .build();

// Check contract status
if (org.isContractActive()) {
    // Process active organization
}

// Soft delete
org.setDeleted(true);
org.setDeletedAt(Instant.now());
```
