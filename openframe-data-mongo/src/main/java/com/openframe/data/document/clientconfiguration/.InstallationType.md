Simple enumeration defining the types of installations supported by the OpenFrame client configuration system.

## Key Components

- **STANDARD**: Represents a standard command-line or server-based installation
- **GUI_APP**: Represents a graphical user interface application installation

## Usage Example

```java
// Setting installation type for client configuration
InstallationType type = InstallationType.GUI_APP;

// Using in conditional logic
if (clientConfig.getInstallationType() == InstallationType.STANDARD) {
    // Configure for headless/CLI environment
    setupCommandLineInterface();
} else if (clientConfig.getInstallationType() == InstallationType.GUI_APP) {
    // Configure for GUI environment
    setupGraphicalInterface();
}

// Switch statement usage
switch (installationType) {
    case STANDARD:
        return new StandardInstallationHandler();
    case GUI_APP:
        return new GuiInstallationHandler();
    default:
        throw new UnsupportedOperationException("Unknown installation type");
}
```

This enum is typically used within the client configuration document structure to specify how the OpenFrame client should be installed and configured, allowing the system to apply appropriate settings and behaviors based on the installation context.
