A MongoDB document entity representing physical devices in the OpenFrame system, tracking device metadata, status, and health information.

## Key Components

- **Device** - Main document class with Spring Data MongoDB annotations
- **@Document(collection = "devices")** - Maps to MongoDB "devices" collection
- **@Id** - MongoDB document identifier field
- **DeviceType** - Enum reference for device categorization (DESKTOP, LAPTOP, SERVER)
- **DeviceConfiguration** - Embedded configuration settings object
- **DeviceHealth** - Embedded health monitoring data object

## Core Fields

- `id` - Unique MongoDB document identifier
- `machineId` - Foreign key linking to Machine entity
- `serialNumber` - Hardware serial number for identification
- `model` - Device model information
- `osVersion` - Operating system version string
- `status` - Current device state (ACTIVE, OFFLINE, MAINTENANCE)
- `type` - Device category enum value
- `lastCheckin` - Timestamp of last communication with the device
- `configuration` - Device-specific configuration settings
- `health` - Device health metrics and monitoring data

## Usage Example

```java
@Autowired
private MongoTemplate mongoTemplate;

// Create new device
Device device = new Device();
device.setMachineId("machine-123");
device.setSerialNumber("SN123456789");
device.setModel("Dell OptiPlex 7090");
device.setOsVersion("Windows 11 Pro");
device.setStatus("ACTIVE");
device.setType(DeviceType.DESKTOP);
device.setLastCheckin(Instant.now());

mongoTemplate.save(device);

// Query devices by status
List<Device> activeDevices = mongoTemplate.find(
    Query.query(Criteria.where("status").is("ACTIVE")), 
    Device.class
);
```
