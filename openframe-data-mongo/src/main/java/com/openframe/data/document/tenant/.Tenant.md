A Spring Data document entity representing a tenant in a multi-tenant architecture system, extending CoreTenant with additional metadata and business logic for organization management.

## Key Components

**Main Class:**
- `Tenant` - Document entity with tenant-specific fields and status management

**Key Fields:**
- `ownerId` - User ID of the tenant owner
- `status` - Current tenant status (defaults to ACTIVE)
- `plan` - Subscription plan (defaults to FREE)
- `hubspotId` - Integration identifier for HubSpot
- `createdAt`/`updatedAt` - Audit timestamps

**Utility Methods:**
- `generateTenantId()` - Static method to create UUID-based tenant IDs
- `isActive()` - Convenience method to check if tenant status is ACTIVE

## Usage Example

```java
// Create a new tenant
Tenant tenant = Tenant.builder()
    .ownerId("user-123")
    .status(TenantStatus.ACTIVE)
    .plan(TenantPlan.PREMIUM)
    .hubspotId("hs-456")
    .build();

// Generate unique tenant ID
String tenantId = Tenant.generateTenantId();

// Check tenant status
if (tenant.isActive()) {
    // Process active tenant operations
}

// Update tenant with builder pattern
Tenant updatedTenant = tenant.toBuilder()
    .status(TenantStatus.SUSPENDED)
    .build();
```
