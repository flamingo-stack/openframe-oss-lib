A domain entity representing a tenant in OpenFrame's multi-tenant architecture, extending the base CoreTenant class with ownership tracking, status management, and audit capabilities.

## Key Components

- **ownerId**: String field storing the ID of the first registered user who becomes the tenant owner
- **status**: TenantStatus enum field managing tenant state (defaults to ACTIVE)
- **plan**: TenantPlan enum field for subscription management (defaults to FREE)
- **hubspotId**: String field for CRM integration tracking
- **createdAt/updatedAt**: LocalDateTime fields for audit timestamps
- **generateTenantId()**: Static utility method for creating unique tenant identifiers
- **isActive()**: Boolean method to check if tenant status is active

## Usage Example

```java
// Create a new tenant with SuperBuilder
Tenant tenant = Tenant.builder()
    .ownerId("user-123")
    .status(TenantStatus.ACTIVE)
    .plan(TenantPlan.PREMIUM)
    .hubspotId("hubspot-456")
    .build();

// Generate unique tenant ID
String tenantId = Tenant.generateTenantId();

// Check tenant status
if (tenant.isActive()) {
    // Process active tenant operations
}

// Update tenant with builder pattern
Tenant updatedTenant = tenant.toBuilder()
    .status(TenantStatus.SUSPENDED)
    .build();
```

The class uses Lombok annotations for reduced boilerplate code and Spring Data annotations for automatic timestamp management, making it suitable for NoSQL document databases in multi-tenant SaaS applications.
