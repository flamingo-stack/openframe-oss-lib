A MongoDB document entity that stores client configuration data for OpenFrame applications, including version information and download configurations with automatic timestamp tracking.

## Key Components

- **@Document** - Maps to MongoDB collection `openframe_client_configuration`
- **id** - Primary key field annotated with `@Id`
- **version** - String field storing configuration version
- **downloadConfiguration** - List of `DownloadConfiguration` objects
- **createdAt/updatedAt** - Automatic timestamp fields for audit tracking
- **Lombok annotations** - Provides boilerplate code generation (@Data, @Builder, etc.)

## Usage Example

```java
// Creating a new configuration
OpenFrameClientConfiguration config = OpenFrameClientConfiguration.builder()
    .version("1.2.0")
    .downloadConfiguration(List.of(
        new DownloadConfiguration("app", "https://example.com/app.zip"),
        new DownloadConfiguration("assets", "https://example.com/assets.zip")
    ))
    .build();

// Repository usage
@Repository
public interface ConfigRepository extends MongoRepository<OpenFrameClientConfiguration, String> {
    Optional<OpenFrameClientConfiguration> findByVersion(String version);
}

// Service layer example
@Service
public class ConfigService {
    public OpenFrameClientConfiguration getLatestConfig() {
        return configRepository.findTopByOrderByCreatedAtDesc();
    }
}
```

The timestamps (`createdAt`, `updatedAt`) are automatically managed by Spring Data MongoDB when the entity is persisted or updated.
