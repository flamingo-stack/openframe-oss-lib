MongoDB document entity that represents client configuration settings for the OpenFrame platform, managing versioned download configurations with automatic timestamping and publish state tracking.

## Key Components

- **OpenFrameClientConfiguration** - Main document entity with MongoDB collection mapping
- **downloadConfiguration** - List of `DownloadConfiguration` objects containing client settings
- **version** - Version identifier for configuration management
- **publishState** - Enum controlling configuration deployment status
- **Automatic timestamps** - `@CreatedDate` and `@LastModifiedDate` for audit tracking

## Usage Example

```java
// Create new client configuration
OpenFrameClientConfiguration config = OpenFrameClientConfiguration.builder()
    .version("1.2.0")
    .downloadConfiguration(Arrays.asList(
        // Add download configurations here
    ))
    .publishState(PublishState.DRAFT)
    .build();

// Save to MongoDB
configurationRepository.save(config);

// Query by version
Optional<OpenFrameClientConfiguration> result = 
    configurationRepository.findByVersion("1.2.0");

// Update publish state
config.setPublishState(PublishState.PUBLISHED);
configurationRepository.save(config);
```

The entity uses Lombok annotations for boilerplate reduction and Spring Data MongoDB for persistence with automatic timestamp management.
