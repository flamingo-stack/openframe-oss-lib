A MongoDB document entity representing users in the OpenFrame MSP platform system, with support for role-based access control, email verification, and automatic timestamping.

## Key Components

- **@Document(collection = "users")** - MongoDB collection mapping for user data persistence
- **id** - Unique identifier (MongoDB ObjectId)
- **email** - Indexed, normalized email address (trimmed and lowercased)
- **firstName/lastName** - User's personal information
- **roles** - List of UserRole objects for permission management
- **emailVerified** - Boolean flag for email verification status (defaults to false)
- **status** - UserStatus enum for account state management (defaults to ACTIVE)
- **createdAt/updatedAt** - Automatic timestamp fields managed by Spring Data

## Usage Example

```java
// Create a new user with builder pattern
User user = User.builder()
    .email("john.doe@company.com")
    .firstName("John")
    .lastName("Doe")
    .emailVerified(true)
    .roles(List.of(UserRole.TECHNICIAN, UserRole.USER))
    .status(UserStatus.ACTIVE)
    .build();

// Email normalization happens automatically
user.setEmail("  ADMIN@COMPANY.COM  "); // Becomes "admin@company.com"

// Check user permissions
if (user.getRoles().contains(UserRole.ADMIN)) {
    // Grant admin access
}

// Verify account status
if (user.getStatus() == UserStatus.ACTIVE && user.isEmailVerified()) {
    // Allow login
}
```
