This MongoDB document entity represents user data with role-based access control and automatic timestamping. The class uses Lombok annotations for code generation and Spring Data MongoDB for persistence operations.

## Key Components

- **User Entity**: Main document class mapped to "users" collection
- **Core Fields**: `id`, `email` (indexed/normalized), `firstName`, `lastName`  
- **Access Control**: `roles` list of `UserRole` enum values with default empty list
- **Status Management**: `status` field with `UserStatus` enum (defaults to ACTIVE)
- **Audit Fields**: `createdAt` and `updatedAt` timestamps managed automatically
- **Email Normalization**: Custom setter that trims and lowercases email input

## Usage Example

```java
// Create new user
User user = User.builder()
    .email("john.doe@example.com")
    .firstName("John")
    .lastName("Doe")
    .roles(List.of(UserRole.ADMIN, UserRole.USER))
    .build();

// Save to MongoDB
userRepository.save(user);

// Query by email (leverages index)
User foundUser = userRepository.findByEmail("john.doe@example.com");

// Update user status
foundUser.setStatus(UserStatus.INACTIVE);
userRepository.save(foundUser);
```
