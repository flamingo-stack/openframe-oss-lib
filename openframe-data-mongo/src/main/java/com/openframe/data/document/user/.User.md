A MongoDB document entity representing system users with automatic email normalization and audit timestamps.

## Key Components

- **@Document("users")** - Maps to MongoDB "users" collection
- **@Id String id** - MongoDB document identifier
- **@Indexed String email** - Indexed email field with automatic lowercase normalization
- **firstName/lastName** - User personal information fields
- **List<UserRole> roles** - User permission roles (defaults to empty list)
- **UserStatus status** - User account status (defaults to ACTIVE)
- **@CreatedDate/@LastModifiedDate** - Automatic timestamp tracking
- **Custom setEmail()** - Normalizes email to lowercase and trims whitespace

## Usage Example

```java
// Create a new user with builder pattern
User user = User.builder()
    .email("John.Doe@Example.Com")  // Will be normalized to "john.doe@example.com"
    .firstName("John")
    .lastName("Doe")
    .roles(List.of(UserRole.USER, UserRole.ADMIN))
    .status(UserStatus.ACTIVE)
    .build();

// Email normalization happens automatically
user.setEmail("  TEST@DOMAIN.COM  ");
System.out.println(user.getEmail()); // Outputs: "test@domain.com"

// Save to MongoDB (timestamps auto-populated)
userRepository.save(user);
```

The class uses Lombok annotations for boilerplate code generation and Spring Data MongoDB for persistence with automatic auditing.
