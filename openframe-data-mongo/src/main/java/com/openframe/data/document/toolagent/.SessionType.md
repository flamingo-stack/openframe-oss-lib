Simple enumeration defining the different types of sessions available in the OpenFrame tool agent system.

## Key Components

- **SERVICE** - Represents automated service-based sessions
- **CONSOLE** - Represents console/terminal-based sessions  
- **USER** - Represents user-initiated interactive sessions

## Usage Example

```java
// Creating session instances with different types
SessionType serviceSession = SessionType.SERVICE;
SessionType consoleSession = SessionType.CONSOLE;
SessionType userSession = SessionType.USER;

// Using in conditional logic
public void handleSession(SessionType type) {
    switch (type) {
        case SERVICE:
            // Handle automated service operations
            break;
        case CONSOLE:
            // Handle console/terminal operations
            break;
        case USER:
            // Handle user interactive operations
            break;
    }
}

// Comparing session types
if (currentSessionType == SessionType.USER) {
    enableInteractiveFeatures();
}
```

This enum provides type safety when categorizing different session contexts throughout the OpenFrame tool agent system, ensuring consistent handling of service automation, console operations, and user interactions.
