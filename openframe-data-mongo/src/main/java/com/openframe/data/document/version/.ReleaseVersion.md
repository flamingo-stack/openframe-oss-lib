A MongoDB document entity that stores application release version information with automatic timestamp tracking.

## Key Components

- **@Document(collection = "release_versions")** - Maps to MongoDB collection for version storage
- **version** - String field with database index for version identifiers (e.g., "4.2.0", "1.0.0-beta")
- **createdAt/updatedAt** - Automatic timestamp fields managed by Spring Data MongoDB
- **@Data** - Lombok annotation providing getters, setters, equals, hashCode, and toString methods

## Usage Example

```java
// Create a new release version
ReleaseVersion release = new ReleaseVersion();
release.setVersion("4.2.0");

// Save to MongoDB (timestamps auto-populated)
releaseVersionRepository.save(release);

// Query by version (uses indexed field)
Optional<ReleaseVersion> found = releaseVersionRepository.findByVersion("4.2.0");

// Version will have auto-generated ID and timestamps
System.out.println("Version: " + found.get().getVersion());
System.out.println("Created: " + found.get().getCreatedAt());
```

The indexed `version` field enables efficient lookups, while Spring Data MongoDB automatically manages the audit timestamps for tracking version lifecycle events.
