MongoDB document entity for storing application release version information with automatic timestamp tracking.

## Key Components

- **ReleaseVersion**: Main entity class representing a release version document
- **version**: Indexed string field for version identifiers (e.g., "4.2.0", "1.0.0-beta")
- **createdAt/updatedAt**: Automatic timestamp fields managed by Spring Data MongoDB
- **id**: MongoDB document identifier

## Usage Example

```java
// Creating a new release version
ReleaseVersion release = new ReleaseVersion();
release.setVersion("4.2.0");

// Save to MongoDB (timestamps auto-populated)
repository.save(release);

// Querying by version (leverages indexed field)
ReleaseVersion found = repository.findByVersion("4.2.0");

// Version comparison
List<ReleaseVersion> versions = repository.findAllByOrderByCreatedAtDesc();
```

The entity uses Lombok's `@Data` annotation for automatic getter/setter generation and Spring Data MongoDB annotations for collection mapping and automatic auditing. The indexed version field enables efficient querying of specific releases.
