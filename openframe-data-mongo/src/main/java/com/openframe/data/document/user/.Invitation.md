A MongoDB document entity that represents user invitations to the OpenFrame platform, handling invitation lifecycle management with automatic email normalization and role assignment.

## Key Components

- **Primary Fields**: `id`, `email` (indexed), `roles`, `expiresAt`, `status`
- **Audit Fields**: `createdAt`, `updatedAt` with automatic timestamping
- **Email Normalization**: Custom setter that trims and converts email to lowercase
- **Default Values**: Empty roles list and PENDING status via `@Builder.Default`
- **MongoDB Integration**: Uses Spring Data MongoDB annotations for collection mapping

## Usage Example

```java
// Create a new invitation
Invitation invitation = Invitation.builder()
    .email("  USER@EXAMPLE.COM  ")  // Will be normalized to "user@example.com"
    .roles(List.of(UserRole.USER, UserRole.VIEWER))
    .expiresAt(Instant.now().plusSeconds(86400 * 7)) // 7 days
    .build();

// Check invitation status
if (invitation.getStatus() == InvitationStatus.PENDING) {
    // Process pending invitation
    System.out.println("Invitation expires at: " + invitation.getExpiresAt());
}

// Update invitation status
invitation.setStatus(InvitationStatus.ACCEPTED);
```

The entity leverages Lombok annotations for boilerplate reduction and Spring Data MongoDB for persistence, with built-in email sanitization ensuring data consistency across the platform.
