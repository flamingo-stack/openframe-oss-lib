Data model class that encapsulates contact information for organizations, including contact persons and address details. This is designed as an embedded document rather than a standalone collection.

## Key Components

- **ContactInformation**: Main data class with Lombok annotations for automatic getter/setter generation
- **contacts**: List of ContactPerson objects representing organization contacts
- **physicalAddress**: Address object for the organization's physical location
- **mailingAddress**: Address object for mailing purposes
- **mailingAddressSameAsPhysical**: Boolean flag to indicate address relationship

## Usage Example

```java
// Create contact information with builder pattern
ContactInformation contactInfo = ContactInformation.builder()
    .physicalAddress(Address.builder()
        .street("123 Main St")
        .city("Springfield")
        .build())
    .mailingAddressSameAsPhysical(true)
    .contacts(List.of(
        ContactPerson.builder()
            .name("John Doe")
            .email("john@company.com")
            .build()
    ))
    .build();

// Add additional contacts
contactInfo.getContacts().add(new ContactPerson("Jane Smith", "jane@company.com"));

// Check if mailing address is same as physical
if (contactInfo.getMailingAddressSameAsPhysical()) {
    // Use physical address for mailing
}
```
