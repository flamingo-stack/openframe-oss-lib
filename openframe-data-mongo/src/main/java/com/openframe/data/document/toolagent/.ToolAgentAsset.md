Represents an asset configuration for tool agents in the OpenFrame platform, defining downloadable resources and their deployment characteristics.

## Key Components

- **Core Identity**: `id` and `version` fields uniquely identify the asset
- **Download Management**: `downloadConfigurations` list specifies how to obtain the asset from various sources
- **Source Configuration**: `source` defines the origin type and location of the asset
- **File Handling**: `localFilenameConfiguration` manages local file naming and organization
- **Deployment Properties**: `path` specifies installation location and `executable` flag indicates if the asset should have execution permissions

## Usage Example

```java
ToolAgentAsset asset = new ToolAgentAsset();
asset.setId("monitoring-tool");
asset.setVersion("2.1.0");
asset.setPath("/opt/tools/monitoring");
asset.setExecutable(true);

// Configure download options
List<DownloadConfiguration> downloads = Arrays.asList(
    createWindowsDownloadConfig(),
    createLinuxDownloadConfig()
);
asset.setDownloadConfigurations(downloads);

// Set source information
ToolAgentAssetSource source = new ToolAgentAssetSource();
source.setType("github");
source.setRepository("openframe/monitoring-tools");
asset.setSource(source);
```

This class serves as a configuration blueprint for automated tool deployment, enabling the OpenFrame system to download, install, and configure various utilities across different environments while maintaining version control and proper file permissions.
