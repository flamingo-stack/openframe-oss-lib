A MongoDB document entity that represents OAuth 2.0 registered clients for Spring Security integration, storing client configurations and authentication settings.

## Key Components

- **MongoRegisteredClient**: Main document class mapped to `oauth_registered_clients` collection
- **Client Identifiers**: `id` (MongoDB document ID) and `clientId` (unique OAuth client identifier)
- **Authentication Configuration**: Client secret, supported authentication methods and grant types
- **Authorization Settings**: Redirect URIs, scopes, PKCE and consent requirements
- **Token Configuration**: Access/refresh token TTL settings and reuse policies

## Usage Example

```java
@Repository
public class OAuth2ClientRepository {
    
    @Autowired
    private MongoTemplate mongoTemplate;
    
    public MongoRegisteredClient createClient() {
        return MongoRegisteredClient.builder()
            .clientId("my-oauth-client")
            .clientSecret("encoded-secret")
            .authenticationMethods(Set.of("client_secret_basic"))
            .grantTypes(Set.of("authorization_code", "refresh_token"))
            .redirectUris(Set.of("https://app.example.com/callback"))
            .scopes(Set.of("read", "write"))
            .requireProofKey(true)
            .requireAuthorizationConsent(false)
            .accessTokenTtlSeconds(3600)
            .refreshTokenTtlSeconds(86400)
            .reuseRefreshTokens(false)
            .build();
    }
    
    public MongoRegisteredClient findByClientId(String clientId) {
        Query query = Query.query(Criteria.where("clientId").is(clientId));
        return mongoTemplate.findOne(query, MongoRegisteredClient.class);
    }
}
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder patterns, making it ideal for MongoDB-based OAuth 2.0 client management systems.
