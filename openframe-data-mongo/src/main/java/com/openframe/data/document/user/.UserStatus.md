Simple enumeration defining the possible status states for users in the authorization server system.

## Key Components

- **ACTIVE**: Represents users who are currently active and can access the system
- **DELETED**: Represents users who have been marked as deleted/deactivated

## Usage Example

```java
import com.openframe.data.document.user.UserStatus;

// Setting user status
UserStatus currentStatus = UserStatus.ACTIVE;

// Checking user status in conditionals
if (user.getStatus() == UserStatus.ACTIVE) {
    // Allow user access
    processUserRequest();
} else if (user.getStatus() == UserStatus.DELETED) {
    // Deny access or redirect
    handleDeletedUser();
}

// Using in switch statements
switch (user.getStatus()) {
    case ACTIVE:
        enableUserFeatures();
        break;
    case DELETED:
        showUserDeletedMessage();
        break;
}
```

This enum provides type safety and clarity when managing user lifecycle states throughout the authorization system.
