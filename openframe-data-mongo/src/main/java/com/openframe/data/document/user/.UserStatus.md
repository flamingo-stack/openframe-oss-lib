A simple enumeration that defines the possible status values for users in the authorization server system.

## Key Components

- **ACTIVE** - Indicates a user account is currently active and operational
- **DELETED** - Indicates a user account has been marked as deleted

## Usage Example

```java
// Setting user status
User user = new User();
user.setStatus(UserStatus.ACTIVE);

// Checking user status
if (user.getStatus() == UserStatus.ACTIVE) {
    // Process active user
    processActiveUser(user);
} else if (user.getStatus() == UserStatus.DELETED) {
    // Handle deleted user
    logDeletedUserAccess(user);
}

// Switch statement usage
switch (user.getStatus()) {
    case ACTIVE:
        enableUserFeatures(user);
        break;
    case DELETED:
        restrictUserAccess(user);
        break;
}
```

This enum provides a type-safe way to manage user account states throughout the authorization system, ensuring consistent status handling across the application.
