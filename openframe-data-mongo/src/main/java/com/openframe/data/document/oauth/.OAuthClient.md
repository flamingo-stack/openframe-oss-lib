MongoDB document entity that represents OAuth 2.0 client configurations for authentication and authorization within the OpenFrame system.

## Key Components

- **@Document(collection = "oauth_clients")** - Maps to MongoDB collection for OAuth client storage
- **@Id** - MongoDB document identifier
- **clientId/clientSecret** - OAuth 2.0 client credentials for authentication
- **machineId** - Associates the OAuth client with a specific machine/device
- **redirectUris** - Valid redirect URLs for OAuth flows
- **grantTypes** - Supported OAuth 2.0 grant types (authorization_code, password, client_credentials, refresh_token)
- **scopes** - Available OAuth scopes for access control
- **roles** - User roles assigned to this client (defaults to empty array)
- **enabled** - Flag to activate/deactivate the OAuth client

## Usage Example

```java
// Create new OAuth client
OAuthClient client = new OAuthClient();
client.setClientId("webapp-client");
client.setClientSecret("secret123");
client.setMachineId("machine-001");
client.setRedirectUris(new String[]{"https://app.example.com/callback"});
client.setGrantTypes(new String[]{"authorization_code", "refresh_token"});
client.setScopes(new String[]{"read", "write", "admin"});
client.setRoles(new String[]{"USER", "ADMIN"});
client.setEnabled(true);

// Save to MongoDB (via repository)
oauthClientRepository.save(client);
```

This entity supports standard OAuth 2.0 flows and integrates with Spring Data MongoDB for persistence in the OpenFrame authentication system.
