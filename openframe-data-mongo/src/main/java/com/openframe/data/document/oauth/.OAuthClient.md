A MongoDB document model representing OAuth 2.0 client configurations for authentication and authorization within the OpenFrame system.

## Key Components

- **OAuthClient**: Main entity class annotated with `@Document` for MongoDB persistence
- **clientId/clientSecret**: Standard OAuth 2.0 client credentials for authentication
- **machineId**: Unique identifier linking the client to a specific machine or device
- **redirectUris**: Array of allowed redirect URIs for authorization flows
- **grantTypes**: Supported OAuth 2.0 grant types (authorization_code, password, client_credentials, refresh_token)
- **scopes**: Permission scopes the client can request
- **roles**: Array of assigned roles for role-based access control
- **enabled**: Flag to activate/deactivate the client

## Usage Example

```java
// Create a new OAuth client
OAuthClient client = new OAuthClient();
client.setClientId("webapp-client");
client.setClientSecret("secret123");
client.setMachineId("web-server-01");
client.setRedirectUris(new String[]{"https://app.example.com/callback"});
client.setGrantTypes(new String[]{"authorization_code", "refresh_token"});
client.setScopes(new String[]{"read", "write"});
client.setRoles(new String[]{"USER", "ADMIN"});

// Save to MongoDB repository
oauthClientRepository.save(client);
```
