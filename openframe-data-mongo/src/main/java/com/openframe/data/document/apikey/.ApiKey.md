A MongoDB document entity representing API keys with authentication, authorization, and lifecycle management capabilities.

## Key Components

**Main Class:**
- `ApiKey` - MongoDB document entity with Lombok annotations for boilerplate code generation

**Key Fields:**
- `keyId` - Unique identifier and MongoDB primary key
- `hashedKey` - Stored hash of the actual API key for security
- `userId` - Indexed field linking the key to a user
- `scopes/roles` - Permission and authorization lists
- `enabled/createdAt/updatedAt/expiresAt` - Lifecycle management fields

**Core Methods:**
- `isExpired()` - Checks if the key has passed its expiration date
- `isActive()` - Validates if the key is both enabled and not expired

## Usage Example

```java
// Create a new API key
ApiKey apiKey = ApiKey.builder()
    .keyId("key_123")
    .hashedKey("hashed_value")
    .name("Production API Key")
    .userId("user_456")
    .scopes(List.of("read", "write"))
    .roles(List.of("admin"))
    .enabled(true)
    .createdAt(Instant.now())
    .expiresAt(Instant.now().plusSeconds(86400)) // 24 hours
    .build();

// Check key status
if (apiKey.isActive()) {
    // Process API request
} else {
    // Reject - key is disabled or expired
}
```
