A simple enumeration that defines the possible security compliance states for a device within the OpenFrame system.

## Key Components

- **COMPLIANT** - Device meets all security requirements and policies
- **NON_COMPLIANT** - Device fails to meet one or more security requirements
- **UNKNOWN** - Device security state cannot be determined or is pending evaluation

## Usage Example

```java
// Setting device security state
SecurityState deviceState = SecurityState.COMPLIANT;

// Checking security compliance in conditional logic
if (device.getSecurityState() == SecurityState.NON_COMPLIANT) {
    // Trigger security remediation
    securityManager.initiateRemediation(device);
}

// Switch statement for handling different states
switch (deviceSecurityState) {
    case COMPLIANT:
        allowAccess(device);
        break;
    case NON_COMPLIANT:
        blockAccess(device);
        break;
    case UNKNOWN:
        requestSecurityEvaluation(device);
        break;
}
```
