Enumeration that defines the security compliance status of a device within the OpenFrame system. This enum is used to categorize devices based on their adherence to security policies and requirements.

## Key Components

- **COMPLIANT** - Device meets all security requirements and policies
- **NON_COMPLIANT** - Device fails to meet one or more security requirements
- **UNKNOWN** - Device security status cannot be determined or has not been evaluated

## Usage Example

```java
// Setting device security state
Device device = new Device();
device.setSecurityState(SecurityState.COMPLIANT);

// Checking security compliance
if (device.getSecurityState() == SecurityState.NON_COMPLIANT) {
    // Take remediation action
    securityService.quarantineDevice(device);
} else if (device.getSecurityState() == SecurityState.UNKNOWN) {
    // Schedule security assessment
    securityService.scheduleAssessment(device);
}

// Switch statement for handling different states
switch (device.getSecurityState()) {
    case COMPLIANT:
        allowDeviceAccess(device);
        break;
    case NON_COMPLIANT:
        restrictDeviceAccess(device);
        break;
    case UNKNOWN:
        requestSecurityScan(device);
        break;
}
```
