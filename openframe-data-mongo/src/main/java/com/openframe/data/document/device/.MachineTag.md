A MongoDB document entity that represents the many-to-many relationship between machines and tags in the OpenFrame system, with automatic indexing for efficient querying.

## Key Components

**Main Entity:**
- `MachineTag` - Document class mapping machine-tag associations
- `@CompoundIndex` - Unique composite index on `machineId` and `tagId` fields
- MongoDB collection: `machine_tags`

**Core Fields:**
- `id` - Primary key (MongoDB ObjectId)
- `machineId` - Reference to the associated machine
- `tagId` - Reference to the associated tag
- `taggedAt` - Timestamp when the tag was applied
- `taggedBy` - User who applied the tag

## Usage Example

```java
// Create a new machine-tag association
MachineTag machineTag = new MachineTag();
machineTag.setMachineId("machine-123");
machineTag.setTagId("tag-production");
machineTag.setTaggedAt(Instant.now());
machineTag.setTaggedBy("admin-user");

// The compound index ensures uniqueness
// This prevents duplicate machine-tag relationships
// Query optimization for finding tags by machine:
// db.machine_tags.find({"machineId": "machine-123"})
```

The compound index `machine_tag_idx` ensures that each machine can only have one instance of a specific tag while providing efficient lookups for both machine-to-tags and tag-to-machines queries.
