This is a MongoDB document entity representing tenant information in a multi-tenant application, using Spring Data MongoDB for persistence and Lombok for boilerplate reduction.

## Key Components

- **CoreTenant**: Main entity class with MongoDB document mapping
- **id**: Primary key field for MongoDB document identification
- **name**: Indexed tenant display name for organization identification
- **domain**: Unique indexed domain field for subdomain-based routing
- **Lombok annotations**: `@Data`, `@SuperBuilder`, `@NoArgsConstructor`, `@AllArgsConstructor` for automatic code generation

## Usage Example

```java
// Create a new tenant
CoreTenant tenant = CoreTenant.builder()
    .name("Acme Corporation")
    .domain("acme.openframe.io")
    .build();

// Update existing tenant
CoreTenant updatedTenant = existingTenant.toBuilder()
    .name("Acme Corp Updated")
    .build();

// Repository usage (typical Spring Data MongoDB)
public interface TenantRepository extends MongoRepository<CoreTenant, String> {
    Optional<CoreTenant> findByDomain(String domain);
    List<CoreTenant> findByNameContainingIgnoreCase(String name);
}
```

The unique domain constraint ensures no duplicate subdomains exist, while indexed fields enable efficient querying for tenant lookup operations.
