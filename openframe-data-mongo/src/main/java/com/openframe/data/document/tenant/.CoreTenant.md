A MongoDB document entity representing a tenant in the multi-tenant OpenFrame system, used for organization-level data isolation and subdomain-based routing.

## Key Components

- **CoreTenant** - Main entity class with MongoDB document mapping
- **id** - Primary identifier (MongoDB ObjectId)
- **name** - Organization display name with indexing for search performance
- **domain** - Unique subdomain identifier for routing (e.g., "company.openframe.io")

## Usage Example

```java
// Create a new tenant
CoreTenant tenant = CoreTenant.builder()
    .name("Acme Corporation")
    .domain("acme.openframe.io")
    .build();

// Update tenant using toBuilder
CoreTenant updatedTenant = tenant.toBuilder()
    .name("Acme Corp")
    .build();

// Access tenant properties
String tenantDomain = tenant.getDomain();
String organizationName = tenant.getName();
```

The class uses Lombok annotations for automatic generation of getters, setters, and builders. The `domain` field has a unique index to ensure no duplicate subdomains exist across tenants, while the `name` field is indexed for efficient tenant lookup operations.
