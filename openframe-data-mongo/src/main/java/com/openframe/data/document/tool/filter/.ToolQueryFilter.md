A data transfer object for filtering tool queries with criteria such as enabled status, type, and category information.

## Key Components

- **enabled** - Boolean filter for tool availability status
- **type** - String filter for tool type classification
- **category** - String filter for general tool category
- **platformCategory** - String filter for platform-specific categorization

## Usage Example

```java
// Create a filter for enabled AI tools
ToolQueryFilter filter = ToolQueryFilter.builder()
    .enabled(true)
    .type("AI")
    .category("assistant")
    .platformCategory("openai")
    .build();

// Create filter with only type constraint
ToolQueryFilter typeFilter = new ToolQueryFilter();
typeFilter.setType("database");
typeFilter.setEnabled(true);

// Use in repository or service layer
List<Tool> results = toolRepository.findByFilter(filter);
```

The class uses Lombok annotations to automatically generate boilerplate code including getters, setters, constructors, and builder pattern implementation, making it ideal for use in query operations and API request filtering.
