This MongoDB document entity represents a machine/device in the OpenFrame system, storing comprehensive device information including hardware specs, security status, and compliance data.

## Key Components

- **Machine Entity**: MongoDB document mapped to "machines" collection with Spring Data annotations
- **Device Identification**: Core fields like `machineId`, `ip`, `macAddress`, and hardware identifiers
- **System Information**: Operating system details, hostname, manufacturer, and model information
- **Security & Compliance**: Security state tracking, compliance requirements, and alert management
- **Status Tracking**: Device status, last seen timestamps, and scan history
- **Organization Mapping**: Links devices to organizations via `organizationId`

## Usage Example

```java
// Create a new machine record
Machine machine = new Machine();
machine.setMachineId("DEVICE-12345");
machine.setHostname("workstation-01");
machine.setIp("192.168.1.100");
machine.setStatus(DeviceStatus.ACTIVE);
machine.setOrganizationId("org-abc123");
machine.setType(DeviceType.WORKSTATION);
machine.setOsType("Windows");
machine.setRegisteredAt(Instant.now());

// Update security state
machine.setSecurityState(SecurityState.SECURE);
machine.setLastSecurityScan(Instant.now());

// Save to MongoDB
machineRepository.save(machine);

// Query by organization
List<Machine> orgMachines = machineRepository.findByOrganizationId("org-abc123");
```

The entity uses Lombok's `@Data` annotation for automatic getter/setter generation and includes validation constraints and MongoDB indexing for optimal query performance.
