A MongoDB document entity that represents a managed device/machine in the OpenFrame MSP platform, storing comprehensive device information including hardware details, operating system data, security status, and compliance state.

## Key Components

- **Machine Entity**: Main document class annotated with `@Document(collection = "machines")`
- **Device Identification**: Fields for `machineId`, `ip`, `macAddress`, `osUuid`, and `serialNumber`
- **Hardware Information**: Properties for `hostname`, `manufacturer`, `model`, and device `type`
- **Operating System Details**: Fields for `osType`, `osVersion`, `osBuild`, and `timezone`
- **Security & Compliance**: `SecurityState`, `ComplianceState`, security alerts, and compliance requirements
- **Audit Fields**: Automatic timestamp tracking with `@CreatedDate` and `@LastModifiedDate`
- **Indexing**: Strategic `@Indexed` annotations on `status`, `organizationId`, `type`, and `osType` for query performance

## Usage Example

```java
// Create a new machine registration
Machine machine = new Machine();
machine.setMachineId("DESKTOP-ABC123");
machine.setHostname("john-workstation");
machine.setIp("192.168.1.100");
machine.setMacAddress("AA:BB:CC:DD:EE:FF");
machine.setOsType("Windows");
machine.setOsVersion("11");
machine.setType(DeviceType.WORKSTATION);
machine.setStatus(DeviceStatus.ONLINE);
machine.setOrganizationId("org_12345");

// Security and compliance tracking
machine.setSecurityState(SecurityState.SECURE);
machine.setComplianceState(ComplianceState.COMPLIANT);
machine.setLastSeen(Instant.now());

// Save to MongoDB
machineRepository.save(machine);
```

The `machineId` field serves as both the primary identifier and OAuth client ID for device authentication, enabling secure communication between managed devices and the OpenFrame platform.
