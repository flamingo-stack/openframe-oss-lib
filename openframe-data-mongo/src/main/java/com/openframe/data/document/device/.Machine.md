A MongoDB document entity representing a machine/device in the system, containing device identification, hardware/software specifications, security status, and compliance information.

## Key Components

**Main Entity:**
- `Machine` - MongoDB document with comprehensive device tracking capabilities

**Key Fields:**
- `machineId` - Primary identifier used for authentication and device linking
- `organizationId` - Indexed field for multi-tenant organization filtering  
- `status` - Current device status (indexed for efficient queries)
- `type` - Device type classification (indexed)
- `securityState/complianceState` - Security and compliance status tracking
- `lastSeen/registeredAt/updatedAt` - Temporal tracking fields

**Security & Compliance:**
- `SecurityAlert` list for tracking security issues
- `ComplianceRequirement` list for compliance management
- Separate scan timestamps for security and compliance checks

## Usage Example

```java
// Create a new machine record
Machine machine = new Machine();
machine.setMachineId("WORKSTATION-001");
machine.setOrganizationId("org-123");
machine.setHostname("dev-laptop");
machine.setStatus(DeviceStatus.ACTIVE);
machine.setType(DeviceType.WORKSTATION);
machine.setOsType("Windows");
machine.setRegisteredAt(Instant.now());

// Query machines by organization
@Query("{'organizationId': ?0, 'status': ?1}")
List<Machine> findActiveOrgMachines(String orgId, DeviceStatus status);

// Update last seen timestamp
machine.setLastSeen(Instant.now());
machine.setUpdatedAt(Instant.now());
```
