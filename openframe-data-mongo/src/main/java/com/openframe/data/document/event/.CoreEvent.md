A MongoDB document model representing system events in the OpenFrame application, tracking event lifecycle with status transitions and user attribution.

## Key Components

- **CoreEvent**: Main entity class annotated with `@Document` for MongoDB persistence
- **EventStatus**: Enum defining event lifecycle states (CREATED, PROCESSING, COMPLETED, FAILED)
- **Core Fields**:
  - `id`: MongoDB document identifier
  - `type`: Event classification string
  - `payload`: JSON or serialized event data
  - `timestamp`: Event creation/occurrence time
  - `userId`: Associated user identifier
  - `status`: Current event processing state

## Usage Example

```java
@Service
public class EventService {
    
    @Autowired
    private MongoTemplate mongoTemplate;
    
    public CoreEvent createEvent(String type, String payload, String userId) {
        CoreEvent event = new CoreEvent();
        event.setType(type);
        event.setPayload(payload);
        event.setUserId(userId);
        event.setTimestamp(Instant.now());
        event.setStatus(CoreEvent.EventStatus.CREATED);
        
        return mongoTemplate.save(event);
    }
    
    public void updateEventStatus(String eventId, CoreEvent.EventStatus status) {
        CoreEvent event = mongoTemplate.findById(eventId, CoreEvent.class);
        if (event != null) {
            event.setStatus(status);
            mongoTemplate.save(event);
        }
    }
}
```

This model provides a foundation for event-driven architecture, enabling audit trails, asynchronous processing tracking, and user activity monitoring within the OpenFrame system.
