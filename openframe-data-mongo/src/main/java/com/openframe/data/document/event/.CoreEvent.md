A MongoDB document model representing system events with tracking capabilities for event-driven architectures.

## Key Components

- **CoreEvent**: Main entity class with MongoDB document mapping
- **EventStatus**: Enum defining event lifecycle states (CREATED, PROCESSING, COMPLETED, FAILED)
- **Key Fields**:
  - `id`: Unique MongoDB identifier
  - `type`: Event classification string
  - `payload`: Event data as string
  - `timestamp`: Event creation time
  - `userId`: Associated user identifier
  - `status`: Current processing state

## Usage Example

```java
// Creating a new event
CoreEvent event = new CoreEvent();
event.setType("USER_LOGIN");
event.setPayload("{\"username\":\"john_doe\",\"ip\":\"192.168.1.1\"}");
event.setTimestamp(Instant.now());
event.setUserId("user123");
event.setStatus(CoreEvent.EventStatus.CREATED);

// Updating event status during processing
event.setStatus(CoreEvent.EventStatus.PROCESSING);

// Repository save operation
eventRepository.save(event);

// Querying by status
List<CoreEvent> failedEvents = eventRepository.findByStatus(EventStatus.FAILED);
```

This model supports event sourcing patterns and provides audit trails for system activities with clear status tracking throughout the event lifecycle.
