A data class representing compliance requirements for device security standards and controls, tracking compliance status and verification details.

## Key Components

**Main Class:**
- `ComplianceRequirement` - Lombok data class for compliance tracking

**Core Fields:**
- `standard` - Compliance standard identifier (e.g., "NIST", "ISO27001")
- `control` - Specific control or requirement within the standard
- `compliant` - Boolean flag indicating current compliance status
- `details` - Additional information about compliance state
- `lastCheck` - Timestamp of most recent compliance verification

## Usage Example

```java
// Create a new compliance requirement
ComplianceRequirement requirement = new ComplianceRequirement();
requirement.setStandard("NIST");
requirement.setControl("AC-2.1");
requirement.setCompliant(true);
requirement.setDetails("User account management implemented");
requirement.setLastCheck(Instant.now());

// Check compliance status
if (requirement.isCompliant()) {
    System.out.println("Device meets " + requirement.getStandard() + 
                      " control " + requirement.getControl());
} else {
    System.out.println("Compliance issue: " + requirement.getDetails());
}

// Update compliance after remediation
requirement.setCompliant(true);
requirement.setDetails("Security controls updated and verified");
requirement.setLastCheck(Instant.now());
```

This class is typically used within device management systems to track and verify that devices meet various security and compliance standards required by organizational policies.
