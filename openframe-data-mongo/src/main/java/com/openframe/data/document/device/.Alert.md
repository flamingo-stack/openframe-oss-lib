A simple data model representing system alerts or notifications with severity levels and resolution status tracking.

## Key Components

- **Alert Class**: Lombok-powered data model with automatic getters/setters/equals/hashCode
- **id**: Unique identifier for the alert
- **severity**: Alert priority level (HIGH, MEDIUM, LOW)
- **message**: Human-readable alert description
- **timestamp**: When the alert was created using Java 8+ Instant
- **resolved**: Boolean flag indicating if the alert has been addressed

## Usage Example

```java
// Create a new high-severity alert
Alert alert = new Alert();
alert.setId("ALT-001");
alert.setSeverity("HIGH");
alert.setMessage("Device temperature exceeded threshold");
alert.setTimestamp(Instant.now());
alert.setResolved(false);

// Mark alert as resolved
alert.setResolved(true);

// Check alert properties
if ("HIGH".equals(alert.getSeverity()) && !alert.getResolved()) {
    System.out.println("Critical unresolved alert: " + alert.getMessage());
}
```

The class uses Lombok's `@Data` annotation to automatically generate boilerplate code, making it ideal for use in device monitoring systems, logging frameworks, or notification services where alerts need to be tracked and managed.
