A simple data model representing system alerts with severity levels and resolution status tracking.

## Key Components

- **Alert** - Main data class using Lombok's `@Data` annotation for automatic getters/setters
- **id** - Unique identifier for the alert
- **severity** - Alert priority level (HIGH, MEDIUM, LOW)
- **message** - Descriptive alert content
- **timestamp** - When the alert was created using `Instant`
- **resolved** - Boolean flag indicating if the alert has been addressed

## Usage Example

```java
// Create a new alert
Alert alert = new Alert();
alert.setId("ALERT-001");
alert.setSeverity("HIGH");
alert.setMessage("Database connection failed");
alert.setTimestamp(Instant.now());
alert.setResolved(false);

// Check alert status
if (!alert.isResolved() && "HIGH".equals(alert.getSeverity())) {
    System.out.println("Urgent alert: " + alert.getMessage());
}

// Mark as resolved
alert.setResolved(true);
```

The class leverages Lombok to automatically generate boilerplate code, making it a clean POJO suitable for JSON serialization, database persistence, or API responses in device monitoring systems.
