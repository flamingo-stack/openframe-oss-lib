This MongoDB document model represents Single Sign-On (SSO) configuration settings for authentication providers in the OpenFrame system. It stores provider-specific credentials, user provisioning rules, and domain restrictions.

## Key Components

- **Basic Configuration**: `provider`, `clientId`, `clientSecret` for OAuth/OIDC integration
- **Feature Toggles**: `enabled` flag and `autoProvisionUsers` for user management control
- **Microsoft Integration**: `msTenantId` for Office 365/Entra ID specific configurations
- **Domain Security**: `allowedDomains` list for restricting user provisioning by email domain
- **MongoDB Integration**: `@Document` annotation mapping to "sso_configs" collection

## Usage Example

```java
// Create SSO configuration for Microsoft Office 365
SSOConfig officeConfig = new SSOConfig();
officeConfig.setProvider("microsoft");
officeConfig.setClientId("your-app-client-id");
officeConfig.setClientSecret("your-app-secret");
officeConfig.setEnabled(true);
officeConfig.setAutoProvisionUsers(true);
officeConfig.setMsTenantId("company-tenant-id");
officeConfig.setAllowedDomains(List.of("company.com", "subsidiary.com"));

// Create SSO configuration for Google with restricted provisioning
SSOConfig googleConfig = new SSOConfig();
googleConfig.setProvider("google");
googleConfig.setClientId("google-client-id");
googleConfig.setClientSecret("google-secret");
googleConfig.setEnabled(true);
googleConfig.setAutoProvisionUsers(false); // Only existing users can sign in
```

The `autoProvisionUsers` flag controls whether new users can be automatically created during SSO authentication, while `allowedDomains` provides an additional security layer for organizations wanting to restrict access by email domain.
