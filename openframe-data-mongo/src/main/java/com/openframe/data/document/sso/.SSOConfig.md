A MongoDB document entity that stores Single Sign-On (SSO) provider configurations, enabling authentication through external identity providers with configurable auto-provisioning and domain restrictions.

## Key Components

- **SSOConfig** - Main entity class representing SSO configuration documents
- **provider** - String identifier for the SSO provider type
- **clientId/clientSecret** - OAuth2/OIDC client credentials
- **enabled** - Boolean flag to activate/deactivate the SSO provider
- **autoProvisionUsers** - Controls whether new users can be automatically created during SSO authentication
- **msTenantId** - Microsoft Entra (Azure AD) tenant identifier for Office 365 integration
- **allowedDomains** - List of email domains permitted for auto-provisioning

## Usage Example

```java
// Create a new SSO configuration
SSOConfig config = new SSOConfig();
config.setProvider("azure_ad");
config.setClientId("your-client-id");
config.setClientSecret("your-client-secret");
config.setEnabled(true);
config.setAutoProvisionUsers(true);
config.setMsTenantId("tenant-uuid");
config.setAllowedDomains(List.of("company.com", "partner.org"));

// Save to MongoDB
mongoTemplate.save(config);

// Query enabled SSO providers
Query query = Query.query(Criteria.where("enabled").is(true));
List<SSOConfig> activeProviders = mongoTemplate.find(query, SSOConfig.class);
```
