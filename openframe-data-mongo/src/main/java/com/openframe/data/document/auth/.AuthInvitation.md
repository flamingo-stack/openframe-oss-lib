Extends the base `Invitation` document to add tenant-specific authentication capabilities with MongoDB indexing for multi-tenant applications.

## Key Components

- **AuthInvitation**: Main class extending `Invitation` with tenant-aware functionality
- **tenantId**: String field identifying the tenant scope for the invitation
- **Compound Index**: Unique constraint on `tenantId` and `email` combination
- **Partial Filter**: Ensures uniqueness only when `tenantId` exists

## Usage Example

```java
// Create a tenant-specific invitation
AuthInvitation invitation = AuthInvitation.builder()
    .tenantId("tenant-123")
    .email("user@example.com")
    .build();

// The compound index ensures no duplicate invitations 
// for the same email within a tenant
repository.save(invitation);

// Query invitations by tenant
List<AuthInvitation> tenantInvitations = repository.findByTenantId("tenant-123");
```

The compound index with partial filtering allows multiple invitations with the same email across different tenants while preventing duplicates within a single tenant scope.
