A MongoDB document class that extends the base `Invitation` entity to support multi-tenant authentication systems by associating invitations with specific tenant contexts.

## Key Components

- **AuthInvitation**: Main entity class extending `Invitation` with tenant-specific functionality
- **tenantId**: Indexed field that associates the invitation with a specific tenant
- **Compound Index**: Ensures unique email addresses per tenant while allowing cross-tenant duplicates
- **Lombok Annotations**: Provides boilerplate code generation for data access and builder patterns

## Usage Example

```java
// Create a tenant-specific invitation
AuthInvitation invitation = AuthInvitation.builder()
    .tenantId("tenant-123")
    .email("user@example.com")
    .role("USER")
    .expiresAt(LocalDateTime.now().plusDays(7))
    .build();

// Repository operations will automatically enforce tenant isolation
authInvitationRepository.save(invitation);

// Find invitations for a specific tenant
List<AuthInvitation> tenantInvitations = 
    authInvitationRepository.findByTenantId("tenant-123");

// The compound index ensures email uniqueness within tenant scope
// This allows the same email to be invited to multiple tenants
```

The compound index with partial filter ensures data integrity by preventing duplicate email invitations within the same tenant while allowing the same email to exist across different tenants.
