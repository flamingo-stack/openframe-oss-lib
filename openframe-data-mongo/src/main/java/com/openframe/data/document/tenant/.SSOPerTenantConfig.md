A MongoDB document class that stores SSO configuration settings specific to individual tenants, extending the base SSOConfig with tenant linkage and audit timestamps.

## Key Components

- **SSOPerTenantConfig** - Main entity class extending `SSOConfig`
- **tenantId** - Unique indexed field linking SSO config to a specific tenant
- **createdAt/updatedAt** - Automatic timestamp fields for audit tracking
- **Lombok annotations** - Generates boilerplate code (getters, setters, constructors, equals/hashCode)
- **MongoDB annotations** - Handles indexing and automatic date population

## Usage Example

```java
// Create SSO config for a tenant
SSOPerTenantConfig tenantSSOConfig = new SSOPerTenantConfig();
tenantSSOConfig.setTenantId("tenant-123");
tenantSSOConfig.setEnabled(true);
tenantSSOConfig.setProvider("OKTA");
tenantSSOConfig.setClientId("client-abc");
tenantSSOConfig.setClientSecret("secret-xyz");

// Save to MongoDB - timestamps will be auto-populated
mongoTemplate.save(tenantSSOConfig);

// Query by tenant
SSOPerTenantConfig config = mongoTemplate.findOne(
    Query.query(Criteria.where("tenantId").is("tenant-123")), 
    SSOPerTenantConfig.class
);
```

The unique index on `tenantId` ensures each tenant can have only one SSO configuration, while inheriting all SSO provider settings from the parent `SSOConfig` class.
