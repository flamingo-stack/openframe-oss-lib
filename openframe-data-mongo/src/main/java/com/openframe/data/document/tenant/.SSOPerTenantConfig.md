A MongoDB document entity that extends base SSO configuration with tenant-specific association and audit timestamps for multi-tenant SSO management.

## Key Components

- **SSOPerTenantConfig** - Main entity class extending `SSOConfig`
- **tenantId** - Unique indexed field linking SSO config to specific tenant
- **createdAt/updatedAt** - Automatic audit timestamps via Spring Data annotations
- **MongoDB Annotations** - `@Indexed` for unique tenant constraint, `@CreatedDate`/`@LastModifiedDate` for automatic timestamping

## Usage Example

```java
// Create SSO config for a specific tenant
SSOPerTenantConfig tenantSSO = new SSOPerTenantConfig();
tenantSSO.setTenantId("tenant-123");
tenantSSO.setProvider("google");
tenantSSO.setClientId("oauth-client-id");
tenantSSO.setClientSecret("oauth-secret");
tenantSSO.setEnabled(true);

// Save to MongoDB - timestamps will be automatically set
ssoRepository.save(tenantSSO);

// Query by tenant
SSOPerTenantConfig config = ssoRepository.findByTenantId("tenant-123");

// The unique index ensures one SSO config per tenant
// Inherits all base SSO functionality from SSOConfig parent class
```

This entity enables isolated SSO configurations per tenant in a multi-tenant architecture, with automatic audit trails and database-level uniqueness constraints.
