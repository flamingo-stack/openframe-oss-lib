A MongoDB document entity representing an integrated tool agent configuration with installation, runtime, and asset management settings.

## Key Components

- **`@Document`** - Maps to MongoDB collection "integrated_tool_agents"
- **Tool Identity** - `id`, `toolId`, `version` for agent identification
- **Session Management** - `sessionType` enum for execution context
- **Asset Management** - `downloadConfigurations` and `assets` lists for resources
- **Command Configuration** - Installation, runtime, and uninstallation command arguments
- **Update Control** - Flags for version and configuration updates
- **Status Tracking** - `status` field for monitoring agent state

## Usage Example

```java
// Create a new integrated tool agent
IntegratedToolAgent agent = new IntegratedToolAgent();
agent.setToolId("my-tool-v1");
agent.setReleaseVersion(true);
agent.setVersion("1.0.0");
agent.setSessionType(SessionType.PERSISTENT);

// Configure installation commands
agent.setInstallationCommandArgs(List.of("npm", "install", "-g", "my-tool"));
agent.setRunCommandArgs(List.of("my-tool", "--config", "${config.json}"));
agent.setUninstallationCommandArgs(List.of("npm", "uninstall", "-g", "my-tool"));

// Set update permissions
agent.setAllowVersionUpdate(true);
agent.setAllowConfigurationUpdate(false);
agent.setStatus(ToolAgentStatus.READY);

// Repository save
toolAgentRepository.save(agent);
```
