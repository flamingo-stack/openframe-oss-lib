A data model representing contact person information that is embedded within organization documents rather than stored as a separate collection.

## Key Components

**Main Class:**
- `ContactPerson` - Lombok-powered POJO with builder pattern support for contact information

**Fields:**
- `contactName` - Full name of the contact person
- `title` - Job title or position within the organization
- `phone` - Contact phone number
- `email` - Contact email address

**Annotations:**
- `@Data` - Generates getters, setters, toString, equals, and hashCode
- `@Builder` - Enables fluent builder pattern for object creation
- `@NoArgsConstructor/@AllArgsConstructor` - Provides default and full constructors

## Usage Example

```java
// Using builder pattern (recommended)
ContactPerson contact = ContactPerson.builder()
    .contactName("John Smith")
    .title("Technical Director")
    .phone("+1-555-123-4567")
    .email("john.smith@company.com")
    .build();

// Using constructor
ContactPerson contact2 = new ContactPerson(
    "Jane Doe", 
    "Project Manager", 
    "+1-555-987-6543", 
    "jane.doe@company.com"
);

// Accessing fields
String name = contact.getContactName();
contact.setTitle("Senior Technical Director");
```
