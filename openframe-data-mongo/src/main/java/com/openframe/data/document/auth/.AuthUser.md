A MongoDB document model for multi-tenant authentication that extends the base User class with authentication-specific fields including tenant isolation, password management, and external provider support.

## Key Components

- **AuthUser** - Main authentication user document extending `User`
- **tenantId** - Indexed field for multi-tenant isolation
- **passwordHash** - Hashed password storage for local authentication
- **loginProvider** - Authentication provider type (LOCAL, GOOGLE, etc.)
- **externalUserId** - External provider user identifier
- **lastLogin** - Timestamp tracking for security monitoring
- **getFullName()** - Utility method for display name generation
- **@CompoundIndex** - Ensures unique email per tenant with partial filtering

## Usage Example

```java
// Create a new authenticated user for a tenant
AuthUser authUser = AuthUser.builder()
    .tenantId("tenant-123")
    .email("user@example.com")
    .firstName("John")
    .lastName("Doe")
    .passwordHash("$2a$10$hashed_password")
    .loginProvider("LOCAL")
    .lastLogin(Instant.now())
    .build();

// Get display name
String displayName = authUser.getFullName(); // "John Doe"

// External provider authentication
AuthUser googleUser = AuthUser.builder()
    .tenantId("tenant-123")
    .email("user@gmail.com")
    .loginProvider("GOOGLE")
    .externalUserId("google-user-id-123")
    .lastLogin(Instant.now())
    .build();
```

The compound index ensures email uniqueness within each tenant while supporting global users without tenant restrictions through partial filtering.
