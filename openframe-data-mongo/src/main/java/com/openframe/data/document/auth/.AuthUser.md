Extends the base User model to provide authentication-specific functionality for multi-tenant authorization servers with domain-based tenancy support.

## Key Components

- **AuthUser** - Main entity class extending User with authentication fields
- **Compound Index** - Ensures unique email per tenant with partial filtering
- **Authentication Fields** - Password hash, email verification, and login provider tracking
- **External Auth Support** - Fields for OAuth/SSO integration (Google, etc.)
- **Tenant Isolation** - Multi-tenant support with indexed tenant ID
- **getFullName()** - Utility method for display name generation

## Usage Example

```java
// Create a new authenticated user for a tenant
AuthUser authUser = AuthUser.builder()
    .tenantId("tenant-123")
    .email("user@example.com")
    .firstName("John")
    .lastName("Doe")
    .passwordHash("$2a$10$...")
    .emailVerified(true)
    .loginProvider("LOCAL")
    .lastLogin(Instant.now())
    .build();

// Get display name with fallback logic
String displayName = authUser.getFullName(); // Returns "John Doe"

// For external OAuth users
AuthUser oauthUser = AuthUser.builder()
    .tenantId("tenant-123")
    .email("oauth@example.com")
    .loginProvider("GOOGLE")
    .externalUserId("google-user-id-123")
    .emailVerified(true)
    .build();
```

The compound index ensures that users can have the same email across different tenants but not within the same tenant, enabling proper multi-tenant isolation while supporting standard authentication workflows.
