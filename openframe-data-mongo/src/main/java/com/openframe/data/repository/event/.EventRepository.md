This repository interface provides MongoDB data access operations for Event documents, combining Spring Data's standard CRUD operations with custom query methods.

## Key Components

- **EventRepository** - Main repository interface extending MongoRepository
- **MongoRepository<Event, String>** - Provides standard CRUD operations for Event entities with String IDs
- **CustomEventRepository** - Interface for custom query methods and complex operations

## Usage Example

```java
@Service
public class EventService {
    
    @Autowired
    private EventRepository eventRepository;
    
    public Event saveEvent(Event event) {
        return eventRepository.save(event);
    }
    
    public List<Event> getAllEvents() {
        return eventRepository.findAll();
    }
    
    public Optional<Event> findEventById(String id) {
        return eventRepository.findById(id);
    }
    
    public void deleteEvent(String id) {
        eventRepository.deleteById(id);
    }
    
    // Custom methods from CustomEventRepository
    public List<Event> findEventsByCustomCriteria() {
        return eventRepository.customQueryMethod();
    }
}
```

The repository automatically provides MongoDB operations through Spring Data, while custom methods are implemented in the CustomEventRepository interface.
