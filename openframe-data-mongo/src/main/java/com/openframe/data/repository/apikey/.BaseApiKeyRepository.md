This interface defines a technology-agnostic base contract for API key repository operations, using generics to support both blocking and reactive implementations.

## Key Components

- **Generic Type Parameters**: 
  - `T` - Return wrapper (Optional for blocking, Mono for reactive)
  - `B` - Boolean return type (boolean for blocking, Mono<Boolean> for reactive)
  - `L` - List return type (List for blocking, Flux for reactive)
  - `ID` - Identifier type

- **Core Methods**:
  - `findByIdAndUserId()` - Retrieves specific API key for a user
  - `findByUserId()` - Gets all API keys for a user
  - `findExpiredKeys()` - Finds keys that have expired

## Usage Example

```java
// Blocking implementation
public class BlockingApiKeyRepository implements BaseApiKeyRepository<Optional<ApiKey>, Boolean, List<ApiKey>, String> {
    @Override
    public Optional<ApiKey> findByIdAndUserId(String keyId, String userId) {
        // Implementation for blocking database access
        return Optional.ofNullable(database.findApiKey(keyId, userId));
    }
}

// Reactive implementation
public class ReactiveApiKeyRepository implements BaseApiKeyRepository<Mono<ApiKey>, Mono<Boolean>, Flux<ApiKey>, String> {
    @Override
    public Mono<ApiKey> findByIdAndUserId(String keyId, String userId) {
        // Implementation for reactive database access
        return databaseClient.select().from("api_keys")
            .where("key_id").is(keyId).and("user_id").is(userId)
            .as(ApiKey.class).first();
    }
}
```
