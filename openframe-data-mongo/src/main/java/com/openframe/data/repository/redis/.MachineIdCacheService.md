Service for caching machine and organization information using Spring Cache abstraction with lightweight DTOs to optimize performance and reduce serialization overhead. Only enabled when MongoDB repositories are available in servlet-based applications.

## Key Components

- **@Cacheable Methods**: Two caching operations using Spring's declarative cache annotations
  - `getMachine(String agentId)` - Caches machine info by agent ID using "machineCache"
  - `getOrganization(String organizationId)` - Caches organization info by ID using "organizationCache"
- **Repository Dependencies**: 
  - `ToolConnectionRepository` - Retrieves tool connections by agent ID
  - `MachineRepository` - Fetches machine details by machine ID
  - `OrganizationRepository` - Retrieves organization information
- **Cache DTOs**: Returns `CachedMachineInfo` and `CachedOrganizationInfo` with only essential fields
- **Conditional Loading**: Enabled via `openframe.machine-id.cache.enabled=true` property

## Usage Example

```java
@Autowired
private MachineIdCacheService cacheService;

// Get cached machine information by agent ID
public void processAgentRequest(String agentId) {
    CachedMachineInfo machineInfo = cacheService.getMachine(agentId);
    if (machineInfo != null) {
        log.info("Machine: {} on host: {}", 
            machineInfo.getMachineId(), 
            machineInfo.getHostname());
        
        // Get organization details for the machine
        CachedOrganizationInfo orgInfo = cacheService.getOrganization(
            machineInfo.getOrganizationId());
        
        if (orgInfo != null) {
            log.info("Organization: {}", orgInfo.getName());
        }
    }
}
```
