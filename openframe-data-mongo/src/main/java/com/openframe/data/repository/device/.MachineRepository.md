Repository interface for managing Machine entities in the OpenFrame device management system. This interface extends MongoDB repository functionality to provide machine-specific query operations with flexible filtering and search capabilities.

## Key Components

**Core Repository Methods:**
- `findByMachineId()` - Retrieves machine by unique machine identifier
- `findByHostnameContainingIgnoreCase()` - Case-insensitive hostname search
- `findByTypeAndHostnameContainingIgnoreCase()` - Combined device type and hostname filtering
- `findByType()` - Filters machines by device type
- `findByMachineIdInAndStatus()` - Bulk machine lookup by IDs and status
- `existsByOrganizationId()` - Checks organization association

**Inheritance:**
- `MongoRepository<Machine, String>` - Standard MongoDB CRUD operations
- `CustomMachineRepository` - Extended custom query methods

## Usage Example

```java
@Service
public class MachineService {
    
    @Autowired
    private MachineRepository machineRepository;
    
    public List<Machine> searchMachines(String hostname, DeviceType type) {
        if (type != null) {
            return machineRepository.findByTypeAndHostnameContainingIgnoreCase(type, hostname);
        }
        return machineRepository.findByHostnameContainingIgnoreCase(hostname);
    }
    
    public List<Machine> getActiveMachines(List<String> machineIds) {
        return machineRepository.findByMachineIdInAndStatus(machineIds, DeviceStatus.ACTIVE);
    }
    
    public boolean hasOrganizationMachines(String orgId) {
        return machineRepository.existsByOrganizationId(orgId);
    }
}
```
