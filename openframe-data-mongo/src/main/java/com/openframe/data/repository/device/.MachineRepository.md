A Spring Data MongoDB repository interface providing data access operations for Machine entities with custom query methods for device management.

## Key Components

- **MachineRepository** - Main repository interface extending `MongoRepository<Machine, String>` and `CustomMachineRepository`
- **findByMachineId()** - Retrieves a machine by its unique machine identifier
- **findByHostnameContainingIgnoreCase()** - Searches machines by hostname substring (case-insensitive)
- **findByTypeAndHostnameContainingIgnoreCase()** - Filters machines by device type and hostname pattern
- **findByType()** - Retrieves all machines of a specific device type
- **existsByOrganizationId()** - Checks if any machines exist for an organization

## Usage Example

```java
@Autowired
private MachineRepository machineRepository;

// Find a specific machine
Optional<Machine> machine = machineRepository.findByMachineId("device-001");

// Search machines by hostname pattern
List<Machine> webServers = machineRepository.findByHostnameContainingIgnoreCase("web");

// Get all machines of a specific type
List<Machine> servers = machineRepository.findByType(DeviceType.SERVER);

// Filter by type and hostname
List<Machine> prodServers = machineRepository.findByTypeAndHostnameContainingIgnoreCase(
    DeviceType.SERVER, "prod"
);

// Check organization has machines
boolean hasDevices = machineRepository.existsByOrganizationId("org-123");
```

The repository leverages Spring Data's query method naming conventions to automatically generate MongoDB queries, providing efficient device lookup and filtering capabilities for the OpenFrame platform.
