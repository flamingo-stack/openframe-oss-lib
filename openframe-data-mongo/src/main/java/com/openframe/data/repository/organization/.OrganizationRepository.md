A Spring Data MongoDB repository interface for managing Organization documents, providing standard CRUD operations plus custom queries for organization-specific operations like finding by organization ID, name, and default status.

## Key Components

- **MongoRepository<Organization, String>**: Inherits standard MongoDB repository operations
- **CustomOrganizationRepository**: Extended interface for custom query operations
- **findByOrganizationId()**: Retrieves organization by unique business identifier
- **findByName()**: Locates organization by name field
- **findByIsDefaultTrue()**: Finds the designated default organization
- **findByOrganizationIdIn()**: Batch retrieval of multiple organizations

## Usage Example

```java
@Service
public class OrganizationService {
    
    @Autowired
    private OrganizationRepository organizationRepository;
    
    public Organization getDefaultOrganization() {
        return organizationRepository.findByIsDefaultTrue()
            .orElseThrow(() -> new RuntimeException("No default organization found"));
    }
    
    public Organization findByBusinessId(String orgId) {
        return organizationRepository.findByOrganizationId(orgId)
            .orElse(null);
    }
    
    public List<Organization> loadMultipleOrganizations(Set<String> orgIds) {
        return organizationRepository.findByOrganizationIdIn(orgIds);
    }
}
```

The repository is conditionally loaded only for servlet-based web applications and combines Spring Data's automatic query derivation with custom repository extensions for specialized organization queries.
