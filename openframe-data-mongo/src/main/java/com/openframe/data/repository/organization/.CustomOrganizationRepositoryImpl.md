Custom repository implementation for MongoDB organization queries that provides advanced filtering and cursor-based pagination capabilities at the database level for optimal performance.

## Key Components

- **CustomOrganizationRepositoryImpl**: Main repository class implementing `CustomOrganizationRepository`
- **buildOrganizationQuery()**: Builds complex MongoDB queries with multiple filter criteria
- **findOrganizationsWithFilters()**: Executes filtered organization queries
- **findOrganizationsWithCursor()**: Implements cursor-based pagination with sorting
- **isSortableField()**: Validates sortable field names
- **getDefaultSortField()**: Returns default sorting field

## Usage Example

```java
@Service
public class OrganizationService {
    
    @Autowired
    private CustomOrganizationRepository organizationRepository;
    
    public List<Organization> searchOrganizations() {
        // Create filter criteria
        OrganizationQueryFilter filter = new OrganizationQueryFilter();
        filter.setCategory("Technology");
        filter.setMinEmployees(10);
        filter.setMaxEmployees(500);
        filter.setHasActiveContract(true);
        
        // Search with text and filters
        return organizationRepository.findOrganizationsWithFilters(
            filter, "Software"
        );
    }
    
    public List<Organization> getOrganizationsPaginated(String cursor) {
        Query query = organizationRepository.buildOrganizationQuery(null, null);
        
        // Get next page with cursor pagination
        return organizationRepository.findOrganizationsWithCursor(
            query, cursor, 20, "name", "ASC"
        );
    }
}
```

The implementation supports filtering by category, employee count, active contracts, and text search across name, ID, and category fields. It automatically excludes soft-deleted records and provides efficient cursor-based pagination with configurable sorting.
