Custom MongoDB repository implementation that provides advanced querying capabilities for Organization documents with filtering, searching, and cursor-based pagination support.

## Key Components

**Main Class:**
- `CustomOrganizationRepositoryImpl` - Repository implementation with MongoDB-specific query building

**Core Methods:**
- `buildOrganizationQuery()` - Constructs MongoDB queries with filters, search terms, and cursor pagination
- `findOrganizationsWithFilters()` - Executes filtered queries without pagination
- `findOrganizationsWithCursor()` - Executes queries with cursor-based pagination and sorting

**Query Features:**
- Soft delete exclusion (always filters out deleted organizations)
- Category filtering with case-insensitive regex matching
- Employee count range filtering (min/max employees)
- Active contract date validation
- Multi-field text search (name, organizationId, category)
- Cursor-based pagination using ObjectId comparison

## Usage Example

```java
@Autowired
private CustomOrganizationRepository organizationRepository;

// Create filter for active organizations with 10-100 employees
OrganizationQueryFilter filter = OrganizationQueryFilter.builder()
    .category("Technology")
    .minEmployees(10)
    .maxEmployees(100)
    .hasActiveContract(true)
    .build();

// Search with text query
List<Organization> results = organizationRepository
    .findOrganizationsWithFilters(filter, "software");

// Paginated search with cursor
Query query = organizationRepository
    .buildOrganizationQuery(filter, "software", "507f1f77bcf86cd799439011");
List<Organization> pagedResults = organizationRepository
    .findOrganizationsWithCursor(query, "507f1f77bcf86cd799439011", 20);
```

The implementation uses MongoDB's `$and` operator to combine multiple criteria efficiently and performs database-level filtering for optimal performance.
