Custom MongoDB repository implementation for Organization entities that provides advanced querying and filtering capabilities with database-level optimization for better performance.

## Key Components

**Main Class:**
- `CustomOrganizationRepositoryImpl` - Repository implementation with MongoDB query building and filtering

**Core Methods:**
- `buildOrganizationQuery()` - Constructs MongoDB queries with filters, search, and cursor pagination
- `findOrganizationsWithFilters()` - Executes filtered organization queries
- `findOrganizationsWithCursor()` - Handles cursor-based pagination with sorting

**Filter Support:**
- Category filtering (case-insensitive regex)
- Employee count range filtering (min/max)
- Active contract status filtering (date-based)
- Text search across name, organizationId, and category
- Cursor-based pagination support
- Soft delete exclusion

## Usage Example

```java
@Autowired
private CustomOrganizationRepository organizationRepository;

// Create filter for organizations with 50-200 employees and active contracts
OrganizationQueryFilter filter = OrganizationQueryFilter.builder()
    .category("Technology")
    .minEmployees(50)
    .maxEmployees(200)
    .hasActiveContract(true)
    .build();

// Find organizations with search term
List<Organization> results = organizationRepository
    .findOrganizationsWithFilters(filter, "software");

// Build query for pagination
Query query = organizationRepository
    .buildOrganizationQuery(filter, "search", "cursorId");
List<Organization> pagedResults = organizationRepository
    .findOrganizationsWithCursor(query, "cursorId", 20);
```
