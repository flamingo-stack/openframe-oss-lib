A generic base interface that defines common user repository operations, designed to be technology-agnostic and support both blocking and reactive implementations through parameterized types.

## Key Components

- **BaseUserRepository<T, B, ID>** - Generic interface with type parameters for return wrapper (T), boolean type (B), and ID type (ID)
- **findByEmail()** - Retrieves a user by email address
- **existsByEmail()** - Checks if a user exists with given email
- **existsByEmailAndStatus()** - Checks user existence by email and UserStatus

## Usage Example

```java
// Blocking implementation
public class JpaUserRepository implements BaseUserRepository<Optional<User>, Boolean, String> {
    @Override
    public Optional<User> findByEmail(String email) {
        return userJpaRepository.findByEmail(email);
    }
    
    @Override
    public Boolean existsByEmail(String email) {
        return userJpaRepository.existsByEmail(email);
    }
    
    @Override
    public Boolean existsByEmailAndStatus(String email, UserStatus status) {
        return userJpaRepository.existsByEmailAndStatus(email, status);
    }
}

// Reactive implementation
public class ReactiveUserRepository implements BaseUserRepository<Mono<User>, Mono<Boolean>, String> {
    @Override
    public Mono<User> findByEmail(String email) {
        return mongoTemplate.findOne(query(where("email").is(email)), User.class);
    }
    
    @Override
    public Mono<Boolean> existsByEmail(String email) {
        return mongoTemplate.exists(query(where("email").is(email)), User.class);
    }
}
```
