A Spring Data MongoDB repository interface for managing release version documents, designed to maintain a single document representing the current application release version.

## Key Components

- **ReleaseVersionRepository** - Main repository interface extending `MongoRepository<ReleaseVersion, String>`
- **findFirstByOrderByCreatedAtAsc()** - Custom query method to retrieve the oldest release version document
- **@ConditionalOnWebApplication** - Conditionally enables the repository only for servlet-based web applications
- **@Repository** - Spring stereotype annotation for repository beans

## Usage Example

```java
@Service
public class VersionService {
    
    @Autowired
    private ReleaseVersionRepository versionRepository;
    
    public Optional<ReleaseVersion> getCurrentVersion() {
        return versionRepository.findFirstByOrderByCreatedAtAsc();
    }
    
    public ReleaseVersion createInitialVersion(String version) {
        ReleaseVersion releaseVersion = new ReleaseVersion();
        releaseVersion.setVersion(version);
        return versionRepository.save(releaseVersion);
    }
    
    public boolean hasVersionDocument() {
        return versionRepository.findFirstByOrderByCreatedAtAsc().isPresent();
    }
}
```

The repository is conditionally loaded only in servlet web applications and provides basic CRUD operations through MongoDB integration. The custom finder method ensures retrieval of the primary version document by creation date ordering.
