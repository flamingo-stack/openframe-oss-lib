Repository interface for managing authentication user data with MongoDB integration and tenant-aware user lookup capabilities.

## Key Components

- **AuthUserRepository** - Spring Data MongoDB repository interface extending `MongoRepository<AuthUser, String>`
- **findByEmailAndStatus()** - Retrieves active users by email address
- **findByEmailAndTenantIdAndStatus()** - Tenant-specific active user lookup
- **existsByEmailAndTenantId()** - Checks user existence within tenant scope
- **findByEmailAndTenantId()** - Tenant-specific user lookup regardless of status

## Usage Example

```java
@Service
public class AuthService {
    
    @Autowired
    private AuthUserRepository authUserRepository;
    
    public AuthUser authenticateUser(String email, String tenantId) {
        return authUserRepository
            .findByEmailAndTenantIdAndStatus(email, tenantId, UserStatus.ACTIVE)
            .orElseThrow(() -> new UserNotFoundException("User not found"));
    }
    
    public boolean isUserRegistered(String email, String tenantId) {
        return authUserRepository.existsByEmailAndTenantId(email, tenantId);
    }
    
    public Optional<AuthUser> findUserInTenant(String email, String tenantId) {
        return authUserRepository.findByEmailAndTenantId(email, tenantId);
    }
}
```

This repository supports multi-tenant authentication scenarios where users can exist across different tenants with various status states, providing both status-aware and status-agnostic query methods.
