A MongoDB repository interface for managing Single Sign-On (SSO) configuration documents. This repository provides data access operations for SSO provider configurations including queries for specific providers and enabled configurations.

## Key Components

- **SSOConfigRepository** - Spring Data MongoDB repository interface extending `MongoRepository<SSOConfig, String>`
- **findByProvider()** - Finds SSO configuration by provider name
- **findByEnabledTrue()** - Retrieves all enabled SSO configurations

## Usage Example

```java
@Service
public class SSOService {
    
    @Autowired
    private SSOConfigRepository ssoConfigRepository;
    
    // Get configuration for a specific SSO provider
    public Optional<SSOConfig> getProviderConfig(String provider) {
        return ssoConfigRepository.findByProvider(provider);
    }
    
    // Get all active SSO providers
    public List<SSOConfig> getActiveProviders() {
        return ssoConfigRepository.findByEnabledTrue();
    }
    
    // Save new SSO configuration
    public SSOConfig createSSOConfig(SSOConfig config) {
        return ssoConfigRepository.save(config);
    }
    
    // Check if Google SSO is available
    public boolean isGoogleSSOEnabled() {
        return ssoConfigRepository.findByProvider("google")
            .map(SSOConfig::isEnabled)
            .orElse(false);
    }
}
```

This repository leverages Spring Data's query method naming conventions to automatically generate MongoDB queries, eliminating the need for manual query implementation.
