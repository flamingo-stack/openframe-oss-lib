Implementation class providing custom MongoDB query operations for Event documents with advanced filtering and cursor-based pagination support.

## Key Components

- **buildEventQuery()** - Constructs MongoDB Query objects from EventQueryFilter parameters and search terms
- **findEventsWithCursor()** - Executes cursor-based pagination queries with configurable limits
- **findDistinctUserIds()** - Retrieves unique user IDs from the Event collection
- **findDistinctEventTypes()** - Retrieves unique event types from the Event collection

## Usage Example

```java
@Autowired
private CustomEventRepository eventRepository;

// Build filtered query
EventQueryFilter filter = EventQueryFilter.builder()
    .userIds(List.of("user1", "user2"))
    .eventTypes(List.of("LOGIN", "LOGOUT"))
    .startDate(LocalDate.of(2024, 1, 1))
    .endDate(LocalDate.of(2024, 12, 31))
    .build();

Query query = eventRepository.buildEventQuery(filter, "login");

// Get paginated results with cursor
List<Event> events = eventRepository.findEventsWithCursor(query, "lastCursorId", 50);

// Get distinct values for filtering
List<String> userIds = eventRepository.findDistinctUserIds();
List<String> eventTypes = eventRepository.findDistinctEventTypes();
```

The implementation handles date range filtering with proper UTC conversion, text search across event type and data fields, and provides efficient cursor-based pagination for large datasets.
