Custom MongoDB repository implementation for event querying with filtering, pagination, and cursor-based navigation. Provides flexible query building capabilities for event data retrieval with support for user filtering, event type filtering, date ranges, and text search.

## Key Components

- **buildEventQuery()** - Constructs MongoDB queries from filter criteria and search terms
- **findEventsWithCursor()** - Implements cursor-based pagination for efficient large dataset navigation
- **findDistinctUserIds()** - Retrieves unique user IDs from events collection
- **findDistinctEventTypes()** - Retrieves unique event types from events collection
- **MongoTemplate** - Spring Data MongoDB template for database operations

## Usage Example

```java
@Autowired
private CustomEventRepository eventRepository;

// Build query with filters
EventQueryFilter filter = EventQueryFilter.builder()
    .userIds(List.of("user1", "user2"))
    .eventTypes(List.of("LOGIN", "LOGOUT"))
    .startDate(LocalDate.now().minusDays(7))
    .endDate(LocalDate.now())
    .build();

Query query = eventRepository.buildEventQuery(filter, "login");

// Get paginated results with cursor
List<Event> events = eventRepository.findEventsWithCursor(query, null, 50);

// Get distinct values for filtering
List<String> userIds = eventRepository.findDistinctUserIds();
List<String> eventTypes = eventRepository.findDistinctEventTypes();
```

The implementation handles date range conversion to UTC timestamps, case-insensitive text search across event type and data fields, and maintains sorted results for consistent pagination behavior.
