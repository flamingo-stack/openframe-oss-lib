A Spring Data MongoDB repository interface for managing Single Sign-On (SSO) configurations on a per-tenant basis, providing methods to query SSO settings by tenant, provider, and allowed domains.

## Key Components

- **SSOPerTenantConfigRepository** - Main repository interface extending MongoRepository
- **findByTenantIdAndEnabledTrue()** - Retrieves all enabled SSO configs for a specific tenant
- **findByProvider()** - Finds SSO config by provider name
- **findFirstByTenantIdAndProviderAndEnabledTrue()** - Gets the first enabled SSO config for a tenant-provider combination
- **findByAllowedDomainsIn()** - Finds SSO configs that match any of the specified allowed domains

## Usage Example

```java
@Autowired
private SSOPerTenantConfigRepository ssoRepository;

// Get all enabled SSO configs for a tenant
List<SSOPerTenantConfig> tenantConfigs = ssoRepository.findByTenantIdAndEnabledTrue("tenant-123");

// Find SSO config by provider
Optional<SSOPerTenantConfig> googleConfig = ssoRepository.findByProvider("google");

// Get specific tenant-provider config
Optional<SSOPerTenantConfig> config = ssoRepository
    .findFirstByTenantIdAndProviderAndEnabledTrue("tenant-123", "okta");

// Find configs by allowed domains
List<String> domains = Arrays.asList("company.com", "subsidiary.com");
List<SSOPerTenantConfig> matchingConfigs = ssoRepository.findByAllowedDomainsIn(domains);
```
