A Spring Data MongoDB repository interface that provides database operations for managing Single Sign-On (SSO) configurations on a per-tenant basis. This repository enables queries for SSO providers by tenant, provider type, enabled status, and allowed domains.

## Key Components

- **SSOPerTenantConfigRepository**: Main repository interface extending `MongoRepository<SSOPerTenantConfig, String>`
- **findByTenantIdAndEnabledTrue()**: Retrieves all enabled SSO configurations for a specific tenant
- **findByProvider()**: Finds SSO configuration by provider name
- **findFirstByTenantIdAndProviderAndEnabledTrue()**: Gets the first enabled SSO config for a tenant-provider combination
- **findByAllowedDomainsIn()**: Finds configurations that allow access from specified domains

## Usage Example

```java
@Service
public class SSOConfigService {
    
    @Autowired
    private SSOPerTenantConfigRepository ssoRepository;
    
    public List<SSOPerTenantConfig> getEnabledSSOConfigs(String tenantId) {
        return ssoRepository.findByTenantIdAndEnabledTrue(tenantId);
    }
    
    public Optional<SSOPerTenantConfig> getProviderConfig(String tenantId, String provider) {
        return ssoRepository.findFirstByTenantIdAndProviderAndEnabledTrue(tenantId, provider);
    }
    
    public List<SSOPerTenantConfig> getConfigsByDomains(List<String> allowedDomains) {
        return ssoRepository.findByAllowedDomainsIn(allowedDomains);
    }
}
```

This repository follows Spring Data JPA naming conventions for automatic query generation, providing type-safe database operations for SSO configuration management in multi-tenant applications.
