A MongoDB repository interface for managing user invitations with status-based filtering and email lookup capabilities.

## Key Components

- **InvitationRepository** - Spring Data MongoDB repository interface extending `MongoRepository<Invitation, String>`
- **findByEmailAndStatus()** - Finds a specific invitation by email address and status
- **findByStatusNotIn()** - Retrieves paginated invitations excluding specific status values

## Usage Example

```java
@Service
public class InvitationService {
    
    @Autowired
    private InvitationRepository invitationRepository;
    
    public Optional<Invitation> getPendingInvitation(String email) {
        return invitationRepository.findByEmailAndStatus(email, "PENDING");
    }
    
    public Page<Invitation> getActiveInvitations(Pageable pageable) {
        Collection<InvitationStatus> excludedStatuses = List.of(
            InvitationStatus.EXPIRED, 
            InvitationStatus.REVOKED
        );
        return invitationRepository.findByStatusNotIn(excludedStatuses, pageable);
    }
}
```

This repository enables efficient querying of invitation data by email/status combinations and supports paginated retrieval of invitations while filtering out unwanted statuses, making it ideal for invitation management workflows in the OpenFrame platform.
