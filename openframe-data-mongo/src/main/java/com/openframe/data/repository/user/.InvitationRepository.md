Repository interface for managing user invitation data in MongoDB, providing standard CRUD operations and custom queries for invitation entities.

## Key Components

- **InvitationRepository** - MongoDB repository interface extending `MongoRepository<Invitation, String>`
- **findByEmailAndStatus()** - Custom query method to find invitations by email address and status

## Usage Example

```java
@Service
public class InvitationService {
    
    @Autowired
    private InvitationRepository invitationRepository;
    
    // Find pending invitation for a specific email
    public Optional<Invitation> findPendingInvitation(String email) {
        return invitationRepository.findByEmailAndStatus(email, "PENDING");
    }
    
    // Save new invitation
    public Invitation createInvitation(Invitation invitation) {
        return invitationRepository.save(invitation);
    }
    
    // Check if invitation exists and is active
    public boolean hasActiveInvitation(String email) {
        return invitationRepository.findByEmailAndStatus(email, "ACTIVE")
                .isPresent();
    }
}
```

This repository provides standard MongoDB operations through Spring Data's `MongoRepository` along with a specialized method for querying invitations by both email and status fields simultaneously.
