A custom repository interface for advanced querying and filtering of integrated tools in the OpenFrame system, extending beyond basic MongoDB operations.

## Key Components

- **buildToolQuery()** - Constructs MongoDB Query objects based on filter criteria and search terms
- **findToolsWithFilters()** - Retrieves filtered IntegratedTool collections using custom query logic
- **findDistinctTypes()** - Returns unique tool type values from the database
- **findDistinctCategories()** - Returns unique category values for categorization features
- **findDistinctPlatformCategories()** - Returns unique platform-specific category values

## Usage Example

```java
@Autowired
private CustomIntegratedToolRepository customToolRepo;

// Build custom query for filtered search
ToolQueryFilter filter = new ToolQueryFilter();
filter.setEnabled(true);
Query query = customToolRepo.buildToolQuery(filter, "docker");

// Find tools with complex filtering
List<IntegratedTool> filteredTools = customToolRepo
    .findToolsWithFilters(filter, "container");

// Get available filter options
List<String> availableTypes = customToolRepo.findDistinctTypes();
List<String> categories = customToolRepo.findDistinctCategories();
List<String> platforms = customToolRepo.findDistinctPlatformCategories();

// Use distinct values for UI dropdowns or validation
populateFilterDropdowns(availableTypes, categories, platforms);
```

This interface enables complex tool discovery and filtering capabilities that would be difficult to achieve with standard Spring Data MongoDB repository methods alone.
