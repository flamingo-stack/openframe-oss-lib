Defines a custom repository interface for advanced querying and filtering of integrated tools in the OpenFrame platform.

## Key Components

- **buildToolQuery(ToolQueryFilter, String)** - Constructs MongoDB queries based on filter criteria and search terms
- **findToolsWithFilters()** - Retrieves tools using filter and search parameters
- **findToolsWithFiltersAndSort()** - Enhanced tool retrieval with custom sorting options
- **findDistinctTypes/Categories/PlatformCategories()** - Extracts unique values for filter dropdowns
- **isSortableField()** - Validates whether a field can be used for sorting
- **getDefaultSortField()** - Returns the default field for sorting operations

## Usage Example

```java
@Service
public class ToolService {
    
    @Autowired
    private CustomIntegratedToolRepository toolRepository;
    
    public List<IntegratedTool> searchTools(String searchTerm, String category) {
        ToolQueryFilter filter = new ToolQueryFilter();
        filter.setCategory(category);
        
        return toolRepository.findToolsWithFiltersAndSort(
            filter, 
            searchTerm, 
            "name", 
            "ASC"
        );
    }
    
    public List<String> getAvailableCategories() {
        return toolRepository.findDistinctCategories();
    }
}
```

This interface extends Spring Data MongoDB's capabilities to support complex tool discovery and filtering operations, enabling the OpenFrame platform to provide sophisticated search and categorization features for integrated MSP tools.
