Custom repository interface that extends Spring Data MongoDB capabilities for event querying with advanced filtering and cursor-based pagination support.

## Key Components

- **`buildEventQuery()`** - Constructs MongoDB queries from filter criteria and search terms
- **`findEventsWithCursor()`** - Implements cursor-based pagination for efficient event retrieval
- **`findDistinctUserIds()`** - Retrieves unique user identifiers from event collection
- **`findDistinctEventTypes()`** - Returns distinct event type values for filtering options

## Usage Example

```java
@Service
public class EventService {
    
    @Autowired
    private CustomEventRepository eventRepository;
    
    public List<Event> getFilteredEvents(EventQueryFilter filter, String searchTerm) {
        Query query = eventRepository.buildEventQuery(filter, searchTerm);
        return eventRepository.findEventsWithCursor(query, null, 20);
    }
    
    public List<String> getAvailableEventTypes() {
        return eventRepository.findDistinctEventTypes();
    }
    
    public List<Event> getNextPage(String cursor, int pageSize) {
        Query query = eventRepository.buildEventQuery(new EventQueryFilter(), "");
        return eventRepository.findEventsWithCursor(query, cursor, pageSize);
    }
}
```

This interface should be implemented alongside a standard Spring Data repository to provide custom query capabilities for event data management.
