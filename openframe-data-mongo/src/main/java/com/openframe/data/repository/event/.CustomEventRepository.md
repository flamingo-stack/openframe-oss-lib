A custom repository interface that extends MongoDB query capabilities for Event entities, providing advanced filtering, cursor-based pagination, and query building functionality.

## Key Components

- **buildEventQuery()** - Constructs MongoDB queries from EventQueryFilter and search parameters
- **findEventsWithCursor()** - Implements cursor-based pagination with configurable sorting
- **findDistinctUserIds()** - Retrieves unique user identifiers from event collection
- **findDistinctEventTypes()** - Retrieves distinct event types for filtering/categorization
- **isSortableField()** - Validates if a field can be used for sorting operations
- **getDefaultSortField()** - Returns the default field for sorting when none specified

## Usage Example

```java
@Autowired
private CustomEventRepository eventRepository;

public void queryEvents() {
    // Build filtered query
    EventQueryFilter filter = new EventQueryFilter();
    filter.setUserId("user123");
    Query query = eventRepository.buildEventQuery(filter, "search term");
    
    // Get paginated results with cursor
    List<Event> events = eventRepository.findEventsWithCursor(
        query, 
        "cursor_token", 
        20, 
        "timestamp", 
        "desc"
    );
    
    // Get distinct values for filtering
    List<String> userIds = eventRepository.findDistinctUserIds();
    List<String> eventTypes = eventRepository.findDistinctEventTypes();
    
    // Validate sorting field
    if (eventRepository.isSortableField("customField")) {
        // Use custom sorting
    } else {
        String defaultSort = eventRepository.getDefaultSortField();
    }
}
```
