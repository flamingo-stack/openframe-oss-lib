This reactive repository interface provides MongoDB data access for integrated tools using Spring WebFlux's reactive programming model. It extends both Spring Data's ReactiveMongoRepository and a custom base repository interface for tool-specific operations.

## Key Components

- **ReactiveIntegratedToolRepository**: Main repository interface for reactive integrated tool data access
- **@ConditionalOnWebApplication(REACTIVE)**: Ensures this repository is only active in reactive web applications
- **IntegratedTool**: MongoDB document entity representing integrated tools
- **BaseIntegratedToolRepository**: Custom base interface providing common tool operations with reactive return types
- **findByType()**: Query method to find tools by their type identifier

## Usage Example

```java
@Service
public class ToolService {
    
    @Autowired
    private ReactiveIntegratedToolRepository toolRepository;
    
    public Mono<IntegratedTool> getToolByType(String toolType) {
        return toolRepository.findByType(toolType)
            .switchIfEmpty(Mono.error(new ToolNotFoundException("Tool not found: " + toolType)));
    }
    
    public Mono<IntegratedTool> saveTool(IntegratedTool tool) {
        return toolRepository.save(tool);
    }
    
    public Mono<Boolean> toolExists(String toolId) {
        return toolRepository.existsById(toolId);
    }
}
```

The repository automatically provides standard CRUD operations through ReactiveMongoRepository while adding custom tool-specific queries. All operations return reactive types (Mono/Flux) for non-blocking data access in reactive applications.
