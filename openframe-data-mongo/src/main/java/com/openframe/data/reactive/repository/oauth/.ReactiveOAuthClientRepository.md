A reactive MongoDB repository interface for managing OAuth client documents with non-blocking operations.

## Key Components

- **ReactiveOAuthClientRepository**: Main repository interface extending `ReactiveMongoRepository<OAuthClient, String>`
- **findByClientId()**: Custom query method to find OAuth clients by their client ID
- **Inherited CRUD operations**: Standard reactive operations like `save()`, `findById()`, `deleteById()`, etc.

## Usage Example

```java
@Service
public class OAuthService {
    
    @Autowired
    private ReactiveOAuthClientRepository clientRepository;
    
    public Mono<OAuthClient> getClientByClientId(String clientId) {
        return clientRepository.findByClientId(clientId);
    }
    
    public Mono<OAuthClient> saveClient(OAuthClient client) {
        return clientRepository.save(client);
    }
    
    public Mono<Void> removeClient(String id) {
        return clientRepository.deleteById(id);
    }
}
```

The repository provides reactive, non-blocking database operations for OAuth client management, returning `Mono` and `Flux` types for asynchronous processing in Spring WebFlux applications.
