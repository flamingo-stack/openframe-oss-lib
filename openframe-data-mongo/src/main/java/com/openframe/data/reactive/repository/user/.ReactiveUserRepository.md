Reactive MongoDB repository interface for user operations using Spring WebFlux reactive programming model. Only available in reactive web applications.

## Key Components

- **ReactiveUserRepository** - Extends `ReactiveMongoRepository` and `BaseUserRepository` with reactive return types
- **@ConditionalOnWebApplication(REACTIVE)** - Ensures repository is only active in reactive web contexts
- **Reactive return types** - All methods return `Mono<T>` for non-blocking operations

## Key Methods

- `findByEmail(String email)` - Finds user by email address reactively
- `existsByEmail(String email)` - Checks if user exists by email
- `existsByEmailAndStatus(String email, UserStatus status)` - Checks user existence by email and status

## Usage Example

```java
@Service
public class ReactiveUserService {
    
    @Autowired
    private ReactiveUserRepository userRepository;
    
    public Mono<User> getUserByEmail(String email) {
        return userRepository.findByEmail(email)
            .switchIfEmpty(Mono.error(new UserNotFoundException()));
    }
    
    public Mono<Boolean> isActiveUser(String email) {
        return userRepository.existsByEmailAndStatus(email, UserStatus.ACTIVE);
    }
    
    public Mono<User> createUser(User user) {
        return userRepository.existsByEmail(user.getEmail())
            .filter(exists -> !exists)
            .switchIfEmpty(Mono.error(new UserAlreadyExistsException()))
            .then(userRepository.save(user));
    }
}
```
