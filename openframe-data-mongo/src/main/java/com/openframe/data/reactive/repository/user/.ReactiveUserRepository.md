A reactive MongoDB repository interface for User entities that provides asynchronous, non-blocking database operations using Spring WebFlux.

## Key Components

- **ReactiveUserRepository**: Main repository interface extending `ReactiveMongoRepository` and `BaseUserRepository`
- **@ConditionalOnWebApplication(REACTIVE)**: Ensures this repository is only active in reactive web applications
- **Mono<User> findByEmail(String email)**: Finds user by email address reactively
- **Mono<Boolean> existsByEmail(String email)**: Checks if user exists by email
- **Mono<Boolean> existsByEmailAndStatus(String email, UserStatus status)**: Validates user existence with specific status

## Usage Example

```java
@Service
public class ReactiveUserService {
    
    @Autowired
    private ReactiveUserRepository userRepository;
    
    public Mono<User> getUserByEmail(String email) {
        return userRepository.findByEmail(email)
            .switchIfEmpty(Mono.error(new UserNotFoundException()));
    }
    
    public Mono<Boolean> isActiveUser(String email) {
        return userRepository.existsByEmailAndStatus(email, UserStatus.ACTIVE);
    }
    
    public Mono<User> createUser(User user) {
        return userRepository.existsByEmail(user.getEmail())
            .flatMap(exists -> exists ? 
                Mono.error(new UserAlreadyExistsException()) : 
                userRepository.save(user));
    }
}
```
