This reactive MongoDB repository interface provides non-blocking, asynchronous operations for managing tenant entities in multi-tenant applications.

## Key Components

- **ReactiveTenantRepository**: Main repository interface extending `ReactiveMongoRepository` and `BaseTenantRepository`
- **findByDomain()**: Asynchronously finds a tenant by domain name, returning `Mono<Tenant>`
- **existsByDomain()**: Asynchronously checks if a tenant exists for a given domain, returning `Mono<Boolean>`
- **@ConditionalOnWebApplication**: Ensures this repository is only active in reactive web applications
- **Reactive Types**: All operations return `Mono` types for non-blocking execution

## Usage Example

```java
@Service
public class TenantService {
    
    @Autowired
    private ReactiveTenantRepository tenantRepository;
    
    public Mono<Tenant> getTenantByDomain(String domain) {
        return tenantRepository.findByDomain(domain)
            .switchIfEmpty(Mono.error(new TenantNotFoundException()));
    }
    
    public Mono<Boolean> validateTenant(String domain) {
        return tenantRepository.existsByDomain(domain);
    }
    
    public Mono<Tenant> createTenant(Tenant tenant) {
        return tenantRepository.existsByDomain(tenant.getDomain())
            .flatMap(exists -> exists 
                ? Mono.error(new DuplicateTenantException())
                : tenantRepository.save(tenant));
    }
}
```
