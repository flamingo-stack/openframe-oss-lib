A reactive Spring Data MongoDB repository interface for managing tenant entities in multi-tenant applications, specifically designed for reactive web applications.

## Key Components

- **ReactiveTenantRepository** - Main repository interface extending ReactiveMongoRepository and BaseTenantRepository
- **findByDomain()** - Reactive method to find a tenant by domain name
- **existsByDomain()** - Reactive method to check if a tenant exists by domain
- **Conditional Loading** - Only active in reactive web application contexts

## Usage Example

```java
@Service
public class TenantService {
    
    @Autowired
    private ReactiveTenantRepository tenantRepository;
    
    public Mono<Tenant> getTenantByDomain(String domain) {
        return tenantRepository.findByDomain(domain)
            .switchIfEmpty(Mono.error(new TenantNotFoundException("Tenant not found for domain: " + domain)));
    }
    
    public Mono<Boolean> validateTenantExists(String domain) {
        return tenantRepository.existsByDomain(domain);
    }
    
    public Mono<Tenant> createTenant(Tenant tenant) {
        return tenantRepository.existsByDomain(tenant.getDomain())
            .flatMap(exists -> exists ? 
                Mono.error(new DuplicateTenantException("Tenant already exists")) :
                tenantRepository.save(tenant));
    }
}
```

The repository automatically provides reactive CRUD operations through ReactiveMongoRepository and includes custom domain-based lookup methods that return Mono types for non-blocking operations.
