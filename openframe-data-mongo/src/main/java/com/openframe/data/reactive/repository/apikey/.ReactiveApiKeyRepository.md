Reactive MongoDB repository interface for API key management in Spring WebFlux applications. Extends both ReactiveMongoRepository and BaseApiKeyRepository to provide reactive CRUD operations and custom query methods for API key entities.

## Key Components

- **ReactiveApiKeyRepository** - Main repository interface extending ReactiveMongoRepository
- **findByIdAndUserId()** - Finds API key by ID and user ID with reactive Mono return type
- **findByUserId()** - Retrieves all API keys for a specific user as reactive Flux
- **findExpiredKeys()** - Queries for expired but enabled keys using custom MongoDB query

## Usage Example

```java
@Service
public class ReactiveApiKeyService {
    
    @Autowired
    private ReactiveApiKeyRepository apiKeyRepository;
    
    public Mono<ApiKey> getApiKey(String keyId, String userId) {
        return apiKeyRepository.findByIdAndUserId(keyId, userId);
    }
    
    public Flux<ApiKey> getUserApiKeys(String userId) {
        return apiKeyRepository.findByUserId(userId);
    }
    
    public Flux<ApiKey> cleanupExpiredKeys() {
        Instant now = Instant.now();
        return apiKeyRepository.findExpiredKeys(now)
            .flatMap(apiKey -> {
                apiKey.setEnabled(false);
                return apiKeyRepository.save(apiKey);
            });
    }
}
```

The repository is automatically configured for reactive web applications and provides type-safe reactive operations with Mono for single results and Flux for multiple results. Custom MongoDB queries use Spring Data's @Query annotation for optimized database operations.
