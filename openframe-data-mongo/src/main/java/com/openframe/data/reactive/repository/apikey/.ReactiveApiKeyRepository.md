A reactive MongoDB repository interface for managing API key documents, designed for Spring WebFlux applications with non-blocking operations.

## Key Components

- **ReactiveApiKeyRepository** - Main repository interface extending `ReactiveMongoRepository` and `BaseApiKeyRepository`
- **findByIdAndUserId()** - Finds an API key by ID and user ID using custom MongoDB query
- **findByUserId()** - Retrieves all API keys for a specific user
- **findExpiredKeys()** - Locates expired but enabled API keys based on timestamp

## Usage Example

```java
@Service
public class ApiKeyService {
    
    @Autowired
    private ReactiveApiKeyRepository apiKeyRepository;
    
    public Mono<ApiKey> getUserApiKey(String keyId, String userId) {
        return apiKeyRepository.findByIdAndUserId(keyId, userId);
    }
    
    public Flux<ApiKey> getAllUserKeys(String userId) {
        return apiKeyRepository.findByUserId(userId);
    }
    
    public Mono<Void> cleanupExpiredKeys() {
        return apiKeyRepository.findExpiredKeys(Instant.now())
            .flatMap(apiKeyRepository::delete)
            .then();
    }
}
```

The repository is conditionally activated only for reactive web applications and provides non-blocking database operations using MongoDB's reactive driver with Reactor types (`Mono` and `Flux`).
