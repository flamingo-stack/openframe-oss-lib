A Jakarta Bean Validation constraint validator that ensures tenant domain names conform to a specific format based on a configurable base domain.

## Key Components

- **TenantDomainValidator**: Main validator class implementing `ConstraintValidator<TenantDomain, String>`
- **baseDomain**: Configurable property (`openframe.tenancy.base-domain`) defining the required domain suffix
- **isValid()**: Core validation method that checks domain format and slug compliance

## Usage Example

```java
// Define the constraint annotation (typically in a separate file)
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = TenantDomainValidator.class)
public @interface TenantDomain {
    String message() default "Invalid tenant domain format";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// Apply validation to entity fields
public class TenantConfig {
    @TenantDomain
    private String domain;
    
    // getters/setters
}

// Configure base domain in application.properties
// openframe.tenancy.base-domain=.example.com

// Valid domains: "tenant1.example.com", "my-tenant.example.com"
// Invalid domains: "tenant1.other.com", "sub.tenant.example.com", "example.com"
```

The validator ensures domains end with the configured base domain and have a valid slug prefix without subdomain nesting.
