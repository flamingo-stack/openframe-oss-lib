A Spring component that validates tenant domain strings against a configurable base domain pattern, ensuring they follow proper slug formatting rules.

## Key Components

- **TenantDomainValidator**: Main validator class implementing `ConstraintValidator<TenantDomain, String>`
- **baseDomain**: Configurable property from `openframe.tenancy.base-domain`
- **isValid()**: Core validation method that checks domain format and slug compliance

## Usage Example

```java
// Configuration
// application.properties: openframe.tenancy.base-domain=.example.com

// Entity with validation
public class TenantConfig {
    @TenantDomain
    private String domain;
    
    // Valid examples:
    // "tenant1.example.com"
    // "my-company.example.com"
    
    // Invalid examples:
    // "tenant.subdomain.example.com" (contains dots in slug part)
    // "tenant@example.com" (invalid slug characters)
    // "example.com" (missing slug part)
    // "tenant.wrongdomain.com" (doesn't match base domain)
}

// Validation usage
@RestController
public class TenantController {
    @PostMapping("/tenants")
    public ResponseEntity<?> createTenant(@Valid @RequestBody TenantConfig config) {
        // Domain is automatically validated
        return ResponseEntity.ok(config);
    }
}
```

The validator ensures tenant domains have exactly one subdomain level before the configured base domain and that the subdomain follows URL-safe slug formatting rules.
