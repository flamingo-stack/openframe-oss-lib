A custom Bean Validation annotation that validates email format using a configurable regular expression pattern. This annotation integrates with Jakarta Bean Validation to provide email validation for fields and method parameters.

## Key Components

- **@ValidEmail** - Custom validation annotation with configurable regex pattern
- **Target Elements** - Can be applied to fields and method parameters
- **Default Regex** - Comprehensive email pattern matching standard email formats
- **ValidEmailValidator** - Referenced validator class that performs the actual validation logic

## Usage Example

```java
// Field validation
public class User {
    @ValidEmail
    private String email;
    
    @ValidEmail(message = "Please provide a valid corporate email")
    private String workEmail;
}

// Method parameter validation
public class UserService {
    public void registerUser(@ValidEmail String email, String name) {
        // Method implementation
    }
}

// Custom regex pattern
public class Contact {
    @ValidEmail(
        regex = "^[a-zA-Z0-9._%+-]+@company\\.com$",
        message = "Must be a company email address"
    )
    private String corporateEmail;
}
```

The annotation automatically integrates with validation frameworks and can be customized with different regex patterns and error messages as needed.
