A custom Bean Validation annotation for validating email addresses with configurable regex patterns and error messages.

## Key Components

- **@ValidEmail** - Custom validation annotation that can be applied to fields and parameters
- **message()** - Customizable validation error message (defaults to "Invalid email format")
- **regex()** - Configurable email validation regex pattern with a comprehensive default
- **groups()** - Standard Bean Validation groups for conditional validation
- **payload()** - Standard Bean Validation payload for metadata

## Usage Example

```java
public class User {
    @ValidEmail
    private String email;
    
    @ValidEmail(message = "Please provide a valid work email")
    private String workEmail;
    
    @ValidEmail(regex = "^[a-zA-Z0-9._%+-]+@company\\.com$")
    private String corporateEmail;
}

// Method parameter validation
public void registerUser(@ValidEmail String email, String name) {
    // Registration logic
}

// Custom validation groups
public class WorkGroup {}

@ValidEmail(groups = WorkGroup.class, message = "Work email required")
private String businessEmail;
```

The annotation integrates seamlessly with Jakarta Bean Validation framework and requires the `ValidEmailValidator` class for actual validation logic implementation.
