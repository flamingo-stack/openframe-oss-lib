Custom validation annotation for validating tenant domain formats in multi-tenant applications. Uses Jakarta Bean Validation framework with a dedicated validator implementation.

## Key Components

- **@TenantDomain** - Custom validation annotation that can be applied to fields and parameters
- **TenantDomainValidator** - Referenced validator class that contains the actual validation logic
- **Default message** - "Invalid domain format" error message for validation failures
- **Jakarta Validation** - Built on standard Bean Validation constraints

## Usage Example

```java
public class TenantRegistration {
    @TenantDomain
    private String domainName;
    
    @TenantDomain(message = "Company domain must be valid")
    private String companyDomain;
}

// Method parameter validation
public void createTenant(@TenantDomain String domain) {
    // Implementation
}

// Custom validation groups
public class TenantUpdate {
    @TenantDomain(groups = ValidationGroups.Update.class)
    private String newDomain;
}
```

The annotation integrates seamlessly with Spring Boot's validation framework and can be used anywhere standard Jakarta validation annotations are supported. Validation logic is encapsulated in the `TenantDomainValidator` class.
