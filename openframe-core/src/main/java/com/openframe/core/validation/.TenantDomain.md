A custom Bean Validation annotation that validates tenant domain formats in OpenFrame applications. This annotation can be applied to fields or method parameters to ensure domain strings meet specific tenant domain requirements.

## Key Components

- **@TenantDomain** - Custom validation annotation interface
- **TenantDomainValidator** - Referenced validator class (implements validation logic)
- **Standard validation attributes** - message, groups, and payload for JSR-303 compliance
- **Target constraints** - Applicable to fields and parameters only

## Usage Example

```java
public class TenantConfiguration {
    @TenantDomain
    private String primaryDomain;
    
    @TenantDomain(message = "Custom domain validation failed")
    private String secondaryDomain;
    
    public void updateDomain(@TenantDomain String newDomain) {
        this.primaryDomain = newDomain;
    }
}

// In a service class
@Service
public class TenantService {
    public void createTenant(@Valid TenantConfiguration config) {
        // Validation automatically triggered by @Valid
        // TenantDomainValidator will validate domain fields
    }
}
```

The annotation integrates with Spring's validation framework and can be used with `@Valid` to automatically trigger domain format validation during object binding or method parameter validation.
