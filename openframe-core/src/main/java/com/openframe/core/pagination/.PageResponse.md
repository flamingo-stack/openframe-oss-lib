Generic pagination response wrapper used to standardize paginated API responses across the OpenFrame platform, encapsulating a page of items alongside metadata like totals and navigation state.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `items` | `List<T>` | The current page's data payload |
| `page` | `int` | Current page index (zero or one-based per convention) |
| `size` | `int` | Number of items per page |
| `totalElements` | `long` | Total number of records across all pages |
| `totalPages` | `int` | Total number of available pages |
| `hasNext` | `boolean` | Whether a subsequent page exists |

Lombok annotations `@Data`, `@SuperBuilder`, `@NoArgsConstructor`, and `@AllArgsConstructor` auto-generate getters, setters, builder, and constructors.

## Usage Example

```java
// Building a PageResponse from a service layer result
List<TicketDto> tickets = ticketService.findTickets(page, size);
long total = ticketService.countTickets();

PageResponse<TicketDto> response = PageResponse.<TicketDto>builder()
    .items(tickets)
    .page(page)
    .size(size)
    .totalElements(total)
    .totalPages((int) Math.ceil((double) total / size))
    .hasNext(page + 1 < (int) Math.ceil((double) total / size))
    .build();

return ResponseEntity.ok(response);
```

> **Tip:** Because `PageResponse` uses `@SuperBuilder`, it can be safely subclassed and extended with domain-specific fields while retaining full builder support via Lombok's `@SuperBuilder` inheritance chain.
