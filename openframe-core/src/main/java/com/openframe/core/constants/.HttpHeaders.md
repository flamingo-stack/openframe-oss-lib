A utility class that centralizes HTTP header name constants for consistent usage across the OpenFrame application, preventing hardcoded strings and reducing typos.

## Key Components

**Header Categories:**
- **Authentication Headers**: API keys, user IDs, and refresh tokens (`X_API_KEY`, `X_USER_ID`, `X_REFRESH_TOKEN`)
- **User Context Headers**: User profile information (`X_USER_EMAIL`, `X_USER_FIRST_NAME`, `X_USER_LAST_NAME`)
- **Client Context Headers**: Client identification and permissions (`X_CLIENT_ID`, `X_CLIENT_SCOPES`)
- **Standard HTTP Headers**: Common protocol headers (`AUTHORIZATION`, `CONTENT_TYPE`, `ACCEPT`)
- **Rate Limiting Headers**: Custom rate limit tracking per time period (`X_RATE_LIMIT_*`)
- **Proxy Headers**: Forwarded request information (`X_FORWARDED_PROTO`, `X_FORWARDED_HOST`)
- **Media Types**: Content type constants (`APPLICATION_JSON`)

## Usage Example

```java
import static com.openframe.core.constants.HttpHeaders.*;

// Setting request headers
request.setHeader(X_API_KEY, apiKey);
request.setHeader(X_USER_ID, userId);
request.setHeader(CONTENT_TYPE, APPLICATION_JSON);

// Reading response headers
String remainingRequests = response.getHeader(X_RATE_LIMIT_REMAINING_HOUR);
String userEmail = response.getHeader(X_USER_EMAIL);

// Configuring proxy headers
response.setHeader(X_FORWARDED_PROTO, "https");
response.setHeader(X_FORWARDED_HOST, "api.openframe.com");
```

This constants class ensures type safety and consistency when working with HTTP headers throughout the application, making code more maintainable and reducing the risk of header name mismatches.
