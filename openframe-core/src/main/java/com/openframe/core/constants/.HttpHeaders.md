A utility class that centralizes HTTP header name constants used throughout the OpenFrame application to avoid hardcoded strings and ensure consistency.

## Key Components

- **Authentication Headers**: API key, user ID, and token-related headers (`X_API_KEY`, `X_USER_ID`, `X_REFRESH_TOKEN`)
- **User Context Headers**: User profile information headers (`X_USER_EMAIL`, `X_USER_FIRST_NAME`, `X_USER_LAST_NAME`)
- **Client Context Headers**: Client identification and scope headers (`X_CLIENT_ID`, `X_CLIENT_SCOPES`)
- **Standard HTTP Headers**: Common headers like `AUTHORIZATION`, `CONTENT_TYPE`, `ACCEPT`
- **Rate Limiting Headers**: Custom rate limit tracking for minute/hour/day intervals
- **Forwarding Headers**: Proxy-related headers (`X_FORWARDED_PROTO`, `X_FORWARDED_HOST`)
- **Media Types**: Content type constants like `APPLICATION_JSON`

## Usage Example

```java
// Setting headers in a request
request.setHeader(HttpHeaders.AUTHORIZATION, "Bearer " + token);
request.setHeader(HttpHeaders.CONTENT_TYPE, HttpHeaders.APPLICATION_JSON);
request.setHeader(HttpHeaders.X_API_KEY, apiKey);

// Reading headers in a controller
@GetMapping("/api/users")
public ResponseEntity<?> getUsers(
    @RequestHeader(HttpHeaders.X_USER_ID) String userId,
    @RequestHeader(HttpHeaders.X_CLIENT_ID) String clientId) {
    // Handle request
}

// Adding rate limit headers to response
response.setHeader(HttpHeaders.X_RATE_LIMIT_REMAINING_HOUR, "45");
```
