A standardized Data Transfer Object (DTO) for representing error responses in API communications, providing consistent error formatting across the OpenFrame application.

## Key Components

- **code**: Optional error code field for categorizing errors
- **message**: Required error message providing human-readable error details
- **@JsonInclude(NON_NULL)**: Ensures null fields are excluded from JSON serialization
- **Lombok annotations**: Automatically generates constructors, getters, setters, and utility methods

## Usage Example

```java
// Create error response with message only
ErrorResponse simpleError = new ErrorResponse("User not found");

// Create error response with code and message
ErrorResponse detailedError = new ErrorResponse("USER_NOT_FOUND", "User with ID 123 not found");

// Create empty error response and set fields
ErrorResponse error = new ErrorResponse();
error.setCode("VALIDATION_ERROR");
error.setMessage("Invalid input parameters");

// JSON serialization example
// {"message": "User not found"} - code field omitted when null
// {"code": "USER_NOT_FOUND", "message": "User with ID 123 not found"}
```

The class supports flexible error reporting by allowing either simple message-only errors or detailed errors with specific error codes, making it suitable for both internal error handling and client-facing API responses.
