A generic paginated response wrapper that encapsulates paginated data along with metadata for pagination controls and navigation.

## Key Components

- **Generic Type Parameter `<T>`** - Allows wrapping any type of data in paginated responses
- **items** - The list of data items for the current page
- **page** - Current page number (typically 0-indexed)
- **size** - Number of items per page
- **totalElements** - Total count of all items across all pages
- **totalPages** - Total number of pages available
- **hasNext** - Boolean flag indicating if more pages exist

## Usage Example

```java
// Create a paginated response for user data
PageResponse<User> userPage = PageResponse.<User>builder()
    .items(userList)
    .page(0)
    .size(10)
    .totalElements(150L)
    .totalPages(15)
    .hasNext(true)
    .build();

// Access pagination metadata
int currentPage = userPage.getPage();
boolean canPaginate = userPage.isHasNext();
List<User> users = userPage.getItems();

// Typical REST API response
@GetMapping("/users")
public PageResponse<User> getUsers(@RequestParam int page, @RequestParam int size) {
    // Service returns PageResponse<User>
    return userService.findPaginated(page, size);
}
```

The class uses Lombok annotations for automatic generation of getters, setters, constructors, and builder patterns, making it ideal for REST API responses that need pagination support.
