A generic pagination response wrapper that encapsulates paginated data along with metadata for pagination controls and navigation.

## Key Components

- **Generic Type `T`**: Allows wrapping any type of data in the pagination response
- **`items`**: List of actual data items for the current page
- **`page`**: Current page number (0-based indexing)
- **`size`**: Number of items per page
- **`totalElements`**: Total count of all items across all pages
- **`totalPages`**: Total number of pages available
- **`hasNext`**: Boolean flag indicating if more pages exist

## Usage Example

```java
// Creating a paginated response for user data
List<User> users = userService.getUsersForPage(0, 10);
long totalUsers = userService.getTotalUserCount();

PageResponse<User> response = PageResponse.<User>builder()
    .items(users)
    .page(0)
    .size(10)
    .totalElements(totalUsers)
    .totalPages((int) Math.ceil((double) totalUsers / 10))
    .hasNext(totalUsers > 10)
    .build();

// Accessing pagination metadata
boolean canLoadMore = response.isHasNext();
int currentPage = response.getPage();
List<User> currentUsers = response.getItems();
```

This DTO is commonly used in REST APIs to return paginated results with complete pagination context, enabling frontend applications to build proper pagination controls.
