A custom runtime exception class designed to handle encryption-related errors with structured error codes and messages.

## Key Components

- **EncryptionException**: Main exception class extending `RuntimeException`
- **errorCode**: String field to store specific error codes for categorizing encryption failures
- **Constructors**: Two overloaded constructors supporting message-only and message-with-cause scenarios
- **@Getter**: Lombok annotation providing automatic getter method for the errorCode field

## Usage Example

```java
// Throwing encryption exception with error code
if (keyGenerationFailed) {
    throw new EncryptionException("ENC001", "Failed to generate encryption key");
}

// Throwing with underlying cause
try {
    cipher.doFinal(data);
} catch (BadPaddingException e) {
    throw new EncryptionException("ENC002", "Decryption failed due to invalid padding", e);
}

// Catching and handling
try {
    encryptionService.encrypt(sensitiveData);
} catch (EncryptionException e) {
    logger.error("Encryption error [{}]: {}", e.getErrorCode(), e.getMessage());
    // Handle based on error code
    switch (e.getErrorCode()) {
        case "ENC001" -> handleKeyError();
        case "ENC002" -> handlePaddingError();
    }
}
```
