A custom runtime exception class used to handle failures in API key operations within the OpenFrame core system.

## Key Components

- **ApiKeyException** - Custom runtime exception class extending `RuntimeException`
- **Constructor (String)** - Creates exception with error message
- **Constructor (String, Throwable)** - Creates exception with error message and underlying cause

## Usage Example

```java
// Throwing an API key exception with message only
if (apiKey == null || apiKey.isEmpty()) {
    throw new ApiKeyException("API key cannot be null or empty");
}

// Throwing an API key exception with cause
try {
    validateApiKey(apiKey);
} catch (ValidationException e) {
    throw new ApiKeyException("API key validation failed", e);
}

// Catching and handling API key exceptions
try {
    authenticateUser(apiKey);
} catch (ApiKeyException e) {
    logger.error("Authentication failed: " + e.getMessage());
    // Handle the exception appropriately
}
```

This exception is typically thrown during API key validation, authentication, or other API key-related operations that fail due to invalid keys, missing keys, or underlying system errors.
