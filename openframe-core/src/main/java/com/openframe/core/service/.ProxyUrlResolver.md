Service responsible for resolving proxy URLs to route requests from the OpenFrame gateway to integrated external tools, with environment-aware host mapping.

## Key Components

**Main Methods:**
- `resolve()` - Constructs target URIs for proxying requests to integrated tools
- `isLocalProfile()` - Determines if application is running in local development mode

**Core Dependencies:**
- `Environment` - Spring environment for profile detection
- `UriComponentsBuilder` - URI construction and manipulation
- `@Service` - Spring service component annotation

## Usage Example

```java
@Autowired
private ProxyUrlResolver proxyUrlResolver;

public void routeToolRequest() {
    // Original request to /tools/my-tool/api/data?param=value
    URI originalUri = URI.create("/tools/my-tool/api/data?param=value");
    String toolUrl = "https://external-tool.com";
    String toolPort = "8080";
    
    // Resolve target URL for proxying
    URI targetUri = proxyUrlResolver.resolve(
        "my-tool",           // Tool ID
        toolUrl,             // Tool base URL from config
        toolPort,            // Tool port from config  
        originalUri,         // Original request URI
        "/tools"             // Prefix to strip
    );
    
    // Result: http://localhost:8080/api/data?param=value (local profile)
    // Result: https://external-tool.com:8080/api/data?param=value (other profiles)
}
```

The service automatically maps external tool hosts to `localhost` when running in local development profile, enabling seamless development workflows while maintaining production URL configurations.
