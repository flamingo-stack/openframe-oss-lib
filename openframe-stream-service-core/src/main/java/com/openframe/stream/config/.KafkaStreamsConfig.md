Configures Kafka Streams for the OpenFrame stream processing service, defining serialization, consumer/producer tuning, and application identity with support for both tenant and SaaS deployment models.

## Key Components

| Bean / Method | Description |
|---|---|
| `kStreamsConfig()` | Primary `KafkaStreamsConfiguration` bean wiring all Streams properties (bootstrap servers, threading, state dir, processing guarantee) |
| `activityMessageSerde()` | JSON `Serde<ActivityMessage>` using the shared `ObjectMapper` for Debezium activity messages |
| `hostActivityMessageSerde()` | JSON `Serde<HostActivityMessage>` for Debezium host-activity messages |
| `outgoingActivityMessageSerde()` | Outgoing `JsonSerde<ActivityMessage>` with type-info headers disabled for downstream consumers |
| `buildStreamsApplicationId()` | Namespaces the Streams `application.id` as `{app-name}-{cluster-id}`, falling back to `{app-name}` when `clusterId` is blank |

**Configuration properties:**

| Property | Purpose |
|---|---|
| `spring.oss-tenant.kafka.bootstrap-servers` | Tenant-specific Kafka broker (takes precedence) |
| `spring.saas.kafka.bootstrap-servers` | Shared/SaaS Kafka broker fallback |
| `openframe.cluster-id` | Tenant/cluster identifier for Streams `application.id` namespacing |
| `kafka.stream.enabled` | Feature flag; defaults to `true` |

**Notable tuning defaults:** `AT_LEAST_ONCE` processing guarantee, 1 stream thread, `MAX_TASK_IDLE_MS = 5000 ms` (ensures `leftJoin` windows close when `host-activities` topic is sparse), consumer `auto.offset.reset = earliest`.

## Usage Example

```java
// application.yml (tenant deployment)
spring:
  application:
    name: openframe-stream
  oss-tenant:
    kafka:
      bootstrap-servers: broker:9092
openframe:
  cluster-id: tenant-y0-1
kafka:
  stream:
    enabled: true

// Resulting Streams application.id → "openframe-stream-tenant-y0-1"
```

```java
// Inject custom Serdes into a Streams topology
@Autowired
private Serde<ActivityMessage> activityMessageSerde;

KStream<String, ActivityMessage> stream = builder.stream(
    "activities",
    Consumed.with(Serdes.String(), activityMessageSerde)
);
```
