Configuration class that sets up Kafka Streams processing for OpenFrame's fleet activity data, defining serializers, stream properties, and tenant-aware application identifiers.

## Key Components

- **Serde Beans**: Custom serializers/deserializers for `ActivityMessage` and `HostActivityMessage` using Jackson ObjectMapper
- **Stream Configuration**: Core Kafka Streams settings including bootstrap servers, processing guarantees, and state management
- **Tenant Support**: Multi-tenant application ID generation using cluster-id for namespace isolation
- **Performance Tuning**: Optimized consumer/producer settings for batch processing and memory management

## Usage Example

```java
@Service
public class FleetDataProcessor {
    
    @Autowired
    private Serde<ActivityMessage> activityMessageSerde;
    
    @StreamListener("input-topic")
    public void processFleetActivity(
        KStream<String, ActivityMessage> activityStream) {
        
        activityStream
            .filter((key, value) -> value.getAfter() != null)
            .mapValues(activity -> enrichActivity(activity))
            .to("enriched-activities", 
                Produced.with(Serdes.String(), activityMessageSerde));
    }
}
```

The configuration automatically handles:
- **At-least-once processing** guarantees for reliable data delivery
- **Window operations** with 5-second idle timeout for join operations
- **State store management** in `/tmp/kafka-streams` directory
- **Multi-tenant isolation** via cluster-id prefixed application IDs
