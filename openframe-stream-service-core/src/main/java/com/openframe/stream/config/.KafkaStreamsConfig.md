Configuration class that sets up Kafka Streams processing with custom serialization for fleet activity messages and optimized stream processing properties.

## Key Components

- **Serde Beans**: Custom serializers/deserializers for `ActivityMessage` and `HostActivityMessage` using Jackson ObjectMapper
- **Stream Configuration**: Main Kafka Streams configuration with performance tuning for processing guarantees, threading, and batch settings
- **Property Injection**: Bootstrap servers and application name from Spring configuration properties

## Usage Example

```java
@Component
public class ActivityStreamProcessor {
    
    @Autowired
    private Serde<ActivityMessage> activityMessageSerde;
    
    @StreamListener
    public void processActivityStream(StreamsBuilder builder) {
        builder.stream("activity-topic", Consumed.with(Serdes.String(), activityMessageSerde))
               .filter((key, activity) -> activity.getAfter() != null)
               .to("processed-activity-topic");
    }
}
```

The configuration provides optimized settings including:
- **At-least-once processing guarantee** for reliability
- **Single stream thread** for simplified processing
- **Batch processing** with 100 max poll records and 10ms linger time
- **Early offset reset** to process all available messages
- **Custom JSON serialization** without type information for cleaner output
