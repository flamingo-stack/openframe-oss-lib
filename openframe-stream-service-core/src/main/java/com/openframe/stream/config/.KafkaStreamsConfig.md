Configuration class that sets up Kafka Streams processing with custom serializers and stream processing properties for handling fleet activity messages.

## Key Components

- **Custom Serdes**: `activityMessageSerde()`, `hostActivityMessageSerde()`, and `outgoingActivityMessageSerde()` for JSON serialization of fleet activity messages
- **Stream Configuration**: `kStreamsConfig()` bean that configures Kafka Streams with processing guarantees, threading, and consumer/producer settings
- **Dependencies**: Integrates with Spring Boot using `@Value` properties and ObjectMapper for JSON handling

## Usage Example

```java
@Autowired
private Serde<ActivityMessage> activityMessageSerde;

@Autowired
private KafkaStreamsConfiguration streamsConfig;

// Using the custom serde in a stream topology
StreamsBuilder builder = new StreamsBuilder();
KStream<String, ActivityMessage> activityStream = builder
    .stream("activity-topic", Consumed.with(Serdes.String(), activityMessageSerde));

// The configuration is automatically applied to all Kafka Streams in the application
// Stream processing will use:
// - AT_LEAST_ONCE processing guarantee
// - Single stream thread
// - Consumer starting from earliest offset
// - Optimized producer batching (16KB batch size, 10ms linger)
```

The configuration enables `@EnableKafkaStreams` and provides the default streams configuration bean that Spring Kafka will automatically use for all stream processing topologies in the application.
