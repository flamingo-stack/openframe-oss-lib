Abstract base class for deserializing Kafka/Debezium change-data-capture events from integrated MSP tools into a unified `DeserializedDebeziumMessage` format.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `deserialize()` | Method | Core deserialization pipeline: extracts timestamp, event type, composite ID, and builds the unified message |
| `buildDetailsJson()` | Private Method | Merges error, result, and additional dynamic fields into a single JSON details blob |
| `generateCompositeId()` | Private Method | Produces a deterministic UUID from `tool_table_id_value` or falls back to a content-hash key |
| `extractTableName()` | Private Method | Resolves table/collection name from Debezium source metadata (PostgreSQL/MySQL vs. MongoDB) |
| `getAgentId()` | Abstract | Tool-specific agent ID extraction |
| `getSourceEventType()` | Abstract | Tool-specific event type string extraction |
| `getEventToolId()` | Abstract | Tool-specific primary key extraction |
| `getMessage()` | Abstract | Tool-specific human-readable message extraction |
| `getDetails()` | Abstract | Tool-specific additional details JSON |
| `getError()` / `getResult()` | Overridable | Optional error/result JSON; default returns `null` |
| `getSourceEventTimestamp()` | Overridable | Tool-specific timestamp; falls back to Debezium processing time |
| `parseStringField()` | Utility | Null-safe `JsonNode` field extraction shared across subclasses |
| `putIfPresent()` | Static Utility | Type-aware `ObjectNode` population, skips `null` values |

## Usage Example

```java
public class MyToolEventDeserializer extends IntegratedToolEventDeserializer {

    public MyToolEventDeserializer(ObjectMapper mapper) {
        super(mapper,
            List.of("SKIP_EVENT_TYPE"),     // events to skip processing
            List.of("INTERNAL_EVENT_TYPE")  // events hidden from clients
        );
    }

    @Override
    protected Optional<String> getAgentId(JsonNode after) {
        return parseStringField(after, "agent_id");
    }

    @Override
    protected Optional<String> getSourceEventType(JsonNode after) {
        return parseStringField(after, "event_type");
    }

    @Override
    protected Optional<String> getEventToolId(JsonNode after) {
        return parseStringField(after, "id");
    }

    @Override
    protected Optional<String> getMessage(JsonNode after) {
        return parseStringField(after, "description");
    }

    @Override
    protected String getDetails(JsonNode after) {
        ObjectNode node = mapper.createObjectNode();
        putIfPresent(node, "severity", after.path("severity").asText(null));
        return node.toString();
    }

    @Override
    protected Optional<Long> getSourceEventTimestamp(JsonNode after) {
        return parseStringField(after, "created_at")
            .map(Long::parseLong);
    }
}
```

> **Idempotency:** `generateCompositeId()` always produces the same UUID for the same event — safe for Kafka at-least-once delivery. If no primary key is available, a content hash of `tool + table + payload` is used as the fallback key.
