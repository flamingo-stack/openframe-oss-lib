A Spring component that deserializes and processes MeshCentral event messages, extracting key fields like agent ID, event type, and message content from JSON data structures.

## Key Components

- **MeshCentralEventDeserializer**: Main deserializer class extending `IntegratedToolEventDeserializer`
- **Field extraction methods**: `getAgentId()`, `getSourceEventType()`, `getEventToolId()`, `getMessage()`
- **JSON parsing utilities**: `parseJson()`, `parseAndExtractField()`, `extractField()`, `extractEventId()`
- **Timestamp handling**: `getSourceEventTimestamp()` using ISO8601 format
- **Message type**: Returns `MessageType.MESHCENTRAL_EVENT`

## Usage Example

```java
@Autowired
private MeshCentralEventDeserializer deserializer;

// Process MeshCentral event data
JsonNode eventData = // ... raw event JSON node
Optional<String> agentId = deserializer.getAgentId(eventData);
Optional<String> eventType = deserializer.getSourceEventType(eventData);
Optional<String> message = deserializer.getMessage(eventData);

// The deserializer handles complex JSON structures like:
// {"nodeid": "agent123", "etype": "node", "action": "connect", 
//  "_id": {"$oid": "507f1f77bcf86cd799439011"}, "msg": "Device connected"}
```

The class handles MeshCentral's specific JSON structure, including MongoDB ObjectId extraction from nested `_id.$oid` fields and combines event type and action fields into composite event identifiers.
