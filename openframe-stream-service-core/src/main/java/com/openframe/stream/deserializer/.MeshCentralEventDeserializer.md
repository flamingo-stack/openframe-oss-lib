Deserializes MeshCentral integration events from raw JSON node payloads into structured OpenFrame stream messages, extending the base `IntegratedToolEventDeserializer`.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `MeshCentralEventDeserializer(ObjectMapper)` | Constructor | Registers the deserializer for `SERVER_TIMELINE_STATS` source event type |
| `getType()` | Method | Returns `MessageType.MESHCENTRAL_EVENT` as the canonical message type |
| `getAgentId(JsonNode)` | Method | Extracts the `nodeid` field identifying the managed device/agent |
| `getSourceEventType(JsonNode)` | Method | Combines `etype` and `action` fields into a composite event type (e.g. `node.connected`) |
| `getEventToolId(JsonNode)` | Method | Extracts the event `_id`, supporting both MongoDB `$oid` format and plain string values |
| `getMessage(JsonNode)` | Method | Extracts the human-readable `msg` field |
| `getTenantId(JsonNode)` | Method | Extracts the `domain` field as the tenant identifier |
| `getSourceEventTimestamp(JsonNode)` | Method | Parses the `time.$date` epoch millisecond timestamp |
| `parseJson(JsonNode)` | Private | Safely deserializes a `JsonNode` containing a JSON string into a nested `JsonNode` |

## Usage Example

```java
// Injected automatically via Spring component scan
@Autowired
MeshCentralEventDeserializer deserializer;

// Deserialize a raw MeshCentral CDC event payload
JsonNode afterNode = objectMapper.readTree("{\"nodeid\":\"node//abc123\",\"etype\":\"node\",\"action\":\"connected\",\"msg\":\"Device online\",\"domain\":\"acme\",\"_id\":{\"$oid\":\"64a1f0c2b3e4d5f6a7b8c9d0\"},\"time\":{\"$date\":1720000000000}}");

Optional<String> agentId  = deserializer.getAgentId(afterNode);        // "node//abc123"
Optional<String> eventType = deserializer.getSourceEventType(afterNode); // "node.connected"
Optional<Long>   timestamp = deserializer.getSourceEventTimestamp(afterNode); // 1720000000000L
```

## Notes

- Raw field values arrive as JSON strings embedded inside a `JsonNode`; `parseJson()` handles the double-deserialization step before field extraction.
- `_id` resolution supports both MongoDB extended JSON (`$oid`) and plain string identifiers.
- `getDetails()` returns `null` — MeshCentral events carry no additional detail payload in the current implementation.
