Deserializer for Fleet MDM policy membership events, transforming raw Kafka/stream JSON payloads into structured `FLEET_MDM_POLICY_MEMBERSHIP_EVENT` messages with pass/fail classification and enriched policy metadata.

## Key Components

| Member | Type | Description |
|---|---|---|
| `FleetPolicyMembershipEventDeserializer` | `@Component` | Spring-managed deserializer extending `IntegratedToolEventDeserializer` |
| `getType()` | Method | Returns `MessageType.FLEET_MDM_POLICY_MEMBERSHIP_EVENT` |
| `getAgentId()` | Method | Extracts `host_id` as the target agent identifier |
| `getSourceEventType()` | Method | Maps `passes` boolean to `POLICY_MEMBERSHIP_PASS` or `POLICY_MEMBERSHIP_FAIL` |
| `getEventToolId()` | Method | Builds composite ID from `policy_id + "_" + host_id` |
| `getMessage()` | Method | Generates human-readable message, enriched with policy name from cache |
| `getResult()` / `getError()` | Methods | Routes policy detail JSON to result or error field based on pass/fail status |
| `buildPolicyDetailsJson()` | Method | Constructs JSON with policy name, query, resolution, description, and critical flag |
| `getPolicyInfo()` | Method | Looks up `Policy` object via `FleetMdmCacheService` using `policy_id` |

## Usage Example

```java
// Injected automatically by Spring; invoked by the stream processing pipeline
@Autowired
FleetPolicyMembershipEventDeserializer deserializer;

// Determine message type handled by this deserializer
MessageType type = deserializer.getType();
// → MessageType.FLEET_MDM_POLICY_MEMBERSHIP_EVENT

// Example: resolving event type from a parsed JsonNode payload
// afterField = { "policy_id": 42, "host_id": 7, "passes": false, "updated_at": "2024-01-15T10:30:00Z" }
Optional<String> eventType = deserializer.getSourceEventType(afterField);
// → Optional.of("POLICY_MEMBERSHIP_FAIL")

Optional<String> toolId = deserializer.getEventToolId(afterField);
// → Optional.of("42_7")

Optional<String> message = deserializer.getMessage(afterField);
// → Optional.of("Policy 'Disk Encryption' failed on host")
```

## Notes

- Falls back to `POLICY_MEMBERSHIP_FAIL` when `passes` is `null`, treating unknown states as failures.
- Timestamp resolution prefers `updated_at`, falling back to `created_at` via `TimestampParser.parseIso8601`.
- Policy metadata enrichment is cache-backed via `FleetMdmCacheService`; missing policies degrade gracefully with a generic message.
