A specialized deserializer that processes Fleet MDM (Mobile Device Management) events from JSON data streams, extracting agent information, activity types, and timestamps for event processing.

## Key Components

- **FleetEventDeserializer** - Main class extending `IntegratedToolEventDeserializer`
- **getAgentId()** - Extracts agent ID from Fleet event data
- **getSourceEventType()** - Retrieves activity type from the "activity_type" field
- **getMessage()** - Maps activity types to human-readable messages using `FleetActivityTypeMapping`
- **getSourceEventTimestamp()** - Parses ISO8601 timestamps from "created_at" field
- **getType()** - Returns `MessageType.FLEET_MDM_EVENT` for type identification

## Usage Example

```java
@Autowired
private FleetEventDeserializer fleetDeserializer;

// Process Fleet MDM event from JSON
JsonNode fleetEventJson = objectMapper.readTree(jsonString);
Optional<String> agentId = fleetDeserializer.getAgentId(fleetEventJson);
Optional<String> activityType = fleetDeserializer.getSourceEventType(fleetEventJson);
Optional<String> message = fleetDeserializer.getMessage(fleetEventJson);

// Get message type for routing
MessageType type = fleetDeserializer.getType(); // Returns FLEET_MDM_EVENT
```

The deserializer handles Fleet-specific field mappings and provides fallback mechanisms for missing data, with built-in logging for unmapped activity types.
