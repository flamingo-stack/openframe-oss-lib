A Spring component that deserializes Fleet MDM events from JSON messages, extending the base integrated tool event deserializer to handle Fleet-specific event data structures and field mappings.

## Key Components

- **FleetEventDeserializer** - Main deserializer class extending `IntegratedToolEventDeserializer`
- **Field Constants** - Defines Fleet-specific JSON field names (`agentId`, `activity_type`, `id`, etc.)
- **Override Methods** - Implements abstract methods for extracting Fleet-specific data:
  - `getAgentId()` - Extracts agent identifier from Fleet events
  - `getSourceEventType()` - Maps to Fleet's `activity_type` field
  - `getEventToolId()` - Uses Fleet's unique `id` field
  - `getMessage()` - Converts activity types to human-readable messages via `FleetActivityTypeMapping`
  - `getSourceEventTimestamp()` - Parses ISO8601 timestamps from `created_at` field
  - `getDetails()` - Extracts additional event details
  - `getType()` - Returns `MessageType.FLEET_MDM_EVENT`

## Usage Example

```java
@Autowired
private FleetEventDeserializer fleetDeserializer;

// Process Fleet MDM event JSON
String fleetEventJson = """
{
  "agentId": "agent-123",
  "activity_type": "app_installed", 
  "id": "fleet-event-456",
  "details": "Chrome browser installed",
  "created_at": "2024-01-15T10:30:00Z"
}
""";

// Deserialize Fleet event
JsonNode eventData = objectMapper.readTree(fleetEventJson);
Optional<String> agentId = fleetDeserializer.getAgentId(eventData);
Optional<String> message = fleetDeserializer.getMessage(eventData);
MessageType type = fleetDeserializer.getType(); // Returns FLEET_MDM_EVENT
```
