A specialized event deserializer for processing Fleet MDM query execution results, extending IntegratedToolEventDeserializer to handle query success/failure messages and metadata enrichment.

## Key Components

- **FleetQueryResultEventDeserializer** - Main deserializer class for Fleet query result events
- **getAgentId()** - Extracts host_id as the agent identifier from query results
- **getMessage()** - Generates descriptive messages based on query execution status (success/failure)
- **getError()** - Processes error information when query execution fails
- **getResult()** - Extracts and formats successful query output data
- **getQueryInfo()** - Retrieves cached query metadata using query_id
- **FleetMdmCacheService** - Injected service for accessing query metadata

## Usage Example

```java
@Autowired
private FleetQueryResultEventDeserializer deserializer;

// The deserializer processes Fleet query result events automatically
// When a query result event is received, it will:

// 1. Extract agent ID from host_id field
String agentId = deserializer.getAgentId(eventData);

// 2. Generate appropriate message based on execution status
String message = deserializer.getMessage(eventData);
// Returns: "Query 'disk_usage' executed successfully" 
//       or "Query 'process_list' execution failed: timeout error"

// 3. Format error information if query failed
String errorDetails = deserializer.getError(eventData);

// 4. Format successful query results with metadata
String results = deserializer.getResult(eventData);
```
