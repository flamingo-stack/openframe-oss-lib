A specialized deserializer for processing Fleet MDM query result events from the change data capture (CDC) stream, handling both successful query executions and error scenarios while enriching results with cached query metadata.

## Key Components

- **FleetQueryResultEventDeserializer** - Main deserializer class extending `IntegratedToolEventDeserializer`
- **getType()** - Returns `MessageType.FLEET_MDM_QUERY_RESULT_EVENT` message type identifier
- **getAgentId()** - Extracts the host/device ID that executed the query from `host_id` field
- **getMessage()** - Generates user-friendly messages based on query success/failure status
- **getError()** - Processes error information and enriches with query metadata
- **getResult()** - Handles successful query results, parsing JSON data and adding query SQL
- **getQueryInfo()** - Retrieves full query metadata from cache using `query_id`

## Usage Example

```java
@Autowired
private FleetQueryResultEventDeserializer deserializer;

// Process a Fleet query result CDC event
JsonNode cdcEvent = // ... CDC event from Kafka
Optional<IntegratedToolEvent> event = deserializer.deserialize(cdcEvent);

if (event.isPresent()) {
    IntegratedToolEvent toolEvent = event.get();
    
    // Access enriched query result information
    String message = toolEvent.getMessage(); // "Query 'system_info' executed successfully"
    String result = toolEvent.getResult();   // {"output": [...], "query": "SELECT * FROM system_info"}
    String agentId = toolEvent.getAgentId(); // "host-123-abc"
}
```

The deserializer automatically enriches query results with cached query names and SQL, providing comprehensive context for downstream event processing and user notifications.
