Deserializes Fleet MDM query result events from raw JSON into structured stream messages, mapping query execution outcomes (success, error, or empty result) to the `FLEET_MDM_QUERY_RESULT_EVENT` message type.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `FleetQueryResultEventDeserializer` | `@Component` | Spring-managed deserializer extending `IntegratedToolEventDeserializer` |
| `getType()` | Method | Returns `MessageType.FLEET_MDM_QUERY_RESULT_EVENT` |
| `getAgentId()` | Method | Extracts `host_id` as the device/agent identifier |
| `getSourceEventType()` | Method | Always returns `EXECUTE_SCHEDULED_QUERY` |
| `getEventToolId()` | Method | Extracts the query result's unique `id` field |
| `getMessage()` | Method | Builds a human-readable status string; enriched with query name from cache |
| `getError()` | Method | Serializes error field + query SQL into a structured JSON string |
| `getResult()` | Method | Serializes `data` field + query SQL into a structured JSON string |
| `getQueryInfo()` | Private | Resolves full `Query` metadata from `FleetMdmCacheService` using `query_id` |
| `getQueryName()` | Private | Convenience wrapper returning only the query name from cache |

## Usage Example

```java
// Resolved automatically by Spring; triggered during stream event processing
FleetQueryResultEventDeserializer deserializer =
    new FleetQueryResultEventDeserializer(objectMapper, fleetMdmCacheService);

// Identify which event type this handles
MessageType type = deserializer.getType();
// → MessageType.FLEET_MDM_QUERY_RESULT_EVENT

// Extract a human-readable message from the raw event payload
JsonNode afterField = objectMapper.readTree("""
    {
      "id": "42",
      "host_id": "device-001",
      "query_id": 7,
      "data": "[{\\"cpu\\":12}]",
      "error": null
    }
""");

Optional<String> message = deserializer.getMessage(afterField);
// → Optional["Query 'CPU Usage' executed successfully"]
// (query name resolved from FleetMdmCacheService by query_id=7)
```

> **Cache dependency:** Query names and SQL are resolved at runtime via `FleetMdmCacheService.getQueryById()`. If the cache is uninitialized or the query has been deleted, name enrichment is skipped and a warning is logged — the event is still processed with a generic message.
