A Spring service that processes generic JSON messages from Kafka using Debezium CDC (Change Data Capture) patterns, providing a flexible pipeline for deserialization, enrichment, and handling based on message type.

## Key Components

- **GenericJsonMessageProcessor**: Main service class with `@Service` annotation for Spring dependency injection
- **Constructor**: Builds lookup maps from injected lists of handlers, enrichment services, and deserializers
- **process()**: Main processing method that orchestrates deserialization, enrichment, and handling
- **deserialize()**: Converts raw Debezium messages to typed objects using appropriate deserializers
- **getExtraParams()**: Enriches messages with additional data using configured enrichment services

## Usage Example

```java
@Autowired
private GenericJsonMessageProcessor processor;

public void handleIncomingMessage(CommonDebeziumMessage message) {
    // Process a user data change message
    processor.process(message, MessageType.USER_UPDATE);
    
    // Process an order event message
    processor.process(message, MessageType.ORDER_CREATED);
}

// The processor automatically:
// 1. Deserializes the message based on type
// 2. Enriches data using appropriate service
// 3. Routes to all configured destination handlers
// 4. Handles errors for unsupported types
```

The processor uses a strategy pattern with dependency injection to handle different message types, destinations, and enrichment requirements without tight coupling.
