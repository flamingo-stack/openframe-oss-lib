A Spring service that processes Kafka messages using a flexible chain of deserialization, enrichment, and handling components. It routes messages to appropriate handlers based on message type and destination.

## Key Components

- **Constructor**: Builds lookup maps for handlers (grouped by event type and destination), data enrichment services, and deserializers
- **process()**: Main processing method that deserializes messages, enriches data, and routes to handlers
- **deserialize()**: Converts CommonDebeziumMessage to DeserializedDebeziumMessage using type-specific deserializers
- **getExtraParams()**: Enriches messages with additional data using configured enrichment services

## Usage Example

```java
@Autowired
private GenericJsonMessageProcessor processor;

public void handleKafkaMessage(CommonDebeziumMessage message) {
    // Process a fleet data message
    processor.process(message, MessageType.FLEET_UPDATE);
    
    // The processor will:
    // 1. Deserialize the message using the appropriate deserializer
    // 2. Enrich data using the configured enrichment service
    // 3. Route to all handlers for the message's destination list
}
```

The processor requires lists of MessageHandler, DataEnrichmentService, and KafkaMessageDeserializer implementations to be injected via constructor for proper routing and processing.
