A Spring component that handles Debezium change data capture (CDC) messages by transforming them into IntegratedToolEvent objects and publishing them to Kafka topics.

## Key Components

- **DebeziumKafkaMessageHandler**: Main handler class extending `DebeziumMessageHandler`
- **transform()**: Converts `DeserializedDebeziumMessage` to `IntegratedToolEvent` 
- **handleCreate/Read/Update()**: CRUD operation handlers that publish to Kafka
- **isValidMessage()**: Message validation based on visibility flag
- **buildMessageBrokerKey()**: Generates Kafka message keys using device/user ID and tool type
- **OssTenantRetryingKafkaProducer**: Injected Kafka producer for reliable message publishing

## Usage Example

```java
@Configuration
public class StreamConfig {
    
    @Bean
    public DebeziumKafkaMessageHandler messageHandler(
            OssTenantRetryingKafkaProducer kafkaProducer,
            ObjectMapper objectMapper) {
        return new DebeziumKafkaMessageHandler(kafkaProducer, objectMapper);
    }
}

// The handler processes Debezium messages automatically
// when integrated into a stream processing pipeline
DeserializedDebeziumMessage debeziumMsg = // ... from CDC stream
IntegratedToolEnrichedData enrichedData = // ... enrichment data
IntegratedToolEvent event = handler.transform(debeziumMsg, enrichedData);
// Event is automatically published to configured Kafka topic
```
