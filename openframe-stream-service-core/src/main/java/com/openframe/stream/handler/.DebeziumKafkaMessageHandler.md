A specialized Debezium message handler that transforms database change events into `IntegratedToolEvent` messages and publishes them to Kafka. This handler processes tool-related database changes and forwards them as structured events to downstream consumers.

## Key Components

- **`transform()`** - Converts Debezium messages into `IntegratedToolEvent` objects with enriched organizational and device data
- **`handleCreate/Read/Update()`** - CRUD operation handlers that publish events to Kafka (delete operations are ignored)
- **`buildMessageBrokerKey()`** - Generates Kafka message keys based on device ID, user ID, or tool type for proper partitioning
- **`isValidMessage()`** - Filters messages based on visibility flag to process only relevant events
- **`getType()/getDestination()`** - Returns handler metadata for routing and classification

## Usage Example

```java
@Configuration
public class StreamConfig {
    
    @Bean
    public DebeziumKafkaMessageHandler kafkaHandler(
            OssTenantRetryingKafkaProducer producer,
            ObjectMapper mapper) {
        return new DebeziumKafkaMessageHandler(producer, mapper);
    }
}

// The handler automatically processes Debezium messages:
// 1. Validates message visibility
// 2. Transforms to IntegratedToolEvent with enriched data
// 3. Publishes to configured Kafka topic with partitioning key
// 4. Handles all CRUD operations except deletes
```

The handler extends `DebeziumMessageHandler` to provide Kafka-specific functionality, ensuring reliable message delivery through the retrying producer and proper message routing via generated keys.
