Interface defining the contract for processing Debezium messages with enriched data in the OpenFrame streaming system. Handlers implement specific processing logic for different event types and destinations.

## Key Components

- **Generic Type Parameters**: 
  - `U extends DeserializedDebeziumMessage` - The deserialized Debezium message type
  - `V extends IntegratedToolEnrichedData` - Additional enriched data from integrated tools
- **getType()** - Returns the specific event handler type for routing
- **getDestination()** - Specifies the target destination for processed messages  
- **handle()** - Core processing method that transforms messages with enriched context

## Usage Example

```java
@Component
public class UserEventHandler implements MessageHandler<UserDebeziumMessage, UserEnrichedData> {
    
    @Override
    public EventHandlerType getType() {
        return EventHandlerType.USER_EVENTS;
    }
    
    @Override
    public Destination getDestination() {
        return Destination.ANALYTICS_SERVICE;
    }
    
    @Override
    public void handle(UserDebeziumMessage message, UserEnrichedData enrichedData) {
        // Process user change events with enriched profile data
        processUserUpdate(message.getPayload(), enrichedData.getProfileMetadata());
        forwardToAnalytics(message, enrichedData);
    }
}
```

This interface enables polymorphic message processing where different handlers can be registered and invoked based on event type and routing requirements.
