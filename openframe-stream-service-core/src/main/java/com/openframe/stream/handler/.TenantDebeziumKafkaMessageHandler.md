Tenant-specific implementation of `DebeziumKafkaMessageHandler` that routes Debezium CDC (Change Data Capture) events to the configured outbound Kafka topic for integrated tool events in a multi-tenant OpenFrame environment.

## Key Components

| Element | Description |
|---|---|
| `TenantDebeziumKafkaMessageHandler` | Spring component extending `DebeziumKafkaMessageHandler` for tenant-scoped CDC event routing |
| `@ConditionalOnBean(OssTenantRetryingKafkaProducer.class)` | Bean only activated when `OssTenantRetryingKafkaProducer` is present in the context |
| `topic` | Injected via `openframe.oss-tenant.kafka.topics.outbound.integrated-tool-events` property |
| `getTopic()` | Overrides parent method to supply the tenant-specific outbound topic name |

## Usage Example

```java
// application.yml / application.properties configuration required:
// openframe.oss-tenant.kafka.topics.outbound.integrated-tool-events=tenant.integrated-tool-events

// Bean is auto-registered by Spring when OssTenantRetryingKafkaProducer is available.
// Inject and use via the parent handler interface:

@Autowired
private TenantDebeziumKafkaMessageHandler tenantHandler;

// The handler will automatically publish Debezium CDC events
// to the configured tenant outbound topic.
tenantHandler.handle(debeziumEvent);
```

## Notes

- Requires `OssTenantRetryingKafkaProducer` to be registered as a Spring bean; otherwise this component is skipped entirely.
- The outbound topic must be defined in the application properties or environment config.
- Delegates producer logic, serialization, and validation to the parent `DebeziumKafkaMessageHandler` via constructor injection.
