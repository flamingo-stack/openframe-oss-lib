Validates incoming Debezium events by ensuring a `tenantId` is present before downstream processing. Acts as the primary `DebeziumEventValidator` implementation in the Spring context.

## Key Components

| Element | Description |
|---|---|
| `TenantIdRequiredDebeziumEventValidator` | Primary Spring component implementing `DebeziumEventValidator` |
| `isValid(DeserializedDebeziumMessage)` | Returns `false` and logs an error if `tenantId` is null or blank; `true` otherwise |

## Usage Example

```java
@Component
public class DebeziumEventHandler {

    private final DebeziumEventValidator validator;

    public DebeziumEventHandler(DebeziumEventValidator validator) {
        this.validator = validator;
    }

    public void handle(DeserializedDebeziumMessage message) {
        if (!validator.isValid(message)) {
            return; // Event dropped — missing tenantId
        }
        // Continue processing...
    }
}
```

## Notes

- Annotated with `@Primary` — this implementation takes precedence if multiple `DebeziumEventValidator` beans exist.
- Failed validation logs at `ERROR` level including the `toolEventId` for traceability.
- Designed to guard multi-tenant event pipelines, preventing tenant-less events from propagating through the stream.
