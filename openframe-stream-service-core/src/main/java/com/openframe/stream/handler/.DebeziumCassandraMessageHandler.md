Handles the persistence of Debezium CDC (Change Data Capture) messages into Apache Cassandra by transforming deserialized Debezium events into `UnifiedLogEvent` entities and saving them to the repository.

## Key Components

| Element | Description |
|---|---|
| `DebeziumCassandraMessageHandler` | Spring component extending `DebeziumMessageHandler`, conditionally active when Cassandra is enabled |
| `transform()` | Maps a `DeserializedDebeziumMessage` and `IntegratedToolEnrichedData` into a `UnifiedLogEvent` |
| `createKey()` | Builds the `UnifiedLogEventKey` composite key from tenant, tool type, event type, timestamp, and ingest day |
| `handleCreate/Read/Update()` | All write operations delegate to `repository.save()` |
| `handleDelete()` | No-op — delete events are intentionally ignored |
| `getDestination()` | Returns `Destination.CASSANDRA` to route messages to this handler |
| `getType()` | Returns `EventHandlerType.COMMON_TYPE` for general event handling |

## Usage Example

```java
// Bean is auto-registered when Cassandra is on the classpath
// and spring.data.cassandra.enabled=true is set in application properties.

// application.yml
// spring:
//   data:
//     cassandra:
//       enabled: true

// The handler is invoked internally by the stream processing pipeline:
DeserializedDebeziumMessage msg = ...; // deserialized Kafka message
IntegratedToolEnrichedData enriched = ...; // enriched device/org context

// transform() produces a UnifiedLogEvent ready for Cassandra persistence
UnifiedLogEvent event = handler.transform(msg, enriched);

// Key structure example:
// tenantId | ingestDay | toolType | eventType | eventTimestamp | toolEventId
```

> **Note:** This handler is conditionally loaded via `@ConditionalOnClass(CassandraRepository.class)` and `@ConditionalOnProperty`. Ensure both the Cassandra driver is on the classpath and the property is set to `true`, otherwise the bean will not be registered.
