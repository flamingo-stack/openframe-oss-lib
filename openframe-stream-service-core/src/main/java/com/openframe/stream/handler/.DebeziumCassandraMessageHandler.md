A Spring component that handles Debezium CDC messages and transforms them into unified log events for storage in Cassandra database.

## Key Components

- **DebeziumCassandraMessageHandler** - Main handler class extending `DebeziumMessageHandler` for Cassandra-specific message processing
- **transform()** - Core method that converts Debezium messages into `UnifiedLogEvent` objects with enriched metadata
- **createKey()** - Generates composite keys for Cassandra storage using timestamp, tool type, and event identifiers
- **CRUD handlers** - Methods for handling create, read, update, and delete operations (all redirect to save except delete)
- **Type descriptors** - Returns `COMMON_TYPE` handler type and `CASSANDRA` destination for routing

## Usage Example

```java
// Inject the handler in a Kafka consumer or message processor
@Autowired
private DebeziumCassandraMessageHandler handler;

// Process incoming Debezium message
DeserializedDebeziumMessage debeziumMsg = // ... from Kafka
IntegratedToolEnrichedData enrichedData = // ... enrichment service

// Handler will transform and save to Cassandra
UnifiedLogEvent logEvent = handler.transform(debeziumMsg, enrichedData);

// The handler automatically routes based on operation type:
// CREATE, READ, UPDATE -> saves to Cassandra
// DELETE -> no-op (data retention policy)
```

The handler maps Debezium CDC events to a unified log format, enriching them with organizational metadata and storing them in Cassandra with time-partitioned composite keys for efficient querying.
