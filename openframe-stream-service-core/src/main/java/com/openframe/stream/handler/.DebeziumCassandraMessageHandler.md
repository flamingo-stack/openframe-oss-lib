Handles Debezium messages by transforming them into UnifiedLogEvent objects and persisting them to Cassandra. This handler extends the base DebeziumMessageHandler to provide Cassandra-specific data transformation and storage logic.

## Key Components

- **DebeziumCassandraMessageHandler**: Main handler class extending `DebeziumMessageHandler<UnifiedLogEvent, DeserializedDebeziumMessage>`
- **transform()**: Transforms Debezium messages into UnifiedLogEvent objects using enriched data
- **createKey()**: Creates composite keys for Cassandra storage with timestamp, tool type, and event details
- **handleCreate/Read/Update()**: CRUD operations that save data to Cassandra repository
- **getType()**: Returns `EventHandlerType.COMMON_TYPE`
- **getDestination()**: Returns `Destination.CASSANDRA`

## Usage Example

```java
@Autowired
private DebeziumCassandraMessageHandler handler;

@Autowired
private CassandraRepository repository;

@Autowired
private ObjectMapper objectMapper;

// Handler is automatically instantiated by Spring
DebeziumCassandraMessageHandler cassandraHandler = 
    new DebeziumCassandraMessageHandler(repository, objectMapper);

// Process a Debezium message (typically called by framework)
DeserializedDebeziumMessage debeziumMessage = // ... obtained from stream
IntegratedToolEnrichedData enrichedData = // ... enrichment data
UnifiedLogEvent logEvent = cassandraHandler.transform(debeziumMessage, enrichedData);

// Data is automatically saved to Cassandra via handleCreate/Read/Update methods
```
