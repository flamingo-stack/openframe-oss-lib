Pluggable validation interface for Debezium CDC events, allowing custom validation logic to be applied during Kafka message handling beyond the built-in visibility check.

## Key Components

| Component | Description |
|-----------|-------------|
| `DebeziumEventValidator` | Functional interface consumed by `DebeziumKafkaMessageHandler` |
| `isValid(DeserializedDebeziumMessage)` | Single validation method; return `true` to pass the event downstream, `false` to drop it |

## Usage Example

```java
// Default pass-through impl (tenant cluster)
DebeziumEventValidator passThroughValidator = message -> true;

// SaaS shared cluster: drop events without a resolved tenantId
DebeziumEventValidator tenantAwareValidator = message -> {
    String tenantId = message.getTenantId();
    return tenantId != null && !tenantId.isBlank();
};

// Wiring into the handler (Spring bean example)
@Bean
public DebeziumEventValidator debeziumEventValidator() {
    return tenantAwareValidator;
}
```

## Notes

- Designed as a **Strategy pattern** — swap implementations per deployment without modifying the handler.
- Tenant cluster deployments typically use the identity (`message -> true`) implementation.
- SaaS shared cluster implementations should guard against events missing a resolved `tenantId` to prevent cross-tenant data leakage.
- Pairs with `DeserializedDebeziumMessage` for type-safe access to CDC event fields.
