Defines string constants for event types emitted by integrated MSP tools — MeshCentral and Fleet MDM — used for event stream mapping within the OpenFrame platform.

## Key Components

| Interface | Description |
|---|---|
| `SourceEventTypes` | Root interface grouping all tool-specific event type namespaces |
| `SourceEventTypes.MeshCentral` | Event type constants for MeshCentral remote management (server, user, node, relay, mesh, user-group events) |
| `SourceEventTypes.Fleet` | Event type constants for Fleet MDM (user, device, MDM, software, script, profile, and platform events) |

## Usage Example

```java
// Route an incoming event to the correct handler based on its source type
public void handleMeshCentralEvent(String eventType, JsonObject payload) {
    switch (eventType) {
        case SourceEventTypes.MeshCentral.NODE_ADD:
            deviceService.onDeviceAdded(payload);
            break;
        case SourceEventTypes.MeshCentral.USER_LOGIN:
            auditService.recordLogin(payload);
            break;
        default:
            log.warn("Unhandled MeshCentral event: {}", eventType);
    }
}

// Check Fleet MDM software events
if (SourceEventTypes.Fleet.INSTALLED_SOFTWARE.equals(incomingEvent)) {
    complianceService.recordInstall(payload);
}
```

## Notes

- All constants are `String` literals matching the raw event identifiers from each tool's API or webhook payload
- The nested interface pattern provides namespace isolation, preventing collisions between tools that share similar event names (e.g., `MeshCentral.USER_LOGIN` vs `Fleet.USER_LOGIN`)
