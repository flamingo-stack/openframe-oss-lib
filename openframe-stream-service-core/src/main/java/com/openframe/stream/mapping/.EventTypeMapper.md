Translates tool-specific event type strings from integrated MSP tools (MeshCentral, Tactical RMM) into a normalized `UnifiedEventType` enum, enabling consistent event processing across the OpenFrame platform.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `mappings` | `Map<String, UnifiedEventType>` | Static registry of `"toolName:sourceEvent"` → `UnifiedEventType` entries |
| `mapToUnifiedType()` | `static` method | Resolves a tool-specific event string to its `UnifiedEventType`; returns `UNKNOWN` if no mapping exists |
| `registerMapping()` | `static` method | Registers a single tool/event → unified type entry into the map |
| `initializeDefaultMappings()` | `static` method | Populates all default mappings at class load time for MeshCentral and Tactical RMM |

## Supported Event Categories

```text
MeshCentral     → user.*, mesh.*, node.*, relay.*, ugrp.*, server.*
Tactical RMM    → auth, device, user, script, task, policy, monitoring, group
```

## Usage Example

```java
// Resolve a MeshCentral login event to its unified type
UnifiedEventType type = EventTypeMapper.mapToUnifiedType(
    IntegratedToolType.MESHCENTRAL,
    SourceEventTypes.MeshCentral.USER_LOGIN
);
// Returns: UnifiedEventType.LOGIN

// Resolve a Tactical RMM script execution event
UnifiedEventType type = EventTypeMapper.mapToUnifiedType(
    IntegratedToolType.TACTICAL,
    SourceEventTypes.Tactical.AGENT_EXECUTE_SCRIPT
);
// Returns: UnifiedEventType.SCRIPT_EXECUTED

// Unmapped event falls back gracefully
UnifiedEventType type = EventTypeMapper.mapToUnifiedType(
    IntegratedToolType.MESHCENTRAL,
    "unknown.custom.event"
);
// Returns: UnifiedEventType.UNKNOWN
```

## Notes

- Keys follow the format `"toolDbName:sourceEventType"` to prevent collisions across tools
- All mappings are registered at class initialization via a `static` block — no runtime overhead
- Extend coverage by calling `registerMapping()` with new `IntegratedToolType` values as additional MSP tools are onboarded
