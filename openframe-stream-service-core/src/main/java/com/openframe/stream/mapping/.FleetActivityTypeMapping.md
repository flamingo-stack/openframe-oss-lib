A utility class that provides mapping between Fleet MDM activity type identifiers and their corresponding human-readable messages. This class serves as a centralized lookup table for translating Fleet's internal activity type codes into user-friendly descriptions for logging and display purposes.

## Key Components

**Static Methods:**
- `getMessage(String activityType)` - Returns an Optional containing the human-readable message for a given activity type
- `isSupported(String activityType)` - Checks if an activity type has a corresponding message mapping

**Static Fields:**
- `ACTIVITY_TYPE_MESSAGES` - Immutable map containing all 100+ activity type mappings organized by categories like user management, device enrollment, policy management, etc.

**Private Methods:**
- `createActivityTypeMessages()` - Factory method that builds the complete mapping of activity types to messages

## Usage Example

```java
import com.openframe.stream.mapping.FleetActivityTypeMapping;
import java.util.Optional;

// Get human-readable message for an activity type
Optional<String> message = FleetActivityTypeMapping.getMessage("created_policy");
if (message.isPresent()) {
    System.out.println("Activity: " + message.get()); // "Activity: Created policy"
}

// Check if activity type is supported
if (FleetActivityTypeMapping.isSupported("mdm_enrolled")) {
    System.out.println("Supported activity type");
}

// Handle unknown activity types
String unknownType = "unknown_activity";
String displayMessage = FleetActivityTypeMapping.getMessage(unknownType)
    .orElse("Unknown activity: " + unknownType);
```
