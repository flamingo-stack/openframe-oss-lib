A Spring Kafka listener service that consumes JSON messages from multiple integrated tool topics and delegates processing to a generic message processor.

## Key Components

- **JsonKafkaListener**: Main service class annotated with `@Service` for Spring dependency injection
- **messageProcessor**: Injected `GenericJsonMessageProcessor` that handles the actual message processing logic
- **listenIntegratedToolsEvents()**: Kafka listener method that consumes messages from four configured topics (MeshCentral, Tactical RMM, Fleet MDM events and query results)

## Usage Example

This listener automatically processes messages from integrated MSP tools when they arrive on the configured Kafka topics:

```java
// The listener is automatically registered with Spring Kafka
// Messages are consumed from these topics:
// - meshcentral-events
// - tactical-rmm-events  
// - fleet-mdm-events
// - fleet-mdm-query-result-events

// Example of what gets processed when a message arrives:
CommonDebeziumMessage message = // Debezium change data capture message
MessageType type = MessageType.CREATE; // From Kafka header

// The listener delegates to the processor:
messageProcessor.process(message, type);
```

The service uses Spring's `@KafkaListener` annotation with configurable topic names and consumer group ID through application properties. Each consumed message includes a Debezium payload and message type header that gets passed to the generic processor for handling.
