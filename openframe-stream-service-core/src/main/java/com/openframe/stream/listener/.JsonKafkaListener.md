Kafka listener bean that consumes Debezium change-data-capture events from multiple integrated-tool topics on the **tenant-cluster** and delegates processing to `GenericJsonMessageProcessor`.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Activates only when `openframe.cluster.mode=tenant` (default); skipped in shared SaaS deployments handled by `SharedMeshCentralKafkaListener` |
| `@KafkaListener` | Subscribes to four inbound topics: MeshCentral events, Fleet MDM events, Fleet MDM query results, and Fleet MDM policy membership |
| `listenIntegratedToolsEvents` | Entry point for all incoming Debezium messages; extracts `MessageType` from the Kafka header and forwards the payload to the processor |
| `GenericJsonMessageProcessor` | Injected dependency that owns the actual processing logic |

## Usage Example

```java
// Spring auto-instantiates this bean when the condition is met.
// No manual wiring required — ensure the following properties are set:

// application.yml (tenant mode)
openframe:
  cluster:
    mode: tenant
  oss-tenant:
    kafka:
      topics:
        inbound:
          meshcentral-events:
            name: meshcentral.events
          # ... remaining topics

spring:
  oss-tenant:
    kafka:
      consumer:
        group-id: openframe-stream-consumer-group

// Incoming Kafka record (producer side)
ProducerRecord<String, CommonDebeziumMessage> record =
    new ProducerRecord<>("meshcentral.events", debeziumMessage);
record.headers().add(KafkaHeader.MESSAGE_TYPE_HEADER, MessageType.INSERT.name().getBytes());
producer.send(record);
```

> **Deployment note:** For shared SaaS environments, set `openframe.cluster.mode=shared` to disable this bean and activate `SharedMeshCentralKafkaListener` instead.
