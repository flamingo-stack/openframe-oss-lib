A Spring Kafka listener service that consumes JSON messages from multiple integrated tools topics and delegates processing to a generic message processor.

## Key Components

- **JsonKafkaListener** - Main service class annotated with `@Service`
- **GenericJsonMessageProcessor** - Injected dependency for message processing logic
- **listenIntegratedToolsEvents()** - Kafka listener method that handles messages from multiple topics

## Usage Example

```java
// The listener automatically processes messages from configured topics
// No direct instantiation needed - Spring handles the lifecycle

// Example of what the listener receives:
@KafkaListener(topics = {"meshcentral-events", "tactical-rmm-events"})
public void listenIntegratedToolsEvents(
    @Payload CommonDebeziumMessage debeziumMessage, 
    @Header(KafkaHeader.MESSAGE_TYPE_HEADER) MessageType messageType
) {
    // Delegates to messageProcessor.process(debeziumMessage, messageType)
}

// Configuration in application.yml:
// openframe:
//   oss-tenant:
//     kafka:
//       topics:
//         inbound:
//           meshcentral-events: "meshcentral.events"
//           tactical-rmm-events: "tactical.rmm.events"
```

The listener consumes Debezium messages from four different integrated tool topics (MeshCentral, Tactical RMM, and Fleet MDM) and processes them using the injected message processor with the appropriate message type header.
