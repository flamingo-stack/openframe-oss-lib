A Spring service that provides cached access to Fleet MDM data for enriching stream processing activities with host and query information.

## Key Components

- **`getAgentId(Integer hostId)`** - Retrieves agent UUID for a given host ID with caching
- **`getQueryById(Long queryId)`** - Fetches query definitions by ID with caching  
- **`getFleetMdmClient()`** - Lazy initialization of Fleet MDM client using integrated tool credentials
- **Cache configurations**: `hostAgentCache` and `fleetQueryCache` for performance optimization

## Usage Example

```java
@Autowired
private FleetMdmCacheService fleetCacheService;

// Get agent ID for host enrichment
String agentId = fleetCacheService.getAgentId(12345);
if (agentId != null) {
    // Use agent ID for activity enrichment
    activity.setAgentId(agentId);
}

// Fetch query metadata for processing
Query queryDef = fleetCacheService.getQueryById(67890L);
if (queryDef != null) {
    // Use query definition for validation or metadata
    String queryName = queryDef.getName();
}
```

The service automatically handles Fleet MDM client initialization, caching for performance, and graceful error handling. Cache entries are only stored for successful (non-null) responses to avoid caching failures.
