Caching service for Fleet MDM data that wraps Spring Cache annotations around Fleet MDM API calls, reducing redundant API requests during activity stream processing.

## Key Components

| Method | Cache Name | Description |
|--------|------------|-------------|
| `getAgentId(Integer hostId)` | `hostAgentCache` | Resolves host ID → agent UUID |
| `getQueryById(Long queryId)` | `fleetQueryCache` | Fetches query metadata by ID |
| `getPolicyById(Long policyId)` | `fleetPolicyCache` | Fetches policy definition, returns `Optional<Policy>` |
| `evictPolicyCache(Long policyId)` | `fleetPolicyCache` | Evicts a stale policy entry on mutation events |
| `getFleetMdmClient()` *(private)* | — | Lazy-initializes `FleetMdmClient` from stored credentials |

**Cache behavior:** All `@Cacheable` methods skip caching on `null` results (`unless = "#result == null"`), preventing negative caching for missing or deleted resources.

## Usage Example

```java
@Autowired
private FleetMdmCacheService fleetMdmCacheService;

// Resolve host UUID (cached after first call)
String agentId = fleetMdmCacheService.getAgentId(1042);

// Fetch query metadata (cached after first call)
Query query = fleetMdmCacheService.getQueryById(7L);

// Fetch policy with Optional handling
Optional<Policy> policy = fleetMdmCacheService.getPolicyById(3L);
policy.ifPresent(p -> System.out.println("Policy: " + p.getName()));

// Evict stale policy after an edited_policy event
fleetMdmCacheService.evictPolicyCache(3L);
```

> **Note:** `FleetMdmClient` is lazily initialized on first use, requiring a valid Fleet integration entry with API credentials stored via `IntegratedToolService`. If credentials are missing, all lookups return `null`/`Optional.empty()` gracefully without throwing.
