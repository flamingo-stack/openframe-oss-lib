Resolves a canonical `tenantId` from a cluster-level identifier found in integrated-tool event payloads (e.g. MeshCentral `domain` fields), enabling shared SaaS deployments where a single upstream cluster serves multiple tenants.

## Key Components

| Member | Description |
|--------|-------------|
| `resolveTenantId(String clusterName)` | Maps a cluster-scoped identifier extracted from an event payload to a canonical tenant ID; returns `null` if no matching tenant is found |

## Usage Example

```java
// Implement for shared-cluster deployments
@Component
@ConditionalOnSharedCluster
public class MeshCentralTenantIdResolver implements ClusterTenantIdResolver {

    @Override
    public String resolveTenantId(String clusterName) {
        // Map MeshCentral domain -> tenantId
        return tenantRegistry.findByClusterName(clusterName)
                             .map(Tenant::getId)
                             .orElse(null);
    }
}

// Consumed by IntegratedToolDataEnrichmentService
@Service
public class IntegratedToolDataEnrichmentService {

    private final Optional<ClusterTenantIdResolver> clusterResolver;
    private final TenantIdProvider fallbackProvider;

    public String resolveTenant(String clusterName) {
        return clusterResolver
            .map(r -> r.resolveTenantId(clusterName))
            .orElseGet(fallbackProvider::getTenantId);
    }
}
```

## Notes

- **Shared cluster only** — this bean is not registered in per-tenant cluster deployments; `IntegratedToolDataEnrichmentService` falls back to `TenantIdProvider` when the bean is absent.
- Returning `null` signals no tenant match, allowing the caller to apply its own fallback strategy.
