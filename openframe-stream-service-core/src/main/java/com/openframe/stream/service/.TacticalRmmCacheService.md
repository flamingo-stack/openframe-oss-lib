This service provides caching functionality for Tactical RMM API operations, including agent and script data retrieval with Spring Cache abstraction for improved performance.

## Key Components

- **getAgentIdByPrimaryKey()** - Retrieves agent ID by primary key with cache-first strategy
- **getScriptNameById()** - Fetches script names by ID with caching support  
- **getAllAgentsFromCache()** - Returns all agents from cache or API
- **evictAllAgentsCache()** - Manually clears the agents cache
- **Cache configurations** - Uses Spring's `@Cacheable` and `@CacheEvict` annotations for three cache regions: `agentIdCache`, `scriptNameCache`, and `allAgentsCache`

## Usage Example

```java
@Autowired
private TacticalRmmCacheService cacheService;

// Get agent ID by primary key (cached)
String agentId = cacheService.getAgentIdByPrimaryKey(12345);

// Get script name by ID (cached)
String scriptName = cacheService.getScriptNameById(67890);

// Get all agents (cached)
List<AgentListItem> agents = cacheService.getAllAgentsFromCache();

// Force cache refresh
cacheService.evictAllAgentsCache();
agents = cacheService.getAllAgentsFromCache(); // Fresh data
```

The service automatically handles Tactical RMM client initialization using integrated tool configuration and provides fallback mechanisms when cache misses occur.
