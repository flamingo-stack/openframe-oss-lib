Caching layer for Tactical RMM API data, reducing repeated network calls by storing agent lists, script names, and task names using Spring Cache abstractions.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `getAgentIdByPrimaryKey(Integer)` | `@Cacheable` | Resolves an agent's string ID from its database primary key; auto-refreshes cache on miss |
| `getScriptNameById(Integer)` | `@Cacheable` | Returns a script's display name by script ID |
| `getTaskNameById(Integer)` | `@Cacheable` | Returns an automated task's display name by task ID |
| `getAllAgentsFromCache()` | `@Cacheable` | Fetches and caches the full agent list from the Tactical RMM API |
| `evictAllAgentsCache()` | `@CacheEvict` | Invalidates the agent list cache to force a fresh API call |
| `getTacticalRmmClient()` | `private` | Lazily initializes `TacticalRmmClient` using credentials from `IntegratedToolService` |

**Cache names:** `agentIdCache`, `scriptNameCache`, `taskNameCache`, `allAgentsCache`

## Usage Example

```java
// Resolve a Tactical RMM agent_id from a database primary key
String agentId = tacticalRmmCacheService.getAgentIdByPrimaryKey(42);

// Look up a script name (result cached after first call)
String scriptName = tacticalRmmCacheService.getScriptNameById(101);

// Look up an automated task name
String taskName = tacticalRmmCacheService.getTaskNameById(55);

// Manually invalidate the agent list (e.g., after an agent is added/removed)
tacticalRmmCacheService.evictAllAgentsCache();
```

> **Note:** The `TacticalRmmClient` is lazily initialized on first use. If the `TACTICAL_SERVER_ID` tool is not configured in `IntegratedToolService`, all methods return `null` gracefully. Cache entries with `null` results are never stored (`unless = "#result == null"`).
