This service provides cached access to Tactical RMM (Remote Monitoring and Management) data, using Spring Cache abstraction to optimize API calls and improve performance when retrieving agent and script information.

## Key Components

**Main Methods:**
- `getAgentIdByPrimaryKey(Integer primaryKey)` - Retrieves agent ID by primary key with cache fallback
- `getScriptNameById(Integer scriptId)` - Gets script name by ID from cache or API
- `getAllAgentsFromCache()` - Fetches all agents with caching enabled
- `evictAllAgentsCache()` - Forces cache refresh for agents

**Cache Configuration:**
- `agentIdCache` - Caches agent ID lookups by primary key
- `scriptNameCache` - Caches script name lookups by script ID  
- `allAgentsCache` - Caches the complete agent list

**Dependencies:**
- `TacticalRmmClient` - SDK client for Tactical RMM API calls
- `IntegratedToolService` - Manages tool configurations
- `ToolUrlService` - Handles tool URL management

## Usage Example

```java
@Autowired
private TacticalRmmCacheService tacticalService;

// Get agent ID by primary key (cached)
String agentId = tacticalService.getAgentIdByPrimaryKey(12345);
if (agentId != null) {
    log.info("Found agent: {}", agentId);
}

// Get script name by ID (cached)
String scriptName = tacticalService.getScriptNameById(678);

// Get all agents (cached list)
List<AgentListItem> allAgents = tacticalService.getAllAgentsFromCache();

// Force cache refresh when needed
tacticalService.evictAllAgentsCache();
```

The service automatically handles client initialization using integrated tool configuration and provides intelligent cache refresh when data is not found in the initial cache lookup.
