A specialized Debezium message wrapper that provides type-safe handling of Activity objects from the activities Kafka topic, eliminating the need for manual JSON parsing.

## Key Components

- **ActivityMessage**: Extends `DebeziumMessage<Activity>` to provide strongly-typed access to Activity data
- **Type Safety**: Replaces generic `JsonNode` fields with concrete `Activity` objects
- **Lombok Annotations**: Uses `@Data` and `@EqualsAndHashCode` for automatic code generation

## Usage Example

```java
// Processing activity messages from Kafka
@KafkaListener(topics = "activities")
public void handleActivityMessage(ActivityMessage message) {
    // Direct access to typed Activity objects
    Activity currentActivity = message.getAfter();
    Activity previousActivity = message.getBefore();
    
    if (currentActivity != null) {
        String activityId = currentActivity.getId();
        String status = currentActivity.getStatus();
        // No need for JSON parsing or type casting
    }
    
    // Handle different CDC operations
    switch (message.getOp()) {
        case "c": // Create
            processNewActivity(currentActivity);
            break;
        case "u": // Update
            processActivityUpdate(previousActivity, currentActivity);
            break;
        case "d": // Delete
            processActivityDeletion(previousActivity);
            break;
    }
}
```
