A strongly-typed wrapper for Debezium change data capture messages containing Activity objects from the fleet activities topic.

## Key Components

- **ActivityMessage** - Main class extending `DebeziumMessage<Activity>`
- **Type-safe accessors** - `getAfter()` and `getBefore()` methods return `Activity` objects instead of generic `JsonNode`
- **Lombok annotations** - `@Data` for getters/setters, `@EqualsAndHashCode` with super class inclusion

## Usage Example

```java
// Processing activity change events from Kafka
public void processActivityChange(ActivityMessage message) {
    // Get typed Activity objects directly
    Activity currentActivity = message.getAfter();
    Activity previousActivity = message.getBefore();
    
    // Access activity properties without casting
    if (currentActivity != null) {
        String activityType = currentActivity.getType();
        Long fleetId = currentActivity.getFleetId();
        
        // Handle create, update, or delete operations
        switch (message.getOp()) {
            case "c" -> handleCreate(currentActivity);
            case "u" -> handleUpdate(previousActivity, currentActivity);
            case "d" -> handleDelete(previousActivity);
        }
    }
}
```

This eliminates the need for manual JSON deserialization when consuming activity change events from the Debezium CDC pipeline.
