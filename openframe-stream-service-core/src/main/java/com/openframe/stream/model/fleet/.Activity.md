A data model representing fleet activity records with user information and system details, designed for JSON serialization and database mapping.

## Key Components

- **Core Fields**: `id`, `createdAt`, `userId`, `userName`, `activityType`
- **Activity Data**: `details` (JSON string), `streamed` status flag
- **User Information**: `userEmail` for contact/notification purposes
- **Enrichment Fields**: `agentId` (from Redis/Fleet DB) and `hostId` (from joins)
- **Jackson Annotations**: `@JsonProperty` mappings for snake_case JSON compatibility
- **Lombok Integration**: `@Data` annotation for automatic getters/setters/toString

## Usage Example

```java
// Creating an activity record
Activity activity = new Activity();
activity.setId(12345);
activity.setUserId(67890L);
activity.setUserName("john.doe");
activity.setActivityType("DEPLOYMENT");
activity.setDetails("{\"action\":\"deploy\",\"target\":\"production\"}");
activity.setStreamed(1);
activity.setUserEmail("john.doe@company.com");

// Enrichment after database retrieval
activity.setAgentId("agent-abc-123");
activity.setHostId(999);

// JSON serialization preserves snake_case format
// {
//   "id": 12345,
//   "user_id": 67890,
//   "activity_type": "DEPLOYMENT",
//   "created_at": "2024-01-15T10:30:00Z"
// }
```

The model supports both Debezium change data capture (core fields) and application-level enrichment (agentId, hostId) for comprehensive fleet activity tracking.
