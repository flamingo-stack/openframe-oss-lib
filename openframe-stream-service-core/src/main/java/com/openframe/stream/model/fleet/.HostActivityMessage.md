A specialized Debezium message wrapper for the `host_activities` Kafka topic that provides type-safe access to `HostActivity` objects instead of generic JSON nodes.

## Key Components

- **`HostActivityMessage`** - Generic specialization of `DebeziumMessage<HostActivity>` that inherits all Debezium change data capture functionality
- **Type Safety** - `getAfter()` and `getBefore()` methods return strongly-typed `HostActivity` objects rather than `JsonNode`
- **Lombok Annotations** - Uses `@Data` for auto-generated getters/setters and `@EqualsAndHashCode(callSuper = true)` for proper inheritance equality

## Usage Example

```java
// Consuming host activity changes from Kafka
@KafkaListener(topics = "host_activities")
public void handleHostActivityChange(HostActivityMessage message) {
    // Type-safe access to host activity data
    HostActivity currentState = message.getAfter();
    HostActivity previousState = message.getBefore();
    
    if (currentState != null) {
        String operation = message.getOp(); // INSERT, UPDATE, DELETE
        String hostname = currentState.getHostname();
        
        // Process the host activity change
        processHostActivity(hostname, operation, currentState, previousState);
    }
}

// No casting or JSON parsing required - direct object access
String hostStatus = message.getAfter().getStatus();
Long hostId = message.getAfter().getId();
```
