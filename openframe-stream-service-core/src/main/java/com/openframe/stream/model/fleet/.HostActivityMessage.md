A typed wrapper for Debezium change data capture messages specifically for the host_activities topic. This class extends the generic DebeziumMessage to provide strongly-typed access to HostActivity objects instead of raw JSON nodes.

## Key Components

- **HostActivityMessage**: Main class that extends `DebeziumMessage<HostActivity>`
- **Type Safety**: Provides compile-time type checking for host activity data
- **Lombok Annotations**: Uses `@Data` and `@EqualsAndHashCode` for boilerplate code generation

## Usage Example

```java
// Processing host activity messages from Kafka
@KafkaListener(topics = "host_activities")
public void handleHostActivityChange(HostActivityMessage message) {
    // Get the current state of the host activity (strongly typed)
    HostActivity currentActivity = message.getAfter();
    
    // Get the previous state before the change
    HostActivity previousActivity = message.getBefore();
    
    // Access operation type (INSERT, UPDATE, DELETE)
    String operation = message.getOp();
    
    // Process based on operation type
    if ("c".equals(operation) && currentActivity != null) {
        // Handle new host activity creation
        processNewActivity(currentActivity);
    }
}
```
