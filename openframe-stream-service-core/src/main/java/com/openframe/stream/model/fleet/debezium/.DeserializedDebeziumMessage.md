A specialized Debezium message model that extends the common message structure with fleet-specific metadata and processing flags for deserialized change data capture events.

## Key Components

- **DeserializedDebeziumMessage** - Main model class extending `CommonDebeziumMessage`
- **unifiedEventType** - Standardized event categorization using `UnifiedEventType` enum
- **integratedToolType** - Source tool identification via `IntegratedToolType` enum
- **Processing flags** - `skipProcessing` and `isVisible` for event handling control
- **Metadata fields** - Agent ID, tool event ID, timestamps, and source event details

## Usage Example

```java
// Creating a deserialized message
DeserializedDebeziumMessage message = DeserializedDebeziumMessage.builder()
    .unifiedEventType(UnifiedEventType.USER_ACTION)
    .integratedToolType(IntegratedToolType.JENKINS)
    .agentId("agent-123")
    .toolEventId("build-456")
    .sourceEventType("build_completed")
    .eventTimestamp(System.currentTimeMillis())
    .skipProcessing(false)
    .isVisible(true)
    .message("Build completed successfully")
    .build();

// Processing logic
if (!message.getSkipProcessing() && message.getIsVisible()) {
    processFleetEvent(message);
}
```
