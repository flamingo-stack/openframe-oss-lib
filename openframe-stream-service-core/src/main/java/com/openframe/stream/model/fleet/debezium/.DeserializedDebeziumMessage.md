Extends `CommonDebeziumMessage` to represent a fully deserialized Debezium CDC event enriched with OpenFrame-specific metadata for fleet event processing.

## Key Components

| Field | Type | Description |
|-------|------|-------------|
| `unifiedEventType` | `UnifiedEventType` | Normalized event classification across tools |
| `integratedToolType` | `IntegratedToolType` | Source MSP tool that generated the event |
| `toolEventId` | `String` | Original event ID from the source tool |
| `agentId` | `String` | Agent associated with the event |
| `ingestDay` | `String` | Partition/routing key for Cassandra ingestion |
| `eventTimestamp` | `Long` | Epoch timestamp of the original event |
| `tenantId` | `String` | Multi-tenant isolation identifier |
| `skipProcessing` | `Boolean` | Flag to bypass downstream processing logic |
| `isVisible` | `Boolean` | Controls event visibility in client-facing surfaces |
| `debeziumMessage` | `String` | Raw serialized Debezium payload for audit/replay |

## Usage Example

```java
DeserializedDebeziumMessage event = DeserializedDebeziumMessage.builder()
    .unifiedEventType(UnifiedEventType.TICKET_CREATED)
    .integratedToolType(IntegratedToolType.CONNECTWISE)
    .toolEventId("CW-98765")
    .agentId("agent-001")
    .tenantId("tenant-abc")
    .ingestDay("2024-01-15")
    .eventTimestamp(System.currentTimeMillis())
    .skipProcessing(false)
    .isVisible(true)
    .message("New ticket opened by client")
    .build();
```

> **Note:** `skipProcessing` should be set to `true` for duplicate or malformed events detected during stream normalization to prevent downstream side effects without dropping the message from the audit trail.
