A data model class that extends CommonDebeziumMessage to represent deserialized Debezium CDC (Change Data Capture) messages with additional metadata for stream processing and fleet management.

## Key Components

**Main Class:**
- `DeserializedDebeziumMessage` - Extends `CommonDebeziumMessage` with fleet-specific fields
- Uses Lombok annotations for automatic generation of getters, setters, builders, and constructors

**Key Fields:**
- `unifiedEventType` - Standardized event type classification
- `integratedToolType` - Type of integrated tool that generated the event  
- `toolEventId` / `agentId` - Identifiers for tracking event sources
- `eventTimestamp` - Timestamp of the original event
- `skipProcessing` / `isVisible` - Processing control flags
- `debeziumMessage` - Raw Debezium message content
- `ingestDay` - Partitioning field for data organization

## Usage Example

```java
// Building a deserialized message
DeserializedDebeziumMessage message = DeserializedDebeziumMessage.builder()
    .unifiedEventType(UnifiedEventType.USER_ACTION)
    .integratedToolType(IntegratedToolType.ANALYTICS_TOOL)
    .agentId("agent-123")
    .toolEventId("event-456")
    .eventTimestamp(System.currentTimeMillis())
    .message("User clicked button")
    .skipProcessing(false)
    .isVisible(true)
    .ingestDay("2024-01-15")
    .build();

// Accessing inherited CommonDebeziumMessage fields
String operation = message.getOp(); // from parent class
Map<String, Object> payload = message.getPayload(); // from parent class
```

This model serves as a bridge between raw Debezium CDC events and the OpenFrame streaming pipeline, providing structured access to both standard Debezium fields and application-specific metadata.
