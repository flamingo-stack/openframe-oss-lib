Scheduled health check component that monitors Debezium CDC connectors, auto-recovers failed connectors, and reconciles missing ones against registered tenant tools using distributed locking to prevent duplicate execution across cluster nodes.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `checkAndRestartFailedTasks()` | `@Scheduled` method | Main health check loop; runs connector reconciliation and recovery on a configurable interval (default: 5 min) |
| `reconcileMissingConnectors()` | Private method | Compares expected connectors (derived from integrated tools) against actual Kafka Connect state and triggers re-creation for any missing ones |
| `init()` | `@PostConstruct` | Logs startup confirmation including the active `ConnectorNameStrategy` |
| `@ConditionalOnProperty` | Class annotation | Only activates when `openframe.debezium.health-check.enabled=true` |
| `@SchedulerLock` | Method annotation | ShedLock-based distributed lock (`debeziumHealthCheck`) prevents concurrent execution across multiple instances |

## Key Behaviors

- **Tenant guard**: Skips connector creation on empty clusters (no registered tenants) to avoid spurious provisioning before the first customer signs up.
- **Transient outage guard**: If `listConnectors()` returns empty but connectors are expected, reconciliation is skipped to avoid noisy 409 re-creation attempts during Kafka Connect downtime.
- `IntegratedToolService` is injected as `required = false`, making the reconciliation path optional.

## Configuration Properties

| Property | Default | Description |
|----------|---------|-------------|
| `openframe.debezium.health-check.enabled` | — | Must be `true` to activate the scheduler |
| `openframe.debezium.health-check.interval` | `300000` ms | Fixed delay between health checks |
| `openframe.debezium.health-check.lock-at-most-for` | `5m` | Max ShedLock hold duration |
| `openframe.debezium.health-check.lock-at-least-for` | `1m` | Min ShedLock hold duration |

## Usage Example

```java
// application.yml
openframe:
  debezium:
    health-check:
      enabled: true
      interval: 300000        # 5 minutes
      lock-at-most-for: 5m
      lock-at-least-for: 1m
```

```java
// Spring auto-discovers and schedules this component when enabled.
// No manual wiring required — ensure ShedLock and @EnableSchedulerLock
// are configured on your application class:

@SpringBootApplication
@EnableScheduling
@EnableSchedulerLock(defaultLockAtMostFor = "10m")
public class OpenFrameApplication { ... }
```
