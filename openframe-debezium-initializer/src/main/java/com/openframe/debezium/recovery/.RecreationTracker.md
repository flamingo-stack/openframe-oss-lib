Rate-limits connector recreation attempts within a rolling time window, preventing the recovery manager from repeatedly recreating the same connector.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `canRecreate(String baseName)` | Method | Returns `true` if recreation is permitted for the given connector base name (rolling window limit not exceeded) |
| `record(String baseName)` | Method | Records a recreation event against the rolling window counter for the given base name |

## Usage Example

```java
@Autowired
RecreationTracker recreationTracker;

public void attemptRecovery(String connectorBaseName) {
    if (recreationTracker.canRecreate(connectorBaseName)) {
        // Proceed with connector recreation
        recreateConnector(connectorBaseName);
        // Record the attempt against the rolling window
        recreationTracker.record(connectorBaseName);
    } else {
        log.warn("Recreation limit exceeded for connector: {}", connectorBaseName);
    }
}
```

> **Note:** This interface has no default implementation — it is absent by default (no recreation occurs). A concrete bean must be supplied by the SaaS layer for shared cluster environments.
