Default fallback implementation of `ConnectorNameStrategy` that preserves connector names verbatim without modification or version suffixes.

## Key Components

| Member | Description |
|--------|-------------|
| `matchesBase` | Returns `true` only when `baseName` exactly equals `connectorName` (strict equality via `Objects.equals`) |
| `extractBaseName` | Returns the connector name unchanged as its own base name |
| `resolveNextName` | Always returns `baseName` as-is, ignoring any existing connectors list |

**Bean Registration:** Registered as a Spring `@Component` with `@ConditionalOnMissingBean`, meaning it activates only when no other `ConnectorNameStrategy` bean is present. The `ignored` attribute ensures this class itself doesn't satisfy that condition, preventing circular resolution.

## Usage Example

```java
// Injected automatically when no other ConnectorNameStrategy is defined
@Autowired
ConnectorNameStrategy strategy;

String base = strategy.extractBaseName("my-connector");
// → "my-connector"

boolean matches = strategy.matchesBase("my-connector", "my-connector");
// → true

String next = strategy.resolveNextName("my-connector", List.of("my-connector"));
// → "my-connector"  (no versioning applied)
```

## Behavior Notes

- **Tenant clusters** use this strategy by default, experiencing no naming behavior change
- **Alternative strategies** (e.g., versioned naming) override this bean by simply implementing `ConnectorNameStrategy` — no extra configuration needed
- Acts as a safe no-op baseline; all name transformations are identity operations (`input == output`)
