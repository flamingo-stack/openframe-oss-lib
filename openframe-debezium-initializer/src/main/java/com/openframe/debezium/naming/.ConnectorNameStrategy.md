Defines the contract for resolving and recognising Kafka Connect connector names, supporting both simple identity-based and versioned (`_v1`, `_v2`, …) naming schemes used across Flamingo's Debezium integration.

## Key Components

| Member | Type | Description |
|--------|------|-------------|
| `matchesBase` | Abstract method | Returns `true` if a connector name equals or is a versioned form of the given base name |
| `extractBaseName` | Abstract method | Strips any version suffix (e.g. `_v3`) and returns the bare base name |
| `resolveNextName` | Abstract method | Computes the next connector name to use on creation or recreation (`baseName` or `baseName_v(max+1)`) |
| `supportsRecreation` | Default method | Indicates whether the strategy allows renaming a connector during recovery; defaults to `false` |
| `hasAnyVersion` | Default method | Runaway-protection guard — returns `true` if any existing connector matches the base name |
| `staleVersions` | Default method | Streams all connectors matching the base name except the current one, used for post-recreate cleanup |
| `currentVersion` | Default method | Returns the active connector name for a base from the existing list; versioned implementations return the highest `_vN` |

## Usage Example

```java
// Identity strategy (default) — name used as-is
ConnectorNameStrategy strategy = new IdentityConnectorNameStrategy();

List<String> existing = List.of("orders-cdc", "inventory-cdc");

// Check if any version exists before creating
if (!strategy.hasAnyVersion("orders-cdc", existing)) {
    String name = strategy.resolveNextName("orders-cdc", existing);
    // name == "orders-cdc"
    kafkaConnect.create(name, config);
}

// Versioned strategy (SaaS shared cluster)
// existing == ["orders-cdc_v1", "orders-cdc_v2"]
String next = versionedStrategy.resolveNextName("orders-cdc", existing);
// next == "orders-cdc_v3"

// Clean up stale versions after recreation
versionedStrategy.staleVersions("orders-cdc", "orders-cdc_v3", existing)
    .forEach(kafkaConnect::delete);
// deletes "orders-cdc_v1", "orders-cdc_v2"
```

## Implementation Notes

- The default implementation is `IdentityConnectorNameStrategy` — name passes through unchanged and `supportsRecreation()` returns `false`
- SaaS shared-cluster deployments override with a versioned strategy, enabling zero-downtime connector rotation by incrementing the `_vN` suffix rather than reusing a name that would cause a collision
