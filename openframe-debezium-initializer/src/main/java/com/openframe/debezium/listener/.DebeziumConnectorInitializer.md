Initializes Debezium connectors on application startup by restoring connector configurations from MongoDB, ensuring CDC pipelines are active after service restarts.

## Key Components

| Element | Description |
|---|---|
| `@ConditionalOnProperty` | Only activates when `openframe.debezium.health-check.enabled=true` |
| `onApplicationReady()` | Event listener triggered on `ApplicationReadyEvent` to kick off initialization |
| `initializeConnectorsIfEmpty()` | Core logic that checks for existing connectors and bootstraps from MongoDB if none are found |

## Behavior & Guards

The initializer applies three early-exit checks before creating any connectors:

1. **`IntegratedToolService` availability** — skips if the service bean is absent (`required = false`)
2. **Tenant registration** — skips on empty/uninitialized clusters with no registered tenant
3. **Existing connectors** — skips if Debezium already has active connectors, preventing duplicates on warm restarts

## Usage Example

```java
// application.properties — enable the initializer
openframe.debezium.health-check.enabled=true
```

```java
// When active, the bean auto-runs on startup — no manual invocation needed.
// To trigger programmatically (e.g., in tests):
@Autowired
DebeziumConnectorInitializer initializer;

initializer.initializeConnectorsIfEmpty();
```

```java
// IntegratedTool must have debeziumConnectors configured for connectors to be created:
tool.getDebeziumConnectors(); // non-null, non-empty array triggers connector creation
```

> **Note:** `IntegratedToolService` is injected with `required = false`, making this component safe to include in deployments where tool management is not available.
