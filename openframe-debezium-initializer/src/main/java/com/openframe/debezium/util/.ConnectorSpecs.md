Utility class providing helper methods for working with raw `Object[]` connector specs stored in `IntegratedTool#getDebeziumConnectors()`. Centralises unsafe cast boilerplate and null-handling to a single location.

## Key Components

| Member | Description |
|--------|-------------|
| `KEY_NAME` / `KEY_CONFIG` | String constants for spec map keys (`"name"`, `"config"`) |
| `asMap(Object)` | Safe cast of an unknown value to `Map<String, Object>`; returns `null` on type mismatch |
| `nameOf(Map)` | Extracts the `"name"` string from a spec map |
| `configOf(Map)` | Extracts and casts the nested `"config"` map from a spec map |
| `specStreamOf(IntegratedTool)` | Returns a `Stream` of non-null spec maps for a given tool |
| `configWithName(Map, String)` | Shallow-copies a config map and injects or overrides `"name"` |
| `connectorPayload(String, Map)` | Builds the Kafka Connect connector-creation payload `{name, config}` with `name` embedded in both places as Debezium requires |

## Usage Example

```java
// Stream all valid connector specs for a tool and build payloads
ConnectorSpecs.specStreamOf(integratedTool)
    .forEach(spec -> {
        String name = ConnectorSpecs.nameOf(spec);
        Map<String, Object> config = ConnectorSpecs.configOf(spec);

        // Build Kafka Connect creation payload
        Map<String, Object> payload = ConnectorSpecs.connectorPayload(name, config);
        kafkaConnectClient.createConnector(payload);
    });

// Build a config map with name embedded (e.g. for an update)
Map<String, Object> updatedConfig = ConnectorSpecs.configWithName(existingConfig, "my-connector");
```

> **Note:** `@SuppressWarnings("unchecked")` is intentionally confined to `asMap()` only — all other methods rely on this single guarded cast, keeping the rest of the codebase clean.
