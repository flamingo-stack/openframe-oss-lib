Defines a single shared constant for the Debezium log-line prefix, ensuring that application log output and Grafana/Loki LogQL alert rules match against an identical, stable string.

## Key Components

| Member | Type | Value | Description |
|--------|------|-------|-------------|
| `PREFIX` | `public static final String` | `"[DEBEZIUM]"` | Log-line prefix matched by Loki alert rules in `manifests/platform/grafana/alerting/rules/debezium-health.yaml` |

## Usage Example

```java
import com.openframe.debezium.util.DebeziumLog;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class DebeziumConnectorService {

    private static final Logger log = LoggerFactory.getLogger(DebeziumConnectorService.class);

    public void onConnectorStarted() {
        log.info("{} Connector started successfully", DebeziumLog.PREFIX);
    }

    public void onConnectorError(Exception ex) {
        log.error("{} Connector encountered a fatal error: {}", DebeziumLog.PREFIX, ex.getMessage());
    }
}
```

> **Important:** Do not change the value of `PREFIX` without also updating the matching LogQL expressions in the Grafana alerting rules manifest. The two must remain in sync — a mismatch will silently break health alerts.
