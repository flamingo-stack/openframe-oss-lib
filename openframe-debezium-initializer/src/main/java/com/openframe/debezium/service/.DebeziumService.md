Manages lifecycle operations for Debezium connectors via the Kafka Connect REST API, including creation, versioned updates, drift detection, deletion, and status retrieval.

## Key Components

| Method | Description |
|--------|-------------|
| `createOrUpdateDebeziumConnector` | Entry point for applying one or more connector specs; delegates to `applyConnectorSpec` |
| `applyConnectorSpec` | Routes connector creation/update based on naming strategy (identity vs. versioned) |
| `createInitialVersion` | Creates a new connector when no version exists yet |
| `recreateOnConfigDrift` | Versioned-strategy path: compares running config to incoming spec and recreates on drift, gated by `RecreationTracker` |
| `tryUpdateExistingIdentityConnector` | Identity-strategy path: pushes updated config in-place via `PUT /connectors/{name}/config` |
| `configsEquivalent` | Normalises both config maps to `Map<String, String>` (dropping `name`) for semantic comparison |
| `createConnector` | POSTs a new connector payload to Kafka Connect |
| `deleteConnector` | DELETEs a connector; tolerates 404 |
| `listConnectors` | Returns all registered connector names; returns empty list on failure |
| `getConnectorStatus` | Returns `ConnectorStatus` or `null` if the connector is gone |
| `getConnectorConfig` | Returns the currently-deployed config map or `null` on 404/error |

## Usage Example

```java
// Inject the service
@Autowired
private DebeziumService debeziumService;

// Apply connector specs loaded from MongoDB (e.g. IntegratedTool config)
Object[] specs = tool.getDebeziumConnectors();
debeziumService.createOrUpdateDebeziumConnector(specs);

// Manually create a connector
Map<String, Object> config = new HashMap<>();
config.put("connector.class", "io.debezium.connector.postgresql.PostgresConnector");
config.put("database.hostname", "localhost");
debeziumService.createConnector("my-connector-v1", config);

// Check connector health
ConnectorStatus status = debeziumService.getConnectorStatus("my-connector-v1");
if (status == null) {
    log.warn("Connector not found or Kafka Connect unreachable");
}

// List all active connectors
List<String> connectors = debeziumService.listConnectors();
```

> **Note:** `RecreationTracker` is optional (`required = false`). When absent, drift-driven recreation proceeds without rate-limiting. The service reads `openframe.debezium.base-url`, `connect-timeout` (default `5s`), and `read-timeout` (default `30s`) from application properties.
