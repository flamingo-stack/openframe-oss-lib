A Spring Web MVC argument resolver that automatically converts JWT authentication tokens into custom `AuthPrincipal` objects, enabling cleaner controller method signatures when working with authenticated users.

## Key Components

- **`supportsParameter()`** - Determines if the resolver applies to method parameters annotated with `@AuthenticationPrincipal` and typed as `AuthPrincipal`
- **`resolveArgument()`** - Extracts the JWT token from the security context and converts it to an `AuthPrincipal` instance
- **JWT Token Handling** - Specifically works with Spring Security's `JwtAuthenticationToken` to access the underlying `Jwt` object

## Usage Example

Register the resolver in your Spring configuration:

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addArgumentResolvers(List<HandlerMethodArgumentResolver> resolvers) {
        resolvers.add(new AuthPrincipalArgumentResolver());
    }
}
```

Use in controllers with clean method signatures:

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public UserProfile getProfile(@AuthenticationPrincipal AuthPrincipal auth) {
        // Direct access to custom AuthPrincipal instead of raw Jwt
        return userService.getProfile(auth.getUserId());
    }
}
```

The resolver eliminates the need to manually convert `Jwt` tokens to domain objects in every controller method, providing a consistent authentication principal across your application.
