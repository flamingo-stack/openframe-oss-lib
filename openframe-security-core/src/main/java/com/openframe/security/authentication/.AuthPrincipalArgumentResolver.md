A Spring Web MVC argument resolver that automatically converts JWT authentication tokens into `AuthPrincipal` objects for controller method parameters. This enables cleaner controller code by allowing direct injection of custom principal objects instead of raw JWT tokens.

## Key Components

- **`supportsParameter()`** - Determines if the resolver can handle parameters annotated with `@AuthenticationPrincipal` and of type `AuthPrincipal`
- **`resolveArgument()`** - Extracts JWT from the authentication context and converts it to an `AuthPrincipal` instance
- **JWT Token Processing** - Handles `JwtAuthenticationToken` extraction and delegates to `AuthPrincipal.fromJwt()`

## Usage Example

Register the resolver in your Spring configuration:

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addArgumentResolvers(List<HandlerMethodArgumentResolver> resolvers) {
        resolvers.add(new AuthPrincipalArgumentResolver());
    }
}
```

Use in controllers:

```java
@RestController
public class UserController {
    
    @GetMapping("/profile")
    public UserProfile getProfile(@AuthenticationPrincipal AuthPrincipal principal) {
        // Direct access to custom principal object instead of raw JWT
        return userService.getProfile(principal.getUserId());
    }
}
```

This eliminates the need to manually extract user information from JWT tokens in every controller method.
